# F7 권한(RBAC) 분기 플로우 — SCR-050 락커 관리

## 1. 목적
6개 역할별 접근 가능 범위와 버튼 활성화 조건을 정의한다.

## 2. 전제조건
- 로그인 상태, /locker 접근 시도

## 3. 다이어그램

```mermaid
flowchart LR
    Login([로그인 사용자]) --> RoleCheck{역할 확인}

    RoleCheck -->|": primary/owner"|RolePO[슈퍼관리자/센터장]
    RoleCheck -->|": manager"|RoleMgr[매니저]
    RoleCheck -->|": staff"|RoleStaff[스태프]
    RoleCheck -->|": fc/coach"|RoleFC[FC/코치]
    RoleCheck -->|": readonly"|RoleRO[읽기전용]

    RolePO -->|": 전체 권한"|FullAccess[접근O 조회O 생성O 수정O 삭제O\n모든 버튼 활성]
    RoleMgr -->|": 삭제 제외"|MgrAccess[접근O 조회O 생성O 수정O 삭제X\n락커추가X]
    RoleStaff -->|": 삭제 제외"|StaffAccess[접근O 조회O 생성O 수정O 삭제X\n락커추가X]
    RoleFC -->|": 접근 불가"|Blocked[접근X → 403\n사이드바 메뉴 숨김]
    RoleRO -->|": 읽기 전용"|ROAccess[접근O 조회O\n배정/회수/고장 버튼 비활성\n일괄배정 버튼 비활성]

    FullAccess --> ActionFull[배정·회수·이동·고장·일괄배정·일괄해제·락커추가·엑셀·새로고침]
    MgrAccess --> ActionMgr[배정·회수·이동·고장·일괄배정·일괄해제·엑셀·새로고침]
    StaffAccess --> ActionStaff[배정·회수·이동·고장·일괄배정·일괄해제·엑셀·새로고침]
    ROAccess --> ActionRO[구역탭·검색·통계카드·기록모달\n수정 액션 전부 비활성]
```

## 4. 엣지 설명

| 역할 | 접근 수준 | |---------|------|-----------| | | primary/owner | 전체 CRUD + 락커추가 | | | manager | CRUD(삭제·락커추가 제외) | | | staff | CRUD(삭제·락커추가 제외) | | | fc/coach | 접근 불가 → 403 | | | readonly | 조회만, 수정 버튼 비활성 |
