## 3. 다이어그램

```mermaid
flowchart TD
  ([상태 선택])
   --> VALIDATE{상태 전환 유효성 검증}

  VALIDATE -->|유효한 전환|VALID[변경 확인 버튼 활성화]
  VALIDATE -->|현재 상태와 동일|ERR_SAME[인라인 경고\n현재 상태와 동일합니다.]
  VALIDATE -->|사유 필요 상태|REQUIRE_REASON[사유 입력 필드 표시\n휴직/전보 사유 필수]

  REQUIRE_REASON -->|사유 입력|VALID
  REQUIRE_REASON -->|사유 미입력 blur|ERR_REASON[인라인 에러\n사유를 입력하세요.]

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class VALID success
  class ERR_SAME,ERR_REASON error
  class REQUIRE_REASON warning
  class VALIDATE system
```
