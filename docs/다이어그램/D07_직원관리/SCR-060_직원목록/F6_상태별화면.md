## 1. 목적

SCR-060의 로딩/빈/에러/권한없음/오프라인 등 UI 상태별 분기를 명세한다. UI 상태 TC 원천.

## 2. 전제조건

- SCR-060 진입 시도 상태이다.

## 3. 다이어그램

```mermaid
flowchart TD
  ENTER([SCR-060 진입 시도])

  ENTER --> PERM{권한 확인}
  PERM -->|매니저/FC/스태프|STATE_BLOCKED[권한없음 화면\n접근 불가 메시지]
  PERM -->|primary|STATE_LOADING[로딩 상태\\n스피너 표시]

  STATE_LOADING -->|API 응답 성공 데이터 있음|STATE_NORMAL[정상 상태\n직원 목록 테이블]
  STATE_LOADING -->|API 응답 성공 데이터 없음|STATE_EMPTY[빈 상태\n직원 데이터가 없습니다.\n+ 직원 등록 버튼 활성]
  STATE_LOADING -->|API 오류|STATE_ERROR[에러 상태\ntost 발생\n에러 배너 + 다시 시도]
  STATE_LOADING -->|네트워크 없음|STATE_OFFLINE[오프라인 상태\n네트워크 연결 확인 메시지]

  STATE_ERROR -->|다시 시도|STATE_LOADING
  STATE_OFFLINE -->|네트워크 복구|STATE_LOADING

  STATE_NORMAL -->|선택 없음|STATE_NO_SELECT[액션 버튼 disabled\n메시지/상태변경/퇴사]
  STATE_NORMAL -->|1개 이상 선택|STATE_SELECTED[액션 버튼 활성화]

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef rbacBlocked fill:#F5F5F5,stroke:#9E9E9E,color:#616161
  classDef info fill:#E0F7FA,stroke:#00838F

  class STATE_NORMAL,STATE_SELECTED screen
  class STATE_LOADING,STATE_NO_SELECT system
  class STATE_EMPTY warning
  class STATE_ERROR,STATE_OFFLINE error
  class STATE_BLOCKED rbacBlocked
  class PERM info
```
