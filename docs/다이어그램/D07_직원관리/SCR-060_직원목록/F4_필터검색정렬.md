## 1. 목적

SCR-060의 필터/검색/정렬/페이지네이션 조작 흐름을 명세한다. 쿼리 TC 원천.

## 2. 전제조건

- SCR-060 진입 완료, 직원 목록 표시 상태이다.

## 3. 다이어그램

```mermaid
flowchart TD
  LIST[직원 목록 표시]

  SEARCH[🔍 이름/연락처 검색 입력]
  FILTER_ROLE[역할 필터 Select]
  FILTER_STATUS[재직 상태 필터 Select]
  SORT[컬럼 헤더 클릭 — 정렬]
  PAGE[페이지네이션 클릭]

  LIST -->|텍스트 입력|SEARCH
  LIST -->|역할 선택|FILTER_ROLE
  LIST -->|상태 선택|FILTER_STATUS
  LIST -->|컬럼 헤더 클릭|SORT
  LIST -->|페이지 버튼 클릭|PAGE

  SEARCH -->|필터링 로직 실행|FILTERED[필터링된 결과 표시]
  FILTER_ROLE --> FILTERED
  FILTER_STATUS --> FILTERED

  FILTERED -->|결과 없음|EMPTY[빈 상태 표시]
  FILTERED -->|결과 있음|RESULT[필터 결과 테이블]

  SORT -->|1회 클릭 — null→asc|SORT_ASC[오름차순 정렬 ChevronUp]
  SORT_ASC -->|2회 클릭 — asc→desc|SORT_DESC[내림차순 정렬 ChevronDown]
  SORT_DESC -->|3회 클릭 — desc→null|SORT_RESET[정렬 해제 ChevronsUpDown]

  PAGE -->|이전 페이지|PAGE_PREV[이전 10건 표시]
  PAGE -->|다음 페이지|PAGE_NEXT[다음 10건 표시]
  PAGE -->|특정 번호|PAGE_NUM[해당 페이지 표시]

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef info fill:#E0F7FA,stroke:#00838F

  class LIST,RESULT screen
  class SEARCH,FILTER_ROLE,FILTER_STATUS,SORT,FILTERED system
  class EMPTY warning
  class PAGE,PAGE_PREV,PAGE_NEXT,PAGE_NUM,SORT_ASC,SORT_DESC,SORT_RESET info
```

## 4. 엣지 설명 테이블

| 출발 | 도착 | 라벨 / 조건 |
|------|------|-------------|
| 목록 | 검색 입력 | 검색창 텍스트 입력 |
| 검색 입력 | 필터 결과 | 이름 or 연락처 포함 필터링 |
| 목록 | 역할 필터 | 역할 Select 변경 |
| 목록 | 상태 필터 | 재직 상태 Select 변경 |
| 정렬 | 오름차순 | 컬럼 헤더 1회 클릭 |
| 오름차순 | 내림차순 | 컬럼 헤더 2회 클릭 |
| 내림차순 | 정렬 해제 | 컬럼 헤더 3회 클릭 |
| 페이지 | 이전 | 이전 버튼 |
| 페이지 | 다음 | 다음 버튼 |
| 페이지 | 특정 페이지 | 번호 클릭 |
