## 1. 목적

SCR-060에서 6개 역할별 접근/액션 가능 범위를 명세한다. RBAC TC 원천.

## 2. 전제조건

- 사용자가 로그인 상태이다.

## 3. 다이어그램

```mermaid
flowchart LR
  USER([로그인 사용자])

  USER --> ROLE_PRIMARY{primary\n슈퍼관리자}
  USER --> ROLE_OWNER{owner\n최고관리자}
  USER --> ROLE_MANAGER{manager\n센터장}
  USER --> ROLE_FC{fc\/코치}
  USER --> ROLE_STAFF{staff\n스태프}
  USER --> ROLE_READONLY{readonly}

  ROLE_PRIMARY -->|전체 권한|ACCESS_FULL[접근 O\n조회/생성/수정/삭제/퇴사 처리]
  ROLE_OWNER -->|전체 권한|ACCESS_FULL
  ROLE_MANAGER -->|전체 권한|ACCESS_FULL

  ROLE_FC --> BLOCKED[접근 X\n권한없음 화면]
  ROLE_STAFF --> BLOCKED
  ROLE_READONLY --> BLOCKED

  ACCESS_FULL --> ACT_VIEW[목록 조회 O]
  ACCESS_FULL --> ACT_CREATE[직원 등록 O]
  ACCESS_FULL --> ACT_EDIT[직원 수정 O]
  ACCESS_FULL --> ACT_RESIGN[퇴사 처리 O]
  ACCESS_FULL --> ACT_STATUS[상태 변경 O]
  ACCESS_FULL --> ACT_EXPORT[엑셀 다운로드 O]

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef rbacBlocked fill:#F5F5F5,stroke:#9E9E9E,color:#616161
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class ACCESS_FULL screen
  class BLOCKED rbacBlocked
  class ACT_VIEW,ACT_CREATE,ACT_EDIT,ACT_RESIGN,ACT_STATUS,ACT_EXPORT success
  class ROLE_PRIMARY,ROLE_OWNER,ROLE_MANAGER,ROLE_FC,ROLE_STAFF,ROLE_READONLY system
```

## 4. 엣지 설명 테이블

| 역할 | 결과 | |---------|------|------| | | primary | 전체 접근 허용 | | | owner | 전체 접근 허용 | | | manager | 전체 접근 허용 | | | fc | 접근 차단 | | | staff | 접근 차단 | | | readonly | 접근 차단 |
