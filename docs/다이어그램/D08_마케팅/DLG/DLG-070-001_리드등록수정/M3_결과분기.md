## 3. 다이어그램

```mermaid
flowchart TD
  API_CALL[API 호출\nPOST or PATCH /leads] --> RESP{응답 코드}

  RESP -->|E_M3_201 201 Created| OK_CREATE[모달 닫힘\ntoast.success 리드가 등록되었습니다\n목록 갱신]
  RESP -->|E_M3_200 200 OK| OK_UPDATE[모달 닫힘\ntoast.success 리드가 수정되었습니다\n목록 갱신]
  RESP -->|E_M3_400 400 Bad Request| ERR_400[필드 에러 표시\n모달 유지]
  RESP -->|E_M3_500 500 Server Error| ERR_500[toast.error 서버 오류\n모달 유지]
  RESP -->|E_M3_TIMEOUT Timeout| ERR_TIMEOUT[toast.warning 타임아웃\n재시도 안내]

  OK_CREATE --> SCR_070_REFRESH[SCR-070 목록 갱신]
  OK_UPDATE --> SCR_070_REFRESH
```
