# A09 — 근태 집계 → 급여 자동 계산

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매월 25일 02:00 |
| 대상 엔티티 | Attendance, Salary, Staff |
| 조건 | 당월 근태 데이터가 확정된 직원 |
| 결과 | 월 급여 자동 계산, 명세서 초안 생성 |
| 관련 화면 | SCR-063 직원 근태 관리, SCR-064 급여 관리, SCR-065 급여 명세서 |

## 2. 발생 조건

- `Staff = ACTIVE`
- 당월 근태 기록(AttendanceRecord) 존재
- 급여 계산 기준: 기본급 + 수당 - 공제 (지점 정책 기반)
- `Salary = DRAFT` 인 경우만 (기확정 제외)

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매월 25일 02:00]) --> QUERY[당월 급여 계산 대상<br/>ACTIVE 직원 조회]

    QUERY -->|없음|EMPTY[대상 없음<br/>종료]
    QUERY -->|있음|LOOP[직원 순회]

    LOOP --> EXIST_CHECK{당월 급여<br/>이미 존재?}
    EXIST_CHECK -->|확정됨|SKIP[스킵: 이미 확정 E409701]
    EXIST_CHECK -->|초안 있음|RECALC[재계산 덮어쓰기]
    EXIST_CHECK -->|없음|ATTEND_AGG[근태 집계<br/>출근일/지각/결근/초과근무]

    RECALC --> ATTEND_AGG

    ATTEND_AGG --> SALARY_CALC[급여 계산<br/>기본급 + 수당 - 공제]
    SALARY_CALC --> VALIDATE{계산 금액<br/ 이상?}

    VALIDATE -->|음수|CALC_ERR[E422700 음수 오류<br/>수동 검토 대기]
    VALIDATE -->|정상|TXN[트랜잭션 시작]

    TXN --> UPSERT_SALARY[Salary UPSERT<br/>status=DRAFT]
    UPSERT_SALARY --> COMMIT{커밋}
    COMMIT -->|성공|LOG_OK[급여 초안 생성 로그]
    COMMIT -->|실패|ROLLBACK[롤백<br/>오류 로그]

    SKIP & CALC_ERR & LOG_OK & ROLLBACK --> NEXT{다음 직원?}
    NEXT --> LOOP
    NEXT -->|완료|NOTIFY_MGR[지점장 검토 알림<br/>급여 초안 확인 요청]
    NOTIFY_MGR --> END([크론 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17

    class CRON,END cron
    class QUERY,LOOP,EXIST_CHECK,VALIDATE,COMMIT,NEXT system
    class LOG_OK,UPSERT_SALARY success
    class ROLLBACK,CALC_ERR error
    class SKIP,EMPTY warning
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 급여 금액 음수 | E422700 오류, 수동 검토 대기 상태로 표시 |
| 트랜잭션 실패 | 롤백, 오류 로그, 관리자 수동 재실행 |
| 이미 확정된 급여 | E409701 스킵, 수정 불가 |
| 크론 실패 | 관리자 수동 실행 (SCR-064 급여 관리) |
