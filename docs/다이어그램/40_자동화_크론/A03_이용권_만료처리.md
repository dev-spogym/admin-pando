# A03 — 이용권 자동 만료 처리

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매일 00:10 |
| 대상 엔티티 | Membership |
| 조건 | ` < NOW()` AND `status = ACTIVE` |
| 결과 | Membership = EXPIRED 전환, 회원 알림 |
| 관련 화면 | SCR-M004-02 이용권 탭, SCR-S001 매출 현황 |

## 2. 발생 조건

- `Membership = ACTIVE`
- `Membership. < NOW()` (당일 00:10 기준)
- 홀딩(HOLDING) 상태 이용권 제외 (A04에서 처리)
- 잔여 횟수가 있어도 만료일 기준 우선

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매일 00:10]) --> QUERY[만료일 도래 이용권 조회<br/> < NOW AND status=ACTIVE]

    QUERY -->|없음|EMPTY[대상 없음<br/>정상 종료]
    QUERY -->|있음|LOOP[대상 이용권 순회]

    LOOP --> HOLDING_CHECK{홀딩 여부 확인}
    HOLDING_CHECK -->|홀딩중|SKIP_HOLDING[스킵: A04에서 처리]
    HOLDING_CHECK -->|일반|TXN_START[트랜잭션 시작]

    TXN_START --> UPDATE_MEM[Membership<br/>ACTIVE → EXPIRED]
    UPDATE_MEM --> UPDATE_MEMBER_STATUS{회원 전체 이용권<br/>모두 만료?}

    UPDATE_MEMBER_STATUS -->|모두 만료|UPDATE_MEMBER[Member 상태<br/>→ EXPIRED_MEMBER 표시]
    UPDATE_MEMBER_STATUS -->|일부 유효|KEEP_MEMBER[Member 상태 유지]

    UPDATE_MEMBER & KEEP_MEMBER --> COMMIT{트랜잭션<br/>커밋}
    COMMIT -->|성공|LOG_HISTORY[만료 이력 저장<br/>MembershipHistory]
    COMMIT -->|실패|ROLLBACK[롤백<br/>오류 로그]

    LOG_HISTORY --> NOTIFY[만료 알림 발송]
    NOTIFY --> LOG_OK[알림 이력 저장]
    NOTIFY --> LOG_NOTIFY_FAIL[알림 실패 로그]

    SKIP_HOLDING & LOG_OK & LOG_NOTIFY_FAIL & ROLLBACK --> NEXT{다음 이용권?}
    NEXT --> LOOP
    NEXT -->|완료|REPORT[실행 리포트]
    REPORT --> END([크론 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17

    class CRON,END cron
    class QUERY,LOOP,HOLDING_CHECK,UPDATE_MEMBER_STATUS,COMMIT,NEXT system
    class LOG_HISTORY,LOG_OK,REPORT success
    class ROLLBACK,LOG_NOTIFY_FAIL error
    class SKIP_HOLDING,EMPTY warning
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 트랜잭션 실패 | 롤백, 해당 이용권 스킵, 다음 처리 계속 |
| 크론 실패 | 00:10에 재실행 불가 시 다음 날 재처리 |
| 홀딩 상태 오판 | A04 크론(00:20) 에서 홀딩 연장 후 재평가 |
