# A12 — 감사로그 아카이빙

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매월 1일 04:00 |
| 대상 엔티티 | AuditLog |
| 조건 | 2개월 이전 감사로그 (기본 보관 정책) |
| 결과 | Hot DB → Cold Storage (S3) 이관 후 DB 정리 |
| 관련 화면 | SCR-097 감사로그, SCR-090 본사 대시보드 |

## 2. 발생 조건

- `AuditLog.개월 전 (설정 가능)`
- `AuditLog = false`
- 압축(gzip) 후 S3 버킷 저장
- DB에는 메타데이터 + S3 경로만 유지

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매월 1일 04:00]) --> POLICY[보관 정책 로드<br/> 설정]

    POLICY --> QUERY[아카이빙 대상 조회<br/> < N개월 전<br/>archived=false]

    QUERY -->|없음|EMPTY[대상 없음<br/>종료]
    QUERY -->|있음|SUMMARY[대상 건수/용량 집계]

    SUMMARY --> BATCH[배치 단위 분할<br/만 건씩]
    BATCH --> LOOP[배치 순회]

    LOOP --> COMPRESS[gzip 압축]
    COMPRESS --> UPLOAD[S3 업로드<br/>audit/{year}/{month}/]
    UPLOAD -->|성공|UPDATE_DB[DB 레코드 업데이트<br/>archived=true, s3Key 저장]
    UPLOAD -->|실패|UPLOAD_ERR{재시도<br/>?}

    UPLOAD_ERR --> UPLOAD
    UPLOAD_ERR -->|초과|BATCH_ERR[배치 실패 로그<br/>해당 배치 스킵]

    UPDATE_DB --> DELETE_OLD[원본 DB 레코드 삭제<br/>메타데이터만 유지]
    DELETE_OLD -->|성공|LOG_BATCH_OK[배치 완료 로그]
    DELETE_OLD -->|실패|DELETE_ERR[삭제 실패 로그<br/>S3 업로드는 유지]

    BATCH_ERR & LOG_BATCH_OK & DELETE_ERR --> NEXT_BATCH{다음 배치?}
    NEXT_BATCH --> LOOP
    NEXT_BATCH -->|완료|INTEGRITY[무결성 검증<br/>S3 파일 수 대조]

    INTEGRITY -->|정상|REPORT[아카이빙 리포트 생성<br/>건수/용량/S3경로]
    INTEGRITY -->|불일치|INTEG_ALERT[운영팀 긴급 알림<br/>수동 검증 요청]

    REPORT & INTEG_ALERT --> END([크론 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef external fill:#ECEFF1,stroke:#455A64,stroke-dasharray:3 3
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17

    class CRON,END cron
    class QUERY,LOOP,BATCH,NEXT_BATCH,INTEGRITY system
    class UPLOAD,COMPRESS external
    class LOG_BATCH_OK,REPORT,UPDATE_DB success
    class UPLOAD_ERR,BATCH_ERR,DELETE_ERR,INTEG_ALERT error
    class EMPTY warning
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| S3 업로드 실패 | 최대 3회 재시도, 실패 시 배치 스킵 |
| DB 삭제 실패 | S3는 유지, 삭제 실패 로그, 다음 크론에서 재시도 |
| 무결성 검증 실패 | 운영팀 긴급 알림, 수동 복구 |
| 크론 중단 | 다음 달 크론에서 미처리 대상 포함 |
