# A06 — 주간/월간 리포트 자동 생성

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 주 월요일 06:00 / 매월 1일 06:00 |
| 대상 엔티티 | Branch, Report |
| 조건 | 활성 지점 전체 |
| 결과 | 주간/월간 KPI 리포트 생성, 본사/지점장 발송 |
| 관련 화면 | SCR-093 지점 성과 리포트, SCR-099 리포트 생성 |

## 2. 발생 조건

- 주간: 매주 월요일 06:00 — 직전 주 월~일 집계
- 월간: 매월 1일 06:00 — 직전 월 1일~말일 집계
- 지점 status = ACTIVE인 지점만 대상
- 집계 항목: 신규회원수, 매출, 수업수, 출석률, 해지율, 리드전환율

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>월요일 06:00 / 매월 1일 06:00]) --> TYPE{리포트 유형}
    TYPE -->|주간|PERIOD_W[집계 기간 설정<br/>직전 주 월~일]
    TYPE -->|월간|PERIOD_M[집계 기간 설정<br/>직전 월 전체]

    PERIOD_W & PERIOD_M --> BRANCH_QUERY[활성 지점 목록 조회]
    BRANCH_QUERY --> LOOP[지점 순회]

    LOOP --> AGG[KPI 집계 쿼리 실행<br/>신규회원/매출/수업/출석/해지/리드]
    AGG -->|성공|BUILD[리포트 데이터 빌드]
    AGG -->|실패|AGG_ERR[집계 오류 로그<br/>해당 지점 스킵]

    BUILD --> RENDER[리포트 렌더링<br/>PDF + JSON]
    RENDER --> SAVE[Report 저장<br/>DB + S3]
    SAVE -->|성공|NOTIFY[담당자 발송<br/>지점장/본사 이메일]
    SAVE -->|실패|SAVE_ERR[저장 실패 로그]

    NOTIFY --> LOG_OK[발송 이력 저장]
    NOTIFY --> LOG_FAIL[발송 실패 로그]

    AGG_ERR & SAVE_ERR & LOG_OK & LOG_FAIL --> NEXT{다음 지점?}
    NEXT --> LOOP
    NEXT -->|완료|ROLLUP[본사 롤업 리포트 생성<br/>전 지점 합산]
    ROLLUP --> HQ_SEND[본사 담당자 발송]
    HQ_SEND --> END([크론 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

    class CRON,END cron
    class TYPE,LOOP,AGG,NEXT system
    class LOG_OK,SAVE,ROLLUP success
    class AGG_ERR,SAVE_ERR,LOG_FAIL error
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 집계 쿼리 실패 | 해당 지점 스킵, 오류 로그, 관리자 알림 |
| 저장 실패 | 재시도 1회, 실패 시 오류 로그 |
| 발송 실패 | 발송 실패 로그, 관리자 수동 재발송 |
