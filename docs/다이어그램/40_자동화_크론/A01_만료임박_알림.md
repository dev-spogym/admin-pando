# A01 — 만료 임박 알림 자동 발송

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매일 00:00 |
| 대상 엔티티 | Membership (이용권) |
| 조건 | 만료일까지 D-30, D-7, D-1 |
| 결과 | 대상 회원에게 알림(SMS/푸시/카카오) 발송 |
| 관련 화면 | SCR-072 자동 알림 설정, SCR-M004 회원 상세 |

## 2. 발생 조건

- `Membership = ACTIVE`
- `Membership. - NOW() IN (30일, 7일, 1일)`
- 동일 회원에게 동일 D-day 알림은 하루 1회 중복 방지
- 알림 수신 거부() 회원 제외

## 3. 다이어그램

```mermaid

    QUERY --> EMPTY[대상 없음<br/>로그 기록 후 종료]
    QUERY --> LOOP[대상 목록 순회]

    LOOP --> OPTOUT{알림 수신<br/>거부 여부}
    OPTOUT -->|거부|SKIP[발송 스킵<br/>로그: opt-out]
    OPTOUT -->|수신 허용|DUPCHECK{당일 중복<br/>발송 여부}

    DUPCHECK -->|중복|SKIP2[발송 스킵<br/>로그: already-sent]
    DUPCHECK -->|최초|DDAY{D-day 분기}

    DDAY -->|D-30|MSG30[메시지 생성<br/>이용권 만료 30일 전 안내]
    DDAY -->|D-7|MSG7[메시지 생성<br/>이용권 만료 7일 전 안내]
    DDAY -->|D-1|MSG1[메시지 생성<br/>이용권 만료 1일 전 안내]

    MSG30 & MSG7 & MSG1 --> CHANNEL{발송 채널 선택}

    CHANNEL -->|앱 푸시|PUSH[EXT: 푸시 발송]

    NEXT -->|완료|REPORT[실행 리포트 저장<br/>성공/실패/스킵 수]
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 채널 발송 실패 | 최대 3회 자동 재시도 (exponential backoff) |
| 3회 모두 실패 | 실패 로그 저장, 관리자 경보 발송 |
| 크론 자체 실패 | 다음 날 동일 조건 재실행 (D-day 재계산) |
| DB 조회 실패 | 크론 즉시 중단, 에러 로그 적재, 운영팀 알림 |
