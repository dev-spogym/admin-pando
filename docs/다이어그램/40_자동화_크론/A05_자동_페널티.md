# A05 — 자동 페널티 발생

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 이벤트 기반 — 수업 종료 시 노쇼 감지 |
| 대상 엔티티 | Lesson, Reservation, Penalty |
| 조건 | 예약 있음 + 수업 종료 후 출석 미체크 = 노쇼 |
| 결과 | Penalty 생성, 예약 횟수 차감, 알림 발송 |
| 관련 화면 | SCR-C008 페널티 관리, DLG-C013 자동페널티 정책 |

## 2. 발생 조건

- `Lesson = COMPLETED`
- `Reservation = BOOKED` (출석 처리 안 됨)
- 수업 종료 후 N분 이내 자동 트리거 (기본 30분)
- 페널티 정책(AutoPenaltyPolicy)이 활성화된 지점

## 3. 다이어그램

```mermaid
flowchart TD
    EVENT([이벤트 트리거<br/>Lesson = COMPLETED]) --> QUERY[노쇼 예약 조회<br/>BOOKED + 미출석]

    QUERY -->|없음|EMPTY[노쇼 없음<br/>종료]
    QUERY -->|있음|LOOP[노쇼 예약 순회]

    LOOP --> POLICY_CHECK{자동 페널티<br/>정책 활성?}
    POLICY_CHECK -->|비활성|SKIP[스킵: 수동 처리]
    POLICY_CHECK -->|활성|EXEMPTION{면제 조건<br/>확인}

    EXEMPTION -->|면제 해당|SKIP_EXEMPT[스킵: 면제 처리<br/>로그 기록]
    EXEMPTION -->|면제 아님|PENALTY_CALC[페널티 계산<br/>정책 기준 횟수/포인트]

    PENALTY_CALC --> TXN[트랜잭션 시작]
    TXN --> CREATE_PENALTY[Penalty 생성<br/>type=NO_SHOW]
    CREATE_PENALTY --> UPDATE_RES[Reservation<br/>= NO_SHOW]
    UPDATE_RES --> DEDUCT{횟수 차감<br/>정책 여부}

    DEDUCT -->|차감|DEDUCT_SESSION[수강권 잔여 횟수 -1]
    DEDUCT -->|미차감|COMMIT_TXN[트랜잭션 커밋]
    DEDUCT_SESSION --> COMMIT_TXN

    COMMIT_TXN -->|성공|NOTIFY[페널티 알림 발송<br/>회원에게 노쇼 안내]
    COMMIT_TXN -->|실패|ROLLBACK[롤백<br/>오류 로그]

    NOTIFY --> LOG_OK[알림 이력 저장]
    NOTIFY --> LOG_FAIL[알림 실패 로그]

    SKIP & SKIP_EXEMPT & LOG_OK & LOG_FAIL & ROLLBACK --> NEXT{다음 예약?}
    NEXT --> LOOP
    NEXT -->|완료|END([처리 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 트랜잭션 실패 | 롤백, 오류 로그, 이벤트 큐 재투입 |
| 정책 변경 후 소급 적용 | 변경 시점 이후 수업에만 적용 |
| 면제 조건 오판 | 관리자가 SCR-C008에서 수동 취소 가능 |
