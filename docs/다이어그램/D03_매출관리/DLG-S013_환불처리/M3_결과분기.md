## 1. 목적
DLG-S013 확인 후 승인 요청 또는 즉시 처리 분기를 표현한다.

## 2. 전제조건
- DLG-S013에서 확인 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
    CONFIRM[확인 버튼 클릭] --> ROUTE{처리 경로}

    ROUTE -->|": 즉시 처리"|PG_CANCEL[PG 취소 요청]
    ROUTE -->|": 승인 필요"|SEND_APPROVAL[승인 요청 발송\nowner/primary 알림]

    PG_CANCEL --> CANCEL_DONE[취소 완료\n영수증 발행]
    PG_CANCEL --> PG_ERR[PG 오류 토스트]

    SEND_APPROVAL --> WAIT_STATE[승인 대기 상태\n정보 토스트]

    CANCEL_DONE --> CLOSED([모달 닫힘\-S012 갱신])
    WAIT_STATE --> CLOSED

    classDef newFeature fill:#F3E5F5,stroke:#7B1FA2,color:#4A148C,stroke-dasharray:4 4
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef info fill:#E0F7FA,stroke:#00838F
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

    class SEND_APPROVAL newFeature
    class ROUTE,PG_CANCEL system
    class CANCEL_DONE success
    class WAIT_STATE info
    class PG_ERR error
```
