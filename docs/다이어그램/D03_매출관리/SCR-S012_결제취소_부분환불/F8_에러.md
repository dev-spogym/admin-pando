## 1. 목적
SCR-S012에서 발생 가능한 에러 및 예외 상황을 표현한다.

## 2. 전제조건
- SCR-S012에서 취소/환불 요청 실행 중

## 3. 다이어그램

```mermaid
flowchart TD
    PROCESS[취소/환불 처리 중]

    PROCESS -->|": PG 취소 오류"|PG_ERR[PG 취소 실패\n오류코드 표시\n수동 처리 안내]
    PROCESS -->|": 이미 취소된 건"|DUPLICATE_ERR[중복 취소 오류\n이미 취소된 결제입니다.]
    PROCESS -->|": 환불액 > 결제액"|EXCEED_ERR[금액 초과 오류\n환불금액이 결제금액을 초과합니다.]
    PROCESS -->|": 네트워크 오류"|NET_ERR[네트워크 오류\n잠시 후 재시도 안내]
    PROCESS -->|": 처리 타임아웃"|TIMEOUT_ERR[타임아웃\n관리자 확인 요청]

    NET_ERR --> NET_RETRY{자동 재시도 3회}
    NET_RETRY -->|": 성공"|PROCESS_OK[처리 완료]
    NET_RETRY -->|": 3회 실패"|MANUAL_PROCESS[수동 처리 안내\n관리자 알림]

    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
    classDef system fill:#EDE7F6,stroke:#5E35B2

    class PG_ERR,DUPLICATE_ERR,EXCEED_ERR,TIMEOUT_ERR error
    class PROCESS_OK success
    class NET_ERR,MANUAL_PROCESS warning
    class NET_RETRY,PROCESS system
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| PROCESS | PG_ERR | PG 취소 실패 |
| PROCESS | DUPLICATE_ERR | 중복 취소 시도 |
| PROCESS | EXCEED_ERR | 부분환불액 초과 |
| PROCESS | NET_ERR | 네트워크 오류 |
| PROCESS | TIMEOUT_ERR | 처리 타임아웃 |
| NET_ERR | NET_RETRY | 자동 재시도 |
| NET_RETRY | MANUAL_PROCESS | 3회 실패 → 수동 처리 |
