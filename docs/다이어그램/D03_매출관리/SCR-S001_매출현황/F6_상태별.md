## 1. 목적
SCR-S001의 로딩/빈/에러/권한없음/오프라인 등 모든 UI 상태를 표현한다.

## 2. 전제조건
- SCR-S001 진입 시도

## 3. 다이어그램

```mermaid
flowchart TD
    ENTRY([SCR-S001 진입]) --> LOAD_STATE{데이터 로드 상태}

    LOAD_STATE -->|": "|SKELETON[로딩 상태\n스켈레톤 행 5개 표시]
    LOAD_STATE -->|": 데이터 있음"|NORMAL[정상 상태\ + 요약바]
    LOAD_STATE -->|": 데이터 없음"|EMPTY[빈 상태\ 아이콘\n매출 데이터가 없습니다.]
    LOAD_STATE -->|": API 오류"|ERROR[에러 상태\n매출 데이터를 불러오지 못했습니다. 토스트\nconsole]
    LOAD_STATE -->|": 네트워크 없음"|OFFLINE[오프라인 상태\n네트워크 연결을 확인하세요.]
    LOAD_STATE -->|": trainer 역할"|TRAINER_STATE[트레이너 상태\n담당자 필터 자동 적용\n안내 배너 표시]
    LOAD_STATE -->|": fc/staff 역할"|FC_STATE[프론트 상태\ 버튼만 활성화]

    SKELETON -->|": 로드 완료"|LOAD_STATE
    ERROR -->|": 재시도"|ENTRY
    OFFLINE -->|": 연결 복구"|ENTRY

    NORMAL --> TAB_STATE{탭별 상태}
    TAB_STATE -->|": 탭 필터 결과 없음"|TAB_EMPTY[탭 빈 상태\n해당 탭 데이터 없음]
    TAB_STATE -->|": 데이터 있음"|TAB_NORMAL[탭 정상 표시]

    classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef info fill:#E0F7FA,stroke:#00838F

    class NORMAL,TAB_NORMAL screen
    class ERROR,EMPTY,TAB_EMPTY error
    class SKELETON,OFFLINE,TRAINER_STATE,FC_STATE warning
    class LOAD_STATE,TAB_STATE system
    class ENTRY info
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| LOAD_STATE | SKELETON | → 스켈레톤 |
| LOAD_STATE | NORMAL | 정상 데이터 로드 |
| LOAD_STATE | EMPTY | 데이터 0건 |
| LOAD_STATE | ERROR | API 오류 |
| LOAD_STATE | OFFLINE | 네트워크 없음 |
| LOAD_STATE | TRAINER_STATE | 트레이너 자동 필터 |
| LOAD_STATE | FC_STATE | 프론트 제한 |
| ERROR | ENTRY | 재시도 |
| OFFLINE | ENTRY | 연결 복구 후 재시도 |
