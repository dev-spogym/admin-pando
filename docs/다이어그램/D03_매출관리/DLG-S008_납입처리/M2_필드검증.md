## 1. 목적
DLG-S008 납입 금액 및 결제수단 입력 검증 규칙을 표현한다.

## 2. 전제조건
- DLG-S008 열림 상태

## 3. 다이어그램

```mermaid
flowchart TD
    OPEN[DLG-S008 열림] --> PREFILL[납입금액 자동 입력\n해당 회차 금액]

    PREFILL -->|": 금액 수정"|AMOUNT_CHECK{금액 검증}
    AMOUNT_CHECK -->|": 0 < 금액 ≤ 회차금액"|AMT_VALID[금액 유효]
    AMOUNT_CHECK -->|": 금액 = 0"|AMT_ERR[0원 불가 오류]
    AMOUNT_CHECK -->|": 초과"|EXCEED_ERR[회차금액 초과 오류]

    AMT_VALID -->|": 결제수단 선택"|METHOD_CHECK{수단 선택 확인}
    METHOD_CHECK -->|": 선택됨"|READY[처리 버튼 활성]
    METHOD_CHECK -->|": 미선택"|METHOD_ERR[결제수단 선택 오류]

    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

    class AMOUNT_CHECK,METHOD_CHECK system
    class AMT_VALID,READY success
    class AMT_ERR,EXCEED_ERR,METHOD_ERR error
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| OPEN | PREFILL | 회차 금액 자동 입력 |
| AMOUNT_CHECK | AMT_VALID | 금액 유효 |
| AMOUNT_CHECK | EXCEED_ERR | 회차 금액 초과 |
| METHOD_CHECK | READY | 결제수단 선택 완료 |
