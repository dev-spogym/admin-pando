## 1. 목적
DLG-S009 할부 등록 폼의 필드별 검증 규칙을 표현한다.

## 2. 전제조건
- DLG-S009 열림 상태

## 3. 다이어그램

```mermaid
flowchart TD
    SUBMIT[등록 버튼 클릭] --> MEMBER_CHECK{회원 선택 여부}
    MEMBER_CHECK --> AMOUNT_CHECK{총 할부금액 검증}
    MEMBER_CHECK --> ERR_MEMBER[회원 선택 필수]

    AMOUNT_CHECK -->|": 금액"|INSTALLMENT_CHECK{할부 회차 검증\n2~24회}
    AMOUNT_CHECK --> ERR_AMOUNT[금액 필수]

    INSTALLMENT_CHECK -->|": 2~24"|DATE_CHECK{첫 납기일 검증\n오늘 이후}
    INSTALLMENT_CHECK -->|": 범위 초과"|ERR_COUNT[2~24회 입력 오류]

    DATE_CHECK --> ALL_VALID[모든 검증 통과\n등록 버튼 활성]
    DATE_CHECK -->|": 과거 날짜"|ERR_DATE[과거 날짜 불가]

    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

    class MEMBER_CHECK,AMOUNT_CHECK,INSTALLMENT_CHECK,DATE_CHECK system
    class ALL_VALID success
    class ERR_MEMBER,ERR_AMOUNT,ERR_COUNT,ERR_DATE error
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| MEMBER_CHECK | ERR_MEMBER | 회원 미선택 |
| AMOUNT_CHECK | ERR_AMOUNT | 금액 0 |
| INSTALLMENT_CHECK | ERR_COUNT | 2~24회 범위 초과 |
| DATE_CHECK | ERR_DATE | 과거 날짜 |
