## 1. 목적
SCR-S002에서 6개 역할별 접근 범위와 제한을 표현한다.

## 2. 전제조건
- 로그인 완료

## 3. 다이어그램

```mermaid
flowchart LR
    AUTH{역할 분기}

    AUTH -->|": primary"|P[슈퍼관리자\n전체 접근\n상품 등록 가능]
    AUTH -->|": owner"|O[센터장\n전체 접근\n상품 등록 가능]
    AUTH -->|": manager"|M[매니저\n전체 접근\n상품 등록 가능]
    AUTH -->|": fc/staff"|FC[프론트\n전체 접근\n상품 등록 가능]
    AUTH -->|": trainer"|TR[트레이너\n접근 가능\n상품 등록 버튼 숨김]
    AUTH -->|": readonly"|RO[읽기전용\n접근 차단]

    P --> FULL_ACCESS[전체 기능\n상품선택+장바구니+결제+등록]
    O --> FULL_ACCESS
    M --> FULL_ACCESS
    FC --> FULL_ACCESS
    TR --> LIMITED[제한 기능\n상품선택+장바구니+결제\n상품 등록 불가]
    RO --> BLOCKED[접근 차단]

    classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
    classDef rbacBlocked fill:#F5F5F5,stroke:#9E9E9E,color:#616161

    class AUTH system
    class FULL_ACCESS success
    class LIMITED warning
    class BLOCKED rbacBlocked
```
