# X02 — 회원 재등록 + 이용권 연장 + 쿠폰 적용

## 1. 시나리오 개요

이용권이 만료되었거나 만료 임박한 기존 회원이 재등록을 요청. 매니저가 회원 상세 → 이용권 탭에서 연장 등록 처리 시 보유 쿠폰을 적용하여 할인 결제하는 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 회원 재등록 요청 (방문/전화) |
| 종료 조건 | 이용권 연장 완료 + 만료일 갱신 |
| 참여 도메인 | 회원관리(D2), 매출관리(D3), 마케팅(D8) |

## 2. 전제조건

- 매니저 계정 로그인 상태
- 대상 회원이 시스템에 등록되어 있음
- 회원이 유효한 쿠폰을 보유하고 있음 (또는 쿠폰 없이 정가 결제 가능)
- PG 연동 정상 상태

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원 | 재등록을 원하는 기존 회원 |
| 매니저 | 이용권 연장 및 결제 처리 담당 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| PG | 결제 대행사 |
| 알림서비스 | SMS/카카오톡 발송 |

## 4. 시퀀스 다이어그램

```mermaid

    actor MB as 회원
    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    participant PG as PG사
    participant N as 알림서비스

    MB->>M: 1. 재등록 요청 %% M->>API: 2. GET /memberships %% API->>DB: 3. memberships WHERE member_id=?
    DB-->>API: 4. 이용권 목록 (만료된 이용권 포함)
    API-->>M: 5. 이용권 현황
    Note over M: SCR-M004 회원 상세 > 이용권 탭

    M->>API: 6. GET /?status=VALID %% API->>DB: 7. WHERE member_id=? AND status=VALID
    DB-->>API: 8. 유효 쿠폰 목록
    API-->>M: 9. 쿠폰 목록 반환

    alt 쿠폰 보유
        M->>API: 10. GET %% API-->>M: 11. 상품 정보 (정가)
        Note over M: DLG-M018 연장 등록 모달 열기
        M->>API: 12. POST {product_id, coupon_id} %% API->>DB: 13. 쿠폰 할인율/금액 조회
        DB-->>API: 14. 할인 정보
        API-->>M: 15. 최종 결제금액 미리보기
        M->>API: 16. POST /sales {member_id, product_id, coupon_id, payment_method} %% else 쿠폰 없음 — 정가 결제
        Note over M: DLG-M018 연장 등록 모달 — 쿠폰 없음
        M->>API: 16. POST /sales {member_id, product_id, payment_method} %% end

    API->>PG: 17. 결제 승인 요청

    alt PG 승인 성공
        PG-->>API: 18. 승인 응답 %% API->>DB: 19. sales
        API->>DB: 20. memberships SET expiry_date=new_date, status=ACTIVE
        API->>DB: 21. SET status=USED (쿠폰 사용 시)
        DB-->>API: 22. OK
        API->>N: 23. 연장 완료 알림 발송 트리거
        N-->>MB: 24. SMS/카카오톡 — 이용권 연장 완료 + 새 만료일
        API-->>M: 25. 200 OK {sale_id, new_expiry_date}
        Note over M: 만료일 갱신 확인
    else PG 승인 실패
        PG-->>API: 26. 에러 응답 %% API-->>M: 27. 결제 실패 {error_code}
        Note over M: 결제 수단 변경 후 재시도
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 6 | GET /?status=VALID | 해당 회원의 유효 쿠폰만 필터링하여 조회 |
| 12 | POST | 결제 전 쿠폰 적용 후 최종금액 미리보기. 실제 결제는 아님 |
| 16 | POST /sales | 쿠폰 ID 포함 시 서버에서 자동 할인 적용 |
| 20 | memberships | 기존 이용권 만료일을 연장. 동일 상품이면 날짜 누적, 다른 상품이면 신규 레코드 생성 |
| 21 | | 쿠폰 사용 처리. 한 번 사용된 쿠폰은 USED 상태로 전환 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 쿠폰 만료 | status=EXPIRED 쿠폰은 목록에 미표시, 선택 불가 |
| 쿠폰 적용 불가 상품 | 서버에서 쿠폰-상품 호환성 검증 → 422 에러 + 안내 메시지 |
| 이미 활성 이용권 보유 | 중복 등록 경고 토스트 표시 후 확인 시 진행 |
| PG 실패 후 쿠폰 롤백 | 결제 실패 시 쿠폰 상태 VALID로 자동 복구 |
