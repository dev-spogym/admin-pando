# X22 — 체성분 임계치 초과 → 자동 상담 트리거

## 1. 시나리오 개요

X04 체성분 측정 결과가 임계값(예: 체지방률 35% 초과 또는 전월 대비 +5% 이상)을 초과 → 자동으로 FC 상담 트리거 생성 → FC 알림 발송 → FC가 상담 일정 잡는 시나리오.

| 항목 | 내용 | |------|------| | 트리거 | 체성분 데이터 임계값 초과 감지 | | 종료 조건 | FC 상담 일정 등록 | | 참여 도메인 | 통합운영(D11), 수업관리(D4), 마케팅(D8) |

## 2. 전제조건

- 체성분 측정 데이터가 DB에 저장됨 (X04 시나리오 완료)
- 체성분 임계값 정책이 설정되어 있음
- 담당 FC가 배정되어 있음

## 3. 참여 액터

| 액터 | 설명 | |------|------| | CRM API | FitGenie CRM 백엔드 | | DB | 데이터베이스 | | 분석엔진 | 임계값 초과 감지 | | FC | 담당 피트니스 컨설턴트 | | 회원 | 상담 대상자 | | 알림서비스 | FC 및 회원 알림 |

## 4. 시퀀스 다이어그램

```mermaid

    participant API as CRM API
    participant DB as DB
    participant AN as 분석엔진
    actor FC as FC
    actor MB as 회원
    participant N as 알림서비스

    Note over API,AN: [X04 체성분 저장 직후 트리거]
    API->>AN: 1. 임계값 초과 분석 요청 {member_id, composition_id} %% AN->>DB: 2. body_threshold_policy WHERE center_id=?
    DB-->>AN: 3. 임계값 정책 {fat_rate_max, fat_increase_max, muscle_decrease_max}
    AN->>DB: 4. body_compositions WHERE member_id=? ORDER BY measured_at DESC LIMIT 2
    DB-->>AN: 5. 최근 2회 측정 데이터

    AN->>AN: 6. 임계값 초과 여부 계산 %% Note over AN: 체지방률 35% 초과 OR 전월 대비 체지방 +5% 이상

    alt 임계값 초과
        AN-->>API: 7. 초과 결과 {triggered: true, reasons: [...]} %% API->>DB: 8. fc_id FROM member_assignments WHERE member_id=?
        DB-->>API: 9. 담당 FC ID
        API->>DB: 10. consultation_triggers {member_id, fc_id, reason, composition_id, status=PENDING} %% DB-->>API: 11. trigger_id
        API->>N: 12. FC 긴급 상담 알림 {fc_id, member_id, reason_summary} %% N-->>FC: 13. 앱/웹 알림 — "⚠️ 회원 OOO 체성분 임계 초과 — 상담 필요"

        Note over FC,API: [FC 상담 일정 등록]
        FC->>API: 14. GET /consultation-triggers?status=PENDING %% API-->>FC: 15. 상담 트리거 목록
        FC->>MB: 16. 회원 연락 (전화/앱 메시지)
        FC->>API: 17. POST /consultations {trigger_id, member_id, scheduled_at, type=INBODY} %% API->>DB: 18. consultations {status=SCHEDULED}
        API->>DB: 19. consultation_triggers SET status=SCHEDULED, consultation_id=?
        DB-->>API: 20. consultation_id
        API->>N: 21. 상담 예약 확인 알림 %% N-->>MB: 22. SMS — "체성분 상담이 {scheduled_at}으로 예약되었습니다"
        API-->>FC: 23. 200 OK {consultation_id}

    else 임계값 미초과
        AN-->>API: 24. {triggered: false} %% Note over API: 정상 범위, 추가 액션 없음
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 | |------|--------|------| | 6 | 임계값 계산 | 절댓값 초과 + 변화율 초과 복합 조건 평가 | | 10 | consultation_triggers | 자동 생성된 상담 트리거. FC가 확인하고 처리 | | 17 | POST /consultations | trigger_id 연결로 어떤 체성분 이슈 때문인지 추적 가능 |

## 6. 예외/분기

| 상황 | 처리 방법 | |------|-----------| | 담당 FC 미배정 | 센터 매니저에게 알림 fallback | | 이미 PENDING 트리거 존재 | 중복 트리거 생성 방지, 기존 트리거 갱신 | | 임계값 정책 미설정 | 트리거 미생성, 임계값 설정 유도 | | FC가 장기간 미처리 | N일 경과 시 매니저에게 에스컬레이션 알림 |
