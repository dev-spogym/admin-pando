# X12 — 그룹수업 일괄 생성 → 수강 오픈 → 만석 처리

## 1. 시나리오 개요

매니저가 그룹수업 시간표를 일괄 등록 → 수강 신청 오픈 → 회원들이 예약 → 정원 초과 시 만석 처리 및 대기열 등록까지의 시나리오.

| 항목 | 내용 | |------|------| | 트리거 | 매니저의 그룹수업 일괄 생성 | | 종료 조건 | 수업 생성 완료 + 예약 오픈 | | 참여 도메인 | 수업관리(D4), 회원관리(D2) |

## 2. 전제조건

- 매니저 계정 로그인 상태 (수업 관리 권한)
- 강사가 시스템에 등록되어 있음
- 그룹수업 템플릿이 있거나 직접 입력

## 3. 참여 액터

| 액터 | 설명 | |------|------| | 매니저 | 수업 일괄 생성 담당 | | 회원 | 수강 신청자 | | CRM API | FitGenie CRM 백엔드 | | DB | 데이터베이스 | | 알림서비스 | 수업 오픈/만석 알림 |

## 4. 시퀀스 다이어그램

```mermaid

    actor M as 매니저
    actor MB as 회원
    participant API as CRM API
    participant DB as DB
    participant N as 알림서비스

    Note over M,API: [그룹수업 일괄 생성]
    M->>API: 1. GET /class-templates %% API-->>M: 2. 템플릿 목록
    Note over M: SCR-C003 시간표 일괄 등록 화면
    M->>API: 3. POST {template_id, start_date, end_date, repeat_rule, instructor_id} %% API->>DB: 4. 반복 규칙 파싱 → 개별 수업 레코드 생성
    Note over DB: 예: 매주 월/수/금 18:00, 8주 → 24개 수업 
    DB-->>API: 5. 생성된 수업 ID 목록
    API->>DB: 6. lessons SET status=OPEN (예약 오픈)
    DB-->>API: 7. OK
    Note over M: DLG-C008 일괄 생성 확인 모달

    alt 일괄 생성 성공
        API->>N: 8. 수업 오픈 알림 발송 (구독 회원 대상) %% N-->>MB: 9. SMS/앱 알림 — "새 그룹수업 오픈"
        API-->>M: 10. 201 Created {lesson_ids, count}
        Note over M: SCR-C005 그룹수업 현황에서 확인
    else 강사 일정 충돌
        API-->>M: 11. 409 Conflict {conflicting_dates}
        Note over M: 충돌 날짜 확인 후 강사 변경 또는 해당 날짜 제외
    end

    Note over MB,API: [회원 수강 신청]
    MB->>API: 12. GET /lessons?type=GROUP&status=OPEN %% API-->>MB: 13. 오픈 그룹수업 목록
    MB->>API: 14. POST / {lesson_id, member_id} %% API->>DB: 15. lessons WHERE id=? FOR 
    DB-->>API: 16. 수업 정보 (현재 인원, 최대 정원)

    alt 정원 미달
        API->>DB: 17. {status=CONFIRMED} %% API->>DB: 18. lessons SET current_count = current_count + 1
        DB-->>API: 19. reservation_id
        API->>N: 20. 예약 확인 알림
        N-->>MB: 21. SMS — "수업 예약 완료"
        API-->>MB: 22. 201 Created {reservation_id}
    else 정원 초과 — 만석
        API->>DB: 23. waitlists {lesson_id, member_id, position} %% DB-->>API: 24. waitlist 순번
        API->>N: 25. 대기 등록 알림
        N-->>MB: 26. SMS — "만석. 대기 N번 등록 완료"
        API-->>MB: 27. 200 OK {waitlist_position}
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 | |------|--------|------| | 3 | POST | repeat_rule: RRULE 또는 커스텀 반복 규칙 (매주/격주/매일) | | 4 | 반복 규칙 파싱 | 서버에서 각 날짜 계산 후 개별 레코드 | | 15 | FOR | 동시 예약 시 race condition 방지. 비관적 락 사용 | | 23 | waitlists | 대기열 순번은 등록 시각 기준 자동 부여. X26 대기열 시나리오 연계 |

## 6. 예외/분기

| 상황 | 처리 방법 | |------|-----------| | 강사 일정 충돌 | 409 반환, 충돌 날짜 목록 표시, 제외 또는 강사 변경 선택 | | 동시 예약 경쟁 | FOR 로 정원 초과 방지 | | 회원 이용권 없음 | 예약 불가, 이용권 구매 유도 | | 일괄 생성 중 오류 | 트랜잭션 전체 롤백 또는 생성 성공한 건만 커밋 (정책 설정) |
