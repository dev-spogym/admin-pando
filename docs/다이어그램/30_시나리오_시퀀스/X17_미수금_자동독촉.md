# X17 — 미수금 발생 → 자동 독촉 → 결제 완료

## 1. 시나리오 개요

결제 실패 또는 후불 정책으로 미수금 발생 → 자동 독촉 알림 발송 → 회원이 결제 완료 → 미수금 해소까지의 시나리오.

| 항목 | 내용 | |------|------| | 트리거 | 결제 실패 또는 후불 미납 | | 종료 조건 | 미수금 완납 + 이용권 정상화 | | 참여 도메인 | 매출관리(D3), 마케팅(D8) |

## 2. 전제조건

- 미수금 독촉 정책이 설정되어 있음 (D+1, D+3, D+7)
- 회원 연락처가 등록되어 있음

## 3. 참여 액터

| 액터 | 설명 | |------|------| | 회원 | 미수금 당사자 | | CRM API | FitGenie CRM 백엔드 | | DB | 데이터베이스 | | 스케줄러 | 미수금 독촉 크론잡 | | 매니저 | 수동 독촉 및 결제 처리 | | PG | 결제 대행사 | | 알림서비스 | 독촉 알림 발송 |

## 4. 시퀀스 다이어그램

```mermaid

    actor MB as 회원
    participant API as CRM API
    participant DB as DB
    participant SCH as 스케줄러
    actor M as 매니저
    participant PG as PG사
    participant N as 알림서비스

    Note over API,DB: [미수금 발생 — 결제 실패 시]
    API->>DB: 1. unpaid_balances {member_id, amount, due_date, status=UNPAID} %% DB-->>API: 2. unpaid_id
    API->>DB: 3. memberships SET status=SUSPENDED (정책에 따라)
    DB-->>API: 4. OK

    Note over SCH,DB: [자동 독촉 크론잡 — 매일 실행]
    SCH->>API: 5. 미수금 독촉 대상 조회 %% API->>DB: 6. unpaid_balances WHERE status=UNPAID AND due_date <= NOW(동작)-dunning_day
    DB-->>API: 7. 독촉 대상 목록

    loop 각 미수금별
        API->>DB: 8. dunning_policy WHERE overdue_days=? %% DB-->>API: 9. 독촉 정책 (D+1: 알림, D+3: 경고, D+7: 이용권 정지)
        API->>N: 10. 독촉 알림 발송 {member_id, amount, overdue_days} %% N-->>MB: 11. SMS — "미납금 {amount}원 납부 요청 (D+N)"
        API->>DB: 12. dunning_logs {unpaid_id, sent_at, level}
    end

    alt 회원 직접 결제 (링크/방문)
        MB->>API: 13. POST {payment_method} %% API->>PG: 14. 결제 승인 요청
        PG-->>API: 15. 승인 응답
        API->>DB: 16. unpaid_balances SET status=PAID, paid_at=NOW(동작)
        API->>DB: 17. memberships SET status=ACTIVE (이용권 복구)
        DB-->>API: 18. OK
        API->>N: 19. 납부 완료 알림 %% N-->>MB: 20. SMS — "미납금 납부 완료. 이용권이 복구되었습니다"
        API-->>MB: 21. 200 OK
    else 매니저 수동 처리
        M->>API: 22. POST {method, memo} %% API->>DB: 23. unpaid_balances SET status=PAID
        API->>DB: 24. memberships SET status=ACTIVE
        DB-->>API: 25. OK
        API->>N: 26. 납부 완료 알림
        N-->>MB: 27. SMS 발송
        API-->>M: 28. 200 OK
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 | |------|--------|------| | 3 | memberships SUSPENDED | 미수금 발생 즉시 또는 D+3 이후 정지 (정책 설정) | | 9 | 독촉 정책 | 단계별 강도 조절: D+1 안내, D+3 경고, D+7 이용권 정지 안내 | | 12 | dunning_logs | 중복 발송 방지. 동일 단계 재발송 여부 체크 | | 17 | memberships ACTIVE | 납부 완료 즉시 이용권 복구 |

## 6. 예외/분기

| 상황 | 처리 방법 | |------|-----------| | 분할 납부 | partial payment 허용 시 amount 부분 업데이트 | | 독촉 수신 거부 | 알림 skip, 매니저 수동 연락 필요 | | PG 재결제 실패 | 에러 코드 안내, 수동 납부 유도 | | 미수금 탕감 | 매니저가 status=WAIVED 처리 가능 |
