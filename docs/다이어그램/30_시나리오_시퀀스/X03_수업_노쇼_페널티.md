# X03 — 수업 노쇼 → 페널티 발생 → 알림 발송

## 1. 시나리오 개요

예약된 수업에 회원이 나타나지 않음(노쇼). 강사 또는 시스템이 노쇼 처리 → 자동 페널티 정책에 따라 페널티 부과 → 회원에게 알림 발송하는 시나리오.

| 항목 | 내용 | |------|------| | 트리거 | 수업 시작 시간 경과 후 미출석 확인 | | 종료 조건 | 페널티 기록 저장 + 회원 알림 발송 | | 참여 도메인 | 수업관리(D4), 회원관리(D2), 마케팅(D8) |

## 2. 전제조건

- 회원이 해당 수업에 예약 완료 상태
- 자동 페널티 정책이 설정되어 있음 (DLG-C014)
- 강사 계정이 `fc` 또는 `manager` 역할로 로그인 상태

## 3. 참여 액터

| 액터 | 설명 | |------|------| | 강사 | 수업 진행 및 노쇼 처리 담당 | | 회원 | 노쇼 당사자 | | CRM API | FitGenie CRM 백엔드 | | DB | 데이터베이스 | | 스케줄러 | 자동 노쇼 처리 크론잡 | | 알림서비스 | SMS/카카오톡 발송 |

## 4. 시퀀스 다이어그램

```mermaid

    actor T as 강사
    actor MB as 회원
    participant API as CRM API
    participant DB as DB
    participant SCH as 스케줄러
    participant N as 알림서비스

    Note over SCH,API: [자동 노쇼 감지 — 수업 시작 후 N분 경과]
    SCH->>API: 1. 미출석 예약 조회 트리거 %% API->>DB: 2. WHERE status=CONFIRMED AND start_time < NOW(동작)-interval
    DB-->>API: 3. 미출석 예약 목록 반환

    alt 자동 노쇼 처리 (정책: 자동)
        API->>DB: 4. SET status=NOSHOW %% API->>DB: 5. penalty_policy WHERE lesson_type=?
        DB-->>API: 6. 페널티 정책 (횟수차감/포인트차감/경고)
        API->>DB: 7. penalties {member_id, reason=NOSHOW, amount}
        DB-->>API: 8. penalty_id 반환
        API->>N: 9. 노쇼 페널티 알림 트리거 %% N-->>MB: 10. SMS/카카오톡 — 노쇼 페널티 안내
    else 수동 노쇼 처리 (강사 직접 처리)
        T->>API: 4. PATCH {status=NOSHOW} %% Note over T: SCR-C001 캘린더 또는 DLG-C013에서 처리
        API->>DB: 5. SET status=NOSHOW
        API->>DB: 6. penalty_policy
        DB-->>API: 7. 페널티 정책
        API->>DB: 8. penalties
        DB-->>API: 9. OK
        API->>N: 10. 노쇼 알림 트리거
        N-->>MB: 11. 페널티 알림 발송
        API-->>T: 12. 200 OK {penalty_id}
        Note over T: SCR-C008 페널티 관리에 기록 반영
    end

    Note over MB,API: [회원 이의 제기 플로우 — 선택]
    MB->>T: 13. 노쇼 페널티 이의 제기 %% T->>API: 14. PATCH {status=DISPUTED, memo} %% API->>DB: 15. penalties SET status=DISPUTED
    DB-->>API: 16. OK
    API-->>T: 17. 200 OK
    Note over T: 매니저 검토 후 취소 또는 유지 결정
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 | |------|--------|------| | 1 | 스케줄러 트리거 | 수업 시작 후 N분(정책 설정값) 경과 시 자동 실행 | | 5 | penalty_policy | 수업 유형별 페널티 정책 조회 (횟수차감/포인트차감/경고 누적) | | 7 | penalties | 페널티 기록 생성. reason=NOSHOW, 정책에 따라 amount 결정 | | 10 | 알림 발송 | 페널티 내용, 누적 횟수, 정책 안내 포함 | | 14 | PATCH /penalties | 페널티 이의 제기 상태로 변경. 매니저 검토 대기 |

## 6. 예외/분기

| 상황 | 처리 방법 | |------|-----------| | 페널티 정책 미설정 | 경고만 발송, 실제 페널티 미부과 (정책 설정 안내) | | 이미 취소된 예약 | 노쇼 처리 불가, "취소된 예약" 안내 | | 알림 발송 실패 | 알림 실패 로그 기록, 다음 배치에서 재발송 시도 | | 누적 페널티 임계 도달 | 자동으로 이용권 정지 또는 매니저 알림 발송 |
