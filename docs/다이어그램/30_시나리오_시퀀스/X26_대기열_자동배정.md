# X26 — 대기열 등록 → 자리 발생 → 자동 배정 🆕

## 1. 시나리오 개요

만석 수업에 대기열 등록 → 예약 취소 또는 정원 증가로 자리 발생 → 대기 순번에 따라 자동 배정 → 배정 알림 발송하는 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 수업 예약 취소 또는 정원 증가 이벤트 |
| 종료 조건 | 대기자 수업 예약 확정 |
| 참여 도메인 | 수업관리(D4), 마케팅(D8) |
| 구현 상태 | 🆕 신규 기능 |

## 2. 전제조건

- X12에서 대기열에 등록된 회원이 있음
- 수업에 예약 취소가 발생하거나 매니저가 정원을 늘림
- 대기 회원이 유효한 이용권을 보유하고 있음

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원A | 예약 취소자 |
| 회원B | 대기 등록자 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 알림서비스 | 대기 배정 알림 |

## 4. 시퀀스 다이어그램

```mermaid

    actor MA as 회원A(취소자)
    actor MB as 회원B(대기자)
    participant API as CRM API
    participant DB as DB
    participant N as 알림서비스

    Note over MA,API: [예약 취소 — 자리 발생]
    MA->>API: 1. DELETE %% API->>DB: 2. SET status=CANCELLED
    API->>DB: 3. lessons SET current_count = current_count - 1
    DB-->>API: 4. current_count, max_capacity

    API->>API: 5. 대기열 처리 트리거 {lesson_id} %% API->>DB: 6. waitlists WHERE lesson_id=? AND status=WAITING ORDER BY position ASC LIMIT 1
    DB-->>API: 7. 최우선 대기자 (회원B)

    alt 대기자 존재
        API->>DB: 8. memberships WHERE member_id=MB AND status=ACTIVE %% DB-->>API: 9. 이용권 유효 여부

        alt 이용권 유효
            API->>DB: 10. {member_id=MB, lesson_id, status=CONFIRMED} %% API->>DB: 11. lessons SET current_count = current_count + 1
            API->>DB: 12. waitlists SET status=ASSIGNED WHERE member_id=MB AND lesson_id=?
            DB-->>API: 13. reservation_id
            API->>N: 14. 자동 배정 알림 {member_id=MB} %% N-->>MB: 15. SMS/앱 알림 — "대기 수업 자리가 생겼습니다! 예약이 확정되었습니다"
            API-->>MA: 16. 취소 완료 응답 (200 OK)

        else 이용권 만료/없음 — 다음 대기자
            API->>DB: 17. waitlists SET status=SKIPPED WHERE member_id=MB %% API->>N: 18. 이용권 만료 알림 (회원B)
            N-->>MB: 19. SMS — "대기 자리가 생겼으나 이용권이 만료되어 배정 불가"
            Note over API: 다음 대기자로 반복 처리
            API->>DB: 20. waitlists WHERE lesson_id=? AND status=WAITING ORDER BY position LIMIT 1
        end

    else 대기자 없음
        Note over API: 수업 current_count 감소만 처리, 추가 액션 없음 %% end

    Note over MB,API: [회원B 대기 취소]
    MB->>API: 21. DELETE %% API->>DB: 22. waitlists SET status=CANCELLED
    API->>DB: 23. waitlists SET position=position-1 WHERE lesson_id=? AND position > cancelled_position
    DB-->>API: 24. OK
    API-->>MB: 25. 200 OK
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 6 | LIMIT 1 ORDER BY position | 대기 순번 1번부터 순차 처리 |
| 8 | 이용권 유효성 재확인 | 대기 등록 후 이용권이 만료됐을 수 있어 배정 전 재검증 |
| 12 | waitlists ASSIGNED | 배정 완료 상태로 변경. 이후 중복 배정 방지 |
| 23 | position 재정렬 | 취소 발생 시 뒷순번 position -1 처리 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 배정 알림 수신 거부 | 알림 skip, 대기 취소 유도 안내 |
| 전체 대기자 이용권 만료 | 모두 SKIPPED, 수업 인원 미채움 |
| 응답 제한 시간 내 미확인 | 자동 배정 (확인 불필요 정책) 또는 N시간 내 확인 요구 |
| 동시 취소 race condition | DB 트랜잭션으로 단일 배정 보장 |
