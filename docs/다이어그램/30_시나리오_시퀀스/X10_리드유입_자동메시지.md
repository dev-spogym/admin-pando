# X10 — 리드 유입 → 자동 메시지 → 상담 예약

## 1. 시나리오 개요

외부 채널(SNS 광고, 랜딩 페이지, 전화)을 통해 리드가 유입 → 자동 알림 설정에 따라 환영 메시지 자동 발송 → FC 배정 알림 → 상담 예약까지의 리드 관리 플로우.

| 항목 | 내용 |
|------|------|
| 트리거 | 신규 리드 등록 이벤트 |
| 종료 조건 | 상담 예약 완료 또는 팔로업 예약 |
| 참여 도메인 | 마케팅(D8), 수업관리(D4) |

## 2. 전제조건

- 자동 알림 규칙이 설정되어 있음 (SCR-072)
- 리드 유입 채널이 CRM에 연동되어 있음
- FC 배정 로직이 설정되어 있음 (순번/담당자 고정)

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 리드 | 잠재 고객 |
| 외부채널 | SNS 광고 / 랜딩 페이지 / 전화 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 자동화엔진 | 알림 규칙 평가 및 실행 |
| FC | 배정된 피트니스 컨설턴트 |
| 알림서비스 | SMS/카카오톡 발송 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor L as 리드
    participant CH as 외부채널
    participant API as CRM API
    participant DB as DB
    participant AUT as 자동화엔진
    actor FC as FC
    participant N as 알림서비스

    L->>CH: 1. 상담 신청 (폼 작성 / 전화) %% E_L_INQUIRY_01
    CH->>API: 2. POST /leads { name, phone, channel, source_utm } %% E_CH_CREATE_LEAD_01
    API->>DB: 3. INSERT leads { status=INQUIRY, created_at=NOW() }
    DB-->>API: 4. lead_id 반환

    API->>AUT: 5. 리드 등록 이벤트 발행 { lead_id, channel } %% E_API_LEAD_EVENT_01
    AUT->>DB: 6. SELECT automation_rules WHERE trigger=LEAD_CREATED AND active=true
    DB-->>AUT: 7. 자동화 규칙 목록

    alt 환영 메시지 규칙 존재
        AUT->>N: 8. 환영 메시지 발송 { lead_id, template_id } %% E_AUT_WELCOME_01
        N-->>L: 9. SMS/카카오톡 — "상담 신청 감사합니다. 빠른 시간 내 연락드리겠습니다"
        AUT->>DB: 10. INSERT automation_logs { rule_id, lead_id, sent_at }
    end

    AUT->>DB: 11. FC 배정 로직 실행 (순번/부하 기반) %% E_AUT_ASSIGN_FC_01
    DB-->>AUT: 12. 배정 FC ID
    AUT->>DB: 13. UPDATE leads SET fc_id=?, status=ASSIGNED
    AUT->>N: 14. FC 내부 알림 { fc_id, lead_id } %% E_AUT_NOTIFY_FC_01
    N-->>FC: 15. 앱/웹 알림 — "새 리드 배정: OOO님"

    Note over FC,API: [FC 상담 예약 진행]
    FC->>API: 16. GET /leads/{lead_id} %% E_FC_GET_LEAD_01
    API-->>FC: 17. 리드 상세 정보
    FC->>L: 18. 아웃바운드 연락 (전화/문자)
    FC->>API: 19. POST /consultations { lead_id, scheduled_at, type } %% E_FC_BOOK_CONSULT_01
    API->>DB: 20. INSERT consultations { lead_id, fc_id, scheduled_at, status=SCHEDULED }
    API->>DB: 21. UPDATE leads SET status=CONSULTING
    DB-->>API: 22. consultation_id
    API->>N: 23. 상담 예약 확인 알림 %% E_API_CONSULT_CONFIRM_01
    N-->>L: 24. SMS — "상담 예약 확인: OO월 OO일 OO시"
    API-->>FC: 25. 200 OK { consultation_id }

    alt 연락 불가 — 팔로업 예약
        FC->>API: 26. PATCH /leads/{id} { status=PENDING, follow_up_at } %% E_FC_FOLLOWUP_01
        API->>DB: 27. UPDATE leads SET status=PENDING, follow_up_at=?
        API->>AUT: 28. 팔로업 알림 예약 등록
        API-->>FC: 29. 200 OK
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 2 | POST /leads | source_utm으로 광고 채널 추적. 마케팅 ROI 분석에 활용 |
| 5 | 이벤트 발행 | 이벤트 기반 자동화. 규칙은 SCR-072에서 설정 |
| 11 | FC 배정 로직 | 순번 배정 또는 특정 채널-담당자 매핑 정책 적용 |
| 14 | FC 내부 알림 | 리드 이름, 채널, 연락처 요약 포함 |
| 19 | POST /consultations | type: 방문상담/전화상담/화상상담 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 동일 번호 중복 리드 | 기존 리드로 병합 또는 신규 생성 선택 |
| FC 전원 부재 | 매니저에게 배정 알림 fallback |
| 자동화 규칙 없음 | 수동 처리만 진행, 자동 메시지 미발송 |
| 상담 예약 시간 중복 | FC 캘린더 충돌 감지 → 다른 시간 제안 |
