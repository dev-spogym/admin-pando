# X25 — 쿠폰 발행 → 배포 → 사용 → 만료 회수

## 1. 시나리오 개요

매니저가 쿠폰 발행 → 대상 회원에게 배포 → 회원이 결제 시 쿠폰 적용 → 만료 시 자동 회수까지의 전체 쿠폰 생명주기 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 매니저의 쿠폰 발행 |
| 종료 조건 | 쿠폰 사용 완료 또는 만료 회수 |
| 참여 도메인 | 마케팅(D8), 매출관리(D3), 회원관리(D2) |

## 2. 전제조건

- 매니저 계정 로그인 상태
- 쿠폰 적용 가능 상품이 등록되어 있음
- 배포 대상 회원이 확정되어 있음

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 매니저 | 쿠폰 발행 및 배포 담당 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 회원 | 쿠폰 수령 및 사용 |
| PG | 결제 처리 (쿠폰 적용 결제) |
| 스케줄러 | 만료 쿠폰 자동 회수 크론잡 |
| 알림서비스 | 쿠폰 발급/만료 임박 알림 |

## 4. 시퀀스 다이어그램

```mermaid

    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    actor MB as 회원
    participant PG as PG사
    participant SCH as 스케줄러
    participant N as 알림서비스

    Note over M,API: [쿠폰 발행]
    M->>API: 1. POST / {name, discount_type, discount_value, valid_days, applicable_products} %% API->>DB: 2. coupon_templates {discount_type, value, valid_days}
    DB-->>API: 3. template_id
    API-->>M: 4. 201 Created {template_id}
    Note over M: SCR-073 쿠폰 관리 화면

    Note over M,API: [회원 배포]
    M->>API: 5. POST {template_id, member_ids, expire_at} %% API->>DB: 6. member_coupons {member_id, template_id, status=VALID, expire_at} (대상별)
    DB-->>API: 7. 발급 건수
    API->>N: 8. 쿠폰 발급 알림 트리거 %% N-->>MB: 9. SMS/카카오톡 — "할인 쿠폰이 발급되었습니다"
    API-->>M: 10. 200 OK {issued_count}

    Note over MB,PG: [회원 쿠폰 사용 — 결제 시]
    MB->>API: 11. GET /?status=VALID %% API-->>MB: 12. 유효 쿠폰 목록
    MB->>API: 13. POST {product_id, coupon_id} %% API-->>MB: 14. 할인 적용 최종 금액 미리보기
    MB->>API: 15. POST /sales {member_id, product_id, coupon_id, payment_method} %% API->>DB: 16. member_coupons WHERE id=? AND status=VALID FOR 
    DB-->>API: 17. 쿠폰 유효성 확인
    API->>PG: 18. 할인 적용 금액으로 결제 요청
    PG-->>API: 19. 승인 응답
    API->>DB: 20. sales {discount_amount, coupon_id}
    API->>DB: 21. member_coupons SET status=USED, used_at=NOW(동작) %% DB-->>API: 22. OK
    API->>N: 23. 쿠폰 사용 완료 알림
    N-->>MB: 24. SMS — "쿠폰 사용 완료. 할인금액: {discount}원"
    API-->>MB: 25. 200 OK {sale_id}

    Note over SCH,DB: [쿠폰 만료 자동 회수 — 매일]
    SCH->>API: 26. 만료 쿠폰 회수 트리거 %% API->>DB: 27. member_coupons SET status=EXPIRED WHERE expire_at < NOW(동작) AND status=VALID
    DB-->>API: 28. 만료 처리 건수
    API->>N: 29. 만료 임박 알림 (D-3) — 별도 트리거
    N-->>MB: 30. SMS — "쿠폰이 3일 후 만료됩니다"
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 5 | POST /distribute | member_ids 배열로 일괄 배포. 개별 발급 코드 생성 |
| 16 | FOR | 쿠폰 중복 사용 방지. 동시 요청 시 락으로 보호 |
| 21 | status=USED | 결제 완료와 동시에 쿠폰 사용 처리. 실패 시 롤백 |
| 27 | 만료 자동 처리 | 매일 00:00 실행. expire_at 기준 일괄 EXPIRED 처리 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 쿠폰 중복 사용 시도 | FOR 락 + status=VALID 체크로 차단 |
| 결제 실패 시 쿠폰 롤백 | 트랜잭션으로 묶어 결제 실패 시 쿠폰 VALID 유지 |
| 적용 불가 상품 | 서버에서 applicable_products 검증 → 422 반환 |
| 만료 임박 알림 중복 | D-3 알림 이미 발송된 쿠폰 skip |
