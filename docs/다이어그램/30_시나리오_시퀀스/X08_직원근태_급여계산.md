# X08 — 직원 근태 체크인 → 급여 자동 계산 → 명세서 발송

## 1. 시나리오 개요

직원이 출근 체크인 → 근태 시스템에 기록 → 월말 급여 계산 크론잡이 전월 근태 집계 → 급여 자동 계산 → 직원에게 명세서 발송하는 시나리오.

| 항목 | 내용 | |------|------| | 트리거 1 | 직원 출퇴근 체크인/아웃 | | 트리거 2 | 매월 급여 계산일(기본: 말일) 크론잡 | | 종료 조건 | 급여 명세서 발송 완료 | | 참여 도메인 | 직원관리(D7) |

## 2. 전제조건

- 직원 계정이 시스템에 등록되어 있음
- 급여 계산 정책(시급/고정급/인센티브)이 설정되어 있음
- 알림 수신 동의한 직원

## 3. 참여 액터

| 액터 | 설명 | |------|------| | 직원 | 체크인/아웃 당사자 | | 매니저 | 근태 수정 및 급여 최종 승인 | | CRM API | FitGenie CRM 백엔드 | | DB | 데이터베이스 | | 스케줄러 | 월말 급여 계산 크론잡 | | 알림서비스 | 명세서 발송 |

## 4. 시퀀스 다이어그램

```mermaid

    actor E as 직원
    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    participant SCH as 스케줄러
    participant N as 알림서비스

    Note over E,API: [일일 근태 기록]
    E->>API: 1. POST {employee_id} %% API->>DB: 2. attendances {employee_id, checkin_at=NOW(동작), status=WORKING}
    DB-->>API: 3. attendance_id
    API-->>E: 4. 200 OK {checkin_at}
    Note over E: SCR-063 근태 관리 화면

    E->>API: 5. POST {attendance_id} %% API->>DB: 6. attendances SET checkout_at=NOW(동작), work_hours=DIFF, status=DONE
    DB-->>API: 7. OK
    API-->>E: 8. 200 OK {work_hours}

    Note over M,API: [근태 수정 — 필요 시]
    M->>API: 9. PATCH {checkin_at, checkout_at, reason} %% API->>DB: 10. attendances (수정 이력 함께 기록)
    DB-->>API: 11. OK
    API-->>M: 12. 200 OK

    Note over SCH,API: [월말 급여 계산 — 매월 말일]
    SCH->>API: 13. 급여 계산 트리거 {year_month} %% API->>DB: 14. attendances WHERE month=? GROUP BY employee_id
    DB-->>API: 15. 직원별 근무 시간 집계
    loop 각 직원별
        API->>DB: 16. salary_policy WHERE employee_id=? %% DB-->>API: 17. 급여 정책 (시급/고정/인센티브)
        API->>API: 18. 급여 계산 (기본급 + 초과근무 + 인센티브 - 공제)
        API->>DB: 19. payrolls {employee_id, month, base, overtime, incentive, deduction, total}
        DB-->>API: 20. payroll_id
    end
    API-->>SCH: 21. 계산 완료 {count}

    Note over M,API: [매니저 검토 및 승인]
    M->>API: 22. GET /payrolls?month={year_month} %% API-->>M: 23. 직원별 급여 목록
    M->>API: 24. POST {month} %% API->>DB: 25. payrolls SET status=APPROVED WHERE month=?
    DB-->>API: 26. OK

    API->>N: 27. 급여 명세서 발송 트리거 %% N-->>E: 28. SMS/이메일 — 급여 명세서 링크 발송
    API-->>M: 29. 200 OK {sent_count}
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 | |------|--------|------| | 9 | PATCH /attendances | 수동 수정 시 원본 + 수정 이력 모두 보존 | | 14 | attendances GROUP BY | 전월 전체 근무일/시간 집계 | | 18 | 급여 계산 | 시급제: 시간 × 시급 + 초과근무 할증. 고정제: 기본급 + 인센티브 | | 24 | POST | 전직원 일괄 승인. 개별 승인도 가능 | | 27 | 명세서 발송 | SCR-065 급여명세서 URL 포함. 직원이 웹에서 확인 가능 |

## 6. 예외/분기

| 상황 | 처리 방법 | |------|-----------| | 체크아웃 누락 | 다음 날 체크인 시 전일 체크아웃 경고. 매니저 수동 수정 | | 급여 정책 미설정 | 계산 skip, 매니저 알림 발송 | | 초과근무 계산 오류 | 매니저 검토 단계에서 수동 조정 가능 | | 퇴사 직원 포함 | 재직 기간만큼 일할 계산 |
