# X28 — 세금계산서 발행 → 국세청 전송 → 상태 업데이트 🆕

## 1. 시나리오 개요

사업자 회원의 결제 건에 대해 세금계산서 발행 요청 → 전자세금계산서 생성 → 국세청 API 전송 → 발행 상태 업데이트 → 회원에게 발송하는 시나리오.

| 항목 | 내용 | |------|------| | 트리거 | 매니저 또는 회원의 세금계산서 발행 요청 | | 종료 조건 | 국세청 전송 완료 + 상태 ISSUED | | 참여 도메인 | 매출관리(D3) | | 구현 상태 | 🆕 신규 기능 |

## 2. 전제조건

- 전자세금계산서 발행 API(e-세로 또는 연동사)가 연동 설정되어 있음
- 회원의 사업자등록번호가 등록되어 있음
- 발행 대상 결제 건이 부가세 과세 거래임

## 3. 참여 액터

| 액터 | 설명 | |------|------| | 매니저 | 세금계산서 발행 요청 | | CRM API | FitGenie CRM 백엔드 | | DB | 데이터베이스 | | 세금계산서API | 전자세금계산서 발행 외부 서비스 | | 국세청 | 전자세금계산서 수신 기관 | | 회원 | 세금계산서 수령자 | | 알림서비스 | 발행 완료 알림 |

## 4. 시퀀스 다이어그램

```mermaid

    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    participant TAX as 세금계산서API
    participant NTS as 국세청
    actor MB as 회원
    participant N as 알림서비스

    M->>API: 1. GET %% API->>DB: 2. sales, WHERE sale_id=? (사업자번호 포함)
    DB-->>API: 3. 결제 및 회원 정보
    API-->>M: 4. 세금계산서 미리보기 {supply_amount, tax_amount, total}
    Note over M: SCR-S010 세금계산서 발행 화면

    M->>API: 5. POST /tax- {sale_id, business_no, email} %% API->>DB: 6. tax_invoices {sale_id, status=DRAFT, created_at}
    DB-->>API: 7. invoice_id
    API->>TAX: 8. POST /issue {invoice_data} %% Note over TAX: 전자세금계산서 XML 생성

    TAX->>NTS: 9. 국세청 전송 %% NTS-->>TAX: 10. 접수 응답 {nts_id, status}

    alt 국세청 접수 성공
        TAX-->>API: 11. 발행 성공 {nts_id, issued_at} %% API->>DB: 12. tax_invoices SET status=ISSUED, nts_id=?, issued_at=?
        DB-->>API: 13. OK
        API->>N: 14. 세금계산서 발행 완료 알림 %% N-->>MB: 15. 이메일 — 세금계산서 PDF 첨부
        API-->>M: 16. 200 OK {invoice_id, nts_id}
        Note over M: 발행 완료 상태 확인

    else 국세청 전송 실패 (사업자번호 오류 등)
        TAX-->>API: 17. 발행 실패 {error_code, message} %% API->>DB: 18. tax_invoices SET status=FAILED, error_message=?
        DB-->>API: 19. OK
        API-->>M: 20. 발행 실패 안내 {error_code, message}
        Note over M: 오류 수정 후 재발행 가능

    else 국세청 처리 지연 (비동기)
        TAX-->>API: 21. 처리 중 응답 {status=PENDING} %% API->>DB: 22. tax_invoices SET status=PENDING
        Note over API: Webhook 또는 폴링으로 상태 추적
    end

    Note over M,API: [발행 취소 — 익일 24시 이전]
    M->>API: 23. DELETE %% API->>TAX: 24. POST /cancel {nts_id}
    TAX->>NTS: 25. 취소 전송
    NTS-->>TAX: 26. 취소 확인
    TAX-->>API: 27. 취소 완료
    API->>DB: 28. tax_invoices SET status=CANCELLED
    DB-->>API: 29. OK
    API-->>M: 30. 200 OK
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 | |------|--------|------| | 8 | POST /issue | 공급가액, 세액, 공급자/수급자 정보 포함 전자세금계산서 발행 | | 9 | 국세청 전송 | 외부 세금계산서 API가 국세청 ASP를 통해 전송 | | 12 | status=ISSUED | nts_id: 국세청 승인번호. 법적 효력 기준 | | 23 | 발행 취소 | 발행일 익일 24시 이전만 취소 가능 (세법 규정) |

## 6. 예외/분기

| 상황 | 처리 방법 | |------|-----------| | 사업자번호 오류 | 국세청 검증 실패, 번호 수정 후 재발행 | | 국세청 서버 장애 | PENDING 상태로 저장, 복구 후 자동 재시도 | | 이미 발행된 건 | 중복 발행 방지. 동일 sale_id 재발행 차단 | | 발행 취소 기한 초과 | 수정세금계산서 발행으로 대응 |
