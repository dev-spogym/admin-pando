## 1. 목적

DLG-M024의 필드별 유효성 검증 흐름을 명세한다.

## 2. 트리거/전제조건

- DLG-M024 열린 상태

## 3. 다이어그램

```mermaid
flowchart TD
  DLG[DLG-M024 열림]

  DLG --> FLD_DATE[평가일 date 필수 기본=오늘]
  DLG --> FLD_CAT[카테고리 select 필수 기본=체력]
  DLG --> FLD_SCORE[점수 number 필수 1~10 정수]
  DLG --> FLD_OPT[평가내용/평가자 선택]

  FLD_DATE -->|비어있음|ERR_DATE[평가일을 입력하세요]
  FLD_SCORE -->|1미만 OR 10초과|ERR_SCORE[점수는 1~10 사이로 입력하세요]
  FLD_SCORE -->|숫자 아님|ERR_SCORE

  FLD_DATE --> CHECK_ALL{AND 1<=점수}
  FLD_SCORE --> CHECK_ALL

  CHECK_ALL --> BTN_ACTIVE[저장 버튼 활성]
  CHECK_ALL --> BTN_DISABLED[저장 버튼 disabled]

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

  class DLG,FLD_DATE,FLD_CAT,FLD_SCORE,FLD_OPT,CHECK_ALL system
  class BTN_ACTIVE success
  class ERR_DATE,ERR_SCORE,BTN_DISABLED error
```
