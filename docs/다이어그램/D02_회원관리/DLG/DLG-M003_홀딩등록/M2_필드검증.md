## 1. 목적

DLG-M003의 필드별 유효성 검증 흐름을 명세한다.

## 2. 트리거/전제조건

- DLG-M003 열린 상태

## 3. 다이어그램

```mermaid
flowchart TD
  DLG[DLG-M003 열림]

  DLG --> VER{버전 분기}
  VER -->|상세 버전|FLD_DAYS[홀딩 기간 number]
  VER -->|탭 버전|FLD_PRODUCT[상품명 text]

  FLD_DAYS --> CHECK_DAYS{1 ≤ 기간 ≤ 90}
  CHECK_DAYS --> BTN_OK[버튼 활성]
  CHECK_DAYS --> ERR_DAYS[1~90일 사이로 입력하세요]

  FLD_PRODUCT --> FLD_START[홀딩 시작일]
  FLD_START --> FLD_END[홀딩 종료일]
  FLD_END --> CHECK_TAB{상품명+시작일+종료일 입력}
  CHECK_TAB -->|모두 입력|BTN_OK
  CHECK_TAB -->|미입력|BTN_DISABLED[버튼 비활성]

  BTN_OK --> SUBMIT[API 호출]
  BTN_DISABLED --> NO_ACTION[동작 없음]

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

  class VER,FLD_DAYS,FLD_PRODUCT,FLD_START,FLD_END,CHECK_DAYS,CHECK_TAB system
  class BTN_OK,SUBMIT success
  class ERR_DAYS,BTN_DISABLED,NO_ACTION error
```
