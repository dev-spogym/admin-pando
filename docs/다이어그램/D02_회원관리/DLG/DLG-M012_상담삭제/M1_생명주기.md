## 1. 목적

DLG-M012 상담 이력 삭제 확인 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 상담이력 탭 > 행 > "삭제" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([상담이력 행 삭제 버튼 클릭])

  TRIGGER -->|E_OPEN_01| DLG_OPEN[DLG-M012 상담 이력 삭제 ConfirmDialog]

  DLG_OPEN --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소| CLOSE[모달 닫기]
  USER_ACT -->|E_CONFIRM_01 삭제| API_CALL[DELETE /api/consultations/:id]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success 삭제되었습니다))
  TOAST_OK -->|E_REFETCH_01| REFETCH[상담이력 refetch]

  API_CALL -->|E_FAIL_01 오류| TOAST_ERR((toast.error 삭제 실패 error))

  classDef modal   fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

  class DLG_OPEN modal

```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_OPEN_01 | 삭제 버튼 | 모달 열기 | - |
| E_CANCEL_01 | 취소 | 모달 닫기 | - |
| E_CONFIRM_01 | 삭제 | API | 확인 클릭 |

| E_FAIL_01 | API | toast.error | 오류 |
