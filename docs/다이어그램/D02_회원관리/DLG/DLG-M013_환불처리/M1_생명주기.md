## 1. 목적

DLG-M013 환불 처리 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 결제이력/결제내역 탭 > "환불" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([결제이력 환불 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M013 환불 처리 ConfirmDialog]
  DLG_OPEN --> DESC[amount원 결제 건 환불 처리 확인 메시지]

  DESC --> USER_ACT{사용자 액션}

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|환불 처리|API_CALL[환불 레코드 음수 + PUT 원본 status=REFUNDED]

  API_CALL -->|200|TOAST_OK((toast 환불 처리가 완료되었습니다))
  TOAST_OK --> REFETCH[결제이력 refetch]

  API_CALL -->|오류|TOAST_ERR((toast 환불 처리 실패 error))
  TOAST_ERR --> DLG_OPEN

  classDef modal fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,DESC modal
  class TOAST_OK,REFETCH success
  class TOAST_ERR error
  class API_CALL system
```
