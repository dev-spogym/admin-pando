## 1. 목적

DLG-M011 상담 이력 추가/수정 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 상담이력 탭 > "상담 추가" 버튼 클릭 (신규)
- 또는 기존 행 "수정" 버튼 클릭 (수정)

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER_ADD([상담 추가 버튼 클릭])
  TRIGGER_EDIT([기존 행 수정 버튼 클릭])

  TRIGGER_ADD  -->|E_OPEN_ADD_01| DLG_ADD[DLG-M011 상담 이력 추가 max-w-480px]
  TRIGGER_EDIT -->|E_OPEN_EDIT_01| DLG_EDIT[DLG-M011 상담 이력 수정 기존값 로드]

  DLG_ADD  --> USER_ACT{사용자 액션}
  DLG_EDIT --> USER_ACT

  USER_ACT -->|E_CANCEL_01 취소/X/ESC| CLOSE[모달 닫기]
  USER_ACT -->|E_SUBMIT_01 저장 consultedAt 있음| API_CALL{INSERT or UPDATE}

  API_CALL -->|E_INSERT_01 신규| API_INSERT[POST /api/consultations INSERT]
  API_CALL -->|E_UPDATE_01 수정| API_UPDATE[PUT /api/consultations/:id UPDATE]

  API_INSERT -->|E_ADD_OK_01 200| TOAST_ADD_OK((toast.success 상담 이력이 저장되었습니다))
  API_UPDATE -->|E_EDIT_OK_01 200| TOAST_EDIT_OK((toast.success 상담 이력이 수정되었습니다))

  TOAST_ADD_OK  -->|E_CLOSE_OK_01| POST_OK[모달 닫기 + refetch]
  TOAST_EDIT_OK -->|E_CLOSE_OK_02| POST_OK

  API_INSERT -->|E_ADD_FAIL_01 오류| TOAST_ERR((toast.error 저장 실패 error))
  API_UPDATE -->|E_EDIT_FAIL_01 오류| TOAST_ERR
  TOAST_ERR  -->|E_STAY_01| DLG_ADD

  classDef modal   fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

  class DLG_ADD,DLG_EDIT modal
  class TOAST_ADD_OK,TOAST_EDIT_OK,POST_OK success
  class TOAST_ERR error
  class API_CALL,API_INSERT,API_UPDATE system
```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_OPEN_ADD_01 | 상담 추가 | 모달(신규) | - |
| E_OPEN_EDIT_01 | 수정 버튼 | 모달(수정) | 기존값 로드 |
| E_SUBMIT_01 | 저장 | API 분기 | consultedAt 있음 |
| E_ADD_OK_01 | INSERT API | toast.success | 200 |
| E_EDIT_OK_01 | UPDATE API | toast.success | 200 |
