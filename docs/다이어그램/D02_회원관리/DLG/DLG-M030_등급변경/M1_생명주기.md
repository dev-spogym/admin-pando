## 1. 목적

DLG-M030 회원 등급 변경 다이얼로그의 열기/닫기/완료 생명주기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- 회원 상세 > 등급 배지 클릭 (관리자) 또는 등급 관리 > "수동 변경" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER_A([회원 상세 등급 배지 클릭])
  TRIGGER_B([등급 관리 수동 변경 버튼 클릭])

  TRIGGER_A --> DLG_OPEN[DLG-M030 회원 등급 변경 🆕]
  TRIGGER_B --> DLG_OPEN

  DLG_OPEN --> FLD_CURRENT[현재 등급 readonly]
  DLG_OPEN --> FLD_NEXT[변경 등급 select 필수]
  DLG_OPEN --> FLD_REASON[변경 사유 text 필수]

  FLD_CURRENT --> USER_ACT{사용자 액션}
  FLD_NEXT --> USER_ACT
  FLD_REASON --> USER_ACT

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|변경 클릭|VALID{필드 유효성}

  VALID -->|현재 등급 동일|ERR_SAME[현재 등급과 동일합니다]
  VALID -->|사유 빈값|ERR_REASON[변경 사유를 입력하세요]
  ERR_SAME --> DLG_OPEN
  ERR_REASON --> DLG_OPEN

  VALID -->|통과|API_CALL[🆕]

  API_CALL -->|200|TOAST_OK((toast 등급이 변경되었습니다))
  TOAST_OK --> CLOSE_OK[모달 닫기 + 등급 정보 갱신]

  API_CALL -->|오류|TOAST_ERR((toast 등급 변경에 실패했습니다))
  TOAST_ERR --> DLG_OPEN

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,FLD_CURRENT,FLD_NEXT,FLD_REASON newFeature
  class TOAST_OK,CLOSE_OK success
  class TOAST_ERR,ERR_SAME,ERR_REASON error
  class API_CALL system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 등급 배지 클릭 | 모달 열기 | 회원 상세 |
| 수동 변경 버튼 | 모달 열기 | 등급 관리 |
| 취소 | 모달 닫기 | - |
| 변경 버튼 | 유효성 검사 | 클릭 |
| 유효성 실패 | 에러 표시 | 현재 등급 동일 |
| 유효성 실패 | 에러 표시 | 사유 빈값 |
| 유효성 통과 | API 호출 | - |
| API | toast | 200 |
| toast | 모달 닫기 + 갱신 | - |
| API | toast | 오류 |
