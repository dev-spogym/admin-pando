## 3. 다이어그램

```mermaid
flowchart TD
  API[등급 수동 변경 🆕]

  API -->|200|SUCCESS[등급 변경 성공]
  API -->|오류|FAIL[등급 변경 실패]

  SUCCESS --> TOAST_OK((toast 등급이 변경되었습니다))
  TOAST_OK --> CLOSE[모달 닫기 + 등급 정보 갱신]

  FAIL --> TOAST_ERR((toast 등급 변경에 실패했습니다))
  TOAST_ERR --> MODAL_STAY[모달 유지]

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

  class API newFeature
  class SUCCESS,TOAST_OK,CLOSE success
  class FAIL,TOAST_ERR,MODAL_STAY error
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| API | 성공 | 200 |
| API | 실패 | 오류 |
| 성공 | toast | - |
| toast | 모달 닫기 + 등급 갱신 | - |
| 실패 | toast | - |
| toast | 모달 유지 | - |
