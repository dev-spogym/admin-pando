## 2. 트리거/전제조건

## 3. 다이어그램

```mermaid
flowchart TD
  API_INSERT[동작]
  API_UPDATE[덮어쓰기]

  API_INSERT -->|200|SUCCESS[저장 성공]
  API_UPDATE -->|200|SUCCESS

  API_INSERT -->|오류|FAIL[저장 실패]
  API_UPDATE -->|오류|FAIL

  SUCCESS --> TOAST_OK((toast 체성분이 저장되었습니다))
  TOAST_OK --> POST_OK[모달 닫기 + 목록/차트 갱신]

  FAIL --> TOAST_ERR((toast 저장에 실패했습니다))
  TOAST_ERR --> MODAL_STAY[모달 유지]

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class SUCCESS,TOAST_OK,POST_OK success
  class FAIL,TOAST_ERR,MODAL_STAY error
  class API_INSERT,API_UPDATE system
```
