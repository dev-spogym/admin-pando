## 3. 다이어그램

```mermaid
flowchart TD
  API[동작]

  API -->|200|SUCCESS[성공]
  API -->|4xx|FAIL_CLIENT[클라이언트 오류]
  API -->|5xx|FAIL_SERVER[서버 오류]

  SUCCESS --> TOAST_OK((toast N명의 상태를 label로 변경했습니다))
  TOAST_OK --> CLOSE[모달 닫기]
  CLOSE --> REFETCH[회원 목록 refetch]
  REFETCH --> RESET[초기화]

  FAIL_CLIENT --> TOAST_ERR((toast 상태 변경에 실패했습니다))
  FAIL_SERVER --> TOAST_ERR
  TOAST_ERR --> MODAL_STAY[모달 유지]

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class SUCCESS,TOAST_OK,CLOSE,REFETCH,RESET success
  class FAIL_CLIENT,FAIL_SERVER,TOAST_ERR,MODAL_STAY error
  class API system
```
