## 1. 목적

DLG-M028 회원 병합 확인 다이얼로그의 열기/닫기/완료 생명주기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- 회원 병합(SCR-M007) > "병합 실행" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([SCR-M007 병합 실행 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M028 회원 병합 확인 🆕]
  DLG_OPEN --> DESC[회원 데이터가 회원에 병합됩니다 되돌릴 수 없습니다]

  DESC --> USER_ACT{사용자 액션}

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|병합 실행 danger|API_CALL[데이터 병합 + 대상 soft delete]

  API_CALL -->|200|TOAST_OK((toast 회원이 병합되었습니다))
  TOAST_OK --> REDIRECT[기준 회원 상세 이동]

  API_CALL -->|오류|TOAST_ERR((toast 병합에 실패했습니다))
  TOAST_ERR --> DLG_OPEN

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,DESC newFeature
  class TOAST_OK,REDIRECT success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 병합 실행 | 모달 열기 | - |
| 취소 | 모달 닫기 | - |
| 병합 실행 | API | danger 버튼 클릭 |
| API | toast | 200 |
| toast | 기준 회원 상세 이동 | - |
| API | toast | 오류 |
