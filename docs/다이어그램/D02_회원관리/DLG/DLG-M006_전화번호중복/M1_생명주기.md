## 1. 목적

DLG-M006 전화번호 중복 확인(인라인)의 트리거/결과 생명주기를 명세한다.

## 2. 트리거/전제조건

- 회원 등록/수정 > 연락처 필드 입력 후 "중복확인" 버튼 클릭
- phone AND 

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([연락처 입력 중복확인 버튼 클릭])

  TRIGGER -->|phone자 미확인|API_CALL[중복 조회]
  TRIGGER -->|phone자 OR 확인완료|BTN_DISABLED[버튼 비활성 동작없음]

  API_CALL -->|중복없음|INLINE_OK[인라인 초록 사용 가능한 번호입니다]
  API_CALL -->|중복있음|INLINE_ERR[인라인 빨강 이미 등록된 번호입니다]

  INLINE_OK --> STATE_OK[동작]
  INLINE_ERR --> STATE_ERR[동작]

  STATE_OK --> STEP_PROCEED[다음 단계 진행 가능]
  STATE_ERR --> STEP_BLOCKED[폼 제출 차단]

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class INLINE_OK,STATE_OK,STEP_PROCEED success
  class INLINE_ERR,STATE_ERR,STEP_BLOCKED,BTN_DISABLED error
  class API_CALL system
```
