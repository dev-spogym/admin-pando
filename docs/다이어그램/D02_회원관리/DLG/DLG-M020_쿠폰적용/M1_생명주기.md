## 1. 목적

DLG-M020 쿠폰 적용 다이얼로그의 열기/닫기/완료 생명주기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- 상세내역 탭 > 쿠폰 서브탭 > "쿠폰 적용" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([쿠폰 적용 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M020 쿠폰 적용 🆕]
  DLG_OPEN --> INIT[쿠폰 코드 빈값 초기화]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|적용|API_VALIDATE[코드 검증]

  API_VALIDATE -->|유효|API_APPLY[쿠폰 적용]
  API_VALIDATE -->|유효하지 않음|ERR_CODE[유효하지 않은 쿠폰 코드입니다]

  API_APPLY -->|200|TOAST_OK((toast 쿠폰이 적용되었습니다))
  TOAST_OK --> POST_OK[모달 닫기 + refetch]

  API_APPLY -->|오류|TOAST_ERR((toast 쿠폰 적용에 실패했습니다))
  TOAST_ERR --> DLG_OPEN

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT newFeature
  class TOAST_OK,POST_OK success
  class ERR_CODE,TOAST_ERR error
  class API_VALIDATE,API_APPLY system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 적용 | 코드 검증 API | 클릭 |
| 검증 | 쿠폰 적용 API | 유효 |
| 검증 | 에러 메시지 | 유효하지 않음 |
| 적용 API | toast | 200 |
