## 1. 목적

출석이력 탭(SCR-M004-03)의 히트맵 캘린더와 출석 리스트 표시 플로우를 정의한다.

## 2. 전제조건

- tab= 활성, attendances 데이터 로드 완료

## 3. 다이어그램

```mermaid
flowchart TD
  TAB[Tab03 출석이력 활성] --> STATS[StatCard 2개\n최근6개월출석 / 전체출석기록]

  STATS --> HEATMAP[출석 히트맵 캘린더\n최근 6개월\ 스타일]

  HEATMAP --> CELL{셀 상태}
  CELL -->|": count=0/미래"|CELL_EMPTY[투명 셀]
  CELL -->|": 과거 미방문"|CELL_GRAY[bg-surface-tertiary]
  CELL -->|": count=1"|CELL_LIGHT[bg-primary/40]
  CELL -->|": count"|CELL_FULL[bg-primary]
  CELL -->|": hover"|TOOLTIP[툴팁: 날짜 + n회 출석]

  HEATMAP --> LIST[최근 출석 내역 최근20회\n초록 dot + 날짜 + 체크인~체크아웃]

  LIST --> LIST_STATE{attendances}
  LIST_STATE -->|": 없음"|LIST_EMPTY[출석 기록이 없습니다.]
  LIST_STATE -->|": 있음"|LIST_ITEMS[출석 항목 렌더링\nhover bg-surface-secondary/40]

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  class TAB,STATS,HEATMAP,LIST,LIST_ITEMS screen
  class CELL,LIST_STATE system
  class CELL_FULL,CELL_LIGHT success
```

## 4. 엣지 설명

| 조건 | 결과 |
|------|------|
| count=0 또는 미래 날짜 | 투명 셀 |
| 과거 날짜, 방문 없음 | 회색 셀 |
| count=1 | 연한 primary 셀 |
| count | 진한 primary 셀 |
| 셀 hover | 날짜+횟수 툴팁 |
| 출석 없음 | 빈 상태 메시지 |
| 출석 있음 | 리스트 렌더링 |
