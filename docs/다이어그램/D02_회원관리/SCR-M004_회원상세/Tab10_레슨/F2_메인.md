## 1. 목적

레슨 탭(SCR-M004-10)의 레슨 기록 조회 + 추가 + 수정/삭제 플로우를 정의한다.

## 2. 전제조건

- tab=lesson 활성
- 기반 데이터 로드 (🆕 Supabase 마이그레이션 예정)

## 3. 다이어그램

```mermaid
flowchart TD
  TAB[Tab10 레슨 활성] --> STAT[StatCard 3종\n총 레슨 횟수 / 이번 달 / 최다 수업]
  TAB --> LIST{lessons}

  LIST -->|": 0"|EMPTY[아이콘+레슨 기록이 없습니다.]
  LIST -->|": 1+"|CARDS[레슨 기록 카드 목록\n시간순 내림차순]

  CARDS --> CARD[카드: 수업명 / 날짜 / 트레이너\n서명상태 / 메모 + 수정/삭제 버튼]

  TAB --> ADD_BTN[레슨 추가 버튼]
  ADD_BTN -->|": 클릭"|MODAL[LessonModal 열기\n날짜/수업명/트레이너/운동내용메모\ height=160]

  MODAL --> SAVE_BTN{저장 버튼\ AND trainer}
  SAVE_BTN -->|": 조건 충족"|[\n🆕 Supabase]
   -->|": 성공"|TOAST_OK((토스트: 레슨이 저장되었습니다.\n목록 갱신))
   -->|": 실패"|TOAST_ERR((토스트: 저장 실패))
  MODAL -->|": 취소/ESC"|TAB

  CARD --> EDIT_BTN[수정 버튼]
  EDIT_BTN -->|": 클릭"|EDIT_MODAL[LessonModal 수정 모드\n기존 데이터 프리필]
  EDIT_MODAL --> [동작]
   -->|": 성공"|UPDATE_OK((토스트: 수정되었습니다.\n카드 갱신))

  CARD --> DEL_BTN[삭제 버튼]
  DEL_BTN -->|": 클릭"|DEL_CONFIRM[삭제 확인]
  DEL_CONFIRM -->|": 확인"|DEL_OK((토스트: 삭제되었습니다.\n목록 갱신))
  DEL_CONFIRM -->|": 취소"|CARDS
```
