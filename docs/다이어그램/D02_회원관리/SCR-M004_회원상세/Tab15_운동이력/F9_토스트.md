## 1. 목적

운동이력 탭에서 발생하는 토스트를 정의한다.

## 2. 전제조건

- Tab15 운동이력 활성

## 3. 다이어그램

```mermaid
flowchart TD
  ACTIONS[Tab15 운동이력 액션] --> T_OK[성공]
  ACTIONS --> T_ERR[에러]

  T_OK -->|": 기록 성공"|TS1((운동이 기록되었습니다.))
  T_OK -->|": 수정 성공"|TS2((수정되었습니다.))
  T_OK -->|": 삭제 성공"|TS3((삭제되었습니다.))
  T_ERR -->|": 기록 실패"|TE1((기록 실패: 에러))
  T_ERR -->|": 수정 실패"|TE2((수정 실패: 에러))
  T_ERR -->|": 삭제 실패"|TE3((삭제 실패: 에러))

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2
  class TS1,TS2,TS3 success
  class TE1,TE2,TE3 error
  class ACTIONS,T_OK,T_ERR system
```

## 4. 엣지 설명

| 상황 | 타입 | 메시지 | |---------|------|------|--------| | | 기록 성공 | success | "운동이 기록되었습니다." | | | 수정 성공 | success | "수정되었습니다." | | | 삭제 성공 | success | "삭제되었습니다." | | | 기록 실패 | error | "기록 실패: ${error}" | | | 수정 실패 | error | "수정 실패: ${error}" | | | 삭제 실패 | error | "삭제 실패: ${error}" |
