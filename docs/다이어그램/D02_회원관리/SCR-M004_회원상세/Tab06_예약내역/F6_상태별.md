## 1. 목적

예약내역 탭의 데이터/예약 상태별 화면 분기를 정의한다.

## 2. 전제조건

- Tab06 예약내역 활성

## 3. 다이어그램

```mermaid
flowchart TD
  TAB[Tab06 예약내역] --> LOAD{로딩}
  LOAD -->|": 로딩 중"|SPINNER[로딩 스피너]
  LOAD -->|": 완료"|DATA{예약 데이터}
  DATA -->|": 없음"|EMPTY[캘린더 dot 없음\n테이블 비어있음]
  DATA -->|": 있음"|NORMAL[캘린더 dot + 테이블 정상]

  NORMAL --> BOOK_ST{예약 상태}
  BOOK_ST -->|": SHOW"|SHOW_ROW[success 배지]
  BOOK_ST -->|": NOSHOW"|NOSHOW_ROW[error 배지]
  BOOK_ST -->|": 미처리"|PEND_ROW[warning 배지 미처리]

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  class TAB,NORMAL screen
  class LOAD,DATA,BOOK_ST system
  class SHOW_ROW success
  class NOSHOW_ROW error
  class PEND_ROW warning
```

## 4. 엣지 설명

| 조건 | 화면 | |---------|------|------| | | 로딩 중 | 스피너 | | | 로드 완료 | 레이아웃 렌더링 | | | 예약 없음 | 빈 상태 | | | 예약 있음 | 정상 | | | SHOW | success 배지 | | | NOSHOW | error 배지 | | | 미처리 | warning 배지 |
