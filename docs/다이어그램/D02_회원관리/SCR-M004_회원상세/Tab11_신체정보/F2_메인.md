## 1. 목적

신체정보 탭(SCR-M004-11)의 신체 데이터 조회 + 추가 + 수정/삭제 플로우를 정의한다.

## 2. 전제조건

- tab=body 활성
- 신체정보 데이터 로드 완료

## 3. 다이어그램

```mermaid
flowchart TD
  TAB[Tab11 신체정보 활성] --> STAT[StatCard 4종\n키 / 체중 / BMI / 체지방률]
  TAB --> CHART[WeightLineChart\n체중 추이 라인차트]
  TAB --> LIST{records}

  LIST -->|": 0"|EMPTY[아이콘+신체정보가 없습니다.]
  LIST -->|": 1+"|TABLE[DataTable\n날짜/키/체중/BMI/체지방률 + 수정/삭제]

  TAB --> ADD_BTN[신체정보 추가 버튼]
  ADD_BTN -->|": 클릭"|FORM[인라인 폼\n날짜/키/체중/체지방률 입력]

  FORM --> SAVE_BTN{저장}
  SAVE_BTN -->|": 성공"|INSERT_OK((토스트: 신체정보가 저장되었습니다.\n테이블+차트 갱신))
  SAVE_BTN -->|": 실패"|INSERT_ERR((토스트: 저장 실패))

  TABLE --> EDIT_BTN[수정 버튼]
  EDIT_BTN -->|": 클릭"|EDIT_FORM[인라인 수정 폼]
  EDIT_FORM -->|": 저장 성공"|UPDATE_OK((토스트: 수정되었습니다.\n테이블+차트 갱신))

  TABLE --> DEL_BTN[삭제 버튼]
  DEL_BTN -->|": 클릭"|DEL_CONFIRM[삭제 확인]
  DEL_CONFIRM -->|": 확인"|DEL_OK((토스트: 삭제되었습니다.\n테이블+차트 갱신))
  DEL_CONFIRM -->|": 취소"|TABLE

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

  class TAB,STAT,CHART,TABLE screen
  class INSERT_OK,UPDATE_OK,DEL_OK success
```
