## 1. 목적

결제이력 탭에서 발생하는 토스트를 정의한다.

## 2. 전제조건

- Tab04 결제이력 활성

## 3. 다이어그램

```mermaid
flowchart TD
  ACTIONS[Tab04 결제이력 액션] --> T_OK[성공 토스트]
  ACTIONS --> T_ERR[에러 토스트]

  T_OK -->|": 환불 성공"|TS1((환불이 처리되었습니다.))
  T_ERR -->|": 환불 실패"|TE1((환불 처리에 실패했습니다.))

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2
  class TS1 success
  class TE1 error
  class ACTIONS,T_OK,T_ERR system
```
