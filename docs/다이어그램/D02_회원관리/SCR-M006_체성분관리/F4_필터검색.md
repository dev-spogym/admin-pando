## 1. 목적

SCR-M006의 차트 메트릭 토글, 테이블 정렬, 데이터 조회 조건을 명세한다.

## 2. 트리거/전제조건

- SCR-M006 데이터 로드 완료

## 3. 다이어그램

```mermaid
flowchart TD
  SCR_M006[SCR-M006 체성분 관리]

  SCR_M006 --> LOAD_DATA[GET 전체 목록]

  LOAD_DATA -->|기본 날짜 내림차순|TABLE_SORTED[테이블 날짜 최신순 표시]

  TABLE_SORTED --> METRIC_TOGGLE{메트릭 토글}
  METRIC_TOGGLE -->|체중 ON|CHART_W[차트 체중 라인 #FF7F6E]
  METRIC_TOGGLE -->|골격근량 ON|CHART_M[차트 골격근량 라인 #48D1CC]
  METRIC_TOGGLE -->|체지방률 ON|CHART_P[차트 체지방률 라인 #F59E0B]

  METRIC_TOGGLE -->|체중 OFF|CHART_W_HIDE[체중 라인 숨김]
  METRIC_TOGGLE -->|골격근량 OFF|CHART_M_HIDE[골격근량 라인 숨김]
  METRIC_TOGGLE -->|체지방률 OFF|CHART_P_HIDE[체지방률 라인 숨김]

  TABLE_SORTED --> DATA_COUNT{데이터 건수}
  DATA_COUNT -->|0건|UI_EMPTY[통계 카드 - 차트 미표시 테이블 빈 상태]
  DATA_COUNT -->|1건|UI_ONE[통계 카드 change 없음 차트 미표시]
  DATA_COUNT -->|2건+|UI_FULL[통계 카드 change + 라인 차트 + 테이블]

  classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef info fill:#E0F7FA,stroke:#00838F

  class SCR_M006 screen
  class LOAD_DATA,METRIC_TOGGLE,DATA_COUNT system
  class UI_EMPTY,UI_ONE info
  class UI_FULL,TABLE_SORTED screen
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 데이터 로드 | 테이블 정렬 | 기본 날짜 내림차순 |
| 메트릭 토글 | 체중 라인 표시 | 체중 ON |
| 메트릭 토글 | 골격근량 라인 표시 | 골격근량 ON |
| 메트릭 토글 | 체지방률 라인 표시 | 체지방률 ON |
| 데이터 건수 | 빈 상태 | 0건 |
| 데이터 건수 | 1건 UI | 1건 |
| 데이터 건수 | 전체 UI | 2건+ |
