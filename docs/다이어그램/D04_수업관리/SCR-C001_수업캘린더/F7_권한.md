## 1. 목적
SCR-C001에서 6개 역할별 접근 가능 기능 범위를 정의한다.

## 2. 전제조건
- 로그인 완료, 역할(role) 보유

## 3. 다이어그램

```mermaid
flowchart LR
  Enter[SCR-C001 진입] --> RoleCheck{역할 확인}

  RoleCheck -->|": primary/owner"|RolePrimary[슈퍼관리자/센터장\n전 지점 캘린더 조회\n수업 등록/수정/삭제\n승인/거절\n일괄변경\n드래그앤드롭]

  RoleCheck -->|": manager"|RoleManager[매니저\n본 지점 캘린더 조회\n수업 등록/수정/삭제\n승인/거절\n일괄변경\n드래그앤드롭]

  RoleCheck -->|": fc 트레이너"|RoleFC[트레이너\n본인 수업만 수정 가능\n다른 강사 수업 조회만\n드래그앤드롭 본인수업만]

  RoleCheck -->|": staff 프론트"|RoleStaff[프론트\n조회만\n수업 등록/수정/삭제 불가\n드래그앤드롭 불가]

  RoleCheck -->|": readonly"|RoleReadonly[읽기전용\n캘린더 접근 차단]

  RolePrimary --> CanReg_P[수업 등록 버튼 활성]
  RolePrimary --> CanEdit_P[수정/삭제/드래그 활성]
  RolePrimary --> CanApprove_P[승인/거절 활성]
  RolePrimary --> CanBulk_P[일괄변경 활성]

  RoleManager --> CanReg_M[수업 등록 버튼 활성]
  RoleManager --> CanEdit_M[수정/삭제/드래그 활성]
  RoleManager --> CanApprove_M[승인/거절 활성]

  RoleFC --> OwnOnly[본인 수업\n수정/삭제/드래그 활성]
  RoleFC --> OtherRO[타강사 수업\n조회만]
  RoleFC --> NoReg_FC[수업 등록 버튼 숨김 or 비활성]

  RoleStaff --> ViewOnly_S[모든 수업 조회만\n버튼류 비활성]

  RoleReadonly --> Blocked[권한없음 페이지 리다이렉트]
```
