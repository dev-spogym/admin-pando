## 1. 목적
DLG-C016 대안 일정 전송 API 결과 분기를 정의한다.

## 2. 전제조건
- 유효성 통과 후 API 호출

## 3. 다이어그램

```mermaid
flowchart TD
  API{-->|": 200"|Success[전송 완료]
  API -->|": 400"|BadReq[toast 입력값을 확인해주세요.]
  API -->|": 404"|NotFound[toast 요청을 찾을 수 없습니다.]
  API -->|": 403"|PermErr[toast 권한이 없습니다.]
  API -->|": 500"|SysErr[toast 처리에 실패했습니다.]

  Success --> CloseModal[모달 닫힘 + 목록 갱신]
  CloseModal --> Toast_S[toast 대안 일정이 전송되었습니다.]

  BadReq --> Stay[모달 유지]
  NotFound --> CloseModal
  PermErr --> Stay
  SysErr --> Stay

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef modal fill:#FFF3E0,stroke:#F57C00,color:#E65100
  class API system
  class Success,CloseModal,Toast_S success
  class SysErr error
  class BadReq,NotFound,PermErr warning
  class Stay modal
```

## 4. 엣지 설명

| 응답 | 동작 |
|------|------|
| 200 | 닫힘 + success 토스트 |
| 400/403 | 경고 + 유지 |
| 404 | 경고 + 닫힘 (요청 이미 처리됨) |
| 500 | 에러 + 유지 |
