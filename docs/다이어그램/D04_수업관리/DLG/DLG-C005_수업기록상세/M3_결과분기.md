## 1. 목적
DLG-C005 상태변경 API 결과 분기를 정의한다.

## 2. 전제조건
- 상태변경 API 호출 후

## 3. 다이어그램

```mermaid
flowchart TD
  StatusAPI{-->|": 200"|Success[상태변경 완료]
  StatusAPI -->|": 400 잘못된전환"|InvalidTrans[toast 유효하지 않은 상태 전환입니다.]
  StatusAPI -->|": 403"|PermErr[toast 권한이 없습니다.]
  StatusAPI -->|": 500"|SysErr[toast 처리에 실패했습니다.]

  Success -->|": no_show/cancelled"|CloseModal[모달 닫힘 + 목록 갱신]
  Success -->|": in_progress"|RefreshModal[모달 내 상태 갱신]
  Success -->|": completed via DLG-C006"|CloseModal

  InvalidTrans --> StayModal[모달 유지]
  PermErr --> StayModal
  SysErr --> StayModal

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef modal fill:#FFF3E0,stroke:#F57C00,color:#E65100
  class StatusAPI system
  class Success,CloseModal,RefreshModal success
  class SysErr error
  class InvalidTrans,PermErr warning
  class StayModal modal
```

## 4. 엣지 설명

| 응답 | 동작 | |------|------| | 200 no_show/cancelled | 모달 닫힘 | | 200 in_progress | 모달 내 갱신 | | 400 | 상태전환 불가 경고 | | 403 | 권한 경고 | | 500 | 에러 |
