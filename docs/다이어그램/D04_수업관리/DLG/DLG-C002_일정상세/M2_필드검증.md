## 1. 목적
DLG-C002에서 삭제 권한 검증 로직을 정의한다.

## 2. 전제조건
- DLG-C002 열림 상태

## 3. 다이어그램

```mermaid
flowchart TD
  DeleteBtn[삭제 버튼 클릭] --> RoleCheck{역할 확인}
  RoleCheck -->|": manager/owner"|StatusCheck{수업 상태 확인}
  RoleCheck -->|": fc"|FCOwnerCheck{본인 수업?}
  RoleCheck -->|": staff/readonly"|PermBlocked[버튼 비활성 - 표시 없음]

  FCOwnerCheck -->|": 본인"|StatusCheck
  FCOwnerCheck -->|": 타강사"|PermErr[toast 권한이 없습니다.]

  StatusCheck -->|": scheduled"|AllowDelete[삭제 허용]
  StatusCheck -->|": in_progress/completed"|BlockDelete[toast\n진행/완료된 수업은 삭제할 수 없습니다.]

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  class DeleteBtn,RoleCheck,FCOwnerCheck,StatusCheck system
  class AllowDelete success
  class PermErr,BlockDelete warning
  class PermBlocked error
```

## 4. 엣지 설명

| 역할 | 삭제 가능 조건 | |------|--------------| | manager/owner | scheduled 상태만 | | fc | 본인 수업 + scheduled | | staff/readonly | 불가 |
