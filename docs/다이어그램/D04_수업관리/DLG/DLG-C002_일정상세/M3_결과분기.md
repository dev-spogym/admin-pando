## 1. 목적
DLG-C002 삭제/수정 액션의 결과 분기를 정의한다.

## 2. 전제조건
- 삭제/수정 API 호출 후

## 3. 다이어그램

```mermaid
flowchart TD
  DeleteAPI{-->|": 200"|DelSuccess[모달 닫힘\n캘린더 갱신\ntost 수업이 삭제되었습니다.]
  DeleteAPI -->|": 403"|DelPerm[toast 권한이 없습니다.]
  DeleteAPI -->|": 404"|DelNotFound[toast 수업을 찾을 수 없습니다.]
  DeleteAPI -->|": 500"|DelErr[toast 처리에 실패했습니다.]

  DelPerm --> StayModal[모달 유지]
  DelNotFound --> CloseModal[모달 닫힘 + 갱신]
  DelErr --> StayModal

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef modal fill:#FFF3E0,stroke:#F57C00,color:#E65100
  class DeleteAPI system
  class DelSuccess,CloseModal success
  class DelErr error
  class DelPerm,DelNotFound warning
  class StayModal modal
```

## 4. 엣지 설명

| 응답 | 동작 | |------|------| | 200 | success 토스트 + 모달 닫힘 + 캘린더 갱신 | | 403 | warning + 모달 유지 | | 404 | warning + 모달 닫힘 | | 500 | error + 모달 유지 |
