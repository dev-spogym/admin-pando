## 1. 목적
DLG-C009 저장 API 결과 분기를 정의한다.

## 2. 전제조건
- 유효성 통과 후 API 호출

## 3. 다이어그램

```mermaid
flowchart TD
  API{POST/ -->|": 201/200"|Success[저장 완료]
  API -->|": 409"|Dup[toast 동일한 템플릿명이 존재합니다.]
  API -->|": 403"|Perm[toast 권한이 없습니다.]
  API -->|": 500"|Err[toast 처리에 실패했습니다.]

  Success --> CloseModal[모달 닫힘 + 목록 갱신]
  CloseModal --> Toast_S[toast 템플릿이 저장되었습니다.]

  Dup --> Stay[모달 유지]
  Perm --> Stay
  Err --> Stay

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef modal fill:#FFF3E0,stroke:#F57C00,color:#E65100
  class API system
  class Success,CloseModal,Toast_S success
  class Err error
  class Dup,Perm warning
  class Stay modal
```

## 4. 엣지 설명

| 응답 | 동작 | |------|------| | 201/200 | 닫힘 + success | | 409 | 중복 경고 + 유지 | | 403/500 | 에러/경고 + 유지 |
