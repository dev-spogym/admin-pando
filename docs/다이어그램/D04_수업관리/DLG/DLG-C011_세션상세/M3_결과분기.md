## 1. 목적
DLG-C011 횟수 차감 API 결과 분기를 정의한다.

## 2. 전제조건
- 차감 유효성 통과 후 API 호출

## 3. 다이어그램

```mermaid
flowchart TD
  API{-->|": 200"|Success[차감 완료]
  API -->|": 400 잔여없음"|NoRemain[toast 잔여 횟수가 없습니다.]
  API -->|": 403"|PermErr[toast 권한이 없습니다.]
  API -->|": 500"|SysErr[toast 처리에 실패했습니다.]

  Success --> RefreshModal[잔여횟수 갱신\n이력 추가]
  RefreshModal --> Toast_S[toast 횟수가 차감되었습니다.]

  NoRemain --> StayModal[모달 유지]
  PermErr --> StayModal
  SysErr --> StayModal

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef modal fill:#FFF3E0,stroke:#F57C00,color:#E65100
  class API system
  class Success,RefreshModal,Toast_S success
  class SysErr error
  class NoRemain,PermErr warning
  class StayModal modal
```

## 4. 엣지 설명

| 응답 | 동작 |
|------|------|
| 200 | 잔여횟수 갱신 + success 토스트 |
| 400 | 잔여 없음 경고 + 유지 |
| 403/500 | 에러/경고 + 유지 |
