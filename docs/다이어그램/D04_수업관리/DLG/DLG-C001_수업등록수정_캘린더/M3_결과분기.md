## 1. 목적
DLG-C001 제출 후 API 응답에 따른 결과 분기를 정의한다.

## 2. 전제조건
- 유효성 검사 통과 후 API 호출

## 3. 다이어그램

```mermaid
flowchart TD
  API{POST/ -->|": 201/200"|Success[수업 등록/수정 완료]
  API -->|": 409 충돌"|Conflict[toast 동일한 시간에 수업이 존재합니다.]
  API -->|": 403 권한없음"|Permission[toast 권한이 없습니다.]
  API -->|": 400 검증실패"|BadRequest[toast 입력값을 확인해주세요.]
  API -->|": 500"|SystemErr[toast 처리에 실패했습니다.]

  Success --> CloseModal[모달 닫힘]
  CloseModal --> CalRefresh[캘린더 이벤트 갱신]
  CalRefresh --> Toast_S[toast\n수업이 등록되었습니다. / 수업이 수정되었습니다.]

  Conflict --> StayModal[모달 유지\n시간 수정 유도]
  Permission --> StayModal
  BadRequest --> StayModal
  SystemErr --> StayModal

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  classDef modal fill:#FFF3E0,stroke:#F57C00,color:#E65100
  class API system
  class Success,CloseModal,CalRefresh,Toast_S success
  class SystemErr error
  class Conflict,Permission,BadRequest warning
  class StayModal modal
```
