## 1. 목적
SCR-C014에서 QR코드 기반 출석 체크인 생성/스캔/확인 메인 플로우를 정의한다.

## 2. 전제조건
- 로그인 완료
- lesson_status = scheduled 또는 in_progress 수업 존재

## 3. 다이어그램

```mermaid
flowchart TD
  SCR_C014[SCR-C014 출석 QR 체크인 🆕]

  SCR_C014 --> ModeSelect{모드 선택}
  ModeSelect -->|": QR 생성"|QRGenerate[QR코드 생성\nlesson_id + timestamp + token]
  ModeSelect -->|": QR 스캔"|QRScan[카메라 스캔 모드\n유효시간 5분]

  QRGenerate --> DisplayQR[QR 화면 표시\n300px + 유효시간 카운트다운]
  DisplayQR -->|": 만료"|QRExpired[QR 만료\n재생성 버튼]
  QRExpired -->|": 재생성"|QRGenerate

  QRScan -->|": 스캔 성공"|ValidateAPI{ValidateAPI -->|": 200 유효"|CheckinOK[출석 처리 완료\nattendance_status = present]
  ValidateAPI -->|": 400 만료"|ExpiredErr[toast QR이 만료되었습니다.]
  ValidateAPI -->|": 404 불일치"|InvalidErr[toast 유효하지 않은 QR입니다.]
  ValidateAPI -->|": 409 중복"|DupErr[toast 이미 출석 처리되었습니다.]
  ValidateAPI -->|": 500"|SysErr[toast 처리에 실패했습니다.]

  CheckinOK --> AttendanceList[출석 현황 목록 갱신\n회원명/체크인시간/상태]

  SCR_C014 --> ManualCheckin[수동 출석 처리\n매니저 전용]
  ManualCheckin --> ValidateAPI

  AttendanceList -->|": 엑셀 다운로드"|ExcelExport[출석 현황 엑셀]
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 | |---------|------|------|------| | | ModeSelect | QRGenerate | QR 생성 모드 선택 | | | ModeSelect | QRScan | QR 스캔 모드 선택 | | ~06 | DisplayQR | QRExpired → QRGenerate | 5분 만료 후 재생성 | | ~12 | QRScan | ValidateAPI | 스캔 후 서버 검증 (6가지 응답) | | ~15 | ManualCheckin | ValidateAPI | 매니저 수동 출석 처리 |
