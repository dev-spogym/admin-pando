## 화면 목적

퇴사 예정인 직원의 퇴사일을 설정하고, 해당 직원이 담당하던 미완료 업무·회원·수업을 인수인계한 뒤 시스템 계정을 비활성화하는 화면입니다. 퇴사 처리가 완료되면 해당 직원은 재직 상태에서 제외되고 이후 로그인이 차단됩니다.

## 주요 구성요소

- **퇴사 대상 직원 정보**: 퇴사 처리할 직원의 이름·역할·입사일을 요약해 표시합니다.
- **퇴사일 설정**: 실제 퇴사 확정일을 날짜 선택기로 입력합니다.
- **퇴사 예정일 설정**: 미리 퇴사 예정일을 등록해 사전 안내에 활용합니다.
- **퇴사 사유 입력**: 퇴사 사유를 자유 텍스트로 입력합니다. (내부 관리용)
- **미완료 업무 인수인계**: 해당 직원이 담당하던 회원, 예약된 수업, 진행 중인 계약 등을 인계받을 담당자를 지정합니다.
- **계정 비활성화 확인**: 퇴사 처리 완료 시 시스템 로그인 계정이 즉시 비활성화된다는 안내를 표시합니다.
- **처리 완료 / 취소 버튼**: 최종 확인 후 퇴사 처리를 완료하거나 취소합니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 센터장 / 매니저 | 퇴사 처리 전체 폼 | 퇴사일 설정, 인수인계 지정, 계정 비활성화 |
| 기타 직원 | 접근 불가 | 퇴사 처리 화면 진입 불가 |

## 주요 UX 흐름

1. 직원 목록(SCR-060)에서 대상 직원을 체크박스로 선택한 뒤 "퇴사 처리" 버튼을 클릭합니다.
2. 퇴사 처리 화면에서 퇴사일과 퇴사 사유를 입력합니다.
3. 미완료 업무가 있다면 인계받을 담당자를 지정합니다.
4. 계정 비활성화 안내를 확인하고 "처리 완료" 버튼을 클릭합니다.
5. 최종 확인 다이얼로그(DLG-060-002)가 표시되며, 확인 시 퇴사 처리가 완료됩니다.
6. 처리 완료 후 직원 목록(SCR-060)으로 돌아가며 해당 직원은 "퇴사" 상태로 표시됩니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 정상 | 퇴사 대상 직원 정보와 입력 폼이 표시됩니다. |
| 미완료 업무 있음 | 인수인계 대상 목록이 추가로 표시됩니다. |
| 미완료 업무 없음 | 인수인계 섹션이 "담당 업무 없음"으로 표시됩니다. |
| 처리 중 | 처리 완료 버튼이 로딩 상태로 변경됩니다. |
| 처리 완료 | 성공 알림 후 직원 목록으로 이동합니다. |

## 관련 다이어그램

- DLG-060-002 삭제(퇴사) 확인 다이얼로그
- SCR-060 직원 목록으로 복귀
