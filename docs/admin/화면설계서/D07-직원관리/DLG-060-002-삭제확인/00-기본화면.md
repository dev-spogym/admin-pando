## 목적

선택한 직원의 퇴사 처리를 최종 확정하기 전에 관리자의 의도를 재확인합니다. 퇴사 처리는 해당 직원의 시스템 접근을 차단하는 중요한 작업이므로 실수 방지를 위해 텍스트 입력 확인 절차를 포함합니다.

## 구성요소

- **제목**: "직원 퇴사 처리"
- **안내 메시지**: "선택한 N명의 직원을 퇴사 처리하시겠습니까? 퇴사 처리 시 해당 직원의 계정 접근이 제한됩니다."
- **확인 텍스트 입력 필드**: "퇴사처리"를 직접 입력해야 최종 확인 버튼이 활성화됩니다.
- **퇴사 처리 버튼 (위험)**: 텍스트 입력 확인 후 퇴사 처리를 실행합니다. 빨간색 계열로 표시됩니다.
- **취소 버튼**: 다이얼로그를 닫고 아무 처리도 하지 않습니다.

## 언제 표시되나요

- 직원 목록(SCR-060)에서 1명 이상의 직원을 선택한 뒤 "퇴사 처리" 버튼을 클릭할 때 표시됩니다.
- SCR-062 퇴사 처리 화면에서 "처리 완료" 버튼을 클릭할 때도 표시됩니다.

## 주요 흐름

1. 관리자가 대상 직원을 선택하고 퇴사 처리 버튼을 클릭합니다.
2. 이 다이얼로그가 표시되며 처리 대상 인원 수가 안내됩니다.
3. "퇴사처리" 텍스트를 입력 필드에 직접 타이핑합니다.
4. 텍스트가 일치하면 "퇴사 처리" 버튼이 활성화됩니다.
5. 버튼을 클릭하면 처리가 완료되고 직원 상태가 "퇴사"로 변경됩니다.
6. "취소"를 클릭하면 다이얼로그가 닫히고 이전 화면으로 돌아갑니다.
