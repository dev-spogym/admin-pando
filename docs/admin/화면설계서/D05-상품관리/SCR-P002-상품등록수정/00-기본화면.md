## 화면 목적

새로운 상품을 등록하거나 기존 상품을 수정하는 모달형 패널입니다. SCR-P001 상품 목록에서는 우측 패널로 열리고, `/products/new` 또는 `/products/edit` 직접 경로에서는 동일한 상품 등록 모달 레이아웃을 단독 화면으로 확인합니다. 상품 구분(레슨/이용/락커/판매)에 따라 필요한 항목이 동적으로 표시되며, 저장 시 Supabase `products` 테이블에 즉시 반영됩니다.

---

## 주요 구성요소

### 상품 구분 선택

| 구분 | 설명 | 저장 유형 | 추가 표시 항목 |
|------|------|-----------|----------------|
| 레슨 | PT, GX 등 수업형 상품 | `category=PT/GX`, `productType=LESSON` | 레슨 시간, 레슨 유효기간, 횟수, 수업구분(개인/정규클래스) |
| 이용 | 기간제 또는 횟수제 회원권 | `category=MEMBERSHIP`, `productType=MEMBERSHIP` | 이용구분(기간/횟수/포인트), 기간 또는 횟수 입력 |
| 락커 | 락커 이용권 | `category=PRODUCT`, `productType=RENTAL` | 기간 입력 |
| 판매 | 실물 상품 판매 | `category=PRODUCT`, `productType=GENERAL` | 태그 입력 |

### 기본 정보 입력

| 항목 | 필수 | 설명 |
|------|------|------|
| 상품 대분류 | 필수 | `MEMBERSHIP` / `PT` / `GX` / `PRODUCT` / `SERVICE` 중 선택 |
| 상품그룹 | 선택 | SCR-P001 분류 관리 탭에서 등록한 활성 그룹 중 직접 선택 |
| 종목·세부종목 | 조건부 필수 | `GX` 선택 시 요가 / 필라테스 / 스피닝 / 줌바 / 에어로빅 / GX 기타 중 필수 선택 |
| 상품명 | 필수 | 중복 불가 |
| 현금가 | 필수 | 현금 결제 기준 가격 (0 초과) |
| 카드가 | 선택 | 미입력 시 현금가와 동일하게 저장 |
| 기간 | 조건부 | 기간제 이용권 또는 락커 상품에서 필수 |
| 횟수/포인트 | 조건부 | 레슨, 횟수제, 포인트 상품에서 필수 |
| 설명 | 선택 | 상품 상세 설명 |
| 태그 | 선택 | 검색·분류용 자유 입력 |

### 2단계 분류 선택 규칙

| 1단계 상품 대분류 | 2단계 종목·세부종목 표시 |
|------|------|
| `MEMBERSHIP` | 헬스 / 골프 / 수영 / 복싱 / 크로스핏 / 기타 |
| `PT` | PT / 재활PT / 그룹PT / 기타 |
| `GX` | 요가 / 필라테스 / 스피닝 / 줌바 / 에어로빅 / GX 기타 |
| `PRODUCT` | 운동복 / 락커 / 식음료 / 용품 / 기타 |
| `SERVICE` | 부가서비스 / 대여 / 기타 |

- 사용자가 1단계에서 `GX`를 선택하면 2단계 드롭다운은 반드시 6개 GX 세부종목만 표시합니다.
- `GX` 상품 저장 시 2단계 세부종목이 비어 있으면 저장 버튼을 비활성화하고 "GX 세부종목을 선택해 주세요" 오류를 표시합니다.
- 저장 데이터는 `category=GX`와 `sportType` 조합으로 상품 마스터에 기록합니다. 상품그룹은 별도 `productGroupId`로 선택 저장합니다.

### Supabase 저장 매핑

| UI/상태 | 저장 컬럼 | 규칙 |
|---------|----------|------|
| 현재 지점 | `branchId` | 현재 선택된 지점 ID |
| 상품명 | `name` | 동일 지점 + 동일 대분류에서 중복 불가 |
| 상품 구분 | `category`, `productType` | 이용=`MEMBERSHIP`, 레슨=`LESSON`, 락커=`RENTAL`, 판매=`GENERAL` |
| 현금가/카드가 | `price`, `cashPrice`, `cardPrice` | `price`는 현금가 기준, 카드가 미입력 시 현금가와 동일 |
| 기간 | `duration` | 기간제 이용권·락커 상품의 일수 |
| 횟수/포인트 | `sessions`, `totalCount` | 레슨·횟수제·포인트 상품의 사용 가능 수량 |
| 상품그룹 | `productGroupId` | 분류 관리 탭의 `product_groups.id`를 직접 저장. 자동 추론으로 고정하지 않음 |
| GX 세부종목 | `sportType` | GX 6개 고정 옵션 중 하나 |
| 요일·시간 | `usage_restrictions.availableDays`, `availableTimeStart`, `availableTimeEnd` | 선택된 요일과 시작/종료 시간 저장 |
| 시설/예약/휴회 옵션 | `usage_restrictions` JSON | 시설 이용, 예약 가능, 예약 간격, 휴회 횟수·기간 저장 |
| 홀딩/양도/포인트 | `holdingEnabled`, `transferEnabled`, `pointAccrual` | 목록 컬럼 및 회원권 정책 판단에 사용 |
| 키오스크/판매채널 | `kioskVisible`, `salesChannel` | 키오스크 노출 및 판매 채널 기준 |

### 이용 요일 · 시간 설정

- 요일별(월~일) 체크박스로 이용 가능 요일 선택
- 체크된 요일에 시작·종료 시간 지정 (10분 단위)
- 체크하지 않은 요일은 이용 불가로 처리

### 패키지 구성 (선택)

- "이 상품을 패키지 상품으로 등록" 체크 시 활성화
- 구성 상품 선택 후 추가
- 구성 상품별 개별 가격 표시
- 합산 정가 자동 계산
- 패키지 할인가 별도 입력

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 상품 편집 권한 없는 직원 | 진입 불가 (버튼 자체 비표시) | 없음 |
| 상품 편집 권한 보유 직원 | 전체 등록/수정 폼 | 상품 등록, 수정, 삭제 |
| 슈퍼관리자 | 전체 등록/수정 폼 | 상품 등록, 수정, 삭제 |

## 주요 UX 흐름

1. **등록 모드**: SCR-P001에서 "+ 상품 등록" 클릭 → 빈 폼으로 패널 열림 → 정보 입력 → "등록" 버튼 클릭 → 저장 완료 후 목록 갱신
2. **수정 모드**: SCR-P001에서 상품 행 클릭 → 기존 정보가 채워진 폼으로 패널 열림 → 수정 → "저장" 버튼 클릭
3. **상품 정보 가져오기**: 하단 "상품정보 가져오기" 버튼 → DLG-P005-상품가져오기 → 선택한 상품 정보 자동 입력 (새 상품으로 등록 처리)
4. **삭제**: 수정 모드에서 "삭제" 버튼 클릭 → 판매 이력 확인 후 삭제 확인 또는 비활성화 안내 다이얼로그 표시
5. **작업 취소**: "닫기" 버튼 또는 패널 외부 영역 클릭 → DLG-P002-작업취소확인 (변경 사항 있을 경우)

## 화면 상태

| 상태 | 조건 | 표시 내용 |
|------|------|-----------|
| 등록 모드 | 신규 등록 진입 | 헤더 "상품 등록", 저장 버튼 "등록", 삭제 버튼 없음 |
| 수정 모드 | 기존 상품 선택 | 헤더 "상품 수정" + 현재 활성 상태 배지, 저장 버튼 "저장", 삭제 버튼 표시 |
| 가져오기 모드 | 상품 정보 가져온 직후 | 수정 모드 외형이지만 "등록" 버튼으로 표시 (새 상품으로 저장) |
| 저장 중 | 저장 버튼 클릭 후 완료 전 | 저장 버튼 로딩 상태, 모든 입력 비활성화 |
| 저장 실패 | Supabase 오류 또는 중복 상품명 | 오류 토스트 표시, 입력값 유지 |
| 삭제 확인 중 | 삭제 버튼 클릭 후 | 하단에 "정말 삭제하시겠습니까?" 인라인 확인 버튼 표시 |
| 비활성화 확인 중 | 판매 이력 있는 상품 삭제 시도 | "이용 중인 회원이 N명 있습니다. 미사용으로 전환하시겠습니까?" 표시 |
