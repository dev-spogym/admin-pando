## 화면 목적

상품 판매 시 적용할 할인 정책을 등록·수정·삭제하는 화면입니다. 정률(%) 또는 정액(원) 방식의 할인을 설정하고, 최소 계약 기간 조건이나 최대 할인 한도를 지정할 수 있습니다. 변경 이력을 통해 최근 수정 내역도 확인할 수 있습니다.

---

## 주요 구성요소

### 할인 정책 목록 테이블

| 컬럼 | 내용 |
|------|------|
| 할인명 | 할인 정책 이름 |
| 유형 | 정률(%) / 정액(원) |
| 할인 값 | 할인율 또는 할인 금액 |
| 조건 | 최소 계약 기간 (없으면 "-") |
| 한도 | 최대 할인금액 (없으면 "-") |
| 상태 | 활성 / 비활성 |
| 액션 | 수정 / 삭제 버튼 |

### 변경 이력 타임라인

- 최근 등록·수정·삭제 이력을 시간순으로 표시
- 이력이 없으면 타임라인 섹션 자체가 표시되지 않음
- 각 이력: 날짜·시간, 작업 구분(등록/수정/삭제), 할인명, 요약 정보

---

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 일반 직원 | 할인 목록 조회, 변경 이력 | 조회만 가능 |
| 관리자 권한 보유 직원 | 전체 화면 | 할인 정책 등록, 수정, 삭제 |
| 슈퍼관리자 | 전체 화면 | 할인 정책 등록, 수정, 삭제 |

---

## 주요 UX 흐름

1. **할인 정책 등록**: "할인 추가" 버튼 → [DLG-P004-할인규칙등록] 모달에서 정보 입력 → 저장
2. **할인 정책 수정**: 목록 행의 수정 아이콘 클릭 → [DLG-P005-할인규칙수정] 모달 → 수정 후 저장
3. **할인 정책 삭제**: 목록 행의 삭제 아이콘 클릭 → [DLG-P006-할인규칙삭제확인] → 확인 후 삭제
4. **이력 확인**: 화면 하단 변경 이력 타임라인에서 최근 작업 내역 확인

---

## 화면 상태

| 상태 | 조건 | 표시 내용 |
|------|------|-----------|
| 로딩 중 | 할인 정책 불러오는 중 | 테이블 로딩 표시 |
| 빈 상태 | 등록된 할인 정책 없음 | "등록된 할인 정책이 없습니다." 안내 |
| 정상 | 할인 정책 목록 존재 | 테이블 표시 |
| 이력 있음 | 변경 이력 1건 이상 | 하단 타임라인 표시 |
| 이력 없음 | 변경 이력 0건 | 타임라인 섹션 숨김 |
