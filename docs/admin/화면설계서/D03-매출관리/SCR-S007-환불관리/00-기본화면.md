## 화면 목적

처리된 환불 내역을 조회하고 통계를 확인하는 화면입니다. 이번 달 환불 총액·건수·평균 환불액·위약금 합계를 파악하고, 기간별로 환불 이력을 조회합니다. 환불 사유와 처리 상태, 원판매 담당자별 환불 책임 현황을 함께 확인할 수 있어 고객 불만 패턴 파악 및 환불 정책 점검에 활용합니다.

## 주요 구성요소

- **요약 지표 카드 (4개)**: 이번 달 환불 총액 / 환불 건수 / 평균 환불액 / 위약금 합계를 상단에 표시합니다.
  - 위약금 합계: 조회 기간 내 환불 건에서 공제된 위약금 총액을 표시합니다.
- **날짜 필터**: 이번 달·지난 달·최근 3개월 프리셋 버튼과 시작일~종료일 직접 입력, 조회 버튼을 제공합니다.
- **환불 목록 테이블**: No / 환불일 / 회원명 / 상품명 / 환불금액 / 환불방법 / 사유 / 처리자 / 상태 컬럼을 제공합니다.
  - 환불금액: 붉은색으로 강조 표시합니다.
  - 상태 배지: 완료(초록) / 처리중(노랑) / 거절(빨강) 색상으로 구분합니다.
  - 사유: 작은 글씨의 보조 텍스트로 표시합니다.
- **원판매 담당자별 환불 책임 탭**: 환불 처리자와 별도로 원매출 판매 귀속 담당자별 환불 건수, 환불 금액, 위약금 합계, 원매출 대비 환불률을 표시합니다.
- **엑셀 내보내기**: 현재 조회된 환불 내역 전체를 엑셀 파일로 내려받습니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 기간 조회, 엑셀 내보내기 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 기간 조회, 엑셀 내보내기 |
| FC | 담당 회원 결제 처리 가능 | 조회만 가능 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

## 주요 UX 흐름

1. 사이드바에서 "매출관리 > 환불 관리"를 클릭해 진입합니다.
2. 상단 지표 카드로 이번 달 환불 현황과 위약금 회수 규모를 파악합니다.
3. 날짜 프리셋 또는 직접 입력으로 조회 기간을 설정하고 "조회" 버튼을 누릅니다.
4. 테이블에서 환불 이력·사유·처리 상태를 확인합니다.
5. 원판매 담당자별 환불 책임 탭에서 특정 담당자에게 환불이 집중되는지 확인합니다.
6. 필요 시 엑셀 버튼으로 내역을 파일로 저장합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 로딩 중 | 데이터를 불러오는 동안 테이블 영역에 로딩 상태를 표시합니다. |
| 정상 표시 | 환불 내역이 있을 때 지표 카드와 테이블을 표시합니다. |
| 데이터 없음 | 해당 기간에 환불 내역이 없을 때 빈 테이블 안내를 표시합니다. |
| 오류 | 데이터 조회 실패 시 오류 메시지 알림(토스트)을 표시합니다. |

## 데이터 매핑

| 항목 | 외부 템플릿 | 셀 / 컬럼 | 산출 기준 |
|---|---|---|---|
| 위약금 합계 | `[환불재결제]`, `[본사취합-환불재결제]` | 환불 탭 H열 또는 지점 템플릿 G열 | 환불 처리 과정에서 공제된 위약금 총액 |
| 환불 처리자 | `[환불재결제]`, `[본사취합-환불재결제]` | 환불 탭 담당자 열 | 환불 승인 또는 처리 담당자 |
| 원판매 담당자별 환불 합계 | `[환불재결제]`, `[매출종합]` | 환불 행 + 원거래ID 또는 카드 승인번호 | 원매출 판매 귀속 담당자별 환불 건수·환불금액·위약금 합계 |

## 비즈니스 규칙

- 환불 목록의 `처리자`는 환불 승인 또는 환불 처리 담당자를 의미합니다.
- `원판매 담당자`는 원결제의 판매 귀속 담당자이며, `[매출종합]` 회원권·락카 K열 `담당자`, 강습·골프 K열 `등록강사`를 기준으로 합니다.
- 담당자별 환불 책임 지표는 처리자 기준이 아니라 원판매 담당자 기준으로 집계합니다.
- 환불 행과 원매출 행의 공식 연결 키는 원거래ID 또는 카드 승인번호입니다.
- 원거래ID 또는 승인번호가 없는 환불 행은 회원명+종목+등록일+금액 조합으로 자동 확정하지 않고, 다중 후보 검수 대상으로 분리합니다.
