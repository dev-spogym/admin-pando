## 화면 목적

POS 판매 화면(SCR-S002)에서 장바구니에 담은 상품의 결제를 완료하는 화면입니다. 결제수단(카드·현금·마일리지·복합결제)을 선택하고 최종 결제 금액을 확인한 뒤 결제를 처리합니다. 결제 완료 후에는 영수증 출력·문자 발송·계속 판매 등의 후속 동작을 안내합니다.

## 주요 구성요소

- **결제 상품 목록 (좌측)**: POS에서 담아온 상품명·수량·소계를 목록으로 표시하고, 하단에 총 결제 금액을 크게 표시합니다.
- **회원 검색 (좌측)**: 마일리지 결제 시 회원 지정이 필수입니다. 이름 또는 전화번호로 검색해 회원을 선택하면 보유 마일리지가 표시됩니다. 선택된 회원은 X 버튼으로 해제할 수 있습니다.
- **결제수단 선택 (우측)**: 카드 / 현금 / 마일리지 / 복합결제 중 하나를 선택합니다. 선택된 수단은 강조 표시됩니다.
  - **복합결제**: 카드·현금·마일리지 금액을 각각 입력해 합계가 총액과 일치해야 결제 버튼이 활성화됩니다. 입력 합계가 부족하거나 초과하면 차액이 붉은색으로 표시됩니다.
  - **마일리지 결제**: 회원 미선택 시 안내 문구를 표시하고 결제 버튼이 비활성화됩니다. 보유 마일리지가 부족하면 부족액을 붉은색으로 표시합니다.
- **결제 확인 버튼**: 모든 조건이 충족되면 활성화됩니다. 클릭 시 결제 확인 모달이 열립니다.
- **결제 확인 모달**: 상품 내역·합계·결제수단·선택 회원을 최종 확인하고 "결제 완료" 버튼으로 처리합니다. 처리 중에는 버튼이 "처리 중..."으로 변경되고 비활성화됩니다.
- **중복 결제 경고**: 동일 회원이 최근 같은 금액을 결제한 이력이 있으면 경고 다이얼로그가 표시됩니다. "계속 진행" 또는 "취소"를 선택할 수 있습니다.
- **결제 완료 화면**: 결제 성공 시 체크 아이콘과 완료 메시지가 표시됩니다. 영수증 출력 / 문자 발송 / 계속 판매하기 / 매출 현황으로 이동 버튼을 제공합니다.
- **초기화 버튼**: 상단에 위치하며, 클릭 시 결제 상태를 초기 상태로 리셋합니다.
- **이전으로 돌아가기**: POS 판매 화면(SCR-S002)으로 돌아갑니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 모든 결제수단으로 결제 처리, 회원 검색, 영수증 출력, 문자 발송 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 모든 결제수단으로 결제 처리, 회원 검색, 영수증 출력, 문자 발송 |
| FC | 담당 회원 결제 처리 가능 | 모든 결제수단으로 결제 처리, 회원 검색 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 결제 처리 화면 접근 불가 |

## 주요 UX 흐름

1. POS 판매 화면에서 "결제하기"를 눌러 진입합니다. 장바구니 상품이 자동으로 불러와집니다.
2. 좌측에서 상품 목록과 총 결제 금액을 확인합니다.
3. 우측에서 결제수단(카드·현금·마일리지·복합결제)을 선택합니다.
4. 마일리지 또는 복합결제(마일리지 포함) 선택 시, 좌측에서 회원을 검색해 지정합니다.
5. 복합결제의 경우 카드·현금·마일리지 금액을 각각 입력해 합계를 맞춥니다.
6. 조건이 모두 충족되면 "결제 확인" 버튼이 활성화됩니다. 클릭하면 확인 모달이 열립니다.
7. 모달에서 내역을 최종 확인하고 "결제 완료"를 누릅니다.
8. 동일 회원의 중복 결제가 감지되면 경고 다이얼로그에서 진행 여부를 선택합니다.
9. 결제가 완료되면 완료 화면으로 전환됩니다. 영수증 출력·문자 발송·계속 판매·매출 현황 이동 중 선택합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 결제 대기 | 결제수단 선택 및 조건 미충족 상태. 결제 확인 버튼 비활성화. |
| 결제 준비 완료 | 모든 조건(상품·결제수단·금액 등)이 충족된 상태. 결제 확인 버튼 활성화. |
| 마일리지 부족 | 보유 마일리지가 총액보다 적을 때 부족액을 붉은색으로 표시하고 결제 버튼 비활성화. |
| 복합결제 금액 불일치 | 입력한 금액 합계가 총액과 다를 때 차액(부족/초과)을 붉은색으로 표시하고 결제 버튼 비활성화. |
| 결제 처리 중 | 결제 완료 버튼 클릭 후 서버 처리 중. 버튼이 "처리 중..."으로 변경되고 재클릭 불가. |
| 중복 결제 경고 | 동일 회원·금액 중복 감지 시 경고 다이얼로그 표시. |
| 결제 완료 | 결제 성공 후 완료 화면(체크 아이콘 + 후속 버튼)으로 전환. |

- 결제수단별 분기 흐름 (카드 / 현금 / 마일리지 / 복합결제)
- 중복 결제 감지 및 경고 흐름
- 결제 처리 → 완료 화면 전환 흐름
- 마일리지 차감 및 적립 흐름
- 계정 권한별 접근 분기 흐름
