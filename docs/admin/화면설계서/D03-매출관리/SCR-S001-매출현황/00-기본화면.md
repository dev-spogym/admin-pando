## 화면 목적

센터에서 발생한 전체 매출 거래를 한눈에 조회하고 분석하는 매출관리 도메인의 핵심 허브 화면입니다. 담당자는 오늘·이번 주·이번 달 매출을 즉시 확인하고, 구매자나 상품명으로 원하는 거래를 빠르게 찾을 수 있습니다. 매출 내역 외에도 기간별·상품별·결제수단별·담당자별 집계, 환불 내역, 미납 내역을 탭으로 전환하며 관리합니다.

## 주요 구성요소

- **요약 지표 카드 (4개)**: 순 매출 / 카드 결제 / 현금 결제 / 미수금을 상단에 크게 표시합니다.
- **기간 프리셋 버튼**: 오늘 / 이번 주 / 이번 달을 클릭하면 날짜 범위가 자동 설정됩니다.
- **검색 및 필터 바**: 구매자 이름 또는 상품명 통합 검색, 유형(이용권·PT·상품·기타) 필터, 상태(완료·환불·미납) 필터, 날짜 범위 직접 지정.
- **매출유형 필터 탭**: 전체 / 신규 / 재등록 / 휴면복귀 / 종목추가 / 업그레이드 중 하나를 선택해 해당 유형의 거래만 표시합니다.
- **분석 탭 (7개)**: 매출 내역 / 기간별 매출 / 상품별 내역 / 결제수단별 / 담당자별 / 환불 내역 / 미납 내역을 탭으로 전환합니다. 환불·미납 탭에는 해당 건수가 배지로 표시됩니다.
  - 담당자별 탭은 담당자별 총매출뿐 아니라 신규 매출, 재등록 매출, 기타 매출, 판매 건수, 평균 단가를 함께 표시합니다.
- **매출 목록 테이블**: 구매일 / 매출유형 / 유형 / 상품명 / 구매자 / 담당자 / 금액 / 결제수단 / 현금·카드·마일리지 / 미수금 / 상태 등을 컬럼으로 제공합니다. 상품명 클릭 시 상품 상세로, 구매자 클릭 시 회원 상세로 이동합니다.
- **하단 요약 바**: 현재 필터 기준의 총매출 / 순매출 / 현금 / 카드 / 환불 / 미납금 / 할인 합계를 한 줄로 표시합니다.
- **매출 상세 모달**: 테이블 행을 클릭하면 결제일·상태·구매 정보·금액 정보·결제 내역·카드 승인 정보·메모를 담은 모달이 열립니다. 모달 하단의 "회원 상세 보기" 버튼으로 해당 회원 페이지로 이동할 수 있습니다.
- **신규 결제 버튼**: POS 판매 화면(SCR-S002)으로 바로 이동합니다.
- **엑셀 내보내기**: 현재 필터가 적용된 전체 매출 내역을 엑셀 파일로 내려받습니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 엑셀 내보내기, 신규 결제(POS) 이동, 모든 필터·탭 사용 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 엑셀 내보내기, 신규 결제(POS) 이동, 모든 필터·탭 사용 |
| FC | 담당 회원 결제 처리 가능 | 신규 결제(POS) 이동, 조회 및 필터 사용 |
| 트레이너 / 스태프 | 소속 지점 매출 조회 | 조회·필터·탭 전환만 가능 (처리 불가) |

## 주요 UX 흐름

1. 사이드바에서 "매출관리 > 매출 현황"을 클릭해 진입합니다.
2. 상단 지표 카드로 이번 달 순매출·결제수단별 합계·미수금 현황을 파악합니다.
3. 기간 프리셋(오늘/이번 주/이번 달) 또는 날짜 직접 입력으로 조회 기간을 설정합니다.
4. 검색창에 구매자 이름이나 상품명을 입력하거나, 유형·상태 필터를 적용해 원하는 거래를 찾습니다.
5. 매출유형 탭(신규/재등록 등)을 선택해 특정 유형만 볼 수 있습니다.
6. 분석 탭을 전환해 기간별·상품별·결제수단별·담당자별 집계를 확인합니다.
7. 담당자별 탭에서는 신규/재등록 매출 분리를 확인해 신규 획득과 기존 회원 유지 성과를 구분합니다.
8. 테이블 행을 클릭하면 해당 거래의 상세 정보를 모달로 확인합니다.
9. 필요 시 "신규 결제(POS)" 버튼으로 현장 판매 화면으로 이동합니다.
10. 엑셀 버튼으로 현재 조회 결과를 파일로 저장합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 로딩 중 | 데이터를 불러오는 동안 테이블 영역에 스켈레톤(뼈대 화면)을 표시합니다. |
| 정상 목록 | 조건에 맞는 거래가 1건 이상일 때 테이블과 하단 요약 바를 표시합니다. |
| 검색 결과 없음 | 필터·검색 적용 후 결과가 0건일 때 "검색 결과가 없습니다" 안내를 표시합니다. |
| 전체 데이터 없음 | 해당 기간에 매출 데이터가 없을 때 빈 화면 안내를 표시합니다. |
| 오류 | 데이터 조회에 실패하면 오류 메시지 알림(토스트)을 표시합니다. |
| 매출 상세 모달 열림 | 테이블 행 클릭 시 상세 정보 모달이 화면 위에 나타납니다. |

## 데이터 매핑

| 항목 | 외부 템플릿 | 셀 / 컬럼 | 산출 기준 |
|---|---|---|---|
| 담당자별 신규/재등록 매출 | `[매출종합]` | 회원권·락카 K열 `담당자`, 강습·골프 K열 `등록강사`, E열 `구분/유형`, F열 `결제금액` | 판매 귀속 담당자별로 `신규`, `재등록`, 기타 유형 매출을 분리 합산 |

## 비즈니스 규칙

- 담당자별 집계의 담당자는 환불 처리자가 아니라 원매출 판매 귀속 담당자입니다.
- 신규/재등록 분리 매출은 `[매출종합]`의 E열 `구분/유형` 값이 `신규`, `재등록`인 거래를 기준으로 산출합니다.
- 강습·골프 매출은 K열 `등록강사`를 판매 귀속 담당자로 사용하고, 회원권·락카 매출은 K열 `담당자`를 사용합니다.

## 관련 다이어그램

- 매출유형(신규/재등록/휴면복귀/종목추가/업그레이드) 분기 흐름
- 탭별 데이터 집계 로직 흐름 (매출 내역 / 기간별·상품별·결제수단별·담당자별 집계 / 환불·미납)
- 매출 상세 모달 전환 흐름
- 계정 권한별 접근 분기 흐름
