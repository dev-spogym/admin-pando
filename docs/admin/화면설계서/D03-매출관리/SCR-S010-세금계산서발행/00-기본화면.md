> **퍼블리싱 미구현**: 이 화면은 기획·설계 단계이며 아직 화면이 구현되지 않았습니다.

## 화면 목적

법인 회원 또는 사업자 회원에게 세금계산서를 발행하고, 발행 이력을 조회·관리하는 화면입니다. 세금계산서 발행이 필요한 거래를 선택해 발행 처리하고, 발행된 세금계산서의 상태(발행 완료·전송 완료·오류)를 추적합니다. 회계 마감 및 세무 신고에 필요한 발행 이력을 한 곳에서 관리할 수 있습니다.

## 주요 구성요소

- **요약 지표 카드 (3개)**: 이번 달 발행 건수 / 이번 달 발행 총액 / 미발행 대기 건수를 상단에 표시합니다.
- **발행 탭 / 이력 탭**: 발행 탭에서 세금계산서 발행을 처리하고, 이력 탭에서 기발행된 세금계산서 목록을 조회합니다.
- **발행 대상 조회**: 세금계산서 발행이 필요한 결제 건을 기간·상태 필터로 조회합니다.
- **세금계산서 발행 폼**: 공급받는 자의 사업자번호·상호·대표자·업태·종목·이메일을 입력하고, 공급 품목(상품명·수량·단가·금액)을 확인한 뒤 발행합니다.
- **발행 이력 테이블**: 발행일 / 공급받는 자 / 공급가액 / 부가세 / 합계 / 상태 컬럼을 제공합니다. 상태 배지로 발행 완료·전송 완료·오류를 구분합니다.
- **이메일 전송**: 발행 완료된 세금계산서를 공급받는 자 이메일로 전송합니다.
- **엑셀 내보내기**: 발행 이력 목록을 엑셀 파일로 내려받습니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 세금계산서 발행, 이력 조회, 이메일 전송, 엑셀 내보내기 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 세금계산서 발행, 이력 조회, 이메일 전송, 엑셀 내보내기 |
| FC | 담당 회원 결제 처리 가능 | 이력 조회만 가능 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

## 주요 UX 흐름

1. 사이드바에서 "매출관리 > 세금계산서 발행"을 클릭해 진입합니다.
2. 상단 지표 카드로 이번 달 발행 현황과 미발행 대기 건수를 파악합니다.
3. 발행 탭에서 발행 대상 결제 건을 선택합니다.
4. 발행 폼에서 공급받는 자 정보와 품목 내역을 확인·입력하고 발행 버튼을 누릅니다.
5. 발행 완료 후 이메일 전송 버튼으로 상대방에게 세금계산서를 전달합니다.
6. 이력 탭에서 기발행 목록을 조회하고 상태를 확인합니다.
7. 엑셀 버튼으로 이력을 파일로 저장합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 로딩 중 | 데이터를 불러오는 동안 테이블 영역에 로딩 상태를 표시합니다. |
| 발행 대기 | 세금계산서를 발행할 수 있는 결제 건이 있을 때 목록을 표시합니다. |
| 발행 완료 | 세금계산서가 발행된 건의 상태를 "발행 완료" 배지로 표시합니다. |
| 전송 완료 | 이메일 전송까지 완료된 건의 상태를 "전송 완료" 배지로 표시합니다. |
| 오류 | 발행 또는 전송 중 오류 발생 시 "오류" 배지와 오류 안내를 표시합니다. |
| 데이터 없음 | 발행 대상 또는 발행 이력이 없을 때 빈 상태 안내를 표시합니다. |

## 관련 다이어그램

- 발행 상태(발행 완료·전송 완료·오류) 분기 흐름
- 계정 권한별 접근 분기 흐름
