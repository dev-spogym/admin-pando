## 목적

회원 또는 직원이 환불을 요청하는 첫 번째 단계 팝업입니다. 환불 대상 매출 건과 요청 사유를 입력하여 환불 요청을 접수합니다. 실제 환불 처리는 담당자 확인 후 DLG-S013(환불 처리)에서 별도로 진행됩니다.

## 구성요소

| 구성요소 | 내용 |
|---|---|
| 환불 대상 매출 | 환불을 요청할 매출 건 정보 (날짜, 상품명, 결제 금액) |
| 요청자 구분 | 요청자가 회원인지 직원인지 구분 표시 |
| 환불 요청 금액 | 전체 환불 또는 부분 환불 금액 입력 |
| 환불 요청 사유 | 사전 정의된 사유 목록에서 선택 또는 직접 입력 |
| 추가 요청 사항 | 요청자가 전달하고 싶은 추가 내용 입력 (선택) |
| 요청 접수 버튼 | 환불 요청을 시스템에 접수 |
| 취소 버튼 | 요청을 취소하고 이전 화면으로 복귀 |

## 언제 표시되나요

- 회원이 앱 또는 키오스크에서 환불 요청 버튼을 눌렀을 때
- 직원이 SCR-S007(환불 관리) 화면에서 새 환불 요청 등록 버튼을 클릭했을 때
- DLG-S001(매출 상세)에서 환불 요청 버튼을 클릭했을 때

## 주요 흐름

1. 환불을 요청할 매출 건을 선택합니다.
2. 환불 요청 다이얼로그가 열립니다.
3. 환불 요청 금액(전체 또는 부분)과 사유를 입력합니다.
4. 요청 접수 버튼을 눌러 요청을 시스템에 등록합니다.
5. 요청이 접수되면 SCR-S007(환불 관리) 목록에 미처리 상태로 등록됩니다.
6. 담당자가 검토 후 DLG-S013(환불 처리)에서 실제 환불을 진행합니다.

## 관련 다이어그램

```
[DLG-S001 매출 상세] 또는 [SCR-S007 환불 관리]
        │ 환불 요청 클릭
        ▼
[DLG-S015 환불 요청]
        │              │
        │ 취소          │ 요청 접수
        ▼              ▼
[이전 화면]    [SCR-S007 환불 관리] (미처리 목록에 추가)
                        │ 담당자 처리 클릭
                        ▼
               [DLG-S013 환불 처리]
```
