## 목적

미수금 또는 할부 미납 건에 대해 납입 처리를 확인하고 완료합니다. 실제 금액 수령이 이루어진 시점에 직원이 시스템에 납입 완료를 기록하여 미납 현황이 정확하게 유지되도록 합니다.

## 구성요소

| 구성요소 | 내용 |
|---|---|
| 납입 대상 정보 | 회원 이름, 상품명, 해당 납입 회차 |
| 납입 금액 | 이번 회차에 납입해야 할 금액 |
| 실수령 금액 입력 | 실제 수령한 금액 입력 (기본값: 납입 예정 금액) |
| 수령 일시 | 납입 처리 일시 (기본값: 현재 날짜·시각) |
| 결제 수단 선택 | 납입 방법 선택 (카드, 현금, 계좌이체 등) |
| 처리 확인 버튼 | 납입 처리를 최종 확정 |
| 취소 버튼 | 처리 없이 이전 화면으로 복귀 |

## 언제 표시되나요

- DLG-S007(할부 상세)에서 미납 회차를 선택하고 납입 처리 버튼을 눌렀을 때
- SCR-S008(미수금 관리) 목록에서 납입 처리 버튼을 선택했을 때

## 주요 흐름

1. 할부 상세 또는 미수금 목록에서 납입 처리 버튼을 선택합니다.
2. 납입 처리 다이얼로그가 열리며 해당 회차 정보가 표시됩니다.
3. 실수령 금액, 수령 일시, 결제 수단을 확인하거나 수정합니다.
4. 처리 확인 버튼을 눌러 납입을 완료합니다.
5. 처리 완료 후 DLG-S007(할부 상세) 또는 미수금 목록으로 돌아가며 납입 상태가 갱신됩니다.

## 관련 다이어그램

```
[DLG-S007 할부 상세] 또는 [SCR-S008 미수금 관리]
        │ 납입 처리 클릭
        ▼
[DLG-S008 납입 처리]
        │              │
        │ 취소          │ 처리 확인
        ▼              ▼
[이전 화면]         [이전 화면] (납입 상태 갱신)
```
