> **퍼블리싱 현황**: 퍼블리싱 완료 / 데이터 미연동. 현재 UI는 목업·임시 데이터 기준이며 실데이터 연동은 후속 단계입니다.

## 화면 목적

완료된 결제 건에 대해 전체 취소 또는 일부 금액 환불을 처리하는 화면입니다. 단순 변심·결제 오류·서비스 불만 등의 사유로 환불 요청이 들어왔을 때, 담당자가 해당 결제 건을 조회하고 전체 취소 또는 부분 환불 금액을 지정해 처리합니다. 처리된 내역은 환불 관리 화면(SCR-S007)에 자동으로 반영됩니다.

## 주요 구성요소

- **결제 건 검색**: 회원명·상품명·결제일로 환불 처리할 결제 건을 조회합니다.
- **결제 내역 확인**: 선택한 결제 건의 결제일·상품명·결제금액·결제수단·결제자 정보를 확인합니다.
- **처리 유형 선택**: "전체 취소"와 "부분 환불" 중 하나를 선택합니다.
  - **전체 취소**: 결제 금액 전액을 환불 처리합니다. 사유 입력이 필요합니다.
  - **부분 환불**: 환불할 금액을 직접 입력합니다. 원래 결제 금액을 초과할 수 없습니다. 사유 입력이 필요합니다.
- **환불 사유 입력**: 단순 변심·결제 오류·회원 요청·중복 결제·서비스 불만 등 프리셋 사유를 선택하거나 직접 입력합니다.
- **환불 수단 확인**: 원래 결제수단(카드·현금·계좌이체·마일리지)에 따라 환불 방법을 안내합니다.
- **처리 확인 모달**: 환불 금액·사유·수단을 최종 확인하고 "처리 완료" 버튼으로 확정합니다.
- **처리 완료 안내**: 환불 처리가 완료되면 완료 메시지와 함께 환불 관리 화면으로 이동하는 버튼을 제공합니다.
- **처리 이력**: 해당 결제 건에 대해 이전에 처리된 환불 이력이 있으면 하단에 목록으로 표시합니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 전체 취소·부분 환불 처리, 결제 건 조회 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 전체 취소·부분 환불 처리, 결제 건 조회 |
| FC | 담당 회원 결제 처리 가능 | 담당 회원 결제 건 조회, 환불 요청 신청 (최종 처리는 매니저 이상) |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

## 주요 UX 흐름

1. 사이드바에서 "매출관리 > 결제 취소 / 부분 환불"을 클릭해 진입합니다.
2. 회원명·상품명·결제일로 환불 처리할 결제 건을 검색합니다.
3. 해당 결제 건을 선택해 결제 내역(금액·수단·일자)을 확인합니다.
4. "전체 취소" 또는 "부분 환불"을 선택합니다.
5. 부분 환불의 경우 환불 금액을 직접 입력합니다.
6. 환불 사유를 프리셋에서 선택하거나 직접 입력합니다.
7. 환불 수단을 확인하고 "처리 확인" 버튼을 누릅니다.
8. 최종 확인 모달에서 내역을 검토하고 "처리 완료"를 누릅니다.
9. 완료 후 환불 관리 화면(SCR-S007)으로 이동하거나 계속 처리합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 초기 상태 | 결제 건이 선택되지 않은 상태. 검색 영역만 활성화됩니다. |
| 결제 건 선택됨 | 결제 내역이 표시되고 처리 유형·사유 입력 영역이 활성화됩니다. |
| 전체 취소 선택 | 환불 금액이 원래 결제 금액 전액으로 자동 입력됩니다. |
| 부분 환불 선택 | 환불 금액 입력란이 활성화됩니다. 원래 금액 초과 입력 시 경고를 표시합니다. |
| 처리 확인 모달 열림 | 최종 내역 확인 모달이 표시됩니다. 처리 중에는 버튼이 비활성화됩니다. |
| 처리 완료 | 완료 메시지와 후속 이동 버튼이 표시됩니다. |
| 이미 취소된 건 | 이미 전액 환불된 결제 건 선택 시 "이미 취소된 결제 건입니다" 안내를 표시합니다. |
| 오류 | 처리 실패 시 오류 메시지 알림(토스트)을 표시합니다. |

## 관련 다이어그램

- 전체 취소 vs. 부분 환불 분기 흐름
- 환불 금액 유효성 검사 흐름 (원래 금액 초과 방지)
- 처리 확인 모달 흐름
- 환불 처리 완료 후 환불 관리 화면 연동 흐름
- 계정 권한별 접근 분기 흐름
