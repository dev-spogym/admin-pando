## 목적

결제를 최종 진행하기 전 구매자, 상품, 금액 정보를 한 번 더 검토합니다. 실수로 인한 잘못된 결제를 방지하고, 직원과 고객 모두 동의한 상태에서 결제가 이루어지도록 합니다.

## 구성요소

| 구성요소 | 내용 |
|---|---|
| 구매자 정보 | 선택된 회원 이름 및 연락처 |
| 상품 목록 | 결제 예정 상품명, 수량, 단가 |
| 할인 내역 | 적용된 할인 항목 및 할인 금액 |
| 최종 결제 금액 | 할인 적용 후 실제 결제될 금액 (크게 강조) |
| 결제 수단 | 선택된 결제 방법 (카드·현금·계좌이체 등) |
| 결제 확인 버튼 | 내용 동의 후 결제를 최종 실행 |
| 취소 버튼 | 결제를 중단하고 POS 화면으로 복귀 |

## 언제 표시되나요

- SCR-S002(POS 판매) 또는 SCR-S003(결제 처리) 화면에서 결제 진행 버튼을 눌렀을 때
- 상품·금액·결제 수단 설정이 완료된 상태에서 최종 확인 단계로 진입할 때

## 주요 흐름

1. 상품과 결제 수단 설정을 마친 후 결제 진행 버튼을 누릅니다.
2. 결제 확인 다이얼로그가 열리며 최종 내역이 표시됩니다.
3. 직원이 내용을 확인합니다.
4. 동일 회원·상품의 중복 결제가 감지되면 DLG-S004(중복 결제 경고)로 이동합니다.
5. 이상이 없으면 결제 확인 버튼을 눌러 결제를 완료합니다.
6. 취소 버튼을 누르면 결제 없이 POS 화면으로 돌아갑니다.

## 관련 다이어그램

```
[SCR-S002 POS 판매]
        │ 결제 진행 클릭
        ▼
[DLG-S003 결제 확인]
        │                    │
        │ 중복 감지           │ 확인
        ▼                    ▼
[DLG-S004 중복결제경고]   결제 완료
```
