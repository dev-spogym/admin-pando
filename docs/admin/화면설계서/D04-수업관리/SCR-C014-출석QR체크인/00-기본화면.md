> **퍼블리싱 현황**: 퍼블리싱 완료 / 데이터 미연동. 현재 UI는 목업·임시 데이터 기준이며 실데이터 연동은 후속 단계입니다.

## 화면 목적

회원이 QR 코드를 스캔해 수업 출석을 직접 처리할 수 있도록 하는 화면입니다. 강사나 스태프가 일일이 출석 처리하는 대신, 회원이 수업 시작 전 QR 체크인 화면에서 자신의 QR 코드를 스캔해 자동으로 출석 등록이 완료됩니다. 실시간 출석 현황도 이 화면에서 확인합니다.

## 주요 구성요소

- **QR 스캔 영역**: 카메라를 통한 QR 코드 스캔 UI (또는 태블릿 설치형)
- **수업 선택**: 현재 진행 중이거나 곧 시작할 수업 목록에서 체크인 대상 수업 선택
- **실시간 출석 현황 패널**:
  - 총 예약 인원, 출석 완료 인원, 미출석 인원
  - 출석한 회원 목록 (이름, 체크인 시간)
- **수동 체크인 버튼**: QR 스캔 실패 시 회원명 또는 연락처 검색으로 수동 출석 처리
- **체크인 완료 피드백**: 스캔 성공 시 회원명과 수업명 표시, 성공 사운드/진동 피드백

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 센터장 / 매니저 | 전체 수업 QR 체크인 화면 | 수업 선택, 수동 체크인 처리 |
| 트레이너 | 담당 수업 QR 체크인 화면 | 담당 수업 체크인 운영 |
| FC / 스태프 | 전체 수업 QR 체크인 화면 | 수업 선택, 수동 체크인 처리 |

## 주요 UX 흐름

1. 프런트 데스크 태블릿 또는 관리자 기기에서 QR 체크인 화면 진입
2. 현재 또는 곧 시작할 수업 선택
3. 회원이 QR 코드를 스캔 → 출석 자동 등록 + 성공 피드백 표시
4. QR 스캔 불가 회원은 수동 체크인으로 처리
5. 실시간 출석 현황 패널에서 출석 완료 인원 모니터링
6. 수업 종료 후 미출석 회원 자동 노쇼 처리 (정책에 따라)

## 화면 상태

| 상태 | 설명 |
|---|---|
| 대기 중 | QR 스캔 대기 상태, 카메라 활성 |
| 스캔 성공 | 회원명 + 수업명 표시, 출석 등록 완료 |
| 스캔 실패 | 인식 오류 안내, 재시도 또는 수동 처리 안내 |
| 수업 없음 | 현재 진행 예정 수업 없음 안내 |
| 만석 처리 | 정원 초과 시 체크인 제한 안내 |
