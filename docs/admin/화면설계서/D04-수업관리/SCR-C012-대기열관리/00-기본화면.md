> **퍼블리싱 미구현**: 이 화면은 기획 완료 상태이나 아직 개발·퍼블리싱이 진행되지 않았습니다.

## 화면 목적

만석이 된 그룹 수업에 대해 대기 신청한 회원 목록을 관리하는 화면입니다. 예약 취소 등으로 자리가 생기면 대기 순번에 따라 자동으로 예약을 배정하거나, 관리자가 수동으로 배정할 수 있습니다. 인기 수업의 공정한 자리 배분을 위해 사용합니다.

## 주요 구성요소

- **수업 선택 드롭다운**: 대기열을 조회할 수업 선택
- **대기 신청자 목록**:
  - 대기 순번, 회원명, 연락처, 대기 신청 일시
  - 이용권 유효 여부 표시
  - 자동 배정 여부 (자동 배정 동의 / 알림만)
- **자동 배정 설정**: 자리 발생 시 자동 배정 활성화 여부 토글
- **수동 배정 버튼**: 특정 대기자를 관리자가 직접 예약 확정 처리
- **대기 취소 버튼**: 대기 신청 취소 처리
- **알림 발송 버튼**: 자리 생겼음을 대기자에게 수동으로 알림 발송

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 센터장 / 매니저 | 전체 수업 대기열 목록 | 수동 배정, 대기 취소, 자동 배정 설정 |
| 트레이너 | 담당 수업 대기열 조회 | 조회만 가능 |
| FC / 스태프 | 전체 수업 대기열 조회 | 수동 배정, 알림 발송 |

## 주요 UX 흐름

1. 사이드바 > 수업관리 > 대기열 관리 진입
2. 대기열 확인할 수업 선택
3. 대기자 목록 및 순번 확인
4. 자리 발생 시 자동 배정 활성화 상태면 순번 1번에게 자동 예약 처리
5. 자동 배정 비활성 상태면 관리자가 "수동 배정" 버튼으로 직접 처리
6. 배정 후 해당 회원에게 예약 확정 알림 자동 발송

## 화면 상태

| 상태 | 설명 |
|---|---|
| 대기자 없음 | 해당 수업 대기 신청자 없음 안내 |
| 대기자 있음 | 순번별 대기자 목록 표시 |
| 자동 배정 활성 | 자리 발생 시 즉시 자동 처리 안내 |
| 배정 처리 중 | 배정 처리 중 로딩 표시 |
