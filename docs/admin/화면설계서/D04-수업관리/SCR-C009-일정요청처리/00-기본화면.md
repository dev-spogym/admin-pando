## 화면 목적

회원이 앱에서 요청한 수업 변경·취소 요청을 관리자가 검토하고 수락 또는 거절하는 화면입니다. 회원이 직접 수업을 취소하거나 다른 일정으로 변경하고 싶을 때 요청을 보내면, 센터 관리자가 이 화면에서 일괄 처리합니다.

## 주요 구성요소

- **요청 목록 테이블**:
  - 요청 유형 (변경 / 취소)
  - 회원명, 연락처
  - 원래 수업 정보 (수업명, 일시, 강사)
  - 요청 내용 (변경 시: 희망 변경 일시, 취소 시: 취소 사유)
  - 요청 일시
  - 처리 상태 (대기 / 수락 / 거절)
- **요청 유형 필터**: 전체 / 변경 요청 / 취소 요청
- **처리 상태 필터**: 전체 / 대기 / 처리 완료
- **수락 버튼**: 요청 수락 → 수업 일정 자동 변경 또는 취소 처리
- **거절 버튼**: 거절 사유 입력 후 회원에게 안내
- **대안 일정 제시 버튼**: 취소 요청 시 대안 일정 제안(DLG-C016) 팝업

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 센터장 / 매니저 | 전체 회원 요청 목록 | 수락·거절·대안 일정 제시 |
| 트레이너 | 담당 수업 관련 요청 | 수락·거절 (담당 수업 한정) |
| FC / 스태프 | 전체 요청 목록 | 조회만 가능 |

## 주요 UX 흐름

1. 사이드바 > 수업관리 > 일정 요청 처리 진입
2. 대기 중인 요청 필터로 미처리 요청 우선 확인
3. 요청 내용 검토 후 수락 또는 거절 선택
4. 취소 요청에 대해 대안 일정을 제시하려면 "대안 제시" 버튼 → DLG-C016
5. 수락 시 수업 일정 자동 반영, 회원에게 알림 발송
6. 거절 시 거절 사유 입력 후 저장, 회원에게 거절 안내 알림 발송

## 화면 상태

| 상태 | 설명 |
|---|---|
| 대기 요청 있음 | 미처리 요청 상단에 건수 배지 표시 |
| 대기 요청 없음 | "처리할 요청이 없습니다" 안내 |
| 처리 완료 목록 | 수락·거절 완료 요청 이력 조회 |
| 처리 중 | 수락/거절 처리 중 버튼 비활성 |
