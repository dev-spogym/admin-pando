## 화면 목적

전체 회원의 체성분 측정 현황을 통합 관리한다. InBody 등 측정 장비 연동, 파일 업로드, 수기 입력 세 가지 채널로 수집된 데이터를 검수하고 확정 처리한다.

## 주요 구성요소

### 상단 현황 카드

| 항목 | 설명 |
|------|------|
| 오늘 측정 건수 | 금일 체성분 측정 완료 건수 |
| 검수 대기 | 매칭 실패·중복·이상치로 검수 필요한 건수 |
| 미확정 | 수집됐지만 아직 확정 처리되지 않은 건수 |
| 연동 장비 | 현재 온라인 상태인 InBody 장비 수 |

### 탭 구조

| 탭 | 설명 |
|----|------|
| 측정 결과 | 확정된 체성분 결과 목록 및 상세 조회 |
| 업로드 센터 | 장비 파일(CSV/XLSX 등) 업로드 및 파싱 결과 확인 |
| 실시간 연동 | InBody API 수신 로그 확인 |
| 오류/검수 | 중복·회원 매칭 실패·값 이상치 건 검수 처리 |

### 측정 결과 탭 테이블

| 컬럼명 | 설명 |
|--------|------|
| 측정일시 | 체성분 측정 완료 시각 |
| 회원명 | 측정 회원 이름 |
| 체중 | 측정 체중 (kg) |
| 골격근량 | 골격근량 (kg) |
| 체지방률 | 체지방 비율 (%) |
| BMI | 체질량지수 |
| 입력 채널 | 수기 / 파일 업로드 / 실시간 API |
| 확정 여부 | 확정 / 미확정 배지 표시 |

### 오류/검수 탭 항목

| 항목 | 설명 |
|------|------|
| 오류 유형 | 회원 매칭 실패 / 중복 / 값 이상치 |
| 수집 시각 | 데이터 수신 시각 |
| 원본 값 | 수집된 원본 측정값 |
| 처리 | 회원 직접 매칭 / 무시 / 확정 처리 |

### 버튼

| 버튼 | 설명 |
|------|------|
| 수기 등록 | DLG-I003 팝업 호출, 수기 체성분 입력 |
| 파일 업로드 | 업로드 센터 탭으로 이동 또는 파일 선택 |
| 연동 로그 | 실시간 연동 탭으로 이동 |
| 확정 처리 | 검수 완료 항목을 확정 상태로 변경 |
| 회원 상세 이동 | 해당 행 회원 상세 화면으로 이동 |

## 계정별 차이

| 계정 유형 | 차이 |
|-----------|------|
| 센터 관리자 | 전체 탭 접근, 검수 처리, 수기 등록 가능 |
| 프런트 직원 | 측정 결과 조회 및 수기 등록 가능, 검수 처리는 관리자만 가능 |
| 강사 | 담당 회원의 측정 결과 조회만 가능 |

## 주요 UX 흐름

1. 화면 진입 → 측정 결과 탭 기본 표시
2. 검수 대기 카드 클릭 → 오류/검수 탭 자동 이동
3. 수기 등록 버튼 → DLG-I003 팝업 → 회원 선택 → 수치 입력 → 저장
4. 파일 업로드 탭: 파일 선택 → 업로드 → 파싱 결과 미리보기 → 확정 처리
5. 오류/검수 탭: 오류 항목 선택 → 회원 매칭 또는 무시 → 확정
6. 측정 결과 행 클릭 → 해당 회원 상세 화면으로 이동

## 화면 상태

| 상태 | 설명 |
|------|------|
| 기본(데이터 있음) | 측정 결과 테이블 정상 표시 |
| 검수 대기 있음 | 검수 탭 배지 수치 강조 |
| 장비 오프라인 | 연동 장비 카드 경고 표시 |
| 업로드 파싱 중 | 파일 처리 중 로딩 표시 |
| 빈 상태 | '측정 데이터가 없습니다.' 안내 |
