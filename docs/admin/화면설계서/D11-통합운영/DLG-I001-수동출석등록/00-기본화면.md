## 화면 목적

키오스크·QR 등 기기를 통한 출석이 불가한 상황에서 관리자(프런트 직원)가 직접 회원 또는 직원의 출석을 수동으로 등록한다. 별도 기기 없이 시스템에서 직접 출석 처리가 가능하다.

## 주요 구성요소

### 팝업 구조

모달 팝업 형태로 통합 출석 관리 화면(SCR-I001) 위에 오버레이로 표시된다.

### 입력 항목

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| 구분 | 필수 | 회원 / 직원 선택 |
| 이름 또는 번호 검색 | 필수 | 회원명, 회원번호, 사번으로 검색 |
| 출석 시각 | 필수 | 기본값: 현재 시각 (수정 가능) |
| 비고 | 선택 | 수동 등록 사유 메모 |

### 버튼

| 버튼 | 설명 |
|------|------|
| 등록 | 입력값 유효성 검사 후 출석 저장 |
| 취소 | 팝업 닫기, 입력값 초기화 |

## 계정별 차이

| 계정 유형 | 차이 |
|-----------|------|
| 센터 관리자 | 회원·직원 모두 수동 등록 가능 |
| 프런트 직원 | 회원 수동 등록 가능, 직원 등록은 관리자만 가능 |
| 강사 | 수동 출석 등록 불가 |

## 주요 UX 흐름

1. 통합 출석 관리 화면 → '수동 출석 등록' 버튼 클릭 → 팝업 열림
2. 구분(회원/직원) 선택
3. 이름 또는 번호 입력 → 자동완성 목록에서 선택
4. 출석 시각 확인 (필요 시 수정)
5. 비고 입력 (선택)
6. 등록 버튼 클릭 → 성공 시 팝업 닫힘 + 출석 목록 즉시 갱신
7. 실패 시 인라인 오류 메시지 표시

## 화면 상태

| 상태 | 설명 |
|------|------|
| 기본(입력 대기) | 입력 항목 비어있음, 등록 버튼 비활성 |
| 검색 중 | 자동완성 드롭다운 표시 |
| 검색 결과 없음 | '일치하는 회원/직원이 없습니다.' 메시지 |
| 입력 완료 | 등록 버튼 활성화 |
| 등록 성공 | 팝업 닫힘 + 토스트 '출석이 등록되었습니다.' |
| 등록 실패 | 인라인 오류 메시지 표시 |
