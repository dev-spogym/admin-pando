## 화면 목적

두 가지 버전의 메시지나 쿠폰 중 어느 것이 더 효과적인지 비교 실험하는 A/B 테스트를 설정하고 결과를 분석하는 화면입니다. 데이터 기반으로 마케팅 전략을 최적화합니다.

> **퍼블리싱 현황**: 퍼블리싱 완료 / 데이터 미연동. 현재 UI는 목업·임시 데이터 기준이며 실데이터 연동은 후속 단계입니다.

## 주요 구성요소

- **A/B 테스트 목록**: 테스트명 / 유형(메시지·쿠폰) / 기간 / 상태(준비·진행·완료) / 승리 버전 컬럼으로 구성됩니다.
- **테스트 등록 버튼**: DLG-079-001을 열어 새 A/B 테스트를 등록합니다.
- **결과 비교 패널**: A안과 B안의 도달 수 / 클릭율 / 전환율 / 매출 기여를 나란히 비교합니다.
- **승리 버전 채택**: 결과 분석 후 더 효과적인 버전을 정식 캠페인에 적용합니다.
- **테스트 삭제**: DLG-079-002를 통해 삭제를 확인합니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 센터장 / 매니저 | 전체 A/B 테스트 목록 및 결과 | 테스트 생성·삭제, 결과 분석, 승리 버전 채택 |
| FC / 트레이너 / 스태프 | 접근 제한 | A/B 테스트 관리 불가 |

## 주요 UX 흐름

1. 사이드바 "마케팅 > A/B 테스트"로 진입합니다.
2. "테스트 등록"(DLG-079-001)으로 A안·B안의 내용과 대상 비율, 기간을 설정합니다.
3. 테스트가 시작되면 무작위로 분배된 수신자에게 각 버전이 발송됩니다.
4. 기간 종료 후 결과 비교 패널에서 두 버전의 성과를 분석합니다.
5. 효과가 우수한 버전을 선택해 정식 마케팅에 적용합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 데이터 미연동 | 화면은 표시되지만 테스트 성과·승리 버전 데이터는 목업 기준으로 표시됩니다. |
| 테스트 없음 | 등록된 A/B 테스트가 없을 때 생성 안내가 표시됩니다. |
| 진행 중 | 현재 실행 중인 테스트와 실시간 지표가 표시됩니다. |
| 분석 가능 | 테스트가 완료되어 결과를 비교할 수 있는 상태입니다. |

## 관련 다이어그램

- DLG-079-001 A/B 테스트 등록
- DLG-079-002 A/B 테스트 삭제 확인
