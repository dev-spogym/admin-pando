## 목적
특정 결제 건에 대해 환불을 처리할 때, 환불 금액과 사유를 입력하고 최종 확인을 받는 다이얼로그입니다.

## 구성요소
- 제목: "환불 처리"
- 원결제 정보 표시 (결제일, 결제 금액, 결제 수단)
- 환불 금액 입력 (기본값: 원결제 금액, 부분 환불 시 직접 입력)
- 환불 사유 입력 (드롭다운 + 직접 입력)
- 환불 후 잔액 표시 (부분 환불 시)
- 확인 버튼 (환불 실행)
- 취소 버튼 (다이얼로그 닫기)

## 언제 표시되나요
- 회원 상세 > 결제 탭에서 특정 결제 건의 환불 버튼을 클릭했을 때
- 이미 환불 처리된 건은 환불 버튼 비활성화

## 주요 흐름
1. 관리자가 환불 버튼 클릭 → 원결제 정보가 채워진 상태로 다이얼로그 표시
2. 환불 금액 확인 또는 수정, 사유 입력
3. 확인 → 환불 처리 완료, 다이얼로그 닫힘, 결제 목록 상태 갱신
4. 취소 → 다이얼로그 닫힘, 환불 없음
