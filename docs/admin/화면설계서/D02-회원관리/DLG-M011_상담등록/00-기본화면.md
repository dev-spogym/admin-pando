## 목적
회원과의 상담 내용을 체계적으로 기록하거나 수정하는 다이얼로그입니다. 상담 일시, 유형, 채널, 상태, 결과, 연결 매출, 후속조치를 입력하여 상담 이력을 관리합니다.

## 구성요소
- 제목: "상담 이력 추가" 또는 "상담 이력 수정"
- 상담 일시 (자동 입력, 수정 가능)
- 상담 유형 선택 (상담, OT, 체험, 재등록상담)
- 상담 채널 선택 (방문, 전화, 카카오톡, DM, SNS, 기타)
- 담당자 입력 (현재 로그인 관리자 기본값)
- 상태 선택 (예정, 완료, 취소, 노쇼)
- 결과 선택 (등록, 미등록, 보류)
- 연결 매출 선택 (선택 사항)
- 상담 내용 입력 (텍스트 영역)
- 후속조치 입력 (선택 사항)
- 저장 버튼
- 취소 버튼

## 언제 표시되나요
- 회원 상세 > 상담이력 탭에서 상담 추가 버튼을 클릭했을 때
- 기존 상담 이력의 수정 버튼을 클릭했을 때

## 주요 흐름
1. 관리자가 상담 추가 또는 수정 버튼 클릭 → 다이얼로그 표시
2. 상담 일시, 유형, 채널, 상태, 결과, 내용, 후속조치 입력
3. 필요 시 연결 매출 선택
4. 저장 버튼 클릭 → 상담 기록 저장 완료, 다이얼로그 닫힘, 목록 즉시 갱신
5. 취소 → 다이얼로그 닫힘, 저장 없음

## 데이터 및 로그 규칙

- 저장 대상은 Supabase `consultations`이다.
- 지점 컬럼은 `branchId`, 회원 컬럼은 `memberId`, 상담 일시는 `scheduledAt`/`completedAt`을 사용한다.
- 상담 채널은 `channel`에 저장한다.
- 상담 수정 성공 시 히스토리 로그에 `상담 이력 수정됨` 문구를 반드시 남긴다.
