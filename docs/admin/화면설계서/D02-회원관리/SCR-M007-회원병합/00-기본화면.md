> **퍼블리싱 현황**: 퍼블리싱 완료 / 데이터 미연동. 현재 UI는 목업·임시 데이터 기준이며 실데이터 연동은 후속 단계입니다.

## 화면 목적

동일인으로 판단되는 중복 회원 계정 두 개를 하나로 합치는 화면입니다. 연락처 변경이나 재등록 등의 이유로 같은 회원이 중복 등록된 경우, 양쪽 데이터를 하나의 주 계정으로 통합합니다. 출석 이력·결제 이력·상담 메모 등 중요한 이력 데이터는 병합 후에도 유지됩니다.

## 주요 구성요소

- **중복 회원 검색**: 이름·연락처·생년월일로 중복이 의심되는 회원을 검색합니다.
- **주 계정 지정**: 병합 결과로 남길 계정(주 계정)과 삭제될 계정(부 계정)을 지정합니다.
- **병합 전 정보 비교 화면**: 주 계정과 부 계정의 주요 정보(이름·연락처·이용권·등록일·최근 방문일)를 나란히 표시해 최종 확인을 돕습니다.
- **병합 항목 선택**: 이름, 연락처, 프로필 사진 등 일부 항목에서 어느 쪽 계정 값을 채택할지 선택할 수 있습니다.
- **병합 후 처리 안내**:
  - 부 계정의 출석 이력·결제 이력·상담 메모·체성분 기록은 주 계정으로 이전
  - 부 계정은 병합 처리 완료 후 비활성 상태로 전환 (즉시 삭제 아님)
- **최종 확인 다이얼로그**: 병합은 되돌리기 어려운 작업임을 명시하고 "병합" 입력 확인 후 처리.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 회원 병합 가능 | 병합 처리 전체 |
| 센터장 (owner) | 소속 지점 내 회원 병합 | 병합 처리 |
| 매니저 / FC / 트레이너 / 스태프 / 프런트 | 접근 불가 | 이 기능 사용 불가 |

## 주요 UX 흐름

1. 회원 목록 또는 회원 상세에서 "회원 병합" 메뉴로 진입합니다.
2. 중복으로 의심되는 회원을 검색하여 주 계정과 부 계정을 각각 지정합니다.
3. 나란히 비교된 두 계정의 정보를 확인하고, 병합 후 남길 정보를 선택합니다.
4. "병합" 버튼을 클릭하면 최종 확인 다이얼로그가 나타납니다. 확인 문구를 입력 후 처리합니다.
5. 병합 완료 후 주 계정의 회원 상세(SCR-M004)로 이동합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 초기 검색 | 두 계정 모두 미선택 상태, 검색 안내 표시 |
| 주 계정만 선택 | 주 계정은 지정됨, 부 계정 선택 대기 |
| 두 계정 모두 선택 | 비교 화면 표시, 병합 버튼 활성화 |
| 처리 중 | 병합 진행 중 버튼 비활성 + 로딩 표시 |
| 병합 완료 | 주 계정 상세 페이지로 이동 |
