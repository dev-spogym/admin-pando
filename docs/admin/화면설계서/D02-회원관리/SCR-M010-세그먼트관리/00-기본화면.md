> **퍼블리싱 현황**: 퍼블리싱 완료 / 데이터 미연동. 현재 UI는 목업·임시 데이터 기준이며 실데이터 연동은 후속 단계입니다.

## 화면 목적

공통된 특성이나 행동 패턴을 가진 회원들을 그룹(세그먼트)으로 묶어 타겟 마케팅과 맞춤 관리에 활용하는 화면입니다. "30일 이상 미방문 활성 회원", "이번 달 만료 예정 회원", "신규 등록 후 한 번도 방문하지 않은 회원" 등 다양한 조건 조합으로 세그먼트를 생성하고, 해당 그룹에 메시지 발송이나 혜택 제공 등의 액션을 연결합니다.

## 주요 구성요소

- **세그먼트 목록**: 생성된 세그먼트를 카드 형태로 표시. 각 카드에는 세그먼트 이름·설명·현재 해당 회원 수·생성일·마지막 업데이트 시각이 표시됩니다.
- **세그먼트 생성 버튼**: 새 세그먼트 만들기. 이름, 설명, 조건을 입력합니다.
- **조건 빌더**: AND / OR 조합으로 여러 조건을 추가합니다.
  - 회원 상태 (활성 / 홀딩 / 만료 / 미등록 등)
  - 최근 방문일 기준 (예: 30일 이상 방문 없음)
  - 이용권 만료일 기준 (예: 정책 step 대상, 만료 30일 이내 등 운영 조건)
  - 등록일 기준 (예: 최근 30일 내 등록)
  - 성별 / 연령대 / 회원구분 / 유입경로
  - 누적 결제 금액 범위
- **미리보기**: 조건 설정 중 "미리보기" 버튼을 누르면 현재 조건에 해당하는 회원 수와 샘플 명단이 표시됩니다.
- **자동 갱신 설정**: 세그먼트 조건을 매일·주간·월간 자동으로 재계산할지 설정합니다.
- **세그먼트 액션 연결**: 세그먼트 카드에서 "메시지 발송" 또는 "이용권 혜택 적용" 등의 액션을 바로 실행할 수 있는 버튼을 제공합니다.
- **세그먼트 수정·삭제**: 기존 세그먼트의 조건이나 이름을 변경하거나 삭제합니다.

## 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 브랜드 세그먼트 | 세그먼트 생성·수정·삭제, 액션 실행 |
| 센터장 (owner) | 소속 지점 세그먼트 | 세그먼트 생성·수정·삭제, 액션 실행 |
| 매니저 | 소속 지점 세그먼트 | 세그먼트 생성·수정, 액션 실행 (삭제 불가) |
| FC (피트니스 코치) | 소속 지점 세그먼트 조회 | 기존 세그먼트 기반 메시지 발송만 가능 |
| 트레이너 / 스태프 / 프런트 | 접근 불가 | 이 화면 진입 차단 |

## 주요 UX 흐름

1. 사이드바 "회원관리 > 세그먼트 관리"로 진입합니다.
2. 기존 세그먼트 목록을 확인하거나 "새 세그먼트 만들기"를 클릭합니다.
3. 세그먼트 이름과 설명을 입력하고, 조건 빌더로 필터 조건을 설정합니다.
4. "미리보기"로 현재 조건에 해당하는 회원 수를 확인합니다.
5. 자동 갱신 주기를 설정하고 저장합니다.
6. 저장된 세그먼트 카드에서 "메시지 발송" 등의 액션을 바로 실행합니다.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 세그먼트 없음 | 생성된 세그먼트가 없을 때 "세그먼트가 없습니다" 안내 + 생성 버튼 |
| 목록 조회 | 생성된 세그먼트 카드 목록 표시 |
| 조건 설정 중 | 조건 빌더 열린 상태, 조건 항목 추가·삭제 가능 |
| 미리보기 | 조건에 해당하는 회원 수 및 샘플 명단 표시 |
| 저장 중 | 세그먼트 저장 처리 중 로딩 표시 |
| 자동 갱신 중 | 조건 재계산 중 세그먼트 카드에 업데이트 중 표시 |
