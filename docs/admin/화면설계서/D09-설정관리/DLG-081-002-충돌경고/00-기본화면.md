## 화면 목적
권한 설정 저장 시 논리적으로 모순된 권한 조합이 감지되었을 때 표시되는 경고 다이얼로그입니다.
충돌 내용을 구체적으로 안내하고, 사용자가 수정하거나 그대로 저장할지 선택할 수 있습니다.

## 주요 구성요소
- "권한 설정 충돌 감지" 제목
- 충돌 항목 목록 (메뉴명과 충돌 이유 설명)
  - 예: "회원 목록 — 삭제 권한이 있지만 수정 권한이 없습니다."
  - 예: "매출 현황 — 쓰기 권한이 있지만 접근 권한이 없습니다."
- 충돌을 무시하고 저장 버튼
- 취소 버튼 (권한 설정 화면으로 돌아가 수정)

## 주요 UX 흐름
1. 저장 버튼 클릭 → 시스템이 권한 충돌 자동 감지
2. 충돌 발견 시 다이얼로그 표시 및 충돌 목록 나열
3. 취소 → 다이얼로그 닫힘, 권한 설정 화면에서 직접 수정
4. 무시하고 저장 → 충돌이 있는 상태 그대로 저장 완료

## 화면 상태
| 상태 | 설명 |
|---|---|
| 표시 | 충돌 목록과 선택지 표시 |
| 저장 처리 중 | 저장 버튼 클릭 후 처리 중 (버튼 비활성) |
