## 화면 목적

고장·이상이 발견된 운동 장비의 수리를 접수하고, 수리 완료 후 결과를 기록하는 다이얼로그입니다. 수리 접수부터 완료까지의 이력을 추적합니다.

## 주요 구성요소

- **다이얼로그 제목**: "수리 접수" (신규) 또는 "수리 완료 기록" (완료 처리)
- **고장 내용**: 발견된 고장 또는 이상 내용을 입력합니다.
- **수리 접수일**: 수리를 접수한 날짜를 입력합니다 (기본값: 오늘).
- **담당 수리업체/담당자**: 수리를 맡긴 업체 또는 담당자를 입력합니다.
- **예상 완료일**: 수리 완료 예정일을 입력합니다 (선택 항목).
- **수리 완료일** (완료 처리 시): 실제 수리가 완료된 날짜를 입력합니다.
- **수리 결과**: 수리 완료 내용을 기록합니다.
- **비용**: 수리 비용을 입력합니다 (선택 항목).
- **저장 버튼**: 수리 정보를 저장합니다.
- **취소 버튼**: 변경 없이 다이얼로그를 닫습니다.

## 주요 UX 흐름

1. 장비 행의 [수리 접수] 버튼 클릭 → 수리 접수 다이얼로그 열림.
2. 고장 내용·담당자·예상 완료일을 입력하고 [저장] → 장비 상태가 "수리중"으로 변경.
3. 수리 완료 후 [수리 완료 기록] 버튼 클릭 → 완료일·결과·비용 입력 → [저장] → 장비 상태 "정상"으로 복구.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 수리 접수 성공 | 장비 상태 "수리중" 변경 + 목록 갱신 |
| 수리 완료 기록 성공 | 장비 상태 "정상" 복구 + 수리 이력 기록 |
