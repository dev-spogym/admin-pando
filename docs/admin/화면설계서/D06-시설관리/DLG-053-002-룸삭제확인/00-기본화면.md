## 화면 목적

운동룸을 삭제하기 전 최종 의사를 확인하는 다이얼로그입니다. 삭제 시 해당 룸의 예약 슬롯 정보도 함께 삭제될 수 있음을 안내합니다.

## 주요 구성요소

- **다이얼로그 제목**: "운동룸 삭제"
- **안내 메시지**: "[룸명]을 삭제하시겠습니까? 삭제 후에는 복구할 수 없습니다."
- **삭제 버튼**: 위험(danger) 스타일로 표시됩니다.
- **취소 버튼**: 변경 없이 다이얼로그를 닫습니다.

## 주요 UX 흐름

1. 룸 카드의 [삭제] 버튼 클릭 → 다이얼로그 열림.
2. 삭제 대상 룸명을 확인합니다.
3. [삭제] 클릭 → 룸 정보 삭제 → "운동룸이 삭제되었습니다" 알림 + 카드 목록에서 제거.
4. [취소] 클릭 시 변경 없이 닫힘.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 삭제 성공 | "운동룸이 삭제되었습니다" 알림 + 카드 목록 갱신 |
| 삭제 실패 | "저장에 실패했습니다" 오류 알림 |
