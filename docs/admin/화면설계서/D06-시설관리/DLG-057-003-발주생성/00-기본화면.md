## 화면 목적

재고가 부족한 소모품에 대해 재발주를 요청하는 다이얼로그입니다. 발주 수량과 공급업체를 입력하면 발주서가 생성되고 담당자에게 알림이 발송됩니다.

## 주요 구성요소

- **다이얼로그 제목**: "발주 생성 — [품목명]"
- **현재 재고 표시**: 현재 재고 수량을 읽기 전용으로 표시합니다.
- **안전 재고 기준 표시**: 설정된 안전 재고 기준 수량을 표시합니다.
- **발주 수량 입력**: 주문할 수량을 입력합니다. 안전 재고 이상의 권장 수량이 기본값으로 표시됩니다.
- **공급업체 선택/입력**: 발주할 공급업체를 선택하거나 직접 입력합니다.
- **희망 납기일**: 납품 받기를 원하는 날짜를 입력합니다 (선택 항목).
- **비고**: 발주 시 특이사항을 입력합니다 (선택 항목).
- **발주서 생성 버튼**: 발주를 확정합니다.
- **취소 버튼**: 변경 없이 다이얼로그를 닫습니다.

## 주요 UX 흐름

1. 재고 부족 품목의 [발주 생성] 버튼 클릭 → 다이얼로그 열림.
2. 발주 수량과 공급업체를 확인·수정합니다.
3. [발주서 생성] 클릭 → 발주 상태로 변경 + 담당자 알림 발송.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 수량 미입력 | [발주서 생성] 버튼 비활성화 |
| 발주 생성 성공 | "발주가 생성되었습니다" 알림 + 품목 상태 "발주 대기"로 변경 |
