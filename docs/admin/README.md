# 기획자 최신통합본

## 목적

- `기획자`를 관리자 상세 원본으로 유지한다.
- `기획자2`의 전사 정합성 기준과 신규 운영 화면을 반영한다.
- 회원앱은 `회원앱2`를 정본으로 연결한다.

## 문서 우선순위

1. 관리자 상세 화면/기능: `기획자`
2. 상태, 권한, 자동화, 본사 정책 구조: `기획자2`
3. 회원앱 IA/화면/기능: `회원앱2`

## 권한/역할 적용 원칙

- 관리자웹 권한 키는 `superAdmin / primary / owner / manager / fc / staff / readonly`를 기준으로 해석한다.
- 관리자 기획서 본문에 보이는 `최고관리자`, `센터장`, `매니저`, `FC`, `스태프`는 위 권한 키의 운영 라벨 또는 예시 명칭으로 병기될 수 있다.
- `트레이너`, `골프강사`, `프런트`는 기본적으로 직원의 직무 또는 운영 주체를 뜻하며, 관리자웹 기본 권한 키와 동일 개념으로 해석하지 않는다.
- 실제 메뉴 접근 권한은 `D09-설정관리 > SCR-081 권한 설정` 또는 커스텀 역할 매핑 기준으로 결정한다.

## 이 폴더에 반영한 내용

- `기획자` 전체를 베이스로 복제
- `기획자2`의 전사 기준 문서 01~05 반영
- `기획자2`의 `운영용_기획문서` 반영
- `기획자2` 전용 신규 화면을 `기획자` ID 체계로 재번호화해 추가
- `관리자문서_통합_ID대응표.md` 포함

## 회원앱 연동 기준

- `화면설계서/D12-회원앱`은 보조 참조 영역이다.
- 회원앱의 최종 IA, 기능명세, 화면설계는 `J:\\MyProject\\외주\\외주\\기획서\\회원앱2`를 기준으로 본다.
- D12의 로컬 `SCR-MA-*` 문서는 관리자 문서에서 필요한 연결 참고용으로만 유지한다.

## 현재 통합 규칙

- 기존 `기획자` ID는 유지
- `기획자2`에만 있는 화면은 `기획자` 체계로 신규 번호 부여
- D12 신규 화면은 `MA` ID를 그대로 따른다
