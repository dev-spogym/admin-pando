# X16 — 선수익금 이연 → 월말 매출 인식

## 1. 시나리오 개요

이용권 결제 시 전액 선수익금(이연 수익)으로 적립 → 매일 또는 월말 크론잡이 실제 서비스 제공량에 따라 매출 인식(accrual) → 선수익금 잔액 감소 → 회계 반영하는 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 결제 시 선수익금 적립 / 매월 말 인식 크론잡 |
| 종료 조건 | 당월 매출 인식 완료 + 잔액 업데이트 |
| 참여 도메인 | 매출관리(D3) |

## 2. 전제조건

- 이용권 결제 시 선수익금 계정이 자동 생성됨
- 이용권 기간/횟수 정보가 정확히 등록되어 있음
- 매출 인식 정책이 설정되어 있음 (일할 계산 또는 횟수 기준)

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 스케줄러 | 매월 말 매출 인식 크론잡 |
| 회계모듈 | 선수익금/매출 계정 관리 |

## 4. 시퀀스 다이어그램

```mermaid

    participant SCH as 스케줄러
    participant API as CRM API
    participant DB as DB
    participant ACC as 회계모듈

    Note over DB,ACC: [결제 시점 — 선수익금 적립]
    Note over DB: POST /sales 처리 시 자동 실행
    API->>DB: 1. sales {amount=500000, status=PAID} %% API->>ACC: 2. 선수익금 계정 적립 {amount=500000, membership_id} %% ACC->>DB: 3. deferred_revenues {membership_id, total=500000, recognized=0, remaining=500000}
    DB-->>ACC: 4. deferred_id
    ACC-->>API: 5. OK

    Note over SCH,DB: [매월 말 매출 인식 크론잡]
    SCH->>API: 6. 선수익금 인식 트리거 {year_month} %% API->>DB: 7. deferred_revenues WHERE remaining %% DB-->>API: 8. 미인식 선수익금 목록

    loop 각 이용권별
        API->>DB: 9. memberships WHERE id=? (기간/횟수 확인) %% DB-->>API: 10. 이용권 정보 (start_date, expiry_date, total_sessions, used_sessions)
        API->>API: 11. 당월 인식 금액 계산 (일할 또는 횟수 비례)
        Note over API: 일할: total × (당월 일수 / 전체 일수)
        API->>ACC: 12. 매출 인식 처리 {deferred_id, recognize_amount, month} %% ACC->>DB: 13. revenue_recognitions {deferred_id, month, amount}
        ACC->>DB: 14. deferred_revenues SET recognized=recognized+amount, remaining=remaining-amount
        DB-->>ACC: 15. OK
        ACC-->>API: 16. 인식 완료 {recognize_amount}
    end

    API->>DB: 17. monthly_revenue_summary {month, total_recognized, total_deferred_remaining} %% DB-->>API: 18. summary_id
    API-->>SCH: 19. 처리 완료 {count, total_recognized}

    Note over API,DB: [SCR-S006 선수익금 조회]
    API->>DB: 20. deferred_revenues + revenue_recognitions for report %% DB-->>API: 21. 선수익금 현황 데이터
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 3 | deferred_revenues | 결제와 동시에 생성. total=결제금액, recognized=0 초기화 |
| 11 | 당월 인식 금액 계산 | 기간제: 일할 계산. 횟수제: 사용횟수/총횟수 × 결제금액 |
| 13 | revenue_recognitions | 월별 인식 내역 상세 기록. 회계 감사 대비 |
| 17 | monthly_revenue_summary | 월별 집계 요약. SCR-S006 대시보드 데이터 소스 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 이용권 중도 환불 | 잔여 선수익금 즉시 취소 처리, deferred_revenues 상태 REFUNDED |
| 이용권 홀딩 기간 | 홀딩 일수만큼 인식 제외 (기간 연장 반영) |
| 계산 오류 | 인식 금액 합계가 결제금액 초과 방지 검증 |
| 재계산 요청 | 매니저가 특정 월 재계산 요청 가능 |
