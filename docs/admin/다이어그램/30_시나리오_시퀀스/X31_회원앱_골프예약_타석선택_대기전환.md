# X31 — 회원앱 골프 예약 → 타석 선택 → 대기 전환

## 1. 시나리오 개요

회원이 골프 수업 상세에서 사용 가능한 타석을 선택해 예약하거나, 만석일 경우 대기 예약으로 전환되는 흐름.

## 2. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor M as 회원
    participant APP as 회원앱
    participant API as CRM API
    participant DB as DB
    participant N as 알림서비스

    M->>APP: 1. MA-121에서 [타석 선택]
    APP->>API: 2. GET /classes/{id}/golf-booths
    API->>DB: 3. SELECT lesson_schedules same slot
    DB-->>API: 4. 타석 상태 목록
    API-->>APP: 5. 사용 가능/마감/대기 가능 응답

    alt 사용 가능한 타석 있음
        M->>APP: 6. 타석 선택 + 예약 확정
        APP->>API: 7. POST /members/me/reservations {schedule_id, booth_code}
        API->>DB: 8. 정원/타석 재검증
        API->>DB: 9. INSERT lesson_bookings(status=BOOKED)
        API->>DB: 10. UPDATE lesson_schedules currentCount + 1
        DB-->>API: 11. reservation_id
        API->>N: 12. 예약 완료 푸시
        N-->>M: 13. "골프 예약이 완료되었습니다"
        API-->>APP: 14. 201 Created

    else 모든 타석 만석 + 대기 가능
        M->>APP: 15. [대기 등록]
        APP->>API: 16. POST /classes/{id}/waitlist
        API->>DB: 17. INSERT lesson_bookings(status=WAITLIST, waitlist_position)
        DB-->>API: 18. 순번 반환
        API->>N: 19. 대기 등록 확인 푸시
        N-->>M: 20. "현재 대기 2번입니다"
        API-->>APP: 21. 201 Created (waitlist)

        Note over API,DB: 이후 자리 발생 시 시스템 배치/트리거 처리
        API->>DB: 22. 최우선 대기자 조회 + 이용권 검증
        API->>DB: 23. WAITLIST -> BOOKED 승격
        API->>N: 24. 자동 확정 푸시
        N-->>M: 25. "자리가 생겨 예약이 확정되었습니다"
    end
```

## 3. 분기 요약

| 분기 | 조건 | 결과 |
|------|------|------|
| 타석 예약 | 사용 가능한 타석 존재 | 즉시 BOOKED |
| 대기 등록 | 전체 만석 + 대기 허용 | WAITLIST 생성 |
| 예약 실패 | 타석 동시 선점 충돌 | 재조회 후 재선택 유도 |
