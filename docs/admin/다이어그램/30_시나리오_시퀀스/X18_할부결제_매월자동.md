# X18 — 할부 결제 → 매월 자동승인 → 완료 🆕

## 1. 시나리오 개요

회원이 이용권을 할부로 결제 → 최초 회차 결제 후 매월 지정일에 자동 결제 → 전 회차 완납 시 할부 완료 처리하는 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 1 | 회원의 할부 결제 선택 |
| 트리거 2 | 매월 지정일 자동결제 크론잡 |
| 종료 조건 | 전 회차 완납 |
| 참여 도메인 | 매출관리(D3) |
| 구현 상태 | 🆕 신규 기능 |

## 2. 전제조건

- 할부 결제 상품이 등록되어 있음
- 회원이 자동결제 동의 완료
- PG사 빌링키(자동결제 키)가 발급되어 있음

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원 | 할부 결제 신청자 |
| 매니저 | 할부 계약 등록 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| PG | 결제 대행사 (빌링키 발급 및 자동결제) |
| 스케줄러 | 매월 자동결제 크론잡 |
| 알림서비스 | 결제 알림 발송 |

## 4. 시퀀스 다이어그램

```mermaid

    actor MB as 회원
    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    participant PG as PG사
    participant SCH as 스케줄러
    participant N as 알림서비스

    Note over MB,PG: [최초 할부 계약 — 빌링키 발급]
    M->>API: 1. POST /installments {member_id, product_id, total_months, monthly_amount} %% API->>PG: 2. 빌링키 발급 요청 (카드 정보 포함) %% PG-->>API: 3. billing_key 반환
    API->>DB: 4. installment_plans {member_id, total_months, monthly_amount, billing_key, next_payment_date}
    API->>DB: 5. installment_payments {plan_id, month=1, status=PENDING}
    DB-->>API: 6. plan_id

    Note over API,PG: [1회차 즉시 결제]
    API->>PG: 7. 1회차 결제 요청 {billing_key, amount} %% PG-->>API: 8. 승인 응답
    API->>DB: 9. installment_payments SET status=PAID, paid_at=NOW(동작) WHERE month=1
    API->>DB: 10. memberships {status=ACTIVE}
    DB-->>API: 11. OK
    API->>N: 12. 1회차 결제 완료 알림 %% N-->>MB: 13. SMS — "1/{total}회차 결제 완료 {amount}원"
    API-->>M: 14. 200 OK {plan_id}

    Note over SCH,PG: [매월 자동결제 크론잡]
    SCH->>API: 15. 자동결제 대상 조회 {today} %% API->>DB: 16. installment_plans WHERE next_payment_date=TODAY AND status=ACTIVE
    DB-->>API: 17. 자동결제 대상 목록

    loop 각 할부 계획별
        API->>DB: 18. installment_payments WHERE plan_id=? AND status=PENDING LIMIT 1
        DB-->>API: 19. 다음 회차 정보
        API->>PG: 20. 자동결제 요청 {billing_key, amount} %% alt 자동결제 성공
            PG-->>API: 21. 승인 응답 %% API->>DB: 22. installment_payments SET status=PAID
            API->>DB: 23. installment_plans SET next_payment_date=next_month, paid_months=paid_months+1
            DB-->>API: 24. OK
            API->>N: 25. 결제 완료 알림 %% N-->>MB: 26. SMS — "N/{total}회차 자동결제 완료"

            alt 전 회차 완납
                API->>DB: 27. installment_plans SET status=COMPLETED %% API->>N: 28. 할부 완납 알림
                N-->>MB: 29. SMS — "할부 결제가 모두 완료되었습니다"
            end
        else 자동결제 실패
            PG-->>API: 30. 결제 실패 응답 %% API->>DB: 31. installment_payments SET status=FAILED
            API->>DB: 32. unpaid_balances {member_id, amount, reason=AUTO_PAY_FAIL}
            API->>N: 33. 결제 실패 + 재결제 안내 %% N-->>MB: 34. SMS — "자동결제 실패. 카드 정보를 확인해 주세요"
        end
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 2 | 빌링키 발급 | 카드 정보를 PG에 등록하여 이후 카드 없이 자동결제 가능한 키 발급 |
| 4 | installment_plans | 전체 할부 계획 저장. total_months, monthly_amount, next_payment_date 포함 |
| 23 | next_payment_date 업데이트 | 다음 달 동일 일자로 설정 (말일 처리 포함) |
| 32 | unpaid_balances | X17 미수금 시나리오와 연계 — 자동 독촉 대상으로 편입 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 카드 만료 | 빌링키 갱신 요청. 새 카드 등록 유도 |
| 자동결제 실패 | X17 미수금 시나리오 연계. 3회 실패 시 할부 계획 SUSPENDED |
| 회원 중도 탈퇴 | 잔여 할부 일시불 처리 또는 이용권 해지 + 환불 정책 적용 |
| 말일 결제일 설정 | 2월 등 일수 부족한 달은 해당 월 말일로 처리 |
