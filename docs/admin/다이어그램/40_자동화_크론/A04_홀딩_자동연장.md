# A04 — 홀딩 자동 연장 계산

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매일 00:20 |
| 대상 엔티티 | Membership, MembershipHolding |
| 조건 | 홀딩 종료일이 도래한 이용권 ( <= NOW()) |
| 결과 | 홀딩 기간만큼 연장, status = ACTIVE 복원 |
| 관련 화면 | SCR-M004-02 이용권 탭, DLG-M003 홀딩 등록/해제 |

## 2. 발생 조건

- `Membership = HOLDING`
- `MembershipHolding. <= NOW()`
- 홀딩 일수 = ` - `
- 새 만료일 = 기존 만료일 + 홀딩 일수

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매일 00:20]) --> QUERY[홀딩 종료 도래 이용권 조회<br/>status=HOLDING AND <=NOW]

    QUERY -->|없음|EMPTY[대상 없음<br/>정상 종료]
    QUERY -->|있음|LOOP[대상 이용권 순회]

    LOOP --> CALC[홀딩 일수 계산<br/>days = -]
    CALC --> NEW_DATE[새 만료일 계산<br/> = + days]

    NEW_DATE --> TXN[트랜잭션 시작]
    TXN --> UPDATE_EXPIRY[Membership. 업데이트<br/>=]
    UPDATE_EXPIRY --> UPDATE_STATUS[Membership<br/>HOLDING → ACTIVE]
    UPDATE_STATUS --> CLOSE_HOLDING[MembershipHolding<br/>= COMPLETED]

    CLOSE_HOLDING --> COMMIT{트랜잭션 커밋}
    COMMIT -->|성공|LOG_HISTORY[연장 이력 저장<br/>days / 기록]
    COMMIT -->|실패|ROLLBACK[롤백<br/>오류 로그]

    LOG_HISTORY --> NOTIFY[홀딩 해제 알림 발송<br/>새 만료일 안내]
    NOTIFY --> LOG_OK[알림 이력 저장]
    NOTIFY --> LOG_NOTIFY_FAIL[알림 실패 로그]

    LOG_OK & LOG_NOTIFY_FAIL & ROLLBACK --> NEXT{다음 이용권?}
    NEXT --> LOOP
    NEXT -->|완료|REPORT[실행 리포트]
    REPORT --> END([크론 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17

    class CRON,END cron
    class QUERY,LOOP,CALC,COMMIT,NEXT system
    class LOG_HISTORY,LOG_OK,REPORT success
    class ROLLBACK,LOG_NOTIFY_FAIL error
    class EMPTY warning
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 트랜잭션 실패 | 롤백, 오류 로그, 다음 크론 재처리 |
| 홀딩 일수 0 이하 | 연장 없이 ACTIVE 복원만 수행 |
| 중복 실행 방지 | COMPLETED 상태 확인으로 중복 방지 |
