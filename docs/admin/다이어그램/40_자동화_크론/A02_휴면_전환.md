# A02 — 90일 미방문 회원 휴면 자동 전환

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매일 01:00 |
| 대상 엔티티 | Member |
| 조건 | 마지막 방문일로부터 90일 초과, status = ACTIVE |
| 결과 | Member = DORMANT 전환, 휴면 안내 알림 발송 |
| 관련 화면 | SCR-M001 회원 목록, SCR-M004 회원 상세 |

## 2. 발생 조건

- `Member = ACTIVE`
- `NOW() - Member.일`
- 이미 DORMANT/SUSPENDED/WITHDRAWN 상태 제외
- 유효한 이용권(ACTIVE Membership) 보유 여부 무관

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매일 01:00]) --> QUERY[미방문 90일 초과<br/>ACTIVE 회원 조회]

    QUERY -->|없음|EMPTY[대상 없음<br/>정상 종료]
    QUERY -->|있음|LOOP[대상 회원 순회]

    LOOP --> STATUS_CHECK{현재 상태 재확인<br/>ACTIVE?}
    STATUS_CHECK -->|변경됨|SKIP[스킵: 상태 불일치]
    STATUS_CHECK -->|ACTIVE|MEMBERSHIP{유효 이용권<br/>존재 여부}

    MEMBERSHIP -->|있음|UPDATE_DORMANT[Member<br/>ACTIVE → DORMANT]
    MEMBERSHIP -->|없음|UPDATE_DORMANT

    UPDATE_DORMANT -->|성공|LOG_STATE[상태전이 이력 저장<br/>MemberStatusHistory]
    UPDATE_DORMANT -->|실패|LOG_ERR[DB 오류 로그<br/>롤백]

    LOG_STATE --> NOTIFY[휴면 전환 알림 발송<br/>SMS/카카오]
    NOTIFY -->|성공|LOG_NOTIFY[알림 이력 저장]
    NOTIFY -->|실패|LOG_NOTIFY_FAIL[알림 실패 로그<br/>상태 전환은 유지]

    SKIP & LOG_NOTIFY & LOG_NOTIFY_FAIL & LOG_ERR --> NEXT{다음 회원?}
    NEXT --> LOOP
    NEXT -->|완료|REPORT[실행 리포트<br/>전환 수 / 실패 수]
    REPORT --> END([크론 완료])

    classDef cron fill:#E0F2F1,stroke:#00695C
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17

    class CRON,END cron
    class QUERY,LOOP,STATUS_CHECK,MEMBERSHIP,NEXT system
    class LOG_STATE,LOG_NOTIFY,REPORT success
    class LOG_ERR,LOG_NOTIFY_FAIL error
    class SKIP,EMPTY warning
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| DB 업데이트 실패 | 트랜잭션 롤백, 해당 회원 스킵, 오류 로그 |
| 알림 발송 실패 | 상태 전환은 유지, 알림만 실패 로그 기록 |
| 크론 중단 | 다음 날 재실행 시 미처리 회원 재포함 (조건 재평가) |
