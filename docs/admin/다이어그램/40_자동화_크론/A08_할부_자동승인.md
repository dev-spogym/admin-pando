# A08 — 할부 자동 승인 🆕

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매월 지정일 09:00 |
| 대상 엔티티 | Installment, Payment |
| 조건 | 당일이 할부 결제일인 미결제 할부 항목 |
| 결과 | PG 자동 결제 승인, Payment 생성, 회원 알림 |
| 관련 화면 | SCR-S009 할부 결제 관리 🆕 |

## 2. 발생 조건

- `Installment. = TODAY`
- `Installment = ACTIVE`
- `Installment.`
- 등록된 자동 결제 수단(빌링키) 유효

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매월 지정일 09:00]) --> QUERY[당일 결제 예정<br/>Installment 조회]

    QUERY -->|없음|EMPTY[결제 예정 없음<br/>종료]
    QUERY -->|있음|LOOP[할부 항목 순회]

    LOOP --> BILLING_CHECK{빌링키<br/>유효 여부}
    BILLING_CHECK -->|무효|NOTIFY_FAIL_KEY[결제 수단 오류 알림<br/>회원 갱신 요청]
    BILLING_CHECK -->|유효|PG_REQUEST[PG 자동 승인 요청<br/>빌링키 결제]

    PG_REQUEST -->|승인|TXN[트랜잭션 시작]
    PG_REQUEST -->|거절|RETRY{재시도 횟수<br/>?}

    RETRY --> PG_REQUEST
    RETRY -->|초과|FAIL_PROCESS[결제 실패 처리<br/>Installment=OVERDUE]

    TXN --> CREATE_PAY[Payment <br/>할부 결제 생성]
    CREATE_PAY --> UPDATE_INSTALL[Installment 업데이트<br/>-1<br/>+1개월]
    UPDATE_INSTALL --> COMPLETE_CHECK{<br/?}

    COMPLETE_CHECK -->|완납|CLOSE_INSTALL[Installment<br/>= COMPLETED]
    COMPLETE_CHECK -->|계속|COMMIT[커밋]
    CLOSE_INSTALL --> COMMIT

    COMMIT -->|성공|NOTIFY_OK[결제 완료 알림<br/>영수증 발송]
    COMMIT -->|실패|ROLLBACK[롤백<br/>오류 로그]

    NOTIFY_OK & NOTIFY_FAIL_KEY & FAIL_PROCESS & ROLLBACK --> NEXT{다음 항목?}
    NEXT --> LOOP
    NEXT -->|완료|REPORT[실행 리포트<br/>성공/실패/완납 수]
    REPORT --> END([크론 완료])
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| PG 결제 실패 | 최대 3회 재시도, 초과 시 OVERDUE 처리 |
| 빌링키 만료 | 즉시 회원 알림, 수동 결제 유도 |
| 트랜잭션 실패 | 롤백, PG 취소 요청, 오류 로그 |
| OVERDUE 상태 | SCR-S009에서 관리자 수동 처리 |
