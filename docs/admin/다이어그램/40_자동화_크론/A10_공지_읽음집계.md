# A10 — 공지 읽음률 집계

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매일 03:00 |
| 대상 엔티티 | Notice, NoticeReadLog |
| 조건 | status = PUBLISHED 인 모든 공지 |
| 결과 | Notice. 갱신, 미읽음 독촉 알림 (선택) |
| 관련 화면 | SCR-085 공지사항 관리 |

## 2. 발생 조건

- `Notice = PUBLISHED`
- `NoticeReadLog` 에서 읽음 수 집계
- 읽음률 = 읽은 수 / 발송 대상 수 × 100
- 읽음률 < 설정 임계치(기본 30%) 이면 독촉 알림 발송 (정책 설정 시)

## 3. 다이어그램

```mermaid
flowchart TD
    CRON([크론 트리거<br/>매일 03:00]) --> QUERY[PUBLISHED 공지 조회]

    QUERY -->|없음|EMPTY[대상 없음<br/>종료]
    QUERY -->|있음|LOOP[공지 순회]

    LOOP --> AGG[읽음 수 집계<br/> NoticeReadLog]
    AGG --> CALC[읽음률 계산<br/> /]

    CALC --> [Notice. 업데이트]
     -->|성공|THRESHOLD_CHECK{읽음률 임계치<br/>미달?}
     -->|실패|UPDATE_ERR[업데이트 오류 로그]

    THRESHOLD_CHECK -->|충족|LOG_OK[집계 완료 로그]
    THRESHOLD_CHECK -->|미달|REMIND_POLICY{독촉 알림<br/>정책 활성?}

    REMIND_POLICY -->|비활성|LOG_OK
    REMIND_POLICY -->|활성|SEND_REMIND[미읽음 대상자<br/>독촉 알림 발송]
    SEND_REMIND -->|성공|LOG_REMIND[알림 발송 이력]
    SEND_REMIND -->|실패|LOG_REMIND_FAIL[알림 실패 로그]

    UPDATE_ERR & LOG_OK & LOG_REMIND & LOG_REMIND_FAIL --> NEXT{다음 공지?}
    NEXT --> LOOP
    NEXT -->|완료|SUMMARY[일간 집계 요약 저장]
    SUMMARY --> END([크론 완료])
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 업데이트 실패 | 오류 로그, 다음 크론에서 재집계 |
| 알림 발송 실패 | 알림 실패 로그, 읽음률은 유지 |
| 임계치 정책 없음 | 집계만 수행, 독촉 알림 없음 |

## 5. 사용자 노출 메시지

| 대상 | 메시지 |
|------|--------|
| 미읽음 직원 | "[FitGenie] 미확인 공지가 있습니다. 확인해주세요: {공지 제목}" |
| 관리자 화면 | SCR-085 공지사항 관리 목록 읽음률 컬럼 갱신 |
