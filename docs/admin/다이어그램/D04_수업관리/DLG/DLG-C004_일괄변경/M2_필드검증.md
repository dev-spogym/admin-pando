## 1. 목적
DLG-C004에서 변경 항목별 유효성 검사를 정의한다.

## 2. 전제조건
- DLG-C004 열림 상태

## 3. 다이어그램

```mermaid
flowchart TD
  Apply[적용 버튼] --> FieldCheck{변경 항목 선택 여부}
  FieldCheck -->|": 미선택"|E_NoField[변경 항목을 선택해주세요.]
  FieldCheck -->|": 선택"|ValueCheck{변경값 입력 여부}
  ValueCheck -->|": 빈값"|E_NoValue[변경값을 입력해주세요.]
  ValueCheck -->|": 입력됨"|CancelCheck{수업취소 선택?}
  CancelCheck -->|": 취소 선택"|ConfirmBox[빨간 경고박스 표시\n확인 필요]
  CancelCheck -->|": 다른 항목"|AllPass[통과 → API 호출]
  ConfirmBox -->|": 체크박스 확인"|AllPass

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
  class Apply,FieldCheck,ValueCheck,CancelCheck system
  class E_NoField,E_NoValue error
```

## 4. 엣지 설명

| 검증 단계 | 규칙 |
|-----------|------|
| 항목 선택 | 최소 1개 항목 선택 필수 |
| 변경값 | 선택 항목에 맞는 값 필수 |
| 수업취소 | 빨간 경고박스 + 체크박스 확인 필수 |
