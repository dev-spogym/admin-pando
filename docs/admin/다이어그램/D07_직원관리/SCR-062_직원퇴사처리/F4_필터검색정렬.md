## 1. 목적

SCR-062 각 Step의 Select/검색 조작 흐름. 필터/정렬이 없는 위자드이므로 Step별 선택 플로우로 대체.

## 3. 다이어그램

```mermaid
flowchart TD
  STEP1([Step 1])
  STEP2([Step 2])

  STEP1 -->|직원 Select 변경|STAFF_LOAD[호출\n선택 카드 표시]
  STAFF_LOAD --> STAFF_CARD[직원 정보 카드\n이름/역할/입사일]
  STAFF_LOAD --> ERR_STAFF[직원 정보 로드 실패]

  STEP1 -->|퇴사 예정일 입력|DATE_VALID{min=today 검증}
  DATE_VALID --> DATE_OK[유효한 퇴사 예정일]
  DATE_VALID -->|과거|ERR_DATE[오늘 이후 날짜 오류]

  STEP2 -->|일괄 배정 FC 선택|FC_BULK[미배정 회원 일괄 FC 배정]
  STEP2 -->|개별 회원 재배정 FC 선택|MEMBER_ASSIGN[개별 회원 담당 FC 변경]
  MEMBER_ASSIGN -->|PT잔여 + 미배정|PT_WARN[error border 표시]

  classDef system fill:#EDE7F6,stroke:#5E35B2
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef info fill:#E0F7FA,stroke:#00838F

  class STEP1,STEP2 info
  class STAFF_LOAD,FC_BULK,MEMBER_ASSIGN,DATE_VALID system
  class ERR_STAFF,ERR_DATE,PT_WARN error
  class STAFF_CARD,DATE_OK success
```
