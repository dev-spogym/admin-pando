## 1. 목적

SCR-M010의 로딩/빈상태/에러/정상 등 각 UI 상태 분기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- SCR-M010 진입 시 API 응답 상태에 따라 분기

## 3. 다이어그램

```mermaid
flowchart TD
  ENTER[SCR-M010 진입]

  ENTER --> LOADING[로딩 스피너 표시]
  LOADING --> API_CALL[동작]

  API_CALL -->|200 데이터 있음|STATE_NORMAL[정상 상태 🆕]
  API_CALL -->|200 데이터 없음|STATE_EMPTY[빈 상태 🆕]
  API_CALL -->|401|STATE_AUTH[인증 오류]
  API_CALL -->|403|STATE_FORBIDDEN[접근 금지]
  API_CALL -->|500|STATE_ERROR[서버 오류]

  STATE_NORMAL --> SYS_CARDS[기본 세그먼트 카드 5개]
  STATE_NORMAL --> USR_TABLE[사용자 정의 테이블]

  STATE_EMPTY --> EMPTY_MSG[세그먼트가 없습니다 안내]
  STATE_EMPTY -->|primary/owner|BTN_ADD[세그먼트 추가 버튼]

  STATE_AUTH --> LOGIN[로그인 화면 이동]
  STATE_FORBIDDEN --> MSG_FORBIDDEN[접근 권한 없음 메시지]
  STATE_ERROR --> RETRY[재시도 버튼]
  RETRY --> API_CALL

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

  class STATE_NORMAL,STATE_EMPTY newFeature
  class SYS_CARDS,USR_TABLE success
  class STATE_AUTH,STATE_FORBIDDEN,STATE_ERROR,MSG_FORBIDDEN error
  class LOADING,API_CALL,RETRY system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| API | 정상 상태 | 200 + 데이터 |
| API | 빈 상태 | 200 + 빈 배열 |
| API | 인증 오류 | 401 |
| API | 접근 금지 | 403 |
| API | 서버 오류 | 500 |
| 서버 오류 | 재시도 버튼 | 오류 상태 |
