## 1. 목적

DLG-M010 메모 삭제 확인 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 상담메모 탭 > 메모 카드 > "삭제" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([메모 카드 삭제 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M010 열기 메모 삭제 ConfirmDialog]

  DLG_OPEN --> USER_ACT{사용자 액션}

  USER_ACT -->|취소 클릭|CLOSE[모달 닫기]
  USER_ACT -->|삭제 클릭|API_CALL[동작]

  API_CALL -->|200|REFETCH[메모 목록 refetch]
  API_CALL -->|오류|TOAST_ERR((toast 삭제 실패 error))

  classDef modal fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN modal
  class REFETCH success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 삭제 버튼 | 모달 열기 | - |
| 취소 | 모달 닫기 | - |
| 삭제 | API | 확인 클릭 |
| API | 목록 갱신 | 200 |
| API | toast | 오류 |
