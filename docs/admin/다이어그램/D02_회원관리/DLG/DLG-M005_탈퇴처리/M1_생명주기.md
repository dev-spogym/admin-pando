## 1. 목적

DLG-M005 회원 탈퇴 처리 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 회원 상세 > 프로필 헤더 > "탈퇴" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([회원 상세 프로필 헤더 탈퇴 클릭])

  TRIGGER --> DLG_OPEN[DLG-M005 열기]
  DLG_OPEN --> INIT[탈퇴 사유 textarea 빈값]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|취소/X/ESC|CLOSE_RESET[모달 닫기 + 사유 초기화]
  USER_ACT -->|탈퇴 처리|API_CALL[status=WITHDRAWN]

  API_CALL -->|200|TOAST_OK((toast 회원이 탈퇴 처리되었습니다))
  TOAST_OK --> REDIRECT[/ 이동]

  API_CALL -->|4xx/5xx|TOAST_ERR((toast 탈퇴 처리에 실패했습니다))
  TOAST_ERR --> DLG_OPEN

  classDef modal fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT modal
  class TOAST_OK,REDIRECT success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 탈퇴 버튼 | 모달 열기 | - |
| 취소/X/ESC | 모달 닫기+초기화 | - |
| 탈퇴 처리 | API | 클릭 |
| API | toast | 200 |
| toast | / 이동 | - |
| API | toast | 오류 |
