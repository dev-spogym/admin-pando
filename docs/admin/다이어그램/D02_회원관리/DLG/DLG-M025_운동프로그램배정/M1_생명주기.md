## 1. 목적

DLG-M025 운동 프로그램 배정 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 운동프로그램 탭 > "프로그램 배정" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([운동프로그램 탭 프로그램 배정 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M025 운동 프로그램 배정 max-w-520px]
  DLG_OPEN --> LOAD[지점 프로그램 목록 로드]

  LOAD -->|목록 있음|LIST[radio list 표시 max-h-400px scroll]
  LOAD -->|목록 없음|EMPTY[등록된 운동 프로그램이 없습니다]

  LIST --> USER_ACT{사용자 액션}

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|프로그램 선택|SELECTED[설정]
  SELECTED -->|배정 클릭|API_CALL[0]

  API_CALL -->|200|TOAST_OK((toast 프로그램이 배정되었습니다))
  TOAST_OK --> POST_OK[모달 닫기 + refetch]

  API_CALL -->|오류|TOAST_ERR((toast 배정 실패 error))
  TOAST_ERR --> DLG_OPEN

  classDef modal fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,LIST modal
  class TOAST_OK,POST_OK success
  class EMPTY,TOAST_ERR error
  class LOAD,API_CALL system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 배정 버튼 | 모달 열기 | - |
| 목록 로드 | 빈 상태 | 프로그램 없음 |
| radio | 선택됨 | 클릭 |
| 배정 버튼 | API | 있음 |
| API | toast | 200 |
