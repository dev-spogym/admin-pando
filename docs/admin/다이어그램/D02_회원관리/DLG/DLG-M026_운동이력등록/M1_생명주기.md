## 1. 목적

DLG-M026 운동 이력 추가 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 운동이력 탭 > "운동 추가" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([운동이력 탭 운동 추가 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M026 운동 이력 추가 max-w-480px]
  DLG_OPEN --> INIT[날짜=오늘 운동명/기타 빈값]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|저장 + 있음|API_CALL[동작]

  API_CALL -->|200|TOAST_OK((toast 운동 이력이 저장되었습니다))
  TOAST_OK --> POST_OK[모달 닫기 + refetch]

  API_CALL -->|오류|TOAST_ERR((toast 저장 실패 error))
  TOAST_ERR --> DLG_OPEN

  classDef modal fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT modal
  class TOAST_OK,POST_OK success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 출발 | 도착 | 조건 |
|------|------|------|
| 운동 추가 | 모달 열기 | - |
| 저장 | API | + 있음 |
| API | toast | 200 |
| API | toast | 오류 |
