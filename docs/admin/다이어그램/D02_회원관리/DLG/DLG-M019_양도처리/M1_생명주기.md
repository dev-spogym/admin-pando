## 1. 목적

DLG-M019 양도 처리 다이얼로그의 열기/닫기/완료 생명주기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- 상세내역 탭 > 양도 서브탭 > "양도하기" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([상세내역 탭 양도하기 버튼 클릭])

  TRIGGER --> DLG_OPEN[DLG-M019 양도 처리 🆕]
  DLG_OPEN --> INIT[상품명/양수인/사유 초기화]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|취소|CLOSE[모달 닫기]
  USER_ACT -->|저장 필수 충족|API_CALL[동작]

  API_CALL -->|200|TOAST_OK((toast 양도가 처리되었습니다))
  TOAST_OK --> POST_OK[모달 닫기 + refetch]

  API_CALL -->|오류|TOAST_ERR((toast 양도 처리에 실패했습니다))
  TOAST_ERR --> DLG_OPEN

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT newFeature
  class TOAST_OK,POST_OK success
  class TOAST_ERR error
  class API_CALL system
```

|------|------|------|
