## 1. 목적

DLG-M013 환불 API 응답별 결과 분기를 명세한다.

## 2. 트리거/전제조건

- (환불 레코드) + PUT 원본 status=REFUNDED 호출 후

## 3. 다이어그램

```mermaid
flowchart TD
  API_REFUND[환불 레코드 음수 금액]
  API_UPDATE[status=REFUNDED]

  API_REFUND -->|200|API_UPDATE
  API_UPDATE -->|200|SUCCESS[환불 성공]
  API_REFUND -->|오류|FAIL[환불 실패]
  API_UPDATE -->|오류|FAIL

  SUCCESS --> TOAST_OK((toast 환불 처리가 완료되었습니다))
  TOAST_OK --> REFETCH[결제이력 refetch]

  FAIL --> TOAST_ERR((toast 환불 처리 실패 error))
  TOAST_ERR --> MODAL_STAY[모달 유지]

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system fill:#EDE7F6,stroke:#5E35B2

  class SUCCESS,TOAST_OK,REFETCH success
  class FAIL,TOAST_ERR,MODAL_STAY error
  class API_REFUND,API_UPDATE system
```

| 출발 | 도착 | 조건 |
|------|------|------|

| 환불 | 실패 | 오류 |
| 성공 | toast | - |
| 실패 | toast | - |
