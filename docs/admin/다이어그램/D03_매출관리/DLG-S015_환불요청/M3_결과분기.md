## 1. 목적
DLG-S015 부분환불 처리 후 승인 대기 또는 즉시 완료 분기를 표현한다.

## 2. 전제조건
- DLG-S015에서 제출 완료

## 3. 다이어그램

```mermaid
flowchart TD
    SUBMIT_OK[제출 완료] --> ROUTE{처리 경로}

    ROUTE -->|": 10만원 미만"|PG_PARTIAL[부분 PG 취소\]
    ROUTE -->|": 10만원 이상"|SEND_APPROVAL[승인 요청 발송\n알림 전송]

    PG_PARTIAL -->|": 성공"|PARTIAL_DONE[부분환불 완료\nsales PARTIAL_REFUNDED]
    PG_PARTIAL -->|": 실패"|PG_ERR[PG 오류 토스트]

    SEND_APPROVAL --> APPROVAL_WAIT[승인 대기 상태]

    PARTIAL_DONE --> DLG_S014[DLG-S014 환불상세 표시]
    APPROVAL_WAIT --> CLOSED_WAIT([모달 닫힘\n승인 대기 토스트])

    classDef newFeature fill:#F3E5F5,stroke:#7B1FA2,color:#4A148C,stroke-dasharray:4 4
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef info fill:#E0F7FA,stroke:#00838F
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

    class DLG_S014,SEND_APPROVAL newFeature
    class ROUTE,PG_PARTIAL system
    class PARTIAL_DONE success
    class APPROVAL_WAIT,CLOSED_WAIT info
    class PG_ERR error
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| ROUTE | PG_PARTIAL | 즉시 부분 PG 취소 |
| ROUTE | SEND_APPROVAL | 승인 요청 발송 |
| PG_PARTIAL | PARTIAL_DONE | PG 성공 → 부분환불 완료 |
| PARTIAL_DONE | DLG_S014 | 환불 상세 모달 표시 |
