## 1. 목적
SCR-S012에서 역할별 접근 범위와 승인 권한을 표현한다.

## 2. 전제조건
- 로그인 완료

## 3. 다이어그램

```mermaid
flowchart LR
    AUTH{역할 분기}

    AUTH -->|": primary"|P[슈퍼관리자\n전체 취소/환불\n승인 권한 보유]
    AUTH -->|": owner"|O[센터장\n전체 취소/환불\n승인 권한 보유]
    AUTH -->|": manager"|M[매니저\n취소/환불 실행\n10만원 미만 즉시처리\n10만원 이상 승인 요청]
    AUTH -->|": fc"|FC[프론트\n본인 담당 건만\n10만원 미만 즉시처리]
    AUTH -->|": trainer"|TR[트레이너\n본인 담당 건만\n조회 전용]
    AUTH -->|": readonly"|RO[읽기전용\n접근 차단]

    P --> FULL_AUTH[전체 취소/환불 + 승인]
    O --> FULL_AUTH
    M --> LIMITED_AUTH[취소/환불 실행\n승인은 불가]
    FC --> LIMITED_AUTH
    TR --> VIEW_ONLY[조회만 가능]
    RO --> BLOCKED[접근 차단]

    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
    classDef rbacBlocked fill:#F5F5F5,stroke:#9E9E9E,color:#616161

    class AUTH system
    class FULL_AUTH success
    class LIMITED_AUTH,VIEW_ONLY warning
    class BLOCKED rbacBlocked
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| AUTH | P | 슈퍼관리자 — 전체 권한 + 승인 |
| AUTH | O | 센터장 — 전체 권한 + 승인 |
| AUTH | M | 매니저 — 실행 가능, 승인 불가 |
| AUTH | FC | 프론트 — 담당 건 제한 |
| AUTH | TR | 트레이너 — 조회 전용 |
| AUTH | RO | readonly — 접근 차단 |
