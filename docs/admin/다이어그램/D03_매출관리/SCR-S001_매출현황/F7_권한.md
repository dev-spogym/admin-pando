## 1. 목적
SCR-S001에서 6개 역할별 접근 가능 범위와 제한 사항을 표현한다.

## 2. 전제조건
- 로그인 완료

## 3. 다이어그램

```mermaid
flowchart LR
    AUTH{역할 분기}

    AUTH -->|": primary"|PRIMARY[슈퍼관리자\n전체 접근\n전 지점 조회 가능\n모든 버튼 활성]
    AUTH -->|": owner"|OWNER[센터장\n전체 접근\n본 지점만\n모든 버튼 활성]
    AUTH -->|": manager"|MANAGER[매니저\n전체 접근\n본 지점만\n모든 버튼 활성]
    AUTH -->|": trainer"|TRAINER[트레이너\n조회만\n담당자 필터 자동\n신규결제 POS만 가능]
    AUTH -->|": fc/staff"|FC[프론트\n조회만\n신규결제 POS 버튼만 활성\n통계/엑셀 비활성]
    AUTH -->|": readonly"|READONLY[읽기전용\n접근 차단\n권한없음 토스트]

    PRIMARY --> ACTION_FULL[전체 기능\n조회+엑셀+POS+탭전환]
    OWNER --> ACTION_FULL
    MANAGER --> ACTION_FULL
    TRAINER --> ACTION_LIMITED[제한 기능\n담당자 매출 조회\ 진입 가능]
    FC --> ACTION_POS_ONLY[POS 전용\ 버튼만 클릭 가능]
    READONLY --> ACTION_BLOCKED[접근 차단\n/sales 리다이렉트 차단]

    classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
    classDef rbacBlocked fill:#F5F5F5,stroke:#9E9E9E,color:#616161
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

    class ACTION_FULL success
    class ACTION_LIMITED,ACTION_POS_ONLY warning
    class ACTION_BLOCKED rbacBlocked
    class AUTH system
    class PRIMARY,OWNER,MANAGER,TRAINER,FC,READONLY screen
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| AUTH | PRIMARY | 슈퍼관리자 분기 |
| AUTH | OWNER | 센터장 분기 |
| AUTH | MANAGER | 매니저 분기 |
| AUTH | TRAINER | 트레이너 분기 — 담당자 자동 필터 |
| AUTH | FC | 프론트 분기 — POS만 활성 |
| AUTH | READONLY | 읽기전용 — 접근 차단 |
