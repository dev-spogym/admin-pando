## 1. 목적
DLG-S011 세금계산서 발행 폼의 필드별 검증 규칙을 표현한다.

## 2. 전제조건
- DLG-S011 열림 상태

## 3. 다이어그램

```mermaid
flowchart TD
    SUBMIT[발행 버튼] --> BIZ_CHECK{사업자번호 검증\n10자리 숫자}
    BIZ_CHECK --> CORP_CHECK{상호/대표자 입력 확인}
    BIZ_CHECK --> ERR_BIZ[사업자번호 형식 오류]

    CORP_CHECK --> AMOUNT_CHECK{공급가액 검증\n금액}
    CORP_CHECK --> ERR_CORP[상호/대표자 필수]

    AMOUNT_CHECK --> DATE_CHECK{발행일자 검증}
    AMOUNT_CHECK --> ERR_AMOUNT[공급가액 필수]

    DATE_CHECK --> ALL_VALID[전체 유효\n발행 버튼 활성]
    DATE_CHECK -->|": 미래 날짜"|ERR_DATE[발행일자 오류\n오늘 이전만 가능]

    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C

    class BIZ_CHECK,CORP_CHECK,AMOUNT_CHECK,DATE_CHECK system
    class ALL_VALID success
    class ERR_BIZ,ERR_CORP,ERR_AMOUNT,ERR_DATE error
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| BIZ_CHECK | ERR_BIZ | 사업자번호 형식 오류 |
| CORP_CHECK | ERR_CORP | 상호/대표자 미입력 |
| AMOUNT_CHECK | ERR_AMOUNT | 공급가액 0 |
| DATE_CHECK | ERR_DATE | 미래 날짜 |
