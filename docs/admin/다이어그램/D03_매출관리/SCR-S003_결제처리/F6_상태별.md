## 1. 목적
SCR-S003의 모든 UI 상태(로딩/완료/에러/마일리지부족 등)를 표현한다.

## 2. 전제조건
- SCR-S003 진입 완료

## 3. 다이어그램

```mermaid
flowchart TD
    S003[SCR-S003 결제 처리] --> PAY_STATE{결제 상태}

    PAY_STATE -->|": 대기"|IDLE[대기 상태\n결제수단 선택 대기]
    PAY_STATE -->|": 결제 처리중"|PROCESSING[처리중 상태\n처리 중... + Spinner\n결제 확인 disabled]
    PAY_STATE -->|": 완료"|COMPLETE[완료 상태\n체크 아이콘 + 성공 메시지\n영수증/문자/계속/매출현황 버튼]
    PAY_STATE -->|": 장바구니 없음"|NO_CART[오류 상태\n결제할 상품이 없습니다.\ 판매로 이동]

    IDLE --> VALID_STATE{유효성 상태}
    VALID_STATE --> ERR_MILEAGE[마일리지 부족\n부족: NP 표시]
    VALID_STATE --> ERR_MEMBER[회원 미선택 안내]
    VALID_STATE --> ERR_MIXED[금액 불일치 표시\원 부족/초과]
    VALID_STATE --> VALID_OK[금액 일치 표시\nsuccess 색상]
    VALID_STATE -->|": 정상"|CONFIRM_ACTIVE[결제 확인 버튼 활성]

    classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17
    classDef system fill:#EDE7F6,stroke:#5E35B2

    class S003 screen
    class ERR_MILEAGE,ERR_MEMBER,ERR_MIXED,NO_CART error
    class COMPLETE,VALID_OK,CONFIRM_ACTIVE success
    class PROCESSING warning
    class PAY_STATE,VALID_STATE system
```

## 4. 엣지 설명

| 출발 | 도착 | 설명 |
|------|------|------|
| PAY_STATE | PROCESSING | 결제 처리 중 |
| PAY_STATE | COMPLETE | 결제 완료 |
| PAY_STATE | NO_CART | 장바구니 없음 |
| VALID_STATE | ERR_MILEAGE | 마일리지 부족 |
| VALID_STATE | ERR_MIXED | 복합결제 불일치 |
