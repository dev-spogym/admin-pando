# E06 — 서버 내부 오류 (500)

## 1. 개요

| 항목 | 내용 |
|------|------|
| 에러코드 | E500001 |
| HTTP | 500 Internal Server Error |
| 발생 모듈 | 전 모듈 |
| 영향 화면 | 전체 화면, SCR-108 에러 페이지 |

## 2. 발생 조건

- 서버 unhandled exception
- DB 쿼리 오류
- 메모리 부족, 스레드 데드락
- 예상치 못한 null 참조, 타입 오류

## 3. 다이어그램

```mermaid
flowchart TD
    REQ([API 요청]) --> SERVER[서버 처리]

    SERVER -->|성공|RESP_OK[200 응답]
    SERVER -->|예외 발생|GLOBAL_HANDLER[전역 예외 핸들러]

    GLOBAL_HANDLER --> LOG_SERVER[서버 로그 적재<br/>스택 트레이스 + 요청 정보]
    LOG_SERVER --> ALERT{오류 빈도<br/>임계치 초과?}

    ALERT -->|초과|OPS_ALERT[운영팀 슬랙/이메일 알림<br/>자동 인시던트 생성]
    ALERT -->|정상|RESP_ERR[E500001 응답 반환]
    OPS_ALERT --> RESP_ERR

    RESP_ERR --> CLIENT_HANDLER[클라이언트 전역 핸들러]
    CLIENT_HANDLER --> TOAST[전역 에러 토스트<br/>일시적인 오류가 발생했습니다]

    TOAST -->|재시도 클릭|RETRY[동일 요청 재시도]
    TOAST -->|닫기|STAY[현재 화면 유지<br/>입력값 보존]

    RETRY -->|성공|RESP_OK
    RETRY -->|재실패|FINAL_ERR[에러 화면 표시<br/>SCR-108 또는 인라인]

    classDef error fill:#FFEBEE,stroke:#C62828,color:#B71C1C
    classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
    classDef system fill:#EDE7F6,stroke:#5E35B2
    classDef screen fill:#E3F2FD,stroke:#1976D2,color:#0D47A1
    classDef warning fill:#FFF8E1,stroke:#F9A825,color:#F57F17

    class GLOBAL_HANDLER,RESP_ERR,TOAST,FINAL_ERR,OPS_ALERT error
    class RESP_OK success
    class SERVER,ALERT,CLIENT_HANDLER system
    class STAY warning
```

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| 단발성 오류 | 재시도 버튼 제공 |
| 반복 오류 | 운영팀 알림, 인시던트 생성 |
| 폼 입력 중 발생 | 입력값 보존, 재시도 가능 |
| 크리티컬 오류 | 서버 재시작, 롤백 |

## 5. 사용자 노출 메시지

| 에러코드 | 메시지 |
|----------|--------|
| E500001 | 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요 |
