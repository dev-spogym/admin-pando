# 문서정비 우선순위

> 목적: 기존 문서는 이번 턴에서 수정하지 않고, 추후 어떤 순서로 정비해야 하는지만 정리한다.

---

## 1. 정비 원칙

- 이 문서는 "추후 수정 대상 목록"이다.
- 이번 작업에서는 기존 문서를 수정하지 않는다.
- 정비 시 반드시 `docs/HWANG폴더` 문서를 기준으로 반영한다.

---

## 2. 1차 정비 대상

### 2.1 상태 체계 관련

우선 정비 대상:

- 리드 상태전이도
- 마케팅 기능명세서 리드 파트
- 리드 관리 화면설계서
- 회원 상태전이도
- 회원관리 기능명세서
- 회원 목록/회원 상세 화면설계서
- 결제 상태전이도
- 매출관리 기능명세서

정비 목표:

- 리드 상태 1벌화
- 회원 `INACTIVE/미등록` 분리
- 결제 `COMPLETED/UNPAID`와 `APPROVED/INSTALLMENT` 통합

---

## 3. 2차 정비 대상

### 3.1 권한 체계 관련

정비 대상:

- 권한 매트릭스 인덱스
- `_FRONTMATTER_SCHEMA.md`
- 관리자 웹 화면설계서의 `trainer/front/admin` 혼용 부분
- 회원앱 공통 화면설계서의 `별도 CRM 앱` 문구

정비 목표:

- 관리자 웹 표준 권한 고정
- 회원앱 표준 역할 고정
- 채널 간 역할 혼용 제거

---

## 4. 3차 정비 대상

### 4.1 자동화/만료 알림 관련

정비 대상:

- 자동화 크론 A01
- 설정관리 알림설정
- 마케팅 자동 알림 설정
- 회원앱 만료 알림 정책 서술
- 본사관리 자동화/KPI 연계 문구

정비 목표:

- 고정 D-day 문구 제거
- 본사 정책 세트 -> 지점 선택 구조 반영
- 필수/권장/선택 step 구조 반영

---

## 5. 4차 정비 대상

### 5.1 인덱스/참조/마스터 문서

정비 대상:

- `docs/화면설계서/README.md`
- `docs/다이어그램/00_사이트맵/README.md`
- `docs/다이어그램/30_시나리오_시퀀스/README.md`
- 깨진 경로를 참조하는 각 `00-기본화면.md`
- 마스터 파일이 없는 SCR/DLG 폴더

정비 목표:

- 현재 실제 폴더 수량과 인덱스 일치
- 존재하지 않는 경로 참조 제거
- 모든 유닛에 마스터 파일 보강

## 7. 정비 순서 제안

1. 상태 체계 반영
2. 권한 체계 반영
3. 자동화/만료 알림 정책 반영
4. 인덱스/참조 경로 정비
5. 누락된 마스터 파일 보강

## 8. 퍼블리싱팀 전달용 결론

퍼블리싱팀은 우선 아래만 고정해도 화면 정합성이 크게 좋아진다.

- 상태 배지 문구
- 권한별 CTA 노출 기준
- 만료 알림 설정 화면의 본사/지점 관계
- 회원앱이 단일 회원앱 셸인지 여부

즉, 먼저 개념을 고정하고 나서 개별 화면을 정리해야 한다.
