# 관리자 문서 통합 ID 대응표

## 목적

- `기획자`의 기존 화면 ID 체계를 기준으로 유지한다.
- `기획자2`의 화면 중 `기획자`에 없는 화면만 `기획자` 체계로 재번호화한다.
- `D12-회원앱`은 `회원앱2`의 `MA` 화면 ID를 최종 기준으로 사용한다.

## 기준 원칙

1. 같은 도메인에서 화면 목적이 동일하면 `기획자`의 기존 ID를 그대로 사용한다.
2. `기획자2`가 기존 두 화면을 하나로 합친 경우:
   기존 상세 화면은 `기획자` ID를 유지하고, 합쳐진 운영 허브가 필요하면 신규 확장 ID를 부여한다.
3. 숫자 슬롯이 모두 사용된 도메인에서 신규 화면이 필요하면 `A` suffix 확장 규칙을 사용한다.
   예: `SCR-072A`, `SCR-080A`, `SCR-082A`
4. `D12-회원앱`은 더 이상 `H120x`를 유지하지 않고 `회원앱2`의 `MA-xxx`로 귀속한다.

## 4. 후속 정리 우선순위

1. `기획자` 권한 표기에서 `superAdmin / primary / owner / manager / fc / staff / readonly` 체계 반영
2. 세부 다이어그램과 보조 문서의 잔여 용어를 최신 통합 기준으로 재점검

## 5. 현재 판단

- 관리자 상세 원본은 `기획자`
- 전사 정합성 기준은 `기획자2`
- 회원앱 원본은 `회원앱2`

따라서 최신 통합본은 `기획자`를 베이스로 하고, 이 문서의 재번호화 규칙과 `회원앱2`의 `MA` 체계를 덮어쓰는 방식으로 정리한다.
