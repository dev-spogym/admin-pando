# 20260429 퍼블리싱 정합화 기록

> 기준일: 2026-04-29
> 목적: 화면설계서·기능명세서·`src/app` route를 기획 우선 기준으로 맞추고, UI 존재 여부와 데이터 연동 상태를 구분해 기록한다.

---

## 1. 처리 원칙

- 화면설계서 route를 정본으로 보고 `src/app` route를 맞춘다.
- 이미 UI가 있는 화면은 `퍼블리싱 완료 / 데이터 미연동`으로 정리한다.
- 기획은 있으나 실데이터가 없는 화면은 `계획 있음 + 데이터 미연동`으로 기록한다.
- 기획 단계, 예정, 제외로만 남아 있고 UI도 없는 항목은 퍼블리싱을 진행하지 않는다.
- 내부 운영용 화면 또는 에러 페이지는 클라이언트 기획 화면과 분리한다.
- Cmd+/ 오버레이는 이번 판단 범위에서 제외하되, route drift가 남지 않도록 매핑 파일은 함께 정리했다.

## 8. 최종 판정

- 화면설계서에 route가 있는 screen 중 `src/app`에 없는 항목은 0개로 정리했다.
- `미구현`으로 남아 있던 화면 중 실제 UI가 있는 항목은 모두 `퍼블리싱 완료 / 데이터 미연동`으로 바꿨다.
- 남은 app-only route는 내부 전용 또는 1개 화면설계서가 2개 route를 대표하는 정상 예외다.
- 현재 범위에서 계획은 있는데 UI가 없는 화면은 남기지 않았다.
