# 상품 · 결제 · 이용권 운영기획서

## 1. 문서 목적

- 상품 판매부터 계약, 결제, 이용권 생성, 환불까지를 하나의 운영 흐름으로 정리한다.
- 매출 화면과 회원 상세 화면이 서로 다른 상태를 말하지 않도록 기준을 통일한다.

## 2. 운영 대상

- 상품 관리
- 할인 정책
- POS 판매
- 결제 처리
- 매출 현황
- 환불 관리
- 미수금 관리
- 선수익금 조회

## 3. 상품 체계

### 3.1 상품 분류

- 정기 이용권
- PT 이용권
- GX 이용권
- 골프 이용권
- 락커/부가서비스
- 패키지 상품
- 단일 판매 상품

### 3.2 상품 마스터 분류 체계

04/29 기준으로 상품 마스터는 1단계 대분류와 2단계 세부 분류를 구분한다.

| 단계 | 항목 | 운영 기준 |
| --- | --- | --- |
| 1단계 | 상품 대분류 | 정기 이용권, PT, GX, 골프, 락커/부가서비스, 패키지, 단일 판매 상품 |
| 2단계 | 세부 분류 | 대분류에 따라 표시되는 하위 종목 또는 상품군 |

- GX 상품은 2단계 세부종목을 필수로 가진다.
- GX 세부종목은 요가, 필라테스, 스피닝, 줌바, 에어로빅, GX 기타 6종으로 고정한다.
- 이 6종은 상품 등록, 상품 목록 필터, 매출통계, KPI 대시보드의 GX 세부 종목 분포에 같은 기준으로 사용한다.

### 3.3 상품 운영 원칙

- 상품은 판매 단위이고, 실제 사용 권한은 이용권에서 관리한다.
- 패키지 상품은 결제는 1건이지만 이용권은 여러 개 생성될 수 있다.
- 전 지점 공통 상품은 본사에서 배포하고, 지점 전용 상품은 지점에서만 운영한다.
- 할인 정책은 상품 자체가 아니라 판매 시 적용되는 정책으로 분리한다.

## 4. 결제 상태 기준

| 결제 상태 | 의미 |
| --- | --- |
| 결제완료 | 승인 완료된 정상 결제 |
| 분할결제중 | 일부 수납 완료, 잔액 남음 |
| 미수금 | 결제 예정이나 미수 상태 |
| 부분환불 | 일부 금액 환불 |
| 환불완료 | 전체 환불 완료 |
| 취소 | 승인 취소 또는 무효 처리 |

## 5. 이용권 상태 기준

| 이용권 상태 | 의미 |
| --- | --- |
| 사용예정 | 시작일 전 |
| 사용중 | 정상 이용 가능 |
| 홀딩 | 일시 정지 |
| 만료예정 | 만료일 임박 |
| 만료 | 사용 종료 |
| 소진완료 | 횟수형 상품 사용 완료 |
| 환불종료 | 환불에 의해 종료 |

## 6. 핵심 운영 원칙

- 결제와 이용권은 같은 개념이 아니다.
- `결제완료` 또는 운영 승인된 `분할결제중` 상태만 이용권을 활성화한다.
- `미수금`은 회원에게 판매 의사가 있는 상태이지, 자동으로 사용 권한이 열리는 상태가 아니다.
- 환불은 금액 처리와 이용권 처리, 매출 집계 반영이 함께 가야 한다.
- 선수익금은 결제 기준이 아니라 서비스 제공 전 금액 기준으로 계산한다.

## 7. 화면 운영 기준

### 7.1 상품 관리

- 상품 목록은 분류, 사용 여부, 지점 범위, 가격, 판매 가능 여부를 바로 보여줘야 한다.
- 상품 상세/등록 화면은 현장 직원이 이해하기 쉬운 언어를 사용한다.
- 사용 제한 조건은 텍스트 설명보다 구조화된 배지/옵션으로 보이게 한다.

### 7.2 POS 판매/결제 처리

- POS는 현장 직원이 빠르게 판매를 끝내는 화면이다.
- 구매자 검색, 상품 선택, 할인, 결제 수단, 최종 금액 확인을 1개 흐름에 넣는다.
- 중복 결제 경고는 강하게 주되, 매니저 이상은 승인 후 진행할 수 있게 한다.

### 7.3 매출 현황

- 매출 현황은 회계 화면이 아니라 운영 화면으로 본다.
- 오늘 매출, 기간 매출, 환불액, 미수금, 선수익금이 같은 기준일로 보이게 해야 한다.
- 결제 상태 배지와 이용권 상태 배지를 혼동하지 않도록 구분한다.
- 통계관리의 `받을금액`은 VAT 제외 순매출로 표시한다.
- 법인권 매출은 일반 회원권/B2C 매출과 별도 집계한다.
- 매출통계에는 개월별 판매 비율과 GX 세부 종목 분포를 표시한다.
- 담당자별 매출은 신규 매출과 재등록 매출을 분리해 볼 수 있어야 한다.
- 골프 매출은 골프프로별 매출로 별도 확인할 수 있어야 한다.

### 7.4 환불/미수금

- 환불 관리와 미수금 관리는 별도 메뉴로 유지한다.
- 환불 화면에서는 환불 사유, 원결제, 환불 방식, 남은 이용권 정리 결과를 함께 보여준다.
- 미수금 화면에서는 추심 메모와 예정일이 핵심이다.
- 환불 화면 상단에는 환불 총액, 환불 건수, 평균 환불액, 위약금 합계를 표시한다.
- 위약금 합계는 환불 총액과 별도로, 환불 처리 과정에서 공제된 금액만 집계한다.
- 담당자별 환불 책임은 환불 처리자가 아니라 원판매 담당자 기준으로 표시한다.
- 원판매 담당자 연결은 원거래ID 또는 카드 승인번호를 기준으로 한다.

## 8. 매출 분석 KPI 기준

04/29 기준으로 매출/재무 화면은 아래 지표가 퍼블리싱 기준에 포함된다.

| 지표 | 퍼블리싱 기준 |
| --- | --- |
| VAT 제외 매출 | `받을금액`을 VAT 제외 순매출로 명시 |
| 법인권 매출 | 법인권 거래를 B2B 매출로 별도 카드 또는 탭에 표시 |
| 개월별 판매 비율 | 1개월, 3개월, 6개월, 12개월, 기타 5개 버킷으로 표시 |
| GX 세부 종목 분포 | 요가·필라테스·스피닝·줌바·에어로빅·GX 기타 6종 분포 표시 |
| 위약금 합계 | 환불관리 요약 KPI에 별도 표시 |
| 담당자별 환불 책임 | 원판매 담당자별 환불 건수, 환불 금액, 위약금 합계, 환불률 표시 |
| 트레이너별 신규/재등록 매출 | 강습 매출을 신규와 재등록으로 분리 |
| 골프프로별 매출 | 골프 매출을 프로별로 분리 |

### 8.1 제외 확정 지표

- 환불관리의 재결제 건수와 재결제율은 추가하지 않는다.
- 취소재결제 건수는 매출관리 신규 KPI로 추가하지 않는다.
- 매입사별 수수료 비교는 현재 퍼블리싱 범위에 포함하지 않는다.

## 9. 계약 흐름 기준

```text
회원 선택
-> 상품 선택
-> 할인/프로모션 적용
-> 결제 방식 선택
-> 승인
-> 이용권 생성
-> 회원 상세/앱에 반영
```

- 전자계약 위자드는 위 흐름을 단계형으로 제공한다.
- POS와 전자계약은 입력 UX는 달라도 최종 결과는 같은 운영 구조를 만든다.
- 앱 결제가 열리더라도 상품/결제/이용권 기준은 관리자 웹과 동일해야 한다.

## 10. 운영 예외 처리

- 시작일이 미래인 상품은 결제완료여도 이용권 상태는 `사용예정`이다.
- 횟수형 상품은 잔여 횟수가 0이면 만료일이 남아 있어도 `소진완료`로 본다.
- 환불 시 홀딩 중인 이용권도 함께 종료 여부를 판단해야 한다.
- 패키지 중 일부만 환불하는 경우 하위 이용권 정리 기준을 별도 안내문으로 제공해야 한다.

## 11. 퍼블리싱 기준 메모

- 숫자 중심 화면은 상태 배지와 금액 정렬을 우선한다.
- 환불/미수금은 경고성 컬러를 명확히 구분한다.
- 이용권 카드와 결제 이력 카드는 같은 의미로 보이지 않게 컴포넌트 구조를 분리한다.
- 매출통계 하위 탭은 단순 차트 나열이 아니라 상품, 결제수단, 기간 믹스, GX 세부 종목, 법인권, 담당자 성과를 구분해 이동 가능해야 한다.
- 환불관리의 처리자와 원판매 담당자는 같은 컬럼처럼 보이지 않게 분리한다.
