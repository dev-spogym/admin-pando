# 리드 · 회원 운영기획서

## 1. 문서 목적

- 리드 유입부터 회원 등록, 회원 상세 운영까지의 기준을 하나로 묶는다.
- FC, 매니저, 지점 운영자가 가장 자주 쓰는 흐름을 우선 정의한다.

## 2. 운영 대상

- 리드 관리
- 회원 목록
- 회원 등록/수정
- 회원 상세
- 회원 이관
- 상담/메모

## 3. 리드 운영 기준

### 3.1 리드 상태

| 운영 상태 | 의미 |
| --- | --- |
| 신규문의 | 아직 첫 접촉 전 |
| 연락완료 | 1차 연락 완료 |
| 상담예정 | 상담 일정 확정 또는 진행 중 |
| 방문완료 | 방문/체험/상담 완료 |
| 등록완료 | 회원 등록 또는 결제 전환 완료 |
| 보류 | 추후 재접촉 대상 |
| 미전환 | 종료된 리드 |

### 3.2 문의 유형과 경로 기준

04/29 기준으로 상담 퍼널은 문의 유형과 가입 경로를 분리해 본다.

| 구분 | 화면 표기 | 운영 의미 |
| --- | --- | --- |
| WI | 방문 문의 | 예약 없이 방문 상담으로 유입된 사람 |
| TI | 전화 문의 | 전화 문의 이후 상담으로 유입된 사람 |
| 체험 | 체험 | 체험 프로그램을 통해 유입된 사람 |
| 일일입장 | 일일입장 | 1회권 또는 일일 이용으로 유입된 사람 |

- 신규 DB 기준으로 C열 `경로`는 문의 유형으로 본다.
- D열 `상세 경로1`과 E열 `상세 경로2`는 간판, 인터넷, 홍보물, 회원소개 같은 세부 유입 경로로 본다.
- 리드 유입 경로와 실제 등록자의 가입경로는 다를 수 있으므로, 등록 완료 기준 가입경로 분포를 별도 표시한다.

### 3.3 리드 운영 원칙

- 리드는 전화번호를 기준으로 중복 여부를 먼저 확인한다.
- 이미 회원으로 존재하는 전화번호는 신규 리드 생성보다 "기존 회원 연결"을 우선 제안한다.
- 리드 담당자는 FC를 기본으로 하되, 지점 매니저가 재배정할 수 있다.
- `등록완료`로 전환되는 시점에 회원 생성 또는 기존 회원 연결이 반드시 발생해야 한다.
- `미전환`은 종료 상태이며, 재유입 시 기존 리드를 재활성화할지 새 리드로 받을지 지점 정책에 따라 선택한다.
- TI 문의는 전화 응대에서 끝났는지, 실제 대면 상담으로 이어졌는지를 분리해서 본다.
- WI 문의는 현장 방문 고객의 즉시 등록 전환 성과로 본다.

### 3.4 상담 퍼널 KPI

04/29 기준으로 FC와 온보딩 화면에는 아래 퍼널 지표를 분리해 표시한다.

| 지표 | 퍼블리싱 기준 |
| --- | --- |
| WI vs TI 건수 | 방문 문의와 전화 문의를 합산하지 않고 2분할 카드 또는 비교 막대로 표시 |
| TI 방문 전환율 | 전화 문의 중 1차·2차·3차 상담 방식에 `대면`이 기록된 비율 |
| TI 등록률 | 전화 문의 대상자 중 등록 완료 비율 |
| WI 등록률 | 방문 문의 대상자 중 등록 완료 비율 |
| 체험 -> 등록 전환율 | 체험 참여율과 분리해, 체험자가 본상품 등록으로 이어진 비율 |
| 일일입장 -> 회원 전환율 | 1회권 또는 일일 이용 고객이 회원 등록으로 전환된 비율 |
| 회원소개 유입 비중 | 세부 경로가 회원소개인 리드와 등록자의 비중 |
| 가입경로 분포 | 리드 전체가 아니라 등록 완료자를 기준으로 한 경로 분포 |

## 4. 회원 운영 기준

### 4.1 회원 상태

| 운영 상태 | 의미 |
| --- | --- |
| 활성 | 현재 이용 가능한 회원 |
| 홀딩 | 이용권 사용이 일시 정지된 회원 |
| 만료예정 | 이용권 만료가 임박한 회원 |
| 만료 | 이용권이 종료된 회원 |
| 비활성 | 탈퇴, 장기 중단, 관리 제외 회원 |
| 미등록 | 앱 연동 또는 일부 정보만 존재하고 정식 등록이 완료되지 않은 상태 |

### 4.2 회원 상태 원칙

- `미등록`은 온보딩 미완료 상태이며 `비활성`과 합치지 않는다.
- `만료예정`과 `만료`는 이용권 기준으로 계산되며, 회원 상세에서는 대표 배지로 노출한다.
- 홀딩은 이용권 종료일을 보정하는 운영 행위로 본다.
- 탈퇴는 회원 이력 삭제가 아니라 운영 종료 처리로 본다.

## 5. 핵심 화면 구조

### 5.1 회원 목록

- 기본 탭은 `전체 / 활성 / 만료예정 / 만료 / 홀딩 / 비활성`으로 구성한다.
- 상단 통계 카드는 오늘 운영에 필요한 숫자만 노출한다.
  - 전체 회원
  - 활성 회원
  - 만료예정 회원
  - 금일 상담 예정
- 리스트의 1차 컬럼은 이름, 연락처, 담당자, 상태, 이용권 현황, 최근 방문일로 구성한다.
- 고급 건강 데이터 탭은 목록 1차 진입 지표에서 제외한다.

### 5.2 회원 등록/수정

- 회원 등록은 `기본정보 -> 운영정보 -> 저장`의 단순 구조로 유지한다.
- 필수값은 이름, 연락처, 지점, 성별/생년월일 중 운영상 반드시 필요한 최소값으로 제한한다.
- 초기 등록 화면에서 과도한 상세정보 수집은 지양한다.

### 5.3 회원 상세

- 운영용 기본 탭 우선순위
  - 회원정보
  - 이용권
  - 결제이력
  - 예약/출석
  - 상담/메모
- 확장 탭
  - 체성분
  - 운동프로그램
  - 평가
  - 운동이력
- 운영상 가장 중요한 상단 영역에는 아래를 고정 노출한다.
  - 회원 상태
  - 대표 이용권 상태
  - 만료일 또는 홀딩 여부
  - 담당자
  - 빠른 액션 버튼

## 6. 빠른 액션 기준

- 회원 상세에서 바로 가능한 액션
  - 홀딩 등록/해제
  - 탈퇴 처리
  - 결제 등록 이동
  - 상담 등록
  - 예약 확인
  - 이용권 추가 구매
- 회원 목록 일괄 액션은 최소화한다.
- 상태 일괄 변경은 운영 실수 위험이 높으므로 매니저 이상 권한으로 제한한다.

## 7. 회원 이관/병합/가족회원 기준

- 지점 이관은 매니저 이상 승인 흐름을 둔다.
- 회원 병합은 중복 회원 정리용 고급 기능으로 두고, 운영 1차 핵심 흐름에서 분리한다.
- 가족회원은 마케팅/부가 혜택 연동까지 확정되기 전까지는 독립 회원 구조를 유지하고 관계만 표시한다.

## 8. 상담/메모 기준

- 상담과 일반 메모는 분리한다.
- 상담은 후속 액션과 다음 일정이 남는 구조여야 한다.
- 메모는 운영 참고 기록이며 필수 후속 일정은 없다.
- 리드 단계 상담과 회원 단계 상담은 같은 UX 패턴을 쓰되 대상 엔티티만 다르게 가져간다.
- FC 개인 KPI에서는 담당자별 상담 건수와 상담 -> 등록 전환율을 표시한다.
- 담당자별 상담 건수의 분모는 `등록`, `관리`, `실패` 상태를 모두 포함한 전체 상담 건수로 본다.
- 담당자별 상담 -> 등록 전환율은 담당자별 `등록` 건수 ÷ 담당자별 전체 상담 건수로 표시한다.

## 9. 운영 예외 처리

- 동일 번호로 신규 리드와 신규 회원 생성이 동시에 시도되면 중복 경고를 우선한다.
- 탈퇴 회원 재등록은 신규 생성보다 기존 회원 재활성화를 우선 검토한다.
- 만료 회원이 재결제하면 회원 상태는 즉시 활성로 전환되고 기존 이력은 유지한다.

## 10. 재등록/이탈 회복 기준

04/29 기준으로 재등록 관리는 이탈 정의가 완전히 확정되기 전까지 아래 두 지표를 우선 표시한다.

| 지표 | 퍼블리싱 기준 |
| --- | --- |
| 재등록 상담 완료 건수 | 재등록 대상자 중 실제 상담 결과가 기록된 건수 |
| 재등록 성공률 | `등록` ÷ `등록+확정+예상+관리+실패` |

- 만료 예정은 기존처럼 30일 이내 단일 지표를 유지한다.
- D-7/D-14/D-30 만료일별 대상자 분포 위젯은 추가하지 않는다.
- 환불 -> 재결제 복귀율은 재등록/이탈 회복 KPI에 추가하지 않는다.

## 11. 퍼블리싱 기준 메모

- 회원 리스트는 "한눈에 분류"가 중요하므로 상태 배지를 강하게 쓴다.
- 회원 상세는 탭 수가 많더라도 첫 화면에서 핵심 요약이 끝나야 한다.
- FC 화면은 리드와 만료예정 회원이 같은 업무 흐름 안에서 연결되도록 설계한다.
- FC 화면은 WI와 TI를 합산한 상담 전환율만 보여주지 않고, TI 방문 전환율·TI 등록률·WI 등록률을 분리한다.
- 온보딩 화면은 체험 참여율과 체험 -> 등록 전환율을 같은 지표처럼 보이게 하지 않는다.
