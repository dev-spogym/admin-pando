# 수업 · 예약 · 출석 운영기획서

## 1. 문서 목적

- 수업 생성부터 예약, 출석, 횟수 차감, 노쇼 처리까지 센터 운영의 실제 흐름을 정리한다.
- PT, GX, 골프 등 유형이 달라도 공통 운영 틀을 유지한다.

## 2. 운영 대상

- 캘린더/수업 일정
- 수업 관리
- 예약/대기예약
- 출석 처리
- 서명 처리
- 횟수 차감
- 노쇼/패널티

## 3. 수업 유형

- PT
- GX
- 골프 레슨
- 자유 예약형 시설 수업

### 3.1 강습 세션 유형

04/29 기준으로 KPI용 강습세션수는 기존 수업 유형과 별도 축으로 관리한다.

| 강습 세션 유형 | 운영 의미 |
| --- | --- |
| PT | 개인 트레이닝 세션 |
| 필라테스 | 필라테스 강습 세션 |
| 스트레칭 | 스트레칭 강습 세션 |
| 테라피 | 테라피 강습 세션 |

- 강습세션수 KPI는 PT, 필라테스, 스트레칭, 테라피 4종의 월간·주차별 수업수를 표시한다.
- GX 세부 종목 6종은 상품/매출 분류 축이고, 강습 세션 유형 4종과 섞지 않는다.
- 수업 등록/수정 화면에는 KPI 집계 대상 수업일 때 강습 세션 유형 선택값이 보이도록 한다.

## 4. 공통 상태 기준

### 4.1 수업 상태

| 상태 | 의미 |
| --- | --- |
| 예정 | 정상 운영 예정 |
| 마감 | 정원 완료 |
| 진행중 | 현재 진행 |
| 완료 | 정상 종료 |
| 취소 | 운영 취소 |
| 승인대기 | 본사/지점 승인 필요한 일정 |

### 4.2 예약 상태

| 상태 | 의미 |
| --- | --- |
| 예약완료 | 정상 예약 |
| 대기예약 | 빈자리 대기 |
| 취소 | 사용자가 취소 |
| 출석완료 | 실제 출석 |
| 노쇼 | 미방문 |

## 5. 운영 원칙

- 수업은 일정 단위, 예약은 좌석 단위, 출석은 실제 이용 단위다.
- 횟수 차감 시점은 수업 유형과 정책에 따라 다르지만 운영 문서에서는 "출석 확정 후 차감"을 기본값으로 둔다.
- 대기예약은 자동 승격 가능 구조를 유지한다.
- 예약 취소 가능 시간은 본사 기본 정책 + 지점 적용값 구조로 관리한다.
- 골프 레슨은 양방향 서명 정책을 지원하되 일반 PT는 강사 서명 중심으로 간소화한다.

## 6. 핵심 화면 구조

### 6.1 캘린더

- 운영자는 캘린더에서 오늘 수업, 정원, 미승인 일정, 강사 배치를 한눈에 봐야 한다.
- 월/주/일/리스트 뷰를 제공하되 기본 운영 뷰는 주간 또는 일간으로 둔다.
- 우측 상세 패널에서는 선택 날짜의 예약 현황과 빠른 수정 액션을 제공한다.

### 6.2 수업 상세

- 수업명
- 담당 강사
- 시간
- 정원/예약 현황
- 대상 상품
- 예약 가능한 회원군
- 취소 규칙
- 메모

### 6.3 출석 관리

- 출석은 아래 입력원을 모두 받을 수 있어야 한다.
  - 앱 QR 체크인
  - 키오스크
  - 관리자 수동 처리
  - IoT/게이트 기반 로그 연계
- 출석 화면에서는 오늘 기준의 예외 처리가 중요하다.
  - 미체크인 예약자
  - 지각
  - 노쇼 전환 대상
  - 수동 출석 요청

## 7. 운영 상세 규칙

### 7.1 횟수 차감

- 예약만으로 즉시 차감하지 않는다.
- 출석완료 또는 수업완료 기준으로 차감한다.
- 노쇼 차감 여부는 본사 정책 템플릿을 따르되, 지점이 허용 범위 안에서 선택 적용한다.

### 7.2 대기예약

- 결원 발생 시 자동 승격을 기본 지원한다.
- 자동 승격 시 알림이 함께 발송되어야 한다.
- 수동 승격도 가능하되 이력은 남겨야 한다.

### 7.3 서명

- 일반 PT/GX
  - 강사 서명만으로 종료 가능
- 골프 레슨
  - 강사 + 회원 쌍방서명 지원
  - 회원 부재 시 원격 후속 서명 플로우 지원

### 7.4 노쇼/패널티

- 노쇼는 단순 상태 변경이 아니라 정책성 운영 행위다.
- 차감, 경고, 패널티 포인트는 하나의 정책 세트로 표현한다.
- 패널티는 수업 운영 화면에서 갑자기 튀어나오는 것이 아니라 예약/출석 흐름 안에서 자연스럽게 안내되어야 한다.

### 7.5 OT 운영 기준

04/29 기준으로 OT는 정규 PT 세션과 분리한 영업 파이프라인으로 본다.

| 지표 | 퍼블리싱 기준 |
| --- | --- |
| OT 신규 배정 건수(강습) | 강습 OT 신규 배정 규모를 별도 카드로 표시 |
| OT 신규 배정 건수(골프) | 골프 OT 신규 배정 규모를 강습과 분리 표시 |
| OT 등록 전환율 | `등록 / OT 진행`으로 표시 |
| OT 거부율 | 거절 건은 `거부+보류`를 함께 집계했다고 툴팁에 명시 |
| OT 이월률 | 이월된 OT 재고 규모를 표시하되, 최종 분모는 확정 대기 항목으로 표시 |
| OT 체험 -> 등록률 | 일반 체험 -> 등록 전환율과 별도 지표로 표시 |
| 골프 OT 전체 현황 | 골프 OT는 별도 섹션 또는 탭으로 표시 |
| 트레이너별 OT 1/2차 완료·예정 | 트레이너별 배정, 1차, 2차, 완료·예정 상태를 표로 표시 |

- OT 등록 전환율은 일반 PT 완료율과 합산하지 않는다.
- OT 거부율의 거절 건은 04/29 기준으로 `거부+보류`를 함께 집계한다.
- OT 이월률의 분모는 우선 OT 배정 인원으로 표시하되, `OT 진행` 기준과 구분이 필요한 산식이므로 확정 대기 주석을 유지한다.

## 8. 관리자와 앱 연계 기준

- 앱 예약 화면과 관리자 캘린더는 같은 잔여 좌석 값을 본다.
- 앱에서 취소 가능한 시간 규칙은 관리자 정책과 동일해야 한다.
- 강사/트레이너 앱 흐름은 관리자 웹 수업 운영 기준과 충돌하지 않아야 한다.

## 9. 운영 예외 처리

- 강사 변경 시 기존 예약은 유지하고 강사만 교체할 수 있어야 한다.
- 수업 취소 시 예약자 일괄 알림을 기본 제공한다.
- 홀딩 회원은 예약 가능 여부를 상품 정책에 따라 막거나 경고한다.
- 만료예정 회원은 예약은 가능하되 수업일 기준 이용권 유효성 검사를 다시 한다.

## 10. 퍼블리싱 기준 메모

- 캘린더는 정보가 많아도 핵심은 시간표와 정원 상태다.
- 수업 상세 모달은 읽기보다 빠른 판단이 되도록 카드형 구조를 우선한다.
- 출석/노쇼 상태는 같은 색계열로 처리하지 말고 분명히 분리한다.
- 수업 운영 KPI에서는 PT 세션, GX 지표, 강습세션수 4종, OT 파이프라인을 같은 카드 그룹 안에서 섞지 않는다.
- 트레이너 개인 현황에는 OT 배정 합계와 OT 1/2차 완료·예정이 같이 보여야 업무 부하 편중을 판단할 수 있다.
