# 본사 · 지점 · 권한 운영기획서

## 1. 문서 목적

- 본사와 지점, 관리자 웹과 회원앱의 권한 구조를 실제 운영 기준으로 정리한다.
- 문서마다 달랐던 역할 명칭을 운영 기준으로 하나로 맞춘다.

## 2. 조직 레벨

| 레벨 | 역할 |
| --- | --- |
| 본사 | 전사 표준, 템플릿, 배포, 정책, 감사, 통합 KPI |
| 지점 | 회원 운영, 현장 판매, 수업 운영, 시설 운영 |
| 앱 사용자 | 회원 또는 현장 인력의 역할 기반 기능 사용 |

## 3. 관리자 웹 표준 역할

| 역할 | 설명 |
| --- | --- |
| superAdmin | 전사 최고 권한 |
| primary | 본사 운영 총괄 |
| owner | 지점 최고 책임자 |
| manager | 지점 운영 관리자 |
| fc | 상담/영업 담당 |
| staff | 현장 운영 담당 |
| readonly | 조회 전용 |

## 4. 역할 운영 원칙

- 관리자 웹 표준 역할은 위 7개를 기준으로 한다.
- `front`는 별도 표준 역할로 두지 않고 `staff`에 포함한다.
- `trainer`는 관리자 웹의 기본 역할이 아니라 앱 또는 수업 운영 관점의 역할로 본다.
- 권한은 "페이지 접근", "조회", "수정", "승인", "일괄처리"로 나눠서 본다.

## 5. 회원앱 표준 역할

| 역할 | 설명 |
| --- | --- |
| member | 일반 회원 |
| trainer | PT 트레이너 |
| golf_trainer | 골프 강사 |
| fc | 상담 담당 |
| staff | 현장 스태프 |

## 6. 본사와 지점의 소유권 분리

### 6.1 본사 소유

- 전사 권한 템플릿
- 공통 상품 마스터
- 자동화 정책 세트
- 알림 템플릿
- 공지 템플릿
- KPI 및 감사 기준

### 6.2 지점 소유

- 회원/리드 실제 운영
- 현장 결제와 미수금 처리
- 수업 개설과 스케줄 운영
- 시설 배정
- 본사 정책 범위 안에서의 적용 설정

## 7. 핵심 기능별 권한 기준

| 기능 | 최소 운영 권한 |
| --- | --- |
| 리드 등록/수정 | fc |
| 회원 등록/수정 | staff |
| 회원 상태 일괄 변경 | manager |
| 상품 배포 | primary 이상 |
| 결제 승인/환불 | manager 이상 |
| 지점 이관 승인 | manager 이상 |
| 자동화 정책 생성 | primary 이상 |
| 자동화 정책 적용 | owner 또는 manager |
| 권한 템플릿 관리 | superAdmin 또는 primary |

## 8. 화면 노출 기준

- 메뉴를 숨길지, 읽기전용으로 둘지는 기능별로 다르게 판단한다.
- 위험한 기능은 메뉴를 보여주되 액션 버튼을 잠그는 방식보다, 최초 진입부터 제한하는 편이 낫다.
- 조회 전용 역할은 다운로드 가능 범위를 별도로 정의한다.

## 9. 퍼블리싱 기준 메모

- 같은 화면이라도 권한에 따라 버튼 세트가 달라져야 한다.
- 관리자 웹 상단 컨텍스트에는 현재 조직 범위와 권한 수준이 보이도록 한다.
- 본사 기능과 지점 기능이 같은 화면에서 뒤섞이면 안 된다.

## 10. 운영 결론

- 관리자 웹은 "본사/지점 운영용 도구"
- 회원앱은 "회원/현장인력 사용 도구"
- 두 시스템의 역할 이름이 같더라도 같은 권한 체계로 간주하지 않는다.
