# 본사 · 지점 · 권한 운영기획서

## 1. 문서 목적

- 본사와 지점, 관리자 웹과 회원앱의 권한 구조를 실제 운영 기준으로 정리한다.
- 문서마다 달랐던 역할 명칭을 운영 기준으로 하나로 맞춘다.

## 2. 조직 레벨

| 레벨 | 역할 |
| --- | --- |
| 본사 | 전사 표준, 템플릿, 배포, 정책, 감사, 통합 KPI |
| 지점 | 회원 운영, 현장 판매, 수업 운영, 시설 운영 |
| 앱 사용자 | 회원 또는 현장 인력의 역할 기반 기능 사용 |

## 3. 관리자 웹 표준 역할

| 역할 | 설명 |
| --- | --- |
| superAdmin | 전사 최고 권한 |
| primary | 본사 운영 총괄 |
| owner | 지점 최고 책임자 |
| manager | 지점 운영 관리자 |
| fc | 상담/영업 담당 |
| staff | 현장 운영 담당 |
| readonly | 조회 전용 |

## 4. 역할 운영 원칙

- 관리자 웹 표준 역할은 위 7개를 기준으로 한다.
- `front`는 별도 표준 역할로 두지 않고 `staff`에 포함한다.
- `trainer`는 관리자 웹의 기본 역할이 아니라 앱 또는 수업 운영 관점의 역할로 본다.
- 권한은 "페이지 접근", "조회", "수정", "승인", "일괄처리"로 나눠서 본다.

## 5. 회원앱 표준 역할

| 역할 | 설명 |
| --- | --- |
| member | 일반 회원 |
| trainer | PT 트레이너 |
| golf_trainer | 골프 강사 |
| fc | 상담 담당 |
| staff | 현장 스태프 |

## 6. 본사와 지점의 소유권 분리

### 6.1 본사 소유

- 전사 권한 템플릿
- 공통 상품 마스터
- 자동화 정책 세트
- 알림 템플릿
- 공지 템플릿
- KPI 및 감사 기준

### 6.2 지점 소유

- 회원/리드 실제 운영
- 현장 결제와 미수금 처리
- 수업 개설과 스케줄 운영
- 시설 배정
- 본사 정책 범위 안에서의 적용 설정

## 7. 본사 통합 KPI 기준

04/29 기준으로 본사 전용 통합 대시보드는 지점 일반 대시보드와 분리한다. 본사 통합 대시보드는 전사 관점에서 지점 간 편차, 위험 지점, 유지율을 보는 화면이다.

| 지표 | 퍼블리싱 기준 |
| --- | --- |
| 전사 환불 규모 | 전지점 환불 총액과 지점별 환불 분산을 함께 표시 |
| 전사 재등록률 | 전지점 재등록 성공률을 카드 또는 추이로 표시 |
| 지점 간 매출 편차 | 지점별 매출 막대 옆에 표준편차 또는 분산도 카드를 표시 |

- 전사 환불 규모는 환불 집중 지점을 조기 포착하는 경고 지표로 본다.
- 전사 재등록률은 본사 차원의 리텐션 프로그램 효과를 보는 유지율 지표로 본다.
- 지점 간 매출 편차는 단순 순위가 아니라 운영 표준화·교육 필요성을 판단하는 지표로 본다.

## 8. 지점 KPI 대시보드 기준

지점 KPI 대시보드는 현장 운영자가 이번 달 성과와 병목을 보는 화면이다.

| 영역 | 퍼블리싱 기준 |
| --- | --- |
| 상담 퍼널 | WI vs TI 건수, TI 방문 전환율, TI 등록률, WI 등록률 |
| 재등록/이탈 회복 | 재등록 상담 완료 건수, 재등록 성공률 |
| GX | GX 수업 KPI와 GX 세부 종목 6종 분포를 분리 표시 |
| 강습세션수 | PT, 필라테스, 스트레칭, 테라피 4종의 월간·주차별 수업수 |
| OT/수업운영 | 강습 OT, 골프 OT, OT 등록 전환율, OT 거부율, OT 이월률, OT 체험 등록률 |
| 운영 모니터링 | 이번 달 1~6주차 매출 추이를 부서별로 표시 |

- 주차별 매출 추이는 월별 KPI 추이와 다르게, 월중 목표 달성 가능성을 조기 판단하기 위한 미니 차트로 표시한다.
- 강습세션수는 PT 세션 하나로 뭉치지 않고 4종 세션 유형을 분리한다.
- GX 세부 종목 분포는 요가, 필라테스, 스피닝, 줌바, 에어로빅, GX 기타 6종으로 고정한다.

## 9. 개인 KPI 기준

04/29 기준으로 개인 KPI는 담당자별 업무량과 전환 성과를 같이 보도록 구성한다.

| 대상 | 퍼블리싱 기준 |
| --- | --- |
| 트레이너 | OT 배정 합계, 신규 매출, 재등록 매출, OT 등록 전환율, OT 체험 등록률 |
| FC | 상담 건수, 상담 -> 등록 전환율, TI 방문 전환율, TI 등록률, WI 등록률 |
| 골프프로 | 골프프로별 매출 |

- 트레이너별 신규/재등록 매출의 공식 기준은 매출 거래 단위로 본다.
- FC 상담 건수의 분모는 등록, 관리, 실패 상태를 모두 포함한다.
- 환불 책임 지표가 필요한 경우 처리자가 아니라 원판매 담당자 기준으로 본다.

## 11. 화면 노출 기준

- 메뉴를 숨길지, 읽기전용으로 둘지는 기능별로 다르게 판단한다.
- 위험한 기능은 메뉴를 보여주되 액션 버튼을 잠그는 방식보다, 최초 진입부터 제한하는 편이 낫다.
- 조회 전용 역할은 다운로드 가능 범위를 별도로 정의한다.
- 본사 통합 대시보드는 지점 운영자가 보는 일반 KPI 대시보드와 메뉴·권한·카드 구성을 분리한다.
- 개인 KPI는 본인 실적용 화면과 관리자 평가용 화면의 노출 범위를 다르게 설계한다.

## 12. 퍼블리싱 기준 메모

- 같은 화면이라도 권한에 따라 버튼 세트가 달라져야 한다.
- 관리자 웹 상단 컨텍스트에는 현재 조직 범위와 권한 수준이 보이도록 한다.
- 본사 기능과 지점 기능이 같은 화면에서 뒤섞이면 안 된다.
- 본사 통합 대시보드는 카드 수를 늘리는 것보다 전사 위험, 편차, 유지율을 빠르게 읽게 하는 구성이 우선이다.
- 지점 KPI 대시보드는 상담, 매출, 수업, OT, 재등록을 한 화면에서 보되 섹션 경계를 명확히 둔다.

## 13. 운영 결론

- 관리자 웹은 "본사/지점 운영용 도구"
- 회원앱은 "회원/현장인력 사용 도구"
- 두 시스템의 역할 이름이 같더라도 같은 권한 체계로 간주하지 않는다.
