# FitGenie CRM 운영용 기획문서

## 목적

- 이 폴더는 기존 `docs`를 수정하지 않고, 실제 운영과 퍼블리싱 기준으로 바로 사용할 수 있도록 재정리한 운영용 문서 세트다.
- 기존 `docs/기능명세서`, `docs/화면설계서`, `docs/다이어그램`, `docs/회원앱`은 참고 원본으로 유지한다.
- 이 문서 세트는 화면 구조, 상태, 권한, 운영 흐름, 알림 정책을 통합한 기준서이며 백엔드 라우터 연결 상세는 범위에서 제외한다.

## 사용 원칙

- 기획, 디자인, 퍼블리싱, QA는 이 폴더 문서를 우선 기준으로 본다.
- 기존 문서와 충돌하면 운영 기준 판단은 이 폴더를 따른다.
- 단, 데이터 모델 구현 방식이나 API 연결 방식은 별도 개발 판단 영역으로 둔다.
- 본사 정책과 지점 운영 권한을 분리해서 본다.

## 문서 구성

- `00_통합운영_개요.md`
  실제 운영 기준의 범위, 서비스 골격, 공통 원칙
- `01_리드_회원_운영기획서.md`
  리드 유입부터 회원 등록, 회원 상세 운영까지의 기준
- `02_상품_결제_이용권_운영기획서.md`
  상품, 계약, 결제, 이용권, 환불, 미수금 운영 기준
- `03_수업_예약_출석_운영기획서.md`
  수업 운영, 예약, 대기예약, 출석, 서명, 노쇼 정책
- `04_시설_부가서비스_운영기획서.md`
  락커, RFID, 룸, 타석, 운동복 등 부가 운영 기준
- `05_본사_지점_권한_운영기획서.md`
  조직 체계, 역할, 메뉴 노출, 정책 소유권 기준
- `06_자동화_만료알림_운영기획서.md`
  자동화, 만료 알림, 본사 정책 세트, 지점 적용 규칙
- `07_회원앱_운영기획서.md`
  회원앱 역할별 운영 범위와 관리자 기능 연계 기준
