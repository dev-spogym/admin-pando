# FitGenie CRM 통합 운영 개요

## 1. 문서 목적

- 이 문서는 관리자 웹과 회원앱을 포함한 전체 서비스의 운영 기준을 한 번에 파악하기 위한 상위 문서다.
- 목표는 "기능이 많다"가 아니라 "센터가 실제로 굴러간다"를 기준으로 문서를 정렬하는 데 있다.

## 2. 범위

- 포함
  - 리드, 회원, 상품, 결제, 이용권, 수업, 출석, 시설, 자동화, 회원앱
  - 본사와 지점의 권한 차이
  - 화면 노출 우선순위와 상태 기준
- 제외
  - 백엔드 라우터 연결
  - DB 스키마 확정
  - 외부 결제사, 알림사, IoT 연동 상세 구현

## 3. 서비스 골격

```text
리드 유입
-> 상담/후속관리
-> 회원 등록
-> 상품 선택/계약/결제
-> 이용권 생성
-> 예약/출석/이용
-> 만료 임박 알림
-> 재등록/업셀
-> 환불/종료/휴면
```

## 4. 운영 기준 원칙

### 4.1 단일 운영 기준

- 상태값은 운영 기준으로 한 벌만 쓴다.
- 화면 라벨과 내부 enum이 다르더라도, 기획 문서에서는 먼저 운영 라벨을 고정한다.
- 같은 개념이 문서마다 다른 이름으로 등장하면 이 문서 세트 기준으로 통합한다.

### 4.2 본사와 지점의 역할 분리

- 본사는 정책, 템플릿, 표준, 기본값을 관리한다.
- 지점은 본사가 허용한 범위 안에서 적용하고 운영한다.
- 지점이 전사 기준을 새로 만드는 구조는 피한다.

### 4.3 운영 중심 화면 우선순위

- 1순위: 리드, 회원, 결제, 이용권, 출석, 예약, 만료 알림
- 2순위: 수업 통계, 시설 운영, 직원 운영, 본사 KPI
- 3순위: 고도화 탭, 실험적 대시보드, 확장형 분석 기능

### 4.4 퍼블리싱 기준

- 화면은 실제 센터 운영 동선을 따라 설계한다.
- 한 화면에서 처리해야 할 핵심 행동은 최대 1~2개로 제한한다.
- 운영자가 많이 보는 리스트 화면은 필터, 상태 배지, 즉시 액션을 우선한다.
- 상세 화면은 탭이 많아도 핵심 탭 우선순위를 명확히 둔다.

## 5. 핵심 엔티티 운영 기준

| 엔티티 | 운영 기준 |
| --- | --- |
| 리드 | 아직 회원 등록 전인 잠재 고객 |
| 회원 | 실제 센터에서 관리하는 등록 대상 |
| 상품 | 판매 단위 |
| 결제 | 실제 금액 수납 단위 |
| 이용권 | 사용 권한 단위 |
| 수업 | 예약과 출석이 발생하는 운영 단위 |
| 시설자산 | 락커, 카드, 룸, 타석, 운동복 등 물리 자산 |
| 자동화 정책 | 본사가 정의하고 지점이 선택 적용하는 알림/후속 규칙 |

## 6. 공통 상태 원칙

- `미등록`과 `비활성`은 같은 뜻으로 취급하지 않는다.
- `만료예정`은 회원 전체 상태라기보다 이용권/계약 상태에 가까운 개념으로 본다.
- `환불`, `미수금`, `부분환불`은 결제 상태이며 회원 상태와 분리한다.
- `보류`는 운영 대기 상태이고 `미전환`은 종료 상태다.

## 7. 화면 구조 원칙

- 리스트 화면: 현황 확인 + 빠른 액션
- 등록/수정 화면: 검증과 저장
- 상세 화면: 이력과 운영 처리
- 설정 화면: 정책과 기본값 관리
- 앱 화면: 회원 또는 현장 인력이 일상적으로 쓰는 단순 흐름

## 8. 이번 운영용 문서에서 확정한 사항

- 리드 상태 체계를 운영 기준으로 하나로 통일한다.
- 회원/결제/이용권 상태를 서로 다른 의미로 분리한다.
- 관리자 웹 역할과 회원앱 역할을 분리 정의한다.
- 만료 알림은 고정 D-day가 아니라 본사 정책 세트 방식으로 설계한다.
- 기존 문서의 누락 화면이나 깨진 참조는 이번 문서 세트에서 운영 우선순위 중심으로 재해석한다.
- 04/29 기준으로 매출, 상담 퍼널, OT, 재등록, 본사 통합 KPI의 추가 지표를 운영용 기획서에도 반영한다.
- 04/29 추가 지표는 우선 퍼블리싱 검수 기준으로 적용하며, 외부 템플릿·DB 연동은 별도 구현 단계에서 확정한다.

## 9. 실무 적용 방식

1. 운영 요구사항 검토는 이 문서 세트에서 결정한다.
2. 퍼블리싱 화면 구조는 이 문서 세트 기준으로 잡는다.
3. 기존 원본 문서 반영이 필요하면 별도 작업으로 옮긴다.

## 10. 04/29 퍼블리싱 선반영 범위

아래 항목은 화면에 카드, 탭, 표 컬럼, 차트, 툴팁 중 하나로 먼저 표현한다. 실제 수집 원천과 자동 집계는 후속 연동 단계에서 검증한다.

| 구분 | 퍼블리싱 반영 기준 |
| --- | --- |
| 매출/재무 | 위약금 합계, VAT 제외 순매출, 법인권 매출, 개월별 판매 비율, 담당자별 환불 책임 |
| 상품/GX | 상품 마스터 1단계 `category`와 2단계 `subCategory`를 구분하고, GX는 요가·필라테스·스피닝·줌바·에어로빅·GX 기타 6종으로 표시 |
| 상담 퍼널 | WI는 방문 문의, TI는 전화 문의로 화면 표기하고, TI 방문 전환율·TI 등록률·WI 등록률을 분리 |
| OT/수업 | 강습 OT와 골프 OT를 분리하고, OT 등록 전환율·거부율·이월률·OT 체험 등록률을 일반 PT 지표와 분리 |
| 개인 KPI | 트레이너별 OT 배정 합계, 트레이너별 신규/재등록 매출, FC별 상담 건수·상담 등록 전환율, 골프프로별 매출 |
| 재등록/이탈 회복 | 재등록 상담 완료 건수와 재등록 성공률을 표시하고, 환불 -> 재결제 복귀율과 만료일별 D-7/D-14/D-30 분포는 추가하지 않음 |
| 본사 통합 KPI | 전사 환불 규모, 전사 재등록률, 지점 간 매출 편차를 본사 전용 통합 대시보드에서 표시 |

## 11. 04/29 제외 확정 항목

- 환불관리의 재결제 건수와 재결제율은 운영용 KPI에서 제외한다.
- 취소재결제 건수는 신규 매출 KPI로 추가하지 않는다.
- 매입사별 수수료 비교는 현재 퍼블리싱 범위에서 제외한다.
- 환불 -> 재결제 복귀율은 재등록/이탈 회복 KPI와 환불관리 확장 지표에 추가하지 않는다.
- 만료일별 대상자 분포 D-7/D-14/D-30 위젯은 추가하지 않고, 기존 30일 이내 만료 예정 단일 지표를 유지한다.
