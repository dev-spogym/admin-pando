## 권한/직무 해석 원칙

- 관리자웹 권한 키는 `superAdmin / primary / owner / manager / fc / staff / readonly`를 기준으로 해석한다.
- 본문에서 사용하는 `최고관리자`, `센터장`, `매니저`, `FC`, `스태프`는 운영 라벨 또는 예시 명칭으로 병기될 수 있다.
- `트레이너`, `프런트` 같은 표현은 매출 귀속 주체나 현장 직무를 뜻하며, 관리자웹 기본 권한 키와 동일 개념이 아니다.
- 실제 메뉴 접근 범위는 `SCR-081 권한 설정` 또는 커스텀 역할 매핑 기준으로 결정한다.

## 1. 매출 현황

**화면 경로**: `/sales`

### 화면 목적

센터에서 발생한 전체 매출 거래를 한눈에 조회하고 분석하는 매출관리 핵심 화면입니다. 당일·이번 주·이번 달 매출을 즉시 확인하고, 구매자나 상품명으로 거래를 검색하며, 기간별·상품별·결제수단별·담당자별 집계와 환불·미납 내역을 탭으로 관리합니다.

### 주요 기능

- **요약 지표**: 순 매출 / 카드 결제 / 현금 결제 / 미수금을 상단 카드로 표시합니다.
- **기간 프리셋**: 오늘 / 이번 주 / 이번 달 버튼으로 조회 기간을 즉시 설정합니다.
- **통합 검색**: 구매자 이름 또는 상품명을 입력해 실시간으로 결과를 좁힙니다.
- **복합 필터**: 유형(이용권·PT·상품·기타), 상태(완료·환불·미납), 날짜 범위를 조합해 적용합니다.
- **매출유형 필터**: 전체·신규·재등록·휴면복귀·종목추가·업그레이드 중 하나를 선택합니다.
- **분석 탭 7종**: 매출 내역 / 기간별 / 상품별 / 결제수단별 / 담당자별 / 환불 내역 / 미납 내역을 탭으로 전환합니다. 환불·미납 탭에는 해당 건수 배지가 표시됩니다.
  - 담당자별 탭은 총매출, 신규 매출, 재등록 매출, 기타 매출, 판매 건수, 평균 단가를 함께 표시합니다.
- **매출 목록 테이블**: 구매일·매출유형·유형·상품명·구매자·담당자·금액·결제수단·미수금·상태 등을 제공합니다. 상품명 클릭 시 상품 상세로, 구매자 클릭 시 회원 상세로 이동합니다.
- **하단 요약 바**: 현재 필터 기준 총매출·순매출·현금·카드·환불·미납금·할인 합계를 한 줄로 표시합니다.
- **매출 상세 모달**: 테이블 행 클릭 시 결제일·상태·구매 정보·금액·결제 내역·카드 승인 정보·메모를 팝업으로 확인합니다.
- **신규 결제 버튼**: POS 판매 화면으로 바로 이동합니다.
- **엑셀 내보내기**: 현재 필터가 적용된 매출 내역 전체를 파일로 내려받습니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 엑셀 내보내기, 신규 결제(POS) 이동, 모든 필터·탭 사용 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 엑셀 내보내기, 신규 결제(POS) 이동, 모든 필터·탭 사용 |
| FC | 담당 회원 결제 처리 가능 | 신규 결제(POS) 이동, 조회 및 필터 사용 |
| 트레이너 / 스태프 | 소속 지점 매출 조회 | 조회·필터·탭 전환만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 매출 현황" 진입 → 이번 달 기간이 기본 설정됩니다.
2. 상단 지표 카드로 순매출·결제수단별 합계·미수금 현황을 파악합니다.
3. 기간 프리셋 또는 날짜 직접 입력으로 조회 범위를 조정합니다.
4. 검색·필터·매출유형 탭을 조합해 원하는 거래를 찾습니다.
5. 분석 탭을 전환해 기간별·상품별·결제수단별·담당자별 집계를 확인합니다.
6. 테이블 행 클릭 → 상세 모달에서 결제 내역 전체를 확인합니다.
7. 엑셀 버튼으로 현재 조회 결과를 파일로 저장합니다.

### 비즈니스 규칙

- 기본 조회 기간은 이번 달 전체이며, 기간 변경은 클라이언트 필터링으로 즉시 반영됩니다.
- 환불 건수는 순매출 계산에서 제외됩니다. (순매출 = 총매출 - 환불액)
- 집계 탭(기간별·상품별·결제수단별·담당자별)에서는 행 클릭이 비활성화됩니다.
- 매출유형 탭(신규·재등록 등)은 매출 내역 탭에서만 동작합니다.
- 담당자별 신규/재등록 매출은 `[매출종합]` E열 `구분/유형`, F열 `결제금액`, K열 `담당자/등록강사`를 기준으로 산출합니다.
- 회원권·락카 매출은 K열 `담당자`, 강습·골프 매출은 K열 `등록강사`를 판매 귀속 담당자로 사용합니다.
- 엑셀 내보내기 컬럼: No·구매일·매출유형·유형·상품명·구매자·담당자·수량·정가·판매가·할인·결제수단·현금·카드·마일리지·미수금·카드사·카드번호·승인번호·서비스일수·서비스횟수·상태·메모.

## 2. POS 판매

**화면 경로**: `/pos`

### 화면 목적

센터 현장에서 직원이 상품을 선택하고 구매자를 지정해 결제 화면으로 넘기는 현장 판매(POS) 화면입니다. 이용권·PT·GX 등 상품을 장바구니에 담고 현금가·카드가를 선택한 뒤 결제 처리 화면으로 이동합니다.

### 주요 기능

- **카테고리 탭**: 이용권·PT·GX·기타 탭으로 상품을 구분하며 각 탭에 상품 수 배지를 표시합니다.
- **상품 검색**: 상품명으로 실시간 검색합니다.
- **상품 카드 그리드**: 상품 유형 배지·상품명·기간·횟수·현금가·카드가를 카드로 표시합니다. 클릭하면 장바구니에 추가됩니다.
- **장바구니 패널**: 담긴 상품 목록, 수량 조절(-/+), 현금/카드 가격 유형 전환, 개별 삭제, 전체 삭제 기능을 제공합니다.
- **구매자 선택**: 회원 검색 모달에서 이름 또는 전화번호로 회원을 지정합니다. 비워두면 비회원으로 처리합니다.
- **총 합계 및 결제하기**: 장바구니에 상품이 1개 이상일 때 결제 버튼이 활성화됩니다. 클릭 시 결제 처리 화면으로 이동합니다.
- **상품 등록 단축 버튼**: 상품 등록 화면으로 바로 이동합니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 상품 선택, 장바구니 구성, 구매자 지정, 결제 이동, 상품 등록 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 상품 선택, 장바구니 구성, 구매자 지정, 결제 이동, 상품 등록 |
| FC | 담당 회원 결제 처리 가능 | 상품 선택, 장바구니 구성, 구매자 지정, 결제 이동 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | POS 화면 접근 불가 또는 조회만 |

### 주요 UX 흐름

1. 사이드바 또는 매출 현황의 "신규 결제(POS)" 버튼으로 진입합니다.
2. 카테고리 탭 선택 또는 상품명 검색으로 원하는 상품을 찾습니다.
3. 상품 카드 클릭 → 장바구니에 추가. 동일 상품 재클릭 시 수량 1 증가.
4. 장바구니에서 수량·가격 유형(현금/카드)을 조정합니다.
5. 필요 시 구매자 검색으로 회원을 지정합니다.
6. 총 합계 확인 후 "결제하기" 클릭 → 결제 처리 화면으로 이동합니다.

### 비즈니스 규칙

- 동일 상품을 카드 클릭으로 추가하면 수량이 증가하고, 장바구니에서 다시 추가하면 별도 행으로 추가됩니다.
- 품절(재고 0) 상품은 클릭이 비활성화되고 흐리게 표시됩니다.
- 마일리지 결제는 결제 처리 화면에서 진행하며, POS 화면에서 구매자를 지정해야 합니다.
- 장바구니 정보는 결제 처리 화면으로 전달된 후 즉시 초기화됩니다.

## 3. 결제 처리

**화면 경로**: `/pos/payment`

### 화면 목적

POS 화면에서 담아온 상품의 결제수단을 선택하고 최종 결제를 완료하는 화면입니다. 결제 완료 후 영수증 출력·문자 발송·계속 판매 등의 후속 동작을 안내합니다.

### 주요 기능

- **결제 상품 목록**: POS에서 담아온 상품명·수량·소계와 총 결제 금액을 표시합니다.
- **회원 검색**: 이름·전화번호로 회원을 검색해 보유 마일리지를 확인하고 지정합니다.
- **결제수단 선택 (4종)**: 카드 / 현금 / 마일리지 / 복합결제 중 하나를 선택합니다.
  - **복합결제**: 카드·현금·마일리지 금액을 각각 입력하며 합계가 총액과 일치해야 결제 버튼이 활성화됩니다.
  - **마일리지**: 회원 미선택 시 결제 버튼 비활성화. 보유 마일리지 부족 시 부족액 표시.
- **결제 확인 버튼**: 모든 조건 충족 시 활성화됩니다.
- **결제 확인 모달**: 상품 내역·합계·결제수단·회원을 최종 확인하고 처리합니다.
- **중복 결제 경고**: 동일 회원·금액 중복 감지 시 경고 다이얼로그로 진행 여부를 선택합니다.
- **결제 완료 화면**: 결제 성공 시 완료 메시지와 영수증 출력·문자 발송·계속 판매·매출 현황 이동 버튼을 제공합니다.
- **초기화 버튼**: 전체 결제 상태를 리셋합니다.
- **이전으로 돌아가기**: POS 판매 화면으로 돌아갑니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 모든 결제수단 처리, 회원 검색, 영수증 출력, 문자 발송 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 모든 결제수단 처리, 회원 검색, 영수증 출력, 문자 발송 |
| FC | 담당 회원 결제 처리 가능 | 모든 결제수단 처리, 회원 검색 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 결제 처리 화면 접근 불가 |

### 주요 UX 흐름

1. POS 판매 화면에서 "결제하기" 클릭 → 장바구니 정보를 불러와 진입합니다.
2. 결제수단을 선택합니다.
3. 마일리지·복합결제 선택 시 회원 검색 후 지정합니다.
4. 복합결제는 금액 합계를 총액에 맞춥니다.
5. "결제 확인" 클릭 → 확인 모달에서 최종 검토 후 "결제 완료"를 누릅니다.
6. 중복 결제 감지 시 경고 다이얼로그에서 진행 여부를 선택합니다.
7. 결제 성공 → 완료 화면에서 영수증 출력·문자 발송·계속 판매 등을 선택합니다.

### 비즈니스 규칙

- 마일리지 결제는 회원 지정이 필수입니다. 보유 마일리지가 총액보다 적으면 처리가 불가합니다.
- 복합결제에서 마일리지를 포함하면 회원 지정이 필수입니다.
- 결제 처리 중에는 버튼이 비활성화되어 중복 처리를 방지합니다.
- 마일리지 결제 외 모든 결제수단에 대해 결제 금액의 1%가 마일리지로 자동 적립됩니다.
- 복합결제 내역은 메모란에 "복합결제 - 카드: N원, 현금: N원, 마일리지: NP" 형태로 자동 기록됩니다.
- 영수증 출력은 인쇄 기능을 통해 처리됩니다. 문자 발송은 처리 후 완료 안내를 제공합니다.

## 4. 매출 통계

**화면 경로**: `/sales/stats`

### 화면 목적

상품별·상품 유형별·결제수단별·종목별·개월별 판매 비율·GX 세부 종목·법인권·직군별·담당자별로 매출 데이터를 시각적으로 분석하는 통계 화면입니다. 차트와 표를 통해 어떤 상품이 잘 팔리는지, 어떤 결제수단이 주로 사용되는지, 단기/장기 회원권 판매 믹스가 어떤지, GX 하위 종목별 수요가 어떤지, 법인권(B2B) 매출이 어느 정도인지, 어느 담당자의 매출이 높은지를 파악합니다.

### 주요 기능

- **최상위 탭 3종**: 전체 통계 / 직군별 / 담당자별을 전환합니다.
- **날짜 필터**: 이번 달·지난 달·최근 3개월·최근 6개월·올해 프리셋과 날짜 직접 입력을 제공합니다. 조회 버튼으로 반영합니다.
- **하위 탭 7종 (전체 통계)**: 상품별(가로 막대 차트, 상위 10개) / 상품타입별(도넛 차트) / 결제수단별(도넛 차트) / 종목별(가로 막대 차트) / 개월별(1·3·6·12·기타 비율) / GX종목별(6종 분포) / 법인권(가로 막대 차트 또는 요약 테이블)을 선택합니다.
- **전월 대비 토글**: 켜면 전월 동기 비교 데이터를 추가 표시합니다. 증가는 초록, 감소는 빨강으로 강조합니다.
- **총 매출 표시**: 현재 필터 기준 총 매출액을 차트 영역 상단에 표시합니다.
- **집계 테이블**: 항목명·판매 건수·매출액·비율(%) 컬럼을 제공합니다. 담당자별 탭에는 신규 매출, 재등록 매출, 기타 매출, 평균 단가 컬럼이 추가됩니다.
- **개월별 판매 비율**: `[주간월간보고]` 주차별 V5:Z5 값을 1개월·3개월·6개월·12개월·기타 버킷으로 매핑해 전체 판매 대비 비율을 표시합니다.
- **GX 세부 종목 분포**: 상품 마스터의 `category=GX`와 2단계 `subCategory` 값을 기준으로 요가·필라테스·스피닝·줌바·에어로빅·GX 기타 6종 판매 분포를 표시합니다. 외부 원천은 `[월매출]` C열과 `[주간월간보고]` 주차별 T5:U5를 매핑합니다.
- **법인권 매출**: `[월매출]` 시트 E열 `구분` 값이 `법인권`인 거래를 별도 집계해 B2B 계약·갱신성 매출 규모를 확인합니다.
- **담당자별 신규/재등록 매출**: `[매출종합]`의 판매 귀속 담당자별 신규 매출과 재등록 매출을 분리해 표시합니다.
- **골프프로별 매출**: `[매출종합]` `골프매출` 탭 K열 `등록강사` 기준으로 골프 프로 개인별 매출을 표시합니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 모든 탭·차트·날짜 필터 사용 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 모든 탭·차트·날짜 필터 사용 |
| FC | 담당 회원 결제 처리 가능 | 조회·필터 사용 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 매출 통계" 진입합니다.
2. 최상위 탭(전체·직군별·담당자별)을 선택합니다.
3. 전체 통계 탭에서 날짜 프리셋 또는 직접 입력 후 조회합니다.
4. 하위 탭(상품별·상품타입별·결제수단별·종목별·개월별·GX종목별·법인권)으로 분석 뷰를 전환합니다.
5. 전월 대비 토글로 지난달과 비교합니다.
6. 집계 테이블에서 수치를 정밀 확인합니다.

### 비즈니스 규칙

- 환불 건은 통계 계산에서 제외됩니다.
- 가로 막대 차트는 상위 10개 항목만 표시합니다.
- 집계 결과는 매출액 내림차순으로 정렬됩니다.
- 개월별 판매 비율은 1개월·3개월·6개월·12개월·기타 5개 버킷으로 고정하며, 단기(1개월) 비중이 높을수록 락인 약화 신호로 해석합니다.
- 개월별 판매 비율 원천은 `[주간월간보고]` 주차별 V5:Z5이며, 매출 원장에 `durationMonths`가 있는 경우 해당 값을 우선 사용합니다.
- GX 세부 종목은 상품 마스터 2단계 분류(`category=GX` + `subCategory`)를 기준으로 요가·필라테스·스피닝·줌바·에어로빅·GX 기타 6종으로 고정합니다.
- GX 세부 종목 분포는 상품 마스터의 `subCategory`를 우선 사용하고, 외부 시트 연동 시 `[월매출]` C열 상품/종목명과 `[주간월간보고]` 주차별 T5:U5를 함께 참조합니다.
- 법인권 매출은 `[월매출]` E열 `구분`이 `법인권`인 거래의 매출액 합계로 산출하며, 일반 회원권/B2C 매출과 별도 집계합니다.
- 담당자별 신규/재등록 분리 매출은 `[매출종합]` E열 `구분/유형` 값이 `신규`, `재등록`인 거래의 F열 `결제금액`을 K열 판매 귀속 담당자별로 합산합니다.
- 트레이너별 신규/재등록 매출의 공식 원천은 `[매출종합]` 거래 단위 데이터이며, `[강습업무보고]` 주간보고 J열 `주간신규`, K열 `주간재등록`은 검산 또는 KPI 센터 운영 보조 원천으로만 사용합니다.
- 골프프로별 매출은 `[매출종합]` `골프매출` K열 `등록강사`별 F열 `결제금액` 합계로 산출합니다.
- 직군은 FC·스태프를 "프론트", 매니저·센터장을 "매니저", 트레이너를 "트레이너"로 구분합니다.
- 전월 데이터는 전월 대비 토글을 켤 때만 별도로 조회합니다.

## 5. 통계 관리

**화면 경로**: `/sales/statistics-management`

### 화면 목적

수업·매출·예약·이용 현황을 하나의 화면에서 통합 조회하고 CSV 파일로 내려받는 통계 보고서 화면입니다. 매출현황의 `받을금액`은 VAT 제외 순매출로 명확히 정의해 재무 보고와 수익성 분석에 활용합니다. 레거시 업무 시스템 스타일로 설계되어 기존 PC 기반 관리 프로그램에 익숙한 운영자도 빠르게 활용할 수 있습니다.

### 주요 기능

- **기간 입력**: 시작일~종료일 직접 입력. 기본값은 이번 달 전체입니다.
- **드롭다운 필터 4종**: 수업 유형 / 담당자 / 구분 / 회원 상태. 각 선택지는 현재 조회 데이터에서 동적으로 생성됩니다.
- **현황 탭 4종**: 수업현황·매출현황·예약현황·이용현황을 탭으로 전환합니다.
- **통계 테이블**: 일자·담당자·회원명·성별·휴대폰·구분·판매금액·할인금액·받을금액(VAT 제외 순매출)·카드·현금·상품권·포인트·미수발생·미수입금·받은금액·비고 컬럼을 제공합니다.
- **VAT제외 매출**: `[월매출]` 시트 I열 `ARRAYFORMULA` 산출값을 `받을금액`에 매핑하며, 부가세를 제외한 순매출 기준으로 표시합니다.
- **합계 행**: 테이블 하단에 전체 건수와 금액 합계를 노란색 배경으로 표시합니다.
- **조회 버튼**: 설정한 기간과 필터로 데이터를 불러옵니다.
- **신규 버튼**: 모든 필터와 조회 결과를 초기화합니다.
- **CSV 내보내기**: 현재 결과를 CSV 파일로 내려받습니다. 파일명에 탭명과 기간이 포함됩니다.
- **종료 버튼**: 이전 화면으로 돌아갑니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 기간 조회, 모든 필터, 탭 전환, CSV 내보내기 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 기간 조회, 모든 필터, 탭 전환, CSV 내보내기 |
| FC | 담당 회원 결제 처리 가능 | 기간 조회, 필터, 탭 전환 |

1. 사이드바 "매출관리 > 통계 관리" 진입합니다.
2. 시작일~종료일 입력 후 필터를 설정합니다.
3. "조회" 버튼으로 데이터를 불러옵니다.
4. 현황 탭(수업·매출·예약·이용)을 전환하며 원하는 뷰를 확인합니다.
5. 합계 행에서 건수와 VAT 제외 순매출을 포함한 금액 총계를 확인합니다.
6. "엑셀" 버튼으로 CSV 파일을 내려받습니다.

### 비즈니스 규칙

- 드롭다운 필터는 조회 결과 데이터에서 유니크 값을 추출해 동적으로 목록을 생성합니다.

## 6. 선수익금 조회

**화면 경로**: `/deferred-revenue`

### 화면 목적

회원이 선납한 금액 중 아직 서비스가 제공되지 않은 금액(선수익금)의 현황을 조회하는 화면입니다. 총 선수익금·인식 완료·잔여·이번 달 인식 예정 금액과 계약별 진행률을 확인합니다. 회원권 기간 믹스(1·3·6·12개월·기타)를 서비스일수 판정 기준으로 연결해 선수익금 인식 기간을 일관되게 계산합니다.

### 주요 기능

- **요약 지표 카드 4종**: 총 선수익금 / 인식 완료 금액 / 잔여 선수익금 / 이번 달 인식 예정 금액을 상단에 표시합니다.
- **기간 필터**: 시작일~종료일 입력. 기본값은 이번 달 전체입니다.
- **선수익금 목록 테이블**: 회원명·상품명·총액·인식 완료·잔여·시작일·종료일·진행률 컬럼을 제공합니다.
  - 진행률: 프로그레스 바와 숫자(%)를 함께 표시합니다. 80% 초과 시 붉은색, 50~80% 시 주황, 50% 미만 시 초록으로 표시합니다.
  - 인식 완료 금액: 초록색으로 표시합니다.
  - 잔여 금액: 포인트 컬러로 굵게 표시합니다.
- **서비스일수 판정**: 매출통계의 개월별 판매 비율과 동일한 1·3·6·12개월·기타 버킷을 사용해 선수익금 인식 기간을 결정합니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 기간 필터, 전체 조회 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 기간 필터, 지점 조회 |
| FC | 담당 회원 결제 처리 가능 | 조회만 가능 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 선수익금 조회" 진입합니다.
2. 상단 지표 카드로 전체 현황을 파악합니다.
3. 기간 필터를 설정해 원하는 계약 시작일 범위로 좁힙니다.
4. 테이블에서 계약별 인식 현황과 진행률을 확인합니다.

### 비즈니스 규칙

- 선수익금은 상품 기간 기준 서비스일수로 일할 계산합니다. 1개월=30일, 3개월=90일, 6개월=180일, 12개월=365일을 기본값으로 사용합니다.
- 기타 기간은 상품 마스터의 실제 서비스일수 또는 계약 종료일을 우선 사용하고, 값이 없으면 상품 유형별 기본값을 fallback으로 사용합니다.
- 기간 버킷 원천은 `[주간월간보고]` 주차별 V5:Z5이며, 매출 원장에 `durationMonths`가 있는 경우 해당 값을 우선합니다.
- 인식 금액 = 총액 × (경과일 / 총 서비스일수), 0~100% 범위.
- 이번 달 인식 예정 금액 = 이번 달에 종료일이 도래하는 계약의 잔여 금액 합산.
- 환불 건은 선수익금 계산에서 제외됩니다.
- 기간 필터는 계약 시작일 기준으로 적용됩니다.

## 7. 환불 관리

**화면 경로**: `/refunds`

### 화면 목적

처리된 환불 내역을 조회하고 통계를 확인하는 화면입니다. 이번 달 환불 총액·건수·평균 환불액·위약금 합계를 파악하고, 기간별 환불 이력을 통해 고객 불만 패턴 및 환불 정책을 점검합니다.

### 주요 기능

- **요약 지표 카드 4종**: 이번 달 환불 총액 / 환불 건수 / 평균 환불액 / 위약금 합계를 표시합니다.
- **위약금 합계**: 조회 기간 내 환불 건에서 공제된 위약금 총액을 표시하여 계약 조건이 실제 환불 처리에 반영되는지 확인합니다.
- **날짜 필터**: 이번 달·지난 달·최근 3개월 프리셋과 날짜 직접 입력, 조회 버튼을 제공합니다.
- **환불 목록 테이블**: No·환불일·회원명·상품명·환불금액·환불방법·사유·처리자·상태 컬럼을 제공합니다.
  - 환불금액: 붉은색으로 강조합니다.
  - 상태 배지: 완료(초록)·처리중(노랑)·거절(빨강)으로 구분합니다.
- **담당자별 환불 책임**: 처리자 기준 목록과 별도로 원판매 담당자별 환불 건수, 환불 금액, 위약금 합계, 원매출 대비 환불률을 표시합니다.
- **엑셀 내보내기**: 현재 조회된 환불 내역 전체를 파일로 내려받습니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 기간 조회, 엑셀 내보내기 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 기간 조회, 엑셀 내보내기 |
| FC | 담당 회원 결제 처리 가능 | 조회만 가능 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 환불 관리" 진입합니다.
2. 상단 지표 카드로 이번 달 환불 현황과 위약금 회수 규모를 파악합니다.
3. 날짜 프리셋 또는 직접 입력으로 조회 기간을 설정하고 조회합니다.
4. 테이블에서 환불 이력·사유·처리 상태를 확인합니다.
5. 엑셀 버튼으로 내역을 파일로 저장합니다.

### 비즈니스 규칙

- 환불 방법(카드·현금·계좌이체·마일리지)은 원래 결제수단을 기준으로 표시됩니다.
- 위약금 합계는 환불 총액과 별도로 표시하며, 환불 처리 과정에서 공제된 금액만 집계합니다.
- `처리자`는 환불 승인 또는 처리 담당자이고, `원판매 담당자`는 원결제의 판매 귀속 담당자입니다.
- 담당자별 환불 책임은 원판매 담당자 기준으로 집계하며, `[환불재결제]` 환불 행을 `[매출종합]`의 원거래ID 또는 카드 승인번호와 연결해 산출합니다.
- 원거래ID 또는 승인번호가 없는 환불 행은 회원명+종목+등록일+금액 조합으로 자동 확정하지 않고, 다중 후보 검수 대상으로 분리합니다.
- 환불 등록 및 승인·거절 처리는 이 화면에서 지원하지 않습니다. 조회 전용 화면입니다.
- 부분 환불 처리는 SCR-S012 결제 취소 / 부분 환불 화면에서 진행합니다.
- 엑셀 내보내기 컬럼: 환불일·회원명·상품명·환불금액·환불방법·사유·처리자·원판매 담당자·상태.

## 8. 미수금 관리

**화면 경로**: `/unpaid`

### 화면 목적

회원이 결제를 완료하지 못한 미수금을 추적하고 관리하는 화면입니다. 미결제·일부결제·연체·완료 상태를 탭으로 구분해 현황을 파악하고, 납입이 이루어지면 상태를 직접 변경하며 독촉 이력을 메모로 기록합니다.

### 주요 기능

- **요약 지표 카드 4종**: 미수금 총액 / 미수금 건수 / 연체(30일 이상) 건수 / 이번 달 회수액을 표시합니다. 연체 건수가 있을 때 해당 카드에 경고 강조 테두리를 표시합니다.
- **상태 탭 5종**: 전체·미결제·일부결제·연체·완료. 각 탭에 해당 건수 배지가 표시됩니다.
- **회원명 검색**: 실시간 검색으로 원하는 미수금 건을 찾습니다.
- **미수금 목록 테이블**: No·회원명·상품명·미수금액·결제기한·상태·메모·등록일·액션 컬럼을 제공합니다.
  - 미수금액: 붉은색으로 강조합니다.
  - 결제기한: 연체 건은 붉은색 굵은 글씨로 표시합니다.
  - 상태 배지: 완료(초록)·연체(빨강)·일부결제(노랑)·미결제(기본)으로 구분합니다.
- **상태 변경**: 현재 상태에 따라 전환 가능한 다음 상태만 드롭다운에 표시됩니다.
- **메모 편집 모달**: 메모 버튼 클릭 시 내용을 입력·수정하고 즉시 저장합니다.
- **엑셀 내보내기**: 현재 탭·검색 조건이 적용된 목록을 파일로 내려받습니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 상태 변경, 메모 편집, 엑셀 내보내기 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 상태 변경, 메모 편집, 엑셀 내보내기 |
| FC | 담당 회원 결제 처리 가능 | 상태 변경, 메모 편집 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 미수금 관리" 진입합니다.
2. 상단 지표 카드로 미수금 총액과 연체 건수를 파악합니다.
3. 상태 탭으로 원하는 상태의 건만 필터링합니다.
4. 회원명 검색으로 특정 회원의 미수금 건을 찾습니다.
5. 납입이 이루어진 건의 드롭다운에서 새 상태로 변경합니다.
6. 메모 버튼으로 독촉 이력이나 납입 협의 내용을 기록합니다.
7. 엑셀 버튼으로 목록을 저장합니다.

### 비즈니스 규칙

- 연체 기준: 결제기한으로부터 30일 이상 경과한 건.
- 상태 전환 규칙: 미결제→일부결제·완료, 일부결제→완료, 연체→일부결제·완료, 완료→변경 불가.
- 미수금 총액·건수 집계에서 완료 상태는 제외됩니다.
- 이번 달 회수액 = 이번 달 등록 + 완료 상태 건의 금액 합산.
- 엑셀 내보내기 컬럼: 회원명·상품명·미수금액·결제기한·상태·메모·등록일.

## 10. 세금계산서 발행

**화면 경로**: `/sales/invoice` _(퍼블리싱 완료 / 데이터 미연동)_

### 화면 목적

법인 회원 또는 사업자 회원에게 세금계산서를 발행하고 발행 이력을 조회·관리하는 화면입니다. 회계 마감 및 세무 신고에 필요한 발행 이력을 한 곳에서 관리합니다.

### 주요 기능

- **요약 지표 카드 3종**: 이번 달 발행 건수 / 발행 총액 / 미발행 대기 건수.
- **발행 탭**: 세금계산서 발행 대상 결제 건을 조회하고 발행 폼을 작성합니다.
- **이력 탭**: 기발행 세금계산서 목록(발행일·공급받는 자·공급가액·부가세·합계·상태)을 조회합니다.
- **발행 폼**: 공급받는 자의 사업자번호·상호·대표자·업태·종목·이메일 입력 후 발행합니다.
- **이메일 전송**: 발행 완료된 세금계산서를 공급받는 자 이메일로 전송합니다.
- **상태 배지**: 발행 완료·전송 완료·오류로 구분합니다.
- **엑셀 내보내기**: 발행 이력 목록을 파일로 내려받습니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 세금계산서 발행, 이력 조회, 이메일 전송, 엑셀 내보내기 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 세금계산서 발행, 이력 조회, 이메일 전송, 엑셀 내보내기 |
| FC | 담당 회원 결제 처리 가능 | 이력 조회만 가능 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 세금계산서 발행" 진입합니다.
2. 발행 탭에서 발행 대상 결제 건을 선택합니다.
3. 발행 폼에서 공급받는 자 정보를 입력하고 발행합니다.
4. 발행 완료 후 이메일 전송으로 상대방에게 전달합니다.
5. 이력 탭에서 발행 목록과 상태를 확인합니다.

### 비즈니스 규칙

- 세금계산서는 사업자등록번호가 있는 거래처에만 발행할 수 있습니다.
- 발행 후 이메일 전송이 완료되어야 "전송 완료" 상태로 변경됩니다.
- 발행 오류 발생 시 오류 상태로 표시하고 재발행이 가능합니다.

## 11. 매출 예측

**화면 경로**: `/sales/forecast` _(퍼블리싱 완료 / 데이터 미연동)_

### 화면 목적

과거 매출 데이터를 기반으로 향후 매출을 예측하고, 설정한 목표 대비 달성률을 시각적으로 확인하는 화면입니다. 선제적인 운영 계획 수립에 활용합니다.

### 주요 기능

- **요약 지표 카드 4종**: 이번 달 예측 매출 / 목표 달성률 / 전월 대비 증감 / 이번 분기 누적 매출.
- **예측 기간 선택**: 다음 달·다음 분기·연간 예측을 선택합니다.
- **매출 목표 설정**: 기간별 목표 금액을 입력하면 달성률이 실시간으로 업데이트됩니다.
- **매출 예측 차트**: 과거 실적(선 그래프)·예측값(점선)·목표 라인을 하나의 차트에 표시합니다.
- **달성률 게이지**: 현재 누적 매출이 목표의 몇 %인지 프로그레스 바와 숫자로 표시합니다.
- **상품별 예측 기여도**: 향후 매출 기여 예상 상품 순위 목록을 제공합니다.
- **월별 예측 테이블**: 월별 예측 매출·목표·달성률·전년 동기 대비 컬럼을 표로 제공합니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 전체 지점 예측 조회, 목표 설정 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 지점 예측 조회, 목표 설정 |
| FC | 담당 회원 결제 처리 가능 | 조회만 가능 |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 매출 예측" 진입합니다.
2. 상단 카드로 이번 달 예측 매출과 목표 달성률을 확인합니다.
3. 예측 기간을 선택합니다.
4. 목표 금액을 입력해 달성률 게이지 변화를 확인합니다.
5. 차트에서 과거 추세와 예측 궤적을 비교합니다.
6. 상품별 기여도와 월별 예측 테이블로 상세 수치를 확인합니다.

### 비즈니스 규칙

- 과거 데이터가 충분하지 않으면 예측 정확도가 낮을 수 있음을 안내합니다.
- 목표 달성률이 100% 초과 시 게이지 색상이 성공 색으로 변경됩니다.
- 예측값은 과거 동기 실적과 최근 추세를 기반으로 산출됩니다.

## 12. 결제 취소 / 부분 환불

**화면 경로**: `/sales/cancel-refund` _(퍼블리싱 완료 / 데이터 미연동)_

### 화면 목적

완료된 결제 건에 대해 전체 취소 또는 일부 금액 환불을 처리하는 화면입니다. 처리된 내역은 환불 관리 화면(SCR-S007)에 자동으로 반영됩니다.

### 주요 기능

- **결제 건 검색**: 회원명·상품명·결제일로 환불 처리할 결제 건을 조회합니다.
- **결제 내역 확인**: 선택한 결제 건의 결제일·상품명·결제금액·결제수단·결제자 정보를 표시합니다.
- **처리 유형 선택**: 전체 취소 또는 부분 환불 중 하나를 선택합니다.
  - **전체 취소**: 결제 금액 전액을 환불 처리합니다.
  - **부분 환불**: 환불 금액을 직접 입력합니다. 원래 결제 금액을 초과할 수 없습니다.
- **환불 사유 입력**: 단순 변심·결제 오류·회원 요청·중복 결제·서비스 불만 프리셋 또는 직접 입력.
- **환불 수단 안내**: 원래 결제수단에 따라 환불 방법을 안내합니다.
- **처리 확인 모달**: 환불 금액·사유·수단을 최종 확인 후 처리합니다.
- **처리 이력**: 해당 결제 건의 이전 환불 이력을 하단에 목록으로 표시합니다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 본사 최고관리자 | 전체 지점 매출 조회·분석 가능 | 전체 취소·부분 환불 처리, 결제 건 조회 |
| 센터장 / 매니저 | 소속 지점 매출 전체 | 전체 취소·부분 환불 처리, 결제 건 조회 |
| FC | 담당 회원 결제 처리 가능 | 담당 회원 결제 건 조회, 환불 요청 신청 (최종 처리는 매니저 이상) |
| 트레이너 / 스태프 | 매출 조회만 (처리 불가) | 조회만 가능 |

### 주요 UX 흐름

1. 사이드바 "매출관리 > 결제 취소 / 부분 환불" 진입합니다.
2. 회원명·상품명·결제일로 환불 처리할 결제 건을 검색합니다.
3. 결제 건을 선택해 내역을 확인합니다.
4. 전체 취소 또는 부분 환불을 선택합니다. 부분 환불은 금액을 직접 입력합니다.
5. 환불 사유를 선택하거나 입력합니다.
6. "처리 확인" 클릭 → 최종 확인 모달에서 "처리 완료"를 누릅니다.
7. 완료 후 환불 관리 화면으로 이동하거나 계속 처리합니다.

### 비즈니스 규칙

- 부분 환불 금액은 원래 결제 금액을 초과할 수 없습니다. 초과 입력 시 경고를 표시합니다.
- 이미 전액 환불된 결제 건은 재처리가 불가합니다.
- 처리 완료된 환불 내역은 환불 관리 화면(SCR-S007)에 자동으로 반영됩니다.
- 부분 환불 후 잔여 미환불 금액이 있는 경우 별도 추가 환불이 가능합니다.
- 처리 중 버튼이 비활성화되어 중복 처리를 방지합니다.
