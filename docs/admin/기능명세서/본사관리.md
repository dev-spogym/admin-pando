- 관리자웹 권한 키는 `superAdmin / primary / owner / manager / fc / staff / readonly`를 기준으로 해석한다.

## 1. 대시보드 (`/`)

### 화면 목적
로그인 직후 진입하는 첫 화면으로, 현재 지점의 운영 현황을 한눈에 파악할 수 있다. 회원 현황, 매출, 출석, 주의 대상 회원(만료 예정·생일자·미수금·홀딩), 최근 활동 로그를 종합적으로 표시한다.

### 주요 기능
- **운영 현황 요약 카드**: 전체 회원 수, 활성 회원 수, 만료 예정 회원 수, 오늘 출석 수, 이번 달 매출을 상단 카드 형태로 표시
- **분포 차트**: 회원의 성별 분포와 연령대 분포를 시각적으로 확인
- **추이 차트**: 월별 매출 추이와 주간 출석 현황을 그래프로 표시
- **주의 대상 위젯**: 오늘 생일인 회원, 미수금 발생 회원, 홀딩 중인 회원, 만료 임박 회원 목록을 별도 위젯으로 표시
- **최근 활동 로그**: 시스템 내 최근 활동 이력 요약 표시
- **수동 새로고침**: 버튼 클릭으로 전체 데이터를 최신 상태로 갱신

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 현재 선택된 지점의 대시보드 | 새로고침, 위젯 내 회원 상세 이동 |
| 지점 관리자 / 직원 | 자신이 속한 지점의 대시보드 | 새로고침, 위젯 내 회원 상세 이동 |

### 주요 UX 흐름
1. 로그인 완료 시 자동으로 이 화면으로 이동한다.
2. 첫 방문 시 환영 배너가 상단에 표시되며, X 버튼으로 닫을 수 있다.
3. 데이터를 불러오는 동안 각 카드와 위젯에 로딩 스켈레톤이 표시된다.
4. 위젯 내 회원 이름을 클릭하면 해당 회원의 상세 화면으로 이동한다.
5. 위젯 하단의 "전체보기" 링크를 클릭하면 해당 메뉴 화면으로 이동한다.

### 비즈니스 규칙
- 만료 예정 회원은 이용권 만료일 기준 30일 이내인 활성 회원을 대상으로 한다.
- 매출 금액은 1억 이상이면 "X.X억", 1만 이상이면 "X만" 형식으로 축약 표시한다.
- 매일 자동으로 만료 처리가 실행되어 이용권이 만료된 회원의 상태가 갱신된다.

## 2. 슈퍼 대시보드 (`/super-dashboard`)

### 화면 목적
슈퍼관리자 전용 화면으로, 전체 지점의 통합 KPI와 지점별 상세 현황을 한 화면에서 파악한다. 지점 전환 기능을 통해 특정 지점의 관리 모드로 빠르게 진입할 수 있다.

### 주요 기능
- **전사 통합 KPI 카드 7종**: 전체 회원 수, 전체 매출, 전체 출석, 전체 직원 수, 이번 달 신규 회원, 만료 예정 회원(30일 이내), 만료 회원을 한 화면에서 확인
- **전월 대비 성장률 표시**: 전체 회원·매출·출석 지표에 전월 또는 전일 대비 증감률(%)을 화살표와 색상으로 표시
- **지점별 현황 카드**: 각 지점의 활성 회원 수, 직원 수, 월 매출, 만료 회원 수, 오늘 출석 수와 운영 상태 배지를 카드 형태로 표시
- **최근 히스토리 로그**: 시스템 전체에서 발생한 최근 10건의 주요 활동 이력 표시
- **지점 전환**: 지점 카드의 버튼을 클릭하여 해당 지점 관리 모드로 즉시 전환
- **수동 새로고침**: 버튼 클릭으로 전체 데이터를 최신 상태로 갱신

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 전체 지점 통합 현황 및 지점별 카드 | 지점 전환, 지점 상세 보기, 새로고침 |
| 일반 관리자 / 직원 | 접근 불가 메시지 화면 | 대시보드로 이동 버튼만 사용 가능 |

### 주요 UX 흐름
1. 슈퍼관리자가 화면에 진입하면 전체 지점 데이터를 자동으로 불러온다.
2. KPI 카드는 데이터 로딩 중 스켈레톤으로 표시되며, 완료 후 실제 수치로 전환된다.
3. 성장률이 양수이면 초록색 상승 화살표, 음수이면 빨간색 하락 화살표로 표시된다.
4. "지점 전환" 버튼을 클릭하면 해당 지점 모드로 전환되고, 대시보드로 이동한다.
5. 슈퍼관리자가 아닌 사용자가 접근하면 권한 없음 안내 메시지와 대시보드 이동 버튼이 표시된다.

### 비즈니스 규칙
- 이 화면은 슈퍼관리자 계정만 접근할 수 있다.
- 전월 회원 수는 현재 전체 회원 수에서 이번 달 신규 회원을 빼고 만료 회원을 더한 근사치로 계산한다.
- 성장률은 전월(또는 전일) 대비 현재 수치의 변화율(%)로 표시한다.
- 지점 운영 상태는 운영중 / 휴업 / 폐점 세 가지로 구분하며, 각각 다른 색상 배지로 표시한다.

## 3. 지점 관리 (`/branches`)

### 화면 목적
전체 지점의 현황을 조회하고, 신규 지점을 등록하거나 기존 지점의 상태를 변경한다. 지점 간 회원 이동 신청 및 지점별 통합 성과 비교도 이 화면에서 처리한다.

### 주요 기능
- **지점 목록 조회**: 지점명, 지점 코드, 주소, 연락처, 회원 수, 직원 수, 운영 상태, 등록일을 목록 형태로 조회
- **지점 검색 및 필터**: 지점명 또는 주소로 검색하고, 운영 상태(운영중 / 임시휴업 / 폐점)로 필터링
- **신규 지점 등록**: 지점명, 지점 코드, 연락처, 주소, 관리자, 운영 시간, 수용 인원 등 정보를 입력하여 신규 지점 등록
- **지점 상태 변경**: 운영 중인 지점을 임시휴업 또는 폐점으로 변경하거나 비활성화
- **통합 현황 비교**: 지점별 회원 수, 매출, 출석률을 월간/주간/연간 기준으로 비교
- **지점 간 회원 이동**: 특정 회원을 다른 지점으로 이동 신청하고, 이동 이력 조회
- **엑셀 다운로드**: 지점 목록을 엑셀 파일로 내보내기

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 전체 지점 목록 및 모든 탭 | 지점 등록, 상태 변경, 비활성화, 지점 전환, 엑셀 다운로드 |
| 지점 관리자 | 자신의 지점 정보 | 지점 이동 신청 |

### 주요 UX 흐름
1. 화면 진입 시 전체 지점 목록이 표시된다.
2. 상단 탭으로 "지점 목록", "통합 현황", "지점 간 이동" 화면을 전환한다.
3. "신규 지점 등록" 버튼 클릭 시 등록 모달이 열리며, 필수 항목 입력 후 저장한다.
4. 지점 코드는 자동 생성 버튼으로 만들거나 직접 입력할 수 있다.
5. 지점을 임시휴업 또는 폐점으로 변경하면 영향받는 회원 수와 직원 수를 확인하는 단계를 거친 후 처리된다.
6. 비활성화는 2단계 확인을 거쳐 처리되며, 이후 해당 지점의 데이터는 조회 전용으로 전환된다.
7. 폐점 지점이 목록에 포함된 경우 상단에 안내 배너가 표시된다.

### 비즈니스 규칙
- 지점 코드는 영문 대문자 2~5자로만 구성하며, 중복 등록이 불가하다.
- 지점명은 2~30자 이내로 입력해야 한다.
- 연락처는 지정된 형식(예: 02-1234-5678)에 맞게 입력해야 한다.
- 지점 등록 시 지점명, 지점 코드, 연락처, 주소, 관리자는 필수 항목이다.
- 폐점 처리 시 해당 지점의 회원 및 직원 데이터는 조회만 가능하며 수정 및 신규 등록이 제한된다.
- 비활성화된 지점의 데이터도 조회 전용으로 유지된다.

## 4. 지점 리포트 (`/branch-report`)

### 화면 목적
월별로 지점 간 성과를 비교하는 리포트 화면이다. 회원 수, 신규 가입, 매출, 출석률을 지점별로 한눈에 비교하여 우수 지점과 부진 지점을 파악할 수 있다.

### 주요 기능
- **월 선택**: 최근 12개월 중 원하는 월을 선택하여 해당 기간의 지점별 성과를 조회
- **통합 요약 카드**: 전체 지점 합산 기준 총 매출, 총 회원, 신규 회원, 활성 회원을 카드로 표시
- **지점별 매출 및 회원 비교 차트**: 지점별 매출과 회원 수를 막대 차트로 비교
- **지점별 성과 테이블**: 지점명(순위 아이콘 포함), 총회원, 신규가입, 활성회원, 만료회원, 매출, 객단가, 출석률을 표 형태로 조회
- **컬럼 정렬**: 각 컬럼 헤더 클릭으로 오름차순/내림차순 정렬
- **엑셀 다운로드**: 조회 결과를 엑셀 파일로 내보내기

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 전체 지점 비교 리포트 | 월 선택, 정렬, 엑셀 다운로드 |
| 지점 관리자 | 접근 권한에 따라 제한 가능 | 동일 |

### 주요 UX 흐름
1. 화면 진입 시 현재 월 기준으로 전체 지점 성과가 자동으로 조회된다.
2. 월 선택 드롭다운에서 다른 월을 선택하면 해당 기간의 데이터로 전환된다.
3. 매출 기준 1~3위 지점에는 금·은·동 메달 아이콘이 표시된다.
4. 컬럼 헤더를 클릭하면 해당 컬럼 기준으로 정렬이 전환된다.

### 비즈니스 규칙
- 매출 순위는 선택한 월의 완료된 매출 합계를 기준으로 산정한다.
- 출석률은 활성 회원 수와 해당 월 일수를 기준으로 계산한다.
- 객단가는 해당 지점의 총 매출을 총 회원 수로 나눈 평균 값이다.

## 5. KPI 대시보드 (`/kpi`)

### 화면 목적
지점의 성장 지표와 운영 건강도를 종합적으로 파악하는 화면이다. 회원 현황, 매출, 상담 전환, 재등록/이탈 회복, PT 수업, GX 수업, 강습 세션 유형별 운영량, OT 영업 파이프라인 등 핵심 지표와 코호트 분석, 퍼널 분석, 상담 채널별 WI/TI 분리, GX 세부 종목 분포까지 제공한다.

### 주요 기능
- **회원 현황 지표 6종**: 전체 회원, 활성 회원, 활성 비율, 이번 달 신규 회원, 만료 예정 회원, 만료 회원
- **매출 지표 4종**: 이번 달 매출, 전월 매출, 전월 대비 성장률, 신규 회원 전월 대비 성장률
- **매출 목표 달성률 게이지**: 설정한 월 매출 목표 대비 현재 달성률을 시각적으로 표시. 목표 미설정 시 안내 문구 표시
- **상담 및 전환 지표**: 이번 달 상담 건수, 상담 완료 건수, 상담 완료율, 일평균 출석, WI vs TI 건수, TI 방문 전환율, TI 등록률, WI 등록률 표시
- **재등록/이탈 회복 지표**: 재등록 상담 완료 건수, 재등록 성공률 표시
- **PT 수업 지표 4종**: 이번 달 PT 세션, PT 완료 건수, PT 완료율, PT 노쇼율
- **강습세션수 지표**: PT, 필라테스, 스트레칭, 테라피 4종의 월간/주차별 수업수를 표시
- **OT / 수업운영 지표**: 강습 OT 신규 배정 건수, 골프 OT 신규 배정 건수, OT 등록 전환율, OT 거부율, OT 이월률, OT 체험 → 등록률, 골프 OT 전체 현황, 트레이너별 OT 1/2차 완료·예정 현황 표시
- **담당자 성과(개인 KPI) 지표**: 트레이너별 OT 배정 합계, 트레이너별 신규/재등록 매출, FC별 상담 건수, FC별 상담 → 등록 전환율, 골프프로별 매출 표시
- **GX 수업 지표 4종**: 이번 달 수업 건수, 예약 수, 출석 수, 수업 출석률
- **GX 세부 종목 분포 6종**: 상품 마스터의 `category=GX`와 2단계 `subCategory`를 기준으로 요가·필라테스·스피닝·줌바·에어로빅·GX 기타의 판매/수요 분포를 표시하여 인기 클래스 증설과 비인기 클래스 폐지 판단에 활용
- **코호트 분석**: 최근 6개월 등록 회원의 월별 유지율을 색상 테이블로 시각화
- **퍼널 분석**: 리드 유입 → 상담 완료 → 회원 등록 → 3개월 유지 → 재등록까지 5단계 전환율 표시
- **매출 목표 설정**: 해당 월의 매출 목표 금액을 직접 입력하여 저장
- **수업 데이터 원천 표시**: 수업/출석/일정 원장의 강습 세션 유형을 기준으로 PT·필라테스·스트레칭·테라피별 강습세션수를 산출하며, `[주간월간보고]` 주차별 `N5(강습)` 및 달성 `G열(강습)`, `[강습업무보고]` 주간보고 `O열(수업수)`는 합계 검산용으로 표시
- **상담/재등록 데이터 원천 표시**: `[신규DB]` C열 `경로`, D열 `상세 경로1`, N/R/V열 `상담 방식`, `[재등록DB]` B열 `상담결과`, F열 `만료일`, I열 `담당자`와 `[주간월간보고]` 주차별 상담 집계 셀을 기준으로 지표 산출 근거를 확인

### 주요 UX 흐름
1. 화면 진입 시 현재 월 기준으로 전체 KPI가 자동으로 조회된다.
2. "목표 설정" 버튼 클릭 시 월 매출 목표 입력 모달이 열린다.
3. 목표 금액을 입력하고 저장하면 달성률 게이지에 즉시 반영된다.
4. 코호트 분석 테이블에서 아직 경과하지 않은 기간은 "-"로 표시된다.
5. 퍼널 분석에서 각 단계의 수치와 직전 단계 대비 전환율이 함께 표시된다.

### 비즈니스 규칙
- 매출 목표는 지점과 월 기준으로 별도 저장되며, 기기에 저장되어 유지된다.
- 달성률이 100% 이상이면 "목표 달성!" 문구와 초록색, 70~99%는 파란색, 50~69%는 노란색, 50% 미만은 빨간색으로 표시한다.
- PT 완료율이 85% 이상이면 긍정 표시, 미만이면 경고 표시한다.
- PT 노쇼율이 5% 이하이면 긍정 표시, 초과하면 경고 표시한다.
- 강습 세션 유형은 `PT`, `필라테스`, `스트레칭`, `테라피` 4종으로 고정한다. 화면에서는 `PT`를 `P.T`, `필라테스`를 `필라`로 축약 표기할 수 있다.
- 강습세션수는 실제 수업/출석/일정 원장에 저장된 `강습 세션 유형`을 공식 원천으로 산출한다.
- `[주간월간보고]` 주차별 시트 `N5(강습)` 및 `달성` 시트 `G열(강습)`, `[강습업무보고]` 주간보고 `O열(수업수)`는 PT·필라테스·스트레칭·테라피 4종 분해 공식 원천이 아니라 합계 검산 및 운영 보조 원천으로 사용한다.
- 기존 `수업 유형(PT/GX/개인레슨/기타)`과 상품 마스터의 GX 6종 세부종목은 강습 세션 유형과 별도 축으로 관리한다.
- GX 수업 출석률이 80% 이상이면 긍정 표시, 미만이면 경고 표시한다.
- GX 세부 종목 분포는 상품 마스터 `category=GX` + `subCategory`를 우선 기준으로 삼고, 외부 시트 연동 시 `[월매출]` C열 상품/종목명과 `[주간월간보고]` 주차별 T5:U5를 함께 참조한다.
- GX 세부 종목은 요가·필라테스·스피닝·줌바·에어로빅·GX 기타 6종으로 고정 분류한다.
- `WI`는 방문 문의, `TI`는 전화 문의로 해석한다. 외부 템플릿에서는 `[신규DB]` C열 `경로` 값 `WI`/`TI`로 구분한다.
- TI 방문 전환율은 `[신규DB]`의 1차/2차/3차 상담 방식 컬럼인 N/R/V열 중 하나라도 `대면`인 TI 문의를 방문 전환으로 본다. 상담 방식 드롭다운 값은 `대면`, `유선`, `부재`이다.
- `[주간월간보고]` 주차 시트의 R10:Z10은 WI 상세 경로별 건수, AA10:AI10은 TI 상세 경로별 건수로 집계한다.
- `[주간월간보고]` AC14:AE14는 TI 방문 전환율, AF14:AI14는 TI 등록률, AJ14:AM14는 WI 등록률의 주차별 산출 기준이다.
- 재등록 상담 완료 건수는 `[재등록DB]` B열 `상담결과`가 `등록`, `확정`, `예상`, `관리`, `실패` 중 하나로 입력된 건수로 산출한다.
- 재등록 성공률은 `[재등록DB]` B열 `등록` 건수 ÷ B열 `등록+확정+예상+관리+실패` 전체 상담 건수로 산출한다.
- `[재등록DB]` F열 `만료일`을 D-7/D-14/D-30 버킷으로 분해하는 만료일별 대상자 분포 위젯은 추가하지 않고, 기존 만료 예정 30일 이내 단일 지표를 유지한다.
- 환불 → 재결제 복귀율은 재등록/이탈 회복 KPI에서 제외하며, 환불관리 확장 지표로도 추가하지 않는다.
- OT 신규 배정 건수는 `[OT관리-강습]`과 `[OT관리-골프]`의 `OT현황` 시트에서 `Z6(OT배정 인원)`을 기본 원천으로 삼고, `[주간월간보고]` 주차 시트의 강습 OT `I24`, 골프 OT `I28`을 주차별 보조 원천으로 사용한다.
- OT 등록 전환율은 `등록 / OT 진행`으로 산출한다. 여기서 `등록`은 `OT현황` `V열 등록여부=등록` 또는 요약 `Z9`, `OT 진행`은 월간 강습 업무보고의 `OT 진행` 기준처럼 신규 OT 배정에서 진행중·미진행을 제외한 실제 진행 건으로 해석한다.
- OT 거부율은 `거절 / OT 진행`으로 산출하며, `거절`은 운영 확정에 따라 `거부+보류`를 함께 집계한다. 이 기준은 `[OT관리-강습/골프]` `OT현황`의 `V열 등록여부=거부 또는 보류` 및 요약 `Z7`과 연결한다.
- OT 이월률은 `[OT관리-강습/골프]` `OT현황`의 `V열 등록여부=이월` 및 요약 `Z8`을 사용한다. 분모는 우선 `OT배정 인원`으로 기록하되, `OT 진행` 기준과 구분이 필요한 산식으로 별도 관리한다.
- OT 체험 → 등록률은 온보딩의 일반 체험 전환율과 별도 지표로 분리한다. 원천은 `[OT관리-강습/골프]` `OT현황`의 체험 요약 `AC5(체험)`와 `AC6(등록)`이며, 산식은 `체험 등록 / 체험 대상`이다.
- `[주간월간보고]` 주차 시트의 강습 OT 현황은 `I24:R24`, 골프 OT 현황은 `I28:R28`을 사용한다. `달성` 시트에서는 강습 OT `CI:CR`, 골프 OT `CS:DB` 영역으로 월간 취합한다.
- 골프 OT 전체 현황은 `[OT관리-골프]` `OT현황` 전체와 본사 취합 파일 `골프부서_OT현황취합_YYYYMM`을 기준으로 강습 OT와 별도 섹션에서 표시한다.
- 트레이너별 OT 1/2차 완료·예정은 `[OT관리-강습]` `OT현황`의 담당자·일자·진행 여부와 `[강습업무보고]` 주간보고 `E열 O.T 배정`, `F열 O.T 1`, `G열 O.T 2`를 연결해 산출한다.
- 트레이너별 OT 배정 합계는 `[강습업무보고]` 주간보고 `E열 O.T 배정`을 트레이너별로 합산해 산출한다.
- 트레이너별 신규/재등록 매출의 공식 원천은 `[매출종합]` 거래 단위 데이터이며, `[강습업무보고]` 주간보고 `J열 주간신규`, `K열 주간재등록`은 검산 또는 운영 보조 원천으로만 사용한다.
- FC별 상담 건수는 `[신규DB]` `K열 담당자` 기준의 전체 상담 건수로 산출하며, A열 상태가 `등록`, `관리`, `실패`인 건을 모두 포함한다.
- FC별 상담 → 등록 전환율은 `[신규DB]`에서 담당자별 `A열=등록` 건수 ÷ 담당자별 `등록+관리+실패` 전체 상담 건수로 산출한다.
- 골프프로별 매출은 `[매출종합]` `골프매출` K열 `등록강사`와 F열 `결제금액`을 기준으로 산출한다.
- `[주간월간보고]` 주차별 `등록률` 셀이 현재 `등록 / 신규` 구조로 해석될 수 있으므로, 대시보드 표준 산식인 `등록 / OT 진행`과 다를 경우 추후 사용자 확인 후 산식을 확정한다.
- 코호트 유지율은 90% 이상 초록, 70~89% 연두, 50~69% 노랑, 30~49% 주황, 30% 미만 빨강으로 색상 구분한다.
- 향후 추가 예정 지표: 이탈률, NPS, LTV, CAC, 영업이익률, 직원 생산성, 회원 만족도, Today Tasks 완료율

## 6. KPI 센터 (`/kpi-preview`)

### 화면 목적
역할별로 맞춤화된 KPI 보드를 제공하는 종합 센터 화면이다. 본사, 지점, FC, 트레이너, 운영 5가지 관점에서 핵심 지표, 우선 업무 큐, 성과 랭킹, 단계별 퍼널, 데이터 인사이트를 한 화면에서 확인할 수 있다.

### 주요 기능
- **역할별 5개 보드 탭**: 본사 / 지점 / FC / 트레이너 / 운영 탭을 전환하며 각 역할에 맞는 KPI 보드 조회
- **핵심 지표 카드 4종**: 각 보드별 가장 중요한 지표 4개를 카드 형태로 표시
- **FC 보드 상담 퍼널 지표**: FC 보드 핵심 지표에는 신규 리드, TI 방문 전환율, WI 등록률, TI 등록률을 포함
- **재등록 파이프라인 지표**: 재등록 상담 완료 건수와 재등록 성공률을 지점/FC 보드 보조 KPI로 표시
- **트레이너 보드 OT 지표**: 트레이너 보드 핵심 지표에는 OT 등록 전환율, OT 체험 → 등록률, 트레이너별 OT 1/2차 완료·예정 현황을 포함
- **담당자 성과 패널**: 트레이너별 OT 배정 합계·신규/재등록 매출, FC별 상담 건수·상담 → 등록 전환율, 골프프로별 매출을 개인 KPI로 표시
- **액션 큐**: 오늘 처리해야 할 우선순위 업무 목록을 긴급/우선/일반/완료가능 단계로 구분하여 표시
- **성과 랭킹 패널**: 지점 성과 랭킹, 팀원 KPI 스냅샷, 회원 상태 등 보드별 맞춤 랭킹 정보 표시
- **퍼널 차트**: 각 보드의 업무 흐름을 4단계 퍼널로 시각화
- **데이터 인사이트**: 현황 데이터를 기반으로 한 분석 인사이트 3개 제공
- **바로가기 버튼**: 자동알림, Today Tasks, 매출 보기, 회원 보기로 빠르게 이동

| FC 역할 | 진입 시 FC 보드가 기본 선택 | 모든 보드 탭 전환 가능 |
| 지점 관리자 / 오너 | 진입 시 지점 보드가 기본 선택 | 모든 보드 탭 전환 가능 |
| 트레이너 | 진입 시 트레이너 보드가 기본 선택 | 모든 보드 탭 전환 가능 |

### 주요 UX 흐름
1. 화면 진입 시 로그인한 계정의 역할에 맞는 보드 탭이 자동으로 선택된다.
2. 상단 탭을 클릭하면 해당 역할의 KPI 보드로 전환된다.
3. 각 보드에서 핵심 지표, 액션 큐, 랭킹, 퍼널, 인사이트를 순서대로 확인할 수 있다.
4. 상단 바로가기 버튼으로 자동알림, Today Tasks, 매출, 회원 화면으로 이동할 수 있다.

### 비즈니스 규칙
- 진입 시 기본으로 선택되는 보드는 계정 역할에 따라 자동으로 결정된다.
- 탭 전환은 모든 계정에서 자유롭게 가능하다.
- 액션 큐의 우선순위는 긴급 / 우선 / 일반 / 완료가능 4단계로 구분된다.
- FC 보드는 WI와 TI를 합산한 상담 전환율만 보여주지 않고, 전화 문의가 실제 대면 상담으로 이어졌는지와 방문 문의·전화 문의의 등록률을 분리해 표시한다.
- TI 등록률은 TI 전체 문의 중 등록 완료 비율, WI 등록률은 WI 방문 문의 중 등록 완료 비율로 산출한다.
- TI 방문 전환율은 전화 문의 대상자 중 N/R/V 상담 방식 컬럼에 `대면`이 기록된 비율로 산출한다.
- 재등록 상담 완료 건수와 재등록 성공률은 `[재등록DB]` B열 `상담결과` 기준으로 산출하며, 재등록 성공률은 `등록 / (등록+확정+예상+관리+실패)`로 표시한다.
- 트레이너 보드의 OT 등록 전환율은 `등록 / OT 진행`으로 산출하며, 일반 PT 완료율과 합산하지 않는다.
- 트레이너 보드의 OT 체험 → 등록률은 `[OT관리-강습/골프]` 체험 대상 중 등록 완료 비율로 산출하며, 온보딩의 일반 `체험 → 등록 전환율`과 별도 지표로 표시한다.
- OT 거부율에서 `거절`은 `거부+보류`를 함께 집계한다고 명시한다.
- 트레이너 개인 KPI는 OT 배정 합계, 신규 매출, 재등록 매출, OT 등록 전환율, OT 체험 → 등록률을 같은 행에서 비교한다.
- FC 개인 KPI는 담당자별 상담 건수와 상담 → 등록 전환율을 같은 행에서 비교해 업무량과 등록 성과를 분리한다.
- 골프 프로 KPI는 `[매출종합]` `골프매출` 탭의 K열 `등록강사`별 매출을 기준으로 별도 표시한다.
- 현재 보드 데이터는 시범 운영 단계로 실제 시스템 데이터와 연동 전 상태이다. 실데이터 연동은 추후 지원 예정이다.

## 7. 온보딩 대시보드 (`/onboarding`)

### 화면 목적
신규 회원의 초기 정착 과정을 단계별로 추적하는 화면이다. 리드 유입부터 등록, 초기 출석 정착까지의 퍼널과 이탈 위험 회원을 조기에 파악하여 적절한 조치를 취할 수 있도록 지원한다.

### 주요 기능
- **프리-온보딩 지표**: 이번 달 신규 리드 수, 연락 완료, 방문 완료, 등록 전환, 리드 전환율, WI vs TI 건수 표시
- **신규 유치 지표**: 이번 달 신규 등록 회원 수, 리드 누락률, 회원소개 유입 비중, 가입경로(등록자 기준) 분포 표시
- **신규 안정 지표**: 7일 이용률, 30일 활동률, PT 체험 참여율, 체험 → 등록 전환율, 일일입장 → 회원 전환율, GX 첫 참여율, 초기 이탈 회원 수 표시
- **온보딩 단계별 진행률 바**: 가입 → 초기 상담 → 체성분 측정 → 프로그램 배정 → 첫 수업 완료 5단계를 진행률 막대로 시각화
- **신규 회원 목록**: 이번 달 등록한 신규 회원의 등록일, 경과일, 방문 횟수, PT 체험 완료 여부, GX 참여 여부, 온보딩 상태를 테이블로 조회
- **이탈 위험 회원 별도 표시**: 위험 또는 이탈위험 상태의 회원을 별도 섹션으로 강조 표시

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 현재 선택된 지점의 온보딩 현황 | 데이터 조회 |
| 지점 관리자 | 자신의 지점 온보딩 현황 | 데이터 조회 |

### 주요 UX 흐름
1. 화면 진입 시 현재 지점의 이번 달 신규 회원 온보딩 현황이 자동으로 표시된다.
2. 상단 KPI 카드에서 각 단계별 지표를 빠르게 파악할 수 있다.
3. 온보딩 단계 진행률 바를 통해 각 단계의 이탈 지점을 시각적으로 확인한다.
4. 이탈 위험 회원이 있으면 하단에 별도 섹션이 자동으로 나타난다.
5. 신규 회원 테이블에서 방문 횟수에 따라 다른 색상으로 상태를 구분한다.

### 비즈니스 규칙
- 7일 이용률은 등록 후 7일 이내에 2회 이상 출석한 신규 회원의 비율이다.
- 30일 활동률은 등록 후 30일 이내에 4회 이상 출석한 신규 회원의 비율이다.
- 체험 → 등록 전환율은 `[신규DB]` C열 `경로=체험` 또는 `[주간월간보고]` U14:V14를 기준으로 체험 대상자 중 등록 완료 비율을 산출한다.
- 일일입장 → 회원 전환율은 `[신규DB]` C열 `경로=일일입장` 또는 `[주간월간보고]` Y14:Z14를 기준으로 일일 입장 고객 중 회원 등록 비율을 산출한다.
- 회원소개 유입 비중은 `[신규DB]` D열 `상세 경로1=회원소개` 또는 `[주간월간보고]` Q14:R14를 기준으로 산출한다.
- 가입경로(등록자 기준) 분포는 `[신규DB]` A열 `등록` 완료 건의 상세 경로를 집계하거나 `[주간월간보고]` I10:Q10의 등록자 기준 가입 경로 집계를 사용한다.
- 온보딩 상태 분류 기준:
  - 정상: 기준 출석 횟수를 충족한 회원
  - 미방문: 등록 후 7일 이내이며 방문 이력이 없는 회원
  - 주의: 등록 후 7일 이내이며 방문이 1회인 회원
  - 위험: 등록 후 7일 초과이며 방문이 1회 이하인 회원
  - 이탈위험: 등록 후 14일 초과이며 방문이 3회 이하인 회원
- 7일 이용률이 40% 이상이면 정상, 미만이면 주의로 색상 구분한다.
- 30일 활동률이 50% 이상이면 정상, 미만이면 주의로 색상 구분한다.

## 8. 히스토리 로그 (`/audit-log`)

### 화면 목적
시스템 내에서 발생한 모든 중요 활동의 이력을 조회하는 화면이다. 누가, 언제, 어떤 동작을 수행했는지 추적하여 보안 감사와 이상 행동 모니터링에 활용한다.

### 주요 기능
- **활동 이력 조회**: 로그인, 로그아웃, 데이터 생성/수정/삭제, 지점 전환, 권한 변경, 환불, 내보내기 등 주요 활동 이력 조회
- **기간 필터**: 시작일~종료일 범위를 지정하여 기간 내 로그 조회 (기본값: 최근 30일)
- **액션 타입 필터**: 로그인, 데이터변경, 권한변경, 지점전환, 내보내기, 생성, 삭제 등 특정 유형의 활동만 필터링
- **대상 유형 필터**: 회원, 직원, 매출, 설정, 지점 중 특정 대상과 관련된 로그만 필터링
- **지점 필터**: 슈퍼관리자의 경우 특정 지점의 활동 로그만 선택하여 조회 가능
- **사용자 검색**: 특정 사용자 이름으로 해당 사용자의 활동 이력 검색
- **CSV 내보내기**: 조회된 히스토리 로그를 CSV 파일로 내보내기
- **페이지네이션**: 한 페이지에 20건씩 표시하며 페이지 이동 가능

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 전체 테넌트의 모든 지점 로그 | 지점 필터 사용, CSV 내보내기 |
| 지점 관리자 | 자신의 지점 범위 로그 | 기간·액션·대상·사용자 필터, CSV 내보내기 |

### 주요 UX 흐름
1. 화면 진입 시 최근 30일의 전체 활동 로그가 자동으로 조회된다.
2. 필터를 조합하여 원하는 조건의 로그만 조회할 수 있다.
3. 검색 버튼 클릭 시 설정된 조건으로 조회가 실행된다.
4. 전체 건수와 현재 조회된 로그 목록이 표시된다.
5. CSV 내보내기 버튼으로 조회 결과 전체를 파일로 저장할 수 있다.

### 비즈니스 규칙
- 히스토리 로그는 테넌트(회사) 단위로 격리되며, 다른 회사의 로그는 조회할 수 없다.
- 슈퍼관리자는 지점 필터로 특정 지점의 로그만 조회할 수 있다.
- 로그는 수정하거나 삭제할 수 없으며 조회 전용이다.
- 기록되는 주요 활동 유형: 로그인/로그아웃, 로그인 실패, 데이터 생성/수정/삭제, 지점 전환, 권한 변경, 퇴직, 이동, 휴직 시작/종료, 회원 이적, 회원 탈퇴, 환불, 내보내기, 설정 변경, 지점 생성/폐점, 슈퍼관리자 권한 부여/회수

## 9. Today Tasks (`/today-tasks`)

### 화면 목적
오늘 처리해야 할 업무를 역할과 지점 기준으로 배정받아 확인하는 화면이다. 본사 업무 풀에서 날짜와 역할에 맞게 배정된 업무를 보여주며, 슈퍼관리자는 직접 업무를 추가하거나 수정할 수 있다.

### 주요 기능
- **오늘의 업무 목록**: 상담, 재등록, 출석회복, PT관리, 운영, 매출, 수업 등 7가지 카테고리의 업무가 역할별로 배정되어 표시
- **업무 현황 요약 카드**: 오늘 배정된 전체 업무 수, 대기 중, 진행중, 완료 건수를 카드로 표시
- **탭 필터**: 전체, 대기, 진행중, 완료 탭으로 업무 목록 필터링
- **업무 카드**: 각 업무의 카테고리, 우선순위, 상태, 제목, 설명, 마감 시간, 담당 회원, 지점명 정보 표시
- **슈퍼관리자 업무 직접 추가**: 슈퍼관리자는 업무 제목, 설명, 카테고리, 우선순위, 마감, 담당 회원, 상태를 입력하여 직접 업무를 추가하거나 수정·삭제 가능

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 전체 지점 대상 18건의 업무 + 직접 추가한 업무 | 업무 추가, 수정, 삭제, 탭 필터링 |
| 지점 관리자 / 직원 | 자신의 역할과 지점에 배정된 12건의 업무 | 탭 필터링 (상태 변경 기능 추후 지원) |
| FC | FC 역할에 배정된 업무 | 탭 필터링 |
| 트레이너 | 트레이너 역할에 배정된 업무 | 탭 필터링 |

### 주요 UX 흐름
1. 화면 진입 시 오늘 날짜와 자신의 역할, 지점을 기준으로 배정된 업무 목록이 자동으로 표시된다.
2. 상단 탭으로 상태별 업무를 필터링하여 확인한다.
3. 슈퍼관리자는 화면 상단의 입력 폼을 통해 새 업무를 추가하거나 기존 업무를 수정할 수 있다.
4. 슈퍼관리자가 추가한 업무에는 "본사 등록" 배지가 표시된다.
5. 업무 카드에서 우선순위(긴급/높음/보통)와 상태(대기/진행중/완료)를 배지로 확인할 수 있다.

### 비즈니스 규칙
- 업무는 날짜, 지점, 역할을 조합한 기준으로 매일 자동 배정되며, 같은 조건에서는 항상 동일한 업무가 배정된다.
- 슈퍼관리자에게는 18건, 일반 역할에게는 12건이 배정된다.
- 슈퍼관리자가 직접 추가한 업무는 해당 역할과 지점에 속하는 사용자에게 표시된다.
- 업무 카테고리: 상담, 재등록, 출석회복, PT관리, 운영, 매출, 수업
- 업무 우선순위: 긴급, 높음, 보통
- 업무 상태: 대기, 진행중, 완료 (상태 변경 기능은 추후 지원 예정)

## 10. 리포트 (`/reports`)

### 화면 목적
일간, 주간, 월간 단위로 지점의 운영 성과를 자동으로 집계하고 조회하는 화면이다. 기간별 KPI 요약을 확인하고 필요 시 수동으로 리포트를 생성할 수 있다.

### 주요 기능
- **자동 생성 리포트 목록**: 최근 7일의 일간 리포트, 최근 4주의 주간 리포트, 최근 3개월의 월간 리포트가 자동으로 생성되어 목록에 표시
- **리포트 유형 구분**: 일간(파랑), 주간(초록), 월간(보라) 색상 배지로 유형 구분
- **리포트 상세 조회**: 리포트를 클릭하면 상세 모달에서 총 매출, 전체 회원, 신규 회원, 총 출석 KPI 카드 확인
- **수동 리포트 생성**: 원하는 기간과 유형(일간/주간/월간)을 선택하여 리포트를 직접 생성
- **CSV 내보내기**: 리포트 상세 내용을 CSV 파일로 내보내기

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 | 현재 선택 지점의 리포트 목록 | 리포트 조회, 생성, CSV 내보내기 |
| 지점 관리자 | 자신의 지점 리포트 목록 | 리포트 조회, 생성, CSV 내보내기 |

### 주요 UX 흐름
1. 화면 진입 시 자동 생성된 최근 리포트 목록이 표시된다.
2. 리포트 항목을 클릭하면 해당 기간의 KPI 상세 모달이 열린다.
3. "리포트 생성" 버튼 클릭 시 유형과 기간을 입력하는 모달이 열린다.
4. 시작일이 종료일보다 늦으면 생성이 불가하며 안내 메시지가 표시된다.
5. 상세 모달에서 CSV 내보내기 버튼으로 해당 리포트를 파일로 저장할 수 있다.

### 비즈니스 규칙
- 리포트는 매출, 회원, 출석 데이터를 지정 기간 기준으로 자동 집계한다.
- 수동 생성 시 시작일은 종료일보다 이전이어야 한다.
- 매출 금액은 1억 이상이면 "X.X억", 1만 이상이면 "X만" 형식으로 표시한다.

## 11. 로그인 (`/login`)

### 화면 목적
FitGenie CRM에 접속하기 위한 인증 화면이다. 지점 선택 후 아이디와 비밀번호를 입력하여 로그인하며, 반복 실패 시 계정이 일시적으로 잠긴다.

### 주요 기능
- **지점 선택**: 로그인 전 소속 지점을 먼저 선택
- **아이디 / 비밀번호 입력**: 발급받은 계정 정보로 로그인
- **비밀번호 표시 토글**: 눈 아이콘 클릭으로 비밀번호 표시/숨김 전환
- **로그인 유지(아이디 저장)**: 체크 시 다음 방문에 아이디가 자동으로 입력됨
- **계정 잠금**: 로그인 5회 연속 실패 시 30분간 계정이 잠김
- **고객센터 안내**: 도움이 필요한 경우 고객센터 연락처 확인

### 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 모든 사용자 | 동일한 로그인 화면 | 지점 선택, 아이디/비밀번호 입력, 로그인 유지 설정 |
| 슈퍼관리자 | 동일 | 지점 선택 없이 로그인 가능(지점 미지정 상태로 진입) |

### 주요 UX 흐름
1. 화면에 접속하면 지점 선택, 아이디, 비밀번호 입력란이 표시된다.
2. 지점을 선택하고 아이디와 비밀번호를 입력한 후 로그인 버튼을 클릭하거나 Enter 키를 누른다.
3. 로그인 중에는 버튼에 로딩 표시가 나타나며 중복 클릭이 방지된다.
4. 로그인 성공 시 대시보드로 자동 이동한다.
5. 로그인 실패 시 남은 시도 횟수를 포함한 오류 메시지가 표시된다.
6. 5회 연속 실패 시 계정이 잠기며, 잠금 안내 메시지와 함께 로그인이 차단된다.
7. 30분 후 잠금이 자동으로 해제된다.
8. "비밀번호 찾기" 클릭 시 관리자에게 문의하도록 안내한다.

### 비즈니스 규칙
- 로그인 5회 연속 실패 시 해당 기기에서 30분간 로그인이 차단된다.
- 잠금은 30분 경과 후 자동으로 해제된다.
- 로그인 성공 시 실패 횟수가 초기화된다.
- "로그인 유지" 체크 시 아이디가 해당 기기에 저장되어 다음 방문 시 자동 입력된다.
- 비밀번호 찾기 기능은 별도 플로우 없이 관리자 문의로 안내한다.
- 2단계 인증(2FA) 및 간편 로그인(OAuth/SSO)은 추후 지원 예정이다.
