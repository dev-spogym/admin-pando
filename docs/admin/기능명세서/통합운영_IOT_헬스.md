## 권한/직무 해석 원칙

- 관리자웹 권한 키는 `superAdmin / primary / owner / manager / fc / staff / readonly`를 기준으로 해석한다.
- 본문에서 보이는 `센터장`, `매니저`, `스태프`, `프런트`는 현장 운영 라벨 또는 직무 명칭으로 병기될 수 있다.
- 통합운영 문서의 `트레이너`, `직원`, `프런트`는 기기 사용 주체를 뜻하며, 관리자웹 기본 권한 키와 동일 개념이 아니다.
- 실제 메뉴 접근 범위는 `SCR-081 권한 설정` 또는 커스텀 역할 매핑 기준으로 결정한다.

## 1. 통합 출석 관리

**화면 경로**: 출석 관리

### 기능 개요

회원과 직원의 출석 현황을 하나의 화면에서 실시간으로 통합 관리한다. 키오스크 QR, 얼굴인식, 앱 QR, 직원 수동 입력 등 모든 출석 채널의 데이터를 통합 집계하며, 출석 후 옷 락커 배정 상태까지 함께 확인할 수 있다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| 통합 출석 현황 조회 | 회원·직원 구분 없이 오늘의 출석 현황을 실시간으로 확인 |
| 수동 출석 등록 | 기기 없이 관리자가 회원 또는 직원의 출석을 직접 등록 |
| 퇴실 처리 | 입장 상태의 회원·직원을 퇴실로 전환 |
| 출석 채널 구분 | 앱 QR / 키오스크 QR / 얼굴인식 / 수동 입력 채널 구분 표시 |
| 락커 배정 상태 표시 | 출석 후 옷 락커 미배정·수동배정·자동배정 상태를 함께 표시 |
| 실패 사유 세분화 | 이용권 만료·중복 출석·미등록 등 실패 원인 구분 표시 |

### 조회 필터

| 필터 | 설명 |
|------|------|
| 기간 | 오늘·이번 주·이번 달·직접 선택 |
| 구분 | 회원 / 직원 / 게스트 |
| 결과 | 성공 / 실패 / 중복 / 퇴실 |
| 채널 | 앱 QR / 키오스크 QR / 얼굴인식 / 수동 |
| 락커 상태 | 미배정 / 수동배정 / 자동배정 / 대상 아님 |
| 검색 | 이름, 연락처, 회원번호, 사번 |

### 운영 정책

- 출석 실패 시에도 시도 이력은 저장하며, 실패 사유를 운영 실패(정책 위반)와 인증 실패(미등록·기기 오류)로 구분한다.
- 중복 출석 방지 시간 내 재시도는 '중복'으로 별도 처리한다.
- 자동 락커 배정이 실패해도 출석 자체는 성공으로 저장하며, 미배정 상태로 표시한다.

### 관련 팝업

- 수동 출석 등록 팝업 (DLG-I001)
- 옷 락커 배정 팝업 (DLG-I002)

## 2. 키오스크 설정

**화면 경로**: 설정 > 키오스크 설정

### 기능 개요

지점별 키오스크 운영 방식을 설정한다. 출석 인식 수단, 화면 구성, 음성 안내, 출입 규칙, 출석 후 락커 처리 정책을 탭별로 관리한다.

### 설정 탭 구성

| 탭 | 주요 설정 항목 |
|----|--------------|
| 기본 설정 | 키오스크 활성 여부, 체크인 수단(QR·RFID·얼굴인식), 직원 출석 허용 |
| 화면 설정 | 배경 이미지, 로고, 환영 문구, 공지 메시지 |
| TTS 설정 | 대기·성공·실패·만료 임박 상황별 음성 안내 문구 |
| 출입 규칙 | 출입 허용 시간, 만료 회원 허용 여부, 중복 방지 시간 |
| 락커 후처리 | 출석 후 락커 처리 정책, 자동 배정 존, 미배정 안내 문구 |

### 출석 후 락커 처리 정책

| 정책 | 설명 |
|------|------|
| 출석만 완료 | 락커 관련 후속 처리 없음 |
| 수동 배정 대기 | 출석 성공 후 인포데스크 안내 문구 노출, 직원이 수동 배정 |
| 자동 배정 시도 | 지정된 락커 존에서 빈 락커를 자동 탐색하여 배정 |

### 운영 정책

- 직원 출석 허용이 꺼진 지점은 직원 QR·얼굴인식 모두 실패 처리된다.
- 얼굴인식 비활성화 지점은 관련 화면과 메시지를 표시하지 않는다.
- 자동 배정 실패 시에도 출석은 성공으로 처리하며, 미배정 안내 문구를 키오스크에 표시한다.

## 3. IoT 연동 관리

**화면 경로**: 설정 > IoT 연동 관리

### 기능 개요

지점에 설치된 모든 IoT 장비(출입 게이트, 키오스크, 락커 컨트롤러, InBody 측정기)의 연결 상태를 모니터링하고 관리한다.

### 관리 대상 장비

| 장비 유형 | 설명 |
|-----------|------|
| 출입 게이트 | 출입문 자동 개폐 장치 |
| 키오스크 | 회원 체크인 단말기 |
| 락커 컨트롤러 | 락커 자동 배정 연동 장치 |
| InBody 측정기 | 체성분 실시간 수신 연동 장치 |

### 주요 기능

| 기능 | 설명 |
|------|------|
| 기기 등록·삭제 | 신규 장비 등록 및 비활성화 |
| 연결 상태 모니터링 | 온라인·오프라인·오류·점검 중 상태 실시간 확인 |
| 연결 테스트 | 개별 기기 연결 상태 즉시 확인 |
| 원격 게이트 개방 | 예외 상황 시 관리자가 원격으로 출입문 개방 (감사 로그 자동 생성) |
| 출입 로그 조회 | 출입 성공·실패·예외 이벤트 이력 조회 |
| 출입 규칙 설정 | 시간대·요일·게이트별 출입 허용 규칙 설정 |
| 자동 전원 스케줄 | 키오스크·기기 전원 ON/OFF 일정 관리 |

### InBody 데이터 수집 채널

| 채널 | 설명 |
|------|------|
| 수기 입력 | 직원이 측정지를 보고 수동 입력 |
| 파일 업로드 | 장비에서 추출한 파일을 시스템에 업로드 |
| 실시간 API | InBody 장비에서 직접 시스템으로 데이터 전송 |

### 운영 정책

- 오프라인·오류·점검 중 장비는 자동 배정·자동 처리 대상에서 제외된다.
- 락커 컨트롤러 장애 시 자동 락커 배정 정책은 수동 배정 모드로 전환된다.
- 게이트 원격 개방은 반드시 감사 로그에 기록된다.

## 4. 옷 락커 운영 관리

**화면 경로**: 락커 관리 > 옷 락커

### 기능 개요

출석과 연동되는 당일용 옷 락커를 실시간으로 운영한다. 락커 그리드에서 전체 사용 현황을 확인하고, 출석 후 미배정 회원에게 락커를 배정하거나 회수한다.

### 옷 락커와 고정 물품 락커 구분

| 항목 | 옷 락커 | 고정 물품 락커 |
|------|---------|----------------|
| 용도 | 운동복·개인용품 당일 보관 | 개인 소지품 장기 보관 |
| 배정 시점 | 출석 후 | 상품 계약 시 |
| 기본 배정 방식 | 직원 수동 배정 | 관리자 고정 배정 |
| 만료 기준 | 당일 종료·퇴실 | 계약 만료일 |

### 락커 상태 구분

| 상태 | 설명 |
|------|------|
| 빈 락커 | 배정 가능 |
| 사용 중 | 회원 배정 완료 |
| 만료 임박 | 당일 마감 시간 근접 |
| 반납 지연 | 퇴실 후 미반납 |
| 점검 중 | 고장 또는 청소 처리 중 |

### 주요 기능

| 기능 | 설명 |
|------|------|
| 락커 그리드 조회 | 상태별 락커 전체 현황 조회 |
| 미배정 회원 패널 | 오늘 출석 후 락커 미배정 상태 회원 목록 |
| 개별 배정 | 회원 선택 후 빈 락커 배정 |
| 회수 | 배정된 락커에서 회원 연결 제거 |
| 고장·복구 처리 | 락커를 점검 상태로 변경 또는 정상 복구 |
| 일괄 해제 | 만료·반납 지연 락커 일괄 회수 |

### 운영 정책

- 기본 배정 방식은 직원 수동 배정이며, 지점 설정에 따라 자동 배정을 활성화할 수 있다.
- 자동 배정 실패 시 미배정 목록으로 이동하여 직원이 수동 처리한다.
- 점검 중·청소 중 락커는 배정 불가 상태이다.

## 5. 고정 물품 락커 관리

**화면 경로**: 락커 관리 > 사물함 배정

### 기능 개요

상품 계약과 연결된 장기 보관 락커를 회원 단위로 관리한다. 신규 배정, 만료 연장, 회수를 처리하며 만료 예정 회원을 사전에 확인한다.

### 락커 유형

| 유형 | 설명 |
|------|------|
| 개인 물품 | 일반 소형 개인 소지품 보관 |
| 골프 | 골프백 보관용 대형 락커 |
| 프리미엄 | 계약 기반 프리미엄 락커 |

### 주요 기능

| 기능 | 설명 |
|------|------|
| 배정 현황 조회 | 유형별·상태별 배정 현황 조회 |
| 신규 배정 | 회원 + 락커 + 만료일 등록 |
| 만료일 연장 | 계약 갱신 시 만료일 연장 처리 |
| 회수 | 회원 연결 제거 (이전 이력은 보존) |
| 상태 동기화 | 만료 상태 재계산 및 배지 갱신 |
| 엑셀 다운로드 | 현재 배정 목록 파일 다운로드 |

### 배정 상태 구분

| 상태 | 설명 |
|------|------|
| 사용 중 | 정상 배정 중 |
| 만료 예정 | 7일 이내 만료 예정 |
| 만료 | 계약 만료, 미연장 |
| 회수 대기 | 만료 후 미회수 상태 |

### 운영 정책

- 락커 상품을 보유하지 않은 회원에게는 신규 배정이 불가하다.
- 계약 만료 후 미연장 상태는 회수 대기로 전환된다.
- 회수 시 이전 배정 이력은 삭제하지 않고 보존한다.

## 6. 체성분 통합 관리

**화면 경로**: 체성분 관리

### 기능 개요

전체 회원의 체성분 측정 데이터를 수기 입력, 파일 업로드, 실시간 장비 연동 세 가지 채널로 수집하고 검수·확정하여 관리한다.

### 데이터 수집 채널

| 채널 | 설명 |
|------|------|
| 수기 입력 | 직원이 측정지를 보고 수동으로 수치 입력 |
| 파일 업로드 | 장비에서 추출한 CSV·XLSX 파일 업로드 및 자동 파싱 |
| 실시간 API | InBody 장비에서 측정 즉시 시스템으로 데이터 전송 |

### 탭 구성

| 탭 | 설명 |
|----|------|
| 측정 결과 | 확정된 체성분 결과 목록 조회 |
| 업로드 센터 | 파일 업로드 및 파싱 결과 확인·확정 |
| 실시간 연동 | InBody API 수신 로그 확인 |
| 오류/검수 | 중복·매칭 실패·이상치 건 검수 처리 |

### 주요 측정 항목

| 항목 | 설명 |
|------|------|
| 체중 (kg) | 측정 체중 |
| 골격근량 (kg) | 근육량 |
| 체지방량 (kg) | 체지방 절대량 |
| 체지방률 (%) | 체지방 비율 |
| BMI | 체질량지수 |
| 기초대사량 (kcal) | 기초 에너지 소비량 |
| 체수분 (L) | 체내 수분량 |

### 운영 정책

- 동일 회원·동일 측정 시각의 중복 데이터는 자동 확정하지 않고 검수 대기 처리한다.
- 회원 앱과 회원 상세 화면에는 확정 처리된 최신 데이터만 표시된다.
- 수기 수정이 발생하면 원본 소스는 유지하고 수정 이력을 별도로 기록한다.

### 관련 팝업

- 체성분 수기 등록 팝업 (DLG-I003)

## 7. 회원 건강 연동 요약

**화면 경로**: 회원 건강 연동 (미구현 예정)

### 기능 개요

회원별 체성분, 운동 이력, 건강 목표, 외부 건강 앱 연동 상태를 하나의 화면에서 통합 요약한다. 관리자가 회원 상담 시 건강 데이터를 빠르게 참고할 수 있도록 지원한다.

### 요약 정보 항목

| 항목 | 설명 |
|------|------|
| 최근 출석 | 마지막 체크인 시각 |
| 오늘 옷 락커 | 당일 배정된 락커 번호 또는 미배정 |
| 고정 락커 | 보유 중인 고정 물품 락커 번호 |
| 최근 InBody | 체중·골격근량·체지방률 최신 수치 |
| 최근 7일 활동량 | 걸음수 합계, 운동 일수 |

### Health Connect 연동

회원 앱(Android)을 통해 Google Health Connect와 연동하여 외부 활동량 데이터를 수집한다.

| 항목 | 설명 |
|------|------|
| 연동 상태 | 연결됨 / 동기화 대기 / 미연결 |
| 최근 동기화 시각 | 마지막 데이터 수신 시각 |
| 활동량 비교 | 센터 출석 운동 vs 외부 활동량 비교 |

> Health Connect 연동은 Android 회원 앱을 통한 간접 연동이며, 웹 직접 연동이 아님을 안내 화면에 명시한다.

### 운영 정책

- 최근 InBody는 확정 처리된 최신 데이터만 표시한다.
- 오늘 옷 락커는 퇴실 처리 후 회수 상태로 표기한다.
- 고정 물품 락커는 만료 후에도 최근 상태를 확인할 수 있다.

## 8. 공통 운영 정책

### 락커 운영 원칙

| 항목 | 정책 |
|------|------|
| 기본 출석 후 처리 | 직원 수동 배정 |
| 자동 락커 배정 | 지점 설정에서 활성화 시에만 적용 |
| 락커 유형 구분 | 옷 락커와 고정 물품 락커는 완전히 분리 운영 |

### 체성분 수집 원칙

| 채널 | 수집 방식 |
|------|-----------|
| 수기 입력 | 직원이 직접 입력 |
| 파일 업로드 | 장비 파일 파싱 |
| 실시간 API | 장비 자동 전송 |

### 권장 운영 순서

1. 키오스크 설정에서 지점별 출석 방식 및 락커 후처리 정책 설정
2. IoT 연동 관리에서 장비 등록 및 연결 테스트
3. 출석 후 옷 락커 운영 관리에서 미배정 회원 확인 및 배정
4. 고정 물품 락커 만료 예정 회원 확인 및 연장·회수 처리
5. 체성분 데이터 수집 채널 설정 및 검수 처리
6. 회원 건강 연동 요약을 통한 상담 자료 활용
