# 제1장. 조직 구조 및 역할 체계

## 1.1 본사-지점 2계층 구조

```
본사 (HQ)
  ├── 회장 ──────────── 최고 의사결정권자
  ├── 대표 ──────────── 본사 총괄 + 지점 겸임 가능
  └── 본사 직원 ──────── 본사 운영/관리 업무
       │
       ├── 지점 A (강남점)
       │     ├── 지점장
       │     ├── 스태프
       │     ├── FC (피트니스 컨설턴트)
       │     └── 트레이너
       │
       ├── 지점 B (역삼점)
       │     ├── 지점장
       │     ├── 스태프
       │     ├── FC
       │     └── 트레이너
       │
       └── 지점 C (판교점)
             └── ...
```

## 1.2 역할 정의 (7종)

### 본사 역할 (3종)

| 코드 | 한글명 | 권한 레벨 | 소속 | 설명 |
|---|---|:---:|---|---|
| `chairman` | 회장 | 100 | 본사 | 최고 의사결정권자. 전 지점 모든 권한. 시스템 설정/구독/조직 관리 |
| `ceo` | 대표 | 90 | 본사 + 지점 겸임 | 본사 총괄 운영 + 겸임 지점 직접 관리. 전 지점 데이터 접근 가능 |
| `hq_staff` | 본사 직원 | 80 | 본사 | 본사 운영 업무. 전 지점 데이터 조회/관리, 결재 처리, 리포트 관리 |

### 지점 역할 (4종)

| 코드 | 한글명 | 권한 레벨 | 소속 | 설명 |
|---|---|:---:|---|---|
| `branch_mgr` | 지점장 | 70 | 지점 | 지점 최고 책임자. 지점 내 전체 권한. 본사 결재 요청 |
| `staff` | 스태프 | 40 | 지점 | 프론트 데스크. 회원 접수, POS 결제, 출석, 시설 운영 |
| `fc` | FC | 30 | 지점 | 피트니스 컨설턴트. 회원 상담, 영업, PT 판매, 회원 관리 |
| `trainer` | 트레이너 | 20 | 지점 | 수업 진행, 운동 지도, 체성분 측정, 운동 프로그램 작성 |

### FC vs 트레이너 역할 차이

| 구분 | FC (피트니스 컨설턴트) | 트레이너 |
|---|---|---|
| 핵심 업무 | 회원 상담, 영업, PT/이용권 판매 | PT/GX 수업 진행, 운동 지도 |
| 매출 관여 | 직접 판매 (매출 귀속) | 서비스 제공 (수업 인센티브) |
| 회원 접점 | 상담/계약/재등록 유도 | 수업/운동 프로그램/체성분 |
| 매출 데이터 | 본인 판매 실적 조회 가능 | 본인 수업 실적만 조회 |
| 상품 관련 | 상품 조회 + POS 판매 가능 | 상품 조회만 |

## 1.3 역할 계층 원칙

- 본사 역할(chairman/ceo/hq_staff)은 **전 지점 데이터에 대한 CRUD 권한** 보유
- 지점 역할은 **소속 지점 데이터만** 접근 가능
- `ceo`는 본사 소속이면서 특정 지점을 겸임 가능 (해당 지점의 지점장 역할 동시 수행)
- `chairman`은 모든 권한 자동 부여 (코드에서 bypass)
- 지점 역할은 본사 결재를 통해서만 특정 행위 가능 (1.5절 참조)

## 1.4 권한 유형 (5종)

| 유형 | 코드 | 설명 |
|---|---|---|
| 접근 | `access` | 해당 메뉴/페이지에 진입 가능 여부 |
| 조회 | `read` | 목록·상세 데이터 열람 가능 여부 |
| 등록 | `create` | 새 데이터 생성 가능 여부 |
| 수정 | `update` | 기존 데이터 변경 가능 여부 |
| 삭제 | `delete` | 데이터 삭제(비활성화) 가능 여부 |

## 1.5 본사 결재(승인) 프로세스

### 핵심 원칙

> 지점에서 경영/재무/정책에 영향을 미치는 행위를 하려면, **지점장이 본사에 결재를 올리고 승인을 받아야** 해당 행위를 수행할 수 있다.

### 결재 흐름

```
[지점장: 결재 요청 작성]
    ↓
[본사 직원: 검토 + 의견 첨부] (선택)
    ↓
[대표 또는 회장: 승인/반려/보류]
    ↓
  승인 → 지점에서 해당 행위 수행 가능
  반려 → 사유 첨부 → 지점장에게 반환
  보류 → 추가 자료 요청
```

### 결재 필요 항목

| 카테고리 | 결재 필요 행위 | 요청자 | 승인자 |
|---|---|---|---|
| **할인/프로모션** | 할인 쿠폰 생성 | 지점장 | 대표/회장 |
| | 프로모션 신규 등록 | 지점장 | 대표/회장 |
| | 10% 초과 할인 적용 | 지점장 | 대표/회장 |
| | 제휴 업체 할인 계약 | 지점장 | 대표/회장 |
| **상품/가격** | 신규 상품 등록 | 지점장 | 대표/회장 |
| | 상품 가격 변경 | 지점장 | 대표/회장 |
| | 상품 폐지/비활성화 | 지점장 | 대표/회장 |
| **환불** | 50만 원 초과 환불 | 지점장 | 대표/회장 |
| | 위약금 면제 환불 | 지점장 | 대표/회장 |
| **인사** | 직원 채용 (정규직) | 지점장 | 대표/회장 |
| | 직원 해고/퇴사 처리 | 지점장 | 대표/회장 |
| | 급여 체계 변경 | 지점장 | 회장 |
| | 인센티브율 변경 | 지점장 | 대표/회장 |
| **시설** | 대형 장비 구매 (100만 원+) | 지점장 | 대표/회장 |
| | 시설 리모델링/확장 | 지점장 | 회장 |
| **운영** | 영업 시간 변경 | 지점장 | 대표/회장 |
| | 지점 일시 중단 | 지점장 | 회장 |
| | 센터 설정 주요 변경 | 지점장 | 대표/회장 |
| **재무** | 미수금 탕감 | 지점장 | 대표/회장 |
| | 매출 마감 후 수정 | 지점장 | 대표/회장 |
| | 월 마감 확정 취소 | 지점장 | 회장 |

### 결재 상태

| 상태 | 설명 |
|---|---|
| `DRAFT` | 작성 중 (지점장) |
| `SUBMITTED` | 제출됨 (본사 검토 대기) |
| `REVIEWING` | 본사 직원 검토 중 |
| `APPROVED` | 승인 완료 → 해당 행위 수행 가능 |
| `REJECTED` | 반려 (사유 첨부) → 수정 후 재요청 가능 |
| `ON_HOLD` | 보류 (추가 자료/논의 필요) |

### 결재 규칙

| 항목 | 규칙 |
|---|---|
| 요청 권한 | 지점장만 (스태프/FC/트레이너는 지점장에게 구두 요청) |
| 긴급 결재 | 대표/회장 전화 승인 후 사후 결재 처리 가능 (24시간 내) |
| 결재 유효 기간 | 승인일로부터 30일 (기간 내 미이행 시 재요청 필요) |
| 결재 이력 | 전 건 AuditLog 기록 (요청/승인/반려 이력 보존) |
| 본사 직원 권한 | 검토 + 의견 첨부 가능, 승인/반려는 대표/회장만 |
| 대표 부재 시 | 회장이 직접 처리 또는 본사 직원에게 임시 승인 위임 (회장 승인) |

### 결재 없이 지점장이 단독 수행 가능한 항목

| 카테고리 | 행위 |
|---|---|
| 회원 | 회원 등록/수정/탈퇴, 휴회/정지 처리, 이관 요청 |
| 수업 | 수업 등록/수정/삭제, 시간표 관리, 페널티 부과 |
| 매출 | POS 결제, 50만 원 이하 환불, 미수금 수납 |
| 시설 | 락커/RFID/운동룸/운동복 일상 관리 |
| 메시지 | 메시지 발송 (100건 미만), 쿠폰 발급 (기 승인된 쿠폰) |
| 직원 | 근태 관리, 파트타임 채용, 직원 일정 관리 |
| 설정 | 키오스크 설정, IoT 설정, 소소한 센터 설정 |

# 제2장. 지점 구조와 데이터 격리

## 2.1 멀티테넌트 구조

```
Tenant (법인/브랜드)
  ├── 본사 (HQ)
  │     ├── 회장
  │     ├── 대표 ─── 겸임 → 강남점, 역삼점
  │     └── 본사 직원 A, B, C
  │
  ├── 지점 A (강남점) ─ ACTIVE
  │     ├── 지점장
  │     ├── 스태프 × 2
  │     ├── FC × 2
  │     └── 트레이너 × 3
  │
  ├── 지점 B (역삼점) ─ ACTIVE
  └── 지점 C (판교점) ─ PAUSED
```

## 2.2 지점 상태 (4종)

| 상태 | 코드 | 설명 |
|---|---|---|
| 운영중 | `ACTIVE` | 정상 운영. 모든 기능 사용 가능 |
| 일시중단 | `PAUSED` | 임시 휴점. 결재 필요 (회장 승인) |
| 폐점진행 | `CLOSING` | 폐점 절차 진행 중. 결재 필요 (회장 승인) |
| 폐점완료 | `INACTIVE` | 폐점 완료. 아카이브 (조회 전용) |

## 2.3 데이터 격리 원칙

| 원칙 | 설명 |
|---|---|
| 본사 → 전 지점 | 본사 역할(chairman/ceo/hq_staff)은 전 지점 데이터 CRUD 가능 |
| 지점 → 소속 지점만 | 지점 역할은 소속 지점 데이터만 접근 |
| 지점 ↔ 타 지점 | 교차 접근 불가 (이관/타 지점 출석 등 명시된 예외 제외) |
| 테넌트 격리 | 다른 테넌트(법인) 데이터는 어떤 경우에도 접근 불가 |

## 2.4 사용자 소속 규칙

| 역할 | 소속 | 지점 전환 |
|---|---|---|
| chairman | 본사 (branchId null) | `switchBranch()`로 전 지점 전환 가능 |
| ceo | 본사 + 겸임 지점 | `switchBranch()`로 전 지점 전환 가능 |
| hq_staff | 본사 (branchId null) | `switchBranch()`로 전 지점 전환 가능 |
| branch_mgr | 특정 지점 1개 | 불가 (소속 지점만) |
| staff / fc / trainer | 특정 지점 1개 | 불가 (소속 지점만) |

# 제4장. 회원 라이프사이클 정책

## 4.1 회원 상태 전이 규칙

### 전체 상태 흐름도

```
[신규 등록] → ACTIVE
                 │
        ┌────────┼────────┬──────────┬──────────┐
        ↓        ↓        ↓          ↓          ↓
    HOLDING   SUSPENDED  EXPIRED  WITHDRAWN  TRANSFERRED
     (휴회)     (정지)    (만료)     (탈퇴)      (이관)
        │        │        │
        ↓        ↓        ↓
     ACTIVE   ACTIVE   DORMANT ──→ [데이터 파기]
    (복귀)    (해제)    (휴면)       (3년 후)
```

### 상태별 허용 행위

| 행위 | ACTIVE | HOLDING | SUSPENDED | EXPIRED | DORMANT | WITHDRAWN |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| 출입 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 수업 예약 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 결제 | ✅ | ❌ | 🔸 | ✅ | ✅ | ❌ |
| 환불 | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| 휴회 신청 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 양도 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 이관 | ✅ | ✅ | ❌ | ✅ | ❌ | ❌ |

- SUSPENDED 결제: 미납 해소 목적만 허용

## 4.2 휴회(HOLDING) 정책

| 항목 | 규칙 |
|---|---|
| 신청 조건 | ACTIVE 상태, 미수금 없음, 잔여 기간 30일+ |
| 최소/최대 기간 | 7일 ~ 90일 |
| 연간 횟수 | 최대 2회 |
| 이용 기간 | 휴회 일수만큼 자동 연장 |
| 시설 이용 | 전면 차단 |
| 자동 복귀 | 종료일 00:00 자동 ACTIVE 전환 |
| 조기 복귀 | 가능 (잔여 휴회 기간 소멸) |
| 신청 권한 | 스태프+ 또는 회원 앱 |
| 승인 권한 | 일반 휴회: 즉시 승인, 특수 휴회(90일 초과): 지점장 승인 |

### 특수 휴회 (90일 초과)

| 사유 | 최대 기간 | 필요 서류 | 승인 |
|---|---|---|---|
| 의료 (수술/입원) | 180일 | 진단서 | 지점장 |
| 군 입대 | 복무 기간 | 입영통지서 | 지점장 |
| 임신/출산 | 365일 | 임신확인서/출생증명서 | 지점장 |
| 해외 장기 체류 | 180일 | 항공권/비자 | 지점장 |

## 4.3 정지(SUSPENDED) 정책

| 분류 | 사유 | 정지 기간 | 부과 |
|---|---|---|---|
| 미납 | 미수금 30일 초과 | 수납 시까지 | 시스템 자동 |
| 자동결제 실패 | 3회 연속 실패 | 결제 성공 시까지 | 시스템 자동 |
| 경미 위반 | 시설 규칙 위반 | 7일 | 지점장 |
| 중대 위반 | 시설 파손, 위협 | 14~30일 | 지점장 |
| 폭력/성범죄 | 물리적 폭행 등 | 영구 (→ 블랙리스트) | 지점장 + 본사 |

- 미납 정지: 이용 기간 정지 (복귀 시 잔여 기간 유지)
- 규정 위반 정지: 이용 기간 계속 소진
- 해제: 미납→수납 시 자동, 기간 만료→자동, 조기 해제→지점장

## 4.4 만료(EXPIRED) 자동 전환 정책

| 항목 | 규칙 |
|---|---|
| 기간제 | endDate 도래 시 자동 전환 (매일 00:00 배치) |
| 횟수제 | 잔여 횟수 0 시 즉시 전환 |
| 혼합형 | 기간/횟수 중 먼저 도래한 조건 |
| 사전 알림 | 30일, 14일, 7일, 3일, 1일 전 |
| 유예 기간 | 만료 후 7일 (재등록 시 기존 데이터 연계, 등록비 면제) |
| 출입 | 만료일 당일까지 허용, 익일 차단 |

## 4.5 휴면(DORMANT) 전환 정책

| 항목 | 규칙 |
|---|---|
| 전환 조건 | EXPIRED 상태에서 90일 경과 |
| 처리 | 매일 배치 자동 전환 |
| 전환 전 알림 | 60일, 75일, 85일 경과 시 SMS |
| 데이터 파기 | 휴면 후 3년 경과 시 개인정보 파기 (개인정보보호법) |
| 파기 전 알림 | 파기 30일 전 SMS |
| 휴면 해제 | 재등록 시 자동 ACTIVE |

## 4.6 회원 양도/양수 정책

| 항목 | 규칙 |
|---|---|
| 양도 조건 | ACTIVE 상태, 잔여 기간 30일+/횟수 5회+, 미수금 없음 |
| 양도 대상 | 동일 지점 내 |
| 수수료 | 잔여 가치의 10% (최소 3만 원, 최대 10만 원) |
| 양도 횟수 | 1개 이용권당 1회 (재양도 불가) |
| 신청 방법 | 양도인+양수인 방문 필수 |
| 이전 항목 | 잔여 기간, 잔여 횟수, 락커(선택) |

# 제6장. 환불/위약금 정책

## 6.1 환불 계산 공식

### 기간제 상품

```
환불 금액 = 결제 금액 - (일일 이용료 × 사용일수) - 위약금
일일 이용료 = 결제 금액 ÷ 총 이용일수
위약금 = (결제 금액 - 일일 이용료 × 사용일수) × 위약금률
```

### 횟수제 상품

```
환불 금액 = 결제 금액 - (1회 단가 × 사용횟수) - 위약금
1회 단가 = 정가 기준 (할인 전 정가 적용 — 공정거래위원회 고시)
```

### 혼합형

```
사용액 = MAX(기간 비례, 횟수 비례)
환불 금액 = 결제 금액 - 사용액 - 위약금
```

## 6.2 위약금 정책

> **법적 근거:** 소비자분쟁해결기준 (공정거래위원회)

| 조건 | 위약금률 |
|---|---|
| 이용 시작 전 | 0% (전액 환불) |
| 이용 시작 후 | 잔여 금액의 10% |

- **위약금 상한:** 잔여 이용 금액의 10%를 초과할 수 없음

### 위약금 면제 사유

| 사유 | 필요 서류 |
|---|---|
| 센터 귀책 (폐쇄/이전/축소) | 없음 |
| 계약일 7일 내 청약철회 | 없음 |
| 질병/부상 (3개월+ 이용 불가) | 진단서 |
| 주소 이전 (직선거리 20km+) | 전입확인서 |
| 해외 이주 | 출국 증빙 |

- 위약금 면제: 지점장 판단 + 본사 결재 (사유 기록)

## 6.3 청약철회(쿨오프) 정책

> **법적 근거:** 소비자기본법

| 항목 | 규칙 |
|---|---|
| 철회 가능 기간 | 계약일로부터 7일 이내 |
| 이용 시작 전 | 전액 환불 (위약금 없음) |
| 이용 시작 후 (7일 내) | 결제금액 - 일일이용료×사용일수 (위약금 없음) |
| 처리 기한 | 접수 후 3영업일 이내 |
| 처리 권한 | 지점장 (청약철회는 거부 불가 — 법적 의무) |

## 6.4 환불 승인 단계

| 환불 금액 | 승인 |
|---|---|
| 50만 원 이하 | 지점장 단독 |
| 50만 원 초과 | 본사 결재 (대표/회장 승인) |
| 위약금 면제 건 | 본사 결재 필수 (금액 무관) |

## 6.5 환불 수단

| 원 결제 수단 | 환불 수단 | 소요 기간 |
|---|---|---|
| 카드 | 카드 취소 | 3~7영업일 |
| 현금 | 계좌 이체 | 3영업일 |
| 계좌이체 | 원 계좌로 이체 | 3영업일 |
| 마일리지 | 마일리지 복원 | 즉시 |
| 복합 결제 | 각 수단별 비례 환불 | 수단별 상이 |

# 제9장. 정산/급여 정책

## 9.1 급여 체계

| 유형 | 대상 | 구성 |
|---|---|---|
| 월급제 | 지점장, 스태프 | 기본급 + 직책수당 + 인센티브 - 공제 |
| 시급제 | 파트타임 스태프 | 시급 × 근무시간 + 인센티브 - 공제 |
| 건별제 | 트레이너 (프리랜서) | 수업 건당 단가 + 인센티브 - 공제 |
| 혼합제 | FC, 트레이너 (정규직) | 기본급 + 성과 수당 + 인센티브 - 공제 |

- 급여 체계 결정: 본사 (회장/대표)
- 급여 체계 변경: 지점장 → 본사 결재

## 9.2 인센티브 산정 공식

### PT 수업 인센티브 (트레이너)

```
PT 인센티브 = PT 매출 × 인센티브율

인센티브율 (매출 구간별):
- 월 300만 원 이하: 30%
- 월 300~500만 원: 35%
- 월 500~800만 원: 40%
- 월 800만 원 초과: 45%
```

### GX 수업 인센티브 (트레이너)

```
GX 인센티브 = 수업 횟수 × 회당 단가

회당 단가 (인원별):
- 5인 이하: 20,000원
- 6~10인: 25,000원
- 11~15인: 30,000원
- 16인+: 35,000원
```

### FC 인센티브

```
판매 인센티브 = 본인 판매 매출 × 수수료율

수수료율:
- 이용권 판매: 8~15% (상품 유형별)
- 재등록 유도: 건당 20,000원
- 신규 회원 유치 (상담→계약): 건당 30,000원
```

### 인센티브 관리 권한

| 행위 | 권한 |
|---|---|
| 인센티브 구간/율 설정 | 본사 (회장/대표), 지점장 → 결재 |
| 인센티브 조회 (전 직원) | 본사, 지점장 |
| 인센티브 조회 (본인) | 전 역할 |
| 인센티브 정산 주기 | 매월 (익월 급여에 합산) |

## 9.4 급여 마감/확정 프로세스

```
[자동 산출] → [지점장 검토] → [본사 검토/수정] → [대표/회장 확정] → [지급] → [명세서]
  1일(시스템)    1~3일(지점장)    3~5일(본사)       5일(대표)       10일    10일(자동)
```

| 상태 | 설명 | 수정 권한 |
|---|---|---|
| DRAFT | 시스템 자동 산출 | 본사 자유 수정 |
| PENDING | 지점장 검토 완료 → 본사 검토 대기 | 본사 수정 가능 |
| CONFIRMED | 대표/회장 확정 | 회장만 수정 (사유 필수) |
| PAID | 지급 완료 | 수정 불가 (환수/추가 지급으로 처리) |

**핵심:** 급여 확정 권한은 **본사(대표/회장)**에 있음. 지점장은 검토만.

## 9.5 퇴직금 산정

> **법적 근거:** 근로자퇴직급여보장법

```
퇴직금 = 1일 평균임금 × 30일 × (총 근속일수 ÷ 365)
1일 평균임금 = 퇴직 전 3개월간 총 급여 ÷ 해당 기간 총 일수
```

| 항목 | 규칙 |
|---|---|
| 지급 대상 | 1년 이상 근속 |
| 인센티브 포함 | 정기적 인센티브는 평균임금에 포함 |
| 지급 기한 | 퇴직일로부터 14일 이내 |
| 산출 | 시스템 자동 + 본사 검토 |
| 승인 | 대표/회장 |

## 9.6 카드 수수료 정산

| 항목 | 규칙 |
|---|---|

| 역할 | 최대 할인율 | 초과 시 |
|---|---|---|
| 스태프 | 사전 설정 정책만 (임의 불가) | N/A |

## 12.2 쿠폰/프로모션 생성 프로세스

```
[지점장: 쿠폰/프로모션 기획]
    ↓

    ↓
[본사 검토 → 대표/회장 승인]
    ↓
[승인 → 시스템에 쿠폰/프로모션 등록]
    ↓
[지점에서 승인된 쿠폰 자유 발급, 프로모션 자동 적용]
```

| 항목 | 규칙 |

# 제14장. 알림/커뮤니케이션 정책

## 14.1 만료 사전 알림

| 시점 | 채널 |
|---|---|
| 30일 전 | 앱 푸시 |
| 14일 전 | SMS + 앱 푸시 |
| 7일 전 | SMS |
| 3일 전 | SMS |
| 당일 | SMS |
| 만료 후 3일 | SMS (재등록 유도) |

- 발송 시간: 09:00~21:00, 수신 거부 회원도 만료 알림은 발송 (서비스 안내)
- 채널 우선순위: 앱 푸시(무료) → 카카오 → SMS(유료)

## 14.2 미출석 알림

| 기간 | 대상 | 행동 |
|---|---|---|
| 7일 | 회원 | 앱 푸시 |
| 14일 | 회원 + FC/트레이너 | SMS + 시스템 알림 |
| 21일 | 회원 + FC/트레이너 | FC 전화 상담 필수 |
| 30일 | 지점장 | 특별 혜택 검토 |
| 60일 | 본사 | 이탈 위험 리포트 |

## 14.3 미수금 독촉 알림

| 미납 기간 | 톤 |
|---|---|
| 3일 | 안내: "결제가 확인되지 않았습니다" |
| 7일 | 안내: 금액, 납부 방법 |
| 14일 | 독촉 + 지점장/본사 알림 |
| 30일 | 경고: 이용 정지 예정 + SUSPENDED |

- 발송: 09:00~20:00, 일 최대 1회, 야간/휴일 금지 (채권추심법)

## 14.4 생일/이벤트 알림

| 이벤트 | 시점 | 내용 |
|---|---|---|
| 생일 | 당일 09:00 | 축하 + 쿠폰 |
| 등록 1주년 | 해당일 | 기념 혜택 |
| 100회 출석 | 달성 즉시 | 마일리지 적립 |

## 14.5 내부 알림 (직원 간)

| 이벤트 | 수신 |
|---|---|
| 환불 요청 접수 | 지점장 + 본사 (50만 원+ 시) |
| 50만 원+ 결제 | 지점장 + 본사 |
| 수기 매출 등록 | 본사(대표) |
| 수업 취소 (강사) | 예약 회원 + 지점장 |
| 직원 지각/결근 | 지점장 |
| 블랙리스트 출입 시도 | 지점장 + 본사 |
| 결재 요청 도착 | 본사 (대표/회장) |
| 결재 승인/반려 | 지점장 |
| 급여 확정 | 전 직원 (본인 것) |

## 14.6 알림 비용 관리

| 항목 | 규칙 |
|---|---|
| 월간 SMS 한도 | 본사 설정 (지점별) |
| 한도 초과 | 지점장 알림 + 본사 승인 |
| 대량 발송 (100건+) | 본사 결재 |
| 비용 리포트 | 월간 채널별 발송/비용 (본사+) |

# 제15장. 지점 운영 정책

## 15.1 영업 시간

| 항목 | 규칙 |
|---|---|
| 설정 | 지점장 (소소한 조정), 주요 변경은 본사 결재 |
| 기본값 | 06:00~23:00 |
| 공휴일/휴무 | 지점장 등록 + 회원 앱/SMS 공지 |
| 영업 시간 외 | 출입 차단 + 키오스크 비활성화 |

## 15.2 지점 일시중단(PAUSED)

| 항목 | 규칙 |
|---|---|
| 결재 | 회장 승인 필수 |
| 허용 행위 | 조회, 환불, 이관 |
| 차단 행위 | 신규 등록, POS, 수업, 출입 |
| 회원 이용 기간 | 중단 일수만큼 자동 연장 |
| 직원 급여 | 본사 결정 (유급 휴업/무급 휴직) |
| 공지 | 중단 7일 전 전 회원/직원 SMS |

## 15.3 타 지점 이용

| 항목 | 규칙 |
|---|---|
| 허용 | 동일 테넌트 내 ACTIVE 지점 (본사 설정) |
| 횟수 제한 | 월 4회 |
| 추가 요금 | 없음 (기본), 본사 설정 가능 |
| 출석 기록 | `isOtherBranch=true`, 원 소속 지점 집계 |
| 락커/시설 | 타 지점 락커 불가, 일반 시설만 |

## 15.4 지점별 상품/가격 차등

| 항목 | 규칙 |
|---|---|
| 가격 차등 | 허용 (본사 승인 하에 지점별 가격 설정) |
| 동일 상품 관리 | 본사 레벨 상품 템플릿 + 지점별 가격 override (본사 결재) |
| 통합 관리 | 본사가 전 지점 가격 일괄 조정 가능 |

# 제16장. 감사 로그 정책

## 16.1 기록 대상

| 카테고리 | 기록 행위 |
|---|---|
| 인증 | 로그인, 로그아웃, 비밀번호 변경, 지점 전환 |
| 회원 | 등록, 수정, 삭제, 이관, 휴회, 정지, 양도, 블랙리스트 |
| 매출 | 결제, 환불, 수정, 미수금, 수기 등록 |
| 급여 | 등록, 수정, 확정, 지급, 퇴직 정산 |
| 직원 | 등록, 수정, 퇴사, 역할 변경, 지점 이동 |
| 설정 | 센터 설정, 권한, 가격 변경 |
| 시설 | 락커, RFID, 강제 반납 |
| 보안 | 로그인 실패, 잠금, 비정상 접근 |
| **결재** | **결재 요청, 승인, 반려, 보류 전 건** |

## 16.2 접근 권한

|---|---|

| 지점장 | 소속 지점 로그 |
| 그 외 | 접근 불가 |
