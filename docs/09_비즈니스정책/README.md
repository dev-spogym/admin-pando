## 1. 인증/보안 정책

### 1.1 비밀번호 정책

**정의**
- 로그인 시 사용하는 관리자 계정 비밀번호의 보안 규칙

**정책 요구사항**

| 항목 | 규칙 |
|------|------|
| 최소 길이 | 8자 이상 |
| 복잡도 | 영문 + 숫자 + 특수문자 조합 필수 |
| 재사용 금지 | 이전 3개 비밀번호 재사용 불가 |
| 만료 기간 | 90일 경과 시 변경 권장 팝업 |
| 강제 변경 | 90일 초과 시 로그인 시 변경 강제 or 건너뛰기 옵션 제공 |

**구현 요구사항**
- 비밀번호 변경 시 이전 비밀번호 이력 저장
- 90일 경과 회원 조회 API
- 변경 알림 팝업 UI

---

### 1.2 계정 잠금 정책

**정의**
- 반복된 잘못된 로그인 시도로부터 계정 보호

**정책 요구사항**

| 항목 | 규칙 |
|------|------|
| 실패 횟수 | 5회 연속 실패 시 계정 잠금 |
| 잠금 기간 | 30분 |
| 남은 시간 표시 | 로그인 화면에 "남은 시간" 표시 |
| 잠금 해제 방법 | 비밀번호 찾기 또는 관리자 수동 해제 |
| 실패 횟수 초기화 | 성공 로그인 시 카운트 초기화 |

**구현 요구사항**
- 로그인 시도 실패 기록 저장 (admin_login_failures 테이블)
- 현재 시각 vs 잠금 해제 시각 비교 로직
- 남은 시간 계산 및 표시 (분 단위)

---

### 1.3 세션 관리 정책

**정의**
- JWT 토큰 기반 로그인 세션 관리

**정책 요구사항**

| 항목 | 규칙 |
|------|------|
| 동시 세션 제한 | 1개만 허용 |
| 새로운 로그인 처리 | 기존 세션 강제 종료 |
| 액세스 토큰 만료 | 2시간 |
| 토큰 갱신 | refresh token 사용 (만료 시간 동안 갱신 가능) |
| 로그인 유지 | 체크 시 7일 유효 |
| 로그아웃 시 | 토큰 무효화 (블랙리스트 저장) |

**구현 요구사항**
- JWT 토큰: access_token + refresh_token 쌍 발급
- localStorage에 토큰 저장 (refresh token은 httpOnly cookie 권장)
- 토큰 갱신 엔드포인트 (/auth/refresh)
- 토큰 유효성 검증 미들웨어

| 매출/POS | CRUD | CRUD | CRU | CR | CR | R |
| 상품관리 | CRUD | CRUD | R | - | - | R |
| 시설관리 | CRUD | CRUD | CRU | R | R | R |
| 직원관리 | CRUD | CRUD | R | - | - | - |
| 급여관리 | CRUD | CRUD | - | - | - | - |
| 메시지 | CRUD | CRUD | CRU | R | - | - |
| 설정 | CRUD | CRU | R | - | - | - |

- C: Create (생성)

## 2. 회원 관리 정책

### 2.1 회원 상태 정의

**상태 목록**

| 상태 | 조건 | 설명 |
|------|------|------|
| 활성 | 유효한 이용권 1개 이상 보유 | 현재 이용 가능한 상태 |
| 만료 | 모든 이용권 만료 | 마지막 만료일 기준 |
| 예정 | 시작일이 미래인 이용권 보유 | 아직 시작되지 않은 이용권 |
| 임박 | 만료일 D-30 이내 | 설정값 조정 가능 |
| 홀딩 | 이용권 정지 중 | 회원의 요청 or 센터 지정 |
| 미등록 | 회원 등록만, 이용권 없음 | 프로필만 생성된 상태 |

**상태 전이 규칙**

```
미등록 → 활성 (이용권 구매)
활성 → 임박 (D-30 도달)
활성 → 만료 (D-0 도달)
활성 → 홀딩 (회원 요청)
홀딩 → 활성 (해제)
만료 → 활성 (재구매)
```

**구현 요구사항**
- 회원 상태 자동 판정 로직 (매일 배치 실행)
- 대시보드에 상태별 카운트 표시
- 회원 목록에서 상태 필터링 가능

---

### 2.2 회원 등록 정책

**필수 입력 항목**

| 항목 | 형식 | 제약 |
|------|------|------|
| 이름 | 한글/영문 | 2~20자 |
| 성별 | 남/여 | 필수 선택 |
| 연락처 | 010-XXXX-XXXX | 중복 불가 (동일 지점 내) |
| 회원구분 | 일반/법인/가족 | 필수 선택 |

**선택 입력 항목**

| 항목 | 형식 | 제약 |
|------|------|------|
| 이메일 | user@domain.com | 형식 검증 |
| 생년월일 | YYYY-MM-DD | 미래 날짜 불가 |
| 주소 | 도로명/지번 | Kakao 주소 API |
| 프로필 사진 | JPG/PNG | 5MB 이하 |
| 운동목적 | 자유 입력 | 선택 |
| 메모 | 자유 입력 | 선택 |

**자동 생성 항목**

| 항목 | 규칙 |
|------|------|
| 회원번호 | 지점코드-YYMMDD-순번 (예: SJ-260311-001) |
| 출석번호 | 4자리 랜덤, 중복 체크 필수 |
| first_reg_date | 등록 시점 자동 설정 |
| 기본 상태 | "미등록" (이용권 구매 시 "활성"으로 변경) |

**유효성 검증**

| 항목 | 검증 로직 |
|------|---------|
| 연락처 중복 | 동일 지점 내에서만 체크 |
| 이메일 형식 | 정규표현식: `^[^\s@]+@[^\s@]+\.[^\s@]+$` |
| 생년월일 | 현재보다 미래 불가 |
| 프로필 사진 | MIME type 확인 (image/jpeg, image/png) |

**담당 FC 배정**
- 등록 시 선택 가능 (선택사항)
- 수정 시 변경 가능
- 담당 FC 퇴사 시 재배정 필요

**구현 요구사항**
- React Hook Form + Zod 스키마 검증
- 2단계 폼 (필수정보 → 추가정보)
- 중복 확인 버튼 (실시간 API 호출)

### 2.3 회원 수정 정책

**수정 가능 항목**
- 모든 선택 입력 항목
- 담당 FC 변경 가능
- 메모 수정

**수정 불가 항목**
- 회원번호 (시스템 생성)
- 출석번호 (시스템 생성)
- first_reg_date (생성 시점)

**변경 이력**

| 항목 | 내용 |
|------|------|
| 기록 대상 | 모든 필드 변경 |
| 저장 정보 | 변경 전/후 값, 변경자, 변경일시 |
| 보존 기간 | 무제한 |
| 조회 권한 | 센터장 이상만 조회 가능 |

**연락처 변경 시**
- 수정 전 중복 체크 (동일 지점 내)
- 이미 등록된 회원이면 경고 후 중단

**구현 요구사항**
- React Hook Form으로 기존 데이터 프리필
- 변경 필드 강조 표시 (색상 or 배경)
- 변경 이력 조회 탭 추가

### 2.4 회원 삭제 정책

**삭제 가능 조건**

| 조건 | 상태 |
|------|------|
| 미납 건 | 없음 |
| 유효 이용권 | 없음 |
| 미사용 마일리지 | 환불 필요 |

**삭제 처리**
- 소프트 삭제 (is_deleted 플래그)
- 회원 정보 암호화 (개인정보 보호)
- 출석/결제 이력은 유지 (감사 목적)

**복구 정책**
- 삭제 후 30일 이내 복구 가능
- 복구 시 모든 정보 복원

**구현 요구사항**
- 삭제 전 조건 검증 (미납, 유효권, 마일리지)
- 삭제 확인 다이얼로그 (회원명 재입력)
- 삭제 이력 감사로그 기록

### 2.5 회원 검색/필터 정책

**검색 기준**

| 검색어 | 매칭 방식 | 최소 글자 |
|-------|---------|---------|
| 이름 | 부분일치 | 2글자 이상 |
| 연락처 | 숫자만 추출 후 부분일치 | 1글자 이상 |
| 회원번호 | 완전일치 | - |

**검색 성능**
- debounce: 300ms (키 입력 후 300ms 경과 후 검색)
- 결과 최대 1,000건 제한
- 삭제 회원 제외 (옵션으로 포함 가능)

**필터 조합 규칙**
- 모든 필터는 AND 조합
- 상태 필터 + 상세 필터 동시 적용 가능
- 필터 초기화 버튼 필수

**적용 가능한 필터**

| 필터 | 옵션 | AND 결합 |
|------|------|--------|
| 상태 | 활성/만료/예정/임박/홀딩/미등록 | 1개만 선택 |
| 회원구분 | 일반/법인/가족 | 복수 선택 |
| 계약상품 | 시설이용권/PT/GX 등 | 복수 선택 |
| 최종만료일 | YYYY-MM-DD 범위 | AND |
| 최근방문일 | YYYY-MM-DD 범위 | AND |
| 성별 | 남/여 | AND |

**구현 요구사항**
- 필터 상태를 URL 파라미터로 유지
- 필터 초기화 시 모든 값 제거
- 활성 필터 카운트 표시

### 2.6 회원 일괄 처리 정책

**동시 선택 제한**
- 최대 100명 동시 선택

**일괄 처리 기능**

| 기능 | 대상 | 제약 |
|------|------|------|
| 상태 변경 | 선택 회원 | 활성↔비활성, 홀딩 처리 |
| 메시지 발송 | 선택 회원 | SMS/카카오톡 선택 |
| 출석 처리 | 선택 회원 | 수동 출석 일괄 등록 |
| 다운로드 | 선택 회원 | Excel 내보내기 |

**상태 변경 규칙**
- 활성 → 비활성: 즉시 반영, 알림 없음
- 비활성 → 활성: 이용권 유효성 재확인 필수
- 홀딩 처리: 사유 필수 입력

**메시지 발송**
- 발송 비용 사전 계산 및 표시
- 발송 내역 저장 (발송일시, 수신자, 내용)

**출석 등록**
- 일괄 선택 회원 대상 특정 날짜 출석 등록
- 기존 출석 기록 중복 여부 확인

**구현 요구사항**
- 체크박스로 다중 선택
- 선택된 행 강조 표시
- 일괄 처리 버튼 활성화 (1명 이상 선택 시)
- 처리 결과 토스트 알림

## 3. 매출/결제 정책

### 3.1 매출 집계 정책

**매출 계산 규칙**

| 항목 | 계산식 |
|------|-------|
| 총매출 | 현금 + 카드 + 마일리지 - 환불 |
| 순매출 | 총매출 - 부가세 |
| 부가세 | 총매출 × 10% (또는 제외 설정) |

**집계 단위**
- 일별 매출
- 주별 매출 (월요일 기준)
- 월별 매출
- 연별 매출

**데이터 격리**
- 지점별 독립 집계 (branch_id 기준)
- 권한별 조회 범위 제한

**매출 상태**

| 상태 | 조건 |
|------|------|
| 완료 | 결제 완료 및 확인됨 |
| 환불 | 환불 처리됨 |
| 미납 | 예약 결제 미완료 |
| 미완료 | 결제 대기 중 |

**구현 요구사항**
- 차트: 막대/원형 그래프 (Recharts)
- 필터: 기간 프리셋 (오늘/이번주/이번달/커스텀)
- 다운로드: Excel 포맷 (.xlsx)

---

### 3.2 환불 정책

**환불 기간별 규칙**

| 기간 | 환불 비율 | 조건 |
|------|---------|------|
| 결제 후 7일 이내 | 100% (전액) | 이용 미시작 |
| 7일 초과 ~ 이용 시작 전 | 90% (위약금 10%) | 환불 신청 시점 기준 |
| 이용 시작 후 | 일할 계산 | 잔여일수 기준 환불 |
| 이용 시작 후 30일 이상 경과 | 0% (환불 불가) | 정책 결정 |

**환불 조건**
- 결제 수단 확인 (원래 결제 수단으로 환불)
- 환불 사유 기록 (선택사항)
- 환불 요청일시 기록

**환불 처리 프로세스**

```
환불 요청 → 승인 대기 → 승인 (or 거절) → 환불 실행 → 완료 알림
```

**환불 불가 사유**
- 이용권 모두 소진된 경우
- 환불 기한 초과
- 분쟁 중인 거래

**구현 요구사항**
- 환불 계산 자동화 (기간별 비율 적용)
- 환불 요청 폼 및 승인 프로세스
- 환불 이력 조회 가능

### 3.4 결제 수단별 정책

#### 현금 결제

| 항목 | 규칙 |
|------|------|
| 거스름돈 | 자동 계산 |
| 영수증 | 현금영수증 발행 (선택) |
| 기록 | 입금 액수 + 거스름돈 기록 |

**현금영수증 발행**
- 휴대폰 번호 or 사업자등록번호 필수
- 발행 후 취소 불가능
- 환불 시 별도 취소 처리 필요

#### 카드 결제

| 항목 | 규칙 |
|------|------|
| 단말기 | PG사 연동 or 수기 입력 |
| 승인 | 즉시 or 정기 설정 |
| 환불 | 원래 카드로 취소 |
| 수수료 | PG사 정책에 따름 |

**카드 환불**
- 당일 취소: 승인 취소 (수수료 없음)
- 익일 이후: 환불 프로세스 (수수료 발생 가능)

#### 마일리지 결제

| 항목 | 규칙 |
|------|------|
| 최소 사용 | 1,000원 이상 |
| 최대 사용 | 결제금액의 50% (설정 가능) |
| 초과 사용 | 불가 (잔액 초과 시 거절) |
| 복원 | 환불 시 마일리지로 복원 |
| 현금 전환 | 불가 |

**마일리지 사용 검증**
- 잔액 확인 (실시간)
- 잔액 초과 차감 불가
- 사용 후 잔액 즉시 차감

#### 혼합결제 (현금 + 카드 + 마일리지)

| 항목 | 규칙 |
|------|------|
| 총액 검증 | 각 수단 합계 = 결제금액 (필수) |
| 순서 | 제약 없음 |
| 영수증 | 통합 영수증 1장 (각 수단별 명시) |

**혼합결제 검증**

```
현금(3,000) + 카드(7,000) + 마일리지(5,000) = 15,000 (일치)
```

- 불일치 시 결제 불가 (에러 메시지)
- 수정 후 재계산

**구현 요구사항**
- 결제 수단별 필드 입력
- 각 수단별 소계 자동 계산
- 총액 일치 검증
- 혼합결제 총액 표시

**구현 요구사항**
- 상품명 중복 체크 (동일 카테고리 내)

분실 신고 → 기존 카드 비활성화 → 신규 카드 발급 → 새 카드 활성화

- 회원 탈퇴/이동 시 카드 반납

| 미할당 | 미등록 상태 |

**출입 연동**

```
RFID 태그 → 회원 조회 → 이용권 유효 확인 → 출입 허용/거부
```

**출입 거부 사유**
- 미등록 카드: "등록되지 않은 카드입니다"
- 만료 회원: 센터 설정에 따라 허용/차단
- 미납 회원: 센터 설정에 따라 허용/경고/차단

- RFID 리더 시뮬레이션 or 실제 연동

### 6.3 타지점 출석 정책

**전제 조건**
- 전지점 이용권 보유 시만 타지점 출석 허용
- 단일 지점 이용권: 해당 지점만 출석 가능

**타지점 출석 절차**

```
회원이 A 지점에 등록 → B 지점 키오스크 체크인 → 출석 기록 B 지점에 저장
```

**알림**
- 타지점 출석 시 원 지점(A)에 실시간 알림
- 알림 내용: "회원명이 B지점에서 출석했습니다"

**출석 기록**
- 실제 방문 지점에만 기록 (B 지점)
- 원 지점(A)에는 참고용 기록 제공

**구현 요구사항**
- 이용권 유효성 검증 (전지점 여부)
- 타지점 출석 시 알림 발송
- 출석 조회에 지점 필터 추가

#### 예약/취소/노쇼 정책

**PT 예약 취소**

|------|------|

| 5회 이상 | 계약 재검토 권고 |

**취소 불가 상황**
- 이미 취소된 예약
- 완료된 수업

**구현 요구사항**
- 예약 취소 시간 계산 (현재 시각 vs 수업 시작)
- 노쇼 자동 감지 배치
- 페널티 누적 리포트

### 7.3 퇴사 처리 정책

**퇴사 처리 절차**

```
퇴사 의사 → 퇴사 처리 실행 → 계정 비활성화 → 담당 회원 재배정 → 급여 정산
```

**퇴사 처리 항목**

| 항목 | 처리 |
|------|------|
| 관리자 계정 | 비활성화 (로그인 불가) |
| 담당 회원 | 재배정 필요 (알림 발송) |
| 미지급 급여 | 정산 (최종 급여 산정) |
| 데이터 보관 | 설정에 따라 6개월 or 영구 보관 |

**담당 회원 재배정**
- 퇴사 직원이 담당하던 회원 목록 추출
- 센터장에게 알림 (재배정 필요)
- 재배정 기한: 3일 이내 권고

**급여 정산**
- 최종 근무일 기준 급여 계산
- 미사용 휴일 수당 (기업 정책에 따름)
- 4대보험 정산 관련 서류 준비

**데이터 보관 정책**
- 기본값: 6개월 보관
- 설정 변경 가능: 설정 → 기본 정보
- 보관 기간 종료 후 자동 삭제 (또는 수동)

**퇴사 후 접근**
- 해당 직원 계정으로 로그인 불가
- 다른 직원이 기존 데이터 조회 가능 (권한에 따라)

**구현 요구사항**
- 퇴사 처리 버튼 (센터장 이상)
- 퇴사 처리 확인 모달
- 담당 회원 목록 조회
- 급여 정산 페이지

### 7.4 급여 계산 정책

**실지급액 공식**

```
실지급액 = 기본급 + 인센티브 + 수당 - 공제
```

**각 항목 정의**

| 항목 | 설명 | 예시 |
|------|------|------|
| 기본급 | 고용 계약상 월급 | 2,500,000원 |
| 인센티브 | PT 건당 커미션 | 30% × PT 매출 |
| 수당 | 식사/교통/기타 | 설정값 |
| 공제 | 4대보험 + 소득세 | 자동 계산 |

**4대보험 공제**

| 보험 | 조건 | 비율 |
|------|------|------|
| 국민연금 | 정규/계약직 | 기본급의 9% |
| 건강보험 | 정규/계약직 | 기본급의 6.5% |
| 고용보험 | 정규/계약직 | 기본급의 0.8% |
| 산재보험 | 정규/계약직 | 기본급의 0.5% |

**소득세 계산**
- 월 과세 표준 기준
- 기본 공제 적용
- 누진세율 적용

**인센티브 계산 예시**

```
PT 매출 (월): 2,000,000원
인센티브율: 30%
인센티브 = 2,000,000 × 30% = 600,000원

실지급액 = 2,500,000 + 600,000 - 공제액 = 최종액
```

**급여 마감일**
- 매월 25일 (설정 가능)
- 마감 후 편집 불가 (관리자 해제 가능)

**구현 요구사항**
- 기본급 + 인센티브 입력 필드
- 공제액 자동 계산
- 실지급액 자동 표시
- 급여 마감 버튼 (센터장 이상)

### 7.5 급여 상태 흐름

**상태 전이**

```
미정산 → 임시저장 → 확정대기 → 확정완료
         ↑           ↓
         └─ 이의제기
```

**각 상태 설명**

| 상태 | 설명 | 권한 | 비고 |
|------|------|------|------|
| 미정산 | 급여 데이터 미입력 | 관리자 | 초기 상태 |
| 임시저장 | 급여 입력, 미확정 | 관리자 | 수정 가능 |
| 확정대기 | 직원 서명 대기 | 직원 | 명세서 조회 가능 |
| 확정완료 | 급여 확정 | 관리자 | 수정 불가 |

**확정완료 후 수정**
- 기본적으로 불가능
- 센터장 이상이 "미정산"으로 롤백 가능
- 롤백 시 사유 필수 기록

**이의제기**
- 직원이 급여에 이의 있을 경우
- 관리자에게 메시지 발송
- 관리자가 처리 후 다시 확정

**구현 요구사항**
- 상태별 버튼 활성화 제어
- 상태 전이 로그 기록
- 이의제기 메시지 UI

## 8. 메시지/쿠폰/마일리지 정책

### 8.1 메시지 발송 정책

**메시지 채널별 규칙**

| 채널 | 단가 | 글자수 제한 | 비용 표시 |
|------|------|-----------|---------|
| SMS | 70원/건 | 90자 | 사전 계산 |
| LMS(장문) | 30원/건 | 2,000자 | 사전 계산 |
| 카카오 알림톡 | 8원/건 | 1,000자 | 사전 계산 |

**발송 비용 계산**

```
총 비용 = 발송 수신자 수 × 채널당 단가 × 발송 횟수
```

**예약 발송**
- 최소: 현재 시각 + 5분
- 최대: 현재 시각 + 30일
- 예약 발송 시간 수정 가능 (발송 전)
- 예약 발송 취소 가능

**발송 대상 선택**

| 대상 | 선택 방법 |
|------|---------|
| 개별 | 회원 검색 후 선택 |
| 그룹 | 필터(상태/회원구분) 선택 |
| 전체 | 체크박스 1개 |

**변수 삽입**

| 변수 | 치환값 |
|------|--------|
| #{이름} | 회원명 |
| #{연락처} | 회원 연락처 |
| #{만료일} | 이용권 만료일 |
| #{잔여일} | 이용권 잔여일수 |

**발송 이력**

| 항목 | 기록 |
|------|------|
| 발송일시 | YYYY-MM-DD HH:MM:SS |
| 수신자 | 회원명/연락처 |
| 채널 | SMS/LMS/카카오톡 |
| 내용 | 발송된 메시지 (변수 치환 후) |
| 상태 | 발송 완료/실패 |

**발송 실패 처리**
- 실패 건 목록 표시
- 재발송 버튼 제공
- 실패 사유 기록 (연락처 오류 등)

**구현 요구사항**
- 메시지 작성 폼 + 변수 삽입 버튼
- 발송 비용 자동 계산
- 예약 발송 달력 선택
- 발송 이력 조회 (DataTable)

### 8.3 쿠폰 정책

**쿠폰 유형**

| 유형 | 기준 | 예시 |
|------|------|------|
| 정률 | 할인율 (%) | 10% 할인 |
| 정액 | 할인액 (원) | 5,000원 할인 |
| 무료 이용권 | 무료 제공 | 이용권 1회 무료 |

**유효기간 설정**

| 방식 | 규칙 |
|------|------|
| 특정 기간 | "2026-03-01 ~ 2026-03-31" |
| 발급일+N일 | 발급일 기준 30일 유효 |

**사용 한도**

| 방식 | 규칙 |
|------|------|
| 전체 한도 | 전체 N회 (예: 총 100회만 사용 가능) |
| 회원당 한도 | 회원당 1회 (1인 1회만 사용 가능) |

**추가 조건**

| 항목 | 규칙 |
|------|------|
| 최소 주문 금액 | 10,000원 이상 구매 시만 적용 (선택) |
| 중복 사용 | 1 거래당 1쿠폰만 사용 가능 |

**쿠폰 코드 생성**
- 자동 생성: 8~16자리 영숫자 (중복 체크)
- 수동 입력: 사용자가 직접 설정

**쿠폰 상태**

| 상태 | 조건 |
|------|------|
| 활성 | 유효기간 내 + 한도 남음 |
| 비활성 | 유효기간 만료 |
| 마감 | 전체 한도 소진 |
| 삭제 불가 | 발급 건이 1건 이상 있으면 삭제 불가 |

**자동 비활성화**
- 유효기간 만료 시 자동 비활성화 (배치)
- 한도 소진 시 자동 마감 (실시간)

**쿠폰 발급**

| 항목 | 규칙 |
|------|------|
| 대상 선택 | 필터(상태/회원구분) 선택 or 개별 |
| 발급 방식 | 즉시 or 예약 발송 |
| 발급 이력 | 발급자, 발급일시, 수신자 기록 |

**구현 요구사항**
- 쿠폰 CRUD 모달
- 코드 자동/수동 생성
- 유효기간 + 한도 설정
- 발급 모달 (대상 선택)
- 사용 이력 조회
- 자동 비활성화 배치

## 9. 계약/구독 정책

### 9.1 계약 프로세스 정책

**5단계 마법사 (Stepper)**

```
Step 1: 회원 선택 (필수)
  ↓
Step 2: 상품 선택 (1개 이상 필수)
  ↓
Step 3: 기간/금액 설정 (시작일, 할인, 서비스일수)
  ↓
Step 4: 결제 처리 (현금/카드/마일리지/혼합)
  ↓
Step 5: 전자서명 (약관 동의 + 서명)
```

**Step 1: 회원 선택**

| 옵션 | 설명 |
|------|------|
| 기존 회원 | 회원 검색 후 선택 |
| 신규 회원 | 회원 정보 입력 + 등록 |

**Step 2: 상품 선택**
- 카테고리별 상품 검색 + 선택
- 1개 이상 필수
- 선택된 상품 확인 가능

**Step 3: 기간/금액 설정**

| 항목 | 설명 |
|------|------|
| 시작일 | 구매일 or 지정일 |
| 할인 | 설정 가능 |
| 서비스일수 | 자동 계산 |
| 최종가 | 원가 - 할인 |

**Step 4: 결제 처리**
- 결제 수단 선택 (현금/카드/마일리지/혼합)
- 예약 결제 옵션 (구독의 경우)
- 결제 확인 모달

**Step 5: 전자서명**
- 약관 동의 체크박스
- 서명 패드 (WebSignaturePad)
- 계약서 미리보기

### 9.2 할인 정책

**할인 유형**

| 유형 | 최대값 | 사용 사례 |
|------|--------|---------|
| 재등록 할인 | 30% | 만료 후 재구매 |
| 신규 할인 | 20% | 신규 회원 |
| 이벤트 할인 | 50% | 프로모션 |
| 관리자 할인 | 50% | 관리자 재량 |

**할인 적용 규칙**

| 항목 | 규칙 |
|------|------|
| 최대 할인 | 50% 초과 불가 |
| 할인 사유 | 필수 입력 |
| 할인가 표시 | 원가 / 할인가 / 최종가 모두 표시 |
| 할인 이력 | 할인 유형 + 사유 기록 |

**할인 계산 예시**

**구현 요구사항**
- 할인율/금액 선택 입력
- 할인 계산 자동화

**자동 생성 항목**

| 항목 | 타이밍 | 설명 |
|------|--------|------|

- 기본: "회원님 가입을 환영합니다. [센터명]"

- 저장 경로: `/contracts/{회원ID}/{년월}/contract.pdf`
- 이메일 발송 (회원 이메일)

**구현 요구사항**
- 자동 처리 설정 폼 (설정 페이지)

- PDF 생성 로직
- 이메일 발송 큐

### 9.4 구독 플랜 정책

**구독 플랜 3가지**

| 플랜 | 회원 제한 | 지점 제한 | 가격 (월) | 연간 할인 |
|------|---------|---------|---------|---------|
| Starter | 200명 | 1개 | 50,000원 | 20% |
| Pro | 1,000명 | 3개 | 150,000원 | 20% |
| Enterprise | 무제한 | 무제한 | 커스텀 | 20% |

**결제 방식**

| 방식 | 가격 | 할인 |
|------|------|------|
| 월간 | 정가 | 없음 |
| 연간 | 정가 × 12 | 20% |

**다운그레이드 정책**
- 조건: 현재 사용량이 하위 플랜 제한 이하
- 예: 현재 회원 수 150명 → Starter로 다운그레이드 가능
- 예: 현재 회원 수 250명 → Pro로만 다운그레이드 가능

**다운그레이드 불가**
- 월간 구독 중에는 다운그레이드 불가
- 연간 구독: 갱신 시점에만 변경 가능
- 차액 환불 없음 (정책)

**구독 해지**
- 언제든 해지 가능
- 해지 다음 월부터 적용
- 현재 월 비용 환불 불가

**용량 초과 시**
- 지점/회원 수 제한 초과 → 업그레이드 권고
- 즉시 업그레이드 옵션 제공
- 차액만 납부 (월할 계산)

**구현 요구사항**
- 플랜 비교 표 UI
- 현재 사용량 표시
- 업그레이드/다운그레이드 버튼
- 해지 확인 모달
- 사용량 모니터링 (대시보드)

### 10.6 회원 지점 이동

**이동 조건**

| 항목 | 규칙 |
|------|------|
| 권한 | 최고관리자만 |
| 대상 | 활성 회원 |
| 방식 | 개별 or 일괄 |

**이동 옵션**

| 옵션 | 설명 |
|------|------|
| 이용권 유지 | 기존 이용권 그대로 유지 (타지점 이용권 전환) |
| 신규 발급 | 기존 이용권 환불 후 신규 발급 |

**이동 프로세스**

```
회원 선택 → 이동할 지점 선택 → 이용권 처리 방식 선택 → 이동 확인 → 실행
```

**이용권 유지 시**
- 기존 이용권 유지
- 이용 가능 지점: 원래 지점 + 신규 지점 (전지점 이용권으로 변환)

**신규 발급 시**
- 기존 이용권 상태 확인
  - 활성: 잔여일수 환불 (일할 계산)
  - 만료: 환불 없음
- 신규 지점에서 새 이용권 발급

**이용권 환불 금액**

```
환불액 = 결제액 × (잔여일수 / 전체일수)
```

**이동 이력**

| 항목 | 기록 |
|------|------|
| 이동일시 | YYYY-MM-DD HH:MM:SS |
| 원래 지점 | 지점명 |
| 이동 지점 | 지점명 |
| 처리 방식 | 이용권 유지/신규 발급 |
| 처리자 | 최고관리자 |

**구현 요구사항**
- 회원 검색 (일괄)
- 이동할 지점 선택 (드롭다운)
- 이용권 처리 방식 라디오 버튼
- 이동 이력 조회
