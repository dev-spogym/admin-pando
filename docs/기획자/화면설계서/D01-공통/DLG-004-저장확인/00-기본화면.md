## 화면 목적
등록/수정 내용을 저장하기 전에 최종 확인을 요청하는 공통 다이얼로그입니다.
중요한 데이터 변경 시 실수를 방지합니다.

## 주요 구성요소
- "저장하시겠습니까?" 확인 메시지
- 저장 대상 요약 정보 (선택적)
- 저장 확인 버튼
- 취소 버튼

## 주요 UX 흐름
1. 저장 버튼 클릭 → 이 다이얼로그 표시
2. 저장 확인 → 저장 처리 → 완료 토스트 메시지
3. 취소 → 다이얼로그 닫고 편집 계속

## 화면 상태
| 상태 | 설명 |
|---|---|
| 표시 | 저장 확인 메시지 |
| 저장 중 | 처리 중 (버튼 비활성) |
| 저장 완료 | 완료 토스트 후 화면 유지 또는 상세로 이동 |
| 저장 실패 | 오류 안내 (충돌, 필수값 누락 등) |
| 충돌 감지 | 다른 사람이 동시에 수정한 경우 안내 |
