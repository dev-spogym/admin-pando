## 화면 목적
지점을 비활성화하기 전 영향 범위를 안내하고 최종 확인을 받는 다이얼로그입니다.
비활성화 시 해당 지점의 모든 직원 계정 접근이 차단되므로 신중한 확인이 필요합니다.

## 주요 구성요소
- "지점을 비활성화하시겠습니까?" 제목
- 비활성화 대상 지점명 표시
- 비활성화 시 발생하는 영향 안내:
  - 해당 지점 직원 계정 접근 차단
  - 회원 앱에서 해당 지점 표시 중단
  - 예약 중인 수업 처리 방법 안내
- 비활성화 예정일 선택 (즉시 / 날짜 지정)
- 취소 버튼
- 비활성화 확인 버튼

## 주요 UX 흐름
1. 지점 관리 화면에서 비활성화 버튼 클릭 → 다이얼로그 표시
2. 영향 범위 확인 및 비활성화 예정일 선택
3. 취소 → 다이얼로그 닫힘, 지점 상태 유지
4. 비활성화 확인 → 지점 상태를 "임시휴업"으로 변경 → 관련 계정 접근 차단 처리

## 화면 상태
| 상태 | 설명 |
|---|---|
| 표시 | 영향 범위 안내 및 예정일 선택 |
| 처리 중 | 확인 버튼 클릭 후 처리 중 (버튼 비활성) |
| 완료 | 다이얼로그 닫히고 지점 상태 배지 변경 |
