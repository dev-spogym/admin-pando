## 화면 목적
역할을 삭제하기 전 최종 확인을 받고, 해당 역할에 배정된 직원이 있을 경우 처리 방법을 안내하는 다이얼로그입니다.

## 주요 구성요소
- "역할을 삭제하시겠습니까?" 제목
- 삭제 대상 역할명 표시
- 해당 역할 배정 직원 수 안내 (예: "이 역할을 사용 중인 직원이 3명 있습니다.")
- 배정 직원이 있을 경우: "삭제 전 해당 직원의 역할을 변경해주세요" 안내 및 역할 재배정 이동 버튼
- 배정 직원이 없을 경우: 삭제 확인 버튼 활성
- 취소 버튼

## 주요 UX 흐름
1. 역할 목록에서 삭제 버튼 클릭 → 다이얼로그 표시
2. 배정 직원 없는 경우 → 삭제 확인 버튼 클릭 → 역할 삭제 완료, 목록에서 제거
3. 배정 직원 있는 경우 → 역할 재배정 이동 버튼 클릭 → DLG-087-001 역할 재배정 다이얼로그로 이동
4. 취소 → 다이얼로그 닫힘

## 화면 상태
| 상태 | 설명 |
|---|---|
| 직원 없음 | 삭제 확인 버튼 활성 상태 |
| 직원 있음 | 삭제 확인 버튼 비활성, 재배정 안내 표시 |
| 삭제 처리 중 | 확인 버튼 클릭 후 처리 중 |
| 삭제 완료 | 다이얼로그 닫히고 역할 목록 갱신 |
