| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-001 |
| 화면명 | 로그인 |
| 경로 | `/login` |
| 플랫폼 | Web |
| 설명 | 이메일/비밀번호 기반 로그인 화면. 지점 선택 드롭다운 포함 |
| 상태 | 확정 |

## 2. 네비게이션

### 진입 경로
- URL 직접 접근, 세션 만료 시 리다이렉트

### 이동 가능 화면
- SCR-002 (대시보드)

## 3. UI 요소

| 요소 ID | 요소명 | 유형 | 설명 | 인터랙션 | 유효성검사 | 에러메시지 |
|--------|--------|------|------|---------|---------|-----------|
| UI-001 | 이메일 입력 | Input | 로그인 이메일 주소 입력 필드 | 포커스 시 라벨 상단 이동, 실시간 이메일 형식 검증 | 이메일 형식 검증 (regex) | 올바른 이메일 주소를 입력하세요 |
| UI-002 | 비밀번호 입력 | Input | 로그인 비밀번호 입력 필드 | 마스킹 토글 버튼, Enter키 로그인 트리거 | 최소 1자 이상 | 비밀번호를 입력하세요 |
| UI-003 | 지점 선택 | Select | 로그인 시 접근할 지점 선택 드롭다운 | 선택 시 지점 로고/색상 변경 | 필수 선택 | 지점을 선택하세요 |
| UI-004 | 로그인 버튼 | Button | 로그인 실행 버튼 | 클릭/Enter 시 로그인 API 호출, 로딩 스피너 | 이메일/비밀번호/지점 모두 입력 시 활성화 | 로그인에 실패했습니다 |
| UI-005 | 비밀번호 찾기 링크 | Link | 비밀번호 재설정 페이지 이동 링크 | 클릭 시 비밀번호 찾기 모달 오픈 |  |  |

## 4. 인터랙션 정의

비밀번호 마스킹 토글, Enter키 로그인, 지점 선택 시 로고 변경

## 5. 에러 및 상태 처리

### 에러 메시지

- 이메일 미입력: '이메일을 입력하세요'
- 비밀번호 오류: '비밀번호가 올바르지 않습니다'
- 계정 잠금: '5회 실패로 잠금됨'

## 6. 접근성 체크리스트

- Tab 이동 순서, aria-label

## 7. 반응형 체크

### Web
- 1024px 이상 중앙 정렬

### Web 특이사항
- JWT 토큰 기반 세션 관리
