## 목적

메시지를 실제로 발송하기 전에 수신자 화면에서 어떻게 보이는지 미리 확인하는 다이얼로그입니다. 변수(회원명, 만료일 등)가 실제 값으로 치환된 최종 메시지 형태를 검토해 오발송을 방지합니다.

## 구성요소

- **채널별 미리보기 탭**: 선택한 채널(알림톡·SMS·앱 푸시)에 따라 해당 UI 형태로 메시지를 미리 표시합니다.
- **발송 대상 요약**: 수신자 수와 선택된 채널을 표시합니다.
- **메시지 본문 미리보기**: 변수가 샘플 값으로 치환된 실제 발송 내용을 보여줍니다.
- **예상 비용**: 수신자 수와 채널 기준으로 계산된 예상 발송 비용을 표시합니다.
- **발송 버튼**: 미리보기를 확인하고 즉시 발송합니다.
- **수정 버튼**: 미리보기를 닫고 메시지 작성 화면으로 돌아갑니다.

## 언제 표시되나요

- 메시지 발송 화면(SCR-071)에서 "미리보기" 버튼을 클릭할 때 표시됩니다.

## 주요 흐름

1. 담당자가 메시지 내용과 수신자 설정을 완료한 뒤 "미리보기"를 클릭합니다.
2. 이 다이얼로그가 열리며 실제 수신 화면 형태로 메시지가 표시됩니다.
3. 내용이 올바르면 "발송"을 클릭해 즉시 전송합니다.
4. 수정이 필요하면 "수정"을 클릭해 작성 화면으로 돌아갑니다.
