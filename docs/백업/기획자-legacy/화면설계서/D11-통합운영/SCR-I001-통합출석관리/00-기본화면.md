## 화면 목적

회원과 직원의 출석 현황을 하나의 화면에서 실시간으로 통합 조회·관리한다. 키오스크 QR, 얼굴인식, 앱 QR, 수동 입력 등 다양한 출석 채널의 데이터를 통합 집계하여 운영 효율을 높이고, 출석 후 옷 락커 배정 상태까지 함께 확인할 수 있다.

## 주요 구성요소

### 상단 통계 카드

| 항목 | 설명 |
|------|------|
| 오늘 총 출석 | 회원 + 직원 합산 입장 수 |
| 현재 센터 내 | 입장 후 퇴실 미처리 인원 수 |
| 미배정 락커 | 출석은 완료됐으나 옷 락커 미배정 인원 수 |
| 실패 건수 | 금일 출석 실패(만료·중복·미등록 등) 건수 |

### 필터 영역

| 필터 | 선택 옵션 |
|------|-----------|
| 기간 | 오늘(기본값), 이번 주, 이번 달, 직접 선택 |
| 구분 | 전체, 회원, 직원, 게스트 |
| 결과 | 전체, 성공, 실패, 중복, 퇴실 |
| 채널 | 앱 QR, 키오스크 QR, 얼굴인식, 수동 |
| 락커 상태 | 전체, 미배정, 수동배정, 자동배정, 대상 아님 |
| 검색 | 이름, 연락처, 회원번호, 사번 |

### 출석 테이블

| 컬럼명 | 설명 |
|--------|------|
| 시간 | 입장 시각 |
| 구분 | 회원 / 직원 / 게스트 |
| 이름 | 이름 + 회원번호 또는 사번 |
| 채널 | 앱 QR / 키오스크 QR / 얼굴인식 / 수동 |
| 결과 | 성공 / 실패 / 중복 / 퇴실 |
| 옷 락커 | 미배정 / 수동배정(번호) / 자동배정(번호) / 대상 아님 |
| 고정 락커 | 보유 시 락커 번호, 없으면 '-' |
| 비고 | 실패 사유 또는 운영 메모 |

### 헤더 버튼

| 버튼 | 설명 |
|------|------|
| 수동 출석 등록 | DLG-I001 팝업 호출 |
| 실패 로그 | 실패 건만 필터링하여 표시 |
| 키오스크 모니터 | 최근 키오스크 이벤트 사이드 패널 열기 |

### 행 단위 액션

| 액션 | 설명 |
|------|------|
| 퇴실 처리 | 입장 상태의 회원·직원을 퇴실로 전환 |
| 락커 배정 | 미배정 상태에서 옷 락커 배정 화면 또는 인라인 연동 |
| 회원 상세 | 해당 회원의 상세 화면으로 이동 |

## 계정별 차이

| 계정 유형 | 차이 |
|-----------|------|
| 센터 관리자 | 전체 기능 사용 가능, 수동 출석 등록 및 퇴실 처리 권한 |
| 프런트 직원 | 출석 조회 및 락커 배정 가능, 설정 변경 불가 |
| 강사 | 조회만 가능, 수동 등록·퇴실 처리 불가 |
| 본사 관리자 | 지점 선택 후 전 지점 현황 조회 가능 |

## 주요 UX 흐름

1. 화면 진입 시 기본값: 오늘 날짜, 전체 구분, 전체 결과
2. 키오스크·앱에서 출석 이벤트 발생 → 테이블 상단에 즉시 반영
3. 미배정 락커 카드 클릭 → 미배정 인원만 필터링
4. 출석 행 클릭 → 해당 회원 상세 화면으로 이동
5. 행 내 락커 배정 버튼 클릭 → DLG-I002(옷 락커 배정) 팝업 호출
6. 수동 출석 등록 버튼 → DLG-I001 팝업 호출 → 등록 완료 시 목록 자동 갱신

## 화면 상태

| 상태 | 설명 |
|------|------|
| 기본(데이터 있음) | 출석 목록 정상 표시 |
| 빈 상태 | 해당 기간 출석 없음 안내 |
| 실시간 업데이트 중 | 새 출석 이벤트 진입 시 행 강조 표시 |
| 필터 결과 없음 | 조건에 맞는 데이터 없음 안내 |
| 로딩 | 데이터 조회 중 스켈레톤 표시 |

## 관련 다이어그램

- DLG-I001: 수동 출석 등록 팝업
- DLG-I002: 옷 락커 배정 팝업
- SCR-I002: 키오스크 설정 (출석 후 처리 정책 연동)
- SCR-I004: 옷 락커 운영 관리 (락커 배정 상태 연동)
