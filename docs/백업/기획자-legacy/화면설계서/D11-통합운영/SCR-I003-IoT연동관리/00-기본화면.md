## 화면 목적

지점에 설치된 IoT 장비(출입 게이트, 키오스크, 락커 컨트롤러, InBody 측정기)의 연결 상태를 모니터링하고 관리한다. 기기 등록·삭제, 연결 테스트, 출입 로그 조회, 원격 게이트 제어를 지원한다.

## 주요 구성요소

### 탭 구조

| 탭 | 설명 |
|----|------|
| 게이트 관리 | 출입문 목록, 실시간 연결 상태, 원격 개방 |
| IoT 기기 | 지점 전체 기기 목록 및 상태 배지 |
| 출입 로그 | 출입 성공·실패·예외 이벤트 이력 조회 |
| 출입 규칙 | 시간대·요일·게이트별 출입 허용 규칙 설정 |
| 자동 전원 | 키오스크·기기 전원 ON/OFF 스케줄 설정 |

### IoT 기기 탭 주요 항목

| 항목 | 설명 |
|------|------|
| 기기명 | 장비 식별 이름 |
| 유형 | 게이트 / 키오스크 / 락커 컨트롤러 / InBody |
| IP/포트 | 네트워크 연결 정보 |
| 상태 | 온라인 / 오프라인 / 오류 / 점검 중 |
| 마지막 통신 | 최근 heartbeat 시각 |
| 펌웨어 버전 | 현재 설치된 펌웨어 버전 |

### InBody 채널 상태 배지

| 채널 | 설명 |
|------|------|
| 수기 | 직원이 수동 입력한 데이터 채널 |
| 파일 업로드 | 장비 파일을 시스템에 업로드한 채널 |
| 실시간 API | InBody 장비에서 직접 전송하는 실시간 채널 |

### 게이트 관리 탭 주요 항목

| 항목 | 설명 |
|------|------|
| 게이트명 | 출입문 식별 이름 |
| 연결 상태 | 온라인 / 오프라인 표시 |
| 원격 개방 | 예외 상황 시 관리자가 원격으로 문 개방 |
| 마지막 이벤트 | 최근 출입 이벤트 시각 |

### 버튼

| 버튼 | 설명 |
|------|------|
| 기기 추가 | 신규 장비 등록 |
| 연결 테스트 | 선택 기기 연결 상태 재확인 |
| 원격 게이트 개방 | 특정 게이트 강제 개방 (감사 로그 자동 생성) |
| 삭제 | 기기 비활성 또는 목록에서 제거 |
| 전체 동기화 | 모든 기기 상태 일괄 새로고침 |

## 계정별 차이

| 계정 유형 | 차이 |
|-----------|------|
| 센터 관리자 | 기기 등록·삭제·원격 제어 전체 가능 |
| 프런트 직원 | 출입 로그 조회 가능, 기기 등록·삭제·원격 제어 불가 |
| 본사 관리자 | 전 지점 기기 현황 조회 가능, 개별 제어는 해당 지점 관리자가 수행 |

## 주요 UX 흐름

1. 화면 진입 → 게이트 관리 탭 기본 표시
2. 기기 추가 → 유형·이름·IP·포트 입력 → 저장
3. 연결 테스트 버튼 → 선택 기기 ping 후 상태 배지 갱신
4. 원격 게이트 개방 → 확인 팝업 → 실행 → 감사 로그 자동 생성
5. 출입 로그 탭: 기간·게이트·결과 필터로 이벤트 조회
6. 출입 규칙 탭: 요일·시간 범위 설정 후 저장

## 화면 상태

| 상태 | 설명 |
|------|------|
| 기본(기기 있음) | 기기 목록 및 상태 배지 표시 |
| 기기 없음 | '등록된 기기가 없습니다.' 안내 및 기기 추가 유도 |
| 오프라인 기기 있음 | 해당 기기 행 빨간 상태 배지 강조 |
| 오류 기기 있음 | 오류 내용 툴팁 표시 |
| 동기화 중 | 전체 동기화 버튼 로딩 스피너 표시 |
