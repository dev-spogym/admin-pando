## 화면 목적

락커의 고장 상태를 활성화하거나 해제하기 전 최종 의사를 확인하는 다이얼로그입니다. 현재 상태에 따라 "고장 처리"와 "고장 해제" 두 가지 방향 모두 이 다이얼로그를 통해 처리됩니다.

## 주요 구성요소

- **다이얼로그 제목**: 현재 상태에 따라 "고장 처리" 또는 "고장 해제"
- **안내 메시지**:
  - 고장 처리: "[락커 번호]번 락커를 고장 처리하시겠습니까?"
  - 고장 해제: "[락커 번호]번 락커의 고장 상태를 해제하시겠습니까?"
- **확인 버튼**: 상태를 변경합니다.
- **취소 버튼**: 변경 없이 닫습니다.

## 주요 UX 흐름

1. 락커 셀 호버 시 [고장] 버튼 클릭 → 현재 상태에 따라 고장 처리 또는 해제 다이얼로그 열림.
2. 안내 메시지 확인 후 [확인] 클릭 → 상태 변경 → 알림 표시 후 그리드 갱신.

## 화면 상태

| 상태 | 설명 |
|---|---|
| 고장 처리 성공 | "고장 처리되었습니다" 알림 |
| 고장 해제 성공 | "고장 해제되었습니다" 알림 |
| 변경 실패 | "고장 상태 변경에 실패했습니다" 오류 알림 |
