## 화면 목적
특정 날짜의 회원 출석 현황을 관리자가 한눈에 파악하고, 수동 체크인·체크아웃 처리를 수행하는 화면입니다.
이용 중인 회원, 이용 완료 회원, 미출석 회원을 구분하여 현장 운영을 지원합니다.

## 주요 구성요소
- **날짜 선택기**: 조회할 날짜를 선택 (기본값: 오늘)
- **통계 카드**: 총 출석 수 / 이용 중 / 이용 완료를 요약 표시
- **출석 목록 테이블**: 회원별 체크인 시각, 체크아웃 시각, 이용 시간, 상태 배지 표시
- **수동 체크인 버튼**: 미출석 회원에 대해 수동으로 체크인 처리
- **수동 체크아웃 버튼**: 이용 중인 회원에 대해 수동으로 체크아웃 처리

## 출석 행 상태 구분
| 상태 | 설명 | 표시 방식 |
|---|---|---|
| 이용 중 | 체크인만 완료, 체크아웃 미완료 | 파란색 배지 "이용 중" |
| 이용 완료 | 체크인 + 체크아웃 모두 완료 | 초록색 배지 "완료" |
| 미출석 | 예약은 있으나 체크인 없음 | 회색 배지 "미출석", 흐린 표시 |

## 계정별 차이
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| 슈퍼관리자 / 지점관리자 | 해당 지점 전체 출석 현황 | 수동 체크인·체크아웃, 날짜 변경 |
| FC / 스태프 | 해당 지점 전체 출석 현황 | 수동 체크인·체크아웃, 날짜 변경 |
| 트레이너 | 자신의 수업 참여 회원 출석 현황 | 조회만 가능 |

## 주요 UX 흐름
1. 화면 진입 시 오늘 날짜 기준으로 출석 목록이 로드됩니다.
2. 날짜 선택기를 변경하면 해당 날짜의 출석 목록으로 전환됩니다.
3. 이용 중인 회원 행에서 "체크아웃" 버튼 클릭 → 체크아웃 시각이 기록되고 상태가 "완료"로 변경됩니다.
4. 미출석 회원 행에서 "수동 체크인" 버튼 클릭 → 현재 시각으로 체크인이 기록됩니다.
5. 처리 결과는 화면 하단 알림으로 즉시 안내됩니다.

## 비즈니스 규칙
- 이용 시간은 체크아웃 시각 - 체크인 시각(분 단위)으로 자동 계산됩니다.
- 수동 체크인·체크아웃은 FC 이상 권한 계정만 가능합니다.
- 출석 목록은 실시간으로 업데이트되어 다른 관리자의 처리 내용도 즉시 반영됩니다.
