## 1. 회원 목록

### 화면 목적

센터에 등록된 전체 회원을 한눈에 조회하고, 필터·검색·정렬을 통해 원하는 회원을 빠르게 찾는다.
목록에서 바로 출석 처리, 메시지 발송, 상태 변경, 관심회원 지정 등의 일괄 작업을 수행한다.

### 주요 기능

- **요약 통계 카드**: 전체 회원 수 / 활성 회원 수 / 만료 예정(D-30) 회원 수 / 이번 달 만료 회원 수를 상단에 표시한다.
- **메인 탭 전환**
  - 회원 전체: 상태별 서브탭(전체 / 활성 / 만료 / 미등록 / 홀딩 / 정지)으로 분류 조회
  - 보유상품별: 이용권 종류 기준으로 그룹화하여 조회
  - 이용권 목록: 이용권 단위로 조회 (시작일, 만료일, D-Day 포함)
- **검색**: 회원명 또는 연락처로 실시간 검색
- **필터**: 계약상품, 성별, 최종만료일 범위, 최근방문일 범위, 미방문 기간, 회원구분, 유입경로, 관심회원 여부, 만료 숨기기
- **일괄 작업**: 여러 회원 선택 후 상태 변경 / 메시지 발송 / 수동 출석 처리 / 관심회원 등록
- **엑셀 다운로드**: 전체 회원 데이터를 엑셀 파일로 내려받는다.
- **세그먼트 표시**: 각 회원의 현재 상황을 신규 / 만료 후 미등록 / 이탈 위험 / 만료 임박 / 관심 필요 / 충성 회원 / 활발한 회원 중 하나로 자동 분류하여 목록에 표시한다.
- **관심회원 즐겨찾기**: 목록에서 별표를 클릭해 바로 관심회원으로 지정하거나 해제한다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 (primary) | 전체 회원 목록, 전체 통계 | 모든 기능 (엑셀 다운로드, 회원 추가, 일괄 작업 포함) |
| 센터장 (owner) | 전체 회원 목록, 전체 통계 | 모든 기능 |
| 매니저 (manager) | 전체 회원 목록, 전체 통계 | 모든 기능 |
| 피트니스코치 (fc) | 전체 회원 목록, 전체 통계 | 목록 조회 및 일괄 작업 (엑셀 다운로드, 회원 추가 불가) |
| 스태프 (staff) | 전체 회원 목록 | 회원 추가, 일괄 작업 (엑셀 다운로드 불가) |
| 조회전용 (readonly) | 접근 불가 | 접근 불가 |

### 주요 UX 흐름

1. 사이드바 > 회원관리 > 회원 목록 진입
2. 상단 통계 카드로 현재 현황 파악
3. 상태 탭 또는 필터를 선택하여 대상 회원 범위 설정
4. 검색어 입력으로 특정 회원 찾기
5. 회원명 클릭 시 해당 회원의 상세 페이지로 이동
6. 체크박스로 여러 회원 선택 후 일괄 작업 수행

### 비즈니스 규칙

- 관심회원 필터와 다른 필터는 동시에 적용할 수 있다.
- 만료 숨기기를 체크하면 상태가 만료인 회원은 목록에서 제외된다.
- 일괄 출석 처리는 수동 처리 방식으로 기록된다.
- 메시지 발송은 선택된 회원 ID를 전달하여 메시지 화면으로 이동한다.
- 엑셀 다운로드는 현재 필터·페이지와 무관하게 전체 회원 데이터를 내려받는 것을 원칙으로 한다.

## 2. 회원 등록/수정

### 화면 목적

새로운 회원을 시스템에 등록하거나, 기존 회원의 기본 정보를 수정한다.
필수 정보와 선택 정보를 2단계(Step)로 나누어 입력 부담을 줄인다.

### 주요 기능

- **2단계 입력 구조**
  - Step 1 (필수 정보): 이름, 성별, 연락처, 회원구분, 생년월일, 키, 담당 FC, 방문경로, 유입경로, 운동목적, 프로필 이미지
  - Step 2 (추가 정보): 별칭, 이메일, 주소, 출석번호, 광고성 수신 동의, 메모
- **전화번호 중복 확인**: 등록 전 동일 연락처가 이미 등록되어 있는지 확인한다. 수정 모드에서는 자기 자신은 중복으로 판단하지 않는다.
- **프로필 이미지 업로드**: 사진을 등록하거나 변경할 수 있다. 이미지 파일만 허용하며, 5MB 이하여야 한다.
- **주소 검색**: 우편번호 검색 기능을 통해 주소를 입력할 수 있다.
- **법인 회원 구분**: 회원구분이 기명법인 또는 무기명법인인 경우 법인 회사명을 추가로 입력한다.
- **초기화**: 입력 중인 내용을 전부 지우고 처음 상태로 되돌린다.
- **등록 완료 후 바로 결제 이동**: 등록 성공 시 결제 화면으로 바로 이동할 수 있는 버튼이 단기간 제공된다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 / 매니저 | 등록·수정 화면 전체 | 회원 등록 및 수정 |
| 피트니스코치 (fc) | 접근 불가 | 접근 불가 |
| 스태프 (staff) | 등록·수정 화면 전체 | 회원 등록 및 수정 |
| 조회전용 | 접근 불가 | 접근 불가 |

### 주요 UX 흐름

**신규 등록:**
1. 회원 목록 상단 우측 [회원 추가] 버튼 클릭
2. Step 1: 이름, 성별, 연락처 입력 → [중복확인] 클릭
3. 나머지 Step 1 항목 입력 후 [다음] 클릭
4. Step 2: 추가 정보 입력 후 [저장] 클릭
5. 완료 메시지 확인 → 회원 상세 또는 결제 화면으로 이동

**수정:**
1. 회원 상세 프로필 영역의 [수정] 버튼 클릭
2. 기존 정보가 자동으로 채워진 상태에서 원하는 항목 변경
3. [저장] 클릭 → 회원 상세로 복귀

### 비즈니스 규칙

- 연락처는 중복 확인을 완료해야 다음 단계로 진행할 수 있다. (신규 등록 한정)
- 생년월일에 미래 날짜는 입력할 수 없다.
- 이름은 한글·영문·한자 2~20자 이내로 제한한다.
- 메모는 500자를 초과할 수 없다.
- 출석번호를 입력하지 않으면 시스템이 자동 생성한다.

## 3. 회원 상세

### 화면 목적

특정 회원의 모든 정보를 하나의 화면에서 탭으로 구분하여 조회하고 관리한다.
회원 상태 관리(홀딩, 탈퇴 등), 출석 처리, 메시지 발송 등 주요 액션을 프로필 헤더에서 바로 수행할 수 있다.

### 주요 기능

- **프로필 헤더 (고정)**: 이름, 등급, 상태, 만료 D-Day, 연락처, 이메일, 마일리지, 미수금, 계약 건수, 락커 정보를 요약 표시
- **만료 알림 배너**: 이용권 만료 30일 이내 시 주황, 7일 이내 시 빨간 배너를 상단에 표시
- **헤더 버튼**: 수동출석 / 수정 / 상품구매 / 메시지 / 지점이관 / 탈퇴
- **15개 탭**: 아래 탭 목록 참조
- **홀딩 처리**: 현재 활성 상태인 회원에 대해 홀딩 기간과 사유를 입력하고 홀딩을 시작한다. 홀딩 해제도 같은 위치에서 수행한다.
- **탈퇴 처리**: 탈퇴 사유를 입력하고 회원을 탈퇴 처리한다. 처리 후 목록으로 이동한다.
- **회원 삭제**: 최고관리자·센터장에 한해 회원 데이터를 완전히 삭제할 수 있다. 삭제 전 회원명을 직접 입력하는 2차 확인 절차가 있다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 | 모든 탭 | 모든 액션 (수정, 탈퇴, 삭제, 지점이관 포함) |
| 매니저 | 모든 탭 | 수동출석, 상품구매, 메시지, 탈퇴 |
| 피트니스코치 (fc) | 모든 탭 | 수동출석, 상품구매, 메시지 |
| 스태프 (staff) | 모든 탭 | 수동출석, 수정, 상품구매, 메시지 |
| 조회전용 | 조회만 가능 | 없음 |

### 15개 탭 기능 설명

| 탭 | 내용 |
|---|---|
| 회원정보 | 기본 인적사항(연락처, 생년월일, 이메일, 키), 가입일, 이용권 기간, 운영 정보(마일리지, 담당자, 방문경로, 운동목적, 광고 수신 동의), 특이사항, 메모를 조회한다. 수정은 헤더의 [수정] 버튼으로 수행한다. |
| 이용권 | 보유 중인 이용권(계약) 목록을 카드 형태로 표시한다. 상품명, 계약일, 이용 상태를 확인할 수 있다. [신규 이용권 / 상품 구매] 버튼으로 구매 화면으로 이동한다. |
| 출석이력 | 최근 6개월 출석 현황을 히트맵 캘린더로 시각화하고, 최근 20회의 체크인·체크아웃 시각을 목록으로 표시한다. 방문 패턴 파악에 활용한다. |
| 결제이력 | 결제 건을 간략하게 목록으로 조회한다. 건별로 결제 상세 내용을 팝업으로 확인하거나 환불 처리를 수행할 수 있다. |
| 결제내역 | 결제이력보다 상세한 뷰를 제공한다. 누적 결제건수, 누적 결제금액, 미수금, 환불금액 요약 카드와 함께 정가·판매금액·할인금액·미수금 컬럼이 포함된 전체 결제 테이블을 조회한다. |
| 예약내역 | 월별 미니 캘린더와 테이블로 수업 예약 현황을 조회한다. SHOW(출석), NOSHOW(노쇼), 미처리 상태를 구분하여 표시한다. |
| 상세내역 | 홀딩 / 연장 / 양도 / 쿠폰 / 마일리지 이용 내역을 서브탭으로 구분하여 조회한다. 홀딩과 연장은 신규 등록도 가능하다. |
| 체성분 | 체중, 골격근량, 체지방률 측정 기록을 라인 차트와 테이블로 표시한다. [측정 추가]로 새 측정값을 입력하고 변화 추이를 추적한다. |
| 상담메모 | 카테고리(상담 / 특이사항 / 일반)별로 자유 메모를 작성하고 관리한다. 메모는 수정 및 삭제가 가능하다. |
| 레슨 | PT·GX 레슨 기록을 관리한다. 수업명, 날짜, 트레이너, 운동 내용을 기록하고, 고객 서명도 받을 수 있다. |
| 신체정보 | 키, 몸무게, 혈압(수축기/이완기), 심박수를 날짜별로 기록한다. 몸무게 변화 추이를 라인 차트로 시각화한다. |
| 종합평가 | 트레이너가 체력 / 자세 / 유연성 / 근력 / 목표달성의 5개 카테고리에 대해 1~10점으로 평가한다. 카테고리별 평균 점수 카드와 평가 이력 테이블을 제공한다. |
| 상담이력 | 상담(초기상담, OT, 체험, 재등록상담) 일정과 결과를 기록한다. 상담 결과(등록/미등록/보류), 후속조치, 연결 매출을 함께 관리하며 등록 전환율을 자동 계산한다. |
| 운동프로그램 | 회원에게 운동 프로그램을 배정하거나 해제한다. 배정된 프로그램의 카테고리, 난이도, 배정일, 배정자를 카드로 표시한다. |
| 운동이력 | 일별 운동 수행 기록(종목, 세트, 횟수, 중량, 시간, 거리)을 날짜 그룹으로 조회한다. [운동 추가]로 새 기록을 입력한다. |

### 주요 UX 흐름

1. 회원 목록에서 회원명 클릭 → 상세 화면 진입
2. 프로필 헤더에서 회원 현황 요약 파악
3. 원하는 탭 클릭으로 상세 정보 확인
4. 헤더 버튼으로 출석 처리, 수정, 구매, 메시지 등 액션 수행
5. 홀딩이 필요한 경우 하단 [홀딩 처리] 버튼 클릭

### 비즈니스 규칙

- 이용권 만료 D-30부터 배너로 담당자에게 안내한다.
- 홀딩 처리 시 만료일이 홀딩 일수만큼 자동 연장된다.
- 탈퇴 처리는 회원 상태를 WITHDRAWN으로 변경하며, 데이터는 보존된다.
- 회원 삭제는 실제 데이터를 지우는 것이 아닌 논리적 삭제(soft delete) 방식이다. 삭제된 회원은 목록에서 보이지 않는다.
- 레슨 탭의 데이터는 기기 로컬에 저장되므로 다른 기기에서는 조회할 수 없다. (향후 서버 저장으로 전환 예정)

## 4. 회원 이관

### 화면 목적

회원을 현재 지점에서 다른 지점으로 이관한다.
이관 전 미납금, 락커, 홀딩 상태, PT 잔여 횟수 등 체크리스트를 자동으로 확인하여 이관 가능 여부를 안내한다.

### 주요 기능

- **대상 회원 정보 표시**: 이관 대상 회원의 이름, 연락처, 현재 지점, 상태, 이용권 종류, 만료일을 확인한다.
- **이관 전 체크리스트**: 미납금 / 락커 / 홀딩 상태 / PT 잔여 / 보유 쿠폰 / 마일리지를 자동 점검하고 결과를 시각적으로 표시한다.
  - 차단(block): 미납금, 락커 → 해결 전 이관 불가
  - 주의(warn): 홀딩 진행 중, PT 잔여 → 확인 후 진행 가능
  - 안내(info): 쿠폰 보유 → 이관 시 소멸 가능성 안내
- **이관 설정**: 이관할 지점 선택, 이관 유형 선택(잔여 이용권 유지 / 환불 후 신규 계약 / 소진 후 이관), 사유 입력
- **이관 확인 모달**: 최종 확인 화면에서 이관 대상과 유형을 재확인한 후 이관을 실행한다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 | 이관 화면 전체 | 이관 실행 |
| 매니저 / 피트니스코치 / 스태프 / 조회전용 | 접근 불가 | 접근 불가 |

### 주요 UX 흐름

1. 회원 상세 헤더 > [지점이관] 버튼 클릭
2. 대상 회원 정보 및 체크리스트 확인
3. 차단 항목이 있으면 해결 후 재진행 / 주의 항목은 확인 후 진행 가능
4. 이관할 지점과 이관 유형 선택, 사유 입력
5. [이관 실행] 클릭 → 확인 모달에서 최종 확인
6. 이관 성공 시 회원 목록으로 이동

### 비즈니스 규칙

- 미납금이 있거나 락커를 보유한 경우 이관이 차단된다. 반드시 해결 후 이관해야 한다.
- 홀딩 중이거나 PT 잔여 횟수가 있어도 이관 자체는 가능하지만, 담당자가 확인 후 진행해야 한다.
- 쿠폰은 지점 한정인 경우 이관 시 소멸될 수 있음을 사전에 안내한다.
- 마일리지는 테넌트 전체 공유이므로 이관 후에도 그대로 유지된다.
- 이관 실행은 되돌릴 수 없다.

---

## 5. 체성분 관리

### 화면 목적

특정 회원의 체성분(체중, 골격근량, 체지방률) 변화를 시계열 차트와 테이블로 추적한다.
InBody 측정 결과를 정기적으로 입력하여 회원의 목표 달성 여부를 모니터링한다.

### 주요 기능

- **요약 통계 카드**: 최신 측정값 기준 체중, 골격근량, 체지방률, BMI를 표시한다.
- **변화 추이 차트**: 체중 / 골격근량 / 체지방률 3개 지표를 라인 차트로 시각화한다. 지표별 표시/숨김 토글이 가능하다. 차트 위에 마우스를 올리면 해당 날짜의 측정값을 툴팁으로 확인한다.
- **측정 기록 테이블**: 날짜별 측정값(체중, 골격근량, 체지방률, 체지방량, BMI, 기초대사량)을 테이블로 조회한다.
- **측정 추가**: 측정일, 체중, 골격근량, 체지방률을 입력하면 BMI와 체지방량은 자동 계산된다.
- **추이 분석 탭**: (탭 전환) 측정 기록 외 추가 분석 뷰를 제공한다.

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 / 매니저 / 피트니스코치 | 전체 화면 | 측정 추가 |
| 스태프 / 조회전용 | 접근 불가 | 접근 불가 |

### 주요 UX 흐름

1. 사이드바 > 회원관리 > 체성분 관리 진입
2. 대상 회원 선택 (또는 회원 상세 체성분 탭에서 접근)
3. 요약 카드와 차트로 현재 상태 파악
4. [측정 추가] 클릭 → 측정일과 수치 입력 → 저장
5. 차트와 테이블에 새 데이터 반영 확인

### 비즈니스 규칙

- 체중은 20~300kg, 골격근량은 5~80kg, 체지방률은 3~60% 범위만 입력할 수 있다.
- BMI는 체중과 키를 기반으로 자동 계산한다. (키는 회원정보에서 가져옴)
- 기초대사량(BMR)은 체중, 키, 나이를 기반으로 자동 계산한다.
- 측정 데이터가 2건 미만이면 차트는 표시되지 않는다.

## 6. 회원 병합

### 화면 목적

동일인이 실수로 중복 등록된 경우, 두 회원 계정을 하나로 병합하여 데이터를 통합한다.

### 주요 기능

- 병합 대상 회원 검색 및 선택
- 기준 회원(유지할 계정)과 병합 대상(삭제할 계정) 지정
- 이용권, 출석 이력, 결제 이력 등 주요 데이터 이전 여부 선택
- 병합 전 두 계정의 정보 비교 화면 제공
- 병합 실행 전 최종 확인 절차

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 | 전체 | 병합 실행 |
| 매니저 이하 | 접근 불가 | 접근 불가 |

### 주요 UX 흐름

1. 병합할 두 회원 계정을 검색하여 선택
2. 기준 계정과 병합 대상 계정 지정
3. 양쪽 정보를 비교 화면에서 확인
4. 이전할 데이터 항목 선택
5. [병합 실행] → 확인 후 처리

### 비즈니스 규칙

- 병합 후 기준 계정은 유지되고, 병합 대상 계정은 비활성 처리된다.
- 병합된 계정의 이용권, 출석 이력, 결제 이력은 기준 계정으로 통합된다.
- 병합은 되돌릴 수 없으므로 반드시 확인 절차를 거친다.

## 7. 가족 회원

### 화면 목적

가족 관계에 있는 회원들을 하나의 그룹으로 연결하여 통합 관리한다.
가족 할인, 이용권 공유, 통합 결제 등 관련 혜택 적용 시 활용한다.

### 주요 기능

- 회원 상세에서 가족 그룹 생성 및 구성원 추가·제거
- 가족 구성원 목록 조회 (이름, 관계, 상태, 이용권)
- 가족 그룹 단위 통계 조회 (총 결제금액, 방문 횟수 등)
- 가족 구성원 개별 상세 페이지로 이동

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 / 매니저 | 가족 그룹 정보 | 그룹 생성·편집 |
| 피트니스코치 / 스태프 | 가족 그룹 정보 | 조회만 |
| 조회전용 | 조회만 | 없음 |

### 주요 UX 흐름

1. 회원 상세 > 가족 회원 섹션에서 [가족 추가] 클릭
2. 연결할 회원 검색 및 관계 입력 (예: 배우자, 자녀 등)
3. 가족 그룹 생성 확인
4. 목록에서 구성원 개별 페이지로 이동하거나 제거

### 비즈니스 규칙

- 한 회원은 하나의 가족 그룹에만 속할 수 있다.
- 가족 그룹 내 구성원 수는 최대 10명으로 제한한다.
- 가족 그룹은 그룹 내 모든 구성원이 탈퇴하면 자동 해산된다.

## 8. 등급 관리

### 화면 목적

회원의 가입 기간 또는 결제 금액 기반으로 등급을 자동 산정하고, 등급별 혜택을 정의하여 회원 리텐션에 활용한다.

### 주요 기능

- **등급 현황 조회**: 현재 등급별 회원 수를 한눈에 확인
- **등급 기준 설정**: 가입 기간 또는 누적 결제금액 기준으로 등급 구간 설정
- **등급별 혜택 정의**: 할인율, 무료 이용권, 우선 예약 등 혜택 항목 설정
- **등급 자동 산정**: 기준 충족 시 등급이 자동으로 갱신됨
- **등급 수동 조정**: 관리자가 특정 회원의 등급을 수동으로 변경 가능

### 현재 적용 중인 등급 (가입 기간 기준)

| 등급 | 기준 |
|------|------|
| 브론즈 | 가입 후 30일 이내 |
| 실버 | 가입 후 31~90일 |
| 골드 | 가입 후 91~180일 |
| 플래티넘 | 가입 후 181~365일 |
| 다이아몬드 | 가입 후 366일 이상 |

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 | 등급 현황 및 설정 | 기준 변경, 수동 조정 |
| 매니저 | 등급 현황 | 수동 조정 |
| 피트니스코치 / 스태프 | 개별 회원 등급 확인 | 조회만 |

### 주요 UX 흐름

1. 설정 > 등급 관리 진입
2. 현재 등급 기준 및 회원 분포 확인
3. 기준 수정이 필요한 경우 [기준 편집] 클릭 후 저장
4. 특정 회원 등급 수동 변경: 회원 상세 > 등급 항목에서 수정

### 비즈니스 규칙

- 등급은 회원 상세 프로필 헤더에 항상 표시된다.
- 등급 기준 변경 시 전체 회원에 대해 재산정이 즉시 적용된다.
- 수동 조정된 등급은 다음 자동 산정 주기에 덮어씌워질 수 있으므로 주의가 필요하다.

## 9. 세그먼트 관리

### 화면 목적

회원을 행동 패턴과 이용 상태에 따라 자동으로 세그먼트(그룹)로 분류하여, 맞춤형 리텐션 전략을 수립하고 타깃 메시지를 발송한다.

### 주요 기능

- **자동 세그먼트 분류**: 각 회원의 현재 상태를 기준으로 아래 세그먼트 중 하나에 자동 배정
- **세그먼트별 회원 조회**: 특정 세그먼트에 속한 회원 목록 확인
- **세그먼트 기반 메시지 발송**: 세그먼트를 선택하고 일괄 메시지 발송
- **세그먼트별 통계**: 각 세그먼트의 회원 수 변화 추이 조회

### 현재 적용 중인 자동 세그먼트

| 세그먼트 | 분류 기준 |
|----------|----------|
| 신규 | 가입 후 30일 이내 회원 |
| 만료 후 미등록 | 이용권이 만료된 상태 |
| 이탈 위험 | 활성 상태이나 마지막 방문 후 30일 이상 경과 |
| 만료 임박 | 활성 상태이며 이용권 만료까지 30일 이하 |
| 관심 필요 | 활성 상태이나 마지막 방문 후 14일 이상 경과 |
| 충성 회원 | 가입 180일 이상이며 최근 7일 내 방문 |
| 활발한 회원 | 최근 14일 내 방문 |

### 계정별 차이

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| 최고관리자 / 센터장 / 매니저 | 전체 세그먼트 현황 | 세그먼트 조회, 메시지 발송, 기준 편집 |
| 피트니스코치 | 전체 세그먼트 현황 | 세그먼트 조회, 메시지 발송 |
| 스태프 / 조회전용 | 회원 목록에서 세그먼트 라벨 확인 | 조회만 |

### 주요 UX 흐름

1. 회원 목록에서 각 회원 행의 세그먼트 컬럼을 통해 현황 파악
2. 세그먼트 관리 화면에서 특정 세그먼트 선택
3. 해당 세그먼트 회원 목록 확인
4. [메시지 발송]으로 타깃 메시지 발송 화면으로 이동

### 비즈니스 규칙

- 세그먼트는 판정 우선순위에 따라 하나만 배정된다. (예: 신규 조건을 먼저 확인하고, 이후 만료·이탈 순으로 판정)
- 세그먼트는 실시간으로 계산되므로 회원의 상태나 방문 기록이 변경되면 즉시 반영된다.
- 세그먼트 기준은 최고관리자·센터장만 변경할 수 있다.
