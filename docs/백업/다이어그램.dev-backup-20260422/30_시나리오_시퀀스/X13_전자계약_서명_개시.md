# X13 — 전자계약 발송 → 서명 → 이용권 자동 개시

## 1. 시나리오 개요

매니저가 전자계약서를 회원에게 발송 → 회원이 모바일/웹에서 서명 → 서명 완료 Webhook 수신 → 이용권 자동 개시까지의 전자계약 플로우.

| 항목 | 내용 |
|------|------|
| 트리거 | 매니저의 전자계약 발송 |
| 종료 조건 | 서명 완료 + 이용권 ACTIVE 개시 |
| 참여 도메인 | 마케팅(D8), 회원관리(D2), 매출관리(D3) |

## 2. 전제조건

- 전자계약 서비스(외부 API)가 연동 설정되어 있음
- 계약 템플릿이 등록되어 있음
- 회원 이메일 또는 휴대폰 번호가 등록되어 있음
- 결제가 먼저 완료되어 있거나 계약 후 결제 정책 선택

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 매니저 | 전자계약 발송 담당 |
| 회원 | 계약서 서명 당사자 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 전자계약API | 외부 전자서명 서비스 (e.g. 모두싸인) |
| 알림서비스 | 서명 요청/완료 알림 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor M as 매니저
    actor MB as 회원
    participant API as CRM API
    participant DB as DB
    participant ESIGN as 전자계약API
    participant N as 알림서비스

    Note over M,API: [계약서 발송]
    M->>API: 1. GET /contract-templates %% E_M_GET_TEMPLATES_01
    API-->>M: 2. 계약서 템플릿 목록
    Note over M: SCR-075 전자계약 화면
    M->>API: 3. POST /contracts { member_id, template_id, product_id } %% E_M_CREATE_CONTRACT_01
    API->>DB: 4. INSERT contracts { member_id, status=DRAFT }
    API->>ESIGN: 5. POST /documents { template, signer_email, signer_phone } %% E_API_ESIGN_SEND_01
    ESIGN-->>API: 6. document_id, sign_url 반환
    API->>DB: 7. UPDATE contracts SET esign_doc_id=?, sign_url=?, status=SENT
    DB-->>API: 8. OK
    API->>N: 9. 서명 요청 알림 %% E_API_SIGN_REQUEST_01
    N-->>MB: 10. SMS/이메일 — "계약서 서명 요청 (링크 포함)"
    API-->>M: 11. 200 OK { contract_id, sign_url }

    Note over MB,ESIGN: [회원 서명]
    MB->>ESIGN: 12. 서명 링크 접속 %% E_MB_SIGN_ACCESS_01
    ESIGN->>MB: 13. 계약서 표시
    MB->>ESIGN: 14. 전자 서명 완료 %% E_MB_SIGN_COMPLETE_01
    ESIGN->>ESIGN: 15. 서명 저장 + PDF 생성

    Note over ESIGN,API: [Webhook 수신]
    ESIGN->>API: 16. POST /webhooks/contract { event=SIGNED, document_id } %% E_ESIGN_WEBHOOK_01
    API->>DB: 17. SELECT contracts WHERE esign_doc_id=?
    DB-->>API: 18. contract 정보 (member_id, product_id)
    API->>DB: 19. UPDATE contracts SET status=SIGNED, signed_at=NOW()

    alt 계약 후 이용권 자동 개시 정책
        API->>DB: 20. INSERT memberships { member_id, product_id, status=ACTIVE, start_date=TODAY } %% E_API_START_MEMBERSHIP_01
        DB-->>API: 21. membership_id
        API->>N: 22. 계약 완료 + 이용권 개시 알림 %% E_API_CONTRACT_DONE_01
        N-->>MB: 23. SMS — "계약 완료. 이용권이 활성화되었습니다"
        API-->>ESIGN: 24. 200 OK (Webhook 응답)
    else 결제 먼저 필요 정책
        API->>N: 25. 계약 완료 알림 (결제 안내 포함) %% E_API_CONTRACT_PENDING_01
        N-->>MB: 26. SMS — "계약 완료. 결제 후 이용 가능합니다"
        API-->>ESIGN: 27. 200 OK
    end

    Note over M,API: [계약서 만료 — 미서명 처리]
    Note over API: 서명 기한 초과 시 contracts.status=EXPIRED 자동 처리
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 5 | POST /documents | 외부 전자계약 API로 계약서 생성 + 서명 링크 발급 |
| 16 | Webhook 수신 | 회원 서명 완료 시 외부 서비스가 CRM으로 이벤트 전달 |
| 20 | INSERT memberships | 계약 서명을 이용권 개시 조건으로 설정한 경우 자동 개시 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 서명 기한 초과 | contracts.status=EXPIRED, 재발송 안내 |
| Webhook 미수신 | 주기적 폴링으로 상태 확인 fallback |
| 이메일/SMS 발송 실패 | 재발송 큐 등록 |
| 외부 계약 API 장애 | 내부 계약 임시 처리 또는 종이 계약 fallback |
