# X23 — 키오스크 체크인 → 출입문 개방 → 출석

## 1. 시나리오 개요

회원이 프론트 키오스크에서 본인 인증(QR/전화번호/지문) → CRM 이용권 유효성 검증 → 출입문 개방 명령 → 출석 자동 기록하는 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 회원의 키오스크 인증 시도 |
| 종료 조건 | 출입문 개방 + 출석 기록 저장 |
| 참여 도메인 | 설정관리(D9), 통합운영(D11) |

## 2. 전제조건

- 키오스크가 CRM에 연동 설정되어 있음 (SCR-082)
- 출입문 IoT 컨트롤러가 키오스크와 연결되어 있음
- 회원이 유효한 이용권을 보유하고 있음

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원 | 키오스크 인증 당사자 |
| 키오스크 | 프론트 무인 체크인 단말 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 출입제어 | IoT 출입문 컨트롤러 |
| 알림서비스 | 이상 상황 알림 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor MB as 회원
    participant KI as 키오스크
    participant API as CRM API
    participant DB as DB
    participant DOOR as 출입제어
    participant N as 알림서비스

    MB->>KI: 1. 인증 시도 (QR/전화번호/지문) %% E_MB_KIOSK_AUTH_01
    KI->>API: 2. POST /kiosk/checkin { auth_type, auth_value, kiosk_id } %% E_KI_CHECKIN_01
    API->>DB: 3. SELECT members WHERE {phone|qr_token|fingerprint}=?
    DB-->>API: 4. 회원 정보 (member_id, status)

    alt 회원 식별 성공
        API->>DB: 5. SELECT memberships WHERE member_id=? AND status=ACTIVE AND expiry_date>=TODAY %% E_API_CHECK_MEMBERSHIP_01
        DB-->>API: 6. 유효 이용권 여부

        alt 유효 이용권 보유
            API->>DB: 7. INSERT attendances { member_id, kiosk_id, type=ENTRY, timestamp=NOW() } %% E_API_INSERT_ATTEND_01
            DB-->>API: 8. attendance_id
            API->>DOOR: 9. POST /door/open { door_id, duration=5 } %% E_API_DOOR_OPEN_01
            DOOR-->>API: 10. 개방 완료
            API-->>KI: 11. 200 OK { access=GRANTED, member_name }
            KI->>MB: 12. 환영 화면 표시 "OOO님 안녕하세요"
            Note over KI: 5초 후 대기 화면 복귀
        else 이용권 만료
            API-->>KI: 13. 200 OK { access=DENIED, reason=EXPIRED } %% E_API_DENY_EXPIRED_01
            KI->>MB: 14. "이용권이 만료되었습니다. 프론트에 문의하세요"
            API->>N: 15. 만료 알림 트리거 %% E_API_EXPIRE_NOTIFY_01
            N-->>MB: 16. SMS — "이용권 만료 안내"
        else 회원 정지/탈퇴
            API-->>KI: 17. 200 OK { access=DENIED, reason=INACTIVE } %% E_API_DENY_INACTIVE_01
            KI->>MB: 18. "이용 제한 상태입니다. 프론트에 문의하세요"
        end
    else 회원 미식별
        API-->>KI: 19. 404 Not Found %% E_API_MEMBER_NOT_FOUND_01
        KI->>MB: 20. "회원 정보를 찾을 수 없습니다. 전화번호를 다시 입력해 주세요"
    end

    Note over KI,DOOR: [키오스크 오프라인 모드]
    Note over KI: 네트워크 장애 시 로컬 캐시(최근 24시간 인증 목록)로 판단
    Note over KI: 복구 후 오프라인 출석 일괄 동기화
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 2 | POST /kiosk/checkin | auth_type: QR/PHONE/FINGERPRINT. 단일 API로 인증 방식 추상화 |
| 7 | INSERT attendances | IoT 밴드(X14)와 동일 테이블. kiosk_id로 출석 방식 구분 |
| 9 | POST /door/open | duration=5(초). 개방 시간 키오스크 설정에서 관리 |
| 12 | 환영 화면 | 회원 사진 표시 가능 (개인정보 동의 시) |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| QR 코드 만료 | 새 QR 발급 안내 |
| 지문 인식 실패 3회 | 전화번호 인증으로 fallback 안내 |
| 출입문 개방 실패 | 프론트 직원 수동 개방 요청 알림 |
| 오프라인 모드 | 로컬 캐시로 판단, 복구 후 동기화 |
