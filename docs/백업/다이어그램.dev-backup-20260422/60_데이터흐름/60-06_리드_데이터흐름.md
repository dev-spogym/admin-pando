## 1. 엔티티 개요

리드(`Lead`)는 유입→상담→전환 파이프라인으로 관리. 전환 시 `Member`로 승격. S7 LeadStatus 참조.

## 2. ER 다이어그램

```mermaid
erDiagram
  Lead ||--o{ Consultation : 상담
  Lead ||--o{ LeadActivity : 활동로그
  Lead }o--o| Member : 전환
  Lead }o--|| LeadSource : 유입경로
  Lead }o--|| Staff : 담당자
  Consultation }o--|| Staff : 상담자
```

## 3. 쓰기 경로 (리드 등록→전환)

```mermaid
flowchart LR
  Form[SCR-070 리드등록] -->|E_DF06_01| API1[POST /api/leads]
  API1 --> DB1[(INSERT leads status=NEW)]
  DB1 --> Evt1[이벤트: lead.created] --> AutoMsg[자동 메시지 A11]
  DB1 --> UI1[리드 목록 갱신]

  UI1 -->|상담 예약| API2[POST /api/leads/:id/consultations]
  API2 --> DB2[(INSERT consultations)]
  DB2 --> DB3[(UPDATE leads SET status=CONSULTING)]

  DB3 -->|전환 버튼| API3[POST /api/leads/:id/convert]
  API3 --> DB4[(INSERT members)]
  DB4 --> DB5[(UPDATE leads SET status=CONVERTED, converted_member_id)]
```

## 4. 읽기 경로 (리드 파이프라인)

```mermaid
flowchart LR
  UI[SCR-070 리드관리] -->|GET /api/leads?status| DB[(SELECT leads GROUP BY status)]
  DB --> Agg[칸반 카드 배치]
  Agg --> UI
```

## 5. 주요 필드

| 필드 | 비고 |
|------|------|
| leads.source_id | 유입경로 (광고/추천/내방/온라인) |
| leads.status | 7종 S7 |
| leads.assigned_to | 담당자 staff_id |
| leads.converted_member_id | 전환 결과 회원 |
| consultations.scheduled_at | 상담 예약일 |

## 6. 인덱스/제약

- `INDEX(leads.status, leads.created_at DESC)`
- `INDEX(leads.assigned_to)` — FC별 리드
