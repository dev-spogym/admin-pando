## 1. 엔티티 개요

모든 중요 이벤트는 감사로그(`AuditLog`)에 적재되며, 사용자 대상 알림(`Notification`)은 수신자(`NotificationRecipient`)별로 전파된다. S14 NotificationStatus 참조.

## 2. ER 다이어그램

```mermaid
erDiagram
  AuditLog }o--|| Staff : 행위자
  AuditLog }o--o| Branch : 지점
  Notification ||--o{ NotificationRecipient : 수신자
  NotificationRecipient }o--|| Staff : 대상직원
  NotificationRecipient }o--o| Member : 대상회원
```

## 3. 감사로그 적재 흐름

```mermaid
flowchart LR
  AnyAPI[[모든 쓰기 API]] -->|E_DF10_01| Middleware[감사 미들웨어]
  Middleware --> DB[(INSERT audit_logs<br/>actor, action, entity, before, after, ip, ua)]
  DB --> Archive{월별 아카이빙<br/>크론 A12}
  Archive -->|30일+| Cold[(cold storage)]
```

## 4. 알림 발송 흐름

```mermaid
flowchart LR
  Evt[[도메인 이벤트]] --> Rule{알림 규칙 매칭}
  Rule --> DB1[(INSERT notifications)]
  DB1 --> Fan[수신자 팬아웃]
  Fan --> DB2[(INSERT notification_recipients)]
  DB2 --> Push{채널}
  Push -->|IN_APP| Ws[WebSocket push]
  Push -->|SMS| Provider1[[SMS 사업자]]
  Push -->|KAKAO| Provider2[[카카오 알림톡]]
  Push -->|EMAIL| Provider3[[SMTP]]
  Ws --> UI[SCR-104 알림센터]
```

## 5. 읽음 처리

```mermaid
flowchart LR
  UI[SCR-104 알림센터] -->|클릭| API[PATCH /api/notifications/:id/read]
  API --> DB[(UPDATE notification_recipients SET read_at)]
  DB --> UI
```

## 6. 주요 필드

| 필드 | 비고 |
|------|------|
| audit_logs.action | CREATE/UPDATE/DELETE/LOGIN/EXPORT |
| audit_logs.entity | members/sales/... |
| audit_logs.diff | before/after JSON |
| notifications.channel | IN_APP/SMS/KAKAO/EMAIL |
| notification_recipients.status | S14 |

## 7. 인덱스/제약

- `INDEX(audit_logs.actor_staff_id, created_at DESC)`
- `INDEX(audit_logs.entity, entity_id, created_at)`
- `INDEX(notification_recipients.staff_id, status, created_at)`
- 감사로그는 append-only (수정/삭제 불가)
