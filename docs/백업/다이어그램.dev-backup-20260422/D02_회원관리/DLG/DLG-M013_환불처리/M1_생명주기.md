## 1. 목적

DLG-M013 환불 처리 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 결제이력/결제내역 탭 > "환불" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([결제이력 환불 버튼 클릭])

  TRIGGER -->|E_OPEN_01| DLG_OPEN[DLG-M013 환불 처리 ConfirmDialog]
  DLG_OPEN --> DESC[productName amount원 결제 건 환불 처리 확인 메시지]

  DESC --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소| CLOSE[모달 닫기]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success 환불 처리가 완료되었습니다))
  TOAST_OK -->|E_REFETCH_01| REFETCH[결제이력 refetch]

  API_CALL -->|E_FAIL_01 오류| TOAST_ERR((toast.error 환불 처리 실패 error))
```
