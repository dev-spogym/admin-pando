## 1. 목적

DLG-M022 수동 출석 등록(인라인 즉시 처리)의 트리거/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 회원 상세 > 프로필 헤더 > "수동출석" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([회원 상세 프로필 헤더 수동출석 버튼 클릭])

  TRIGGER -->|E_CLICK_01 모달 없이 즉시| API_CALL[POST /api/attendances attendance INSERT type=MANUAL checkInMethod=MANUAL checkInAt=now]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success 출석이 기록되었습니다))
  TOAST_OK -->|E_REFETCH_01| REFETCH[출석 refetch]

  API_CALL -->|E_FAIL_01 오류| TOAST_ERR((toast.error 출석 처리에 실패했습니다))

  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error   fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system  fill:#EDE7F6,stroke:#5E35B2

  class TOAST_OK,REFETCH success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_CLICK_01 | 수동출석 버튼 | API 즉시 호출 | 모달 없음 |
| E_SUCCESS_01 | API | toast.success | 200 |
| E_FAIL_01 | API | toast.error | 오류 |
| E_REFETCH_01 | toast | 출석 갱신 | - |
