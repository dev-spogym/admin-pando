## 1. 목적

DLG-M020 쿠폰 적용 다이얼로그의 열기/닫기/완료 생명주기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- 상세내역 탭 > 쿠폰 서브탭 > "쿠폰 적용" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([쿠폰 적용 버튼 클릭])

  TRIGGER -->|E_OPEN_01| DLG_OPEN[DLG-M020 쿠폰 적용 🆕]
  DLG_OPEN --> INIT[쿠폰 코드 빈값 초기화]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소| CLOSE[모달 닫기]
  USER_ACT -->|E_SUBMIT_01 적용| API_VALIDATE[POST /api/coupons/validate 코드 검증]

  API_VALIDATE -->|E_VALID_01 유효| API_APPLY[POST /api/member-coupons 쿠폰 적용]
  API_VALIDATE -->|E_INVALID_01 유효하지 않음| ERR_CODE[유효하지 않은 쿠폰 코드입니다]

  API_APPLY -->|E_SUCCESS_01 200| TOAST_OK((toast.success 쿠폰이 적용되었습니다))
  TOAST_OK  -->|E_CLOSE_OK_01| POST_OK[모달 닫기 + refetch]

  API_APPLY -->|E_FAIL_01 오류| TOAST_ERR((toast.error 쿠폰 적용에 실패했습니다))
  TOAST_ERR -->|E_STAY_01| DLG_OPEN
```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_SUBMIT_01 | 적용 | 코드 검증 API | 클릭 |
| E_VALID_01 | 검증 | 쿠폰 적용 API | 유효 |
| E_INVALID_01 | 검증 | 에러 메시지 | 유효하지 않음 |
| E_SUCCESS_01 | 적용 API | toast.success | 200 |
