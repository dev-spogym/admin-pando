# FN-027~028: 계약/구독

## FN-027: 계약 등록

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-027 |
| **기능명** | 계약 등록 (위자드) |
| **분류** | 계약/구독 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-080 |
| **관련 API ID** | API-016 |

### 기능 설명
회원과 상품 간 계약을 체계적으로 등록합니다. 5단계 위자드(회원선택→상품선택→기간/금액→할인→결제)를 거쳐 계약서를 자동 생성하고 매출을 등록합니다.

### 상세 요구사항
1. Step 1: 회원 선택 (검색)
2. Step 2: 상품 선택 (이용권/PT/GX - 카테고리별)
3. Step 3: 기간/금액 설정 (상품별 기본값 노출)
4. Step 4: 할인 적용 (금액/할인율)
5. Step 5: 결제 수단 선택 및 처리
6. 계약서 자동 생성 (PDF)
7. 이용권 자동 생성 (계약 시작일 기준)
8. 매출 자동 등록
9. 영수증 발급

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 회원ID | UUID | 필수 | Step 1에서 검색 선택 |
| 상품ID | UUID | 필수 | Step 2에서 선택 |
| 계약시작일 | Date | 필수 | Step 3 (기본값: 오늘) |
| 계약기간 | String | 필수 | "1개월", "3개월" 등 |
| 기본금액 | Number | 계산됨 | 상품의 현금가/카드가 |
| 할인타입 | String | 선택 | "금액" \| "할인율" |
| 할인값 | Number | 선택 | 할인액 또는 할인율(%) |
| 최종금액 | Number | 계산됨 | 기본금액 - 할인액 |
| 결제수단 | String | 필수 | Step 5에서 선택 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 계약 정보 | Object | {id, member_id, product_id, start_date, end_date, basic_amount, discount_amount, final_amount, payment_method, contract_date} |
| 이용권 정보 | Object | {id, member_id, product_id, start_date, end_date, status: "활성"} |
| 매출 정보 | Object | {id, contract_id, amount, payment_method, transaction_id, created_at} |
| 계약서 PDF | Blob | 다운로드 가능 |
| 영수증 | Object | {receipt_number, items, total, member, timestamp} |

### 비즈니스 규칙
1. **이용권 자동 생성**:
   - 계약 시작일 기준으로 이용권 자동 생성
   - 이용권 상태: "활성"
   - 만료일 = 계약 시작일 + 기간
2. **할인 제약**:
   - 최대 50%까지만 할인 가능 (그 이상 신청 시 승인 필요 - 선택사항)
   - 할인 후 금액은 0원 이상이어야 함
3. **계약서 자동 생성**:
   - 계약일, 회원명, 상품명, 계약 기간, 금액 포함
   - PDF 형식으로 저장
4. **결제 완료 시**:
   - 이용권 활성화
   - 매출 자동 등록
   - 영수증 발급
5. **이전 단계 미완료**: 다음 단계 진행 불가

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 회원 미존재 | Step 1에서 "존재하지 않는 회원입니다" |
| 상품 선택 안 함 | Step 2에서 "상품을 선택해주세요" |
| 할인 50% 초과 | "할인율이 50%를 초과합니다. 관리자 승인이 필요합니다" (선택사항) |
| 결제 실패 | Step 5에서 "결제에 실패했습니다. 다시 시도해주세요" + 재시도 |
| 이전 단계 미완료 | "이전 단계를 완료해주세요" + 해당 단계로 돌아가기 |

### Web 구현 사항
- **위자드**: Stepper 컴포넌트 (5단계)
- **Step 1**: Combobox (회원 검색) + 현재 상태 표시 (활성/만료 등)
- **Step 2**: 카테고리 탭 + 상품 그리드 (이미지, 상품명, 가격)
- **Step 3**: 기본값 프리필 + DatePicker + Number Input
- **Step 4**: 할인 입력 (금액/율 라디오) + 최종 금액 계산
- **Step 5**: 결제 수단 선택 + 확인 모달 + 결제 처리
- **PDF 생성**: html2pdf 라이브러리

### 단계별 유효성 검증
```
Step 1: 회원 선택 완료
Step 2: 상품 선택 완료
Step 3: 기간 설정 완료, 금액 >= 0
Step 4: 최종 금액 >= 0
Step 5: 결제 수단 선택 완료
```

## FN-028: 구독 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-028 |
| **기능명** | 구독 관리 |
| **분류** | 계약/구독 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-080 |
| **관련 API ID** | API-029 (미정) |

### 기능 설명
정기 구독 상품을 관리합니다. 자동 결제를 설정하고, 구독 현황 대시보드를 제공하며, 해지 시 환불 계산을 지원합니다.

### 상세 요구사항
1. 구독 상품 CRUD
2. 구독 상품 설정 (월간/분기/연간)
3. 자동 결제 설정 (on/off)
4. 구독 현황 대시보드 (활성/해지/일시정지 구독 수)
5. 구독 해지 (즉시/월말)
6. 결제 이력 조회
7. 결제 실패 시 자동 재시도 (3회)
8. 3회 실패 시 자동 일시정지
9. 환불 계산 (해지 시)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 구독상품ID | UUID | 필수 | 기존 상품 선택 또는 신규 생성 |
| 회원ID | UUID | 필수 | 구독 회원 선택 |
| 자동결제 | Boolean | 필수 | true \| false (기본값: true) |
| 결제주기 | String | 필수 | "monthly" \| "quarterly" \| "yearly" |
| 시작일 | Date | 필수 | 구독 시작일 |
| 카드정보 | Object | 조건부 | 자동 결제 ON 시 필수 (카드 토큰) |
| 해지사유 | String | 조건부 | 구독 해지 시 필수 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 구독 목록 | Array | [{id, member_id, product_id, status, start_date, next_payment_date, payment_method}, ...] |
| 구독 통계 | Object | {active_count, paused_count, canceled_count, mrr (월간 반복 수익)} |
| 결제 이력 | Array | [{id, subscription_id, amount, status, attempted_at, next_retry}, ...] |

### 비즈니스 규칙
1. **자동 결제 실패 시**:
   - 1차 실패: 3일 후 자동 재시도
   - 2차 실패: 6일 후 자동 재시도
   - 3차 실패: 구독 자동 일시정지 + 회원 알림
2. **구독 해지**:
   - 즉시: 즉시 해지 (환불 계산)
   - 월말: 현재 기간 종료 후 해지
3. **환불 계산** (해지 시):
   - 남은 기간 비례 계산
   - 예: 30일 중 10일 사용 → 20/30 × 금액 = 환불액
4. **구독 상태**:
   - "활성": 자동 결제 진행 중
   - "일시정지": 결제 실패 3회 (수동 재개 가능)
   - "해지": 구독 종료
5. **결제일**: 매월 같은 날짜 (월간), 분기, 연간

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 자동 결제 실패 | 회원 알림 + 이메일 "결제에 실패했습니다. 재시도 예정: {{next_retry}}" |
| 3회 연속 실패 | "구독이 일시정지되었습니다. 카드 정보를 업데이트하고 재개해주세요" |
| 해지 시 환불액 음수 | "환불액이 없습니다 (이미 모두 사용됨)" |
| 구독 없음 | "구독 정보가 없습니다" |

### Web 구현 사항
- **구독 목록**: DataTable (회원, 상품, 상태, 다음 결제일)
- **구독 상세**: 모달 또는 별도 페이지 (상태, 결제 주기, 카드 정보, 다음 결제일)
- **해지 모달**: 사유 입력, 해지 방식 선택 (즉시/월말), 환불액 표시
- **결제 이력**: 테이블 (날짜, 금액, 상태, 재시도 여부)
- **대시보드**: 통계 카드 (활성/일시정지/해지 구독 수)
- **카드 정보 관리**: 안전한 토큰 기반 저장 (직접 카드번호 저장 금지)

## 계약/구독 공통 사항

### 권한 검증
| 기능 | 센터장 | PT담당자 | 직원 | 조회전용 |
|------|--------|---------|------|---------|
| FN-027 계약 등록 | O | O | O | X |
| FN-028 구독 관리 | O | X | O | X |
| FN-028 조회 | O | O | O | O |

### 데이터 격리
- 모든 계약/구독 데이터는 branch_id 기준
- 회원도 자신의 계약/구독만 조회 가능 (앱)

### 자동화 작업 (백엔드)
- **자동 결제**: 매일 자정 또는 설정된 시간
- **자동 재시도**: 실패 시 3일, 6일 후 재시도
- **자동 일시정지**: 3회 실패 시
- **이용권 만료 처리**: 만료일 자정에 상태 변경

### 환불 정책
- **전액 환불**: 계약 후 7일 이내 + 이용권 미사용
- **비례 환불**: 사용 기간 제외하고 반환
- **환불 불가**: 해지 신청 후 일정 기간 경과

### 공통 기술 스택
- **폼**: React Hook Form + Zod
- **테이블**: TanStack Table
- **모달**: Stepper (계약), Modal (구독)
- **결제**: 결제 게이트웨이 API (토큰 기반)
- **PDF**: html2pdf
- **스케줄러**: 백엔드 (Node-schedule, Bull Queue 등)
