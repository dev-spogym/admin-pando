## 상태 고유 인터랙션
1. **[+ 리드 등록] 버튼**: 로딩 중에도 클릭 가능 — 모달 오픈 시 기존 데이터 없이 빈 폼 표시 가능
2. **필터/검색 입력**: `disabled` 처리, 완료 시 자동 활성
3. **뷰 토글(목록/칸반)**: 클릭 가능 (viewMode는 UI state로 즉시 반영)
4. **3초 이상 로딩 지속**: 화면 하단 `<p className="text-xs text-gray-400 text-center">데이터를 불러오는 중입니다...</p>` 노출
5. **10초 초과**: 자동으로 `04-에러-서버오류`로 전환(timeout) + 재시도 버튼

## 상태 고유 비즈니스 룰 델타
- 통계 카드값: 임시 `—` 표시 아님, **skeleton** 표시
- refetch 트리거 시(새로고침) 기존 데이터 유지 + 상단 `isRefetching` 인디케이터만 표시 (정식 로딩 아님)
- `branchId` 없으면 로딩 유지 (절대 빈 목록 단정 금지)

## 에지 케이스
- **캐시 있는 재진입**: React Query `staleTime` 내 재진입 → 즉시 `02-정상-데이터있음`, 로딩 화면 스킵
- **네트워크 느림**: 로딩 유지, 사용자가 새로고침 눌러도 기존 요청 취소 후 재발행
- **동시 mutate**: createMutation pending 중 refetch 발생 시 skeleton 금지, 기존 리스트 유지
