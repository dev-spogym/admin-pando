## 상태 고유 인터랙션
1. **[재시도] 버튼** → `refetch()` → `01-로딩` 전환 → 성공 시 `02-정상`, 실패 시 이 상태 유지
2. **재시도 쿨다운**: 연속 3회 실패 시 30초 쿨다운 + 카운트다운 표시
3. **[관리자에게 문의] 클릭** → `mailto:support@fitgenie.kr?subject=리드조회실패&body=RequestID={reqId}`
4. **페이지 자동 재시도**: React Query 기본 3회 재시도 후 중단
5. **ErrorBanner [x] 닫기**: 배너만 닫고 ErrorCard는 유지
6. **Esc 키**: ErrorBanner만 닫기

## 상태 고유 비즈니스 룰 델타
- 에러 발생 시 감사 로그 `AUDIT.LEAD_FETCH_FAILURE`(errorCode, requestId) 기록
- Sentry/관찰성 도구 자동 전송
- React Query `retry: 3, retryDelay: exponentialBackoff(1000)`
- mutation 실패는 toast로만 표시(inline error), 이 상태로 전환 안 함
- `E503001`(점검): 배너 tone `warn`, "서비스 점검 중입니다"

### 에러 코드별 UI 분기

## 에지 케이스
- **부분 실패**: 리드 목록은 성공인데 stats RPC만 실패 → 이 상태 아님, stats 카드만 "—"
- **인증 만료 (401)**: 이 상태 아님 → 전역 인터셉터로 `/login`
- **권한 없음 (403)**: 이 상태 아님 → `06-권한없음`
- **재시도 중 네트워크 복구**: 자동 `02-정상` 전환
- **mutation 도중 서버 오류**: toast "저장에 실패했습니다" + 기존 목록 유지

```
조건: !isLoading && error != null

const ERROR_CONFIG = {
  E500001: { tone: 'error', icon: AlertTriangle, title: '서버 오류', message: '서버 오류가 발생했습니다.', canRetry: true },
  E500950: { tone: 'error', icon: AlertTriangle, title: '리드 조회 실패', message: '리드 데이터를 불러오지 못했습니다.', canRetry: true },
  E503001: { tone: 'warn',  icon: Wrench, title: '서비스 점검', message: '서비스 점검 중입니다.', canRetry: false },
  NETWORK: { tone: 'warn',  icon: WifiOff, title: '네트워크 단절', message: '네트워크 연결을 확인해주세요.', canRetry: true },
  E429:    { tone: 'warn',  icon: Clock, title: '요청 과다', message: '요청이 많습니다. 잠시 후 다시 시도해주세요.', canRetry: true },
};

if (error) {
  const code = error.errorCode ?? (navigator.onLine ? 'E500001' : 'NETWORK');
  const cfg = ERROR_CONFIG[code];
  return (
    <div className="p-6 lg:p-8 space-y-6">
      <header className="flex items-center justify-between">
        <div>
          <h1 className="text-2xl font-bold text-gray-900">리드 관리</h1>
          <p className="text-sm text-gray-500">잠재 고객(문의/상담) 파이프라인을 관리합니다.</p>
        </div>
        <Button variant="primary" disabled title="데이터 조회 복구 후 가능">
          <Plus className="size-4 mr-1"/> 리드 등록
        </Button>
      </header>
      <ErrorBanner code={code} requestId={error.requestId} onClose={() => setDismissed(true)}/>
      <ErrorCard   code={code} onRetry={handleRetry} cooldown={cooldown}/>
    </div>
  );
}

━━ 재시도 쿨다운 ━━
const [retryCount, setRetryCount] = useState(0);
const [cooldown, setCooldown] = useState(0);
const handleRetry = () => {
  if (retryCount >= 3) {
    setCooldown(30);
    setRetryCount(0);
    const id = setInterval(() => {
      setCooldown(c => { if (c<=1) { clearInterval(id); return 0; } return c-1; });
    }, 1000);
    return;
  }
  setRetryCount(c => c+1);
  refetch();
};

━━ 접근성 ━━
- ErrorBanner role="alert" aria-live="assertive"
- ErrorCard role="status"
- 배너 발생 시 배너 DOM에 자동 포커스
- 재시도 버튼 disabled 시 aria-disabled
```
