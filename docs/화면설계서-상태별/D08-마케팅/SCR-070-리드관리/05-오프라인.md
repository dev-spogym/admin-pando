## 상태 고유 인터랙션
1. **[재연결 시도] 버튼** → `refetch()` → 네트워크 복구 시 `01-로딩` → `02-정상`
2. **브라우저 `online` 이벤트** 자동 감지 → 즉시 refetch + 배너 제거 + toast "연결이 복구되었습니다"
3. **mutation 시도(수정/삭제/등록)** → `toast.error("오프라인 상태에서는 변경할 수 없습니다.")`
4. **필터/검색**: 로컬 캐시 기준 동작 (서버 요청 없음)
5. **뷰 토글**: 정상 동작 (UI state)
6. **배너 [x] 닫기**: 임시 숨김 (30초 후 자동 재표시)

## 상태 고유 비즈니스 룰 델타
- React Query `cacheTime` 충분히 길게(예: 10분) → 캐시 유지
- ServiceWorker(PWA) 등록된 경우 해당 캐시 사용 (Phase 2)
- 오프라인 중 mutation은 **큐잉하지 않음** (동기화 리스크) — 단순 거부
- `online` 이벤트 자동 refetch + toast
- localStorage `lastSyncedAt: ISO` 저장 → "15분 전" 상대 시간 표시
- 캐시 없음 + 오프라인: 전용 `EmptyOffline` 카드 "오프라인 상태이며 캐시된 데이터가 없습니다"

## 에지 케이스
- **오프라인 진입 직후 최초 로딩**: 캐시 있으면 정상 렌더(배너만 추가), 캐시 없으면 EmptyOffline 카드
- **오프라인 중 다른 탭에서 수정**: 해당 탭도 오프라인이므로 동기화 안됨, 복구 시 일괄 refetch
- **오프라인 중 페이지 이동**: 다른 화면도 동일 배너(전역 OfflineBanner)
- **간헐적 끊김**: 1분 이내 복구 시 별도 배너 표시 없이 조용히 refetch
- **서버 정상/클라 네트워크만 끊김**: `navigator.onLine` false만 신뢰

export function OfflineBanner() {
  const [online, setOnline] = useState(true);
  useEffect(() => {
    setOnline(navigator.onLine);
    const onOnline = () => {
      setOnline(true);
      toast.success('연결이 복구되었습니다.');
      queryClient.invalidateQueries();
    };
    const onOffline = () => setOnline(false);
    window.addEventListener('online', onOnline);
    window.addEventListener('offline', onOffline);
    return () => {
      window.removeEventListener('online', onOnline);
      window.removeEventListener('offline', onOffline);
    };
  }, []);

  if (online) return null;
  return (
    <div role="alert" aria-live="assertive"
         className="sticky top-0 z-50 bg-amber-50 border-b border-amber-200 text-amber-800 px-4 py-2 flex items-center justify-between">
      <span className="inline-flex items-center gap-2 text-sm">
        <WifiOff className="size-4"/> 오프라인 — 네트워크 연결이 끊겼습니다. 복구 시 자동 동기화됩니다.
      </span>
      <button className="text-xs font-medium hover:underline"
              onClick={() => queryClient.invalidateQueries()}>
        <RefreshCw className="size-3 inline mr-1"/> 재연결 시도
      </button>
    </div>
  );
}
