## 상태 진입 조건

## 상태 고유 비주얼 델타
- 모달 유지
- 해당 필드 `border-red-500` + `aria-invalid="true"`
- 토스트:
  - 이름: `toast.error("이름을 입력하세요.")`
  - 유입경로: `toast.error("유입경로를 선택하세요.")`
- 저장 버튼 다시 활성

## 역할별 차이
공통.

## 상태 고유 인터랙션
1. 토스트 직후 해당 필드 자동 포커스
2. 인라인 메시지 하단 렌더(옵션)
3. 값 입력 시 border/aria 즉시 해제

## 상태 고유 비즈니스 룰 델타
- 서버 호출 없음 (클라 차단)
- 다중 누락 시 이름 → 유입경로 순 포커스
- 전화번호 형식 검증은 선택

## 에지 케이스
- 한글 공백 입력 → `trim().length > 0` 판정
- select 초기값이 blank로 보임 → 기본값 강제
- 날짜 포맷 오류는 서버에서 `05-API오류`로

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00 §14. 상태: validation-error

const errors: Record<string,string> = {};
if (!form.name.trim()) errors.name = '이름을 입력하세요.';
if (!form.source)      errors.source = '유입경로를 선택하세요.';
if (Object.keys(errors).length) {
  const first = Object.keys(errors)[0];
  toast.error(errors[first]);
  document.getElementById(first)?.focus();
  setFieldErrors(errors);
  return;
}
className={cx("...border...", fieldErrors.name && "border-red-500")}
aria-invalid={!!fieldErrors.name}
```
