# DLG-070-001 리드등록수정 — 상태: 유효성 오류

## 메타

| 항목 | 값 |
|------|----|
| 모달 ID | DLG-070-001 |
| 상태 코드 | `validation-error` |
| 트리거 | 저장 클릭 시 필수 필드 누락 |
| 역할 | owner, manager, fc |
| 우선순위 | P0 |
| 이전 상태 | `01-열림-등록모드` 또는 `02-열림-수정모드` |
| 다음 상태 | 수정 후 재시도 |

## 🧩 바이브코딩 프롬프트

```
유효성 오류 UI (LeadFormModal.tsx 내부):

이름 필드 오류 (TC-070-003):
<div>
  <Input
    {...register('name')}
    className={errors.name ? 'border-red-500 focus:ring-red-500' : ''}
    placeholder="이름을 입력하세요"
  />
  {errors.name && (
    <p className="mt-1 text-xs text-red-500">{errors.name.message}</p>
  )}
</div>

유입경로 오류:
- Select border-red-500
- "유입경로를 선택해주세요" 에러 텍스트

연락처 형식 오류 (TC-070-M2-01):
- "올바른 연락처 형식을 입력해주세요 (예: 010-1234-5678)"

Zod 스키마:
const schema = z.object({
  name: z.string().min(1, '이름을 입력해주세요'),
  phone: z.string().regex(/^[0-9-]*$/, '숫자와 하이픈만 입력 가능합니다').optional().or(z.literal('')),
  source: z.string().min(1, '유입경로를 선택해주세요'),
  status: z.string().default('신규'),
  fcId: z.string().optional(),
  inquiryDate: z.string(),
  followUpDate: z.string().optional(),
  memo: z.string().optional(),
})

오류 시 동작:
- 모달 유지 (닫히지 않음)
- 오류 필드 focus + red border
- 저장 버튼 loading 해제
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 이름 빈값 (TC-070-003)
- 유입경로 미선택
- 연락처 형식 오류 (TC-070-M2-01)

### 인터랙션 (User Actions)
1. 오류 필드 수정 후 재시도
2. "취소" → 모달 닫힘 (오류 무시)

### 비즈니스 룰
- 클라이언트 검증 우선, 서버 응답 에러도 인라인 표시
- 최초 submit 전에는 에러 미표시 (touched 기준)
