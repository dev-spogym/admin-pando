## 역할별 차이
- 모든 역할 공통 동일 skeleton (권한 검증은 loading 이전 단계에서 완료)

## 상태 고유 인터랙션
- 탭 전환: 허용 (각 탭 고유 skeleton 유지)
- 버튼 클릭: 모두 `disabled`, Tab 포커스는 가능하나 Enter 무시
- 모달 트리거 버튼 비활성

## 상태 고유 비즈니스 룰 델타
- `staleTime: 60_000` 유지되므로 재방문 시 캐시 데이터 있으면 **skeleton 생략**하고 `02-정상-*`로 즉시 진입
- 최소 skeleton 노출 시간 없음 (0ms) — flash 허용

## 에지 케이스
- 3초 이상 loading 지속: 상단에 "불러오는 중..." 토스트 옵션 (UX 가이드)
- 부분 성공: points만 먼저 도착 → PointBadge는 실제값, 폼/이력은 skeleton 유지

## 바이브코딩 프롬프트 (델타)

```
마스터 00-기본화면.md를 기반으로, 모든 쿼리가 isLoading=true인 상태의 skeleton UI를 렌더하라.

- 카드 3개 모두 내부 텍스트 대신 <Skeleton /> 컴포넌트 사용
- Skeleton: <div className="bg-gray-200 rounded animate-pulse" style={{width,height}} />
- 발송 버튼: <button disabled className="h-12 w-full rounded-lg bg-gray-200 animate-pulse" />
- PointBadge: <span className="inline-block h-6 w-20 rounded-full bg-gray-200 animate-pulse" />
- 이력 테이블: thead 실제 렌더, tbody는 tr×6을 pulse 처리
- aria-busy="true" on <main>
- prefers-reduced-motion: animate-pulse 제거, bg-gray-200 고정
- isLoading 판정: const isLoading = recent.isLoading || points.isLoading
```
