## 상태 고유 인터랙션

| 인터랙션 | 동작 |
|---|---|
| 눈 토글 클릭 | `isPwVisible=true` + `audit_logs` INSERT (`action='password_view'`) + 3초 후 자동 재마스킹 |
| 비밀번호 입력 변경 | `dirty=true` → 저장 활성 |
| 메모 입력 변경 | `dirty=true` + 문자 카운터 갱신 |

| 리스트 스크롤 | `max-h-[50vh] overflow-y-auto`, 포커스 링 유지 |
| ESC/배경/X | 닫힘 (미저장 변경 있으면 `window.confirm` 선택적) |

## 상태 고유 비즈니스 룰 델타

1. **정렬**: `created_at DESC`, 동일 시각은 `id DESC`
2. **액션 라벨/배지**: 마스터 §8.4 ACTION_LABEL/ACTION_TONE 맵 적용
3. **시간 포맷**: `YYYY-MM-DD HH:mm` (분까지), `tabular-nums` 고정폭
4. **상세 필드 분기**:
   - `assign` / `reclaim` / `bulk_assign` / `bulk_release` → `details.member_name` 표기
   - `move` → `details.from_number→details.to_number`
   - `password_change` / `memo_change` → 값 미노출(보안)
5. **시스템 이벤트**: `operator IS NULL` → "시스템" 배지
6. **50건 초과**: `hasMore=true` 시 하단 "더 보기" 링크(옵션)
7. **낙관적 업데이트**: 메타 저장 성공 시 부모 캐시를 즉시 반영

## 에지 케이스

- 이력 1건뿐 → EmptyState 아님, `02` 유지
- `details.member_name` 결측 → "(탈퇴 회원)" 폴백
- 담당자 레코드 삭제 → operator join null → "(삭제된 사용자)" 폴백
- 비밀번호 필드 공백 저장 → `null` 처리(락커 비밀번호 제거)
- 50건 초과: 현재는 최신 50건만, 페이지네이션은 옵션(별도 티켓)
