## 상태 진입 조건

- `query.status === 'success'` 이며 `data.items.length === 0`
- 주로 **신규 추가된 락커** 또는 **오래된 이력이 감사 로그 보관 기간 초과로 파기된 경우**

## 상태 고유 인터랙션

- 일반 인터랙션 동일 (ESC/배경/X/닫기)
- 메타 저장 후에도 이력은 여전히 `0`건 (저장은 자체 감사 트리거로 새 이력 생성 → 다음 오픈 시 1건 표시)
- 사용자에게 추가 액션 유도 없음 (정상 상황)

## 상태 고유 비즈니스 룰 델타

1. **이력 없음 정의**: `audit_logs` 쿼리 결과가 빈 배열일 때 (에러 아님)
2. **부가 설명**: 감사 로그 보관 기간(기본 90일) 초과 시에도 이 상태로 표시. 기획 정책 변경 가능.
3. **서버 성능**: 빈 결과도 캐시(staleTime 30s). 불필요한 재쿼리 방지.
4. **다음 액션 유도 없음**: "배정하기" 등 버튼 제공하지 않음. 부모 SCR-050에서 수행.
5. **감사 시점**: 본 다이얼로그 오픈 자체를 감사할지는 추가 정책 (현재 미기록).

## 에지 케이스

- 락커는 있지만 `audit_logs` 테이블 마이그레이션 이전 생성 → `03` (정상 분기)
- 방금 추가된 락커(Zero-State) → `03` (자연스러움)
- 감사 로그 파티셔닝 실패 등 서버 이상 → `04-에러`로 분기해야 함 (이 상태 아님)
- 비밀번호/메모 저장 성공 직후 재오픈 시 → `02-이력있음`(`memo_change` 1건)
