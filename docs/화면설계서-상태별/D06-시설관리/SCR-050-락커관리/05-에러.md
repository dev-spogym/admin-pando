## 상태 진입 조건

- `useLockers` fetch 실패 (`isError === true`)
- HTTP 500 (`E500001`) / 503 (`E503001`) / 네트워크 4xx 기타
- 10초 타임아웃 초과 (`01-로딩`에서 자동 전환)
- 3회 이상 재시도 실패

### ErrorState 컴포넌트
```tsx
<ErrorState
  icon={<AlertCircle size={72} className="text-red-500"/>}
  title="락커 데이터를 불러오지 못했습니다"
  errorCode={errorCode}
  description={ERROR_MESSAGES[errorCode] ?? "일시적인 오류가 발생했습니다."}
  actions={[
    <Button variant="primary" onClick={retry} disabled={retryCount>=3}>
      <RefreshCcw/> 다시 시도 {retryCount>0 && `(${retryCount}/3)`}
    </Button>,
    <Button variant="ghost" onClick={contactSupport}>
      지원 문의
    </Button>,
  ]}
  footer={`마지막 시도: ${format(lastTriedAt, 'HH:mm:ss')}`}
/>
```

## 역할별 차이

| 역할 | 에러 세부 노출 |
|---|---|
| super/primary | errorCode + detail + 서버 시간 + [로그 복사] 버튼 |
| owner/manager | errorCode + 기본 메시지 + 재시도 |
| staff/front | 기본 메시지만, "관리자에게 문의하세요" 안내 |
| fc/trainer | 이 상태 도달 전 /forbidden |

## 상태 고유 인터랙션

- **[다시 시도]**: `queryClient.invalidateQueries(['lockers'])` + `retryCount++`
- **3회 실패 후 버튼 disabled**: "잠시 후 다시 시도해주세요" 표기
- **[지원 문의]**: `mailto:support@fitgenie.com?subject=Locker%20Error%20E500001&body=...` 또는 지원 페이지
- **[로그 복사] (super만)**: 에러 스택 클립보드 복사 + toast "로그가 복사되었습니다"

## 비즈니스 룰 델타

- 재시도 간격: 즉시 → 2초 → 5초 (exponential backoff)
- 3회 실패 시: disabled + 30초 쿨다운 후 재활성
- 503 수신 시: 점검 모드 배너 추가 "서비스 점검 중입니다"
- 500 에러: 자동 감사 로그 기록 (`AUDIT.LOCKER_FETCH_FAIL`)
- 세션 만료(401): 이 상태 미진입, 즉시 `/login` 리다이렉트

## 에지 케이스

| 상황 | errorCode | 처리 |
|---|---|---|
| 서버 내부 오류 | E500001 | 기본 에러 스테이트 + 재시도 |
| 서비스 점검 | E503001 | `bg-amber-50 border-amber-200` 점검 배너 |
| Rate limit 429 | E429001 | "요청이 너무 많습니다. 30초 후 다시 시도" + 카운트다운 |
| DB 타임아웃 | E500001 | 기본 에러 + 지원 문의 |
| 권한 만료 중간 | 401 | 즉시 `/login` 리다이렉트 (이 상태 무시) |
| 브라우저 오프라인 | NETWORK | 즉시 `06-오프라인` 전환 (이 상태 skip) |
