## 상태 진입 조건

- `navigator.onLine === false` (`offline` 이벤트)
- 또는 fetch가 `NETWORK_ERROR` (`Failed to fetch`)
- SW 캐시 존재 시 stale 데이터 fallback
- SW 캐시 없음 시 안내 화면

## 역할별 차이

역할 무관 (네트워크 이슈는 전 역할 동일). 단:
- super/primary: 배너에 "네트워크 진단" 버튼 추가 (개발자 도구 유도)

## 상태 고유 인터랙션

- **online 이벤트 감지**: `window.addEventListener('online', ...)` → 자동 refetch → `02-정상` 전환
- **[연결 상태 확인]**: `navigator.onLine` + 간단한 `/health` ping → 복귀 확인
- **쓰기 액션 시도**: 즉시 toast "오프라인 상태에서는 사용할 수 없습니다"
- **캐시 데이터 클릭**: 읽기 전용 상세 모달만 표시 (액션 버튼 숨김)

## 비즈니스 룰 델타

- Service Worker 캐시: 마지막 성공한 `GET /lockers` 응답을 IndexedDB에 저장 (TTL 24h)
- 캐시 TTL 초과 시 → 캐시 없음 상태
- 복귀 시 차이점 감지: `If-None-Match` ETag로 증분 갱신
- 오프라인 중 쓰기 시도는 **큐잉하지 않음** (일관성 우선). 사용자에게 즉시 안내

## 에지 케이스

| 상황 | 처리 |
|---|---|
| VPN/프록시 부분 단절 | `/health` ping으로 실제 연결 확인 |
| 일시적 끊김 (<3초) | 배너 표시하되 자동 복귀 시 배너 제거 |
| 캐시 있지만 만료 | "데이터가 최신이 아닐 수 있습니다" 경고 |
| 캐시 없는 첫 진입 + 오프라인 | 캐시 없음 안내 화면 |
| 복귀 후 refetch 실패 | `05-에러` 전환 |
| 모바일 데이터 ↔ WiFi 전환 | 배너 깜빡임 방지 (debounce 1s) |
