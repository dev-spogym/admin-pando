## 📝 디스크립션

### 사용 시점 (Trigger)
- `E_F6_08`: 개별 락커 액션 API 4xx/5xx 응답
- 배정/이동/회수 중 특정 락커만 실패

### 필수 데이터
- 실패한 락커 ID 목록 (에러 표시용)
- 성공한 나머지 락커는 정상 표시

### 인터랙션 (User Actions)
1. "재시도" 버튼 → 실패 락커만 재조회
2. 에러 배너 닫기 (X) → 배너 숨김 (에러 셀은 유지)
3. 나머지 정상 셀은 계속 인터랙션 가능

### 비즈니스 룰
- 전체 실패가 아닌 부분 실패이므로 전체 에러 상태로 전환하지 않음
- 에러 셀 클릭 시 "데이터를 불러오지 못했습니다. 재시도해주세요." toast

### 에지 케이스
- 3회 연속 실패 → "관리자에게 문의하세요" CTA로 변경
- 재시도 성공 → 에러 표시 제거, 정상 셀로 복구

### 접근성 (A11y)
- 에러 배너 `role="alert"` + `aria-live="polite"`
- 에러 셀 `aria-label="락커 {번호} - 데이터 오류"`
