### EmptyState 컴포넌트
```tsx
<EmptyState
  icon={<Lock size={64} className="text-gray-300" />}
  title="등록된 락커가 없습니다"
  description="락커를 추가하여 시설 관리를 시작하세요."
  action={canAddLocker(role) ?
    <Button variant="primary" size="lg" onClick={openAddLocker}>
      <Plus/> 락커 추가
    </Button> : null
  }
/>
```

## 역할별 차이

| 역할 | EmptyState CTA |
|---|---|
| super/primary/owner/manager | "+ 락커 추가" Primary 버튼 |
| staff | "+ 락커 추가" 버튼 **숨김**, 대신 "관리자에게 문의하세요" 텍스트 |
| front | 읽기 전용 안내 "관리자 권한이 필요합니다" |
| fc/trainer | 이 상태 도달 전 /forbidden |

## 상태 고유 인터랙션

- 검색창 숨김 (filtering 의미 없음)
- 탭 전환 가능하되 모든 구역 빈 상태
- 새로고침: `refetch()` → 락커 생성 감지 시 `02-정상` 전환
- [+ 락커 추가] CTA 클릭: `openAddLocker()` → AddLockerModal (번호/구역 입력)

## 비즈니스 룰 델타

- 락커 추가 성공 후 즉시 `02-정상` 상태로 전환 (refetch)
- 3개 이상 연속 추가 시 배치 등록 힌트 toast: "대량 등록은 CSV 가져오기를 이용하세요" (P2 기능)
- 초기 등록 시 지점 내 번호 순차 (최소 번호부터)

## 에지 케이스

| 상황 | 처리 |
|---|---|
| 전체 지점은 락커 있지만 현재 zone만 0개 | EmptyState 대신 "A구역에 락커가 없습니다. 다른 구역을 확인하세요" (zone별) |
| branchId 전환 직후 | 로딩 → 새 지점의 empty 상태 |
| 락커 추가 실패 (네트워크) | AddLockerModal 내부 에러 처리, 본 화면 상태 유지 |
