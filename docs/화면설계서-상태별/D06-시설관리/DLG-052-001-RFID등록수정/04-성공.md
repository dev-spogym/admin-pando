## 상태 진입 조건
- `POST /rfid` → 201 Created (등록)
- `PUT /rfid/:id` → 200 OK (수정)
- 응답 본문에 완성된 `RfidCard` 객체 포함

## 상태 고유 비주얼 델타
- 이 상태는 **거의 순간적** — 다이얼로그 즉시 언마운트 + 토스트만 노출
- 다이얼로그 본체: 렌더 없음
- 토스트:
  - 등록: `toast.success('RFID 카드가 등록되었습니다')`
  - 수정: `toast.success('RFID 카드가 수정되었습니다')`
  - 수정(상태 변경 동반): `toast.success('RFID 카드가 수정되었습니다. 상태: 분실')`

## 상태 고유 인터랙션
- 사용자 입력 없음(자동)
- 토스트 자동 소멸(3초)
- 목록 자동 갱신 — 새로 등록된 행 상단에 하이라이트(1초 amber-50 flash)

## 역할별 차이
- 없음. 단 감사로그에 `role`, `userId`, `branchId` 기록.

## 상태 고유 비즈니스 룰 델타

### 후속 처리 체인
```
onSuccess(savedCard) =>
  1) 토스트 노출 (등록/수정 구분)
  2) queryClient.setQueryData(['rfid-cards', branchId], cache => cache + savedCard) — 낙관적 캐시 병합
  3) queryClient.invalidateQueries(['rfid-cards', branchId]) — 실제 재조회
  4) onSaved?.(savedCard)  — 부모 콜백 (행 하이라이트 등)
  5) onClose() — 다이얼로그 언마운트
  6) (선택) 사물함 연결된 경우 queryClient.invalidateQueries(['lockers', branchId])
```

### 감사로그
- 서버 자동 기록:
  - `AUDIT.RFID_CREATE { userId, userName, cardNo, memberId|staffId, branchId, createdAt }`
  - `AUDIT.RFID_UPDATE { userId, userName, cardId, changes: {...}, branchId, updatedAt }`
- 클라이언트 추가 호출 없음

### 행 하이라이트(부모 화면 SCR-052)
- 새로 등록된 행 또는 수정된 행: `className="animate-[highlightFade_1s_ease-out]"`
- CSS: `@keyframes highlightFade { from { background: #FEF3C7; } to { background: transparent; } }`

## 에지 케이스
- **onSuccess 내부 에러**: 저장은 성공이므로 실패로 취급하지 않음. 콘솔 경고만.
- **낙관적 캐시 병합 실패**: `invalidateQueries` 가 폴백 보장.
- **연결 사물함 동시 편집**: `lockerNo` 저장 후 SCR-050 동기화를 위해 `['lockers']` 쿼리도 invalidate
- **브라우저 탭 여러 개**: BroadcastChannel 또는 Supabase Realtime 구독 시 중복 토스트 방지 — `lastSavedId` 세션스토리지 기록 후 중복 무시

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: rfid-save-success

onSuccess 핸들러:
function onSuccess(savedCard: RfidCard) {
  // 1) 토스트
  const msg = mode === 'create'
    ? 'RFID 카드가 등록되었습니다'
    : `RFID 카드가 수정되었습니다${savedCard.status === 'lost' ? '. 상태: 분실' : ''}`;
  toast.success(msg);

  // 2) 캐시 병합(낙관)
  qc.setQueryData<RfidCard[]>(['rfid-cards', branchId], (prev = []) => {
    if (mode === 'create') return [savedCard, ...prev];
    return prev.map(c => c.id === savedCard.id ? savedCard : c);
  });

  // 3) 실제 refetch
  qc.invalidateQueries({ queryKey: ['rfid-cards', branchId] });

  // 4) 사물함 동기화
  if (savedCard.lockerNo) qc.invalidateQueries({ queryKey: ['lockers', branchId] });

  // 5) 부모 콜백 → 행 하이라이트
  onSaved?.(savedCard);

  // 6) 닫기
  onClose();
}

행 하이라이트 (부모 화면):
const [highlightId, setHighlightId] = useState<number | null>(null);
const handleSaved = (card) => { setHighlightId(card.id); setTimeout(() => setHighlightId(null), 1000); };
<tr key={c.id} className={c.id === highlightId ? 'animate-[highlightFade_1s_ease-out]' : ''}>
```
