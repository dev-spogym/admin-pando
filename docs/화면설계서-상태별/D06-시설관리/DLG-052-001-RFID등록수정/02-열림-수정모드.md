# DLG-052-001 RFID 등록/수정 — 상태 02: 열림 (수정 모드)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 상태 진입 조건
- SCR-052 행 메뉴 `[수정]` 클릭
- `canUpdateRfid(role, card.branchId)` 통과
- 카드 데이터 전체 로드(404가 아님)

## 상태 고유 비주얼 델타
- 제목: `"RFID 카드 수정"` (CreditCard 아이콘)
- `cardNo` 입력: **readonly** (`disabled:bg-gray-50 disabled:text-gray-500`), 스캔 버튼 비노출
- 기존 데이터 채워짐: `{ cardNo, userType, memberId|staffId, lockerNo, status }`
- 추가 필드 노출: `상태(status) Select` — 활성/분실/해제
- 첫 포커스: 첫 편집 가능 필드(`userType` RadioGroup)
- `isDirty=false` 상태에서 저장 버튼 `disabled` (의미 없는 PUT 방지)

## 상태 고유 인터랙션
1. **오픈 직후**: `initialData`로 `reset(initialData)` + 첫 편집 필드 포커스
2. **상태 변경 Select**: `lost`/`released` 선택 시 하단에 **경고 문구** 표시 "분실/해제 처리 시 회원 출입이 차단됩니다"
3. **Enter**: 어느 필드에서든 Enter → 폼 submit
4. **ESC**: `isDirty` 여부에 따라 DLG-002 이탈 경고 또는 즉시 닫기
5. **회원 변경**: AutoComplete로 다른 회원 선택 시 기존 mapping 자동 해제(서버 처리)

## 역할별 차이
- `manager`: 상태 Select에서 `lost`/`released` 선택 가능, 단 삭제는 불가 (삭제는 DLG-052-003)
- `front`: 상태 Select 자체 **비노출** (읽기 전용 필드로 렌더)
- `superAdmin/primary`: 다른 지점 카드 수정 시 제목에 `[{branchName}]` 부기

## 상태 고유 비즈니스 룰 델타
- `mode='edit'` 고정, `cardNo` 불변
- `isDirty===false` 에서 저장 클릭 시 API 호출 생략 → 토스트 없이 닫기
- 상태를 `released` 로 변경 → 연결된 사물함(`lockerNo`)이 있으면 경고 배너 "연결된 사물함 {N} 함께 해제됩니다"
- PUT 응답 성공 시 목록 `queryClient.invalidateQueries(['rfid-cards'])`
- 수정 중 카드 삭제됨(다른 세션) → 404 → 토스트 "이미 삭제된 카드입니다" + 닫기 + 목록 refetch

## 에지 케이스
- **회원 비활성화 상태**: 수정 대상 카드의 `memberId`가 비활성/탈퇴 회원 → AutoComplete 초기값에 "비활성 회원" 뱃지 표시, 저장은 가능(운영 편의)
- **사물함 번호 변경 후 충돌**: 서버에서 `lockerNo` 유일성 검증 → 422 `E422051` → 필드 인라인
- **낙관적 업데이트 후 409**: 다른 관리자가 동시 수정 → `E409052` → 최신 데이터 재조회 후 사용자에게 "데이터가 변경되었습니다. 다시 확인하세요" 배너
