## 상태 고유 인터랙션
- 버튼/인풋 전부 `disabled` + `aria-busy="true"`
- 사이드바 이동은 허용(페이지 언마운트)
- 새로고침 연타 방지 `React Query dedupe`(60초)

## 상태 고유 비즈니스 룰 델타
- 로딩 3초 초과 시 `toast.info "불러오는 중..."` (1회만)
- 8초 초과 시 `04-에러`로 전환 후보(타임아웃 처리)
- 이전 페이지 캐시 존재 시 `placeholderData: keepPreviousData`로 stale 데이터 flash 방지

## 에지 케이스
- 네트워크 단절 중 진입: 로딩 유지 → `navigator.online=false` 감지 시 즉시 `04-에러` 오프라인 배너
- primary/super의 branchId 미선택: 로딩 없이 "지점을 선택해주세요" 안내 (빈상태 아님)
- 백엔드 slow (>5s): progressive hint "데이터가 많아 조금 느릴 수 있어요"

## 바이브코딩 프롬프트 (델타)
```
마스터 프롬프트를 따르되 로딩 스테이트만 구현:
- isLoading 분기에서 <TableSkeleton rows={10}/> + <StatCardSkeleton count={3}/>
- PageHeader 액션 버튼은 disabled={isLoading}
- FilterBar 인풋도 disabled={isLoading}
- 8초 타임아웃 시 throw Error → onError에서 상태 전환
```
