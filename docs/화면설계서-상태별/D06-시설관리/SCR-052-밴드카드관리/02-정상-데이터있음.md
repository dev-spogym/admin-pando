## 상태 고유 인터랙션
- **회원명 셀**: member-detail 이동 (`/members/detail?id={memberId}`)
- **📜 이력**: DLG-052-002 오픈, cardId 전달
- **❌ 분실 (ACTIVE)**: ConfirmModal "카드 {cardNo}를 분실 처리하시겠습니까?" → `POST /rfid/cards/:id/lost` → toast.success "카드가 분실 처리되었습니다" → 상태 배지 즉시 LOST로 갱신
- **⏺ 복구 (LOST/RELEASED)**: DLG-052-001 수정 모드로 오픈 후 상태 ACTIVE 변경
- **✏ 수정**: DLG-052-001 수정 모드 (cardNo readonly, 회원 매핑·사물함·메모 편집)
- **🗑 삭제**: DLG-052-003 "삭제" 확인 텍스트 입력 → `DELETE /rfid/cards/:id`
- **필터 변경**: 300ms debounce → page=1로 리셋
- **검색**: 카드번호/회원명 부분 일치 + 연락처 뒷 4자리

## 상태 고유 비즈니스 룰 델타
- 통계(전체/활성/미사용)는 서버가 `stats` 필드에 병합 반환 또는 클라 집계 허용
- 페이지네이션: `total / 10` 올림
- 새로고침: `invalidateQueries(['rfid'])` → 로딩 상태 재진입
- Supabase Realtime 구독(선택): `rfid_cards` 테이블 INSERT/UPDATE 이벤트 수신 시 해당 행만 patch
- 가상 스크롤 불필요(페이지당 10건)

## 에지 케이스
- 200개 넘는 결과: 페이지네이션 12페이지+ 표시
- 분실 처리 도중 네트워크 실패: 낙관적 업데이트 롤백 + toast.error
- 동시 편집(다른 스태프가 수정 중): 저장 시 optimistic concurrency → "다른 사용자가 수정했습니다. 새로고침" 안내
- 삭제 후 현재 페이지가 비면 page-1로 자동 이동

## 바이브코딩 프롬프트 (델타)
```
isLoading=false && items.length>0 분기만 구현:
- DataTable rows={items} 매핑
- STATUS_VARIANT = { ACTIVE:'success', LOST:'error', RELEASED:'default' }
- STATUS_LABEL   = { ACTIVE:'활성', LOST:'분실', RELEASED:'해제' }
- 분실 액션: mutation POST /lost + optimistic update + 롤백
- 삭제 액션: mutation DELETE + 현재 페이지 empty 처리
- Realtime 구독(선택): supabase.channel('rfid').on('UPDATE',...).subscribe()
- 새 행 하이라이트: useRef + setTimeout(()=>clear, 500)
```
