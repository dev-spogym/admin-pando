## 📝 디스크립션

### 사용 시점 (Trigger)
- `E_F6_OK_01`: API 200 + rfid_cards.length > 0

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| RFID 카드 | `rfid_cards` | `branch_id = ?` |
| 회원 | `members` | JOIN on member_id |
| 통계 | `rfid_cards` | 상태별 COUNT |

### 인터랙션 (User Actions)
1. "카드 등록" 버튼 → DLG-052-001 등록 모달
2. "수정" 버튼 → DLG-052-001 수정 모달 (기존 데이터 채워진 폼)
3. "이력" 버튼 → DLG-052-002 이력 조회 모달
4. "삭제" 버튼 → DLG-052-003 삭제 확인 모달
5. 상태 필터 → 해당 상태만 표시

### 비즈니스 룰
- lost 카드: 모든 액션 버튼 비활성화 (삭제만 가능)
- revoked 카드: 수정 불가, 이력/삭제만 가능
- card_number 중복 불가 (branch_id 내)

### 에지 케이스
- 분실 카드 다수: 상단에 "분실 카드 N건" 경고 배너
- 카드 없는 회원: 회원명 "미배정" 표시

### 접근성 (A11y)
- 테이블 `role="table"` + 컬럼 `aria-sort`
- 상태 배지 `aria-label="{상태명} 카드"`
