## 📝 디스크립션

### 사용 시점 (Trigger)
- `E_F6_OK_01`: API 200 + clothing.length > 0

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 운동복 | `clothing` | `branch_id = ?` |
| 대여자 | `members` | JOIN on rented_by |
| 통계 | `clothing` | 상태별 COUNT |

### 인터랙션 (User Actions)
1. "대여" 버튼 (AVAILABLE) → 회원 검색 + 대여 처리 → RENTED 전환
2. "반납" 버튼 (RENTED) → 반납 확인 → WASHING 전환 (세탁 필요)
3. "세탁완료" 버튼 (WASHING) → AVAILABLE 전환
4. 상태 TabNav → 해당 상태만 필터링
5. "운동복 추가" → 신규 운동복 코드 등록 모달

### 비즈니스 룰
- 반납 시 자동으로 WASHING 상태로 전환 (세탁 필요 가정)
- 반납예정일 초과 → 대여자명 옆에 "연체" 배지 (text-red-600)
- 사이즈별 재고 부족 (AVAILABLE 0) → 해당 사이즈 행 bg-amber-50

### 에지 케이스
- 전체 RENTED: "재고 없음" 경고 배너
- 연체 운동복 다수: 상단 "연체 운동복 N건" 알림 배너
