## 📝 디스크립션

### 사용 시점 (Trigger)
- `session === null` → `/login` 리다이렉트
- `hasPermission('attendance:read') === false` → 403 UI 렌더

### 필수 데이터
- useAuthStore: session, role
- hasPermission 유틸

### 인터랙션 (User Actions)
1. "대시보드로 이동" 버튼 → `/dashboard`
2. 브라우저 뒤로가기 가능

### 비즈니스 룰
- SCR-I001은 모든 역할(superAdmin~staff) 읽기 허용이므로 실질적 403은 미인증 케이스
- 미인증: redirect('/login?returnUrl=/attendance')
- 로그: 403 접근 시도 서버 로그 기록 (optional)

### 에지 케이스
- 세션 만료 중 접근: 자동 로그인 시도 후 실패 → 로그인 페이지
- 권한 변경 후 새로고침: 정상 진입

### 연결 화면
- 이전: 사이드바 메뉴 / 직접 URL 입력
- 다음: `/dashboard` / `/login`
