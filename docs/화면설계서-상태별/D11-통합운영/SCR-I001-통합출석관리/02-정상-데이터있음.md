## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 + `attendance.length > 0`
- 실시간 WebSocket으로 신규 출석 INSERT 수신 시 로그 상단 추가

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| KPI | `attendance` | `branch_id=?`, 오늘 날짜, 채널별 GROUP BY |
| 로그 | `attendance` JOIN `members` | `branch_id=?`, 날짜·채널 필터 |
| IoT | `iot_devices` | `branch_id=?`, `type='entrance'` |

### 인터랙션 (User Actions)
1. 수동 출석 등록 버튼 → DLG-I001 열기
2. 필터 변경 → 재조회 (debounce 300ms)
3. 회원 행 클릭 → `/members/[id]`
4. IoT 장애 배지 클릭 → `/settings/iot`

### 비즈니스 룰
- 권한 `attendance:write` 없으면 수동 등록 버튼 숨김
- 실시간 수신 시 KPI 카드 자동 +1 갱신
- 중복 출석(30분 이내 동일 회원) → StatusBadge "중복" 표시

### 에지 케이스
- WebSocket 끊김 → `07-WebSocket재연결` 상태
- IoT 장비 오류 감지 → `06-IoT장애` 배지 오버레이
- 필터 결과 0건 → `03-빈상태`로 전환

### 연결 화면
- 이전: `01-로딩`
- 다음: DLG-I001(수동등록) / `03-빈상태` / `06-IoT장애` / `07-WS재연결` / `/members/[id]`
