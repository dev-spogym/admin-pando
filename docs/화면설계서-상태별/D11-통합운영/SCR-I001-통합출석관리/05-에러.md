## 📝 디스크립션

### 사용 시점 (Trigger)
- Supabase 네트워크 오류 (offline)
- API 500/503 응답
- RLS 정책 위반 (예외적 권한 문제)
- Promise.all reject

### 필수 데이터
- error 객체 메시지 (사용자 친화적 변환)
- 재시도 횟수 카운터 (3회 초과 시 고객센터 CTA 표시)

### 인터랙션 (User Actions)
1. "다시 시도" 버튼 → loadData() 재실행
2. 3회 연속 실패 → "고객센터 문의" 링크 노출
3. 필터 변경 → 재조회 시도 가능

### 비즈니스 룰
- 재시도 간격 최소 1초 (rate limit 방지)
- 3회 연속 실패: 에러 코드 노출 + 고객센터 링크
- 토스트 id='attendance-error' 로 중복 방지

### 에지 케이스
- 오프라인 → `08-오프라인` 배너 별도 처리
- JWT 만료 에러 → 자동 리다이렉트 to `/login`
- 부분 실패 (KPI만 실패): 해당 영역만 에러 표시

### 연결 화면
- 이전: `01-로딩`
- 다음: 재시도 성공 → `02-정상` / 반복 실패 유지 / 인증 만료 → `/login`
