```
파싱 상태 관리:
const [parsingState, setParsingState] = useState<{
  status: 'idle' | 'parsing' | 'done' | 'error'
  fileName: string
  progress: number // 0~100
  parsedCount: number
  errorCount: number
}>({ status: 'idle', fileName: '', progress: 0, parsedCount: 0, errorCount: 0 })

파싱 중 프로그레스바:
{parsingState.status === 'parsing' && (
  <div className="bg-blue-50 border border-blue-200 rounded-lg px-4 py-3 mb-5">
    <div className="flex items-center justify-between mb-2">
      <p className="text-sm font-semibold text-blue-700">
        파일 파싱 중: {parsingState.fileName}
      </p>
      <span className="text-xs text-blue-500">{parsingState.progress}%</span>
    </div>
    <div className="w-full h-2 bg-blue-100 rounded-full overflow-hidden">
      <div
        className="h-full bg-blue-500 rounded-full transition-all duration-300"
        style={{ width: `${parsingState.progress}%` }}
      />
    </div>
    <p className="mt-1.5 text-xs text-blue-500">
      처리 중: {parsingState.parsedCount}건 / 오류: {parsingState.errorCount}건
    </p>
  </div>
)}

파싱 완료 처리:
- toast.success(`파싱 완료: ${parsedCount}건 등록, ${errorCount}건 오류`)
- loadBodyCompositionData() // 목록 갱신
- 검수대기 상태로 자동 전환 (review_status = 'pending')

파싱 실패:
- toast.error('파일 형식을 확인해 주세요 (CSV/XLSX 지원)')
- parsingState.status = 'error'

파일 형식 오류:
- 즉시 toast.error('지원하지 않는 파일 형식입니다')
- 파일 선택 초기화
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- InBody CSV/XLSX 파일 드래그앤드롭 또는 파일 선택
- POST /api/body-composition/parse 요청 후

### 필수 데이터
- 파싱 진행 상태 (WebSocket 또는 폴링)
- 파일명, 진행률, 처리/오류 건수

### 인터랙션 (User Actions)
- 파싱 중 테이블 조회 가능 (읽기 차단 없음)
- 파싱 중 추가 파일 업로드 차단
- "파싱 취소" 버튼 (서버에서 cancel 처리, 향후)

### 비즈니스 룰
- 파싱 완료 데이터: `review_status = 'pending'` 저장 → 검수 필요
- 중복 날짜 데이터: 덮어쓰기 확인 모달 (M3 DLG-I003 규칙 동일)
- 파싱 오류 행: 개별 오류 리포트 (errorCount 집계)
- 지원 형식: InBody CSV, 표준 엑셀 양식

### 에지 케이스
- 빈 파일: 즉시 "데이터가 없는 파일입니다" 오류
- 헤더 불일치: "InBody 표준 양식이 아닙니다" 오류
- 100건 초과: 분할 처리 (50건씩 배치)

### 연결 화면
- 이전: `02-정상` / `03-빈상태`
- 다음: 파싱 완료 → `04-검수대기배지` / 실패 → `02-정상` + 에러 토스트
