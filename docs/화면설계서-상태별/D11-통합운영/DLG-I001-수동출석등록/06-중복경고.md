## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: DLG-I001 수동출석등록 — 상태: 중복 출석 경고

파일: src/components/attendance/ManualAttendanceModal.tsx

조건: error.code === '23505' (unique constraint) 또는 409 응답

중복 경고 UI (모달 하단 오버레이 방식):
{duplicateWarning && (
  <div className="mt-4 p-4 bg-amber-50 border border-amber-200 rounded-xl">
    <div className="flex items-start gap-2 mb-3">
      <svg ... className="w-5 h-5 text-amber-500 flex-shrink-0 mt-0.5" />
      <div>
        <p className="text-sm font-semibold text-amber-700">이미 체크인된 회원입니다</p>
        <p className="text-xs text-amber-600 mt-0.5">
          {selectedMember?.name}님은 {format(lastCheckInTime, 'HH:mm')}에 이미 출석이 등록되어 있습니다.
          강제로 추가 등록하시겠습니까?
        </p>
      </div>
    </div>
    <div className="flex gap-2">
      <button
        onClick={() => submitAttendance(true)}  // force=true
        disabled={isSubmitting}
        className="flex-1 px-3 py-2 bg-amber-500 text-white text-xs font-medium rounded-lg hover:bg-amber-600 disabled:opacity-60"
      >
        {isSubmitting ? '등록 중...' : '강제 등록'}
      </button>
      <button
        onClick={() => setDuplicateWarning(false)}
        className="flex-1 px-3 py-2 bg-white border border-amber-300 text-amber-700 text-xs font-medium rounded-lg hover:bg-amber-50"
      >
        취소
      </button>
    </div>
  </div>
)}

강제 등록 API:
submitAttendance(force=true):
- payload: { ...formData, force: true, force_reason: 'manual_override' }
- 강제 등록 성공: 정상 성공 처리 (05-성공닫힘)
- 강제 등록 실패: toast.error + 경고 유지
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 동일 회원 동일 날짜 중복 출석 시도 (30분 이내 또는 하루 1회 제한)
- Supabase unique constraint 위반

### 필수 데이터
- 기존 출석 시각 (lastCheckInTime): 에러 응답에 포함
- duplicateWarning: boolean

### 인터랙션 (User Actions)
1. "강제 등록" → submitAttendance(force=true)
2. "취소" → duplicateWarning=false (폼으로 복귀)

### 비즈니스 룰
- 중복 기준: 동일 회원 + 동일 날짜 (하루 중복 방지) 또는 30분 이내
- 강제 등록 시 `force: true`, `force_reason: 'manual_override'` 기록
- 강제 등록 이력: attendance 테이블 별도 컬럼으로 추적

### 에지 케이스
- 강제 등록도 실패: toast.error + 경고 UI 유지 + isSubmitting=false
- 중복 + 회원권 만료 동시: 중복 경고 우선 표시
