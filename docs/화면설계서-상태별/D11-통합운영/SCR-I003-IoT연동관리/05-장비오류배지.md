## 📝 디스크립션

### 사용 시점 (Trigger)
- 30초 폴링에서 `status IN ('error', 'offline')` 감지
- WebSocket 이벤트로 장비 상태 변경 수신

### 필수 데이터
| 블록 | 조건 |
|------|------|
| 오류 장비 | `status IN ('error', 'offline')` |
| 오류 상세 | `error_code`, `error_message`, `last_ping_at` |

### 인터랙션 (User Actions)
1. "재시작" 버튼 → POST /api/iot/restart/:deviceId
2. 오류 카드 클릭 → 상세 오류 정보 표시
3. SCR-I001 연동: IoT 배지 → 이 화면으로 랜딩

### 비즈니스 룰
- 오류 장비 재시작 시도 후 3분 대기
- 재시작 후에도 오류: "장비 점검 요청" 버튼 표시
- SCR-I001 IoT 장애 배지와 데이터 공유 (같은 `iot_devices` 테이블)

### 에지 케이스
- 재시작 성공: toast.success + 카드 StatusBadge 갱신
- 재시작 실패: toast.error + 오류 유지

### 연결 화면
- 이전: `02-정상`
- 다음: 오류 해소 → `02-정상` / 전체 다운 → `06-전체오프라인`
