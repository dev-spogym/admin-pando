## 7. 데이터 계약

```ts
interface IoTDevice {
  id: string;
  branchId: number;
  branchName: string;
  type: 'kiosk'|'locker'|'inbody'|'camera'|'sensor';
  name: string;
  status: 'online'|'reconnecting'|'offline';
  lastPing: string;
  firmware?: string;
  envWarn?: 'temp'|'humidity'|null;
  error?: string;
}

interface IoTAlert {
  id: number;
  deviceId: string;
  deviceName: string;
  branchName: string;
  level: 'info'|'warn'|'error';
  message: string;
  createdAt: string;
}
```

### API
| 메서드 | 경로 |
|---|---|
| GET | `/iot/devices?branchId=&type=&status=` |
| GET | `/iot/alerts?limit=20` |
| POST | `/iot/devices/:id/reconnect` |
| DELETE | `/iot/devices/:id` (super) |
| WS | `/ws/iot?branchId=` (실시간 상태 스트림) |

## 8. 비즈니스 룰
1. 장비 `offline` > 5분 → `error` 알림 자동 생성 + 관리자 알림
2. `envWarn` (온도/습도 임계치 초과) → 배지 + 알림
3. 재연결 시도 실패 3회 → "현장 점검 필요" 뱃지
4. 전 장비 offline 감지 → 특수 "전체 오프라인" 상태(06)
5. 필터 URL 동기화 (?branch=&type=&status=)
6. 멀티테넌트: owner는 자기 지점만, super/primary는 전체
