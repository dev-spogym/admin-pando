## 📝 디스크립션

### 사용 시점 (Trigger)
- `expires_at < NOW()` AND `status != 'returned'` 존재
- 만료임박 상태에서 날짜 경과 후

### 필수 데이터
- 회수대상 목록: 만료 경과 + 미반납
- 경과 일수: `differenceInDays(today, expires_at)`

### 인터랙션 (User Actions)
1. "목록 보기" → 필터 'overdue'
2. "반납 처리" → 확인 모달 → PUT /locker/:id/return
3. 반납 완료: 행 회수대상 → 반납완료 스타일 전환

### 비즈니스 룰
- 회수 대상 락커는 신규 배정 불가 (반납 처리 선행 필수)
- 반납 처리 시 회원에게 SMS/앱 알림 (향후)
- 회수대상 배지는 배너 최상단 위치 (만료임박보다 우선)

### 에지 케이스
- 장기 미처리 회수대상: 경과일 증가 표시 → 관리자 주의 유도
- 회수대상 전체 반납: 배지 제거, `02-정상` 복귀

### 연결 화면
- 이전: `04-만료임박배지` (만료 경과)
- 다음: 반납 완료 → `02-정상` / 일부 남음 → 배지 유지
