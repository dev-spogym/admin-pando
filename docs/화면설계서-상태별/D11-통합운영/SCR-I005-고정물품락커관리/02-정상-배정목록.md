## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 + 배정 데이터 존재
- 만료임박/회수대상 없는 정상 상태

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 배정 목록 | `lockers` JOIN `members` | `branch_id=?`, `type='fixed'` |
| 만료임박 | — | `expires_at BETWEEN NOW() AND NOW()+7d` |
| 회수대상 | — | `expires_at < NOW()` AND `status != 'returned'` |

### 인터랙션 (User Actions)
1. "반납" → 반납 확인 모달 → PUT /locker/:id/return
2. "연장" → 연장 날짜 선택 모달 → PUT /locker/:id/extend
3. "새 배정" → 배정 폼 (SCR-I004 DLG-I002 연동)
4. 필터 → 클라이언트 사이드 또는 재조회

### 비즈니스 룰
- 만료일 기준 자동 상태 전환 (DB 또는 클라이언트 계산)
- 회수 대상: 반납 처리 필수 (자동 해제 없음)
- 만료임박 있으면 → `04-만료임박배지` 오버레이
