| 요소 | 변경 |
|---|---|
| 에러 배너 | `bg-red-50 text-red-700 border-red-200` + `role="alert" aria-live="assertive"` |
| 에러 코드 | `<code className="text-xs">E500001</code>` 우측 노출 |
| 저장 버튼 | 텍스트 "재시도", `bg-red-600 hover:bg-red-700` |
| 필드 오류 | `border-red-300` + 인라인 메시지 |
| 409 충돌 시 | 배너 내 "재로드" 버튼 추가 |

## 상태 고유 인터랙션
1. **재시도** → 04-저장중
2. **배너 닫기(X)** → 03-변경됨 (값 유지)
3. **재로드 (409)** → invalidateQueries → 01-로딩 → 02-정상
4. **로그로 이동** (super/primary) → `/audit-log?filter=SETTINGS&branchId=...`
5. **탭 이동** → dirty 가드 유지(DLG-080-001)

## 상태 고유 비즈니스 룰 델타
- 에러 코드별 메시지 매핑 (§00 §10)
- 필드 검증(E900001~E900006) → 필드 포커스 + 인라인
- 서버 에러(E500/E503) → 상단 배너
- 409(E900007) → "재로드" 강조
- Sentry captureException(err, { tags: { screen:'SCR-080', state:'error' } })
- PATCH 실패 → 폼 값 보존 / GET 실패 → EmptyState

## 에지 케이스
- **GET 자체 실패**: TabPanel 대신 `<EmptyState>` + 재시도 버튼만
- **연속 실패 3회**: "지속적 문제" 배너 + 문의 링크
- **부분 성공(로고만 업로드 + PATCH 실패)**: 로고 URL 유지
- **클라이언트 검증 실패**: 05-에러 아님, 03-변경됨에서 인라인만
