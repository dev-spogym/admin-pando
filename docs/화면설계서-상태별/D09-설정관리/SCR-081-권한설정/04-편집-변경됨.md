# SCR-081 권한 설정 — 상태 04: 편집 (변경됨, dirty)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — **델타**만 기술.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `perm-dirty` |
| 이전 | `02-정상-역할선택` (cell 토글), `07-에러` (재시도 값 보존) |
| 다음 | `05-저장중` / `02-정상-역할선택` (초기화) / DLG-081-002 (충돌) |

## 진입 조건
- 최소 1개 cell이 `savedPermissions[roleCode][menuId][action]` 기준값과 다름
- `dirtyCells.size > 0`

## 인터랙션
1. **저장 클릭** → 충돌 검증 → 없으면 `05-저장중`, 있으면 `DLG-081-002`
2. **초기화 클릭** → `DLG-081-001` → savedPermissions 복원
3. **역할 전환 시도** → "다른 역할로 이동하면 변경사항이 사라집니다" 경고
4. **cell 추가 토글** → dirtyCells 갱신
5. **원본값으로 되돌림** → 해당 cell dirty 해제 (Set에서 제거)
6. **Ctrl/Cmd+S** → 저장 트리거

## 비즈니스 룰 델타
- `dirtyCells: Set<string>` 키 = `${menuId}:${action}`
- 실시간 `validateConflicts(draft[roleCode])` 실행 → conflicts 표시
- 저장 가능 조건: `isDirty && !isSaving` (충돌은 경고만, 저장 가능)
- 충돌 무시 저장 시 `DLG-081-002` "무시하고 저장" → PATCH + `AUDIT.PERMISSION_CONFLICT_IGNORED`

## 에지 케이스
- **전 cell 원본 복귀**: dirtyCells 비어짐 → DirtyBadge/저장 버튼 자동 비활성화
- **충돌 해소**: 사용자가 규칙 지켜 수정 → conflicts 자동 사라짐
- **복수 역할 동시 편집 (draft store)**: 역할 전환 시 각 roleCode별 draft 보존 (선택)

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14

상태: perm-dirty

toggle:
function togglePermission(menuId: string, action: PermissionAction) {
  const cur = draft[selectedRoleCode][menuId]?.[action] ?? false;
  const next = !cur;
  setDraft(prev => ({
    ...prev,
    [selectedRoleCode]: {
      ...prev[selectedRoleCode],
      [menuId]: { ...prev[selectedRoleCode]?.[menuId], [action]: next }
    }
  }));
  const key = `${menuId}:${action}`;
  const savedVal = savedPermissions[selectedRoleCode]?.[menuId]?.[action] ?? false;
  setDirtyCells(prev => {
    const s = new Set(prev);
    if (next === savedVal) s.delete(key); else s.add(key);
    return s;
  });
}

cell 변경됨 스타일:
<button role="checkbox" ...
        className={cn(
          'size-8 rounded inline-flex items-center justify-center transition-colors',
          on ? 'bg-blue-50 text-blue-700' : 'bg-white text-gray-300',
          dirtyCells.has(`${m.id}:${action}`) && 'ring-2 ring-amber-500 bg-amber-50'
        )}>

ConflictList:
const conflicts = useMemo(() => validateConflicts(draft[selectedRoleCode]), [draft, selectedRoleCode]);
{conflicts.length > 0 && (
  <div role="alert" aria-live="polite"
       className="rounded-lg bg-amber-50 text-amber-800 border border-amber-200 p-3 space-y-1">
    <div className="font-medium flex items-center gap-1"><AlertTriangle className="size-4" /> 충돌 감지</div>
    <ul className="list-disc list-inside text-sm">
      {conflicts.map((c,i) => (
        <li key={i}>[{MENU_MAP[c.menuId].label}] {c.msg}</li>
      ))}
    </ul>
  </div>
)}

저장 버튼 클릭:
async function handleSave() {
  if (conflicts.length > 0) { setDialog('conflict'); return; }
  setIsSaving(true);
  try { /* 05-저장중 */ } catch { /* 07-에러 */ }
}

역할 전환 가드:
function selectRole(next) {
  if (isDirty) {
    if (!confirm('다른 역할로 이동하면 변경사항이 사라집니다. 계속하시겠습니까?')) return;
    setDraft(prev => ({ ...prev, [selectedRoleCode]: savedPermissions[selectedRoleCode] }));
    setDirtyCells(new Set());
  }
  setSelectedRoleId(next.id);
}

Ctrl/Cmd+S:
useEffect(() => {
  const h = (e: KeyboardEvent) => {
    if ((e.ctrlKey || e.metaKey) && e.key === 's') {
      e.preventDefault();
      if (isDirty && !isSaving) handleSave();
    }
  };
  window.addEventListener('keydown', h);
  return () => window.removeEventListener('keydown', h);
}, [isDirty, isSaving]);
```
