# SCR-081 권한 설정 — 상태 07: 에러

> 상속: [`00-기본화면.md`](./00-기본화면.md) — **델타**만 기술.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `perm-error` |
| 이전 | `01-로딩`(5xx), `05-저장중`(4xx/5xx) |
| 다음 | `01-로딩`(재시도) / `04-편집-변경됨`(배너 닫기) |

## 진입 조건
- GET/PUT 실패 (E401, E403, E901xxx, E500, E503, NETWORK)
- 타임아웃 15초

## 비주얼 델타

| 요소 | 변경 |
|---|---|
| 에러 배너 | `bg-red-50 text-red-700 border-red-200 rounded-lg p-3` + `role="alert"` |
| 에러 코드 | 배너 우측 `<code>E500001</code>` |
| 저장 버튼 | 텍스트 "재시도", `bg-red-600` |
| 매트릭스 | 마지막 draft 유지, 편집 가능 |
| 409 충돌 | 배너에 "재로드" 버튼 추가 |
| GET 실패 | 전체 EmptyState로 치환 |

## 인터랙션
1. **재시도** → 동일 요청 재발사 → `05-저장중`
2. **배너 닫기** → `04-편집-변경됨` (값 보존)
3. **재로드 (409)** → invalidateQueries → `01-로딩` → `02-정상`
4. **로그로 이동** (super/primary) → `/audit-log?filter=PERMISSION`

## 비즈니스 룰 델타
- 에러코드 매핑 (§00 §10):
  - E901001: 충돌 무시 저장했는데 서버 거부 (추가 규칙 위반)
  - E901002: primary 수정 시도 거부
  - E901003: 커스텀 역할 삭제 시 배정 직원 존재 → DLG-087-001 유도
  - E901004: 역할 코드 중복
  - E901005: 역할 이름 중복
- Sentry: `captureException(err, { tags:{screen:'SCR-081'} })`
- PUT 실패 시 draft 값 보존

## 에지 케이스
- **GET 실패 (권한/네트워크/기타)**: 매트릭스 대신 `<EmptyState icon={AlertTriangle} title="권한 데이터를 불러올 수 없습니다" action={재시도} />`
- **연속 3회 실패**: "지속적 문제" 배너 + 지원 연락처
- **부분 저장 성공**: 일부 cell만 서버 반영 → "일부 저장됨" 경고 + 서버 값으로 동기화
