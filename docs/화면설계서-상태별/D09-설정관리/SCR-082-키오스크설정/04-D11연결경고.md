## 진입 조건
- `d11Status === 'disconnected'` AND `cfg.integration.d11Enabled === true`
- WebSocket `onclose` 또는 polling 3회 연속 실패

## 인터랙션
1. **재연결 클릭** → `POST /iot/d11/reconnect` → 성공 시 배너 제거
2. **자동 재연결** → WebSocket `onopen` 수신 → 배너 제거
3. **30초마다 polling fallback** → 3회 성공 시 'connected'
4. **배너 닫기(X)** — 없음 (상태 복구 전까진 표시 유지)

## 비즈니스 룰 델타
- 저장 자체는 허용 (IoT 연결과 무관)
- 실시간 원격 제어만 비활성 (토글 상태는 유지하되 툴팁 "연결 복구 후 적용")
- D11 끊김 상태 지속 시간 10분 이상 → 이메일 알림 (owner/super)
- 마지막 통신 시각 표시: `relative(lastHeartbeat)` (KST)

## 에지 케이스
- **feature flag off**: `integration.d11Enabled === false`면 배너 미표시
- **captive portal 유사**: WebSocket OPEN이나 heartbeat 실패 → disconnected 간주
- **재연결 타임아웃(10초)**: "연결 실패. IoT 허브를 확인하세요" 추가 메시지

```
async function retryD11() {
  setRetrying(true);
  try {
    const ctrl = new AbortController();
    const t = setTimeout(() => ctrl.abort(), 10_000);
    await fetch('/api/iot/d11/reconnect', { method: 'POST', signal: ctrl.signal });
    clearTimeout(t);
    // WebSocket이 곧 onopen 발사해 상태 자동 갱신
  } catch {
    toast.error('재연결 실패. IoT 허브를 확인하세요.');
  } finally {
    setRetrying(false);
  }
}

연동 섹션 내 배지:
<StatusBadge status={d11Status}>
  {d11Status === 'connected' ? '연결됨' : '연결 끊김'}
</StatusBadge>

프리뷰 오버레이:
{d11Status === 'disconnected' && (
  <div className="absolute inset-0 bg-black/40 flex items-center justify-center text-white text-sm">
    <WifiOff className="size-4 mr-1" /> 단말 연결 끊김
  </div>
)}
```
