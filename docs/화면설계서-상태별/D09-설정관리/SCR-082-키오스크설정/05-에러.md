# SCR-082 키오스크 설정 — 상태 05: 에러

> 상속: [`00-기본화면.md`](./00-기본화면.md) — **델타**만 기술.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `kiosk-error` |
| 이전 | `01-로딩`(5xx), `03-변경됨`(저장 실패) |
| 다음 | `01-로딩`(재시도) / `03-변경됨`(닫기, 값 보존) |

## 진입 조건
- GET/PATCH 4xx/5xx, 타임아웃, 네트워크 실패

## 비주얼 델타
| 요소 | 변경 |
|---|---|
| 에러 배너 | `bg-red-50 text-red-700 border-red-200` + role="alert" |
| 에러 코드 | 우측 `<code>` 태그 |
| GET 실패 | 전체 EmptyState (6섹션 대체) |
| PATCH 실패 | 폼 값 보존 + 상단 배너 |
| 저장 버튼 | "재시도", `bg-red-600` |

## 인터랙션
1. 재시도 → 04-저장중 (PATCH) 또는 01-로딩 (GET)
2. 배너 닫기 → 값 보존
3. 로그로 이동 (super/primary)

## 비즈니스 룰 델타
- E900005 로고 용량 초과 → LogoUploader 인라인 에러
- E900006 파일 형식 → LogoUploader 인라인 에러
- E910001 D11 연결 실패 (저장 직후 push 실패)
- Sentry 기록

## 에지 케이스
- GET 실패 시 `<EmptyState>` + 재시도
- WebSocket 장기 단절 시 polling으로 fallback
