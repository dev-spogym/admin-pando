## 상태 진입 조건
- API `200 OK` + `response.items.length === 0`
- 또는 공개 상품이 전혀 없는 신규 지점의 최초 미리보기

## 역할별 차이
- owner/manager/primary/superAdmin: "카탈로그 설정"(→ DLG-P010) + "상품 관리로"(`/products` 이동) 2개 CTA
- fc/trainer/staff/front/readonly: CTA 숨김, "관리자에게 문의하세요" 헬퍼 텍스트만 노출

## 상태 고유 인터랙션
- "카탈로그 설정" → 현재 모달 닫기 + DLG-P010 오픈
- "상품 관리로" → 현재 모달 닫기 + `router.push('/products')`
- "다시 로드" 버튼 → `01-로딩` 재전이

## 비즈니스 룰 델타
- 빈 상태에서도 디바이스 전환 가능 (UI 레이아웃 점검 목적)
- 공개 상품 발생 시 실시간 반영: DLG-P011 저장 후 자동 invalidate → 로딩 경유 후 `02-시뮬표시`
- 토스트는 띄우지 않음 (UX 노이즈 방지)

## 에지 케이스
- 지점 전환 후 해당 지점에 공개 상품 0 → `03-공개상품없음` 유지 (상태 초기화 없이 지점 배지만 갱신)
- 카탈로그 설정은 있으나 상품 0 → 동일 상태
- 상품은 있으나 `catalogVisible=false` 전부 → 동일 상태

```
━━ 델타: EmptyCatalogState ━━
import { BookOpen } from 'lucide-react';

export function EmptyCatalogState({ canManage, onOpenSettings, onGoProducts }: {
  canManage: boolean;
  onOpenSettings: () => void;
  onGoProducts: () => void;
}) {
  return (
    <div className="flex flex-col items-center justify-center text-center px-6 py-12 max-w-sm">
      <BookOpen className="size-12 text-gray-300" aria-hidden />
      <h3 className="mt-4 text-base font-semibold text-gray-700">공개된 상품이 없습니다.</h3>
      <p className="mt-2 text-sm text-gray-500 leading-relaxed">
        카탈로그 편집에서 "공개" 상태를 활성화하거나<br />
        상품 관리에서 상품을 먼저 등록하세요.
      </p>
      {canManage ? (
        <div className="mt-5 flex items-center gap-2">
          <button onClick={onOpenSettings}
            className="h-10 px-4 rounded-lg border border-gray-300 text-gray-700 text-sm font-medium hover:bg-gray-50">
            카탈로그 설정
          </button>
          <button onClick={onGoProducts}
            className="h-10 px-4 rounded-lg bg-blue-600 text-white text-sm font-medium hover:bg-blue-700
                       focus:outline-none focus:ring-2 focus:ring-offset-2 focus:ring-blue-500">
            상품 관리로
          </button>
        </div>
      ) : (
        <p className="mt-5 text-xs text-gray-400">관리자에게 문의하세요.</p>
      )}
    </div>
  );
}

━━ 조건부 렌더 ━━
const canManage = ['superAdmin','primary','owner','manager'].includes(role ?? '');
```
