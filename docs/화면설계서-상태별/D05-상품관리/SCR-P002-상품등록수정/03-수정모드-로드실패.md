# SCR-P002 — 상태: 03-수정모드-로드실패 (edit-load-fail)

> 마스터 `00-기본화면.md` 델타.

## 메타

| 항목 | 값 |
|---|---|
| 화면 ID | SCR-P002 |
| 상태 코드 | `edit-load-fail` |
| 다이어그램 노드 | F6 → `STATE_EDIT_LOAD_FAIL` / F8 → `ERR_404_PRODUCT` |
| 경로 | `/products/edit?id=N` |
| 역할 | superAdmin, primary, owner, manager |
| 이전 상태 | `loading` (진입 직후) |
| 다음 상태 | 재시도 → `02-수정모드` / 목록으로 → `/products` |

## 상태 진입 조건

- `/products/edit?id=N` 진입 → `GET /products/:id` 응답 **404 / 500 / NETWORK**
- 세션 만료(401)는 전역 인터셉터가 `/login`으로 이동시켜 이 상태에 도달하지 않음
- 403은 페이지 진입 전 서버 가드에서 차단

## 에러 코드 매핑 (델타)

| errorCode | 제목 | 본문 | 재시도 |
|---|---|---|---|
| E404301 | 상품을 찾을 수 없습니다 | 삭제되었거나 접근 권한이 없을 수 있습니다 | ✗ (목록으로만) |
| E500001 | 상품 정보를 불러오지 못했습니다 | 일시적인 오류가 발생했습니다 | ● |
| NETWORK | 네트워크 연결을 확인해주세요 | 연결 확인 후 다시 시도하세요 | ● |

## 상태 고유 인터랙션

- `[다시 시도]` → `loadQ.refetch()`, 성공 시 `02-수정모드` 전환
- `[목록으로]` → `moveToPage('/products')`
- Esc 키 → `[목록으로]`와 동일

## 에지 케이스

- 재시도 중 다시 실패 → 같은 상태 유지 + toast "다시 시도해주세요"
- 404 시 URL 교체 없이 현재 URL 유지 (사용자 뒤로가기 정상 동작 보장)

## 바이브코딩 프롬프트 (델타)

```tsx
if (loadQ.isError || loadQ.data === null) {
  const code = loadQ.error?.errorCode;
  const is404 = code === 'E404301';
  return (
    <AppLayout role={user.role}>
      <main className="min-h-screen bg-gray-50">
        <div className="h-[3px] w-full bg-blue-600" aria-hidden />
        <header className="h-14 px-6 border-b bg-white flex items-center justify-between">
          <h1 className="text-xl font-semibold">상품 수정</h1>
          <button aria-label="닫기" onClick={() => moveToPage('/products')}>
            <X className="size-5"/>
          </button>
        </header>
        <div className="flex justify-center mt-24">
          <div role="alert" aria-live="polite"
               className="max-w-md w-full bg-white rounded-xl ring-1 ring-gray-200 p-8 text-center">
            {is404 ? <PackageX className="mx-auto size-10 text-amber-500" />
                   : <AlertTriangle className="mx-auto size-10 text-red-500" />}
            <h2 className="mt-4 text-lg font-semibold text-gray-900">
              {is404 ? '상품을 찾을 수 없습니다'
                     : '상품 정보를 불러오지 못했습니다'}
            </h2>
            <p className="mt-2 text-sm text-gray-600">
              {is404 ? '삭제되었거나 접근 권한이 없을 수 있습니다.'
                     : '일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요.'}
            </p>
            <div className="mt-6 flex gap-2 justify-center">
              {!is404 && (
                <button onClick={() => loadQ.refetch()}
                        className="h-10 px-4 rounded-lg border border-gray-300 bg-white hover:bg-gray-50 text-sm">
                  다시 시도
                </button>
              )}
              <button onClick={() => moveToPage('/products')}
                      className="h-10 px-4 rounded-lg bg-blue-600 hover:bg-blue-700 text-white text-sm font-medium">
                목록으로
              </button>
            </div>
          </div>
        </div>
      </main>
    </AppLayout>
  );
}
```
