## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-P001 DataTable 수정 버튼 클릭 → `/products/edit?id=N`
- `editId` URL 파라미터 존재 시

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 기존값 | `products` | `id = editId` |
| 카테고리 목록 | `product_groups` | `branch_id = ?` |

### 인터랙션 (User Actions)
1. 기존값 수정 후 "저장" → `04-저장중`
2. 취소 → 변경 감지 후 DLG-P002

### 비즈니스 룰
- 기존 상품 데이터 프리필 후 편집
- UPDATE API 호출 (INSERT 아님)

### 에지 케이스
- editId의 상품이 삭제된 경우: `03-수정모드-로드실패`
- 다른 관리자가 동시 수정: 저장 시 충돌 처리

### 연결 화면
- 저장 성공: SCR-P001 목록
- 로드 실패: `03-수정모드-로드실패`
