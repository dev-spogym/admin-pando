## 상태 고유 인터랙션

- submit 실패 즉시 첫 에러 필드로 스크롤 + 포커스 (`scrollIntoView({ block:'center' })`)
- 사용자가 에러 필드 입력 시작 → `form.clearErrors('name')` (react-hook-form 기본 동작)
- 모든 에러 해소 시 ValidationBanner 자동 숨김
- `신규` 버튼은 여전히 DLG-P002 트리거 (dirty 상태 유지)
- ValidationBanner 항목 클릭 → 해당 필드 스크롤 + 포커스

## 상태 고유 비즈니스 룰 (델타)

1. 인라인 에러 + 상단 배너 **이중 표시** (접근성 + UX)
2. 서버 에러 주입 시 해당 필드 `border-red-300`만 적용, 다른 필드는 기존 상태 유지
3. `priceCard < priceCash * 0.5` 같은 cross-field 경고는 warning(amber)로 별도 표시 (에러 아님)
4. 포커스 링 색상은 에러 필드일 때 `ring-red-500`, 아니면 기본 `ring-blue-500`

## 에지 케이스

- 모든 에러 해소 후 제출 → 재검증 통과 시 `04-저장중`으로 전환
- 클라 검증 통과했으나 서버에서 추가 검증 실패 → 이 상태 재진입
- 빠른 Tab 이동으로 연쇄 에러 발생 → 모든 에러 한 번에 배너에 누적 표시

## 바이브코딩 프롬프트 (델타)

```tsx
<input id="name" {...register('name')}
       aria-invalid={!!errors.name}
       aria-describedby={errors.name ? 'err-name' : undefined}
       className={cn(
         'h-10 w-full rounded-lg border px-3 text-sm transition-colors',
         errors.name
           ? 'border-red-300 bg-red-50 focus:ring-2 focus:ring-red-500 focus:border-red-500'
           : 'border-gray-300 focus:ring-2 focus:ring-blue-500 focus:border-blue-500'
       )} />
{errors.name && (
  <p id="err-name" role="alert" aria-live="polite"
     className="text-xs text-red-600 mt-1">
    {errors.name.message}
  </p>
)}

{Object.keys(errors).length > 0 && (
  <div role="alert" aria-live="polite"
       className="rounded-lg border border-red-200 bg-red-50 p-3 text-sm text-red-700">
    <div className="flex items-center gap-2 font-semibold mb-1">
      <AlertCircle className="size-4" /> 입력 오류를 확인해주세요
    </div>
    <ul className="list-disc pl-5 space-y-0.5">
      {flattenErrors(errors).map(({ path, message }) => (
        <li key={path}>
          <button type="button" onClick={() => focusField(path)}
                  className="underline hover:no-underline">
            {message}
          </button>
        </li>
      ))}
    </ul>
  </div>
)}

const onInvalid = (errors) => {
  const firstKey = flattenErrors(errors)[0]?.path ?? Object.keys(errors)[0];
  setTimeout(() => document.getElementById(firstKey)?.focus(), 0);
};
<form onSubmit={handleSubmit(onSubmit, onInvalid)}>
```
