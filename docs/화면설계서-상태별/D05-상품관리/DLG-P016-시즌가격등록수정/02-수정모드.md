## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-P008 시즌가격 목록에서 특정 행의 "수정" 버튼 클릭
- 기존 값이 폼에 프리필된 상태로 오픈

### 레이아웃 구성
- 모달 너비: 520px (고정)
- 헤더: "시즌가격 수정" 타이틀 + X 닫기 버튼
- 본문: 기존값 프리필된 4개 필드
- 하단 액션: 취소(secondary) / 저장(primary)

### 필드 정의

| 필드 | 타입 | 필수 | 기본값 |
|------|------|------|--------|
| 시즌명 | text input | Y | policy.name |
| 적용 기간 | DateRangePicker | Y | policy.start_date ~ policy.end_date |
| 적용 단가 | number input | Y | policy.unit_price |
| 적용 상품 | MultiSelect | Y | policy.product_ids |

### 인터랙션 (User Actions)
1. 필드 수정 → 실시간 유효성 검사
2. "저장" 클릭 → `seasonal_prices` UPDATE → `03-저장결과`
3. "취소" / X 클릭 → 모달 닫기 (변경사항 폐기)

### 비즈니스 룰
- 기간 수정 시 다른 시즌과 충돌 여부 재검사
- 활성 중인 시즌가격 수정 시 즉시 반영
- owner, manager만 접근 가능

### 연결 화면
- 저장 성공: `03-저장결과` (success)
- 기간 충돌: `03-저장결과` (conflict)
- 서버 오류: `03-저장결과` (error)
- 취소: SCR-P008 복귀
