## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S001 매출현황 — 상태: FC/프론트 역할 제한 (RBAC)

파일: src/app/sales/page.tsx (role === 'fc' || role === 'staff' 분기)

RBAC 처리:
const isFC = ['fc', 'staff'].includes(user?.role ?? '');

FC 제한 사항:
1. PageHeader 버튼:
   - [신규 결제 POS]: 활성 (primary)
   - [엑셀 다운로드]: hidden 또는 disabled + tooltip "권한 없음"
2. DataTable 행 클릭:
   - 허용 → DLG-S001 열림 (읽기 전용)
3. DLG-S001 내:
   - [결제 취소] 버튼: hidden
   - [메모 편집] 버튼: hidden
4. 탭:
   - TAB-006 환불내역: hidden
   - TAB-007 미납내역: hidden

안내 배너:
{isFC && (
  <div className="flex items-center gap-2 bg-yellow-50 border border-yellow-200 rounded-lg px-4 py-2 mb-4">
    <ShieldOff className="h-4 w-4 text-yellow-600 flex-shrink-0" />
    <p className="text-sm text-yellow-700">
      열람 권한으로 접속 중입니다. 일부 기능이 제한됩니다.
    </p>
  </div>
)}

사용 유틸:
- useAuthStore().user.role
- hasPermission('sales:export') — false → 엑셀 숨김
- ShieldOff from 'lucide-react'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- user.role === 'fc' 또는 'staff' 로 SCR-S001 진입
- E_STATE_RBAC_02 엣지 발생 시

### 필수 데이터
| 항목 | 테이블 | 비고 |
|------|--------|------|
| 매출 목록 | `sales` | 전체 조회 (지점 내) |

### 인터랙션 (User Actions)
1. [신규 결제 POS] → `/pos` (허용)
2. 행 클릭 → DLG-S001 읽기 전용
3. 엑셀 다운로드: 불가 (숨김 또는 disabled)

### 비즈니스 룰
- fc 역할: 매출 조회 O, 수정/취소/환불 X
- StatCard 모두 표시 (읽기는 허용)
- 탭 TAB-006, TAB-007 숨김 (환불/미납 민감 정보)

### 에지 케이스
- fc가 직접 URL로 `/sales?tab=refunds` 접근: 쿼리 무시 또는 빈 탭 표시

### 접근성 (A11y)
- 배너 `role="note"` + `aria-label="제한된 접근 권한 안내"`
- 숨긴 버튼은 DOM에서 완전 제거 (aria-hidden 아닌 display:none)

### 연결 화면
- 이전: `01-로딩`
- 다음: `02-정상` (제한 UI 적용 상태)
