## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S001 매출현황 — 상태: 트레이너 자동 필터 (RBAC)

파일: src/app/sales/page.tsx (role === 'trainer' 분기)

RBAC 처리:
const { user } = useAuthStore();
const isTrainer = user?.role === 'trainer';

// 트레이너: 담당자 필터 자동 적용
const query = supabase.from('sales').select('*').eq('branch_id', branchId);
if (isTrainer) {
  query.eq('manager_id', user.id);
}

안내 배너 (DataTable 위, 트레이너 전용):
{isTrainer && (
  <div className="flex items-center gap-2 bg-blue-50 border border-blue-200 rounded-lg px-4 py-2 mb-4">
    <Info className="h-4 w-4 text-blue-500 flex-shrink-0" />
    <p className="text-sm text-blue-700">
      담당자 필터가 자동 적용되어 본인 담당 매출만 표시됩니다.
    </p>
  </div>
)}

트레이너 제한 사항:
- 담당자 필터 Select: disabled (본인으로 고정)
- 엑셀 다운로드: 허용 (본인 데이터만)
- POS 버튼: 허용
- 전체 통계(StatCard): 본인 담당 합계만 표시

사용 유틸:
- useAuthStore from '@/store/authStore'
- Info from 'lucide-react'
- hasPermission 체크 (role 기반)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- user.role === 'trainer' 로 로그인 후 SCR-S001 진입
- E_STATE_RBAC_01 엣지 발생 시

### 필수 데이터
| 항목 | 테이블 | 조건 |
|------|--------|------|
| 매출 목록 | `sales` | branch_id + manager_id = 본인 |

### 인터랙션 (User Actions)
1. 담당자 필터: disabled (변경 불가)
2. 탭 전환: 가능 (전체 탭)
3. 날짜 범위 변경: 가능

### 비즈니스 룰
- 트레이너는 타 담당자 매출 조회 불가 (RLS 레벨에서도 차단)
- StatCard 금액: 본인 담당 합계
- 안내 배너는 항상 표시 (닫기 불가)

### 에지 케이스
- 트레이너 본인 담당 건 없음: `03-빈상태` + 배너 유지

### 접근성 (A11y)
- 배너 `role="note"` + `aria-label="트레이너 권한 안내"`
- 비활성 필터 `aria-disabled="true"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `02-정상` (담당 데이터 표시) 또는 `03-빈상태`
