## 📝 디스크립션

### 사용 시점 (Trigger)
- Supabase API 500/503 오류
- 네트워크 오프라인 (E_STATE_OFFLINE_01)
- JWT 만료 후 재로그인 실패

### 필수 데이터
| 항목 | 설명 |
|------|------|
| error 객체 | console.error 로깅용 |

### 인터랙션 (User Actions)
1. [다시 시도] 버튼 → fetchSales() 재호출 → `01-로딩`
2. 오프라인 복구 → 자동 재시도 없음 (수동 클릭 필요)

### 비즈니스 룰
- 3회 연속 실패: "고객센터 문의" 링크 노출 (향후)
- JWT 만료(401): `/login` 자동 리다이렉트
- toast.error는 중복 발사 방지 (hasToasted ref 사용)

### 에지 케이스
- 오프라인 상태: 동일 에러 UI + "네트워크 연결을 확인해주세요" 메시지 변경
- 부분 실패(통계만 실패): StatCard 0 표시, DataTable 정상

### 접근성 (A11y)
- 에러 메시지 `role="alert"`
- 재시도 버튼 `aria-label="매출 데이터 다시 불러오기"`

### 연결 화면
- 이전: `01-로딩`
- 다음: 재시도 성공 → `02-정상` / 인증 실패 → `/login`
