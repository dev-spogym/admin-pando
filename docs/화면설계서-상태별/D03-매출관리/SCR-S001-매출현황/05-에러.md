## 상태 진입 조건
- `useSales` 훅의 `error !== null`
- HTTP 500/502/503/504, 네트워크 단절, Supabase Postgrest 오류
- `error.code` 에 따라 하위 분기:
  - `E500001` 일반 서버 오류
  - `E503001` 결제 단말기 연결 실패
  - `E503002` PG사 연동 오류 (경고 톤)
  - `NETWORK` 오프라인 (navigator.onLine=false)
  - `401 만료` 인터셉터 → `/login` (본 상태 미노출)
  - `403 권한` → `/forbidden` (본 상태 미노출)

## 상태 고유 인터랙션
- 진입 직후 `toast.error('매출 데이터를 불러오지 못했습니다.')` 1회 (`hasToasted` ref 중복 방지)
- [다시 시도] 클릭 → `refetch()` → `01-로딩`
- [홈으로] 클릭 → `moveToPage('/')`
- `NETWORK` 상태에서 온라인 복구 감지(`window.addEventListener('online')`) 시 자동 refetch
- ErrorBanner는 `onClose` 지원 (X 버튼). 닫아도 본문 에러 일러스트는 유지

## 상태 고유 비즈니스 룰 델타
- **캐시 fallback**: 이전 `useSales` 결과 캐시가 있으면 grayscale로 희미하게 노출 + "최신 데이터 불러오기 실패 (캐시 표시)" 배너
- 재시도 상한: 연속 실패 3회 시 자동 재시도 중단 + 사용자에게 수동 재시도 안내 ("고객센터 문의" 링크는 Phase 2)
- 에러 발생 시 `console.error('[SCR-S001] 매출 데이터 로드 실패', error)` + 로그 수집 (Sentry 등)
- 엑셀 다운로드 기능: 캐시가 있어도 비활성 (stale 방지)
- 부분 실패(통계만 실패): StatCard `—` 표시, DataTable 정상 (Phase 2; 현재 단일 쿼리 실패면 전체 에러)

## 에지 케이스
- 503 + 2초 내 201 성공: 첫 에러 토스트는 취소 (`toast.dismiss`)
- 필터 변경으로 refetch 중 에러: 이전 성공 데이터 유지 + 상단 경고 배너만
- 5분 이상 체류 시 자동 주기 재시도 유지 (백오프: 5s → 15s → 60s)
- HTTP 418 등 비표준 코드: `errorCode=UNKNOWN` 처리, 기본 메시지
- Supabase RLS 거부: 403으로 매핑 → `/forbidden`

```

```
