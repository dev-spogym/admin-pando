## 📝 디스크립션

### 사용 시점 (Trigger)
- isLoading=false AND sales.length > 0
- 필터 변경 후 데이터 재로드 완료 시

### 필수 데이터
| 항목 | 테이블 | 조건 |
|------|--------|------|
| 매출 목록 | `sales` | branch_id, 날짜 범위 |
| 요약 통계 | `sales` | SUM(sale_price), SUM(card), SUM(cash), SUM(unpaid) |

### 인터랙션 (User Actions)
1. TAB-001~007 탭 클릭 → 해당 탭 데이터 표시
2. 행 클릭 → DLG-S001 매출상세 모달
3. [신규 결제 POS] → `/pos`
4. [엑셀 다운로드] → exportToExcel() → 성공/실패 토스트
5. 날짜 프리셋 변경 → 데이터 재조회

### 비즈니스 룰
- trainer 역할: 담당자 필터 자동 적용 (본인 데이터만)
- fc/staff 역할: POS 버튼만 활성, 엑셀 다운로드 비활성
- 미수금 > 0: StatCard 빨간색 강조
- 엑셀 최대 10,000건

### 에지 케이스
- 탭 전환 시 해당 탭 데이터 0건 → `04-탭-빈상태`
- 엑셀 0건: "다운로드할 데이터가 없습니다" 토스트

### 접근성 (A11y)
- DataTable `role="grid"`, 각 행 `role="row"`
- 행 클릭 `aria-label="매출 상세 보기: {상품명}"`
- StatCard `aria-label="{카드명}: {금액}"`

### 연결 화면
- DLG-S001 매출상세 (행 클릭)
- SCR-S002 POS판매 (신규 결제 버튼)
- SCR-S012 결제취소/부분환불 (DLG-S001 내 버튼)
