## 상태 진입 조건
- API 응답 성공 + `data.length === 0` (신규 지점 또는 해당 기간에 매출 없음)
- 즉, **모든 탭·모든 필터에서 0건**

> 차이: 특정 탭만 0건이면 `04-탭-빈상태` 적용. 본 상태는 **전체 0건**에서만 표시.

## 역할별 차이
- **fc**: [신규 결제 POS] primary 버튼 활성, [기간 변경] 도 활성
- **trainer**: 기본적으로 `06-트레이너-제한` 으로 분기. 본 상태 노출 시 [POS] 버튼 숨김, [기간 변경]만 활성, 안내 문구: "담당 매출 데이터가 없습니다."

## 상태 고유 인터랙션
- [기간 변경] 클릭 → SearchFilter 날짜 범위 input에 포커스
- [신규 결제 (POS)] 클릭 → `moveToPage(982)`
- 탭 전환 가능하나 모든 탭이 동일 0건 (클릭해도 빈 일러스트 유지)
- 필터 조정 후 재조회 시 `01-로딩` → `02-정상`/`03-빈상태`/`05-에러` 로 전이

## 상태 고유 비즈니스 룰 델타
- 엑셀 다운로드 버튼 disabled (클릭 시 미호출)
- 요약바의 모든 합계는 `₩0` 렌더 (tabular-nums 유지로 정렬)
- 매출유형 필터 버튼은 렌더하되 클릭 시 아무 효과 없음 (데이터 없음)
- 진입 시 조용히(`role="status"`) 안내만, 오류성 배너 금지

## 에지 케이스
- 기간 필터가 미래 날짜라서 0건인 경우: 일러스트 설명에 `"설정한 기간이 미래입니다"` 서브텍스트
- 삭제된 지점의 자료로 0건: 서버가 먼저 403 처리 → 본 상태 노출 X
- 서버가 `null` 응답: `data ?? []` 로 방어, 본 상태 진입
- 매우 오래된 기간(수년 전)으로 0건: 기본 메시지 유지
