## 3. 상태 고유 비주얼 델타

### 3.1 PENDING 상태 vs ISSUED/CANCELLED 분기

```
PENDING 분기:
  Footer: [ 닫기 ]  [ 발행(primary) ]
  상태 배지: [대기](warning, amber)
  발행일: "-"

ISSUED 분기:
  Footer: [ 닫기(전체폭) ]                 ← 발행 버튼 숨김
  상태 배지: [발행완료](success, emerald)
  발행일: 2026-04-18

CANCELLED 분기:
  Footer: [ 닫기(전체폭) ]                 ← 발행 버튼 숨김
  상태 배지: [취소](error, rose)
  발행일: 2026-04-18 (취소 시점 표시 옵션)
```

### 3.2 변경 요소 표

| 요소 | 값 | 비고 |
|---|---|---|
| 본문 | `DetailView(taxInvoice)` | InfoRow × 8 + ItemsList |
| 합계 강조 | `text-lg font-bold tabular-nums` | 총액 |
| 상태 배지 | 동적 | INV_STATUS_KO / VARIANT |
| [닫기] | 항상 활성 | `flex-1` 또는 `w-full` |
| [발행] | `canIssue(taxInvoice, role)` | PENDING + 권한 시에만 |
| `showIssueModal` | 기본 `false` | [발행] 클릭 시 `true` |
| ESC/배경/X | 즉시 닫힘 허용 | 조회 전용 |

## 5. 상태 고유 인터랙션

| 이벤트 | 트리거 | 결과 |
|---|---|---|
| `E_CLICK_ISSUE` | [발행] 클릭(PENDING+권한) | `setShowIssueModal(true)` → DLG-S011 오픈(중첩) |
| `E_CLOSE` | [닫기]/ESC/배경/X | `onClose()` |

| `E_DLG_S011_CANCEL` | DLG-S011 [취소] | `setShowIssueModal(false)` (본 모달 유지) |

## 6. 상태 고유 비즈니스 룰 델타

1. **세액 자동 검산**: `Math.abs(supply*0.1 - tax) > 1` 이면 상단 amber 경고 배너 표시(옵션).
2. **발행 후 refetch**: DLG-S011 발행 성공 → `onIssued` 콜백에서 본 모달은 detail refetch. 닫히지 않음.
3. **ISSUED 후 [발행] 숨김**: status 변화 시 즉시 Footer 재렌더.
4. **품목 합계 검증**: `Σ items.amount === supply_amount` UI 경고(옵션).
5. **조회 전용**: 모든 input 없음. 복사 가능 텍스트만.
6. **외부 링크(Phase 2)**: 발행된 세금계산서의 국세청 확인번호 링크(`https://www.hometax.go.kr/...`).

---

## 7. 에지 케이스

| 케이스 | 처리 |
|---|---|
| `items` 배열 비어있음 | "품목 없음" 안내 텍스트 렌더 |
| `items` 100건 이상 | `max-h-[200px] overflow-auto`, 상단 "총 N건" |
| `issued_at` 타임존 | 서버 UTC → 클라이언트 로컬 `toLocaleDateString('ko-KR')` |
| 사업자번호가 10자리가 아님 | raw 그대로 표시, 포맷 함수는 길이 체크 |
| PENDING 인데 발행 불가 역할 | [발행] 미렌더, [닫기] 전체폭 |
| DLG-S011 오픈 중 본 모달 ESC | 본 모달은 닫히지 않음(중첩 최상위만 ESC 반응) |

## 8. 바이브코딩 프롬프트 (델타)

```
상태: DLG-S010 — 02-열림

const inv = taxInvoice!;
const issuable = inv.status === 'PENDING'
  && ['superAdmin','primary','owner','manager'].includes(role);

<>
  <section className="p-6 space-y-3 border-b">
    <InfoRow label="발행 번호" value={inv.invoice_number} mono />
    <InfoRow label="거래처명" value={inv.company_name} />
    <InfoRow label="사업자번호" value={formatBizNumber(inv.business_number)} mono />
    <hr className="border-gray-100" />
    <InfoRow label="공급가액" value={formatKRW(inv.supply_amount)} amount />
    <InfoRow label="세액" value={formatKRW(inv.tax_amount)} amount />
    <InfoRow label="합계"
      value={formatKRW(inv.total_amount)}
      valueClassName="text-lg font-bold text-gray-900 tabular-nums" />
    <hr className="border-gray-100" />
    <InfoRow label="발행일"
      value={inv.issued_at ? new Date(inv.issued_at).toLocaleDateString('ko-KR') : '-'} />
    <InfoRow label="상태"
      value={<StatusBadge label={INV_STATUS_KO[inv.status]} variant={INV_STATUS_VARIANT[inv.status]} />} />
  </section>

  <section className="p-6">
    <h4 className="text-sm font-semibold mb-3">품목 ({inv.items?.length ?? 0}건)</h4>
    {inv.items?.length ? (
      <ul className="space-y-1 text-sm max-h-[200px] overflow-auto">
        {inv.items.map((it, i) => (
          <li key={i} className="flex justify-between p-2 bg-gray-50 rounded">
            <span>{it.name}</span>
            <span className="tabular-nums">{formatKRW(it.amount)}</span>
          </li>
        ))}
      </ul>
    ) : (
      <p className="text-sm text-gray-400 text-center py-4">품목 없음</p>
    )}
  </section>

  <footer className="p-4 border-t flex gap-3">
    <Button variant="outline" className={issuable ? 'flex-1' : 'w-full'}
            onClick={onClose}>닫기</Button>
    {issuable && (
      <Button variant="primary" className="flex-1"
              onClick={() => setShowIssueModal(true)}>발행</Button>
    )}
  </footer>
</>
```
