## 4. 역할별 차이

- RLS 위반(`42501`) 시 fc 역할에서 가장 흔히 발생 가능(지점 외 회원 시도 등). 이 경우 **자동 닫힘** + 토스트로만 통지.
- owner/manager 는 재시도 메시지 그대로 노출.

## 6. 상태 고유 비즈니스 룰 델타

1. **에러 코드 매핑** (마스터 §10 보강):
   - `23502`(NOT NULL), `23514`(CHECK) → "입력값을 확인해주세요."
   - `23505`(UNIQUE) → "이미 등록된 약정입니다."
   - `42501`(RLS) → "등록 권한이 없습니다." + 자동 닫힘
   - `PGRST*`(RPC) → "서버 통신 오류"
   - `AbortError` → "응답이 지연됩니다. 다시 시도해주세요."
   - 기타 → "할부 등록에 실패했습니다."
2. **입력값 보존**: 모든 필드 상태 유지(초기화 금지). 사용자가 즉시 재시도 가능.
3. **재시도 카운트**: 클라이언트 측 별도 제한 없음. 서버 rate limit(30/min/IP) 에 위임.
4. **권한 에러 처리**: `42501` 수신 즉시 닫힘(재시도해도 무의미).
5. **에러 토스트 단일화**: id `'installment-create-error'` 로 중복 방지.
6. **감사로그**: 실패도 `AUDIT.INSTALLMENT_CREATE.fail`(code 포함) 로 기록.

## 7. 에지 케이스

| 케이스 | 처리 |
|---|---|
| 네트워크 끊김 후 복귀 | 토스트 "네트워크 오류" 노출 후 상태 유지. 사용자가 재시도 |
| 세션 만료(401) | DLG-000 세션만료 우선 오픈, 본 다이얼로그 자동 정리 |
| 브라우저 오프라인 이벤트 | `navigator.onLine === false` 감지 시 [등록] 비활성 + 배너 권장 |
| 동일 오류 반복 3회 | 토스트 누적 방지(id 재사용) |
| 이상 응답(null + null) | 방어적으로 error 로 취급: "알 수 없는 오류" |
| 개월 수 0으로 재설정 후 재시도 | `canSubmit=false` 로 버튼 disabled → 재시도 차단 |

## 8. 바이브코딩 프롬프트 (델타)

```
상태: DLG-S009 — 04-에러

// 에러 분기 (02-처리중 onSubmit 내부)
if (error) {
  const message = mapInstallmentError(error);
  toast.error(message, { id: 'installment-create-error' });
  setIsProcessing(false);   // 재시도 가능 상태
  setLastError(error);

  // 권한 오류 → 자동 닫힘
  if (error.code === '42501') {
    setTimeout(() => onClose(), 200);
  }
  return;
}

function mapInstallmentError(err: any): string {
  switch (err?.code) {
    case '23502':
    case '23514': return '입력값을 확인해주세요.';
    case '23505': return '이미 등록된 약정입니다.';
    case '42501': return '등록 권한이 없습니다.';
    case 'PGRST301': return '세션이 만료되었습니다.';
    default:
      if (err?.name === 'AbortError') return '응답이 지연됩니다. 다시 시도해주세요.';
      if (err?.message?.includes('Failed to fetch')) return '네트워크 오류. 연결을 확인해주세요.';
      return '할부 등록에 실패했습니다.';
  }
}
```
