## 상태 진입 조건
- `paymentMethod === 'mileage'`
- `selectedMember !== null`
- `memberMileage < finalAmount`
- 또는 `paymentMethod === 'mixed' && mixed.mileage > memberMileage`(복합결제 마일리지 초과)

## 역할별 차이
전 역할 동일. trainer/fc도 마일리지 결제는 허용되지만 잔액 부족 시 동일하게 차단.

## 상태 고유 인터랙션
- 결제수단 변경(카드/현금/복합) → `01` 복귀 또는 `02-유효성-통과`
- 회원 변경(DLG-S002) → 새 memberMileage 조회 → 충분하면 `02`
- 마일리지 충전 링크(Phase 2) → `/members/detail?id=` (회원 상세 → 마일리지 충전)
- "부족 N P" 클릭 시 toast.info "마일리지를 충전하거나 다른 결제수단을 이용해주세요."

## 상태 고유 비즈니스 룰 델타
- `isValid = false` 고정
- 결제 확인 버튼 disabled + aria-disabled=true
- 복합결제에서 mileage>0이고 > memberMileage인 경우도 동일 상태로 진입
- 서버 validate 중복 수행(클라 UX + 서버 최종 권위)

## 에지 케이스
- 다른 탭/화면에서 마일리지 차감이 발생해 도중에 잔액 변경 → `useMemberMileage` refetch 트리거(포커스 복귀 시)
- `memberMileage === 0` → 부족 N P = finalAmount 전체
- 회원 해제 후 재선택 시 잔액 충분하면 즉시 `02`
- 마일리지 결제를 복합결제로 전환 → MixedPaymentPanel에서 mileage 입력 시 maxMileage 표시
