## 📝 디스크립션

### 사용 시점 (Trigger)
- sales INSERT 성공 후 paymentStatus='complete'
- E_STATE_COMPLETE_01

### 필수 데이터
| 항목 | 설명 |
|------|------|
| saleResult | INSERT 결과 (sale_id, buyer_name, sale_price, payment_method) |

### 인터랙션 (User Actions)
1. [영수증 출력] → window.print() 또는 영수증 API
2. [문자 발송] → SMS API (buyer 전화번호 필요)
3. [계속 판매] → clearCart() + moveToPage('/pos')
4. [매출 현황으로] → moveToPage('/sales')

### 비즈니스 룰
- 마일리지 적립: 결제금액의 1% (설정값 기반)
- 영수증: 브라우저 프린트 다이얼로그
- 문자 발송: buyer?.phone 없으면 버튼 disabled

### 에지 케이스
- 뒤로가기 시: 완료 화면 유지 (장바구니 이미 초기화)
- 새로고침 시: `/pos`로 리다이렉트 (state 소실)

### 접근성 (A11y)
- 완료 메시지 `role="status"` + `aria-live="polite"`
- 체크 아이콘 `aria-label="결제 완료"`

### 연결 화면
- SCR-S002 POS판매 (계속 판매)
- SCR-S001 매출현황 (매출 현황으로)
