## 역할별 차이
전 역할 동일. trainer도 적립률 동일 1%.

## 상태 고유 인터랙션
- 진입 시 체크 아이콘 → [영수증 출력] 버튼으로 포커스 자동 이동
- [🖨 영수증 출력]: 200ms 딜레이 후 `window.print()` + `toast.info('영수증 인쇄를 시작합니다.')`
- [📱 문자 발송]: `toast.info('문자 영수증을 발송합니다.')` (Phase 2 실 API)
- [🛒 계속 판매하기]: `handleReset()` → 대기 or NoCartState
- [📊 매출 현황으로]: `moveToPage(970)` → `/sales`
- ESC 무시(실수 방지)
- 뒤로가기 → `/pos` 이동하되 sessionStorage 비어있어 빈 장바구니

## 상태 고유 비즈니스 룰 델타
- 새로고침 시 `isComplete` 유지 X → NoCartState
- 영수증 printer-friendly CSS: `@media print { .no-print { display: none } }`
- 회원 전화번호 없으면 문자 발송 disabled + tooltip "회원 전화번호가 없습니다."
- completedSale.id를 hidden으로 유지 → 영수증 서버 재조회 가능
- 마일리지 적립 = 0인 경우 적립 섹션 자체 렌더 X (마일리지 전액 결제)

## 에지 케이스
- 프린터 없음 → `window.print()` 실행되지만 PDF 저장 다이얼로그
- 문자 발송 실패(Phase 2) → toast.error "문자 발송에 실패했습니다."
- completedSale=undefined + isComplete=true 방어 → fallback "결제 정보를 가져올 수 없습니다." + [매출 현황으로]만 표시
- 비회원 결제 → 부제 "결제가 정상 처리되었습니다.", 문자 disabled
