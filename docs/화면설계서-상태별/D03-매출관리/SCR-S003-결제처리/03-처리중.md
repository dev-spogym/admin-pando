## 상태 고유 비즈니스 룰 델타
- 최소 500ms 표시(깜빡임 방지, Promise.all+sleep)
- 타임아웃 30초 → 에러 토스트 + isProcessing=false + `02`
- 실패 분기:
  - E402001/002 → toast.error, `02` 유지, 재시도 가능
  - E402003(마일리지 차감 실패) → toast.error "마일리지 차감에 실패했습니다."
  - E503001/002 → amber 배너 + toast.warning
  - E500001 → toast.error "결제 저장에 실패했습니다."
  - NETWORK → toast.error "네트워크를 확인해주세요."
- 성공 → `04-완료`(setIsComplete(true))
- 서버 트랜잭션: 마일리지 차감 성공 + sales.insert 실패 시 서버 rollback 또는 보상 accruePoints

## 에지 케이스
- 동시 클릭/연타 방지: isProcessing=true인 동안 setShowConfirm 무시
- 새로고침 → sessionStorage posCart 복원되나 결제 결과 소실. `/sales`에서 확인 안내
- 탭 전환 → 결제 요청 유지, 완료 시 포커스 복귀 후 `04` 렌더
- 마일리지 차감 완료 + sales.insert 실패 → 서버 보상 실패 시 관리자 수동 개입
