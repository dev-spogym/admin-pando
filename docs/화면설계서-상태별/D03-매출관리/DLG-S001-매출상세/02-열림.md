## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: DLG-S001 매출상세 — 상태: 열림 (상세 데이터 표시)

파일: src/app/sales/page.tsx 내 SaleDetailModal 컴포넌트

모달 크기: 400px × 560px (max-h-[90vh] overflow-y-auto)

모달 구성:
<Modal isOpen={isOpen} onClose={onClose} title="매출 상세">
  <div className="divide-y">
    {/* 기본 정보 섹션 */}
    <div className="p-6 space-y-3">
      <h4 className="text-xs font-semibold text-gray-400 uppercase tracking-wide mb-3">
        기본 정보
      </h4>
      <InfoRow label="결제일" value={sale.purchaseDate} />
      <InfoRow label="구매자" value={sale.buyer} />
      <InfoRow label="상품명" value={sale.productName} />
      <InfoRow label="구분" value={sale.type} />
      <InfoRow label="담당자" value={sale.manager} />
    </div>

    {/* 금액 정보 섹션 */}
    <div className="p-6 space-y-3">
      <h4 className="text-xs font-semibold text-gray-400 uppercase tracking-wide mb-3">
        금액 정보
      </h4>
      <InfoRow label="정가" value={formatKRW(sale.originalPrice)} />
      <InfoRow label="판매가" value={formatKRW(sale.salePrice)} />
      <InfoRow label="할인금액" value={`- ${formatKRW(sale.discountPrice)}`} />
      <InfoRow label="결제수단" value={sale.paymentMethod} />
      <InfoRow label="카드" value={formatKRW(sale.card)} />
      <InfoRow label="현금" value={formatKRW(sale.cash)} />
      <InfoRow label="마일리지" value={`${formatNumber(sale.mileage)}P`} />
      <InfoRow
        label="미수금"
        value={formatKRW(sale.unpaid)}
        valueClassName={sale.unpaid > 0 ? "text-red-600 font-bold" : ""}
      />
    </div>

    {/* 상태 */}
    <div className="p-6">
      <div className="flex items-center justify-between">
        <span className="text-sm text-gray-500">상태</span>
        <StatusBadge
          label={sale.status}
          variant={sale.status === '완료' ? 'success' : sale.status === '환불' ? 'error' : 'warning'}
        />
      </div>
    </div>
  </div>

  {/* 하단 버튼 */}
  <div className="p-4 border-t flex gap-2">
    <Button variant="outline" className="flex-1" onClick={onClose}>
      닫기
    </Button>
    {hasPermission('sales:cancel') && sale.status === '완료' && (
      <Button
        variant="danger"
        className="flex-1"
        onClick={() => moveToPage(`/sales/cancel?saleId=${sale.id}`)}
      >
        결제 취소
      </Button>
    )}
  </div>
</Modal>

InfoRow 컴포넌트:
const InfoRow = ({ label, value, valueClassName }: {...}) => (
  <div className="flex justify-between text-sm">
    <span className="text-gray-500">{label}</span>
    <span className={cn("font-medium text-gray-800", valueClassName)}>{value}</span>
  </div>
);

사용 유틸:
- formatKRW, formatNumber from '@/lib/format'
- StatusBadge, Modal, Button
- hasPermission from '@/lib/auth'
- moveToPage from '@/internal'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- E_LOAD_OK_01: saleDetail 데이터 로드 완료
- 모달 400px × 560px, 오버레이 위

### 필수 데이터
| 항목 | 테이블 | 설명 |
|------|--------|------|
| 매출 상세 | `sales` | id=saleId 단건 |

### 인터랙션 (User Actions)
1. [닫기] → 모달 닫힘
2. ESC / X / 오버레이 클릭 → 모달 닫힘
3. [결제 취소] → SCR-S012 (owner, manager 전용)

### 비즈니스 룰
- [결제 취소] 버튼: hasPermission('sales:cancel') AND status='완료' 인 경우만 표시
- fc/staff 역할: 읽기 전용 (취소 버튼 숨김)
- 미수금 > 0: 빨간 텍스트 강조

### 에지 케이스
- 긴 상품명: truncate 처리
- 모달 내 스크롤: max-h 초과 시 overflow-y-auto

### 접근성 (A11y)
- 모달 `role="dialog"` + `aria-labelledby="modal-title"`
- 닫기 버튼 `aria-label="매출 상세 모달 닫기"`

### 연결 화면
- 닫힘: SCR-S001 유지
- SCR-S012 결제취소/부분환불 ([결제 취소])
