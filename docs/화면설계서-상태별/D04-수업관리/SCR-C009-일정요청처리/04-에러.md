## 상태 진입 조건
- 목록 GET 호출 실패:
  - HTTP 401 → 전역 인터셉터 처리(이 상태 미진입)
  - HTTP 403 → 이 상태(권한 오류 전용 UI)
  - HTTP 5xx / 서버 오류 → 이 상태(서버 오류)
  - 네트워크 오프라인 / 타임아웃 / CORS → 이 상태(`NETWORK`)
- Mutation 실패는 행 단위 토스트로 처리하며 이 상태로 진입하지 않음.

## 상태 고유 인터랙션

| 액션 | 동작 |
|---|---|
| 다시 시도 버튼 | `refetch()` → `01-로딩` 전환 |
| 새로고침 버튼(Header) | 동일 |
| Esc 키 | 없음 (모달 아님) |
| 온라인 복귀 감지 | `window.online` 이벤트 → 자동 `refetch()` 1회 시도 |
| 탭 포커스 복귀 | `refetchOnWindowFocus=true`로 자동 재시도 |

## 상태 고유 비즈니스 룰 델타
1. **exponential backoff**: 재시도 버튼 연타 방지 — 첫 재시도 즉시, 두 번째 2s, 세 번째 5s, 네 번째 10s 이후 disabled(수동 새로고침 유도).
2. **503 자동 폴링**: 30초 주기 3회 자동 재시도 후 멈춤.
3. **에러 로깅**: Sentry `captureException(err, { tags: { screen:'SCR-C009' }, extra: { filter, user.role } })`.
4. **네트워크 복귀 감지**: `navigator.onLine` + `window.addEventListener('online', refetch)`.
5. **세션 만료**: 401은 이 상태에 오지 않고 전역 handler가 `/login?redirect=/schedule-requests`로 이동.
6. **stale 데이터 숨김**: 이전 데이터 `keepPreviousData` 끄고 완전한 에러 패널로 대체(사용자 혼선 방지).

## 에지 케이스

- **로딩 중 네트워크 단절**: 즉시 `04-에러(NETWORK)` 전환, 복귀 시 자동 재시도.
- **무한 503**: 3회 자동 재시도 후 "관리자 문의" 링크만 남김.
- **CORS 오류(개발 환경)**: 같은 NETWORK 코드로 처리, 콘솔에는 상세 로그.
- **IDB 캐시 가능 여부**: Phase 2 — 직전 성공 응답을 읽기 전용 표시하는 폴백(stale-while-error).
- **동일 에러 반복**: 동일 errorCode가 3회 연속이면 Sentry 우선순위 조정.
- **403 임시 처리**: 브랜드 전환/지점 변경 직후 JWT가 갱신되기 전 → 1초 후 1회 자동 재시도.

function mapError(err: unknown): ErrView {
  const code = (err as any)?.response?.data?.errorCode ?? 'NETWORK';
  const status = (err as any)?.response?.status;
  if (status === 403) return { code:'403', icon:'shield-off', title:'접근 권한이 없습니다',
                                sub:'관리자에게 문의해주세요', cta:'대시보드로', href:'/' };
  if (status === 503) return { code:'E503001', icon:'wrench', title:'서비스 점검 중입니다', ... };
  if (!navigator.onLine) return { code:'NETWORK', icon:'wifi-off', title:'네트워크 연결을 확인해주세요', ... };
  if ((err as any).name === 'AbortError') return { code:'TIMEOUT', ... };

const [retryCount, setRetryCount] = useState(0);
const retryDelay = [0, 2000, 5000, 10000][Math.min(retryCount, 3)];
const retryDisabled = retryCount >= 4;
const onRetry = async () => {
