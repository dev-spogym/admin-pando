## 역할별 차이
- 모든 역할 동일. trainer도 본인 건수 스켈레톤이므로 동일 행 수(8) 유지.
- readonly는 애초 접근 차단이므로 이 상태에 진입하지 않음.

## 상태 고유 인터랙션
- 필터 pill / 기간 입력: 클릭 무시 (`pointer-events-none`). Tab 포커스 이동은 허용.
- RefreshButton: 연속 클릭 방지(`disabled={isFetching}`)
- 2초 초과 로딩 시 aria-live 영역에 "요청 내역을 불러오는 중입니다" 1회 공지.

## 상태 고유 비즈니스 룰 델타
- **10초 타임아웃**: `AbortController` 설정. 초과 시 `04-에러` 전환(`errorCode='NETWORK'`).
- **백그라운드 refetch**: `isFetching && data` 이면 이 상태가 아닌 `02-정상`에서 테이블 상단 얇은 프로그레스 바만 표시.
- **Preflight**: 진입 시 `user.role`이 `readonly`면 이 상태로 진입하지 않고 403 리다이렉트.

## 에지 케이스
- **API 500ms 내 즉시 완료**: 스켈레톤 최소 200ms 유지(깜빡임 방지, CSS `delay-[200ms]`).
- **네트워크 오프라인 상태로 진입**: 즉시 `04-에러` 전환 (navigator.onLine=false).
- **재필터 중 이전 요청 진행 중**: `AbortController` 로 이전 요청 취소.
- **권한 403**: jwt 유효하나 scope 부족 → 로딩 → 403 → 전역 handler → `/forbidden` 리다이렉트.
