화면: SCR-C009 일정요청처리 — 상태: 정상

  · pending: bg-amber-100 text-amber-700 "대기중"

- 액션 (pending 상태만):
  · "승인" Button green (PATCH status=approved)
  · "거절" Button red outline → confirm

- 회원명 검색
- 날짜 범위

- PATCH /api/schedule-requests/:id { status: 'approved' }
- toast.success "일정 요청이 승인되었습니다."
- 목록 갱신

거절 처리:
- confirm("일정 요청을 거절하시겠습니까?")
- PATCH /api/schedule-requests/:id { status: 'rejected' }
- toast.success "일정 요청이 거절되었습니다."

데이터:
- schedule_requests JOIN users (member)

사용 유틸:

## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 `schedule_requests.length > 0`

### 인터랙션 (User Actions)
1. "승인" → PATCH approved + toast
2. "거절" → confirm → PATCH rejected
3. "대안제시" → DLG-C016
4. 탭 전환 → 해당 상태 필터

### 비즈니스 룰
- 처리된 요청(approved/rejected): 액션 버튼 숨김
- 미처리 요청 있으면 PageHeader에 빨간 배지
- 승인 시 회원 앱에 알림 발송 (향후)

### 에지 케이스
- 이미 처리된 요청 중복 클릭: "이미 처리된 요청입니다." toast.warning

- 대안제시: DLG-C016
