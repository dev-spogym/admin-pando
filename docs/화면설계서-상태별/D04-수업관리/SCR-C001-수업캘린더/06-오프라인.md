## 📝 디스크립션

### 사용 시점 (Trigger)
- `navigator.onLine === false` (페이지 진입 시)
- 페이지 로드 후 네트워크 단절 (`offline` 이벤트)
- fetch 실패 + network error 타입

### 필수 데이터
- 없음 (오프라인 상태 자체가 트리거)

### 인터랙션 (User Actions)
1. 네트워크 복구 → 자동 `fetchCalendarData()` 재실행
2. 수동 새로고침 → 로딩 상태로 전환

### 비즈니스 룰
- 연결 복구 시 자동 재시도 (사용자 액션 불필요)
- 오프라인 중 필터/버튼 모두 disabled
- toast.warning 1회만 발사

### 에지 케이스
- 캐시된 데이터가 있는 경우: 캐시 데이터 표시 + "오프라인" 배너만 노출 (향후 기능)
- 간헐적 연결: online/offline 이벤트 디바운스 처리 (300ms)

### 접근성 (A11y)
- 오프라인 배너 `role="alert"` + `aria-live="assertive"`
- 아이콘 `aria-hidden="true"`

### 연결 화면
- 이전: `01-로딩`
- 복구 후: `01-로딩` → `02-정상`
