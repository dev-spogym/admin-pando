## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 `pendingCount > 0`
- 일정 요청이 새로 들어온 경우 (실시간 구독으로 자동 갱신)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 미승인 건수 | `lesson_schedules` | `status='pending'`, `branch_id=?` |

### 인터랙션 (User Actions)
1. "확인하기" 클릭 → `/schedule-requests` (SCR-C009)
2. 미승인 이벤트 클릭 → DLG-C002 (상세 + 승인/거절 액션)
3. 모든 건 승인/거절 후 → 배너 자동 숨김 (`pendingCount === 0`)

### 비즈니스 룰
- manager, fc만 배너 노출 (staff, readonly는 숨김)
- pendingCount 실시간 갱신: supabase realtime channel
- 배너 닫기(X) 버튼 없음 — 처리 완료 시에만 사라짐

### 에지 케이스
- pendingCount 99 초과: "99+" 표시
- 배너 노출 중 다른 사용자가 승인: 실시간으로 카운트 감소

### 접근성 (A11y)
- 배너 `role="alert"` + `aria-live="polite"`
- 배너 텍스트 스크린리더: "미승인 일정 N건이 있습니다."

### 연결 화면
- 이전: `02-정상-데이터있음`
- 확인하기: SCR-C009 일정요청처리
- 처리 완료: `07-미승인없음` (배너 숨김)
