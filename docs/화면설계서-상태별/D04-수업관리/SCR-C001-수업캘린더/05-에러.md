## 📝 디스크립션

### 사용 시점 (Trigger)
- Supabase API 호출 실패 (500, network error)
- RLS 정책 위반 (403)
- 타임아웃 (10초 초과)

### 필수 데이터
- error 객체 (로깅 및 메시지 분기용)

### 인터랙션 (User Actions)
1. "다시 시도" 버튼 → `fetchCalendarData()` 재호출
2. 뒤로가기 → 이전 화면

### 비즈니스 룰
- toast.error 최초 1회만 발사 (재시도 시 미발사)
- 403 에러: 자동 로그인 페이지 이동
- 3회 연속 실패 시 "고객센터 문의" CTA 추가 노출

### 에지 케이스
- 403 에러 → 로그인 페이지 자동 리다이렉트
- 부분 실패(수업은 로드됐지만 강사목록 실패): 강사 필터만 비활성화, 캘린더는 표시

### 접근성 (A11y)
- 에러 컨테이너 `role="alert"`
- 재시도 버튼 `aria-label="수업 데이터 다시 불러오기"`

### 연결 화면
- 이전: `01-로딩`
- 재시도 성공: `02-정상` 또는 `03-우측패널빈상태`
- 403: `/login`
