## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-C001 최초 진입 시 (라우터 push 또는 직접 접근)
- 필터 변경 후 데이터 재요청 시
- 월/주 이동 시 날짜 범위 변경으로 재fetch 시

### 필수 데이터
| 쿼리 | 테이블 | 조건 |
|------|--------|------|
| 수업 일정 | `lesson_schedules` JOIN `classes` | branch_id, 날짜 범위 |
| 강사 목록 | `users` | role='instructor', branch_id |
| 미승인 건수 | `lesson_schedules` | status='pending', branch_id |

### 인터랙션 (User Actions)
- 로딩 중에는 버튼/필터 모두 disabled
- 뒤로가기 가능 (네비게이션 차단 없음)

### 비즈니스 룰
- 최초 로드 범위: 현재 월의 1일 ~ 말일
- 병렬 fetch로 3개 쿼리 동시 실행
- 타임아웃 10초 초과 시 에러 상태로 전환

### 에지 케이스
- 느린 네트워크: Skeleton이 10초 이상 유지될 수 있음 → 타임아웃 처리
- 권한 없는 사용자: 로딩 직후 403 → 에러 상태

### 접근성 (A11y)
- Skeleton 영역 `aria-busy="true"` 속성
- 스크린리더: "수업 캘린더 데이터를 불러오는 중입니다"
