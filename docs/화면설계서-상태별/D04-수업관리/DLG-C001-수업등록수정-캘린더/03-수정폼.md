## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 모달 컴포넌트를 작성하라.
화면: DLG-C001 수업등록/수정(캘린더) — 상태: 수정 폼 (프리필)

파일: src/components/classes/LessonCalendarModal.tsx (mode='edit' 조건)

모달 헤더:
- 제목: "수업 수정" (text-lg font-semibold)

프리필 처리 (useEffect):
useEffect(() => {
  if (lesson && mode === 'edit') {
    reset({
      title: lesson.class.name,
      type: lesson.class.type,
      instructorId: lesson.instructor_id,
      date: new Date(lesson.start_at),
      startTime: format(new Date(lesson.start_at), 'HH:mm'),
      duration: differenceInMinutes(new Date(lesson.end_at), new Date(lesson.start_at)),
      capacity: lesson.capacity,
      memo: lesson.memo || '',
    })
  }
}, [lesson, mode])

변경 감지 (isDirty):
- react-hook-form formState.isDirty 사용
- 변경 없으면 X 클릭 시 바로 닫힘
- 변경 있으면 `04-미저장확인` 다이얼로그

저장 시:
- PUT /api/lesson-schedules/:id
- 성공: 캘린더 갱신 + toast.success "수업이 수정되었습니다."

폼 필드: 신규와 동일 (프리필 값 포함)

모달 푸터:
- "취소" Button outline
- "수정 저장" Button primary (isDirty=false일 때 disabled)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 캘린더 이벤트 클릭 → mode='edit', lesson 객체 전달

### 필수 데이터
| 항목 | 소스 |
|------|------|
| 기존 수업 데이터 | lesson prop (이미 로드됨) |
| 강사 목록 | `users` role=instructor |

### 인터랙션 (User Actions)
1. 프리필 값 수정 → isDirty=true
2. "수정 저장" → 유효성 → PUT API
3. X/취소 → isDirty 체크 → `04-미저장확인` 또는 바로 닫힘
4. 변경 없이 닫기 → 바로 닫힘

### 비즈니스 룰
- 완료(completed) 상태 수업: 수정 불가 (읽기 전용 모달 표시)
- 시간 충돌 재발 시: confirm 경고 (신규와 동일)
- 저장 버튼: 변경 없으면 disabled

### 에지 케이스
- in_progress 중인 수업 수정: 경고 "진행 중인 수업을 수정하시겠습니까?"
- 과거 수업 수정: 날짜 필드 비활성화 (상태만 수정 가능)

### 연결 화면
- 저장: `05-저장성공`
- 닫기(변경있음): `04-미저장확인`
- 닫기(변경없음): `01-닫힘`
