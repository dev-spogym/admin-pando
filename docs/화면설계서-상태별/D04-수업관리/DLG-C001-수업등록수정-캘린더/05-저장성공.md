```
저장 성공 처리 흐름:
const handleSubmit = async (data: FormValues) => {
  setIsSubmitting(true)
  try {
    if (mode === 'create') {
      await supabase.from('lesson_schedules').insert({ ...data })
      toast.success("수업이 등록되었습니다.")
    } else {
      await supabase.from('lesson_schedules').update({ ...data }).eq('id', lesson!.id)
      toast.success("수업이 수정되었습니다.")
    }
    onSuccess() // → 캘린더 리프레시 + 모달 닫힘
  } catch (error) {
    handleApiError(error) // 에러 분기 처리
  } finally {
    setIsSubmitting(false)
  }
}

저장 중 상태 (isSubmitting=true):
- 저장 버튼: disabled + Loader2 스피너
- 폼 필드: pointer-events-none

성공 후:
- toast.success (sonner)
- onSuccess() 호출:
  · 모달 닫힘 (setIsOpen(false))
  · 캘린더 리프레시 (refetchCalendar())
  · selectedLesson 초기화

API:
- 신규: POST /api/lesson-schedules
- 수정: PUT /api/lesson-schedules/:id
- Body: { class_id, instructor_id, start_at, end_at, capacity, memo, branch_id }
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- POST/PUT API 201/200 응답 수신

### 성공 후 동작
1. `toast.success("수업이 등록/수정되었습니다.")` 발사
2. 모달 자동 닫힘
3. SCR-C001 캘린더 이벤트 갱신 (refetch)
4. 신규 등록: 해당 날짜로 캘린더 포커스

### 비즈니스 룰
- 409 충돌: 모달 유지 + toast.warning
- 403: toast.warning "권한이 없습니다."
- 500: toast.error + 모달 유지

### 연결 화면
- 성공: `01-닫힘` + SCR-C001 갱신
- API 에러: `06-유효성에러` 또는 `07-API에러`
