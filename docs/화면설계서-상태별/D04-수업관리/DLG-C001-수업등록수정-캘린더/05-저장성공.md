## 상태 고유 인터랙션

- 사용자 입력 차단 (모달 unmount 진행 중)
- 토스트는 우측 상단에 stack
- 새 수업이 캘린더에 즉시 표시됨 (로컬 낙관적 추가 + 서버 데이터 머지)

## 상태 고유 비즈니스 룰 델타

- **API 응답 처리**:
  - INSERT: `setEvents(prev => [...prev, fromRow(data)])`, 신규 ID로 캘린더 키 갱신
  - UPDATE: `setEvents(prev => prev.map(e => e.id===data.id ? fromRow(data) : e))`
  - LOCAL: `id.startsWith('local-')`인 경우 INSERT 후 local id를 server id로 교체
- **반복 시리즈 저장**: 백엔드가 N개 row 반환 → 모두 events 추가
- **참여자 변경**: 백엔드 트랜잭션, `class_participants` diff 자동 처리
- **감사 로그**: `AUDIT.CLASS_CREATE` / `AUDIT.CLASS_UPDATE` 서버 자동 기록
- **알림 트리거**: 미승인(pending) 수업이면 매니저 알림 큐에 enqueue (서버)
- **자동 닫힘 타이밍**: 토스트 발사 직후(`~120ms`), 사용자 인지 시간 확보. `motion-reduce` 시 즉시
- **신규 등록 후 캘린더 포커스 이동**: `calendarRef.current.getApi().changeView('timeGridDay', saved.start)` 옵션 (UX 정책)

## 에지 케이스

- **응답 200 + data null**: 무결성 위반 → 토스트 "응답이 비어 있습니다. 캘린더를 새로고침합니다" + refetch
- **2xx 후 캘린더 refetch 실패**: 토스트는 성공으로 표시(서버 저장은 완료됨). 캘린더 재시도 토스트
- **반복 시리즈 부분 실패**: 서버가 `{succeeded:[], failed:[]}` 반환 → 토스트 "5개 중 3개 등록됨, 2개 실패" + 실패 사유 dialog 옵션
- **네트워크 단절 직후 응답 도착**: Sonner 토스트는 발사되나 캘린더는 stale → SWR/React Query revalidate
- **빠른 연속 저장 시 토스트 중복**: Sonner는 dedupe 자동
- **알림 큐 발사 실패**: 사일런트 로그 (UI 영향 X)
