> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 상태 진입 조건

- `formState.isDirty === true`
- 사용자가 X / 취소 / ESC / 배경 클릭 → 닫기 시도
- `isSaving === false` (저장 중일 때는 닫기 시도 자체가 차단)

## 역할별 차이

- 없음 (모든 역할 동일 흐름)

## 상태 고유 인터랙션

1. **계속 편집** (autoFocus): 다이얼로그 닫힘, `02`/`03` 폼 복귀, 변경값 보존
2. **닫기**: `reset(DEFAULTS)` → `setShowDiscard(false)` → `onClose()` → `01-닫힘`
3. **ESC**: "계속 편집" 동작 (안전한 기본값, 데이터 보존)
4. **배경 클릭(중첩 backdrop)**: "계속 편집"
5. **본 모달 백드롭 클릭**: 무시 (다이얼로그 활성 동안)

## 상태 고유 비즈니스 룰 델타

- **저장 중 차단**: `isSaving=true`이면 본 다이얼로그 자체가 표시되지 않음 (닫기 시도 무시)
- **신규/수정 모드 공통**: 흐름 동일
- **포커스 트랩 중첩**: 본 모달 트랩 일시 해제, 본 다이얼로그로 포커스 이동
- **autoFocus**: "계속 편집" 버튼 (실수로 Enter 시 데이터 보존)
- **닫기 확정 시 행동**:
  - `02-신규폼`에서 진입: form reset → 캘린더 그대로
  - `03-수정폼`에서 진입: form reset → 변경 무시 → 캘린더 원본 유지
- **a11y**: `role="alertdialog"`, `aria-labelledby`, `aria-describedby`

## 에지 케이스

- **isDirty=false인데 표시되면 안 됨**: 시스템 가드 — 컴포넌트 진입 조건 확인
- **저장 중 닫기 시도**: 다이얼로그 미표시, 닫기 무시
- **빠른 더블클릭(취소 → 다이얼로그 → ESC)**: 디바운스 또는 onAnimationEnd 가드
- **세션 만료(401) 동시 발생**: DLG-000 우선, 본 다이얼로그 즉시 정리
- **외부 라우팅 (router push)**: `next/navigation` `beforeunload` 이벤트와 분리. 모달 내 닫기에만 작동
- **모바일 가로 회전**: 다이얼로그 카드 가운데 정렬 유지

## 🧩 바이브코딩 프롬프트 (델타)

```

핵심 추가:

    else onClose();
  }
- 본 모달 backdrop/X/ESC → handleClose
- 중첩 다이얼로그 JSX (showDiscard 일 때):

          </p>
        </div>
      </div>
      <div className="flex justify-end gap-2 pt-2">

ESC 가드:

a11y:

- 닫힘 시 본 모달 활성화 + 직전 트리거(닫기 버튼)에 포커스 복귀
```
