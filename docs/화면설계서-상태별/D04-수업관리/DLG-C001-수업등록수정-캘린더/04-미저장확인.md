## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 확인 다이얼로그를 작성하라.
화면: DLG-C001 — 상태: 미저장 확인 다이얼로그

파일: src/components/classes/LessonCalendarModal.tsx (내부 상태)

미저장 확인 다이얼로그 (showUnsavedDialog=true):
- 모달 위에 중첩된 소형 다이얼로그 (z-60):
  <div className="absolute inset-0 bg-black/30 flex items-center justify-center rounded-xl">
    <div className="bg-white rounded-lg p-6 w-80 shadow-xl">
      <h3 className="font-semibold text-gray-800">변경사항이 있습니다</h3>
      <p className="mt-2 text-sm text-gray-500">
        저장하지 않고 닫으시겠습니까? 변경사항이 사라집니다.
      </p>
      <div className="mt-4 flex justify-end gap-3">
        <Button variant="outline" onClick={() => setShowUnsavedDialog(false)}>
          계속 편집
        </Button>
        <Button variant="destructive" onClick={handleForceClose}>
          닫기
        </Button>
      </div>
    </div>
  </div>

트리거 조건:
- formState.isDirty === true && (X 클릭 || ESC || 오버레이 클릭)

handleForceClose:
- reset() // 폼 초기화
- setShowUnsavedDialog(false)
- onClose() // 모달 닫기
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 폼에 변경사항이 있는 상태에서 닫기 시도

### 인터랙션 (User Actions)
1. "계속 편집" → 다이얼로그 닫힘, 폼 유지
2. "닫기" → 폼 초기화 + 모달 닫힘

### 비즈니스 룰
- formState.isDirty=false: 이 다이얼로그 건너뜀
- 신규/수정 모드 모두 동일하게 적용

### 연결 화면
- 계속 편집: `02-신규폼` 또는 `03-수정폼`
- 닫기: `01-닫힘`
