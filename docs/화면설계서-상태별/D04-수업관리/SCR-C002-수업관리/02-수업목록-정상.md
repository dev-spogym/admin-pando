## 상태 고유 인터랙션
1. 체크박스 클릭 → `selectedIds` 토글
2. 전체 체크 → 모든 표시된 행 선택
3. `수정` (Edit2) → `DLG-C003` (수정 모달)
4. `삭제` (Trash2) → `window.confirm("이 수업을 삭제하시겠습니까?")` → DELETE → 성공 토스트
5. `[N개 선택 · 일괄변경]` → `DLG-C004`
6. `[+ 수업 등록]` → `DLG-C003` (create)
7. `[노쇼/취소 정책]` → `DLG-C007`
8. 행 클릭 (staff/front) → 상세 모달 (조회 전용)

## 상태 고유 비즈니스 룰 델타
- **정렬**: 수업명 오름차순 기본, 헤더 클릭으로 변경
- **검색**: 없음 (수업 목록은 소수 항목 가정). 필요시 수업명 filter (클라이언트)
- **삭제 가드**: classes 테이블에 참조된 수업은 삭제 시 "이 수업에 예약된 기록이 있습니다" 경고 → 서버 errorCode 반환 시 토스트
- **trainer 제약**: `lesson.instructorId !== user.id`인 행은 수정/삭제 버튼 비활성 + tooltip
- **일괄변경 대상**: 선택된 lesson id 배열 → DLG-C004로 전달

## 에지 케이스
- 1건만 선택: [1개 선택 · 일괄변경] 표기
- 100건 이상 선택: 경고 + 확인 ("많은 수업을 한번에 변경합니다")
- 삭제 성공 후 0건: `04-수업목록-빈상태`
- 필터 후 0건: `04`와 다른 메시지 ("필터에 해당하는 수업 없음") — 본 케이스는 04에서 처리
- 다른 사용자 동시 편집: 낙관적 잠금 (updatedAt mismatch) → 409 재시도 안내

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14
상태: lessons-list-ready

델타:
- LESSON_COLUMNS:
  [
    { key:'select', w:40, render:(l)=><Checkbox checked={selectedIds.includes(l.id)}
        disabled={!canBulkEdit(role)} onChange={()=>toggle(l.id)} /> },
    { key:'no', w:50, align:'center', render:(_,i)=>i+1 },
    { key:'name', render:l=>(
        <div className="flex items-center gap-2">
          <span className="size-1.5 rounded-full" style={{background:l.color}} />
          <span className="text-sm font-medium text-gray-900">{l.name}</span>
        </div>
    )},
    { key:'type', w:90, align:'center', render:l=>(
        <StatusBadge variant={l.type==='GROUP'?'info':l.type==='PERSONAL'?'peach':'mint'}>
          {l.type==='GROUP'?'그룹':l.type==='PERSONAL'?'1:1':'세미'}
        </StatusBadge>
    )},
    { key:'instructorName', w:100 },
    { key:'capacity', w:70, align:'center', render:l=>`${l.capacity}명` },
    { key:'durationMin', w:70, align:'center', render:l=>`${l.durationMin}분` },
    { key:'status', w:80, align:'center', render:l=>(
        <StatusBadge dot variant={l.status==='ACTIVE'?'success':'default'}>
          {l.status==='ACTIVE'?'활성':'비활성'}
        </StatusBadge>
    )},
    { key:'actions', w:100, align:'center', render:l=>{
        const editable = canEditRecord(role, l as any, user.id);
        return (
          <div className="flex items-center gap-1 justify-center">
            <IconButton disabled={!editable} onClick={()=>openEdit(l)} aria-label="수정">
              <Edit2 className="size-4" />
            </IconButton>
            <IconButton disabled={!editable} onClick={()=>handleDelete(l)} aria-label="삭제">
              <Trash2 className="size-4 text-rose-500" />
            </IconButton>
          </div>
        );
    }},
  ]

- 삭제 핸들러:
  const handleDelete = async (l: Lesson) => {
    if (!window.confirm(`${l.name} 수업을 삭제하시겠습니까?`)) return;
    try {
      await remove(l.id);
      toast.success('수업이 삭제되었습니다.');
    } catch (e:any) {
      if (e.code === 'E409001') toast.error('이 수업에 예약된 기록이 있어 삭제할 수 없습니다.');
      else toast.error('처리에 실패했습니다.');
    }
  };

- 선택/일괄 UI:
  const toggle = (id:number) => setSelectedIds(p=>p.includes(id)?p.filter(x=>x!==id):[...p,id]);
  const selectAll = () => setSelectedIds(lessons.map(l=>l.id));
  const clearAll = () => setSelectedIds([]);

  {canBulkEdit(role) && selectedIds.length>0 && (
    <div className="flex items-center gap-2">
      <span className="text-xs text-gray-500">{selectedIds.length}개 선택</span>
      <Button variant="secondary" size="sm" onClick={openBulk}>일괄 변경</Button>
      <Button variant="ghost" size="sm" onClick={clearAll}>선택 해제</Button>
    </div>
  )}

- trainer 제약 tooltip:
  <IconButton disabled={!editable}
    aria-label="수정"
    title={!editable ? '본인 담당 수업만 수정할 수 있습니다' : '수정'}>
```
