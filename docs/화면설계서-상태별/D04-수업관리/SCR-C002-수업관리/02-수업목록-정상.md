## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 `classes.length > 0`
- 검색/필터 후 결과 있음
- 수업 등록/수정/삭제 후 목록 갱신

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 수업 목록 | `classes` | branch_id, 검색어, 분류, 상태 |
| 담당 강사 | `users` | role='instructor' |

### 인터랙션 (User Actions)
1. 수업 등록 버튼 → DLG-C003 (빈 폼)
2. 수정(Edit2) 버튼 → DLG-C003 (프리필)
3. 삭제(Trash2) → confirm → DELETE API
4. 체크박스 선택 → 일괄변경 버튼 활성
5. 일괄변경 → DLG-C004
6. 노쇼/취소정책 → DLG-C007
7. 수업 기록 탭 클릭 → `03-수업기록-정상`

### 비즈니스 룰
- 체크박스 선택 없이 일괄변경 클릭: toast.warning "수업을 선택해주세요."
- staff 역할: 등록/수정/삭제/일괄변경 버튼 숨김
- 삭제 시 확인 다이얼로그: "수업을 삭제하면 관련 기록도 함께 삭제됩니다."

### 에지 케이스
- 정원 초과: 현재인원/정원 텍스트 red-600
- 비활성 수업: 행 전체 opacity-60
- 검색 결과 없음: `04-빈상태`로 전환

### 연결 화면

- 일괄변경: DLG-C004
- 노쇼정책: DLG-C007
