```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C012 대기열관리 — 상태: 정상

파일: src/app/(classes)/waitlist/page.tsx

레이아웃:
- AppLayout + PageHeader (제목 "대기열 관리")

StatCardGrid (grid-cols-3 gap-4):
- "대기 중인 회원": N명
- "이번주 확정 전환": N건
- "평균 대기 순위": N위

DataTable:
컬럼: 순위 | 회원명 | 수업명 | 수업일시 | 대기 신청일 | 상태 | 액션
- 순위: 1위부터 순서 (드래그 순서 변경 가능)
- 상태 배지: 대기중(amber) / 확정(green) / 취소(gray)
- 액션:
  · "확정" Button green → PATCH status=confirmed + lesson_bookings 생성
  · "취소" Button red outline → PATCH status=cancelled
  · 순위 위/아래 이동 버튼 (manager만)

수업별 그룹핑 (accordion):
- 각 수업별로 그룹 헤더 + 대기 인원 목록
- 수업 헤더: 수업명 | 날짜 | 총 대기 N명

확정 처리:
- 잔여석 확인 → 있으면 lesson_bookings INSERT + waitlists PATCH(confirmed)
- 없으면 toast.warning "잔여석이 없어 확정할 수 없습니다."

데이터:
- waitlists JOIN users JOIN lesson_schedules JOIN classes
- status='waiting' 기본 필터

사용 유틸:
- DataTable, StatusBadge from '@/components/ui'
- ChevronUp, ChevronDown from 'lucide-react'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 대기 데이터 있음

### 필수 데이터
| 블록 | 테이블 |
|------|--------|
| 대기 목록 | `waitlists` (신규) |
| 수업 정보 | `lesson_schedules` JOIN `classes` |
| 회원 정보 | `users` |

### 인터랙션 (User Actions)
1. "확정" → 잔여석 확인 → lesson_bookings 생성
2. "취소" → 대기 취소
3. 순위 이동 (위/아래) → position 업데이트
4. 수업별 아코디언 펼치기/접기

### 비즈니스 룰
- 잔여석 없으면 확정 불가
- 순위 1위 자동 확정 옵션 (설정 기반)
- 확정 시 회원 앱 알림 (향후)

### 에지 케이스
- 동시 확정 시도: 낙관적 락 처리
- 수업 취소 시: 해당 수업 대기열 전체 취소

### 연결 화면
- 결과 없음: `03-빈상태`
