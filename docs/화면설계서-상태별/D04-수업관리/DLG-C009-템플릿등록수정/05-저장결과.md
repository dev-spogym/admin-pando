## 1. 상태 진입 조건

- Submit 버튼 클릭 → `handleSubmit(onSubmit)` 실행 중
- `isSubmitting === true`

## 2. 비주얼 델타

### 2.1 제출 중 (submitting)
- Submit 버튼: `disabled + 스피너` (`<Loader2 className="size-4 animate-spin" />`)
- Submit 라벨: "저장" → 유지 (스피너 좌측 prepend)
- 모든 인풋: `readOnly=true`, 팔레트/요일 버튼 `aria-disabled`
- 취소 버튼: `disabled`
- Esc / Overlay: 무시 (`isSubmitting`이면 `onEscapeKeyDown` preventDefault)

### 2.2 성공 (result: success)
- 모달 자동 닫힘 → `01-닫힘`
- 페이지 상단 Toast 표시:
  - create: `"템플릿이 등록되었습니다."`
  - edit: `"템플릿이 수정되었습니다."`
- 부모 목록(`queryClient.invalidateQueries(['lesson-templates', branchId])`) 자동 갱신
- 포커스: 트리거 버튼 복원

### 2.3 실패 (result: error)
- 모달 유지 (이전 상태 복귀)
- 에러 종류별 분기:

| errorCode | 표시 | 토스트 |
|---|---|---|
| E400001~003 | 필드 인라인 에러 (`aria-invalid`) | 없음 |
| E403001 | — | warning "권한이 없습니다." |
| E404001 | — | error "삭제된 템플릿입니다." + 모달 닫힘 |
| E409001 | `name` 필드 인라인 `"동일한 템플릿명이 존재합니다."` | 없음 |
| E500001 | — | error "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요." |
| NETWORK | — | error "네트워크 연결을 확인해주세요." |

## 3. 역할별 차이

- 동일 (모든 역할에서 Submit 후 UX 동일)
- `E403001` 응답은 `readonly/front`에서만 발생 가능 (기타 역할은 사전 차단)

## 4. 상태 고유 인터랙션

- **Submit 재클릭 차단**: `disabled + isSubmitting` 가드
- **뒤로가기**: `isSubmitting` 중에는 `window.history` 경고
- **실패 후 재시도**: 폼 값 유지 → 사용자가 에러 수정 후 다시 Submit 가능
- **성공 후 Focus**: 부모의 최초 트리거 버튼(`triggerRef`)에 복원

## 5. 비즈니스 룰 델타

1. **낙관적 업데이트 금지**: 응답 전에 목록을 수정하지 않는다 (롤백 복잡성 방지).
2. **invalidateQueries 범위**: `['lesson-templates', branchId]` + `['class-templates-count']` (대시보드 카운트).
3. **감사 로그**: 성공 시 `AUDIT.LESSON_TEMPLATE.CREATE|UPDATE` + 페이로드 diff.
4. **에러 로그**: 실패 시 `AUDIT.LESSON_TEMPLATE.ERROR` + errorCode.
5. **idempotency**: create에 클라이언트 UUID 송신(옵션) 하여 중복 생성 방지.
6. **Toast 중복 방지**: 연속 실패 시 `toast.dismiss(prevId)` 후 재표시.

## 6. 에지 케이스

- **10초 이상 지연**: 스피너 유지 + "저장 중..." 텍스트 추가 표시 고려
- **응답 직후 모달이 이미 언마운트**: `useMutation.onSuccess`의 `onSuccess` 가드로 stale closure 방지
- **성공 직후 사용자가 뒤로가기**: 페이지가 변해도 toast는 유지 (전역 Toaster)
- **E500001 후 E500001 반복**: 3회 연속 실패 시 "잠시 후 다시 시도해주세요." 쿨다운 UI (10초)
- **CORS/401 만료**: 공통 인터셉터가 `DLG-000 세션만료`로 분기

## 7. 바이브코딩 프롬프트 (델타)

```
핸들러:
const onSubmit = async (data: LessonTemplateForm) => {
  try {
    const branchId = useAuthStore.getState().branchId
    const payload = { ...data, branchId }
    const res = mode === 'create'
      ? await api.post('/api/lesson-templates', payload)
      : await api.put(`/api/lesson-templates/${template!.id}`, payload)

    toast.success(mode === 'create' ? '템플릿이 등록되었습니다.' : '템플릿이 수정되었습니다.')
    queryClient.invalidateQueries({ queryKey: ['lesson-templates', branchId] })
    queryClient.invalidateQueries({ queryKey: ['class-templates-count'] })
    onSuccess()
    onClose()
  } catch (e: any) {
    const code = e?.response?.data?.errorCode
    switch (code) {
      case 'E400001':
      case 'E400002':
      case 'E400003':
        Object.entries(e.response.data.fieldErrors ?? {}).forEach(([k,v]) =>
          setError(k as any, { message: v as string })
        )
        break
      case 'E403001': toast.warning('권한이 없습니다.'); break
      case 'E404001': toast.error('삭제된 템플릿입니다.'); onClose(); break
      case 'E409001': setError('name', { message: '동일한 템플릿명이 존재합니다.' }); break
      case 'E500001': toast.error('일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요.'); break
      default:
        if (!e?.response) toast.error('네트워크 연결을 확인해주세요.')
        else toast.error('처리에 실패했습니다.')
    }
  }
}

Submit 버튼:
<button type="submit"
  disabled={!isValid || isSubmitting || (mode==='edit' && !isDirty)}
  className="h-10 px-4 rounded-lg bg-blue-600 hover:bg-blue-700 text-white text-sm font-medium
             disabled:bg-blue-400 disabled:cursor-not-allowed inline-flex items-center gap-2">
  {isSubmitting && <Loader2 className="size-4 animate-spin" />}
  {mode === 'create' ? '저장' : '수정 저장'}
</button>

폼 필드 readOnly 처리:
<fieldset disabled={isSubmitting} className="contents">
  {/* 모든 인풋 래핑 */}
</fieldset>
```
