# DLG-C009 템플릿 등록/수정 — 상태: 닫힘

> 이 문서는 `00-기본화면.md` 마스터의 **델타**만 기술합니다. 레이아웃/토큰/권한/데이터 등은 마스터 기본값을 따릅니다.

## 0. 메타

| 항목 | 값 |
|---|---|
| 상태 코드 | `closed` |
| 다이어그램 노드 | `STATE_CLOSED` |
| 이전 상태 | 초기 / `05-저장결과`(성공) / `04-미저장확인`(버리기) |
| 다음 상태 | `02-신규폼` / `03-수정폼` |

## 1. 상태 진입 조건

- 페이지 최초 렌더 (`isOpen=false`)
- 저장 성공 후 (`onClose()` 호출)
- 미저장 변경 확인에서 "버리기" 선택
- 부모에서 강제 언마운트

## 2. 비주얼 델타

- **다이얼로그 자체가 렌더되지 않음** (Radix Portal 비활성, `<Dialog.Root open={false}>`)
- 부모 화면 `SCR-C004 그룹수업템플릿` 정상 노출
- body scroll lock 해제 (`overflow:auto` 복원)
- 포커스 복원: 이전 트리거 버튼(`+ 템플릿 등록` 또는 행의 `Edit2`)에 `focus()` 이동

```
(모달 없음) — 부모 페이지 정상 표시
```

## 3. 역할별 차이

- 모든 역할에서 동일 (부모 화면 노출). RBAC 차이는 부모 화면 기준.

## 4. 상태 고유 인터랙션

- `+ 템플릿 등록` 버튼 클릭 → `setMode('create'); setIsOpen(true)` → `02-신규폼`
- 행 `Edit2` 클릭 → `setMode('edit'); setTemplate(row); setIsOpen(true)` → `03-수정폼`

## 5. 비즈니스 룰 델타

- 부모 목록이 `queryClient.invalidateQueries(['lesson-templates', branchId])` 결과로 최신화되어 있어야 함.
- 저장 성공 시 `toast.success` 이미 트리거된 상태.

## 6. 에지 케이스

- 뒤로가기 중 실수 트리거: 없음(모달 닫힘 상태는 route 영향 없음).
- 직전 미저장 변경 버림 후 재클릭: 이전 입력값 남지 않고 새 폼으로 초기화(`reset()` 호출).
- 저장 성공 직후 부모 페이지 언마운트(라우트 변경) → 리프레시 쿼리 취소 허용.

## 7. 바이브코딩 프롬프트 (델타)

```
상태: 닫힘
변경점:
- <Dialog.Root open={false}> → Portal 내부 미렌더
- 트리거 버튼 포커스 복원: useEffect(() => { if(!isOpen && triggerRef.current) triggerRef.current.focus() }, [isOpen])
- 폼 상태 초기화: useEffect(() => { if(!isOpen) reset(defaultValues); setShowUnsaved(false) }, [isOpen])
- body scroll lock 해제: Radix 자동
```
