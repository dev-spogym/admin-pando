```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 수업기록 상세 표시를 작성하라.
화면: DLG-C005 수업기록상세 — 상태: 정상 상세표시

파일: src/components/classes/LessonRecordDetailModal.tsx

모달 헤더:
- 제목: record.class_name (text-lg font-semibold)
- StatusBadge: record.status
- X 버튼: onClose()

상세 정보 (p-6 space-y-3):
- 회원명 / 강사명 / 날짜·시간 / 수업 분류 / 서명이미지(completed 시)

상태별 버튼 표시:
{record.status === 'scheduled' && (
  <>
    <Button onClick={handleStart}>수업 시작</Button>
    <Button variant="outline" onClick={handleNoShow}>노쇼</Button>
    <Button variant="outline" onClick={handleCancel}>취소</Button>
  </>
)}
{record.status === 'in_progress' && (
  <>
    <Button onClick={() => setOpenSign(true)}>완료 처리</Button>
    <Button variant="outline" onClick={handleCancel}>취소</Button>
  </>
)}
{['completed','no_show','cancelled'].includes(record.status) && (
  <p className="text-sm text-gray-500">읽기 전용</p>
)}

역할별 제한:
- manager/owner: 모든 버튼 활성
- fc (본인 수업): 모든 버튼 활성
- fc (타강사) / staff: 버튼 없음

서명 이미지 (completed):
<img src={record.signature_url} alt="서명" className="max-h-24 border rounded" />
```

## 📝 디스크립션

### 상태별 활성 버튼
| lesson_status | 활성 버튼 |
|---------------|----------|
| scheduled | 수업시작 / 노쇼 / 취소 |
| in_progress | 완료처리 / 취소 |
| completed / no_show / cancelled | 없음 (읽기전용) |

### 인터랙션
1. 수업시작 → PATCH lesson_status=in_progress → 모달 내 상태 갱신
2. 완료처리 → DLG-C006 서명 모달 열림
3. 노쇼 → PATCH lesson_status=no_show → toast.success + 모달 닫힘
4. 취소 → PATCH lesson_status=cancelled → toast.success + 모달 닫힘

- 완료처리: DLG-C006
- 닫기: `01-닫힘`
