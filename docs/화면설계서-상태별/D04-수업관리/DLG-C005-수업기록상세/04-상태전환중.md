```
상태 전환 핸들러:
const handleStatusChange = async (newStatus: 'in_progress' | 'no_show' | 'cancelled') => {
  setIsUpdating(true)
  try {
    await fetch(`/api/lesson-records/${recordId}/status`, {
      method: 'PATCH',
      body: JSON.stringify({ status: newStatus }),
    })
    if (newStatus === 'in_progress') {
      toast.success("수업이 시작되었습니다.")
      refetch()  // 모달 내 상태 갱신
    } else if (newStatus === 'no_show') {
      toast.success("노쇼 처리되었습니다.")
      onSuccess(); onClose()
    } else if (newStatus === 'cancelled') {
      toast.success("수업이 취소되었습니다.")
      onSuccess(); onClose()
    }
  } catch (error) {
    if (error.status === 400) toast.warning("유효하지 않은 상태 전환입니다.")
    else if (error.status === 403) toast.warning("권한이 없습니다.")
    else toast.error("처리에 실패했습니다.")
  } finally {
    setIsUpdating(false)
  }
}

처리 중 UI (isUpdating=true):
- 버튼들: disabled + opacity-50
- 클릭한 버튼: Loader2 스피너
```

## 📝 디스크립션

### API 결과 분기
| 응답 | 동작 |
|------|------|
| 200 in_progress | 모달 내 상태 갱신 (refetch) |
| 200 no_show/cancelled | toast.success + 모달 닫힘 |
| 400 | toast.warning "유효하지 않은 상태 전환" |
| 403 | toast.warning "권한이 없습니다." |
| 500 | toast.error + 모달 유지 |

- no_show/cancelled 성공: `01-닫힘` + SCR-C002 갱신
- 실패: `05-에러`
