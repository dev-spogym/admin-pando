## 📝 디스크립션

### 사용 시점 (Trigger)
- 목록/상세 화면의 삭제 버튼 클릭
- 권한 검증 통과 후 모달 열림

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| targetName | props | 삭제 대상 이름 표시 |
| hasRelatedData | M2 검증 결과 | 연관 데이터 경고 여부 |
| relatedCount | M2 검증 결과 | 경고 문구용 |

### 인터랙션 (User Actions)
1. "삭제" 버튼 → `02-삭제중`
2. "취소" / X / ESC / 배경 클릭 → 닫힘

### 비즈니스 룰
- 권한 없는 역할 삭제 버튼 클릭 → 모달 미열림 + 권한없음 토스트 (M2 검증)
- 참조 데이터 있음 → 경고 문구 포함 모달 (삭제는 허용)
- 삭제 불가 항목 → 별도 안내 토스트 (모달 미열림)

### 에지 케이스
- 이미 삭제된 항목 → 404 에러 토스트 + 목록 갱신
- 네트워크 오류 → `02-삭제중`에서 에러 처리

### 접근성 (A11y)
- `role="alertdialog"` `aria-modal="true"` `aria-labelledby="delete-confirm-title"`
- 모달 열림 시 "삭제" 버튼에 포커스 (위험 액션 확인 강요)
- 닫힘 시 삭제 버튼으로 포커스 복원

### 연결 화면
- 삭제: `02-삭제중`
- 취소: 현재 화면
