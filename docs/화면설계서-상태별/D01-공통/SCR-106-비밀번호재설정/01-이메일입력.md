## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-100 로그인의 "비밀번호 재설정" 링크 클릭
- `/reset-password` 직접 접근 (token 파라미터 없음)
- 423 계정 잠금 시 재설정 링크 강조 후 클릭

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 이메일 | 사용자 입력 | 가입 시 등록된 이메일 |

### 인터랙션 (User Actions)
1. 이메일 입력
2. "재설정 링크 발송" 클릭 또는 Enter → `02-발송중`
3. 형식 오류 → 인라인 에러 표시
4. "로그인으로 돌아가기" → /login

### 비즈니스 룰
- 미등록 이메일 입력 시에도 동일 성공 화면 표시 (보안: 이메일 존재 여부 노출 금지)
- 발송 후 5분 내 재발송 제한 (서버 정책)

### 에지 케이스
- 이미 로그인 상태에서 접근 → 동일 화면 표시 (리다이렉트 없음)
- 이메일 공백 제출 → "이메일을 입력하세요" 에러

### 접근성 (A11y)
- input `autocomplete="email"` `aria-label="이메일 주소"`
- 에러 `role="alert"` `aria-live="polite"`

### 연결 화면
- 제출: `02-발송중`
- 뒤로: SCR-100 로그인
