# DLG-002 이탈 경고 — 상태 01: 열림

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 메타
| 항목 | 값 |
|------|----|
| 상태 코드 | `leave-warning-open` |
| 상태 ID | STATE_PENDING |
| 다이어그램 노드 | `M1_생명주기.md` → STATE_PENDING |

| 다음 상태 | `02-이탈허용` ("저장 안 함") / Idle("계속 편집" / ESC) / save 성공 후 이동 |

## 상태 진입 조건
- `useBlockNavigation(isDirty=true)` 훅에서 `pendingUrl !== null` 로 전환
- 트리거 케이스: 사이드바 링크 클릭, `router.push` 차단, `popstate`(뒤로가기), 탭 전환 가드, 기타 LinkGuard 가로채기
- DLG-000/DLG-001 열려 있지 않음

## 상태 고유 비주얼 델타
- 마스터 기본 렌더 그대로
- "계속 편집" 버튼 오토포커스(안전 기본값)
- "저장 후 이동" 버튼은 `onSaveAndLeave` prop 있을 때만 렌더 (2-버튼 vs 3-버튼 분기)
- `saving=false` 상태

## 상태 고유 인터랙션
1. **오픈 직후**: "계속 편집" 포커스 → Enter 즉시 닫힘(안전한 기본 동작)
2. **ESC 키**: `onContinueEditing()` 호출 (닫힘)
3. **배경 클릭**: 동일
4. **X 버튼**: 동일
5. **"계속 편집"**: 닫힘, pendingUrl 폐기
6. **"저장 후 이동"**(옵션): save → 성공 시 이동 → 언마운트. 실패 시 다이얼로그 유지 + 토스트
7. **"저장 안 함"**: `02-이탈허용` 전환 후 언마운트(리다이렉트)
8. **Tab 순환**: 계속 편집 → 저장 후 이동 → 저장 안 함 → X

## 역할별 차이
- 기본 동일. 단, 부모 폼이 저장 권한 없다고 판단하면 "저장 후 이동" 버튼을 아예 노출하지 않음(RBAC).

## 상태 고유 비즈니스 룰 델타
- 모달 오픈 시점에 `document.body.style.overflow = 'hidden'`
- `popstate` 트리거된 경우 `pendingUrl = '__back__'` 표식 사용 → 확정 시 `history.back()`
- DLG-000 이 뒤늦게 오픈되면 이 다이얼로그는 자동 언마운트
- 저장 후 이동 구현 시 부모 폼 `onSubmit` 가 `resolver` 검증 실패하면 save 안 함 + 토스트

## 에지 케이스
- **pendingUrl 이 현재 URL**: noop (닫기)
- **여러 번 연속 popstate**: 가짜 프레임을 계속 `pushState` 로 채워 넣어 방어
- **저장 후 이동 중 세션 만료(401)**: DLG-000 이 우선 오픈되며 DLG-002 는 자동 정리

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: leave-warning-open (열림)

주요 사용 패턴:
const form = useForm<Profile>({ mode:'onChange', defaultValues });
const isDirty = form.formState.isDirty;
const { isOpen, cancelLeave, confirmLeave } = useBlockNavigation(isDirty);

<LeaveWarningDialog
  isOpen={isOpen}
  onContinueEditing={cancelLeave}
  onDiscard={confirmLeave}
  onSaveAndLeave={form.handleSubmit(async (v) => { await save(v); confirmLeave(); })}
/>

- 오픈 시 "계속 편집" autoFocus
- ESC/배경/X → cancelLeave
- saving=false 기본 렌더
```
