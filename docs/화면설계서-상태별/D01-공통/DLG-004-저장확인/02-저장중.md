## 상태 고유 비즈니스 룰 델타
- **낙관적 잠금 헤더**:
  - 지원 API: `If-Match: <etag>` 또는 body `version` 필드
  - 서버가 mismatch 감지 시 `409 E409xxx` + 최신 서버값 포함
- 타임아웃 10초 → `04-저장실패` + 토스트 "요청 시간이 초과되었습니다"
- 응답 200/201 → `03-저장성공`
- 응답 409 → `05-충돌` (conflict 상태 set)
- 응답 4xx 기타/5xx/NETWORK → `04-저장실패`

## 에지 케이스
- **부분 성공(207 Multi-Status)**: 저장 1건 + 실패 1건 → 토스트 warning, 다이얼로그 유지 + 실패 항목 하이라이트(부모 책임)
- **서버 202 Accepted(비동기 처리)**: 폴링 필요 → 현재 스펙은 동기 응답 가정
- **세션 만료(401)**: 인터셉터가 DLG-000 오픈 → 이 다이얼로그 자동 정리
- **네트워크 재연결**: fetch 재시도 없음(명시적 재시도 버튼만 제공)

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: save-confirm-saving

핵심:
- loading=true 전달 → ConfirmDialog 내부에서 버튼 disabled + 스피너
- ESC/배경 차단 가드:
  onClick={(e) => { if (e.target === e.currentTarget && !loading) onCancel(); }}
  useEffect: if (loading) e.preventDefault() on Escape

API 호출 패턴 (useMutation):
const save = useMutation({
  mutationFn: async (values) => {
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), 10_000);
    try {
      const res = await fetch('/resource/'+id, {
        method:'PATCH', signal: controller.signal,
        headers:{ 'Content-Type':'application/json', 'If-Match': etag },
        body: JSON.stringify(values),
      });
      const body = await res.json();
      if (res.status === 409) throw { __conflict: true, server: body.data?.server, body };
      if (!res.ok) throw { errorCode: body.errorCode, message: body.message, status: res.status };
      return body;
    } finally { clearTimeout(timeout); }
  },
});
```
