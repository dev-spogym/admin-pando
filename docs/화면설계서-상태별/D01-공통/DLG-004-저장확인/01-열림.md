## 📝 디스크립션

### 사용 시점 (Trigger)
- 편집 폼에서 저장 버튼 클릭 (일부 화면에서 중요 변경 시 확인 유도)
- DLG-002 이탈 경고에서 "저장 후 이동" 선택 시 (선택적 패턴)

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| afterSaveAction | props | stay(계속 편집) 또는 navigate(이동) |
| destination | props | navigate 시 목적지 URL |

### 인터랙션 (User Actions)
1. "저장" 버튼 → `02-저장중`
2. "취소" / X / ESC / 배경 클릭 → 닫힘, 편집 유지

### 비즈니스 룰
- 모든 폼이 이 모달을 사용하지 않음 (중요 작업에만 적용)
- 일반 저장은 버튼 직접 처리 (모달 없음)
- 이탈 경고(DLG-002)와 연계: "저장하고 나가기" 패턴 지원

### 에지 케이스
- 폼 검증 실패 → 저장 버튼 비활성 (이 모달 미표시)
- 저장 중 ESC → 차단 (`02-저장중`에서 처리)

### 접근성 (A11y)
- `role="dialog"` `aria-modal="true"` `aria-labelledby="save-confirm-title"`
- 모달 열림 시 "저장" 버튼에 포커스
- 닫힘 시 저장 버튼으로 포커스 복원

### 연결 화면
- 저장: `02-저장중`
- 취소: 현재 편집 화면
