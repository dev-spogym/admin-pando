## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장 API → 409 Conflict 응답
- 동시 편집 충돌 (낙관적 잠금 실패)

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| conflictData | 409 응답 본문 | 서버 최신 데이터 (옵셔널) |
| onReload | props | 폼 데이터 재로드 함수 |

### 인터랙션 (User Actions)
1. "최신 데이터 불러오기" → 서버 최신 데이터로 폼 갱신 (사용자 입력 손실)
2. "취소" → 모달 닫힘 + 사용자 입력 유지 (재시도 가능)

### 비즈니스 룰
- 409 충돌은 낙관적 잠금(optimistic locking) 실패를 의미
- 최신 데이터 불러오기 → 사용자 입력 덮어쓰기 경고
- 취소 시 사용자가 직접 변경 내용 비교 후 재시도 가능

### 에지 케이스
- 충돌 데이터 미제공 → "최신 데이터 불러오기" 버튼만 제공 (비교 불가)
- 재로드 실패 → 에러 토스트

### 접근성 (A11y)
- `role="alertdialog"` `aria-labelledby="conflict-title"`
- 충돌 안내 `aria-live="assertive"`
- 포커스: "최신 데이터 불러오기" 버튼

### 연결 화면
- 재로드: 편집 화면 (최신 데이터 반영)
