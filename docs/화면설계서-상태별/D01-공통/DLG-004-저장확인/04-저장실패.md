# DLG-004 저장 확인 — 상태 04: 저장 실패

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 상태 진입 조건
- 응답 코드별:
  - 400/422 → 필드/룰 위반
  - 403 → 권한 없음
  - 500/502/503 → 서버 오류
  - NETWORK/타임아웃

## 상태 고유 비주얼 델타

### 에러 코드별 동작

## 상태 고유 인터랙션
- 에러 토스트 노출(5초 auto-dismiss 또는 X)
- 다이얼로그 유지 시 Primary 버튼 재활성 → 재시도 가능

## 역할별 차이
- E403은 주로 역할 변경/세션 브랜치 변경 시 발생
- 일반 사용자는 거의 E500/NETWORK 경험

## 상태 고유 비즈니스 룰 델타
- **ERR_MAP** 기반 분기(DLG-003 §04 와 유사 구조):
  ```ts
  const SAVE_ERR_MAP = {
    E403001: { message: '저장 권한이 없습니다', closeDialog: true },
    E422xxx: { message: '저장할 수 없는 데이터입니다', closeDialog: false },
    E500001: { message: '일시적인 오류가 발생했습니다', closeDialog: false },
    E503001: { message: '서비스 점검 중입니다', closeDialog: false },
    NETWORK: { message: '네트워크 오류가 발생했습니다', closeDialog: false },
  };
  ```
- 낙관적 업데이트 롤백: 부모가 `onMutate` 에서 store 변경을 했다면 `onError` 에서 snapshot 복원
- **E400 필드 에러**: 서버가 필드별 에러를 반환하면 `form.setError(field, { message })` 로 인라인 표시 → 다이얼로그 닫고 폼으로 포커스 이동
- **3회 연속 500** 시 다이얼로그 자동 닫기(악순환 방지)

## 에지 케이스
- **409 는 `05-충돌` 로 분기** (`04-저장실패` 에서 처리하지 않음)
- **응답 본문 파싱 실패**: 기본 메시지 "저장에 실패했습니다"
- **토스트 중복**: dedup 로직 적용
- **DLG-000 우선**: 401이면 세션만료가 우선

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: save-confirm-error

onError 핸들러:
function onError(err: any) {
  if (err?.__conflict) return; // 05-충돌에서 처리 (여기서는 건너뜀)
  const code = err.errorCode ?? (err.status >= 500 ? 'E500001' : 'NETWORK');
  const info = SAVE_ERR_MAP[code] ?? { message: '저장에 실패했습니다', closeDialog: false };

  // 필드 에러 분기 (E400 + details)
  if (err.errorCode?.startsWith('E400') && Array.isArray(err.details)) {
    err.details.forEach((d: {field:string; message:string}) => {
      form.setError(d.field as any, { message: d.message });
    });
    toast.error(info.message);
    setOpen(false); // 폼으로 돌려보냄
    return;
  }

  toast.error(info.message);
  if (info.closeDialog) setOpen(false);
  setLoading(false);
}

3회 500 방어:
if (code === 'E500001') {
  setFailCount(c => c + 1);
  if (failCount + 1 >= 3) { setOpen(false); toast.error('잠시 후 다시 시도해 주세요'); return; }
}
```
