## 📝 디스크립션

### 사용 시점 (Trigger)
- KPI 데이터 API 500/502/504 응답
- Promise.all 중 하나 이상 실패

### 필수 데이터
- errorMessage: 서버에서 받은 오류 메시지 (옵셔널)
- 재시도 함수

### 인터랙션 (User Actions)
1. 재시도 버튼 → 데이터 다시 로딩
2. 사이드바 네비게이션 → 다른 화면 이동 가능

### 비즈니스 룰
- 에러 발생 시 에러 토스트도 함께 표시
- 재시도 횟수 제한 없음
- 부분 실패(일부 KPI만 실패) → 실패 KPI 카드에만 에러 표시 + 나머지 정상 표시

### 에지 케이스
- 재시도 중에도 에러 → 에러 상태 유지
- 네트워크 복구 후 재시도 성공 → 정상 상태 전환

### 접근성 (A11y)
- 에러 영역 `role="alert"` `aria-live="assertive"`
- 재시도 버튼 `aria-label="대시보드 데이터 다시 불러오기"`

### 연결 화면
- 재시도: `01-로딩` → `02-정상`
