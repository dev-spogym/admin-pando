## 상태 고유 인터랙션
1. **"다시 시도"**: `refetchAll()` + 로딩 상태 재진입
2. **"새로고침"**: `window.location.reload()` (최후 수단)
3. **위젯 재시도**: 각 `WidgetError` 가 해당 쿼리 invalidate
4. **사이드바 네비**: 활성 (다른 화면 이동 가능)

## 역할별 차이
- 없음. 단 primary/super 는 시스템 장애 시 `/super-dashboard` 로 이동 권장 링크 추가

## 상태 고유 비즈니스 룰 델타
- **치명 vs 부분 구분**:
  - 치명: `/dashboard/stats` 실패 → 전체 에러 배너 + 위젯들은 스켈레톤 or 숨김
  - 부분: 개별 쿼리 실패 → WidgetError 표시, 나머지는 정상
- **재시도 쿨다운**: 5초 간 중복 클릭 방지
- **연속 실패 감지**: 3회 이상 연속 실패 시 `"서버 점검 중일 수 있습니다. 잠시 후 다시 시도해 주세요."` 문구 교체
- **로그 수집**: `console.error` 또는 서버 측 에러 로깅 서비스로 전송(선택)
- **감사로그**: 에러 자체는 기록 안 함

## 에지 케이스
- **오프라인 + 에러 동시 발생**: `05-오프라인` 우선(오프라인 감지가 더 상위)
- **세션 만료(401)**: DLG-000 우선 오픈 → 이 상태 자동 정리
- **403 조작**: `/forbidden` 리다이렉트
- **503 점검**: 점검 배너로 대체(warn 톤)

  return (
    <div role="alert" className="bg-white rounded-xl ring-1 ring-rose-100 p-5 text-center">
      <AlertCircle className="size-5 text-rose-500 mx-auto" />
      <p className="mt-2 text-sm text-gray-700">{message ?? '이 위젯을 불러오지 못했습니다'}</p>

  );
}
