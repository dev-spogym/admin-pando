## 📝 디스크립션

### 사용 시점 (Trigger)
- `/` 경로 최초 진입 (로그인 후 리다이렉트)
- 페이지 새로고침
- 기간 필터 변경 시 재로딩

### 필수 데이터
| 블록 | 테이블/API | 비고 |
|------|-----------|------|
| 매출 KPI | `transactions` 집계 | 역할별 브랜치 필터 |
| 회원 KPI | `members` 집계 | 활성 회원 수 |
| 출석 KPI | `attendance` 집계 | 오늘/이번주 |
| 수업 KPI | `classes` 집계 | 오늘 예정 수업 |

### 인터랙션 (User Actions)
- 로딩 중 모든 상호작용 차단 (사이드바 네비게이션은 허용)
- 헤더의 새로고침 버튼 → 재로딩 가능

### 비즈니스 룰
- 인증 없으면 SCR-100으로 리다이렉트 (middleware 처리)
- 역할별 접근 가능한 KPI 필터 적용
- 데이터 로드 병렬 실행 (Promise.all)

### 에지 케이스
- 일부 KPI만 실패 → 실패한 KPI 카드에 에러 표시 + 나머지 정상 표시
- 로딩 5초 초과 → 타임아웃 에러 처리

### 접근성 (A11y)
- `aria-busy="true"` on 메인 컨텐츠 영역
- 스켈레톤에 `aria-label="로딩 중"` `role="status"`

### 연결 화면
- 완료: `02-정상`
- 실패: `04-에러` 또는 `05-오프라인`
