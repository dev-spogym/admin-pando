## 상태 고유 인터랙션
- **모든 위젯 클릭 불가** (`pointer-events-none opacity-70`)
- BranchSwitcher/PeriodFilter/새로고침/엑셀/레이아웃 전부 `disabled` (중복 호출 방지)
- 사이드바 네비(SCR-102)는 항상 활성 — 다른 페이지 이동 허용
- 로딩 5초 초과 시 PageHeader 하단에 보조 텍스트 "조회에 시간이 걸리고 있습니다..."
- 10초 초과 시 → `04-에러` 분기 또는 계속 로딩(타임아웃 useQuery 레벨 10s)

## 상태 고유 비즈니스 룰 델타
- **skeleton 수량**: 역할별 §5.1 매트릭스와 동일 카드 수 유지
- **CLS 방지**: 모든 스켈레톤은 실제 컴포넌트와 동일 치수
- **첫 진입**: `runDailySync()` 동시 호출 → 종료 시 refetch 트리거
- **Stream 렌더링(옵션)**: Next 15 Suspense로 header → KPI → 차트 → 위젯 순차 경계

## 에지 케이스
- **부분 완료**: 완료된 위젯은 즉시 실제 데이터로 전환, 나머지는 스켈레톤 유지
- **타임 슬로우**: 일부 쿼리 10초 이상 pending → 해당 위젯만 ErrorBoundary fallback
- **role 미확정**: user 객체 로드 전에는 전체 스켈레톤(기본 5개 KPI 추정)
- **realtime 초기 연결 지연**: IoT 상태 판단 전이므로 배너 미렌더
