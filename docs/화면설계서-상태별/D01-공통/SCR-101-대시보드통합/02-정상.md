```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-101 대시보드 통합 — 상태: 정상 (KPI + 차트 + 위젯)

컴포넌트 구조:
1. PageHeader (title, period filter, action buttons)
2. StatCardGrid (4 KPI StatCard: 클릭 시 도메인 이동)
3. QuickAction 버튼 행
4. MainChart (매출 추이 LineChart)
5. TodayScheduleWidget (오늘 수업 목록)
6. UnreadNotifWidget (미읽음 알림 목록)

데이터:
- salesKpi: { today, week, month, trend }
- memberKpi: { active, new, expiring }

- classKpi: { todayCount, upcomingCount }
- todayClasses: Class[]

- periodFilter: 'today' | 'week' | 'month'

인터랙션:
- 매출 KPI 클릭 → moveToPage('/sales')
- 회원 KPI 클릭 → moveToPage('/members')
- 출석 KPI 클릭 → moveToPage('/attendance')
- 수업 KPI 클릭 → moveToPage('/classes')
- 기간 필터 변경 → 데이터 갱신 (01-로딩 재진입)
- IoT 배지 클릭 → moveToPage('/iot')
- 새로고침 버튼 → 전체 재로드
- 엑셀 내보내기 → KPI 엑셀 다운로드

사용 유틸:
- useAuthStore, supabase, toast, formatNumber, formatDate, moveToPage, hasPermission, lucide-react
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 1개 이상 KPI 데이터 존재
- 기간 필터 변경 후 갱신 완료
- 수동 새로고침 완료

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 매출 KPI | `transactions` | branch_id, 기간 필터 |
| 회원 KPI | `members` | branch_id, status='active' |
| 출석 KPI | `attendance` | branch_id, 오늘/이번주 |
| 수업 KPI | `classes` | branch_id, 오늘 예정 |
| 오늘 일정 | `classes` JOIN `instructors` | 오늘 00:00~23:59 |
| 미읽음 알림 | `notifications` | user_id, is_read=false |

### 인터랙션 (User Actions)
1. KPI 카드 클릭 → 해당 도메인 이동
2. 기간 필터 탭 전환 → 데이터 재로딩
3. 새로고침 → 전체 재로딩
4. 엑셀 내보내기 → 성공 토스트 + 파일 다운로드
5. 퀵액션 버튼 → 해당 화면 이동
6. IoT 배지 → IoT 관리 화면 이동

### 비즈니스 룰
- superAdmin은 지점 전환 드롭다운으로 타 지점 데이터 열람 가능
- staff 역할은 매출 KPI 미표시 (hasPermission 체크)
- IoT 장애 감지(isIotError=true) 시 상단 경고 배너 노출

### 에지 케이스
- 일부 KPI 로딩 실패 → 해당 카드만 에러 표시 + 재시도 버튼
- 엑셀 내보내기 실패 → "내보내기에 실패했습니다" 에러 토스트
- 실시간 알림 수신 → 미읽음 뱃지 즉시 업데이트

### 접근성 (A11y)
- KPI 카드 `role="button"` `aria-label="{KPI명}: {값}"`
- 퀵액션 `aria-label="퀵액션: {액션명}"`
- IoT 배너 `role="alert"` `aria-live="assertive"`
