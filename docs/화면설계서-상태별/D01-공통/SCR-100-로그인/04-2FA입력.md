## 📝 디스크립션

### 사용 시점 (Trigger)
- POST /auth/login → 200 응답 + `requires2FA: true` 반환 시
- 이메일/비밀번호 검증 성공 후 2FA 단계로 진입

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| OTP 코드 | 사용자 입력 | 6자리 숫자 |
| email | 이전 단계 유지 | 재발송/검증 API에 사용 |

### 인터랙션 (User Actions)
1. 6자리 OTP 입력 → 자동 제출 또는 확인 버튼
2. OTP 재발송 버튼 → 새 OTP 발송, 60초 쿨다운
3. "다른 계정으로 로그인" → `01-기본` 초기화
4. OTP 오류 → 필드 초기화 + 에러 메시지

### 비즈니스 룰
- OTP 유효 시간: 5분 (서버 검증)
- OTP 재발송 쿨다운: 60초 (클라이언트 카운트다운)
- OTP 5회 연속 실패 시 계정 잠금 (423 응답)
- ESC/배경 클릭으로 이 단계 이탈 불가

### 에지 케이스
- OTP 만료(5분 경과) → "OTP가 만료되었습니다. 재발송해주세요" 에러
- 잘못된 OTP 입력 → 필드 초기화 + "올바르지 않은 코드입니다" 에러
- 재발송 성공 → "새 코드가 발송되었습니다" 토스트

### 접근성 (A11y)
- OTP 입력 `aria-label="6자리 인증 코드"` `autocomplete="one-time-code"`
- 재발송 버튼 disabled 시 `aria-disabled="true"` + 카운트다운 `aria-live="polite"`
- 에러 `role="alert"`

### 연결 화면
- 성공: SCR-101 대시보드 (역할별)
- 실패: 동일 화면 (에러 표시)
- 취소: `01-기본` 초기화
