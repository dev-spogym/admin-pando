## 📝 디스크립션

### 사용 시점 (Trigger)
- POST /auth/login → 503 Service Unavailable 응답
- 진입 전 헬스체크 실패 시
- 점검 예고 배너 클릭 → 점검 안내 전환

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 점검 종료 예정 시각 | 503 응답 본문 | 없을 수 있음 (옵셔널) |
| 점검 안내 메시지 | 503 응답 본문 | 없으면 기본 문구 표시 |

### 인터랙션 (User Actions)
1. 자동 30초 카운트다운 → 헬스체크 재시도
2. "새로고침" 버튼 클릭 → 즉시 헬스체크 재시도
3. 서비스 정상 응답 → `01-기본` 폼 복원

### 비즈니스 룰
- 서비스 점검 중에는 로그인 폼 비활성화
- 자동 재시도 간격: 30초
- 재시도 성공 시 폼 복원 + "서비스가 재개되었습니다" 토스트

### 에지 케이스
- 점검 종료 시각 미제공 → "점검 시간 미정" 표시
- 반복 재시도 실패 → 횟수 제한 없이 계속 재시도

### 접근성 (A11y)
- 카운트다운 `aria-live="polite"` `aria-label="재시도까지 N초"`
- 새로고침 버튼 `aria-label="서비스 연결 재시도"`

### 연결 화면
- 점검 종료 후: `01-기본` 복원
