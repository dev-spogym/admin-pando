## 📝 디스크립션

### 사용 시점 (Trigger)
- 로그인 버튼 클릭 또는 Enter 키 입력 후 클라이언트 검증 통과 시
- POST /auth/login API 호출 진행 중

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| email | 폼 입력값 | 이미 검증 완료 |
| password | 폼 입력값 | 이미 검증 완료 |

### 인터랙션 (User Actions)
- 모든 UI 상호작용 차단 (disabled)
- API 응답 도착까지 대기
- 성공 응답 → 세션 저장 → 역할별 대시보드 이동
- 실패 응답 → `03-에러` 상태 복원

### 비즈니스 룰
- 버튼 disabled 처리로 중복 제출 방지
- API 타임아웃 10초 설정 (초과 시 에러 상태 전환)
- 로딩 중 뒤로가기 → 폼 초기화 후 `01-기본` 복원

### 에지 케이스
- 네트워크 단절 → 타임아웃 후 에러 토스트 + `01-기본` 복원
- 응답 지연 2초 초과 → 스피너 유지 (UX 안내 없음, 타임아웃 전까지)

### 접근성 (A11y)
- 버튼 `aria-disabled="true"` + `aria-busy="true"`
- 스피너 `aria-label="로그인 처리 중"` `role="status"`
- `aria-live="polite"` 영역에 "로그인 중입니다" 공지

### 연결 화면
- 성공: SCR-101 대시보드 (역할별 분기)
- 실패: `03-에러` (폼 활성화 복원)
- 2FA 필요: `04-2FA입력`
