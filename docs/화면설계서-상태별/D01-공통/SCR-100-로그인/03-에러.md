## 📝 디스크립션

### 사용 시점 (Trigger)
- POST /auth/login → 401 잘못된 자격증명
- POST /auth/login → 423 계정 잠금
- POST /auth/login → 500 서버 오류
- 네트워크 타임아웃

### 필수 데이터
| 에러 코드 | 메시지 | 추가 액션 |
|-----------|--------|-----------|
| 401 | 이메일 또는 비밀번호가 올바르지 않습니다 | 재시도 |
| 423 | 계정이 잠겼습니다. 관리자에게 문의하세요 | 재설정 링크 강조 |
| 500 | 서버 오류가 발생했습니다 | 재시도 |
| network | 네트워크 연결을 확인해주세요 | 재시도 |

### 인터랙션 (User Actions)
1. 에러 배너 확인 후 필드 수정
2. 로그인 버튼 재클릭 → `02-제출중`
3. 423 에러 시 비밀번호 재설정 링크 클릭 → SCR-106

### 비즈니스 룰
- 에러 배너는 폼 최상단에 표시 (인라인 에러와 병행)
- 423 계정 잠금 시 비밀번호 입력 필드 disabled 처리
- 연속 5회 실패 시 클라이언트에서 30초 쿨다운 안내 (선택적)

### 에지 케이스
- 동시에 여러 에러 발생 불가 (단일 에러 코드 처리)
- 화면 새로고침 → `01-기본` 초기화 (에러 상태 휘발)

### 접근성 (A11y)
- 에러 배너 `role="alert"` `aria-live="assertive"`
- 에러 발생 시 에러 배너로 포커스 이동 (`focus()`)
- 에러 필드 `aria-invalid="true"` `aria-describedby="error-msg"`

### 연결 화면
- 재시도: `02-제출중` → 성공 시 대시보드
- 비밀번호 재설정: SCR-106
