## 📝 디스크립션

### 사용 시점 (Trigger)
- `/login` 직접 접근
- 세션 없음 또는 만료로 인한 리다이렉트
- 유효 세션 존재 시 → SCR-101 대시보드로 즉시 리다이렉트

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 이메일 | 사용자 입력 | RFC 5322 형식 검증 |
| 비밀번호 | 사용자 입력 | 공백 불가 |

### 인터랙션 (User Actions)
1. 이메일 입력 → 포커스 아웃 시 형식 검증
2. 비밀번호 입력 → 눈 아이콘으로 표시/숨김 토글
3. 로그인 버튼 또는 Enter → 제출 (`02-제출중` 전환)
4. 비밀번호 재설정 링크 → SCR-106

### 비즈니스 룰
- 유효 세션이 있으면 이 화면 진입 불가 (자동 리다이렉트)
- 로그인 성공 후 역할에 따라 분기:
  - `superAdmin` → `/super-dashboard`
  - 나머지 → `/` (지점 대시보드)
- 2FA 활성 계정 → 로그인 API 200 응답 후 `04-2FA입력` 전환

### 에지 케이스
- 이메일 미입력 제출 → "이메일을 입력하세요" 인라인 에러
- 비밀번호 미입력 제출 → "비밀번호를 입력하세요" 인라인 에러
- 서비스 점검(503) → `05-서비스점검` 상태 전환

### 접근성 (A11y)
- `<form>` role, autocomplete="email" / autocomplete="current-password"
- 에러 메시지 `aria-live="polite"` `role="alert"`
- 비밀번호 토글 버튼 `aria-label="비밀번호 표시/숨기기"`
- 탭 순서: 이메일 → 비밀번호 → 로그인 버튼 → 재설정 링크
