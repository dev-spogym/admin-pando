## 상태 진입 조건

- API 응답 실패:
  - 401 → `E401002` 세션 만료 → DLG-000 세션만료 모달
  - 403 → `E403001` 권한 없음 → SCR-108 403 페이지
  - 500 → `E500001` 서버 오류 → 에러 상태 (인라인)
  - 503 → `E503001` 서비스 불가 → 폴링 폴백 + 경고 배너
  - 네트워크 타임아웃 → `NETWORK`
  - WebSocket 끊김 → `WS_DISCONNECTED` (경고 배너만, 목록은 유지)

### 에러 코드별 UI/액션 분기

| errorCode | HTTP | UI 표현 | 액션 | 결과 |
|---|---|---|---|---|
| `E401002` | 401 | 인라인 X | DLG-000 세션만료 모달 open | 모달 → /login |
| `E403001` | 403 | 인라인 X | `router.push('/forbidden')` | SCR-108 |
| `E500001` | 500 | 중앙 에러 블록 + 재시도 버튼 | 수동 재시도 | 01-로딩 |
| `E503001` | 503 | 상단 경고 배너(`bg-amber-50`) + 목록은 마지막 캐시 표시 | 자동 폴링(30초) | 복구 시 02-정상 |
| `NETWORK` | — | 중앙 에러 블록 + "네트워크 확인" 문구 | 자동 재시도(3회, 2s/4s/8s) | 성공 시 02-정상 |
| `WS_DISCONNECTED` | — | 상단 경고 배너만(`bg-yellow-50`) + 목록 유지 | 재연결 시도 | 복구 시 배너 제거 |

## 역할별 차이

- **모든 역할 공통**: 에러 UI 동일
- **403 발생**: readonly가 manager 전용 액션 호출 시(전체읽음/전체삭제) → 토스트 + 버튼 disable 유지, 페이지 이탈 없음
- **슈퍼관리자**: 503 발생 시 "시스템 점검 여부 확인" 보조 링크 노출 (→ /audit-log)

## 상태 고유 인터랙션

1. **"다시 시도" 클릭** → `refetch()` → `01-로딩`
2. **DLG-000 세션만료 모달** (E401002):
   - "다시 로그인" → `useAuthStore.logout()` → `/login?redirect=/notifications`
   - 모달 외부 클릭 차단
3. **403 수신** → 즉시 `/forbidden` 이동 (이 상태 렌더 X)
4. **503 상단 배너**:
   - 메시지: "알림 서비스 연결이 원활하지 않습니다. 30초 후 자동 복구를 시도합니다."
   - [X] 닫기 버튼 → 배너 dismiss (세션 동안 재표시 안 함)
5. **네트워크 복구** → 자동 재시도 성공 시 토스트 "알림을 다시 불러왔습니다"
6. **WebSocket 재연결** → 배너 자동 제거 + 토스트 "실시간 알림이 복구되었습니다"

## 상태 고유 비즈니스 룰 델타

- **자동 재시도 전략**:
  - NETWORK: 3회 (2s, 4s, 8s 지수 백오프) → 실패 시 수동 버튼 노출
  - E500001: 1회 자동 → 실패 시 수동 버튼 노출
  - E503001: 30초 간격 폴링 (최대 5분), 성공 시 종료
  - WS_DISCONNECTED: 3회 재연결 (1s/3s/9s) → 실패 시 폴링 폴백 모드 유지
- **낙관적 업데이트 롤백**: 후속 API 실패 시(개별 읽음/삭제/전체읽음) `react-query` 캐시 롤백 + 에러 토스트(3초)
- **에러 상태에서도 실시간 구독 유지 시도**: 단, 500/NETWORK 이면 일시 중단
- **에러 로그 전송**: `Sentry.captureException(err, { extra: { userId, errorCode } })`
- **연속 실패 방지**: 같은 errorCode 3회 연속 실패 시 "관리자에게 문의" 추가 안내

## 에지 케이스

- **인증 만료 중 다른 탭 열려 있음** → 모든 탭에서 DLG-000 동기 표시 (BroadcastChannel)
- **403 → /forbidden 이동 후 뒤로가기** → `/notifications`로 돌아오면 재시도 실행
- **NETWORK 자동 재시도 중 수동 재시도** → 기존 요청 abort + 새 요청
- **503 배너 닫은 후 에러 재발** → 재발 시 배너 다시 표시 (dismiss는 성공 후 리셋)
- **WS 끊김 + 폴링도 실패** → 양쪽 배너 아닌 중앙 에러 블록으로 격상
- **토큰 갱신 성공 후 재시도** → 무한 루프 방지 가드(최대 2회)
