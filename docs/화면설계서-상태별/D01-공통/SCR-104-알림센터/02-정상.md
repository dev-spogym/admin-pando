## 📝 디스크립션

### 사용 시점 (Trigger)
- 알림 목록 로드 완료 + 1개 이상 알림 존재

### 필수 데이터
| 항목 | 테이블 | 조건 |
|------|--------|------|
| 알림 목록 | `notifications` | user_id = ? ORDER BY created_at DESC |
| 미읽음 수 | `notifications` COUNT | is_read = false |

### 인터랙션 (User Actions)
1. 알림 항목 클릭 → 읽음 처리 + link_url 이동
2. 탭 "미읽음" → 필터링
3. "모두 읽음" → PATCH /notifications/read-all
4. 실시간 수신 → 목록 상단 삽입

### 비즈니스 룰
- 알림 타입별 아이콘/색상 구분
- 읽음 처리는 낙관적 업데이트 적용
- 알림 30일 이후 자동 삭제 (서버 정책)
- Supabase realtime 채널 구독 (컴포넌트 언마운트 시 해제)

### 에지 케이스
- 알림 클릭 후 link_url 404 → 에러 페이지 이동
- "모두 읽음" 실패 → 에러 토스트 + 상태 롤백

### 접근성 (A11y)
- 목록 `role="feed"` `aria-label="알림 목록"`
- 미읽음 항목 `aria-label="{제목} (미읽음)"`
- 탭 `role="tablist"` / 각 탭 `role="tab"` `aria-selected`

### 연결 화면
- 알림 link_url → 각 도메인 화면
- 헤더 뱃지: 미읽음 수 실시간 반영
