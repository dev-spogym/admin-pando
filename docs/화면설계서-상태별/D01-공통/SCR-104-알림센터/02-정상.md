## 상태 고유 인터랙션

1. **항목 클릭**
   - `!isRead` → 낙관적 업데이트(`isRead=true`) → PATCH `/notifications/:id/read` → 실패 시 롤백 + 에러 토스트
   - `linkUrl` 존재 → `router.push(linkUrl)`
   - `linkUrl` 없음 → 머무름(읽음만 처리)
2. **삭제 버튼 클릭**
   - `stopPropagation` → 낙관적 제거 → DELETE `/notifications/:id` → 실패 시 복구 + 에러 토스트
3. **"모두 읽음" 클릭** (manager+)
   - Confirm 없이 즉시 PATCH `/notifications/read-all` → 성공 토스트 "모든 알림을 읽음 처리했습니다" → `unreadCount = 0`
4. **"설정" 클릭** → `router.push('/profile?tab=notifications')` (SCR-105)
5. **탭 전환** → `setActiveTab` → `useQuery` refetch (로딩 스켈레톤 X, `keepPreviousData` 적용 → stale 데이터 표시 + 상단 프로그레스)
6. **페이지네이션** → `setPage(n)` → 새 쿼리
7. **Enter 키** (항목 포커스) → 클릭과 동일
8. **Delete 키** (항목 포커스) → 삭제와 동일
9. **실시간 수신** → `05-실시간수신` 으로 일시 전환 후 복귀

## 상태 고유 비즈니스 룰 델타

- **정렬**: `createdAt DESC` (최신이 상단). 동시각 생성 시 `id DESC`.
- **미읽음 우선 표시**: `activeTab==='all'`일 때 1차 `isRead ASC` 2차 `createdAt DESC`.
- **낙관적 업데이트**: 읽음/삭제는 UI 즉시 반영, 실패 시 롤백.
- **알림 타입별 분기**: `type.startsWith('audit.')`이면 staff/front/readonly에게 미표시(서버 필터).
- **미읽음 뱃지 동기화**: `unreadCount` 변경 시 Zustand store 갱신 → 헤더/사이드바 뱃지 즉시 반영.
- **토스트 중복 방지**: 같은 ID 토스트는 `toast.update()` 사용.

## 에지 케이스

- **미읽음 12개, 페이지 20개 중 미읽음 탭 전환**: `total` 기준 재조회, `page=1` 리셋
- **시간 경계**: "1분 전" → "방금 전"(60초 미만), "1시간 전" → "N시간 전", 24h 초과 시 `YYYY-MM-DD HH:MM`
- **linkUrl이 현재 화면**: `router.push` 무시 + 읽음만 처리
- **linkUrl이 외부 URL**: `https://` 포함 → 새 탭 (`window.open`)
- **삭제 중 새 WS INSERT 도착**: 삭제 애니메이션 완료 후 prepend
- **네트워크 플래키**: `react-query` `retry: 1`로 한 번만 재시도
- **낙관적 업데이트 시 페이지 벗어남**: AbortController로 요청 취소 (링크 이동이 우선)

항목 클릭 핸들러:
const handleClick = async (item: Notification) => {
  if (!item.isRead) {
    // 낙관적 업데이트
    qc.setQueryData(['notifications', activeTab, page, branchId], (old: any) => ({
      ...old,
      items: old.items.map((n: Notification) => n.id === item.id ? { ...n, isRead: true } : n),
      unreadCount: Math.max(0, old.unreadCount - 1),
    }));
    useNotificationStore.getState().decrementUnread();
    try {
      await fetch(`/api/notifications/${item.id}/read`, { method: 'PATCH' });
    } catch (e) {
      // 롤백
      qc.invalidateQueries({ queryKey: ['notifications'] });
      toast.error('읽음 처리에 실패했습니다');
    }
  }
  if (item.linkUrl) {
    if (item.linkUrl.startsWith('http')) window.open(item.linkUrl, '_blank');
    else router.push(item.linkUrl);
  }
};

전체 읽음 핸들러 (manager+ only):
const handleMarkAllRead = async () => {
  if (unreadCount === 0) return;
  setIsMarkingAll(true);
  try {
    await fetch('/api/notifications/read-all', { method: 'PATCH' });
    qc.invalidateQueries({ queryKey: ['notifications'] });
    useNotificationStore.getState().resetUnread();
    toast.success('모든 알림을 읽음 처리했습니다');
  } catch (e: any) {
    if (e.errorCode === 'E403001') toast.error('접근 권한이 없습니다');
    else toast.error('처리에 실패했습니다');
  } finally {
    setIsMarkingAll(false);
  }
};

삭제 핸들러:
const handleDelete = async (id: string) => {
  const prev = qc.getQueryData(['notifications', activeTab, page, branchId]);
  qc.setQueryData(['notifications', activeTab, page, branchId], (old: any) => ({
    ...old,
    items: old.items.filter((n: Notification) => n.id !== id),
    total: Math.max(0, old.total - 1),
  }));
  try {
    await fetch(`/api/notifications/${id}`, { method: 'DELETE' });
  } catch (e) {
    qc.setQueryData(['notifications', activeTab, page, branchId], prev);
    toast.error('삭제에 실패했습니다');
  }
};
