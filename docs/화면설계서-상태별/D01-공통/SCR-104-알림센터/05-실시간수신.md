```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-104 알림 센터 — 상태: 실시간 알림 수신

- 새 알림 슬라이드인 애니메이션 (상단에서 아래로)

컴포넌트 구조:
1. 새 알림 항목 (목록 상단 삽입):
   - 슬라이드인 애니메이션: translate-y-[-100%] → translate-y-0 (transition 300ms)
   - 미읽음 스타일 (bg-blue-50 border-l-4 border-blue-500)
   - "새 알림" 뱃지 (absolute, top-2 right-2, text-xs bg-blue-500 text-white rounded-full px-2)
2. 헤더 미읽음 뱃지 카운트 증가

데이터:
- Supabase realtime channel: supabase.channel('notifications').on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'notifications', filter: `user_id=eq.${user.id}` }, handler)
- newNotification: Notification
- 타입: interface Notification (동일)

인터랙션:
- 새 알림 삽입 → 목록 상단 추가 + 미읽음 뱃지 +1
- 새 알림 클릭 → 읽음 처리 + link_url 이동
- 슬라이드인 후 2초 → "새 알림" 뱃지 제거

사용 유틸:
- useAuthStore, supabase, toast, moveToPage, lucide-react
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- Supabase realtime `notifications` 테이블 INSERT 이벤트 수신
- 현재 사용자의 알림 센터 화면 열람 중

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 새 알림 데이터 | Supabase realtime payload | INSERT 이벤트 |

### 인터랙션 (User Actions)
1. 새 알림 자동 삽입 (사용자 액션 불필요)
2. 새 알림 클릭 → 읽음 처리 + 이동
3. 슬라이드인 완료 후 일반 `02-정상` 상태로 귀환

### 비즈니스 룰
- 알림 센터 미열람 중(다른 화면) → 헤더 뱃지만 업데이트 (토스트 표시)
- 알림 센터 열람 중 → 목록 자동 업데이트 (토스트 미표시)
- 컴포넌트 언마운트 시 realtime 채널 unsubscribe 필수

### 에지 케이스
- 빠른 연속 알림(< 1초) → 순서대로 삽입
- 탭 비활성 중 수신 → 탭 활성화 시 반영

### 접근성 (A11y)
- 새 알림 삽입 시 `aria-live="polite"` 공지 ("새 알림: {제목}")
- 헤더 뱃지 `aria-label="미읽음 알림 N개"`

### 연결 화면
- 알림 link_url → 각 도메인 화면
