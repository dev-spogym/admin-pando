```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-105 프로필/계정 — 상태: 저장 중

- 저장 버튼: disabled + Loader2 스피너 + "저장 중..."
- 모든 폼 필드: disabled (pointer-events-none)
- 카드 영역 opacity-75

컴포넌트 구조:
1. PageHeader: "프로필 편집" + 저장 버튼(disabled, 스피너) + 취소 버튼(disabled)

- isSaving: boolean = true
- PATCH /users/{id} 호출 중
- 성공: toast.success("프로필이 저장되었습니다.") + isEditing=false
- 실패: toast.error("저장에 실패했습니다.") + isSaving=false

인터랙션:
- 모든 UI 상호작용 차단
- 성공 → `02-읽기모드` + 성공 토스트
- 실패 → `03-편집모드` + 에러 토스트

사용 유틸:
- useAuthStore, supabase, toast, lucide-react (Loader2)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `03-편집모드`에서 저장 버튼 클릭 + 검증 통과

### 필수 데이터
- PATCH 요청 payload: { name, phone, avatarUrl? }

### 인터랙션 (User Actions)
- 모든 상호작용 차단 (저장 완료까지 대기)

### 비즈니스 룰
- 아바타 변경 시 Supabase Storage 업로드 후 URL 저장
- 저장 성공 → useAuthStore 사용자 정보 업데이트
- 저장 실패 → 원본 데이터 복원 없음 (폼값 유지)

### 에지 케이스
- 네트워크 단절 중 저장 → 타임아웃 후 에러
- 아바타 업로드 성공 + 프로필 저장 실패 → 孤立 이미지 (Storage 롤백 고려)

### 접근성 (A11y)
- 저장 버튼 `aria-busy="true"` `aria-disabled="true"`
- `aria-live="polite"` 저장 상태 공지
