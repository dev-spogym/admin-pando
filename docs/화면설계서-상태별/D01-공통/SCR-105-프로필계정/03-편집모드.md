## 🧩 바이브코딩 프롬프트
```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-105 프로필/계정 — 상태: 편집 모드

파일: src/app/(dashboard)/profile/page.tsx (isEditing=true 분기)

레이아웃:
- AppLayout (사이드바 + 헤더)
- max-w-2xl mx-auto
- 편집 중 이탈 시도 → DLG-002 이탈 경고

컴포넌트 구조:
1. PageHeader: "프로필 편집" + "저장" 버튼(primary) + "취소" 버튼(ghost)
2. 아바타 편집 영역:
   - 아바타 + 오버레이 (Camera 아이콘)
   - 이미지 업로드 input (hidden, accept="image/*")
3. 편집 가능 폼 필드:
   - 이름 Input (required)
   - 전화번호 Input (type="tel")
   - (이메일은 disabled, 읽기 전용 표시)
4. 변경사항 dirty 감지 (useEffect로 원본 비교)
5. 하단 저장/취소 버튼 (중복, 모바일 대응)

데이터:
- formData: { name: string; phone: string }
- originalData: { name: string; phone: string }
- isDirty: boolean (formData !== originalData)
- 타입: interface ProfileForm { name: string; phone: string }
- 저장: PATCH /users/{id} { name, phone }

인터랙션:
- 값 변경 → isDirty = true
- 이탈 시도 (isDirty=true) → DLG-002 이탈 경고
- 저장 버튼 → `04-저장중`
- 취소 버튼 (isDirty=false) → `02-읽기모드`
- 취소 버튼 (isDirty=true) → DLG-002 또는 직접 취소

사용 유틸:
- useAuthStore, supabase, toast, lucide-react (Camera, Save, X)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-읽기모드`에서 "편집" 버튼 클릭

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 현재 프로필 | `02-읽기모드` 데이터 | 폼 초기값 |
| isDirty 상태 | 클라이언트 | 원본 대비 변경 감지 |

### 인터랙션 (User Actions)
1. 필드 수정 → isDirty = true
2. 저장 버튼 → `04-저장중`
3. 취소 (변경 없음) → 즉시 `02-읽기모드`
4. 취소 (변경 있음) → DLG-002 이탈 경고
5. 이탈 시도 (뒤로가기/메뉴클릭, isDirty=true) → DLG-002

### 비즈니스 룰
- 이메일 변경 불가 (Input disabled)
- 이름 필수 (공백 불가)
- 전화번호 형식 검증: 010-XXXX-XXXX
- 아바타 이미지: 최대 5MB, JPG/PNG/WebP

### 에지 케이스
- 이름 공백 제출 → "이름을 입력하세요" 인라인 에러
- 전화번호 형식 오류 → "올바른 전화번호를 입력하세요" 에러
- 아바타 용량 초과 → "5MB 이하의 이미지를 선택해주세요" 토스트

### 접근성 (A11y)
- 편집 폼 `role="form"` `aria-label="프로필 편집"`
- 필수 필드 `aria-required="true"`
- 에러 메시지 `aria-describedby` 연결

### 연결 화면
- 저장: `04-저장중`
- 취소/이탈: `02-읽기모드` (DLG-002 경유 가능)
