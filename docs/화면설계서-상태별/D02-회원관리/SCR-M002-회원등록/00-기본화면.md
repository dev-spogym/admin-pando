## 1. 화면 목적 (Why)

새로운 회원을 시스템에 등록한다. 2단계 스텝 폼(Step 1: 필수 정보, Step 2: 추가 정보)으로 구성되어 빠른 등록과 상세 정보 입력을 분리한다.
- Step 1: 이름/성별/연락처/회원구분/생년월일/키 + 관리정보(담당 FC/트레이너/경로)
- Step 2: 이메일/주소/메모/닉네임/광고성 동의 등

등록 성공 시 "바로 결제" 유도로 POS/전자계약으로 자연스럽게 연결.

### 7.1 Zod 스키마
```ts
// src/schemas/member.ts
export const memberFormSchema = z.object({
  name: z.string().regex(/^[가-힣a-zA-Z一-鿿\s]{2,20}$/, '한글/영문/한자 2~20자로 입력하세요'),
  gender: z.enum(['male','female'], { required_error: '성별을 선택하세요' }),
  phone: z.string().regex(/^010-\d{4}-\d{4}$/, '올바른 연락처를 입력하세요 (010-xxxx-xxxx)'),
  memberType: z.string().min(1, '회원구분을 선택하세요').default('일반'),
  birthDate: z.string().optional().refine(v => !v || new Date(v) < new Date(), '올바른 날짜를 입력하세요 (미래 날짜 불가)'),
  height: z.string().optional().refine(v => !v || (+v >= 100 && +v <= 250), '100~250cm 사이로 입력하세요'),
  referralSource: z.string().optional(),
  companyName: z.string().optional(),
  email: z.union([z.string().email('올바른 이메일을 입력하세요'), z.literal('')]).optional(),
  address: z.string().optional().refine(v => !v || v.length >= 5, '주소를 5자 이상 입력하세요'),
  addressDetail: z.string().optional(),
  notes: z.string().max(500, '500자 이내로 입력하세요').optional(),
  fc: z.string().optional(),
  trainer: z.string().optional(),
  visitPath: z.string().optional(),
  exerciseGoal: z.string().optional(),
  nickname: z.string().optional(),
  company: z.string().optional(),
  marketingConsent: z.boolean().default(false),
  attendanceNumber: z.string().optional(),
  profileImage: z.string().nullable().default(null),
});
export type MemberForm = z.infer<typeof memberFormSchema>;
```

### 7.4 중복확인 로직
```ts
async function handleDuplicateCheck() {
  const ok = await trigger('phone'); if (!ok) return;
  const phone = watch('phone');
  const { duplicate } = await checkPhoneDuplicate(phone);
  setPhoneChecked(true); setPhoneDuplicate(duplicate);
  if (duplicate) toast.error('이미 등록된 번호입니다.');
}
```

### 7.5 Step 1 → Step 2 전환
```ts
async function handleNext() {
  const ok = await trigger(['name','gender','phone','memberType','birthDate','height']);
  if (!ok) return;
  if (!phoneChecked) return toast.error('전화번호 중복확인이 필요합니다.');
  setStep('step2'); window.scrollTo(0,0);
}
```

## 8. 비즈니스 룰

1. Step 1 필수: 이름, 성별, 연락처, 회원구분. Step 2는 모두 선택.
2. 전화번호: `formatPhone()` 자동 포맷 `010-xxxx-xxxx`. 중복확인 필수.
3. 번호 변경 시 `phoneChecked/phoneDuplicate` 초기화.
4. 법인 회원구분 선택 시 `companyName` 필드 추가 노출.
5. 프로필 이미지: jpeg/png, 5MB 이하, Supabase Storage `profiles/members/{branchId}/{timestamp}_{filename}`.
6. 저장 시 서버 추가 검증 — 성공 → 상세 페이지 이동 + "바로 결제" 토스트.
7. `status = 'INACTIVE'` 초기 (이용권 구매 시 ACTIVE 전환 — 상태전이도 §1).
8. `branchId`는 서버에서 JWT로 강제.
9. 감사 로그: `AUDIT.MEMBER_CREATE` (actorId, memberId).
10. `handleCancel`: `isDirty` true면 DLG-M007 표시, false면 즉시 `/members`.
11. `handleReset`: DLG-M008 표시 → 확인 시 `reset()` + `phoneChecked/phoneDuplicate` false + Step 1로.
12. 자동 저장(🆕 미구현): 5분 주기 localStorage에 draft.
