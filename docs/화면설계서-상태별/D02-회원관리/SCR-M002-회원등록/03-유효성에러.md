## 📝 디스크립션

### 사용 시점
- "다음" 버튼 클릭 시 Step1 필드 유효성 검증 실패
- react-hook-form trigger() 실행 후 errors 존재

### 필수 데이터
| 항목 | 상태 |
|------|------|
| formState.errors | 1개 이상 에러 |
| 에러 필드 | border-red-500, 에러 메시지 표시 |

### 인터랙션
1. 에러 필드 수정 → onChange 시 에러 즉시 제거
2. 모든 에러 해소 + 다음 클릭 → `06-Step2`

### 비즈니스 룰
- 필수 필드: 이름, 연락처(중복확인 완료)
- 선택 필드 형식 오류도 에러 표시 (이메일 등)
- 에러 메시지는 각 필드 바로 아래 표시

### 에지 케이스
- 다중 에러 동시 발생: 모든 에러 필드에 표시
- 연락처 중복 에러는 DLG-M006 인라인 처리와 연동

### 접근성
- `aria-invalid="true"` + `aria-describedby` 연결
- 에러 요약 `role="alert"` (폼 상단에 에러 개수 고지 고려)

### 연결 화면
- 에러 수정 → `02-입력중`
