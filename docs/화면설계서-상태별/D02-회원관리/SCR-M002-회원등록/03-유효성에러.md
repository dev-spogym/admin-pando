## 인터랙션
| 액션 | 동작 |
|---|---|
| 에러 필드 수정 → Blur | 재검증, 통과 시 에러 해제 |
| [다음] 재클릭 | 모든 필수 필드 재검증 |
| [저장] (Step 2) | 400 응답 시 첫 에러 필드로 `scrollIntoView` + focus |
| 상단 배너 클릭 | 첫 에러 필드로 포커스 이동 |
| ESC (IME 아님) | 에러 요약 배너 닫기 (선택) |

## 룰 델타
1. `mode: 'onBlur'` — 실시간 타이핑 중 에러 노출 안 함.
2. 다중 에러 시 상단 배너에 건수 표시 + "자세히 보기" 클릭 시 첫 에러 포커스.
3. 첫 에러 필드로 `scrollIntoView({ behavior: 'smooth', block: 'center' })`.
4. 서버 400 에러 → RHF `setError(field, { message })`로 맵핑.
5. SR `role="alert" aria-live="assertive"`로 에러 공지.
6. 폼 제출 실패 시 analytics `member_new_validation_failed` (errors 배열) 기록.

## 에지
- 중복확인 후 번호 재수정 → 에러 자동 해제 + phoneChecked=false.
- 중복 + 형식 오류 동시 → 형식 오류 먼저 해결 후 중복 확인.
- 한국어 IME 조합 중 Blur → 잘못된 검증 방지 (compositionend 이후 trigger).
- 자동완성 → setValue 시 isDirty 갱신 + 재검증.

## 🧩 바이브코딩 (델타)
```
// RHF + zodResolver로 자동 처리

// 첫 에러 필드로 scroll
useEffect(() => {
  const firstError = Object.keys(errors)[0];
  if (firstError) {
    document.getElementById(firstError)?.scrollIntoView({ behavior: 'smooth', block: 'center' });
    document.getElementById(firstError)?.focus();
  }
}, [errors]);

// 서버 400 맵핑
onError: (err) => {
  if (err.errorCode?.startsWith('E400')) {
    const fieldMap: Record<string,string> = {
      E400100: 'name', E400101: 'memberType', E400102: 'gender',
      E400103: 'phone', E400104: 'email', E400105: 'notes'
    };
    setError(fieldMap[err.errorCode], { message: err.message, type: 'server' });
  } else if (err.errorCode === 'E409100') {
    setError('phone', { message: '이미 등록된 번호입니다.', type: 'duplicate' });
    setPhoneChecked(true); setPhoneDuplicate(true);
  }
}

// 상단 에러 요약 배너
{Object.keys(errors).length > 1 && (
  <div role="alert" aria-live="assertive"
       className="bg-red-50 border border-red-200 text-red-700 rounded-lg p-3 text-sm flex items-start gap-2">
    <AlertCircle className="w-4 h-4 mt-0.5" />
    <div>
      입력 내용을 확인해주세요 ({Object.keys(errors).length}건)
      <button className="ml-2 underline" onClick={scrollToFirstError}>자세히 보기</button>
    </div>
  </div>
)}
```
