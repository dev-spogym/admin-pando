사용 유틸: supabase.storage, lucide-react (Loader2), toast

## 📝 디스크립션

### 사용 시점
- 프로필 이미지 파일 선택 후 Supabase Storage 업로드 진행 중

### 필수 데이터
| 항목 | 상태 |
|------|------|
| isImageUploading | true |
| previewUrl | 선택한 파일의 로컬 ObjectURL |
| 업로드 진행률 | (선택) 0~100% |

### 인터랙션
1. 업로드 완료 → photoUrl 저장 → `02-입력중`
2. 업로드 실패 → toast.error → 이미지 영역 초기화 → `02-입력중`

### 비즈니스 룰
- 이미지 크기 제한: 5MB 이하
- 허용 형식: jpg, jpeg, png, webp
- 업로드 경로: `member-photos/{centerId}/{uuid}.{ext}`

### 에지 케이스
- 네트워크 끊김 중 업로드: Supabase Storage 자동 재시도 후 실패 처리
- 5MB 초과 파일: 업로드 시작 전 클라이언트 사이드 검증

### 접근성
- 이미지 영역 `aria-busy="true"` + `aria-label="이미지 업로드 중"`

### 연결 화면
- 완료/실패: `02-입력중`

### 다이어그램 참조
- 엣지: `E_IMG_START_01` → `E_IMG_DONE_01` / `E_IMG_FAIL_01`
- 상태: `STATE_IMG_LOADING`
