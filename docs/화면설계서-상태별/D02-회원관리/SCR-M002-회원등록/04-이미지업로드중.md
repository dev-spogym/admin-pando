## 인터랙션
| 액션 | 동작 |
|---|---|
| 파일 재선택 | 차단 (disabled) |
| 취소 (🆕 지원 가능) | `AbortController.abort()` → 이전 이미지 복원 |
| 다른 필드 입력 | 정상 (업로드 백그라운드) |
| [저장] 클릭 | 업로드 완료 대기 권장. 또는 `profileImage = null`로 저장 후 나중에 추가 |

## 룰 델타
1. **파일 검증 (클라이언트)**:
   - 타입: `image/jpeg`, `image/png` 만 허용 (`accept` 속성 + JS 재검증)
   - 크기: ≤ 5MB (`file.size > 5 * 1024 * 1024` → 차단)
   - 차단 시 `04` 상태 미진입 → toast.error
2. **경로**: `profiles/members/{branchId}/{Date.now()}_{filename}` (충돌 방지)
3. **Supabase Storage**:
   ```ts
   const { data, error } = await supabase.storage
     .from('profiles')
     .upload(path, file, { cacheControl: '3600', upsert: false });
   ```
4. **성공**: `supabase.storage.from('profiles').getPublicUrl(path).data.publicUrl` → `setValue('profileImage', url)` → toast.success + `02-입력중`
5. **실패**: `setValue('profileImage', prev)` + toast.error(`이미지 업로드 실패: ${error.message}`) + `02-입력중`
6. **진행률**: Supabase JS SDK는 현재 native progress 미지원 → XHR 또는 multipart fetch로 구현 (선택)
7. **EXIF 제거**: 클라이언트에서 `canvas` 기반 재인코딩 (개인정보 보호, 🆕)
8. **리사이즈**: 원본 > 1024×1024 시 클라이언트 리사이즈 후 업로드 (🆕)
9. **대용량**: 5MB 초과 시 즉시 toast.error "파일 크기는 5MB 이하여야 합니다." + 업로드 스킵.

## 에지
- 네트워크 단절 중 업로드 → 10초 타임아웃 → 실패 처리.
- 같은 파일 재선택 (input 값 동일) → onChange 안 터짐 → `e.target.value = ''` 트릭으로 재트리거 허용.
- 이전 이미지 존재 + 업로드 실패 → 이전 이미지 복원.
- 브라우저 HEIC 파일 선택 (iOS 기본) → 지원 안 함 + toast.error "JPG/PNG만 업로드 가능합니다."

## 🧩 바이브코딩 (델타)
```
// ProfileImageUploader 내부

async function handleFileSelect(e: ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0];
  e.target.value = ''; // 같은 파일 재선택 허용
  if (!file) return;

  if (!['image/jpeg','image/png'].includes(file.type)) {
    return toast.error('JPG/PNG 형식만 업로드 가능합니다.');
  }
  if (file.size > 5 * 1024 * 1024) {
    return toast.error('파일 크기는 5MB 이하여야 합니다.');
  }

  setIsUploadingImage(true);
  const prev = value;
  const path = `profiles/members/${branchId}/${Date.now()}_${file.name}`;
  const res = await uploadFile('profiles', path, file);
  setIsUploadingImage(false);

  if (res.error) {
    onChange(prev);
    toast.error(`이미지 업로드 실패: ${res.error.message}`);
  } else {
    onChange(res.publicUrl);
    toast.success('프로필 이미지가 등록되었습니다.');
  }
}

// UI
<label className="relative w-[120px] h-[120px] rounded-full cursor-pointer group
                  bg-gray-100 overflow-hidden block" aria-label="프로필 사진 선택">
  {value && <img src={value} alt="" className={cx('w-full h-full object-cover', isUploadingImage && 'blur-sm')} />}
  {!value && !isUploadingImage && <Camera className="w-10 h-10 text-gray-400 absolute inset-0 m-auto" />}
  {isUploadingImage && (
    <div className="absolute inset-0 bg-black/30 flex items-center justify-center">
      <Loader2 className="w-8 h-8 text-white animate-spin" />
    </div>
  )}
  <input type="file" accept="image/jpeg,image/png" className="sr-only"
         onChange={handleFileSelect} disabled={isUploadingImage} />
</label>
```
