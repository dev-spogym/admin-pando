## 상태 고유 인터랙션
1. 모든 입력 차단
2. `beforeunload`: 창 닫기 경고 (선택)
3. 타임아웃 30초: 자동 에러 전환

## 비즈니스 룰 델타
1. **멱등성 보장 필요**: 백엔드에서 `member.deletedAt != null` 재진입 시 409/no-op 처리
2. **트랜잭션**: 연관 테이블 FK 업데이트/해제는 서버 트랜잭션으로 보장
3. **감사 로그**: 성공 전후 양쪽 기록 (시도/완료)
4. **낙관적 UI 금지**: 삭제는 destructive → 서버 확정 후 반영

## 에지 케이스
- **서버 멱등 처리**: 중복 호출 시 409 E409xxx 방지 (기존 deletedAt 존재 시 200 반환 또는 409 적절 처리)
- **네트워크 단절 중간**: 요청 완료 여부 모호 → 재시도 전 부모 refetch로 확인
- **동시 세션 삭제**: 다른 사용자가 이미 완료 → 404 E404100

## 바이브코딩 프롬프트 (델타)
```
상태: submitting
조건: mutation.isPending === true
UI 잠금:
  - <Dialog.Close disabled>
  - input readOnly className+=' bg-gray-50 text-gray-500'
  - 버튼 disabled
  - 확인 버튼: <Loader2 animate-spin/> "삭제 중..."
  - onEscapeKeyDown / onPointerDownOutside preventDefault()
Unmount 가드:
  useEffect(() => {
    if (!mutation.isPending) return;
    const h = (e: BeforeUnloadEvent) => { e.preventDefault(); e.returnValue=''; };
    window.addEventListener('beforeunload', h);
    return () => window.removeEventListener('beforeunload', h);
  }, [mutation.isPending]);
Timeout:
  AbortSignal.timeout(30_000) 또는 fetch + setTimeout
```
