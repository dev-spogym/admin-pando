## 상태 진입 조건
1. SCR-M001 `selectedRows.size > 0`
2. 역할: `superAdmin | primary | owner | manager`
3. BulkActionBar "상태 변경" 버튼 클릭 → `setIsOpen(true)`
4. `selectedIds`가 모두 `user.branchId` 범위 내 (클라 1차 검증, 서버 최종)

## 역할별 차이
| 역할 | 노출 옵션 | 비고 |
|---|---|---|
| superAdmin/primary/owner | 6개 전부 | ACTIVE, HOLDING, SUSPENDED, EXPIRED, INACTIVE, WITHDRAWN |
| manager | 5개 | WITHDRAWN 제외 |
| 그 외 | 다이얼로그 진입 불가 | 트리거 자체 미노출 |

## 상태 고유 인터랙션
1. **RadioItem 클릭/방향키** → `setStatus(value)` → `02-선택중` 전이
2. **Tab**: 닫기(X) → 라디오(방향키 내부 이동) → textarea → 취소 → 변경 확인
3. **Esc**: 모달 닫기 (제출 중 아니므로 허용)
4. **Overlay 클릭**: 모달 닫기
5. **X 클릭**: 모달 닫기
6. **취소 클릭**: 모달 닫기, selectedRows 유지

## 비즈니스 룰 델타
- 모달 열리는 순간 `setStatus(undefined)` + `setReason('')` + `setErrorCode(null)` 초기화
- `selectedIds.length === 0` 이면 즉시 `onOpenChange(false)` + `toast.error('선택된 회원이 없습니다')` (방어 코드)
- 감사 로그: `AUDIT.DLG_OPEN` 기록 (선택 사항)

## 에지 케이스
- **selectedIds 포함 회원 중 일부가 다른 지점**: 서버가 최종 거부. 클라는 경고 배너 프리플라이트 권장.
- **초기 포커스 안 먹힘** (스크린 리더): `autoFocus`를 첫 라디오에 명시.
- **React StrictMode 이중 마운트**: `open` prop 기반 제어이므로 문제 없음.

## 바이브코딩 프롬프트 (델타)
```
상태: open (초기)
초기값:
  status = undefined
  reason = ''
  errorCode = null
  isSubmitting = false
확인 버튼 disabled={!status || isSubmitting}
열림 effect:
  useEffect(() => { if (open) { setStatus(undefined); setReason(''); setErrorCode(null); } }, [open])
방어: if (selectedIds.length === 0) { onOpenChange(false); toast.error('대상 회원 없음'); return null; }
```
