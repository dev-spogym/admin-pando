## 상태 진입 조건

- `02-입력중`에서 `canSubmit === true` 상태로 "조정" 클릭
- `mut.mutate(form)` 호출 직후 `mut.isPending === true`

## 상태 고유 비주얼 델타

- 모든 입력 필드 `readOnly` + `opacity-60` + `cursor-not-allowed`
- 유형 radio `disabled`
- "조정" 버튼: `<Loader2 className="size-4 animate-spin" />` + 텍스트 `"처리 중..."`
- "취소" 버튼 `disabled` (ESC/배경/X 모두 차단)
- 헤더 X 버튼 `disabled`
- 배경 opacity 유지, 추가 시각 신호 없음

## 상태 고유 인터랙션

1. **ESC 차단**: `e.key === 'Escape'` 이벤트 무시
2. **배경 클릭 차단**: `onClick` no-op
3. **X 버튼 차단**: `disabled` 속성
4. **Enter 키 차단**: 버튼 `disabled`로 자동 차단
5. **페이지 언로드 경고**: `beforeunload` 이벤트 등록하여 "처리 중인 작업이 있습니다" 경고 (optional)

## 역할별 차이

- 마스터와 동일

## 상태 고유 비즈니스 룰 델타

- 타임아웃: 기본 10초 (`signal: AbortSignal.timeout(10_000)`) 초과 시 네트워크 에러로 처리
- 중복 제출 방지: `mut.isPending` 기반 버튼 disabled + 이벤트 무시
- 취소 API 없음: 일단 제출된 요청은 중단 불가 (서버는 멱등성 보장하지 않음)
- 로딩 최소 시간: UX상 150ms 이하로 끝나도 스피너 잠깐 보이게 하지 말 것 (즉시 결과 표시 허용)

## 에지 케이스

- **네트워크 끊김 중 제출**: 30초 대기 후 NETWORK 에러 → `04-성공또는실패`
- **사용자 탭 전환**: 백그라운드에서도 요청 계속 진행, 복귀 시 결과 반영
- **탭 닫기**: 브라우저 기본 경고 외 처리 불가 (요청 취소 또는 진행)
- **중복 클릭**: 버튼 disabled로 방어, 이벤트 레벨에서도 `if (mut.isPending) return;`
