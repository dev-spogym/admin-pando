## 상태 진입 조건

- SCR-M004 > 상세내역 탭 > 마일리지 서브탭에서 "수동 조정" 버튼 클릭
- `hasFeature(role, 'memberMileageAdjust')` 통과 (superAdmin/primary/owner/manager)
- `member.status !== 'WITHDRAWN'` (탈퇴 회원 차단 — 부모 가드)
- `useSessionStore.open === false`

## 상태 고유 비주얼 델타

- 기본 렌더(마스터 §2) 그대로
- 유형 radio 기본값 `EARN` 선택
- 포인트 필드 placeholder `예: 100`, 값 비어있음
- 사유 필드 placeholder `예: 이벤트 보상, 결제 오류 정정`
- "조정" 버튼 `disabled` (points=0 또는 reason='')
- 현재 잔액 카드 `bg-gray-50` + `{formatNumber(member.mileage)} P`

## 상태 고유 인터랙션

1. **오픈 직후**: 유형 radio "적립" 자동 포커스 (아님 → 첫 입력 필드 `points`로도 가능하나 정책상 radio 권장)
2. **Tab 순환**: 유형 radio → 포인트 → 사유 → 취소 → 조정 → X → 유형
3. **ESC / 배경 / X**: `onClose()`
4. **필드 변경**: `02-입력중` 상태로 전환 (본질적으로는 같은 DOM)
5. **빈 필드 제출 시도**: Enter 눌러도 `disabled`로 차단

## 역할별 차이

- `superAdmin/primary`: 타 지점 회원의 마일리지도 조정 가능 (branchId 서버 검증)
- `owner/manager`: 본인 지점 회원만
- `fc/trainer/staff/front/readonly`: 부모 화면에서 버튼 자체 미렌더

## 상태 고유 비즈니스 룰 델타

- 열림 즉시 회원 최신 잔액을 `GET /members/:id` 또는 props로 동기화 — 최신이 아니면 refetch 후 렌더
- 다른 탭에서 동시 조정 시 동시성 이슈 방지: 서버는 `balance_version` optimistic lock 권장 (서버 구현부)
- i18n: 본문의 `"{name}" 회원의 ...`에서 한국어 조사 처리

## 에지 케이스

- **잔액 0 USE 시도**: 차단 (type=USE + balance=0 시 radio USE 자체는 선택 가능하나 제출 시 E422100)
- **회원 탈퇴 중간 변경**: 오픈 후 회원이 탈퇴되면 서버 422101 → 토스트 + 다이얼로그 닫기
- **권한 중간 박탈**: 서버 403으로 방어
