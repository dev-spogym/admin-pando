## 상태 진입 조건
- `mutation.isPending === false`
- `mutation.isSuccess` 또는 `mutation.isError`

## 상태 고유 비주얼 델타

### 성공 (SUCCESS)
- 토스트 `toast.success("삭제되었습니다.")` 3초
- 모달 즉시 닫힘
- 상담 목록에서 해당 행 제거(refetch 또는 낙관적)

### 실패 (FAIL)
- 모달 동작:
  - 403/404 → 닫기 + 토스트
  - 500/NETWORK → 유지 + 토스트 + 재시도 가능(버튼 활성 복귀)
- 토스트 3초, 에러코드별 분기

## 상태 고유 인터랙션

### 성공
- 없음 (자동 정리)

### 실패(유지)
1. **재시도**: "삭제" 재클릭 → `03-제출중`
2. **취소**: 닫힘
3. **치명 자동 닫기**: 2초 후 `onClose()`

## 상태 고유 비즈니스 룰 델타

### 성공 후처리
```ts
onSuccess: () => {
  qc.invalidateQueries({ queryKey: ['consultations', memberId] });
  qc.invalidateQueries({ queryKey: ['member', memberId] }); // 상담 카운트
  toast.success('삭제되었습니다.');
  onClose();
}
```

### 실패 처리 매핑
| errorCode | 액션 |
|---|---|
| E403001 | 토스트 "삭제 권한이 없습니다" + 닫기(2s) |
| E404001/E404100 | 토스트 "이미 삭제된 상담입니다" + refetch + 닫기 |
| E500001 | 토스트 "일시 오류" + 유지 |
| NETWORK | 토스트 "네트워크 오류" + 유지 |
| E401002 | DLG-000 우선, 본 모달 자동 정리 |

### 감사로그
- 서버가 `AUDIT.DELETE` (consultationId, memberId, actorId) 기록

## 역할별 차이
- `fc`: 403 시 "본인 상담만 삭제 가능" 힌트 토스트
- `manager+`: 원문 메시지

## 에지 케이스
- **낙관적 제거 후 에러**: `onError` 에서 rollback `setQueryData(previous)`
- **목록 재정렬**: 삭제로 페이지네이션 변화 가능 → `invalidateQueries` 로 재호출
- **다중 삭제 연타**: mutationKey로 동일 id 중복 호출 방지

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14
상태: consult-del-done

onSuccess():
  invalidateQueries(['consultations', memberId]);
  invalidateQueries(['member', memberId]);
  toast.success('삭제되었습니다.');
  onClose();

onError(e):
  if (e.errorCode === 'E403001') { toast.error('삭제 권한이 없습니다'); setTimeout(onClose, 2000); }
  else if (e.status === 404) { toast.error('이미 삭제된 상담입니다'); invalidateQueries(['consultations', memberId]); onClose(); }
  else { toast.error(e.message ?? '삭제 실패'); }

낙관적 UI(옵션):
onMutate(async (id) => {
  await qc.cancelQueries(['consultations', memberId]);
  const previous = qc.getQueryData(['consultations', memberId]);
  qc.setQueryData(['consultations', memberId], (old:any) => old?.filter((c:any) => c.id !== id));
  return { previous };
},
onError(err, id, context) {
  qc.setQueryData(['consultations', memberId], context.previous);
},
```
