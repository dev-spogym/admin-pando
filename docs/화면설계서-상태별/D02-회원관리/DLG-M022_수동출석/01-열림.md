# DLG-M022 수동 출석 등록 — 상태 01: 열림 (idle 버튼)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.
> ⚡ 본 DLG는 **인라인 버튼**이므로 "열림"은 부모 화면에서 버튼이 idle 상태로 렌더된 시점을 말한다.

| 상태 ID | STATE_IDLE |

| 이전 상태 | (부모 화면 진입) |
| 다음 상태 | `03-제출중`(클릭) / Disabled(조건 불충족) |

## 상태 진입 조건

- SCR-M004 회원 상세 헤더 렌더 완료
- `hasFeature(role, 'attendance')` (대부분 역할 통과)
- 부모 가드:
  - `member.status !== 'WITHDRAWN'`
  - `member.status !== 'SUSPENDED'`
  - `withinBusinessHours === true`

## 상태 고유 비주얼 델타

### idle (정상)
- `bg-emerald-600 hover:bg-emerald-700`
- 아이콘 `CheckCircle2(13px)` + 라벨 `수동출석`
- hover: 약간의 밝기 변화
- focus-visible: `ring-2 ring-emerald-500 ring-offset-1`

### disabled (조건 불충족)
- `bg-gray-300 text-gray-500 cursor-not-allowed`
- 툴팁(`title`): "탈퇴 회원" / "정지 회원" / "영업시간 외"
- 클릭 무시

## 상태 고유 인터랙션

1. **클릭**: 즉시 `03-제출중` 전환
2. **Enter/Space**: 동일
3. **Hover**: 색상 변화
4. **Disabled**: 포인터 커서 변경 + 툴팁만 표시

## 역할별 차이

- `readonly`: 버튼 자체 렌더되지 않음 (부모에서 `if (role!=='readonly') render`)
- `superAdmin/primary`: 타 지점 회원 헤더에서도 버튼 노출, 서버가 branchId 검증

## 상태 고유 비즈니스 룰 델타

- 영업시간 체크: 지점 `business_hours` 테이블 기반 (요일/시간대). 클라 훅 `useBusinessHours()` 캐시
- 탈퇴 회원 차단: 서버도 E422101 반환하나 UX상 사전 차단
- 중복 출석 사전 힌트: 당일 마지막 출석 시각이 < 5분 이내면 "최근 출석 {m}분 전" 툴팁 (optional)

## 에지 케이스

- **영업시간 경계**: 18:00 정확히 종료인 경우 17:59:59까지 허용
- **시간대 설정 변경**: 영업시간 조회 실패 시 기본 `true` (서버가 최종 판단)
- **회원 상태 중간 변경**: 부모 화면 세션 중 `WITHDRAWN` 전환 시 실시간 disabled (Realtime 또는 invalidate)

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: manual-attend-idle

부모 가드 훅:
const withinBH = useBusinessHours(currentBranchId);
const canAttend =
  member.status !== 'WITHDRAWN' &&
  member.status !== 'SUSPENDED' &&
  withinBH;

<ManualAttendanceButton
  memberId={member.id}
  disabled={!canAttend}
  disabledReason={
    member.status==='WITHDRAWN' ? '탈퇴 회원' :
    member.status==='SUSPENDED' ? '정지 회원' :
    !withinBH ? '영업시간 외' : undefined
  }
/>

툴팁 표시(aria):
aria-describedby={disabled ? 'attend-disabled-hint' : undefined}
<span id="attend-disabled-hint" className="sr-only">{disabledReason}</span>
```
