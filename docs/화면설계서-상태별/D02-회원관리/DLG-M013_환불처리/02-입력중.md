# DLG-M013 환불 처리 — 상태 02: 입력 중(사유 입력)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 델타만 기술.

## 메타

## 상태 진입 조건
- 사용자가 사유 입력 필드 터치 또는 입력

## 상태 고유 비주얼 델타
- 사유 입력 필드 `focus:ring-2 focus:ring-rose-500 border-rose-500`
- 입력 길이 카운터(선택): `{reason.length}/200`
- 200자 초과 시 `text-rose-600` + 버튼 disabled
- 요약 카드 고정(변경 없음)

## 상태 고유 인터랙션
1. **사유 입력**: 자유 텍스트, 200자 제한
2. **200자 초과**: `maxLength` 자동 절단 또는 토스트
3. **Enter**: 사유 필드 안에서는 줄바꿈 불가(input 이므로 폼 submit 안함)
4. **Tab**: 사유 → 취소 → 환불 처리

## 역할별 차이
- 없음 (사유는 모든 권한자 공통 선택 입력)

## 상태 고유 비즈니스 룰 델타
- **사유 검증**:
  - 빈 문자열 → 허용 (선택 필드)
  - 200자 초과 → 자동 절단 or 인라인 에러
  - 스크립트 패턴 차단(서버 sanitize)
- **dirty 판단**: 사유 입력 시 `isDirty=true` → ESC 시 DLG-002 이탈경고 선택적

## 에지 케이스
- **복사-붙여넣기 긴 텍스트**: 200자 초과 시 `onChange` 가드 + 토스트 "최대 200자"
- **한글 입력 조합(IME)**: `onCompositionEnd` 기반 카운트 권장
- **너무 오래 입력 후 저장**: 세션 만료 → 서버 401 → DLG-000

```
  if (v.length > 200) { toast.error('최대 200자'); return; }
  setReason(v);
}}

버튼 활성:
disabled={refund.isPending || reason.length > 200}
```
