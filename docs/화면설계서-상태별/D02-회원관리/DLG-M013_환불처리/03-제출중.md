## 상태 고유 인터랙션
1. **재클릭 방지**: disabled 버튼
2. **ESC**: preventDefault
3. **배경**: preventDefault
4. **네트워크 대기 표시**: sr-only live region "환불 처리 중입니다"

## 상태 고유 비즈니스 룰 델타
- **서버 트랜잭션** (atomic):
  1. INSERT sale (amount=-original, refType='REFUND', parentSaleId)
  2. UPDATE original sale SET status='REFUNDED'
  3. (옵션) PG 환불 호출
  4. AUDIT.REFUND 기록
- **PG 연동**: `paymentMethod==='카드'` 시 동기 PG API 호출, 실패 시 전체 롤백 → `E503002`
- **타임아웃**: 30초 (PG 응답 포함). 초과 시 AbortController → `04` 실패
- **idempotency key**: `Idempotency-Key: refund-{saleId}-{userId}-{ts}` 헤더 권장

## 역할별 차이
- 없음 (서버 권한 재검증)

## 에지 케이스
- **PG 타임아웃**: 서버가 내부 재시도 후 실패 응답 → `E503002`
- **원본 이미 환불(동시성)**: 트랜잭션 내 check → `E409300`
- **매출 삭제됨(희박)**: `E404300`
- **세션 만료**: 401 → DLG-000
