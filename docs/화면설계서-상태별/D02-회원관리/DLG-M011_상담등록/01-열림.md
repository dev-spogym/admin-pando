# DLG-M011 상담 등록 — 상태 01: 열림

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 메타
| 항목 | 값 |
|------|----|
| 상태 코드 | `consult-open` |
| 상태 ID | STATE_OPEN |
| 다이어그램 노드 | `M1_생명주기.md` → DLG_OPEN/INIT |
| 이전 상태 | Closed (상담이력 탭 [상담 추가] 또는 행 [수정] 클릭) |
| 다음 상태 | `02-입력중` (필드 터치) / Closed (취소/ESC) |

## 상태 진입 조건
- `memberId` 유효(SCR-M004 스코프)
- `auth.role` ∈ {primary, owner, manager, fc}
- 수정 모드: `editTarget?.counselorId === auth.user.id || auth.role !== 'fc'`

## 상태 고유 비주얼 델타
- 마스터 레이아웃 초기 렌더. 첫 필드(`consultedAt`)에 `autoFocus`
- 신규: 제목 "상담 이력 추가", 기본값 채워진 상태 (일시=now, 상담, 방문, 완료)
- 수정: 제목 "상담 이력 수정", `form.reset(editTarget)` 완료 상태
- `isSubmitting=false`, "저장" 버튼 — `formState.isValid` 에 따라 enabled/disabled
- 하단 힌트 텍스트 없음, 필드 에러 모두 비노출

## 상태 고유 인터랙션
1. **오픈 직후**: `consultedAt` 포커스, datetime picker 즉시 열기 가능
2. **ESC / 배경 / X**: 닫힘 (dirty 없으면 즉시, dirty 있으면 DLG-002 이탈경고)
3. **필드 터치**: `02-입력중` 전이
4. **저장 클릭**: 검증 후 `03-제출중` 전이

## 역할별 차이
- `fc`: 담당자 필드 readonly(자기 이름 고정), 수정 모드에서 타인 상담 접근 시 서버 403
- `manager+`: 담당자 자유 지정 가능

## 상태 고유 비즈니스 룰 델타
- 연결 매출 쿼리 `enabled=isOpen` → 오픈 시에만 호출
- 수정 모드에서 `editTarget.consultedAt` 과거 3개월 초과 시 저장 서버 403(선택)
- 폼 기본값이 모두 채워진 상태에서도 `formState.isDirty=false` 로 시작

## 에지 케이스
- **sales 로드 실패**: 연결 매출 select 에 "로드 실패" placeholder, 다른 필드 정상
- **editTarget 로드 지연**: 수정 모드에서 `editTarget=undefined` 이면 스켈레톤
- **오픈 중 세션 만료**: DLG-000 우선, 본 다이얼로그 자동 정리
