## 📝 디스크립션

### 사용 시점
- GET /api/members 응답이 4xx 또는 5xx
- 네트워크 연결 끊김 (fetch 자체 실패)
- Supabase 연결 오류

### 필수 데이터
| 항목 | 상태 |
|------|------|
| error 객체 | message, status code |
| members | undefined 또는 마지막 캐시 |

### 인터랙션
1. "다시 시도" 버튼 → refetch() → `01-로딩` 재진입
2. 검색어/필터 변경 → 자동 재fetch 트리거
3. 탭 변경 → 각 탭별 재시도

### 비즈니스 룰
- 에러 발생 시 toast.error도 동시 표시 (배너 + 토스트 병행)
- 네트워크 오류는 자동 재시도 1회 (useQuery retry 설정)
- 401 오류: 자동 로그아웃 + 로그인 페이지 이동

### 에지 케이스
- 부분 성공 (페이지네이션 중 오류): 이전 페이지 유지 + 에러 배너
- 500 오류 지속: "관리자에게 문의하세요" CTA 표시

### 접근성
- ErrorBanner `role="alert"` (즉각 스크린리더 고지)
- 다시 시도 버튼 `aria-label="회원 목록 다시 불러오기"`

### 연결 화면
- 재시도 성공 → `02-정상목록`
- 재시도 실패 → 에러 상태 유지

### 다이어그램 참조
- 엣지: `E_RESP_ERR_01` (4xx/5xx/네트워크 → 에러 배너)
- 상태: `STATE_ERROR` (🆕 신규 기능)
- 재시도: `E_RETRY_01`
