## 📝 디스크립션

### 사용 시점
- GET /api/members 200 응답 + 데이터 1건 이상 존재
- 필터 변경 후 재fetch 완료 + 결과 있음

### 필수 데이터
| 테이블 | 조건 |
|--------|------|
| members | branch_id = 현재 지점, status 필터 |
| contracts | 최신 이용권 JOIN |
| users | 담당 FC (fc_id) JOIN |

### 인터랙션
1. 탭 변경 → status 쿼리 업데이트 → 로딩 재진입
2. 검색 입력 후 엔터/검색 버튼 → 검색 결과 fetch
3. 체크박스 선택 → BulkActionBar 슬라이드업 표시
4. BulkAction "상태 변경" → DLG-M001 트리거 (selectedRows.size > 0 필수)
5. 행 클릭 → `/members/detail?id={id}`
6. "회원 등록" → `/members/new`
7. 엑셀 다운로드 → exportToExcel()

### 비즈니스 룰
- primary/owner: 모든 지점 회원 조회 가능
- manager/fc/staff: 소속 지점 회원만 조회
- fc: 자신이 담당한 회원만 기본 필터 적용
- readonly: 이 화면 진입 불가 (권한 없음으로 분기)
- 페이지당 기본 20건 표시

### 에지 케이스
- 검색 결과 0건 → `03-빈상태-검색` 상태로 전환
- 전체 회원 0명 (신규 센터) → `04-빈상태-전체`
- BulkAction 중 API 오류 → toast.error, 목록 유지

### 접근성
- DataTable `role="grid"`, 헤더 `scope="col"`
- BulkActionBar `aria-live="polite"` + 선택 인원수 고지
- 페이지네이션 버튼 `aria-label="다음 페이지"`
