## 상태 진입 조건

- JWT `role === 'readonly'` — 페이지 접근 차단
- JWT 역할이 화면 접근 허용 목록 밖 → E403001
- 지점 branchId 조작 (다른 브랜드/지점) → HTTP 403 (E403003)
- 비로그인은 이 상태 미도달 — 미들웨어가 `/login?redirect=` 이동

## 상태 고유 비주얼 델타

레이아웃 구조 자체 변경 — **AppLayout 내 ForbiddenView만 렌더** 또는 `/forbidden`으로 미들웨어 리다이렉트.

## 역할별 차이

| 역할 | 분기 |
|---|---|
| readonly | 즉시 ForbiddenView |
| fc (접근 허용) | 본 상태 미도달 |
| fc가 `?staffId=otherStaff` 조작 | 서버 403 → ForbiddenView |
| manager/owner 다른 지점 `?branch=99` 조작 | 403 |
| primary 잘못된 브랜드 id | 403 |
| 비로그인 | 미들웨어가 `/login?redirect=`로 이동 |

## 상태 고유 인터랙션

| 액션 | 동작 |
|---|---|
| [대시보드로 이동] | `router.replace('/')` |
| [관리자에게 문의] | `mailto:admin@{domain}?subject=접근권한요청&body=...` 또는 사내 메신저 딥링크 |
| [로그아웃] (readonly) | `useAuthStore.logout()` → `/login` |
| 사이드바 다른 메뉴 | 정상 이동 |

## 상태 고유 비즈니스 룰 델타

1. 가드 체인 (depth-in-depth):
   - 서버 미들웨어: JWT 파싱 → role 체크 → 미허용 → `redirect('/forbidden')`
   - 클라이언트 가드: `useAuthGuard({allow: MEMBERS_ALLOWED, redirect: '/forbidden'})`
   - 서버 API: 재검증
2. 감사 로그 `AUDIT.ACCESS_DENIED` (role, path, branchId) 기록.
3. 지점 조작은 다른 문구 "해당 지점에 대한 접근 권한이 없습니다."
4. 검색엔진 index 방지: `<meta name="robots" content="noindex">`.
5. analytics `forbidden_view` 이벤트.
6. SR `role="alert"` + 포커스 `대시보드로 이동` 버튼.
7. 가장(impersonate) 중이면 상단 배너 "가장 중: {role}" 표시.

## 에지 케이스

- 세션 만료 + 권한 문제 → 세션 만료 우선 → `/login` 리다이렉트.
- 역할 변경 직후 진입 → 403 받은 뒤 JWT refresh 후 재평가.
- 사이드바에 원래 이 항목 노출 → 사이드바도 role 필터링 (본 상태는 URL 직접 접근 방어).

## 🧩 바이브코딩 프롬프트 (델타)

```
※ 델타: ForbiddenView + 가드 체인

// middleware.ts 또는 서버 layout:
import { redirect } from 'next/navigation';
const MEMBERS_ALLOWED: Role[] = ['primary','superAdmin','owner','manager','fc','trainer','staff','front'];
export default async function Layout({ children }) {
  const { role } = await getServerSession();
  if (!role) redirect('/login?redirect=/members');
  if (!MEMBERS_ALLOWED.includes(role)) redirect('/forbidden?from=/members');
  return children;
}

// 클라이언트 가드
useAuthGuard({ allow: MEMBERS_ALLOWED });

// ForbiddenView.tsx
export function ForbiddenView({ role, errorCode = 'E403001', from }: Props) {
  useEffect(() => {
    analytics.track('forbidden_view', { role, from, errorCode });
  }, []);
  return (
    <main role="alert" aria-live="assertive"
          className="flex flex-col items-center justify-center min-h-[60vh] gap-4 text-center px-4">
      <Lock className="w-14 h-14 text-red-400" />
      <h1 className="text-xl font-bold text-gray-900">접근 권한이 없습니다.</h1>
      <p className="text-sm text-gray-500 max-w-md">
        이 페이지에 접근할 권한이 없습니다. 문제가 있다면 관리자에게 문의해주세요.
      </p>
      <div className="mt-4 flex gap-2">
        <Button variant="primary" autoFocus onClick={() => router.replace('/')}>
          대시보드로 이동
        </Button>
        <a href="mailto:admin@fitgenie.kr?subject=접근권한요청"
           className="inline-flex items-center gap-2 h-10 px-4 rounded-lg border border-gray-300 hover:bg-gray-50 text-sm">
          <Mail className="w-4 h-4" /> 관리자에게 문의
        </a>
        {role === 'readonly' && <Button variant="ghost" onClick={logout}>로그아웃</Button>}
      </div>
      <p className="mt-4 text-xs text-gray-400 font-mono">
        Error code: {errorCode} · Role: {role}
      </p>
    </main>
  );
}

// SEO
<Head><meta name="robots" content="noindex" /></Head>
```
