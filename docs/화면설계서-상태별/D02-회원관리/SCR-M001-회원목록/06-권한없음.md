## 📝 디스크립션

### 사용 시점
- 로그인 사용자의 역할이 `readonly`인 경우
- `hasPermission(role, 'members:read') === false`

### 필수 데이터
| 항목 | 상태 |
|------|------|
| user.role | readonly |
| members 데이터 | 미조회 (API 호출 차단) |

### 인터랙션
1. "홈으로 돌아가기" → `/` (대시보드)
2. 사이드바 메뉴에서 다른 허용 화면 접근 가능

### 비즈니스 룰
- readonly 역할은 API 호출 자체를 차단 (데이터 노출 방지)
- 미들웨어 레벨에서도 redirect 처리 권장
- 이 화면은 클라이언트 사이드 fallback으로도 동작

### 에지 케이스
- JWT 위조로 접근 시도 → 서버사이드에서도 403 반환
- 역할 변경 후 즉시 재접근 시: 세션 갱신 필요

### 접근성
- `role="alert"` 또는 `aria-live="assertive"` 적용
- ShieldOff 아이콘 `aria-hidden="true"`

### 연결 화면
- 홈(대시보드): `/`

### 다이어그램 참조
- 엣지: `E_AUTH_READONLY_01` (readonly → 권한없음 화면)
- 상태: `STATE_BLOCKED` (🆕 신규 기능)
