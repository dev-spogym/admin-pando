## 📝 디스크립션

### 사용 시점
- 검색 필터(이름/연락처/이메일/담당FC/상태)를 적용했으나 API 응답이 200 + 빈 배열
- 탭 필터(홀딩/만료 등)와 검색 키워드를 동시 적용 시 0건

### 필수 데이터
| 항목 | 상태 |
|------|------|
| members (검색 결과) | 빈 배열 [] |
| 통계 카드 수치 | 정상 표시 유지 (검색 전 전체 수) |
| searchQuery | 현재 검색어 (빈 상태 메시지에 표시) |

### 인터랙션
1. "검색 초기화" 버튼 → 필터 전체 리셋 → 재fetch
2. 검색어 직접 수정 → debounce 후 재검색
3. 탭 변경 → 다른 status 필터로 재검색

### 비즈니스 룰
- 검색 결과 없음과 전체 회원 없음은 다른 상태로 분기
- 현재 검색어를 메시지에 노출하여 UX 명확화
- 통계 카드는 검색 결과와 무관하게 전체 수치 유지

### 에지 케이스
- 특수문자 포함 검색 → XSS 방지 처리 후 표시
- 매우 긴 검색어 → `truncate` 처리

### 접근성
- EmptyState `role="status"` + `aria-live="polite"`
- "검색 결과가 없습니다" 스크린리더 고지

### 연결 화면
- 검색 초기화 → `02-정상목록`
- 재검색 성공 → `02-정상목록`
