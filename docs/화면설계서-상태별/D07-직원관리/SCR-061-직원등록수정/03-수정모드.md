## 역할별 차이
- **자기 자신 수정**(`user.id === editId`): role Select disabled + hint "본인 정보는 일부 필드만 수정 가능합니다."
- manager: "센터장(owner)" 옵션 disabled.
- owner: 자기 지점 직원만 (`02`에서 차단).

## 고유 인터랙션
- `formState.isDirty` 기반 취소 시 DLG 스킵 가능 (변경 없으면 바로 이동).
- 연락처 편집 시 재포맷.
- 역할 변경: 08 Preview 재렌더.
- 이름 필드 autofocus + `setSelectionRange(end,end)` (선택).
- beforeunload: isDirty=true && !isSaving 시 경고.

## 비즈니스 룰 델타
1. `isDirty` 플래그로 취소 흐름 최적화.
2. 역할 변경 시 추가 감사 `AUDIT.STAFF_ROLE_CHANGE`.
3. 기본급 변경 시 `AUDIT.STAFF_SALARY_CHANGE`.
4. 낙관적 concurrency: updatedAt 체크 선택. 충돌 시 409 → dialog.
5. 부분 업데이트: Partial<StaffRequest>, 변경 필드만 포함(선택 최적화).

## 에지 케이스
- 로드 직후 이탈: isDirty=false → 경고 없음.
- 동시 편집(다른 탭): 마지막 저장 승리.
- 본인 상위 역할 변경 시도: 서버 403.
- 퇴사 예정일 있는 직원: 기본급 등만 수정 허용(옵션).

## 바이브코딩 프롬프트 (델타)
```
editId && !isLoadingStaff:
<PageHeader title="직원 정보 수정"
  description="직원의 정보를 수정하고 저장합니다."
  actions={<>
    <Button variant="secondary" onClick={()=>{
      if (formState.isDirty) setShowCancelDialog(true);
      else moveToPage(974);
    }}>취소</Button>
    <Button type="submit" variant="primary" disabled={isSaving}>
      <Save size={16}/> {isSaving ? '저장 중...' : '수정 저장'}
    </Button>
  </>} />

const isSelf = user.id === editId;
<Select disabled={isSelf} {...register('role')}/>
{isSelf && <Hint>본인 정보는 역할/지점을 수정할 수 없습니다.</Hint>}

// beforeunload 경고
useEffect(() => {
  const h = (e: BeforeUnloadEvent) => {
    if (formState.isDirty && !isSaving) { e.preventDefault(); e.returnValue=''; }
  };
  window.addEventListener('beforeunload', h);
  return () => window.removeEventListener('beforeunload', h);
}, [formState.isDirty, isSaving]);

// 감사 로그 (변경 필드별)
if (before.role !== after.role) await createAuditLog({ action:'STAFF_ROLE_CHANGE', ... });
if (before.salary !== after.salary) await createAuditLog({ action:'STAFF_SALARY_CHANGE', ... });
```
