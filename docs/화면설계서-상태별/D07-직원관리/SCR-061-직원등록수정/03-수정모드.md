## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-수정모드-로딩` 완료 후 `form.reset(staffDetail)` 호출
- `isLoadingStaff === false && staffDetail !== null`

### 필수 데이터
| 필드 | preloaded 값 |
|------|-------------|
| 이름 | staffDetail.name |
| 연락처 | staffDetail.phone |
| 부서 | staffDetail.department |
| 직책 | staffDetail.position |
| 역할 | staffDetail.role |
| 입사일 | staffDetail.hire_date |
| 급여 | staffDetail.salary |

### 인터랙션 (User Actions)
1. 필드 수정 → `isDirty=true`
2. 역할 변경 → 권한 미리보기 업데이트
3. 수정 저장 클릭 → 검증 → `04-저장중`
4. 취소: `isDirty=true` → DLG-061-001

### 비즈니스 룰
- staffStatus 필드: disabled (목록 화면의 상태 변경 모달로만 변경 가능)
- RESIGNED 직원 수정 시도: toast.warning + 일부 필드 읽기 전용
- 자기 자신(현재 로그인 사용자) 수정: 역할 변경 불가

### 에지 케이스
- 수정 중 다른 관리자가 동일 직원 수정: 저장 시 409 conflict → toast.error
- 입사일 과거로 변경: 허용 (근속 연수 재계산)

### 접근성 (A11y)
- 알림 배너 `role="note"`
- disabled Select `aria-disabled="true"`
