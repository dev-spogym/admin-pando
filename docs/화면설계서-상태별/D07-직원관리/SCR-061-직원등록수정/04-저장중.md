```
Next.js 14 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-061 직원 등록/수정 — 상태: 저장 중 (isSaving=true)

파일: src/app/staff/new/page.tsx + src/app/staff/edit/page.tsx (공통 처리)

조건: isSaving === true (저장 버튼 클릭 후 API 응답 전)

UI 변화:
1. 저장 버튼:
   <Button disabled className="opacity-70 cursor-not-allowed">
     <Spinner className="mr-2 h-4 w-4 animate-spin" />
     저장 중...
   </Button>

2. 취소 버튼: disabled (isSaving 중 이탈 방지)

3. 모든 input 필드: disabled (readOnly 처리)

4. 폼 전체 오버레이:
   <div className="absolute inset-0 bg-white/50 z-10 rounded-lg" />

저장 핸들러 (등록):
  const onSubmit = async (data: StaffFormValues) => {
    setIsSaving(true)
    try {
      const { error } = await supabase.from('staff').insert({
        ...data,
        center_id: user.centerId,
        staff_status: 'ACTIVE',
        created_by: user.id,
      })
      if (error) throw error
      toast.success('직원이 등록되었습니다.')
      moveToPage('/staff')
    } catch (err: any) {
      if (err.code === '23505') {
        toast.error('이미 등록된 연락처입니다.')
        setError('phone', { message: '이미 사용 중인 연락처입니다.' })
      } else {
        toast.error('저장에 실패했습니다. 다시 시도해주세요.')
      }
      setIsSaving(false)
    }
  }

상태 변수:
- isSaving: true
- 모든 폼 필드: disabled
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 검증 통과 후 저장/수정저장 버튼 클릭 → Supabase insert/update API 호출 중

### 인터랙션 (User Actions)
- 없음 (모든 입력/버튼 비활성)
- 브라우저 탭 닫기: 저장 중단 가능 (경고 다이얼로그 없음 — 브라우저 기본 동작)

### 비즈니스 룰
- 저장 중 중복 클릭 방지 (`isSaving=true` 동안 submit 핸들러 guard)
- API 타임아웃 10초 초과 시 `setIsSaving(false)` + 에러 메시지

### 에지 케이스
- 저장 중 네트워크 끊김: catch → toast.error('네트워크 오류') + isSaving=false
- 연락처 중복(409): 인라인 에러 + isSaving=false → `05-폼에러`
- 시스템 에러(500): toast.error + isSaving=false → `07-에러`

### 접근성 (A11y)
- 저장 버튼 `aria-busy="true"` 동안
- 폼 `aria-disabled="true"`

### 연결 화면
- 이전: `01-등록모드` 또는 `03-수정모드`
- 다음: `06-완료` / `05-폼에러` / `07-에러`
