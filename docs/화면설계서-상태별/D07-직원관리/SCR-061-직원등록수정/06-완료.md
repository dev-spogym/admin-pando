## 📝 디스크립션

### 사용 시점 (Trigger)
- Supabase insert(등록) 또는 update(수정) API 200 OK 응답

### 완료 처리 순서
1. `toast.success(메시지)` 표시
2. `moveToPage('/staff')` 즉시 실행
3. SCR-060 진입 → `fetchStaffList()` 재호출
4. 테이블 갱신 완료

### 인터랙션 (User Actions)
- 이 상태에서 사용자 액션 없음 (자동 리다이렉트)
- toast 닫기 버튼 클릭 가능 (toast 자체 기능)

### 비즈니스 룰
- 등록 완료 후 신규 직원 staffStatus: `ACTIVE` 자동 설정
- 수정 완료: 변경된 필드만 update (partial update)
- 완료 후 SCR-060 탭 필터: 마지막 선택 탭 유지 (sessionStorage)

### 에지 케이스
- navigate 실패 (라우터 오류): 수동 새로고침 유도 toast
- SCR-060 갱신 실패: 별도 에러 처리 없음 (SCR-060 자체 에러 처리로 위임)
