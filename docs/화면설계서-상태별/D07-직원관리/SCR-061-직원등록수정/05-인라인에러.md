```
Next.js 14 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-061 직원 등록/수정 — 상태: 인라인 에러 (필드 검증 실패)

파일: src/app/staff/new/page.tsx

   <input className="border-red-500 focus:ring-red-500 bg-red-50" aria-invalid="true" />

   {errors.name && (
     <p className="mt-1 text-xs text-red-600" role="alert" id="name-error">
       {errors.name.message}
     </p>
   )}

3. 저장 버튼 위 요약 배너 (에러 2개 이상):
   <div className="rounded-md bg-red-50 border border-red-200 p-3 text-sm text-red-700" role="alert">
     입력 값을 확인해주세요. {Object.keys(errors).length}개 항목에 오류가 있습니다.
   </div>

zod 스키마 에러 메시지:
  const staffSchema = z.object({
    name: z.string().min(2, '이름은 2자 이상 입력하세요.'),
    phone: z.string().regex(/^010-\d{4}-\d{4}$/, '010-xxxx-xxxx 형식으로 입력하세요.'),
    department: z.string().min(1, '부서를 입력하세요.'),
    position: z.string().min(1, '직책을 입력하세요.'),
    role: z.enum(['OWNER','MANAGER','FC','STAFF'], { required_error: '역할을 선택하세요.' }),
    hire_date: z.string().min(1, '입사일을 입력하세요.'),
    email: z.string().email('올바른 이메일 형식을 입력하세요.').optional().or(z.literal('')),
    salary: z.number().min(0, '급여는 0 이상이어야 합니다.').optional(),
  })

서버 에러 (API 409 중복):
  setError('phone', { type: 'server', message: '이미 사용 중인 연락처입니다.' })

사용 컴포넌트:
- react-hook-form formState.errors
- zod zodResolver
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장 버튼 클릭 시 zod 검증 실패
- API 응답 400/409 후 `setError()` 호출

### 에러 유형별 메시지
| 필드 | 에러 조건 | 메시지 |
|------|-----------|--------|
| 이름 | 2자 미만 | "이름은 2자 이상 입력하세요." |
| 연락처 | 패턴 불일치 | "010-xxxx-xxxx 형식으로 입력하세요." |
| 연락처 | 중복(API 409) | "이미 사용 중인 연락처입니다." |
| 부서 | 빈 값 | "부서를 입력하세요." |
| 역할 | 미선택 | "역할을 선택하세요." |
| 입사일 | 빈 값 | "입사일을 입력하세요." |
| 이메일 | 형식 불일치 | "올바른 이메일 형식을 입력하세요." |

### 인터랙션 (User Actions)
1. 에러 필드 수정 → 해당 에러 클리어 (onChange validation)
2. 저장 버튼 재클릭 → 전체 재검증 → 통과 시 `04-저장중`

### 비즈니스 룰
- 에러 필드로 자동 포커스 이동 (첫 번째 에러 필드)
- `onBlur` 모드로 실시간 검증 (타이핑 중 에러 미표시)

### 에지 케이스
- 여러 필드 동시 에러: 요약 배너 + 개별 에러 메시지 모두 표시
- 서버 에러 후 클라이언트 에러 같은 필드 충돌: 서버 에러 우선

### 접근성 (A11y)
- `aria-invalid="true"` + `aria-describedby="{field}-error"`
- 요약 배너 `role="alert"` + `aria-live="assertive"`
- 첫 번째 에러 필드 자동 포커스

### 연결 화면
- 이전: `01-등록모드` / `03-수정모드`
- 다음: 수정 완료 후 재저장 → `04-저장중`
