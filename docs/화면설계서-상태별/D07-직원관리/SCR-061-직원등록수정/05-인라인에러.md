## 비주얼 델타

```
┌──────────────────────────────────────────────────────────┐
│ FormField: 이름 *                                        │
│ [👤 홍          ]  ← input.error border + ring-error/30  │
│ ⚠ 이름은 2글자 이상 입력해주세요.  role="alert"          │
├──────────────────────────────────────────────────────────┤
│ FormField: 역할 *                                        │
│ [역할 선택                                              ▼] │
│ ⚠ 역할을 선택하세요.                                      │
├──────────────────────────────────────────────────────────┤
│ FormField: 연락처 *                                      │
│ [📞 010-12      ]                                        │
│ ⚠ 010-0000-0000 형식으로 입력하세요.                      │
└──────────────────────────────────────────────────────────┘
```

| 변경 요소 | From → To |
|---|---|
| input border | `border-line` → `border-state-error` |
| focus ring | `ring-primary` → `ring-state-error/30` |
| 에러 메시지 | — | `<p role="alert" className="text-sm text-state-error mt-1">` |
| aria-invalid | false → true |
| aria-describedby | — → `err-{field}` |
| 저장 버튼 | 활성 | 제출 시 errors 존재면 첫 번째 에러 필드 focus |

## 역할별 차이
- 공통.

## 고유 인터랙션
- 에러 메시지는 `role="alert" aria-live="polite"` (동적 추가).
- 제출 시 errors 있으면 **첫 번째 에러 필드로 자동 포커스** 이동.
- 수정 시작 → errors 유지, blur 시 재검증 → 통과하면 에러 제거.
- 필수 필드 라벨 `*` 색상 error.

## 비즈니스 룰 델타
1. **검증 모드**: `mode:'onBlur'` + 제출 시 전체 재검증.
2. **인라인 에러 메시지**:
   - `name.min(2)`: "이름은 2글자 이상 입력해주세요."
   - `role.min(1)`: "역할을 선택하세요."
   - `contact.regex`: "010-0000-0000 형식으로 입력하세요."
   - `joinDate.min(1)`: "입사일을 입력하세요."
   - `email.email`: "올바른 이메일 형식이 아닙니다."
   - `memo.max(5000)`: "메모는 5,000자 이내로 입력해주세요."
3. **서버 에러 매핑**:
   - E409100(전화번호 중복) → `setError('contact', {message:'이미 등록된 전화번호입니다.'})`
   - E400103 → `setError('contact', ...)`
   - E400104 → `setError('email', ...)`
4. **첫 에러 자동 포커스**: `setFocus(Object.keys(errors)[0])`.

## 에지 케이스
- 여러 필드 동시 에러: 모두 표시, 첫 번째로 포커스.
- 필드 간 의존 검증 없음(모두 독립).
- 에러 메시지 변경(다국어 전환): `zod.messages` 외부화.
- aria-live 중복 선언 피하기: 한 번만 announce.

## 바이브코딩 프롬프트 (델타)
```
errors[field] 존재 시:
<FormField label="이름" required error={errors.name?.message}>
  <InputWithIcon id="name" icon={<User size={16}/>}
    className={cn(
      'rounded-input h-10 px-3 focus:ring-2 transition-colors',
      errors.name
        ? 'border-state-error focus:ring-state-error/30 focus:border-state-error'
        : 'border-line focus:ring-primary focus:border-primary'
    )}
    aria-required aria-invalid={!!errors.name}
    aria-describedby={errors.name ? 'err-name' : undefined}
    {...register('name')} />
</FormField>

{errors.name && (
  <p id="err-name" role="alert" aria-live="polite"
     className="text-sm text-state-error mt-1 flex items-center gap-1">
    <AlertCircle size={14}/> {errors.name.message}
  </p>
)}

// 제출 시 첫 에러 필드로 포커스
const onInvalid = (errs) => {
  const first = Object.keys(errs)[0];
  if (first) setFocus(first as keyof StaffFormData);
};
<form onSubmit={handleSubmit(onSubmit, onInvalid)}>

// 서버 에러 → 필드 에러로 매핑
if (error.code === 'E409100') setError('contact', { type:'server', message:'이미 등록된 전화번호입니다.' });
if (error.code === 'E400103') setError('contact', { type:'server', message:'올바른 전화번호를 입력해주세요.' });
if (error.code === 'E400104') setError('email',   { type:'server', message:'올바른 이메일 형식이 아닙니다.' });
```
