## 🧩 바이브코딩 프롬프트

```
Next.js 14 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-061 직원 등록/수정 — 상태: 수정 모드 로딩

파일: src/app/staff/edit/page.tsx

조건: searchParams.get('id') !== null → 수정 모드 진입 → 데이터 fetch 중

useEffect(() => {
  if (editId) {
    setIsLoadingStaff(true)
    fetchStaffDetail(editId)
  }
}, [editId])

fetchStaffDetail:
  const { data, error } = await supabase
    .from('staff')
    .select('*')
    .eq('id', editId)
    .single()
  if (error) { setLoadError(error); toast.error('직원 정보를 불러오지 못했습니다.'); moveToPage('/staff') }
  else { form.reset(data); setIsLoadingStaff(false) }

로딩 UI:
- PageHeader: title="직원 정보 수정" (breadcrumb 동일)
- 폼 영역 전체에 오버레이 (bg-white/70) + 중앙 스피너
- 각 input 필드 disabled + skeleton 느낌 (opacity-50)
- 하단 버튼 disabled

상태 변수:
- isLoadingStaff: true
- editId: string (searchParams에서 추출)
- form: react-hook-form (초기값 빈 상태 → reset 후 채워짐)

사용 컴포넌트:
- AppLayout, PageHeader, Button
- useSearchParams, supabase, toast, moveToPage
```

### 필수 데이터
| 항목 | 값 |
|------|-----|
| editId | URL searchParams 'id' |
| isLoadingStaff | true |
| staffDetail | null (fetch 전) |

### 인터랙션 (User Actions)
- 없음 (로딩 중 폼 비활성)

### 비즈니스 룰

### 에지 케이스
- editId URL 파라미터 변조: Supabase RLS에서 차단
