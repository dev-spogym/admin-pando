## 📝 디스크립션

### 사용 시점 (Trigger)
| 에러 유형 | 발생 시점 | 처리 |
|-----------|-----------|------|
| 로드 에러 | fetchStaffDetail API 실패 | toast + 즉시 목록 복귀 |
| 저장 에러(500) | insert/update API 500 | toast + 폼 에러 배너 + 재시도 |
| 저장 에러(403) | RLS 권한 오류 | toast.error('권한 없음') + isSaving=false |

### 인터랙션 (User Actions)
1. 로드 에러: 자동 `/staff` 복귀 (사용자 액션 불필요)
2. 저장 에러: 폼 수정 후 "수정 저장" 재클릭 → `04-저장중`
3. 취소 버튼 클릭 → DLG-061-001 또는 직접 목록 복귀

### 비즈니스 룰
- 에러 메시지에 내부 error.code 노출 금지
- 403 에러: 권한 없음 toast + 목록 복귀 (폼 유지 불필요)
- 로드 에러 시 URL 파라미터 id 존재하지만 데이터 없음: 404 처리 동일

### 에지 케이스
- 저장 에러 반복 3회 이상: "고객센터 문의" CTA 표시 (카운트 로컬 변수)
- 네트워크 오류와 시스템 에러 구분: `error.name === 'FetchError'` 체크

### 접근성 (A11y)
- 에러 배너 `role="alert"` + `aria-live="assertive"`
