    })
    if (error) toast.error('이메일 발송에 실패했습니다.')

## 📝 디스크립션

### 사용 시점 (Trigger)
- `statements.length > 0 && !isLoadingData && !error`

### 필수 데이터
| 테이블 | 조건 |
|--------|------|
| `payroll_statements` (가상) | `year_month = selectedMonth` |
| JOIN `staff` | name, department, position, email |

### 인터랙션 (User Actions)
1. MonthPicker 변경 → 해당 월 재fetch
2. SearchFilter 입력 → 이름 필터링 (클라이언트 사이드)
3. "전체 이메일 발송" → 미발송 전체 일괄 발송 (progress toast)
4. "PDF 일괄 생성" → 전체 PDF 생성 + ZIP 다운로드
5. 개별 "이메일 발송" → `handleSendEmail(id)`
6. 개별 "PDF 다운로드" → `handleGeneratePdf(id)`

### 비즈니스 룰
- 이메일 발송: `hasPermission('payroll:send')` — primary / owner
- manager: 명세서 조회 가능, 발송 불가
- 이미 발송된 명세서 재발송: 허용 (재발송 확인 토스트)
- 이메일 주소 없는 직원: 발송 버튼 disabled + 툴팁 "이메일 주소 없음"

### 에지 케이스
- 직원 이메일 미등록: 발송 버튼 disabled
- PDF 생성 실패: toast.error + 재시도 버튼
- 전체 발송 중 일부 실패: 성공/실패 건수 요약 toast

### 접근성 (A11y)
- StatusBadge `aria-label="발송 상태: {status}"`
- "이메일 발송" 버튼 `aria-label="{이름} 급여 명세서 이메일 발송"`
