# SCR-060 직원 목록 — 05 권한 없음 (델타)

> 이 문서는 `00-기본화면.md`의 **델타**입니다.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `staff-list-forbidden` |
| 상태 ID | 05 |
| 다이어그램 노드 | F7 권한분기 > DENY |
| 이전 상태 | (진입 시도) |
| 다음 상태 | `/forbidden` 화면 or 역할별 기본 대시보드 |

## 진입 조건
- 세션 role ∈ {fc, trainer, staff, front, readonly}
- 또는 owner/manager가 타 지점 branchId 조작 시 (E403003)
- 클라이언트 라우트 가드 + 서버 RLS 이중 차단

## 비주얼 델타

본 화면은 **렌더되지 않고** `/forbidden` 페이지로 즉시 리다이렉트된다. 아래는 `/forbidden` 공통 페이지 참조.

```
┌──────────────────────────────────────────────────────────┐
│              [FORBIDDEN 전용 페이지 /forbidden]          │
│                                                          │
│              🚫 403 — 접근 권한이 없습니다                │
│                                                          │
│       "이 화면은 센터장 이상만 접근할 수 있습니다."       │
│                                                          │
│       [대시보드로 돌아가기]   [이전 페이지]              │
└──────────────────────────────────────────────────────────┘
```

| 변경 요소 | From → To |
|---|---|
| 렌더 | 본 SCR-060 트리 | 렌더 중단 → `redirect('/forbidden')` |
| 사이드바 "직원 관리" 메뉴 | 표시 | 숨김 (역할별 menuConfig에 따라) |
| 토스트 | — → (최초 진입 시만) `toast.error("접근 권한이 없습니다.")` |

## 역할별 차이
| 역할 | 시나리오 |
|---|---|
| fc | 메뉴 자체 숨김. URL 직접 입력 시 `/forbidden` |
| trainer | 동일 |
| staff | 동일 |
| front | 동일 |
| readonly | 동일 |
| manager (2026-04 이후) | 접근은 허용 — `05`는 아님. 단, 퇴사 버튼은 disabled (비즈니스 룰) |
| owner | `?branch=<타지점>` 조작 시 E403003 → 본 상태 |

## 고유 인터랙션
- `/forbidden`에서 "대시보드로 돌아가기" → `moveToPage('/')` 역할별 기본 대시보드.
- "이전 페이지" → `router.back()`.
- 자동 포커스: "대시보드로 돌아가기" 버튼.

## 비즈니스 룰 델타
1. **클라이언트 가드**: 페이지 최상단 `if (!canAccess(role)) redirect('/forbidden')`.
2. **서버 RLS**: jwt.role/branchId로 staff 테이블 행 스코프 강제. 데이터 유출 방지.
3. **감사 로그**: 접근 차단 이벤트를 `audit_logs.action = 'ACCESS_DENIED'`로 기록 (actorId, targetPath, ip).
4. **메뉴 가드**: `getMenuForRole(role)`에서 "직원 관리" 제거.

## 에지 케이스
- **역할 다운그레이드 중 세션**: owner → manager로 역할 변경된 순간 새로고침 시 버튼 제약 재적용.
- **primary가 하위 관리 계정 로그인 시뮬레이션**: 실제 jwt 역할 반영됨.
- **staff가 사이드바 메뉴 숨겨진 상태에서 공유받은 URL 진입**: 최초 1회 toast + 리다이렉트.

## 바이브코딩 프롬프트 (델타)
```
// src/app/staff/page.tsx 최상단 가드 (Server Component 또는 Client early return)
const { user } = useAuth();
const allowed = ['superAdmin','primary','owner','manager'];
if (!user) return redirect(`/login?redirect=/staff`);
if (!allowed.includes(user.role)) {
  toast.error('접근 권한이 없습니다.');  // 1회
  return redirect('/forbidden');
}

// 서버 측 (Supabase RLS 정책)
CREATE POLICY staff_read_policy ON staff FOR SELECT
USING (
  (auth.jwt()->>'role') IN ('superAdmin','primary','owner','manager')
  AND (
    (auth.jwt()->>'role') IN ('superAdmin','primary')
    OR branchId = (auth.jwt()->>'branchId')::int
  )
);

// 사이드바: getMenuForRole(role)에서 '/staff' 항목을 allowed에 한해 노출
```
