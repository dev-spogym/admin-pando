## 📝 디스크립션

### 사용 시점 (Trigger)
- FC / STAFF 역할 사용자가 `/staff` 라우트 직접 접근 시도
- `hasPermission('staff:view') === false`

### 권한 매트릭스
| 역할 | 접근 가능 |
|------|----------|
| primary (본사) | O |
| owner (센터장) | O |
| manager (매니저) | O |
| fc (FC) | X |
| staff (스태프) | X |

### 인터랙션 (User Actions)
1. "대시보드로 이동" 버튼 → `/dashboard`
2. 브라우저 뒤로가기 → 이전 라우트

### 비즈니스 룰
- 서버사이드 미들웨어에서도 동일 권한 체크 (`middleware.ts`)
- 클라이언트 권한 체크는 UX용, 실제 보안은 서버사이드 RLS
- 권한 없음 화면 자체는 로그인 세션 유효 상태에서만 표시

### 에지 케이스
- 로그인 세션 만료 후 접근: 로그인 페이지로 리다이렉트 (useAuthStore 처리)
- URL 직접 입력: 미들웨어에서 차단 후 이 화면 표시

### 접근성 (A11y)
- 에러 영역 `role="main"` + heading `aria-level="1"`
- "대시보드로 이동" 버튼 명확한 aria-label

### 연결 화면
- 이전: 진입 시도
- 다음: `/dashboard`
