## 📝 디스크립션

### 사용 시점 (Trigger)
- Supabase API 호출 실패 (4xx / 5xx)
- 네트워크 오류 (fetch throw)
- RLS 정책 예외 (403)

### 필수 데이터
| 항목 | 값 |
|------|----|
| error | Error 객체 (message 포함) |
| isLoadingData | false |

### 인터랙션 (User Actions)
1. "다시 시도" 버튼 → `handleRetry()` → `01-로딩` 전환
2. PageHeader "직원 등록" 버튼 → `/staff/new` (에러 중에도 접근 가능)
3. 뒤로가기 → 이전 라우트

### 비즈니스 룰
- toast.error는 fetch 실패 1회만 발사 (재시도 중 중복 방지)
- 에러 메시지에 내부 스택 트레이스 노출 금지
- 403 에러: "권한이 없습니다" 별도 메시지 표시

### 에지 케이스
- 재시도 후 재실패: 에러 배너 유지, toast 재발사
- 500ms 이내 재시도 클릭 방지 (debounce/disabled 처리)
- 부분 데이터 로드 중 오류: 이미 로드된 데이터는 표시 (부분 에러)

### 접근성 (A11y)
- 에러 배너 `role="alert"` + `aria-live="assertive"`
- "다시 시도" 버튼 `aria-label="직원 목록 다시 불러오기"`
