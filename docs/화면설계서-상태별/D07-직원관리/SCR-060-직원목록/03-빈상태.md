## 📝 디스크립션

### 사용 시점 (Trigger)
- fetch 완료 후 `staffList.length === 0`
- 탭 필터 결과 없음 (예: '퇴사' 탭인데 퇴사자 없음)
- 검색어 입력 후 일치하는 직원 없음

### 두 가지 빈 상태 구분
| 구분 | 조건 | 메시지 |
|------|------|--------|
| 최초 빈 상태 | `search === '' && staffList.length === 0` | "등록된 직원이 없습니다." + 등록 버튼 |
| 검색 결과 없음 | `search !== '' && result.length === 0` | "검색 결과가 없습니다." + 검색어 초기화 안내 |

### 인터랙션 (User Actions)
1. "직원 등록하기" 버튼 → `/staff/new` (SCR-061 등록 모드)
2. PageHeader "직원 등록" 버튼 → 동일
3. 탭 전환 → 해당 탭 재조회
4. 검색어 삭제 → 전체 목록 재조회

### 비즈니스 룰
- manager 역할도 빈 상태 메시지 확인 가능
- 직원 등록 버튼: `hasPermission('staff:create')` true인 경우만 표시
- 검색 빈 상태에서는 등록 버튼 숨김

### 에지 케이스
- 탭 필터링 중 전환 시 이전 결과 잠깐 노출 방지 → transition 처리
- 검색 + 탭 조합 결과 없음: 두 조건 모두 안내 메시지에 반영

### 접근성 (A11y)
- 빈 상태 영역 `role="status"` + `aria-live="polite"`
- 빈 상태 메시지 `aria-label` 명확히
