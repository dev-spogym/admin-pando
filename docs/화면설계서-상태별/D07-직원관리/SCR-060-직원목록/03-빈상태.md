## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `staff-list-empty` |
| 상태 ID | 03 |
| 다이어그램 노드 | F6 > STATE_EMPTY |
| 이전 상태 | `01-로딩` / `02-정상` (필터 결과 0) |
| 다음 상태 | `02-정상` (등록/필터 해제 후) |

## 진입 조건
- `isLoadingData === false`
- `filtered.length === 0`
- 전체 0명 or 필터 결과 0명

## 비주얼 델타

```
┌──────────────────────────────────────────────────────────┐

│        [+ 첫 직원 등록하기] (전체 0명 CTA)                │
```

| 변경 요소 | From → To |
|---|---|
| DataTable body | row → `emptyMessage` + 중앙 Illustration |
| 액션 버튼 바 | 메시지/상태/퇴사/엑셀 disabled |
| StatCard | 실제 값 → 전부 `0명` |
| CTA(A) | — | `<Button variant="primary" onClick={()=>moveToPage(998)}>+ 첫 직원 등록하기</Button>` |
| CTA(B) | — | `<Button variant="outline" onClick={handleResetFilters}>필터 초기화</Button>` |

## 역할별 차이
- 등록 권한 없는 역할: CTA 숨김, 안내 텍스트만.
- primary/superAdmin: "다른 지점 확인" 힌트 텍스트 (옵션).

## 고유 인터랙션
- "+ 첫 직원 등록하기" → `moveToPage(998)` → SCR-061 등록.
- "필터 초기화" → `handleResetFilters()`.
- 키보드 포커스 트랩: 빈 테이블에서도 CTA 접근 가능.

## 비즈니스 룰 델타
1. **두 분기 구분**:
   - (A) `staffData.length === 0` — 전체 0명 (신규 지점)
   - (B) `staffData.length > 0 && filtered.length === 0` — 필터 결과 0
2. (A) → CTA "+ 첫 직원 등록하기". (B) → "필터 초기화".
3. (A) 상태에서 온보딩 가이드 오버레이 연동 가능(선택).

## 에지 케이스
- **일시적 0건 → 검색 변경**: 즉시 `02-정상` 전이.
- **신규 지점 (primary)**: (A) + "지점 관리" 안내.
- **엑셀 시도**: filtered=0 → 버튼 disabled.

## 바이브코딩 프롬프트 (델타)
```
filtered.length === 0 상태:
분기 (A) staffData.length===0:

  description="첫 직원을 등록해 운영을 시작하세요."

분기 (B) 필터 적용:
  title="검색 결과가 없습니다."
  description="필터를 변경하거나 초기화하세요."
  action=<Button variant="outline" onClick={handleResetFilters}>필터 초기화</Button>

- 액션 버튼 바 전부 disabled
- StatCard 0으로 렌더 (숨기지 않음)
```
