## 📝 디스크립션

### 사용 시점 (Trigger)
- fetch 완료 후 `staffList.length > 0`
- 탭 필터/검색 변경 후 결과 있음

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 직원 목록 | `staff` | `center_id = ?`, 탭 필터, 검색어 |
| 역할 표시 | `staff.role` | OWNER/MANAGER/FC/STAFF |
| 상태 배지 | `staff.staff_status` | ACTIVE/ON_LEAVE/RESIGNED/TRANSFERRED/LOCKED |

### 인터랙션 (User Actions)
1. 탭 클릭 → staffStatus 필터 변경 → 재fetch
2. 검색 input 입력 (debounce 300ms) → 재fetch
3. 행 체크박스 선택 → `08-선택있음` 전환
4. 이름 클릭 → 직원 상세 (SCR-060 내 슬라이드오버 또는 별도 라우트)
5. 수정 버튼 → `/staff/edit?id=xxx`
6. 근태 버튼 → `/staff/attendance?staffId=xxx`
7. 급여 버튼 → `/payroll?staffId=xxx` (센터장/manager/본사만, hasPermission 확인)

### 비즈니스 룰
- `hasPermission('payroll:view')` false 시 급여 버튼 hidden
- 퇴사(RESIGNED) 직원은 수정/근태 버튼 disabled
- LOCKED 직원: 수정 가능, 퇴사 처리 불가

### 에지 케이스
- 검색 결과 없음: `03-빈상태` 전환
- 탭 전환 중 검색어 유지
- 50명 초과: 페이지네이션 (pageSize=20)

### 접근성 (A11y)
- DataTable `role="grid"`, 헤더 `scope="col"`
- StatusBadge `aria-label="상태: {label}"`
- 체크박스 `aria-label="{이름} 선택"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `07-선택없음` / `08-선택있음` / `/staff/new` / `/staff/edit`
