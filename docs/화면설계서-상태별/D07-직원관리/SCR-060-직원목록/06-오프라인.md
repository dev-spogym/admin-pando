## 진입 조건
- `navigator.onLine === false`
- `window.addEventListener('offline')` 발생
- fetch 실패 `TypeError: Failed to fetch` → 오프라인으로 판정

## 역할별 차이
- 모든 역할 공통 (오프라인은 role-invariant). 단, primary의 지점 전환은 캐시 지점 목록에 한해 허용.

## 고유 인터랙션
- "다시 연결" → `refetch()` + `navigator.onLine` 재확인.
- 온라인 복귀 자동: `window.ononline` 이벤트 수신 → toast "네트워크 복구됨" + 자동 refetch.
- 쓰기 시도: 버튼 비활성으로 막되 우회 시 toast.error "오프라인에서는 저장할 수 없습니다".

## 비즈니스 룰 델타
1. **Service Worker 캐시**: 직전 응답을 `runtimeCaching`으로 저장. 만료 1시간.
2. **낙관적 업데이트 비활성**: 오프라인 중 mutation 큐잉 하지 않음 (데이터 정합성 리스크 회피).
3. **엑셀은 허용**: 현재 캐시 filtered 데이터로 클라이언트 측 XLSX 생성.
4. **상태 배지**: Sidebar 하단에도 "오프라인" 점 표시.

## 에지 케이스
- **불안정 네트워크 (플리킹)**: 1초 debounce로 배너 깜빡임 방지.
- **캐시 전무**: 최초 진입이 오프라인이면 `03-빈상태` 변형 "오프라인: 캐시 없음" 메시지.
- **시간대 캐시 오래된 경우**: 배너에 "마지막 갱신 N분 전" 표시.
- **VPN 등 부분 연결**: `navigator.onLine=true`여도 API timeout → `04-에러`로.

## 바이브코딩 프롬프트 (델타)
```
useEffect(() => {
  const on = () => { setOffline(false); toast.info('네트워크 복구됨'); refetch(); };
  const off = () => setOffline(true);
  window.addEventListener('online', on);
  window.addEventListener('offline', off);
  setOffline(!navigator.onLine);
  return () => { window.removeEventListener('online', on); window.removeEventListener('offline', off); };
}, []);

{offline && (
  <OfflineBanner tone="warn" role="status" aria-live="polite"
    className="sticky top-0 z-20 flex items-center gap-2 rounded-lg border border-amber-200 bg-amber-50 p-3 text-sm text-amber-800">
    <WifiOff size={16}/>
    <span>오프라인 상태입니다. 캐시된 데이터를 표시 중</span>
    {lastSyncedAt && <span className="ml-auto text-xs text-amber-700">마지막 갱신 {formatRelative(lastSyncedAt)}</span>}
    <Button size="sm" variant="outline" onClick={()=>{ refetch(); }}>다시 연결</Button>
  </OfflineBanner>
)}

- 쓰기 버튼(등록/메시지/상태/퇴사): disabled + aria-disabled + title
- 엑셀 버튼: 활성 (로컬 처리)
- StatCard description: "(캐시)" 접미
- 새 mutation 수행 시도 시 toast.error("오프라인에서는 저장할 수 없습니다.")
```
