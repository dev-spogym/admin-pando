## 📝 디스크립션

### 사용 시점 (Trigger)
- fetch 시도 중 `navigator.onLine === false`
- `window offline` 이벤트 발생
- Supabase 연결 타임아웃 (네트워크 불안정)

### 필수 데이터
| 항목 | 값 |
|------|----|
| isOffline | true |
| staffList | [] |

### 인터랙션 (User Actions)
1. 네트워크 복구 → `window online` 이벤트 → 자동 `fetchStaffList()` 재호출
2. "다시 시도" 버튼 클릭 (네트워크 있을 때만 활성) → 수동 재시도

### 비즈니스 룰
- 오프라인 배너는 전체 앱 레이아웃 상단에 고정 표시
- 네트워크 복구 시 자동 재조회 (사용자 액션 불필요)
- 이미 로드된 데이터 있으면 유지 표시 + "오프라인" 배너만 추가

### 에지 케이스
- VPN 연결 변경: offline → online 순환 시 중복 fetch 방지 (debounce 1초)
- 오프라인 중 탭 전환: 각 탭 재조회 없이 오프라인 UI 유지

### 접근성 (A11y)
- 상단 배너 `role="alert"` + `aria-live="assertive"`
- "다시 시도" `aria-disabled="true"` (오프라인 중)

### 연결 화면
- 이전: `01-로딩`
- 다음: 네트워크 복구 → `01-로딩`
