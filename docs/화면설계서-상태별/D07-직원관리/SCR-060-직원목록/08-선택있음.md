## 📝 디스크립션

### 사용 시점 (Trigger)
- `selectedRows.size >= 1` — 체크박스 1개 이상 선택

### 인터랙션 (User Actions)
1. "메시지 보내기" → 메시지 전송 모달 (SMS/앱 푸시)
2. "상태 변경" → DLG-060-002 오픈 (ACTIVE/ON_LEAVE 등 변경)
3. "퇴사 처리" → DLG-060-001 오픈 (danger variant, 확인 텍스트 필요)
4. 체크박스 재해제 → `07-선택없음` 복귀

### 비즈니스 룰
| 액션 | 허용 역할 |
|------|-----------|
| 메시지 보내기 | primary / owner / manager |
| 상태 변경 | primary / owner / manager |
| 퇴사 처리 | primary / owner (`hasPermission('staff:resign')`) |

- RESIGNED 상태 직원이 선택 포함 시: 퇴사 처리 버튼 disabled + 툴팁 "이미 퇴사한 직원이 포함되어 있습니다."
- LOCKED 직원 선택 포함: 상태 변경 가능, 퇴사 불가

### 에지 케이스
- 50명 전체 선택 후 퇴사 처리 → 확인 모달에서 N명 명시
- 선택 후 검색어 변경 → 선택 초기화 여부: 초기화 (UX 혼란 방지)

### 접근성 (A11y)
- "N명 선택됨" `aria-live="polite"` 영역
- 퇴사 처리 버튼 danger 색상 + `aria-label="선택한 {N}명 퇴사 처리"`
