## 역할별 차이
| 역할 | 선택된 상태에서의 액션 |
|---|---|
| superAdmin/primary | 전 액션 활성 + 타 지점 직원 선택 가능 |
| owner | 전 액션 활성 (본인 지점만) |
| manager | 메시지/엑셀 활성, 상태변경은 "휴직"만, 퇴사 여전히 disabled + "센터장 권한 필요" |
| 기타 | 본 상태 도달 불가 (`05-권한없음`) |

## 고유 인터랙션
- 헤더 체크박스 클릭:
  - none → checked(현재 페이지 전체)
  - indeterminate → checked
  - checked → none(현재 페이지 해제)
- 행 체크박스 클릭: 개별 토글. Shift+클릭으로 범위 선택 (선택 기능).
- "선택 해제" 클릭: `setSelectedRows(new Set())` → `07` 전이.
- 메시지 전송 클릭: `moveToPage(980,{ids:Array.from(selectedRows)})` → SCR-071.
- 상태 변경 클릭: DLG-060-002 오픈 (count=N).
- 퇴사 처리 클릭: DLG-060-001 오픈 (count=N, confirmationText="퇴사처리").

## 비즈니스 룰 델타
1. **카운트 배지**: 버튼 텍스트 옆 `<Badge>{N}</Badge>` 또는 "(N)" 접미.
2. **페이지 이동 정책**: 현재 페이지의 선택만 유지 → 다른 페이지 이동 시 선택 해제(정책).
3. **대량 선택 경고**: 선택 > 20 시 하단에 "대량 퇴사 처리 시 주의" 경고 배너 (선택).
4. **퇴사 처리 호출**:
   - DLG-060-001 `confirmationText="퇴사처리"` 입력 필수.
   - 실행: `supabase.from('staff').update({ isActive:false, staffStatus:'RESIGNED', resignedAt:now }).in('id', ids)`.
   - E422200 담당 회원 존재 → 경고 dialog + "SCR-062 퇴사 위자드로" 권유.
   - 성공 후: `fetchStaff()` + `setSelectedRows(new Set())` → `07`.
5. **상태 변경 호출**: DLG-060-002 적용 후 update in-place → refetch.
6. **메시지 전송**: 선택된 ids URL 쿼리 또는 전역 스토어 전달.

## 에지 케이스
- **선택 후 필터 변경**: filtered에 없는 id는 자동 해제.
- **일부 행이 타 지점 소속**(primary 전 지점 조회 중): 요약 바에 "전 지점 {N}명" 표기.
- **API 실패 후 재시도**: 선택 유지.
- **이미 퇴사된 직원 선택 후 퇴사 재시도**: E409200 → 해당 행 skip, 나머지 성공 toast.
- **체크박스와 행 클릭 이벤트 분리**: row click은 상세로, checkbox는 선택만 (`e.stopPropagation()`).

## 바이브코딩 프롬프트 (델타)
```
selectedRows.size >= 1 (조건):
const n = selectedRows.size;

<Button variant="outline"
        onClick={()=>moveToPage(980,{ids:Array.from(selectedRows)})}>
  <MessageSquare size={16}/> 메시지 전송
  <Badge className="ml-2 bg-primary/10 text-primary">{n}</Badge>
</Button>

<Button variant="outline" disabled={!canChangeStatus(role)}
        onClick={()=>setIsStatusDialogOpen(true)}>
  <Settings2 size={16}/> 상태 변경 <Badge>{n}</Badge>
</Button>

<Button variant="danger" disabled={!canRetire(role)}
        onClick={()=>setIsRetireDialogOpen(true)}>
  <UserMinus size={16}/> 퇴사 처리 <Badge className="bg-white/20 text-white">{n}</Badge>
</Button>

// 선택 요약 바 (선택 옵션)
{n > 0 && (
  <div className="flex items-center justify-between px-4 py-2 rounded-lg bg-primary/5 ring-1 ring-primary/10">
    <span className="text-sm font-medium text-primary">{n}명 선택됨</span>
    <button className="text-sm text-primary hover:underline"
            onClick={()=>setSelectedRows(new Set())}>선택 해제</button>
  </div>
)}

// 행 스타일
<tr className={selectedRows.has(row.id) ? 'bg-primary/5 border-l-2 border-primary' : ''}>
  ...
</tr>

// 헤더 체크박스
const isAll = pageData.length > 0 && pageData.every(r => selectedRows.has(r.id));
const isSome = pageData.some(r => selectedRows.has(r.id)) && !isAll;
<input type="checkbox" checked={isAll} ref={el => el && (el.indeterminate = isSome)}
       onChange={toggleAllOnCurrentPage} />

// 퇴사 처리 (DLG-060-001 onConfirm)
async function handleRetire() {
  const ids = Array.from(selectedRows);
  const { data, error } = await supabase.from('staff')
    .update({ isActive:false, staffStatus:'RESIGNED', resignedAt:new Date().toISOString() })
    .in('id', ids).select('id');
  if (error) {
    if (error.code === 'E422200') { openE422Dialog(); return; }
    toast.error('퇴사 처리에 실패했습니다.'); return;
  }
  toast.success(`${ids.length}명의 퇴사 처리가 완료되었습니다.`);
  setIsRetireDialogOpen(false);
  setSelectedRows(new Set());
  refetch();
}
```
