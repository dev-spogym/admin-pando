## 🧩 바이브코딩 프롬프트

```
API 호출 핸들러:
  const handleResignConfirm = async () => {
    if (!handleValidate()) return
    setIsProcessing(true)

    try {
      // 복수 직원 일괄 처리
      const { error } = await supabase
        .from('staff')
        .update({ staff_status: 'RESIGNED', resigned_at: new Date().toISOString() })
        .in('id', targetStaffIds)

      if (error) throw error

      // 성공
      toast.success(`${targetStaffIds.length}명의 퇴사 처리가 완료되었습니다.`)
      setResignModalOpen(false)
      setSelectedRows(new Set())
      fetchStaffList()  // SCR-060 목록 갱신

    } catch (err: any) {
      if (err.code === '42501') {
        // RLS 권한 오류 (403 equivalent)
        toast.error('퇴사 처리 권한이 없습니다.')
        setResignModalOpen(false)
      } else if (err.status === 400) {
        toast.error('입력 오류가 발생했습니다. 다시 시도해주세요.')
        // 모달 유지
      } else {
        toast.error('서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.')
        // 모달 유지, 재시도 가능
      }
    } finally {
      setIsProcessing(false)
    }
  }
```

## 📝 디스크립션

### API 분기 결과
| HTTP 응답 | 처리 |
|-----------|------|
| 200 OK | toast.success + 모달 닫힘 + SCR-060 갱신 |
| 400 Bad Request | toast.error('입력 오류') + 모달 유지 |
| 403 / RLS 차단 | toast.error('권한 없음') + 모달 닫힘 |
| 500 Server Error | toast.error('서버 오류') + 모달 유지 + 재시도 가능 |

### 성공 후 처리
1. `toast.success('N명의 퇴사 처리가 완료되었습니다.')`
2. `setResignModalOpen(false)` — 모달 닫힘
3. `setSelectedRows(new Set())` — SCR-060 선택 초기화
4. `fetchStaffList()` — 목록 갱신

### 인터랙션 (User Actions)
- 처리 중: 없음 (모든 버튼 disabled)
