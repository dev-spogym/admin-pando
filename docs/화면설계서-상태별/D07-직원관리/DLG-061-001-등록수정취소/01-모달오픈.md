## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-061 취소 버튼 클릭 시 `isDirty === true`
- 변경 없음(`isDirty=false`): 모달 없이 즉시 `/staff` 이동

### 두 가지 진입 경로
| 조건 | 처리 |
|------|------|
| `isDirty=true` | DLG-061-001 모달 오픈 |
| `isDirty=false` | 모달 생략, 즉시 `/staff` 이동 |

### 인터랙션 (User Actions)
1. "계속 작성하기" → 모달 닫힘, 폼 포커스 복귀, 입력 데이터 유지
2. "네, 취소합니다" → `handleCancelConfirm()` → SCR-060 이동
3. ESC 키 → 모달 닫힘 (= 계속 작성하기와 동일)
4. 모달 외부 클릭 → 닫힘

### 비즈니스 룰
- 모달은 정보 제공 + 확인 목적 (API 호출 없음)
- 취소 확인 후 폼 데이터 초기화 (`form.reset()`)
- 브라우저 뒤로가기: isDirty 시 beforeunload 이벤트 (향후 구현)

### 접근성 (A11y)
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby`
- 기본 포커스: "계속 작성하기" 버튼 (안전한 선택 우선)
- ESC 닫기
