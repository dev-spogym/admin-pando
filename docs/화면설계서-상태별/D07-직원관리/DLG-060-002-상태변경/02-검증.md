## 🧩 바이브코딩 프롬프트

```
검증 로직:

  const validateStatusChange = (): boolean => {
    // 1. 상태 미선택
    if (!selectedStatus) {
      toast.warning('변경할 상태를 선택하세요.')
      return false
    }

    // 2. 현재 상태와 동일 (단일 선택 시)
    if (targetStaffIds.length === 1 && selectedStatus === currentStatus) {
      setSameStatusError('현재 상태와 동일합니다. 다른 상태를 선택하세요.')
      return false
    }

    // 3. 사유 필수 상태에서 사유 미입력
    if (requiresReason && !statusReason.trim()) {
      setReasonError('사유를 입력하세요.')
      return false
    }

    setSameStatusError(null)
    setReasonError(null)
    return true
  }

현재 상태 동일 경고:
  {sameStatusError && (
    <p className="mt-2 text-xs text-orange-600" role="alert">{sameStatusError}</p>
  )}

사유 미입력 에러:
  {reasonError && (
    <p className="mt-1 text-xs text-red-600" role="alert">{reasonError}</p>
  )}

"변경 확인" 버튼 비활성 조건:
  disabled={
    !selectedStatus ||
    (targetStaffIds.length === 1 && selectedStatus === currentStatus) ||
    (requiresReason && !statusReason.trim()) ||
    isProcessing
  }
```

## 📝 디스크립션

### 검증 규칙
| 조건 | 메시지 | 처리 |
|------|--------|------|
| 상태 미선택 | "변경할 상태를 선택하세요." | toast.warning |
| 현재 상태와 동일 | "현재 상태와 동일합니다." | 인라인 경고 |
| 사유 미입력(ON_LEAVE/TRANSFERRED) | "사유를 입력하세요." | 인라인 에러 |

### 인터랙션 (User Actions)
1. 상태 변경 → sameStatusError 즉시 클리어
2. 사유 입력 → reasonError 즉시 클리어
3. 검증 통과 → `03-결과분기`
