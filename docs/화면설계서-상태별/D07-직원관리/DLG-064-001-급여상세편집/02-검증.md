## 🧩 바이브코딩 프롬프트

```
급여 상세/편집 모달 — 로드 완료 + 편집 상태:

모달 본문 (isLoadingDetail=false):
  <div className="space-y-6">
    {/* 직원 정보 (읽기 전용) */}
    <div className="rounded-lg bg-gray-50 p-4">
      <p className="font-medium">{payrollDetail.staff.name}</p>
      <p className="text-sm text-gray-500">
        {payrollDetail.staff.department} · {payrollDetail.staff.position}
      </p>
      <p className="text-xs text-gray-400 mt-1">
        입사일: {format(payrollDetail.staff.hire_date, 'yyyy.MM.dd')}
      </p>
    </div>

    {/* 급여 항목 */}
    <div className="grid grid-cols-2 gap-4">
      <div>
        <label className="text-sm font-medium text-gray-700">기본급</label>
        <input
          type="number"
          {...register('base_salary', { min: { value: 0, message: '0 이상의 값을 입력하세요.' } })}
          disabled={payrollDetail.status === 'CONFIRMED' || !hasPermission('payroll:edit')}
          className="mt-1 w-full rounded-lg border px-3 py-2 text-sm"
        />
        {errors.base_salary && (
          <p className="mt-1 text-xs text-red-600" role="alert">{errors.base_salary.message}</p>
        )}
      </div>

      <div>
        <label className="text-sm font-medium text-gray-700">수당</label>
        <input
          type="number"
          {...register('allowances', {
            min: { value: 0, message: '0 이상의 값을 입력하세요.' },
            required: '필수 항목입니다.',
          })}
          disabled={payrollDetail.status === 'CONFIRMED' || !hasPermission('payroll:edit')}
          className="mt-1 w-full rounded-lg border px-3 py-2 text-sm"
        />
        {errors.allowances && (
          <p className="mt-1 text-xs text-red-600" role="alert">{errors.allowances.message}</p>
        )}
      </div>

      <div>
        <label className="text-sm font-medium text-gray-700">공제</label>
        <input
          type="number"
          {...register('deductions', {
            min: { value: 0, message: '0 이상의 값을 입력하세요.' },
          })}
          disabled={payrollDetail.status === 'CONFIRMED' || !hasPermission('payroll:edit')}
          className="mt-1 w-full rounded-lg border px-3 py-2 text-sm"
        />
      </div>

      <div>
        <label className="text-sm font-medium text-gray-700">실수령액 (자동 계산)</label>
        <div className="mt-1 rounded-lg bg-blue-50 border border-blue-200 px-3 py-2 text-sm font-bold text-blue-700">
          {formatNumber(watchBaseSalary + watchAllowances - watchDeductions)}원
        </div>
      </div>
    </div>

    {/* 확정 상태 알림 */}
    {payrollDetail.status === 'CONFIRMED' && (
      <div className="rounded-md bg-yellow-50 border border-yellow-200 p-3" role="note">
        <p className="text-sm text-yellow-700">확정된 급여는 수정할 수 없습니다.</p>
      </div>
    )}
  </div>

실수령액 실시간 계산:
  const watchBaseSalary = watch('base_salary') || 0
  const watchAllowances = watch('allowances') || 0
  const watchDeductions = watch('deductions') || 0
  const netSalary = watchBaseSalary + watchAllowances - watchDeductions

검증 규칙:
  - 음수: "0 이상의 값을 입력하세요."
  - 필수 항목 빈 값: "필수 항목입니다."
  - 한도 초과 (10억 이상): "허용 한도를 초과했습니다."

하단 버튼:
  <Button variant="outline" onClick={() => setDetailModalOpen(false)}>취소</Button>
  {hasPermission('payroll:edit') && payrollDetail.status !== 'CONFIRMED' && (
    <Button
      variant="primary"
      disabled={!formState.isValid || isSaving}
      onClick={handleSavePayroll}
    >
      저장
    </Button>
  )}
```

## 📝 디스크립션

### 검증 규칙
| 필드 | 조건 | 에러 메시지 |
|------|------|-------------|
| 기본급 | 음수 | "0 이상의 값을 입력하세요." |
| 수당 | 빈 값 / 음수 | "필수 항목입니다." / "0 이상의 값을 입력하세요." |
| 공제 | 음수 | "0 이상의 값을 입력하세요." |
| 모든 필드 | 10억 이상 | "허용 한도를 초과했습니다." |

### 편집 가능 조건
| 조건 | 편집 가능 |
|------|-----------|
| status === 'PENDING' + payroll:edit 권한 | O |
| status === 'CONFIRMED' | X (읽기 전용) |
| manager 역할 | X (조회만) |

### 인터랙션 (User Actions)
1. 수당/공제 입력 → 실수령액 실시간 재계산
2. blur → 개별 필드 검증
3. "저장" 클릭 → 전체 검증 → `03-결과분기`
