## 상태 고유 인터랙션
1. StatCard 클릭 → href 있으면 이동 (전체 회원→`/members`, 매출→`/sales`)
2. 지점 카드 "전환" → `switchBranch(id)` → `/` (SCR-090) + toast
3. 지점 카드 "상세" → `/branches?id={id}`
4. 지점 카드 전체 클릭 → 상세 동일
5. 감사로그 행 클릭 → `/audit-log?id={id}`
6. 감사로그 "전체보기" → `/audit-log`
7. 새로고침 → 전 쿼리 invalidate, 60초 쿨다운
8. 지점 폐점/휴업 드롭다운 → 2단계 확인 다이얼로그

## 비즈니스 룰 델타
- **MoM/DoD**: `prev > 0 ? Math.round(((curr-prev)/prev)*100) : 0`
- **지점 상태 배지**:
  - `ACTIVE` → `🟢 운영중` emerald
  - `SUSPENDED` → `🟡 휴업` amber
  - `CLOSED` → `🔴 폐점` rose, opacity-70
- **매출 포맷**: 1억+ `"X.X억"`, 1천만+ `"N,NNN만"`, 그 외 콤마+원
- **감사로그 scope**: super/primary 모두 `scope='all'`
- **지점 전환 감사**: 자동 `BRANCH_SWITCH` 기록

## 에지 케이스
- **전 지점 운영 정지**: 모든 SUSPENDED → UI 정상, KPI 0 가능
- **지점 1개 primary**: 카드 1개 렌더
- **매출 전월 0원**: 델타 0% (분모 0)
- **감사로그 0건**: "최근 활동 없음" + 전체보기 링크
- **일부 쿼리 실패**: 해당 섹션만 `WidgetError`

function calcDelta(curr: number, prev: number, label: string) {
  if (prev <= 0) return { value: 0, direction: 'zero' as const, label };
  const pct = Math.round(((curr - prev) / prev) * 100);
  return { value: pct, direction: pct > 0 ? 'up' : pct < 0 ? 'down' : 'zero', label };
}
