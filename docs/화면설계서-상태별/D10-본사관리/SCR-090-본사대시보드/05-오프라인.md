## 역할별 차이
- **공통**: 모든 역할에 동일한 stale 경고
- **super/primary**: stale 배지 hover 시 "이 데이터는 오프라인 캐시입니다. 지점 전환 불가"
- **owner/manager**: 매출 카드에 추가 경고 "결제·환불은 온라인 상태에서만 가능합니다"

## 상태 고유 인터랙션
1. **새로고침 버튼 시도**: 온라인 체크 → 오프라인이면 스낵바 "오프라인 상태입니다"
2. **StatCard 클릭**: 상세 페이지로 이동 시도 → 해당 페이지가 자체 오프라인 UI 표시
3. **위젯 항목 클릭**: 동일
4. **지점 전환**: 불가 (drop-down disabled)
5. **자동 복구**: `online` 이벤트 → silent refetch → stale 상태 해제 → `01-로딩` → `02-정상`

## 상태 고유 비즈니스 룰 델타
- **캐시 TTL**: React Query `staleTime: 60_000`, `gcTime: 600_000` (10분). SW 캐시는 `workbox` 설정에 따름.
- **Stale 배지 조건**: 캐시 나이(age) > 1분 또는 오프라인 진입 후
- **Optimistic UI 금지**: 이 상태에서 mutation 모두 차단(실패 확정이므로)
- **갱신 시간 표시**: `formatDistanceToNow(lastUpdatedAt, { addSuffix: true, locale: ko })`
- **쓰기 액션 큐잉 (선택)**: 오프라인에서 시도한 액션을 queue에 저장 → 복구 시 재시도 (complexity 높음, 우선순위 P2)

## 에지 케이스
- **캐시 부분 있음**: 있는 위젯은 stale, 없는 위젯은 "데이터 없음" (03-빈상태 규칙 적용 X — 대신 "오프라인으로 불러올 수 없음")
- **매우 오래된 캐시(1시간+)**: 배지 색상을 더 짙게 (amber → orange) + "오래된 데이터입니다" 안내
- **로그인 전환 중 오프라인**: 이전 사용자 캐시 노출 금지 → 완전 로그아웃 UI
- **권한 변경 중 오프라인**: 이전 역할 뷰 유지 (새 역할 적용은 온라인 복구 후)
