## 상태 진입 조건
**전역 빈 상태 (Global Empty)**
- `stats.totalMembers === 0` **AND** 주요 위젯 결과 배열이 모두 빈 배열
- 전형적 시나리오: 신규 지점 개설 직후, 본사가 새 지점 전환 시 아직 데이터 없음

**부분 빈 상태 (Partial Empty)**
- 특정 위젯만 빈 (예: 생일자 0명이지만 회원은 존재) → 해당 위젯만 빈 상태 UI
- 마스터 §2 정상 레이아웃에서 위젯 단위로 `<WidgetEmpty>` 렌더


## 상태 고유 인터랙션
1. **"회원 등록하기"** 버튼 → `/members/new`
2. **"도움말 보기"** → 온보딩 페이지 `/onboarding` (P0 역할) 또는 외부 가이드
3. **위젯 내부 "전체보기"**: 빈 상태에서는 링크 비활성(클릭 불가) 또는 "회원이 없습니다"로 교체
4. **지점 전환**: 다른 지점에 데이터가 있으면 즉시 `02-정상`으로 전환

## 상태 고유 비즈니스 룰 델타
- **완전 신규 지점 감지**: `createdAt > now - 7d` 이고 stats 모두 0 → 온보딩 권유 배너
- **지점 폐점/휴업**: branches.status !== 'ACTIVE'이면 빈 상태 위 "이 지점은 현재 휴업 중입니다" 배너 추가
- **트레이너 담당 회원 0명**: "담당 회원이 배정되지 않았습니다. 매니저에게 문의하세요"
- **프론트 오늘 출석 0명**: "아직 오늘 출석이 없습니다" (시간대 감안, 새벽이면 정상)

## 에지 케이스
- **날짜 필터로 인한 빈 상태**: 매출 차트에서 선택 기간 매출 0일 경우 "해당 기간 매출이 없습니다" + 기간 변경 힌트
- **API는 정상인데 배열만 빈**: 빈 상태로 처리(에러 아님)
- **권한 필터로 인한 빈 상태**: trainer에게 담당 없음 → 분명히 안내
