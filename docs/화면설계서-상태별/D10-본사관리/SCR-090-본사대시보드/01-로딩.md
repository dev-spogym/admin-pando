## 상태 고유 인터랙션
- **모든 위젯 클릭 불가** (`pointer-events-none opacity-70`)
- BranchSwitcher, 새로고침은 `disabled` 처리 (로딩 중 이중 호출 방지)
- 사이드바 네비는 항상 활성 (다른 페이지 이동은 허용)
- 로딩이 **5초 초과**하면 Header 하단에 "조회에 시간이 걸리고 있습니다..." 보조 텍스트 표시
- 10초 초과 시 → `04-에러-API` 또는 계속 로딩(타임아웃 10s, `useQuery`의 `staleTime`과 별개)

## 상태 고유 비즈니스 룰 델타
- **skeleton 수량**: 역할별 KPI 카드 개수와 동일 (§마스터 §5.1 매트릭스)
- **CLS 방지**: 모든 스켈레톤은 실제 컴포넌트와 동일 치수/그리드 적용
- **첫 진입**: `runDailySync()`를 동시 호출 → 종료 시 refetch 트리거
- **Stream 렌더링**: Next 15 Suspense로 header는 먼저, 섹션별 순차 Suspense 경계 (선택)

## 에지 케이스
- **일부 쿼리만 완료**: 완료된 위젯은 즉시 실제 데이터로 전환, 나머지는 스켈레톤 유지 (부분 렌더링)
- **브라우저 타임 슬로우**: 일부 위젯이 10초 이상 pending → 해당 위젯만 ErrorBoundary fallback
- **역할 변경 중 진입**: user 객체 로드 전에는 전체 스켈레톤 (KPI 카드 개수는 기본 4개로 추정)
