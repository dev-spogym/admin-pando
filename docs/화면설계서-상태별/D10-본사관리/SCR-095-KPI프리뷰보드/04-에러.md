## 역할별 차이
- **primary/super**: 에러 카드에 "로그 상세 보기" 링크 추가 → `/audit-log?code=E500001`
- **그 외 역할**: 로그 상세 링크 비표시, 문의처 텍스트 표시

## 상태 고유 인터랙션
1. **[재시도]** → `queryClient.invalidateQueries(['kpi-preview'])` → `01-로딩`
2. **[대시보드로]** → `moveToPage('/')` (SCR-090)
3. **탭 전환** → 해당 탭 쿼리 신규 실행 (다른 탭은 정상일 수 있음)
4. **자동 재시도**: 30초 후 1회 (React Query `retry: 1, retryDelay: 30_000`)
5. **토스트 표시**: 진입 즉시 1회만

## 상태 고유 비즈니스 룰 델타
- **에러 매핑**:
  - `E401002` → 토큰 만료: 로그인 리다이렉트 (이 상태 미도달)
  - `E403001` → 권한 없음: 탭 자체 숨김 + toast, 이 상태 미도달
  - `E500001` → 서버 오류: 이 상태 + 재시도 버튼
  - `E503001` → 점검 중: 에러 카드 대신 점검 배너 "KPI 서비스 점검 중 (14:32~15:00)"
  - `NETWORK` → 네트워크 단절: "오프라인입니다" + 캐시 fallback 시도
- **재시도 쿨다운**: 3회 연속 실패 시 버튼 disabled + "잠시 후 다시 시도해주세요" (60초)
- **탭별 독립 에러**: hq 탭만 에러, 다른 탭은 정상 유지 가능

## 에지 케이스
- **점검 중 (503)**: 에러 카드 대신 warn 배너 + 자동 폴링(5초 간격, 최대 5분)
- **일부 탭만 에러**: 현재 탭이 에러여도 다른 탭 전환 시 정상 가능성
- **모든 탭 에러**: 전역 네트워크 문제 의심 → 오프라인 UI로 전환
- **재시도 중 탭 변경**: 이전 재시도 cancel + 새 탭 로딩

    error={query.error}

    <AlertTriangle className="size-8 text-rose-500" aria-hidden />

      에러코드: {errorCode} · 요청 시각: {formatKST(requestedAt,'HH:mm:ss')}
