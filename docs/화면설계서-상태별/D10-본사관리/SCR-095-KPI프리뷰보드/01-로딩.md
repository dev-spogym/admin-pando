## 상태 고유 비주얼 델타
마스터 §2 레이아웃 골격을 유지하되, 데이터 영역만 **스켈레톤**으로 교체.

```
[TabNav: 5개 탭 정상 렌더 (활성 탭 표시)]

§B KPI: [skel w-16 h-3] [skel w-24 h-8] × 4
§C 액션 큐 / 랭킹:
  [skel w-32 h-4]
  ─── skeleton rows × 4 (w-full h-16 rounded-xl bg-gray-100 animate-pulse)
§D 퍼널: 4단계 각 [skel w-full h-2 rounded-full]
§E 인사이트: [skel w-full h-20 rounded-xl] × 3
```

| 변경 요소 | Before (정상) | After (로딩) |
|---|---|---|
| StatCard 값 | `428명` | `<div className="h-8 w-24 bg-gray-200 rounded animate-pulse" />` |
| 액션 큐 항목 | 4개 행 | `<li className="h-16 bg-gray-100 rounded-xl animate-pulse" />` × 4 |
| 랭킹 행 | 4개 | 동일 스켈레톤 |
| 퍼널 진행률 | 실제 % 바 | `width:0%` + 회색 트랙 pulse |
| 인사이트 | 텍스트 | skeleton 3줄 |
| 탭 | 클릭 가능 | 클릭 가능(전환 허용, 새 로딩 시작) |

## 역할별 차이
- 전 역할 동일한 스켈레톤. 매출 마스킹 역할(fc/trainer/staff/front)은 로딩 후 `***` 표시 예정이므로 스켈레톤은 동일.

## 상태 고유 인터랙션
1. 탭 전환: 허용 (캐시 적중 시 즉시 정상)
2. 액션 버튼(자동알림/Today/매출/회원): 허용
3. 액션 큐/랭킹 항목 클릭: **비활성** (cursor-default, aria-disabled)

## 상태 고유 비즈니스 룰 델타
- 스켈레톤 최소 300ms 유지(깜빡임 방지)
- 10초 초과 시 `04-에러`로 전환 + `toast.error("요청이 지연되고 있습니다")`
- `prefers-reduced-motion` → pulse 제거, `opacity-60`

## 에지 케이스
- 캐시 적중 → 로딩 생략, 즉시 `02-정상`
- 탭 전환 + 캐시 적중 → 로딩 생략
- 네트워크 지연 10초 초과 → 에러
