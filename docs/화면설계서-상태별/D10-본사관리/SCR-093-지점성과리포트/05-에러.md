# SCR-093 지점 성과 리포트 — 05 에러

> 마스터 문서: `00-기본화면.md` (상속). 델타만 기술.

## 메타
| 항목 | 값 |
|---|---|
| 상태 ID | `report-error` |
| 다이어그램 | F6 `STATE_ERROR` / F8 |
| 이전 상태 | `01-로딩` |
| 다음 상태 | `01-로딩` (재시도) / 성공 |

## 상태 진입 조건
- `/branch-report` 또는 `/branch-report/summary` API 실패
- React Query `isError=true` && `data` 없음

### 변경 요소
- PageHeader 유지(월 선택 가능)
- KPI/차트/테이블 → ErrorCard 교체
- 엑셀 disabled
- 재시도 버튼 primary

## 역할별 차이
- primary/owner 동일. primary만 요청 ID 노출

## 인터랙션
| 요소 | 동작 |
|---|---|
| [다시 시도] | `queryClient.invalidateQueries(['branch-report'])` |
| [문의하기] | mailto/Intercom + 에러코드/요청ID 첨부 |
| 월 선택 | 가능 (다른 월로 전환하여 복구 시도) |

## 비즈니스 룰 델타
- Exponential backoff: 1s → 2s → 4s (retry 3회)
- 503 시 30초 자동 재시도 (최대 3회)
- Sentry 로깅
- 부분 실패 (summary만 실패): summary 카드만 "불러오기 실패" + 테이블 정상

## 에지 케이스
- 401: 전역 인터셉터 → `/login`
- 403: `06-권한없음` 대신 `/forbidden` 리다이렉트
- 오프라인: WifiOff 아이콘 + 온라인 복귀 시 자동 refetch
- 월 변경으로 복구: 새 쿼리 성공 시 02/03

## 바이브코딩 프롬프트 (델타)
```
if (isError) {
  const errorCode = error?.errorCode || 'E500001';
  return (
    <div className="rounded-xl bg-rose-50 border border-rose-200 p-8 text-center">
      <AlertCircle className="h-10 w-10 text-rose-600 mx-auto mb-3"/>
      <h2 className="text-lg font-semibold text-gray-900">리포트를 불러오지 못했습니다</h2>
      <p className="mt-2 text-sm text-gray-600">{errorMessageMap[errorCode]}</p>
      <p className="mt-2 font-mono text-xs text-gray-500">에러코드: {errorCode}</p>
      <div className="mt-4 flex gap-2 justify-center">
        <Button variant="primary" onClick={() => refetch()}>
          <RefreshCw className="h-4 w-4 mr-1"/> 다시 시도
        </Button>
        {user.role === 'primary' && (
          <Button variant="secondary" onClick={contactSupport}>문의하기</Button>
        )}
      </div>
    </div>
  );
}

useQuery({
  retry: (fc, e) => e.statusCode < 500 ? false : fc < 3,
  retryDelay: a => Math.min(1000 * 2**a, 30_000),
});
```
