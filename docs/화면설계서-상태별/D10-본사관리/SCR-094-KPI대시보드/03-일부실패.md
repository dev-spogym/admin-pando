### 변경 요소
| 요소 | 변경 |
|---|---|
| 실패 카드 | `ErrorKpiCard` (rose-50 배경, 📉 아이콘, 🔄 재시도) |
| 섹션 전체 실패(코호트) | 섹션 내부 ErrorCard, 다른 섹션은 정상 |
| 상단 경고 배너 | amber-50, 전체 재시도 버튼 |
| 정상 카드 | 기존 그대로 |

## 역할별 차이
- primary: 에러코드 + 요청ID 노출
- owner/manager: 간단한 "불러오기 실패" 문구만

## 인터랙션
| 요소 | 동작 |
|---|---|
| 카드 내 🔄 | 해당 쿼리만 `refetch()` |
| 상단 [전체 재시도] | 실패한 쿼리만 선택적으로 `refetchQueries({ predicate: q => q.isError })` |
| 정상 카드 클릭 | 원래 동작 유지 |
| 목표 설정 | 정상 작동 (매출 쿼리가 성공했다면) |

## 비즈니스 룰 델타
- ErrorBoundary per widget (개별 실패 격리)
- Sentry에 실패 쿼리 키 + 에러코드 기록
- 3회 연속 실패 시 자동 재시도 중단
- **코호트/퍼널**은 데이터 의존성이 커서 실패 시 섹션 전체 에러 카드로 대체
- 성공한 데이터만으로 UI 렌더 (부분 가용성)

## 에지 케이스
- 17개 실패, 1개만 성공: UI상으로는 `05-에러`에 가깝지만 규약상 일부실패로 취급 (배너 문구 강화)
- 매출 쿼리만 실패: 게이지 "데이터 없음" + 목표 설정 disabled
- 재시도 중 또 실패: 카드 유지 + toast
- 부분 성공 시 audit_log에 `PARTIAL_KPI_LOAD` 기록

## 바이브코딩 프롬프트 (델타)
```
function ErrorKpiCard({ label, onRetry, errorCode }) {
  return (
    <div className="rounded-xl bg-rose-50 border border-rose-200 p-4 flex flex-col items-center justify-center min-h-[96px]">
      <AlertCircle className="h-5 w-5 text-rose-600 mb-1"/>
      <p className="text-xs text-gray-500 uppercase">{label}</p>
      <p className="text-xs text-rose-600 mt-1">불러오기 실패</p>
      <button onClick={onRetry} className="mt-2 text-xs text-blue-600 hover:underline flex items-center gap-1">
        <RefreshCw className="h-3 w-3"/> 재시도
      </button>
    </div>
  );
}

// 렌더
{totalMembersQ.isError ? (
  <ErrorKpiCard label="전체 회원" onRetry={() => totalMembersQ.refetch()} errorCode={totalMembersQ.error?.code}/>
) : (
  <KpiCard label="전체 회원" value={formatNumber(totalMembersQ.data)} unit="명" icon={<Users/>}/>
)}

// 상단 배너
const failedCount = queries.filter(q => q.isError).length;
{failedCount > 0 && (
  <div role="status" className="rounded-md bg-amber-50 border border-amber-200 p-3 flex items-center justify-between">
    <span className="text-sm text-amber-800">
      ⚠ 일부 KPI를 불러오지 못했습니다 ({failedCount}/18)
    </span>
    <Button size="sm" variant="secondary" onClick={retryAllFailed}>
      <RefreshCw className="h-4 w-4 mr-1"/> 전체 재시도
    </Button>
  </div>
)}

const retryAllFailed = () => {
  queries.filter(q => q.isError).forEach(q => q.refetch());
  toast.info(`실패한 ${failedCount}개 쿼리 재시도 중...`);
};

// 코호트 섹션 에러
{cohortQ.isError ? (
  <ErrorSection title="코호트 분석" onRetry={() => cohortQ.refetch()} />
) : (
  <CohortAnalysis data={cohortQ.data}/>
)}
```
