## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본` 테이블에서 상태='적용중' 페널티의 [면제] 버튼 클릭

### 필수 데이터
- targetPenalty: Penalty (이미 메모리에 있음)

### 인터랙션 (User Actions)
1. 면제 사유 입력 (최소 10자)
2. [면제 처리] → UPDATE → toast.success → 재fetch
3. [취소] / ESC → 모달 닫기

### 비즈니스 룰
- 면제 사유 필수 (최소 10자)
- 면제 후 회원의 누적 페널티 점수에서 해당 점수 차감
- 면제 처리자: 현재 로그인 스태프 자동 기록

### 에지 케이스
- 이미 면제된 페널티에 [면제] 접근: 버튼 비활성 (UI 방어)

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby`
- 면제 사유 textarea `aria-required="true"` + `aria-describedby="min-length-hint"`
