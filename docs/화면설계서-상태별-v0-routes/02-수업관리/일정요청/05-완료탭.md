## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C009 일정 요청 — 상태: 처리 완료 탭 (승인/반려 이력)

파일: src/app/schedule-requests/page.tsx

레이아웃:
- 동일 AppLayout + PageHeader
- [처리 완료] 탭 활성

컴포넌트 구조:
1. [탭 네비] [대기 중 {pendingCount}] 비활성 | [처리 완료] 활성

2. [필터 바] flex gap-3 mb-4
   - <Select> 처리 유형 (전체/승인/반려)
   - date range picker: 기간 필터 (기본: 이번달)
   - text input: 요청자 검색

3. [DataTable] columns:
   - 요청자 (강사명)
   - 수업명
   - 요청 유형 (<StatusBadge>: 일정추가/수정/삭제/취소)
   - 요청 내용 (기존→변경 요약, text-sm text-gray-500)
   - 처리 결과 (<StatusBadge>:
       · 승인: bg-green-100 text-green-700 CheckCircle 아이콘
       · 반려: bg-red-100 text-red-700 XCircle 아이콘)
   - 반려 사유 (반려 시만, text-sm text-gray-500, truncate, hover: tooltip 전체 표시)
   - 처리자 (staffName)
   - 처리일시 (formatDate)

4. [페이지네이션] 20건씩

데이터:
- supabase.from('schedule_requests').select('*, staff(name), lessons(name), processed_staff:processed_by(name)').eq('branch_id', branchId).in('status',['approved','rejected']).gte('processed_at', periodStart).lte('processed_at', periodEnd).order('processed_at', {ascending:false})
- 필터: typeFilter ('전체'/'approved'/'rejected'), 기간, 요청자 검색

인터랙션:
- [대기 중] 탭 → `02-대기탭`
- 처리 유형 필터 → 재fetch 또는 클라이언트 필터
- 기간 필터 → 재fetch
- 페이지네이션 → .range(page*20, (page+1)*20-1)

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- formatDate from '@/lib/format'
- lucide-react: CheckCircle, XCircle, History
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-대기탭`에서 [처리 완료] 탭 클릭
- 승인/반려 처리 완료 후 이력 확인

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 처리 완료 목록 | `schedule_requests` | `status IN ('approved','rejected')`, 기간 범위 |
| 처리자 | `staff` (processed_by) | JOIN |

### 인터랙션 (User Actions)
1. [대기 중] 탭 → `02-대기탭`
2. 처리 유형 Select → 필터링
3. 기간 필터 → 재fetch
4. 페이지네이션 → 다음/이전 페이지

### 비즈니스 룰
- 처리 완료 목록은 READ ONLY (수정 불가)
- 기본 기간: 이번달 (너무 많은 데이터 방지)
- 반려 사유: truncate + hover tooltip으로 전체 표시

### 에지 케이스
- 이번달 처리 내역 없음: "처리 내역이 없습니다." (기간 변경 안내)
- 반려 사유 없음 (구 데이터): "-" 표시

### 접근성 (A11y)
- 테이블 `caption="처리 완료 일정 요청 목록"`
- 반려 사유 truncate + `title` 속성으로 전체 표시 (tooltip 대안)

### 연결 화면
- 이전: `02-대기탭`
- 다음: `02-대기탭` ([대기 중] 탭 클릭)
