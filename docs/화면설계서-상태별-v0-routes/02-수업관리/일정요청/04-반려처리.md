## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-대기탭`에서 개별 [반려] 버튼 클릭

### 필수 데이터
- targetRequest: ScheduleRequest (이미 메모리에 있음)

### 인터랙션 (User Actions)
1. 반려 사유 입력 (최소 10자)
2. [반려 처리] → UPDATE → toast.info → 행 제거
3. [취소] / ESC → 모달 닫기 (반려 안 함)

### 비즈니스 룰
- 반려 사유 필수 (최소 10자) — 트레이너에게 통보 목적
- 반려 후 lesson_schedules 변경 없음 (원상태 유지)
- processed_by: 현재 로그인 스태프 ID 기록

### 에지 케이스
- 반려 사유 10자 미만: [반려 처리] disabled + "최소 10자 이상 입력해주세요." 인라인

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby`
- 반려 사유 `aria-required="true"` + `aria-describedby="min-length-hint"`
- 포커스: 모달 열리면 textarea에 자동 포커스

### 연결 화면
- 이전: `02-대기탭`
- 다음: 완료/취소 → `02-대기탭`
