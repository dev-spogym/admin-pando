## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 기본 탭 (pending 요청 우선 표시)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 대기 요청 | `schedule_requests` | `status = 'pending'`, `branch_id = ?` |
| 오늘 처리 | `schedule_requests` | `status IN ('approved','rejected')`, 오늘 날짜 |

### 인터랙션 (User Actions)
1. [승인] → `03-승인처리` (즉시 또는 확인 모달)
2. [반려] → `04-반려처리` (반려 사유 입력 모달)
3. 체크박스 선택 + [일괄 승인] → 일괄 UPDATE + 재fetch
4. [처리 완료] 탭 → `05-완료탭`

### 비즈니스 룰
- 오래된 요청 우선 표시 (created_at ASC)
- 일괄 승인: Promise.all INSERT/UPDATE 병렬 처리
- 승인 시 lesson_schedules 자동 업데이트 (requestData 적용)

### 에지 케이스
- pending 0건: 빈상태 UI "대기 중인 요청이 없습니다." (아이콘 + 텍스트)
- 일괄 승인 중 일부 실패: 성공 N건 / 실패 M건 toast

### 접근성 (A11y)
- 체크박스 `aria-label="{요청자명} 요청 선택"`
- 일괄 처리 바 `role="toolbar"`
