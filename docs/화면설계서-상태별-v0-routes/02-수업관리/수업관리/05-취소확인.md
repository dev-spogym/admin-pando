## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C002 수업 관리 — 상태: 비활성화 확인 다이얼로그

파일: src/app/(classes)/lessons/page.tsx

비활성화 확인 모달:
- max-w-sm (384px)

레이아웃:
1. [아이콘] EyeOff (w-12 h-12 text-amber-500 mx-auto mb-4)
2. [제목] "수업 비활성화" (text-lg font-bold text-center)
3. [메시지]
   - "'{lessonName}' 수업을 비활성화하시겠습니까?"
   - "비활성화된 수업은 새 일정을 등록할 수 없습니다." (text-sm text-gray-500 mt-1)
   - scheduleCount > 0 시:
     · "예정된 일정 {scheduleCount}회가 있습니다. 기존 일정은 유지됩니다." (text-sm text-amber-600 mt-2 font-medium)
4. [버튼] flex gap-2 mt-6
   - [취소] variant=ghost w-full
   - [비활성화] variant=warning w-full → UPDATE is_active=false

비활성화 로직:
- supabase.from('lessons').update({ is_active: false }).eq('id', targetLesson.id)
- 성공: toast.success('수업이 비활성화되었습니다.') + 닫기 + 재fetch
- 실패: toast.error('비활성화에 실패했습니다.')

활성화 (역방향):
- 동일 모달, is_active=false인 수업에서 [활성화] 클릭 시
- "'{lessonName}' 수업을 다시 활성화하시겠습니까?"
- supabase.from('lessons').update({ is_active: true })

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: EyeOff, Eye, AlertTriangle, Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본` 테이블에서 [비활성화] 또는 [활성화] 버튼 클릭

### 필수 데이터
- targetLesson: Lesson (이미 메모리에 있음)
- scheduleCount: 예정 일정 수 (비활성화 경고용)

### 인터랙션 (User Actions)
1. [비활성화] → UPDATE is_active=false → toast.success → 재fetch
2. [취소] / ESC → 모달 닫기

### 비즈니스 룰
- 비활성화 후 기존 lesson_schedules는 유지 (취소 아님)
- 비활성 수업: 수업 등록 모달에서 Select 목록에서 제외

### 에지 케이스
- 예정 일정 있는 수업 비활성화: 경고 문구 표시 후 진행 허용

### 접근성 (A11y)
- `role="alertdialog"` + `aria-modal="true"`
- 포커스: [취소] 버튼에 초기 포커스
