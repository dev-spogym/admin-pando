```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C010 운동 프로그램 — 상태: 기본 목록

파일: src/app/(classes)/exercise-programs/page.tsx

레이아웃:
- <AppLayout> 데스크톱 레이아웃
- <PageHeader title="운동 프로그램" subtitle="회원별 운동 프로그램을 관리합니다."
    actions={[<Button variant="primary" onClick={openAddModal}>+ 프로그램 추가</Button>]} />

컴포넌트 구조:
1. [SearchFilter] flex gap-3 mb-4
   - text input: placeholder="프로그램명 검색"
   - <Select> 담당 강사 필터 (전체 + staff 목록)
   - <Select> 난이도 필터 (전체/초급/중급/고급)

2. [DataTable] columns:
   - 프로그램명 (font-medium)
   - 담당 강사
   - 난이도 (<StatusBadge>: 초급=green, 중급=blue, 고급=red)
   - 운동 수 (N개)
   - 배정 회원 수 (N명)
   - 기간 (N주)
   - 생성일 (formatDate)
   - 액션: [배정] 버튼 | 편집 아이콘 | 삭제 아이콘

3. 행 클릭 → `04-편집모달` (상세/편집)
4. [배정] 버튼 → `05-배정모달`

데이터:
- interface ExerciseProgram { id, branchId, name, instructorId, instructorName, difficulty, exerciseCount, assignedCount, durationWeeks, createdAt }
- supabase.from('exercise_programs').select('*, staff(name), member_exercise_programs(count)').eq('branch_id', branchId)

인터랙션:
- [+ 프로그램 추가] → `03-추가모달`
- 행 클릭 → `04-편집모달`
- [배정] 클릭 → `05-배정모달`
- 검색/필터 → 클라이언트 필터링

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- formatDate from '@/lib/format'
- lucide-react: Plus, Pencil, Trash2, UserPlus
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 exercise_programs 1건 이상

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 프로그램 목록 | `exercise_programs` | `branch_id = ?` |
| 배정 수 | `member_exercise_programs` | `program_id IN (...)`, count |
| 강사 목록 | `staff` | `branch_id = ?`, `is_active = true` |

### 인터랙션 (User Actions)
1. [+ 프로그램 추가] → `03-추가모달`
2. 행 클릭 / 편집 아이콘 → `04-편집모달`
3. [배정] 버튼 → `05-배정모달`
4. 삭제 아이콘 → window.confirm → DELETE
5. 검색/강사/난이도 필터 → 즉시 클라이언트 필터

### 비즈니스 룰
- 트레이너: 본인 생성 프로그램만 편집/삭제 가능
- 배정된 회원이 있는 프로그램: 삭제 시 경고

### 에지 케이스
- 배정 회원 > 0인 프로그램 삭제: "배정된 회원이 있습니다. 계속 삭제하시겠습니까?" confirm

### 접근성 (A11y)
- 테이블 `caption="운동 프로그램 목록"`
- [배정] 버튼 `aria-label="{프로그램명} 회원 배정"`
