## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C010 운동 프로그램 — 상태: 프로그램 추가 모달

파일: src/app/(classes)/exercise-programs/page.tsx

모달 크기: max-w-xl (576px), 최대 높이 80vh overflow-y-auto

모달 구조:
1. [헤더] "운동 프로그램 추가" + X 버튼

2. [기본 정보 섹션] p-6 space-y-4
   | 필드 | 타입 | 필수 | 기본값 |
   |------|------|------|--------|
   | 프로그램명 | text | Y | "" |
   | 담당 강사 | Select | Y | 현재 로그인 강사 |
   | 난이도 | Select | Y | "초급" |
   | 기간(주) | number | N | 4 |
   | 설명 | textarea | N | "" |

3. [운동 목록 섹션] p-6 border-t
   - "운동 추가" 소제목 + [+ 운동 추가] 버튼
   - 운동 행 리스트 (동적 추가/제거):
     · 운동명 input (text, required)
     · 세트 수 input (number, 기본 3)
     · 횟수/시간 input (text, "12회" 또는 "30초")
     · 휴식 시간 input (text, "60초")
     · 삭제 X 버튼
   - 행 추가: exercises.push({ name:'', sets:3, reps:'12회', rest:'60초' })
   - 드래그로 순서 변경 (선택 구현)

4. [하단 버튼] flex justify-end gap-2
   - [취소] → 모달 닫기
   - [저장] → INSERT

저장 로직:
- supabase.from('exercise_programs').insert({ branch_id, name, instructor_id, difficulty, duration_weeks, description, exercises: JSON.stringify(exercises), created_by: staffId })
- 성공: toast.success('프로그램이 추가되었습니다.') + 모달 닫기 + 재fetch
- 실패: toast.error('프로그램 추가에 실패했습니다.')

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: X, Plus, GripVertical, Trash2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본`에서 [+ 프로그램 추가] 버튼 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 강사 목록 | `staff` | `branch_id = ?`, `is_active = true` |

### 인터랙션 (User Actions)
1. [+ 운동 추가] → 운동 행 동적 추가
2. 운동 행 X → 행 제거
3. [저장] → 유효성 검증 → INSERT → 목록 재fetch
4. [취소] / ESC → 모달 닫기

### 비즈니스 룰
- 운동 목록 최소 1개 이상 필요 (권장, 강제 아님)
- exercises 데이터는 JSONB 컬럼에 저장
- 트레이너 역할: 담당 강사 본인으로 고정

### 에지 케이스
- 운동 행 0개로 저장 시: 경고 없이 빈 배열 저장 허용

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby`
- 동적 행 추가 시 새 input에 자동 포커스

- 다음: 저장/취소 → `02-기본`
