## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C010 운동 프로그램 — 상태: 편집 모달

파일: src/app/(classes)/exercise-programs/page.tsx

모달: 추가모달과 동일 구조, 차이점:
1. 헤더: "운동 프로그램 수정"
2. 모든 필드에 selectedProgram 데이터 사전 입력
3. exercises: JSON.parse(selectedProgram.exercises) 로 기존 운동 목록 표시
4. 하단 버튼: [취소] + [수정 완료]

추가 섹션 (편집 전용):
[배정 현황] p-6 border-t
- "현재 배정 회원: N명"
- 배정된 회원 칩 목록 (member.name, x 버튼으로 배정 해제)
- [+ 추가 배정] 링크 → `05-배정모달`

수정 로직:
- supabase.from('exercise_programs').update({ name, instructor_id, difficulty, duration_weeks, description, exercises: JSON.stringify(exercises) }).eq('id', selectedProgram.id)
- 성공: toast.success('프로그램이 수정되었습니다.') + 닫기 + 재fetch
- 실패: toast.error('프로그램 수정에 실패했습니다.')

배정 해제:
- supabase.from('member_exercise_programs').delete().eq('program_id', id).eq('member_id', memberId)
- 성공: toast.success('배정이 해제되었습니다.')

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: X, Plus, Trash2, UserMinus
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본` 테이블 행 클릭 또는 편집(Pencil) 아이콘 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 프로그램 상세 | `exercise_programs` | `id = selectedProgram.id` |
| 배정 회원 | `member_exercise_programs` JOIN `members` | `program_id = ?` |
| 강사 목록 | `staff` | `branch_id = ?` |

### 인터랙션 (User Actions)
1. 필드 수정 후 [수정 완료] → UPDATE → toast.success
2. 배정된 회원 X → 배정 해제 → toast.success
3. [+ 추가 배정] → `05-배정모달`
4. [취소] / ESC → 모달 닫기

### 비즈니스 룰
- 트레이너: 본인 생성 프로그램만 편집 가능
- 배정 해제 시 회원의 진행 중인 운동 기록은 유지

### 에지 케이스
- 배정 회원 없음: "아직 배정된 회원이 없습니다." + [+ 배정하기] 링크

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby`
- 배정 해제 버튼 `aria-label="{회원명} 배정 해제"`
