```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C004 수업 템플릿 — 상태: 삭제 확인 다이얼로그

파일: src/app/(classes)/class-templates/page.tsx

- max-w-sm (384px), 중앙 정렬
- bg-white rounded-xl shadow-xl p-6

레이아웃:
1. [아이콘] AlertTriangle (w-12 h-12 text-red-500 mx-auto mb-4)
2. [제목] "템플릿 삭제" (text-lg font-bold text-center)
3. [메시지]
   - "'{templateName}' 템플릿을 삭제하시겠습니까?"
   - "삭제된 템플릿은 복구할 수 없습니다." (text-sm text-gray-500 mt-1)
4. [버튼] flex gap-2 mt-6
   - [취소] variant=ghost w-full → 모달 닫기

- supabase.from('class_templates').delete().eq('id', targetTemplate.id)

isDeleting 상태:
- [삭제] 버튼 클릭 시 isDeleting=true → 버튼 disabled + Loader2 스피너

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: AlertTriangle, Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본-목록` 테이블에서 삭제(Trash2) 아이콘 클릭

### 필수 데이터
- targetTemplate: ClassTemplate (삭제 대상, 이미 메모리에 있음)

### 인터랙션 (User Actions)
1. [삭제] → DELETE → toast.success → 모달 닫기 → 목록 재fetch
2. [취소] / ESC / 배경 클릭 → 모달 닫기 (삭제 안 함)

### 비즈니스 룰
- 해당 템플릿을 사용 중인 수업(lesson_schedules)이 있을 경우 삭제 불가 → "수업에 사용 중인 템플릿은 삭제할 수 없습니다."
- 삭제 전 FK 체크: lesson_schedules.template_id 참조 여부 확인

### 에지 케이스
- FK 제약으로 삭제 실패: "수업에 사용 중인 템플릿은 삭제할 수 없습니다. 비활성화를 사용하세요." toast.error

### 접근성 (A11y)
- `role="alertdialog"` + `aria-modal="true"` + `aria-labelledby`
- 포커스: 모달 열리면 [취소] 버튼에 포커스
- ESC → 취소

### 연결 화면
- 이전: `02-기본-목록`
- 다음: 삭제 성공/취소 → `02-기본-목록`
