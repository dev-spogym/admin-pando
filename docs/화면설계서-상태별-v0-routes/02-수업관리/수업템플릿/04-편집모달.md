```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C004 수업 템플릿 — 상태: 편집 모달

파일: src/app/(classes)/class-templates/page.tsx

모달 크기: max-w-lg (512px)

추가모달과 동일 구조, 차이점:
1. [헤더] "템플릿 수정" (font-bold text-lg) + X 버튼
2. 모든 필드에 selectedTemplate 데이터 사전 입력
3. [하단 버튼]
   - [취소] → 모달 닫기
   - [수정 완료] variant=primary → UPDATE

편집 로직:
- selectedTemplate: ClassTemplate (행 클릭 시 주입)
- useEffect(() => { reset(selectedTemplate) }, [selectedTemplate])
- supabase.from('class_templates').update({ name, type, capacity, duration, instructor_id, color, memo, is_active }).eq('id', selectedTemplate.id)
- 성공: toast.success('템플릿이 수정되었습니다.') + 모달 닫기 + 목록 재fetch
- 실패: toast.error('템플릿 수정에 실패했습니다.')

활성 여부 토글:
- is_active 토글 변경 시 즉시 반영 (모달 내 상태)
- [수정 완료] 클릭 시 일괄 저장

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: X, Pencil
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본-목록`에서 행 클릭 또는 편집(Pencil) 아이콘 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 템플릿 상세 | `class_templates` | `id = selectedTemplate.id` |
| 강사 목록 | `staff` | `branch_id = ?`, `is_active = true` |

### 인터랙션 (User Actions)
1. 필드 수정 후 [수정 완료] → UPDATE → toast.success → 목록 재fetch
2. [취소] / X / ESC → 변경사항 버리고 모달 닫기
3. is_active 토글 → 즉시 UI 반영 (저장 전)

### 비즈니스 룰
- 수정 전 originalData 보관 → 취소 시 롤백
- 변경사항 없이 [수정 완료] 클릭: UPDATE 스킵 + 모달 닫기

### 에지 케이스
- 모달 열린 상태에서 다른 사용자가 같은 템플릿 삭제 → UPDATE 실패 → toast.error

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby="edit-template-title"`
- 포커스 트랩, ESC 닫기

### 연결 화면
- 이전: `02-기본-목록`
- 다음: 저장 성공/취소 → `02-기본-목록`
