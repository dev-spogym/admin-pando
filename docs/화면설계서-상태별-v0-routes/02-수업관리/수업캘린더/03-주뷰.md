## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C001 수업 캘린더 — 상태: 주뷰 (timeGridWeek)

파일: src/app/(classes)/class-schedule/page.tsx

레이아웃:
- 동일 AppLayout + PageHeader
- FullCalendar initialView="timeGridWeek" 로 전환된 상태

캘린더 설정:
- slotMinTime: "06:00:00", slotMaxTime: "22:00:00"
- slotDuration: "00:30:00" (30분 슬롯)
- nowIndicator: true (현재 시간 빨간 줄)
- editable: hasPermission(user, 'class:edit') (드래그앤드롭 이동)
- eventResizableFromStart: false (리사이즈: 종료 시간만)

이벤트 블록 렌더링 (eventContent):
- 좌측 3px 컬러 보더 (카테고리/강사 색상)
- 수업명 (font-semibold text-[12px])
- 분류 태그 배지 (text-[10px])
- 인원/정원 (currentCount/capacity)
- 강사명 (text-[11px] text-content-secondary)
- 장소 (text-[11px])
- 미승인: opacity-70 + "미승인" amber 배지

드래그앤드롭:
- eventDrop 콜백: supabase.from('lesson_schedules').update({ start_at, end_at }).eq('id', eventId)
- 성공: toast.success('일정이 이동되었습니다.')
- 실패: toast.error('일정 이동에 실패했습니다.') + revert()

리사이즈:
- eventResize 콜백: supabase.from('lesson_schedules').update({ end_at }).eq('id', eventId)
- 성공: toast.success('시간이 변경되었습니다.')

빈 시간 셀 클릭:
- dateClick 콜백: 수업 등록 모달 오픈 (날짜+시간 사전 입력)

데이터:
- 주 범위 계산: startOfWeek/endOfWeek({ weekStartsOn: 1 })
- supabase.from('lesson_schedules').select('*, lessons(*), staff(*)').gte('start_at', weekStart).lte('start_at', weekEnd)

인터랙션:
- 이벤트 드래그 → DB 업데이트 + 토스트
- 이벤트 리사이즈 → DB 업데이트 + 토스트
- 시간 셀 클릭 → 수업 등록 모달
- 이벤트 클릭 → 06-예약확인모달

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- hasPermission from '@/lib/permissions'
- toast from 'sonner'
- lucide-react: Bell
- @fullcalendar/react, @fullcalendar/timegrid, @fullcalendar/interaction
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 캘린더 툴바 "주" 버튼 클릭
- 월뷰에서 이벤트 클릭 후 주뷰로 이동 요청

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 주간 이벤트 | `lesson_schedules` JOIN `lessons`, `staff` | `branch_id = ?`, 주 범위 |
| 드래그 업데이트 | `lesson_schedules` | `id = eventId` → `start_at`, `end_at` |

### 인터랙션 (User Actions)
1. 이벤트 드래그앤드롭 → start_at/end_at 업데이트 + toast.success
2. 이벤트 하단 리사이즈 → end_at 업데이트 + toast.success
3. 빈 시간 셀 클릭 → 수업 등록 모달 (날짜+시간 사전 입력)
4. 이벤트 클릭 → `06-예약확인모달`
5. "prev/next" 버튼 → 이전/다음 주 데이터 fetch

### 비즈니스 룰
- 과거 날짜 드래그: "과거 일정은 수정할 수 없습니다." 경고 + revert
- 시작 2시간 이내 수업: "수업 시작 2시간 전까지만 수정할 수 있습니다." + revert
- 강사 시간 충돌: window.confirm 경고 후 진행 여부 선택
- 트레이너 역할: editable = 본인 수업만 true

### 에지 케이스
- 드래그 중 네트워크 에러: 낙관적 UI 롤백 (revert() 호출)
- 동시간대 이벤트 다수: FullCalendar 자동 컬럼 분할

### 접근성 (A11y)
- 드래그 가능 이벤트: `aria-grabbed`, `aria-dropeffect`
- 시간 슬롯: `aria-label="HH:MM 시간 슬롯"`

### 연결 화면
- 이전: `02-월뷰`
- 다음: `04-일뷰` / `06-예약확인모달` / 수업 등록 모달
