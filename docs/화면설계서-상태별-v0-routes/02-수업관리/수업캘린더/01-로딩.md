## 📝 디스크립션

### 사용 시점 (Trigger)
- 사이드바 > 수업관리 > 수업 캘린더 클릭 시 최초 진입
- 페이지 새로고침 시
- 월/주/일 뷰 전환 시 데이터 재fetch 중

### 필수 데이터
| 테이블 | 쿼리 | 목적 |
|--------|------|------|
| `lesson_schedules` | `select('*, lessons(*), staff(*)')` | 캘린더 이벤트 |
| `staff` | `select('id, name, role').eq('branch_id', branchId)` | 강사 필터 목록 |
| `lessons` | `select('id, name, type').eq('branch_id', branchId)` | 수업명 필터 목록 |
| `schedule_requests` | `select('id').eq('status', 'pending')` | 미승인 건수 배지 |

### 인터랙션 (User Actions)
1. 데이터 로드 완료 → `02-월뷰` 상태로 자동 전환
2. 데이터 로드 실패 → toast.error('수업 일정을 불러오지 못했습니다.')

### 비즈니스 룰
- branchId는 useAuthStore().user.branchId 기준
- 트레이너 역할: staffId 기준 본인 수업만 fetch (WHERE instructor_id = staffId)

### 에지 케이스
- 네트워크 지연 3초 이상: 스켈레톤 유지 + 타임아웃 없음
- 로딩 중 뷰 전환 버튼 비활성화

### 접근성 (A11y)
- `role="status"` + `aria-label="수업 일정 로딩 중"` on 스켈레톤 컨테이너
- `aria-busy="true"` on 메인 콘텐츠 영역
