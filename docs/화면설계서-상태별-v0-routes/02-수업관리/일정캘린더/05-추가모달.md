## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader [수업 등록] 버튼 클릭
- 캘린더 빈 시간 셀 클릭 (날짜+시간 사전 입력)
- EventDetailModal [수정] 버튼 클릭 (기존 데이터 사전 입력)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 수업 템플릿 | `class_templates` | `branch_id = ?`, `is_active = true` |
| 강사 목록 | `staff` | `branch_id = ?`, `is_active = true` |
| 룸 목록 | 상수 `ROOMS` | GX룸/스피닝룸/필라테스룸/PT룸 등 |

### 인터랙션 (User Actions)
1. 템플릿 선택 → 관련 필드 자동완성
2. 강사 + 시간 입력 → 실시간 충돌 체크
3. 충돌 감지 → `06-충돌경고` (인라인 또는 confirm)
4. [수업 등록] → INSERT → toast.success → 캘린더 리프레시
5. [취소] / ESC → 모달 닫기

### 비즈니스 룰
- 강사 충돌: window.confirm 경고 후 진행 여부 선택
- 종료 시간 < 시작 시간: "종료 시간은 시작 시간 이후여야 합니다." 인라인 에러
- 반복 설정: 요일 선택 시 해당 요일마다 반복 일정 일괄 INSERT

### 에지 케이스
- 반복 설정 + 기간: 최대 12주 (84일) 제한
- 좌석 행×열 = 0이면 좌석 배치도 미사용

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby` + 포커스 트랩
- 필수 필드 `aria-required="true"`
