## 📝 디스크립션

### 사용 시점 (Trigger)
- `05-추가모달`: 강사 또는 시간 변경 시 자동 충돌 체크 (debounce 500ms)
- `04-드래그이동`: 드롭 완료 후 충돌 체크

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 충돌 이벤트 | `lesson_schedules` | `instructor_id = ?`, 시간 범위 겹침, `id != 현재` |

### 인터랙션 (User Actions)
1. 추가모달 인라인: 충돌 경고 확인 후 [수업 등록] 클릭 → 그대로 INSERT
2. 드래그 confirm: [확인] → DB 업데이트 / [취소] → revert
3. 충돌 해소: 강사 또는 시간 변경 → 충돌 배너 자동 사라짐

### 비즈니스 룰
- 충돌 허용: 시스템이 강제 막지 않음 (경고만)
- 룸 충돌도 동일 패턴 (경고만, 강제 차단 없음)
- 미승인 일정은 충돌 체크 대상 포함

### 에지 케이스
- 동일 강사 복수 충돌: 모든 충돌 이벤트 목록 표시
- 충돌 체크 쿼리 실패: 경고 없이 진행 (안전 실패)

### 접근성 (A11y)
- 인라인 충돌 배너 `role="alert"` + `aria-live="assertive"`
- confirm 다이얼로그: 브라우저 기본 (네이티브 접근성)

### 연결 화면
- 이전: `05-추가모달` / `04-드래그이동`
- 다음: 진행 → DB 업데이트 완료 / 취소 → 이전 상태
