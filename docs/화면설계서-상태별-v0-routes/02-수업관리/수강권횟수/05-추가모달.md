## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader [+ 횟수 추가] 버튼 클릭
- `03-잔여위험` 행의 [+ 횟수 추가] 인라인 버튼 클릭 (회원 사전 입력)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 검색 | `members` | `branch_id = ?`, `name ILIKE %q%` |
| 수업 목록 | `lessons` | `branch_id = ?`, `is_active = true` |
| 강사 목록 | `staff` | `branch_id = ?`, `is_active = true` |
| 현재 잔여 | `lesson_counts` | `member_id = ?`, `lesson_id = ?` |

### 인터랙션 (User Actions)
1. 회원 검색 → 드롭다운 선택
2. 수업/강사 Select → 현재 잔여 조회 + 추가 후 미리보기
3. 추가 횟수 입력 → "추가 후: N회" 실시간 업데이트
4. [저장] → UPSERT + 이력 INSERT → toast.success
5. [취소] / ESC → 모달 닫기

### 비즈니스 룰
- 추가 횟수 최소 1회, 최대 500회
- 만료일 미입력 시 NULL (무기한)
- 트레이너: 회원 추가만 가능, 만료일 수정 불가

### 에지 케이스
- 수업+회원 조합의 lesson_counts 없으면 신규 INSERT
- 만료일 지난 수강권에 추가 시: "만료된 수강권입니다. 만료일을 갱신하시겠습니까?" confirm

### 접근성 (A11y)
- `role="dialog"` + `aria-labelledby`
- 추가 횟수 `aria-describedby="current-remaining"` (현재 잔여 설명 연결)

### 연결 화면
- 이전: `02-기본-목록` / `03-잔여위험`
- 다음: 저장/취소 → 이전 상태
