## 📝 디스크립션

### 사용 시점 (Trigger)
- 존재하지 않는 URL 직접 접근
- 삭제된 리소스 링크 클릭
- 잘못된 ID 파라미터로 페이지 이동 후 404 처리

### 필수 데이터
- 없음 (정적 에러 페이지)

### 인터랙션 (User Actions)
1. "대시보드로 이동" → moveToPage(966)
2. "이전 페이지로" → window.history.back()

### 비즈니스 룰
- AppLayout 없음 — 사이드바/헤더 없이 독립 렌더링
- Next.js App Router의 `not-found.tsx` 와 구분: 이 파일은 명시적 라우트 `/not-found`
- 비인증 사용자도 접근 가능 (미들웨어 예외 처리)

### 에지 케이스
- window.history.back() 이전 페이지 없음 → 아무 동작 없음 (브라우저 기본)
- moveToPage(966) 실패 → router.push('/') fallback 권장

### 접근성 (A11y)
- h1: "페이지를 찾을 수 없습니다" — 페이지 제목
- 404 숫자: `aria-hidden="true"`
- document.title: "404 — 페이지 없음 | FitGenie CRM" 권장

### 연결 화면
- 이전: 잘못된 URL
- 다음: 대시보드(966) / 이전 페이지
