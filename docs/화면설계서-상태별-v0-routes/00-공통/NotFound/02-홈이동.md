```
Next.js 15 App Router + TypeScript + Tailwind 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-404 Not Found — 상태: 홈 이동 (대시보드 이동 버튼 클릭)

파일: src/app/(error)/not-found/page.tsx

이 상태는 "대시보드로 이동" 버튼 클릭 직후의 전환 순간을 설명한다.
별도 UI 상태 없음 — moveToPage(966) 즉시 실행.

추가 개선 구현 (선택):
1. 자동 리다이렉트 카운트다운 (5초 후 자동 이동)
   const [countdown, setCountdown] = useState(5);

   useEffect(() => {
     const timer = setInterval(() => {
       setCountdown(prev => {
         if (prev <= 1) {
           clearInterval(timer);
           moveToPage(966);
           return 0;
         }
         return prev - 1;
       });
     }, 1000);
     return () => clearInterval(timer);
   }, []);

   // 카운트다운 표시
   <p className="mt-4 text-[12px] text-content-tertiary">
     {countdown}초 후 대시보드로 자동 이동합니다.
   </p>

2. 진행 바 (선택)
   <div className="mt-2 h-1 w-48 mx-auto bg-surface-secondary rounded-full overflow-hidden">
     <div
       className="h-full bg-primary rounded-full transition-all duration-1000"
       style={{ width: `${((5 - countdown) / 5) * 100}%` }}
     />
   </div>

버튼 클릭 시:
<button
  onClick={() => moveToPage(966)}
  className="px-6 py-2.5 rounded-lg bg-primary text-white text-sm font-medium hover:opacity-90 transition-opacity"
>
  대시보드로 이동
</button>

사용 유틸:
- moveToPage(966) from '@/internal'
- useState, useEffect (카운트다운 구현 시)
```

### 필수 데이터

### 인터랙션 (User Actions)
1. "대시보드로 이동" 클릭 → moveToPage(966) 즉시 실행
2. 카운트다운 중 "취소" → clearInterval + countdown 리셋 (선택)

### 비즈니스 룰
- moveToPage(966): 대시보드 페이지 ID
- 자동 리다이렉트는 선택적 구현 (UX 개선 목적)
- 비인증 상태 → 미들웨어가 /login으로 차단 (moveToPage 전 처리)

### 에지 케이스
