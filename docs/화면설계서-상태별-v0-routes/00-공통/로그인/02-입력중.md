## 📝 디스크립션

### 사용 시점 (Trigger)
- 아이디 또는 비밀번호 필드에 값 입력 시작
- error state가 ''인 상태 (에러 배너 없음)

### 필수 데이터
| 항목 | 값 |
|------|-----|
| id | 입력 중인 문자열 |
| password | 입력 중인 문자열 |
| isFormValid | id && password 모두 비어있지 않음 |

### 인터랙션 (User Actions)
1. 아이디 입력 → setId + setError('') — 이전 에러 초기화
2. 비밀번호 입력 → setPassword + setError('')
3. Eye 아이콘 → 비밀번호 보이기/숨기기 토글
4. Enter 키 또는 로그인 버튼 → `03-제출중`
5. 지점 Select 변경 → selectedBranchId 업데이트

### 비즈니스 룰
- onChange마다 setError('')로 이전 에러 메시지 초기화
- isFormValid가 false면 버튼 disabled 유지
- 입력값이 공백만 있어도 trim() 후 빈 문자열로 판단 → disabled

### 에지 케이스
- 아이디만 입력하고 비밀번호 미입력 → 버튼 disabled
- 비밀번호만 입력하고 아이디 미입력 → 버튼 disabled
- 이전에 에러가 표시된 상태에서 재입력 시 에러 배너 즉시 숨김

### 접근성 (A11y)
- 포커스된 input: `focus:ring-1 focus:ring-primary/20`
- Eye 버튼: `type="button"` (폼 submit 방지)
- 비밀번호 보이기 전환 시 스크린리더: aria-label 동적 변경 권장

### 연결 화면

- 다음: `03-제출중` (버튼 클릭/Enter) / `04-에러-잘못된자격` (실패 후 재입력)
