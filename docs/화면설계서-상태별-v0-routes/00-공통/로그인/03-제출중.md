## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-LOGIN 로그인 — 상태: 제출중 (loginMutation.isPending = true)

파일: src/app/(auth)/login/page.tsx

상태 변화:
- isLoading = loginMutation.isPending = true
- 모든 input disabled={isLoading}
- Select disabled={branchesLoading || isLoading}
- 로그인 버튼: loading={isLoading} → 스피너 + "로그인 중..." 텍스트
- 버튼 disabled={!isFormValid || isLoading || isLocked}

UI 포인트:
1. Button variant="primary" loading={true} → Loader2 spin 아이콘 자동 표시
2. input들 disabled → opacity-50 cursor-not-allowed 시각 피드백
3. Eye 토글 버튼도 pointer-events 차단 (input disabled 로 충분)
4. 에러 배너 없음 (setError('')는 submit 시작 시 호출)
5. 폼 배경 변화 없음 — 로딩은 버튼 내부 스피너로만 표시

API 호출:
- loginMutation.mutate({ username: id, password })
- onSuccess: res.success === false → incrementFailCount() → 04-에러
- onSuccess: res.success === true → clearFailCount() → authStore.login() → moveToPage(966)
- onError: setError('서버 연결에 실패했습니다. 잠시 후 다시 시도해주세요.')

사용 유틸:
- useLogin() from '@/api/hooks/useAuth'
- useAuthStore().login
- moveToPage(966) — 대시보드
- localStorage.setItem('branchId', branchIdToUse)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 폼 submit 이벤트 발생 (버튼 클릭 또는 Enter)
- `loginMutation.isPending === true`인 동안 유지

### 필수 데이터
| 항목 | 값 |
|------|-----|
| username | id state 값 |
| password | password state 값 |
| selectedBranchId | 선택된 지점 ID |

### 인터랙션 (User Actions)
- 이 상태에서는 모든 입력 비활성 — 사용자 인터랙션 없음
- 뒤로가기 버튼만 가능 (브라우저)

### 비즈니스 룰
- 중복 제출 방지: isPending=true 동안 버튼 disabled
- 슈퍼관리자 로그인 시 selectedBranchId 또는 '' 사용 (branchId null 허용)
- 일반 사용자: selectedBranchId || String(user.branchId ?? 1)
- 로그인 성공 시 localStorage.setItem('branchId', branchIdToUse)
- authStore.login() 후 moveToPage(966) — 대시보드로 이동

### 에지 케이스
- 네트워크 타임아웃 → onError 핸들러 → "서버 연결에 실패했습니다." 에러 표시
- 응답은 오지만 res.success=false → incrementFailCount() → 남은 횟수 안내

### 접근성 (A11y)
- 로딩 버튼: `aria-busy="true"` 추가 권장
- 스크린리더: "로그인 중..." 텍스트가 버튼 내에 있어 읽힘

### 연결 화면
- 이전: `02-입력중`
- 다음: `05-성공-대시보드이동` (성공) / `04-에러-잘못된자격` (실패)
