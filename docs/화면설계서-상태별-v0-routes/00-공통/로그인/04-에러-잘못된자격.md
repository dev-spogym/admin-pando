const REMEMBER_KEY = 'spoGym_rememberedId';
const FAIL_COUNT_KEY = 'login_fail_count';
const LOCKED_UNTIL_KEY = 'login_fail_locked_until';
const MAX_FAIL_COUNT = 5;
const LOCK_DURATION_MS = 30 * 60 * 1000; // 30분

function incrementFailCount(): number {
  const prev = Number(localStorage.getItem(FAIL_COUNT_KEY) ?? '0');
  const next = prev + 1;
  localStorage.setItem(FAIL_COUNT_KEY, String(next));
  if (next >= MAX_FAIL_COUNT) {
    localStorage.setItem(LOCKED_UNTIL_KEY, String(Date.now() + LOCK_DURATION_MS));
  }
  return next;
}

## 📝 디스크립션

### 사용 시점 (Trigger)
- loginMutation onSuccess에서 `res.success === false` 수신
- loginMutation onError 발생 (네트워크 오류)
- 페이지 진입 시 localStorage에 잠금 상태 존재 (LOCKED_UNTIL_KEY)

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| failCount | localStorage[FAIL_COUNT_KEY] | 누적 실패 횟수 |
| lockedUntil | localStorage[LOCKED_UNTIL_KEY] | 잠금 해제 시각 |
| error | state | 에러 메시지 문자열 |
| isLocked | state | 계정 잠금 여부 |

### 인터랙션 (User Actions)
1. 아이디/비밀번호 재입력 → setError('') — 에러 배너 즉시 제거
2. 로그인 재시도 → `03-제출중`
3. 잠금 상태에서는 버튼 비활성 — 입력만 가능

### 비즈니스 룰
- 실패 횟수 누적: localStorage에 저장 (탭/브라우저 재시작 후에도 유지)
- 5회 실패 시 30분 잠금 (LOCK_DURATION_MS)
- 잠금 시간 경과 시 자동 해제 (getLoginLockState() 내에서 처리)
- 로그인 성공 시 clearFailCount() 호출

### 에지 케이스
- 잠금 상태에서 30분 후 페이지 새로고침 → 자동 잠금 해제
- 서버 에러(500) → "서버 연결에 실패했습니다." (failCount 증가 안 함)
- 지점 API 실패와 로그인 에러 동시 → 각각 독립적으로 처리

### 접근성 (A11y)
- 에러 배너: `role="alert"` — 스크린리더 즉시 읽음
- input: `aria-invalid="true"` 설정
- 잠금 버튼: disabled 상태이므로 Tab 순서에서 제외됨

### 연결 화면
- 이전: `03-제출중`
- 다음: `02-입력중` (재입력) / 잠금 상태 30분 후 `01-기본`
