## 📝 디스크립션

### 사용 시점 (Trigger)
- 최초 접속 시 (비인증 상태)
- 세션 만료 후 자동 리다이렉트
- 로그아웃 후 이동
- localStorage의 잠금 상태 없음 + 아이디/비밀번호 미입력

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| 지점 목록 | `getBranches()` API | Select 옵션 |
| 저장된 아이디 | `localStorage[REMEMBER_KEY]` | 로그인 유지 복원 |
| 잠금 상태 | `localStorage[LOCKED_UNTIL_KEY]` | 계정 잠금 확인 |

### 인터랙션 (User Actions)
1. 지점 Select 클릭 → 드롭다운 열림
2. 아이디 필드 클릭 → 02-입력중
3. 로그인 유지 체크박스 토글
4. 비밀번호 찾기 → toast.info
5. 고객센터 문의 → toast.info

### 비즈니스 룰
- 지점 목록 로드 완료 전 Select는 disabled + "지점 로딩 중..." 표시
- 지점이 없으면 "지점 없음" 옵션
- 저장된 아이디가 있으면 id 필드에 자동 복원 + rememberMe=true
- isFormValid = id.trim() !== '' && password.trim() !== ''
- 제출 버튼 disabled 조건: !isFormValid || isLoading || isLocked

### 에지 케이스
- 지점 API 호출 실패 → "지점 없음" 옵션 표시
- localStorage 접근 불가 (SSR) → useEffect 내에서만 접근
- 잠금 상태이면 즉시 에러 메시지 표시 + 버튼 "계정 잠금" 텍스트로 변경

### 접근성 (A11y)
- 아이디/비밀번호 input: `aria-required="true"`
- 에러 시 `aria-invalid="true"`
- 로그인 유지 체크박스: sr-only input + `aria-label="로그인 유지"`
- Tab 순서: 지점 Select → 아이디 → 비밀번호 → 로그인 유지 → 로그인 버튼
