## 📝 디스크립션

### 사용 시점 (Trigger)
- 보호된 라우트에 권한 없는 역할로 접근 시도
- 미들웨어 또는 페이지 내 권한 체크에서 `/forbidden` 리다이렉트
- 예: 스태프가 `/super-dashboard` 접근 시도

### 필수 데이터
| 항목 | 소스 |
|------|------|
| authUser.role | useAuthStore |
| authUser.isSuperAdmin | useAuthStore |
| canAccessDashboard | hasPermission() 결과 |

### 인터랙션 (User Actions)
1. "대시보드로 이동" → moveToPage(966) (canAccessDashboard=true)
2. "급여 명세서로 이동" → /payroll/statements (canAccessDashboard=false)
3. "이전 페이지로" → window.history.back()

### 비즈니스 룰
- canAccessDashboard: hasPermission(role, '/', isSuperAdmin) — 대시보드 접근 가능 여부
- 스태프 계정은 대시보드 접근 가능 → "대시보드로 이동" 버튼 표시
- 급여 명세서 전용 계정 → "급여 명세서로 이동"
- AppLayout 없음: 사이드바 없이 독립 렌더링

### 에지 케이스
- 비인증 사용자가 /forbidden에 직접 접근 → 미들웨어가 /login으로 리다이렉트
- history.back() 시 이전 페이지가 없으면 아무 동작 없음

### 접근성 (A11y)
- h1: "접근 권한이 없습니다" — 페이지 제목
- 403 숫자: `aria-hidden="true"` (장식 요소)
- ShieldX 아이콘: `aria-hidden="true"`
- 버튼: 명확한 텍스트 레이블
