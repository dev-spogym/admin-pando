## 📝 디스크립션

### 사용 시점 (Trigger)
- `/forbidden?required=superAdmin&from=/super-dashboard` 형식으로 리다이렉트
- 사용자가 어떤 권한이 필요한지 명확히 알아야 할 때
- 권한 체크 미들웨어가 필요 권한 정보를 URL에 포함할 때

### 필수 데이터
| 항목 | 소스 |
|------|------|
| required | searchParams.get('required') |
| from | searchParams.get('from') |
| authUser.role | useAuthStore |

### 인터랙션 (User Actions)
1. "관리자에게 문의" → toast.info 권한 문의 안내
2. "대시보드로 이동" → moveToPage(966)
3. "이전 페이지로" → window.history.back()

### 비즈니스 룰
- required 파라미터 없으면 `01-기본`과 동일 UI
- 현재 사용자 역할을 한글로 표시 (normalizeRole 활용)
- 권한 부여는 슈퍼관리자 또는 최고관리자만 가능
- 이 화면에서 직접 권한 요청 기능 없음 (문의 안내만)

### 에지 케이스
- required 파라미터 없음 → 01-기본과 동일 렌더링
- 알 수 없는 required 값 → "해당 권한" 으로 fallback
- from 경로가 없음 → 경로 안내 생략

### 접근성 (A11y)
- 안내 박스: `role="note"` 또는 `role="complementary"`
- 코드 태그: 경로 표시에 `<code>` 태그 사용 (시맨틱)

### 연결 화면
- 이전: `01-기본`
- 다음: 대시보드(966) / 이전 페이지 / 관리자 문의 (외부)
