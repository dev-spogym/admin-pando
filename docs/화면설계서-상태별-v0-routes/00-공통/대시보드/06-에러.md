try { ... } catch (err) {
  console.error("[Dashboard] 데이터 로드 실패:", err);
  toast.error("대시보드 데이터를 불러오지 못했습니다.");

}

현재 코드 동작:
- catch 시 toast.error만 발생, isLoading=false로 전환
- stats는 모두 0 상태로 유지 → 빈 데이터 화면 표시
- 에러 전용 UI 없음 (toast만)

개선 구현 (추가 권장):
const [hasError, setHasError] = useState(false);

catch (err) {
  console.error("[Dashboard] 데이터 로드 실패:", err);
  toast.error("대시보드 데이터를 불러오지 못했습니다.");
  setHasError(true);
} finally {
  setIsLoading(false);
}

// 에러 배너 (hasError=true 시 StatCardGrid 위에 노출)
{hasError && (
  <div className="mb-lg flex items-center gap-md rounded-xl border border-red-200 bg-red-50 px-lg py-md">
    <AlertCircle size={18} className="text-red-500 shrink-0" />
    <div className="flex-1">
      <p className="text-[14px] font-semibold text-red-800">데이터를 불러오지 못했습니다</p>
      <p className="text-[12px] text-red-600 mt-[2px]">네트워크 연결을 확인하거나 다시 시도해주세요.</p>
    </div>
    <Button
      variant="ghost"
      size="sm"
      onClick={() => { setHasError(false); handleRefresh(); }}
    >
      다시 시도
    </Button>
  </div>
)}

새로고침 버튼 동작:
const handleRefresh = useCallback(() => {
  setIsRefreshing(true);
  setHasError(false);
  fetchDashboard().finally(() => setIsRefreshing(false));
}, [fetchDashboard]);

사용 유틸:
- toast.error from 'sonner'

## 📝 디스크립션

### 사용 시점 (Trigger)
- `fetchDashboard()` catch 블록 진입
- Supabase 연결 오류 (네트워크 다운, 서버 500)
- RLS 정책 위반 (권한 없는 branchId 접근)
- JWT 만료 (자동 갱신 실패)

### 필수 데이터
- err 객체: console.error 로깅용
- hasError: boolean state

### 인터랙션 (User Actions)
1. "다시 시도" 버튼 → setHasError(false) + handleRefresh()
2. 새로고침 버튼 (헤더) → handleRefresh()
3. 사이드바 메뉴 → 다른 페이지 이동

### 비즈니스 룰
- JWT 만료 에러 → Supabase 클라이언트가 자동 갱신 시도
- 자동 갱신 실패 → authStore logout → /login 리다이렉트 (미들웨어 처리)
- RLS 에러: 잘못된 branchId → getBranchId() fallback 1 사용
- toast.error는 1회만 (catch 블록 내에서 단 1회 호출)

### 에지 케이스
- 부분 실패: Promise.all 중 일부만 실패해도 전체 catch → 모든 데이터 0
- 재시도 성공 → hasError=false + `02-기본-전체위젯` 전환
- 연속 실패 3회 이상 → 추가 안내 (고객센터 연락처) 노출 권장

### 접근성 (A11y)
- 에러 배너: `role="alert"` — 스크린리더 즉시 공지
- "다시 시도" 버튼: `aria-label="대시보드 데이터 다시 불러오기"`

### 연결 화면
- 이전: `01-로딩`
- 다음: 재시도 → `01-로딩` → `02-기본` / 인증 실패 → `/login`
