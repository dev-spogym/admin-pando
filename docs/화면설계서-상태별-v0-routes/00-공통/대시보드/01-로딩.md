## 📝 디스크립션

### 사용 시점 (Trigger)
- 로그인 직후 대시보드 첫 진입
- 페이지 새로고침
- `handleRefresh()` 수동 새로고침 호출

### 필수 데이터
- branchId: `getBranchId()` — localStorage 기반
- 모든 통계 쿼리는 branchId 필터 적용

### 인터랙션 (User Actions)
1. 새로고침 버튼 클릭 → isRefreshing=true → RefreshCw animate-spin
2. 사이드바 네비게이션 클릭 → 다른 페이지 이동 (데이터 로딩 중단)

### 비즈니스 룰
- 로딩 시작 시 `runDailySync()` 동시 실행 (세션당 1회)
  - sessionStorage['lastDailySync'] !== today 인 경우만
  - 만료 회원/락커/쿠폰 자동 처리
- `getFavoriteVisitsToday()` → 즐겨찾기 회원 방문 알림 toast
- 주간 출석 데이터: 월~일 7일치 순차 쿼리 (Promise.all)
- 월별 매출: 최근 6개월 Promise.all

### 에지 케이스
- branchId=0 또는 NaN → 기본값 1 사용
- Supabase 응답 지연 > 10초 → 스켈레톤 유지 (타임아웃 없음)
- runDailySync 실패 → console.error만, 대시보드 로딩 차단 안 함

### 접근성 (A11y)
- 스켈레톤 영역: `aria-busy="true"` 권장
- 스크린리더: "데이터를 불러오는 중입니다" 숨김 텍스트 제공 권장
