## 📝 디스크립션

### 사용 시점 (Trigger)
- `fetchDashboard()` 완료 후 isLoading=false
- 하나 이상의 위젯 데이터 존재

### 필수 데이터
| 위젯 | 테이블 | 쿼리 조건 |
|------|--------|----------|
| 전체/활성/만료 회원 | members | branchId, status, deletedAt IS NULL |
| 오늘 출석 | attendance | branchId, checkInAt 오늘 범위 |
| 이번달 매출 | sales | branchId, status=COMPLETED, 이번달 |
| 생일자 | members | birthDate LIKE '%-MM-DD' |
| 미수금 | sales | status=UNPAID, limit 5 |
| 홀딩 | members | status=HOLDING, limit 5 |
| 만료임박 | members | ACTIVE, expiry 30일 이내, limit 5 |
| 감사로그 | audit_logs | branchId, 최신 10건 |

### 인터랙션 (User Actions)
1. StatCard 클릭 → 해당 도메인 페이지 이동 (moveToPage)
2. 공지 배너 X → setShowBanner(false)
3. 공지 배너 "기능 보기" → moveToPage(977) 전자계약
4. 생일자/미수금/홀딩/만료임박 행 클릭 → moveToPage(985, { id }) 회원 상세
5. 미수금 "결제" → moveToPage(971) POS
6. 최근 활동 "더보기" → moveToPage(990) 감사로그
7. 새로고침 버튼 → handleRefresh()
8. 회원 신규 등록 → moveToPage(986)

### 비즈니스 룰
- 매출 포맷: 1억+ → "X.X억", 1만+ → "X만", 나머지 → 천단위 콤마
- 만료임박 D-Day 배지: ≤3일 → error, ≤7일 → warning, 나머지 → default
- 미수금 연체 배지: >30일 → "장기미납" error 배지
- 홀딩 잔여일: StatusBadge variant="info" dot
- 성별 도넛: SVG strokeDasharray 기반 (여성=primary, 남성=accent)
- 주간 출석 바: 오늘 → bg-primary, 나머지 → bg-accent/60

### 에지 케이스
- 생일자 0명 → "오늘 생일자가 없습니다" 빈 상태
- 미수금 0건 → "미수금 내역이 없습니다"
- 성별 데이터 없음 → "생년월일 데이터가 없습니다"
- 감사로그 없음 → "최근 활동 내역이 없습니다"

### 접근성 (A11y)
- StatCard: onClick → `role="button"` 또는 button 태그 사용
- 차트 영역: `aria-label` 또는 숨김 텍스트로 데이터 설명 제공
- 배너 닫기 버튼: `aria-label="공지 닫기"`
