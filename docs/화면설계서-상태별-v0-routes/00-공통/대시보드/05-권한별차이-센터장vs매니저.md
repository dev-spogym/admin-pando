화면: SCR-090 대시보드 — 상태: 권한별 차이 (역할에 따른 위젯/버튼 노출 제어)

파일: src/app/(dashboard)/page.tsx

권한 체크:
- const authUser = useAuthStore((s) => s.user)
- const isSuperAdmin = authUser?.isSuperAdmin ?? false
- const userRole = authUser?.role ?? ''
- const isOwnerOrAbove = ['owner','primary','superAdmin'].includes(userRole) || isSuperAdmin
- const isManagerOrAbove = isOwnerOrAbove || userRole === 'manager'

역할별 노출 차이:

[센터장/최고관리자/슈퍼관리자]
- 이번달 매출 StatCard: 전체 금액 표시
- 미수금 총액 StatCard: 전체 금액 표시
- 이번달 환불 StatCard: 건수 표시
- PageHeader actions: "회원 신규 등록" 버튼 표시
- 미수금 위젯 결제 버튼: 표시 (moveToPage(971))
- 매출 추이 차트: 전체 금액 표시

[매니저]
- 이번달 매출: 조회 가능 (읽기 전용)
- 매출 위젯 액션 버튼 제한 없음 (현 코드 기준)
- "회원 신규 등록" 버튼: 표시

[FC/코치]
- 매출 StatCard: 숨김 또는 '-' 표시
  if (!isManagerOrAbove) {
    // 매출 관련 카드 value를 '권한 필요'로 대체하거나 숨김
  }
- 미수금 위젯: 결제 버튼 숨김
- "회원 신규 등록" 버튼: 표시 (FC는 결제 가능)

[스태프]
- 매출 StatCard: 숨김
- 미수금 위젯: 결제 버튼 숨김
- "회원 신규 등록" 버튼: 숨김
- 조회 전용

구현 패턴:
// 매출 카드 조건부
const visibleStats = isManagerOrAbove
  ? memberStats
  : memberStats.filter(s => !['이번달 매출','미수금 총액','이번달 환불'].includes(s.label))

// 결제 버튼 조건부
{isManagerOrAbove && (
  <Button variant="ghost" size="sm" onClick={() => moveToPage(971)}>결제</Button>
)}

// 신규 등록 버튼 조건부 (스태프 제외)
{userRole !== 'staff' && (
  <Button variant="primary" size="sm" onClick={() => moveToPage(986)}>회원 신규 등록</Button>
)}

사용 유틸:
- useAuthStore from '@/stores/authStore'

## 📝 디스크립션

### 사용 시점 (Trigger)
- 대시보드 데이터 로드 완료 후 항상 적용
- authUser.role 에 따라 조건부 렌더링

### 필수 데이터
| 항목 | 소스 |
|------|------|
| authUser.role | useAuthStore |
| authUser.isSuperAdmin | useAuthStore |
| authUser.branchName | useAuthStore |

### 인터랙션 (User Actions)
역할별 접근 가능한 액션:
| 액션 | 센터장 | 매니저 | FC | 스태프 |
|------|:---:|:---:|:---:|:---:|
| 회원 신규 등록 | O | O | O | X |
| 매출 카드 조회 | O | O | X | X |
| 미수금 결제 | O | O | X | X |
| 새로고침 | O | O | O | O |
| 회원 상세 이동 | O | O | O | O(조회) |

### 비즈니스 룰
- 역할 접근 매트릭스는 `공통.md` 섹션 2.2 기준
- 스태프는 대부분 조회 전용 (회원 등록/결제 불가)
- 슈퍼관리자는 모든 기능 접근 가능
- branchName: authUser.branchName || '센터' (PageHeader description에 사용)

### 에지 케이스
- role이 알 수 없는 값 → 가장 낮은 권한(스태프)으로 처리
- isSuperAdmin=true이면 role 무관하게 전체 접근
- 로그인 직후 authStore 미초기화 → 빈 role → 최소 권한 적용

### 접근성 (A11y)
- 숨겨진 버튼은 DOM에서 완전 제거 (visibility:hidden X, display:none O)
- 스크린리더: 권한 없는 영역 접근 시 명확한 메시지 제공
