## 📝 디스크립션

### 사용 시점 (Trigger)
- `fetchDashboard()` 완료 후 `expiringMembers.some(m => m.ddayNum <= 3)`
- D-3 이내 만료 임박 회원이 1명 이상 존재

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| expiringMembers | members 테이블 | ACTIVE, expiry ≤30일, limit 5 |
| ddayNum | 계산값 | Math.floor((expDate - today) / 86400000) |

### 인터랙션 (User Actions)
1. "재등록 상담" 버튼 → moveToPage(985, { id }) 회원 상세
2. 만료임박 위젯 행 클릭 → 회원 상세 이동
3. StatCard 만료임박 클릭 → moveToPage(967) 회원 목록

### 비즈니스 룰
- D-Day 계산: `Math.max(0, Math.floor((expDate - today) / (1000*60*60*24)))`
- D-0 (당일 만료): ddayNum=0 → error 배지
- 만료일 지난 회원은 이미 EXPIRED 상태로 자동 처리됨 (runDailySync)
- 만료임박 위젯은 최대 5명 표시 (limit 5)

### 에지 케이스
- 만료임박 회원이 5명 초과 → 5명까지만 노출, 전체는 회원목록에서 확인
- ddayNum=0인 회원이 당일 EXPIRED로 자동 전환되면 목록에서 사라짐

### 접근성 (A11y)
- D-Day 배지: `aria-label="D-{n}일 남음"` 권장
- error 배지: 색상만으로 구분하지 않도록 텍스트(D-1, D-2 등) 병행
