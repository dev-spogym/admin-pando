## 📝 디스크립션

### 사용 시점 (Trigger)
- `05-Step5-서명`에서 서명 완료 후 [계약 완료 확인] 클릭
- `handleContractSave()` → `doInsert()` 실행 중
- `isSaving=true` 상태

### 필수 데이터
| 단계 | 작업 |
|------|------|
| 서명 업로드 | Storage `contracts/{branchId}/{fileName}.png` |
| 계약 저장 | `contracts` 테이블 INSERT |
| 매출 저장 | `sales` 테이블 INSERT |
| 이용권 갱신 | `updateMembershipPeriod()` |
| 마일리지 적립 | `accruePoints()` |

### 인터랙션 (User Actions)
1. 저장 중 — 버튼 비활성 (중복 클릭 방지)
2. 중복 결제 감지 → ConfirmDialog → [진행] or [취소]
3. 저장 성공 → `toast.success` + `moveToPage(985)` → `07-완료`
4. 서명 업로드 실패 → `toast.error` + `isSaving=false` → Step5 유지

### 비즈니스 룰
- 서명 이미지 양쪽 모두 업로드 성공해야 contracts INSERT 진행
- contracts + sales는 트랜잭션이 아니므로 순차 처리 (실패 시 toast.error)
- 마일리지는 마일리지 단독 결제 제외 조건 확인 후 적립

### 에지 케이스
- 네트워크 오류 → `toast.error("계약 저장에 실패했습니다.")` + `isSaving=false`
- 서명 이미지 null → 업로드 skip + `isSaving=false` + Step5 유지

### 접근성 (A11y)
- [계약 완료 확인] 버튼: `aria-busy="true"` (isSaving=true 시)

### 연결 화면
- 이전: `05-Step5-서명`
- 다음: `07-완료` (성공) / `05-Step5-서명` (실패)
