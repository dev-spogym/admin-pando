## 📝 디스크립션

### 사용 시점 (Trigger)
- `doInsert()` 내 모든 처리 완료 후
- `toast.success` 호출 직후 `moveToPage(985)` 실행

### 피드백 구성
| 피드백 | 내용 |
|--------|------|
| toast.success | "계약이 완료되었습니다." |
| toast.info | "{N}P 마일리지가 적립되었습니다." (적립 시) |
| 화면 전환 | 회원 상세 (memberPage) 자동 이동 |

### 인터랙션 (User Actions)
- 사용자 액션 불필요 — 자동 리다이렉트

### 비즈니스 룰
- 계약 완료 = contracts + sales + 회원상태 + 마일리지 모두 처리 완료
- 리다이렉트 대상: `moveToPage(985, { id: selectedMember.id })` — 회원 상세
- 마일리지 결제 단독 시: accruePoints 호출 생략

### 에지 케이스
- moveToPage 실패 시 현재 페이지 유지 (toast만 표시됨)
- 마일리지 적립량 0 → toast.info 미표시

### 접근성 (A11y)
- toast는 `role="status"` live region으로 스크린리더 읽음
