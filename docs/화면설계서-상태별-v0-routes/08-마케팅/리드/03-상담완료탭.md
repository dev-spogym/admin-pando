## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본-신규탭`에서 [칸반] 버튼 클릭 시 (`setViewMode("kanban")`)
- URL 파라미터 없이 state로만 관리 (새로고침 시 기본 목록 뷰)

### 필수 데이터
| 블록 | 소스 |
|------|------|
| 칸반 카드 | `filtered` (useEffect + leads 배열) |
| 열 건수 | `colLeads.length` |

### 인터랙션 (User Actions)
1. 카드 클릭 → `openEdit(lead)` → 수정 모달
2. [목록] 버튼 → `setViewMode("list")`
3. 상태 필터 변경 → `filtered` 재계산 → 해당 열 카드 수 변동

### 비즈니스 룰
- 칸반은 5개 상태만 표시 (미전환/보류 제외)
- 미전환/보류 리드는 목록 뷰에서만 확인 가능
- 카드 내 drag-and-drop은 현재 미구현 (클릭으로 수정 모달에서 상태 변경)

### 에지 케이스
- 필터 적용 시 특정 열이 0건 → "없음" 표시 (열 자체는 유지)
- 로딩 중 칸반 전환 → "로딩 중..." 표시

### 접근성 (A11y)
- 각 열: `role="region"` + `aria-label="{col.label} 파이프라인"`
- 카드 버튼: `aria-label="{lead.name} 리드 편집"`

### 연결 화면
- 이전: `02-기본-신규탭`
- 다음: `06-담당FC배정` (카드 클릭)
