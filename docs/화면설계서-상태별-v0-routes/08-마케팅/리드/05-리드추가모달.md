## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader의 [+ 리드 등록] 버튼 클릭 시
- `openAdd()` → `setEditTarget(null)`, `setForm({...EMPTY_FORM})`, `setShowModal(true)`

### 필수 데이터
| 항목 | 내용 |
|------|------|
| LEAD_SOURCES | 간판/인터넷/전단지/추천/SNS/카카오톡/전화문의/방문/기타 |
| LEAD_STATUSES | 신규/연락완료/상담예정/방문완료/등록완료/미전환/보류 |
| branchId | getBranchId() |

### 인터랙션 (User Actions)
1. 이름 미입력 → [저장] 클릭 → `toast.error("이름을 입력하세요.")`
2. 정상 입력 → [저장] → saving=true → API 호출 → 성공 → `toast.success("리드가 등록되었습니다.")` → 모달 닫힘 → fetchData()
3. [취소] / X 버튼 → `setShowModal(false)` (입력값 초기화 없음, 재오픈 시 빈 폼)
4. 상태 pill 버튼 클릭 → `setForm(p => ({ ...p, status: s }))`
5. 유입경로 Select 변경 → `setForm(p => ({ ...p, source: v }))`

### 비즈니스 룰
- 이름 필수, 유입경로 필수 (기본값 "전화문의")
- 상태 기본값 "신규"
- 문의일 기본값 = 오늘 날짜
- 저장 중(saving=true) 버튼 중복 클릭 방지

### 에지 케이스
- 저장 실패 시 `toast.error` + 모달 유지 (form 값 보존)
- 연락처 미입력 허용 → null 저장

### 접근성 (A11y)
- 모달 오픈 시 첫 Input(이름)에 focus
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby="modal-title"`
- ESC 키 → 모달 닫힘
