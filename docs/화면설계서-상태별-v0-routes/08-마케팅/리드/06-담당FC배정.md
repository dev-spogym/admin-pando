## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable 행의 Edit(수정) 아이콘 클릭
- 칸반 뷰 카드 클릭
- `openEdit(lead)` 함수 호출

### 필수 데이터
| 항목 | 내용 |
|------|------|
| editTarget | 현재 수정 중인 Lead 객체 |
| form | 기존 lead 데이터로 초기화된 폼 상태 |

### 인터랙션 (User Actions)
1. 상태 pill 버튼 클릭 → 리드 상태 변경 (예: 신규 → 상담예정)
2. 담당 FC 입력 → 배정 담당자 변경
3. 후속 예정일 변경 → 팔로우업 일정 관리
4. [저장] → `updateLead(editTarget.id, payload)` → `toast.success("리드가 수정되었습니다.")`
5. [취소] / X → `setShowModal(false)` → 이전 상태 복귀

### 비즈니스 룰
- 이름 필수 (빈 값 저장 불가)
- 담당 FC는 텍스트 직접 입력 (staff 연동 없음, 현재 구현)
- 리드 상태를 "등록완료"로 변경 시 전환율 통계 자동 반영
- 수정 권한: 센터장/매니저/FC/코치 모두 가능
- 삭제 권한: 센터장/매니저만 (FC/코치는 수정만)

### 에지 케이스
- 저장 실패 → `toast.error` + 모달 유지
- 연락처 변경 → 빈 값 허용 (null 저장)

### 접근성 (A11y)
- 모달 `role="dialog"` + `aria-label="리드 수정"`
- 첫 Input(이름)에 자동 focus

- 다음: `02-기본-신규탭` (저장 성공/취소)
