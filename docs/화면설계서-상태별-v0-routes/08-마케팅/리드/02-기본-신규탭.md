## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 `leads.length > 0` 인 경우
- 필터/검색 적용 후 결과가 1건 이상인 경우
- 리드 등록/수정/삭제 후 재조회 완료 시

### 필수 데이터
| 블록 | 테이블 | 컬럼 |
|------|--------|------|
| 통계 카드 | `leads` | `status`, `branchId` |
| 상태별 분포 | `leads` | `status`, `branchId` |
| DataTable | `leads` | 전체 컬럼 |

### 인터랙션 (User Actions)
1. [+ 리드 등록] → `openAdd()` → `05-리드추가모달` 오픈
2. 수정 아이콘 → `openEdit(lead)` → 폼에 기존 데이터 세팅 → 모달 오픈
3. 삭제 아이콘 → `setDeleteId(row.id)` → ConfirmDialog(danger)
4. 상태 필터 Select 변경 → `filterStatus` 업데이트 → DataTable 재렌더
5. 유입경로 필터 Select 변경 → `filterSource` 업데이트
6. 검색창 입력 → 이름/연락처/담당FC 포함 필터
7. [칸반] 버튼 → `setViewMode("kanban")`

### 비즈니스 룰
- 상태 variant 매핑: 신규=info / 연락완료=default / 상담예정=warning / 방문완료=info / 등록완료=success / 미전환=error / 보류=default
- 전환율 = Math.round((등록완료 / total) × 100)
- 미전환율 = Math.round((미전환 / total) × 100)
- 대기 = 신규 + 연락완료 건수
- FC/코치는 수정 가능, 삭제 불가 (Trash2 hidden or disabled)

### 에지 케이스
- 검색 결과 0건 → `07-빈상태` 표시
- stats.total === 0 → 상태별 분포 섹션 숨김

### 접근성 (A11y)
- DataTable 행: `role="row"` + `aria-label="{이름} 리드"`
- 필터 Select: `aria-label="상태 필터"`

### 연결 화면

- 다음: `05-리드추가모달` / `06-담당FC배정` / `07-빈상태`
