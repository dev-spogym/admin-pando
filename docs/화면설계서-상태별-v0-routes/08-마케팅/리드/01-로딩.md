## 📝 디스크립션

### 사용 시점 (Trigger)
- `/leads` 라우트 최초 진입 시
- `fetchData()` 호출 직후 `setLoading(true)` 실행 시점
- 리드 등록/수정/삭제 후 목록 재조회 시

### 필수 데이터
| 소스 | 상태 |
|------|------|
| `getLeads(branchId)` | 호출 중 (미완료) |
| `getLeadStats(branchId)` | 호출 중 (미완료) |

### 인터랙션 (User Actions)
1. 사용자는 로딩 중 액션 없음 — 스켈레톤 UI만 표시
2. 로딩 완료 → `loading=false` → `02-기본-신규탭` 전환

### 비즈니스 룰
- `Promise.all` 병렬 호출로 지연 최소화
- StatCardGrid의 StatCard는 skeleton shimmer 표시
- DataTable 내부 loading prop 처리로 행 스켈레톤 자동 표시

### 에지 케이스
- 네트워크 오류 시 loading=false + 빈 데이터 → 빈상태로 전환
- 재조회(fetchData) 호출 시 기존 데이터 유지하며 로딩 오버레이 없음

### 접근성 (A11y)
- 스켈레톤 컨테이너에 `aria-busy="true"` 속성
- 로딩 완료 시 `aria-busy="false"` 전환

### 연결 화면
- 이전: 사이드바 마케팅 > 리드 관리
- 다음: `02-기본-신규탭` (성공) / 빈상태 (데이터 없음)
