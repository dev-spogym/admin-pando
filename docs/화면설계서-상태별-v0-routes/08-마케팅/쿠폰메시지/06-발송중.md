## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-073 쿠폰 관리 — 상태: 삭제 확인 다이얼로그

파일: src/app/(marketing)/message/coupon/page.tsx

삭제 트리거:
- handleDeleteClick(coupon) → setSelectedCoupon(coupon) + setIsConfirmDeleteOpen(true)
- 삭제 버튼 활성 조건: row.totalIssued === 0 (발급 이력 없는 쿠폰만)

ConfirmDialog 컴포넌트:
- open={isConfirmDeleteOpen}
- title="쿠폰 삭제"
- description=`"${selectedCoupon?.name}" 쿠폰을 삭제하시겠습니까?\n발급 이력이 없는 쿠폰만 삭제가 가능합니다.`
- confirmLabel="삭제"
- variant="danger"
- confirmationText="삭제"
- onConfirm={confirmDelete}
- onCancel={() => setIsConfirmDeleteOpen(false)}

confirmDelete 함수:
- if (!selectedCoupon) return
- supabase.from('coupons').update({isActive: false}).eq('id', selectedCoupon.id)
- error → toast.error("쿠폰 삭제에 실패했습니다.") + return
- setIsConfirmDeleteOpen(false)
- setSelectedCoupon(null)
- fetchCoupons()
- toast.success("쿠폰이 삭제되었습니다.")

soft delete 방식:
- isActive: false (물리적 삭제 아님)
- computeStatus → 'inactive' 반환
- 목록에서 filterValues.status='inactive'로 필터 가능

삭제 버튼 disabled 조건 (listColumns actions):
- disabled={row.totalIssued > 0}
- title="발급 이력이 있어 삭제 불가" (tooltip)

데이터:
- selectedCoupon: Coupon | null
- isConfirmDeleteOpen: boolean

인터랙션:
- [삭제] 버튼 클릭 → ConfirmDialog 표시
- confirmationText="삭제" 입력 후 [삭제] → confirmDelete()
- [취소] → setIsConfirmDeleteOpen(false)
- 삭제 완료 → fetchCoupons() + toast.success

사용 유틸:
- ConfirmDialog from '@/components/ui/ConfirmDialog'
- supabase from '@/lib/supabase'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 쿠폰 목록 행의 [삭제] 버튼 클릭 (totalIssued===0일 때만 활성)

### 필수 데이터
| 항목 | 소스 |
|------|------|
| selectedCoupon | 삭제 대상 Coupon 객체 |
| confirmationText | "삭제" 입력 확인 |

### 인터랙션 (User Actions)
1. [삭제] 버튼 클릭 → ConfirmDialog 열림
2. confirmationText 입력 → "삭제" 타이핑
3. [삭제 확인] → `confirmDelete()` → soft delete → `fetchCoupons()`
4. [취소] → 다이얼로그 닫힘

### 비즈니스 룰
- Soft delete: `isActive: false` (물리 삭제 아님)
- totalIssued > 0인 쿠폰 → 삭제 버튼 disabled (발급 이력 보호)
- 삭제 후 coupons 목록 재조회 (fetchCoupons)

### 에지 케이스
- supabase update 실패 → toast.error + 다이얼로그 유지
- isActive=false 쿠폰 → computedStatus='inactive' → 목록에 비활성 상태로 표시

### 접근성 (A11y)
- ConfirmDialog: `role="alertdialog"` + `aria-modal="true"`
- confirmationText 입력: `aria-label="삭제 확인 문자 입력"`

- 다음: `02-쿠폰선택` (삭제 완료 또는 취소)
