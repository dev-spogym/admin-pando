## 📝 디스크립션

### 사용 시점 (Trigger)
- 쿠폰 목록 행의 [발급] 버튼 클릭
- `handleOpenIssueModal(coupon)` 실행

### 필수 데이터
| 항목 | 소스 |
|------|------|
| selectedCoupon | 선택된 Coupon 객체 |
| memberResults | supabase members 검색 (debounce 300ms) |

### 인터랙션 (User Actions)
1. 발급 대상 탭 선택 (전체/특정/조건별)
2. 특정 회원 검색 → 드롭다운 선택 → 태그 추가
3. 발급 수량 입력
4. 유효 기간 설정
5. [발급] → handleIssue() → coupons.totalIssued 업데이트 → toast.success

### 비즈니스 룰
- 발급 = `coupons.totalIssued += count` (실제 coupon_issues 테이블 없음)
- individual + selectedMembers.length===0 → 발급 불가
- custom 유효기간 + expiryDate 없음 → 발급 불가

### 에지 케이스
- 검색 결과 없음 → "검색 중..." 표시 후 빈 결과
- issuing=true 중 재클릭 → disabled로 방어

### 접근성 (A11y)
- 모달: `role="dialog"` + `aria-modal="true"`
- 발급 대상 버튼: `role="tablist"` + `role="tab"` + `aria-selected`

### 연결 화면
- 이전: `02-쿠폰선택`
- 다음: `05-발송확인` (발급 완료 → toast.success → 모달 닫힘)
