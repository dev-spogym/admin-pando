## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 (loading=false, coupons 데이터 있음)
- TabNav 쿠폰 목록 탭 선택 시

### 필수 데이터
| 항목 | 소스 |
|------|------|
| coupons | `coupons` 테이블 |
| enrichedCoupons | computeCouponStatus 계산값 |
| filteredCoupons | 검색/필터 적용 |

### 인터랙션 (User Actions)
1. [신규 쿠폰 생성] → CouponFormModal 열기
2. [발급] → IssueCouponModal 열기 (03-수신자선택)
3. [수정] → CouponFormModal(편집 모드) 열기
4. [삭제] → ConfirmDialog (totalIssued=0일 때만 활성)
5. 검색/필터 → filteredCoupons 실시간 업데이트
6. 탭 전환 → 발급 이력 탭

### 비즈니스 룰
- 발급 이력이 있는 쿠폰(totalIssued>0)은 삭제 불가 (버튼 disabled)
- 삭제 = `isActive: false` soft delete
- computedStatus 자동 계산 (DB 저장 안 함)

### 에지 케이스
- coupons 빈 배열 → DataTable emptyMessage="등록된 쿠폰이 없습니다."
- 만료 D-7 이내 쿠폰 → 상태 배지 아래 D-N 경고 텍스트

### 접근성 (A11y)
- DataTable: `role="grid"` + 헤더 `scope="col"`
- 삭제 버튼 disabled: `aria-disabled="true"` + `title="발급 이력이 있어 삭제 불가"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-수신자선택` / `04-메시지편집` / 삭제 확인 다이얼로그
