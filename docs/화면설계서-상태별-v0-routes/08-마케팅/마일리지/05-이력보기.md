```
Next.js 15 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-074 마일리지 관리 — 상태: 마일리지 이력 탭 (activeTab="history")

파일: src/app/(marketing)/mileage/page.tsx

이력 탭 진입 방법:
- TabNav [마일리지 이력] 탭 클릭
- statusColumns [이력] 버튼 클릭 → setActiveTab('history')

mileageLogs 조회 로직 (useEffect [members 의존]):
1차 시도 (mileage_logs 테이블):
- supabase.from('mileage_logs').select('id, memberId, type, amount, balance, reason, createdAt, admin')
  .eq('branchId', branchId).order('createdAt', {ascending:false})
- 성공 시: TYPE_MAP = {earn:'적립', use:'사용', expire:'만료'} 적용
- name: memberMap[row.memberId] ?? '-'

2차 fallback (sales 테이블):
- supabase.from('sales').select('id, memberId, memberName, amount, mileageEarned, mileageUsed, saleDate, staffName')
  .eq('branchId', branchId).order('saleDate', {ascending:false})
- mileageEarned > 0 → 적립 로그
- mileageUsed > 0 → 사용 로그 (amount 음수)
- 날짜 내림차순 정렬

filteredLogs 필터 (useMemo):
- historySearch: row.name 포함
- historyFilters.type !== '전체': row.type 매칭
- historyFilters.dateRange?.start: row.createdAt >= start
- historyFilters.dateRange?.end: row.createdAt <= end+'T23:59:59'

SearchFilter (마일리지 이력):
- searchPlaceholder="회원명, 연락처 검색"
- filters:
  - dateRange: { label:'조회 기간', type:'dateRange' }
  - type: { label:'처리 유형', type:'select', options:[전체/적립/차감/사용] }
- filterValues=historyFilters
- onFilterChange → setHistoryFilters(prev => ({...prev, [key]:val}))

historyColumns (useMemo):
- no: "No" (width:60, center) — index+1
- createdAt: "처리일시" (center)
- name: "회원명"
- type: "처리유형" (center) — StatusBadge:
  - 적립: variant="peach" dot / 사용: variant="mint" dot / 기타: variant="default" dot
- amount: "마일리지" (right):
  - val > 0: text-state-success `+${formatNumber(val)} P`
  - val <= 0: text-state-error `${formatNumber(val)} P`
- balance: "잔액" (right) — `${formatNumber(val)} P`
- reason: "사유"
- admin: "처리자" (center)
- expiryDate: "만료 예정일" (center) — <ExpiryBadge expiryDate={val} />

DataTable:
- columns=historyColumns, data=filteredLogs
- pagination={page:1, pageSize:20, total:filteredLogs.length}
- title="전체 마일리지 이력"
- emptyMessage="이력 데이터 없음"

엑셀 다운로드 (activeTab="history"):
- exportToExcel(filteredLogs, historyExportColumns, {filename:'마일리지이력'})
- exportColumns: createdAt/name/type/amount/balance/reason/admin/expiryDate
- toast.success('마일리지 이력 엑셀 다운로드 완료')

데이터:
- mileageLogs: any[] (mileage_logs 또는 sales fallback)
- filteredLogs: useMemo 계산값
- historySearch: string
- historyFilters: { type:'전체', dateRange:null }

인터랙션:
- 검색/필터 → filteredLogs 실시간 업데이트
- [엑셀 다운로드] → exportToExcel
- 탭 전환 → 현황/정책 탭

사용 유틸:
- supabase from '@/lib/supabase'
- exportToExcel from '@/lib/exportExcel'
- StatusBadge from '@/components/common/StatusBadge'
- SearchFilter, DataTable from '@/components/common'
- formatNumber from '@/lib/format'
- cn from '@/lib/utils'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- TabNav [마일리지 이력] 탭 클릭
- 또는 statusColumns [이력] 버튼 클릭 (`setActiveTab('history')`)

### 필수 데이터
| 항목 | 소스 |
|------|------|
| mileageLogs | `mileage_logs` 테이블 (없으면 `sales` fallback) |
| filteredLogs | useMemo (검색+필터 적용) |

### 인터랙션 (User Actions)
1. 검색어 입력 → historySearch 필터
2. 처리 유형 필터 → 적립/차감/사용 구분
3. 기간 필터 → dateRange 적용
4. [엑셀 다운로드] → exportToExcel(filteredLogs)

### 비즈니스 룰
- mileage_logs 테이블 없으면 sales 테이블로 fallback
- type 매핑: earn→적립, use→사용, expire→만료
- amount: 적립 양수 / 사용 음수

### 에지 케이스
- mileage_logs + sales 모두 없음 → mileageLogs=[] → emptyMessage
- sales fallback 시 balance=0 (누적 잔액 계산 불가)

### 접근성 (A11y)
- amount 음수 표시: `aria-label="차감 {amount}P"`

### 연결 화면
- 이전: `02-기본-회원별`
- 다음: `02-기본-회원별` (현황 탭 클릭)
