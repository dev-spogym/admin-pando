## 📝 디스크립션

### 사용 시점 (Trigger)
- statusColumns actions의 [수동 처리] 버튼 클릭
- `setSelectedMember(row)` + `setIsManualModalOpen(true)`

### 필수 데이터
| 항목 | 소스 |
|------|------|
| selectedMember | statusColumns row (name, contact) |
| type | '적립' | '차감' |
| amount | 숫자 입력 |
| reason | 사유 Select |

### 인터랙션 (User Actions)
1. [적립]/[차감] 선택
2. 금액 입력 (P 단위)
3. 사유 선택
4. 메모 입력 (선택)
5. [처리 확인] → handleManualAdjustment → toast.success

### 비즈니스 룰
- 현재 실제 DB 저장 미구현 (toast.success만 표시)
- 향후 members.mileage 업데이트 + mileage_logs insert 필요
- 차감 시 잔여 마일리지 초과 방어 로직 필요 (미구현)

### 에지 케이스
- amount 비어있으면 [처리 확인] disabled
- isProcessing=true 중 재클릭 방어

### 접근성 (A11y)
- 처리 유형 버튼: `role="radio"` + `aria-checked`
- amount input: `aria-label="마일리지 금액"` + `aria-describedby`
