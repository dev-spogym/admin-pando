## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-074 마일리지 관리 — 상태: 로딩 (Supabase 데이터 불러오는 중)

파일: src/app/(marketing)/mileage/page.tsx

로딩 조건:
- useState isLoading=true 초기값
- useEffect → fetchMembers() 실행
  - supabase.from('members').select('id, name, phone, mileage').eq('branchId', getBranchId())
  - 완료 후 setIsLoading(false)

로딩 중 UI:
- DataTable loading={isLoading} prop 전달
- StatCardGrid는 summary 초기값(0)으로 즉시 렌더
- TabNav, SearchFilter는 즉시 렌더 (인터랙션 가능)

INITIAL_SUMMARY:
- totalIssued: 0
- totalUsed: 0
- currentBalance: 0
- monthlyEarned: 0

StatCardGrid (cols=4):
- "전체 발행 마일리지" / "전체 사용 마일리지" / "잔여 마일리지" / "이번 달 적립"
- 로딩 중: 모두 0 표시

members 데이터 가공:
- id, name, contact(phone 특수문자 치환), earned(mileage), used(0), balance(mileage)
- lastEarnedAt: '-', expiryDate: '-'

summary 집계 (fetchMembers 완료 후):
- totalIssued = mapped.reduce((sum, m) => sum + m.earned, 0)
- totalUsed = 0 (현재 used 데이터 없음)
- currentBalance = mapped.reduce((sum, m) => sum + m.balance, 0)

getBranchId 함수:
- typeof window === 'undefined' → 1 반환
- localStorage.getItem('branchId') ? Number(...) : 1

데이터:
- isLoading: boolean (초기 true)
- members: any[] (초기 빈 배열)
- summary: INITIAL_SUMMARY

인터랙션:
- 로딩 중 사용자 액션 없음 (DataTable 스켈레톤)
- 로딩 완료 → 02-기본-회원별 상태로 자동 전환

사용 유틸:
- supabase from '@/lib/supabase'
- StatCardGrid, StatCard from '@/components/common'
- DataTable from '@/components/common/DataTable'
- Coins, ArrowUpRight, ArrowDownRight, CheckCircle2 from 'lucide-react'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `/mileage` 진입 시 (useEffect → fetchMembers 실행)

### 필수 데이터
| 항목 | 소스 |
|------|------|
| members | `members` 테이블 (id, name, phone, mileage) |
| isLoading | fetchMembers 실행 중 true |

### 인터랙션 (User Actions)
- 없음 (자동 전환)

### 비즈니스 룰
- loading=true 구간: DataTable 스켈레톤 행 표시
- summary는 fetchMembers 완료 후 members 배열 기반 집계

### 에지 케이스
- 네트워크 오류 → isLoading=false, members=[] → 빈 상태 테이블

### 접근성 (A11y)
- DataTable loading: `aria-busy="true"` + `role="status"`
