```
Next.js 15 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-074 마일리지 관리 — 상태: 마일리지 정책 탭 (activeTab="policy")

파일: src/app/(marketing)/mileage/page.tsx

정책 탭 콘텐츠 (max-w-4xl space-y-lg):

변경 감지 배너 (isPolicyDirty=true일 때만):
- flex items-center justify-between p-md rounded-xl
- bg-amber-600/10 border border-amber-600/30 animate-in fade-in duration-300
- AlertCircle icon + "저장되지 않은 변경사항이 있습니다." (text-amber-600)
- [되돌리기] 버튼 → setPolicy(savedPolicy)
- [지금 저장] 버튼 (bg-amber-600 text-white) → setIsPolicySaveDialogOpen(true)

DEFAULT_POLICY:
- earnRate: 5 (기본 적립률 %)
- expiryMonths: 12 (마일리지 유효기간 개월)
- minUsage: 1000 (최소 사용 금액 P)
- maxUsagePerTx: 50000 (1회 최대 사용 P)

isPolicyDirty 계산:
- JSON.stringify(policy) !== JSON.stringify(savedPolicy)

FormSection "기본 적립 정책":
- description="결제 및 서비스 이용에 따른 자동 적립 규칙을 설정합니다."
- 기본 적립률 (%) input:
  - value=policy.earnRate
  - onChange → setPolicy(prev => ({...prev, earnRate: Number(e.target.value)}))
  - 안내: "결제 금액의 일정 비율을 마일리지로 적립합니다."
- 마일리지 유효기간 (개월) input:
  - value=policy.expiryMonths
  - 안내: "적립일로부터 해당 기간이 지나면 자동 소멸됩니다."

FormSection "사용 제한 설정":
- description="마일리지 사용 시 적용되는 최소/최대 기준을 설정합니다."
- 최소 사용 금액 (P) input:
  - value=policy.minUsage
  - 안내: "마일리지를 사용하기 위한 최소 보유 및 사용 포인트입니다."
- 1회 최대 사용 제한 (P) input:
  - value=policy.maxUsagePerTx
  - 안내: "1회 결제 시 사용할 수 있는 최대 마일리지입니다."

FormSection "대상 상품 설정" (columns=1):
- description="적립 및 사용 대상에서 제외하거나 포함할 상품 범위를 지정합니다."
- 사용 가능 상품: border-dashed + "+ 상품 범위 추가 (수강권, PT, 락커 등)"
- 적립 제외 상품: border-dashed + "+ 제외 상품 추가 (일일권, 특가 상품 등)"

저장 버튼 (flex justify-end pt-md):
- isPolicyDirty=true: bg-accent text-white hover:opacity-90 shadow-accent/20
- isPolicyDirty=false: bg-line text-content-secondary cursor-not-allowed opacity-60
- icon=<Save size={18} />
- 텍스트: isPolicyDirty ? "정책 저장하기" : "저장 완료"
- onClick → isPolicyDirty && setIsPolicySaveDialogOpen(true)

ConfirmDialog (정책 저장):
- open={isPolicySaveDialogOpen}
- title="정책 변경 확인"
- description="마일리지 정책을 변경하시겠습니까? 변경된 정책은 이후 발생하는 적립/사용 건부터 즉시 적용됩니다."
- confirmLabel="저장" / cancelLabel="취소"
- onConfirm=handlePolicySave → setSavedPolicy(policy) + toast.success('정책이 성공적으로 저장되었습니다.')

데이터:
- policy: { earnRate, expiryMonths, minUsage, maxUsagePerTx }
- savedPolicy: 저장된 정책 (비교용)
- isPolicyDirty: boolean (derived)
- isPolicySaveDialogOpen: boolean

인터랙션:
- input 변경 → policy state 업데이트 → isPolicyDirty=true → 변경 배너 표시
- [지금 저장] / [정책 저장하기] → ConfirmDialog
- ConfirmDialog [저장] → handlePolicySave → isPolicyDirty=false
- [되돌리기] → setPolicy(savedPolicy) → 변경 배너 사라짐

사용 유틸:
- FormSection from '@/components/common/FormSection'
- ConfirmDialog from '@/components/ui/ConfirmDialog'
- Save, AlertCircle from 'lucide-react'
- toast from 'sonner'
- cn from '@/lib/utils'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- TabNav [마일리지 정책] 탭 클릭
- `setActiveTab('policy')` 실행

### 필수 데이터
| 항목 | 소스 |
|------|------|
| policy | DEFAULT_POLICY 기반 로컬 state |
| savedPolicy | 저장된 정책 (비교 기준) |
| isPolicyDirty | JSON.stringify 비교 |

### 인터랙션 (User Actions)
1. 적립률/유효기간/최소/최대 입력 변경 → isPolicyDirty=true
2. 변경 배너 [되돌리기] → policy 원복
3. [저장] → ConfirmDialog → handlePolicySave → toast.success
4. 상품 범위 추가 버튼 → 미구현 (UI만 표시)

### 비즈니스 룰
- 정책 저장 = 로컬 state 저장 (DB 미연동)
- 향후 settings 테이블 연동 예정
- isPolicyDirty 감지로 불필요한 저장 방지

### 에지 케이스
- 변경 후 탭 전환 → 변경 내용 소실 (저장 안 됨)
- earnRate=0 → 마일리지 미적립

### 접근성 (A11y)
- input: `aria-label="{항목명}"` + `aria-describedby` (안내 텍스트)

### 연결 화면
