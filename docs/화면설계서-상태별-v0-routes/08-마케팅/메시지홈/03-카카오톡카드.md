## 📝 디스크립션

### 사용 시점 (Trigger)
- 발송 채널 기본값 = "kakao"
- 또는 알림톡 라디오 카드 클릭

### 필수 데이터
| 항목 | 값 |
|------|-----|
| effectiveCost | 8원/건 (고정) |
| maxLen | 1000자 |
| smsTypeLabel | null |

### 인터랙션 (User Actions)
1. 메시지 작성 → 글자수 실시간 표시 (N/1,000자)
2. 1000자 초과 → isOverLimit=true → [발송] disabled
3. [메시지 발송] → content 없으면 toast.warning → 있으면 PreviewModal
4. PreviewModal [발송하기] → confirmSend() → toast.success → 폼 초기화

### 비즈니스 룰
- 알림톡은 카카오 비즈메시지 API 연동 필요 (현재 활성 상태 배지 표시)
- 8원/건 고정 (SMS와 달리 글자수에 따른 요금 변동 없음)
- 발송 이력: `type="KAKAO"`로 저장

### 에지 케이스
- 알림톡 비활성화(StatusBadge default) 시 발송 불가 안내 필요 (현재 미구현)
- 예약 발송: `status='SCHEDULED'`, `sentAt=null`, `scheduledAt=예약시간`

### 접근성 (A11y)
- PreviewModal: `role="dialog"` + `aria-modal="true"`

### 연결 화면
- 이전: `01-기본`
- 다음: 발송 완료 → `01-기본` (폼 초기화)
