## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 (loading=false)

### 필수 데이터
| 항목 | 소스 |
|------|------|
| rules | localStorage `settings_{branchId}_auto_alarm` 또는 INITIAL_RULES |
| senderNumber | localStorage (기본: "02-1234-5678") |
| enabledCount | rules.filter(r => r.enabled).length |

### 인터랙션 (User Actions)
1. Toggle 클릭 → `handleToggle(id)` → `persistAlarm` → `04-활성화토글` 상태
2. MoreHorizontal 클릭 → `handleEdit(rule)` → `03-템플릿편집` 모달
3. [모두 사용] → 전체 enabled:true + `persistAlarm`
4. [설정 추가] → 새 규칙 생성 + 편집 모달 열기

### 비즈니스 룰
- localStorage 저장 키: `settings_{branchId}_auto_alarm`
- 규칙 변경 즉시 localStorage 저장 (persistAlarm 호출)
- enabledCount 실시간 계산 (derived state)

### 에지 케이스
- localStorage 없음 → INITIAL_RULES 13종 기본값
- [설정 추가] → id=`custom-{Date.now()}`, enabled:false, template.content="" → 즉시 편집 모달

### 접근성 (A11y)
- Toggle: `role="switch"` + `aria-checked`
- FormSection 각 섹션: `role="region"` + `aria-label`
