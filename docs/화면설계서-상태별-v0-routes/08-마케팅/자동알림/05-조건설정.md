## 📝 디스크립션

### 사용 시점 (Trigger)
- `hasNumberInput=true` 규칙의 숫자 입력 필드 변경 시

### 필수 데이터
| 항목 | 소스 |
|------|------|
| rule.numberValue | AlarmRule.numberValue (현재값) |
| rule.numberLabel | AlarmRule.numberLabel ("일 전" 등) |
| numVal | RuleCard 로컬 state |

### 인터랙션 (User Actions)
1. 숫자 입력 → `setNumVal(v)` + `onNumberChange(v)`
2. `handleNumberChange(id, val)` → `persistAlarm` → localStorage 저장
3. 별도 저장 버튼 없음 (onChange 즉시 저장)

### 비즈니스 룰
- 최솟값 min=1 (0 이하 입력 방지)
- 즉시 저장 (onChange 이벤트)
- numberValue 변경 = 발송 조건 변경 (실제 크론 발송 로직은 백엔드에서 읽음)

### 에지 케이스
- 빈 값 입력 시 Number("") = 0 → min=1 제한으로 방어
- 매우 큰 숫자(999일) → UI 표시만, 실제 제한 없음

### 접근성 (A11y)
- `<input type="number" aria-label="{rule.name} {rule.numberLabel} 값">`
