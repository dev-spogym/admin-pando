## 📝 디스크립션

### 사용 시점 (Trigger)
- RuleCard 우측 Toggle 클릭
- `handleToggle(rule.id)` 실행

### 필수 데이터
| 항목 | 소스 |
|------|------|
| rule.enabled | AlarmRule.enabled (boolean) |
| persistAlarm | useCallback(saveAlarmSettings) |

### 인터랙션 (User Actions)
1. Toggle 클릭 → `enabled` 반전
2. RuleCard 스타일 즉시 변경 (border/opacity/dot)
3. 상단 `enabledCount` 카드 숫자 즉시 업데이트
4. localStorage 자동 저장 (silent)

### 비즈니스 룰
- 토글 변경은 즉시 localStorage에 영속화
- 별도 [저장] 버튼 없음
- enabledCount = derived state (별도 setter 불필요)

### 에지 케이스
- localStorage 쓰기 실패 → toast.error 표시 (rules state는 변경 유지)
- 전체 비활성화(enabledCount=0) → 상단 카드 "0 / 13종" 표시

### 접근성 (A11y)
- Toggle button: `role="switch"` + `aria-checked={rule.enabled}`
- `aria-label="{rule.name} 자동 알림 {활성/비활성}"`
