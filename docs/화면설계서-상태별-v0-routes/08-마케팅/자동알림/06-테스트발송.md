```
Next.js 15 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-072 자동 알림 설정 — 상태: 테스트 발송 버튼 (미리보기 영역)

파일: src/app/(marketing)/message/auto-alarm/page.tsx

테스트 발송 버튼 위치:
- 규칙 설정 모달(isModalOpen=true) 내 미리보기 영역 하단
- Button variant="outline" size="sm" icon={<Smartphone size={13} />} className="mt-md rounded-full"
- 텍스트: "테스트 발송"

현재 구현 상태 (P2, 미구현):
- 버튼 클릭 → 기능 미구현 상태
- 향후 구현 예정:
  1. 현재 로그인한 센터장/매니저 전화번호로 테스트 발송
  2. modalData.content 그대로 발송 (변수 치환 없음)
  3. 발송 채널: modalData.channel 기준

향후 구현 시 테스트 발송 흐름:
- [테스트 발송] 클릭
- → toast.info("테스트 발송 중...")
- → supabase.from('messages').insert({
    branchId, type: channelTypeMap[modalData.channel],
    content: modalData.content,
    recipients: [{ id: adminId, name: adminName, phone: adminPhone }],
    status: 'SENT', sentAt: new Date().toISOString()
  })
- → toast.success("테스트 메시지가 발송되었습니다.")

미리보기 폰 목업 (테스트 발송 버튼 위):
- w-[220px] h-[440px] bg-content rounded-[32px] border-[7px] shadow-xl
- 노치: mt-7 상단 h-7 + w-14 h-3 rounded-full bg-black/30
- 메시지 카드: bg-surface rounded-[14px] p-md shadow-sm
  - 발신자: Smartphone icon (w-5 h-5 bg-primary) + "FitGenie CRM" + "방금 전"
  - 내용: text-[11px] whitespace-pre-wrap → modalData.content || "(내용을 입력하세요)"

채널별 타입 매핑 (향후 구현):
- talk → "KAKAO"
- sms → "SMS"
- lms → "SMS" (LMS 구분)
- push → "PUSH"

데이터:
- modalData.content: 현재 편집 중인 메시지 내용
- modalData.channel: 발송 채널
- 테스트 수신자: 로그인 관리자 정보 (미구현)

인터랙션:
- [테스트 발송] 클릭 → 현재 미구현 (버튼만 렌더)
- 미리보기 폰 목업은 content 실시간 반영

사용 유틸:
- Smartphone from 'lucide-react'
- Button from '@/components/ui/Button'
- toast from 'sonner' (향후 구현 시)
- supabase from '@/lib/supabase' (향후 구현 시)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 규칙 설정 모달 내 [테스트 발송] 버튼 클릭

### 필수 데이터
| 항목 | 소스 |
|------|------|
| modalData.content | 편집 중인 메시지 내용 |
| modalData.channel | 선택된 발송 채널 |

### 인터랙션 (User Actions)
1. [테스트 발송] 클릭 → 현재 미구현 (P2 기능)
2. 미리보기 폰에서 content 실시간 확인

### 비즈니스 룰
- 테스트 발송 = 관리자 본인 번호로 발송 (실제 회원 발송 아님)
- 변수({이름} 등)는 치환 없이 그대로 발송
- 발송 이력에 기록되지 않거나 별도 'TEST' 타입으로 구분 (향후 결정)

### 에지 케이스
- content 비어있을 때 테스트 발송 → toast.warning("내용을 입력하세요.") (향후)
- 관리자 전화번호 없음 → toast.warning 처리 (향후)

### 접근성 (A11y)
- Button: `aria-label="테스트 메시지 발송"`

### 연결 화면
