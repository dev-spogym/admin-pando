## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 logs.length > 0

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| logs | getAuditLogs API | 페이지 단위 로그 배열 |
| totalCount | pagination.total | 전체 건수 |
| branches | getBranches (슈퍼관리자) | 지점 필터 옵션 |

### 인터랙션 (User Actions)
1. 날짜 변경 → useEffect 재조회 → `01-로딩`
2. 액션/대상/지점 Select 변경 → useEffect 재조회
3. 사용자 검색 Enter / 검색 버튼 → handleSearch()
4. 페이지 번호 클릭 → handlePageChange(page)
5. 이전/다음 버튼 → handlePageChange(currentPage ±1)
6. CSV 다운로드 → ExportButton 클릭
7. 로그 행 클릭 → `05-상세모달` (추가 구현 시)

### 비즈니스 룰
- 사용자 검색: 클라이언트 측 필터 (userName 포함 여부)
- toDate 파라미터: `${toDate}T23:59:59` (당일 23:59:59까지 포함)
- 페이지네이션: ±2 페이지 범위 + 첫/마지막 페이지 항상 표시
- CSV 컬럼: 시간/사용자/액션/대상유형/대상ID/상세/IP
- 파일명: `감사로그_${fromDate}_${toDate}`

### 에지 케이스
- totalPages <= 1 → 페이지네이션 숨김 + 하단 총 건수 표시
- userName 없음 → `ID ${userId}` 표시
- detail JSON 없음 → '-' 표시
- fromBranchId/toBranchId 있음 → `지점 N → M` 요약

### 접근성 (A11y)
- 페이지 번호 버튼: `aria-label="N페이지로 이동"`
- 현재 페이지 버튼: `aria-current="page"`
- 테이블: `role="table"` (SimpleTable 내부)
