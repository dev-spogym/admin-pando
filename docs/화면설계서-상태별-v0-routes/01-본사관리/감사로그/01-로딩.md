### 인터랙션 (User Actions)
1. 필터 변경 → useEffect 재실행 → 로딩 상태로 복귀
2. 로딩 중 다른 필터 변경 → 이전 요청 결과 무시 (최신 요청 우선)

### 비즈니스 룰
- 슈퍼관리자: isSuperAdmin=true → 지점 필터 표시 + getBranches() 호출
- 일반 관리자: 지점 필터 숨김 (자신의 지점만 조회)
- 페이지 변경 시에도 로딩 표시
- 필터 변경 시 setCurrentPage(1) 리셋

### 에지 케이스
- API 응답 지연 > 5초 → 스켈레톤 유지 (타임아웃 없음)
- 필터 연속 변경 → 마지막 fetchLogs 결과만 반영

### 접근성 (A11y)
- 스켈레톤 영역: `aria-busy="true"` `aria-label="감사 로그 불러오는 중"`
- 필터 입력은 로딩 중에도 접근 가능 (disabled 처리 안 함)
