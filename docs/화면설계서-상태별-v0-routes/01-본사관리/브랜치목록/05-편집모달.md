편집 상태 추가:
const [isEditBranchOpen, setIsEditBranchOpen] = useState(false);
const [editTarget, setEditTarget] = useState<any>(null);
const [editForm, setEditForm] = useState<BranchForm>(EMPTY_FORM);
const [editErrors, setEditErrors] = useState<FormErrors>({});
const [isEditSaving, setIsEditSaving] = useState(false);

편집 모달 열기:
const openEditBranch = (row: any) => {
  setEditTarget(row);
  setEditForm({
    name: row.name ?? '',
    code: row.code ?? '',
    phone: row.phone ?? '',
    address: row.address ?? '',
    addressDetail: '',
    managerId: row.managerId ?? '',
    openTime: row.openTime ?? '06:00',
    closeTime: row.closeTime ?? '23:30',
    maxCapacity: String(row.maxCapacity ?? ''),
    memo: row.memo ?? '',
  });
  setEditErrors({});
  setIsEditBranchOpen(true);
};

편집 저장 핸들러:
const handleEditBranchSubmit = async () => {
  if (!validateEditForm()) return;
  if (isEditSaving) return;
  setIsEditSaving(true);
  try {
    const { error } = await supabase
      .from('branches')
      .update({
        name: editForm.name,
        phone: editForm.phone,
        address: editForm.address + (editForm.addressDetail ? ` ${editForm.addressDetail}` : ''),
      })
      .eq('id', editTarget.id);
    if (error) { toast.error(`지점 수정 실패: ${error.message}`); return; }
    setBranches(prev => prev.map(b => b.id === editTarget.id ? { ...b, ...editForm } : b));
    toast.success(`지점 "${editForm.name}"이(가) 수정되었습니다.`);
    setIsEditBranchOpen(false);
    setEditTarget(null);
  } finally {
    setIsEditSaving(false);
  }
};

// DataTable actions 컬럼 "수정" 버튼에 onClick={() => openEditBranch(row)} 연결

## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable actions 컬럼 "수정" 버튼 클릭 → openEditBranch(row) → isEditBranchOpen=true

### 필수 데이터
| 항목 | 설명 |
|------|------|
| editTarget | 수정 대상 지점 row 객체 |
| editForm | editTarget 값으로 초기화된 BranchForm |
| editErrors | 필드별 에러 메시지 |
| isEditSaving | 중복 제출 방지 |

### 인터랙션 (User Actions)
1. 필드 수정 → editForm 즉시 업데이트
2. "수정 완료" → validateEditForm() → supabase update → toast.success → 모달 닫기
3. "취소" / X → 모달 닫기 (editForm 변경사항 폐기)

### 비즈니스 룰
- 지점 코드(code): readOnly, 변경 불가
- 수정 가능 필드: 지점명 / 연락처 / 주소 / 지점관리자 / 운영시간
- supabase update 후 setBranches local 상태도 즉시 반영 (낙관적 업데이트)

### 에지 케이스
- supabase update 실패 → toast.error + 모달 유지
- isEditSaving=true 중 "수정 완료" 재클릭 방지

### 접근성 (A11y)
- 모달: `role="dialog"` `aria-modal="true"` `aria-labelledby="edit-modal-title"`
- 코드 필드 readOnly: `aria-readonly="true"` `aria-label="변경 불가"`
