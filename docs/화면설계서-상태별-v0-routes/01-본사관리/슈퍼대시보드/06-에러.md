## 📝 디스크립션

### 사용 시점 (Trigger)
- fetchAll() 내부 getBranches 실패 (branchRes.success=false)
- fetchAll() catch 블록 진입 (네트워크 오류, Supabase 연결 실패 등)

### 필수 데이터
| 항목 | 값 | 설명 |
|------|-----|------|
| loading | false | 에러 후 로딩 종료 |
| branches | [] | 빈 배열 (데이터 없음) |
| kpiData | 초기값 (모두 0) | 패치 실패로 초기값 유지 |

### 인터랙션 (User Actions)
1. toast.error 확인 — 자동 소멸 (Sonner 기본 4초)
2. 새로고침 버튼 클릭 → handleRefresh → fetchAll 재실행
3. (향후) "다시 시도" 버튼 → setError(null) + setLoading(true) + fetchAll()

### 비즈니스 룰
- 현재 구현: 에러 상태를 별도 state로 관리하지 않고 toast로만 표시
- 에러 후 loading=false → 빈 데이터로 기본 UI 렌더 (데이터 0건 상태처럼 보임)
- getBranches 실패: branches=[] → 지점별 현황 "등록된 지점이 없습니다." 표시
- fetchBranchStats 에러: toast 없음 (catch 없음) — 개별 지점 stats undefined 처리

### 에지 케이스
- getAuditLogs 실패: 현재 에러 처리 없음 (auditLogs=[] 유지)
- 부분 성공: getBranches 성공 + fetchBranchStats 일부 실패 → 성공한 지점만 통계 표시
- 네트워크 복구: 새로고침 버튼으로 재시도 가능

- 에러 메시지 (향후): `role="alert"` `aria-live="assertive"`
