## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 branches.length > 0

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| kpiData | fetchBranchStats 집계 | 전 지점 합산 KPI |
| branches | getBranches(tenantId) | 지점 목록 |
| branchStats | fetchBranchStats | 지점별 stats Map |
| auditLogs | getAuditLogs({page:1,size:10}) | 최근 10건 감사 로그 |

### 인터랙션 (User Actions)
1. "상세 보기" 클릭 → handleSwitchBranch(branch) → switchBranch + toast.success + router.push('/')
2. "지점 관리" 클릭 → router.push('/branches')
3. "전체 보기" 클릭 → router.push('/audit-log')
4. 새로고침 버튼 → handleRefresh → isRefreshing=true → fetchAll 재실행

### 비즈니스 룰
- KpiCard change: 전체회원/매출/출석은 MoM/DoD 비교; 직원/신규/만료 예정/만료는 change=0 고정
- branchStats[branch.id]가 undefined면 각 stat 셀에 animate-pulse 스켈레톤 표시
- 감사 로그 없으면 "감사 로그가 없습니다." 표시
- 매출 급감 배너: prevMonthRevenue > 0 && totalRevenue < prevMonthRevenue * 0.8 → 최상단 red AlertTriangle 배너

### 에지 케이스
- branches.length === 0 → "등록된 지점이 없습니다." 빈 상태 카드
- branchStats 부분 로드 중: stats=undefined 셀만 스켈레톤 표시
- auditLogs.length === 0 → "감사 로그가 없습니다."

### 접근성 (A11y)
- "상세 보기" 버튼: `aria-label="[지점명] 상세 보기"`
- KpiCard change 아이콘: `aria-hidden="true"`
- 새로고침: `aria-label="대시보드 새로고침"`
