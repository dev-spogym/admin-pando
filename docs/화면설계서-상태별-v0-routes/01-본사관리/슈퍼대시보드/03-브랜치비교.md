```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-099 슈퍼대시보드 — 상태: 브랜치 비교 (loading=false, branches.length >= 2)

파일: src/app/(admin)/super-dashboard/page.tsx

지점별 현황 섹션에서 branchStats가 모두 로드된 상태:
- 전 지점 매출 평균: avgRevenue = Object.values(branchStats).reduce((s,b)=>s+b.revenue,0) / branches.length
- 각 브랜치 카드에 평균 대비 매출 비율 표시:
  stats.revenue > avgRevenue → 초록색 "평균 +N%" 텍스트
  stats.revenue < avgRevenue * 0.5 → 빨간색 "평균 -N%" + border-red-200 bg-red-50/30 카드 강조
  그 외 → 기본 카드 스타일

지점 카드 통계 그리드 (grid-cols-3):
<div className="rounded-lg bg-surface-secondary p-sm text-center">
  <p className="text-[10px] text-content-tertiary mb-[2px]">활성 회원</p>
  <p className="text-[15px] font-bold text-content">
    {formatNumber(stats.members)}<span className="text-[10px] font-normal text-content-tertiary ml-[2px]">명</span>
  </p>
</div>
// 직원, 이번달 매출 동일 구조

평균 대비 매출 배지 (카드 헤더에 추가):
{stats && avgRevenue > 0 && (
  <span className={cn(
    'text-[10px] font-semibold px-[6px] py-[2px] rounded-full',
    stats.revenue > avgRevenue ? 'bg-green-50 text-green-600' : 'bg-red-50 text-red-500'
  )}>
    {stats.revenue >= avgRevenue ? '+' : ''}{Math.round(((stats.revenue - avgRevenue) / avgRevenue) * 100)}%
  </span>
)}

위험 신호 섹션 (alerts.length > 0일 때):
- 각 alert 카드: bg-amber-50 border-amber-200, AlertTriangle 아이콘
- reasons 목록: 매출 관련 / 만료 회원 급증 / 출석 저조 3가지 규칙
  규칙1: stats.revenue === 0 → '이번달 매출 없음'
         stats.revenue <= avgRevenue * 0.5 → '매출 급감 (평균 대비 N%)'
  규칙2: stats.expiredMembers >= stats.members * 0.3 → '만료 회원 급증 (활성 대비 N%)'
  규칙3: stats.todayAttendance <= stats.members * 0.1 → '출석 저조 (활성 대비 N%)'

사용 유틸:
- AlertTriangle from 'lucide-react'
- cn from '@/lib/utils'
- formatNumber from '@/lib/format'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본-전체브랜치`에서 branchStats가 모두 로드된 후 (branches.length >= 2)
- 위험 신호 섹션: alerts.length > 0

### 필수 데이터
| 항목 | 조건 | 설명 |
|------|------|------|
| branchStats | 모든 지점 stats 로드 완료 | 지점별 5개 지표 |
| avgRevenue | branches.length > 0 | 전 지점 매출 평균 |
| alerts | 규칙 3개 적용 후 | 위험 지점 배열 |

### 인터랙션 (User Actions)
1. 위험 신호 카드 확인 → 해당 지점 카드로 시각적 강조 확인
2. 강조된 지점 "상세 보기" → handleSwitchBranch(branch) → 해당 지점 대시보드
3. 지점 관리 → router.push('/branches')

### 비즈니스 룰
- 매출 평균 대비 배지: branchStats 전체 로드 후 계산 (일부 undefined 시 avgRevenue 계산 제외)
- 위험 신호 섹션: loading=true 또는 alerts.length=0이면 return null
- 지점 카드 강조: stats.revenue <= avgRevenue * 0.5인 경우 border-red-200 bg-red-50/30 적용
- 위험 신호는 "위험 신호" 섹션에만 표시 (지점 카드와 별도)

### 에지 케이스
- branches.length === 1 → 비교 불가, 평균 배지 미표시
- avgRevenue === 0 → 평균 대비 배지 미표시
- 모든 지점 alerts → 전체 지점이 위험 신호 섹션에 표시됨

- 위험 이유 목록: `role="list"`
