## 📝 디스크립션

### 사용 시점 (Trigger)
- fetchAll 완료 후 전 지점 합산 totalRevenue < prevMonthRevenue * 0.8 조건 충족

### 필수 데이터
| 항목 | 조건 | 설명 |
|------|------|------|
| kpiData.totalRevenue | 이번달 전 지점 합산 | 현재 매출 |
| kpiData.prevMonthRevenue | 전월 전 지점 합산 | 전월 매출 |
| dropPct | (1 - curr/prev) * 100 | 감소율 % |

### 인터랙션 (User Actions)
1. 배너 확인 (클릭 없음) — 읽기 전용 정보
2. 새로고침 → fetchAll 재실행 → 조건 재확인
3. KpiCard '전체 매출' → 변동률 시각 확인
4. 위험 신호 섹션 → 지점별 원인 확인

### 비즈니스 룰
- 기준: totalRevenue < prevMonthRevenue * 0.8 (20% 초과 감소)
- loading=true 또는 prevMonthRevenue=0이면 배너 미표시
- 배너는 PageHeader 바로 다음, KPI 카드 섹션 위에 위치
- 위험 신호 섹션의 지점별 매출 급감 alerts와 독립적으로 작동

### 에지 케이스
- prevMonthRevenue=0 → 배너 미표시 (0으로 나누기 방지)
- totalRevenue=0 → 배너 표시 (100% 감소)
- 일부 지점만 매출 급감 → 배너는 합산 기준 / 위험 신호는 지점 기준

### 접근성 (A11y)
- 배너: `role="alert"` `aria-live="assertive"`
- AlertTriangle: `aria-hidden="true"`
