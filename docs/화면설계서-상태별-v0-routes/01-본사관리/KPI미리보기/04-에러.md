## 📝 디스크립션

### 사용 시점 (Trigger)
- authUser가 null인 경우 (세션 만료 / 로그인 필요)
- normalizeRole 실패로 boardKey 결정 불가
- 현재 정적 데이터 기반이므로 실질적 에러 발생 가능성 낮음

### 필수 데이터
| 항목 | 설명 |
|------|------|
| hasError | 에러 상태 플래그 |
| errorMessage | 사용자에게 표시할 에러 메시지 |

### 인터랙션 (User Actions)
1. "새로고침" → window.location.reload()
2. "대시보드로 이동" → moveToPage(966)

### 비즈니스 룰
- authUser null → 에러 UI 표시
- PageHeader는 에러 상태에서도 title/description 렌더 유지
- boardContents 정적 데이터 기반 → 에러 발생 가능성 매우 낮음

### 에지 케이스
- 세션 만료 → authUser null → 에러 메시지 + 재로그인 유도
- 네트워크 장애 → 새로고침 버튼으로 복구 유도

### 접근성 (A11y)
- 에러 아이콘: `role="img"` `aria-label="에러"`
- 에러 메시지: `role="alert"` `aria-live="assertive"`
