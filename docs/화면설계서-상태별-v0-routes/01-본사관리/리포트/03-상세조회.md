## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable 행 클릭 (status=READY) → setSelectedReport(row)
- DataTable actions "보기" 버튼 클릭

### 필수 데이터
| 항목 | 설명 |
|------|------|
| selectedReport | 선택된 Report 객체 (status=READY) |
| report.kpi | totalRevenue / totalMembers / newMembers / totalAttendance / avgDailyAttendance |
| report.type | DAILY / WEEKLY / MONTHLY |
| report.periodLabel | 기간 레이블 텍스트 |

### 인터랙션 (User Actions)
1. X 버튼 → setSelectedReport(null) → 모달 닫기
2. ExportButton → exportData CSV 다운로드 (파일명: `report-{id}`)
3. "이메일 발송" → toast.info('이메일 발송 기능은 준비 중입니다.')

### 비즈니스 룰
- TYPE_COLOR: DAILY→blue, WEEKLY→green, MONTHLY→purple 뱃지
- exportData: 항목/값 2컬럼 CSV
- KPI 카드: 총 매출(formatAmount) / 전체회원 / 신규회원(+N명) / 총출석
- 이메일 발송: 미구현 (toast.info 안내)

### 에지 케이스
- GENERATING/FAILED 행 → onRowClick 무시 (selectedReport 미설정)
- 모달 외부 클릭 닫기: 미구현 (X 버튼으로만 닫기)

### 접근성 (A11y)
- 모달: `role="dialog"` `aria-modal="true"` `aria-labelledby="detail-modal-title"`
- ExportButton: `aria-label="리포트 CSV 다운로드"`
