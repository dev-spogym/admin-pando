## 📝 디스크립션

### 사용 시점 (Trigger)
- `/reports` 첫 진입 (useEffect → loadReports)
- PageHeader "새로고침" 버튼 클릭

### 필수 데이터
| 항목 | 초기값 | 설명 |
|------|--------|------|
| loading | false (loadReports 호출 시 true) | 로딩 플래그 |
| reports | [] | 리포트 배열 |
| branchId | authUser 기반 결정 | 지점 ID (undefined 가능) |

### 인터랙션 (User Actions)
1. "새로고침" disabled=true (로딩 중)
2. "리포트 생성" 버튼: 로딩 중에도 클릭 가능 (모달 열기)
3. 필터 탭: 클릭 가능 (loading 독립)

### 비즈니스 룰
- generateReportList: 7 일간 + 4 주간 + 3 월간 = 14개 리포트 순차 생성
- 각 리포트: fetchReportKpi() 4개 쿼리 (매출/회원/신규/출석)
- 총 쿼리 수: 약 14 × 4 = 56개 → 로딩 시간 길 수 있음

### 에지 케이스
- generateReportList 실패 → toast.error + reports=[] (빈상태 전환)
- authUser null → branchId=undefined → 전체 지점 집계

### 접근성 (A11y)
- RefreshCw 스피너: `aria-label="리포트 불러오는 중"` `role="status"`
- DataTable loading: `aria-busy="true"`

### 연결 화면
- 이전: 사이드바 메뉴
- 다음: `02-기본-리포트목록` (성공) / `05-빈상태` (0건)
