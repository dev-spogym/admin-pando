## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 metrics 데이터 수신

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| metrics | Promise.all 18쿼리 | KpiMetrics 전체 |
| monthlyTarget | localStorage | 매출 목표 (null 가능) |
| branchId | getBranchId() | 현재 지점 |

### 인터랙션 (User Actions)
1. "목표 설정" 버튼 → TargetModal 열기
2. "새로고침" 버튼 → fetchMetrics() 재실행 → `01-로딩`
3. 매출 게이지 "목표 수정" → TargetModal 열기
4. monthlyTarget >= monthlyRevenue → `05-목표달성`
5. monthlyTarget < monthlyRevenue × 0.5 → `06-목표미달-경고`

### 비즈니스 룰
- activeRate = pct(activeMembers, totalMembers)
- revenueMoM = mom(monthlyRevenue, prevMonthRevenue)
- PT 완료율 >= 85% → mint, 미만 → peach
- PT 노쇼율 <= 5% → mint, 초과 → peach
- 수업 출석률 >= 80% → mint, 미만 → peach
- monthlyTarget: `kpi_monthly_target_{branchId}_{yearMonth}` localStorage 키

### 에지 케이스
- monthlyTarget null → RevenueGauge "목표를 설정하세요" 텍스트
- 전월 매출 0 → MoM 0% (0으로 나누기 방지)
- CohortAnalysis / FunnelAnalysis 각자 독립 loading 상태

### 접근성 (A11y)
- 섹션 헤더: `<h3>` 시맨틱 태그
- 게이지 바: `role="progressbar"` `aria-valuenow` `aria-valuemax="100"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `05-목표달성` / `06-목표미달-경고` / 목표 설정 모달
