const storageKey = `kpi_monthly_target_${branchId}_${yearMonth}`;

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본-종합` 렌더 시 매출 섹션에서 MoM 데이터 표시
- 목표 설정 후 RevenueGauge 업데이트

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| monthlyRevenue | supabase sales | 이번 달 매출 합산 |
| prevMonthRevenue | supabase sales | 전월 매출 합산 |
| monthlyTarget | localStorage | 이번 달 목표 금액 |
| yearMonth | 오늘 기준 계산 | YYYY-MM 형식 |

### 인터랙션 (User Actions)
1. "목표 설정" 버튼 → TargetModal 열기 → 금액 입력 → localStorage 저장
2. 목표 저장 후 → RevenueGauge 즉시 업데이트
3. 새로고침 → fetchMetrics 재실행 (yearMonth 동일)

### 비즈니스 룰
- 기간: 항상 이번 달 1일 ~ 말일 (고정, 변경 UI 미구현)
- 전월 비교: prevMonthStart ~ prevMonthEnd 자동 계산
- 목표 저장 단위: 원 (localStorage에 순수 숫자 문자열)
- formatAmount: >=1억 → N.N억, >=1만 → N만, 그 외 → toLocaleString

### 에지 케이스
- prevMonthRevenue = 0 → MoM = 0% (0 나누기 방지)
- monthlyTarget = null → "목표를 설정하세요" 표시
- 목표 입력값 <= 0 → toast.error("올바른 목표 금액을 입력해 주세요.")

### 접근성 (A11y)
- TargetModal input: `type="text"` `inputMode="numeric"` `aria-label="매출 목표 금액"`
- RevenueGauge 바: `role="progressbar"` `aria-valuenow={pctValue}` `aria-valuemax="100"`
