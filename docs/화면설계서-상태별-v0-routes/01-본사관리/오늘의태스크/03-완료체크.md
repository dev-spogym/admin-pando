상태 진행 버튼 핸들러:
onClick={() => {
  if (task.status === "완료") return;
  const nextStatus: TodayTask["status"] = task.status === "대기" ? "진행중" : "완료";

  if ("source" in task) {
    // 커스텀 태스크 (EditableTask): customTasks에서 직접 업데이트
    setCustomTasks(prev => prev.map(t => t.id === task.id ? { ...t, status: nextStatus } : t));
  } else {
    // 랜덤 배정 태스크: customTasks에 상태 오버라이드 항목 추가
    setCustomTasks(prev => {
      const exists = prev.find(t => t.id === task.id);
      if (exists) {
        return prev.map(t => t.id === task.id ? { ...t, status: nextStatus } : t);
      }
      return [{ ...task, status: nextStatus, source: "hq" as const }, ...prev];
    });
  }
}}

## 📝 디스크립션

### 사용 시점 (Trigger)
- "처리 시작" 버튼 클릭 → 대기→진행중
- "완료 처리" 버튼 클릭 → 진행중→완료

### 필수 데이터
| 항목 | 변화 | 설명 |
|------|------|------|
| task.status | 대기→진행중→완료 | 클릭당 1단계 진행 |
| customTasks | 업데이트 | localStorage 자동 동기화 |
| StatCard 카운트 | 즉시 갱신 | waitingCount / progressCount / doneCount |

### 인터랙션 (User Actions)
1. "처리 시작" → nextStatus="진행중" → customTasks 업데이트
2. "완료 처리" → nextStatus="완료" → customTasks 업데이트
3. "완료됨" → disabled (더 이상 진행 불가)
4. 필터 탭 "진행중" → 진행중 태스크만 표시
5. 필터 탭 "완료" → 완료 태스크만 표시

### 비즈니스 룰
- 상태 진행은 단방향: 대기→진행중→완료 (되돌리기 없음)
- 랜덤 태스크 상태 변경: source:"hq"로 customTasks에 복사하여 오버라이드
- localStorage 저장: customTasks useEffect로 매 변경 시 즉시 저장
- 새로고침 후에도 완료 상태 유지 (localStorage 기반)

### 에지 케이스
- 동일 랜덤 태스크 중복 오버라이드: exists 체크로 중복 방지
- filter="진행중" 탭에서 "완료 처리" → 완료 후 해당 탭에서 사라짐 (필터 재적용)
- filter="완료" 탭에서 모든 태스크 → `04-빈상태-모두완료` 로직 확인 필요

### 접근성 (A11y)
- 완료된 버튼: `aria-disabled="true"` `disabled`
- 진행 버튼: `aria-label="[태스크명] 처리 시작"` / `aria-label="[태스크명] 완료 처리"`
