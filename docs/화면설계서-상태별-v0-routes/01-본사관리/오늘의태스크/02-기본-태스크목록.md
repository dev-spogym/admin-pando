## 📝 디스크립션

### 사용 시점 (Trigger)
- 마운트 완료 후 allTasks.length > 0

### 필수 데이터
| 항목 | 소스 | 설명 |
|------|------|------|
| randomTasks | getTodayTasks() useMemo | 날짜+지점+역할 기반 랜덤 |
| customTasks | localStorage(STORAGE_KEY) | 슈퍼관리자 직접 입력 |
| allTasks | visibleCustomTasks + randomTasks | 전체 태스크 |
| filter | useState('all') | 현재 상태 필터 |

### 인터랙션 (User Actions)
1. 탭 클릭 → setFilter → tasks 즉시 필터
2. "처리 시작" → 대기→진행중
3. "완료 처리" → 진행중→완료
4. "완료됨" → disabled (클릭 없음)
5. isSuperAdmin 전용: "수정" → handleEdit / "삭제" → handleDelete

### 비즈니스 룰
- randomTasks: dateKey+branchName+role+isSuperAdmin → 동일 조건 동일 결과 (deterministic random)
- visibleCustomTasks: isSuperAdmin이면 전체 / 아니면 branchName+taskRole 매칭
- 랜덤 태스크 상태 변경: customTasks에 source:"hq"로 복사 후 관리 (원본 불변)
- STORAGE_KEY = "today_tasks_custom" → 새로고침 후에도 커스텀 태스크 유지

### 에지 케이스
- filter='대기' + 대기 태스크 없음 → tasks=[] → "해당 상태의 업무가 없습니다." 빈 상태
- customTasks와 randomTasks 중복 id → visibleCustomTasks 먼저 → 오버라이드 적용

### 접근성 (A11y)
- 태스크 카드: `role="article"`
- 상태 버튼 disabled: `aria-disabled="true"`
- TabNav: `role="tablist"` `aria-selected`
