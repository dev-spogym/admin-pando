```
화면: SCR-083 IOT 설정 — 상태: 특정 기기 연결 테스트 진행 중

연결 테스트 로직:
const [testingDeviceId, setTestingDeviceId] = useState<string | null>(null);

const handleConnectionTest = async (device: IotDevice) => {
  setTestingDeviceId(device.id);
  try {
    // 실제 ping API 대신 타임아웃 시뮬레이션
    await new Promise((resolve, reject) => {
      setTimeout(() => {
        // 시뮬레이션: 80% 성공, 20% 실패
        Math.random() > 0.2 ? resolve(null) : reject(new Error('timeout'));
      }, 2000);
    });
    // 성공
    setDevices(prev => prev.map(d =>
      d.id === device.id
        ? { ...d, status: 'connected', lastPing: new Date().toISOString() }
        : d
    ));
    toast.success(`'${device.name}' 연결 확인 완료`);
  } catch {
    setDevices(prev => prev.map(d =>
      d.id === device.id ? { ...d, status: 'error' } : d
    ));
    // → 04-연결실패 상태
  } finally {
    setTestingDeviceId(null);
  }
};

UI (testingDeviceId === device.id):
- 해당 기기 행: bg-blue-50 border-l-2 border-blue-400
- "연결 테스트" 버튼 → <Loader2 className="animate-spin w-4 h-4" /> "테스트 중..."
- 상태 셀: <span className="flex items-center gap-1 text-blue-500 animate-pulse"><Activity size={14}/>연결 중...</span>
- 다른 행의 연결 테스트 버튼: disabled (동시 1개만 허용)

전체 동기화 (syncAll):
- 모든 기기 순차 ping (3초 간격)
- 진행률 표시: "3 / 5 기기 동기화 중..."

사용 유틸:
- toast from 'sonner'
- lucide-react: Loader2, Activity
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 특정 기기의 "연결 테스트" 버튼 클릭 시
- 전체 동기화 버튼 클릭 시 (순차 실행)

### 필수 데이터
- 없음 (시뮬레이션 ping, 실제 IoT API 미구현)

### 인터랙션 (User Actions)
1. 테스트 진행 중 — 해당 기기 행 강조, 다른 테스트 차단
2. 성공 → `02-기본-기기목록` (status: 'connected')
3. 실패 → `04-연결실패` (status: 'error')

### 비즈니스 룰
- 동시 연결 테스트 1개만 허용
- 타임아웃 3초
- 성공 시 lastPing 업데이트

### 에지 케이스
- 사용자가 테스트 중 페이지 이탈 → finally 블록에서 상태 초기화

### 접근성 (A11y)
- 테스트 중 버튼 `aria-busy="true"`

- 다음: `02-기본-기기목록` (성공) / `04-연결실패`
