interface IotDevice {
  id: string;

  lastPing: string;      // 마지막 통신 시각
  location: string;      // 위치 설명
}

- 최근 통신: formatDate(lastPing, 'HH:mm:ss')

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 기기 1건 이상 존재

### 필수 데이터
| 블록 | 소스 | 설명 |
|------|------|------|
| IOT 기기 목록 | `localStorage: iot_devices_{branchId}` | 가상 데이터 |
| 출입 정책 | `settings` | `blockAfterHours, blockExpiredMembers, alertUnregistered` |

### 인터랙션 (User Actions)
1. 기기 추가 버튼 → `05-기기추가모달`
2. 연결 테스트 → `03-연결중` (해당 기기)
3. 전체 동기화 → 모든 기기 ping 재시도
4. 출입 규칙 토글 → settings UPDATE
5. 탭 전환 → 출입 로그, 출입 규칙 표시

### 비즈니스 룰
- 오류/오프라인 기기: 자동 배정 대상 제외
- 기기 삭제 시 확인 모달 (이름 타이핑 불필요, 단순 confirm)
- 연결 테스트: 3초 timeout

### 에지 케이스
- 기기 0건 → `06-빈상태`
- ping 타임아웃 → `04-연결실패`

### 접근성 (A11y)
- 상태 아이콘 `aria-label="연결됨"` / `"오프라인"` / `"오류"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-연결중` / `05-기기추가모달` / `06-빈상태`
