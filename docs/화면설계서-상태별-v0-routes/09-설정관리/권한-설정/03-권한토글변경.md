```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-081 권한 설정 — 상태: 권한 체크박스 변경 중 (isDirty=true)

파일: src/app/settings/permissions/page.tsx

권한 토글 로직:
const handlePermToggle = (menuId: string, permKey: string) => {
  if (selectedRole?.code === 'primary') return; // 최고관리자 변경 불가

  setPermissions(prev => ({
    ...prev,
    [selectedRole.code]: {
      ...prev[selectedRole.code],
      [menuId]: {
        ...prev[selectedRole.code][menuId],
        [permKey]: !prev[selectedRole.code][menuId][permKey],
      }
    }
  }));
  setIsDirty(true);
};

변경된 셀 스타일 (changedCells Set에 포함된 셀):
<td
  onClick={() => handlePermToggle(menuId, permKey)}
  className={cn(
    "w-16 h-10 text-center cursor-pointer transition-all",
    perm[permKey]
      ? "bg-primary/10 text-primary hover:bg-primary/20"
      : "bg-gray-50 text-gray-300 hover:bg-gray-100",
    isChanged && "ring-2 ring-inset ring-amber-400 bg-amber-50"
  )}>
  {perm[permKey]
    ? <Check size={14} className="mx-auto" />
    : <X size={14} className="mx-auto" />
  }
</td>

충돌 감지 (실시간):
const conflicts = useMemo(() => {
  if (!selectedRole) return [];
  const matrix = permissions[selectedRole.code] || {};
  const result: string[] = [];

  Object.entries(matrix).forEach(([menuId, perm]) => {
    if (perm.delete && !perm.update)
      result.push(`[${menuId}] 삭제 권한이 있지만 수정 권한이 없습니다.`);
    if ((perm.create || perm.update || perm.delete) && !perm.access)
      result.push(`[${menuId}] 쓰기 권한이 있지만 접근 권한이 없습니다.`);
    if ((perm.create || perm.update || perm.delete) && !perm.read)
      result.push(`[${menuId}] 조회 권한 없이 쓰기 권한이 설정되어 있습니다.`);
  });
  return result;
}, [permissions, selectedRole]);

충돌 경고 배너 (conflicts.length > 0):
<div className="flex items-start gap-2 p-3 bg-amber-50 border border-amber-200 rounded-lg">
  <AlertTriangle className="w-4 h-4 text-amber-600 mt-0.5 shrink-0" />
  <div>
    <p className="text-sm font-medium text-amber-700">{conflicts.length}개 권한 충돌이 감지되었습니다.</p>
    <ul className="text-xs text-amber-600 mt-1 space-y-0.5">
      {conflicts.slice(0, 3).map((c, i) => <li key={i}>• {c}</li>)}
      {conflicts.length > 3 && <li>+ {conflicts.length - 3}개 더...</li>}
    </ul>
  </div>
</div>

사용 유틸:
- lucide-react: Check, X, AlertTriangle
- cn from '@/lib/utils'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-역할별탭`에서 체크박스 셀 클릭 시 (isDirty=true 전환)

### 필수 데이터
- 없음 (클라이언트 상태 변경, 저장 전까지 DB 미반영)

### 인터랙션 (User Actions)
1. 체크박스 클릭 → 해당 셀 권한 토글, ring-2 ring-amber 표시
2. 모두 허용/모두 거부 → 전체 셀 일괄 변경
3. 충돌 감지 → 실시간 충돌 배너 표시
4. 저장 → 충돌 있으면 `06-충돌경고`, 없으면 `04-저장중`

### 비즈니스 룰
- 최고관리자(primary): 체크박스 클릭 완전 차단
- 변경된 셀: ring-2 ring-amber-400 시각적 표시
- 충돌 3가지 규칙 실시간 감지

### 에지 케이스
- 모두 허용 후 일부 해제 → changedCells 정확히 추적
- 원래 값으로 복원 → changedCells에서 제거, isDirty 재계산

### 접근성 (A11y)
- 체크박스 셀 `role="checkbox"`, `aria-checked={perm[key]}`
- 충돌 배너 `role="alert"`

### 연결 화면
- 이전: `02-역할별탭`
- 다음: `04-저장중` / `06-충돌경고`
