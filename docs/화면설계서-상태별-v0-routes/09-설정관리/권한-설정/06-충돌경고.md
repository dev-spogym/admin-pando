충돌 감지 규칙:
const getConflicts = (matrix: PermissionMatrix): ConflictItem[] => {
  const conflicts: ConflictItem[] = [];
  Object.entries(matrix).forEach(([menuId, perm]) => {
    if (perm.delete && !perm.update)
      conflicts.push({ menuId, type: 'delete_without_update',
        message: `[${menuId}] 삭제 권한이 있지만 수정 권한이 없습니다.` });
    if ((perm.create || perm.update || perm.delete) && !perm.access)
      conflicts.push({ menuId, type: 'write_without_access',
        message: `[${menuId}] 쓰기 권한이 있지만 접근 권한이 없습니다.` });
    if ((perm.create || perm.update || perm.delete) && !perm.read)
      conflicts.push({ menuId, type: 'write_without_read',
        message: `[${menuId}] 조회 권한 없이 쓰기 권한이 설정되어 있습니다.` });
  });
  return conflicts;
};

## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장 버튼 클릭 시 conflicts.length > 0인 경우
- 실시간 충돌 배너와 별개로 저장 시점에 모달로 재확인

### 필수 데이터
- 없음 (클라이언트 상태 기반 충돌 계산)

### 인터랙션 (User Actions)
1. "취소 (권한 수정)" → 모달 닫힘, `03-권한토글변경` 복귀 (수정 계속)
2. "경고 무시하고 저장" → `04-저장중` (충돌 그대로 저장)

### 비즈니스 룰
충돌 3가지 규칙:
1. delete=true && update=false → "삭제 권한이 있지만 수정 권한이 없습니다."
2. (create|update|delete)=true && access=false → "쓰기 권한이 있지만 접근 권한이 없습니다."
3. (create|update|delete)=true && read=false → "조회 권한 없이 쓰기 권한이 설정되어 있습니다."

### 에지 케이스
- 충돌 항목 많을 때 (10개+) → max-h-48 overflow-y-auto 스크롤
- 강제 저장 후 동일 충돌 남아있음 → 다음 저장 시도에 재경고

### 접근성 (A11y)
- 모달 `role="alertdialog"`, `aria-labelledby`, `aria-describedby`
- focus: "취소" 버튼에 초기 포커스 (파괴적 액션 방지)
- ESC → 모달 닫기

### 연결 화면
- 이전: `03-권한토글변경`
- 다음: `04-저장중` (강제 저장) / `03-권한토글변경` (취소)
