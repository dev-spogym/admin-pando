<div className="space-y-4">

시간 검증:
const openTimeError = useMemo(() => {
  if (!centerInfo.openTime || !centerInfo.closeTime) return false;
  return centerInfo.openTime >= centerInfo.closeTime;
}, [centerInfo.openTime, centerInfo.closeTime]);

저장 시 검증:
if (openTimeError) {
  toast.error('운영 시간을 확인해주세요.');
  return;
}

## 📝 디스크립션

### 사용 시점 (Trigger)
- 기본정보 탭 내 운영시간 필드(Input[time]) 수정 시
- isDirty=true 전환과 동일한 흐름

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 운영시간 저장 | `settings` | `UPDATE businessHoursOpen, businessHoursClose, weekendOpen, weekendClose` |

### 인터랙션 (User Actions)
1. 시간 입력 → 실시간 검증 (오픈 < 마감)
2. 오류 시 → 인라인 오류 메시지
3. 저장 → 검증 통과 후 DB UPDATE

### 비즈니스 룰
- 오픈 시간 < 마감 시간 강제
- 주말 운영 안 함 → 시간 비워도 허용 (null 저장)
- IoT 출입 차단 정책과 연동 (영업시간 외 차단)

### 에지 케이스
- 24시간 운영 → 00:00 ~ 23:59
- 주말 시간 미입력 → null 저장 (영업 안 함으로 처리)

### 접근성 (A11y)
- Input[time] `aria-label="평일 오픈 시간"`
- 오류 메시지 `role="alert"`

### 연결 화면
- 이전: `01-기본-탭메뉴`
- 다음: `05-저장중`
