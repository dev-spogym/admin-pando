```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-080 설정 홈 — 상태: 저장 요청 진행 중 (isSaving=true)

파일: src/app/settings/page.tsx

저장 로직:
const handleSave = async () => {
  setIsSaving(true);
  try {
    // 탭에 따라 다른 필드 저장
    if (activeTab === 'basic') {
      await Promise.all([
        supabase.from('settings').update({
          centerName: centerInfo.name,
          businessNumber: centerInfo.businessNumber,
          description: centerInfo.description,
          businessHoursOpen: centerInfo.openTime,
          businessHoursClose: centerInfo.closeTime,
          weekendOpen: centerInfo.weekendOpenTime,
          weekendClose: centerInfo.weekendCloseTime,
          sectors: centerInfo.sectors,
        }).eq('branchId', branchId),
        supabase.from('branches').update({
          name: centerInfo.name,
          address: centerInfo.address,
          phone: centerInfo.phone,
        }).eq('id', branchId),
      ]);
    } else if (activeTab === 'notification') {
      await supabase.from('settings').update({
        smsEnabled: notificationSettings.smsPayment || notificationSettings.smsExpiry,
        pushEnabled: notificationSettings.pushEntrance || notificationSettings.pushExpiry,
        autoExpireNotify: notificationSettings.pushExpiry || notificationSettings.emailExpiry,
      }).eq('branchId', branchId);
    } else if (activeTab === 'theme') {
      await supabase.from('settings').update({ theme: themeSettings }).eq('branchId', branchId);
    } else if (activeTab === 'supplies') {
      await supabase.from('settings').update({ supplies: suppliesData }).eq('branchId', branchId);
    }

    setIsDirty(false);
    setSaveSuccess(true);
    setTimeout(() => setSaveSuccess(false), 3000);
  } catch {
    toast.error('설정 저장에 실패했습니다. 다시 시도해주세요.');
  } finally {
    setIsSaving(false);
  }
};

UI (isSaving=true):
- 저장 버튼: <Loader2 className="animate-spin w-4 h-4 mr-1" /> "저장 중..." disabled animate-pulse
- 탭 전환 버튼: disabled
- 폼 전체: pointer-events-none opacity-70

성공 후 (saveSuccess=true):
- PageHeader에 "✅ 변경사항이 저장되었습니다." accent 배지 (3초 후 해제)
- isDirty=false → 저장 버튼 다시 disabled

사용 유틸:
- supabase, toast
- lucide-react: Loader2, CheckCircle2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장하기 버튼 클릭 시 (isDirty=true 상태)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 설정 저장 | `settings` | `UPDATE WHERE branchId = ?` |
| 지점 저장 (basic탭) | `branches` | `UPDATE WHERE id = branchId` |

### 인터랙션 (User Actions)
1. 저장 중 — 모든 인터랙션 차단
2. 성공 → saveSuccess=true, 3초 후 해제, isDirty=false
3. 실패 → toast.error, 폼 원복

### 비즈니스 룰
- 탭별 다른 UPDATE 쿼리 실행
- 기본정보 탭: settings + branches 두 테이블 동시 UPDATE
- 성공 시 isDirty=false, 저장 버튼 다시 disabled

### 에지 케이스
- 부분 실패 (settings 성공, branches 실패) → 전체 실패로 처리, toast.error

### 접근성 (A11y)
- 저장 버튼 `aria-busy="true"`
- 성공 배지 `role="status"` `aria-live="polite"`
