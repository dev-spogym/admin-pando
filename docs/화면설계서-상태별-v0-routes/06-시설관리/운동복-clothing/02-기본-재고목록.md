    actions={[<Button>지급 처리</Button>, <Button>반납 처리</Button>]} />

    <StatCard label="총 재고" value={totalStock} icon={Shirt} variant="default" unit="벌" />
    <StatCard label="대여중" value={rentedCount} icon={Users} variant="mint" unit="벌" />
    <StatCard label="세탁중" value={washingCount} icon={Waves} variant="peach" unit="벌" />
    <StatCard label="사용가능" value={availableCount} icon={CheckCircle} variant="default" unit="벌" />
  </StatCardGrid>

컴포넌트 구조:
1. StatCardGrid (4컬럼) — 총 재고 / 대여중 / 세탁중 / 사용가능 수치 표시
2. TabNav — 재고 현황 / 대여 이력 전환
3. DataTable (재고 현황 탭):
   - 컬럼: 품목(상의/하의/수건) | 사이즈 | 총 재고 | 대여중 | 세탁중 | 사용가능 | 일일한도
   - 각 행에 [지급] [반납] 버튼
   - 사용가능 수량 0이면 셀 text-red-500 bold 표시
4. 지급 처리 버튼 → 지급 모달 열기 (→ 03-지급처리 상태)
5. 반납 처리 버튼 → 반납 모달 열기 (→ 04-반납처리 상태)

데이터:
- interface ClothingItem {
    id: number; item: '상의' | '하의' | '수건'; size: string;
    totalStock: number; rentedCount: number; washingCount: number;
    availableCount: number; dailyLimit: number; branchId: number;
  }
- supabase.from('clothing').select('*').eq('branch_id', branchId).order('item').order('size')
- 통계: totalStock = sum(totalStock), rentedCount = sum(rentedCount), 등

- PageHeader [지급 처리] → 전체 지급 모달 (품목/사이즈 선택)
- PageHeader [반납 처리] → 전체 반납 모달

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후, `clothing` 테이블에 1건 이상 데이터 존재
- 기본 진입 상태 (재고 현황 탭 active)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 재고 목록 | `clothing` | `branch_id = ?` ORDER BY item, size |
| 통계 카드 | `clothing` (집계) | SUM per column |

### 인터랙션 (User Actions)
1. [지급 처리] 버튼 → `03-지급처리` 모달 오픈
2. [반납 처리] 버튼 → `04-반납처리` 모달 오픈
3. 탭 전환(재고 현황↔대여 이력) → 뷰 교체
4. 행별 [지급]/[반납] → 해당 품목+사이즈 pre-fill 모달

### 비즈니스 룰
- 사용가능 = 총 재고 - 대여중 - 세탁중
- 사용가능 0건 시 셀 강조 (`text-red-500 font-bold`)
- 일일한도 초과 지급 시 `toast.error("일일 대여 한도를 초과했습니다.")`
- 스태프는 조회/지급/반납만 가능, 재고 수정은 센터장/매니저 전용

### 에지 케이스
- 특정 사이즈 재고 0 → [지급] 버튼 disabled
- 대여중 0인 품목 → [반납] 버튼 disabled

### 접근성 (A11y)
- DataTable `role="table"`, 각 행 `role="row"`
- 숫자 셀 `aria-label="{품목} {사이즈} 사용가능 {N}벌"`
