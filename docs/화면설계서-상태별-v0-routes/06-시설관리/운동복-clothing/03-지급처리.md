## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-055 운동복 관리 — 상태: 지급처리 모달

파일: src/app/(facilities)/clothing/page.tsx

레이아웃:
- 기본 레이아웃 유지 (배경 dim overlay)
- <Modal title="운동복 지급" isOpen={issueModalOpen} onClose={() => setIssueModalOpen(false)}>
    <form onSubmit={handleIssue}>
      회원 검색 AutoComplete (이름/회원번호)
      품목 Select: 상의 / 하의 / 수건
      사이즈 Select: XS / S / M / L / XL / XXL (품목에 따라 동적)
      수량 Input(number, min=1, max=availableCount)
      메모 Textarea (선택)
      footer: [취소] [지급 처리]
    </form>
  </Modal>

컴포넌트 구조:
1. 배경: 02-기본-재고목록 화면 (dim 처리)
2. Modal — 지급 처리 폼
   - 회원 검색: AutoComplete → members 테이블에서 이름/번호 검색
   - 품목/사이즈: cascading Select (품목 선택 시 사이즈 옵션 갱신)
   - 수량: 사용가능 재고 이하 제한 (max 표시)
   - [지급 처리] → handleIssue() 실행

handleIssue():
  1. 유효성 검사 (회원 필수, 수량 > 0, 수량 <= available)
  2. supabase.from('clothing').update({ rentedCount: prev + qty }).eq('id', itemId)
  3. supabase.from('clothing_rentals').insert({ memberId, item, size, qty, issuedAt: new Date() })
  4. toast.success(`${memberName}님께 ${item} ${size} ${qty}벌이 지급되었습니다.`)
  5. 모달 닫기 + 데이터 재조회

데이터:
- interface ClothingRental {
    id: number; clothingId: number; memberId: number; memberName: string;
    item: string; size: string; qty: number; issuedAt: string; returnedAt?: string;
    status: 'rented' | 'returned' | 'washing'; branchId: number;
  }
- supabase.from('members').select('id, name, memberNo').ilike('name', `%${keyword}%`)

인터랙션:
- 품목 변경 → 사이즈 Select 옵션 리셋 + 재조회
- 사이즈 변경 → 해당 품목+사이즈 사용가능 수량 표시
- 수량 > available → 입력 제한 + 경고 텍스트
- [취소] → 모달 닫기 (변경사항 없음)
- [지급 처리] → handleIssue() → 성공/실패 toast

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: Shirt, User
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader [지급 처리] 버튼 클릭
- 재고 테이블 행의 [지급] 버튼 클릭 (해당 품목/사이즈 pre-fill)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 검색 | `members` | `branch_id = ?`, ilike name/memberNo |
| 재고 확인 | `clothing` | `id = ?` → availableCount |

### 인터랙션 (User Actions)
1. 회원 검색 → AutoComplete 드롭다운 선택
2. 품목 선택 → 사이즈 옵션 갱신
3. 수량 입력 → 재고 초과 시 경고
4. [지급 처리] → DB 업데이트 + toast + 모달 닫기
5. [취소] → 모달 닫기

### 비즈니스 룰
- 사용가능 재고 0 → 지급 불가 (버튼 disabled + 경고 메시지)
- 일일 한도 초과 → `toast.error("일일 대여 한도를 초과했습니다.")`
- 스태프는 본인 지급만 처리 가능 (센터장/매니저는 전체)

### 에지 케이스
- 회원 미선택 → [지급 처리] disabled
- 수건은 사이즈 선택 없음 (단일 타입)

### 접근성 (A11y)
- Modal `role="dialog"`, `aria-modal="true"`, `aria-labelledby="modal-title"`
- 포커스 트랩: Modal 내부로 Tab 순환
