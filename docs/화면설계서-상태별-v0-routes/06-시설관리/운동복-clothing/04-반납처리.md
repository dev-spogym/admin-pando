## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-055 운동복 관리 — 상태: 반납처리 모달

파일: src/app/(facilities)/clothing/page.tsx

레이아웃:
- 기본 레이아웃 유지 (배경 dim overlay)
- <Modal title="운동복 반납" isOpen={returnModalOpen} onClose={() => setReturnModalOpen(false)}>
    <form onSubmit={handleReturn}>
      회원 검색 AutoComplete (이름/회원번호)
      대여 중인 운동복 목록 표시 (검색 후 자동 조회)
      반납 항목 체크박스 선택
      반납 후 상태 Select: 세탁 필요 / 즉시 사용가능
      메모 Textarea (선택)
      footer: [취소] [반납 처리]
    </form>
  </Modal>

컴포넌트 구조:
1. 배경: 02-기본-재고목록 화면 (dim 처리)
2. Modal — 반납 처리 폼
   - 회원 검색: AutoComplete → 회원 선택 시 해당 회원의 현재 대여 목록 자동 조회
   - 대여 목록: 체크박스 리스트 (품목 | 사이즈 | 지급일 | 대여일수)
   - 반납 후 상태: '세탁 필요' 선택 시 세탁 상태로, '즉시 사용가능' 시 available 복원
   - [반납 처리] → handleReturn() 실행

handleReturn():
  1. 선택된 반납 항목 유효성 검사 (1건 이상 필수)
  2. supabase.from('clothing_rentals').update({ status: nextStatus, returnedAt: new Date() })
  3. nextStatus === 'washing'
     → supabase.from('clothing').update({ rentedCount: prev - qty, washingCount: prev + qty })
  4. nextStatus === 'available'
     → supabase.from('clothing').update({ rentedCount: prev - qty })
  5. toast.success(`${memberName}님의 운동복 ${N}벌이 반납되었습니다.`)
  6. 모달 닫기 + 데이터 재조회

데이터:
- supabase.from('clothing_rentals')
    .select('*, clothing(*)')
    .eq('member_id', selectedMemberId)
    .eq('status', 'rented')
    .eq('branch_id', branchId)

인터랙션:
- 회원 선택 → 대여 목록 자동 로드
- 체크박스 전체 선택/해제
- 반납 후 상태 → 세탁 / 즉시 사용가능 라디오
- [반납 처리] → handleReturn() → toast → 모달 닫기

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatDate from '@/lib/format'
- lucide-react: RotateCcw, User, CheckSquare
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader [반납 처리] 버튼 클릭
- 재고 테이블 행의 [반납] 버튼 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 검색 | `members` | `branch_id = ?` |
| 대여 목록 | `clothing_rentals` | `member_id = ?`, `status = 'rented'` |

### 인터랙션 (User Actions)
1. 회원 검색 → 대여 목록 자동 로드
2. 반납 항목 체크박스 선택
3. 반납 후 상태 선택 (세탁/즉시)
4. [반납 처리] → DB 업데이트 + toast + 모달 닫기
5. [취소] → 모달 닫기

### 비즈니스 룰
- 대여 이력 없는 회원 선택 시 "대여 중인 운동복이 없습니다." 안내
- 세탁 선택 시 `washingCount` 증가, `rentedCount` 감소
- 즉시 사용가능 선택 시 `rentedCount`만 감소 (세탁 과정 없음)

### 에지 케이스
- 회원 미선택 → [반납 처리] disabled
- 반납 항목 미선택 → [반납 처리] disabled + 안내 텍스트

### 접근성 (A11y)
- 체크박스 `aria-label="{품목} {사이즈} 반납 선택"`
- Modal 포커스 트랩
