## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료, `rooms.length > 0`
- 카드 뷰 기본 진입 상태

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 룸 목록 | `rooms` | `branch_id = ?` ORDER BY type, name |
| 슬롯 정보 | `room_slots` or `rooms.slots` JSON | 당일 예약 현황 |

### 인터랙션 (User Actions)
1. 뷰 전환 → 카드/리스트 모드
2. 유형 필터 → 해당 유형만 표시
3. [상태 변경] → 운영중→점검중→고장 순환
4. [✏] → 수정 모달
5. [🗑] → 삭제 확인 (→ DLG-053-002)
6. [새 운동룸 등록] → 등록 모달 (→ DLG-053-001)

### 비즈니스 룰
- 상태 순환: `active → maintenance → broken → active`
- 슬롯바: 당일 예약 기준 (오전 6시~오후 10시, 2시간 단위 12슬롯)
- 삭제 시 연결된 수업 일정 확인 필요

### 에지 케이스
- 유형 필터 결과 0건 → "해당 유형의 룸이 없습니다." 인라인 메시지
- rooms.length === 0 → `05-빈상태`

### 접근성 (A11y)
- RoomCard `role="article"`, `aria-label="{name} {type}룸 {status}"`
- 슬롯바 `aria-label="예약 현황 {N}%"` (시각적 요소 설명)
