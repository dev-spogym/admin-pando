```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-051 사물함 배정 관리 — 상태: 일괄 배정

파일: src/app/(facilities)/locker/management/page.tsx

레이아웃:
- 기본 레이아웃 유지
- <Modal title="사물함 일괄 배정" isOpen={bulkAssignOpen} onClose={...} size="lg">
    <div className="space-y-4">
      {/* 회원 + 만료일 */}
      <div className="grid grid-cols-2 gap-3">
        <div>
          <label>회원 검색 *</label>
          <AutoComplete ... />
        </div>
        <div>
          <label>만료일 *</label>
          <Input type="date" ... />
        </div>
      </div>
      {/* 선택된 락커 목록 */}
      <div>
        <p className="text-sm font-medium mb-2">선택된 사물함 ({selectedLockers.length}개)</p>
        <div className="flex flex-wrap gap-1.5">
          {selectedLockers.map(id => {
            const l = lockers.find(l => l.id === id);
            return <span key={id} className="inline-flex items-center gap-1 px-2 py-0.5 bg-primary/10 text-primary rounded text-xs">
              {l?.number}번 <button onClick={() => deselectLocker(id)}><X size={10}/></button>
            </span>
          })}
        </div>
      </div>
      <p className="text-xs text-gray-400">그리드에서 빈 사물함을 클릭하여 추가/제거할 수 있습니다.</p>
    </div>
    <div slot="footer" className="flex justify-end gap-2">
      <Button variant="ghost" onClick={closeBulkModal}>취소</Button>
      <Button variant="primary" onClick={handleBulkAssign} disabled={!selectedMember || selectedLockers.length === 0}>
        {selectedLockers.length}개 사물함 배정
      </Button>
    </div>
  </Modal>

handleBulkAssign():
  if (!selectedMember || selectedLockers.length === 0) return;
  await Promise.all(selectedLockers.map(lockerId =>
    supabase.from('lockers').update({
      status: 'IN_USE', memberId: selectedMember.id, memberName: selectedMember.name,
      assignedAt: new Date().toISOString(), expiresAt: expiryDate
    }).eq('id', lockerId)
  ));
  toast.success(`${selectedLockers.length}개 사물함이 ${selectedMember.name}님께 배정되었습니다.`);
  setBulkAssignOpen(false); setSelectedLockers([]); setSelectedMember(null); fetchLockers();

데이터:
  - selectedLockers: number[] (선택된 locker id 목록)
  - selectedMember: { id, name } | null
  - expiryDate: string

인터랙션:
  - 모달 외부/배경 그리드에서 available 셀 클릭 → selectedLockers 토글
  - 선택된 락커 배지 [x] 클릭 → 해제
  - [N개 사물함 배정] → handleBulkAssign() → toast → 모달 닫기 + 재조회
  - [취소] → 모달 닫기 (selectedLockers 유지 또는 초기화)

사용 유틸:
  - useAuthStore, supabase, toast
  - lucide-react: Users, X, CheckSquare
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본`에서 [일괄 배정] 버튼 클릭 (PageHeader 또는 배정 영역)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 선택 락커 | `lockers` | AVAILABLE, `branch_id = ?` |
| 회원 검색 | `members` | `branch_id = ?` |

### 인터랙션 (User Actions)
1. 그리드 셀 클릭 → selectedLockers 토글
2. 모달 내 배지 [x] → 개별 해제
3. 회원 검색 → 선택
4. [N개 배정] → 일괄 처리 + toast

### 비즈니스 룰
- 일괄 배정은 매니저/센터장만 가능
- available 락커만 선택 가능 (in_use/abnormal 클릭 무시)
- 선택 0건 시 [배정] 버튼 disabled

### 에지 케이스
- 일부 락커 배정 실패 → "N개 중 M개 배정 성공, K개 실패" toast

### 접근성 (A11y)
- Modal `role="dialog"`, `aria-modal="true"`
- 선택된 락커 배지 `aria-label="{N}번 락커 선택 해제"`
