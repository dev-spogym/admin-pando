```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-050 락커 관리 — 상태: 배정 모달 (DLG-050-004)

파일: src/app/(facilities)/locker/page.tsx

레이아웃:
- 기본 레이아웃 유지 (dim overlay)
- <Modal title="{N}번 락커 회원 배정" isOpen={assignModalOpen} onClose={...}>
    <form onSubmit={handleAssign}>
      회원 검색 AutoComplete *
      만료일 Input(type="date") * (기본: 오늘 + 3개월)
      비밀번호 Input(type="text") (선택)
      메모 Textarea (선택)
      footer: [취소] [배정]
    </form>
  </Modal>

handleAssign():
  1. 유효성: 회원 필수, 만료일 오늘 이후
  2. supabase.from('lockers').update({
       status: 'IN_USE',
       memberId: selectedMember.id,
       memberName: selectedMember.name,
       assignedAt: new Date().toISOString(),
       expiresAt: expiryDate,
       password: password || null,
       memo: memo || null,
     }).eq('id', selectedLocker.id)
  3. toast.success(`${selectedLocker.number}번 락커가 ${selectedMember.name}님께 배정되었습니다.`)
  4. 모달 닫기 + 데이터 재조회

일괄 배정 모달 (DLG-050-006):
  <Modal title="일괄 배정" isOpen={bulkAssignModalOpen}>
    선택된 락커: {selectedLockers.length}개 표시 (번호 배지 리스트)
    회원 검색 AutoComplete *
    만료일 Input(type="date") *
    footer: [취소] [{N}개 락커 배정]
  </Modal>

  handleBulkAssign():
    Promise.all(selectedLockers.map(lockerId =>
      supabase.from('lockers').update({
        status: 'IN_USE', memberId, memberName, assignedAt, expiresAt
      }).eq('id', lockerId)
    ))
    toast.success(`${N}개 락커가 ${memberName}님께 배정되었습니다.`)
    setBulkMode(false); setSelectedLockers([]); 모달 닫기; 재조회

데이터:
  - selectedLocker: Locker (배정 대상)
  - selectedMember: { id: number; name: string } | null
  - expiryDate: string (기본값: format(addMonths(new Date(), 3), 'yyyy-MM-dd'))
  - members 검색: supabase.from('members').select('id, name, member_no').ilike('name', `%${q}%`)

인터랙션:
  - 회원 검색 AutoComplete → 드롭다운 선택
  - 만료일 입력 (date picker)
  - [배정] → handleAssign() → toast → 모달 닫기
  - [취소] → 모달 닫기 (상태 변경 없음)

사용 유틸:
  - useAuthStore from '@/stores/authStore'
  - supabase from '@/lib/supabase'
  - toast from 'sonner'
  - addMonths, format from 'date-fns'
  - lucide-react: User, Calendar, Lock, FileText
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `AVAILABLE` 락커 셀 호버 → 오버레이 [배정] 클릭
- `04-공실필터`에서 개별 셀 [배정] 클릭
- 일괄 배정: bulkMode에서 [N개 락커 배정] 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 검색 | `members` | `branch_id = ?`, ilike name |
| 선택 락커 | `lockers` | `id = ?` (AVAILABLE 확인) |

### 인터랙션 (User Actions)
1. 회원 검색 → AutoComplete 선택
2. 만료일 입력 (기본 3개월)
3. [배정] → DB 업데이트 + toast + 모달 닫기
4. [취소] → 모달 닫기

### 비즈니스 룰
- 회원 미선택 시 [배정] 버튼 disabled
- 만료일 오늘 이전 설정 불가 (date min 제한)
- 이미 배정된 회원에게 중복 배정 시 경고 toast

### 에지 케이스
- 배정 직전 다른 스태프가 동일 락커 배정 → 충돌 처리 (재조회 후 이미 배정됨 안내)
- 일괄 배정 중 일부 실패 → 성공 N건/실패 M건 toast

### 접근성 (A11y)
- Modal `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
- AutoComplete `aria-autocomplete="list"`, `aria-expanded`
- 포커스 트랩: Modal 내부 Tab 순환
