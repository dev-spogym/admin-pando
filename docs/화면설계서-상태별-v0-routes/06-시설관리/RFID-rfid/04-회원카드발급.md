  {/* 스캔 완료 상태 표시 */}

## 📝 디스크립션

### 사용 시점 (Trigger)
- `03-연결중`에서 카드 스캔 성공 (실제 또는 시뮬레이션)
- `cardNo` 자동 입력 완료 후 발급 확인 단계

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 정보 | `members` | 선택된 회원 |
| 카드 중복 확인 | `rfid_cards` | `card_no = cardNo` |

### 인터랙션 (User Actions)
1. 회원 변경 → AutoComplete 재검색
2. [다시 스캔] → `03-연결중` 복귀
3. [발급 완료] → DB insert + toast + 모달 닫기
4. [취소] → 모달 닫기

### 비즈니스 룰
- 카드번호 중복 시 이 단계 진입 전 차단 (중복 체크 선행)
- 회원 미선택 시 [발급 완료] disabled
- 발급 즉시 `status: 'active'` 설정

### 에지 케이스
- 선택 회원이 이미 활성 카드 보유 → "이미 활성 카드가 있습니다. 계속하시겠습니까?" 확인
- insert 실패 → toast.error + 재시도 옵션

### 접근성 (A11y)
- 발급 완료 영역 `role="status"` (스캔 성공 알림)
- [발급 완료] `aria-label="RFID 카드 발급 완료"`

### 연결 화면
- 이전: `03-연결중`
- 다음: `02-기본-기기목록` (발급 완료)
