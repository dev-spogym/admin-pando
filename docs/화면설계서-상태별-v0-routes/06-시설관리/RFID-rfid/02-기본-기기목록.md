## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료, `rfid_cards` 데이터 존재
- 기본 진입 상태

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 카드 목록 | `rfid_cards` | `branch_id = ?` ORDER BY registered_at DESC |
| 회원명/연락처 | `members` JOIN | - |

### 인터랙션 (User Actions)
1. [신규 등록] → `03-연결중` (스캔) 또는 `04-회원카드발급`
2. [이력] → 이력 조회 모달
3. [분실] → 상태 변경 + toast
4. [수정] → 수정 모달
5. [삭제] → 삭제 확인 + 처리
6. 검색/필터 → 목록 갱신

### 비즈니스 룰
- 활성 카드만 [분실] 버튼 표시 (분실/해제 카드는 버튼 숨김)
- 삭제 시 연결된 회원 관계 해제 안내 (DLG-052-003)

### 에지 케이스
- 검색 결과 0건 → "검색 결과가 없습니다." DataTable 빈 상태
- 분실 처리 중 네트워크 오류 → toast.error

### 접근성 (A11y)
- DataTable `role="table"`, 헤더 `scope="col"`
- 상태 배지 `aria-label="{상태명}"`
