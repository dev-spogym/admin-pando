```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-052 밴드/카드 관리 — 상태: RFID 스캔 대기 (등록/수정 모달 내)

파일: src/app/(facilities)/rfid/page.tsx

레이아웃:
- 기본 레이아웃 유지 (dim overlay)
- <Modal title="RFID 카드 등록" isOpen={registerModalOpen} onClose={...}>
    스캔 대기 UI + 수동 입력 폼
  </Modal>

모달 내부 구조:
  {/* 카드 번호 영역 */}
  <div className="space-y-3">
    <label className="text-sm font-medium">카드 번호 *</label>
    {!isScanning ? (
      <div className="flex gap-2">
        <Input value={cardNo} onChange={e => setCardNo(e.target.value)} placeholder="수동 입력 또는 스캔" className="flex-1" />
        <Button variant="outline" onClick={startScan}>
          <Wifi className="animate-pulse text-blue-500" /> 스캔 대기
        </Button>
      </div>
    ) : (
      /* 스캔 대기 UI */
      <div className="flex flex-col items-center justify-center h-32 border-2 border-dashed border-blue-300 rounded-lg bg-blue-50">
        <Wifi className="w-10 h-10 text-blue-400 animate-pulse" />
        <p className="mt-2 text-sm text-blue-600 font-medium">카드를 리더기에 가져다 주세요...</p>
        <p className="text-xs text-blue-400 mt-1">또는</p>
        <Button size="sm" variant="ghost" onClick={simulateScan} className="mt-1 text-blue-500">
          스캔 시뮬레이션 (테스트용)
        </Button>
        <Button size="sm" variant="ghost" onClick={cancelScan} className="mt-1 text-gray-400">
          취소
        </Button>
      </div>
    )}
  </div>

  {/* 사용자 유형 */}
  <div className="mt-4">
    <label className="text-sm font-medium">사용자 유형 *</label>
    <RadioGroup value={userType} onChange={setUserType} className="flex gap-4 mt-1">
      <RadioItem value="member" label="회원" />
      <RadioItem value="staff" label="직원" />
    </RadioGroup>
  </div>

  {/* 회원/직원 선택 */}
  <div className="mt-3">
    <label className="text-sm font-medium">{userType === 'member' ? '회원' : '직원'} 선택</label>
    <AutoComplete
      placeholder={`${userType === 'member' ? '회원명/번호' : '직원명'} 검색`}
      onSearch={searchMembers}
      onSelect={setSelectedUser}
    />
  </div>

  {/* 연결 사물함 */}
  <div className="mt-3">
    <label className="text-sm font-medium">연결 사물함 (선택)</label>
    <Input value={lockerNo} onChange={e => setLockerNo(e.target.value)} placeholder="사물함 번호 입력" />
  </div>

  <div slot="footer">
    <Button variant="ghost" onClick={closeModal}>취소</Button>
    <Button variant="primary" onClick={handleRegister} disabled={!cardNo || !selectedUser}>
      등록
    </Button>
  </div>

스캔 시뮬레이션:
  simulateScan():
    setIsScanning(true)
    setTimeout(() => {
      const simulatedCardNo = 'RF-' + Math.random().toString(36).substring(2, 10).toUpperCase()
      setCardNo(simulatedCardNo)
      setIsScanning(false)
      toast.success(`카드가 스캔되었습니다: ${simulatedCardNo}`)
    }, 1500)

handleRegister():
  supabase.from('rfid_cards').insert({
    cardNo, memberId: selectedUser.id, memberName: selectedUser.name,
    userType, status: 'active', issuedAt: new Date().toISOString(),
    lockerNo: lockerNo || null, branchId
  })
  toast.success("RFID 카드가 등록되었습니다.")
  closeModal(); fetchCards()

데이터:
  - isScanning: boolean
  - cardNo: string
  - userType: 'member' | 'staff'
  - selectedUser: { id, name } | null
  - lockerNo: string

인터랙션:
  - [스캔 대기] → isScanning = true → 스캔 UI 표시
  - [스캔 시뮬레이션] → 1.5초 후 cardNo 자동 생성 → 04-회원카드발급
  - [취소] → isScanning = false → 수동 입력 UI 복귀
  - [등록] → handleRegister() → toast → 모달 닫기

사용 유틸:
  - useAuthStore, supabase, toast
  - lucide-react: Wifi, CreditCard, User, Building
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- [신규 등록] 클릭 → 모달 오픈 → [스캔 대기] 버튼 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 검색 | `members` | `branch_id = ?` |
| 직원 검색 | `staff` | `branch_id = ?` (userType=staff 시) |

### 인터랙션 (User Actions)
1. [스캔 대기] → 스캔 UI 표시
2. 실제 스캔 또는 [스캔 시뮬레이션] → cardNo 자동 입력
3. 사용자 유형 선택 → 검색 대상 변경
4. 회원/직원 검색 선택
5. [등록] → DB insert + toast

### 비즈니스 룰
- 카드번호 중복 시 `toast.error("이미 등록된 카드 번호입니다.")`
- 스캔 시뮬레이션: `"RF-" + 8자리 랜덤 영숫자`

### 에지 케이스
- 스캔 타임아웃 (30초) → `05-에러` 상태 전환
- 중복 카드번호 → 에러 toast

### 접근성 (A11y)
- 스캔 UI `aria-live="polite"` (상태 변경 시 읽기)
- RadioGroup `role="radiogroup"`, `aria-label="사용자 유형"`

- 이전: `02-기본-기기목록`
- 다음: `04-회원카드발급` (스캔 완료) / `05-에러` (실패)
