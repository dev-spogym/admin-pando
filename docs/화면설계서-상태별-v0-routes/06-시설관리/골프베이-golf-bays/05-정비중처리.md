## 📝 디스크립션

### 사용 시점 (Trigger)
- `available` 타석에서 [🔧 점검] 버튼 클릭
- `maintenance` 타석에서 [점검 해제] 버튼 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 타석 상태 | `golf_bays` | `id = ?` |

### 인터랙션 (User Actions)
1. [🔧 점검] → 확인 다이얼로그 → `maintenance` 전환 + toast
2. [점검 해제] → 확인 다이얼로그 → `available` 전환 + toast
3. [취소] → 상태 변경 없이 닫기

### 비즈니스 룰
- 점검 중 타석은 사용 시작/예약 불가
- StatCard "대기" 카운트에서 점검 타석 제외
- 점검 처리는 매니저/센터장만 가능 (스태프는 버튼 disabled)

### 에지 케이스
- `in_use` 상태 타석은 점검 처리 불가 (종료 후 점검)
- 전체 타석 점검 시 경고 배너 표시

### 접근성 (A11y)
- 확인 다이얼로그 `role="alertdialog"`
- 점검 타석 카드 `aria-label="{N}번 타석 점검중"`
