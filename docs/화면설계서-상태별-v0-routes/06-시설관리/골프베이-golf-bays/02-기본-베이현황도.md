    <StatCard label="전체 타석" value={total} icon={Target} variant="default" />

2. 타석 그리드 (grid grid-cols-4 gap-4):
   각 BayCard:
   - 헤더: "{N}번 타석" + StatusBadge (대기/사용중/예약/점검)
   - 사용중: 회원명 + 남은 시간 카운트다운 (🕐 MM:SS)
   - 프로젝터 상태 표시 (📺 on/off)
   - 액션 버튼:
     available → [▶ 시작] [🔧 점검]
     in_use → [📺 토글] [↔ 이동] [⬜ 종료]
     maintenance → [점검 해제]
     reserved → [예약 확인]
3. 대기열 섹션 (mt-6):
   - 제목: "대기열 ({N}명)"
   - 리스트: 순번 | 회원명 | 대기 시작 | 예상 대기시간
   - [대기 취소] 버튼

실시간 타이머:
  useEffect(() => {
    const interval = setInterval(() => {
      setBays(prev => prev.map(bay => {
        if (bay.status === 'in_use' && bay.remainingMinutes > 0) {
          const next = bay.remainingMinutes - 1;
          if (next === 0) toast.warning(`⏰ ${bay.number}번 타석 시간 종료! (${bay.memberName})`);
          return { ...bay, remainingMinutes: next };
        }
        return bay;
      }));
    }, 60000);
    return () => clearInterval(interval);
  }, []);

handleStart(bayId):
  const name = window.prompt('회원명을 입력하세요:');
  if (!name) return;
  supabase.from('golf_bays').update({ status: 'in_use', memberName: name, startedAt: new Date(), remainingMinutes: 60 }).eq('id', bayId)
  toast.success(`${bay.number}번 타석 사용이 시작되었습니다.`)

handleEnd(bayId):
  supabase.from('golf_bays').update({ status: 'available', memberName: null, startedAt: null, remainingMinutes: null })
  // 대기열 첫 번째 자동 배정
  if (queue.length > 0) autoAssign(queue[0], bayId)
  toast.success(`${bay.number}번 타석이 종료되었습니다.`)

데이터:
- interface GolfBay {
    id: number; number: number; status: 'available' | 'in_use' | 'reserved' | 'maintenance';
    memberName: string | null; startedAt: string | null; remainingMinutes: number | null;
    projectorOn: boolean; branchId: number;
  }
- supabase.from('golf_bays').select('*').eq('branch_id', branchId).order('number')

인터랙션:
- [▶ 시작] → handleStart() → 회원명 입력 → 사용중 전환
- [⬜ 종료] → handleEnd() → 대기 전환 + 대기열 자동 배정
- [📺 토글] → 프로젝터 on/off 전환
- [↔ 이동] → 이동 모달 (→ 04-사용중상태)
- [🔧 점검] → 점검 확인 다이얼로그 (→ 05-정비중처리)
- [예약 확인] → 예약 확인 모달 (→ 03-예약확인)

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 정상 데이터 존재
- 기본 진입 상태

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 타석 목록 | `golf_bays` | `branch_id = ?` ORDER BY number |
| 대기열 | `golf_bay_queue` | `branch_id = ?`, status='waiting' ORDER BY created_at |

### 인터랙션 (User Actions)
1. [▶ 시작] → 회원명 입력 → 사용중 전환
2. [⬜ 종료] → 타석 초기화 + 대기열 자동 배정
3. [📺 on/off] → 프로젝터 상태 토글
4. [↔ 이동] → 이동 모달
5. [🔧 점검] → 점검 처리
6. [예약 확인] → 예약 상세 모달

### 비즈니스 룰
- 타이머: 1분 간격 setInterval, 컴포넌트 언마운트 시 clearInterval
- 시간 종료 시 자동으로 대기열 첫 번째 회원 배정
- 대기열 없으면 `available` 상태로만 전환

### 에지 케이스
- 모든 타석 사용중 → StatCard 대기 = 0, 안내 배너 표시
- 대기열 10명 초과 → 경고 배지 표시

### 접근성 (A11y)
- BayCard `role="article"`, `aria-label="{N}번 타석 {상태}"`
- 카운트다운 `aria-live="polite"` (1분마다 갱신)
