```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S003 POS 결제 — 상태: 결제 실패

파일: src/app/pos/payment/page.tsx

레이아웃:
- 결제 실패 시 인라인 에러 배너 (전체 화면 전환 없음):
  {paymentError && (
    <div className="rounded-lg border border-state-error/30 bg-red-50 p-4 flex items-start gap-3">
      <AlertCircle className="w-5 h-5 text-state-error flex-shrink-0 mt-0.5" />
      <div>
        <p className="text-sm font-semibold text-state-error">결제 처리 실패</p>
        <p className="text-xs text-red-600 mt-0.5">{paymentErrorMessage}</p>
        <button className="text-xs text-state-error underline mt-1" onClick={()=>setPaymentError(null)}>
          다시 시도
        </button>
      </div>
    </div>
  )}
- 결제 폼: 그대로 유지 (재시도 가능)
- [결제] 버튼: isProcessing=false 복귀 → 재클릭 가능

컴포넌트 구조:
1. paymentError: string | null — useState(null)
2. paymentErrorMessage: 에러 유형별 메시지
3. handlePayment() catch 블록:
   catch(error) {
     const msg = error?.message || '알 수 없는 오류가 발생했습니다.'
     setPaymentError(msg)
     toast.error(`결제 처리에 실패했습니다. ${msg}`)
     console.error('결제 실패:', error)
   } finally {
     setIsProcessing(false)
   }
4. 에러 메시지 닫기: setPaymentError(null) → 폼 유지
5. 재시도: 동일 결제 폼에서 다시 [결제] 클릭

에러 유형별 메시지:
- Supabase 오류: '데이터 저장에 실패했습니다. 잠시 후 다시 시도해 주세요.'
- 네트워크 오류: '네트워크 연결을 확인해 주세요.'
- 기타: '결제 처리 중 오류가 발생했습니다.'

데이터:
- paymentError: string | null
- isProcessing: false (실패 후 복귀)

인터랙션:
- 에러 배너 표시 → 결제 폼 유지 (전체 화면 전환 없음)
- [다시 시도] 클릭 → setPaymentError(null) → 폼 재활성화
- 결제 수단 변경 후 재시도 가능
- [← 장바구니로] → router.back() → 장바구니 복귀

사용 유틸:
- toast from 'sonner'
- lucide-react: AlertCircle
- useRouter from 'next/navigation'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- DB insert 실패 (네트워크/Supabase 오류)
- catch 블록 진입

### 필수 데이터
- `paymentError: string` — 에러 메시지
- `isProcessing: false` — 버튼 복귀

### 인터랙션 (User Actions)
1. 에러 배너 [다시 시도] → paymentError=null → 폼 재활성화
2. 결제 수단 변경 → 다른 수단으로 재시도
3. [← 장바구니로] → 장바구니로 복귀
4. toast.error 자동 표시

### 비즈니스 룰
- 실패 시 전체 화면 전환 없이 인라인 에러 표시 (결제 폼 유지)
- isProcessing=false 복귀로 즉시 재시도 가능
- console.error로 상세 오류 로깅
- 부분 insert 성공 후 실패 시 → 롤백 미구현 (향후 트랜잭션)

### 에지 케이스
- 연속 실패 → 동일 에러 배너 갱신 (중복 표시 방지)
- sessionStorage 유실 → `/pos`로 redirect 처리 (별도 guard)

### 접근성 (A11y)
- 에러 배너 `role="alert"` + `aria-live="assertive"`
- 에러 메시지 `aria-describedby` 로 버튼에 연결
