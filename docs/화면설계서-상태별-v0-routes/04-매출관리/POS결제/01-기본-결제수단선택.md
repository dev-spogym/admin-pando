```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S003 POS 결제 — 상태: 기본 (결제수단 선택)

파일: src/app/pos/payment/page.tsx

레이아웃:
- <AppLayout> 데스크톱 레이아웃
- <PageHeader title="결제 처리" subtitle="결제 수단을 선택하고 결제를 완료하세요."
    actions={[<Button variant="outline" onClick={()=>router.back()}>← 장바구니로</Button>]} />
- 2단 레이아웃 (flex gap-6):
  · 좌측 (flex-1): 주문 요약
  · 우측 (w-[400px]): 결제 수단 선택 + 결제 폼

좌측 주문 요약:
- <div className="bg-white rounded-xl border p-6 space-y-4">
    <h3 className="font-semibold text-[15px]">주문 내역</h3>
    구매자: buyer ? `${buyer.name} (${buyer.phone})` : '비회원'
    <hr />
    {cartItems.map(item => (
      <div key={item.cartId} className="flex justify-between text-sm">
        <span>{item.productName} × {item.quantity} ({item.priceType==='cash'?'현금':'카드'})</span>
        <span className="font-medium">{formatKRW(item.subtotal)}</span>
      </div>
    ))}
    <hr />
    {discount && <div className="flex justify-between text-sm text-state-error">
      <span>할인 ({discount.name})</span>
      <span>-{formatKRW(discountAmount)}</span>
    </div>}
    <div className="flex justify-between font-bold text-[16px]">
      <span>총 결제금액</span><span className="text-primary">{formatKRW(totalAmount)}</span>
    </div>
  </div>

우측 결제 수단 선택:
- <div className="bg-white rounded-xl border p-6 space-y-4">
    <h3 className="font-semibold text-[15px]">결제 수단</h3>
    <div className="grid grid-cols-2 gap-3">
      {['카드','현금','분할결제','마일리지'].map(method => (
        <button key={method}
          className={`p-4 border rounded-xl text-center hover:border-primary ${paymentMethod===method?'border-primary bg-primary/5':''}`}
          onClick={()=>setPaymentMethod(method)}>
          {methodIcon[method]}
          <p className="text-sm font-medium mt-1">{method}</p>
        </button>
      ))}
    </div>
    {paymentMethod && <PaymentForm method={paymentMethod} total={totalAmount} />}
  </div>

컴포넌트 구조:
1. useEffect on mount: sessionStorage에서 posCart, posBuyer, posDiscount, posTotal 복원
2. cartItems, buyer, discount, totalAmount 상태 설정
3. paymentMethod: '카드'|'현금'|'분할결제'|'마일리지'|null
4. 결제 수단 버튼 클릭 → paymentMethod 설정 → 해당 폼 표시

데이터:
- cartItems: sessionStorage('posCart') 파싱
- buyer: sessionStorage('posBuyer') 파싱
- discount: sessionStorage('posDiscount') 파싱
- totalAmount: sessionStorage('posTotal') 파싱

인터랙션:
- [카드] → 02-카드결제 폼
- [현금] → 03-현금결제 폼
- [분할결제] → 04-분할결제 폼
- [마일리지] → 마일리지 결제 폼
- [← 장바구니로] → router.back()

사용 유틸:
- useAuthStore from '@/stores/authStore'
- formatKRW / formatNumber from '@/lib/format'
- lucide-react: CreditCard, Wallet, Split, Gift
- useRouter from 'next/navigation'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-S002 POS에서 [결제하기] 클릭 후 이동

### 필수 데이터
| 블록 | 소스 | 내용 |
|------|------|------|
| 장바구니 | `sessionStorage('posCart')` | 상품 목록 |
| 구매자 | `sessionStorage('posBuyer')` | 회원 정보 |
| 할인 | `sessionStorage('posDiscount')` | 할인 정책 |
| 총액 | `sessionStorage('posTotal')` | 최종 결제금액 |

### 인터랙션 (User Actions)
1. 결제 수단 버튼(카드/현금/분할/마일리지) 클릭 → 해당 폼 표시
2. [← 장바구니로] → POS 장바구니로 복귀

### 비즈니스 룰
- sessionStorage 데이터 없으면 `/pos`로 redirect
- 마일리지 결제: buyer 있어야 활성화 (비회원 불가)

### 에지 케이스
- sessionStorage 파싱 실패 → toast.error + `/pos` redirect
- 직접 URL 접근 → sessionStorage 없음 → redirect

### 접근성 (A11y)
- 결제 수단 버튼 `role="radio"` + `aria-checked`
- 선택된 수단 `aria-label="{method} 결제 선택됨"`

### 연결 화면
- 이전: SCR-S002 POS (`05-결제전환`)
- 다음: `02-카드결제` / `03-현금결제` / `04-분할결제`
