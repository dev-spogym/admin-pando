## 📝 디스크립션

### 사용 시점 (Trigger)
- [카드] 결제 수단 버튼 클릭

### 필수 데이터
- `totalAmount` — 최종 결제금액
- `cartItems` — 상품 목록
- `buyer` — 구매자 (null 가능)
- `cardForm` — 카드사/번호/승인번호 (선택사항)

### 인터랙션 (User Actions)
1. 카드사/번호/승인번호 입력 (선택사항)
2. [카드 결제] 클릭 → DB insert → 성공/실패

### 비즈니스 룰
- 카드 정보(카드사/번호/승인번호)는 모두 선택사항
- `sales` 테이블에 각 cartItem별 row 삽입
- `paymentMethod='CARD'`, `card=totalAmount`, `cash=0`, `unpaid=0`
- 결제 성공 시 sessionStorage 클리어 + `/pos?success=true` redirect

### 에지 케이스
- DB insert 실패 → toast.error + isProcessing=false (모달 유지)
- 부분 삽입 후 실패 → 롤백 미구현 (향후 트랜잭션 처리)

### 접근성 (A11y)
- 카드번호 input `inputMode="numeric"` + `autocomplete="cc-number"`
- 결제 버튼 로딩 중 `aria-busy="true"`

### 연결 화면
- 이전: `01-기본-결제수단선택`
- 다음: `05-성공` / `06-실패`
