## 📝 디스크립션

### 사용 시점 (Trigger)
- DB insert 성공 + `paymentSuccess=true`

### 필수 데이터
- `paymentSuccess: true`
- 결제 결과: 수단, 금액, 구매자, 일시
- `changeAmount` — 현금 결제 시 거스름돈

### 인터랙션 (User Actions)
1. [영수증 출력] → window.print()
2. [다음 결제] → `/pos?success=true` → POS 초기화

### 비즈니스 룰
- 결제 성공 즉시 toast.success 표시
- 거스름돈 > 0 → 큰 글씨로 강조 표시 (현금 결제 시)
- 영수증 출력: 브라우저 print dialog

### 에지 케이스
- 영수증 출력 실패 → toast.error 후 유지 (결제 자체는 완료)

### 접근성 (A11y)
- 성공 아이콘 `aria-label="결제 완료"`
- `role="main"` + `aria-live="assertive"` (성공 알림)
