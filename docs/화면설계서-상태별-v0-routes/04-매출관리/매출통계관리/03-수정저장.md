3. isSaving: boolean
4. handleSave():
   setIsSaving(true)
   try {
     if (editItem) {
       await supabase.from('statistics_settings').update({...form, updatedAt: now, updatedBy: user.name}).eq('id', editItem.id)
       toast.success('통계 항목이 수정되었습니다.')
     } else {
       await supabase.from('statistics_settings').insert({...form, branchId, createdAt: now})
       toast.success('통계 항목이 추가되었습니다.')
     }
     setShowEditModal(false)
     fetchStatItems()
   } catch {
     toast.error('저장에 실패했습니다.')
   } finally { setIsSaving(false) }

## 📝 디스크립션

### 사용 시점 (Trigger)
- [수정] 버튼 클릭 → editItem 설정 + 모달 오픈
- [+ 항목 추가] 버튼 클릭 → editItem=null + 빈 폼

### 필수 데이터
- `editItem: StatManagementItem | null`
- `form: { name, type, period, isActive }`
- DB: `statistics_settings` 테이블 (또는 sales 집계)

### 인터랙션 (User Actions)
1. 항목명 입력 (required)
2. 유형 선택 (이용권/PT/상품/기타)
3. 기간 선택 (이번달/이번분기/올해/전체)
4. 활성 Toggle
5. [저장] → DB upsert + toast.success + 목록 갱신
6. [취소] / ESC → 변경 취소 + 모달 닫기

### 비즈니스 룰
- 항목명: 필수, 최대 50자, 중복 불가 (같은 branchId 내)
- 저장 성공 후 fetchStatItems() 재호출로 목록 갱신
- updatedAt, updatedBy 자동 기록

### 에지 케이스
- 저장 실패 → toast.error + 모달 유지 (닫히지 않음)
- 항목명 중복 → DB constraint error → toast.error('이미 존재하는 항목명입니다.')

### 접근성 (A11y)
- 모달 `role="dialog"` + `aria-modal="true"` + 포커스 트랩
- 폼 필드 `aria-required="true"` (필수 항목)
- [저장] 로딩 중 `aria-busy="true"`
