```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S005 매출 통계 관리 — 상태: 일괄 처리 모달

파일: src/app/sales/statistics-management/page.tsx

레이아웃:
- 기본 페이지 위에 <Modal> 오버레이
- Modal title="일괄 처리" size="sm"
- 모달 본문:
  <div className="space-y-4">
    <p className="text-sm text-content-secondary">
      선택한 {selectedIds.length}개 항목에 대해 처리할 작업을 선택하세요.
    </p>
    <div className="space-y-2">
      <label className="flex items-center gap-2 cursor-pointer p-3 rounded-lg border hover:bg-gray-50">
        <input type="radio" name="bulkAction" value="activate" /> 활성화
      </label>
      <label className="flex items-center gap-2 cursor-pointer p-3 rounded-lg border hover:bg-gray-50">
        <input type="radio" name="bulkAction" value="deactivate" /> 비활성화
      </label>
      <label className="flex items-center gap-2 cursor-pointer p-3 rounded-lg border border-state-error/30 text-state-error hover:bg-red-50">
        <input type="radio" name="bulkAction" value="delete" /> 삭제
      </label>
    </div>
  </div>
- 모달 푸터:
  [취소] variant="outline"
  [적용] variant="primary" disabled={!bulkActionType}

컴포넌트 구조:
1. showBulkModal: boolean
2. bulkActionType: 'activate' | 'deactivate' | 'delete' | null
3. isBulkProcessing: boolean
4. handleBulkApply():
   if (bulkActionType === 'delete') {
     // 추가 확인 dialog
     if (!confirm(`${selectedIds.length}개 항목을 삭제하시겠습니까?`)) return
     await supabase.from('statistics_settings').delete().in('id', selectedIds)
     toast.success(`${selectedIds.length}개 항목이 삭제되었습니다.`)
   } else {
     const isActive = bulkActionType === 'activate'
     await supabase.from('statistics_settings').update({ isActive }).in('id', selectedIds)
     toast.success(`${selectedIds.length}개 항목이 ${isActive?'활성화':'비활성화'}되었습니다.`)
   }
   setSelectedIds([])
   setShowBulkModal(false)
   fetchStatItems()

데이터:
- selectedIds: string[] — 체크박스 선택된 ID 목록
- bulkActionType: 라디오 선택값

인터랙션:
- 라디오 선택 → bulkActionType 설정
- [적용] → handleBulkApply() → toast + 목록 갱신 + selectedIds 초기화
- [취소] → 모달 닫기 (selectedIds 유지)

사용 유틸:
- Modal from '@/components/ui/Modal'
- supabase from '@/lib/supabase'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 체크박스로 1개 이상 선택 후 [일괄 처리] 버튼 클릭

### 필수 데이터
- `selectedIds: string[]` — 선택된 항목 ID 목록 (1개 이상)
- `bulkActionType` — 라디오 선택값

### 인터랙션 (User Actions)
1. 라디오 선택 (활성화/비활성화/삭제)
2. [적용] → DB 일괄 처리 → toast.success + 목록 갱신
3. 삭제 선택 시 → 추가 확인 단계
4. [취소] → 모달 닫기

### 비즈니스 룰
- 삭제: 추가 확인 (confirm 또는 별도 확인 모달)
- 처리 완료 후 selectedIds 초기화
- [일괄 처리] 버튼: selectedIds.length === 0 이면 disabled

### 에지 케이스
- 처리 실패 → toast.error + 모달 유지
- 삭제 중 취소 → 처리 중단

### 접근성 (A11y)
- 라디오 그룹 `role="radiogroup"` + `aria-label="일괄 처리 작업 선택"`
- 삭제 옵션 `aria-describedby="delete-warning"`
