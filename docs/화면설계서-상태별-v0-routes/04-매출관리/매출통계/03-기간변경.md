## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S004 매출 통계 — 상태: 기간 변경 (탭 전환 또는 직접 입력)

파일: src/app/sales/stats/page.tsx

레이아웃:
- 기본 레이아웃 동일
- 기간 탭 그룹: [최근 3개월] [최근 6개월] [올해] [직접입력]
  · 선택된 탭: bg-primary text-white rounded-lg
- 직접입력 선택 시: DateRangePicker 표시 (시작월~종료월 선택)
- 차트 + DataTable: 선택 기간으로 필터된 데이터 표시

컴포넌트 구조:
1. periodTab: '3month' | '6month' | 'year' | 'custom'
2. customRange: { start: string, end: string } — periodTab==='custom'일 때 활성
3. getPeriodRange(tab): 탭별 시작~종료 YYYY-MM 계산
   - '3month': 오늘 기준 -3개월 ~ 이번달
   - '6month': 오늘 기준 -6개월 ~ 이번달
   - 'year': 올해 1월 ~ 12월
   - 'custom': customRange.start ~ customRange.end
4. filteredStats: statsData.filter(d => d.month >= start && d.month <= end)
5. 차트 + StatCardGrid + DataTable: filteredStats 기반 재계산

데이터:
- statsData: MonthlyData[] — 전체 데이터 (초기 로드)
- filteredStats: MonthlyData[] — 기간 필터 적용
- summary: 필터된 데이터 기반 집계

인터랙션:
- 기간 탭 클릭 → periodTab 변경 → filteredStats 즉시 업데이트 (클라이언트)
- 직접입력 → DateRangePicker 팝오버 → customRange 설정 → 적용

사용 유틸:
- subMonths, startOfYear, endOfYear from 'date-fns'
- formatKRW from '@/lib/format'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 기간 탭 클릭 (최근 3개월 / 6개월 / 올해 / 직접입력)
- 직접입력 모드에서 날짜 범위 선택

### 필수 데이터
- `statsData: MonthlyData[]` — 전체 월별 데이터 (초기 1회 로드)
- `periodTab` — 선택된 기간 탭
- `customRange` — 직접입력 시 날짜 범위

### 인터랙션 (User Actions)
1. 기간 탭 클릭 → 즉시 클라이언트 필터 적용
2. [직접입력] → DateRangePicker 표시 → 월 선택 → 적용
3. 차트/테이블 → 선택 기간 데이터로 업데이트

### 비즈니스 룰
- 클라이언트 필터링 (DB 재호출 없음)
- 기간 단위: 월(YYYY-MM) 기준
- 최대 조회 기간: 2년 (성능 제한)

### 에지 케이스
- 선택 기간 내 데이터 없음 → 차트 빈 상태 + 안내 메시지
- 직접입력 시작월 > 종료월 → 선택 불가 처리

### 접근성 (A11y)
- 탭 그룹 `role="tablist"`, 각 탭 `role="tab"` + `aria-selected`
- DateRangePicker `aria-label="통계 기간 직접 입력"`

- 다음: `02-기본-차트` (기간 적용)
