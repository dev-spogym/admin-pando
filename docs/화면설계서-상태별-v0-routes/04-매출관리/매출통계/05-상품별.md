## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S004 매출 통계 — 상태: 상품별 매출 분석

파일: src/app/sales/stats/page.tsx

레이아웃:
- 기본 레이아웃 동일
- 분석 탭: [월별 추이(기본)] [상품별] — 상품별 탭 활성
- 상품별 뷰:
  · 좌측: PieChart (Recharts) — 상품별 매출 비율
  · 우측: DataTable (상품명 / 매출 / 건수 / 비율)
- PieChart 구성:
  <PieChart width={300} height={300}>
    <Pie data={productData} dataKey="total" nameKey="productName" cx="50%" cy="50%" outerRadius={120}>
      {productData.map((entry, i) => <Cell key={i} fill={COLORS[i % COLORS.length]} />)}
    </Pie>
    <Tooltip formatter={v=>formatKRW(v)} />
    <Legend />
  </PieChart>

컴포넌트 구조:
1. analysisTab: 'monthly' | 'product' (탭 전환)
2. productData: { productName, total, count, ratio }[]
   - salesData에서 클라이언트 group by productName
   - 환불 건 제외
   - ratio: total / sumAll * 100
   - 상위 10개 표시, 나머지 "기타" 합산
3. DataTable 컬럼:
   | 상품명 | 매출 합계 | 건수 | 비율(%) |
   - 매출 합계 내림차순 정렬
4. 상품명 클릭 → 매출 목록(SCR-S001)에 해당 상품명 필터 적용

데이터:
- salesData: SaleItem[] (SCR-S001과 동일 소스)
- productData: groupBy(salesData.filter(x=>x.status!=='환불'), 'productName')

인터랙션:
- [상품별] 탭 클릭 → analysisTab='product'
- PieChart 슬라이스 hover → 상품명 + 금액 tooltip
- DataTable 행 클릭 → moveToPage(970) with productName 필터

사용 유틸:
- recharts: PieChart, Pie, Cell, Tooltip, Legend
- formatKRW / formatNumber from '@/lib/format'
- moveToPage from '@/internal'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 분석 탭에서 [상품별] 탭 클릭

### 필수 데이터
- `salesData: SaleItem[]` — 전체 매출 데이터
- `productData: { productName, total, count, ratio }[]` — 클라이언트 집계

### 인터랙션 (User Actions)
1. [상품별] 탭 클릭 → PieChart + 상품별 DataTable 표시
2. PieChart 슬라이스 클릭 → 해당 상품 강조
3. DataTable 행 클릭 → 매출 목록 상품명 필터로 이동

### 비즈니스 룰
- 환불 건 제외
- 상위 10개 상품 표시, 나머지 "기타" 합산
- 비율: 해당 상품 매출 / 전체 매출 × 100 (소수점 1자리)

### 에지 케이스
- 상품 1개만 있으면 PieChart 100% 1개 슬라이스
- 상품명 없는 데이터 → "미분류"로 표시

### 접근성 (A11y)
- PieChart `role="img"` + `aria-label="상품별 매출 비율 차트"`
- DataTable로 동일 데이터 텍스트 제공

- 다음: `02-기본-차트` (탭 전환) / 매출 목록 (행 클릭)
