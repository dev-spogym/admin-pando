   try { ... setHasError(false) }
   catch(error) {
     console.error('매출 통계 로드 실패:', error)
     toast.error('매출 통계 데이터를 불러오지 못했습니다.')
     setHasError(true)
   } finally { setIsLoading(false) }
3. handleRetry(): setIsLoading(true) + setHasError(false) + fetchStatsData()
4. hasError 분기: 에러 UI 렌더링 (차트/테이블 숨김)

데이터:

## 📝 디스크립션

### 사용 시점 (Trigger)
- `fetchStatsData()` 실패 (네트워크 오류, Supabase 오류 등)
- `01-로딩` 중 catch 블록 진입

### 필수 데이터
- `hasError: boolean` — true
- `isLoading: boolean` — false

### 인터랙션 (User Actions)
1. [다시 시도] 버튼 클릭 → handleRetry() → `01-로딩` 재진입
2. 브라우저 새로고침 → 페이지 재진입

### 비즈니스 룰
- 에러 발생 시 toast.error도 함께 표시

- 에러 UI는 차트/테이블 영역을 완전히 대체

### 에지 케이스
- 재시도 후 또 실패 → 동일 에러 UI 재표시
