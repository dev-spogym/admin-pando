## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S002 POS — 상태: 결제 전환 ([결제하기] 클릭 시 처리)

파일: src/app/pos/page.tsx

레이아웃:
- [결제하기] 버튼 클릭 시 로딩 스피너 + disabled 처리
- 버튼: <Button variant="primary" className="w-full" onClick={handleCheckout} disabled={isCheckingOut || cart.length===0}>
    {isCheckingOut ? <><Loader2 className="animate-spin w-4 h-4 mr-2" />처리 중...</> : '결제하기'}
  </Button>

컴포넌트 구조:
1. isCheckingOut: boolean — useState(false)
2. handleCheckout():
   if (cart.length===0) return
   setIsCheckingOut(true)
   try {
     // sessionStorage에 장바구니 + 구매자 저장
     sessionStorage.setItem('posCart', JSON.stringify(
       cart.map(item => ({
         cartId: item.cartId,
         productId: item.product.id,
         productName: item.product.name,
         cashPrice: item.product.cashPrice,
         cardPrice: item.product.cardPrice,
         quantity: item.quantity,
         priceType: item.priceType,
         subtotal: (item.priceType==='cash'?item.product.cashPrice:item.product.cardPrice)*item.quantity
       }))
     ))
     sessionStorage.setItem('posBuyer', JSON.stringify(buyer))
     sessionStorage.setItem('posDiscount', JSON.stringify(selectedDiscount))
     sessionStorage.setItem('posTotal', JSON.stringify(totalAmount))
     moveToPage(983) // /pos/payment pageId
   } catch(e) {
     console.error(e)
     toast.error('결제 페이지 이동에 실패했습니다.')
     setIsCheckingOut(false)
   }

데이터:
- posCart: CartItem[] serialized → sessionStorage
- posBuyer: Member | null → sessionStorage
- posDiscount: DiscountPolicy | null → sessionStorage
- posTotal: number → sessionStorage

인터랙션:
- [결제하기] 클릭 → sessionStorage 저장 → moveToPage(983)
- 저장 실패 → toast.error + 버튼 복귀
- cart.length===0 → 버튼 disabled

사용 유틸:
- moveToPage from '@/internal'
- toast from 'sonner'
- lucide-react: Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- [결제하기] 버튼 클릭 (cart.length > 0)

### 필수 데이터
- `cart: CartItem[]` — 직렬화하여 sessionStorage 저장
- `buyer: Member | null`
- `selectedDiscount: DiscountPolicy | null`
- `totalAmount: number`

### 인터랙션 (User Actions)
1. [결제하기] 클릭 → 데이터 sessionStorage 저장 → `/pos/payment` 이동
2. 저장 실패 → toast.error + 버튼 복귀

### 비즈니스 룰
- sessionStorage keys: `posCart`, `posBuyer`, `posDiscount`, `posTotal`
- 결제 페이지에서 sessionStorage 읽어 복원
- cart 비어있으면 버튼 disabled (전환 불가)

### 에지 케이스
- sessionStorage 용량 초과 (대량 상품) → try-catch 처리
- 뒤로 가기 시 sessionStorage 유지 → 장바구니 복원 가능

### 접근성 (A11y)
- 버튼 로딩 중 `aria-busy="true"` + `aria-label="결제 처리 중"`
