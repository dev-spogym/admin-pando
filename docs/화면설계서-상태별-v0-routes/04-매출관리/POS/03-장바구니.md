## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S002 POS — 상태: 장바구니 (상품 1개 이상 담긴 상태)

파일: src/app/pos/page.tsx

레이아웃:
- 기본 레이아웃 동일
- 우측 장바구니 (w-[360px] sticky):
  · 헤더: 🛒 장바구니 [N] + [전체삭제] (cart.length>0일 때 표시)
  · 구매자: buyer 있으면 "{이름}님 | {전화번호}" + X 버튼, 없으면 [회원 검색] 버튼
  · 구분선
  · 장바구니 아이템 목록 (max-h-96 overflow-y-auto space-y-2):
    CartItem 카드:
    <div className="border rounded-lg p-3">
      <div className="flex justify-between items-start">
        <p className="text-[13px] font-semibold truncate flex-1">{item.product.name}</p>
        <button onClick={()=>handleRemoveItem(item.cartId)}><X className="w-4 h-4 text-gray-400" /></button>
      </div>
      <div className="flex items-center justify-between mt-2">
        <div className="flex items-center gap-2 border rounded-lg">
          <button className="px-2 py-1" onClick={()=>handleQuantityChange(item.cartId,-1)}>−</button>
          <span className="text-sm font-medium w-6 text-center">{item.quantity}</span>
          <button className="px-2 py-1" onClick={()=>handleQuantityChange(item.cartId,1)}>+</button>
        </div>
        <div className="flex gap-1">
          <button className={`px-2 py-1 text-xs rounded ${item.priceType==='cash'?'bg-primary text-white':'border'}`}
            onClick={()=>handlePriceType(item.cartId,'cash')}>현금</button>
          <button className={`px-2 py-1 text-xs rounded ${item.priceType==='card'?'bg-primary text-white':'border'}`}
            onClick={()=>handlePriceType(item.cartId,'card')}>카드</button>
        </div>
      </div>
      <p className="text-right text-[13px] font-semibold text-primary mt-1">
        {formatNumber((item.priceType==='cash'?item.product.cashPrice:item.product.cardPrice)*item.quantity)}원
      </p>
    </div>
  · 구분선
  · 총 합계 (text-right text-[16px] font-bold):
    총 합계: {formatNumber(totalAmount)}원
  · [결제하기] Button variant="primary" className="w-full" onClick={handleCheckout}

컴포넌트 구조:
1. cart: CartItem[] — 1개 이상
2. totalAmount: cart.reduce((sum,i)=>sum+(priceType=cash?cashPrice:cardPrice)*quantity, 0)
3. handleQuantityChange(cartId, delta): Math.max(1, prev+delta)
4. handleRemoveItem(cartId): cart에서 제거, 0개 되면 07-빈카트
5. handlePriceType(cartId, type): item.priceType 변경
6. handleCheckout():
   sessionStorage.setItem('posCart', JSON.stringify(cart))
   sessionStorage.setItem('posBuyer', JSON.stringify(buyer))
   moveToPage(983) // pos/payment

데이터:
- CartItem: { cartId: string, product: Product, quantity: number, priceType: 'cash'|'card' }
- totalAmount: number (현금가/카드가 × 수량 합산)

인터랙션:
- − 버튼 → 수량 -1 (최소 1)
- + 버튼 → 수량 +1
- [현금]/[카드] 토글 → priceType 변경 → 소계 업데이트
- X 버튼 → 아이템 제거
- [전체삭제] → setCart([])
- [결제하기] → sessionStorage 저장 + moveToPage(983)

사용 유틸:
- formatNumber from '@/lib/format'
- moveToPage from '@/internal'
- lucide-react: X, ShoppingCart
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 상품 카드 클릭 후 `cart.length > 0`

### 필수 데이터
- `cart: CartItem[]` — 장바구니 아이템
- `buyer: Member | null` — 구매자 (선택 사항)
- `totalAmount: number` — 합계

### 인터랙션 (User Actions)
1. 수량 − / + → 최소 1, 소계/총합 즉시 업데이트
2. 현금/카드 토글 → 가격 전환
3. X 클릭 → 아이템 제거 (0개면 `07-빈카트`)
4. [전체삭제] → `07-빈카트`
5. [결제하기] → sessionStorage 저장 + `/pos/payment` 이동

### 비즈니스 룰
- 수량 최소값: 1 (이하 감소 불가)
- sessionStorage key: `posCart`, `posBuyer`
- 결제 페이지에서 장바구니 데이터 복원

### 에지 케이스
- 장바구니 전체 삭제 → `07-빈카트`
- buyer 없이 결제 → 비회원 결제로 진행 (허용)

### 접근성 (A11y)
- 수량 버튼 `aria-label="수량 감소/증가"`
- X 버튼 `aria-label="{상품명} 제거"`

### 연결 화면
- 이전: `01-기본-상품선택`
- 다음: `04-할인적용` / `05-결제전환` / `07-빈카트`
