```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S002 POS — 상태: 할인 적용 (장바구니 내 할인 설정)

파일: src/app/pos/page.tsx

레이아웃:
- 장바구니 우측 패널 내 할인 섹션 추가 (구분선 아래):
  <div className="space-y-2 py-2">
    <p className="text-sm font-medium text-content-secondary">할인 적용</p>
    <Select
      placeholder="할인 정책 선택..."
      options={discountPolicies.map(p=>({label:`${p.name} (${p.discountRate}%)`,value:p.id}))}
      value={selectedDiscount?.id}
      onChange={handleSelectDiscount}
    />
    <div className="flex gap-2">
      <input type="text" placeholder="할인 코드 입력..." value={discountCode}
        onChange={e=>setDiscountCode(e.target.value)} className="flex-1 border rounded-lg px-2 py-1 text-sm" />
      <Button size="sm" variant="outline" onClick={handleApplyCode}>적용</Button>
    </div>
  </div>
- 총 합계 영역 (할인 적용 시):
  <div className="space-y-1">
    <div className="flex justify-between text-sm text-content-secondary">
      <span>소계</span><span>{formatNumber(subtotal)}원</span>
    </div>
    <div className="flex justify-between text-sm text-state-error">
      <span>할인 ({discountLabel})</span><span>-{formatNumber(discountAmount)}원</span>
    </div>
    <div className="flex justify-between font-bold text-[16px]">
      <span>총 합계</span><span>{formatNumber(totalAmount)}원</span>
    </div>
  </div>

컴포넌트 구조:
1. discountPolicies: DiscountPolicy[] — supabase.from('discount_policies').select('*').eq('branchId',getBranchId()).eq('isActive',true)
2. selectedDiscount: DiscountPolicy | null
3. discountCode: string
4. discountAmount: selectedDiscount ? subtotal * selectedDiscount.discountRate/100 : 0
5. discountLabel: selectedDiscount ? selectedDiscount.name : discountCode
6. handleSelectDiscount(policyId): setSelectedDiscount(policy)
7. handleApplyCode(): 코드 검증 → 매칭 정책 → setSelectedDiscount
8. totalAmount: subtotal - discountAmount

데이터:
- interface DiscountPolicy { id, name, discountRate, discountType:'rate'|'fixed', minAmount, code }
- 소스: supabase.from('discount_policies').select('*').eq('branchId',getBranchId()).eq('isActive',true)

인터랙션:
- 할인 정책 Select → discountAmount 계산 → 총합 업데이트
- 할인 코드 입력 + [적용] → 코드 매칭 → 정책 자동 선택
- 할인 X 제거 → selectedDiscount=null → 원가 복귀

사용 유틸:
- Select from '@/components/ui/Select'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatNumber from '@/lib/format'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 장바구니 내 할인 정책 선택 또는 할인 코드 입력

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 할인 정책 | `discount_policies` | `branchId=?`, `isActive=true` |

### 인터랙션 (User Actions)
1. 할인 정책 Select → discountAmount 계산
2. 할인 코드 입력 → [적용] → 코드 매칭
3. 할인 제거 (X) → selectedDiscount=null

### 비즈니스 룰
- 할인율(rate): `subtotal × discountRate / 100`
- 할인 고정액(fixed): `min(discountFixedAmount, subtotal)`
- 최소 결제금액(minAmount) 미달 시 적용 불가 → toast.warning
- 할인 후 음수 방지: max(0, subtotal - discountAmount)

### 에지 케이스
- 잘못된 코드 → "유효하지 않은 할인 코드입니다." toast.error
- 최소 결제금액 미달 → "최소 {N}원 이상 결제 시 적용 가능합니다."

### 접근성 (A11y)
- 할인 섹션 `aria-label="할인 적용"`
- 코드 input `aria-label="할인 코드 입력"`
