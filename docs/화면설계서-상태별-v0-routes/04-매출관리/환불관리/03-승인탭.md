```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S007 환불 관리 — 상태: 처리완료 탭 (승인/거절 완료 목록)

파일: src/app/refunds/page.tsx

레이아웃:
- 기본 레이아웃 동일
- TabNav: [대기(N)] [처리완료(N, active)]
- DataTable: completedRefunds (status='APPROVED' | status='REJECTED')

DataTable 컬럼 (처리완료 탭):
| No | 신청일 | 처리일 | 회원명 | 상품명 | 환불금액 | 처리결과 | 처리자 |
- 처리일: processedAt (YYYY-MM-DD)
- 처리결과: StatusBadge — APPROVED(success, "승인") / REJECTED(error, "거절")
- 처리자: processedBy
- 환불금액: formatKRW(refundAmount) — APPROVED이면 text-state-error, REJECTED이면 text-content-tertiary line-through

컴포넌트 구조:
1. activeTab: '대기' | '처리완료'
2. completedRefunds: refunds.filter(r => r.status === 'APPROVED' || r.status === 'REJECTED')
   .sort((a,b) => new Date(b.processedAt)-new Date(a.processedAt))
3. 행 클릭 → handleRowClick(row) → 05-환불상세모달 (조회 전용)
4. 처리완료 탭에서는 [승인]/[거절] 버튼 없음 (읽기 전용)

데이터:
- completedRefunds: RefundItem[] (status='APPROVED' or 'REJECTED')
- 처리일 내림차순 정렬

인터랙션:
- [대기] 탭 → 02-대기탭
- 행 클릭 → 05-환불상세모달 (조회 전용)
- 엑셀 다운로드 (optional)

사용 유틸:
- StatusBadge from '@/components/ui/StatusBadge'
- formatKRW / formatDate from '@/lib/format'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- [처리완료] 탭 클릭

### 필수 데이터
- `completedRefunds: RefundItem[]` — status='APPROVED' or 'REJECTED'

### 인터랙션 (User Actions)
1. [대기] 탭 → `02-대기탭`
2. 행 클릭 → `05-환불상세모달` (조회 전용)

### 비즈니스 룰
- 처리완료 탭은 읽기 전용 (승인/거절 버튼 없음)
- 거절된 건: 환불금액 line-through 표시
- 승인된 건: 환불금액 빨간색 표시

### 에지 케이스
- 처리완료 건 없음 → `06-빈상태` 또는 "처리된 환불이 없습니다."

### 접근성 (A11y)
- 처리결과 StatusBadge `aria-label="승인/거절"`

### 연결 화면
