## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S007 환불 관리 — 상태: 대기 탭 (승인 대기 중인 환불 목록)

파일: src/app/refunds/page.tsx

레이아웃:
- <AppLayout> 데스크톱 레이아웃
- <PageHeader title="환불 관리" subtitle="환불 신청을 검토하고 처리합니다." />
- <StatCardGrid cols={3}>
    <StatCard label="대기 건수" value={`${pendingCount}건`} variant="warning" icon={RotateCcw} />
    <StatCard label="처리완료" value={`${completedCount}건`} icon={CheckCircle} />
    <StatCard label="환불 총액" value={formatKRW(totalRefundAmount)} icon={DollarSign} />
  </StatCardGrid>
- <TabNav tabs={[`대기(${pendingCount})`, `처리완료(${completedCount})`]} activeTab='대기' />
- <SearchFilter> 검색어 + 날짜범위
- <DataTable columns={pendingColumns} data={pendingRefunds} onRowClick={handleRowClick} />

DataTable 컬럼 (대기 탭):
| No | 신청일 | 회원명 | 상품명 | 결제금액 | 환불요청액 | 사유 | 신청경로 | 액션 |
- 신청일: requestedAt (YYYY-MM-DD)
- 회원명: text-primary (memberName)
- 결제금액: formatKRW(salePrice) right
- 환불요청액: formatKRW(refundAmount) right font-bold text-state-error
- 사유: truncate max-w-[200px]
- 액션: [승인] Button variant="primary" size="sm" + [거절] Button variant="outline" size="sm" text-state-error

컴포넌트 구조:
1. refunds: RefundItem[] — 전체 로드
2. pendingRefunds: refunds.filter(r => r.status === 'PENDING')
3. completedRefunds: refunds.filter(r => r.status !== 'PENDING')
4. pendingCount, completedCount, totalRefundAmount 집계
5. handleApprove(id): setIsApproving(id) → supabase update status='APPROVED' → toast.success → refetch
6. handleReject(id): setShowRejectModal(true) + setRejectTarget(id) → 04-거절처리

데이터:
- interface RefundItem { id, memberId, memberName, productName, salePrice, refundAmount, reason, status, requestedAt, processedAt, processedBy, branchId }
- 소스: supabase.from('refunds').select('*').eq('branchId', getBranchId()).order('requestedAt', {ascending:false})
  (없으면: supabase.from('sales').select('*').eq('branchId',getBranchId()).eq('status','REFUNDED'))

인터랙션:
- [승인] 클릭 → handleApprove(id) → status='APPROVED' + toast.success
- [거절] 클릭 → 04-거절처리 모달
- 행 클릭 → 05-환불상세모달
- 탭 클릭 → 03-승인탭

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatKRW from '@/lib/format'
- useAuthStore from '@/stores/authStore'
- lucide-react: RotateCcw, CheckCircle, DollarSign
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 + 기본 탭 (대기)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 환불 대기 | `refunds` | `status='PENDING'`, `branchId=?` |
| 환불 완료 | `refunds` | `status!='PENDING'` |

### 인터랙션 (User Actions)
1. [승인] 클릭 → DB update status='APPROVED' + toast.success
2. [거절] 클릭 → `04-거절처리` 모달
3. 행 클릭 → `05-환불상세모달`
4. [처리완료] 탭 → `03-승인탭`

### 비즈니스 룰
- 승인 시: `sales.status='REFUNDED'` 동기화 필요
- 처리자: `processedBy = user.name`, `processedAt = now()`
- 권한: 센터장/매니저만 승인/거절 가능

### 에지 케이스
- 승인 실패 → toast.error + 상태 유지
- 대기 건 없음 → `06-빈상태`

### 접근성 (A11y)
- [승인] `aria-label="{회원명} 환불 승인"`
- [거절] `aria-label="{회원명} 환불 거절"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-승인탭` / `04-거절처리` / `05-환불상세모달`
