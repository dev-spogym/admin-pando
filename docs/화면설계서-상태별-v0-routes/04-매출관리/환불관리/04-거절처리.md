```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S007 환불 관리 — 상태: 거절 처리 모달

파일: src/app/refunds/page.tsx

레이아웃:
- <Modal title="환불 거절" size="sm">
    <div className="space-y-4">
      <div className="bg-gray-50 rounded-lg p-3 text-sm">
        <p className="font-medium">{rejectTarget?.memberName}</p>
        <p className="text-content-secondary">{rejectTarget?.productName}</p>
        <p className="text-state-error font-semibold">{formatKRW(rejectTarget?.refundAmount)}</p>
      </div>
      <div>
        <label className="text-sm font-medium">거절 사유 *</label>
        <textarea value={rejectReason} onChange={e=>setRejectReason(e.target.value)}
          placeholder="거절 사유를 입력하세요..." rows={3}
          className="w-full border rounded-lg px-3 py-2 text-sm resize-none"
          required maxLength={200} />
        <p className="text-xs text-content-tertiary text-right mt-1">{rejectReason.length}/200</p>
      </div>
    </div>
    <div className="flex gap-2 mt-4">
      <Button variant="outline" className="flex-1" onClick={()=>setShowRejectModal(false)}>취소</Button>
      <Button variant="primary" className="flex-1 bg-state-error" onClick={handleRejectConfirm}
        disabled={!rejectReason.trim() || isRejecting}>
        {isRejecting ? <Loader2 className="animate-spin w-4 h-4" /> : '거절 확인'}
      </Button>
    </div>
  </Modal>

컴포넌트 구조:
1. showRejectModal: boolean
2. rejectTarget: RefundItem | null
3. rejectReason: string (필수, 최대 200자)
4. isRejecting: boolean
5. handleRejectConfirm():
   if (!rejectReason.trim()) return
   setIsRejecting(true)
   try {
     await supabase.from('refunds').update({
       status: 'REJECTED',
       rejectReason: rejectReason,
       processedAt: new Date().toISOString(),
       processedBy: user.name
     }).eq('id', rejectTarget.id)
     toast.success('환불 신청이 거절되었습니다.')
     setShowRejectModal(false)
     setRejectReason('')
     fetchRefunds()
   } catch {
     toast.error('처리에 실패했습니다.')
   } finally { setIsRejecting(false) }

데이터:
- rejectTarget: RefundItem (거절할 환불 건)
- rejectReason: string (필수)
- DB update: status='REJECTED', rejectReason, processedAt, processedBy

인터랙션:
- 거절 사유 textarea 입력 (필수, 최대 200자)
- [거절 확인] → handleRejectConfirm → toast.success → 모달 닫기 + 목록 갱신
- [취소] / ESC → 모달 닫기

사용 유틸:
- Modal from '@/components/ui/Modal'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatKRW from '@/lib/format'
- lucide-react: Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 대기 탭에서 [거절] 버튼 클릭

### 필수 데이터
- `rejectTarget: RefundItem` — 거절할 환불 건
- `rejectReason: string` — 필수 입력

### 인터랙션 (User Actions)
1. 거절 사유 입력 (필수, 최대 200자)
2. [거절 확인] → DB update + toast.success
3. [취소] / ESC → 취소

### 비즈니스 룰
- 거절 사유 필수 (빈 값이면 버튼 disabled)
- processedAt, processedBy 자동 기록
- 거절 후 해당 건 처리완료 탭으로 이동

### 에지 케이스
- 거절 사유 공백만 입력 → trim() 후 검증
- 처리 실패 → toast.error + 모달 유지

### 접근성 (A11y)
- 모달 `role="dialog"` + 포커스 트랩
- textarea `aria-required="true"` + 글자수 `aria-describedby`
