```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S008 미수금 — 상태: 수금 처리 모달

파일: src/app/unpaid/page.tsx

레이아웃:
- <Modal title="수금 처리" size="md">
    <div className="space-y-4">
      [미수금 정보 요약] (bg-gray-50 rounded-lg p-4):
      - 회원명: memberName
      - 상품명: productName
      - 결제일: saleDate
      - 미수금액: formatKRW(unpaidAmount) (text-state-error font-bold text-[16px])
      - 연체일수: `${daysOverdue}일`
      [수금 정보 입력]:
      <div>
        <label className="text-sm font-medium">수금 금액 *</label>
        <input type="number" value={collectAmount} onChange={e=>setCollectAmount(Number(e.target.value))}
          max={unpaidAmount} min={1} className="w-full border rounded-lg px-3 py-2 text-right tabular-nums text-[15px]" />
        <div className="flex gap-2 mt-1">
          <button onClick={()=>setCollectAmount(unpaidAmount)} className="text-xs text-primary underline">전액 입력</button>
        </div>
      </div>
      <div>
        <label className="text-sm font-medium">결제 수단</label>
        <Select value={collectMethod} onChange={setCollectMethod}
          options={['카드','현금','계좌이체','마일리지']} />
      </div>
      <div>
        <label className="text-sm font-medium">비고</label>
        <input type="text" value={collectMemo} onChange={e=>setCollectMemo(e.target.value)}
          placeholder="수금 관련 메모..." className="w-full border rounded-lg px-3 py-2 text-sm" maxLength={100} />
      </div>
      {collectAmount < unpaidAmount && collectAmount > 0 && (
        <div className="text-xs text-yellow-600 bg-yellow-50 border border-yellow-200 rounded-lg p-2">
          부분 수금: 잔여 미수금 {formatKRW(unpaidAmount - collectAmount)}이 남습니다.
        </div>
      )}
    </div>
    <div className="flex gap-2 mt-4">
      <Button variant="outline" className="flex-1" onClick={()=>setShowCollectModal(false)}>취소</Button>
      <Button variant="primary" className="flex-1" onClick={handleCollectConfirm}
        disabled={!collectAmount || collectAmount<=0 || isCollecting}>
        {isCollecting ? <Loader2 className="animate-spin w-4 h-4" /> : '수금 완료'}
      </Button>
    </div>
  </Modal>

컴포넌트 구조:
1. collectAmount: number — 수금할 금액 (기본값: unpaidAmount)
2. collectMethod: string — 결제 수단 (기본: '카드')
3. collectMemo: string
4. isCollecting: boolean
5. handleCollectConfirm():
   setIsCollecting(true)
   try {
     const newUnpaid = selectedUnpaid.unpaidAmount - collectAmount
     await supabase.from('sales').update({
       unpaid: Math.max(0, newUnpaid),
       status: newUnpaid<=0 ? 'COMPLETED' : 'UNPAID',
       memo: collectMemo ? `${selectedUnpaid.memo || ''} [수금: ${collectMemo}]` : selectedUnpaid.memo
     }).eq('id', selectedUnpaid.id)
     // 수금 이력 기록 (별도 테이블 있는 경우)
     toast.success(`${formatKRW(collectAmount)} 수금 처리 완료`)
     setShowCollectModal(false)
     fetchUnpaidItems()
   } catch {
     toast.error('수금 처리에 실패했습니다.')
   } finally { setIsCollecting(false) }

데이터:
- selectedUnpaid: UnpaidItem
- collectAmount: 수금 금액 (1 ~ unpaidAmount)
- newUnpaid: unpaidAmount - collectAmount (0 이하면 완납)
- DB update: sales.unpaid, sales.status

인터랙션:
- [전액 입력] → collectAmount=unpaidAmount
- 부분 수금 가능 (collectAmount < unpaidAmount)
- [수금 완료] → handleCollectConfirm → toast.success → 목록 갱신

사용 유틸:
- Modal / Select from '@/components/ui'
- supabase from '@/lib/supabase'
- formatKRW from '@/lib/format'
- toast from 'sonner'
- lucide-react: Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- [수금처리] 버튼 클릭 또는 행 클릭

### 필수 데이터
- `selectedUnpaid: UnpaidItem`
- `collectAmount: number`
- `collectMethod: string`

### 인터랙션 (User Actions)
1. 수금 금액 입력 (1 ~ unpaidAmount)
2. [전액 입력] → collectAmount=unpaidAmount (전액 수금)
3. 결제 수단 선택
4. [수금 완료] → DB update + toast.success
5. [취소] → 모달 닫기

### 비즈니스 룰
- 부분 수금: collectAmount < unpaidAmount → 잔여 미수금 유지 (status='UNPAID')
- 전액 수금: collectAmount >= unpaidAmount → status='COMPLETED'
- 수금 후 목록에서 해당 건 제거 또는 미수금액 업데이트

### 에지 케이스
- collectAmount > unpaidAmount → max(0, newUnpaid) 처리
- 처리 실패 → toast.error + 모달 유지

### 접근성 (A11y)
- `role="dialog"` + 포커스 트랩
- 수금 금액 input `aria-label="수금 금액"` + `aria-describedby="remaining-unpaid"`

### 연결 화면
- 이전: `02-기본` / `03-회원별그룹`
- 다음: `02-기본` (완료)
