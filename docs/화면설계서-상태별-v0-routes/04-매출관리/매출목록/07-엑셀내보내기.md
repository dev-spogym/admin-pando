## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader 우측 [엑셀] 버튼 클릭
- 현재 filteredData (필터 적용된 데이터) 기준으로 내보내기

### 필수 데이터
- `filteredData: SaleItem[]` — 현재 표시 중인 데이터
- `exportColumns` — 23개 컬럼 헤더 + 키 정의

### 인터랙션 (User Actions)
1. [엑셀] 클릭 → 스피너 표시 + 버튼 비활성
2. 다운로드 완료 → toast.success(`${N}건 엑셀 다운로드 완료`)
3. 다운로드 실패 → toast.error(`엑셀 다운로드에 실패했습니다.`)
4. 다운로드 후 → 버튼 정상 복귀

### 비즈니스 룰
- 내보내기 대상: 현재 필터된 `filteredData` (전체가 아님)
- 파일명: `매출내역.xlsx` (또는 `.csv`)
- 컬럼 순서: No, 구매일, 매출유형, 유형, 상품명, 구매자, 담당자, 수량, 정가, 판매가, 할인, 결제수단, 현금, 카드, 마일리지, 미수금, 카드사, 카드번호, 승인번호, 서비스일수, 서비스횟수, 상태, 메모 (23개)
- 권한 제한: 트레이너는 엑셀 다운로드 버튼 숨김 처리 가능 (hasPermission 체크)

### 에지 케이스
- filteredData 0건 → 버튼 disabled 또는 "데이터가 없습니다" toast.warning
- 대용량(1000건 이상) → 처리 시간 증가, isExporting 스피너 유지

### 접근성 (A11y)
- 버튼 `aria-label="엑셀 다운로드"` + 로딩 중 `aria-busy="true"`
- toast 메시지 `aria-live="assertive"`
