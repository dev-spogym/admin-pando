Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-S001 매출 목록 — 상태: 결제수단 필터 + 유형/상태 multiSelect 필터 적용

- DataTable: 필터 적용된 filteredData 표시

   - type multiSelect: ['이용권', 'PT', '상품', '기타']

3. filteredData 계산:
   - type 필터: filterValues.type.length>0 ? item.type이 포함되는지 : 전체

4. TAB-004 결제수단별 집계:
   const paymentData = Object.entries(groupBy(filteredData.filter(x=>x.status!=='환불'), 'paymentTool'))
     .map(([paymentTool, items]) => ({ paymentTool, count: items.length, total: sum(items, 'salePrice') }))
     .sort((a,b) => b.total - a.total)

데이터:

- TAB-004 활성 시: paymentData (결제수단별 집계)
- PAYMENT_KO: { CARD:'카드', CASH:'현금', TRANSFER:'계좌이체', MILEAGE:'마일리지' }

인터랙션:

## 📝 디스크립션

### 사용 시점 (Trigger)
- SearchFilter의 유형 또는 상태 multiSelect에서 항목 선택
- TAB-004(결제수단별) 탭 클릭

### 필수 데이터
- `filterValues.type: string[]` — 선택된 유형 목록
- `filterValues.status: string[]` — 선택된 상태 목록
- `filteredData` — 모든 필터 조건 AND 적용 결과

### 인터랙션 (User Actions)
1. 유형 선택(이용권/PT/상품/기타) → 해당 유형만 표시
2. 상태 선택(완료/환불/미납) → 해당 상태만 표시
3. TAB-004 클릭 → 결제수단별 집계(카드/현금/계좌이체/마일리지) 표시
4. 필터 X 클릭 → 개별 필터 해제
5. 전체 초기화 → 모든 필터 리셋

### 비즈니스 룰
- 유형 + 상태 + 기간 + 검색어 필터는 AND 조건으로 결합
- 집계 탭(TAB-002~005)에서 환불 건은 합계에서 제외
- 결제수단별 탭 행 클릭 이벤트 없음 (집계 탭)

### 에지 케이스
- 필터 적용 후 결과 0건 → `06-빈상태`
- 모든 필터 해제 시 → `02-기본`으로 복귀

### 접근성 (A11y)
- multiSelect 드롭다운 `role="listbox"`, 옵션 `role="option"` + `aria-selected`
- 선택된 칩 `aria-label="{유형} 필터 제거"` 포함 X 버튼
