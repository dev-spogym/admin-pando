```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-DISC 할인 설정 — 상태: 수정 모달 (기존 정책 편집)

파일: src/app/discount-settings/page.tsx

레이아웃:
- <Modal title="할인 정책 수정" size="md">
    (03-할인정책추가와 동일한 폼 구조, 기존 값 pre-fill)
    <div className="flex gap-2 mt-4">
      <Button variant="outline" className="flex-1" onClick={()=>setShowEditModal(false)}>취소</Button>
      <Button variant="primary" className="flex-1" onClick={handleUpdate} disabled={isUpdating}>
        {isUpdating ? <Loader2 className="animate-spin w-4 h-4 mr-2" /> : null}수정 저장
      </Button>
    </div>
  </Modal>

컴포넌트 구조:
1. showEditModal: boolean, editPolicy: DiscountPolicy | null
2. form: DiscountPolicyForm — editPolicy 값으로 초기화 (useEffect)
3. isUpdating: boolean
4. handleUpdate():
   setIsUpdating(true)
   try {
     await supabase.from('discount_policies').update({
       name: form.name,
       discountType: form.discountType,
       discountRate: form.discountRate,
       discountAmount: form.discountAmount,
       minAmount: form.minAmount,
       maxDiscount: form.maxDiscount,
       code: form.code?.toUpperCase(),
       startDate: form.startDate,
       endDate: form.endDate,
       isActive: form.isActive,
       updatedAt: new Date().toISOString()
     }).eq('id', editPolicy.id)
     toast.success('할인 정책이 수정되었습니다.')
     setShowEditModal(false)
     fetchPolicies()
   } catch {
     toast.error('수정에 실패했습니다.')
   } finally { setIsUpdating(false) }

데이터:
- editPolicy: DiscountPolicy — [수정] 버튼 클릭 시 setEditPolicy(policy)
- form: 기존 값으로 초기화
- DB UPDATE: discount_policies.eq('id', editPolicy.id)

인터랙션:
- [수정] 클릭 → editPolicy 설정 + showEditModal=true + form 초기화
- 필드 수정
- [수정 저장] → handleUpdate → toast.success → 모달 닫기 + 목록 갱신
- [취소] / ESC → 변경 취소

사용 유틸:
- Modal, Toggle from '@/components/ui'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable 행의 [수정] 버튼 클릭

### 필수 데이터
- `editPolicy: DiscountPolicy` — 수정할 정책
- `form` — editPolicy 값으로 초기화

### 인터랙션 (User Actions)
1. 폼 필드 수정
2. [수정 저장] → UPDATE + toast.success
3. [취소] / ESC → 취소

### 비즈니스 룰
- 이미 적용된(usageCount>0) 정책 수정 시 경고 없음 (즉시 반영)
- 코드는 대문자 자동 변환
- updatedAt 자동 기록

### 에지 케이스
- 코드 중복 (다른 정책과) → DB constraint error → toast.error
- 업데이트 실패 → 모달 유지

### 접근성 (A11y)
- 모달 `aria-label="할인 정책 수정"` + 포커스 트랩

### 연결 화면
- 이전: `02-기본`
- 다음: `02-기본` (수정 완료)
