## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-DISC 할인 설정 — 상태: 기본 (할인 정책 목록)

파일: src/app/discount-settings/page.tsx

레이아웃:
- <AppLayout> 데스크톱 레이아웃
- <PageHeader title="할인 설정" subtitle="할인 정책을 관리하고 상품에 적용합니다."
    actions={[<Button variant="primary" onClick={handleAddNew}>+ 할인 정책 추가</Button>]} />
- <StatCardGrid cols={3}>
    <StatCard label="활성 정책" value={`${activePolicies.length}개`} icon={Tag} />
    <StatCard label="총 적용 건수" value={`${totalUsageCount}건`} icon={BarChart2} />
    <StatCard label="평균 할인율" value={`${avgDiscountRate.toFixed(1)}%`} icon={Percent} />
  </StatCardGrid>
- <SearchFilter> 검색어 + 유형(rate/fixed) + 활성상태
- <DataTable columns={columns} data={filteredPolicies} />

DataTable 컬럼:
| 정책명 | 유형 | 할인값 | 최소금액 | 코드 | 기간 | 활성 | 적용 건수 | 액션 |
- 정책명: text-primary font-medium
- 유형: StatusBadge — rate(blue,"율 할인") / fixed(green,"정액 할인")
- 할인값: rate → `${discountRate}%` / fixed → formatKRW(discountAmount)
- 최소금액: formatKRW(minAmount) 또는 '-'
- 코드: code ? `<code className="bg-gray-100 px-2 py-0.5 rounded text-xs">{code}</code>` : '-'
- 기간: startDate~endDate 또는 '상시'
- 활성: Toggle switch (isActive)
- 적용 건수: usageCount center
- 액션: [수정] [삭제] [상품 연결] 버튼

컴포넌트 구조:
1. discountPolicies: DiscountPolicy[] — fetch 결과
2. activePolicies: discountPolicies.filter(p=>p.isActive)
3. totalUsageCount: sum(discountPolicies.map(p=>p.usageCount))
4. avgDiscountRate: avg(discountPolicies.filter(p=>p.discountType==='rate').map(p=>p.discountRate))
5. handleToggleActive(id, isActive): supabase.update({isActive})
6. handleEdit(policy): setEditPolicy(policy) + setShowEditModal(true)
7. handleDelete(policy): setDeleteTarget(policy) + setShowDeleteModal(true)
8. handleLinkProducts(policy): setLinkTarget(policy) + setShowLinkModal(true)

데이터:
- supabase.from('discount_policies').select('*').eq('branchId', getBranchId()).order('createdAt',{ascending:false})

인터랙션:
- Toggle → isActive 즉시 업데이트
- [+ 할인 정책 추가] → 03-할인정책추가
- [수정] → 04-수정
- [삭제] → 05-삭제확인
- [상품 연결] → 06-적용상품연결

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatKRW from '@/lib/format'
- lucide-react: Tag, Percent, BarChart2, Edit2, Trash2, Link
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 할인 정책 | `discount_policies` | `branchId=?` |

### 인터랙션 (User Actions)
1. Toggle → isActive 즉시 업데이트 (optimistic)
2. [+ 추가] → `03-할인정책추가`
3. [수정] → `04-수정`
4. [삭제] → `05-삭제확인`
5. [상품 연결] → `06-적용상품연결`

### 비즈니스 룰
- 활성 정책만 POS 할인 선택에 표시
- 기간(startDate~endDate) 지난 정책: isActive 자동 비활성 권장
- 코드: POS에서 직접 입력으로 적용

### 에지 케이스
- 정책 없음 → "할인 정책이 없습니다." 빈 상태 (별도 파일 없음, DataTable 내 emptyMessage)
- Toggle 실패 → rollback + toast.error
