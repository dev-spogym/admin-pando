```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-DISC 할인 설정 — 상태: 삭제 확인 모달

파일: src/app/discount-settings/page.tsx

레이아웃:
- <Modal title="할인 정책 삭제" size="sm">
    <div className="space-y-4">
      <div className="flex items-start gap-3">
        <div className="w-10 h-10 rounded-full bg-state-error/10 flex items-center justify-center flex-shrink-0">
          <AlertTriangle className="w-5 h-5 text-state-error" />
        </div>
        <div>
          <p className="font-semibold text-content-primary">"{deleteTarget?.name}"을 삭제하시겠습니까?</p>
          <p className="text-sm text-content-secondary mt-1">
            삭제 후 복구할 수 없습니다.
            {deleteTarget?.usageCount > 0 && (
              <span className="block text-state-error mt-1">
                ⚠️ 이 정책은 {deleteTarget.usageCount}건에 적용된 이력이 있습니다.
              </span>
            )}
          </p>
        </div>
      </div>
    </div>
    <div className="flex gap-2 mt-4">
      <Button variant="outline" className="flex-1" onClick={()=>setShowDeleteModal(false)}>취소</Button>
      <Button className="flex-1 bg-state-error text-white hover:bg-red-600" onClick={handleDeleteConfirm}
        disabled={isDeleting}>
        {isDeleting ? <Loader2 className="animate-spin w-4 h-4 mr-2" /> : <Trash2 className="w-4 h-4 mr-2" />}
        삭제
      </Button>
    </div>
  </Modal>

컴포넌트 구조:
1. showDeleteModal: boolean, deleteTarget: DiscountPolicy | null
2. isDeleting: boolean
3. handleDeleteConfirm():
   setIsDeleting(true)
   try {
     await supabase.from('discount_policies').delete().eq('id', deleteTarget.id)
     toast.success(`"${deleteTarget.name}" 할인 정책이 삭제되었습니다.`)
     setShowDeleteModal(false)
     setDeleteTarget(null)
     fetchPolicies()
   } catch {
     toast.error('삭제에 실패했습니다.')
   } finally { setIsDeleting(false) }

데이터:
- deleteTarget: DiscountPolicy — [삭제] 버튼 클릭 시 설정
- DB DELETE: discount_policies.eq('id', deleteTarget.id)

인터랙션:
- [삭제] → handleDeleteConfirm → toast.success → 목록 갱신
- [취소] / ESC → 모달 닫기

사용 유틸:
- Modal from '@/components/ui/Modal'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: AlertTriangle, Trash2, Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable [삭제] 버튼 클릭

### 필수 데이터
- `deleteTarget: DiscountPolicy` — 삭제할 정책

### 인터랙션 (User Actions)
1. [삭제] → DB delete + toast.success + 목록 갱신
2. [취소] / ESC → 취소

### 비즈니스 룰
- usageCount > 0 → 이력 있음 경고 표시 (삭제는 허용)
- 삭제 후 복구 불가 → 경고 문구 필수
- 연결된 상품(applicableProducts) 있으면 연결 자동 해제

### 에지 케이스
- 삭제 실패 → toast.error + 모달 유지
- 이미 다른 세션에서 삭제됨 → 오류 처리

### 접근성 (A11y)
- `role="alertdialog"` + `aria-modal="true"`
- [삭제] 버튼 `aria-label="할인 정책 삭제 확인"`

### 연결 화면
- 이전: `02-기본`
- 다음: `02-기본` (삭제 완료 또는 취소)
