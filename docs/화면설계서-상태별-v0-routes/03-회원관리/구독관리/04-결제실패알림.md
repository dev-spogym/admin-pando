     · 실패: 실패 토스트 + 모달 유지

## 📝 디스크립션

### 사용 시점 (Trigger)
- 결제 실패(PAYMENT_FAILED) 상태 구독 행 클릭
- 상단 통계 카드 "결제 실패" 클릭으로 해당 행 강조 후 모달 오픈

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 실패 구독 상세 | `subscriptions` | status='PAYMENT_FAILED', memberId |
| 재시도 이력 | `subscriptions` | retryCount, lastPaymentAttempt |

### 인터랙션 (User Actions)
1. [재결제 시도] → 결제 API 재호출
2. [알림 발송] → SMS/푸시 발송
3. [해지 처리] → 해지 모달로 전환

### 비즈니스 룰
- 재시도 최대 3회 (retryCount >= 3이면 버튼 disabled + "최대 재시도 횟수 초과" 안내)
- D+7 이후 자동 해지 예정 경고 표시
- 알림 발송: 중복 발송 방지 (24시간 내 동일 회원 재발송 제한)

### 에지 케이스
- 재결제 성공 → status='ACTIVE' 업데이트 + 성공 토스트 + 모달 닫기
- 재결제 실패 → retryCount+1 + 에러 메시지 유지
- 결제 API 연동 없는 경우 → [재결제 시도] 버튼 비활성 + "수동 결제 필요" 안내

### 접근성 (A11y)
- 모달 `role="alertdialog"` (긴급 처리 필요)
- [재결제 시도] 로딩 중 `aria-busy="true"` + `aria-label="결제 처리 중"`
- 실패 사유 `aria-label="결제 실패 사유: {reason}"`

- 이전: `02-기본-목록`
- 다음: `02-기본-목록` (처리 완료) / `03-해지처리` (해지 선택)
