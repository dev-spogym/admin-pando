## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-SUB 구독 관리 — 상태: 해지 처리 모달

파일: src/app/subscription/page.tsx (showCancelModal === true)

레이아웃:
- <Modal> 오버레이 (max-w-md)

컴포넌트 구조:
1. [Modal 헤더]
   - 제목: "구독 해지 처리"
   - X 닫기 버튼

2. [대상 구독 정보] (bg-gray-50 rounded-lg p-3)
   - 회원명 / 구독 상품 / 결제 주기 / 다음 결제일

3. [해지 방식] RadioGroup*
   ○ 즉시 해지 — "현재 구독이 즉시 종료됩니다."
   ○ 다음 결제일에 해지 — "다음 결제일({date})까지 서비스 유지 후 자동 해지됩니다." (권장)

4. [해지 사유] Select*
   - 본인 요청 / 결제 문제 / 센터 변경 / 운동 중단 / 기타

5. [해지 메모] Textarea (optional, rows=2)

6. [환불 안내] (연간 구독인 경우만)
   - bg-yellow-50 border border-yellow-200 p-3
   - "연간 구독 중도 해지 시 잔여 기간 환불 정책을 확인하세요."
   - [환불 정책 보기] 링크

7. [버튼 행]
   - [취소] → 모달 닫기
   - [해지 처리] (destructive) → UPDATE

데이터:
- cancelType: 'IMMEDIATE' | 'NEXT_CYCLE'
- cancelReason: string
- cancelMemo: string
- UPDATE: supabase.from('subscriptions').update({ status: cancelType === 'IMMEDIATE' ? 'CANCELLED' : 'CANCEL_PENDING', cancelledAt: now(), cancelReason }).eq('id', sub.id)

인터랙션:
- [해지 처리] → UPDATE → 성공 토스트 + 모달 닫기 + 목록 갱신

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatDate from '@/lib/format'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 활성 구독 행 [해지 요청] 클릭

### 필수 데이터
- selectedSubscription: 해지 대상 구독 전체 데이터

### 인터랙션 (User Actions)
1. 해지 방식 선택 (즉시/다음 결제일)
2. 해지 사유 선택
3. [해지 처리] → UPDATE

### 비즈니스 룰
- 즉시 해지: status='CANCELLED', cancelledAt=now()
- 다음 결제일 해지: status='CANCEL_PENDING'
- 연간 구독 중도 해지: 환불 정책 안내 필수 표시

### 에지 케이스
- UPDATE 실패 → 에러 토스트 + 모달 유지
- CANCEL_PENDING 상태에서 재해지 시도 → "이미 해지 요청된 구독입니다" 안내

### 접근성 (A11y)
- 모달 `role="dialog"` + 포커스 트랩
- RadioGroup `role="radiogroup"` + `aria-label="해지 방식 선택"`
