## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-SUB 구독 관리 — 상태: 기본 목록

파일: src/app/subscription/page.tsx

레이아웃:
- <AppLayout> 데스크톱
- <PageHeader title="구독 관리" subtitle="회원 구독 현황 및 결제 상태를 관리합니다." />
- StatCardGrid 4열 + 필터 + DataTable

컴포넌트 구조:
1. [StatCardGrid 4열]
   - 활성 구독: status='ACTIVE' COUNT
   - 이번달 결제 예정: nextPaymentAt이 이번달 내 COUNT
   - 결제 실패: status='PAYMENT_FAILED' COUNT (red variant)
   - 해지 요청: status='CANCEL_PENDING' COUNT (yellow variant)

2. [필터 행]
   - 회원 검색: Input
   - 구독 상태: Select (전체 / 활성 / 결제실패 / 해지대기 / 해지완료)
   - 결제 주기: Select (전체 / 월간 / 3개월 / 연간)

3. [DataTable]
   컬럼: 회원명 | 연락처 | 구독 상품 | 결제 주기 | 다음 결제일 | 결제금액 | 상태 | 액션
   - 다음 결제일: YYYY-MM-DD (D-7 이하 → text-red-600)
   - 결제금액: formatNumber + "원"
   - 상태: <StatusBadge>
     · ACTIVE → "활성" / success
     · PAYMENT_FAILED → "결제실패" / error
     · CANCEL_PENDING → "해지대기" / warning
     · CANCELLED → "해지완료" / default
   - 액션:
     · ACTIVE: [해지 요청] 버튼
     · PAYMENT_FAILED: [재결제 시도] + [해지] 버튼
     · CANCEL_PENDING: [해지 확정] + [해지 취소] 버튼

4. [Pagination]

데이터:
- interface Subscription { id, memberId, productName, billingCycle, nextPaymentAt, amount, status, members: { name, phone } }
- supabase.from('subscriptions').select('*, members(name, phone)').eq('branchId', branchId)
    .order('nextPaymentAt', { ascending: true })

인터랙션:
- [해지 요청] → `03-해지처리` 모달
- 결제실패 행 → `04-결제실패알림` 모달
- 필터 변경 → 재조회

사용 유틸:
- supabase from '@/lib/supabase'
- formatNumber, formatDate from '@/lib/format'
- differenceInDays from 'date-fns'
- lucide-react: CreditCard, AlertCircle, XCircle
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 구독 데이터 1건 이상 존재

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 구독 목록 | `subscriptions` | branchId, JOIN members |
| 통계 카드 | `subscriptions` | status별 COUNT |

### 인터랙션 (User Actions)
1. [해지 요청] → `03-해지처리` 모달
2. 결제실패 [재결제 시도] → 재결제 API 호출
3. 결제실패 행 클릭 → `04-결제실패알림` 상세
4. 필터 변경 → 상태별 필터링

### 비즈니스 룰
- 결제 실패 D+3 이상 → 행 bg-red-50 강조
- 해지 대기는 다음 결제일까지 서비스 유지
- 연간 구독은 중도 해지 시 환불 정책 안내 팝업

### 에지 케이스
- 구독 0건 → `05-빈상태`
- 결제실패 건이 있을 때 → 상단 경고 배너 노출

### 접근성 (A11y)
- 결제실패 행 `aria-label="결제 실패 회원 행, 주의 필요"`
- 상태 배지 색상 외 텍스트로도 구분

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-해지처리` / `04-결제실패알림`
