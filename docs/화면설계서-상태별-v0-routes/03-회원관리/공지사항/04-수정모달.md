## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-NOTICE 공지사항 — 상태: 수정 모달

파일: src/app/notices/page.tsx (showEditModal === true && selectedNotice !== null)

레이아웃:
- <Modal> 오버레이 (max-w-2xl)
- 03-작성모달과 동일한 폼 구조 + 기존 데이터 pre-fill

컴포넌트 구조:
1. [Modal 헤더]
   - 제목: "공지사항 수정"
   - X 닫기 버튼

2. [공지 수정 폼] — 03-작성모달과 동일, 기존값 pre-fill
   - 제목: selectedNotice.title
   - 내용: selectedNotice.content
   - 게시 여부: selectedNotice.isPublished
   - 고정 공지: selectedNotice.isPinned

3. [최종 수정일 표시]
   - "최종 수정: {formatDate(updatedAt)}" (text-xs text-gray-400)

4. [버튼 행]
   - [취소] → 모달 닫기
   - [저장] (primary) → UPDATE

데이터:
- selectedNotice: Notice (기존 공지 전체 데이터)
- UPDATE: supabase.from('notices').update({ title, content, isPublished, isPinned, updatedAt: now() }).eq('id', selectedNotice.id)

인터랙션:
- 폼 로드 시 기존 데이터 자동 fill (useForm defaultValues)
- [저장] → UPDATE → 성공 토스트 + 목록 갱신 + 모달 닫기

사용 유틸:
- useForm with defaultValues: selectedNotice
- supabase from '@/lib/supabase'
- formatDate from '@/lib/format'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 공지 카드 [수정] 버튼 클릭

### 필수 데이터
- selectedNotice: 선택된 공지 전체 필드 (pre-fill용)

### 인터랙션 (User Actions)
1. 기존 데이터 수정
2. [저장] → UPDATE
3. [취소] → 모달 닫기 (변경사항 버림)

### 비즈니스 룰
- 타인 작성 공지 수정: owner만 가능 (manager는 본인 작성만)
- updatedAt 자동 갱신

### 에지 케이스
- 수정 중 다른 사람이 삭제한 경우 → UPDATE 실패 → "이미 삭제된 공지입니다" 에러

### 접근성 (A11y)
- 모달 포커스 트랩 + ESC 닫기
- 첫 포커스: 제목 Input
