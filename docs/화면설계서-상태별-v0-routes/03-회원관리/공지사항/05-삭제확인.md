## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-NOTICE 공지사항 — 상태: 삭제 확인 모달

파일: src/app/notices/page.tsx (showDeleteConfirm === true)

레이아웃:
- <Modal> 소형 확인 모달 (max-w-sm)

컴포넌트 구조:
1. [삭제 확인 Modal]
   - 아이콘: Trash2 (w-10 h-10 text-red-500 mx-auto)
   - 제목: "공지사항을 삭제하시겠습니까?"
   - 내용: '"{selectedNotice.title}" 공지를 삭제하면 복구할 수 없습니다.'
   - 게시 중인 경우 경고: "현재 게시 중인 공지입니다. 삭제하면 회원 앱에서 즉시 제거됩니다." (text-red-600)
   - 버튼 행:
     · [취소] (outline) → 모달 닫기
     · [삭제] (destructive/red) → DELETE

데이터:
- selectedNotice: Notice (제목, isPublished 상태)
- isDeleting: boolean
- DELETE: supabase.from('notices').delete().eq('id', selectedNotice.id)

인터랙션:
- [삭제] → isDeleting=true → DELETE → 성공 토스트 + 목록에서 제거 + 모달 닫기
- [취소] → 모달 닫기

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: Trash2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 공지 카드 [삭제] 버튼 클릭

### 필수 데이터
- selectedNotice.id (DELETE 대상)
- selectedNotice.isPublished (게시 중 경고)

### 인터랙션 (User Actions)
1. [삭제] → DELETE → 목록 제거 + 성공 토스트
2. [취소] → 모달 닫기

### 비즈니스 룰
- 소프트 딜리트 없음 (물리적 삭제)
- 게시 중 공지 삭제 시 추가 경고 문구 표시
- manager는 본인 작성 공지만 삭제 가능

### 에지 케이스
- DELETE 실패 → 에러 토스트 + 모달 유지
- 삭제 중 [취소] 불가 (isDeleting=true 시 버튼 disabled)

### 접근성 (A11y)
- 모달 `role="alertdialog"` + 포커스 트랩
- 첫 포커스: [취소] 버튼 (파괴적 작업이므로 기본 포커스를 안전한 쪽으로)
- [삭제] `aria-label="공지사항 영구 삭제 확인"`
