## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-NOTICE 공지사항 — 상태: 작성 모달

파일: src/app/notices/page.tsx (showCreateModal === true)

레이아웃:
- <Modal> 오버레이 (max-w-2xl)

컴포넌트 구조:
1. [Modal 헤더]
   - 제목: "공지사항 작성"
   - X 닫기 버튼

2. [공지 작성 폼]
   - 제목*: Input (placeholder="공지 제목을 입력하세요", maxLength=100)
   - 내용*: Textarea (rows=10, placeholder="공지 내용을 입력하세요", 마크다운 간단 지원)
   - 게시 여부: Toggle Switch (isPublished, 기본: true)
     · ON: "게시 중 — 회원 앱에 표시됩니다" (green)
     · OFF: "미게시 — 회원 앱에 표시되지 않습니다" (gray)
   - 고정 공지: Checkbox (isPinned, "이 공지를 목록 상단에 고정합니다")
   - 글자수: N/2000자 카운터

3. [버튼 행]
   - [취소] → 모달 닫기
   - [저장] (primary) → INSERT

데이터:
- interface NoticeInput { title, content, isPublished, isPinned }
- INSERT: supabase.from('notices').insert({ title, content, isPublished, isPinned, branchId, staffId: user.id, createdAt: now() })

인터랙션:
- 게시 여부 Toggle → 상태 즉시 반영
- [저장] → INSERT → 성공 토스트 + 목록 최상단에 추가 + 모달 닫기

사용 유틸:
- supabase from '@/lib/supabase'
- useForm from 'react-hook-form'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- [공지 작성] 버튼 클릭 (owner/manager)

### 필수 데이터
- INSERT: `notices` 테이블

### 인터랙션 (User Actions)
1. 제목/내용 입력
2. 게시 여부 / 고정 설정
3. [저장] → INSERT → 목록 갱신

### 비즈니스 룰
- fc/staff는 버튼 자체 비노출
- 고정 공지는 여러 개 가능 (제한 없음)
- 게시 중 공지만 회원 앱에 표시

### 에지 케이스
- 제목 100자 초과 → 검증 에러
- 내용 2000자 초과 → 검증 에러

### 접근성 (A11y)
- 모달 포커스 트랩 (첫 포커스: 제목 Input)
- Toggle `role="switch"` + `aria-checked`
