## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-ATT 출석 관리 — 상태: 수동 출석 모달

파일: src/app/attendance/page.tsx (showManualModal === true)

레이아웃:
- <Modal> 오버레이 (max-w-md)

컴포넌트 구조:
1. [Modal 헤더]
   - 제목: "수동 출석 처리"
   - X 닫기 버튼

2. [회원 검색 필드]
   - Input (placeholder="회원명 또는 연락처 검색")
   - debounce 300ms → supabase 실시간 검색
   - 검색 결과 드롭다운 리스트 (최대 8개)
     · 각 항목: 아바타 + 이름 + 연락처 + 상태 배지
     · 이미 오늘 출석한 회원: 회색 처리 + "출석 완료" 표시 (선택 불가)
   - 선택된 회원: 카드 형태로 확정 표시

3. [출석 정보]
   - 출석일: DatePicker (기본: 오늘, 미래 불가)
   - 출석 시각: TimePicker (기본: 현재 시각, HH:mm)
   - 출석 방법: Select (수동 / QR / RFID / 키오스크) — 기본: 수동

4. [버튼 행]
   - [취소] → 모달 닫기
   - [출석 처리] (primary, 회원 선택 시 활성) → INSERT

데이터:
- searchQuery: string
- selectedMember: Member | null
- attendanceForm: { memberId, date, time, method: 'MANUAL' }
- INSERT: supabase.from('attendance').insert({ memberId, branchId, date, attendedAt, method: 'MANUAL', staffId: user.id })
- 중복 체크: 기존 attendance 조회

인터랙션:
- 회원 검색 → 드롭다운 → 선택 → selectedMember 설정
- [출석 처리] → 중복 체크 → INSERT → 성공 토스트 + 리스트 갱신 + 모달 닫기

사용 유틸:
- supabase from '@/lib/supabase'
- useDebounce from '@/hooks/useDebounce'
- format from 'date-fns'
- toast from 'sonner'
- lucide-react: Search, X, CheckCircle
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- [수동 출석] 버튼 클릭
- QR/RFID 없이 직접 출석 처리 필요 시

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 검색 | `members` | name/phone ILIKE, branchId, status=ACTIVE |
| 중복 확인 | `attendance` | memberId, date |
| INSERT | `attendance` | 새 출석 기록 |

### 인터랙션 (User Actions)
1. 회원명/연락처 검색 → 드롭다운 선택
2. 날짜/시간/방법 설정
3. [출석 처리] → INSERT → 모달 닫기 + 리스트 갱신

### 비즈니스 룰
- 이미 출석한 회원 재선택 불가 (중복 방지)
- 과거 날짜 수동 출석: owner/manager만 가능
- fc/staff는 오늘 날짜만 수동 출석 가능

### 에지 케이스
- 만료된 회원 선택 → 경고 배지 + 선택은 허용 (수동이므로)
- INSERT 실패 → 에러 토스트 + 모달 유지

### 접근성 (A11y)
- 모달 `role="dialog"` + 포커스 트랩 (첫 포커스: 검색 Input)
- 검색 드롭다운 `role="listbox"` + `role="option"`
- ESC → 모달 닫기

### 연결 화면
- 이전: `02-기본-오늘`
- 다음: `02-기본-오늘` (저장 또는 취소)
