## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-ATT 출석 관리 — 상태: 검색 결과

파일: src/app/attendance/page.tsx (searchQuery !== '' 조건)

레이아웃:
- 02-기본-오늘과 동일
- 검색창에 검색어 표시 + X 버튼
- DataTable: 검색 결과만 표시

컴포넌트 구조:
1. [검색창] — 검색어 입력됨
   - 우측 X 버튼 → 검색 초기화
   - "'{searchQuery}' 검색 결과 N건" 텍스트

2. [DataTable] — 해당 날짜 출석 중 검색어에 해당하는 회원만
   - 회원명 또는 연락처로 필터링
   - 검색어 하이라이트 (mark 태그 또는 bold)

3. [결과 없음] — 검색 결과 0건 → `06-빈상태` 처리

데이터:
- searchQuery: string (debounce 300ms)
- filteredAttendance: attendance.filter(a =>
    a.members.name.includes(searchQuery) ||
    a.members.phone.includes(searchQuery))
- 클라이언트 사이드 필터링 (이미 로드된 당일 출석 데이터 기준)

인터랙션:
- 검색어 입력 → 실시간 필터링 (클라이언트)
- X 클릭 → 검색 초기화 → 전체 목록 복귀
- 결과 0건 → `06-빈상태` 인라인

사용 유틸:
- useDebounce from '@/hooks/useDebounce'
- 검색어 하이라이트: 문자열 replace + <mark> 태그
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 출석 관리 페이지 검색창에 회원명/연락처 입력 시

### 필수 데이터
- 이미 로드된 당일 출석 데이터 (클라이언트 필터링)

### 인터랙션 (User Actions)
1. 검색어 입력 → 실시간 필터링
2. X 버튼 → 검색 초기화
3. 결과 없음 → `06-빈상태`

### 비즈니스 룰
- 클라이언트 사이드 필터링 (API 재호출 없음)
- 대소문자/공백 무시 (trim + toLowerCase)

### 에지 케이스
- 검색 결과 0건 → `06-빈상태` 케이스 B (검색 결과 없음)

### 접근성 (A11y)
- 결과 건수 `aria-live="polite"`
- 검색어 하이라이트 `<mark aria-label="검색어 강조">`

### 연결 화면
- 이전: `02-기본-오늘`
- 다음: `02-기본-오늘` (초기화) / `06-빈상태`
