## 📝 디스크립션

### 사용 시점 (Trigger)
- 회원 상세 페이지 [수정] 버튼 클릭
- URL 직접 접근 `/members/edit?id={memberId}`

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 정보 | `members` | id = memberId (pre-fill) |
| 담당FC 목록 | `staff` | branchId, 활성 직원 |

### 인터랙션 (User Actions)
1. 폼 로드 → 기존 데이터 자동 입력
2. 입력 시작 → `02-입력중`
3. [취소] → 이전 페이지 (회원 상세)

### 비즈니스 룰
- fc는 회원 수정 접근 불가 → `/forbidden`
- 다른 지점 회원 수정 불가 (branchId 검증)

### 에지 케이스
- memberId 없거나 잘못된 id → `/members` 리다이렉트
- 삭제된(softDelete) 회원 → 수정 불가 + 에러 메시지

### 접근성 (A11y)
- 필수 필드 `aria-required="true"` + 레이블 `<label htmlFor>`
- 폼 `role="form"` + `aria-label="회원 정보 수정 폼"`

### 연결 화면
- 이전: 회원 상세 `02-기본정보`
- 다음: `02-입력중` / `03-저장중`
