## 📝 디스크립션

### 사용 시점 (Trigger)
- [저장] 클릭 후 클라이언트 검증 실패 (Zod schema 불충족)
- Supabase 서버 에러 응답 (중복 연락처, RLS 등)

### 필수 데이터
- `formState.errors` 객체 (필드별 에러 메시지)

### 인터랙션 (User Actions)
1. 에러 필드 수정 → 해당 인라인 에러 제거
2. 에러 배너 필드명 클릭 → 해당 필드 포커스
3. 모든 에러 해결 → [저장] 활성화

### 비즈니스 룰
- 연락처 중복 시 서버 에러 → "이미 등록된 연락처입니다" 인라인 에러
- 필수 필드: 이름, 성별, 연락처

### 에지 케이스
- 서버 에러 (500) → 상단 에러 배너에 "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요" 표시

### 접근성 (A11y)
- 에러 배너 `role="alert"` + `aria-live="assertive"`
- 에러 필드 `aria-invalid="true"` + `aria-describedby="{fieldId}-error"`
- Submit 시 첫 에러 필드로 포커스 이동

### 연결 화면
- 이전: `02-입력중`
- 다음: `02-입력중` (수정 중) / `03-저장중` (재시도 성공)
