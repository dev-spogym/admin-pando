## 📝 디스크립션

### 사용 시점 (Trigger)
- [이관 신청] 버튼 클릭 후 최종 확인 모달 오픈

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 이관 실행 | `members` | UPDATE branchId |
| 이관 로그 | `member_transfer_logs` | INSERT |

### 인터랙션 (User Actions)
1. [이관 확인] → members.branchId UPDATE + transfer_log INSERT → `05-완료`
2. [취소] → 모달 닫기

### 비즈니스 룰
- 이관 로그 반드시 기록 (member_transfer_logs)
- 이관 후 현재 지점에서 접근 불가 (branchId 변경)
- 이용권 초기화 선택 시: membershipType/Start/Expiry null 처리

### 에지 케이스
- API 호출 중 [취소] 클릭 불가 (isTransferring=true 시 버튼 disabled)
- 이관 실패 → 에러 토스트 + 모달 유지

### 접근성 (A11y)
- 모달 `role="alertdialog"` (파괴적 작업) + 포커스 트랩
- ESC로 모달 닫기
- [이관 확인] `aria-label="회원 이관 최종 확인"`

### 연결 화면
- 이전: `03-대상브랜치선택`
- 다음: `05-완료` / `02-기본`
