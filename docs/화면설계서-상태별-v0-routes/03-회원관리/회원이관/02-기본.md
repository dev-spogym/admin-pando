## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 회원 정보 + 브랜치 목록 수신

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 이관 대상 회원 | `members` | id = memberId |
| 브랜치 목록 | `branches` | id != currentBranchId |

### 인터랙션 (User Actions)
1. 대상 지점 선택 → 지점 정보 표시
2. 이관 사유 선택 → 필수 충족
3. [이관 신청] → 확인 모달 (`04-이관확인`)
4. [취소] → 이전 페이지

### 비즈니스 룰
- 현재 지점과 동일한 지점으로 이관 불가
- 이관 후 이용권 처리 방식 명확히 선택 필요
- 이관은 owner/primary만 가능

### 에지 케이스
- 브랜치 목록이 1개(현재 지점만) → "이관 가능한 다른 지점이 없습니다" 안내

### 접근성 (A11y)
- RadioGroup `role="radiogroup"` + `aria-label="이관 후 이용권 처리"`
- 읽기 전용 Input `aria-readonly="true"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-대상브랜치선택` / `04-이관확인`
