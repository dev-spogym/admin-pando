## 📝 디스크립션

### 사용 시점 (Trigger)
- 회원 등록 폼에서 첫 번째 필드 입력 시작 (`isDirty === true`)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 연락처 중복 확인 | `members` | phone = 입력값, branchId |

### 인터랙션 (User Actions)
1. 연락처 입력 완료 → 자동 중복 체크
2. [SMS 인증] 클릭 → 인증 플로우
3. 모든 필수 필드 충족 → [등록] 활성화

### 비즈니스 룰
- 동일 브랜치 내 연락처 중복 불가
- SMS 인증은 필수 (설정에 따라 bypass 가능)
- 이용권 만료일 < 시작일 조합 불가

### 에지 케이스
- 연락처 중복 시 기존 회원 링크 제공
- 중복 체크 API 오류 → 체크 건너뜀 + 서버에서 최종 검증

### 접근성 (A11y)
- 중복 확인 결과 `aria-live="polite"`
- [SMS 인증] 버튼 `aria-label="SMS 인증 발송"`

### 연결 화면
- 이전: `01-기본`
- 다음: `03-SMS인증` / `04-저장중`
