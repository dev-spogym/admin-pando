## 📝 디스크립션

### 사용 시점 (Trigger)
- [등록] 클릭 후 클라이언트 Zod 검증 실패
- Supabase INSERT 에러 응답 (연락처 중복, RLS 등)

### 필수 데이터
- `formState.errors` 객체
- `serverError` 문자열 (서버 에러 시)

### 인터랙션 (User Actions)
1. 에러 필드 수정 → 해당 에러 제거
2. [기존 회원 보기] → 기존 회원 상세로 이동 (중복 연락처)
3. [다시 시도] → submit 재실행

### 비즈니스 룰
- SMS 미인증 시 에러 표시 (smsVerified=false)
- 서버 에러와 클라이언트 에러 구분하여 표시

### 에지 케이스
- 중복 연락처: [기존 회원 보기] 링크로 기존 회원 상세 안내
- 서버 타임아웃: "다시 시도" 옵션 제공

### 접근성 (A11y)
- 에러 배너 `role="alert"` + `aria-live="assertive"`
- Submit 실패 시 첫 에러 필드로 포커스 이동
