## 📝 디스크립션

### 사용 시점 (Trigger)
- '이용권' 탭 클릭
- URL 파라미터 `?tab=tickets`

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 계약 목록 | `contract` | memberId = memberId |
| 홀딩 정보 | `member_holdings` | memberId = memberId |
| 연장 이력 | `member_extensions` | memberId = memberId |

### 인터랙션 (User Actions)
1. [홀딩] → 홀딩 모달 → 기간/사유 입력 → 저장
2. [연장] → 연장 모달 → 연장 날짜 선택 → 저장
3. [해지] → 확인 모달 → 취소 처리
4. [이용권 추가] → 계약 추가 모달

### 비즈니스 룰
- fc는 홀딩/연장/해지 버튼 비노출 (조회만)
- 이미 만료된 계약은 [홀딩] 버튼 비노출
- 횟수형 이용권만 프로그레스바 표시
- 기간제 이용권은 D-Day만 표시

### 에지 케이스
- 계약 없는 경우 → EmptyState + [이용권 추가] 버튼
- 홀딩 중인 계약 → 홀딩 배지 + 홀딩 기간 표시

### 접근성 (A11y)
- 프로그레스바 `role="progressbar"` + `aria-valuenow` + `aria-valuemax`
- 카드별 `role="article"` + `aria-label="{이용권명} 카드"`

### 연결 화면
- 이전: `02-기본정보`
- 다음: 홀딩/연장/해지 모달 / 다른 탭
