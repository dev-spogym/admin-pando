## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-M004 회원 상세 — 상태: 결제 탭

파일: src/app/members/detail/page.tsx (activeTab === 'payment' 조건부)

레이아웃:
- Sticky 프로필 헤더 + TabNav (동일)
- 결제 이력 요약 + DataTable

컴포넌트 구조:
1. [결제 요약 카드 행] (3개)
   - 총 결제액: formatNumber(totalPaid) + "원"
   - 결제 횟수: N건
   - 마지막 결제일: formatDate(lastPaidAt)
2. [결제 이력 DataTable]
   컬럼: 결제일 | 상품명 | 결제방법 | 금액 | 할인 | 실결제 | 상태 | 담당자
   - 금액: formatNumber 천단위 콤마 + "원"
   - 상태: <StatusBadge> 완료/취소/부분취소/미수금
   - 환불: 환불된 건은 취소선 + 빨간색
3. [환불 처리] 버튼 — 각 행 우측 (owner/manager만)
4. [미수금 처리] 링크 — status='UNPAID' 건에 노출

데이터:
- interface SaleRecord { id, memberId, productName, paymentMethod, amount, discountAmount, finalAmount, status, paidAt, staffId, staff: { name } }
- supabase.from('sale').select('*, staff(name)').eq('memberId', memberId).order('paidAt', { ascending: false })
- totalPaid: sale.filter(s => s.status === 'COMPLETED').reduce((acc, s) => acc + s.finalAmount, 0)

인터랙션:
- [환불 처리] → 환불 금액/사유 모달 → sale UPDATE (status='CANCELLED')
- 미수금 건 클릭 → 미수금 처리 모달

사용 유틸:
- supabase from '@/lib/supabase'
- hasPermission from '@/lib/permissions'
- formatNumber, formatDate from '@/lib/format'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- '결제' 탭 클릭
- owner/manager/primary만 접근 가능 (fc/staff는 탭 자체 비노출)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 결제 이력 | `sale` | memberId, ORDER BY paidAt DESC |
| 결제 담당자 | `staff` | sale.staffId = staff.id |

### 인터랙션 (User Actions)
1. [환불 처리] → 환불 모달 → sale status 업데이트
2. 미수금 클릭 → 미수금 처리

### 비즈니스 룰
- fc/staff는 결제 탭 비노출
- 환불 처리: owner/manager 권한
- 동일 건 중복 환불 방지 (status 체크)

### 에지 케이스
- 결제 이력 없음 → EmptyState "결제 이력이 없습니다"
- 미수금 건 강조 표시 (bg-red-50 row)

### 접근성 (A11y)
- 금액 `aria-label="결제금액 {N}원"`
- 환불 버튼 `aria-label="{상품명} 환불 처리"`
