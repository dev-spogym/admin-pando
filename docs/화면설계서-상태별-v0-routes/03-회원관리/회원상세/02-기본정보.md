```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-M004 회원 상세 — 상태: 기본정보 탭 (회원정보 탭)

파일: src/app/members/detail/page.tsx

레이아웃:
- <AppLayout> 데스크톱
- [Sticky 프로필 헤더] — 상단 고정 (top-0 z-10 bg-white border-b)
  · 프로필 이미지 원형 (w-16 h-16, 없으면 이니셜 아바타)
  · 회원명 (text-xl font-bold) + <StatusBadge> 상태
  · 연락처 / 이메일 (text-sm text-gray-500)
  · 우측: [수정] [이관] 버튼 (권한 있는 경우)
- [TabNav] 7개 탭: 기본정보(활성) | 이용권 | 수업 | 상담 | 결제 | 메모 | 더보기(드롭다운)
- [기본정보 탭 콘텐츠] 2열 그리드 카드

컴포넌트 구조:
1. [Sticky 프로필 헤더]
   - Avatar: member.profileImage 또는 이니셜 (member.name[0])
   - StatusBadge: ACTIVE→활성/success, EXPIRED→만료/error 등
   - 버튼: hasPermission → [수정] → moveToPage('/members/edit', { id })
   - 버튼: hasPermission(owner) → [이관] → moveToPage('/members/transfer', { id })

2. [TabNav] tabs = ['기본정보', '이용권', '수업', '상담', '결제', '메모']

3. [기본정보 탭] 2열 그리드 섹션
   섹션 A "개인정보":
   - 이름 / 성별 (남/여) / 생년월일 / 연락처 / 이메일
   - 주소 / 유입경로 / 회원구분 / 광고수신동의

   섹션 B "이용권 정보":
   - 이용권종류 / 시작일 / 만료일 / D-Day 배지
   - 담당FC (staff.name) / 마일리지

   섹션 C "특이사항 / 메모":
   - member.specialNote (textarea 스타일, 읽기 전용)
   - member.memo

   섹션 D "방문 통계":
   - 총 방문수 (attendance COUNT)
   - 이번달 방문수
   - 마지막 방문일

데이터:
- interface MemberDetail extends Member { staff?: { name: string } }
- const { data: member } = supabase.from('members').select('*, staff(name)').eq('id', memberId).single()
- const { data: visitStats } = supabase.from('attendance').select('id, date').eq('memberId', memberId)

인터랙션:
- 탭 클릭 → 해당 탭 콘텐츠 렌더
- [수정] → moveToPage('/members/edit?id='+memberId)
- [이관] → moveToPage('/members/transfer?id='+memberId)
- 뒤로가기 → moveToPage('/members')

사용 유틸:

```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 기본 탭 (첫 진입 시 기본정보 탭 활성)
- 다른 탭에서 '기본정보' 탭 클릭

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 회원 정보 | `members` | id = memberId, JOIN staff |
| 방문 통계 | `attendance` | memberId = memberId |
| D-Day | 클라이언트 계산 | differenceInDays(membershipExpiry, today) |

### 인터랙션 (User Actions)
1. [수정] 버튼 → `/members/edit?id={memberId}` (owner/manager/staff)
2. [이관] 버튼 → `/members/transfer?id={memberId}` (owner만)
3. 탭 클릭 → 해당 탭 콘텐츠 전환

### 비즈니스 룰
- fc/staff는 [이관] 버튼 비노출
- readonly 역할은 수정/이관 버튼 모두 비노출
- D-Day 7일 이하 → 빨간색 강조
- 만료된 경우 → "만료됨" 텍스트 빨간색

### 에지 케이스
- profileImage URL 오류 → 이니셜 아바타 폴백
- 방문 통계 로딩 실패 → "-" 표시 (non-blocking)
- 담당FC 삭제된 경우 → "-" 표시

### 접근성 (A11y)
- 탭 `role="tablist"` + `role="tab"` + `aria-selected`
- 탭 패널 `role="tabpanel"` + `aria-labelledby`
- Sticky 헤더 `role="banner"`

### 연결 화면
- 이전: 회원 목록 (`/members`)
- 다음: 회원 수정 / 회원 이관 / 각 탭
