```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-M004 회원 상세 — 상태: 수업 탭

파일: src/app/members/detail/page.tsx (activeTab === 'lessons' 조건부)

레이아웃:
- Sticky 프로필 헤더 + TabNav (동일)
- 수업 예약/출석 이력 리스트

컴포넌트 구조:
1. [필터 행]
   - 기간 선택: Select (이번달 / 지난달 / 최근 3개월 / 전체)
   - 수업 유형: Select (전체 / PT / GX)
   - 상태: Select (전체 / 예약 / 출석 / 결석 / 취소)
2. [요약 통계 카드 행] (3개)
   - 총 예약: N건 / 출석: N건 / 결석: N건
3. [수업 이력 리스트] — DataTable 또는 카드 리스트
   컬럼: 날짜 | 수업명 | 강사 | 유형 | 상태
   - 상태: <StatusBadge> 출석/결석/취소/예약
   - 날짜: YYYY-MM-DD HH:mm
4. [페이지네이션]

데이터:
- interface LessonBooking { id, memberId, lessonName, instructorName, scheduledAt, type, status }
- supabase.from('lesson_bookings').select('*, lessons(name, type, instructorId), staff(name)')
    .eq('memberId', memberId).order('scheduledAt', { ascending: false })

인터랙션:
- 기간/유형/상태 필터 변경 → 즉시 재조회
- 행 클릭 → 수업 상세 (선택적, /lessons/detail?id=lessonId)

사용 유틸:
- supabase from '@/lib/supabase'
- formatDate from '@/lib/format'
- StatusBadge
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- '수업' 탭 클릭
- 예약/출석 이력 확인 필요 시

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 수업 예약 | `lesson_bookings` | memberId, 기간 필터 |
| 수업 정보 | `lessons` (또는 관련 테이블) | JOIN |
| 강사 정보 | `staff` | instructorId |

### 인터랙션 (User Actions)
1. 기간 필터 변경 → 해당 기간 이력 조회
2. 유형/상태 필터 → 추가 필터링

### 비즈니스 룰
- 본인 담당 수업만 조회 가능 (fc 역할): hasFeature('own-lessons-only')
- 취소된 예약도 이력에 표시 (회색 처리)

### 에지 케이스
- 수업 이력 없는 회원 → EmptyState "수업 이력이 없습니다"
- 미래 예약 포함 표시 (날짜 내림차순)

### 접근성 (A11y)
- 필터 Select `aria-label` 명시
- DataTable `role="grid"`

### 연결 화면
- 이전: 다른 탭
- 다음: 다른 탭 / 수업 상세 (선택)
