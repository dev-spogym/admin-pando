## 📝 디스크립션

### 사용 시점 (Trigger)
- '상담' 탭 클릭
- FC/매니저가 상담 후 이력 기록 시

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상담 이력 | `consultations` | memberId, ORDER BY consultedAt DESC |
| 작성자 | `staff` | consultations.staffId = staff.id |

### 인터랙션 (User Actions)
1. 상담 내용 입력 후 [저장] → consultations INSERT
2. 기존 항목 [수정] → 인라인 편집 → UPDATE
3. [삭제] → 확인 후 DELETE

### 비즈니스 룰
- fc도 상담 메모 작성 가능 (fc 권한 포함)
- 타인 작성 기록 삭제: owner/manager만 가능
- 본인 작성 기록 삭제: 24시간 이내만 허용

### 에지 케이스
- 상담 이력 없음 → EmptyState + 상담 추가 폼은 유지
- 내용 500자 초과 → 검증 에러

### 접근성 (A11y)
- 상담 추가 폼 `role="form"` + `aria-label="상담 추가"`
- 삭제 버튼 `aria-label="{날짜} 상담 기록 삭제"`
