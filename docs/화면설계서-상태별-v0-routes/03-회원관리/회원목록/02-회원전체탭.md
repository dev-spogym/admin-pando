## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 members 데이터 1건 이상 존재
- 탭 전환으로 '회원 전체' 탭 재진입

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 통계 카드 | `members` | branchId 필터, deletedAt IS NULL |
| 회원 목록 | `members` | branchId + status 서브탭 필터 |
| 세그먼트 | `members` | registeredAt, lastVisitAt, membershipExpiry, status |
| 담당FC | `staff` | members.staffId = staff.id |

### 인터랙션 (User Actions)
1. 상태 서브탭 클릭 → 해당 status 필터 적용 + 테이블 갱신
2. 회원명 클릭 → `/members/detail?id={id}`
3. ★ 클릭 → isFavorite 토글 (낙관적 업데이트)
4. 컬럼 헤더 클릭 → 정렬 asc/desc 토글
5. 엑셀 다운로드 → exportToExcel 실행 (owner/manager 권한)
6. 회원 추가 버튼 → `/members/new` (owner/manager/staff 권한)

### 비즈니스 룰
- fc/staff는 엑셀 다운로드 버튼 비노출
- fc는 회원 추가 버튼 비노출
- 세그먼트 판정: 신규(30일이내등록) > 만료후미등록(EXPIRED) > 이탈위험(ACTIVE+30일미방문) > 만료임박(ACTIVE+D-30이내) > 관심필요(ACTIVE+14일미방문) > 충성(ACTIVE+180일+7일내방문) > 활발(ACTIVE+14일내방문)
- 페이지당 20건 기본

### 에지 케이스
- 회원 0건 → `08-빈상태` 전환
- 검색어 입력 후 결과 없음 → 인라인 "검색 결과가 없습니다" 표시
- toggleFavorite 실패 → 롤백 + 에러 토스트

### 접근성 (A11y)
- DataTable `role="grid"`, 각 row `role="row"`
- ★ 버튼 `aria-label="{name} 관심회원 등록/해제"`
- 정렬 버튼 `aria-sort="ascending/descending/none"`
