## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-M001 회원 목록 — 상태: 이용권 목록 탭

파일: src/app/members/page.tsx (activeMainTab === 'pass' 조건부 렌더)

레이아웃:
- 메인 TabNav '이용권 목록' 활성
- 이용권 기준 DataTable (이용권 중심, 1회원 N이용권 가능)

컴포넌트 구조:
1. [기간 필터] — 시작일~만료일 날짜 범위 선택 (DateRangePicker)
2. [상태 필터] — Select: 전체 / 활성 / 만료 / 홀딩
3. [DataTable]
   컬럼: 회원명 | 이용권유형 | 시작일 | 만료일 | D-Day | 상태 | 잔여횟수
   - D-Day: 만료일 - 오늘. 양수면 "D-{n}", 음수면 "만료", 0이면 "D-Day"
   - D-Day 색상: D-7 이하 → text-red-600, D-30 이하 → text-yellow-600, 그 외 → text-green-600
   - 잔여횟수: PT 계약만 표시 (contract 테이블 join)
   - 회원명: 클릭 → moveToPage('/members/detail', { id })
4. [Pagination] 하단 페이지네이션

데이터:
- interface PassItem { memberId, memberName, membershipType, membershipStart, membershipExpiry, status, remainCount }
- supabase.from('members').select('id, name, membershipType, membershipStart, membershipExpiry, status').eq('branchId', branchId)
- contract JOIN으로 잔여횟수 추가

인터랙션:
- 날짜 범위 변경 → 만료일 기준 필터
- 상태 필터 변경 → status 기준 필터
- 회원명 클릭 → 상세 페이지

사용 유틸:
- differenceInDays from 'date-fns'
- formatDate from '@/lib/format'
- moveToPage from '@/internal'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 메인 탭에서 '이용권 목록' 탭 클릭
- 만료 임박 이용권 확인이 필요한 경우

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 이용권 목록 | `members` | branchId, membershipType, membershipExpiry |
| 잔여횟수 | `contract` | memberId, remainCount |

### 인터랙션 (User Actions)
1. 날짜 범위 필터 → 해당 기간 만료 이용권만 표시
2. 상태 필터 → 이용권 상태별 필터
3. 회원명 클릭 → 회원 상세 페이지

### 비즈니스 룰
- D-Day 계산: `differenceInDays(membershipExpiry, today)`
- 음수 D-Day는 "만료" 텍스트로 표시
- PT 계약만 잔여횟수 컬럼에 값 노출 (나머지 '-')

### 에지 케이스
- 이용권이 없는 회원은 목록에 미노출
- 동일 회원이 복수 이용권 보유 시 각각 행으로 표시

### 접근성 (A11y)
- D-Day 배지 `aria-label="D-{n}일 남음"` 또는 `aria-label="만료됨"`
- 날짜 범위 입력 `aria-label="이용권 만료일 범위 필터"`

### 연결 화면
- 이전: `02-회원전체탭` / `03-보유상품별탭`
- 다음: 회원 상세 / 탭 전환
