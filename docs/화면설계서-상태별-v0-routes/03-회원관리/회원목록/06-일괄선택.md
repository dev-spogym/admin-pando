## 📝 디스크립션

### 사용 시점 (Trigger)
- 테이블 row의 체크박스 클릭 (1개 이상 선택 시)
- 헤더 체크박스로 전체 선택 시

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 선택 상태 | 클라이언트 Set | selectedRows: Set<number> |
| 일괄 업데이트 | `members` | id IN (selectedRows) |
| 일괄 출석 | `attendance` | memberId IN (selectedRows) |

### 인터랙션 (User Actions)
1. 체크박스 클릭 → BulkActionBar 슬라이드 다운 노출
2. 상태 변경 드롭다운 선택 → 확인 모달 → 일괄 UPDATE
3. 출석 처리 → 오늘 날짜로 선택 회원 전체 출석 INSERT
4. 관심회원 → isFavorite 일괄 토글
5. 선택 취소 → BulkActionBar 숨김

### 비즈니스 룰
- 일괄 상태 변경: owner/manager 권한만 가능
- 출석 처리: 오늘 이미 출석한 회원은 중복 INSERT 방지 (upsert)
- 전체 선택은 현재 페이지 내 회원만 (전체 DB 아님)
- 선택 후 페이지 이동 시 선택 상태 초기화

### 에지 케이스
- 일괄 업데이트 실패 시 롤백 + 에러 토스트
- 선택된 회원 중 일부가 이미 해당 상태인 경우 → skip 처리
- 50명 이상 선택 시 경고 토스트 ("대량 처리는 시간이 걸릴 수 있습니다")

### 접근성 (A11y)
- 헤더 체크박스 `aria-label="전체 선택"` + `aria-checked="mixed"` (indeterminate)
- BulkActionBar `role="toolbar"` + `aria-label="일괄 작업 도구"`
- 선택된 row `aria-selected="true"`
