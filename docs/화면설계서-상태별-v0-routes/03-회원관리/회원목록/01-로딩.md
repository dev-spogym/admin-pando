## 📝 디스크립션

### 사용 시점 (Trigger)
- `/members` 페이지 최초 진입 시
- 브라우저 새로고침 시
- `useMembers` 훅의 `isLoading === true` 구간

### 필수 데이터
| 블록 | 상태 | 설명 |
|------|------|------|
| StatCardGrid | 로딩 중 | stats 데이터 fetch 전 |
| TabNav | 정적 렌더 | 탭 구조는 하드코딩 |
| DataTable | 로딩 중 | members 데이터 fetch 전 |

### 인터랙션 (User Actions)
1. 로딩 중 클릭 → 무반응 (pointer-events-none)
2. 데이터 수신 완료 → `02-회원전체탭` 상태로 전환

### 비즈니스 룰
- 스켈레톤은 실제 레이아웃과 동일한 구조여야 함 (CLS 최소화)
- 통계 카드 4개 / 테이블 행 8개 고정 렌더링

### 에지 케이스
- 네트워크 오류 시 → 에러 토스트 노출 후 빈 상태 유지
- 권한 없는 역할(readonly) → `/forbidden` 리다이렉트

### 접근성 (A11y)
- 스켈레톤에 `aria-busy="true"` + `aria-label="회원 목록 로딩 중"` 적용
- `role="status"` 컨테이너로 감싸기

### 연결 화면
- 이전: 사이드바 메뉴 클릭
- 다음: `02-회원전체탭` (로드 성공) / 에러 상태 (네트워크 실패)
