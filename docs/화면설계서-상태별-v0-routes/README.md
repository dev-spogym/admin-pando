# FitGenie CRM 관리자 화면설계서 (상태별)

> **목적**: 바이브코딩(Vibe Coding) 친화적 상태별 화면설계서 — 각 상태를 독립 파일로 분리, AI 코딩 프롬프트 + 디스크립션 포함
> **버전**: 1.0 (2026-04-21)
> **플랫폼**: Next.js 15 App Router + TypeScript + Tailwind CSS + Supabase (데스크톱 중심)
> **자매 문서**: `client-pando/docs/화면설계서/` (회원앱 — 모바일)

---

## 폴더 구조

```
docs/화면설계서-상태별/
├── README.md                              # 이 파일 (포맷 가이드 + 색인)
├── 00-공통/                                # 로그인, 대시보드, 에러 페이지
├── 01-본사관리/                            # 슈퍼관리자, KPI, 브랜치, 리포트
├── 02-수업관리/                            # 수업, 캘린더, 템플릿, PT 횟수
├── 03-회원관리/                            # 회원 목록/상세/등록/수정/전환, 출석, 공지
├── 04-매출관리/                            # 매출, POS, 환불, 미수금, 할인
├── 05-상품관리/                            # 상품 CRUD
├── 06-시설관리/                            # 락커, 옷장, RFID, 룸, 골프베이
├── 07-직원관리/                            # 직원 CRUD, 근태, 급여, 명세서
├── 08-마케팅/                              # 리드, 계약, 메시지, 마일리지
├── 09-설정관리/                            # 설정, IOT, KIOSK, 권한
└── 10-통합운영/                            # 다이어그램 등
```

## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: {SCR-ID} {화면명} — 상태: {상태명}

파일: src/app/(group)/path/page.tsx

레이아웃:
- <AppLayout> 데스크톱 레이아웃 (사이드바 + 헤더)
- <PageHeader title="..." subtitle="..." actions={...} />
- <StatCardGrid>과 <StatCard> (4컬럼 통계)
- <TabNav> / <SearchFilter> / <DataTable> (리스트)
- <Modal> (추가/수정/삭제 다이얼로그)

컴포넌트 구조:
1. [첫 번째 블록 — 무슨 데이터, 어떻게 표시]
2. [두 번째 블록]
...

데이터:
- 타입: interface Xxx { ... }
- 소스: useXxx() 훅 또는 supabase.from('table').select(...)
- 권한: hasPermission(user, '...')

인터랙션:
- [이벤트 → 결과 상태]

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- toast from 'sonner'
- formatNumber / formatDate from '@/lib/format'
- moveToPage from '@/internal'
- hasPermission / hasFeature from '@/lib/permissions'
- lucide-react 아이콘
```
