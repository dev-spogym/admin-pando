   const loadStaffData = async () => {
     const { data, error } = await supabase
       .from("staff")
       .select("*")
       .eq("id", editId)
       .single();
     if (error || !data) {
       toast.error("직원 정보를 불러오지 못했습니다.");
       moveToPage(974);
       return;
     }
     reset({
       name: data.name,
       role: ROLE_DB_TO_KEY[data.role] ?? data.role,
       contact: data.phone ?? "",
       joinDate: data.hireDate ?? "",
       email: data.email ?? "",
       memo: data.memo ?? "",
       salary: data.salary?.toString() ?? "",
     });
     setIsDataLoaded(true);
   }
3. isDataLoaded = false 동안 스켈레톤 렌더링

## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-060 직원명 ghost 버튼 클릭 → `moveToPage(998, { id })`
- `/staff/edit?id={staffId}` URL 진입
- `editId !== null && !isDataLoaded` 조건

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 직원 상세 | `staff` | `id = editId` |

### 인터랙션 (User Actions)
1. 자동 fetch → reset() → 02-기본 전환
2. fetch 실패 → toast.error + SCR-060으로 이동

### 비즈니스 룰
- editId가 없는 경우 등록 모드로 처리 (같은 컴포넌트)
- 데이터 로드 실패 시 목록으로 강제 이동
- ROLE_DB_TO_KEY로 한글 역할 → 영문 키 변환

### 에지 케이스
- 다른 지점의 staffId 접근 → branchId 검증 후 접근 거부
- staffId가 숫자 아닌 값 → parseInt 실패 → 목록으로 이동

### 접근성 (A11y)
- 스켈레톤 영역 `aria-busy="true"` + `aria-label="직원 정보 로딩 중"`
