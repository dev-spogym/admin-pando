## 📝 디스크립션

### 사용 시점 (Trigger)
- `isDataLoaded === true` (supabase fetch + reset() 완료)
- 폼에 기존 직원 데이터가 채워진 상태

### 필수 데이터
| 블록 | 소스 | 설명 |
|------|------|------|
| 기존 직원 정보 | supabase | name, role, phone, hireDate, salary, email, memo |
| 현재 staffStatus | supabase | ACTIVE/ON_LEAVE/RESIGNED |

### 인터랙션 (User Actions)
1. 필드 수정 → dirty 상태 (react-hook-form)
2. 수정 저장 → 유효성 검사 → 03-저장중
3. 취소 → DLG-061-001 (수정 취소 확인)
4. 직원 삭제 → 05-삭제확인 다이얼로그

### 비즈니스 룰
- 수정 모드: title="직원 정보 수정", 버튼="수정 저장"
- 재직 상태 섹션은 수정 모드에서만 표시
- 퇴사 처리는 DLG-삭제확인을 통해 수행

### 에지 케이스
- 역할 변경 → 권한 미리보기 즉시 업데이트
- 이메일 삭제 → null로 저장 (선택 필드)

### 접근성 (A11y)
- 수정 모드 폼 `aria-label="직원 정보 수정 폼"`
- 삭제 버튼 `aria-label="직원 삭제"` + 위험 색상
