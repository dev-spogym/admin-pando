## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-061 직원 수정 — 상태: 삭제 확인 모달 오픈

파일: src/app/staff/edit/page.tsx

컴포넌트 구조:
1. showDeleteDialog === true → ConfirmDialog 렌더링
2. ConfirmDialog props:
   open={showDeleteDialog}
   onClose={() => setShowDeleteDialog(false)}
   title="직원 삭제"
   description={`${staffName} 직원을 삭제하시겠습니까?\n삭제 시 해당 직원의 모든 데이터 접근이 제한됩니다.\n이 작업은 되돌릴 수 없습니다.`}
   confirmLabel="삭제하기"
   cancelLabel="취소"
   variant="danger"
   confirmationText="삭제"
   onConfirm={handleDelete}
3. handleDelete 함수:
   const handleDelete = async () => {
     try {
       const { error } = await supabase
         .from("staff")
         .update({ isActive: false, staffStatus: "RESIGNED", resignedAt: new Date().toISOString() })
         .eq("id", editId);
       if (error) throw error;
       toast.success(`${staffName} 직원이 삭제(퇴사 처리)되었습니다.`);
       moveToPage(974);
     } catch (err: any) {
       toast.error(`삭제 실패: ${err.message}`);
     }
   }
4. confirmationText="삭제" → 사용자가 "삭제" 텍스트 직접 입력 후 확인 활성화

데이터:
- showDeleteDialog: boolean
- staffName: string (로드된 직원명)
- supabase update: isActive=false, staffStatus="RESIGNED"

인터랙션:
- 삭제하기 확인 → handleDelete() → toast.success → moveToPage(974)
- 취소 → showDeleteDialog=false → 02-기본 복귀
- confirmationText 미일치 → 삭제 버튼 disabled

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- moveToPage from '@/internal'
- ConfirmDialog from '@/components/ui/ConfirmDialog'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader "직원 삭제" 버튼 클릭 → `showDeleteDialog = true`

### 필수 데이터
| 블록 | 테이블 | 동작 |
|------|--------|------|
| 직원 삭제(퇴사 처리) | `staff` | UPDATE isActive=false, staffStatus="RESIGNED" |

### 인터랙션 (User Actions)
1. "삭제" 텍스트 입력 → confirmationText 검증 → 삭제 버튼 활성화
2. 삭제하기 → handleDelete() → 성공 시 SCR-060
3. 취소 → 모달 닫기, 02-기본 복귀

### 비즈니스 룰
- 실제 DB 행 삭제 아닌 소프트 삭제 (isActive=false, staffStatus="RESIGNED")
- confirmationText="삭제" 입력 필수 (의도적 삭제 확인)
- 토스트: `"{staffName} 직원이 삭제(퇴사 처리)되었습니다."`

### 에지 케이스
- confirmationText 미일치 → 버튼 disabled (실수 방지)
- 삭제 중 네트워크 오류 → toast.error + 모달 유지

### 접근성 (A11y)
- ConfirmDialog `role="alertdialog"` + 포커스 트랩
- 위험 동작 `aria-describedby` 경고 텍스트 참조
