## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-064 급여 관리 — 상태: 급여 확정 처리 (DLG-064-002)

파일: src/app/payroll/page.tsx

컴포넌트 구조:
1. DLG-064-002 급여 확정 ConfirmDialog:
   <ConfirmDialog
     open={isConfirmDialogOpen}
     onClose={() => setIsConfirmDialogOpen(false)}
     title="급여 확정"
     description={`${selectedMonth} 급여를 확정하시겠습니까?\n확정 후 직원별 급여 명세서가 자동으로 생성됩니다.\n(미확정 ${pendingCount}명 포함)`}
     confirmLabel="급여 확정"
     cancelLabel="취소"
     variant="primary"
     confirmationText="확정"
     onConfirm={handleConfirmAll}
   />

2. handleConfirmAll():
   setIsConfirming(true);
   try {
     const pendingIds = payrollData
       .filter(p => p.status === "pending")
       .map(p => p.id);

     // 전체 확정 처리
     const { error } = await supabase
       .from("staff")
       .update({ payrollStatus: "confirmed", payrollMonth: selectedMonth })
       .in("id", pendingIds);
     if (error) throw error;

     // 명세서 자동 생성 (가상 테이블)
     await createPayrollStatements(pendingIds, selectedMonth);

     toast.success(`${selectedMonth} 급여가 확정되었습니다. 명세서가 자동 생성되었습니다.`);
     fetchPayroll(selectedMonth);
   } catch (err: any) {
     toast.error(`급여 확정 실패: ${err.message}`);
   } finally {
     setIsConfirming(false);
     setIsConfirmDialogOpen(false);
   }

3. 확정 완료 후 목록 상태:
   - 전체 StatusBadge → variant="mint" label="확정"
   - 급여 확정 버튼 → disabled (이미 전원 확정)

데이터:
- isConfirming: boolean
- pendingCount: status === "pending" 직원 수
- confirmationText="확정" (의도적 확정 확인)

인터랙션:
- "확정" 텍스트 입력 → 확정 버튼 활성화
- 급여 확정 → handleConfirmAll() → 전체 status="confirmed"
- 성공 → toast.success + fetchPayroll 재호출

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- ConfirmDialog from '@/components/ui/ConfirmDialog'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- PageHeader "급여 확정" 버튼 클릭
- `isConfirmDialogOpen === true`

### 필수 데이터
| 블록 | 테이블 | 동작 |
|------|--------|------|
| 급여 확정 | `staff` (또는 payroll 가상) | UPDATE payrollStatus="confirmed" |
| 명세서 생성 | `payroll_statements` (가상) | INSERT |

### 인터랙션 (User Actions)
1. "확정" 텍스트 입력 → 버튼 활성화
2. 급여 확정 → handleConfirmAll() → 전체 확정
3. 성공 → toast.success + 목록 갱신
4. 취소 → 모달 닫기

### 비즈니스 룰
- 확정 토스트: `"{selectedMonth} 급여가 확정되었습니다. 명세서가 자동 생성되었습니다."`
- 확정 후 명세서 자동 생성 → SCR-065에서 조회 가능
- 전원 확정 후 급여 확정 버튼 disabled

### 에지 케이스
- 이미 전원 확정 상태 → 급여 확정 버튼 disabled
- 일부만 확정 → 미확정 직원만 대상으로 처리

### 접근성 (A11y)
- ConfirmDialog `role="alertdialog"` + 포커스 트랩
- confirmationText input `aria-label="확정 텍스트 입력"`
