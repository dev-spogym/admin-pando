## 📝 디스크립션

### 사용 시점 (Trigger)
- 재직상태 Select에서 "퇴사" 선택
- `filterValues.status === "resigned"` 조건

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 퇴사 직원 목록 | `staff` | `staffStatus = "RESIGNED"` OR `isActive = false` |
| 퇴사 정보 | `staff` | `resignedAt`, `resignReason`, `resignScheduledAt` |

### 인터랙션 (User Actions)
1. 재직상태 Select → "전체/재직/휴직" → 해당 필터로 전환
2. 명단 다운로드 → 퇴사 직원 엑셀 출력
3. 직원명 클릭 → 수정 화면 (읽기 전용 모드 권장)

### 비즈니스 룰
- 퇴사 직원은 퇴사 처리 버튼 불필요 (이미 퇴사 상태)
- 퇴사일 표시: `resignedAt` 우선, 없으면 `resignScheduledAt`
- 퇴사 직원 계정은 `isActive = false` 로 접근 제한

### 에지 케이스
- 퇴사 직원 0명 → "퇴사한 직원이 없습니다." 메시지
- 퇴사 사유 미입력 → "-" 표시

### 접근성 (A11y)
- StatusBadge `aria-label="퇴사"`
- 퇴사일 컬럼 `aria-label="퇴사 처리일"`
