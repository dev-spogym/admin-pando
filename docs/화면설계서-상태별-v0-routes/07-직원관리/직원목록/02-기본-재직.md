## 📝 디스크립션

### 사용 시점 (Trigger)
- `fetchStaff()` 완료 후 `staffData.length > 0`
- 기본 필터 상태: role="", status="" (전체 재직 포함)

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 직원 목록 | `staff` | `branchId = getBranchId()` |
| 통계 카드 | 로컬 계산 | staffData 기반 count |

### 인터랙션 (User Actions)
1. 체크박스 단일/전체 선택 → selectedRows 업데이트
2. 직원명 ghost 버튼 클릭 → `moveToPage(998, { id })`
3. + 직원 등록 → `moveToPage(998)`
4. 메시지 전송 → `moveToPage(980)` (선택 직원 ID 전달)
5. 상태 변경 → DLG-060-002 모달 오픈
6. 퇴사 처리 → DLG-060-001 ConfirmDialog 오픈
7. 명단 다운로드 → `exportToExcel` + `toast.success`
8. 컬럼 헤더 클릭 → 3-state 정렬 토글 (null→asc→desc→null)
9. 페이지네이션 → currentPage 변경

### 비즈니스 룰
- `staffStatus: ACTIVE` 기준으로 재직 탭 기본 표시
- `isActive === false` → resigned 매핑
- 역할 배지: ROLE_CONFIG 기반 (한글/영문 키 모두 fallback)
- PAGE_SIZE = 10, 3-state 정렬 지원 컬럼: name, role, joinDate, status

### 에지 케이스
- selectedRows.size === 0 → 메시지전송/상태변경/퇴사처리 disabled
- 검색어 + 필터 동시 적용 시 AND 교집합
- 정렬 + 필터 동시 적용 시 필터 먼저, 정렬 나중

### 접근성 (A11y)
- 테이블 `role="grid"`, 체크박스 `aria-label="직원 선택"`
- 정렬 아이콘 `aria-sort="ascending|descending|none"`
- 역할 배지 `aria-label="{역할명}"`
