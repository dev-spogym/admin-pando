   <Button variant="primary" disabled={isSaving}>

3. onSubmit 함수:
   const onSubmit = async (data: StaffFormData) => {
     setIsSaving(true);
     try {
       const { error } = await supabase.from("staff").insert({
         name: data.name,
         phone: data.contact,
         email: data.email || null,
         role: ROLE_DB_MAP[data.role] ?? data.role,
         hireDate: data.joinDate,
         salary: data.salary ? Number(data.salary) : null,
         memo: data.memo || null,
         isActive: true,
         branchId: getBranchId(),
         staffStatus: "ACTIVE",
       });
       if (error) throw error;
       toast.success("직원이 성공적으로 등록되었습니다.");
       moveToPage(974); // 직원 목록으로 이동
     } catch (err: any) {
       toast.error(`저장 실패: ${err.message}`);
     } finally {
       setIsSaving(false);
     }
   }

- isSaving: boolean = true (supabase insert 중)

## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장하기 버튼 클릭 → zod 유효성 통과 → `isSaving = true`
- `supabase.from("staff").insert()` 실행 중

### 필수 데이터
| 블록 | 테이블 | 동작 |
|------|--------|------|
| 직원 등록 | `staff` | INSERT |

### 인터랙션 (User Actions)
1. 저장 중 → 버튼 disabled, 아이콘 스피너 표시
2. 성공 → `toast.success("직원이 성공적으로 등록되었습니다.")` → SCR-060
3. 실패 → `toast.error("저장 실패: {message}")` → 폼 복원

### 비즈니스 룰
- isSaving 중 저장 버튼 재클릭 방지 (disabled)
- insert 완료 후 `moveToPage(974)` → SCR-060 직원 목록
- 실패 시 isSaving=false 복원 (finally 블록)

### 에지 케이스
- 네트워크 타임아웃 → toast.error 표시 + 폼 복원
- 저장 중 탭 이탈 → 데이터 손실 가능 (경고 없음)

### 접근성 (A11y)
- 저장 버튼 `aria-busy="true"` 저장 중 상태
- 스피너 `aria-label="저장 중"`
