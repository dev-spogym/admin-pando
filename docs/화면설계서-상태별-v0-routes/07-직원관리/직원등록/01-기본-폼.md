## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-060에서 "+ 직원 등록" 버튼 클릭
- `moveToPage(998)` → `/staff/new` 라우팅
- URL에 `?id=` 파라미터 없음 (등록 모드)

### 필수 데이터
| 블록 | 소스 | 설명 |
|------|------|------|
| ROLE_OPTIONS | 상수 | 센터장/매니저/FC/트레이너/스태프 |
| ROLE_PERMISSIONS | 상수 | 역할별 권한 미리보기 |
| staffFormSchema | zod | 필수 필드 유효성 검증 |

### 인터랙션 (User Actions)
1. 역할 Select → ROLE_PERMISSIONS 박스 표시
2. 연락처 입력 → 자동 하이픈 포맷
3. 저장하기 → 유효성 검사 통과 시 02-입력중
4. 취소 → DLG-061-001 (취소 확인 다이얼로그)

### 비즈니스 룰
- 이름, 역할, 연락처, 입사일 필수 (zod 검증)
- 연락처 최대 13자 (010-0000-0000 형식)
- 역할 기본값: "trainer"
- 입사일 기본값: 오늘 날짜

### 에지 케이스
- 페이지 새로고침 → 폼 초기화 (defaultValues)
- 브라우저 뒤로가기 → 취소 확인 없이 이탈 (추후 beforeunload 처리)

### 접근성 (A11y)
- 폼 필드 `aria-required="true"` (필수 항목)
- 에러 메시지 `role="alert"` + `aria-describedby`
