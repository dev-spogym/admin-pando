## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장하기 버튼 클릭 시 zod 유효성 실패
- mode: "onBlur" → 개별 필드 이탈 시 에러 표시

### 필수 데이터
| 필드 | 에러 메시지 |
|------|------------|
| name | "이름을 입력해주세요." |
| role | "역할을 선택해주세요." |
| contact | "연락처를 입력해주세요." / "010-XXXX-XXXX 형식으로 입력해주세요." |
| joinDate | "입사일을 입력해주세요." |
| email (선택) | "올바른 이메일 형식이 아닙니다." |

### 인터랙션 (User Actions)
1. 에러 필드 수정 → onBlur → 에러 해소 (border 정상화)
2. 저장하기 재시도 → handleSubmit 재실행
3. 에러 요약 배너 → 몇 개 항목 오류인지 안내

### 비즈니스 룰
- 저장하기 클릭 시 모든 필드 동시 유효성 검사 (mode 무관)
- 에러 있는 첫 번째 필드로 자동 스크롤
- 에러 해소 후 즉시 border 정상화

### 에지 케이스
- 이메일 형식 오류 → "올바른 이메일 형식이 아닙니다." (선택 필드)
- 저장 API 실패 (DB 에러) → toast.error로 별도 처리

### 접근성 (A11y)
- 에러 메시지 `role="alert"` + `aria-live="assertive"`
- 에러 필드 `aria-invalid="true"` + `aria-describedby="{field}-error"`
- 에러 요약 배너 `role="alert"`
