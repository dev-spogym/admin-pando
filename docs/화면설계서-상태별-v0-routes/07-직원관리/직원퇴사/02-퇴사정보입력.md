## 📝 디스크립션

### 사용 시점 (Trigger)
- Step 1에서 직원 선택 완료 후 퇴사 정보 입력
- `selectedStaff !== null` 조건

### 필수 데이터
| 필드 | 타입 | 필수 | 설명 |
|------|------|:---:|------|
| 퇴사 예정일 | date | * | min=today |
| 퇴사 사유 | textarea | - | 자유 입력 |

### 인터랙션 (User Actions)
1. 퇴사 예정일 입력 → step1.resignDate 업데이트
2. 퇴사 사유 입력 (선택)
3. 다음 → FC이면 Step 2, 비FC이면 Step 3

### 비즈니스 룰
- 퇴사 예정일: 오늘 이후만 선택 가능 (min=today)
- FC 역할: role === "fc" || "FC" 판단
- 퇴사 사유 미입력 허용

### 에지 케이스
- 과거 날짜 선택 → input type="date" min 속성으로 자동 차단
- 퇴사 예정일 없이 다음 → "퇴사 예정일을 입력하세요." 에러

### 접근성 (A11y)
- 퇴사 예정일 `aria-required="true"` + `aria-label="퇴사 예정일"`
- 에러 메시지 `role="alert"`
