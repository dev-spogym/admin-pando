## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-060 "퇴사 처리" 버튼 클릭
- `/staff/resignation` 진입 (currentStep === 1)

### 필수 데이터
| 블록 | 테이블/API | 조건 |
|------|-----------|------|
| 직원 목록 | `getStaff({ size: 100 })` | 재직 중(`isActive=true`) |
| 선택 직원 상세 | `getStaffById(id)` | 직원 선택 시 |

### 인터랙션 (User Actions)
1. 직원 Select → selectedStaff 카드 표시
2. 다음 → handleStep1Next() → 유효성 검사
3. 유효성 실패 → step1Errors.staffId = "퇴사 처리할 직원을 선택하세요."

### 비즈니스 룰
- FC 역할 직원 선택 시 Step 2(담당 회원 재배정) 진입
- 비FC 역할 → Step 2 건너뜀, Step 3으로 직행
- 이미 퇴사/휴직 직원 → Select 옵션에서 제외 권장

### 에지 케이스
- 직원 0명 → Select 비어있음, "등록된 직원이 없습니다." 안내
- URL에 staffId 파라미터 → 자동 선택

### 접근성 (A11y)
- StepIndicator `role="progressbar"` + `aria-valuenow="1"` `aria-valuemax="4"`
- 직원 Select `aria-required="true"` + `aria-label="퇴사 처리 대상 직원"`

- 이전: SCR-060 직원 목록
- 다음: 02-퇴사정보입력
