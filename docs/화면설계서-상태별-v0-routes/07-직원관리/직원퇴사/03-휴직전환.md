```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-062 직원 퇴사 처리 — 변형: 휴직 전환 처리 (퇴사 대신 휴직 선택)

파일: src/app/staff/resignation/page.tsx

컴포넌트 구조:
1. Step 1에서 "퇴사 대신 휴직 전환" 옵션 제공:
   <div className="border border-accent/30 rounded-lg p-4 bg-accent-light">
     <h4 className="font-semibold text-accent mb-2">💡 퇴사 대신 휴직 전환이 가능합니다</h4>
     <p className="text-sm text-content-secondary mb-3">
       퇴사 처리 대신 휴직으로 전환하면 복직이 가능합니다.
     </p>
     <Button variant="outline" onClick={() => setShowLeaveModal(true)}>
       휴직으로 전환하기
     </Button>
   </div>

2. 휴직 전환 Modal (showLeaveModal):
   <Modal open={showLeaveModal} onClose={() => setShowLeaveModal(false)} title="휴직 전환">
     <div className="space-y-4">
       <div>
         <label>휴직 시작일 *</label>
         <Input type="date" value={leaveStart} onChange={...} min={today}/>
       </div>
       <div>
         <label>휴직 종료일 *</label>
         <Input type="date" value={leaveEnd} onChange={...} min={leaveStart}/>
       </div>
       <div>
         <label>휴직 사유</label>
         <Textarea value={leaveReason} onChange={...} placeholder="휴직 사유를 입력하세요"/>
       </div>
     </div>
     <div className="flex gap-2 justify-end mt-6">
       <Button variant="outline" onClick={() => setShowLeaveModal(false)}>취소</Button>
       <Button variant="primary" onClick={handleLeaveConversion}>휴직 처리</Button>
     </div>
   </Modal>

3. handleLeaveConversion():
   const { error } = await supabase
     .from("staff")
     .update({
       staffStatus: "ON_LEAVE",
       leaveStartAt: leaveStart,
       leaveEndAt: leaveEnd,
       leaveReason: leaveReason || null,
     })
     .eq("id", step1.staffId);
   if (error) toast.error("휴직 전환 실패");
   else { toast.success("휴직 전환이 완료되었습니다."); moveToPage(974); }

데이터:
- leaveStart, leaveEnd: string (YYYY-MM-DD)
- leaveReason: string
- supabase update: staffStatus="ON_LEAVE", leaveStartAt, leaveEndAt, leaveReason

인터랙션:
- 휴직으로 전환하기 버튼 → showLeaveModal=true
- Modal 내 휴직 처리 → handleLeaveConversion()
- 성공 → toast.success → moveToPage(974)

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- moveToPage from '@/internal'
- Modal from '@/components/ui/Modal'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- Step 1에서 직원 선택 후 "휴직으로 전환하기" 버튼 클릭
- 퇴사 처리 대신 휴직 전환을 원할 때

### 필수 데이터
| 블록 | 테이블 | 동작 |
|------|--------|------|
| 휴직 전환 | `staff` | UPDATE staffStatus="ON_LEAVE", leaveStartAt, leaveEndAt |

### 인터랙션 (User Actions)
1. "휴직으로 전환하기" → Modal 오픈
2. 휴직 시작일/종료일 입력
3. 휴직 처리 → supabase update → toast.success → SCR-060

### 비즈니스 룰
- 휴직 전환: staffStatus="ON_LEAVE" (RESIGNED 아님)
- 휴직 종료일 이후 자동 복직 처리는 별도 배치 작업
- leaveStartAt/leaveEndAt 모두 필수

### 에지 케이스
- 종료일 < 시작일 → 에러 (min=leaveStart 적용)
- 이미 휴직 중인 직원 → 안내 메시지

### 접근성 (A11y)
- Modal `role="dialog"` + 포커스 트랩
- 날짜 input `aria-label="휴직 시작일/종료일"`
