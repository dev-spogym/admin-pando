```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-062 직원 퇴사 처리 — Step 2: 담당 회원 재배정 (FC 역할만)

파일: src/app/staff/resignation/page.tsx

컴포넌트 구조:
1. currentStep === 2 (FC 역할만 진입)
2. 일괄 배정 섹션:
   <div className="flex items-center gap-3 mb-4">
     <label className="text-sm font-medium">일괄 배정:</label>
     <Select
       value={bulkAssignFcId}
       onChange={setBulkAssignFcId}
       options={activeFcOptions}
       placeholder="FC 선택"
       className="w-48"
     />
     <Button variant="outline" size="sm" onClick={handleBulkAssign}>
       미배정 회원에 적용
     </Button>
   </div>

3. 담당 회원 재배정 테이블:
   <SimpleTable
     columns={[
       { header: "회원명", key: "name" },
       { header: "연락처", key: "phone" },
       {
         header: "PT 잔여",
         key: "hasPtRemaining",
         render: (v) => v
           ? <StatusBadge variant="error" label="PT잔여"/>
           : "-"
       },
       {
         header: "재배정 담당자",
         key: "assignedStaffId",
         render: (v, row) => (
           <Select
             value={assignments[row.id]}
             onChange={val => setAssignments(prev => ({...prev, [row.id]: val}))}
             options={activeFcOptions}
             placeholder="FC 선택"
             className={cn(
               "w-40",
               row.hasPtRemaining && !assignments[row.id] ? "border-state-error" : ""
             )}
           />
         )
       }
     ]}
     rows={memberRows}
   />

4. handleStep2Next():
   // PT 잔여 회원 미배정 확인
   const unassigned = memberRows.filter(m => m.hasPtRemaining && !assignments[m.id]);
   if (unassigned.length > 0) {
     toast.warning(`PT 잔여 회원 ${unassigned.length}명의 재배정 담당자를 지정해주세요.`);
     return;
   }
   setCurrentStep(3);

데이터:
- memberRows: MemberRow[] (getStaffMembers(staffId))
- activeFcOptions: { value: id, label: name }[] (FC 역할 재직 직원)
- assignments: Record<memberId, fcStaffId>
- bulkAssignFcId: number | null

인터랙션:
- 일괄 배정 FC 선택 → [미배정 회원에 적용] → assignments 일괄 업데이트
- 개별 Select → 해당 회원 assignments[id] 업데이트
- 다음 → PT잔여 미배정 확인 → toast.warning 또는 Step 3
- 이전 → setCurrentStep(1)

사용 유틸:
- getStaffMembers from '@/api/endpoints/staff'
- toast from 'sonner'
- SimpleTable from '@/components/common/SimpleTable'
- StatusBadge: variant="error"
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `currentStep === 2` (FC 역할 직원 퇴사 처리 시)
- Step 1에서 FC 역할 감지 → Step 2 진입

### 필수 데이터
| 블록 | API | 조건 |
|------|-----|------|
| 담당 회원 목록 | `getStaffMembers(staffId)` | 퇴사 FC의 담당 회원 |
| 재직 FC 목록 | `getStaff({ role: 'FC' })` | 재배정 대상 |

### 인터랙션 (User Actions)
1. 일괄 FC 선택 → [미배정 회원에 적용] → 전체 적용
2. 개별 회원 FC Select → 해당 회원만 변경
3. 다음 → PT잔여 미배정 체크 → 경고 또는 Step 3
4. 이전 → Step 1

### 비즈니스 룰
- PT 잔여 회원 미배정 시 경고 toast (차단하지 않음)
- `toast.warning("PT 잔여 회원 {N}명의 재배정 담당자를 지정해주세요.")`
- 비FC 역할 → 이 Step 완전히 건너뜀

### 에지 케이스
- 담당 회원 0명 → "담당 회원이 없습니다." 표시 후 자동으로 Step 3 이동
- 재직 FC 0명 → 재배정 불가, 경고 표시

### 접근성 (A11y)
- PT잔여 배지 `aria-label="PT 잔여 회원"`
- 재배정 Select `aria-label="재배정 FC 선택"`
