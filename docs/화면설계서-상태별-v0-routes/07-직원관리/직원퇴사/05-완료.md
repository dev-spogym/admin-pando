     <AlertTriangle size={20} className="text-state-warning shrink-0 mt-0.5"/>
     <p className="text-sm text-content-primary">
       퇴사 예정 등록 후에는 수정이 어려울 수 있습니다. 아래 정보를 확인해주세요.
     </p>
   </div>

3. 요약 정보 카드:
   <div className="bg-surface-secondary rounded-xl p-6 space-y-4">
     <div className="flex justify-between">
       <span className="text-content-secondary text-sm">퇴사 대상</span>
       <span className="font-semibold">{selectedStaff?.name} ({ROLE_CONFIG[selectedStaff?.role]?.label})</span>
     </div>
     <div className="flex justify-between">
       <span className="text-content-secondary text-sm">퇴사 예정일</span>
       <span className="font-semibold">{formatDate(step1.resignDate)}</span>
     </div>
     {step1.resignReason && (
       <div className="flex justify-between">
         <span className="text-content-secondary text-sm">퇴사 사유</span>

       </div>
     )}
     {isFC && (
       <div className="flex justify-between">
         <span className="text-content-secondary text-sm">회원 재배정</span>
         <span>{Object.keys(assignments).length}명 배정 완료</span>
       </div>
     )}
     <div className="flex justify-between">
       <span className="text-content-secondary text-sm">스케줄 처리</span>
       <span>{scheduleAction === "transfer" ? "후임에게 이관" : "일괄 취소"}</span>
     </div>

   <div className="flex justify-between mt-6">

   setIsSubmitting(true);
   try {
     if (isFC) await reassignMembers({ assignments });
     await scheduleResignation(step1.staffId, {
       resignScheduledAt: step1.resignDate,
       resignReason: step1.resignReason,
       scheduleAction,
     });
     toast.success("퇴사 예정이 등록되었습니다.");
     moveToPage(974);
   } catch (err: any) {
     toast.error(`퇴사 예정 등록 실패: ${err.message}`);
   } finally {
     setIsSubmitting(false);
   }

## 📝 디스크립션

### 사용 시점 (Trigger)
- `currentStep === 4` (Step 3에서 다음 클릭)
- 모든 입력 완료 후 최종 확인 화면

### 필수 데이터
| 블록 | API | 동작 |
|------|-----|------|
| 회원 재배정 | `reassignMembers({ assignments })` | FC만 |
| 퇴사 예정 등록 | `scheduleResignation(staffId, data)` | 전원 |

### 인터랙션 (User Actions)
1. 요약 정보 확인
2. 퇴사 예정 등록 → handleSubmit()
3. 이전 → Step 3 복귀

### 비즈니스 룰
- 퇴사 예정 등록: `resignScheduledAt` + `resignReason` 저장
- FC 경우: reassignMembers 먼저 실행
- 성공 토스트: "퇴사 예정이 등록되었습니다."
- 실패 토스트: `"퇴사 예정 등록 실패: {error.message}"`

### 에지 케이스
- reassignMembers 실패 → scheduleResignation 실행 안 함
- 네트워크 오류 → isSubmitting=false 복원

### 접근성 (A11y)
- 퇴사 등록 버튼 `aria-busy="true"` 처리 중
- 경고 배너 `role="alert"`
