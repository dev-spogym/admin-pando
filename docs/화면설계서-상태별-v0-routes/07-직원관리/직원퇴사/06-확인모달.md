## 📝 디스크립션

### 사용 시점 (Trigger)
- `currentStep === 3`
- FC: Step 2 → Step 3 / 비FC: Step 1 → Step 3 (Step 2 건너뜀)

### 필수 데이터
| 블록 | 소스 | 설명 |
|------|------|------|
| 미래 스케줄 | API/supabase | 퇴사 예정일 이후 수업/PT 스케줄 |
| scheduleAction | 로컬 상태 | transfer / cancel |

### 인터랙션 (User Actions)
1. 스케줄 목록 확인 (케이스 A/B)
2. RadioGroup 선택 → scheduleAction 결정
3. 다음 → Step 4
4. 이전 → FC면 Step 2, 비FC면 Step 1

### 비즈니스 룰
- 스케줄 없으면 RadioGroup도 표시 (이관/취소 선택 필요)
- scheduleAction 기본값: "transfer" (후임에게 이관)
- 비FC 뒤로가기: Step 2 건너뜀 (`setCurrentStep(1)`)

### 에지 케이스
- 스케줄 fetch 실패 → 케이스 A로 처리 (스케줄 없음 가정)
- 후임 직원 없는 경우 → "이관" 선택 시 경고

### 접근성 (A11y)
- RadioGroup `role="radiogroup"` + `aria-label="스케줄 처리 방법"`
- 상태 배너 `role="status"`

### 연결 화면
- 이전: 04-재배치처리 (FC) / 02-퇴사정보입력 (비FC)
- 다음: 05-완료 (Step 4)
