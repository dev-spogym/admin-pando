```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-063 직원 근태 관리 — 상태: 수동 수정 (출퇴근 시간 직접 수정)

파일: src/app/staff/attendance/page.tsx

레이아웃:
- 02-기본-당일 레이아웃 유지
- DataTable 행에 수동 수정 모드 추가

컴포넌트 구조:
1. 행 우측 MoreVertical 메뉴 → "수동 수정" 선택 → 해당 행 편집 모드
2. 편집 모드 행:
   - 출근시간: <input type="time" value={editClockIn} onChange={...}/>
   - 퇴근시간: <input type="time" value={editClockOut} onChange={...}/>
   - 상태: <Select options={statusOptions} value={editStatus}/>
   - [저장] [취소] 버튼
3. handleManualEdit 함수:
   const handleManualEdit = async (attendanceId: number) => {
     const clockInISO = new Date(`${selectedDate}T${editClockIn}:00`).toISOString();
     const clockOutISO = editClockOut
       ? new Date(`${selectedDate}T${editClockOut}:00`).toISOString()
       : null;
     const workMinutes = editClockOut
       ? Math.round((new Date(clockOutISO!).getTime() - new Date(clockInISO).getTime()) / 60000)
       : null;
     const { error } = await supabase
       .from("staff_attendance")
       .update({ clockIn: clockInISO, clockOut: clockOutISO, workMinutes, status: editStatus })
       .eq("id", attendanceId);
     if (error) toast.error("수정 실패");
     else { toast.success("근태 기록이 수정되었습니다."); fetchAttendances(); }
   }
4. 수동 추가 (출근 기록 없는 직원):
   supabase.from("staff_attendance").insert({staffId, staffName, date, clockIn, branchId, status})

데이터:
- editMode: Record<number, boolean> (행별 편집 모드)
- editClockIn, editClockOut: string (HH:mm)
- editStatus: AttendanceStatus

인터랙션:
- MoreVertical 클릭 → 드롭다운 메뉴 → "수동 수정"
- time input 수정 → 실시간 미리보기
- [저장] → handleManualEdit() → fetchAttendances()
- [취소] → 편집 모드 해제

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: MoreVertical, Edit2, Check, X
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable 행 MoreVertical 메뉴에서 "수동 수정" 선택
- 센터장/최고관리자만 수동 수정 가능

### 필수 데이터
| 블록 | 테이블 | 동작 |
|------|--------|------|
| 근태 수정 | `staff_attendance` | UPDATE clockIn/clockOut/workMinutes/status |
| 근태 추가 | `staff_attendance` | INSERT (출근 기록 없는 직원) |

### 인터랙션 (User Actions)
1. MoreVertical → "수동 수정" → 행 편집 모드 진입
2. time input 수정 → clockIn/clockOut 업데이트
3. 상태 Select → 수동 상태 지정
4. [저장] → supabase update → 재조회
5. [취소] → 편집 모드 해제

### 비즈니스 룰
- 수동 수정은 센터장/최고관리자만 가능
- 수정 시 수정자/수정시각 로그 (선택적)
- clockOut < clockIn 입력 불가 (validation)

### 에지 케이스
- 퇴근 없이 출근만 기록 → clockOut=null, workMinutes=null
- 퇴근시간이 출근시간보다 이른 경우 → 에러 메시지

### 접근성 (A11y)
- time input `aria-label="출근시간 수정"`
- [저장] 버튼 `aria-label="근태 수정 저장"`
