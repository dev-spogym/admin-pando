## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-065 급여 명세서 — 상태: 이메일 발송 모달

파일: src/app/payroll/statements/page.tsx

컴포넌트 구조:
1. 이메일 발송 Modal (isEmailModalOpen):
   <Modal open={isEmailModalOpen} onClose={() => setIsEmailModalOpen(false)}
     title="급여 명세서 이메일 발송">

   {/* 발송 대상 정보 */}
   <div className="bg-surface-secondary rounded-lg p-4 mb-4">
     <p className="text-sm text-content-secondary">발송 대상</p>
     <p className="font-semibold">{emailTarget?.staffName}</p>
     <p className="text-sm text-content-secondary">{emailTarget?.email || "이메일 미등록"}</p>
   </div>

   {/* 이메일 미등록 경고 */}
   {!emailTarget?.email && (
     <div className="rounded-lg bg-state-warning/10 border border-state-warning/30 p-3 mb-4 flex gap-2">
       <AlertTriangle size={16} className="text-state-warning shrink-0 mt-0.5"/>
       <p className="text-sm text-content-primary">
         직원 이메일이 등록되지 않았습니다. 직원 정보에서 이메일을 등록하세요.
       </p>
     </div>
   )}

   {/* 이메일 내용 커스터마이즈 */}
   <div className="space-y-3">
     <div>
       <label className="text-sm font-medium">받는 이메일</label>
       <Input
         value={emailTo}
         onChange={e => setEmailTo(e.target.value)}
         placeholder="직원 이메일 주소"
         disabled={!!emailTarget?.email}
       />
     </div>
     <div>
       <label className="text-sm font-medium">제목</label>
       <Input
         value={emailSubject}
         onChange={e => setEmailSubject(e.target.value)}
         defaultValue={`[FitGenie] ${emailTarget?.year}년 ${emailTarget?.month}월 급여 명세서`}
       />
     </div>
     <div>
       <label className="text-sm font-medium">메모 (선택)</label>
       <Textarea
         value={emailMemo}
         onChange={e => setEmailMemo(e.target.value)}
         placeholder="추가 안내사항을 입력하세요"
         rows={2}
       />
     </div>
   </div>

   <div className="flex justify-end gap-2 mt-6">
     <Button variant="outline" onClick={() => setIsEmailModalOpen(false)}>취소</Button>
     <Button
       variant="primary"
       onClick={handleEmailSend}
       disabled={!emailTo || isSending}
     >
       {isSending ? <><Loader2 size={16} className="animate-spin"/> 발송 중...</> : <><Mail size={16}/> 발송하기</>}
     </Button>
   </div>
   </Modal>

2. handleEmailSend():
   setIsSending(true);
   try {
     // API 호출 (실제 이메일 발송 서비스 연동)
     await sendPayrollEmail({
       to: emailTo,
       subject: emailSubject,
       staffName: emailTarget?.staffName,
       month: `${emailTarget?.year}년 ${emailTarget?.month}월`,
       netSalary: emailTarget?.netSalary,
       memo: emailMemo,
     });
     toast.success(`${emailTarget?.staffName}에게 급여 명세서를 발송했습니다.`);
     setIsEmailModalOpen(false);
   } catch (err) {
     toast.error("이메일 발송 중 오류가 발생했습니다.");
   } finally {
     setIsSending(false);
   }

데이터:
- emailTarget: PayrollRecord
- emailTo: string (직원 이메일 또는 수동 입력)
- emailSubject: string
- emailMemo: string
- isSending: boolean

인터랙션:
- [발송하기] → handleEmailSend() → toast.success
- 이메일 미등록 → 수동 입력 가능
- 취소 → 모달 닫기

사용 유틸:
- toast from 'sonner'
- Modal from '@/components/ui/Modal'
- lucide-react: Mail, Loader2, AlertTriangle
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- DataTable 행의 [발송] 버튼 클릭
- `isEmailModalOpen === true`

### 필수 데이터
| 블록 | 소스 | 설명 |
|------|------|------|
| 발송 대상 | PayrollRecord | staffName, email |
| 이메일 API | 외부 서비스 | sendPayrollEmail() |

### 인터랙션 (User Actions)
1. 이메일 주소 확인 (자동 입력)
2. 제목/메모 수정 (선택)
3. 발송하기 → sendPayrollEmail() → toast.success
4. 취소 → 모달 닫기

### 비즈니스 룰
- 직원 이메일 자동 입력 (staff.email)
- 이메일 미등록 → 경고 + 수동 입력 허용
- 발송 완료 토스트: `"{직원명}에게 급여 명세서를 발송했습니다."`

### 에지 케이스
- 이메일 서비스 오류 → toast.error
- 빈 이메일 → 발송하기 버튼 disabled

### 접근성 (A11y)
- Modal `role="dialog"` + 포커스 트랩
- 발송 버튼 `aria-busy="true"` 발송 중
