| 항목 | 값 |
|------|----|
| 화면 ID | SCR-P002 |
| 상태 코드 | `save-complete` |
| 경로 | `/products/new` → `/products` |
| 역할 | 센터장 / 매니저 |
| 우선순위 | P0 |
| 이전 상태 | `03-저장중` (INSERT 성공) |
| 다음 상태 | `SCR-P001 02-기본` (상품 목록, 신규 상품 포함) |

## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-P002 상품 등록 — 상태: 저장 완료 후 목록 화면으로 이동

파일: src/app/products/new/page.tsx

완료 플로우:
// INSERT 성공 직후
toast.success(`'${formData.name}' 상품이 등록되었습니다.`);
setIsSaving(false);
setIsDirty(false); // beforeunload 경고 방지
moveToPage('/products'); // router.push('/products')

목록 화면 복귀 후 (/products):
- 신규 등록 상품이 목록 최상단에 표시 (createdAt DESC 정렬)
- 해당 행이 자동 하이라이트: bg-primary/5 border-l-2 border-primary (2초 후 해제)
- 통계 카드 재계산 (activeCount +1, total +1)
- toast.success는 /products 화면에서도 유지 (Sonner 기본 지속시간 4초)

선택적 플로우 (연속 등록 모드):
<Button variant="outline" onClick={handleSaveAndAddNew}>
  저장 후 다음 상품 등록
</Button>
// 클릭 시: 저장 완료 → 폼 초기화 → /products/new 유지

사용 유틸:
- moveToPage from '@/internal'
- toast from 'sonner'
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `03-저장중` 상태에서 Supabase INSERT 응답 성공 시
- 자동으로 `/products`로 라우팅됨

### 필수 데이터
- 없음 (이동 후 SCR-P001이 재fetch)

### 인터랙션 (User Actions)
1. 자동 이동 → `/products` (사용자 조작 불필요)
2. "저장 후 다음 상품 등록" → 폼 초기화 + `/products/new` 유지

### 비즈니스 룰
- isDirty=false 설정 후 이동 (beforeunload 경고 방지)
- 목록 복귀 시 신규 상품 하이라이트 (2초)
- Sonner toast는 화면 이동 후에도 유지됨

### 에지 케이스
- moveToPage 실패 시 → window.location.href fallback

### 접근성 (A11y)
- 이동 중 `aria-live="assertive"` 영역에 "상품이 등록되었습니다" 공지

### 연결 화면
- 이전: `03-저장중`
- 다음: `SCR-P001 02-기본` (상품 목록)
