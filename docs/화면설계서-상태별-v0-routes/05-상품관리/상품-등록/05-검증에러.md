파일: src/app/products/new/page.tsx

검증 로직:
interface FieldError { field: string; message: string; }

const validateForm = (data: ProductFormData): FieldError[] => {
  const errors: FieldError[] = [];
  if (!data.name.trim()) errors.push({ field: 'name', message: '상품명을 입력해주세요.' });
  if (!data.productType) errors.push({ field: 'productType', message: '상품 타입을 선택해주세요.' });
  if (!data.price || data.price <= 0) errors.push({ field: 'price', message: '기본가를 입력해주세요.' });
  if (data.cashPrice && data.cashPrice < 0) errors.push({ field: 'cashPrice', message: '현금가는 0 이상이어야 합니다.' });
  return errors;
};

에러 UI:
- 오류 필드 Input: border-red-500 + ring-1 ring-red-400
- 오류 메시지: <p className="text-xs text-red-500 mt-1 flex items-center gap-1"><AlertCircle size={12}/>{message}</p>
- 폼 상단 인라인 배너:
  <div className="flex items-center gap-2 p-3 bg-red-50 border border-red-200 rounded-lg text-sm text-red-700">
    <AlertCircle size={16} />
    <span>{errors.length}개 항목을 확인해주세요.</span>
  </div>

에러 상태 초기화:
- 해당 필드 onChange 시 → 해당 필드 오류 제거
- 저장 버튼 재클릭 → 전체 재검증

포커스 처리:
// 저장 시도 후 첫 번째 오류 필드로 스크롤 + 포커스
const firstErrorField = document.getElementById(`field-${errors[0].field}`);
firstErrorField?.scrollIntoView({ behavior: 'smooth', block: 'center' });
firstErrorField?.focus();

사용 유틸:

## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장 버튼 클릭 시 클라이언트 검증 실패
- 필수값(상품명, 상품타입, 기본가) 미입력 또는 유효하지 않은 값

### 필수 데이터
- 없음 (클라이언트 사이드 검증, DB 요청 없음)

### 인터랙션 (User Actions)
1. 오류 배너 확인 → 해당 필드로 스크롤 이동
2. 오류 필드 수정 → 해당 필드 오류 즉시 제거
3. 저장 버튼 재클릭 → 전체 재검증 → 통과 시 `03-저장중`

### 비즈니스 룰
- 필수값: 상품명, 상품타입, 기본가(price > 0)
- 선택값: 현금가, 카드가, 기간, 횟수, 설명 등
- 가격 음수 → 개별 필드 오류

### 에지 케이스
- 상품명 공백만 입력 → trim() 후 빈 문자열로 오류 처리
- 가격에 문자 입력 → Input type="number"로 방지

### 접근성 (A11y)
- 오류 필드 `aria-invalid="true"`, `aria-describedby="{field}-error"`
- 오류 배너 `role="alert"`, `aria-live="assertive"`
- 첫 번째 오류 필드로 자동 포커스 이동

### 연결 화면
- 이전: `02-입력중`
- 다음: `02-입력중` (수정 중) / `03-저장중` (수정 완료 후 저장)
