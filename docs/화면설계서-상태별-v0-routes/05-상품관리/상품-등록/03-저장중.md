Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-P002 상품 등록 — 상태: 저장 요청 중 (isSaving=true)

파일: src/app/products/new/page.tsx

저장 로직:
const handleSubmit = async () => {
  // 1. 클라이언트 검증
  const errors = validateForm(formData);
  if (errors.length > 0) { setErrors(errors); return; } // → 05-검증에러

  // 2. 저장 시작
  setIsSaving(true);

  // 3. Supabase INSERT
  const { data, error } = await supabase.from('products').insert({
    branchId,
    name: formData.name.trim(),
    productType: formData.productType,
    category: formData.category,
    sportType: formData.sportType || null,
    price: formData.price,
    cashPrice: formData.cashPrice || formData.price,
    cardPrice: formData.cardPrice || null,
    duration: formData.duration || null,
    sessions: formData.sessions || null,
    totalCount: formData.sessions || null,
    description: formData.description || null,
    tag: formData.tag || null,
    isActive: true,
    kioskVisible: formData.kioskVisible,
    deductionType: formData.deductionType || null,
    holdingEnabled: formData.holdingEnabled,
    transferEnabled: formData.transferEnabled,
    pointAccrual: formData.pointAccrual,
    salesChannel: formData.salesChannel || 'ALL',
    usage_restrictions: formData.usage_restrictions,
    productGroupId: formData.productGroupId || null,
    suspendLimit: formData.suspendLimit || null,
    dailyUseLimit: formData.dailyUseLimit || null,
  }).select().single();

  if (error) {
    toast.error('상품 저장에 실패했습니다. 다시 시도해주세요.');
    setIsSaving(false);
    return;
  }

  // 4. 성공
  toast.success(`'${formData.name}' 상품이 등록되었습니다.`);
  moveToPage('/products'); // → 04-완료이동
};

UI 상태 (isSaving=true):
- 저장 버튼: disabled + Loader2 스피너 + "저장 중..." 텍스트
- 취소 버튼: disabled
- 모든 폼 입력 필드: pointer-events-none opacity-60
- PageHeader: opacity-80

저장 버튼 스타일:
<Button variant="primary" disabled={isSaving}>
  {isSaving ? <><Loader2 className="animate-spin w-4 h-4 mr-2" />저장 중...</> : '저장'}
</Button>

사용 유틸:

## 📝 디스크립션

### 사용 시점 (Trigger)
- 저장 버튼 클릭 후 클라이언트 검증 통과 시
- Supabase INSERT 요청이 진행 중인 동안

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 등록 | `products` | INSERT 실행 |

### 인터랙션 (User Actions)
1. 저장 중 — 모든 인터랙션 차단 (pointer-events-none)
2. 성공 → toast.success + `/products` 이동 (`04-완료이동`)
3. 실패 → toast.error + isSaving=false → 폼 입력 재활성화

### 비즈니스 룰
- 현금가 미입력 → price 값으로 대체 저장
- isActive=true 고정으로 신규 등록
- salesChannel 미선택 → 'ALL' 기본값

### 에지 케이스
- 네트워크 타임아웃 (15초) → toast.error, 폼 원복
- 중복 상품명 → DB unique 제약 없음, 허용

### 접근성 (A11y)
- 저장 버튼 `aria-busy="true"` (진행 중)
- 전체 폼 `inert` 속성 적용 (저장 중 포커스 차단)

### 연결 화면
- 이전: `02-입력중`
- 다음: `04-완료이동` (성공) / 폼 원복 (실패)
