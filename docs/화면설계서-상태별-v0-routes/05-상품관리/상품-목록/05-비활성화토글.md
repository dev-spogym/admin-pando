## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-P001 상품 목록 — 상태: 상품 비활성화/활성화 토글 처리 중

파일: src/app/products/page.tsx (ProductDetailPanel 연동)

토글 플로우:
1. ProductDetailPanel 내 isActive Toggle 클릭
2. supabase.from('products').update({isActive: !current}).eq('id', productId)
3. 저장 중: 해당 행 opacity-60 + Loader2 스피너 (16px inline)
4. 완료: products 배열 클라이언트 업데이트 (immutable replace)
5. StatCard 카운터 즉시 업데이트 (activeCount / inactiveCount)
6. toast.success(`'${name}' 상품이 ${isActive ? '활성화' : '비활성화'}되었습니다.`)

UI 상태:
- 비활성 상품 행: 테이블 행 text-gray-400, 상태 배지 StatusBadge(default, '미사용')
- 활성 상품 행: 정상 색상, StatusBadge(mint, '사용')
- 토글 진행 중 행: opacity-60 + cursor-not-allowed

인터랙션:
- 토글 성공 → products state 업데이트, 통계 카드 recompute
- 토글 실패 → toast.error('상태 변경에 실패했습니다. 다시 시도해주세요.')
- isActive=false 상품도 목록에 유지 (제거 안 함)

권한:
- canEdit=false → Toggle 비활성화 (disabled)

사용 유틸:
- supabase from '@/lib/supabase'
- toast from 'sonner'
- lucide-react: Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- ProductDetailPanel 또는 테이블 행의 상태 토글 버튼 클릭 시
- canEdit=true인 역할(센터장/매니저)에서만 트리거 가능

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 상태 수정 | `products` | `UPDATE isActive WHERE id = ?` |

### 인터랙션 (User Actions)
1. 토글 클릭 → 즉시 로딩 UI (opacity-60, 스피너)
2. 성공 → 상태 반전, toast.success
3. 실패 → 원래 상태 유지, toast.error

### 비즈니스 룰
- 비활성 상품: 키오스크에서 미노출 (`kioskVisible` 자동 false)
- 비활성 상품도 기존 계약에는 유지 (삭제가 아닌 soft-disable)
- 통계 카드 activeCount/inactiveCount 즉시 recompute

### 에지 케이스
- 네트워크 오류 → optimistic update 롤백, 원래 상태 복원
- 이미 계약된 상품 비활성화 → 허용 (계약 자체는 유효 유지)

### 접근성 (A11y)
- 토글 버튼 `aria-checked={isActive}`, `aria-label="상품 활성화 상태 변경"`
- 진행 중 `aria-busy="true"`

- 다음: `02-기본` (토글 완료 후)
