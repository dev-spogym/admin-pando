```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-P001 상품 목록 — 상태: 기본 (데이터 정상 로드, 상세패널 닫힘)

파일: src/app/products/page.tsx

레이아웃:
- <AppLayout> 데스크톱 레이아웃
- <PageHeader
    title="상품 관리"
    subtitle="센터에서 판매하는 이용권, PT, GX 및 기타 상품을 관리합니다."
    actions={[
      <TabNav key="mainTab" tabs={[{key:'products',label:'상품 목록'},{key:'groups',label:'분류 관리'}]} />,
      isSuperAdmin && <Button variant="outline" size="sm" icon={Globe}>전 지점 배포</Button>,
      canEdit && <Button variant="primary" size="sm" icon={Plus}>상품 등록</Button>,
      <Button variant="ghost" size="sm">Excel</Button>
    ]}
  />
- <StatCardGrid> 4열:
    StatCard(전체상품, products.length, Package, peach)
    StatCard(활성상품, activeCount, CheckCircle, mint)
    StatCard(비활성, inactiveCount, XCircle, default)
    StatCard(카테고리수, categoryCount, LayoutGrid, default)
- 상품타입 TabNav: [전체|회원권|수강권|대여권|일반] (각 탭에 건수 배지)
- 분류 필터 pill 버튼 그룹 (전체 + product_groups 동적 목록)
- 종목 필터 pill 버튼 그룹 (전체|헬스|필라테스|요가|수영|복싱|크로스핏|기타)
- 검색창 (w-48, placeholder="상품명 검색", Search 아이콘)
- <DataTable> 마스터-디테일 (panelOpen=false → w-full, panelOpen=true → w-[55%])

컴포넌트 구조:
1. [통계 카드] — 4개 StatCard
2. [상품타입 TabNav] — productTypeFilter 상태 제어
3. [분류 + 종목 필터] — categoryFilter, sportFilter 상태 제어
4. [검색창] — searchQuery 상태 200ms debounce
5. [DataTable] — filteredProducts 렌더링, 행 클릭 시 selectedProduct 설정 + panelOpen=true
6. [ProductDetailPanel] — panelOpen ? 표시 : hidden

DataTable 컬럼 (panelOpen=false 기준):
- 상품명: 타입배지(9px) + 패키지배지(📦) + 상품명(12px)
- 현금가: formatKRW(cashPrice ?? price) right-aligned
- 카드가: formatKRW(cardPrice) 또는 '-'
- 기간: `${duration}일` 또는 '-'
- 횟수: `${sessions}회` 또는 '-'
- 월~일(7열): availableDays ✓(green-600)/✗(red-400)
- 시간: `시작~종료` 또는 '전체'
- 홀딩/양도/포인트: ✓/✗
- 판매: `${N}건`
- 상태: <StatusBadge> 사용(mint)/미사용(default)

데이터:
- interface ProductRow { id, branchId, name, category, productType, price, cashPrice, cardPrice,
    duration, sessions, isActive, kioskVisible, classType, deductionType, sportType,
    holdingEnabled, transferEnabled, pointAccrual, salesChannel, usage_restrictions,
    productGroupId, suspendLimit, dailyUseLimit, description, tag, createdAt }
- supabase.from('products').select('*').eq('branchId', branchId).order('createdAt', {ascending:false})
- supabase.from('product_groups').select('*').eq('branchId', branchId).order('sortOrder')
- Promise.all 병렬 fetch

필터 로직:
- productTypeFilter: product.productType === key (all → 전체)
- categoryFilter: product_groups.id 또는 category 문자열 매칭
- sportFilter: product.sportType === value
- searchQuery: product.name.toLowerCase().includes(q)

권한:
- canEdit = hasFeature(authUser.role, 'productEdit', authUser.isSuperAdmin)
- isSuperAdmin = authUser.isSuperAdmin

사용 유틸:
- useAuthStore from '@/stores/authStore'
- supabase from '@/lib/supabase'
- getBranchId from '@/lib/getBranchId'
- hasFeature from '@/lib/permissions'
- formatKRW from '@/lib/format'
- exportToExcel from '@/lib/exportExcel'
- lucide-react: Package, CheckCircle, XCircle, LayoutGrid, Plus, Search, Globe
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 products 배열 1건 이상 존재하는 경우
- 필터/검색 초기화 상태

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 목록 | `products` | `branchId = ?`, ORDER BY `createdAt DESC` |
| 상품 분류 | `product_groups` | `branchId = ?`, ORDER BY `sortOrder` |

### 인터랙션 (User Actions)
1. 상품타입 탭 클릭 → `03-카테고리필터` 상태
2. 분류 pill 클릭 → categoryFilter 변경, 테이블 즉시 재렌더
3. 종목 pill 클릭 → sportFilter 변경
4. 검색어 입력 → 200ms debounce → 실시간 필터
5. 행 클릭 → selectedProduct 설정, ProductDetailPanel 오픈 (우측 45%)
6. 상품 등록 버튼 → ProductDetailPanel 신규 모드 오픈
7. Excel 버튼 → exportToExcel() 호출, toast.success(`${N}건 엑셀 다운로드 완료`)
8. 전 지점 배포 (슈퍼관리자) → 배포 확인 모달

### 비즈니스 룰
- canEdit=false → 상품 등록 버튼 숨김, 행 클릭 시 조회 전용 패널
- 슈퍼관리자만 "전 지점 배포" 버튼 노출
- 통계 카드: activeCount = products.filter(p => p.isActive).length

### 에지 케이스
- products 0건 → `06-빈상태` 전환
- 필터 결과 0건 → 테이블 내 "검색 결과가 없습니다" 인라인 메시지

### 접근성 (A11y)
- DataTable에 `role="grid"`, 각 행 `role="row"`
- 상태 배지 `aria-label="사용 중"` / `aria-label="미사용"`

### 연결 화면
- 이전: `01-로딩`
- 다음: 행 클릭 → 상품 수정 패널 / 등록 버튼 → 상품 등록 패널 / 필터 → `03-카테고리필터`
