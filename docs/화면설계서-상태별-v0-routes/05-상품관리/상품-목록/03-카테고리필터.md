```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-P001 상품 목록 — 상태: 카테고리/타입 필터 활성

레이아웃: 02-기본과 동일 구조, 아래 변경사항 적용

필터 상태 변경:
- productTypeFilter: 'MEMBERSHIP' | 'LESSON' | 'RENTAL' | 'GENERAL' (기본 'all'에서 변경됨)
- categoryFilter: 선택된 product_group.id 또는 category 문자열
- sportFilter: 선택된 종목 (선택적 조합)

UI 차이점:
- 선택된 탭(productTypeFilter): bg-primary text-white (강조)
- 선택된 분류 pill: bg-primary/10 text-primary border border-primary/30 (강조)
- 선택된 종목 pill: bg-primary/10 text-primary border border-primary/30
- 테이블 상단 "총 {N}개 상품" 카운트 업데이트
- 필터 초기화 버튼 노출: 필터 활성 시 X 버튼 또는 "초기화" 텍스트 버튼 표시

필터 로직:
const filteredProducts = useMemo(() => {
  return products.filter(p => {
    const typeMatch = productTypeFilter === 'all' || p.productType === productTypeFilter;
    const catMatch = categoryFilter === 'all' || p.productGroupId === categoryFilter || p.category === categoryFilter;
    const sportMatch = sportFilter === 'all' || p.sportType === sportFilter;
    const searchMatch = !searchQuery || p.name.toLowerCase().includes(searchQuery.toLowerCase());
    return typeMatch && catMatch && sportMatch && searchMatch;
  });
}, [products, productTypeFilter, categoryFilter, sportFilter, searchQuery]);

인터랙션:
- 다른 탭/pill 클릭 → 해당 필터만 교체, 복합 필터 유지
- 필터 초기화 클릭 → 모든 필터 초기화, 02-기본 상태로
- 결과 0건 → 06-빈상태

사용 유틸:
- useAuthStore, supabase, formatKRW, hasFeature, lucide-react
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 상품타입 TabNav에서 특정 타입(회원권/수강권/대여권/일반) 선택 시
- 분류 pill 버튼에서 특정 분류 선택 시
- 종목 pill에서 특정 종목 선택 시
- 복합 필터 조합 가능

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 목록 | `products` | `branchId = ?` (클라이언트 필터링) |
| 상품 분류 | `product_groups` | `branchId = ?` |

### 인터랙션 (User Actions)
1. 탭 클릭 → productTypeFilter 변경, filteredProducts 즉시 재계산
2. 분류 pill → categoryFilter 변경
3. 종목 pill → sportFilter 변경
4. 필터 초기화 → 전체 필터 reset
5. 행 클릭 → ProductDetailPanel 오픈 (필터 상태 유지)

### 비즈니스 룰
- 복합 필터: AND 조건 (productType AND category AND sport AND search)
- 통계 카드는 필터 무관하게 전체 기준 유지 (filteredProducts 기준 아님)
- 분류 필터: product_groups.id 매핑 우선, 없으면 products.category 직접 비교

### 에지 케이스
- 필터 조합 결과 0건 → `06-빈상태`
- 분류 그룹 삭제 후 해당 필터 선택 시 → 결과 없음으로 처리

### 접근성 (A11y)
- 선택된 필터 pill: `aria-pressed="true"`
- 초기화 버튼: `aria-label="필터 초기화"`
