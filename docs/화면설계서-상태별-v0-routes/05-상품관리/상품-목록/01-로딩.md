## 📝 디스크립션

### 사용 시점 (Trigger)
- 사용자가 사이드바에서 "상품 관리"를 클릭하여 `/products` 경로에 진입한 직후
- `useEffect` 내 Supabase 쿼리가 실행되기 시작한 시점부터 응답 수신 전까지

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 목록 | `products` | `branchId = ?` |
| 상품 분류 | `product_groups` | `branchId = ?` |

### 인터랙션 (User Actions)
1. 로딩 중 — 모든 버튼/필터 비활성화 (pointer-events-none)
2. 로딩 완료 → `02-기본` 상태로 자동 전환

### 비즈니스 룰
- `Promise.all`로 products + product_groups 병렬 fetch
- getBranchId()로 현재 지점 ID 추출 후 쿼리

### 에지 케이스
- 네트워크 timeout (10초) → `04-에러` 상태 전환
- branchId 없음 → 빈 배열로 처리, toast.error 노출

### 접근성 (A11y)
- `aria-busy="true"` 속성을 main 컨테이너에 부여
- 스켈레톤 요소에 `aria-hidden="true"`

### 연결 화면
- 이전: 사이드바 네비게이션
- 다음: `02-기본` (정상) / 에러 시 toast 노출
