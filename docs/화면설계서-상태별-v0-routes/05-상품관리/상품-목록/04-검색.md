```
검색 로직:
const [searchQuery, setSearchQuery] = useState('');
const [debouncedSearch, setDebouncedSearch] = useState('');

useEffect(() => {
  const timer = setTimeout(() => setDebouncedSearch(searchQuery), 200);
  return () => clearTimeout(timer);
}, [searchQuery]);

const filteredProducts = useMemo(() => {
  if (!debouncedSearch) return baseFiltered;
  return baseFiltered.filter(p =>
    p.name.toLowerCase().includes(debouncedSearch.toLowerCase()) ||
    p.tag?.toLowerCase().includes(debouncedSearch.toLowerCase())
  );
}, [baseFiltered, debouncedSearch]);

인터랙션:
- 입력 → 200ms 후 필터 적용
- X 버튼 → searchQuery 초기화, 02-기본
- ESC 키 → 검색창 blur, 검색어 초기화
- 결과 0건 → 06-빈상태

사용 유틸:
- useAuthStore, supabase, formatKRW, hasFeature
- lucide-react: Search, X
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 검색창에 1자 이상 입력 후 200ms 경과 시
- 기존 카테고리/타입 필터와 AND 조합 가능

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 목록 | `products` | `branchId = ?` (클라이언트 사이드 필터) |

### 인터랙션 (User Actions)
1. 검색어 입력 → 200ms debounce → name/tag 포함 검색
2. 검색창 X 버튼 → 초기화
3. ESC → 검색 해제
4. 결과 행 클릭 → ProductDetailPanel 오픈

### 비즈니스 룰
- name + tag 필드 모두 검색 대상
- 대소문자 무시 (toLowerCase)
- 검색은 클라이언트 사이드 필터 (Supabase 재요청 없음)

### 에지 케이스
- 특수문자 입력 → toLowerCase 이외 이스케이프 처리 불필요 (contains 검색)
- 결과 0건 → `06-빈상태` 화면

### 접근성 (A11y)
- 검색창 `aria-label="상품명 검색"`, `role="searchbox"`
- 결과 수 변경 시 `aria-live="polite"` 영역에 "{N}개 결과" 공지

### 연결 화면
- 이전: `02-기본` / `03-카테고리필터`
- 다음: `06-빈상태` (0건) / `02-기본` (검색 해제)
