```
<Modal
  isOpen={showDeleteModal}
  onClose={() => setShowDeleteModal(false)}
  title="상품 삭제"

>

        </p>

    >
      {isDeleting ? <Loader2 className="animate-spin w-4 h-4 mr-1" /> : null}
      삭제
    </Button>
  </div>
</Modal>

삭제 로직:
const handleDelete = async () => {
  if (deleteConfirmText !== formData.name) return;
  setIsDeleting(true);
  const { error } = await supabase.from('products').delete().eq('id', productId);
  if (error) {
    toast.error('상품 삭제에 실패했습니다.');
    setIsDeleting(false);
    return;
  }
  toast.success(`'${formData.name}' 상품이 삭제되었습니다.`);
  moveToPage('/products');
};

사용 유틸:
- supabase, moveToPage, toast
- lucide-react: AlertTriangle, Loader2
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `02-기본-폼`의 삭제 버튼 클릭 시
- canEdit=true인 역할만 접근 가능

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 삭제 | `products` | `DELETE WHERE id = ?` |

### 인터랙션 (User Actions)
1. 모달 열림 → 배경 dim, focus trap
2. 상품명 입력 → deleteConfirmText === formData.name 시 삭제 버튼 활성화
3. 삭제 버튼 → handleDelete() 실행
4. 취소 → 모달 닫힘, `02-기본-폼` 복귀

### 비즈니스 룰
- 상품명 타이핑 확인 필수 (오타 방지)
- 삭제 후 기존 contracts는 유지 (상품 참조는 존재하나 조회만 가능)
- Hard delete (soft delete 없음)

### 에지 케이스
- 이미 삭제된 상품 → DB 오류 → toast.error
- 연계된 진행 중 계약 있어도 삭제 허용 (계약은 별도 관리)

### 접근성 (A11y)
- 모달 `role="dialog"`, `aria-labelledby="modal-title"`, `aria-modal="true"`
- focus trap: 모달 내 요소만 Tab 순환
- ESC → 모달 닫기

- 다음: `SCR-P001 02-기본` (삭제 성공) / `02-기본-폼` (취소)
