## 📝 디스크립션

### 사용 시점 (Trigger)
- `01-로딩` 완료 후 product 데이터 정상 수신

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 상품 상세 | `products` | `id = ?` (이미 로드됨) |
| 분류 목록 | `product_groups` | `branchId = ?` (이미 로드됨) |

### 인터랙션 (User Actions)
1. 필드 수정 → isDirty=true, 저장 버튼 활성화
2. 저장 버튼 → `03-저장중`
3. 삭제 버튼 → `05-삭제확인` 모달
4. 취소 버튼 → isDirty=true 시 경고 모달 / false 시 즉시 `/products`

### 비즈니스 룰
- canEdit=false → 모든 필드 readOnly, 저장/삭제 버튼 숨김
- 저장 버튼: isDirty=false 시 disabled
- isActive 토글은 폼 최상단에 배치 (가시성 확보)

### 에지 케이스
- 다른 관리자가 동시 수정 → 저장 시 최신 데이터로 덮어씌움 (last-write-wins)
- 상품 타입 변경 → 관련 필드 동적 재표시

### 접근성 (A11y)
- readOnly 필드: `aria-readonly="true"`
- 삭제 버튼: `aria-label="상품 삭제"` (danger 색상으로 위험 표시)

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-저장중` / `05-삭제확인` / `/products`
