## 발견된 이슈 목록

### Critical (서비스 불가)

#### [QA-001] 상품 등록/수정 페이지 렌더링 크래시 — ✅ 정상 확인 (HMR 일시적 이슈)
- **심각도**: Critical
- **페이지**: 상품 등록 (/products/new), 상품 수정 (/products/edit)
- **현상**: 페이지 진입 시 흰 화면 (ErrorBoundary에 걸림)
- **콘솔 에러**: `Invalid hook call` → `Cannot read properties of null (reading 'useRef')` at `ProductForm.tsx:93`
- **원인**: Vite HMR 일시적 이슈 (새로고침 후 정상 동작 확인)
- **해결**: 런타임 검증 완료 — 상품 유형 선택 → 폼 렌더링 정상

#### [QA-002] 급여 관리 - DB 컬럼 누락으로 데이터 로드 실패 — ✅ 수정 완료
- **심각도**: Critical
- **페이지**: 급여 관리 (/payroll)
- **현상**: "급여 데이터를 불러오지 못했습니다" 에러 표시, 0건
- **콘솔 에러**: `column payroll.branchId does not exist` (HTTP 400)
- **원인**: payroll 테이블에 branchId 컬럼이 없음 — staff 테이블을 통해 조인 필요
- **해결**: `staff!inner` 조인으로 branchId 필터링. 런타임 검증 완료 — 5명 급여 데이터 정상 로드

### High (핵심 기능 오류)

#### [QA-003] 급여 명세서 - DB 컬럼 누락으로 로드 실패 — ✅ 수정 완료
- **심각도**: High
- **페이지**: 급여 명세서 (/payroll/statements)
- **현상**: 직원 목록은 표시되나, 금액 데이터 모두 "-" 또는 0
- **콘솔 에러**: `column payroll.paymentDate does not exist` (HTTP 400)
- **원인**: 컬럼명이 `paidAt`인데 `paymentDate`로 조회 + branchId 조인 누락
- **해결**: `paidAt` 컬럼으로 변경 + `staff!inner` 조인. 런타임 검증 완료 — 5명 명세서 정상 로드

#### [QA-004] 지점 관리 - 모든 지점 "폐점" 상태 표시
- **심각도**: High
- **페이지**: 지점 관리 (/branches)
- **현상**: 3개 지점 모두 "폐점" 상태, 활성 지점 0개, 회원 수/직원 수 모두 0
- **기대**: 강남 본점/서초점/송파점이 "운영중"으로 표시, 각 지점별 회원/직원 수 표시
- **원인**: 지점 상태 필드가 기본값 또는 `status` 컬럼 매핑 오류

#### [QA-005] 락커 관리 - 데이터 0건 표시
- **심각도**: High
- **페이지**: 락커 현황 (/locker)
- **현상**: 전체 락커 수 0, 사용중 0, 만료임박 0, 고장 0, "검색 결과가 없습니다"
- **기대**: 시드 데이터의 락커 정보가 표시되어야 함
- **원인**: 락커 테이블에 데이터 미삽입 또는 API 쿼리 오류

#### [QA-006] 매출 페이지 - SearchFilter 콘솔 경고 — ✅ 수정 완료
- **심각도**: High
- **페이지**: 매출 현황 (/sales)
- **현상**: 페이지 동작은 하나, React 경고 발생
- **콘솔 에러**: `The value prop supplied to <select> must be a scalar value if multiple is false` at `SearchFilter.tsx`
- **원인**: Sales.tsx에서 유형/상태 필터가 배열 값인데 filter type이 `select`로 정의됨
- **해결**: filter type을 `multiSelect`로 변경. 런타임 검증 완료 — 경고 사라짐

## 2차 상호작용 테스트 (CRUD / 검색 / 필터 / 권한)

### Critical

#### [QA-016] 슈퍼관리자(superadmin) 로그인 불가 — ✅ 수정 완료
- **심각도**: Critical
- **페이지**: 로그인 (/login)
- **현상**: superadmin 계정으로 로그인 시 branchId가 null → "null" 문자열로 변환되어 오류
- **원인**: superadmin의 branchId가 null인데 Login.tsx에서 null 처리 누락
- **해결**: isSuperAdmin 체크 추가, branchId null 시 선택된 지점 또는 빈 문자열 사용, branchName 기본값 '전체 지점'

### High

#### [QA-017] 회원 목록 - 검색 및 상태 필터 미동작 — ✅ 정상 확인 (테스트 오류)
- **심각도**: High
- **페이지**: 회원 목록 (/members)
- **현상**: 초기 QA에서 검색/필터가 미동작으로 보고
- **원인**: 테스트 시 헤더의 글로벌 검색 input에 입력한 것으로 판명 (SearchFilter 검색 input과 별개)
- **해결**: 런타임 검증 완료 — SearchFilter 검색 input에 "김민준" 입력 시 1건만 정상 필터링

### Medium

#### [QA-018] 회원 수정 - 기존 데이터 미로딩 — ✅ 수정 완료
- **심각도**: Medium
- **페이지**: 회원 수정 (/members/edit)
- **현상**: 회원 상세에서 "수정" 클릭 시 수정 폼이 빈 상태로 표시
- **원인**: `window.location.search` 대신 React Router `useSearchParams` 사용 필요 + 초기 isEditMode 상태 설정 누락
- **해결**: useSearchParams로 id 파라미터 읽기 + isEditMode 초기값 설정. 런타임 검증 완료 — 이름/연락처/생년월일 정상 프리필

#### [QA-019] 직원 수정 - 기존 데이터 미로딩 — ✅ 정상 확인
- **심각도**: Medium
- **페이지**: 직원 수정 (/staff/new?id=X)
- **현상**: 초기 QA에서 미로딩으로 보고
- **해결**: 런타임 검증 완료 — 이름(김태희), 역할(trainer), 입사일(2023-03-15), 이메일 정상 프리필. 코드 변경 불필요

#### [QA-020] 회원 등록 - 필수값 입력해도 다음 단계 비활성화
- **심각도**: Medium
- **페이지**: 회원 등록 (/members/new)
- **현상**: 이름, 성별, 연락처(필수 3개) 입력 후에도 "다음 단계" 버튼이 disabled 상태 유지
- **기대**: 필수값 모두 입력 시 버튼이 활성화되어야 함
- **원인**: 추가 필수 검증 조건이 있거나, 입력값 validation 로직 오류
