## 발견된 이슈 목록

### Critical (서비스 불가)

#### [QA-001] 상품 등록/수정 페이지 렌더링 크래시
- **심각도**: Critical
- **페이지**: 상품 등록 (/products/new), 상품 수정 (/products/edit)
- **현상**: 페이지 진입 시 흰 화면 (ErrorBoundary에 걸림)
- **콘솔 에러**: `Invalid hook call` → `Cannot read properties of null (reading 'useRef')` at `ProductForm.tsx:93`
- **원인**: `useForm` 훅 호출 위치 또는 React 중복 인스턴스 문제
- **영향**: 새 상품 등록 및 기존 상품 수정 불가

#### [QA-002] 급여 관리 - DB 컬럼 누락으로 데이터 로드 실패
- **심각도**: Critical
- **페이지**: 급여 관리 (/payroll)
- **현상**: "급여 데이터를 불러오지 못했습니다" 에러 표시, 0건
- **콘솔 에러**: `column payroll.branchId does not exist` (HTTP 400)
- **원인**: API에서 `branchId` 컬럼으로 조회하나, 실제 DB 컬럼명이 다름 (snake_case `branch_id` 등)
- **영향**: 급여 데이터 전체 조회 불가

---

### High (핵심 기능 오류)

#### [QA-003] 급여 명세서 - DB 컬럼 누락으로 로드 실패
- **심각도**: High
- **페이지**: 급여 명세서 (/payroll/statements)
- **현상**: 직원 목록은 표시되나, 금액 데이터 모두 "-" 또는 0
- **콘솔 에러**: `column payroll.paymentDate does not exist` (HTTP 400)
- **원인**: API에서 `paymentDate` 컬럼으로 조회하나, 실제 컬럼명이 다름 (`payment_date` 등)
- **영향**: 급여 명세서 금액 데이터 조회 불가

#### [QA-004] 지점 관리 - 모든 지점 "폐점" 상태 표시
- **심각도**: High
- **페이지**: 지점 관리 (/branches)
- **현상**: 3개 지점 모두 "폐점" 상태, 활성 지점 0개, 회원 수/직원 수 모두 0
- **기대**: 강남 본점/서초점/송파점이 "운영중"으로 표시, 각 지점별 회원/직원 수 표시
- **원인**: 지점 상태 필드가 기본값 또는 `status` 컬럼 매핑 오류

#### [QA-005] 락커 관리 - 데이터 0건 표시
- **심각도**: High
- **페이지**: 락커 현황 (/locker)
- **현상**: 전체 락커 수 0, 사용중 0, 만료임박 0, 고장 0, "검색 결과가 없습니다"
- **기대**: 시드 데이터의 락커 정보가 표시되어야 함
- **원인**: 락커 테이블에 데이터 미삽입 또는 API 쿼리 오류

#### [QA-006] 매출 페이지 - SearchFilter 콘솔 경고
- **심각도**: High
- **페이지**: 매출 현황 (/sales)
- **현상**: 페이지 동작은 하나, React 경고 발생
- **콘솔 에러**: `The value prop supplied to <select> must be a scalar value if multiple is false` at `SearchFilter.tsx`
- **원인**: SearchFilter 컴포넌트의 select value에 배열 등 비스칼라 값 전달

### Medium (사용성 이슈)

#### [QA-007] 대시보드 - 지점명 하드코딩
- **심각도**: Medium
- **페이지**: 대시보드 (/)
- **현상**: "스포짐 종각점의 실시간 센터 운영 현황입니다" 표시
- **기대**: 로그인한 지점명("강남 본점")이 표시되어야 함
- **원인**: 지점명이 하드코딩되어 있음

#### [QA-008] 쿠폰 관리 - 할인 쿠폰 유형/값 오류
- **심각도**: Medium
- **페이지**: 쿠폰 관리 (/message/coupon)
- **현상**:
  - "봄맞이 10% 할인" 쿠폰의 할인 값이 "10원"으로 표시 (10%여야 함)
  - "신규 회원 5만원 할인" 쿠폰의 유형이 "무료"로 표시 (할인 쿠폰이어야 함)
  - 모든 쿠폰이 "무료" 유형으로 표시됨
- **기대**: 할인 쿠폰은 "할인" 유형, 퍼센트 할인은 "%" 단위로 표시

#### [QA-009] 메시지 발송 이력 - 수신자 수 0명
- **심각도**: Medium
- **페이지**: 메시지 발송 (/message)
- **현상**: 최근 발송 이력 5건 모두 수신자 수 "0명" 표시
- **기대**: 발송 메시지별 실제 수신자 수가 표시되어야 함
- **원인**: `recipient_count` 필드 누락 또는 시드 데이터 미설정

#### [QA-010] 센터 설정 - 기본 정보 비어있음
- **심각도**: Medium
- **페이지**: 센터 설정 (/settings)
- **현상**: 센터명, 사업자등록번호, 연락처, 주소 등 모든 필드가 비어있음
- **기대**: 시드 데이터의 센터 기본 정보가 채워져 있어야 함
- **원인**: settings 테이블에 기본 정보 미삽입

#### [QA-011] 전자계약 - 모든 회원 "만료" 상태
- **심각도**: Medium
- **페이지**: 전자계약 (/contracts/new)
- **현상**: 23명 전체 회원의 상태가 "만료"로 표시
- **기대**: 활성 회원은 "이용중" 등 정상 상태 표시
- **원인**: 회원 상품 만료일이 과거 날짜이거나 상태 필드 계산 로직 오류

#### [QA-012] 마일리지 - 적립/만료 날짜 정보 없음
- **심각도**: Medium
- **페이지**: 마일리지 관리 (/mileage)
- **현상**: 모든 회원의 "최근 적립일"과 "만료 예정일"이 "-"로 표시, 사용 마일리지 전원 0
- **기대**: 마일리지 적립 이력에 따른 날짜 정보 표시

## 2차 상호작용 테스트 (CRUD / 검색 / 필터 / 권한)

### Critical

#### [QA-016] 슈퍼관리자(superadmin) 로그인 불가
- **심각도**: Critical
- **페이지**: 로그인 (/login)
- **현상**: superadmin 계정으로 로그인 시 "아이디 또는 비밀번호가 올바르지 않습니다" 에러
- **기대**: 슈퍼관리자도 정상 로그인 가능해야 함
- **원인**: 로그인 시 지점 선택이 필수인데, superadmin은 특정 지점에 매핑되지 않아 인증 실패 추정
- **영향**: 슈퍼관리자 전용 기능 (통합 대시보드, 지점 비교, 감사 로그) 전체 접근 불가

### High

#### [QA-017] 회원 목록 - 검색 및 상태 필터 미동작
- **심각도**: High
- **페이지**: 회원 목록 (/members)
- **현상**:
  - "홍길동" 검색 후에도 23명 전체 목록이 그대로 표시됨
  - "만료" 상태 탭 클릭해도 필터링 안 됨 (23명 전체 표시)
- **기대**: 검색어 입력 시 이름/연락처 매칭 결과만 표시, 상태 탭 클릭 시 해당 상태만 필터링
- **영향**: 회원 검색/필터 기능 전체 미동작

### Medium

#### [QA-018] 회원 수정 - 기존 데이터 미로딩
- **심각도**: Medium
- **페이지**: 회원 수정 (/members/edit)
- **현상**: 회원 상세에서 "수정" 클릭 시 수정 폼이 열리지만, 기존 회원 데이터(이름, 연락처, 성별 등)가 폼에 채워지지 않음 (빈 폼)
- **기대**: 기존 회원의 정보가 폼에 미리 채워져야 함
- **영향**: 수정 시 모든 정보를 처음부터 다시 입력해야 함

#### [QA-019] 직원 수정 - 기존 데이터 미로딩
- **심각도**: Medium
- **페이지**: 직원 수정 (/staff/edit)
- **현상**: 직원 클릭 시 수정 폼으로 이동하나, 기존 직원 데이터(이름, 연락처, 입사일 등)가 폼에 채워지지 않음
- **기대**: 기존 직원 정보가 폼에 미리 채워져야 함
- **영향**: 수정 시 모든 정보를 처음부터 다시 입력해야 함

#### [QA-020] 회원 등록 - 필수값 입력해도 다음 단계 비활성화
- **심각도**: Medium
- **페이지**: 회원 등록 (/members/new)
- **현상**: 이름, 성별, 연락처(필수 3개) 입력 후에도 "다음 단계" 버튼이 disabled 상태 유지
- **기대**: 필수값 모두 입력 시 버튼이 활성화되어야 함
- **원인**: 추가 필수 검증 조건이 있거나, 입력값 validation 로직 오류

### Low
