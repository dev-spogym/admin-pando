# 스포짐 CRM 정보구조(IA) 문서

**버전:** 1.0
**작성일:** 2026-03-11
**상태:** 확정

---

## 1. 개요

스포짐 CRM은 피트니스 센터/골프장 관리를 위한 멀티테넌트 웹 기반 CRM 솔루션입니다. 본 문서는 전체 시스템의 정보구조(IA)를 정의하며, 사이트맵, 화면 목록, 네비게이션 플로우, 권한별 접근 가능 화면을 포함합니다.

## 4. 네비게이션 플로우

### 4.1 인증 플로우
```
로그인 (SCR-001)
    ↓
지점 선택
    ↓
대시보드 (SCR-002)
```

### 4.2 회원 관리 플로우
```
회원 목록 (SCR-010)
    ├─ 검색/필터
    ├─ 선택 → 회원 상세 (SCR-011)
    │   ├─ 탭: 프로필/이용권/출석/결제/체성분/메모
    │   └─ 수정 → 회원 수정 (SCR-013)
    └─ 신규 → 회원 등록 (SCR-012)
        └─ 완료 → 회원 상세 (SCR-011)

체성분 분석 (SCR-014)
    ├─ 회원 선택
    └─ 측정 기록 및 변화 추이 조회
```

### 4.3 출석/일정 플로우
```
출석 기록 (SCR-020)
    └─ 일별/주별/월별 조회

일정 관리 (SCR-021)
    ├─ 월/주/일 캘린더 뷰
    └─ 드래그앤드롭 일정 변경
```

### 4.4 매출/결제 플로우
```
POS (SCR-031)
    ├─ 상품 선택
    ├─ 장바구니 관리
    └─ 결제 (SCR-032)
        ├─ 현금/카드/마일리지 선택
        ├─ 영수증 출력
        └─ 완료

매출 조회 (SCR-030)
    └─ 기간/유형별 필터 및 통계
```

### 4.5 상품/시설 관리 플로우
```
상품 목록 (SCR-040)
    ├─ 필터
    └─ 신규 → 상품 등록 (SCR-041)

락커 현황 (SCR-050)
    └─ 상태 조회
        └─ 관리 → 락커 관리 (SCR-051)
            ├─ 배정
            └─ 해제

RFID 관리 (SCR-052)
    ├─ 카드 등록/해제
    └─ 이력 조회

룸 관리 (SCR-053)
    ├─ 룸 CRUD
    └─ 예약 설정
```

### 4.6 직원/급여 관리 플로우
```
직원 목록 (SCR-060)
    └─ 신규 → 직원 등록 (SCR-061)

급여 현황 (SCR-062)
    └─ 상세 → 급여명세서 (SCR-063)
        └─ PDF 다운로드
```

### 4.7 메시지/마케팅 플로우
```
메시지 발송 (SCR-070)
    ├─ 수신자 선택
    └─ 발송 이력 조회

자동 알림 (SCR-071)
    └─ 규칙 설정 (만료/생일/미출석)

쿠폰 관리 (SCR-072)
    ├─ 생성
    ├─ 발급
    └─ 이력 조회

마일리지 관리 (SCR-073)
    └─ 적립/사용 이력 조회
```

### 4.8 계약/구독 플로우
```
계약 마법사 (SCR-080)
    └─ 단계: 회원 선택 → 상품 선택 → 기간/금액 → 결제 → 계약서 생성

구독 관리 (SCR-081)
    ├─ 구독 상품 CRUD
    ├─ 자동 결제 설정
    └─ 구독 현황 대시보드
```

### 4.9 설정 플로우
```
설정 (SCR-090)
    └─ 센터 기본 설정 (센터명/영업시간/연락처)

권한 설정 (SCR-091)
    └─ 역할별 메뉴/기능 권한 매트릭스

키오스크 설정 (SCR-092)
    └─ 디스플레이/동작 설정

IoT 설정 (SCR-093)
    └─ 출입문/체성분 기기 연동 설정

지점 관리 (SCR-094)
    ├─ 지점 CRUD
    └─ 멀티테넌트 데이터 관리
```

## 6. 초기화면 및 진입점

- **로그인 없음**: `/login` (SCR-001)
- **로그인 후 기본 진입**: `/` (SCR-002 대시보드)
- **권한별 초기 메뉴**: 역할별로 접근 가능한 화면 중 첫 번째 메인 메뉴로 진입

---

## 7. 멀티테넌트 구조

- 시스템은 멀티테넌트 아키텍처로 구성
- 지점(Branch) 단위로 독립된 데이터 관리
- 로그인 시 지점 선택 필수
- 모든 화면은 선택된 지점의 데이터만 표시
- 최고관리자만 지점 관리(SCR-094) 접근 가능
