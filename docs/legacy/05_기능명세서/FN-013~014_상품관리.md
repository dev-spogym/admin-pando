# FN-013~014: 상품관리

## FN-013: 상품 목록 조회

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-013 |
| **기능명** | 상품 목록 조회 |
| **분류** | 상품관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-040 |
| **관련 API ID** | API-018 |

### 기능 설명
이용권(월권, 회차권), PT, GX, 기타 상품을 카테고리별로 조회합니다. 상태 필터와 키오스크 노출 토글 기능을 제공합니다.

### 상세 요구사항
1. 카테고리 필터 (이용권/PT/GX/상품)
2. 상태 필터 (판매중/중지)
3. 키오스크 노출 토글 (ON/OFF)
4. 상품 테이블: 이름/카테고리/현금가/카드가/기간/상태
5. 상품 클릭 → 수정 페이지로 이동
6. 페이지네이션

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 카테고리 | String[] | 선택 | "이용권" \| "PT" \| "GX" \| "상품" |
| 상태 | String[] | 선택 | "판매중" \| "중지" |
| 키오스크노출 | Boolean | 선택 | true \| false \| null(모두) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 상품 목록 | Array | [{id, name, category, cash_price, card_price, duration, status, kiosk_visible}, ...] |
| 총 건수 | Number | 필터 적용된 전체 상품 수 |

### 비즈니스 규칙
1. branch_id 기준 데이터 격리
2. 판매중 상품만 POS에서 노출
3. 키오스크 노출은 별도 설정 (POS와 독립적)
4. 가격 정보: 현금가 vs 카드가 (다를 수 있음)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 상품 없음 | "상품이 없습니다" 빈 상태 UI |
| 조회 실패 | "데이터를 불러올 수 없습니다. 새로고침을 시도해주세요" |

### Web 구현 사항
- **테이블**: TanStack Table
- **필터**: MultiSelect (카테고리), MultiSelect (상태), Toggle (키오스크)
- **페이지네이션**: 기본 구현
- **데이터 페칭**: React Query

## FN-014: 상품 등록/수정

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-014 |
| **기능명** | 상품 등록/수정 |
| **분류** | 상품관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-040 |
| **관련 API ID** | API-019 |

### 기능 설명
신규 상품을 등록하거나 기존 상품을 수정합니다. 카테고리별로 필드가 달라지며, 상품명 중복을 방지합니다.

### 상세 요구사항
1. 카테고리 선택 (이용권/PT/GX/상품)
2. 상품명 입력 (중복 체크)
3. 현금가/카드가 입력 (다를 수 있음)
4. 기간 설정 (이용권/PT/GX만)
5. 키오스크 노출 토글
6. 상태 선택 (판매중/중지)
7. 상품 수정 시 기존 정보 프리필
8. 취소 시 변경 폐기 확인

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 카테고리 | String | 필수 | "이용권" \| "PT" \| "GX" \| "상품" |
| 상품명 | String | 필수 | 1~100자, 동일 카테고리 내 중복 불가 |
| 현금가 | Number | 필수 | 0 이상 |
| 카드가 | Number | 필수 | 0 이상 (현금가와 다를 수 있음) |
| 기간 | String/Number | 조건부 | 이용권/PT/GX: 필수 (예: "1개월", "10회", "4주") |
| 키오스크노출 | Boolean | 선택 | 기본값: true |
| 상태 | String | 선택 | "판매중" \| "중지" (기본값: "판매중") |
| 설명 | String | 선택 | 300자 이내 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 등록/수정된 상품 정보 | Object | 전체 상품 정보 반환 |

### 비즈니스 규칙
1. 동일 카테고리 내 상품명 중복 불가
   - 다른 카테고리는 중복 허용 (예: 상품 "헬스보충제" vs 이용권 "헬스보충제")
2. 가격 0원 이상 (무료 상품 가능)
3. 현금가와 카드가가 다를 수 있음 (카드 할인 등)
4. 수정 시 기존 매출에는 영향 없음 (미래 판매 가격만 변경)
5. 상태 변경: 판매중 → 중지 (POS에서 미표시)
6. 키오스크 노출: POS 자동 노출과 독립적

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 중복 상품명 | "이 이름의 상품이 이미 존재합니다. 다른 이름을 입력해주세요" |
| 필수 미입력 | 필드별 에러 메시지 |
| 가격 형식 오류 | "숫자를 입력해주세요" |
| 음수 입력 | "0 이상의 숫자를 입력해주세요" |
| 저장 실패 | "상품 등록/수정에 실패했습니다. 다시 시도해주세요" |
| 동시 수정 충돌 | "다른 사용자에 의해 수정되었습니다. 새로고침 후 다시 시도해주세요" |

### Web 구현 사항
- **Form 관리**: React Hook Form + Zod validation
- **카테고리별 필드**: 조건부 렌더링 (useWatch)
- **상품명 중복 체크**: onChange 이벤트로 API 호출 (debounce 500ms)
- **레이아웃**: 2열 그리드 (데스크톱), 1열 (모바일)
- **상태 선택**: Radio Button 또는 Switch

### 카테고리별 필드 매트릭스

| 필드 | 이용권 | PT | GX | 상품 |
|------|--------|----|----|------|
| 상품명 | ✓ | ✓ | ✓ | ✓ |
| 현금가 | ✓ | ✓ | ✓ | ✓ |
| 카드가 | ✓ | ✓ | ✓ | ✓ |
| 기간 | ✓(개월) | ✓(회) | ✓(기간) | X |
| 회차/횟수 | ✓ | X | X | X |
| 키오스크 노출 | ✓ | ✓ | ✓ | ✓ |
| 설명 | ✓ | ✓ | ✓ | ✓ |

## 상품관리 공통 사항

### 권한 검증
| 기능 | 센터장 | PT담당자 | 직원 | 조회전용 |
|------|--------|---------|------|---------|
| FN-013 조회 | O | O | O | O |
| FN-014 등록/수정 | O | O | O | X |
| FN-014 삭제* | O | X | X | X |

*상품 삭제는 기본적으로 불가 (기존 매출 데이터 무결성 보호)
대신 상태를 "중지"로 변경하여 판매 중단

### 데이터 격리
- 모든 상품 데이터는 branch_id 기준
- 다지점 관리자는 각 지점별로 상품 관리 가능

### 카테고리 설명
- **이용권**: 월권(1개월), 3개월, 6개월, 12개월 등
- **PT**: Personal Training (개인 운동, 회차 기반: 4회, 8회, 12회 등)
- **GX**: Group Exercise (그룹 운동, 기간 기반: 4주, 8주, 12주 등)
- **상품**: 헬스보충제, 의류, 기타 물품 (기간 개념 없음)

### 가격 전략
- **현금가**: 현금 결제 시 적용 가격
- **카드가**: 카드 결제 시 적용 가격 (보통 현금가와 동일하거나 낮음)
- 예: 이용권 월권 현금가 100,000원 vs 카드가 98,000원

### 공통 기술 스택
- **Form**: React Hook Form + Zod
- **테이블**: TanStack Table
- **데이터 페칭**: React Query
- **필터**: MultiSelect
- **UI**: 커스텀 컴포넌트
