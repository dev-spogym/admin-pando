# FN-015~018: 시설관리

## FN-015: 락커 현황 조회

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-015 |
| **기능명** | 락커 현황 조회 |
| **분류** | 시설관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-050 |
| **관련 API ID** | API-020 |

### 기능 설명
구역별 락커의 사용 현황을 색상 그리드로 표시합니다. 클릭하여 상세 정보를 조회할 수 있습니다.

### 상세 요구사항
1. 구역별 탭 (예: VIP 구역, 일반 구역, 여성 구역)
2. 색상 상태 (사용중/빈/만료임박/고장)
3. 클릭 시 상세 모달 (회원명, 만료일, 액션)
4. 상세 모달에서 "배정 변경" 또는 "해제" 가능
5. 만료임박 강조 (경고 배너 또는 색상)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 구역ID | UUID | 선택 | 탭에서 선택 (기본값: 첫 번째 구역) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 락커 그리드 | Array | [{locker_id, number, status, member_name, expired_date}, ...] |
| 통계 | Object | {total: 10, in_use: 5, empty: 3, expiring: 1, broken: 1} |

### 비즈니스 규칙
1. branch_id 기준
2. 만료임박 = D-3 이내 (빨강 강조)
3. 사용중 = 유효 기간 남음 (파랑)
4. 빈 것 = 미배정 (회색/흰색)
5. 고장 = 사용 불가 (검은색 X 표시)

### 색상 코드
- **사용중**: #3B82F6 (파랑)
- **만료임박**: #EF4444 (빨강)
- **빈 것**: #E5E7EB (밝은 회색)
- **고장**: #1F2937 (어두운 회색)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 락커 미등록 | "등록된 락커가 없습니다. 시설 관리자에 문의하세요" |
| 조회 실패 | "데이터를 불러올 수 없습니다" |

### Web 구현 사항
- **그리드**: CSS Grid (5열 또는 10열)
- **상세 모달**: Modal 컴포넌트
- **색상 매핑**: CSS 클래스 또는 인라인 스타일
- **탭**: Tab 컴포넌트
- **데이터 페칭**: React Query

## FN-016: 락커 배정/관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-016 |
| **기능명** | 락커 배정/관리 |
| **분류** | 시설관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-050 |
| **관련 API ID** | API-021 |

### 기능 설명
회원에게 빈 락커를 배정하거나, 기존 배정을 해제/변경합니다. 일괄 해제(만료 락커) 기능도 제공합니다.

### 상세 요구사항
1. 회원 검색 (이름/연락처)
2. 빈 락커 선택
3. 만료일 설정 필수
4. 배정 완료 후 현황 화면으로 복귀
5. 기존 배정 해제 (확인 모달)
6. 일괄 해제 (만료 락커)
7. 고장 등록/해제

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 회원ID | UUID | 필수 | 검색으로 선택 |
| 락커번호 | UUID | 필수 | 빈 락커 선택 (자동 필터링) |
| 만료일 | Date | 필수 | YYYY-MM-DD |
| 비고 | String | 선택 | 최대 100자 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 배정 결과 | Object | {success: true/false, locker_id, member_id, expired_date} |

### 비즈니스 규칙
1. 이미 배정된 락커 재배정 불가 (해제 후 재배정)
2. 만료일 필수 (빈 채로 두면 저장 불가)
3. 과거 날짜 만료일 설정 불가
4. 일괄 해제: D-0 이하인 모든 락커 한 번에 해제
5. 고장 등록 시 해당 락커는 자동으로 "빈 것"으로 표시되지 않음 (사용 불가)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 중복 배정 | "이 락커는 이미 {{회원명}}에게 배정되어 있습니다. 해제 후 재배정하세요" |
| 회원 미존재 | "존재하지 않는 회원입니다" |
| 빈 락커 없음 | "사용 가능한 빈 락커가 없습니다" |
| 배정 실패 | "배정에 실패했습니다. 다시 시도해주세요" + 재시도 |
| 만료일 미입력 | "만료일을 입력해주세요" |
| 과거 날짜 | "만료일은 오늘 이후 날짜여야 합니다" |

### Web 구현 사항
- **회원 검색**: Combobox (이름/연락처 검색)
- **빈 락커 목록**: 자동 필터링된 드롭다운
- **만료일**: DatePicker (과거 날짜 비활성화)
- **배정 모달**: 단계별 폼 (회원 선택 → 락커 선택 → 만료일 선택 → 확인)
- **일괄 해제**: 확인 모달 (몇 개 해제될지 표시)

## FN-017: RFID 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-017 |
| **기능명** | RFID 관리 |
| **분류** | 시설관리 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-052 |
| **관련 API ID** | API-022 (미정) |

### 기능 설명
RFID 카드를 회원과 매핑하여 출입통제에 활용합니다. 카드 번호 스캔/수동 입력, 분실 처리, 이력 조회를 지원합니다.

### 상세 요구사항
1. 카드 번호 입력 (스캔 또는 수동 입력)
2. 회원 선택 (자동 추천 또는 검색)
3. 1회원 1카드 원칙 (기존 카드 해제 후 새 카드 배정)
4. 분실 처리 (기존 매핑 해제)
5. RFID 이력 조회 (카드별, 회원별)
6. 카드 비활성화

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 카드번호 | String | 필수 | 스캔 또는 수동 입력 (16자리 hex) |
| 회원ID | UUID | 필수 | 검색으로 선택 |
| 카드상태 | String | 선택 | "활성" \| "분실" \| "비활성" (기본값: "활성") |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 매핑 결과 | Object | {card_id, card_number, member_id, status, created_at} |
| RFID 이력 | Array | [{id, card_number, member_id, action, timestamp}, ...] |

### 비즈니스 규칙
1. 1회원 1카드 원칙
   - 새 카드 배정 시 기존 카드 자동 비활성화
   - 사용자 확인 모달 표시: "{{회원명}}의 기존 카드({{old_card}})를 비활성화하고 새 카드({{new_card}})를 활성화하시겠습니까?"
2. 중복 카드 불가 (같은 카드 번호 2회 이상 배정 불가)
3. 분실 처리: 해당 카드 매핑 자동 해제

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 미등록 카드 | "등록되지 않은 카드입니다. 먼저 카드를 등록해주세요" |
| 이미 매핑된 카드 | "이미 {{회원명}}에게 배정된 카드입니다" |
| 회원 미존재 | "존재하지 않는 회원입니다" |
| 잘못된 카드 번호 형식 | "카드 번호 형식이 올바르지 않습니다 (16자리 hex)" |
| 스캔 실패 | "카드 스캔에 실패했습니다. 다시 시도해주세요" |

### Web 구현 사항
- **카드 번호 입력**: Input 필드 (스캔 기기 연동)
- **회원 선택**: Combobox (자동 추천 또는 검색)
- **매핑 모달**: 확인 화면
- **이력 테이블**: TanStack Table (카드번호/회원/액션/시간)
- **분실 처리**: 모달 확인

## FN-018: 룸 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-018 |
| **기능명** | 룸 관리 |
| **분류** | 시설관리 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-053 |
| **관련 API ID** | API-023 (미정) |

### 기능 설명
PT/GX 운동실(룸)을 등록/수정/삭제합니다. 각 룸의 수용 인원과 예약 가능 시간대를 설정할 수 있습니다.

### 상세 요구사항
1. 룸 CRUD
2. 룸명, 유형(PT/GX/기타), 수용 인원 설정
3. 예약 가능 시간대 (예: 06:00~23:00)
4. 룸명 중복 불가
5. 기존 예약이 있으면 삭제 불가 (수정은 가능)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 룸명 | String | 필수 | 1~50자, 중복 불가 |
| 유형 | String | 필수 | "PT" \| "GX" \| "기타" |
| 수용인원 | Number | 필수 | 1~30명 |
| 예약시작시간 | Time | 필수 | HH:MM |
| 예약종료시간 | Time | 필수 | 시작시간보다 나중 |
| 비고 | String | 선택 | 최대 200자 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 룸 정보 | Object | {id, name, type, capacity, start_time, end_time} |

### 비즈니스 규칙
1. 룸명 동일 지점 내 중복 불가
2. 기존 예약 있으면 삭제 불가
3. 수정 가능 (예약 영향 없음)
4. 수용 인원: 1~30명

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 중복 룸명 | "이 이름의 룸이 이미 존재합니다" |
| 예약 존재 시 삭제 | "이 룸에 예약된 일정이 있습니다. 삭제할 수 없습니다" |
| 필수 미입력 | 필드별 에러 메시지 |
| 잘못된 시간 범위 | "종료 시간이 시작 시간보다 나중이어야 합니다" |

### Web 구현 사항
- **룸 목록**: Card 또는 Table (룸명, 유형, 수용인원)
- **생성/수정 모달**: Form (룸명, 유형, 수용인원, 시간)
- **시간 선택**: TimePicker 컴포넌트
- **삭제 버튼**: 예약 존재 시 비활성화 + 안내 메시지
