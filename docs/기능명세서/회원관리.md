### 1.3 데이터 모델

**DB 테이블**: `members`

**API 훅**: `useMembers(params)`, `useMemberStats()`

| 파라미터 | 설명 |
|---------|------|
| `page` | 현재 페이지 (기본: 1) |
| `size` | 페이지 크기 (기본: 20) |
| `search` | 이름/연락처 검색어 (300ms 디바운스) |
| `status` | 상태 필터 (ACTIVE/EXPIRED/INACTIVE/HOLDING/SUSPENDED) |
| `gender` | 성별 필터 |
| `product` | 계약상품 필터 |
| `sortKey` | 정렬 키 (name/membershipExpiry/registeredAt) |
| `sortDirection` | 정렬 방향 (asc/desc) |
| `isFavorite` | 관심회원 필터 |
| `daysNoVisit` | 미방문 N일 필터 |
| `memberType` | 회원구분 필터 |
| `referralSource` | 유입경로 필터 |

**통계 API**: `useMemberStats()` → `MemberStats` 반환

```typescript
interface MemberStats {
  total: number;
  active: number;
  inactive: number;
  expired: number;
  holding: number;
  suspended: number;
  expiredThisMonth: number;
  newThisMonth: number;
  expiringCount: number; // D-30 이내
}
```

### 1.4 필드 상세 — 회원 전체 탭 테이블 컬럼

| 필드명 | 타입 | 필수 | 기본값 | 유효성검증 | 에러메시지 | 비고 |
|--------|------|:---:|--------|-----------|-----------|------|
| ★ (isFavorite) | boolean | - | false | - | - | 클릭 토글, 즉시 DB 반영 |
| 상태 (status) | enum | ✓ | INACTIVE | ACTIVE/INACTIVE/EXPIRED/HOLDING/SUSPENDED | - | StatusBadge dot 표시 |
| 세그먼트 | computed | - | - | - | - | 신규/만료후미등록/이탈위험/만료임박/관심필요/충성회원/활발한회원 |
| 회원명 (name) | string | ✓ | - | - | - | 클릭 시 회원 상세로 이동 |
| 성별 (gender) | enum | - | - | M/F | - | 'M'→'남', 'F'→'여' |
| 생년월일 (birthDate) | date | - | - | - | - | YYYY-MM-DD 형식 |
| 연락처 (phone) | string | ✓ | - | - | - | tabular-nums 스타일 |

| 등록일 (registeredAt) | date | - | - | - | - | 정렬 가능, tabular-nums |

### 1.6 필터/검색/정렬 상세

**메인 탭 (1차 분류)**

| 탭 키 | 라벨 | 설명 |
|-------|------|------|

| `pass` | 이용권 목록 | 이용권 정보 + D-Day 표시 |

**상태 탭 (2차 분류, 회원 전체 탭에서만 표시)**

| 탭 키 | 라벨 | 카운트 소스 |

| 필터 키 | 라벨 | 유형 | 옵션 |

| `expiryDate` | 최종만료일 | dateRange | 시작일~종료일 |
| `visitDate` | 최근방문일 | dateRange | 시작일~종료일 |

**추가 필터 행**

| 필터 | 유형 | 옵션 |
|------|------|------|
| 관심회원만 | checkbox | ★ 관심회원만 |
| 미방문 | select | 전체/7일 초과/14일 초과/30일 초과 |

**정렬**: 회원명(name), 만료일(membershipExpiry), 등록일(registeredAt) 컬럼 헤더 클릭 정렬 (서버사이드)

**검색 디바운스**: 300ms

### 1.11 이용권 목록 탭 D-Day 계산 로직

```
diff = Math.ceil((만료일 - 현재시간) / 86400000)
- diff < 0 → "만료" (빨강)
- diff === 0 → "D-Day" (노랑)
- diff <= 7 → "D-{diff}" (노랑)
- diff > 7 → "D-{diff}" (기본)
```

### 2.3 데이터 모델

**DB 테이블**: `members`

**폼 스키마**: `memberFormSchema` (Zod, `src/lib/validations.ts`)

**등록 API**: `useCreateMember()` 훅 (react-query mutation)

**수정 API**: `supabase.from('members').update(payload).eq('id', urlMemberId)` (직접 호출)

**수정 모드 데이터 로드**: `supabase.from('members').select('*').eq('id', urlMemberId).single()`

**스태프 목록 로드**: `supabase.from('staff').select('id, name, role').eq('branchId', getBranchId())`
- FC: `role === 'FC'`
- 트레이너: `role === '트레이너'`

### 4.3 데이터 모델

**API 함수**:
- `getMember(memberId)` → 회원 정보
- `getBranches()` → 지점 목록 (현재 지점 제외)
- `checkTransferEligibility(memberId)` → 이관 가능성 체크
- `transferMember(request)` → 이관 실행

**TransferCheckResult 타입**:

```typescript

  canTransfer: boolean;
  hasUnpaidAmount: boolean;
  unpaidAmount: number;
  hasLocker: boolean;
  hasActiveHolding: boolean;
  hasActivePt: boolean;
  ptRemainingSessions: number;
  activeCoupons: number;
  mileageBalance: number;
  blockers: string[];
  warnings: string[];
}
```

### 4.4 체크리스트 항목

| 항목 | status 조건 | 상태별 표시 |
|------|-----------|-----------|
| 미납금 | `hasUnpaidAmount` → block | block: "{금액}원 정산 필요", ok: "0원" |
| 락커 | `hasLocker` → block | block: "락커 반납 필요", ok: "없음" |
| 홀딩 상태 | `hasActiveHolding` → warn | warn: "홀딩 중 — 종료 후 이관 권장", ok: "없음" |
| PT 잔여 | `hasActivePt` → warn | warn: "잔여 {N}회 — 이관 유형 확인 필요", ok: "없음" |
| 보유 쿠폰 | 항상 info | info: "{N}장 보유 — 지점 한정 쿠폰은 이관 시 소멸될 수 있습니다." / "없음" |
| 마일리지 | 항상 ok | ok: "{N}P — 테넌트 통합이므로 유지됩니다." |

**상태 배너**:
- blockers.length > 0 → 빨강 "이관 불가" + blocker 목록
- blockers.length === 0 && warnings.length > 0 → 노랑 "주의사항 확인 후 진행 가능" + warning 목록
- canTransfer && warnings.length === 0 → 초록 "모든 항목 통과 — 이관 가능합니다."

### 5.2 화면 레이아웃

```
┌────────────────────────────────────────────────┐
│ ← 회원 상세로 돌아가기                          │
│ PageHeader: "{회원명} 회원의 체성분 정보"        │

│   [전체 기록 추출] [새 측정 기록 추가]            │
├────────────────────────────────────────────────┤
│ StatCardGrid (3열)                              │
│   현재 체중 (변화량) / 골격근량 (변화량) /        │

├────────────────────────────────────────────────┤

```

### 5.3 데이터 모델

**DB 테이블**: `body_compositions`, `member_goals`

**체성분 로드 쿼리**:
```
body_compositions.select('id, memberId, date, weight, muscle, fat, fatRate, bmi, memo')
  .eq('memberId', memberId)
  .order('date', { ascending: false })
```

**회원 정보 로드**: `members.select('id, name, height, birthDate, gender').eq('id', memberId).single()`

**목표 로드**: `member_goals.select('goalWeight, goalPbf').eq('memberId', memberId).single()`

**자동 계산**:
- BMI = weight / (height/100)^2
- BMR = 10 × weight + 6.25 × height - 5 × age - 161 (Mifflin-St Jeor)
- fat = weight × (pbf / 100)
- 제지방량 = weight - fat
- 권장 섭취 열량 = BMR × 1.4

### 5.4 기록 목록 탭

**테이블 컬럼**: 측정일, 체중(kg, 볼드), 골격근(kg), 체지방량(kg), BMI, 체지방률(%), 기초대사량(kcal), 리포트

**행 클릭**: 상세 보기 모달 (구현됨)

### 5.5 변화 그래프 탭

**SVG 라인 차트**: 최근 6회, 3개 지표 토글 (체중/골격근량/체지방률)
- 체중: `#FF7F6E` (코랄)
- 골격근량: `#48D1CC` (민트)
- 체지방률: `#F59E0B` (옐로우)

**포인트 호버**: 툴팁 — 날짜 + 지표값

**변화 요약**: 최초 측정 → 최근 측정 기간 동안 각 지표 변화량 텍스트
- 체중 감소 → 초록, 증가 → 코랄
- 골격근량 증가 → 민트, 감소 → 빨강
- 체지방률 감소 → 초록, 증가 → 노랑

### 5.6 목표 관리 탭

**목표 게이지 컴포넌트**: 프로그레스 바 + 달성률 표시

| 항목 | 현재값 | 목표값 | 계산 |
|------|--------|--------|------|
| 목표 체중 | latest.weight | goals.weight | `current ≤ goal ? 100 : (1 - (current-goal)/goal) × 100` |
| 목표 체지방률 | latest.pbf | goals.pbf | (동일) |

**목표 수정 모달**: weight(kg), pbf(%) 입력 → `member_goals.upsert({ memberId, goalWeight, goalPbf })`

### 5.7 비만 분석 영역

**BMI 분석 바**: 현재값, 표준 범위 18.5~23 kg/m²
**체지방률 분석 바**: 현재값, 표준 범위 18~28 %

**에너지/대사 분석**:
- 기초대사량 (BMR): `{N} kcal`
- 제지방량: `{N} kg`
- 권장 섭취 열량: `{N} kcal` (BMR × 1.4)

### 5.8 측정 추가 모달

| 필드명 | 타입 | 필수 | 유효성검증 | 에러메시지 |
|--------|------|:---:|-----------|-----------|
| date | date | ✓ | 비어있지 않음 | "날짜를 입력해주세요." |
| weight | number | ✓ | 20~300 | "유효 범위: 20~300kg" |
| muscle | number | ✓ | 5~80 | "유효 범위: 5~80kg" |
| pbf | number | ✓ | 3~60 | "유효 범위: 3~60%" |
| bodyWater | number | - | - | - |

**같은 날짜 중복**: 이미 해당 날짜 데이터 존재 시 → ConfirmDialog "기존 데이터를 덮어쓰시겠습니까?" → upsert

**저장**: `body_compositions.insert/update` → 토스트: "체성분 데이터가 저장되었습니다."

**BMI 미리보기**: weight 입력 시 실시간 BMI 계산 표시
