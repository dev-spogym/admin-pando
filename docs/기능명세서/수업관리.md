## 1. 캘린더/수업 일정 (`/calendar`)

**화면 ID**: SCR-021 | **코드**: `src/app/calendar/page.tsx` (2,491줄)
**FullCalendar 기반** 드래그&드롭, 리사이즈, 클릭 생성/편집 지원

---

### 1.1 화면 개요

PT 및 그룹 수업 스케줄을 캘린더 형태로 관리하고 회원의 예약 현황을 확인하는 메인 화면.
5개 탭(일정표 / 수업 관리 / 횟수 관리 / 페널티 관리 / 유효 수업 목록)으로 구성되며,
각 탭은 독립 라우트(`/calendar`, `/lessons`, `/lesson-counts`, `/penalties`, `/valid-lessons`)로 연결된다.

### 1.6 모달/팝업

#### 1.6.1 일정 상세 모달 (EventDetailModal)

**트리거**: 캘린더 이벤트 클릭

**구성**:
```
┌─────────────────────────────────────────────┐
│ 헤더: 수업명 [상태배지] [미승인배지] [분류배지] │
│ ────────────────────────────────────────────│
│ 강사 | 장소                                  │
│ 시간 | 예약 현황                              │
│ 예약 현황 바 (%)                             │
│ 날짜                                         │
│ 수업 유형 배지                                │
│ 대상 정보 (#16 대상 유형 표시)                  │
│ 좌석 배치도 (토글, FN-037)                     │
│ 수정 불가 안내 (과거/시작 임박)                  │
│ ────────────────────────────────────────────│
│ [승인][거절] (미승인시)      [삭제] [수정]      │
└─────────────────────────────────────────────┘
```

**필드 표시**:
| 영역 | 표시 내용 |
|------|---------|
| 헤더 좌 | 수업 유형별 색상 dot + 수업명 + StatusBadge(예약/완료/취소) |
| 헤더 우 | 닫기 버튼 |
| 미승인 배지 | `approvalStatus === 'pending'`일 때 표시 (Bell 아이콘 + "미승인") |
| 분류 배지 | `scheduleCategory` 있을 때 해당 색상으로 표시 |
| 강사 | Users 아이콘 + `{instructor} 강사` |
| 장소 | MapPin 아이콘 + `{room}` |
| 시간 | Clock 아이콘 + `{start시간} ~ {end시간}` |
| 예약 현황 | Users 아이콘 + `예약 {curRes} / {maxCap}명` |
| 예약 바 | 0~80% 초록, 80~100% 노랑, 100% 빨강 |
| 예약 마감 | `reservationDeadline` 존재 시 표시 |
| 날짜 | Calendar 아이콘 + 한국어 날짜 포맷 |
| 수업 유형 | 유형별 색상 배지 (PT/GX/개인레슨/기타) |
| 대상 정보 | `targetType` 또는 `targetName` 존재 시 표시 |
| 좌석 배치도 | `seatRows > 0 && seatCols > 0`일 때 접힌 상태로 표시, 펼치기 토글 |

**좌석 그리드 (SeatGrid)**:
- rows x cols 그리드 렌더링
- 상태별 색상: 예약됨(파랑), 선택 중(노랑), 빈 좌석(회색)
- 좌석 번호 1부터 표시
- 범례 하단 표시

**액션 버튼**:
| 버튼 | 조건 | 동작 |
|------|------|------|
| 승인 | `approvalStatus === 'pending'` | Supabase `approval_status` → `'approved'` |
| 거절 | `approvalStatus === 'pending'` | Supabase `approval_status` → `'rejected'` |
| 삭제 | `editable === true` | 수업 삭제 (로컬/DB) |
| 수정 | `editable === true` | 상세 모달 닫고 등록 모달 열기 (편집 모드) |

### 1.7 상태별 변화

#### 캘린더 뷰별 이벤트 렌더링

| 뷰 | 렌더링 방식 |
|----|----------|
| **월간 (dayGridMonth)** | 색상 dot(6px 원형) + 시간 + 수업명 (10px), 미승인 시 "미승인" 텍스트(8px) |
| **주간/일간 (timeGrid)** | 수업명(11px bold) + 미승인 텍스트 + 분류 라벨 + 인원/강사 + 장소 |
| **목록 (listWeek)** | 수업명(13px bold) + 분류 배지 + 미승인 배지 + 강사/장소 + 인원 |

#### 이벤트 색상 로직

1. `scheduleCategory` 존재 시 → 해당 분류 색상(SCHEDULE_COLORS) 우선 적용
2. 없으면 → 트레이너별 색상(TRAINER_HEX) 적용
3. `approvalStatus === 'pending'` → 반투명 처리 (배경 `#f9fafb`, opacity-70~80)

#### 날짜 셀 커스텀 렌더 (월간뷰)

- 날짜 번호 좌측 정렬
- 해당 날짜 수업 건수 > 0이면 우측에 primary 색상 원형 배지로 건수 표시

#### 수업 편집 가능 여부 (isEventEditable)

| 조건 | 결과 | 메시지 |
|------|------|--------|
| 과거 날짜 | 수정 불가 | "과거 일정은 수정할 수 없습니다." |
| 이미 시작된 수업 | 수정 불가 | "이미 시작된 수업은 수정할 수 없습니다." |
| 시작 2시간 이내 | 수정 불가 | "수업 시작 2시간 전까지만 수정할 수 있습니다." |
| 그 외 | 수정 가능 | - |

### 1.10 비즈니스 로직

#### 시간 충돌 감지 (A-36)

- 수업 등록/수정 시 동일 강사(`instructorId`)의 기존 수업과 시간이 겹치는지 확인
- 겹치는 조건: `newStart < existingEnd && newEnd > existingStart`
- 편집 중인 수업 자체(`editingId`)는 제외
- 충돌 시 `window.confirm` 경고: `"${강사명}의 ${시작}~${종료} 수업과 겹칩니다.\n계속 등록하시겠습니까?"`

#### 데이터 소스 합산

- **classes 테이블**: 직접 등록된 수업 이벤트 (`events` 상태)
- **lesson_schedules 테이블**: 수업 정의 기반 스케줄 이벤트 (`lessonSchedules` 상태)
- **로컬 이벤트**: 아직 DB에 저장되지 않은 이벤트 (`localEvents` 상태)
- 캘린더에는 세 소스를 합산하여 표시 (`calendarEvents = filteredEvents + lessonScheduleEvents`)

| "PT / OT" | PT |
