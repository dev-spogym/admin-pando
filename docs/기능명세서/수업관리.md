### 1.5 일정표 탭 상세 (activeTab === "schedule")

#### 1.5.1 미승인 일정 알림 배너

- **표시 조건**: `pendingCount > 0`
- **UI**: `bg-amber-50 border border-amber-200 rounded-xl` 배너
- **아이콘**: Bell (15px, text-amber-600)
- **텍스트**: "미승인 일정 **{pendingCount}건**이 있습니다."
- **폰트**: text-[13px] font-semibold text-amber-700

#### 1.5.2 필터 바

`bg-surface p-md rounded-xl border border-line shadow-xs` 컨테이너 안에 3단 구성:

**1단: 기본 필터**

| 필터 | 컴포넌트 | 옵션 | 기본값 | 너비 |
|------|----------|------|--------|------|
| 강사 필터 | Select | `"전체 강사"` + `instructors.map(i => i.name (i.type))` | `""` (전체) | w-40 |

| 정원 상태 필터 | 버튼 그룹 (3개) | `"전체"`, `"여유"`, `"마감"` | `"전체"` | - |

**정원 상태 필터 버튼 스타일**:
- 선택됨: `bg-primary text-white border-primary`
- 미선택: `bg-surface-secondary text-content-secondary border-line hover:border-primary hover:text-primary`
- 크기: `h-9 px-md rounded-lg text-[12px] font-semibold`

**2단: 카테고리 멀티필터 (#17)**

`pt-xs border-t border-line` 구분선 아래에 위치.

| 항목 | 설명 |

### 1.15 시간 충돌 감지 (A-36, 124-21)

수업 등록(handleAddClass) 시 동일 강사의 기존 수업과 시간이 겹치는지 검사.

**검사 조건**:
- `formInstructor`가 선택되어 있을 때만 실행
- 비교 대상: `[...events, ...localEvents]` 중 같은 강사(instructorId === formInstructor)이고 수정 중인 이벤트(selectedEvent?.id)가 아닌 것
- 충돌 판정: `newStart < existingEnd && newEnd > existingStart`

**충돌 발견 시**:
- `window.confirm()` 대화상자 표시
- 메시지: `"{강사명}의 {startLabel}~{endLabel} 수업과 겹칩니다.\n계속 등록하시겠습니까?"`
- 확인 → 등록 계속
- 취소 → 등록 중단 (return)

- **수업 목록**: `supabase.from('classes').select('id, staffId, startTime, endTime, capacity').eq('branchId', branchId).gte/lte(startTime, 기간)`
- **예약 통계**: `supabase.from('lesson_bookings').select('scheduleId, memberId, status').in('scheduleId', classIds).neq('status', 'CANCELLED')`
- **매출 기여**: `supabase.from('sales').select('staffName, amount').eq('branchId', branchId).gte/lte(saleDate, 기간)`
- **상세 수업**: `supabase.from('classes').select('id, title, startTime, endTime, room, capacity').eq('branchId', branchId).eq('staffId', instr.id)`

### B.1 캘린더 이벤트 필터링 (filteredEvents)

캘린더 페이지에서 `allEvents = [...events, ...localEvents]`를 기반으로 다음 필터가 순차 적용된다:

```

1. 강사 필터 (selectedInstructor):
   - 값이 빈 문자열이면 → 패스 (전체)
   - 값이 있으면 → e.instructorId === selectedInstructor 인 것만 유지

2. 수업명 필터 (selectedLessonFilter):
   - 값이 빈 문자열이면 → 패스 (전체)
   - 값이 있으면 → e.title === selectedLessonFilter 인 것만 유지

3. 정원 상태 필터 (capacityFilter):
   - "전체" → 패스
   - "여유" → e.currentCount < e.capacity 인 것만 유지
   - "마감" → e.currentCount >= e.capacity 인 것만 유지

4. 카테고리 멀티필터 (selectedCategories):
   - 배열이 비어있으면 → 패스 (전체)
   - 배열에 값이 있으면 → e.scheduleCategory가 존재하고 selectedCategories에 포함된 것만 유지
```

### B.2 lesson_schedules 필터링 (lessonScheduleEvents)

```

출력: lessonScheduleEvents (FullCalendar EventInput[])

각 schedule에 대해:
1. 색상: s.lessonColor ?? "#3b82f6"
2. 정원 체크: isFull = s.currentCount >= s.capacity

3. 강사 필터: selectedInstructor가 있고 String(s.instructorId) !== selectedInstructor → null 반환
4. 수업명 필터: selectedLessonFilter가 있고 s.lessonName !== selectedLessonFilter → null 반환
5. 정원 필터:
   - "여유" && isFull → null 반환
   - "마감" && !isFull → null 반환

6. 이벤트 변환:
   - id: "ls-{s.id}"
   - title: s.lessonName ?? "수업"
   - start: s.startAt
   - end: s.endAt
```

## 부록 D: 노쇼/취소 정책 상세 명세

### D.1 정책 데이터 구조

정책은 `branchStorage`의 `lesson_policy` 키에 JSON으로 저장됨.

```typescript
interface LessonPolicy {
  cancelDeadlineHours: number;         // 취소 마감 (시간), 기본: 3
  noShowDeductsSession: boolean;       // 노쇼 시 횟수 차감, 기본: true
  autoCompleteHours: number;           // 자동 완료 (시간), 기본: 24
  lateCancelPenalty: boolean;          // 마감 후 취소 시 패널티, 기본: true
  maxNoShowCount: number;              // 연속 노쇼 경고 기준, 기본: 3
  reservationAutoOpenHours: number;    // 예약 자동 오픈 (시간), 기본: 48
  waitlistEnabled: boolean;            // 대기열 기능, 기본: true
  waitlistAutoPromote: boolean;        // 자동 승격, 기본: true
}
```

### D.2 정책 로드/저장 메커니즘

- **로드**: `readBranchJson<Record<string, unknown>>('lesson_policy', {}, branchId)` → 각 필드를 state에 설정 (누락 필드는 기본값 사용)
- **저장**: `writeBranchJson('lesson_policy', policyValue, branchId)` → localStorage 기반 영속화

### D.3 정책 적용 범위

| 정책 | 적용 시점 | 동작 |
|------|-----------|------|
| cancelDeadlineHours | 회원이 예약 취소 요청 시 | 수업 시작 N시간 전까지만 취소 가능 |
| lateCancelPenalty | 마감 이후 취소 시 | 취소해도 횟수 차감 (수업 진행으로 간주) |
| noShowDeductsSession | 노쇼 판정 시 | 예약 상태 유지 + 세션 횟수 차감 |
| maxNoShowCount | 회원 누적 노쇼 N회 도달 시 | 회원 메모에 경고 기록 |
| autoCompleteHours | 수업 종료 후 N시간 경과 | 미처리 건 자동 completed 전환 |
| reservationAutoOpenHours | 수업 시작 N시간 전 도달 시 | 예약 자동 오픈 |
| waitlistEnabled | 정원 초과 예약 시 | 대기 등록 허용 |
| waitlistAutoPromote | 예약 취소 발생 시 | 대기 1순위 자동 예약 전환 |

```
attendeeCount = lesson_bookings에서 해당 scheduleId의 ATTENDED 건수

attendRate = bookedCount > 0 ? round((attendeeCount / bookedCount) * 100) : 0
```

### H.4 강사 현황 (instructor-status) — summary

```
totalInstructors = instructors.length
totalClasses = sum(instructors.classCount)
totalHours = round(sum(instructors.totalMinutes) / 60)
avgAttendanceRate = totalBookings > 0 ? round((totalAttended / totalBookings) * 100) : 0
avgNoShowRate = totalBookings > 0 ? round((totalNoShow / totalBookings) * 100) : 0
```

개별 강사 통계:
```
classCount = 해당 강사의 기간 내 수업 수

bookingCount = 해당 강사 수업의 예약 건수 (CANCELLED 제외)
attendedCount = 해당 강사 수업의 ATTENDED 건수
noShowCount = 해당 강사 수업의 NOSHOW/NO_SHOW 건수
attendanceRate = bookingCount > 0 ? round((attendedCount / bookingCount) * 100) : 0
noShowRate = bookingCount > 0 ? round((noShowCount / bookingCount) * 100) : 0
salesAmount = sales 테이블에서 staffName이 강사명과 일치하는 amount 합산
```

### Q.1 Supabase 쿼리 에러 처리

모든 Supabase 쿼리는 다음 패턴을 따른다:

```typescript
// 패턴 1: 조회 (에러 무시, 빈 결과)
const { data, error } = await supabase.from('table').select('*').eq('field', value);
if (!error && data) {
  setState(data);
}

// 패턴 2: 변경 (에러 표시)
const { error } = await supabase.from('table').update(payload).eq('id', id);
if (error) {
  toast.error('작업에 실패했습니다.');
} else {
  toast.success('작업이 완료되었습니다.');
  fetchData(); // 목록 새로고침
}

// 패턴 3: API 엔드포인트 (throw)
try {
  const { data, error } = await supabase.from('table').select('*');
  if (error) throw new Error(error.message);
  return { success: true, data };
} catch (err) {
  console.error('함수명 오류:', err);
  throw err;
}
```
