## 1. 도메인 개요

시설관리 도메인은 피트니스 센터 내 물리적 시설물(락커, 사물함, RFID 밴드/카드, 운동룸, 골프 타석, 운동복)의 배정, 상태, 이력을 통합 관리하는 모듈이다.

### 1.1 페이지 구성

| 경로 | 페이지명 | 화면 코드 | UC |
|------|----------|-----------|-----|
| `/locker` | 락커 관리 | SCR-050 | UC-023 |
| `/locker/management` | 사물함 배정 관리 | SCR-051 | UC-023 |
| `/rfid` | RFID/밴드 관리 | SCR-052 | UC-033 |
| `/rooms` | 운동룸 관리 | SCR-053 | UC-032 |
| `/golf-bays` | 골프 타석 관리 | - | UC-030 |
| `/clothing` | 운동복 관리 | - | UC-031 |

### 1.2 접근 권한

- 사이드바 > 시설 관리 메뉴 그룹에서 진입
- branchId 기반 지점 격리 (localStorage 또는 `getBranchId()` 유틸)
- 슈퍼관리자, 지점관리자, 매니저 권한 접근 가능

### 1.3 기술 스택

| 구분 | 기술 |
|------|------|
| 프레임워크 | Next.js App Router (`'use client'`, `force-dynamic`) |
| 상태관리 | React useState/useMemo/useEffect (로컬 상태 + Supabase 실시간 조회) |
| DB | Supabase (tables: `lockers`, `members`, `clothing`, `users`, `audit_logs`) |
| 영속 저장 | localStorage (`branchStorage`) — 운동룸 설정 등 |
| UI 컴포넌트 | AppLayout, PageHeader, StatCard, DataTable, Modal, ConfirmDialog 등 |
| 알림 | sonner `toast` |
| 엑셀 | `exportToExcel` 유틸 |

## 2. 락커 관리 (`/locker`)

> **소스**: `src/app/(facilities)/locker/page.tsx`
> **화면 코드**: SCR-050
> **UI 명세**: UI-053 ~ UI-056

### 2.1 개요

센터 내 모든 락커의 배정 현황, 상태, 이력을 시각적 그리드(4x8)로 관리하는 메인 화면이다. 구역별(A/B/C) 탭 전환, 개별/일괄 배정·해제, 고장 처리, 번호 이동, 비밀번호/메모 관리 기능을 제공한다.

### 2.2 데이터 모델

```typescript
type LockerStatus = "available" | "in_use" | "expiring" | "broken";

interface LockerData {
  id: string;
  number: number;
  zone: "A" | "B" | "C";
  status: LockerStatus;
  memberName?: string;
  memberId?: string;
  assignedDate?: string;
  expiryDate?: string;
  password?: string;
  memo?: string;
  history?: { date: string; action: string; member: string }[];
}
```

### 2.3 DB 연동

- **테이블**: `lockers`
- **조회 컬럼**: `id, number, status, memberId, memberName, assignedAt, expiresAt, branchId, password, memo`
- **필터**: `branchId = getBranchId()`
- **상태 매핑**: DB status (`AVAILABLE`, `IN_USE`, `MAINTENANCE`) → UI status (`available`, `in_use`, `broken`)
- **자동 판정**: `in_use` 상태 + 만료일 7일 이내 → `expiring` 자동 전환
- **zone 매핑**: DB의 `zone` 또는 `section` 필드 사용, 없으면 기본 `"A"`
- **회원 목록**: `members` 테이블에서 `status = 'ACTIVE'`인 회원 조회 (배정용)
- **이력 조회**: `audit_logs` 테이블, `target_type = 'locker'`, `target_id = locker.id`

- 탭 우측에 만료임박 일괄 해제 / 일괄 배정 기능 버튼 표시

- **셀 표시 내용**:
  - 락커 번호 (13px, bold)
  - 배정된 회원명 (11px, 최대 60px truncate)

- **선택 효과**: `ring-2 ring-primary ring-offset-2`

| 배정 | 회원 배정 모달 오픈 | `status === "available"` |

### 2.6 액션 모달

#### 2.6.1 이력 보기 모달 (`action === "history"`)

- `audit_logs` 테이블에서 해당 락커 이력 조회 (target_type: locker)
- 이력 테이블: 날짜 / 액션 배지(배정=success, 회수=default 등) / 회원명
- 하단에 비밀번호 입력 필드 + 메모 텍스트에어리어 + 저장 버튼
- 저장 시 `lockers` 테이블 `password`, `memo` 컬럼 업데이트

#### 2.6.2 번호 이동 모달 (`action === "move"`)

- 이동할 새 락커 번호 입력 (number input)
- 확인 시 `lockers` 테이블 `number` 컬럼 업데이트
- 취소/이동 버튼 (Button 컴포넌트)

#### 2.6.3 회수 처리 (`action === "reclaim"`)

- ConfirmDialog: "N번 락커를 회수하시겠습니까?"
- 확인 시 `lockers` 테이블 update: `status='available'`, `memberId=null`, `memberName=null`, `assignedAt=null`, `expiresAt=null`

#### 2.6.4 고장 처리/해제 (`action === "broken"`)

- ConfirmDialog: "N번 락커를 고장 처리하시겠습니까?" / "고장을 해제하시겠습니까?"
- `available ↔ broken` 토글
- `lockers` 테이블 `status` 컬럼 업데이트

#### 2.6.5 개별 배정 모달 (`action === "assign"`)

- 회원 검색 AutoComplete (이름/전화번호)
- 만료일 입력 (date input)
- 확인 시 `lockers` 테이블 update: `status='in_use'`, `memberId`, `memberName`, `assignedAt`, `expiresAt`

### 2.7 일괄 기능

#### 2.7.1 만료임박 일괄 해제

1. "만료임박 일괄 해제" 버튼 클릭 → `bulkMode = true`
2. 안내 배너: "만료임박 락커를 클릭하여 선택하세요" (노랑 배경)
3. `expiring` 상태 락커만 클릭 가능, 나머지 `opacity-40` + `cursor-not-allowed`
4. 선택된 셀에 빨간 체크 아이콘
5. "N개 일괄 해제" 버튼 클릭 → ConfirmDialog
6. 확인 시 `lockers` 테이블 `.in("id", ids)` 일괄 update

#### 2.7.2 일괄 배정

1. "일괄 배정" 버튼 클릭 → `bulkMode = true`, `bulkAssignOpen = true`
2. 안내 배너: "빈 락커를 클릭하여 선택하세요" (파랑 배경)
3. `available` 상태 락커만 클릭 가능
4. 우측 패널에서 회원 검색 → 만료일 입력 → 일괄 배정 확정
5. 확인 시 `lockers` 테이블 `.in("id", ids)` 일괄 update

### 2.9 D-Day 계산 로직

```
getDDay(expiryDate):
  today = new Date() (시간 00:00:00)
  expiry = new Date(expiryDate) (시간 00:00:00)
  return floor((expiry - today) / 86400000)

getDDayLabel(expiryDate):
  d < 0  → "D+|d|" (초과)
  d === 0 → "D-0" (당일)
  d > 0  → "D-d" (잔여)
```

**색상 규칙**:
- D-Day <= 0: `text-state-error` (빨강, bold)
- D-Day 1~7: `text-amber-600` (주황, semibold)
- D-Day > 7: 기본 색상

## 3. 사물함 배정 관리 (`/locker/management`)

> **소스**: `src/app/(facilities)/locker/management/page.tsx`
> **화면 코드**: SCR-051
> **UI 명세**: UI-089 ~ UI-092

### 3.1 개요

일일/개인/골프 사물함의 배정 워크플로우를 전담하는 화면이다. 회원 검색 → 빈 사물함 선택 → 배정 완료의 3단계 플로우와, 만료 사물함 일괄 해제 기능을 제공한다.

### 3.2 데이터 모델

```typescript
type LockerStatus = "available" | "in_use" | "overtime" | "abnormal" | "disabled";
type LockerType = "daily" | "personal" | "golf";

interface Locker {
  id: string;
  number: string;
  type: LockerType;
  status: LockerStatus;
  gender?: "M" | "F";
  userName: string | null;
  expiryDate: string | null;
}
```

### 3.3 DB 연동

- **테이블**: `lockers` (join: `members`)
- **조회**: `lockers.*, member:members(name, phone)` where `branchId = getBranchId()`
- **정렬**: `number` 오름차순
- **상태 매핑**: `AVAILABLE → available`, `IN_USE → in_use` (만료 시 `overtime`), `MAINTENANCE → disabled`
- **타입 분류**: 현재 모든 락커를 `daily` 탭에 표시 (향후 `type` 필드로 분류 예정)
- **회원 목록**: `members` 테이블에서 전체 회원 조회 (배정용)

### 3.4 화면 구성

#### 3.4.1 통계 카드 (StatCardGrid, 5열)

| 카드 | 값 | 아이콘 | variant |
|------|-----|--------|---------|
| 총 사물함 | `total` | User | - |
| 사용 가능 | `available` 개수 | CheckCircle2 | mint |
| 사용 중 | `in_use` 개수 | User | - |
| 시간 초과 | `overtime` 개수 | AlertTriangle | peach |
| 상태 비정상 | `abnormal` 개수 | XCircle | - |

#### 3.4.2 탭 구성

| 탭 | key | 설명 |
|-----|-----|------|
| 일일 사물함 | `daily` | 일일 이용 사물함 (카운트 배지) |
| 개인 사물함 | `personal` | 장기 배정 사물함 |
| 골프 사물함 | `golf` | 골프 전용 사물함 |

- 탭 전환 시 선택된 락커 초기화
- 최종 갱신 시각 표시 (`isLoading ? "동기화 중..." : 현재 시각`)

#### 3.4.3 배정 패널 (UI-089 ~ UI-091)

배정 패널은 `bg-surface-secondary/50` 라운드 박스 안에 2열 그리드로 구성:

**회원 검색 (UI-089)**:
- Search 아이콘 + 텍스트 입력 (회원명, 연락처, 회원번호)
- 입력 시 AutoComplete 드롭다운 표시
  - 각 항목: 아바타 원 + 이름(bold) + 회원번호/연락처
  - 클릭 시 선택 완료 → 초록 배지 표시 (CheckCircle2 + 이름 + 회원번호)
  - X 버튼으로 선택 해제

**추천 락커 (FN-050)**:
- 번호가 가장 작은 `available` 상태 사물함을 자동 추천
- "N번 락커 (최소 번호)" 버튼으로 표시, 클릭 시 선택
- 사용 가능 락커 없으면 "사용 가능한 락커 없음" 비활성 표시

**선택된 락커**:
- 락커 선택 시 파란 배경 상태 표시 ("N번 락커 선택됨")
- 미선택 시 "아래 그리드에서 빈 락커를 선택하세요" 안내

**만료일 입력**:
- date input, 기본값: 오늘 + 3개월
- 필수 입력

**배정 버튼 (UI-091)**:
- `회원 선택 + 락커 선택` 모두 완료 시에만 활성화
- 비활성 시: `bg-surface-tertiary text-content-tertiary cursor-not-allowed`
- 활성 시: `bg-primary text-white`
- 배정 완료 시 3초간 성공 메시지 표시

#### 3.4.4 일괄 해제 배너 (UI-092)

- `overtime` 상태 사물함이 1개 이상 존재 시 노란 배너 표시
- "시간 초과 사물함 N개 — 일괄 반납 처리가 필요합니다"
- "N개 일괄 해제" 버튼 → ConfirmDialog → `overtime` 사물함 전체를 `available`로 변경

#### 3.4.5 빈 사물함 선택 그리드 (UI-090)

### 4.2 데이터 모델

## 7. 운동복 관리 (`/clothing`)

> **소스**: `src/app/(facilities)/clothing/page.tsx`
> **UC**: UC-031

### 7.1 개요

운동복 재고 현황과 대여/반납/세탁/파손 상태를 관리한다. QR 코드 스캔으로 빠른 대여/반납 처리를 지원하며, Supabase `clothing` 테이블과 실시간 연동된다.

### 7.2 데이터 모델

```typescript
interface ClothingItem {
  id: number;
  number: string;        // 운동복 번호
  qrCode?: string;       // QR 코드 (자동 생성)
  size: string;          // S / M / L / XL / XXL
  type: string;          // TOP(상의) / BOTTOM(하의) / SET(세트)
  status: string;        // AVAILABLE / RENTED / WASHING / DAMAGED
  memberId?: number;
  memberName?: string;
  rentedAt?: string;     // 대여일 (YYYY-MM-DD)
  returnDue?: string;    // 반납예정일
  memo?: string;
}
```

### 7.3 DB 연동

- **테이블**: `clothing`
- **조회**: `clothing.*` where `branchId = getBranchId()`, order by `number`
- **등록**: `clothing.insert({ branchId, number, size, type, status: 'AVAILABLE', memo })`
- **상태 변경**: `clothing.update({ status, memberId?, memberName?, rentedAt?, returnDue? }).eq('id', id)`
- **QR 조회**: `clothing.select('*').eq('branchId').or('qrCode.eq.{input}, number.eq.{input}').limit(1)`
- **회원 검색**: `members.select('id, name, phone').eq('branchId').ilike('name', '%{query}%').limit(10)`
- **테이블 미존재 대응**: 조회 실패 시 빈 목록 표시 (에러 무시)

```
    ↑                   │                            │
    │                   ↓                            ↓
    ←──(회수/해제)──── in_use ←──────────────── expiring
    │
    ↓
  broken ←──(고장 처리)── available/in_use
    │
    └──(고장 해제)──→ available
```

### 10.2 운동복 상태 머신

```
AVAILABLE ──(대여)──→ RENTED ──(반납)──→ AVAILABLE
    │                   │
    │                   └──(세탁)──→ WASHING ──(세탁완료)──→ AVAILABLE
    │
    └──(파손)──→ DAMAGED ──(복구)──→ AVAILABLE
    
RENTED ──(파손)──→ DAMAGED
```

### 10.3 골프 타석 상태 머신

```
available ──(이용 시작)──→ in_use ──(이용 종료)──→ available
    │                        │                        ↑
    │                        └──(타석 이동)────→ (대상 타석 in_use)
    │
    ├──(점검 처리)──→ maintenance ──(점검 해제)──→ available
    │
    └──(대기열 자동 배정)──→ reserved ──(이용 시작)──→ in_use
```

### 10.4 운동룸 상태 순환

```
운영중 → 점검중 → 고장 → 운영중 (순환)
```

### 10.5 핵심 비즈니스 정책

| 정책 | 규칙 |
|------|------|
| 만료임박 판정 | D-Day 7일 이내 (D-0 ~ D-7) → `expiring` 자동 전환 |
| 추천 락커 | 가용 사물함 중 번호가 가장 작은 것 자동 추천 |
| 대기열 자동 배정 | 타석 종료 시 대기열 첫 번째 대기자를 해당 타석에 `reserved`로 자동 배정 |
| 예상 대기 시간 | 이전 대기자의 예상 시간 + 15분 (없으면 0분) |
| 타석 기본 이용 시간 | 60분 (durationMin) |
| 타석 타이머 | 1분(60초) 간격으로 `remainingMin` 자동 감소 |
| 타석 종료 경고 | 잔여 5분 이내 → 노란 경고, 0분 → 빨간 깜박 + 토스트 |
| 운동복 QR 대여 | QR 코드 또는 번호로 조회 → 회원 검색·선택 → 대여 확정 |
| 운동복 QR 반납 | QR 코드 또는 번호로 조회 → 반납 처리 (한 번에 완료) |
| RFID 스캔 | 리더기 태그 시 카드 번호 자동 입력 (현재 시뮬레이션) |
| 일괄 배정 | 빈 락커 다중 선택 → 단일 회원에게 일괄 배정 |
| 일괄 해제 | 만료임박/시간초과 락커 다중 선택 → 일괄 회수 |
