| 영역 | 설명 |
|------|------|
| 로고 | FitGenie CRM 로고 + "피트니스 센터 통합 관리 솔루션" 설명 |

| 로그인 버튼 | `Button variant="primary" fullWidth` — 로딩/잠금 상태에 따라 텍스트 변경 |
| 고객센터 | 하단 — "02-1234-5678 (평일 09:00~18:00)" |

### 1.2 로그인 잠금 메커니즘

| 항목 | 값 |
|------|-----|
| 최대 실패 횟수 | 5회 (`MAX_FAIL_COUNT`) |
| 잠금 지속 시간 | 30분 (`LOCK_DURATION_MS = 30 * 60 * 1000`) |
| 실패 카운트 저장 | `localStorage` 키: `login_fail_count` |
| 잠금 시각 저장 | `localStorage` 키: `login_fail_locked_until` |
| 잠금 해제 방식 | 자동 — 30분 경과 시 `getLoginLockState()` 에서 자동 해제 + 카운트 초기화 |

**잠금 흐름**:

```
로그인 시도 → 실패 → incrementFailCount()
  ├ failCount < 5 → "이메일 또는 비밀번호가 올바르지 않습니다. (N회 더 실패 시 잠깁니다)"
  └ failCount >= 5 → lockedUntil = now + 30분 저장
                    → "계정이 잠겼습니다. 본사 IT팀에 문의하세요."
                    → 버튼 disabled + 텍스트 "계정 잠금"

30분 경과 → getLoginLockState() → isLocked: false
         → localStorage 키 삭제 → 재시도 가능
```

### 1.3 로그인 성공 처리

```
로그인 성공 → clearFailCount()
            → rememberMe ? localStorage.setItem(아이디) : 삭제
            → authStore.login({
                id, name, email, role, branchId, branchName,
                tenantId, isSuperAdmin, currentBranchId
              }, accessToken)
            → localStorage.setItem('branchId', ...)
            → moveToPage(966) → 대시보드 이동
```

**슈퍼관리자 특이사항**:
- `user.branchId`가 `null`일 수 있음 — 선택한 지점 ID 사용 또는 빈 문자열
- `branchName`: 지점 선택 시 해당 지점명, 미선택 시 "전체 지점"
- `isSuperAdmin: true` 저장 → 사이드바에 본사 관리 메뉴 노출

### 1.4 유효성 검증

| 검증 | 조건 | 처리 |
|------|------|------|
| 폼 유효성 | `id.trim() !== '' && password.trim() !== ''` | 미충족 시 버튼 disabled |
| 잠금 상태 | `isLocked === true` | 버튼 disabled + "계정 잠금" 표시 |
| API 실패 | `onError` 콜백 | "서버 연결에 실패했습니다." 에러 표시 |

| 항목 | 내용 |
|------|------|

| **UC 참조** | UC-002 |

### 2.1 전환 방식

| 트리거 | 동작 |
|--------|------|
| 사이드바 드롭다운 선택 | `authStore.switchBranch(branchId, branchName)` 호출 |
| 통합 대시보드 지점 카드 [상세 보기] | `switchBranch()` + `router.push('/')` → 지점 대시보드 이동 |
| 지점 관리 테이블 지점명 클릭 | `switchBranch()` + 토스트 "N 지점으로 전환되었습니다" |

### 2.2 전환 시 데이터 갱신

- `localStorage.setItem('branchId', selectedBranchId)` — 모든 페이지에서 `getBranchId()` 함수로 참조
- 현재 보고 있는 페이지의 데이터가 선택된 지점 기준으로 자동 리로드
- 토스트 알림: `"${branchName}" 지점으로 전환되었습니다.`

### 2.3 제약사항

- 폐점 상태(`status === 'closed'`) 지점 선택 시 → 조회 전용 모드 (데이터 수정/신규 등록 제한)

### 4.2 KPI 카드 (7개)

| # | 라벨 | 계산 | 변동률 |
|---|------|------|--------|
| 1 | 전체 회원 | 전 지점 활성 회원 합산 | 전월 대비 MoM% |

**MoM 계산 함수**:
```
mom(curr, prev) = prev > 0 ? round((curr - prev) / prev * 100) : 0
```

### 4.3 위험 신호 섹션

**자동 감지 규칙** (3가지):

| # | 규칙 | 조건 | 메시지 예시 |
|---|------|------|------------|

## 비즈니스 규칙 요약

### BR-001. 로그인 잠금 정책

- 5회 연속 실패 시 30분 계정 잠금
- 잠금 기간 경과 시 자동 해제
- 잠금 상태: localStorage 기반 (서버 사이드 잠금 미구현)

### BR-002. 지점 상태 전이

```
active (운영중) ──→ inactive (임시휴업) ──→ active (운영중)
                └─→ closed (폐점)
```

- 폐점 시: 활성 회원 수 + 재직 직원 수 확인 → 영향 범위 안내
- 폐점 지점: 조회 전용 모드

### BR-003. 위험 신호 감지

| 규칙 | 조건 | 심각도 |
|------|------|--------|
| 매출 급감 | 이번달 매출 <= 전 지점 평균의 50% 또는 0원 | 높음 (amber) |
| 만료 회원 급증 | 만료 회원 >= 활성 회원의 30% | 중간 (amber) |
| 출석 저조 | 오늘 출석 <= 활성 회원의 10% | 중간 (amber) |
| 전사 매출 급감 | 이번달 전체 매출 < 전월의 80% | 매우 높음 (red alert) |

### BR-004. 신규 회원 온보딩 상태

| 상태 | 판정 기준 |
|------|----------|
| 정상 | 기본 상태 (위험 조건 미충족) |
| 미방문 | 등록 7일 이내 + 방문 0회 |
| 주의 | 등록 7일 이내 + 방문 1회 |
| 위험 | 등록 7일 초과 + 방문 1회 이하 |
| 이탈위험 | 등록 14일 초과 + 방문 3회 이하 |

### BR-005. 코호트 유지율 계산

- 코호트: 등록월 기준 그룹
- 유지 판정: 해당 월말 시점에 `status === 'ACTIVE'` 또는 `membershipExpiry >= 해당월말`
- 유지율: 유지 회원 / 코호트 등록 회원 * 100
- 미도래 월: null (대시 표시)

### BR-006. 퍼널 전환율

- 전환율 = 현재 단계 수 / 직전 단계 수 * 100
- 바 너비 = 현재 단계 수 / 첫 단계 수 * 100
- 첫 단계(리드 유입) 전환율은 "기준"으로 표시

### BR-007. 매출 달성률 게이지

- 목표: 지점별 + 월별로 localStorage에 저장
- 달성률 = 이번달 매출 / 목표 * 100 (최대 999%)
- 바 너비 = min(달성률, 100)%

### BR-008. Today Tasks 배정

- 본사 업무 풀 100개에서 날짜 + 지점 + 역할 기준으로 결정론적 랜덤 배정
- AI 미사용 — 동일 날짜/지점/역할이면 항상 동일 태스크 배정
- 슈퍼관리자만 CRUD 가능
- 상태 전환: 대기 → 진행중 → 완료 (단방향)

### BR-009. 자동 리포트 생성 규칙

- 일간: 최근 7일 (어제부터)
- 주간: 최근 4주 (일요일 기준 주간)
- 월간: 최근 3개월 (1일~말일)
- 수동 생성: 유형 + 기간 지정 → GENERATING → READY/FAILED

### BR-010. Soft Delete 정책

- `members` 테이블: `deletedAt IS NULL` 필터 필수
- 모든 회원 관련 쿼리에 `.is("deletedAt", null)` 조건 적용
- 삭제된 회원은 통계에서 제외
