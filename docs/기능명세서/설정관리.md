## 1. 도메인 개요

설정관리 도메인은 FitGenie CRM의 센터 운영 인프라를 구성하는 핵심 관리 영역이다. 센터 기본정보, 알림 정책, 테마, 권한, 키오스크, IoT 기기 연동, 구독 플랜, 공지사항, 출석 관리를 포함한다.

### 1.1 대상 페이지 맵

| 경로 | 페이지명 | 소스 파일 | UC 참조 |
|------|----------|-----------|---------|
| `/settings` | 센터 설정 | `src/app/settings/page.tsx` | UC-028 |
| `/settings/permissions` | 권한 설정 | `src/app/settings/permissions/page.tsx` | UC-028, A-61 |
| `/settings/kiosk` | 키오스크 설정 | `src/app/settings/kiosk/page.tsx` | UC-028, A-25 |
| `/settings/iot` | 출입문/IoT 설정 | `src/app/settings/iot/page.tsx` | UC-028, A-86 |
| `/subscription` | 구독 관리 | `src/app/subscription/page.tsx` | UC-007 |
| `/notices` | 공지사항 | `src/app/notices/page.tsx` | UC-029, A-95 |
| `/attendance` | 출석 관리 | `src/app/attendance/page.tsx` | UC-015, A-70 |

### 1.2 기술 스택 요약

| 항목 | 사용 기술 |
|------|-----------|
| 프레임워크 | Next.js (App Router, `'use client'`) |
| 상태 관리 | React `useState` / `useEffect` / Zustand (`useAuthStore`) |
| 데이터 소스 | Supabase (branches, settings, attendance, notices 테이블) |
| 실시간 | Supabase Realtime (attendance INSERT 구독) |

## 2. 센터 설정

**경로**: `/settings`
**소스**: `src/app/settings/page.tsx`
**UC 참조**: UC-028

### 2.1 페이지 구조

4개 탭(TabNav)으로 구성되며, 각 탭별 독립적인 dirty state를 관리한다.

| 탭 키 | 탭 명 | 아이콘 | 설명 |
|--------|--------|--------|------|
| `basic` | 기본정보 | `Info` | 센터 대표 정보 및 영업시간 |
| `notification` | 알림설정 | `Bell` | 푸시/이메일/SMS 알림 토글 |
| `theme` | 테마설정 | `Palette` | 다크모드, 색상, 글자 크기 |
| `supplies` | 물품 관리 | `Package` | 수건/운동복 재고 관리 |

### 2.2 기본정보 탭 (`basic`)

#### 2.2.1 센터 기본정보 섹션

| 필드 | 타입 | 필수 | 비고 |
|------|------|:----:|------|
| 센터 로고 | 이미지 업로드 | | 120x120px, hover 시 변경 버튼 노출 |
| 센터명 | text | * | `branches.name` + `settings.centerName` 동기화 |
| 사업자등록번호 | text | | |
| 센터 소개 | textarea | | |
| 대표 연락처 | text | | `branches.phone` 동기화 |
| 주소 | text + 주소검색 버튼 | | `branches.address` 동기화, readOnly 필드 |
| 업종 선택 | 체크박스 그룹 | | 헬스, 필라테스, PT샵, 골프, 요가, 태권도, 크로스핏, 복싱, 수영, 사우나, 기타 |

#### 2.2.2 영업시간 설정 섹션

| 필드 | 타입 | 기본값 | 비고 |
|------|------|--------|------|
| 평일 오픈~마감 | time (2개) | 09:00 ~ 22:00 | `settings.businessHoursOpen/Close` |
| 주말/공휴일 오픈~마감 | time (2개) | 09:00 ~ 20:00 | |

#### 2.2.3 저장 동작

- `branches` 테이블: `name`, `address`, `phone` 업데이트
- `settings` 테이블: `centerName`, `businessHoursOpen`, `businessHoursClose` 업데이트
- 저장 성공 시 3초간 "변경사항이 저장되었습니다" 알림 표시

### 2.3 알림설정 탭 (`notification`)

3개 섹션(푸시/이메일/SMS)으로 구성되며, 각 항목은 토글 스위치로 제어한다.

#### 2.3.1 푸시 알림

| 항목 키 | 라벨 | 설명 |
|---------|------|------|
| `pushEntrance` | 실시간 입장 알림 | 회원이 센터 입장 시 관리자 앱으로 푸시 알림 |
| `pushExpiry` | 회원 만료 임박 알림 | 이용권 만료 D-7 이내 발생 시 알림 |
| `pushPayment` | 결제 완료 알림 | 새로운 결제 발생 시 알림 |
| `pushReservation` | 수업 예약/취소 알림 | 수업 예약 또는 취소 발생 시 강사에게 알림 |

#### 2.3.2 이메일 알림

| 항목 키 | 라벨 | 설명 |
|---------|------|------|
| `emailWeeklyReport` | 주간 리포트 | 매주 월요일 오전 지난 주 운영 통계 메일 발송 |
| `emailExpiry` | 만료 예정 안내 메일 | 이용권 만료 D-30 회원 목록 메일 발송 |

#### 2.3.3 SMS 알림

| 항목 키 | 라벨 | 설명 |
|---------|------|------|
| `smsPayment` | 결제 완료 SMS | 결제 완료 시 회원에게 SMS 발송 |
| `smsExpiry` | 만료 임박 SMS | 이용권 만료 D-7 회원에게 SMS 자동 발송 |

#### 2.3.4 저장 동작

- `settings.smsEnabled` = `smsPayment || smsExpiry`
- `settings.autoExpireNotify` = `pushExpiry || emailExpiry`

### 2.4 테마설정 탭 (`theme`)

#### 2.4.1 다크모드 설정

3개 옵션 카드 선택 방식:

| 값 | 라벨 | 아이콘 | 설명 |
|----|------|--------|------|
| `light` | 라이트 모드 | `Sun` | 밝은 배경 |
| `dark` | 다크 모드 | `Moon` | 어두운 배경 |
| `system` | 시스템 설정 | `Monitor` | 기기 설정 따름 |

#### 2.4.2 색상 설정

| 필드 | 기본값 | 비고 |
|------|--------|------|
| 메인 컬러 (Primary) | `#3B82F6` | 컬러 피커 + HEX 표시 |
| 보조 컬러 (Accent) | `#10B981` | 컬러 피커 + HEX 표시 |

#### 2.4.3 글자 크기

| 값 | 라벨 | 미리보기 |
|----|------|----------|
| `sm` | 작게 | text-sm |
| `md` | 기본 | text-base |
| `lg` | 크게 | text-lg |

#### 2.4.4 저장 동작

- `settings.theme` 컬럼에 모드 값 저장

### 2.5 물품 관리 탭 (`supplies`)

수건/운동복 발급 재고를 관리하는 테이블 형태 입력 UI.

#### 2.5.1 물품 항목

| 키 | 물품명 | 필드 |
|----|--------|------|
| `towelLarge` | 수건 (대) | 재고 수량, 1일 발급량, 현재 잔여량 |
| `towelSmall` | 수건 (소) | 재고 수량, 1일 발급량, 현재 잔여량 |
| `uniformTop` | 운동복 (상의) | 재고 수량, 1일 발급량, 현재 잔여량 |
| `uniformBottom` | 운동복 (하의) | 재고 수량, 1일 발급량, 현재 잔여량 |

- 모든 필드는 `number` 타입, 최솟값 0
- `settings.supplies` 컬럼에 JSON으로 저장

### 2.6 미저장 경고 시스템

| 기능 | 설명 |
|------|------|
| Dirty State 추적 | 탭별 독립적인 `tabDirty` 상태 관리 |
| 탭 전환 경고 | dirty 상태에서 탭 전환 시 모달 표시 |
| 브라우저 이탈 방지 | `beforeunload` 이벤트 바인딩 |
| 모달 옵션 | 취소 / 저장 후 이동 / 저장 안 함 (3가지) |

## 3. 권한 설정

**경로**: `/settings/permissions`
**소스**: `src/app/settings/permissions/page.tsx`
**UC 참조**: UC-028 (권한 설정), A-61

### 3.1 페이지 구조

좌우 2패널 레이아웃:
- **좌측 (280px)**: 역할 목록 + 배정 직원 현황
- **우측 (flex-1)**: 권한 매트릭스 테이블

### 3.2 시스템 역할 (초기값)

| ID | 코드 | 역할명 | 설명 | 시스템 역할 |
|----|------|--------|------|:-----------:|
| 1 | `primary` | 최고관리자 | 모든 권한 (수정 불가) | O |
| 2 | `owner` | 센터장 | 경영/매출 전체 접근 | O |
| 3 | `manager` | 매니저 | 회원/상품/일정 관리 | O |
| 4 | `fc` | 피트니스 코치 | 담당 회원/수업 접근 | O |
| 5 | `staff` | 스태프 | 출석 체크/기본 조회 | O |
| 6 | `readonly` | 조회전용 | 읽기 전용 전체 | O |

### 3.3 메뉴 그룹 및 권한 항목

#### 3.3.1 메뉴 그룹 구조

| 그룹 | 메뉴 항목 |
|------|-----------|
| 회원 | 회원 목록, 회원 상세, 회원 등록/수정 |
| 수업 | 수업/캘린더 |
| 매출 | 매출 현황, POS 결제 |
| 상품 | 상품 관리 |
| 직원 | 직원 관리 |
| 급여 | 급여 관리 |
| 시설 | 락커 관리, 운동룸 관리, 밴드/카드 관리 |
| 메시지 | 메시지 발송, 쿠폰 관리, 마일리지 관리 |
| 설정 | 센터 설정, 키오스크 설정, IoT 설정, 권한 설정 |

#### 3.3.2 권한 타입 (PermissionType)

| 타입 | 컬럼 헤더 | 설명 |
|------|-----------|------|
| `access` | 접근 | 메뉴 접근 가능 여부 |
| `read` | 조회 | 데이터 열람 가능 여부 |
| `create` | 등록 | 신규 데이터 생성 가능 여부 |
| `update` | 수정 | 기존 데이터 수정 가능 여부 |
| `delete` | 삭제 | 데이터 삭제 가능 여부 |

### 3.4 권한 충돌 검증 로직

저장 시 자동으로 충돌을 감지하고, 경고 모달을 표시한다.

| 충돌 조건 | 경고 메시지 |
|-----------|-------------|
| 삭제 O + 수정 X | `"삭제 권한이 있지만 수정 권한이 없습니다"` |
| 쓰기(등록/수정/삭제) O + 접근 X | `"쓰기 권한이 있지만 접근 권한이 없습니다"` |
| 쓰기(등록/수정/삭제) O + 조회 X | `"조회 권한 없이 쓰기 권한이 설정되어 있습니다"` |

경고 발생 시 옵션:
- **수정하기**: 모달 닫고 수정 계속
- **경고 무시하고 저장**: 충돌 상태 그대로 저장

### 3.5 역할 관리 기능

| 기능 | 설명 |
|------|------|
| **새 역할 생성** | 역할 이름, 코드, 설명, 기준 역할 복사(선택) 입력 |
| **역할 복사** | 기존 역할의 권한 설정을 복제하여 새 역할 생성 |
| **역할 삭제** | 커스텀 역할만 삭제 가능 (시스템 역할 삭제 불가), 확인 다이얼로그 + 역할명 입력 확인 |
| **전체 허용** | 선택된 역할의 모든 메뉴에 전체 권한 부여 |
| **전체 차단** | 선택된 역할의 모든 메뉴 권한 해제 |
| **초기화** | 저장 이전 상태로 되돌리기 (확인 다이얼로그) |

### 3.6 최고관리자 제약

- `primary` 역할은 모든 권한이 `true`로 고정
- 권한 토글 비활성화 (`disabled`)
- hover 시 "최고 관리자 권한은 변경할 수 없습니다" 툴팁 표시
- 전체 허용/전체 차단/역할 복사 버튼 비활성화

### 3.7 배정 직원 현황 (하단 패널)

- Supabase `users` 테이블에서 `branchId` 기준 직원 조회
- 역할(`role`)별 그룹핑하여 배정 인원 수 표시
- 직원 이름 클릭 시 직원 상세로 이동 (`moveToPage(974)`)
- 배정 직원이 없는 경우 "배정된 직원이 없습니다" 표시

### 3.8 변경 표시 (Change Tracking)

- 변경된 셀에 `bg-amber-600/10` 배경 + `ring-2 ring-amber-600` 링 표시
- 헤더에 "저장되지 않은 변경 사항이 있습니다" 경고 배지
- 저장 완료 시 "저장되었습니다" 성공 배지 (3초 후 자동 숨김)

### 3.9 데이터 저장

- `localStorage`에 `permissions_{roleCode}` 키로 저장
- 메뉴 ID별 권한 객체 (`Record<string, MenuPermission['permissions']>`)

## 4. 키오스크 설정

**경로**: `/settings/kiosk`
**소스**: `src/app/settings/kiosk/page.tsx`
**UC 참조**: UC-028 (키오스크 설정), A-25

### 4.1 페이지 구조

4개 탭으로 구성되며, 우측에 키오스크 미리보기(폰 프레임) 제공.

| 탭 키 | 탭 명 | 아이콘 | 설명 |
|--------|--------|--------|------|
| `basic` | 기본 설정 | `Settings` | 활성화, 타입, 입장방식, 시스템 설정, 기능 설정 |
| `screen` | 화면 설정 | `Monitor` | 로고, 배경, 색상, 환영 문구, 공지, 배너 |
| `tts` | TTS 설정 | `Volume2` | 이벤트별 음성 안내 메시지 편집 |
| `access` | 출입 규칙 | `Lock` | 출입 시간, 만료/미납 회원 제어 |

### 4.2 기본 설정 탭 (`basic`)

#### 4.2.1 키오스크 활성화

- 최상단 토글 스위치
- 활성: 녹색 보더 + "키오스크 체크인이 활성화되어 있습니다"
- 비활성: 빨강 보더 + "키오스크 체크인이 비활성화되어 있습니다"

#### 4.2.2 키오스크 타입 선택

| 값 | 라벨 | 아이콘 | 설명 |
|----|------|--------|------|
| `typeA` | 타입 A | `Smartphone` | 입퇴장 전용 키오스크 |
| `typeB` | 타입 B | `Monitor` | 결제 포함 풀 기능 키오스크 |

#### 4.2.3 입장 방식 (체크인 인증 수단)

다중 선택 가능한 체크박스 카드 형태:

| 값 | 라벨 |
|----|------|
| `qr` | QR 코드 |
| `rfid` | RFID/NFC |
| `face` | 얼굴 인식 |
| `pin` | 전화번호 입력 |
| `barcode` | 바코드 |

#### 4.2.4 얼굴 인식 설정 (조건부 표시)

`face` 체크인 활성화 시에만 노출:

| 필드 | 옵션 | 비고 |
|------|------|------|
