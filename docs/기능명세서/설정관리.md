### G. 모달/팝업

#### G-1. 미저장 경고 다이얼로그

- **트리거**: `isDirty=true` 상태에서 다른 탭 클릭
- **동작**: `setPendingTab(tab)` → `setShowUnsavedWarning(true)`
- **옵션**: `"저장하지 않고 이동"` → `handleDiscardAndSwitch()` / 현재 탭 유지

### H. 상태별 변화

| 상태 | UI 변화 |
|------|---------|
| isLoading=true | 초기 로딩 중 |
| isDirty=true | 탭 전환 시 미저장 경고 표시 |
| isSaving=true | 저장 버튼 로딩 상태 |
| saveSuccess=true | 3초간 성공 메시지 표시 후 자동 해제 |
| 브라우저 이탈 시도 (hasAnyDirty) | `beforeunload` 이벤트 → 브라우저 기본 경고 |

### L. 비즈니스 로직

1. **탭별 dirty 추적**: `tabDirty` 객체로 탭별 변경 여부 관리
2. **beforeunload 이벤트**: 미저장 상태에서 브라우저 이탈 방지
3. **markDirty()**: 모든 입력 onChange에서 호출 → 현재 탭 dirty 플래그 설정
4. **supplies JSON 파싱**: settings 테이블의 supplies 컬럼이 문자열이면 JSON.parse, 객체면 그대로 사용

### 미저장 경고 모달 상세

```
┌──────────────────────────────────────┐
│  ⚠ 저장되지 않은 변경사항              │
│  탭을 이동하면 변경사항이 사라집니다.     │
│                                      │
│    [취소]  [저장 후 이동]  [저장 안 함]  │
└──────────────────────────────────────┘
```

- **취소**: `setShowUnsavedWarning(false)` — 현재 탭 유지
- **저장 후 이동**: `handleSave()` → `setActiveTab(pendingTab)` → `setPendingTab(null)`
- **저장 안 함**: `handleDiscardAndSwitch()` → 현재 탭 dirty 해제 → 대기 탭으로 이동

### 저장 버튼 상태별 스타일

| 상태 | className |
|------|-----------|
| 저장 가능 (isDirty=true) | `bg-accent text-white hover:opacity-90` |
| 저장 불가 (isDirty=false) | `opacity-50 cursor-not-allowed` |
| 저장 중 (isSaving=true) | `animate-pulse` |

### PageHeader 우측 상태 표시

| 조건 | 표시 |
|------|------|
| isDirty && !saveSuccess | amber 배지: AlertTriangle + `"저장되지 않은 변경사항이 있습니다"` |
| saveSuccess | accent 배지: CheckCircle2 + `"변경사항이 저장되었습니다."` (3초 후 해제) |

### G. 모달/팝업

#### G-1. 역할 생성 모달

- **필드**: 역할 이름 (`name`), 역할 코드 (`code`), 설명 (`description`)
- **선택사항**: 기존 역할에서 복사 (`createSourceRole`)
- **저장**: `handleCreateRole(e)` → `FormData` 추출 → roles 배열에 추가 → 자동 선택

#### G-2. 역할 복사 모달

- **필드**: 복사할 역할 (`copySourceRoleId`), 새 역할 이름 (`copyNewRoleName`), 새 역할 코드 (`copyNewRoleCode`)
- **검증**: 3개 필드 모두 비어있으면 무시
- **저장**: `handleCopyRole()` → description=`"${sourceRole.name} 복사본"` → 새 역할 생성 → 자동 선택

#### G-3. 초기화 확인 다이얼로그

- **동작**: `handleReset()` → `savedPermissions`로 복원 → `isDirty=false`, `conflictWarnings=[]`

#### G-4. 역할 삭제 확인 다이얼로그

- **동작**: `handleDeleteRole()` → roles 배열에서 제거 → 첫 번째 역할 자동 선택

#### G-5. 권한 충돌 경고 모달

- **트리거**: 저장 시 `validateConflicts(permissions)` 결과 warnings > 0
- **충돌 규칙 3가지**:
  1. `delete=true && update=false`: `"\"${name}\": 삭제 권한이 있지만 수정 권한이 없습니다."`
  2. `(create || update || delete) && access=false`: `"\"${name}\": 쓰기 권한이 있지만 접근 권한이 없습니다."`
  3. `(create || update || delete) && read=false`: `"\"${name}\": 조회 권한 없이 쓰기 권한이 설정되어 있습니다."`
- **옵션**: 경고 무시하고 저장 (`doSave()`) / 취소

### L. 비즈니스 로직

1. **최고관리자 보호**: `code="primary"` 역할은 모든 메뉴 모든 권한 `true` 고정. `handleToggle`, `handleAllAllow`, `handleAllDeny` 모두 `isPrimary` 체크로 차단
2. **충돌 검증 (`validateConflicts`)**: 저장 전 3가지 논리적 충돌 규칙 검사. 충돌 발견 시 경고 모달 표시 후 사용자가 무시/수정 선택
3. **역할별 직원 수 매핑**: `employeesByRole` state → `users` 테이블에서 role 기준 그룹화 → 각 역할 카드에 `userCount` 반영
4. **권한 복원 우선순위**: localStorage 저장값 > INITIAL_PERMISSIONS 기본값 > all false
5. **변경 추적**: `isChanged(menuId, type)` 함수 — `permissions`와 `savedPermissions` 비교하여 셀 단위 변경 여부 판별
6. **시스템 역할 삭제 불가**: `isSystem=true`인 역할은 삭제 옵션 미노출 (사용자 정의 역할만 삭제 가능)

### L. 비즈니스 로직

3. **만료 회원 처리**: `allowExpired=deny` 시 만료 회원 체크인 거부, TTS 메시지 #2 재생

5. **TTS 변수 치환**: `{이름}` → 회원명, `{N}` → 잔여일수
6. **미리보기 실시간 반영**: `KioskPreview` 컴포넌트에 settings prop 전달 → 환영 문구, 테마 색상, 공지 표시 실시간 반영

### branchId 획득 패턴

설정관리 도메인에서 사용하는 3가지 branchId 획득 패턴:

```typescript
// 패턴 1: getBranchId() 유틸 함수 (import)
import { getBranchId } from '@/lib/getBranchId';
const branchId = getBranchId();

// 패턴 2: 인라인 함수 (일부 페이지)
const getBranchId = (): number => {
  if (typeof window === 'undefined') return 1;
  const stored = localStorage.getItem('branchId');
  return stored ? Number(stored) : 1;
};

// 패턴 3: 문자열 반환 (키오스크, IoT, 자동 알림)
function getBranchId() {
  if (typeof window === "undefined") return "1";
  return localStorage.getItem("branchId") || "1";
}
```

**참고**: 패턴 1이 권장되며, 패턴 2/3은 레거시 코드에서 사용됨.

### ToggleSwitch 컴포넌트 (설정관리 공통)

센터 설정 및 키오스크 설정에서 공통으로 사용하는 토글 스위치:

```typescript
function ToggleSwitch({ checked, onChange }: { checked: boolean; onChange: () => void }) {
  return (
    <button
      onClick={onChange}
      className={cn(
        "relative w-12 h-6 rounded-full transition-colors duration-200 outline-none focus:ring-2 focus:ring-accent/30",
        checked ? "bg-accent" : "bg-line"
      )}
    >
      <div className={cn(
        "absolute top-[2px] left-[2px] w-5 h-5 bg-white rounded-full shadow-sm transition-transform duration-200",
        checked ? "translate-x-6" : "translate-x-0"
      )} />
    </button>
  );
}
```

- **활성 상태**: `bg-accent`, 원형 thumb `translate-x-6`
- **비활성 상태**: `bg-line`, 원형 thumb `translate-x-0`
- **포커스 링**: `focus:ring-2 focus:ring-accent/30`
- **transition**: `duration-200` (색상 + 위치 동시)

이 컴포넌트는 `src/components/ui/Toggle.tsx`와 별도로 페이지 내에서 인라인 정의되어 있으며, 센터 설정(`settings/page.tsx`)과 키오스크(`settings/kiosk/page.tsx`)에서 각각 동일한 구현이 중복되어 있다. 자동 알림(`auto-alarm/page.tsx`)에서도 유사한 Toggle 컴포넌트를 인라인으로 정의하여 사용한다.
