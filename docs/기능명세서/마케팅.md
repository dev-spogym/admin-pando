#### LeadStatus 열거형

#### CreateLeadInput 인터페이스

```typescript
export interface CreateLeadInput {
  branchId: number;
  name: string;
  phone?: string | null;
  source: LeadSource;
  status?: LeadStatus;
  assignedFc?: string | null;
  memo?: string | null;
  inquiryDate?: string;
  followUpDate?: string | null;

```

#### 통계 반환값 (`getLeadStats`)

```typescript
{ total, converted, conversionRate, pending, missedRate, byStatus, bySource }
```

- `converted`: `byStatus['등록완료'] ?? 0`
- `conversionRate`: `Math.round((converted / total) * 100)`
- `pending`: `byStatus['신규'] + byStatus['연락완료']`
- `missedRate`: `Math.round((byStatus['미전환'] / total) * 100)`

### L. 비즈니스 로직

1. **리드 → 회원 전환**: status가 `"등록완료"`로 변경되면 전환율 통계에 반영. `convertedMemberId` 필드로 매핑 가능 (현재 UI에서 직접 사용하지 않음)
2. **전환율 계산**: `Math.round((등록완료 건수 / 전체 건수) * 100)`
3. **미전환율 계산**: `Math.round((미전환 건수 / 전체 건수) * 100)`
4. **대기 건수**: 신규 + 연락완료 건수 합산
5. **칸반 뷰 5단계**: 신규 → 상담예약(상담예정) → 상담완료(방문완료) → 등록의향(연락완료) → 등록완료
6. **branchId 기준 격리**: 모든 조회/생성은 `getBranchId()` 기준

### M. 키보드/접근성

- 모달 내 Tab 순서: 이름 → 연락처 → 유입경로 → 상태 버튼 → 담당FC → 문의일 → 후속일 → 메모 → 취소 → 저장
- 칸반 카드는 `<button>` 태그로 구현하여 키보드 접근 가능

### 발송 확인 로직 (`confirmSend`) 상세

```typescript
const confirmSend = async () => {
  if (isSending) return;           // 중복 발송 방지
  setIsSending(true);
  setIsPreviewModalOpen(false);

  const typeMap = { kakao: 'KAKAO', sms: 'SMS', push: 'PUSH' };
  const recipientList = sendForm.useAllMembers
    ? []                            // 전체 발송 시 빈 배열
    : sendForm.recipients.map(r => ({ id: r.id, name: r.name, phone: r.phone }));

  const record = {
    branchId: getBranchId(),
    type: typeMap[sendForm.channel] ?? 'SMS',
    title: null,
    content: sendForm.content,
    recipients: recipientList,
    status: sendForm.isReserved ? 'SCHEDULED' : 'SENT',
    sentAt: sendForm.isReserved ? null : new Date().toISOString(),
    scheduledAt: sendForm.isReserved && sendForm.reserveDate
      ? new Date(sendForm.reserveDate).toISOString()
      : null,
  };

  const { error } = await supabase.from('messages').insert(record);
  if (error) {
    toast.error(`발송 이력 저장 실패: ${error.message}`);
    return;
  }

  toast.success(sendForm.isReserved
    ? "메시지 예약이 완료되었습니다."
    : "메시지가 발송되었습니다.");

  // 폼 초기화
  setSendForm(prev => ({
    ...prev,
    content: "",
    recipients: [],
    useAllMembers: false,
    isReserved: false,
    reserveDate: ""
  }));

  // 이력 갱신
  fetchMessageHistory(getBranchId()).then(setHistory);
};
```

```typescript
function generateCouponCode(): string {
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  const seg = (n: number) => Array.from({ length: n }, () => chars[Math.floor(Math.random() * chars.length)]).join('');
  return `COUP-${seg(4)}-${seg(4)}`;
}
```

#### 쿠폰 상태 자동 계산 (`computeCouponStatus`)

```typescript
function computeCouponStatus(coupon: any): string {
  if (!coupon.isActive) return 'inactive';     // 비활성
  if (coupon.validUntil) {
    const days = getDaysUntil(coupon.validUntil);
    if (days < 0) return 'expired';            // 만료
  }
  if (coupon.totalIssued > 0 && coupon.totalUsed >= coupon.totalIssued) return 'exhausted'; // 소진
  return 'active';                             // 활성
}
```

### L. 비즈니스 로직

1. **검증 단계별**:
   - Step 1: `selectedMember` 필수 → `"계약 대상 회원을 선택해주세요."`
   - Step 2: `selectedProducts.length > 0` 필수 → `"최소 한 개의 상품을 선택해주세요."`
   - Step 3: `contractDetails.startDate` 필수 → `"계약 시작일을 입력해주세요."`
2. **할인율 제한**: 최대 50%, 음수 불가
3. **중복 결제 방지**: `checkDuplicatePayment(memberId, finalPrice)` → INSERT 전 체크
4. **중복 저장 방지**: `isSaving` state로 연타 방지
5. **서명 필수**: 고객 서명 + 센터장 서명 모두 완료 필요
6. **서명 업로드**: dataUrl → Blob → File → `uploadFile('contracts', path, file)`
7. **이용권 자동 갱신**: `updateMembershipPeriod` 호출 → 회원 상태 ACTIVE로 변경
8. **마일리지 자동 적립**: 결제금액의 1% (`accruePoints`), 마일리지 단독 결제 시 제외
9. **결제 수단 매핑**: `card→'CARD'`, `cash→'CASH'`, `mileage→'MILEAGE'`, `transfer→'TRANSFER'`, 복수 시 `+` 연결
10. **취소 시 확인**: step > 1이거나 데이터 입력 시 `window.confirm` 확인

### 에러 핸들링 패턴

마케팅 도메인 전체에서 사용하는 에러 핸들링 패턴:

```typescript
// 1. try-catch + toast.error (리드, 마일리지)
try {
  await apiCall();
  toast.success("성공 메시지");
} catch (e: unknown) {
  toast.error(`실패: ${e instanceof Error ? e.message : "알 수 없는 오류"}`);
} finally {
  setSaving(false);
}

// 2. Supabase error 체크 (메시지, 쿠폰, 전자계약)
const { error } = await supabase.from('table').insert(record);
if (error) {
  toast.error(`저장 실패: ${error.message}`);
  return;
}
toast.success("성공 메시지");

// 3. console.warn + fallback (마일리지 이력)
console.warn('[MileageManagement] mileage_logs 테이블 조회 실패, sales fallback 시도:', logError?.message);
```
