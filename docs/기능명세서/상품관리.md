## 1. 상품 목록 (`/products`)

> 소스: `src/app/products/page.tsx`
> UC 참조: UC-022a, UC-022e

### 1.1 화면 개요

센터에서 판매하는 모든 상품(이용권, PT/GX 수강권, 대여권, 일반 상품)을 마스터-디테일 레이아웃으로 조회·관리하는 화면이다. 최상위 탭으로 **상품 목록**과 **분류 관리** 두 개의 서브화면을 전환할 수 있다.

### 1.10 비즈니스 로직

1. **데이터 로드**: `supabase.from('products').select('*').eq('branchId', getBranchId()).order('id')` + `supabase.from('sales').select('productName').eq('status', 'COMPLETED')` 병렬 로드
2. **가격 정규화**: `price`, `cashPrice`, `cardPrice`를 `Number()`로 변환
3. **판매 건수 집계**: sales 테이블에서 `productName` 기준 Map 생성, 목록에서 건수 표시
4. **동적 카테고리**: `product_groups`(활성)와 실제 상품 `category`의 합집합으로 분류 탭 자동 생성
5. **카테고리 매핑**: DB 영문값 → 한글 변환 (`MEMBERSHIP→이용권`, `PT→PT`, `GX→GX`, `PRODUCT/SERVICE/ETC→기타`)
6. **패키지 판별**: `description` 필드가 JSON이고 `isPackage === true`이면 패키지 상품으로 표시
7. **전 지점 배포**: 현재 지점 상품을 대상 지점에 복사. `id`, `branchId`, `createdAt` 제외 후 INSERT. 동일 이름 중복 시 skip
8. **권한 제어**: `hasFeature(role, 'productEdit', isSuperAdmin)` — 등록/수정/삭제 버튼 표시 여부 결정

## 1-B. 분류 관리 (상품 목록 > 분류 관리 탭)

> 소스: `src/app/products/page.tsx` (mainTab === 'groups')
> UC 참조: UC-022e
> API: `src/api/endpoints/productGroups.ts`

### 1-B.1 화면 개요

상품을 분류할 카테고리를 자유롭게 추가·수정·삭제하는 화면이다. 센터별 종목(헬스+필라테스, 골프+수영 등)에 맞는 분류 체계를 구성할 수 있다.

### 1-B.2 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  분류 등록 / 수정 폼                                         │
│                                                              │
│  분류명 * [          ]  정렬순서 [  ]  활성 [토글]            │
│                                   [등록]  (수정 시 [수정 저장] [취소]) │
├──────────────────────────────────────────────────────────────┤
│  분류명        정렬순서      상태         액션                 │
│  ──────────────────────────────────────────────────────────  │
│  이용권        1            활성         [✏️] [🗑]            │
│  PT           2            활성         [✏️] [🗑]            │
│  GX           3            활성         [✏️] [🗑]            │
│  골프         4            비활성       [✏️] [🗑]            │
└──────────────────────────────────────────────────────────────┘
```

### 1-B.3 필드 상세 — 분류 등록/수정 폼

| # | 필드명 | 타입 | 필수 | 기본값 | 설명 |

### 3.10 비즈니스 로직

1. **카테고리 매핑**:
   - DB→화면: `MEMBERSHIP→이용권`, `PT→PT`, `GX→GX`, `PRODUCT/SERVICE/ETC→기타`
   - 화면→DB: `이용권→MEMBERSHIP`, `PT→PT`, `GX→GX`, `기타→PRODUCT`
   - 화면→productType: `이용권→MEMBERSHIP`, `PT/GX→LESSON`, `기타→GENERAL`

2. **상품구분 → 카테고리 자동 결정**:
   - 레슨 → PT, 이용 → 이용권, 락커/판매 → 기타

3. **중복 검사**: 저장 시 `supabase.from('products').select('id').eq('branchId', branchId).eq('name', productName)` (수정 시 자기 자신 제외 `.neq('id', product.id)`)

4. **패키지 저장**: `isPackage=true`이고 `packageItems.length > 0`이면 `description` 필드에 JSON 직렬화. `packagePrice`, `originalDescription` 포함

5. **usage_restrictions 저장**: 체크된 요일의 인덱스 변환 (배열 인덱스 → 요일 번호: `0~5→1~6`, `6→0`), 첫 번째 체크된 요일의 시간 사용

6. **가격 변경 감지 및 audit_log**: 수정 시 이전 가격과 비교 → 변경된 경우 `createAuditLog({ action: 'UPDATE', targetType: 'product', targetId, beforeValue, afterValue, detail: { field: 'price', userName } })` 기록

7. **삭제 보호**: 삭제 전 `sales` 테이블에서 `productName`으로 COMPLETED 건수 조회 → 회원 있으면 비활성화 유도, 없으면 삭제 허용. 삭제 실패 시 fallback으로 `isActive: false` 업데이트

8. **상품정보 가져오기**: `applyProductToForm(source, { markAsCopy: true })` — 전체 필드 복사 + 패키지 정보 복원 + `copiedFromProductId` 설정으로 INSERT 모드 전환
