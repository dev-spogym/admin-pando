### 1.3 데이터 모델

**DB 테이블**: `sales`

**Supabase 쿼리**:
```sql
SELECT id, memberId, memberName, productId, productName, saleDate, type, round,
       quantity, originalPrice, salePrice, discountPrice, amount, paymentMethod,
       paymentType, cash, card, mileageUsed, cardCompany, cardNumber, approvalNo,
       status, unpaid, staffId, staffName, memo, branchId
FROM sales
WHERE branchId = {현재지점}
ORDER BY saleDate DESC
```

**프론트엔드 타입**: `SaleItem`

|------|------|

### 1.8 인터랙션 상세

#### 필터

| 필터 | 타입 | 동작 |
|------|------|------|
| 날짜 범위 | dateRange | 시작일/종료일 선택 → `filterValues.dateRangeStart/End` 반영 |
| 유형 | multiSelect | 이용권/PT/상품/기타 다중 선택 |
| 상태 | multiSelect | 완료/환불/미납 다중 선택 |
| 검색 | text | 구매자명 또는 상품명 포함 검색 (300ms debounce) |
| 기간 프리셋 | button group | 오늘/이번주/이번달 → 날짜 자동 설정 |
| 매출유형 | button group (TAB-001 전용) | 전체/신규/재등록/휴면복귀/종목추가/업그레이드 토글 |
| 리셋 | button | 모든 필터 초기화 (이번달 기본) |

#### 탭 구성

| 탭 키 | 라벨 | 데이터 | 컬럼 |
|--------|------|--------|------|
| TAB-001 | 매출 내역 | filteredData (개별 행) | 전체 22개 컬럼 |
| TAB-002 | 기간별 매출 | periodData (집계) | 기간, 건수, 매출합계 |
| TAB-003 | 상품별 내역 | productData (집계) | 상품명, 건수, 매출합계 |
| TAB-004 | 결제수단별 | paymentData (집계) | 결제수단, 건수, 매출합계 |
| TAB-005 | 담당자별 | staffData (집계) | 담당자, 건수, 매출합계 |
| TAB-006 | 환불 내역 | status === '환불' 필터 | 전체 22개 컬럼 |
| TAB-007 | 미납 내역 | unpaid > 0 필터 | 전체 22개 컬럼 |

TAB-006/007에는 건수 카운트 배지가 탭 옆에 표시된다.

#### 정렬

DataTable 컴포넌트 내장 정렬 (구매일, 판매가, 상품명 기준).

#### 기간별 집계 단위 (TAB-002)

| 단위 | key 생성 규칙 |
|------|--------------|
| 일 | YYYY-MM-DD |
| 주 | 해당 주 월요일 날짜 + " 주" |
| 월 | YYYY-MM |

| 상황 | 타입 | 메시지 |

### 1.10 비즈니스 로직

**요약 합계 계산**:
- `total`: status !== '환불'인 행의 salePrice 합계
- `card`: 전체 행의 card 합계
- `cash`: 전체 행의 cash 합계
- `mileage`: 전체 행의 mileage 합계
- `unpaid`: 전체 행의 unpaid 합계
- `discount`: 전체 행의 discountPrice 합계
- `refund`: status === '환불'인 행의 salePrice 합계
- `netTotal`: total - refund (순 매출)

**매출유형(round) 배지 색상**:

| 유형 | 배경 | 텍스트 |
|------|------|--------|
| 신규 | bg-blue-100 | text-blue-700 |
| 재등록 | bg-green-100 | text-green-700 |
| 휴면복귀 | bg-yellow-100 | text-yellow-700 |
| 종목추가 | bg-purple-100 | text-purple-700 |
| 업그레이드 | bg-orange-100 | text-orange-700 |

### 2.3 데이터 모델

**DB 테이블**: `products`

**Supabase 쿼리**:
```sql
SELECT *
FROM products
WHERE branchId = {현재지점}
  AND isActive = true
```

**프론트엔드 타입**: `Product`

| 필드 | DB 컬럼 | 타입 | 설명 |
|------|---------|------|------|
| id | id | number | PK |
| name | name | string | 상품명 |
| category | category | string | 카테고리 (MEMBERSHIP/PT/GX/ETC → 한글 변환) |
| cashPrice | cashPrice (fallback: price) | number | 현금가 |
| cardPrice | cardPrice (fallback: price) | number | 카드가 |
| period | duration | string | 기간 |
| count | sessions | string | 횟수 |
| productType | productType | string | null | 상품 타입 (MEMBERSHIP/LESSON/RENTAL/GENERAL) |
| kiosk | kioskVisible | boolean | 키오스크 노출 여부 |
| stock | - | number | null | 재고 (현재 null 고정) |

**카테고리 매핑 (영문 → 한글)**:

| DB 값 | 한글 |
|--------|------|
| MEMBERSHIP | 이용권 |
| PT | PT |
| GX | GX |
| ETC | 기타 |

**상품 타입 배지**:

| 타입 | 라벨 | 색상 |
|------|------|------|

### 2.4 장바구니 타입: `CartItem`

| 필드 | 타입 | 설명 |

### 2.5 버튼/액션 상세

**일할 인식 계산**:
```
elapsedDays = min(serviceDays, max(0, ceil((today - startDate) / 1일)))
progressPct = round((elapsedDays / serviceDays) * 100)  // 0~100
recognizedAmount = round(totalAmount * (progressPct / 100))
remainingAmount = totalAmount - recognizedAmount
```

**별도 API (`calculateDeferredRevenue`)**: 개별 매출 건의 선수익금 계산 (deferred_revenue 테이블 기반). 현재 화면에서는 미사용.
