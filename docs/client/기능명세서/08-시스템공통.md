## 0. 도입 배경

회원앱이 마켓플레이스 / 결제 / 커뮤니티로 도메인을 확장하면서 다음과 같은 공통 인프라가 필요하다.

- 신규 회원에게 핵심 가치(센터 둘러보기 / 결제 / 활동 / 리워드)를 4단계로 안내하는 환영 온보딩
- 네트워크 단절 / 서버 점검 등 예외 상태를 회원에게 명확히 보여주는 시스템 화면
- 비로그인 사용자가 보호 액션(결제 / 스크랩 / 리뷰 작성)을 시도할 때의 통일된 게이트
- 상품 / 강사 / 센터 / 추천 코드 등을 외부로 공유하는 통일된 시트
- 취소 / 삭제 / 이탈 등 결정성 액션 직전의 통일된 확인 모달
- 새 메시지 / 알림 / 주문 상태 변경 등을 비차단(non-blocking)으로 알리는 푸시 토스트

본 문서는 위 항목을 `MA-9XX / MFN-900` 시리즈 시스템 화면과 공통 컴포넌트(MFN-9XX-Cxx)로 신규 정의한다.

## 1. 기본 정책

### 1.1 온보딩 환영 정책

- 노출 시점: CRM 회원 앱 연동 직후 첫 진입(`MA-002` 완료 직후) 또는 `둘러보기 홈(MA-300)` 첫 진입.
- 4단계 슬라이드 구조: `① 둘러보기 / ② 결제·예약 / ③ 활동·리워드 / ④ 메신저·커뮤니티`.
- 각 단계는 일러스트 + 1줄 헤드라인 + 2줄 설명으로 구성한다.
- 중간 종료(`스킵`) 가능, 마지막 단계에서 `시작하기` CTA로 둘러보기 홈 진입.
- 한 회원에게는 1회만 자동 노출하며, 이후엔 설정에서 다시 보기 가능.
- 기존 운동 온보딩(`MA-127 ~ MA-128`)과는 별도 흐름이며 운동 온보딩이 우선 진행된다.

### 1.2 네트워크 오류 정책

- 진입 시점: 네트워크 fetch 실패 또는 단말 오프라인 상태.
- 화면 구성: 일러스트 + `네트워크 연결을 확인해 주세요` + `다시 시도` 버튼.
- `다시 시도` 클릭 시 직전 화면 라우터 재시도.
- 마지막 본 센터 / 강사 카드 1세트는 캐시에서 노출 (마켓 비기능 요구와 동일).

### 1.3 서버 점검 정책

- 진입 시점: 서버에서 `503 + maintenance` 응답 또는 운영팀 강제 점검 모드.
- 화면 구성: 일러스트 + `점검 안내 헤드라인` + 점검 시간 / 사유 / 예상 종료 + `공지 보러 가기`.
- 점검 중에는 모든 보호 라우트(결제 / 예약 / 메신저)가 차단되며, 공지(`MA-152`)와 고객센터만 진입 가능.

### 1.4 공통 컴포넌트 정책

| 컴포넌트 | 사용 시점 |
|---|---|
| LoginGateModal | 비로그인 회원이 결제 / 스크랩 / 리뷰 / 메신저 시도 시 |
| ShareSheet | 상품 / 센터 / 강사 / 추천 코드 / 이벤트 외부 공유 시 |
| ConfirmDialog | 취소 / 삭제 / 이탈 / 환불 신청 등 결정성 액션 직전 |
| PushToast + usePushSimulator | 새 메시지 / 주문 상태 변경 / 시스템 알림 비차단 노출 |

## 3. 화면 / 컴포넌트 상세

### MFN-900. 신규 회원 온보딩 환영

| 항목 | 내용 |
|---|---|
| 목적 | 신규 회원에게 마켓 / 결제 / 활동 / 메신저 4가지 핵심 가치를 한 번에 소개한다 |
| 슬라이드 1 | 둘러보기 — `내 주변 센터 · 강사 · 상품 한 곳에서 발견` |
| 슬라이드 2 | 결제·예약 — `옵션 선택 → 결제 → 예약·이용 자동 추적` |
| 슬라이드 3 | 활동·리워드 — `누적 결제로 등급 상승, 친구 초대로 리워드` |
| 슬라이드 4 | 메신저·커뮤니티 — `센터 / 강사와 직접 소통, 공개 Q&A 활용` |
| 액션 | 좌우 스와이프, 페이지 인디케이터, 하단 `다음 / 시작하기 / 건너뛰기` |
| 노출 정책 | 회원 단위 1회 자동 노출 (`hasSeenWelcome` 플래그) |

### MFN-901. 네트워크 오류 안내

| 항목 | 내용 |
|---|---|
| 목적 | 통신 단절 시 회원이 상황을 이해하고 즉시 재시도할 수 있게 한다 |
| 출력 | 일러스트, `네트워크 연결을 확인해 주세요`, 보조 안내, `다시 시도` 버튼 |
| 액션 | 다시 시도 / 설정 진입 / 캐시된 마지막 카드 보기 |
| 정책 | 라우터 단에서 fetch 실패 시 진입, 복구 즉시 자동 이탈 |

### MFN-902. 서버 점검 안내

| 항목 | 내용 |
|---|---|
| 목적 | 점검 시간을 명확히 안내해 회원의 불필요한 재접속을 줄인다 |
| 출력 | 일러스트, 점검 헤드라인, 점검 시간 / 사유 / 예상 종료, `공지 보러 가기` 버튼 |
| 정책 | 점검 모드에서는 보호 라우트 차단, 공지·고객센터만 진입 가능 |

### MFN-9C1. 비로그인 게이트 모달 (LoginGateModal)

| 항목 | 내용 |
|---|---|
| 목적 | 비로그인 회원이 보호 액션을 시도할 때 즉시 로그인 / 가입 진입을 유도한다 |
| 호출 위치 | 결제 / 스크랩 / 리뷰 / 메신저 / 추천 코드 입력 등 |
| 구성 | 일러스트, 헤드라인(예: `로그인이 필요한 기능이에요`), 보조 설명, `로그인 / 회원가입 / 닫기` 버튼 |
| 동작 | 로그인 진입 시 `MA-001` 라우팅 + 직전 화면 returnUrl 보존 |

### MFN-9C2. 공유 시트 (ShareSheet)

| 항목 | 내용 |
|---|---|
| 목적 | 상품 / 센터 / 강사 / 이벤트 / 추천 코드 등을 외부 채널로 공유한다 |
| 구성 | 카카오톡 / 문자 / 링크 복사 / 시스템 공유 (Web Share API) |
| 입력 | title, description, url, ogImage |
| 정책 | 추천 코드 공유 시 추천 가입 화면 딥링크 자동 포함 |

### MFN-9C3. 확인 / 취소 모달 (ConfirmDialog)

| 항목 | 내용 |
|---|---|
| 목적 | 결정성 액션(취소 / 삭제 / 이탈 / 환불 신청) 직전 회원에게 한 번 더 확인한다 |
| 구성 | 헤드라인, 본문, `확인 / 취소` 2버튼 (위험 톤 / 기본 톤 분리) |
| 입력 | title, description, confirmLabel, cancelLabel, tone(`primary / danger`), onConfirm / onCancel |

### MFN-9C4. 푸시 토스트 + usePushSimulator

| 항목 | 내용 |
|---|---|
| 목적 | 새 메시지 / 주문 상태 변경 / 시스템 알림을 비차단으로 노출한다 |
| 토스트 구성 | 좌측 아이콘, 제목, 본문 1줄, 우측 닫기, 자동 사라짐(5초) |
| 시뮬레이터 | `usePushSimulator()` hook으로 데모 환경에서 토스트 트리거 |
| 정책 | 토스트 클릭 시 알림 대상 화면(`MA-150` 또는 직접 화면)으로 라우팅 |

## 4. 비기능 요구

| 항목 | 기준 |
|---|---|
| 응답 시간 | 모달 / 토스트 진입·사라짐 200ms 이내 |
| 접근성 | 모든 모달은 ESC / 외부 클릭으로 닫히고, 포커스 트랩을 가진다 |
| 다국어 | 1차 한국어, 2차 영어 가능(후속) |
| 일관성 | 모든 시스템 화면은 회원앱 디자인 시스템 컴포넌트(`Card`, `Button`, `EmptyState`)를 재사용 |
