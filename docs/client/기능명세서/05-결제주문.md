## 0. 도입 배경

기존 회원 결제 흐름(`MA-140 결제하기 / MA-141 개인 결제 / MA-142 영수증`)은 단일 센터 가입자 기준의 단건 결제만을 다룬다.

마켓플레이스 v1.0 도입 이후 회원은 다음과 같은 다단계 결제 흐름을 가진다.

- 옵션 결제 — 강사 / 요일·시간 / 시작일 / 회차를 선택한 뒤 단건 결제
- 장바구니 결제 — 여러 센터의 다양한 상품을 한 번에 결제
- 주문 / 예약 통합 추적 — 결제 직후부터 이용 완료까지 4단계 timeline
- 환불 — 잔여 회차 또는 결제 후 경과일 기반 부분 환불
- 결제수단 관리 — 등록 카드 / 페이 (카카오페이 / 네이버페이) 보관 및 기본 결제 지정

본 문서는 기존 결제 도메인(`MFN-133 ~ MFN-135`)을 변경하지 않고, 마켓 결제 후속 흐름을 `MA-4XX / MFN-400` 시리즈로 신규 정의한다.

## 1. 기본 정책

### 1.1 옵션 결제 정책

- 마켓 상품 중 강사 1:1 / 그룹 / 횟수형 상품은 결제 전 옵션 화면(`MA-400`)을 거친다.
- 옵션 항목: `강사`, `요일 + 시간`, `시작일(오늘 / 이번 주 / 다음 주)`, `회차(5 / 10 / 20)`.
- 회차 선택은 단가 배수(예: 10회 ×1.9, 20회 ×3.5)로 가격이 변동된다.
- 옵션 화면에서 `장바구니 담기` 또는 `바로 결제`를 선택할 수 있다.
- 옵션 요약은 한 줄 형태로 저장한다 (예: `김도윤 강사 · 화/목 19:00 · 10회`).

### 1.2 장바구니 정책

- 장바구니는 회원 단위 단일 컬렉션이며, 동일 상품 + 동일 옵션 요약은 수량으로 합쳐 저장한다.
- 항목별 `선택 / 해제` 토글로 부분 결제가 가능하다.
- 쿠폰은 장바구니 합계 기준으로 1장만 적용된다 (`MFN-413`).
- 마일리지는 장바구니 합계 기준으로 차감 입력한다.
- 장바구니에서 결제 시 비례 분배 규칙으로 항목별 최종 결제액을 계산한다.

### 1.3 주문 / 예약 통합 추적 정책

- 결제 완료 시 `결제 + 예약 + 이용` 트래킹이 묶인 주문(`OrderItem`)이 생성된다.
- 주문은 4단계 timeline을 가진다: `결제 완료 → 예약 확정 → 이용 가능 → 이용 완료`.
- 취소 / 환불 시 별도의 종료 단계(`주문 취소` / `환불 완료`)로 timeline을 마무리한다.
- 주문 상태 코드: `pending / confirmed / in_use / completed / cancelled / refunded`.
- 회원 화면 탭 필터는 `전체 / 진행중 / 완료 / 취소` 4탭으로 운용한다.

### 1.4 환불 / 취소 정책

- 환불 가능 상태: `pending / confirmed / in_use`.
- 회차형 상품: `잔여 회차 / 총 회차` 비율로 환불액을 계산한다.
- 기간형 상품: 결제 후 7일 이내 100%, 30일 이내 70%, 30일 초과 30% (위약금 차감).
- 환불 사유: `단순 변심 / 서비스 불만족 / 일정 변경 불가 / 기타`.
- 취소 가능 상태: `pending / confirmed`. 이용중·완료·환불 주문은 취소 불가.

### 1.5 결제수단 정책

- 등록 가능 결제수단: `카드`, `카카오페이`, `네이버페이`.
- 카드는 `카드사 + 끝 4자리 + 만료월/년`으로 식별한다.
- 페이(카카오페이 / 네이버페이)는 ON/OFF 활성 상태를 가진다.
- 기본 결제수단은 회원당 1개만 지정 가능하며, 삭제 시 자동 승계된다.
- 결제 화면(`MA-140`)은 등록된 결제수단을 우선 노출하고, 미등록 시 `간편 등록` 진입을 유도한다.

## 3. 데이터 도메인 정의

### 3.1 신규 엔티티

| 엔티티 | 설명 | 핵심 필드 |
|---|---|---|
| CheckoutOptionDraft | 옵션 화면에서 작성 중인 결제 초안 | productId, trainerId, weekday, time, startOption, planId |
| CartItem | 장바구니 단건 항목 | id, productId, centerId, productName, optionSummary, unitPrice, quantity, selected |
| CartCoupon | 장바구니용 쿠폰 | id, name, discount, minAmount |
| OrderItem | 결제 + 예약 + 이용 통합 주문 | id, productName, optionSummary, trainerName, totalSessions, remainingSessions, status, timeline, paidAt |
| OrderTimelineStep | 주문 단계 timeline | status, label, description, timestamp |
| RefundEstimate | 환불 예상액 | ratio, refundAmount, policy |
| SavedPaymentMethod | 등록된 결제수단 | id, kind(card / kakaopay / naverpay), company, last4, expiry, isDefault, enabled |

### 3.2 기존 엔티티와의 연결

- `MA-140 결제하기 / MA-142 영수증`은 본 도메인 결제의 종착 화면으로 그대로 재사용한다.
- `OrderItem.productId`는 마켓 상품(`MarketProduct`) 또는 기존 상품 스토어(`MA-134`)와 동일 키를 사용한다.
- `OrderItem.trainerName`은 마켓 강사(`MarketTrainer`)와 동일 식별자를 가진다.
- 회차 차감 정책은 기존 `MFN-145 PT 잔여횟수 / 차감 이력 조회`와 동일하게 `수업 완료` 시점에 처리한다.

## 6. 주요 기능 상세

### MFN-400. 결제 옵션 선택

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 강사 / 요일·시간 / 시작일 / 회차를 선택해 최종 결제 금액을 확정한다 |
| 입력 | 상품 ID, 강사 ID(선택), 요일, 시간, 시작일 옵션, 회차 플랜 |
| 처리 | 옵션 변동마다 가격 재계산 → 옵션 요약 문자열 생성 |
| 출력 | 최종 결제액, 옵션 요약, 다음 단계 CTA(장바구니 / 바로결제) |
| 비고 | 강사 1:1 / 횟수형 상품은 옵션 화면 필수, 단건 이용권은 옵션 화면 생략 가능 |

### MFN-410. 장바구니 조회

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 담은 다양한 센터 / 상품을 한 화면에서 본다 |
| 출력 | 항목 카드(이미지 / 상품명 / 옵션 요약 / 단가 / 수량 / 합계), 합계 / 쿠폰 / 마일리지 / 최종액 |
| 액션 | 수량 변경, 선택 토글, 삭제, 일괄 결제 |
| 정책 | 동일 상품 + 동일 옵션 요약은 자동 합산 |

### MFN-420. 주문 / 예약 통합 내역

| 항목 | 내용 |
|---|---|
| 목적 | 결제부터 이용 완료까지 모든 단계를 한 화면에서 추적한다 |
| 입력 | 회원 ID, 탭 필터(전체 / 진행중 / 완료 / 취소) |
| 출력 | 주문 카드(상품명 / 옵션 요약 / 강사 / 잔여 회차 / 결제일 / 상태 칩) |
| 비고 | `진행중`은 `pending / confirmed / in_use`, `완료`는 `completed`, `취소`는 `cancelled / refunded` |

### MFN-421. 주문 상세 조회

| 항목 | 내용 |
|---|---|
| 목적 | 단일 주문의 결제 → 예약 → 이용 4단계 timeline과 옵션·결제수단을 본다 |
| 출력 | 상품 정보, 옵션 요약, 강사명, 회차(총/잔여), 시작일, 결제 정보, timeline 4단계 |
| 액션 | 취소 / 환불 / 후기 작성 / 영수증 보기 |
| 정책 | 상태별로 활성 액션이 다름 (`pending`/`confirmed` → 취소·환불, `completed` → 후기, 모든 단계 → 영수증) |

### MFN-430. 환불 신청

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 잔여 가치를 기준으로 부분 환불을 신청한다 |
| 입력 | 주문 ID, 환불 사유(4종), 추가 의견 |
| 처리 | 회차형 → 잔여 비율 환불 / 기간형 → 결제일 경과 구간 환불 / 사유 + 의견 저장 |
| 출력 | 예상 환불액, 적용 정책 안내, 신청 완료 토스트 |
| 비고 | 환불 신청은 운영 승인 전 `요청 접수` 단계로 정의 |

### MFN-440. 결제수단 목록 조회 / 관리

| 항목 | 내용 |
|---|---|
| 목적 | 등록된 결제수단을 한 화면에서 관리한다 |
| 출력 | 카드(카드사 / 끝 4자리 / 만료) / 페이(카카오페이 / 네이버페이 / ON·OFF) |
| 액션 | 등록 / 삭제 / 기본 지정 / 페이 활성 토글 |
| 정책 | 회원당 기본 결제수단 1개 보장(삭제 시 자동 승계) |
