## 0. 도입 배경

기존 회원앱 IA(v3.0)는 `회원이 이미 가입한 한 센터` 기준으로 정의되어 있다.
그러나 운영 시나리오상 회원이 자기 센터 외에도 `다른 센터의 체험권 / 이용권을 둘러보고 결제`하거나, `위치 기반으로 새로운 센터·강사를 탐색`하거나, `여러 센터의 트레이너와 직접 대화`해야 하는 흐름이 존재한다.

본 문서는 기존 회원 도메인을 변경하지 않은 상태에서, 다음 도메인을 별도 기능군으로 추가 정의한다.

- 발견(Discover) — 위치 / 카테고리 / 추천 기반 센터 · 강사 탐색
- 마켓플레이스 — 멀티센터 상품(이용권, 체험권, 그룹/개인 레슨) 탐색 / 결제 진입
- 스크랩 — 관심 센터 · 강사 · 상품 저장
- 메신저 — 센터 문의 / 강사톡 / 강의노트 통합 채널
- 센터 리뷰 — 센터 평점 / 후기 작성 · 조회

기존 단일 센터 회원 흐름(MA-1XX)은 `내 센터` 컨텍스트, 신규 마켓 흐름(MA-3XX)은 `둘러보기` 컨텍스트로 분리한다.

## 1. 기본 정책

### 1.1 컨텍스트 정책

- 회원은 `내 센터` 컨텍스트와 `둘러보기` 컨텍스트를 동시에 가진다.
- `내 센터` 컨텍스트는 가입한 센터의 이용권 · 예약 · 출석 · QR 등을 다룬다 (`MA-1XX`).
- `둘러보기` 컨텍스트는 가입 여부와 무관하게 모든 공개 센터 / 강사를 탐색한다 (`MA-3XX`).
- 둘러보기에서 결제하면 새 센터에 추가 가입되거나 체험권으로 등록된다.
- 둘러보기 컨텍스트는 비로그인 사용자도 일부 화면을 열람할 수 있다 (검색, 센터 상세, 강사 상세).

### 1.2 위치 정책

- 위치 권한은 앱 첫 진입 시 또는 `MA-310 내 주변 센터` 진입 시 요청한다.
- 위치 권한이 거부되면 `행정구역(시/구/동)` 수동 선택으로 대체한다.
- 거리 계산은 단말 기준 좌표와 센터 좌표 기준의 직선 거리(km)를 사용한다.

### 1.3 카테고리 정책

- 1차 카테고리: `피트니스`, `필라테스`, `요가`, `골프`, `크로스핏`, `스피닝`, `복싱`, `수영`.
- 2차 상품 분류: `수강권`, `이용권`, `체험권`, `그룹 레슨`, `개인 레슨`, `온라인`.
- 1차 + 2차 조합으로 다중 필터를 적용한다.
- 정렬 옵션: `최신순`, `거리순`, `리뷰 좋은 순`, `가격 낮은 순`, `체험권`.

### 1.4 스크랩 정책

- 스크랩 대상: `센터`, `강사`, `상품`, `수업`.
- 스크랩은 회원 단위 단일 컬렉션이며 폴더 분류는 후속 단계에 둔다.
- 스크랩 추가/제거는 비동기 토글로 처리하고 결과 토스트를 표시한다.

### 1.5 메신저 정책

- 메신저는 세 탭으로 구성한다: `센터 문의`, `강사톡`, `강의노트`.
- 센터 문의는 회원 ↔ 센터(센터 통합 응답 계정) 1:1 대화방이다.
- 강사톡은 회원 ↔ 트레이너 / 골프강사 1:1 대화방이다 (트레이너앱과 양방향 연결).
- 강의노트는 트레이너가 회원에게 보낸 수업 노트와 회원의 답신을 묶어 보여주는 읽기 위주 탭이다.
- 빈 상태(가입 센터가 0이거나 강사 매칭이 없을 때) 메시지를 명확히 노출한다 (예: `센터 없습니다`).
- 알림센터(`MA-150`)는 시스템 알림 전용으로 유지하고, 사람과의 대화는 메신저로 분리한다.

### 1.6 리뷰 정책

- 회원은 출석 또는 결제 이력이 있는 센터/강사에 대해 별점(1~5)과 텍스트 후기를 1회 작성할 수 있다.
- 리뷰는 작성 후 30일 이내에만 1회 수정 가능하다.
- 리뷰 신뢰도를 위해 `방문 인증 회원` 배지를 자동 부여한다.
- 별점 평균과 리뷰 수는 센터 상세, 검색 카드, 추천 슬라이더에 모두 반영된다.

### 1.7 비로그인 정책

- `둘러보기 홈`, `내 주변 센터 목록`, `센터 검색`, `센터 상세`, `강사 상세`는 비로그인 열람을 허용한다.
- 결제, 스크랩, 메신저, 리뷰 작성은 로그인 / 앱 연동을 요구한다 (`MA-001` 진입).

## 3. 데이터 도메인 추가 정의

### 3.1 신규 엔티티

| 엔티티 | 설명 | 핵심 필드 |
|---|---|---|
| Center (확장) | 단일 센터 → 멀티 센터 카탈로그 | id, 이름, 카테고리, 위치(lat/lng), 행정구역, 시설 태그, 영업시간, 대표 이미지, 평점, 리뷰 수 |
| CenterProduct | 센터별 판매 상품 | id, centerId, 분류(수강권/이용권/체험권), 카테고리, 정가, 판매가, 할인율, 회차/기간 |
| Trainer (확장) | 마켓 표시용 강사 프로필 | id, 이름, 소속 센터, 카테고리, 자격, 평점, 리뷰 수, 가능 시간, 프로필 이미지 |
| Scrap | 회원 즐겨찾기 | userId, targetType(center/trainer/product/class), targetId, createdAt |
| Conversation | 메신저 대화방 | id, 종류(center/trainer/note), 참가자, 마지막 메시지, 미읽음 수 |
| Message | 메신저 메시지 | id, conversationId, senderId, type(text/image/file), content, readAt |
| CenterReview | 센터 리뷰 | id, centerId, userId, 별점, 시설 항목별 점수, 본문, createdAt |

### 3.2 기존 엔티티와의 연결

- `Center` 식별자는 기존 `회원.branchId`와 연결된 센터를 동일 도메인으로 본다.
- `Trainer`는 기존 트레이너 계정과 동일 식별자를 사용한다.
- `CenterProduct`는 기존 `MA-134 상품 스토어`의 상품과 동일 도메인이며, `centerId` 필드만 추가한다.
- `Scrap` 대상이 `상품 / 수업`인 경우 기존 스토어/예약 도메인을 그대로 가리킨다.

## 4. 비기능 요구

| 항목 | 기준 |
|---|---|
| 응답 시간 | 검색 결과 1초 이내 첫 페이지 표시 |
| 페이지네이션 | 무한 스크롤 또는 20개 단위 페이지 |
| 캐싱 | 위치/카테고리 기반 결과는 5분 캐시 |
| 오프라인 | 마지막으로 본 센터/강사 카드 1세트 캐시 (기본값) |
| 접근성 | 별점, 거리, 가격은 텍스트로도 제공 (스크린리더) |
| 다국어 | 1차 한국어, 2차 영어 가능 (후속) |
