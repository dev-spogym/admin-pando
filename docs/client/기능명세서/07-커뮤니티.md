## 0. 도입 배경

마켓플레이스 환경에서는 다른 회원의 질문 / 답변이 의사결정에 큰 영향을 미친다.
또한 운영 측면에서 광고 / 비방 / 도배 / 허위정보 등 부적절한 콘텐츠를 회원이 직접 신고할 수 있어야 한다.

본 문서는 기존 회원 도메인을 변경하지 않고, 다음 세 화면을 `MA-6XX / MFN-600` 시리즈로 신규 정의한다.

- 공개 Q&A — 회원이 센터 / 강사 / 다른 회원에게 공개로 질문하고 답변을 받는 채널
- FAQ — 자주 묻는 질문 카테고리별 검색 / 인기 질문
- 신고 내역 — 회원 본인이 한 신고와 처리 결과 확인

## 1. 기본 정책

### 1.1 공개 Q&A 정책

- Q&A는 마켓 단위 전체 공개이며, 작성자는 익명 / 실명(마스킹) 중 선택한다.
- 카테고리 5종: `시설 / 가격 / 강사 / 예약 / 기타`.
- 답변자 종류: `센터 공식(official) / 강사(trainer) / 일반 회원(member)`.
- 답변자 종류별 표기 색을 분리한다 (공식 = 청록 강조, 강사 = 보조, 회원 = 기본).
- 답변에는 `도움됨 카운트`를 부여한다.
- 회원 본인 작성글에 한해 24시간 이내 수정 / 삭제 가능.

### 1.2 FAQ 정책

- FAQ는 운영팀 작성 전용이며 회원은 조회만 가능하다.
- 카테고리 6종: `이용 / 결제 / 환불 / 예약 / 회원권 / 기타`.
- `popular` 플래그가 있는 항목은 인기 질문 섹션 상단에 노출.
- 키워드 검색은 질문 + 답변 본문 모두에서 부분 일치로 처리.
- 회원은 `도움됨` 클릭으로 helpfulCount를 증가시킬 수 있다.

### 1.3 신고 정책

- 신고 대상 종류: `회원 / 리뷰 / Q&A / 메시지`.
- 신고 사유 6종: `광고/홍보성 / 욕설/비방 / 음란/선정성 / 도배/스팸 / 허위정보 / 기타`.
- 신고 상태: `접수 → 검토중 → 처리완료 / 반려`.
- 처리 완료 / 반려 시 운영팀 답변(staffReply)을 회원에게 표시한다.
- 회원은 본인 신고 내역만 조회할 수 있다.
- 한 콘텐츠에 대한 중복 신고는 금지하며, 이미 신고한 항목에는 안내를 표시한다.

|---|---|---|---|

| MFN-601 | Q&A 검색 | MA-600 | 제목 + 본문 키워드 검색 |

| MFN-620 | 신고 내역 목록 | MA-603 | 회원 본인 신고 + 처리 상태 |

| MFN-622 | 신고 접수 (외부 진입) | (각 콘텐츠) → MA-603 | 회원/리뷰/Q&A/메시지에서 신고 모달 진입 |

## 3. 데이터 도메인 정의

### 3.1 신규 엔티티

| 엔티티 | 설명 | 핵심 필드 |
|---|---|---|
| QnaItem | 공개 Q&A 질문 | id, category, title, body, authorName, anonymous, createdAt, likeCount, viewCount, answers[] |
| QnaAnswer | Q&A 답변 | id, authorName, role(official/trainer/member), refId, body, createdAt, helpfulCount |
| FaqItem | 자주 묻는 질문 | id, category, question, answer, helpfulCount, popular |
| ReportItem | 신고 내역 | id, targetType, targetLabel, reason, detail, createdAt, status, staffReply, resolvedAt |

### 3.2 기존 엔티티와의 연결

- Q&A 답변자 `role=official`은 센터(`Center`) 식별자, `role=trainer`는 강사(`Trainer`) 식별자와 연결된다.
- FAQ는 기존 `MA-152 공지사항`과는 별개의 운영 콘텐츠로 본다.
- 신고는 기존 1:1 문의(`MA-153`)와 별도 채널로, 콘텐츠 단위 처리를 위한 운영 도구.

## 4. 비기능 요구

| 항목 | 기준 |
|---|---|
| 응답 시간 | 검색 결과 500ms 이내 |
| 모더레이션 | 욕설 / 광고 자동 필터(후속) |
| 마스킹 | 작성자 이름 가운데 글자 마스킹 (예: 김*수) |
| 접근성 | 답변 역할(공식/강사/회원)을 색뿐 아니라 텍스트 라벨로도 제공 |
| 빈 상태 | 카테고리 / 검색 0건 명확한 안내 |

## 6. 주요 기능 상세

### MFN-600. Q&A 목록 조회

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 마켓 전반의 공개 Q&A를 카테고리 / 답변 상태로 좁혀 본다 |
| 입력 | 카테고리 필터(전체 + 5종), 상태 탭(전체/답변완료/답변대기), 검색어 |
| 출력 | 카드(카테고리 칩, 제목, 본문 미리보기, 작성자(마스킹), 좋아요/조회/답변 수, 작성일) |
| 액션 | 카드 클릭 → MA-601, 상단 작성 버튼 → 작성 모달 |

### MFN-602. Q&A 상세 조회

| 항목 | 내용 |
|---|---|
| 목적 | 단일 질문 본문과 모든 답변을 본다 |
| 출력 | 질문(제목 + 본문 + 메타), 답변 리스트(역할별 색 / 본문 / 도움됨), 답변 작성 입력 |
| 정책 | 답변 역할: `공식(센터 강조 색) / 강사(보조 강조) / 회원(기본)` |

### MFN-610. FAQ 카테고리 / 인기 질문

| 항목 | 내용 |
|---|---|
| 목적 | 결제 / 환불 / 예약 등 자주 발생하는 질문을 빠르게 찾는다 |
| 출력 | 카테고리 칩 6종, 인기 질문 5건, 카테고리별 펼침 리스트 |
| 액션 | 항목 클릭 → 답변 펼침, 도움됨 카운트, 1:1 문의 진입 |

### MFN-620. 신고 내역 목록

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 자신이 한 신고의 처리 진행을 확인한다 |
| 출력 | 카드(대상 종류 / 대상 발췌 / 사유 / 신고일 / 상태 칩) |
| 정책 | 상태별 색: 접수(info) / 검토중(warning) / 처리완료(success) / 반려(neutral) |

### MFN-621. 신고 상세 / 운영팀 답변

| 항목 | 내용 |
|---|---|
| 목적 | 회원 신고 건의 운영팀 처리 결과와 답변을 본다 |
| 출력 | 신고 내용, 추가 메모, 처리 상태, 처리일, 운영팀 답변 |
| 정책 | 처리완료 / 반려 시 답변 본문 필수 |
