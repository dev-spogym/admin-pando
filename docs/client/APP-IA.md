## 1. 핵심 원칙

- 회원앱은 `회원 중심 통합 앱`이다.
- 회원가입은 `CRM 등록 + 모바일 앱 휴대폰 인증 연동` 방식으로 해석한다.
- 체크리스트의 `포인트`는 `마일리지`로 통일한다.
- `배지`, `알림센터`, `온보딩`, `강사 상세`, `대기 예약`, `수업 후기`, `재등록 추천`을 회원앱 범위에 포함한다.
- PT 플랜 외에도 `헬스장 이용권`, `골프장 이용권`, `골프 레슨권`, `개인 결제`를 동일 결제 도메인 안에 포함한다.
- 회원은 트레이너의 가능 시간과 자신의 예약 일정을 확인한 뒤 PT/골프 레슨을 요청할 수 있어야 한다.
- 회원이 생성한 트레이너 대상 예약은 `트레이너 승인 대기`를 기본으로 하고, 트레이너가 직접 회원을 지정해 만든 일정은 즉시 확정으로 처리한다.
- PT/골프 레슨 횟수 차감은 예약 승인이나 일정 확정 시점이 아니라 `수업 완료` 시점에 처리한다.
- 화면설계서/기능명세서/IA는 동일한 화면 ID와 기능 ID를 사용한다.

## 2. 역할 구조

| Role | 설명 |
|---|---|
| member | 일반 회원 |
| trainer | 트레이너 |
| golf_trainer | 골프강사 |
| fc | FC |
| staff | 스태프 |

회원앱 체크리스트 반영 작업의 중심은 `member` 영역이다.  
트레이너/골프강사/FC/스태프는 회원 흐름과 연결되는 화면을 함께 사용한다.

- 회원앱 role 키는 `member / trainer / golf_trainer / fc / staff`를 기준으로 해석한다.
- 이 role 체계는 관리자웹 권한 키(`superAdmin / primary / owner / manager / fc / staff / readonly`)와 별도 체계다.
- 같은 표기인 `fc`, `staff`도 채널에 따라 접근 범위와 화면이 다르며, IA 문맥에서는 회원앱 role로만 해석한다.

### 2.1 로그인 진입 구조

| 로그인 구분 | 입력 방식 | 선택 단계 | 진입 모드 |
|---|---|---|---|
| 회원 로그인 | 전화번호 + 비밀번호 | 없음 | 회원 모드 |
| 직원 로그인 > 트레이너 | 아이디 + 비밀번호 | 트레이너 선택 | 트레이너/골프강사 모드 |
| 직원 로그인 > FC | 아이디 + 비밀번호 | FC 선택 | FC 모드 |
| 직원 로그인 > 스태프 | 아이디 + 비밀번호 | 스태프 선택 | 스태프 모드 |

`golf_trainer`는 직원 로그인 화면에서 `트레이너` 계열로 묶어 진입한다.

### 2.2 회원-트레이너 예약 연동 원칙

| 주체 | 진입 화면 | 동작 | 결과 상태 |
|---|---|---|---|
| 회원 | MA-125, MA-123 | 트레이너/골프강사 가능 시간 확인 후 예약 요청 | 승인 대기 |
| 회원 | MA-121 | 공개 수업 또는 즉시 예약형 수업 예약 | 즉시 확정 또는 대기 |
| 트레이너 | MA-210 | 자신의 캘린더에서 회원 지정 후 수업 직접 생성 | 즉시 확정 |
| 트레이너 | MA-210, MA-221 | 회원 요청 예약 승인 / 거절 | 확정 또는 거절 |

회원의 확인 화면은 `MA-122 내 예약 / 수업 이력`, `MA-123 Golf 예약`, `MA-131 이용권 / 잔여 회차`로 연결하고,  
트레이너의 처리 화면은 `MA-210 일정 관리`, `MA-221 회원 상세`로 연결한다.

## 6. 회원 탭 구조

마켓플레이스 v1.0 구현 이후 마켓 5탭 패턴으로 재구성한다.

```text
[ 홈 ] [ 예약 ] [ 센터 ] [ 강사 ] [ MY ]
 MA-100  MA-402  MA-310  MA-320   MA-130
```

- `홈`: MA-100 (기존 단일 센터 대시보드) + 마켓 카드 (둘러보기 / 추천 센터·강사 진입)
- `예약`: MA-402 주문 / 예약 통합 내역 (결제 + 예약 + 이용 4탭). MA-122 / MA-123 등 단일 센터 예약 흐름은 본 탭의 하위 진입으로 본다.
- `센터`: MA-310 내 주변 센터 + MA-311 카테고리 검색
- `강사`: MA-320 강사 둘러보기
- `MY`: MA-130 마이페이지 — 찜(MA-330) / 멤버십 등급(MA-500) / 친구 초대(MA-501) / 활동 이력(MA-502) / 결제수단(MA-405) / Q&A·FAQ·신고(MA-600~603) / 알림 / 설정 / 약관 / 탈퇴 / 단일 센터 컨텍스트(이용권 MA-131, 출석 MA-110, QR 등)는 모두 MY 안의 메뉴로 묶는다.

보조 화면은 탭 내부 스택으로 진입한다.

- 홈 하위: 둘러보기 홈(MA-300) 진입, 추천 / 최근 본 카드, 멤버십 / 리워드 요약, 공지·이벤트 배너
- 예약 하위: 주문 상세(MA-403), 환불(MA-404), 후기 작성(MA-350 / MA-126), 결제 옵션(MA-400) → 결제(MA-140) 흐름
- 센터 하위: 센터 상세(MA-312), 지도(MA-313), 리뷰 목록(MA-351), 메신저 진입(MA-340)
- 강사 하위: 강사 상세(MA-321), 1:1 톡(MA-341)
- MY 하위: 찜(MA-330) / 멤버십(MA-500) / 추천(MA-501) / 활동(MA-502) / 결제수단(MA-405) / Q&A(MA-600) / FAQ(MA-602) / 신고(MA-603) / 설정 / 약관 / 동의관리 / 탈퇴 / 알림센터(MA-150) / 공지(MA-152) / 이용권(MA-131) / 출석 이력(MA-111) / QR(MA-110) / 결제 내역(MA-133) / 영수증(MA-142) / 체성분(MA-132) / 쿠폰함(MA-135) / 마일리지(MA-136) / 배지(MA-137) / 재등록 추천(MA-138)

찜(MA-330)은 별도 탭이 아니라 MY 메뉴 안에 위치한다. (마켓 v0.1 사이트맵 옵션 A의 변형)

## 8. 체크리스트 반영 메모

- `포인트`는 `마일리지`로 통일한다.
- 로그인은 `회원 로그인 / 직원 로그인`으로 나누고, 직원 로그인은 `트레이너 / FC / 스태프` 선택 구조로 유지한다.
- `배지`는 `리워드 센터` 내 별도 컬렉션으로 추가한다.
- `온보딩 설문`과 `첫 루틴 제안`은 `MA-127~128`로 독립시킨다.
- `회원가입`은 공개 회원 생성이 아니라 `CRM 등록 회원의 앱 연동`으로 정의한다.
- `결제`, `영수증`, `개인 결제`, `헬스장/골프장 이용권 구매`는 `MA-133~142` 영역으로 확장한다.
- `회원이 트레이너 스케줄을 보고 예약 요청`, `트레이너 승인 대기`, `골프 예약 일정 조회`, `PT 차감 이력 조회`는 `MA-121`, `MA-122`, `MA-123`, `MA-125`, `MA-131` 영역에서 함께 다룬다.
- `개인정보·로그아웃`은 설정 단일 화면이 아니라 `설정 + 약관/정책 + 동의관리 + 탈퇴` 묶음으로 해석한다.
