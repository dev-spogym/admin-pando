## 4. 핵심 화면 설계

### 4.1 MA-600 Q&A 목록

**목적**

- 회원이 마켓 전반의 공개 Q&A를 카테고리 / 상태 / 키워드로 좁혀 본다.

**필수 요소**

- 상단 헤더: 뒤로 / `Q&A` / 우상단 `+ 작성`
- 카테고리 가로 탭: `전체 / 시설 / 가격 / 강사 / 예약 / 기타`
- 상태 탭(sticky): `전체 / 답변완료 / 답변대기`
- 검색바: `궁금한 점을 검색해 보세요`
- 카드:
  - 카테고리 칩 (좌상단)
  - 제목 (1~2줄)
  - 본문 미리보기 (2줄 ellipsis)
  - 작성자(마스킹/익명), 작성일
  - 우하단: 좋아요 / 조회 / 답변 수
- 카드 클릭 → MA-601

**예외 / 빈 상태**

- 결과 0건: `해당 조건의 질문이 없어요` + `질문 작성하기` 버튼
- 검색 결과 0건: `검색 결과가 없어요` + 다른 키워드 안내

---

### 4.2 MA-601 Q&A 상세

**목적**

- 단일 질문의 본문과 모든 답변을 본다.

**필수 요소**

- 상단: 뒤로 / 우상단 옵션(신고 / 공유)
- 질문 영역:
  - 카테고리 칩, 제목, 본문, 작성자, 작성일
  - 좋아요 / 조회 / 답변 수
- 답변 리스트:
  - 답변자 아바타 + 이름
  - 역할 라벨: `공식(센터)` / `강사` / `회원` (역할별 색)
  - 본문, 작성일
  - 도움됨 토글 (helpfulCount)
  - 강사 / 센터 답변은 우측 `프로필 보기` (refId 진입 → MA-321 / MA-312)
- 하단 답변 입력바:
  - 회원 = 일반 답변, 강사 / 센터 = 역할 라벨 자동 적용
- 빈 답변: `아직 답변이 없어요 — 첫 답변을 남겨 보세요`

**예외**

- 회원 본인 작성글 24시간 이내: 우상단 옵션에 `수정 / 삭제` 추가

---

### 4.3 MA-602 FAQ

**목적**

- 결제 / 환불 / 예약 등 자주 발생하는 질문을 빠르게 찾는다.

**필수 요소**

- 상단 검색바: `자주 묻는 질문에서 찾아보세요`
- 카테고리 칩: `전체 / 이용 / 결제 / 환불 / 예약 / 회원권 / 기타`
- 인기 질문 섹션 (popular 5건): 카드 또는 칩
- 카테고리별 펼침 리스트(아코디언):
  - 클릭 시 답변 펼침 + `도움됨 N` 카운트 + `1:1 문의` CTA
- 하단 floating: `1:1 문의하기` (`MA-153` 진입)

**예외 / 빈 상태**

- 검색 0건: `검색 결과가 없어요 — 1:1 문의로 직접 물어보기` 버튼

---

### 4.4 MA-603 신고 내역

**목적**

- 회원 본인이 한 신고 건의 처리 결과와 운영팀 답변을 본다.

**필수 요소**

- 상단 헤더: 뒤로 / `신고 내역`
- 정렬: `최신순`
- 신고 카드:
  - 좌측: 대상 종류 칩 (`회원 / 리뷰 / Q&A / 메시지`)
  - 본문: 대상 발췌 / 신고 사유 / 신고일
  - 우측 상단: 상태 칩 (`접수 info / 검토중 warning / 처리완료 success / 반려 neutral`)
  - 처리완료 / 반려 시: 운영팀 답변 펼침 박스
- 카드 클릭: 상세 펼침(아코디언) — 추가 메모 / 처리일 / 운영팀 답변 본문 노출

**예외 / 빈 상태**

- 신고 이력 0건: `신고하신 내역이 없어요`
- 신고 시도(외부): 콘텐츠 옵션에서 신고 모달 → 사유 6종 + 상세 입력 → 접수 토스트

## 5. 디자인 시스템 매핑

| 요소 | 사용 컴포넌트 |

## 6. 회원 화면 흐름

### 6.1 센터 상세 → Q&A → 답변

```text
MA-312 센터 상세 → 하단 `Q&A` 진입
  → MA-600 (해당 센터 카테고리 미리 적용 가능)
    → 카드 클릭
    → MA-601 답변 보기 + 본인 답변 추가
```

### 6.2 결제 / 환불 의문 → FAQ → 1:1 문의

```text
MA-403 주문 상세 → 우상단 `?` (도움말)
  → MA-602 FAQ → 결제/환불 카테고리
  → 검색 결과 없음 → `1:1 문의하기` (MA-153)
```

### 6.3 부적절 콘텐츠 신고 → 처리 결과 확인

```text
콘텐츠 옵션 → `신고하기` → 사유 모달 → 접수
  → MA-603 신고 내역 (접수 → 검토중)
  → 운영팀 처리 → 처리완료 / 반려 + 답변 표시
```
