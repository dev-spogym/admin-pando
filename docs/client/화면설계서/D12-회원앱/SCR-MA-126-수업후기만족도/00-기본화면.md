## 화면 목적

수업이 완료된 후 회원이 별점과 텍스트 후기를 작성하여 만족도를 표현한다. 관리자와 강사는 후기를 바탕으로 수업 품질을 개선하는 데 활용한다.

## 주요 구성요소

### 수업 정보 표시 영역 (읽기 전용)

| 항목 | 설명 |
|------|------|
| 수업명 | 수강한 수업 이름 |
| 수업 일시 | 수업 날짜 및 시간 |
| 강사명 | 담당 강사 이름 |
| 강사 사진 | 강사 프로필 이미지 (소형) |

### 별점 입력

5점 만점 별점을 탭하여 선택한다. 선택 전에는 빈 별표, 선택 후에는 채워진 별표로 표시된다.

| 항목 | 설명 |
|------|------|
| 별점 | 1~5점 선택 (필수) |
| 점수 설명 | 선택한 점수에 따른 안내 문구 표시 (예: 5점 → '매우 만족') |

### 후기 텍스트 입력

| 항목 | 설명 |
|------|------|
| 후기 내용 | 자유 텍스트 입력, 최대 500자 |
| 글자 수 표시 | 현재 입력 글자 수 / 최대 글자 수 표시 |

### 태그 선택 (선택사항)

자주 쓰는 후기 키워드를 태그로 빠르게 선택할 수 있다.

| 예시 태그 |
|-----------|
| 설명이 친절해요 |
| 동작 교정을 잘 해줘요 |
| 시간 관리를 잘 해요 |
| 분위기가 좋아요 |

### 버튼

| 버튼 | 조건 | 설명 |
|------|------|------|
| 후기 등록 | 별점 선택 완료 | 후기 저장 처리 |
| 나중에 하기 | 항상 | 후기 작성 건너뛰기 |

## 계정별 차이

회원 앱 화면으로 로그인한 회원 본인이 수강한 수업에 대해서만 후기 작성이 가능하다. 동일 수업에 중복 작성 불가.

## 주요 UX 흐름

1. 수업 완료 후 앱 푸시 알림 수신 → 후기 작성 화면으로 이동 유도
2. 또는 수업 이력 목록에서 '후기 작성' 버튼 탭 → 화면 진입
3. 별점 탭하여 선택 (필수)
4. 태그 선택 (선택사항)
5. 텍스트 후기 입력 (선택사항)
6. 후기 등록 버튼 탭 → 저장 완료 → 완료 안내 화면 또는 이전 화면으로 이동
7. 나중에 하기 탭 → 후기 미작성 상태 유지, 이후 수업 이력에서 재진입 가능

## 화면 상태

| 상태 | 설명 |
|------|------|
| 기본(별점 미선택) | 후기 등록 버튼 비활성 |
| 별점 선택됨 | 후기 등록 버튼 활성화 |
| 후기 텍스트 입력 중 | 글자 수 실시간 표시 |
| 등록 성공 | '후기가 등록되었습니다.' 완료 화면 또는 토스트 |
| 등록 실패 | 오류 안내 및 재시도 유도 |
| 이미 작성됨 | '이미 작성한 후기입니다.' 안내, 수정 버튼 제공 |
