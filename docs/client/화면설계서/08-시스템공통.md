## 4. 핵심 화면 설계

### 4.1 MA-900 신규 회원 온보딩 환영

**목적**

- 신규 회원이 앱의 4가지 핵심 가치(둘러보기 / 결제·예약 / 활동·리워드 / 메신저·커뮤니티)를 첫 진입에 한 번에 이해하도록 한다.

**필수 요소**

- 상단 우측 `건너뛰기` (모든 단계에서 노출)
- 슬라이드 4종:
  1. 둘러보기 — `내 주변 센터 · 강사 · 상품 한 곳에서 발견`
  2. 결제·예약 — `옵션 선택 → 결제 → 예약·이용 자동 추적`
  3. 활동·리워드 — `누적 결제로 등급 상승, 친구 초대로 리워드`
  4. 메신저·커뮤니티 — `센터 / 강사와 직접 소통, 공개 Q&A 활용`
- 각 슬라이드: 일러스트 16:9, 1줄 헤드라인, 2줄 설명
- 페이지 인디케이터 (1/4 ~ 4/4)
- 하단 CTA: `다음` (1~3) / `시작하기` (4)
- 스와이프 좌/우 지원

**예외 / 빈 상태**

- 운동 온보딩(`MA-127`)이 미완료인 경우: 운동 온보딩이 우선 완료된 뒤 진입
- `hasSeenWelcome` 플래그가 true: 자동 진입 차단, 설정에서 다시 보기 가능

---

### 4.2 MA-901 네트워크 오류

**목적**

- 통신 단절 시 회원이 상황을 이해하고 즉시 재시도하거나 캐시된 데이터를 본다.

**필수 요소**

- 일러스트 (단절 / Wi-Fi off 모티프)
- 헤드라인: `네트워크 연결을 확인해 주세요`
- 보조 텍스트: `Wi-Fi 또는 모바일 데이터 상태를 확인해 주세요`
- 주요 CTA: `다시 시도`
- 보조 액션: `설정 열기`, `최근 본 센터 보기` (캐시)
- 하단 작은 안내: `네트워크가 복구되면 자동으로 새로고침됩니다`

**예외**

- 네트워크 복구 즉시 자동 이탈 → 직전 라우터 재시도

---

### 4.3 MA-902 서버 점검 안내

**목적**

- 서버 점검 시 회원에게 점검 시간을 명확히 안내해 불필요한 재접속을 줄인다.

**필수 요소**

- 일러스트 (수리 / 점검 모티프)
- 헤드라인: `시스템 점검 중이에요`
- 점검 정보 박스:
  - 점검 시간 (시작 ~ 종료)
  - 점검 사유 (1~2줄)
  - 예상 종료 시각
- 주요 CTA: `공지 보러 가기` (`MA-152`)
- 보조 액션: `고객센터` (`MA-153`)
- 하단: `점검이 끝나면 자동으로 이용 가능해집니다`

**예외**

- 점검 모드에서는 결제 / 예약 / 메신저 등 보호 라우트 모두 차단
- 공지 / 고객센터만 진입 가능

## 5. 공통 컴포넌트 설계

### 5.1 LoginGateModal (MFN-9C1)

**목적**

- 비로그인 회원이 결제 / 스크랩 / 리뷰 / 메신저 / 추천 코드 입력 등을 시도할 때 통일된 로그인 유도 경험을 제공한다.

**필수 요소**

- 일러스트 (잠금 / 로그인 모티프)
- 헤드라인: `로그인이 필요한 기능이에요`
- 보조 설명 (사용 위치별 동적 텍스트):
  - 결제 시도: `결제는 회원 로그인 후 가능해요`
  - 스크랩 시도: `스크랩은 회원 로그인 후 가능해요`
  - 리뷰 작성: `리뷰는 회원 로그인 후 작성할 수 있어요`
  - 메신저 시도: `메신저는 회원 로그인 후 이용 가능해요`
- 하단 듀얼 CTA: `로그인` (primary) + `회원가입` (보조)
- 우상단 `X` (닫기, 직전 컨텍스트 복귀)

**동작**

- 로그인 진입 시 `MA-001` 라우팅 + `returnUrl` 보존 → 인증 성공 후 직전 화면 자동 복귀

---

### 5.2 ShareSheet (MFN-9C2)

**목적**

- 상품 / 센터 / 강사 / 이벤트 / 추천 코드 / 후기 등 다양한 콘텐츠를 외부 채널로 공유한다.

**필수 요소**

- 상단 핸들 (드래그 가능 표시)
- 미리보기 카드: 썸네일 + 제목 + 한줄 설명
- 채널 아이콘 그리드: `카카오톡 / 문자 / 링크 복사 / 시스템 공유 (Web Share API)`
- 하단 닫기 버튼

**입력**

- `title`, `description`, `url`, `ogImage`
- 추천 코드 공유 시: 추천 가입 화면 딥링크 자동 포함

**예외**

- 시스템 공유 미지원 단말: 해당 아이콘 비활성, 안내 토스트

---

### 5.3 ConfirmDialog (MFN-9C3)

**목적**

- 취소 / 삭제 / 이탈 / 환불 신청 등 결정성 액션 직전에 회원에게 한 번 더 확인을 받는다.

**필수 요소**

- 헤드라인 (1줄)
- 본문 (2~3줄)
- 듀얼 버튼: `확인` (primary 또는 danger) + `취소` (neutral)
- 외부 클릭 / ESC 시 취소로 처리

**입력**

- `title`, `description`, `confirmLabel`, `cancelLabel`, `tone(primary / danger)`, `onConfirm`, `onCancel`

**사용 예**

- 주문 취소: `tone="danger"`, confirmLabel: `취소하기`
- 결제수단 삭제: `tone="danger"`
- 환불 신청: `tone="danger"`
- 작성 중인 글 이탈: `tone="primary"`

---

### 5.4 PushToast + usePushSimulator (MFN-9C4)

**목적**

- 새 메시지 / 주문 상태 변경 / 시스템 알림을 비차단으로 회원에게 전달한다.

**필수 요소 (PushToast)**

- 좌측 아이콘 (알림 종류별)
- 제목 1줄 + 본문 1줄
- 우측 닫기(X)
- 자동 사라짐(5초) + 마우스/탭 호버 시 일시정지

**시뮬레이터 (usePushSimulator)**

- 데모 환경에서 토스트를 트리거하는 hook
- 트리거 시점:
  - 주문 결제 완료 / 예약 확정 / 이용 가능 / 이용 완료
  - 새 메신저 메시지 도착
  - 등급 변경 / 배지 획득

**클릭 동작**

- 메시지: `MA-340 / MA-341` 진입
- 주문: `MA-403` 진입
- 등급 / 배지: `MA-500 / MA-502` 진입

**스택 정책**

- 동시 최대 3개 노출, 새 항목은 위로 push, 5초 자동 dismiss

## 7. 회원 화면 흐름

### 7.1 신규 회원 첫 진입

```text
MA-002 앱 가입 / 연동 완료
  → MA-127 ~ MA-128 운동 온보딩
  → MA-900 환영 (4 슬라이드)
  → MA-300 둘러보기 홈 (또는 MA-100 홈)
```

### 7.2 비로그인 사용자 보호 액션

```text
비로그인 → MA-312 센터 상세 → `결제하기`
  → LoginGateModal (결제 컨텍스트 텍스트)
    → 로그인 진입 (MA-001) + returnUrl 보존
    → 인증 후 MA-312 자동 복귀
```

### 7.3 결정성 액션 직전 확인

```text
MA-403 주문 상세 → `주문 취소`
  → ConfirmDialog (tone=danger)
  → 확인 시 처리 + PushToast 알림
  → 취소 시 직전 컨텍스트 유지
```

### 7.4 주문 상태 변경 → 푸시 토스트 → 진입

```text
주문 상태 변경(예약 확정 / 이용 가능 / 이용 완료)
  → PushToast 알림 (제목 + 본문)
  → 토스트 클릭 → MA-403 주문 상세 진입
```

### 7.5 네트워크 단절 → 복구

```text
화면 진입 시 fetch 실패
  → MA-901 네트워크 오류
  → 다시 시도 / 캐시 카드 / 설정 진입
  → 복구 시 자동 이탈 + 원 화면 재시도
```
