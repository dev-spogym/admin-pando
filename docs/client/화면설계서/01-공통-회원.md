> 작성일: 2026-04-26
> 버전: 3.0
> 문서 성격: 기획/퍼블리싱 기준
> 기준: `APP-IA.md`, `기능명세서/01-공통-회원.md`

---

## 1. 문서 사용 원칙

- 이 문서는 `화면 기획 기준 문서`다.
- 각 화면은 회원이 이해해야 할 구조와 퍼블리싱 기준을 우선해 정리한다.
- 체크리스트 항목이 빠지지 않도록 `화면 ID ↔ 체크리스트` 연결을 명시한다.
- 기술 구현 방식, 파일 구조, 경로 정보는 이 문서 범위에서 제외한다.

## 2. 공통 화면

| 화면 ID | 화면명 | 목적 | 핵심 요소 |
|---|---|---|---|
| MA-001 | 로그인 | 회원 로그인 / 직원 로그인 분기 | 회원 전화번호 로그인, 직원 아이디 로그인, 직원 유형 선택 |
| MA-002 | 앱 가입 / 연동 | CRM 회원의 앱 연동 완료 | 이름/전화 확인, 휴대폰 인증, 비밀번호 설정 |

### 공통 정책 메모

- 회원 기본 생성은 CRM에서 이뤄진다.
- 앱에서는 `휴대폰 인증 기반 연동`으로 가입을 완료한다.
- 공개형 즉시 회원 생성은 현재 기본 시나리오가 아니다.
- 로그인은 `회원 로그인`과 `직원 로그인`으로 나뉜다.
- 직원 로그인은 `트레이너`, `FC`, `스태프` 직군 선택 후 아이디/비밀번호로 인증한다.
- `golf_trainer` 계정은 로그인 화면에서는 `트레이너` 계열로 묶어 진입한다.
- 회원은 트레이너의 가능 시간을 보고 PT/골프 레슨을 요청할 수 있다.
- 회원이 직접 만든 트레이너 대상 예약은 `트레이너 승인 대기`를 기본 상태로 한다.
- 트레이너가 자신의 스케줄에서 회원을 직접 지정해 일정을 만들면 즉시 확정으로 처리한다.

## 4. 핵심 화면 설계 요약

### 4.1 MA-001 로그인

**목적**

- 회원과 직원이 동일 앱 안에서 본인 역할에 맞는 로그인 경로로 진입한다.

**필수 요소**

- 상단 탭: 회원 로그인 / 직원 로그인
- 회원 입력: 전화번호, 비밀번호
- 직원 입력: 직원 유형(트레이너/FC/스태프), 아이디, 비밀번호
- 로그인 성공 후 역할별 기본 홈 이동
- 회원 전용 `앱 연동하기` 링크

### 4.2 MA-100 홈

**목적**

- 회원이 가장 자주 쓰는 `예약`, `출석`, `리워드`, `온보딩`, `공지`를 한 화면에서 본다.

**필수 요소**

- PT 예약 바로가기
- GX 스케줄 바로가기
- Golf 예약 바로가기
- 마일리지/배지 요약 카드
- 공지/이벤트 배너
- 오늘 수업 요약
- 이번 달 출석 요약
- 온보딩 또는 추천 루틴 카드

### 4.3 MA-121 수업 상세 / 예약

**목적**

- 예약 여부, 정원 현황, 강사 진입, 대기 등록 진입을 한 곳에서 처리한다.

**필수 요소**

- 수업명, 시간, 장소, 강사
- 정원 / 잔여석
- 예약하기 / 예약취소
- 트레이너 승인 필요 시 예약 요청 상태
- 대기열 등록
- 강사 상세 보기

### 4.4 MA-122 내 예약 / 수업 이력

**목적**

- 회원이 내 수업 일정, 골프 예약 일정, 승인 대기 상태, 완료 이력을 한 화면에서 확인한다.

**필수 요소**

- 서비스 필터: 전체 / PT / GX / Golf
- 상태 필터: 승인 대기 / 확정 / 완료 / 취소
- 리스트 또는 캘린더 전환
- 트레이너명 / 수업일 / 시간 / 상태
- 후기 작성 진입

### 4.5 MA-123 Golf 예약 / 골프 강사 예약 / 타석 선택

**목적**

- 골프 회원이 `강사 레슨 예약`과 `타석 예약`을 하나의 화면 안에서 처리한다.

**필수 요소**

- 골프 강사 예약 탭
- 강사명 / 레슨명 / 시간대 / 타석 / 가격
- 내 골프 레슨 예약 목록 / 일정
- 골프 타석 예약 탭
- 타석 상태 표시, 만석 시 대기 등록
- 레슨권 / 이용권 구매 CTA

### 4.6 MA-124 대기 예약 관리

**목적**

- 회원이 대기 순번과 자동 확정 상태를 직접 확인하고 취소한다.

**필수 요소**

- 수업명, 일시, 강사
- 현재 순번
- 자동 확정 여부
- 대기 취소

### 4.7 MA-125 강사 상세

**목적**

- 예약 전 강사의 전문분야, 후기 요약, 가능 수업을 확인한다.

**필수 요소**

- 강사명 / 경력
- 전문분야 태그
- 최근 후기 요약
- 예약 가능한 시간대 요약
- 예약 요청 CTA

### 4.8 MA-126 수업 후기 / 만족도

**목적**

- 완료 수업에 대해 별점, 태그, 한줄 후기, 추천 의향을 받는다.

**필수 요소**

- 별점 1~5
- 빠른 태그
- 한줄 후기
- 추천 의향 점수

### 4.9 MA-127 ~ MA-128 온보딩

**목적**

- `운동 목적/성향`, `통증/체형 정보`, `첫 루틴 제안`을 연결한다.

**필수 요소**

- 운동 목적 복수 선택
- 운동 성향 / 강도 선택
- 통증 부위 선택
- 체형/집중 부위 선택
- 선호 요일/운동 시간
- 요약 및 첫 루틴 제안

### 4.10 MA-132 체성분 / FMS

**목적**

- 인바디와 FMS/체형 정보를 한 도메인 안에서 본다.

**필수 요소**

- 최근 체성분 요약
- 측정 이력
- FMS 총점
- 세부 항목 점수
- 코치 메모

### 4.11 MA-131 이용권 / 잔여 회차

**목적**

- 회원이 이용권 만료일뿐 아니라 PT 잔여 횟수와 차감 이력, 다음 예약 예정 일자를 구분해 확인한다.

**필수 요소**

- 이용권명 / 만료일
- 총 횟수 / 사용 횟수 / 잔여 횟수
- 최근 차감 일자
- 다음 예약 예정 일자
- 차감 기준 안내: 수업 완료 후 횟수 차감
- 예약 화면 이동 CTA

### 4.12 MA-133 ~ MA-142 구매 / 결제

**목적**

- PT 플랜뿐 아니라 헬스장 이용권, 골프장 이용권, 골프 레슨권, 개인 결제를 하나의 구매 흐름으로 연결한다.

**필수 요소**

- 결제 내역 목록
- 영수증 진입
- 상품 스토어 카테고리
- 상품 상세 가격/혜택/구성
- 결제수단 선택
- 마일리지 사용
- 개인 결제 직접 입력
- 영수증 상세

### 4.13 MA-136 ~ MA-137 리워드 센터

**목적**

- 체크리스트의 `포인트/배지`를 `마일리지/배지` 구조로 정리한다.

**필수 요소**

- 쿠폰 탭
- 마일리지 탭
- 배지 탭
- 마일리지 정책
- 배지 획득 조건

### 4.14 MA-138 재등록 추천 플랜

**목적**

- 회원이 만료 전후에 재등록 후보 상품 3종을 비교한다.

**필수 요소**

- 추천 플랜 3종
- 각 플랜 가격/혜택
- 결제 CTA
- 결제/영수증 진입 연계

### 4.15 MA-150 알림센터

**목적**

- 예약, 이용권, 리워드, 공지 알림을 한곳에 모으고 액션 화면으로 이동시킨다.

**필수 요소**

- 전체/미읽음 탭
- 알림 카테고리
- 액션 버튼
- 대상 화면으로 이동

### 4.16 MA-155 ~ MA-158 설정 / 개인정보

**목적**

- 알림 설정과 함께 개인정보 관련 화면을 하나의 허브로 묶는다.

**필수 요소**

- 예약/이용권/결제/공지/마케팅 알림 ON/OFF
- 약관 / 정책 진입
- 동의관리 진입
- 개인 결제 / 상품 스토어 진입
- 앱 버전
- 회원 탈퇴
- 로그아웃
