## 1. 문서 사용 원칙

- 본 문서는 기존 결제 화면(`MA-140 ~ MA-142`)을 변경하지 않고, 마켓 결제 후속 화면(`MA-4XX`)만 정의한다.
- 모든 화면은 회원앱 디자인 시스템(Pretendard, primary 청록 #0E7C7B, rounded-card, shadow-card-soft)을 그대로 사용한다.
- 화면 ID는 `MA-4XX`, 기능 ID는 `MFN-4XX`를 따른다.
- 기획 단계 문서이며 구현 경로 / 파일 구조 정보는 포함하지 않는다.

진입 경로: 마켓 상품 상세(`MA-312` / `MA-139`) → `MA-400` → `MA-401` 또는 `MA-140` → `MA-402 / MA-403`.

## 4. 핵심 화면 설계

### 4.1 MA-400 결제 옵션

**목적**

- 마켓 상품(주로 PT / 횟수형 / 그룹 레슨)의 옵션을 결정하고 결제 진입한다.

**필수 요소**

- 상단: 뒤로가기 / 상품 썸네일 / 상품명 / 센터명
- 강사 선택 가로 스크롤 (사진 + 이름 + 별점, 단일 선택)
- 요일 + 시간 그리드 (요일 7칸 × 시간 슬롯 8칸)
- 시작일 옵션 카드 3종 (`오늘 시작 / 이번 주 시작 / 다음 주부터`)
- 회차 플랜 카드 3종 (`5회 / 10회 / 20회`, 단가 배수 표기)
- 옵션 요약 줄 (예: `김도윤 강사 · 화/목 19:00 · 10회`)
- 우상단 또는 하단 가격 sticky (정가 / 할인가)
- 하단 듀얼 CTA: `장바구니 담기` (보조) + `바로 결제` (primary)

**예외 / 빈 상태**

- 강사가 선택되지 않으면 `바로 결제` 비활성
- 가능 시간이 없는 강사 선택 시: `해당 강사의 가능 시간이 없습니다 — 다른 강사를 선택해 주세요`

---

### 4.2 MA-401 장바구니

**목적**

- 여러 센터 / 상품 / 옵션을 한 번에 결제한다.

**필수 요소**

- 상단 헤더: 뒤로가기 / `장바구니` 제목 / 전체선택 체크박스
- 항목 카드:
  - 좌측 체크박스 + 썸네일
  - 중앙: 센터명, 상품명, 옵션 요약, 단가 / 수량 stepper
  - 우상단: 삭제 X
- 쿠폰 섹션: `자동 추천 쿠폰 / 직접 선택 / 미사용`
- 마일리지 입력 (보유 / 사용 / 적용 후 차감액)
- 합계 박스: 상품 합계 / 쿠폰 할인 / 마일리지 사용 / 최종 결제액
- 하단 sticky CTA: `N건 결제하기` (선택 항목 합계)

**예외 / 빈 상태**

- 빈 장바구니: 일러스트 + `장바구니가 비어 있어요` + `상품 둘러보기` 버튼
- 쿠폰 최소금액 미달: 안내 텍스트 + 자동 비활성

---

### 4.3 MA-402 주문 내역

**목적**

- 회원의 결제 + 예약 + 이용 + 환불을 한 화면에서 추적한다.

**필수 요소**

- 상단 헤더: 뒤로 / `주문 / 예약` / 알림
- 탭 (sticky): `전체 / 진행중 / 완료 / 취소`
- 정렬: `최신순 / 결제액 높은 순`
- 카드:
  - 좌측 썸네일
  - 우측: 센터명, 상품명, 옵션 요약, 강사명, 결제일, 결제액
  - 우측 상단: 상태 칩 (대기/확정/이용중/완료/취소/환불)
  - 우측 하단: 잔여 회차 / 다음 예약 일시 (있을 때)
- 카드 클릭 → MA-403

**예외 / 빈 상태**

- 탭별 0건: `해당 상태의 주문이 없어요` + 둘러보기 / 결제 진입

---

### 4.4 MA-403 주문 상세

**목적**

- 단일 주문의 결제 → 예약 → 이용 → 완료를 한 화면에서 추적하고 액션한다.

**필수 요소**

- 상단: 뒤로 / 공유 / 영수증 진입
- 상품 카드: 썸네일, 센터, 상품, 옵션 요약, 강사, 시작일, 회차(총/잔여)
- 4단계 timeline:
  - 결제 완료 → 예약 확정 → 이용 가능 → 이용 완료
  - 각 단계 제목 + 설명 + 발생 시각 (없으면 회색)
  - 환불 / 취소는 별도 종료 단계로 분기
- 결제 정보: 결제수단 카드사 / 끝 4자리 / 결제액 / 정가 / 사용 마일리지 / 쿠폰
- 하단 액션 (상태별):
  - `pending / confirmed`: 주문 취소, 환불 신청
  - `in_use`: 환불 신청, 후기 미리보기
  - `completed`: 후기 작성, 영수증 보기
  - 모든 단계: 영수증 보기

**예외**

- 환불 / 취소된 주문: 액션 비활성 + 처리 결과 표시

---

### 4.5 MA-404 환불 신청

**목적**

- 회원이 환불 사유와 예상 금액을 확인한 뒤 환불을 접수한다.

**필수 요소**

- 상단: 뒤로 / `환불 신청`
- 환불 대상 요약 카드 (썸네일 / 상품명 / 옵션 / 회차)
- 사유 선택 (단일):
  - 단순 변심 / 서비스 불만족 / 일정 변경 불가 / 기타(직접 입력)
- 추가 의견 textarea (선택)
- 예상 환불액 박스: 결제액 / 환불 비율 / 환불 예상액 / 정책 안내 본문
- 동의 체크: `환불 정책에 동의합니다`
- 하단 CTA: `환불 신청`

**예외 / 빈 상태**

- 환불 불가 상태(`completed / cancelled / refunded`): 진입 자체 차단 + 안내
- 잔여 회차 0회: `잔여 회차가 없어 환불이 어려워요` 안내

---

### 4.6 MA-405 결제수단 관리

**목적**

- 등록된 카드 / 페이를 추가, 삭제, 기본 지정한다.

**필수 요소**

- 상단: 뒤로 / `결제수단` / 우상단 `+ 추가`
- 카드 리스트:
  - 카드사 로고, 카드명, 끝 4자리, 만료월/년, `기본` 배지
  - 우측: 메뉴(기본 지정 / 삭제)
- 페이 리스트:
  - 카카오페이 / 네이버페이 (좌측 로고 + 우측 ON/OFF 토글)
- 빈 상태: 일러스트 + `결제수단을 추가해 주세요` + `+ 카드 추가` CTA

**예외**

- 기본 결제수단 삭제 시: 다음 등록 항목으로 자동 승계 + 안내 토스트

## 5. 디자인 시스템 매핑

| 요소 | 사용 컴포넌트 |
|---|---|
| 옵션 칩 / 회차 카드 | `Chip active` / `Card variant="soft"` |

| 빈 상태 | `EmptyState` |
| 확인 / 환불 신청 | `ConfirmDialog (tone=danger)` |

## 6. 회원 화면 흐름

### 6.1 옵션 선택 → 바로 결제 → 추적

```text
MA-312 센터 상세 → 상품 카드 클릭
  → MA-400 결제 옵션 (강사 + 시간 + 회차)
    → MA-140 결제하기 (기존)
      → MA-142 영수증 (기존)
        → MA-402 / MA-403 주문 내역·상세 자동 진입 가능
```

### 6.2 장바구니 다중 결제

```text
다양한 상품 → MA-400 → 장바구니 담기 → ...
  → MA-401 장바구니
    → 항목 선택 + 쿠폰/마일리지
    → MA-140 결제하기
    → MA-402 진행중 탭에 비례 분배 결제 항목들 자동 등재
```

### 6.3 환불 → 처리 결과

```text
MA-403 주문 상세 → `환불 신청`
  → MA-404 환불 신청 (사유 + 예상액)
    → MA-403 timeline에 `환불 완료` 단계 표시
```
