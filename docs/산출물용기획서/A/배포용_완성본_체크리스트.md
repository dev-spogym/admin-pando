## 3. 운영 기준 고정 여부

- 리드 상태 체계 통일
- 회원/이용권/결제 상태 분리
- 본사/지점 권한 소유권 분리
- 자동화/만료 알림 정책 세트 구조 반영
- 회원앱 역할별 운영 범위 반영

## 4. 퍼블리싱 기준 고정 여부

- 핵심 리스트 화면의 상태 배지 우선순위 정의
- 상세 화면의 핵심 탭 우선순위 정의
- 본사용 정책 화면과 지점용 적용 화면 분리
- 앱 회원 홈과 현장 인력 홈의 역할 분기 반영

## 5. 이 패키지의 사용 선언

- 기존 `docs` 원본은 참고 자료로 유지
- 실제 기획/퍼블리싱/QA 기준은 `docs/HWANG폴더`를 우선 사용
- 백엔드 라우터 상세 연결은 이 체크리스트의 범위 밖
