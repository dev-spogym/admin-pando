# 매출관리 기능명세서

## 1. 목적

- 판매, 결제, 환불, 미수금, 선수익금 운영 기준을 통일한다.
- 결제 상태와 이용권 상태가 서로 섞이지 않도록 분리한다.

## 2. 포함 범위

- 매출 현황
- POS 판매
- 결제 처리
- 환불 관리
- 미수금 관리
- 선수익금 조회

## 3. 결제 상태 기준

| 상태 | 설명 |
| --- | --- |
| 결제완료 | 정상 승인된 결제 |
| 분할결제중 | 일부 수납 완료, 잔액 존재 |
| 미수금 | 수납 대기 |
| 부분환불 | 일부 금액 환불 |
| 환불완료 | 전체 환불 완료 |
| 취소 | 승인 취소 |

## 4. 핵심 기능

- 상품 판매 및 구매자 선택
- 결제 수단 선택 및 승인 확인
- 중복 결제 경고
- 환불 처리
- 미수금 메모 및 예정일 관리
- 기간별 매출 현황 조회

## 5. 운영 규칙

- 결제완료 또는 승인된 분할결제만 이용권을 활성화할 수 있다.
- 미수금은 판매 의사 상태이지 이용 시작 허용 상태가 아니다.
- 환불은 금액, 이용권 종료, 매출 반영을 함께 처리해야 한다.
- 선수익금은 서비스 제공 전 잔액 관점으로 계산한다.

## 6. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 매출 조회 | staff |
| 결제 처리 | staff |
| 환불 처리 | manager |
| 미수금 정리 | manager |
| 기간 통계 조회 | owner 이상 |

## 7. 퍼블리싱 포인트

- 결제 화면은 속도가 우선이다.
- 환불과 미수금은 경고성 UI를 명확히 분리한다.
- 결제 상태 배지와 이용권 상태 배지를 같은 컴포넌트 톤으로 처리하지 않는다.
