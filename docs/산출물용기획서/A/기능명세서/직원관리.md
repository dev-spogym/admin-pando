# 직원관리 기능명세서

## 1. 목적

- 직원 등록부터 퇴사, 근태, 급여까지 인사 운영의 기본 흐름을 정리한다.
- 회원 운영과 수업 운영에 직접 영향을 주는 인력 변동을 별도 관리한다.

## 2. 포함 범위

- 직원 목록
- 직원 등록/수정
- 퇴사 처리
- 직원 근태
- 급여 관리
- 급여 명세서

## 3. 핵심 기능

### 3.1 직원 목록

- 재직 상태, 역할, 지점, 담당 회원 수 확인
- 상세 수정 진입

### 3.2 직원 등록/수정

- 기본정보, 역할, 소속, 근무 상태 관리
- 표준 역할 기준 권한 미리보기

### 3.3 퇴사 처리

- 담당 회원 재배정
- 예정 수업 및 스케줄 정리
- 최종 퇴사 확정

### 3.4 근태 관리

- 일별 출퇴근 기록
- 월간 요약
- 수동 정정

### 3.5 급여 관리

- 급여 기준값 확인
- 정산 내역 검토
- 급여 확정

## 4. 운영 규칙

- 역할은 표준 권한 체계를 기준으로 한다.
- 퇴사 처리는 단순 비활성화가 아니라 인수인계 절차를 포함해야 한다.
- FC나 강사 퇴사 시 회원과 수업 재배정이 선행되어야 한다.
- 근태와 급여는 읽는 사람과 수정하는 사람의 권한을 분리한다.

## 5. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 직원 조회 | manager |
| 직원 등록/수정 | owner |
| 퇴사 처리 | owner |
| 근태 조회 | manager |
| 근태 정정 | owner |
| 급여 조회/확정 | owner 이상 |

## 6. 퍼블리싱 포인트

- 직원 목록은 사람 검색보다 상태 파악과 역할 구분이 중요하다.
- 퇴사 처리는 단계형 화면이 적합하다.
- 급여 화면은 금액 신뢰성이 중요하므로 경고와 확정 단계를 분명히 분리한다.
