## 4. UX/접근성 개선 (Nielsen 10 Heuristics + WCAG 2.1 AA)

### 4-1. 일관성 개선
- `window.confirm` → `ConfirmDialog` 전면 교체 (10+ 페이지)
- `alert()` → `toast (sonner)` 전면 교체 (40+ 건)
- 인라인 모달 → `Modal` 공통 컴포넌트 전환
- `text-error` → `text-state-error` 디자인 토큰 통일

### 4-2. 접근성 (WCAG 2.1 AA)
- `aria-required`, `aria-invalid` — 모든 폼 필드
- `role="alert"` — 에러 메시지
- `role="dialog"`, `aria-modal="true"` — 모달
- `role="button"`, `tabIndex`, `onKeyDown` — 락커 그리드
- `sr-only` — 스크린리더 전용 레이블 (사이드바, 로그인)
- 사이드바 축소 모드 `aria-label` / `title`

### 4-3. 반응형/모바일
- `DataTable` 모바일 sticky 첫 번째 컬럼 (`left-0`, `z-10`)
- 계약 마법사 Step 인디케이터 컴팩트 모드 (`<768px`)
- 상품 카드 `div` → `button` (터치 타겟)

### 4-4. 데이터 정합성
- 하드코딩 날짜 → 동적 `new Date()` (Attendance, RFID, Branch, Mileage)
- Prisma Decimal → `Number()` 래핑 (Sale, Payroll, Product, Contract)
- 수신자 수 하드코딩 1240 → Supabase count 쿼리

### 4-5. 사용성
- 페이지네이션 필터 변경 시 1페이지 리셋 (Payroll, StaffList)
- 대시보드 통계 카드 클릭→상세 페이지 이동
- 매출 탭별 집계 뷰 (기간/상품/결제수단/담당자)
- 회원 목록→메시지 발송 연동 (sessionStorage)
