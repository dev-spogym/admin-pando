## 1. 이번 주 진행 목적

- A(관리자/운영 기준 문서)와 B(모바일 앱 기획 문서)의 연동 구조와 문서 정합성을 점검
- B 문서를  `기획/퍼블리싱 기준 문서`로 재정리
- B 기능을  `유지 + A 연계 보강` 관점으로 재정의

## 2. 주요 진행 내용

### 2.1 A-B 연동 검토 및 추적 기준 정리

- A와 B 문서를 비교 검토하여 주요 충돌 및 누락 가능 구간을 확인
- B 기능을 아래 4개 기준으로 재분류하는 `A-B 기능 추적 매트릭스`를 정리
- 분류 기준 1: `유지`
- 분류 기준 2: `A 보강 필요`
- 분류 기준 3: `정책 연동 필요`
- 분류 기준 4: `앱 고유 기능`
- 검토 결과, B는 A의 단순 축소판이 아니라 A와 연동되는 앱 채널로 유지하는 방향이 적합하다고 정리

### 2.2 만료 알림 정책 기준 정렬

- B 문서 내 고정형 `D-30 / D-14 / D-7 / D-3 / D-1` 해석을 본사에서 정하여 셋팅할 수 있도록하는 것
- B는 자체 고정 규칙을 소유하지 않고, A에서 관리하는 `본사 정책 세트 -> 지점 선택 적용` 결과를 표시하는 구조로 기준 변경
- 이에 따라 회원/FC 문서도 `정책 결과 표시 채널` 관점으로 재정리

### 2.3 B 문서 방향성 재정리

- B 기능은 삭제보다 유지가 우선이라는 방향으로 재정리
- 다만 A와 연결되는 운영 기준이 필요한 기능은 `A 보강 필요` 또는 `정책 연동 필요`로 분류
- 온보딩 등 일부 기능은 `앱 고유 기능`으로 별도 성격을 구분

### 2.4 클라이언트 보고용 문서 정리

- IA, 화면설계서, 기능명세서, 연계정의서, 사용자시나리오, README를 `기획/퍼블리싱 기준`으로 재정리
- 문서에서 아래 성격의 항목을 제거 또는 기획 문장으로 치환
- 제거/치환 항목 1: 앱 라우트 경로
- 제거/치환 항목 2: 코드/파일 경로
- 제거/치환 항목 3: 구현 상태값 (`구현`, `보강구현` 등)
- 제거/치환 항목 4: 앱 구현 사항
- 제거/치환 항목 5: 저장 경로, 권한 구현 방식, 기술 스택 상세
- 제거/치환 항목 6: QA/개발 보조 메모
- 결과적으로 B 문서는 클라이언트 공유용 기획 문서와 내부 기술 참고 문서의 경계를 분리하는 방향으로 정리
