## 상태 고유 인터랙션
- 진입 시 `role="status" aria-live="polite" aria-busy="true"` 선언
- 사용자 입력 차단(디바운스 된 검색은 표시는 유지, 제출은 비활성)
- ESC·Enter 등 전역 핫키는 무시

## 상태 고유 비즈니스 룰 델타
- 최소 300ms 표시 후 결과로 전환 (깜빡임 방지). 실제 fetch가 그보다 빠르면 delay-300.
- 필터 변경 → refetch 시엔 **기존 데이터 유지 + 테이블 상단 progress bar**(두께 2px, primary).
- 15초 초과 로딩 시 `05-에러` 전이 + "응답이 느립니다" 안내 (React Query timeout 설정).
- 5초 초과 시 "조금만 더 기다려주세요" 서브 토스트 (sonner info) 1회.

## 에지 케이스
- JWT 만료 상태에서 진입 → 로딩을 거치지 않고 `/login?redirect=/sales`로 즉시 이탈
- 백그라운드 탭에서 refetch → 로딩 오버레이 없이 silent refetch (React Query `isFetching` 만 사용)
- 권한에 따라 스켈레톤 레이아웃 동일 (RBAC는 데이터 로드 후 적용). trainer도 동일 스켈레톤.
- 네트워크 복구 직후 재진입: 기존 skeleton 재사용

## 바이브코딩 프롬프트 (델타)
```
상태: sales-loading. 마스터 프롬프트 기본값에 아래만 override.
- isLoading===true 분기에서만 렌더
- <DataTable loading={true} /> 로 스켈레톤 5행 (실제 컬럼 수 반영 불필요, 대표 3~4 블록)
- StatCard에 loading prop 전달 → value 영역에 <Skeleton class="h-8 w-24"/>
- PageHeader 버튼 {disabled: isLoading}
- <main role="status" aria-busy="true" aria-label="매출 현황 데이터 로딩 중">
- 300ms min-display (useDelay 훅 또는 setTimeout)
- 5초 초과 시 toast.info('조금만 더 기다려주세요.')
- 15초 초과 시 throw → 05-에러
- refetch 시 (isFetching && !isLoading) → 상단 2px progress bar만
```
