## 역할별 차이
마스터 §5 매트릭스 그대로 적용:
- **owner/manager**: 전체 탭 + 모든 액션
- **fc**: 엑셀 버튼 숨김 (`07-FC-제한`)
- **trainer**: `06-트레이너-제한` 파일로 분기
- **staff/front**: 애초에 진입 불가(`/forbidden`)

## 상태 고유 인터랙션
- 행 클릭(TAB-001/006/007) → DLG-S001 매출상세 오픈 + `setSelectedSale(row)`
- 상품명 클릭 → `moveToPage(971)` (event.stopPropagation)
- 구매자 클릭 → `moveToPage(985, {id: row.buyerId})` (event.stopPropagation)
- 컬럼 헤더 클릭 → 정렬 토글, 3번째 클릭 시 원상복구
- 엑셀 버튼 → `exportToExcel(filteredData, EXPORT_COLUMNS_22, {filename:'매출내역'})` + toast
- 필터 변경 → 클라이언트 필터링 (API 재호출 X, staleTime 내)
- Tab 전환 → 컬럼 교체 + 데이터 소스 교체 (periodData/productData/…)

## 상태 고유 비즈니스 룰 델타
- `summary.netTotal = summary.total - summary.refund` 정확 적용
- 집계 탭(TAB-002~005)에서 환불 건 제외
- 매출유형 필터 선택 시 URL 쿼리 반영하지 않음 (세션 단위)
- 페이지네이션 기본 50건/페이지, 옵션 20/50/100
- 엑셀 10,000건 상한, 초과 시 "엑셀은 최대 10,000건까지 지원합니다" warning

## 에지 케이스
- 정렬 변경 후 필터 변경 시: 정렬 유지
- 탭 전환 시 페이지 1로 리셋
- 필터 리셋 시 매출유형/정렬/페이지 모두 초기값 복귀
- 매우 큰 데이터(10,000건 이상): virtual scroll 권장 (Phase 2). 현재는 pageSize 100 상한.
- 당일 결제 직후 진입: React Query invalidate → 즉시 반영
- 탭 전환 시 해당 탭 데이터 0건 → `04-탭-빈상태`
