## 5. 상태 고유 인터랙션

| 이벤트 | 트리거 | 결과 |
|---|---|---|
| `E_OPEN_01` 진입 | 행 클릭 | `fetchTaxInvoiceDetail(id)` 호출 |
| `E_LOAD_OK_01` | supabase 응답 OK | `02-열림` 전환 |
| `E_LOAD_ERR_01` | supabase 응답 err | `03-에러` 전환 + 자동 닫힘 |
| `E_CANCEL` | ESC/X/배경 | `onClose()` 즉시 호출(진행 중 요청은 언마운트로 abort) |

---

## 6. 상태 고유 비즈니스 룰 델타

1. **요청 취소**: 사용자가 로딩 중 ESC/X 로 닫으면, 컴포넌트 언마운트 시 in-flight 요청은 무시(결과를 `setState` 하지 않도록 `abortSignal` 또는 mounted 플래그).
2. **캐싱**: React Query 사용 시 `staleTime: 30_000`으로 재방문 시 즉시 표시. 본 문서는 직접 fetch 가정.
3. **타임아웃**: 3초 초과 시 스켈레톤 유지하되, 5초 초과 시 "응답 지연" 배너(옵션) 표시.
4. **스켈레톤 표시 최소 시간 없음**: 빠른 응답 시 바로 `02-열림` 전환 (플래시 방지용 delay 없음).

## 7. 에지 케이스

| 케이스 | 처리 |
|---|---|
| 로딩 중 다시 같은 행 클릭 | `taxInvoiceId` 변화 없으면 추가 요청 생략 |
| 로딩 중 다른 행 클릭 | `taxInvoiceId` 변경 → 새 fetch, 이전 결과 무시 |
| 세션 만료 | DLG-000 오픈 → 본 다이얼로그 자동 정리 |
| 5초 초과 응답 | 계속 로딩 유지 또는 자동 취소(옵션) |
| 언마운트 후 setState | `mounted` flag 또는 AbortSignal 로 방어 |
| 품목 수가 매우 많음 (>100) | API 측에서 limit 50 권고, UI는 그대로 |

## 8. 바이브코딩 프롬프트 (델타)

```
상태: DLG-S010 — 01-로딩

// 스켈레톤 렌더
{isLoading && (
  <div className="p-6 space-y-3" aria-busy="true" aria-live="polite">
    {Array.from({ length: 6 }).map((_, i) => (
      <div key={i} className="flex justify-between">
        <div className="animate-pulse bg-gray-200 h-4 w-24 rounded" />
        <div className="animate-pulse bg-gray-200 h-4 w-32 rounded" />
      </div>
    ))}
    <div className="mt-4 animate-pulse bg-gray-200 h-24 rounded-xl" />
  </div>
)}

// fetch (마운트 + taxInvoiceId 변경 시)
useEffect(() => {
  if (!isOpen || !taxInvoiceId) return;
  let mounted = true;
  (async () => {
    setIsLoading(true);
    const { data, error } = await supabase
      .from('tax_invoices')
      .select('*')
      .eq('id', taxInvoiceId)
      .single();
    if (!mounted) return;
    if (error) {
      toast.error(mapInvoiceError(error));
      onClose();
    } else {
      setTaxInvoice(data);
    }
    setIsLoading(false);
  })();
  return () => { mounted = false; };
}, [isOpen, taxInvoiceId]);
```
