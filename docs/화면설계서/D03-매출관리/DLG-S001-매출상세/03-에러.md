## 역할별 차이
- 403 에러는 owner/manager가 타 지점 매출에 접근 시 발생. super/primary는 해당 에러 미발생.
- fc/staff는 구매 정보만 허용 역할이므로, 금액 섹션 접근 중 403이 아니라 `02-열림`에서 섹션 마스킹으로 처리 → 이 상태 진입 드뭄.

## 상태 고유 인터랙션
1. 오픈 시 "다시 시도"(Primary)에 오토포커스.
2. "다시 시도" 클릭 → `query.refetch()` → `01-로딩` 재진입.
3. "닫기" 클릭/ESC/배경 → 닫힘(부모 화면 유지).
4. 5회 연속 재시도 실패 → "다시 시도" 버튼 30초 쿨다운(카운트다운 라벨).
5. E404300은 재시도해도 해결되지 않음 → "다시 시도" 버튼 숨김, [닫기]만 노출.

## 상태 고유 비즈니스 룰 델타
- **에러 발생 시 부모 테이블에 회귀 신호**: 부모 테이블에 toast "매출을 불러오지 못했습니다" 별도 노출 금지(모달 내부만 안내).
- **자동 로그**: `console.error` + (옵션) Sentry captureException. 사용자 식별 필드 제외.
- **E409300** (결제 취소 클릭 시 충돌): 이 상태로 전이되지 않음. `02-열림` 유지 + 토스트.
- **재시도 동시성**: 버튼 더블클릭 방지 → `isFetching` 가드.

## 에지 케이스
- **로딩 중 사용자가 네트워크 끊음**: `NETWORK` 에러 → 자동으로 본 상태 진입.
- **네트워크 복구 + 자동 refetch**: `refetchOnReconnect: true` 면 사용자 클릭 없이 `01-로딩` → `02-열림` 복구.
- **E401002(세션 만료)**: 본 에러 상태 대신 DLG-000 세션만료 다이얼로그가 우선 오픈. 이 상태 진입하지 않고 즉시 언마운트.
- **E429001(Rate limit)**: "요청이 많습니다. 잠시 후 재시도" 카피 + 30초 카운트다운.

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: sale-detail-error (에러)

이 상태에서만 적용할 변경점:
- q.isError === true 분기
- Header 아이콘 교체: AlertCircle + bg-rose-50
- Body: <ErrorPanel errorCode={...} onRetry={...} />
- Footer: [다시 시도][닫기] 만 노출

컴포넌트:
function ErrorPanel({ error, onRetry }: { error: any; onRetry: () => void }) {
  const code = error?.errorCode ?? (error?.name === 'NetworkError' ? 'NETWORK' : 'E500001');
  const map = {
    E403300: { Icon: Lock,        title: '조회 권한이 없습니다',         body: '타 지점 매출은 조회할 수 없습니다.' },
    E404300: { Icon: SearchX,     title: '매출 내역을 찾을 수 없습니다', body: '해당 매출이 삭제되었거나 ID가 유효하지 않습니다.' },
    E500001: { Icon: ServerCrash, title: '서버 오류가 발생했습니다',    body: '잠시 후 다시 시도해 주세요.' },
    NETWORK: { Icon: WifiOff,     title: '네트워크 연결을 확인해주세요', body: '오프라인 상태에서는 매출 상세를 조회할 수 없습니다.' },
  } as const;
  const entry = map[code as keyof typeof map] ?? map.E500001;
  const { Icon, title, body } = entry;
  const retriable = code !== 'E404300' && code !== 'E403300';

  return (
    <div role="alert" aria-live="assertive" className="py-8 flex flex-col items-center text-center space-y-3">
      <div className="size-12 rounded-full bg-rose-50 grid place-items-center">
        <Icon className="size-6 text-rose-500" aria-hidden />
      </div>
      <h3 className="text-base font-semibold text-gray-900">{title}</h3>
      <p className="text-sm text-gray-600 max-w-[360px]">{body}</p>
      <p className="text-xs text-gray-400" aria-hidden>{code}</p>
    </div>
  );
}

Footer:
{retriable && (
  <button onClick={onRetry} autoFocus disabled={q.isFetching}
    className="h-10 px-4 rounded-lg bg-blue-600 hover:bg-blue-700 disabled:bg-blue-300
               text-white text-sm font-medium">
    {q.isFetching ? '재시도 중...' : '다시 시도'}
  </button>
)}
<button onClick={onClose}
  className="h-10 px-4 rounded-lg border border-gray-300 bg-white hover:bg-gray-50
             text-sm font-medium text-gray-700">닫기</button>

접근성:
- ErrorPanel role="alert" aria-live="assertive"
- errorCode 는 aria-hidden 으로 스크린리더 생략(개발자용)
```
