## 📝 디스크립션

### 사용 시점 (Trigger)
- 부분환불이 완료되어 잔여 환불 가능금액이 있는 상태 (E_ST_PARTIAL_01, 신규기능)
- 파란 배지 + 추가환불 가능

### 필수 데이터
| 항목 | 설명 |
|------|------|
| sale | 원결제 건 |
| refunds | 해당 매출 건의 환불 이력 (totalRefunded 계산) |

### 인터랙션 (User Actions)
1. [추가 환불] → DLG-S015 환불요청 (잔여금액 기준)
2. [환불 상세보기] → DLG-S014 환불상세결과

### 비즈니스 룰
- 잔여 환불 가능금액 = sale_price - sum(refunds.amount)
- 잔여금액 = 0이면 [추가 환불] disabled → 전액환불 완료 상태
- 부분환불 횟수 제한 없음

### 에지 케이스
- 잔여금액이 정확히 0: 부분환불완료 → 환불완료 상태로 자동 전환

### 연결 화면
- DLG-S015 환불요청 (추가 환불)
- DLG-S014 환불상세결과
