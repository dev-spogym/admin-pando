## 3. 상태 고유 비주얼 델타

### 3.1 ASCII 변경점

```
  (마스터 레이아웃 그대로)
  Footer: [ 닫기 ]  [ 등록 ]   ← [등록] disabled=true 로 시작
                                 (필수값 완비 시 active)
```

### 3.2 변경 요소 표

| 요소 | 상태 | 기본값 | 비고 |
|---|---|---|---|
| `form.product_name` | 빈 문자열 `''` | placeholder "예: PT 30회" | 입력 시 disabled 해제에 기여 |
| `form.total_amount` | `0` | placeholder "0" | ≤0 이면 disabled 유지 |
| `form.total_months` | `0` (미선택) | "개월 수 선택" | 0 이면 disabled 유지 |
| `form.start_date` | `오늘(YYYY-MM-DD)` | `type="date"` | 클라이언트 로컬 타임존 |
| `selectedMember` | `null` | readonly input placeholder | 검색 버튼만 활성 |
| 월 납입액 박스 | `-` (표시만) | `form.total_months === 0` | 계산 불가 표시 |
| [닫기] | 활성 | `disabled={false}` | |
| [등록] | **disabled** | `!canSubmit` | 5가지 필수값 충족 시 활성 |

### 3.3 `canSubmit` 계산

```ts
const canSubmit =
  !!selectedMember &&
  form.product_name.trim().length > 0 &&
  form.total_amount > 0 &&
  form.total_months > 0 &&
  !!form.start_date;
```

## 6. 상태 고유 비즈니스 룰 델타

1. **BuyerSearchModal 중첩**: DLG-S002 오픈 시 본 다이얼로그는 유지(z-index: 40 → 50 중첩). 선택 후 자동 닫힘.
2. **회원 재선택**: 이미 `selectedMember` 존재 상태에서 [검색] 누르면 새 선택으로 덮어쓰기 가능.
3. **과거 날짜 허용**: `start_date`는 과거 허용(소급 등록). UI에서 min 제약 없음.
4. **입력 보존**: ESC로 닫고 재오픈 시 기본 정책은 **초기화**. (메모/임시저장 없음)

---

## 7. 에지 케이스

| 케이스 | 처리 |
|---|---|
| 회원 검색 결과 0건 | DLG-S002에서 "검색 결과 없음" 표시, 본 다이얼로그는 그대로 |
| 회원 목록 로드 실패 | DLG-S002가 에러 토스트 후 닫힘 → 본 다이얼로그 유지 |
| 모바일 키보드 오픈으로 Footer 가림 | 본문 `max-h-[70vh] overflow-auto` + Footer 고정 |
| total_amount 문자열 입력 | `type="number"` + `inputMode="numeric"` 로 한글/특수문자 차단 |
| 개월 수 변경 시 total_amount % total_months ≠ 0 | `Math.ceil` 으로 처리, 마지막 회차 보정은 서버 담당 |
| start_date 미입력(빈 문자열) | `canSubmit=false` 로 등록 차단 |
