## 1. 화면 목적 (Why)

**신규 회원이 14~30일 내 센터에 안정적으로 정착하는지**를 측정하고 이탈 위험을 조기 발견.
- 프리-온보딩 5개: 신규 리드, 연락 완료, 방문 완료, 등록 전환, 리드 전환율
- 신규 유치 3개: 이번달 신규 등록, 온라인 유입 비중(집계 중), 리드 누락률
- 신규 안정 5개: 7일 이용률, 30일 활동률, PT 체험 참여율, GX 첫 참여율, 초기 이탈 수
- 온보딩 단계별 진행률 바 (5단계)
- 이탈 위험 회원 목록 (위험/이탈위험 상태)

## 8. 비즈니스 룰

1. **온보딩 상태 판별**:
   - `daysSince ≤ 7 && visits === 0` → `미방문` (info)
   - `daysSince ≤ 7 && visits < 2` → `주의` (warning)
   - `daysSince > 7 && visits < 2` → `위험` (error)
   - `daysSince > 14 && visits < 4` → `이탈위험` (error)
   - 그 외 → `정상` (success)
2. **진행률 5단계**:
   - 가입 (100%), 초기 상담, 체성분 측정(근사 60%), 프로그램 배정(PT체험+GX참여), 첫 수업 완료(day30Rate)
3. **7일 이용률 variant**: ≥40 mint / <40 peach
4. **30일 활동률 variant**: ≥50 mint / <50 peach
5. **이탈 위험 영역**: `atRisk.length > 0` 시 빨간 배너 + 리스트
6. **리드 누락률**: `(newLeads - contacted) / newLeads * 100`
7. **신규 안정 집계 기준**: 30일 전 이후 등록된 회원 중 활동 기준
8. **새로고침**: 전 쿼리 invalidate + 60초 쿨다운
