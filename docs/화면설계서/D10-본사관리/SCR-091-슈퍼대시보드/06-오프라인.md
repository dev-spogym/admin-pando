## 상태 고유 인터랙션
1. "다시 시도": `/api/health` → 성공 시 전체 refetch + 배너 해제
2. 자동 재시도: 30초 폴링
3. `online` 이벤트: 즉시 refetch + toast
4. 사이드바 이동: 다른 페이지 자체 오프라인 처리
5. 쓰기 액션 시도: disabled + "오프라인 상태입니다" toast

## 비즈니스 룰 델타
- `navigator.onLine` 신뢰 불완전 → 실제 fetch로 확인
- 캐시 TTL: `staleTime: 60_000`, `gcTime: 600_000`
- Stale 판정: age > 60s 또는 offline=true
- 쓰기 차단: 모든 mutation disabled
- 감사 로그 기록 불가 (오프라인)
- 지점 전환 금지 (서버 세션 변경 필요)

## 에지 케이스
- Wi-Fi 연결 + 서버 다운: 배너 "서버 연결 불가" 변경
- VPN 부분 단절: `/05-에러`로 분류
- 짧은 단절(1~2초): debounce 2초 → 배너 안 띄움
- 모바일 셀룰러 ↔ Wi-Fi: online/offline flash 방지

function useOffline() {
  const [offline, setOffline] = useState(!navigator.onLine);
  useEffect(() => {
    const on = () => setOffline(false);
    const off = () => setOffline(true);
    window.addEventListener('online', on);
    window.addEventListener('offline', off);
    return () => {
      window.removeEventListener('online', on);
      window.removeEventListener('offline', off);
    };
  }, []);
  return offline;
}
