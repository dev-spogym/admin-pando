## 상태 고유 인터랙션
1. "대시보드로 이동" → `router.push('/')` → SCR-090
2. "문의" → `mailto:{supportEmail}?subject=슈퍼대시보드 접근 요청`
3. 뒤로가기: 브라우저 기본
4. URL/DOM 조작 방지: 서버가 API 레벨에서 403 강제

## 비즈니스 룰 델타
- URL 조작 대응: 클라에서 URL 바꿔도 서버가 API 403 강제
- 감사 로그: 권한 없는 접근 시 `AUDIT.FORBIDDEN_ACCESS` 기록(IP+UA+path)
- redirect 우선순위: middleware → server-component → client-guard
- 자동 리다이렉트(옵션): 5초 타이머 후 `/`
- permission upgrade 요청(옵션 P1): owner → super 권한 요청 버튼

## 에지 케이스
- **토큰 탈취 + role 조작**: jwt verify 후 DB role 재조회 이중 체크
- **primary인데 브랜드 미배정**: E403 대신 "브랜드 배정 필요" 안내(P2)
- **세션 만료**: 401 → `/login?redirect=/super-dashboard` 우선
- **권한 상승 중**: 역할 변경 후 캐시된 UI → 소프트 리로드
