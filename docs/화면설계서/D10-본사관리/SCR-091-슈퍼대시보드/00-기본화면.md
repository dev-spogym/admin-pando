## 1. 화면 목적 (Why)

본사 운영 책임자(슈퍼관리자/primary)가 **전 지점의 통합 상태**를 한 화면에서 조망하고, 지점 전환·지점 생성/폐점·감사로그 확인 등 본사 운영 액션을 수행하는 대시보드.

- 전 지점 통합 KPI 7개: 전체 회원·매출·출석·직원·신규·만료예정·만료
- 지점별 현황 카드 (지점 상태 배지 + 전환 버튼)
- 최근 감사로그 (전 지점, 전 이벤트)
- 권한 변경/지점 폐점 등 본사 전용 액션

## 8. 비즈니스 룰

1. **접근 가드**: 서버·클라이언트 모두 역할 체크. 실패 시 `04-권한없음` 또는 `/forbidden`.
2. **전월 대비(%)**: `prev > 0 ? Math.round(((curr - prev) / prev) * 100) : 0`
3. **지점 상태 배지**: `ACTIVE=🟢 운영중`, `SUSPENDED=🟡 휴업`, `CLOSED=🔴 폐점`
4. **지점 전환**: `switchBranch(id)` → `/branches/:id/switch` POST → 세션 branchContext 변경 → toast "강남점으로 전환됨" → `router.push('/')` (SCR-090 이동)
5. **지점 폐점**: 2단계 확인 다이얼로그(회원 이관/데이터 보관 정책 선택)
6. **감사로그 액션 배지**: 마스터 §3.1 색상 토큰 사용
7. **감사로그 infinite scroll**: cursor 기반, 20건씩 로드
8. **숫자 포맷**: `Intl.NumberFormat('ko-KR')` + 한국어 단위
9. **매출 포맷**: 1억+ `"X.X억"`, 1천만+ `"N,NNN만"`
10. **새로고침**: 전 쿼리 `invalidate` + 60초 쿨다운

function useSuperDashboard() {
  const statsQ = useQuery({ queryKey:['super-stats'], queryFn: fetchSuperStats, staleTime: 60_000 });
  const branchQ = useQuery({ queryKey:['branches-summary'], queryFn: fetchBranches, staleTime: 60_000 });
  const auditQ = useQuery({ queryKey:['audit-log','super'], queryFn: () => fetchAuditLogs({limit:20}), staleTime: 30_000 });
  const refetchAll = () => Promise.all([statsQ.refetch(), branchQ.refetch(), auditQ.refetch()]);
  return {
    stats: statsQ.data, branches: branchQ.data ?? [], auditLogs: auditQ.data?.items ?? [],
    isLoading: statsQ.isLoading || branchQ.isLoading,
    isRefetching: statsQ.isFetching || branchQ.isFetching,
    errors: { stats: statsQ.error, branches: branchQ.error, audit: auditQ.error },
    refetchAll,
  };
}
