## 인터랙션
| 요소 | 동작 |
|---|---|
| [다시 시도] | `queryClient.invalidateQueries(['branches'])` |
| [문의하기] | mailto/Intercom, 에러코드/요청ID 자동 첨부 |
| 부분 실패 재시도 | 해당 쿼리만 `refetch()` |
| 30초 자동 재시도 | 최대 3회 |
| 오프라인 복귀 | `window online` 이벤트 → 자동 refetch |

## 비즈니스 룰 델타
- Exponential backoff: 1s → 2s → 4s
- Sentry.captureException + 태그 {screen: 'SCR-092'}
- placeholderData = 이전 성공 응답 (stale 유지)
- 부분 실패 격리 (ErrorBoundary per widget)
- fetch 타임아웃 10초 (AbortController)

## 에지 케이스
- 401 만료: 전역 인터셉터 → `/login` (이 상태 안 옴)
- 403 권한: `06-권한없음` 전이
- 재시도 중 또 실패: 에러 유지 + toast "재시도 실패"
- 3회 연속 실패: 자동 재시도 중단

## 바이브코딩 프롬프트 (델타)
```
if (isError) {
  const errorCode = error?.errorCode || 'E500001';
  const isMaintenance = errorCode === 'E503001';
  const isOffline = error?.type === 'NETWORK';

  useEffect(() => {
    if (isMaintenance && retryCount < 3) {
      const t = setTimeout(() => { refetch(); setRetryCount(c => c+1); }, 30_000);
      return () => clearTimeout(t);
    }
  }, [isMaintenance, retryCount]);

  return (
    <div className="rounded-xl bg-rose-50 border border-rose-200 p-8 text-center">
      {isOffline ? <WifiOff/> : isMaintenance ? <Construction/> : <AlertCircle/>}
      <h2>{isOffline ? '네트워크 연결이 끊어졌습니다'
        : isMaintenance ? '서비스 점검 중입니다'
        : '지점 정보를 불러오지 못했습니다'}</h2>
      <p>{errorMessageMap[errorCode]}</p>
      <p className="font-mono text-xs">에러코드: {errorCode} · 요청 ID: {requestId}</p>
      <Button onClick={() => refetch()}><RefreshCw/> 다시 시도</Button>
      {user.role==='primary' && <Button variant="secondary" onClick={contactSupport}>문의하기</Button>}
    </div>
  );
}

useQuery({
  retry: (fc,e) => e.statusCode < 500 ? false : fc < 3,
  retryDelay: a => Math.min(1000 * 2**a, 30_000),
  placeholderData: keepPreviousData,
});

useEffect(() => {
  const h = () => refetch();
  window.addEventListener('online', h);
  return () => window.removeEventListener('online', h);
}, []);
```
