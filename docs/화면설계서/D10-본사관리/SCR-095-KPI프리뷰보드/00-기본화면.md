## 1. 화면 목적 (Why)

역할별(본사/지점/FC/트레이너/운영) 5개 보드를 **탭으로 전환**하며, 각 보드에서:
- **핵심 지표 4개** (StatCard)
- **액션 큐 4개** (Today 처리 필요)
- **랭킹 4개** (지점/팀원/회원 등)
- **퍼널 4단계** (리드→등록, 체험→재구매 등)
- **인사이트 텍스트 3개** (하드코딩 or AI 분석)

를 한 화면에서 제공하는 **종합 KPI 센터**.

모든 역할이 자신의 현재 수행 지표를 **실행 관점**으로 볼 수 있으며, super는 5개 보드 전부 열람, 지점 역할은 자신의 권역 보드만 열람한다.

## 5. 🔐 역할별(RBAC) 뷰 매트릭스

> `●` = 탭 접근+기본 탭, `○` = 탭 접근만(읽기), `—` = 탭 비표시
> 보드 기본 탭은 역할에 따라 자동 선택(`defaultBoardKey(role, isSuperAdmin)`)

| 요소 | primary/super | owner | manager | fc | trainer | staff | front | readonly |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **페이지 접근** | ● | ● | ● | ● | ● | ● | ● | ○ |

| **FC 탭 (fc)** | ● | ● | ● | ●(기본) | — | — | — | — |
| **트레이너 탭 (trainer)** | ● | ● | ● | — | ●(기본) | ●(기본) | — | — |

### 역할별 기본 탭 자동 선택 로직
```ts
function defaultBoardKey(role: Role, isSuperAdmin: boolean): BoardKey {
  if (isSuperAdmin) return 'hq';
  const n = normalizeRole(role);
  if (n === 'fc') return 'fc';
  if (['manager','owner','primary'].includes(n)) return 'branch';
  if (['trainer','staff'].includes(n)) return 'trainer';
  return 'branch';
}
```

### 서버 가드
- `GET /kpi-preview/board?key=hq`: primary/super만 허용 (다른 역할은 403 → 탭 비노출 처리로 UX 차단)
- 매출 카드 값은 `canSeeRevenue(role)` 서버 필터 (fc/trainer/staff/front에는 마스킹)

## 7. 데이터 계약

### 7.1 타입
```ts
type BoardKey = 'hq' | 'branch' | 'fc' | 'trainer' | 'operations';
type ActionLevel = 'error' | 'info' | 'default' | 'success';

interface BoardStat { id: string; label: string; value: string; description?: string; }
interface ActionItem { title: string; owner: string; due: string; level: ActionLevel; href?: string; }
interface RankingItem { label: string; value: string; delta?: string; }
interface FunnelStep { label: string; value: number; rate: number; change?: string; }

interface BoardData {
  key: BoardKey;
  label: string;
  icon: string;
  stats: BoardStat[];       // 4개
  queue: ActionItem[];      // 4개
  ranking: RankingItem[];   // 4개
  rankingTitle: string;
  moreHref: string;
  funnel: FunnelStep[];     // 4단계
  insights: string[];       // 3개
}
```

### 7.2 API
| 엔드포인트 | 파라미터 | 반환 |
|---|---|---|
| `GET /kpi-preview/board` | `{key:BoardKey, branchId?}` | `BoardData` |
| `GET /kpi-preview/boards` | — | `BoardData[]` (탭 일괄 로드, 선택) |

현재 기능명세서 기준 데이터는 **하드코딩 상수** (`boardContents`)이며, MVP에서는 API 미연동, 로컬 모듈에서 반환.

### 7.3 상태 관리
- `useState<BoardKey>(defaultBoardKey(role, isSuperAdmin))`
- `React Query` `useQuery(['kpi-preview', activeTab, branchId])` (staleTime 5분)
- 탭 전환 시 즉시 UI 전환(캐시 적중) + 백그라운드 refetch

### 7.4 멀티테넌트 / 권한
- primary/super: 5개 탭 전체 표시, 본사 탭 기본
- owner/manager: 지점 탭 기본, 본사 탭은 비표시(`visibleTabs`에서 필터)
- fc/trainer/staff: 본인 역할 탭 기본, 상위 탭 비표시

## 8. 비즈니스 룰

1. **탭 기본 선택**: `defaultBoardKey(role, isSuperAdmin)` 로 초기 `activeTab` 설정
2. **탭 필터**: 역할에 허용된 탭만 `TabNav.tabs`에 포함(보안은 서버, UX는 클라)
3. **매출 마스킹**: `canSeeRevenue(role)=false` 시 매출 관련 StatCard는 값 `"***"` + aria-label "권한 없음"
4. **액션 큐 항목 클릭**: `level` 무관, `href` 있으면 `moveToPage(href)`, 없으면 toast `"연결된 화면이 없습니다"`
5. **랭킹 "상세 보기"**: `moveToPage(moreHref)` (예: 매출 랭킹 → `/sales`, 회원 랭킹 → `/members`)
6. **퍼널 "실시간 보드" 배지**: 항상 표시(현재 하드코딩이므로 정적 라벨)
7. **인사이트 갱신**: 하드코딩. 추후 AI 생성 연동 시 `board.insightsGeneratedAt` 표시
8. **숫자 포맷**: 매출 `"X.X억"/"N,NNN만"`, 건수 `Intl.NumberFormat('ko-KR')`
9. **재진입 시**: 마지막 `activeTab` `sessionStorage.getItem('kpi_preview_last_tab')` 복원

## 11. 접근성 (WCAG 2.1 AA)
- `<main role="main">` + section `aria-label`
- `TabNav`: `role="tablist"`, 각 탭 `role="tab"` + `aria-selected`
- StatCard: `role="group"` + `aria-labelledby`
- ActionQueue 항목: `<button>` + 명확한 `aria-label` ("긴급: 재등록률 40% 이하 지점 코칭")
- 퍼널 진행률: `role="progressbar"` + `aria-valuenow/min/max`
- 인사이트: `<article>` 래핑 + 스크린리더용 헤딩
- Tab/Enter/Space 키보드 조작
- `prefers-reduced-motion`: 퍼널 애니메이션 비활성
