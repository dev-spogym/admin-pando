## 상태 진입 조건
- `data.stats.every(s => s.value === '0' || s.value === '0건' || s.value === '0명')`
- 또는 `queue.length === 0 && ranking.length === 0 && funnel.every(f => f.value === 0)`
- 신규 지점(첫 운영 30일 이내) 또는 운영 데이터 미축적

## 상태 고유 비주얼 델타

탭/레이아웃은 정상 구조 유지. 각 영역에 **빈 상태 안내**를 표시한다.

```
§A 탭: 정상 (탭 전환 가능)

§B KPI (4카드): 값은 "0" 또는 "—"로 표시
   [전사 신규]  [유지율]  [전사 매출]  [이탈위험 지점]
      0명         —         0원           0곳
   (description: "데이터 축적 중")

§C 액션 큐 / 랭킹:
   ┌──────────────────┐  ┌──────────────────┐
   │ Today Actions    │  │ 지점 성과 랭킹     │
   │ 📭               │  │ 📭               │
   │ 처리할 작업이      │  │ 집계된 데이터가    │
   │ 아직 없습니다.     │  │ 없습니다.         │
   └──────────────────┘  └──────────────────┘

§D 퍼널: 4단계 라벨 + `rate=0%` 빈 바
   "진행 데이터를 수집하면 퍼널이 표시됩니다."

§E 인사이트: 파란 배너 1개
   💡 "30일 이상 운영 데이터를 기반으로 인사이트가 생성됩니다."

상단 안내 배너:
┌─────────────────────────────────────────────────────────────┐
│ ℹ️ 데이터가 쌓이면 KPI가 자동으로 계산됩니다.                   │
│    [회원 등록하기]  [자동알림 설정]                            │
└─────────────────────────────────────────────────────────────┘
```

| 변경 요소 | Before (정상) | After (빈상태) |
|---|---|---|
| StatCard value | `428명`, `88.6%` | `0` / `—` / `0원` |
| StatCard description | `전월 대비 +12.4%` | `데이터 축적 중` |
| 액션 큐 | 4개 행 | `<EmptyState icon={Inbox} text="처리할 작업이 아직 없습니다." />` |
| 랭킹 | 4개 행 | `<EmptyState icon={BarChart3} text="집계된 데이터가 없습니다." />` |
| 퍼널 | 실제 진행률 | rate=0 바 + 텍스트 "퍼널이 표시됩니다" |
| 인사이트 | 3개 카드 | 안내 배너 1개 |
| 상단 배너 | 없음 | 파란 안내 배너 + 액션 버튼 2개 |

## 역할별 차이
- **primary/super**: 상단 배너 액션 = [회원 등록하기] [자동알림 설정] [지점 관리]
- **owner/manager**: 액션 = [회원 등록하기] [자동알림 설정]
- **fc**: 액션 = [리드 등록하기] [오늘 상담]
- **trainer**: 액션 = [수업 일정] [체험 등록]
- **staff/front/readonly**: 액션 버튼 미표시 (안내 문구만)

## 상태 고유 인터랙션
1. 탭 전환 시 각 보드가 독립적으로 빈/정상 판정 → 다른 탭은 정상일 수 있음
2. 상단 배너 액션 버튼 → 해당 페이지 이동
   - `/members/new`, `/message/auto-alarm`, `/branches`, `/leads/new`, `/today-tasks`, `/calendar`
3. 퍼널 "수집이 시작되면" 안내 클릭 → 자동알림 설정(`/message/auto-alarm`) 안내 가이드

## 상태 고유 비즈니스 룰 델타
- **판정 기준**: 모든 `stats` 값이 0/`—`인 경우에만 빈 상태. 일부만 0이면 정상.
- **신규 지점 30일 가드**: `branch.createdAt < 30d before` → 무조건 빈 상태 UI 사용(`data.length>0`이어도)
- **FC/Trainer 빈상태**: 본인 할당 회원이 없는 경우에만 본인 탭 빈상태
- **매출 마스킹 역할**: 매출 StatCard는 `***`, 빈상태 판정에서 제외
- **새로고침**: 24시간 간격으로 자동 polling → 데이터 수집되면 `02-정상` 전환

## 에지 케이스
- **일부 보드만 빈**: super가 hq=정상, operations=빈 → 탭별 UI 분리
- **데이터 1~2개만 존재**: 빈 상태로 판정하지 않음(정상으로 표시, 값만 작게)
- **캐시된 과거 데이터 있음**: 현재 보드 0이어도 최근 7일 캐시 표시 + "최신 데이터 없음" 서브 배너
