## 1. 화면 목적 (Why)

**시스템 내 모든 중요 활동(로그인/수정/삭제/권한변경/지점전환/환불/내보내기 등)을 시간순 조회**하여 보안·감사·책임 추적.
- tenantId 기준 격리, primary만 전 지점/전 사용자
- 필터: 기간(기본 30일)·액션·대상유형·사용자명·지점(primary)
- 페이지네이션 20건/페이지, CSV 내보내기
- 21종 액션 배지 색상 매핑

## 8. 비즈니스 룰

1. **기본 필터**: 최근 30일 (`fromDate = today - 30days`)
2. **PAGE_SIZE**: 20건, 서버 페이지네이션
3. **tenantId 필수**: 서버에서 jwt 기반 자동 주입
4. **non-super 범위**: `branchId = user.branchId` 서버 강제
5. **액션 배지**: 21종 매핑, 알 수 없는 액션은 gray default
6. **상세(detail)**: beforeValue/afterValue 있으면 `{field}: {before} → {after}` 포맷, 없으면 detail 키-값 요약
7. **IP 마스킹**: owner/manager에게는 마지막 옥텟 `.***`
8. **CSV export**: audit_log에 `EXPORT` 액션 자체 기록
9. **실시간 업데이트**: 폴링 없음, 수동 새로고침
10. **사용자 검색**: 서버 `userName ILIKE %keyword%`
11. **시간 포맷**: `yyyy-MM-dd HH:mm:ss` KST

function ActionBadge({ action }) {
  const cls = ACTION_BADGE[action] ?? 'bg-gray-100 text-gray-600 border-gray-200';
  const label = ACTION_LABELS[action] ?? action;
  return (
    <span aria-label={label}

      {label}
    </span>

function AuditDetailCell({ entry }) {
  if (entry.beforeValue && entry.afterValue) {
    const changed = Object.keys(entry.afterValue)
      .filter(k => entry.beforeValue[k] !== entry.afterValue[k]);
    return (
      <div className="text-xs">
        {changed.slice(0,2).map(k => (
          <div key={k}>
            <span className="text-gray-500">{k}:</span>
            <span className="ml-1 line-through text-red-500">{String(entry.beforeValue[k])}</span>
            <span className="mx-1 text-gray-400">→</span>
            <span className="text-emerald-600">{String(entry.afterValue[k])}</span>
          </div>
        ))}
        {changed.length > 2 && <span className="text-gray-400">외 {changed.length-2}건</span>}
      </div>
    );
  }
  if (entry.detail) {
    return <span className="text-xs text-gray-600 truncate max-w-xs">{summarizeDetail(entry.detail)}</span>;
  }
  return <span className="text-xs text-gray-400">-</span>;
}
