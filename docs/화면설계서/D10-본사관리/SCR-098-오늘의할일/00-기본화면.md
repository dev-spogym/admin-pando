## 1. 화면 목적 (Why)

**오늘 처리해야 할 업무를 역할/지점별로 자동 배정하고 상태를 추적**하여 업무 누락을 방지.
- 자동 생성: TASK_POOL 100개 → `seededShuffle(`${dateKey}-${branchName}-${taskRole}`)` → 역할별 12~18건
- 수동 추가: primary만 가능 (DLG-098-001)
- 상태 전환: 대기 → 진행중 → 완료
- 통계: 전체/대기/진행중/완료 4개 KPI
- 탭 필터, 카테고리/우선순위/검색 필터

### 5.1 배정 규칙
```ts
function roleToTaskRole(role, isSuperAdmin): 'hq'|'branch'|'fc'|'trainer'|'staff' {
  if (isSuperAdmin) return 'hq';
  switch (role) {
    case 'primary': case 'owner': case 'manager': return 'branch';
    case 'trainer': return 'trainer';
    case 'staff': case 'front': return 'staff';
    case 'fc': return 'fc';
    default: return 'branch';
  }
}
```

### 7.3 배정 알고리즘
```ts
function seededShuffle<T>(items: T[], seed: string): T[] { /* hashSeed + LCG */ }
const shuffled = seededShuffle(TASK_POOL, `${dateKey}-${branchName}-${taskRole}`);
const filtered = shuffled.filter(t =>
  (taskRole === 'hq' || t.assigneeRole === taskRole || t.assigneeRole === 'branch') &&
  (branchName === '전체 지점' || t.branchName === branchName));
return filtered.slice(0, taskRole === 'hq' ? 18 : 12);
```

## 8. 비즈니스 룰

1. **날짜 시드**: `dateKey = YYYY-MM-DD` → 당일 내 결정적 같은 배정
2. **배정 크기**: hq=18, 그 외=12
3. **태스크 소스**:
   - `auto`: TASK_POOL에서 배정
   - `hq`: primary가 추가한 커스텀 태스크 (localStorage)
4. **가시성**: primary는 모든 hq 태스크, 그 외는 본인 지점+역할 매칭만
5. **상태 전환**: 체크박스 클릭 시 `대기 → 완료` 즉시, 상세 수정(DLG-098-002)에서 `대기 ↔ 진행중 ↔ 완료`
6. **수동 입력 필드**: primary만 가능
7. **우선순위 배지**: 긴급(error), 높음(info), 보통(default)
8. **상태 배지**: 완료(success), 진행중(info), 대기(default)
9. **탭 카운트**: 동적 계산 (필터와 무관)
10. **audit_log**: 수동 추가/수정/삭제 시 `CREATE`/`UPDATE`/`DELETE` 액션 기록

```
const handleStatusChange = (id, newStatus) => {
  if (id.startsWith('auto-')) {
    // auto 태스크는 customTasks로 복제 후 관리
    const orig = autoTasks.find(t => t.id === id);
    if (!orig) return;
    setCustomTasks(prev => [...prev, { ...orig, status: newStatus, source: 'auto', id: `custom-${Date.now()}` }]);
```
