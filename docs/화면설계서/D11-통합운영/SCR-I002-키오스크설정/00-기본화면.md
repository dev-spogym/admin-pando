지점 키오스크 장비의 **연결 상태·동작 설정·출석 정책**을 관리.

## 7. 데이터 계약

### 7.1 타입
```ts
interface KioskConfig {
  branchId: number;
  device: { id: string; status: 'online'|'reconnecting'|'offline'; lastPing: string; signalStrength: number };
  modes: Array<'rfid'|'qr'|'face'>;
  display: { logoUrl?: string; bgImageUrl?: string; welcomeText: string; screensaverMin: number };
  security: { afterHoursMode: 'locked'|'staffOnly'|'open'; adminPin: string };
  updatedAt: string;
  updatedBy: string;
}
```

### 7.2 API
| 메서드 | 경로 | 용도 |
|---|---|---|
| GET | `/kiosk/config?branchId=` | 조회 |
| PUT | `/kiosk/config` | 저장 (owner 이상) |
| POST | `/kiosk/test` | 연결 테스트 |
| POST | `/kiosk/reboot` | 재부팅 (관리자 PIN 확인 후) |
| WS | `/ws/kiosk?branchId=` | 실시간 장비 상태 스트림 |

### 7.3 스키마 (Zod)
```ts
const kioskSchema = z.object({
  modes: z.array(z.enum(['rfid','qr','face'])).min(1, '최소 1개 모드 선택'),
  display: z.object({
    welcomeText: z.string().max(50),
    screensaverMin: z.number().min(1).max(60),
  }),
  security: z.object({
    afterHoursMode: z.enum(['locked','staffOnly','open']),
    adminPin: z.string().regex(/^\d{4,6}$/, 'PIN은 4~6자리 숫자'),
  }),
});
```

## 8. 비즈니스 룰
1. 장비 미연결 시 저장 가능하되 **"변경사항은 장비 연결 후 적용됩니다" 배너** 표시 (03-미연결경고 상태)
2. 관리자 PIN 변경 시 현재 PIN 재확인 프롬프트
3. 출석 모드 중 `face`는 P2 (UI는 disabled + "준비 중")
4. 근무 외 시간 `open` 선택 시 경고 다이얼로그 (보안 위험 안내)
5. 저장 시 감사로그 `KIOSK_CONFIG_UPDATE` 기록
6. 멀티테넌트: `branchId` 서버 jwt에서 강제, URL 조작 시 403
