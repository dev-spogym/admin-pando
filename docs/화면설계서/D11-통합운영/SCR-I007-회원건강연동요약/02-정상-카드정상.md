## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 + 5개 카드 모두 데이터 존재
- HC 연동 + InBody 측정 + 락커 배정 + 이용권 + 출석 모두 정상

### 필수 데이터
| 카드 | 테이블 | 조건 |
|------|--------|------|
| 체성분 | `bodyComposition` | `member_id=?`, 최신 1건 |
| 출석 | `attendance` | `member_id=?`, 3개월 |
| 락커 | `clothing` JOIN `lockers` | `member_id=?`, active |
| HC | `health_data` (가상) | `member_id=?` |
| 이용권 | `contracts` | `member_id=?`, 서명완료 |

### 인터랙션 (User Actions)
1. "전체 보기" (체성분) → `/body-composition?memberId=[id]`
2. "배정 변경" (락커) → DLG-I002
3. HC 연동 카드 → 상세 동기화 정보 (향후)
4. 이용권 카드 → 이용권 상세 (향후)

### 비즈니스 룰
- 카드별 독립 에러 처리 (한 카드 실패해도 다른 카드 정상 표시)
- staff 역할: 모든 카드 읽기 가능, 배정 변경 불가
- HC 연동: 외부 API 의존 — timeout 5초, 실패 시 미연결 카드로 표시

### 에지 케이스
- HC 미연결 → `04-HC미연결` 카드
- InBody 미측정 → `05-InBody미측정` 카드
- 락커 미배정 → `06-락커미배정` 카드
- 이용권 없음 → 이용권 카드 "이용권 없음" 표시

### 연결 화면
- 이전: `01-로딩`
- 다음: 각 카드별 부분 에러 상태 (`04`~`06`) / 전체 에러 `03`
