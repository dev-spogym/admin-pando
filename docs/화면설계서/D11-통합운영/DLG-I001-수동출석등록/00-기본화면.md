# DLG-I001 수동 출석 등록 — 기본화면 (마스터)

> 이 문서는 **다이얼로그 마스터 스펙**. 상태 델타 파일은 이 문서를 상속.
> QR/NFC 스캔 실패 시 프론트/스태프가 수동으로 출석을 등록하는 모달.

## 0. 메타
| 항목 | 값 |
|---|---|
| 화면 ID | DLG-I001 |
| 화면명 | 수동 출석 등록 |
| 도메인 | D11-통합운영 |
| 트리거 | SCR-I001 출석 관리 → "수동 등록" 버튼 |
| 파일 | `src/components/dialogs/ManualAttendanceDialog.tsx` |
| 역할 | super/primary/owner/manager/staff ● / front ● / fc ○ / trainer — |
| 우선순위 | P0 |

## 1. 목적
QR·NFC 스캔 불가 시 회원을 검색하여 출석을 수동 등록. 사유 기록 필수.

## 5. 컴포넌트 트리
```
<Dialog open={open} onClose={onClose}>
  <DialogHeader title="수동 출석 등록" onClose={onClose} />
  <MemberSearch onSelect={setMember} />
  {member && (
    <>
      <SelectedMemberCard member={member} />
      <AttendanceTypeRadio value={type} onChange={setType} />
      <ReasonInput value={reason} onChange={setReason} required />
    </>
  )}
  <DialogFooter>
    <CancelButton onClick={onClose} />
    <SubmitButton onClick={submit} disabled={!member || !reason} loading={isPending} />
  </DialogFooter>
</Dialog>
```

## 6. 데이터 계약
```ts
interface ManualAttendanceInput {
  memberId: number;
  type: 'enter' | 'exit';
  reason: string;
  branchId: number;
  staffId: number;
}
```

### API
| 메서드 | 경로 |
|---|---|
| POST | `/attendance/manual` |
| GET | `/members/search?q=&branchId=` |

## 7. 비즈니스 룰
1. 사유 입력 필수 (빈 값 제출 불가)
2. 이미 입장 상태 → 퇴장만 선택 가능 (반대도 동일)
3. 등록 후 SCR-I001 출석 목록 자동 갱신
4. 담당자 `staffId` 자동 기록

## 10. 🧩 바이브코딩 프롬프트
```
Next.js 15 + TS + Tailwind + React Query

파일: src/components/dialogs/ManualAttendanceDialog.tsx

<Dialog open={open} onOpenChange={onClose}>
  <DialogContent className="max-w-md" role="dialog" aria-modal="true">
    <DialogHeader>
      <DialogTitle id="dialog-title">수동 출석 등록</DialogTitle>
    </DialogHeader>
    <div className="space-y-4 py-2">
      <MemberSearchInput onSelect={setMember} branchId={branchId} />
      {member && (
        <>
          <div className="rounded-lg bg-blue-50 p-3 text-sm">
            <span className="font-medium">{member.name}</span>
            <span className="text-gray-500 ml-2">#{member.id}</span>
          </div>
          <RadioGroup value={type} onValueChange={v=>setType(v as 'enter'|'exit')}>
            <div className="flex gap-4">
              <RadioGroupItem value="enter" label="입장" />
              <RadioGroupItem value="exit" label="퇴장" />
            </div>
          </RadioGroup>
          <div>
            <label className="text-sm font-medium">등록 사유 *</label>
            <input value={reason} onChange={e=>setReason(e.target.value)}
              aria-required="true"
              placeholder="QR 스캔 불가 등"
              className="mt-1 h-10 w-full rounded-lg border border-gray-300 px-3 text-sm" />
          </div>
        </>
      )}
    </div>
    <DialogFooter>
      <button onClick={onClose} className="h-10 px-6 rounded-lg border border-gray-300 text-sm">취소</button>
      <button onClick={submit} disabled={!member || !reason || isPending}
        className="h-10 px-6 rounded-lg bg-blue-600 text-white text-sm font-medium disabled:opacity-50">
        {isPending ? '등록 중...' : '등록'}
      </button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```
