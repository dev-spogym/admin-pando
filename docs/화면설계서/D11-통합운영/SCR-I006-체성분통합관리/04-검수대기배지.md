## 📝 디스크립션

### 사용 시점 (Trigger)
- `bodyComposition.review_status = 'pending'` 데이터 존재
- 파일 업로드/InBody 자동 수신 후 검수 필요 상태

### 필수 데이터
- 검수대기 목록: `review_status = 'pending'`
- 검수자 정보: useAuthStore().userId

### 인터랙션 (User Actions)
1. "검수 목록 보기" → 필터 'pending' 자동 선택
2. "검수" 버튼 → 검수 모달
3. "검수 완료" → PUT approve + KPI 갱신
4. "삭제" → 오류 데이터 삭제 확인 모달

### 비즈니스 룰
- 검수 완료: `review_status = 'approved'`, `reviewed_by`, `reviewed_at` 기록
- 검수 권한: `bodyComposition:review` (manager, fc, owner 이상)
- 수기 입력은 자동 검수완료 (review_status = 'auto_approved')
- InBody/파일 파싱 데이터: pending 상태로 저장 → 검수 필요

### 에지 케이스
- 전체 검수 완료: 배너 제거 + KPI amber → gray 복귀
- 검수 중 추가 파싱 데이터 수신: pendingCount 증가

### 연결 화면
- 이전: `02-정상`
- 다음: 검수 완료 → `02-정상` (배지 유지 또는 제거)
