## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 + `bodyComposition.length > 0`
- 검수대기·InBody오류 없는 정상 상태

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 측정 결과 | `bodyComposition` JOIN `members` | `branch_id=?`, 필터 |
| InBody 상태 | `iot_devices` | `type='inbody'` |

### 인터랙션 (User Actions)
1. "수기 등록" → DLG-I003
2. "파일 업로드" / 드래그앤드롭 → `07-파싱중` 상태
3. "검수" 버튼 → 검수 모달
4. "상세" → 회원 체성분 상세 페이지

### 비즈니스 룰
- 검수대기 N건 있으면 KPI + 테이블 배지 표시 → `04-검수대기배지`
- InBody 연동 오류 시 → `05-InBody오류배지`
- `bodyComposition:write` 없으면 수기 등록/파일 업로드 버튼 숨김
