## [SCR-S001] 매출 현황

### 1. 화면 목적
센터의 전체 매출 거래를 조회, 분석하는 메인 매출 대시보드. 기간별/상품별/결제수단별/담당자별 집계와 환불/미납 내역을 통합 관리한다.

### 2. 접근 경로
사이드바 > 매출관리 > 매출 현황 (`/sales`)

### 3. 화면 구성

```
+------------------------------------------------------------------+
| PageHeader: "매출 현황"                    [신규 결제(POS)] [엑셀] |
+------------------------------------------------------------------+
| StatCardGrid (4열)                                                |
| [순 매출] [카드 결제] [현금 결제] [미수금]                          |
+------------------------------------------------------------------+
| 기간 프리셋: [오늘] [이번주] [이번달]                               |
| SearchFilter: 검색창 + 날짜범위 + 유형 멀티셀렉트 + 상태 멀티셀렉트 |
+------------------------------------------------------------------+
| 매출유형 필터 (TAB-001 전용):                                      |
| [전체] [신규] [재등록] [휴면복귀] [종목추가] [업그레이드]            |
+------------------------------------------------------------------+
| TabNav: [매출내역] [기간별매출] [상품별내역] [결제수단별] [담당자별]  |
|         [환불내역(N)] [미납내역(N)]                                 |
+------------------------------------------------------------------+
| DataTable (탭별 컬럼 변경)                                         |
+------------------------------------------------------------------+
| 하단 요약바: 총매출 | 순매출 | 현금 | 카드 | 마일리지 |             |
|             환불금액 | 미납금 | 할인합계                            |
+------------------------------------------------------------------+
```

### 4. 통계 카드

| 카드명 | 값 | variant | 아이콘 | 데이터 규칙 |
|--------|-----|---------|--------|-------------|
| 순 매출 | `formatKRW(netTotal)` | peach | DollarSign | 총 매출 - 환불 합계. `filteredData`에서 status!='환불'인 salePrice 합 - status=='환불'인 salePrice 합 |
| 카드 결제 | `formatKRW(summary.card)` | default | CreditCard | filteredData의 card 필드 합계 |
| 현금 결제 | `formatKRW(summary.cash)` | default | Wallet | filteredData의 cash 필드 합계 |
| 미수금 | `formatKRW(summary.unpaid)` | default | AlertCircle | filteredData의 unpaid 필드 합계. unpaid > 0이면 `border-state-error/20` 스타일 적용 |

## [SCR-S002] POS 판매

### 1. 화면 목적
카테고리별 상품을 그리드 형태로 표시하고 장바구니에 담아 결제 페이지로 이동하는 POS 인터페이스.

### 2. 접근 경로
사이드바 > 매출관리 > POS 판매 (`/pos`)
또는 매출 현황 > "신규 결제 (POS)" 버튼

### 3. 화면 구성

```
+------------------------------------------------------------------+
| PageHeader: "POS 판매"                           [+ 상품 등록]    |
+------------------------------------------------------------------+
| 좌측 (flex-1)                    | 우측 (360px)                   |
| +------------------------------+ | +----------------------------+ |
| | TabNav: [이용권(N)] [PT(N)]  | | | 장바구니 헤더         [전체삭제]|
| |         [GX(N)] [기타(N)]    | | | 구매자: [회원 검색]          | |
| | [검색: 상품명 검색...]       | | |----------------------------| |
| +------------------------------+ | | 장바구니 아이템 목록         | |
| | 상품 그리드 (2~4열)          | | | [−] 수량 [+]               | |
| | +------+ +------+ +------+  | | | [현금] [카드]   금액        | |
| | | 카드 | | 카드 | | 카드 |  | | |                            | |
| | | 타입 | | 타입 | | 타입 |  | | | --- 또는 빈 장바구니 ---   | |
| | | 이름 | | 이름 | | 이름 |  | | |----------------------------| |
| | | 기간 | | 기간 | | 기간 |  | | | 총 합계: N원               | |
| | | 현금가| | 현금가| | 현금가|  | | | [결제하기]                 | |
| | | 카드가| | 카드가| | 카드가|  | | +----------------------------+ |
| | +------+ +------+ +------+  | |                                |
| +------------------------------+ +--------------------------------+
```

### 4. 통계 카드
없음

### 5. 테이블 컬럼 정의
해당 없음 (그리드 카드 방식)

**상품 카드 구성 요소:**

| 요소 | 데이터 규칙 |
|------|-------------|
| 상품 타입 배지 | MEMBERSHIP→회원권(blue), LESSON→수강권(green), RENTAL→대여권(orange), GENERAL→일반(gray) |
| 재고 배지 | stock===0: 품절(error), 0<stock<=5: 재고부족(warning, N개), stock>5: 재고 N개(success) |
| 상품명 | 14px semibold, 1줄 말줄임 |
| 기간 | Calendar 아이콘 + `기간: {period}` |
| 횟수 | Clock 아이콘 + `횟수: {count}` |
| 현금가 | 14px bold primary `formatNumber(cashPrice)원` |
| 카드가 | 12px medium secondary `formatNumber(cardPrice)원` |

**장바구니 아이템 구성:**

| 요소 | 동작 |
|------|------|
| 상품명 | 13px semibold, 1줄 말줄임 |
| X 삭제 버튼 | 해당 아이템 장바구니에서 제거 |
| 수량 조절 (−/+) | 최소 1, handleQuantityChange(cartId, delta) |
| 가격타입 토글 | [현금] [카드] 선택 - 선택된 쪽 primary 배경 |
| 소계 | `formatNumber((priceType=cash?cashPrice:cardPrice) * quantity)원` |

### 6. 필터 옵션

| 필터명 | 타입 | 옵션 목록 |
|--------|------|-----------|
| 카테고리 탭 | TabNav | 이용권, PT, GX, 기타 (각각 상품 수 표시) |
| 상품명 검색 | text | 실시간 필터 (toLowerCase 매칭) |

### 7. 버튼 -> 동작 매핑

| 버튼명 | 위치 | 동작 |
|--------|------|------|
| + 상품 등록 | PageHeader 우측 | `moveToPage(997)` - 상품 등록(new) 페이지 이동 |
| 상품 카드 클릭 | 상품 그리드 | 장바구니에 추가 (품절 시 클릭 불가). 동일 상품 존재시 수량+1 |
| 전체 삭제 | 장바구니 헤더 | `setCart([])` - 장바구니 비우기 (카트 1개 이상일 때만 표시) |
| [회원 검색] | 구매자 영역 | 구매자 검색 모달 오픈 |
| X (구매자 해제) | 구매자 영역 | `setBuyer(null)` |
| − (수량 감소) | 장바구니 아이템 | 수량 -1 (최소 1) |
| + (수량 증가) | 장바구니 아이템 | 수량 +1 |
| 현금/카드 토글 | 장바구니 아이템 | priceType 전환 (cash/card) |
| X (아이템 삭제) | 장바구니 아이템 | 해당 아이템 제거 |
| 결제하기 | 장바구니 하단 | sessionStorage에 `posCart`, `posBuyer` 저장 후 `moveToPage(982)`. 카트 비어있으면 disabled |

### 8. 상태별 화면

| 상태 | 화면 |
|------|------|
| 로딩 | 없음 (useEffect에서 직접 로드) |
| 빈 데이터 (카테고리) | ShoppingCart 아이콘 + "해당 카테고리에 상품이 없습니다." |
| 빈 장바구니 | ShoppingCart 아이콘 + "장바구니가 비어 있습니다." (opacity-50) |
| 품절 상품 | opacity-50, grayscale, cursor-not-allowed, 클릭 불가 |
| 에러 | 별도 에러 UI 없음 (fetch 실패시 빈 목록) |
