## [SCR-S001] 매출 현황

### 1. 화면 목적
센터의 전체 매출 거래를 조회, 분석하는 메인 매출 대시보드. 기간별/상품별/결제수단별/담당자별 집계와 환불/미납 내역을 통합 관리한다. 매출 추세를 한눈에 파악하고 신규 결제, 엑셀 내보내기 등 주요 매출 액션의 진입점 역할을 한다.

### 2. 접근 경로
- 사이드바 > 매출 > 매출 현황 (`/sales`)
- 내부 pageId: `970`

### 3. 권한별 접근

| 역할 | 접근 | 제한 사항 |
|------|------|-----------|
| 슈퍼관리자 | 전체 접근 | 없음 |
| 센터장/매니저 | 전체 접근 | 없음 |
| 트레이너 | 조회만 | 본인 담당 매출만 조회 (담당자 필터 자동 적용) |
| 프론트 | 조회만 | POS 판매 버튼만 활성화 |

## [SCR-S002] POS 판매

### 1. 화면 목적
카테고리별 상품을 그리드 형태로 표시하고 장바구니에 담아 결제 페이지로 이동하는 POS 인터페이스. 피트니스 센터 프론트 데스크에서 빠른 판매를 위한 터치 친화적 UI를 제공한다.

### 2. 접근 경로
- 사이드바 > 매출 > POS 결제 (`/pos`)
- 또는 매출 현황 > "신규 결제 (POS)" 버튼
- 내부 pageId: `971`

### 3. 권한별 접근

| 역할 | 접근 | 제한 사항 |
|------|------|-----------|
| 슈퍼관리자 | 전체 접근 | 없음 |
| 센터장/매니저 | 전체 접근 | 없음 |
| 트레이너 | 접근 가능 | 상품 등록 버튼 숨김 |

### [DLG-S013] 환불 처리 모달 🆕

#### 1. 트리거
결제 취소/부분환불 > 액션 열 > "처리" 클릭

#### 2. 크기
- 너비: 520px (md)

#### 3. 필드 테이블

| 필드명 | 타입 | 필수 | 옵션 | 유효성 | 에러메시지 |
|--------|------|------|------|--------|------------|
| 원 결제 정보 | display | - | 상품명, 결제일, 결제금액, 결제수단 표시 | - | - |
| 환불 유형 | radio | Y | 전체 취소, 부분 환불 | - | - |
| 환불 금액 | number | Y (부분환불시) | 기본값: 원 결제금액 | > 0, <= 원결제금액 | "환불 금액이 원 결제금액을 초과합니다." |
| 환불 방법 | Select | Y | 원래 결제수단, 현금, 계좌이체 | - | - |
| 환불 계좌 (계좌이체시) | text | 계좌이체시 Y | 은행명 + 계좌번호 | - | "환불 계좌를 입력하세요." |
| 환불 사유 | Select + text | Y | 단순변심, 서비스불만, 결제오류, 회원요청, 기타(직접입력) | 빈값 불가 | "환불 사유를 입력하세요." |
| 처리 메모 | textarea | N | rows=3 | - | - |

#### 4. 버튼

| 버튼명 | 동작 | 성공 토스트 | 실패 시 동작 |
|--------|------|-------------|-------------|
| 취소 | 모달 닫기 | - | - |
| 환불 처리 | 권한 확인 -> refunds INSERT + sales status UPDATE + 마일리지 반환(해당시). 10만원 이상이면 승인 요청 | `환불이 처리되었습니다.` | `toast.error('환불 처리에 실패했습니다.')` |

**환불 처리 로직:**
1. 권한 체크 (금액별 권한 레벨 확인)
2. 10만원 이상: 승인 요청 (status='pending_approval') -> 슈퍼관리자 알림
3. 10만원 미만 또는 승인 완료: refunds 테이블 INSERT
4. sales 테이블 status='REFUNDED' (전체취소) 또는 partialRefund 금액 기록 (부분환불)
5. 마일리지 결제였으면 마일리지 반환 (accruePoints)
6. 카드 결제였으면 PG 취소 요청 (해당시)

#### 5. 닫기 조건
- 취소 버튼, X 버튼, ESC
