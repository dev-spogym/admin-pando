## SCR-090 본사 대시보드

### 1. 화면 목적
지점 단위 운영 현황 한눈에 보기. 회원 통계, 매출, 출석, 만료예정, 생일자, 미수금, 홀딩, 감사 로그를 종합 표시한다. 로그인 후 첫 화면.

### 2. 접근 경로
`로그인 후 자동 이동` (`/`)

### 3. 접근 권한
전 역할 접근 가능 (읽기 전용)

### 4. ASCII 와이어프레임

```
┌──────────────────────────────────────────────────────────┐
│  PageHeader: "{branchName} 대시보드"       [새로고침]     │
├──────────────────────────────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐          │
│  │전체   │ │활성   │ │만료   │ │오늘   │ │이번달 │          │
│  │회원   │ │회원   │ │예정   │ │출석   │ │매출   │          │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘          │
├──────────────────────────────────────────────────────────┤
│  ┌───────────────┐ ┌───────────────┐                    │
│  │ 성별 분포 차트  │ │ 연령대 분포    │                    │
│  └───────────────┘ └───────────────┘                    │
│  ┌───────────────┐ ┌───────────────┐                    │
│  │ 월별 매출 추이  │ │ 주간 출석 현황  │                    │
│  └───────────────┘ └───────────────┘                    │
├──────────────────────────────────────────────────────────┤
│  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐               │
│  │생일자  │ │미수금  │ │홀딩   │ │만료예정│               │
│  │회원   │ │회원   │ │회원   │ │회원   │               │
│  └───────┘ └───────┘ └───────┘ └───────┘               │
│  ┌───────────────────────────────────────┐              │
│  │ 최근 활동 로그                          │              │
│  └───────────────────────────────────────┘              │
└──────────────────────────────────────────────────────────┘
```

### 5. 상단 통계 카드 (5열)

| 카드 | 값 | 아이콘 | variant |
|------|-----|--------|---------|
| 전체 회원 | `${totalMembers}명` | Users | default |
| 활성 회원 | `${activeMembers}명` | UserCheck | mint |
| 만료 예정 | `${expiringCount}명` | Clock | - |
| 오늘 출석 | `${todayAttendance}명` | UserMinus | - |
| 이번달 매출 | `${formatAmount(monthlyRevenue)}원` | DollarSign | peach |

### 6. 차트 위젯

| 위젯 | 타입 | 데이터 |
|------|------|--------|
| 성별 분포 | 도넛 차트 (div 기반) | male/female 비율 |
| 연령대 분포 | 막대 차트 | 10대~60대+ 6구간 |
| 월별 매출 추이 | 꺾은선 차트 | 최근 6개월 |
| 주간 출석 현황 | 막대 차트 | 요일별 출석 수 |

### 7. 하단 위젯 (4열)

| 위젯 | 내용 | 클릭 동작 |
|------|------|----------|
| 생일자 회원 | 오늘 생일인 회원 목록 | 회원 상세 이동 |
| 미수금 회원 | 미수금 있는 회원 (금액/상품/연체일수) | 회원 상세 이동 |
| 홀딩 회원 | 이용권 일시정지 회원 (기간/잔여일) | 회원 상세 이동 |
| 만료예정 회원 | 30일 이내 만료 회원 (D-Day) | 회원 상세 이동 |

### 8. API 호출 (병렬)

| 데이터 | 테이블 | 쿼리 |
|--------|--------|------|
| 전체 회원 수 | members | `.select('id', { count: 'exact', head: true }).eq('branchId')` |
| 활성 회원 | members | `.eq('status', 'ACTIVE')` |
| 만료 예정 | members | `.eq('status', 'ACTIVE').gte('membershipExpiry', today).lte('membershipExpiry', in30days)` |
| 오늘 출석 | attendance | `.gte('checkInAt', todayStart).lte('checkInAt', todayEnd)` |
| 월 매출 | sales | `.eq('status', 'COMPLETED').gte('saleDate', monthStart)` |

### 9. 비즈니스 로직

- **일일 동기화**: `runDailySync()` — 만료 이용권 회원 상태를 EXPIRED로 자동 변경
- **매출 포맷**: 1억 이상→`"X.X억"`, 1만 이상→`"X만"`, 나머지→천 단위 콤마

## SCR-091 슈퍼 대시보드

### 2. 접근 경로
`본사 관리 > 통합 대시보드` (`/super-dashboard`)

### 3. 접근 권한
`isSuperAdmin=true` 전용. false 시 `"이 페이지는 슈퍼관리자 전용 메뉴입니다."` + 대시보드 이동 버튼.

│  │전체   │ │전체   │ │전체   │ │전체   │                   │

### 5. 7개 KPI 카드

| # | label | value | change 계산 | changeLabel |
|---|-------|-------|-------------|-------------|
| 1 | 전체 회원 | `${formatNumber(totalMembers)}명` | MoM(전월) | 전월 대비 |
| 2 | 전체 매출 | `${formatAmount(totalRevenue)}원` | MoM(전월) | 이번달 전월 대비 |
| 3 | 전체 출석 | `${formatNumber(todayAttendance)}명` | DoD(어제) | 어제 대비 |
| 4 | 전체 직원 | `${formatNumber(totalStaff)}명` | 0 | 전월 대비 |
| 5 | 이번달 신규 | `${formatNumber(newMembers)}명` | 0 | 이번달 |
| 6 | 만료 예정 | `${formatNumber(expiringMembers)}명` | 0 | 30일 이내 |
| 7 | 만료 회원 | `${formatNumber(expiredMembers)}명` | 0 | 이번달 |

**MoM 계산:** `prev > 0 ? Math.round(((curr - prev) / prev) * 100) : 0`

**변동 표시:** 양수 → ArrowUpRight green `"+N%"`, 음수 → ArrowDownRight red `"-N%"`

## SCR-092 지점 관리

### 1. 화면 목적
전체 지점의 목록 조회, 신규 등록, 상태 변경(임시휴업/폐점), 지점 간 회원 이동을 3개 탭으로 관리한다.

### 2. 접근 경로
`본사 관리 > 지점 관리` (`/branches`)

### 3. ASCII 와이어프레임

```
┌──────────────────────────────────────────────────────────┐
│  PageHeader: "지점 관리 (멀티지점)"                        │
│  [지점 이동 신청] [+ 신규 지점 등록]                       │
├──────────────────────────────────────────────────────────┤
│  [지점 목록]  [통합 현황]  [지점 간 이동]                   │
├──────────────────────────────────────────────────────────┤
│  [총 지점 {N}] [총 회원 {N}] [이번달 매출 {N}] [활성 {N}] │
│  [🔍 지점명/주소 검색] [상태 ▼]                           │
├──────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────┐    │
│  │No│지점명│코드│주소│연락처│회원수│직원수│상태│메뉴 │    │
│  └──────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────┘
```

### 4. 3개 탭

| 탭 | 내용 |
|----|------|
| 지점 목록 | 지점 현황 테이블 (등록/비활성/상태변경/전환) |
| 통합 현황 | 지점별 매출/회원/출석률 비교 (월간/주간/연간) |
| 지점 간 이동 | 회원 지점 이동 이력 테이블 |

### 5. 신규 지점 등록 모달 (DLG-092-001)

| 필드 | 타입 | 필수 | 검증 |
|------|------|:---:|------|
| 지점명 | Input(text) | * | 2~30자 |
| 지점 코드 | Input(text) + [자동생성] | * | 영문 대문자 2~5자, 중복 불가 |
| 연락처 | Input(text) | * | `0XX-XXXX-XXXX` 형식 |
| 주소 | Input(text) | * | 필수 |
| 상세주소 | Input(text) | - | - |
| 지점 관리자 | Select | * | 직원 목록 |
| 오픈/마감 시간 | Input(time) | - | 기본: 06:00~23:30 |
| 수용 인원 | Input(text) | - | - |
| 메모 | Textarea | - | - |

**지점 코드 자동생성:** `SG${(branches.length + 1).toString().padStart(3, '0')}`

### 6. 지점 상태 변경

| 동작 | 확인 내용 |
|------|----------|
| 임시휴업 | 단순 확인 → `status: 'inactive'` |
| 폐점 | 활성 회원 수 + 재직 직원 수 조회 → 영향 범위 표시 → 최종 확인 |
| 비활성화 | 2단계 확인 (영향 범위 → 최종) |

### 7. 토스트 메시지

| 트리거 | 유형 | 메시지 |
|--------|------|--------|
| 지점 등록 성공 | success | `"지점 \"${name}\"이(가) 등록되었습니다."` |
| 지점 전환 | success | `"${name} 지점으로 전환되었습니다"` |
| 상태 변경 성공 | success | `"\"${name}\" 지점이 ${label} 처리되었습니다."` |

## SCR-094 KPI 대시보드

### 1. 화면 목적
회사 성장 및 Team Health KPI 24개 핵심 지표를 한 화면에서 확인한다.

### 2. 접근 경로
`본사 관리 > KPI 대시보드` (`/kpi`)

### 3. 24개 KPI 지표

#### 회원 현황 (6개)

| # | label | 계산 |
|---|-------|------|
| 1 | 전체 회원 | `members.count (deletedAt IS NULL)` |
| 2 | 활성 회원 | `members.count (status='ACTIVE')` |
| 3 | 활성 비율 | `활성/전체 × 100` |
| 4 | 이번달 신규 | `members.count (registeredAt >= monthStart)` |
| 5 | 만료 예정 | `members.count (ACTIVE, expiry 30일 이내)` |
| 6 | 만료 회원 | `members.count (status='EXPIRED')` |

#### 매출 (4개 + 달성률 게이지)

| # | label | 계산 |
|---|-------|------|
| 7 | 이번달 매출 | `sales.sum(amount, COMPLETED)` |
| 8 | 전월 매출 | 전월 동일 쿼리 |

### 4. NPS 계산

```
NPS = 추천자 비율(%) - 비추천자 비율(%)
- 추천자: 9~10점
- 중립: 7~8점
- 비추천자: 0~6점
- NPS 범위: -100 ~ +100
```

### 5. NPS 등급

| NPS | 등급 | 해석 |
|-----|------|------|
| 70+ | 세계적 수준 | 업계 최고 수준 |
| 50~69 | 우수 | 대부분의 회원이 만족 |
| 30~49 | 양호 | 개선 여지 있음 |
| 0~29 | 보통 | 적극적 개선 필요 |
| < 0 | 위험 | 즉각적 조치 필요 |

### 6. 설문 생성 모달 (DLG-104-001)

| 필드 | 타입 | 필수 |
|------|------|:---:|
| 설문명 | Input(text) | * |
| 대상 세그먼트 | 조건 빌더 | * |
| 추가 질문 (최대 3개) | Textarea | - |
| 발송 채널 | Select (알림톡/SMS/앱 푸시) | * |
| 발송 일시 | datetime | * |
| 응답 마감일 | date | * |

### 7. 설문 질문 구성

| 순서 | 질문 | 응답 형식 |
|------|------|----------|
| 1 (필수) | "이 센터를 친구나 동료에게 추천할 의향이 얼마나 있으신가요?" | 0~10 점수 선택 |
| 2 (선택) | "가장 만족스러운 점은 무엇인가요?" | 자유 텍스트 |
| 3 (선택) | "개선이 필요한 점은 무엇인가요?" | 자유 텍스트 |
| 4~6 (커스텀) | 관리자가 추가한 질문 | 자유 텍스트 또는 선택형 |

### 8. 피드백 감정 분석

| 점수 | 감정 | 아이콘 |
|------|------|--------|
| 9~10 | 매우 만족 | 😊 |
| 7~8 | 보통 | 😐 |
| 4~6 | 불만족 | 😕 |
| 0~3 | 매우 불만족 | 😞 |

### 9. 자동 후속 조치

| 조건 | 자동 액션 |
|------|----------|
| 비추천자 (0~6점) | 담당 FC에게 "불만 회원 후속 상담" 태스크 자동 생성 |
| 추천자 (9~10점) | 리퍼럴 코드 자동 발송 유도 |
| 특정 키워드 (시설, 위생 등) | 해당 부서 담당자에게 알림 |
