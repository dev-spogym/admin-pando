## SCR-090 본사 대시보드

### 1. 화면 목적
전체 지점 통합 KPI(회원, 매출, 출석, 직원)를 실시간 조회하고, 지점별 현황 카드, 최근 감사 로그를 한 화면에 표시한다.

### 2. 접근 경로
`슈퍼관리자 로그인 > 대시보드` (`/super-dashboard`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 본사 대시보드      [🔄 새로고침] [시각: HH:MM]│
├─────────────────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│ │ 전체 회원 │ │ 전체 매출 │ │ 오늘 출석 │ │ 전체 직원 │    │
│ │ 3,240명  │ │ 4.82억   │ │ 1,124명  │ │ 48명     │    │
│ │ +12.4%   │ │ +8.2%    │ │ +5.1%    │ │ +2.1%    │    │
│ └──────────┘ └──────────┘ └──────────┘ └──────────┘    │
├─────────────────────────────────────────────────────────┤
│ 지점별 현황                                              │
│ ┌─────────────────────┐ ┌─────────────────────┐        │
│ │ 강남점 [운영중]       │ │ 송도점 [운영중]       │        │
│ │ 회원: 850 / 직원: 15 │ │ 회원: 620 / 직원: 12 │        │
│ │ 매출: 1.8억          │ │ 매출: 1.2억          │        │
│ │ [상세보기]            │ │ [상세보기]            │        │
│ └─────────────────────┘ └─────────────────────┘        │
├─────────────────────────────────────────────────────────┤
│ 주의 필요 사항                                           │
│ ⚠ 만료 임박 회원: 45명  ⚠ 신규 가입: 23명               │
│ ⚠ 만료 회원: 12명                                       │
├─────────────────────────────────────────────────────────┤
│ 최근 감사 로그                                           │
│ [시간] [사용자] [액션] [대상] [IP] [상세]                 │
└─────────────────────────────────────────────────────────┘
```

### 4. KPI 카드 (4열)

| 카드 | 값 | 아이콘 | 변동 | 변동 기준 |
|------|-----|--------|------|----------|
| 전체 회원 | totalMembers | Users | (current-prev)/prev × 100 | 전월 대비 |
| 전체 매출 | totalRevenue | DollarSign | (current-prev)/prev × 100 | 전월 대비 |
| 오늘 출석 | todayAttendance | UserCheck | (today-yesterday)/yesterday × 100 | 전일 대비 |
| 전체 직원 | totalStaff | Building2 | 0% (정적) | - |

금액 포맷: >= 1억 → "{N.N}억", >= 1만 → "{N}만"

변동 표시: 양수 → 녹색 ArrowUpRight, 음수 → 빨강 ArrowDownRight

### 5. 지점 상태 배지 (BranchStatusBadge)

| 상태 | 표시 | 색상 |
|------|------|------|
| ACTIVE / active / 운영중 | 운영중 | green |
| SUSPENDED / inactive / 임시휴업 | 휴업 | amber |
| CLOSED / closed / 폐점 | 폐점 | red |

### 6. 주의 사항 영역

| 항목 | 값 | 아이콘 |
|------|-----|--------|
| 신규 가입 | newMembers | UserPlus |
| 만료 임박 | expiringMembers | AlertTriangle |
| 만료 회원 | expiredMembers | AlertTriangle |

## SCR-091 슈퍼 대시보드

### 1. 화면 목적
본사/지점/FC/트레이너/운영 5개 보드를 탭으로 전환하며, 각 역할별 핵심 메트릭, 액션 큐, 성과 랭킹, 퍼널, 인사이트를 표시한다.

### 2. 접근 경로
`슈퍼관리자 > KPI 프리뷰` (`/kpi-preview`)

### 3. 보드 탭 (5종)

| 탭 | key | 아이콘 | 부제목 |
|----|-----|--------|--------|
| 본사 | hq | Building2 | "전사 성장률, 지점 비교, 이탈위험, 자동화 효율을 한 번에 관리합니다." |
| 지점 | branch | LayoutDashboard | "지점장이 오늘 바로 확인해야 할 목표, 팀 실행률, 재등록 우선순위를 모읍니다." |
| FC | fc | Users | FC별 퍼널/실적 |
| 트레이너 | trainer | Activity | 수업/출석 실적 |
| 운영 | operations | BellRing | 운영 지표 |

### 4. 본사 보드 (hq) 상세

**메트릭 카드 (4열):**

| 라벨 | 값 | description | variant |
|------|-----|-------------|---------|
| 전사 신규 | 428명 | 전월 대비 +12.4% | mint |
| 유지율 | 88.6% | 상위 5개 지점 평균 | default |
| 전사 매출 | 4.82억 | 월 누적 / 목표 96% | peach |
| 이탈위험 지점 | 3곳 | 14일 기준 경고 상태 | default |

**액션 큐 (본사 액션 큐):**

| 제목 | 담당 | 기한 | 긴급도 |
|------|------|------|--------|
| 재등록률 40% 이하 지점 코칭 | 운영총괄 | 오늘 14:00 | error |
| 신규 리드 응답 지연 지점 점검 | CRM 매니저 | 오늘 16:00 | info |
| 4월 자동알림 시나리오 승인 | 마케팅 | 내일 | default |
| 운영비 급증 지점 원인 확인 | 재무 | 이번 주 | success |

**지점 성과 랭킹:**

| 지점 | 성과 | 변동 |
|------|------|------|
| 강남점 | 목표 달성 121% | +18% |
| 송도점 | 유지율 92.4% | +6% |
| 분당점 | PT 재구매 38% | +9% |
| 마곡점 | 자동화 실행률 94% | +11% |

**전사 퍼널 요약:**

| 단계 | 값 | 변동 |
|------|-----|------|
| 리드 유입 | 1,280 | +14% |
| 상담 완료 | 742 | +9% |
| 등록 전환 | 286 | +6% |
| 재등록 | 191 | +4% |

**전사 인사이트:**
- PT 재구매율은 강남/분당에서 강하고, 송도/마곡은 체험 참여율이 더 높습니다.
- 리드 응답 속도 30분 이내 지점이 등록 전환율에서도 상위권을 유지하고 있습니다.
- 자동화 규칙이 켜진 지점은 만료 D-7 응답률이 평균보다 높습니다.

### 5. 지점 보드 (branch) 상세

**메트릭 카드:** 월 목표 달성률(84%), 활성 회원(1,124명), 재등록 예정(49명), Today Tasks(27건)

**오늘 우선 액션:** 만료 D-7 상담 배정, PT 노쇼 복구, GX 증설 확인, 미수금 결제 확인

## SCR-092 지점 관리

### 1. 화면 목적
전체 지점 목록 조회, 신규 지점 등록, 지점 간 회원 이동, 지점 비활성화/폐쇄 처리를 관리한다.

### 2. 접근 경로
`슈퍼관리자 > 지점 관리` (`/branches`)

### 3. 탭 구조

| 탭 | key | 내용 |
|----|-----|------|
| 지점 목록 | list | 전체 지점 테이블 + 통합 통계 |
| 통합 통계 | stats | 지점 간 비교 차트/테이블 |

### 4. 지점 등록 폼 (BranchForm)

| 필드명 | key | 타입 | 필수 | 유효성 | 에러 메시지 |
|--------|-----|------|------|--------|------------|
| 지점명 | name | text | Y | 2~30자 | "지점명은 2~30자" |
| 지점 코드 | code | text | Y | 영문 대문자 2~5자 | "영문 대문자 2~5자" |
| 연락처 | phone | tel | Y | 0XX-XXXX-XXXX | "올바른 전화번호 형식" |
| 주소 | address | text | Y | 비어있지 않을 것 | "주소를 입력하세요" |
| 상세 주소 | addressDetail | text | N | - | - |
| 담당자 | managerId | select | Y | - | "담당자를 선택하세요" |
| 오픈 시간 | openTime | time | N | 기본 06:00 | - |
| 마감 시간 | closeTime | time | N | 기본 23:30 | - |
| 최대 수용 인원 | maxCapacity | number | N | - | - |
| 메모 | memo | textarea | N | - | - |

### 5. 지점 상태 변경

**2단계 비활성화 프로세스:**
- Step 1: 영향 범위 확인 (활성 회원 수, 활성 직원 수 표시)
- Step 2: 2차 확인 ("정말 비활성화하시겠습니까?")

**상태 변경 유형:** closed(폐쇄), inactive(임시휴업)

### 6. 회원 이동 기능
지점 간 회원 이동 모달: 출발 지점 → 도착 지점 → 대상 회원 선택

## SCR-093 지점 성과 리포트

### 1. 화면 목적
월별 전체 지점 성과를 비교 분석한다. 회원 수, 매출, 출석률을 바 차트로 시각화한다.

### 2. 접근 경로
`슈퍼관리자 > 지점 리포트` (`/branch-report`)

### 3. 통계 카드 (StatCardGrid cols=4)

| 카드 | 값 | 아이콘 | variant |
|------|-----|--------|---------|
| 전체 지점 수 | 지점 count | Building2 | default |
| 전체 회원 | 합계 totalMembers | Users | mint |
| 전체 매출 | 합계 totalSales | TrendingUp | peach |
| 평균 출석률 | 평균 attendanceRate | UserCheck | default |

### 4. 지점별 비교 테이블

| 컬럼 | key | width | align | 렌더링 |
|------|-----|-------|-------|--------|
| 지점명 | name | auto | left | bold |
| 총 회원 | totalMembers | 100 | right | "{N}명" |
| 신규 가입 | newMembers | 100 | right | "{N}명" |
| 활성 회원 | activeMembers | 100 | right | "{N}명" |
| 만료 회원 | expiredMembers | 100 | right | "{N}명" |
| 총 매출 | totalSales | 140 | right | formatKRW |
| 평균 매출 | avgSales | 120 | right | formatKRW |
| 출석률 | attendanceRate | 100 | right | "{N}%" |

정렬: sortKey/sortAsc 토글 (ArrowUpDown 아이콘)

### 5. 바 차트 (SimpleBarChart)

| 차트 | 데이터 | 색상 |
|------|--------|------|
| 회원 수 비교 | 지점별 totalMembers | bg-primary |
| 매출 비교 | 지점별 totalSales | bg-accent |

## SCR-094 KPI 대시보드

### 1. 화면 목적
회사 성장 및 Team Health KPI 24개 지표를 종합 대시보드로 표시한다. 매출 목표 설정, 코호트 분석, 퍼널 분석을 포함한다.

### 2. 접근 경로
`사이드바 > KPI` (`/kpi`)

### 3. KPI 24개 지표 상세

#### 회원 현황 (6지표)

| # | 지표명 | key | 계산 방식 | 표시 |
|---|--------|-----|----------|------|
| 1 | 전체 회원 수 | totalMembers | members 테이블 count | "{N}명" |
| 2 | 활성 회원 수 | activeMembers | status=ACTIVE count | "{N}명" |
| 3 | 이번달 신규 가입 | newMembersThisMonth | 이번달 registeredAt count | "{N}명" |
| 4 | 전월 신규 가입 | newMembersPrevMonth | 전월 registeredAt count | "{N}명" |
| 5 | 만료 회원 | expiredMembers | expireDate < today count | "{N}명" |
| 6 | 만료 임박 회원 | expiringMembers | 7일 이내 만료 count | "{N}명" |

#### 매출 (2지표)

| # | 지표명 | key | 계산 방식 | 표시 |
|---|--------|-----|----------|------|
| 7 | 이번달 매출 | monthlyRevenue | sales 합계 (이번달) | formatAmount |
| 8 | 전월 매출 | prevMonthRevenue | sales 합계 (전월) | formatAmount |

#### 출석 (2지표)

| # | 지표명 | key | 계산 방식 | 표시 |
|---|--------|-----|----------|------|
| 9 | 주간 평균 출석 | avgWeeklyAttendance | 최근 7일 일평균 | "{N}명" |
| 10 | 오늘 출석 | todayAttendance | 오늘 attendance count | "{N}명" |

#### 상담 (2지표)

| # | 지표명 | key | 계산 방식 | 표시 |
|---|--------|-----|----------|------|
| 11 | 전체 상담 | totalConsultations | consultations count (이번달) | "{N}건" |
| 12 | 완료 상담 | completedConsultations | status=완료 count | "{N}건" |

#### PT (3지표)

| # | 지표명 | key | 계산 방식 | 표시 |
|---|--------|-----|----------|------|
| 13 | 전체 PT 세션 | totalPtSessions | lessons count (이번달) | "{N}건" |
| 14 | 완료 PT 세션 | completedPtSessions | status=완료 count | "{N}건" |
| 15 | PT 노쇼 | noShowPtSessions | status=노쇼 count | "{N}건" |

#### 수업 (3지표)

| # | 지표명 | key | 계산 방식 | 표시 |
|---|--------|-----|----------|------|
| 16 | 전체 수업 | totalClasses | class_schedule count (이번달) | "{N}건" |
| 17 | 수업 참석자 | totalClassAttendees | 수업별 참석 합계 | "{N}명" |
| 18 | 수업 예약자 | totalClassBooked | 수업별 예약 합계 | "{N}명" |

#### 파생 지표 (6지표, 계산)

| # | 지표명 | 계산식 | 표시 |
|---|--------|--------|------|
| 19 | 활성 비율 | activeMembers / totalMembers × 100 | "{N}%" |
| 20 | 신규 가입 MoM 변동 | (thisMonth - prevMonth) / prevMonth × 100 | "+{N}%" / "-{N}%" |
| 21 | 매출 MoM 변동 | (thisMonth - prevMonth) / prevMonth × 100 | "+{N}%" / "-{N}%" |
| 22 | 상담 완료율 | completed / total × 100 | "{N}%" |
| 23 | PT 완료율 | completed / total × 100 | "{N}%" |
| 24 | PT 노쇼율 | noShow / total × 100 | "{N}%" |

### 4. 매출 달성률 게이지 (RevenueGauge)

| 구간 | 색상 | 라벨 |
|------|------|------|
| 100%+ | green | "목표 달성!" |
| 70~99% | blue | "{N}% 달성" |
| 50~69% | yellow | "{N}% 달성" |
| ~49% | red | "{N}% 달성" |

매출 목표: localStorage `kpi_monthly_target_{branchId}_{yearMonth}` 에 저장. DLG-094 목표 설정 모달.

### 5. 코호트 분석 (CohortAnalysis)

최근 6개월 등록 코호트 × 1~6개월 유지율 히트맵

| 코호트 | 등록수 | 1개월 | 2개월 | 3개월 | 4개월 | 5개월 | 6개월 |
|--------|--------|-------|-------|-------|-------|-------|-------|
| 2026-01 | 45명 | 89% | 78% | 65% | 58% | 52% | 48% |
| 2026-02 | 52명 | 92% | 81% | 71% | 63% | - | - |
| ... | | | | | | | |

**셀 색상:** 90%+=녹색, 70~89%=연녹색, 50~69%=노랑, 30~49%=주황, ~29%=빨강

### 6. 퍼널 분석 (FunnelAnalysis)

| 단계 | 소스 | 색상 |
|------|------|------|
| 리드 유입 | leads count (이번달) | bg-primary |
| 상담 완료 | consultations status=완료 (이번달) | bg-primary/80 |
| 회원 등록 | members registeredAt (이번달) | bg-primary/60 |
| 3개월 유지 | members status=ACTIVE, registeredAt < 3개월 전 | bg-primary/40 |
| 재등록 | sales round=재등록 (이번달) | bg-primary/25 |

## SCR-096 온보딩 대시보드

### 1. 화면 목적

## SCR-100 로그인

### 1. 화면 목적
지점 선택 + 아이디/비밀번호 로그인. 실패 횟수 기반 계정 잠금 플로우를 포함한다.

### 2. 접근 경로
`/login` (미인증 시 리다이렉트)

### 3. ASCII 와이어프레임

```
┌──────────────────────────────┐
│          [S]                 │
│      FitGenie CRM            │
│  피트니스 센터 통합 관리 솔루션│
│                              │
│  지점 선택                    │
│  [📍 ▼ 강남점              ] │
│                              │
│  아이디                      │
│  [👤 admin@fitgenie.com    ] │
│                              │
│  비밀번호                    │
│  [🔒 ••••••••         👁️  ] │
│                              │
│  [☐ 로그인 유지]             │
│                              │
│  [        로그인        ]     │
│                              │
│  ⚠ 이메일 또는 비밀번호가     │
│     올바르지 않습니다.         │
│     (3회 더 실패 시 잠금)     │
└──────────────────────────────┘
```

### 4. 로그인 폼 필드

| 필드 | key | 타입 | 아이콘 | 필수 |
|------|-----|------|--------|------|
| 지점 선택 | selectedBranchId | select | MapPin | Y |
| 아이디 | id | text | User | Y |
| 비밀번호 | password | password | Lock | Y |
| 비밀번호 표시 | showPassword | toggle | Eye/EyeOff | - |
| 로그인 유지 | rememberMe | checkbox | - | N |

### 5. 로그인 잠금 플로우 (계정 잠금 메커니즘)

**상수:**

| 상수 | 값 | 설명 |
|------|-----|------|
| MAX_FAIL_COUNT | 5 | 최대 실패 허용 횟수 |
| LOCK_DURATION_MS | 30 * 60 * 1000 | 잠금 시간 (30분) |
| FAIL_COUNT_KEY | 'login_fail_count' | localStorage 키 |
| LOCKED_UNTIL_KEY | 'login_fail_locked_until' | localStorage 키 |
| REMEMBER_KEY | 'spoGym_rememberedId' | 아이디 저장 키 |

**잠금 플로우 시퀀스:**

```
[로그인 시도]
    │
    ├─ 잠금 상태 확인 (getLoginLockState)
    │   ├─ lockedUntil 존재 && remaining > 0
    │   │   → "계정이 잠겼습니다. 본사 IT팀에 문의하세요."
    │   │   → 로그인 차단
    │   └─ lockedUntil 경과
    │       → 자동 해제 (FAIL_COUNT, LOCKED_UNTIL 삭제)
    │
    ├─ 로그인 API 호출
    │   ├─ 성공
    │   │   → clearFailCount() (실패 카운트 초기화)
    │   │   → rememberMe ? localStorage에 아이디 저장 : 삭제
    │   │   → authStore.login(사용자 정보, accessToken)
    │   │   → localStorage.setItem('branchId', selectedBranchId)
    │   │   → moveToPage(966) (대시보드)
    │   │
    │   └─ 실패
    │       → incrementFailCount()
    │       ├─ failCount < MAX_FAIL_COUNT
    │       │   → "이메일 또는 비밀번호가 올바르지 않습니다."
    │       │   → "({remaining}회 더 실패 시 계정이 잠깁니다)"
    │       │
    │       └─ failCount >= MAX_FAIL_COUNT
    │           → LOCKED_UNTIL = now + 30분
    │           → "계정이 잠겼습니다. 본사 IT팀에 문의하세요."
    │           → toast.error 동일 메시지
    │           → 폼 전체 disabled
    │
    └─ 서버 오류
        → "서버 연결에 실패했습니다. 잠시 후 다시 시도해주세요."
```

### 6. 슈퍼관리자 로그인 특성

| 항목 | 일반 사용자 | 슈퍼관리자 |
|------|-----------|-----------|
| branchId | 필수 (user.branchId) | 선택 (selectedBranchId 또는 '') |
| branchName | 선택 지점명 | '전체 지점' (미선택 시) |
| isSuperAdmin | false | true |
| currentBranchId | null | user.currentBranchId |
| tenantId | user.tenantId | user.tenantId |

### 7. 상태별 화면

| 상태 | 표시 |
|------|------|
| 초기 | 저장된 아이디 복원 + rememberMe 체크 |
| 로딩 | 로그인 버튼 Loader2 아이콘 + "로그인 중..." + disabled |
| 지점 로딩 | Select disabled + "지점 로딩 중..." |
| 지점 없음 | Select "지점 없음" |
| 잠금 상태 | 전체 폼 disabled + 빨강 에러 메시지 |
| 실패 (잠금 전) | 노랑 경고 + 잔여 횟수 표시 |
