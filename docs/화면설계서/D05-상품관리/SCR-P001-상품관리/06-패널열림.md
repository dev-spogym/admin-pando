> 델타 문서. 공통 스펙은 `00-기본화면.md` 상속. 우측 패널 자체 스펙은 **SCR-P003 상품상세패널** 참조.

## 상태 진입 조건
- 사용자가 테이블 행 클릭 → `setSelectedProduct(row); setIsNewMode(false); setPanelOpen(true)`
- `+ 상품 등록` 클릭 → `setSelectedProduct(null); setIsNewMode(true); setPanelOpen(true)`
- 복사 모드 → `setSelectedProduct({...row,id:undefined,name:row.name+' 복사'}); setIsNewMode(true)`

## 역할별 차이

| 역할 | 패널 모드 |
|---|---|
| super/primary/owner | 편집 가능 (전체 폼 활성, 저장/삭제/복사 노출) |
| manager | 읽기 전용 (인풋 `readOnly`, 버튼 `disabled` + 툴팁 "권한이 없습니다") |
| fc | 읽기 전용, 가격이력은 조회 가능 |
| trainer | 읽기 전용, 가격이력 조회 가능, 삭제 버튼 비렌더 |
| staff/front | 읽기 전용, 최소 정보만 (가격/상태만) |
| readonly | 패널 열림 자체 차단 (toast "읽기 전용 계정입니다") |

## 상태 고유 인터랙션

### 패널 오픈
- 행 클릭 `onRowClick(row)` → `setSelectedProduct(row); setPanelOpen(true)`; 200ms 슬라이드 인.
- `+ 상품 등록` → `selectedProduct=null; isNewMode=true; panelOpen=true`.

### 패널 닫기
- **X 버튼** / **Esc 키** / **좌측 오버레이 클릭(모바일 Drawer)** → 닫기.
- **폼 더티** 시 DLG-P002-작업취소확인 우선 오픈.
- 저장 성공 시 패널 닫기 + 토스트 + refetch.

### 키보드
- `Esc` → 닫기(더티 검사).
- `Cmd/Ctrl + S` → 저장(폼 유효 시).
- `Cmd/Ctrl + D` → 삭제(권한 있을 때 DLG-P006-상품삭제확인).
- 패널 내부 Tab 포커스 트랩.

### 상품 간 이동
- 좌측 테이블에서 **다른 행 클릭** → 현재 패널 폼 더티면 DLG-P002, 아니면 바로 교체.

## 비즈니스 룰 델타
- 패널 오픈 중 URL 해시 `#product-{id}` 추가 (새로고침 시 복원).
- 패널 폼은 react-hook-form + zod (`productSchema`).
- 신규 모드 `isNewMode=true` → 저장 시 `products.insert`, `branchId = useAuthStore.branchId`.
- 수정 모드 → `products.update({...}).eq('id', selectedProduct.id)`.
- 복사 모드 → id 제거 + 이름에 " 복사" suffix + 저장 시 insert.
- 비활성 토글 시 활성 판매 존재 → DLG-P004-비활성화안내.
- 삭제 시 DLG-P006-상품삭제확인 → 연결 데이터 존재 시 차단.
- 가격 변경 시 price_history 테이블 자동 기록 (서버 트리거).

## 에지 케이스
- 저장 중 네트워크 단절 → 폼 유지 + 에러 토스트 + 재시도 버튼 인라인.
- 다른 사용자가 동일 상품을 동시에 수정하여 `updatedAt` 충돌 → DLG-004-저장확인 `05-충돌` 변형.
- 패널 열린 채 필터 변경 → 선택된 상품이 필터 결과에서 사라져도 패널은 유지(주황 배너 "필터 결과에서 제외됨").
- 새로고침 시 `#product-{id}` 복원 실패(삭제됨) → 토스트 "해당 상품을 찾을 수 없습니다." + 패널 닫힘.
