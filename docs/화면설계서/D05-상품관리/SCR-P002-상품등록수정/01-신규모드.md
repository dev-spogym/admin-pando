# SCR-P002 — 상태: 01-신규모드 (new-mode)

> 이 문서는 마스터 `00-기본화면.md`를 상속하는 **델타(변경점)** 문서입니다.
> 레이아웃/토큰/컴포넌트/데이터 계약/권한/접근성은 마스터 기본값을 따릅니다.

## 메타

| 항목 | 값 |
|---|---|
| 화면 ID | SCR-P002 |
| 상태 코드 | `new-mode` |
| 다이어그램 노드 | F6 → `STATE_NEW_MODE` |
| 경로 | `/products/new` |
| 역할 | superAdmin, primary, owner, manager |
| 이전 상태 | (없음) SCR-P001 "상품 등록" 버튼 |
| 다음 상태 | `04-저장중` (submit) / `05-유효성에러` (검증 실패) / 취소 → DLG-P002 |

## 상태 진입 조건

- `/products/new` 직접 진입 또는 SCR-P001 "상품 등록" 클릭
- 폼 defaultValues 초기화 (`DEFAULT_PRODUCT_FORM`)
- 로드 API 호출 없음 (즉시 렌더)

## 비주얼 델타

```
헤더 타이틀: [📦] 상품 등록
sticky footer 우측: [신규] [등록] [종료]
활성/비활성 배지: 표시 안 함 (신규라 기본값 "활성" 고정)
```

| 요소 | 신규 모드 값 |
|---|---|
| 헤더 타이틀 | "상품 등록" |
| primary 버튼 라벨 | "등록" |
| 모든 입력값 | 초기 defaultValues |
| 가격 이력/복사/삭제 버튼 | 숨김 (SCR-P003 패널 전용) |
| ValidationBanner | 숨김 (submit 전) |
| SavingOverlay | 숨김 |

## 상태 고유 인터랙션

- `신규` 버튼 클릭 → `isDirty=false`면 무시, `isDirty=true`면 DLG-P002 → "돌아가기" 시 폼 resetForm(keep default)
- `종료` 버튼 / `X` 버튼: `isDirty=true`면 DLG-P002, 아니면 `/products` 즉시 이동

## 상태 고유 비즈니스 룰 (델타)

1. 저장 성공 시 **INSERT** 경로. 이후 응답의 `productId`로 `/products?highlight={id}`로 이동 가능.
2. 상품구분 라디오 초기값 `lesson` (요구사항: 레슨 상품이 가장 빈번).
3. 요일 defaults: 월~금 `enabled=true` `09:00~18:00`, 토/일 `enabled=false`.

## 에지 케이스

- 사용자가 바로 "등록" 클릭(공란 상태) → zod 검증 실패 → `05-유효성에러` 전환
- 네트워크 불안정 + 중복 클릭 → `saving` 가드로 재진입 방지

## 바이브코딩 프롬프트 (델타)

```
// 마스터 프롬프트를 기본으로 하되, 다음만 오버라이드
mode = 'new'
defaultValues = DEFAULT_PRODUCT_FORM  // 월~금 09~18, productKind='lesson', isActive=true
header title = '상품 등록'
submit button label = '등록'
onSuccess = (data) => moveToPage(`/products?highlight=${data.id}`)
```
