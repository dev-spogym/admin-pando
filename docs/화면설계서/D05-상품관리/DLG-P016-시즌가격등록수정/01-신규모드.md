## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-P008 시즌가격관리 화면에서 "시즌가격 추가" 버튼 클릭 시 진입
- 폼은 빈 상태로 오픈

### 레이아웃 구성
- 모달 너비: 520px (고정)
- 헤더: "시즌가격 추가" 타이틀 + X 닫기 버튼
- 본문: 4개 필드 수직 배치
- 하단 액션: 취소(secondary) / 등록(primary)

### 필드 정의

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 시즌명 | text input | Y | 최대 50자 |
| 적용 기간 | DateRangePicker | Y | start_date ~ end_date |
| 적용 단가 | number input | Y | 원 단위, 0 이상 |
| 적용 상품 | MultiSelect | Y | products 테이블에서 조회 |

### 인터랙션 (User Actions)
1. 폼 입력 → 실시간 유효성 검사
2. "등록" 클릭 → `seasonal_prices` INSERT → `03-저장결과`
3. "취소" / X 클릭 → 모달 닫기

### 비즈니스 룰
- 기간 중복 시 `03-저장결과` 기간충돌 분기로 이동
- owner, manager만 접근 가능 (front, trainer 접근 불가)
- 등록 후 SCR-P008 목록 자동 갱신

### 연결 화면
- 등록 성공: `03-저장결과` (success)
- 기간 충돌: `03-저장결과` (conflict)
- 서버 오류: `03-저장결과` (error)
- 취소: SCR-P008 복귀
