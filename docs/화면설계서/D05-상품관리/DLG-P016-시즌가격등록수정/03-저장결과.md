[4] 서버 오류 (기타 error):
  toast.error("저장에 실패했습니다. 다시 시도해 주세요.")

  // 폼 유지, 재시도 가능

사용 유틸: toast (react-hot-toast)

## 📝 디스크립션

### 사용 시점 (Trigger)
- 신규 등록 또는 수정 후 API 응답 수신 시

### 분기 정의

| 분기 | 조건 | 처리 방식 |
|------|------|-----------|
| 등록 성공 | INSERT 성공 | toast.success + 모달 닫기 + 목록 갱신 |
| 수정 성공 | UPDATE 성공 | toast.success + 모달 닫기 + 목록 갱신 |
| 기간 충돌 | overlap 에러 | 인라인 경고 배너 표시 (모달 유지) |
| 서버 오류 | 기타 DB 에러 | toast.error + 폼 유지 (재시도 가능) |

### 인터랙션 (User Actions)

**등록/수정 성공 시:**
- 자동으로 모달 닫힘
- SCR-P008 목록 자동 갱신

**기간 충돌 시:**
- 모달 유지, 기간 필드 포커스
- 사용자가 기간 수정 후 재시도 가능

**서버 오류 시:**
- toast 알림 후 폼 유지
- "저장" 버튼 재활성화 → 재시도 가능

### 비즈니스 룰
- 충돌 에러는 모달을 닫지 않고 인라인으로 표시 (UX 일관성)
- 성공 후 SCR-P008의 `refetch()` 호출하여 목록 최신화
- 저장 중(saving=true)에는 버튼 disabled + 스피너 표시

### 연결 화면
- 성공: SCR-P008 (자동 복귀)
- 충돌: `02-수정모드` 또는 `01-신규모드` 유지
- 서버 오류: 현재 폼 상태 유지
