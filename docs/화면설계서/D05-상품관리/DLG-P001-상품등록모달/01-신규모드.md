## 상태 진입 조건
- `canEditProduct === true`
- 세션 유효 (`DLG-000` 미표시)
- `mode='new'`로 `ProductDetailPanel` 마운트

## 상태 고유 비주얼 델타
- 헤더 타이틀 = "상품 등록" (수정 모드와 구분)
- 활성/비활성 배지 **미노출**(수정 모드 전용)
- "상품명" 필드 `autoFocus`
- Primary 버튼 라벨 = "등록" (수정 모드는 "저장")
- 모든 필드 `defaultValues` 초기값 렌더 (폼 clean 상태)
- 에러 배너 / 필드 인라인 에러 **미노출**

## 역할별 차이
- super/primary는 현재 선택된 `branchId` 기준으로 INSERT (BranchSwitcher 값 표시)
- owner/manager: 본인 지점 고정 (`user.branchId`)
- 기타 역할: 진입 불가 (모달 자체가 열리지 않음)

## 상태 고유 인터랙션
1. **오픈 직후**: 상품명 autoFocus, Tab 순서 = 상품구분 → 상품명 → 현금가 → (조건부) → 요일/옵션 → 패키지 → 취소 → 등록
2. **상품구분 변경**: 레슨/이용 조건부 섹션 토글. 전환 시 하위 필드 `setValue(..., undefined)` 초기화
3. **패키지 체크**: 패키지 구성 섹션 펼침 + 합산 가격 자동 계산
4. **취소(X/ESC/배경)**: `isDirty === false` → 즉시 닫기 / `true` → DLG-P002 오픈
5. **Enter**: 포커스된 input에서 제출 트리거, textarea는 줄바꿈

## 상태 고유 비즈니스 룰 델타
- `defaultValues`: `productKind='레슨'`, `isActive=true`, `salesChannel='ALL'`, `weekdayRows=DEFAULT_WEEKDAYS` (월~금 09:00~18:00)
- 제출 전 중복 체크 `SELECT id FROM products WHERE branchId=? AND name=?` → 존재 시 `02-유효성에러`로 전이
- `isDirty` 판정: `formState.isDirty || packageItems.length > 0`
- super/primary가 BranchSwitcher로 지점 전환 시 모달 종료(dirty면 DLG-P002)

## 에지 케이스
- **긴 상품명 입력**: 100자 초과 시 입력 차단(maxLength=100) + hint "최대 100자"
- **금액 한글/특수문자**: input type=number로 차단 + IME 조합 시 onBlur에서 숫자 변환
- **레슨 ↔ 이용 토글**: 이미 입력한 레슨시간/이용구분 값은 초기화 (혼선 방지)
- **패키지 자기 참조**: `packageSelect` 목록에서 편집 중인 상품은 제외 (신규 모드는 자기 자신 없음)
- **네트워크 단절 중 제출**: `useMutation` 실패 → 토스트 "네트워크 오류" + 모달 유지 (입력값 보존)
- **오프라인 큐**: 선택적 구현 — offline이면 제출 차단, 재연결 시 자동 복구
