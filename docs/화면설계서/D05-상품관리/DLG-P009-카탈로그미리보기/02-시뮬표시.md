## 역할별 차이
- super/primary: 우상단에 선택된 지점명 배지 노출 (`강남점`)
- owner/manager: 배지 숨김 (컨텍스트 고정)

## 상태 고유 인터랙션
- 디바이스 클릭 → `01-로딩`으로 전이 후 재렌더
- iframe 내부 스크롤은 iframe 로컬 스크롤 (모달 스크롤 전파 금지 `overscroll-contain`)
- "카탈로그 설정으로" 링크(역할에 따라) 클릭 시 현재 모달 닫고 DLG-P010 오픈

## 비즈니스 룰 델타
- `response.items` 정렬: catalogOrder ASC, 동률 시 createdAt DESC
- 숨김 상품은 응답에 포함되지 않아야 함 (클라이언트 재필터 금지, 서버 신뢰)
- iframe URL은 `generatedAt` 쿼리로 cache-bust: `?branchId=...&device=...&t=...`
- 디바이스 키오스크일 경우 `data-kiosk="true"` 전달 (iframe에서 대형 터치 타겟 스타일 적용)

## 에지 케이스
- iframe `onLoad` 실패 (미디어 404): 카탈로그 프리뷰 HTML 자체는 살아있으면 `02-시뮬표시` 유지 + 이미지 fallback
- 외부 리소스 CORS 실패: console warning만, 상태 전이 없음
- 200초 유휴 후 `staleTime` 초과 시 재오픈하면 자동 refetch

## 바이브코딩 프롬프트 (델타)

```
━━ 델타: DeviceFrame + iframe ━━
{viewState === 'ready' && data && (
  <DeviceFrame device={device}>
    <iframe
      title={`카탈로그 미리보기 · ${device === 'mobile' ? '모바일' : device === 'desktop' ? '데스크톱' : '키오스크'}`}
      sandbox="allow-same-origin allow-scripts"
      src={`/api/catalog/preview/html?branchId=${data.branchId}&device=${device}&t=${encodeURIComponent(data.generatedAt)}`}
      className="w-full h-full border-0"
      onLoad={() => trackEvent('CATALOG_PREVIEW_RENDERED', { device, branchId: data.branchId })}
    />
  </DeviceFrame>
)}

━━ Footer 델타 ━━
<p className="text-xs text-gray-500">
  💡 회원 앱/키오스크 노출 화면과 동일합니다.
  {data && <span className="ml-2 text-gray-400">· {formatTime(data.generatedAt)} · {data.totalCount}개 상품</span>}
</p>

━━ 지점 배지 (super/primary) ━━
{isSuper && data && (
  <span className="ml-2 inline-flex items-center gap-1 h-6 px-2 rounded-full bg-blue-50 text-blue-700 text-xs">
    {branches.find(b => b.id === data.branchId)?.name}
  </span>
)}
```
