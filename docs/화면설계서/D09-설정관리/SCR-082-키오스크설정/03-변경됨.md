# SCR-082 키오스크 설정 — 상태 03: 변경됨 (dirty)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — **델타**만 기술.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `kiosk-dirty` |
| 이전 | `02-정상` (필드 수정) |
| 다음 | `02-정상`(저장 완료) / `05-에러`(저장 실패) |

## 진입 조건
- `form.formState.isDirty === true`

## 비주얼 델타
| 요소 | 변경 |
|---|---|
| DirtyBadge | 표시 (amber) |
| 변경된 섹션 | 카드 좌측 `border-l-2 border-amber-500` |
| 저장 버튼 | 활성 (primary blue) |
| 프리뷰 | 실시간 dirty 값 반영 |
| beforeunload | 등록 |

## 인터랙션
1. 저장 클릭 → PATCH → WebSocket push → `02-정상` + SavedBadge 3초
2. 사이드바 이동 / 탭 변경 → dirty 경고
3. Ctrl/Cmd+S → 저장

## 비즈니스 룰 델타
- `pickDirty(values, dirtyFields)` → PATCH 본문 축약
- D11 비활성이어도 PATCH 저장은 허용 (단 push 실패 경고)
- 저장 + push 결과는 `saved-with-push-warning` 상태 가능

## 에지 케이스
- push 실패: 설정은 저장되지만 키오스크에 적용은 재연결 후 자동
- D11 disconnect 중 저장: "단말에 적용은 재연결 시" 토스트 표시

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14

onSave:
async function handleSave() {
  const diff = pickDirty(form.getValues(), form.formState.dirtyFields);
  const updated = await patchKioskSettings(branchId, diff);
  form.reset(updated);
  try {
    await pushToKiosk(branchId);
    toast.success('저장 + 단말 반영 완료');
  } catch {
    toast.warn('저장됨. 단말 적용은 재연결 후 자동 반영됩니다.');
  }
  setSaved(true); setTimeout(() => setSaved(false), 3000);
}

변경된 섹션 좌측 border:
const sectionsDirty = computeSectionDirty(form.formState.dirtyFields);
<section className={cn(
  'bg-white rounded-xl ring-1 ring-gray-100 p-6',
  sectionsDirty.has('design') && 'border-l-2 border-amber-500'
)}>...</section>

Ctrl+S:
useEffect(() => {
  const h = (e: KeyboardEvent) => {
    if ((e.ctrlKey||e.metaKey) && e.key === 's') { e.preventDefault(); handleSave(); }
  };
  window.addEventListener('keydown', h);
  return () => window.removeEventListener('keydown', h);
}, []);
```
