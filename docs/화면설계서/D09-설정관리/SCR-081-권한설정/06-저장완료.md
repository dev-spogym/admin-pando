# SCR-081 권한 설정 — 상태 06: 저장 완료

> 상속: [`00-기본화면.md`](./00-기본화면.md) — **델타**만 기술.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `perm-saved` |
| 이전 | `05-저장중`(200) |
| 다음 | `02-정상-역할선택` (3초 후 자동) |

## 진입 조건
- `saveMutation.isSuccess === true`
- 배지 타이머 작동 중 (3초)

## 비주얼 델타

| 요소 | 변경 |
|---|---|
| SavedBadge | 표시 — `bg-emerald-50 text-emerald-700 ring-emerald-200` + `<CheckCircle2 />` "저장되었습니다" |
| 모든 ring-amber cell | 제거 (dirtyCells 비움) |
| DirtyBadge | 비표시 |
| 저장/초기화 버튼 | disabled (isDirty=false) |
| 매트릭스 인터랙션 | 복구 (pointer-events-auto) |
| 충돌 리스트 | 재계산 (일반적으로 비표시) |

## 인터랙션
1. **새로운 cell 토글** → 즉시 `04-편집-변경됨` 전환(SavedBadge 즉시 해제)
2. **3초 경과** → SavedBadge 자동 해제 → `02-정상-역할선택`
3. **역할 전환** → 즉시 해당 역할 매트릭스 로드

## 비즈니스 룰 델타
- 배지 타이머: `setTimeout(() => setJustSaved(false), 3000)`
- 사용자 인터랙션 시 즉시 해제 (자연스러운 UX)
- `savedPermissions[roleCode]` 갱신되어 이후 편집 기준값 됨
- 감사로그 기록: 저장 성공 시점에 `AUDIT.PERMISSION_UPDATE`

## 에지 케이스
- **연속 저장**: 이전 배지 타이머 clear → 새 타이머 시작
- **저장 직후 탭 이탈**: 배지는 사라지지만 save 결과는 persistent

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14

상태: perm-saved

SavedBadge:
{justSaved && (
  <div role="status" aria-live="polite"
       className="inline-flex items-center gap-1.5 h-8 px-3 rounded-full text-xs font-medium
                  bg-emerald-50 text-emerald-700 ring-1 ring-emerald-200 animate-[fadeIn_150ms]">
    <CheckCircle2 className="size-3.5" />
    저장되었습니다
  </div>
)}

타이머:
useEffect(() => {
  if (!justSaved) return;
  const t = setTimeout(() => setJustSaved(false), 3000);
  return () => clearTimeout(t);
}, [justSaved]);

필드 변경 시 즉시 해제:
function togglePermission(...) {
  if (justSaved) setJustSaved(false);  // 즉시 해제
  // 04-편집-변경됨 전환
}

prefers-reduced-motion:
@media (prefers-reduced-motion: reduce) {
  .animate-\[fadeIn_150ms\] { animation: none; }
}
```
