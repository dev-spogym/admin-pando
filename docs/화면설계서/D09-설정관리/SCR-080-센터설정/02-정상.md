> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 상태 진입 조건
- GET 200 + 폼 초기화 완료 + `isDirty=false`
- 저장 성공 후 3초 배지 해제 시점
- "저장 안 함"으로 탭 이동 완료 후

## 상태 고유 비주얼 델타

**마스터 레이아웃 기본값.** 아래 요소만 명시적으로 OFF/비표시.

| 요소 | 상태 |
|---|---|
| DirtyBadge | 비표시 |
| SavedBadge | 비표시 (저장 직후 3초만 표시, 이후 해제) |
| 저장 버튼 | `disabled`, `bg-blue-400 cursor-not-allowed` |
| 취소 버튼 | `disabled` |
| 에러 배너 | 비표시 |
| 모든 필드 | 활성(enabled), 기본 테두리 |

## 상태 고유 인터랙션
1. **필드 수정** (input/toggle/color/time) → `03-변경됨` 전환, DirtyBadge 표시
2. **탭 클릭** → 즉시 탭 변경 (dirty 가드 없음)
3. **BranchSwitcher 변경** (super/primary) → 지점 변경 → `01-로딩`
4. **키보드**: Tab으로 필드 순회, Arrow 키로 탭 이동
5. **로고 영역 클릭** → 파일 선택 다이얼로그 (변경 시 → `03-변경됨`)

## 상태 고유 비즈니스 룰 델타
- 진입 시 최초 1회 `runOnboarding7dayBanner()` (owner 한정, 가입 7일 이내)
- 테마 `mode`는 `02-정상` 진입 시점에 `<html>` 클래스에 이미 적용됨
- 폼 값은 `data`(서버 값)와 동기화. 외부 refetch 발생 시 `form.reset(newData)`로 재동기화

## 에지 케이스
- **동시 수정 감지**: 윈도우 포커스 복귀 시 `updatedAt` 재검증 → 서버 값이 더 최신이면 "다른 관리자가 수정함" 배너 + 재로드 버튼
- **체크박스 undefined → false 정규화**: 서버가 null/undefined로 보낼 수 있으므로 초기화 시 false로 정규화
- **저장 완료 배지와 수정 동시 발생**: 배지 3초 타이머 중 필드 수정 → 즉시 배지 해제 + DirtyBadge 표시
