## [SCR-P001] 상품 관리

### 1. 화면 목적
센터에서 판매하는 이용권, PT, GX 및 기타 상품을 관리한다. 마스터-디테일 패턴으로 목록과 상세/편집을 한 화면에서 처리. 분류 관리 탭에서 상품 카테고리를 CRUD 관리.

### 2. 접근 경로
사이드바 > 상품관리 > 상품 관리 (`/products`)

### 3. 화면 구성

```
+------------------------------------------------------------------+
| PageHeader: "상품 관리"                                           |
| [상품목록|분류관리] [전지점배포*] [+상품등록*] [Excel]               |
+------------------------------------------------------------------+
| StatCardGrid (4열)                                                |
| [전체 상품] [활성 상품] [비활성] [카테고리 수]                       |
+------------------------------------------------------------------+
| 상품타입 TabNav: [전체(N)] [회원권(N)] [수강권(N)] [대여권(N)] [일반(N)]|
+------------------------------------------------------------------+
| 분류 필터: [전체] [이용권(N)] [PT(N)] [GX(N)] [기타(N)] ...         |
+------------------------------------------------------------------+
```

### 4. 통계 카드

| 카드명 | 값 | variant | 아이콘 | 데이터 규칙 |
|--------|-----|---------|--------|-------------|
| 전체 상품 | `products.length` | peach | Package | 전체 상품 수 |
| 활성 상품 | `activeCount` | mint | CheckCircle | isActive===true인 상품 수 |
| 비활성 | `inactiveCount` | default | XCircle | isActive===false인 상품 수 |
| 카테고리 수 | `categoryCount` | default | LayoutGrid | 고유 category 값의 개수 |

### 5. 테이블 컬럼 정의 (상품 목록)

| 컬럼명 | key | 정렬 | 데이터 규칙 | 패널 열림시 |
|--------|-----|------|-------------|-------------|
| 상품명 | name | left | 타입배지(9px) + 패키지배지 + 상품명(12px). 선택된 행: primary 색상 | 항상 표시 |
| 현금가 | cashPrice | right | `formatKRW(cashPrice ?? price)` 12px | 항상 표시 |
| 카드가 | cardPrice | right | `formatKRW(cardPrice)` 또는 '-' 12px secondary | 항상 표시 |
| 기간 | duration | center | `${duration}일` 또는 '-' 11px | 항상 표시 |
| 횟수 | sessions | center | `${sessions}회` 또는 '-' 11px | 항상 표시 |
| 월~일 (7열) | usage_restrictions.availableDays | center | 이용 가능: ✓(green), 불가: ✗(red). 제한 없으면 전체 ✓ | 패널 닫힘시만 |
| 시간 | usage_restrictions | center | `시작~종료` 또는 '전체' | 패널 닫힘시만 |
| 홀딩 | holdingEnabled | center | ✓(green) / ✗(red) | 패널 닫힘시만 |
| 양도 | transferEnabled | center | ✓(green) / ✗(red) | 패널 닫힘시만 |
| 포인트 | pointAccrual | center | ✓(green) / ✗(red) | 패널 닫힘시만 |
| 판매 | salesCountMap | center | `${N}건` 또는 '0건' | 패널 닫힘시만 |
| 상태 | isActive | center | StatusBadge. 사용(mint, dot) / 미사용(default) | 항상 표시 |

**상품 타입 배지 스타일:**
- MEMBERSHIP → 회원권 (bg-blue-100 text-blue-700)
- LESSON → 수강권 (bg-green-100 text-green-700)
- RENTAL → 대여권 (bg-orange-100 text-orange-700)
- GENERAL → 일반 (bg-gray-100 text-gray-600)

**패키지 배지:** `📦 패키지` (bg-amber-100 text-amber-700). description JSON 파싱하여 isPackage===true인 경우 표시.

### 6. 필터 옵션

| 필터명 | 타입 | 옵션 목록 |
|--------|------|-----------|
| 상품타입 | TabNav | 전체, 회원권(MEMBERSHIP), 수강권(LESSON), 대여권(RENTAL), 일반(GENERAL) |
| 분류 (카테고리) | button group (pill) | 전체 + 동적 목록 (product_groups + 실제 상품 카테고리 합집합) |
| 종목 | button group | 전체, 헬스, 필라테스, 요가, 수영, 복싱, 크로스핏, 기타 |
| 상품명 검색 | text | 실시간 필터 (toLowerCase 포함 검색, 200px 너비) |

### 7. 버튼 -> 동작 매핑

| 버튼명 | 위치 | 조건 | 동작 |
|--------|------|------|------|
| 상품 목록 / 분류 관리 | PageHeader 토글 | 항상 | mainTab 전환 (products / groups) |
| 전 지점 배포 | PageHeader | 슈퍼관리자(isSuperAdmin)만 | DLG-P001 배포 모달 오픈 |
| + 상품 등록 | PageHeader | canEditProduct 권한 | 상세 패널 신규 모드로 오픈 |
| Excel | PageHeader | 상품목록 탭 | `exportToExcel()` - 6개 컬럼. 성공 토스트: `${N}건 엑셀 다운로드 완료` |
| 행 클릭 | 상품 목록 테이블 | - | 우측 상세 패널 오픈 (수정 모드) |
| 첫 상품 등록하기 | 빈 목록 영역 | canEditProduct | 상세 패널 신규 모드로 오픈 |

**분류 관리 탭 버튼:**

| 버튼명 | 위치 | 동작 |
|--------|------|------|
| 등록 / 수정 저장 | 분류 폼 | `createProductGroup` 또는 `updateProductGroup`. 성공: `toast.success('분류가 등록/수정되었습니다.')`, 실패: `toast.error('등록/수정에 실패했습니다.')`. 분류명 빈값: `toast.error('분류명을 입력하세요.')` |
| 취소 | 분류 폼 (수정시만) | editingGroupId=null, 폼 초기화 |
| 수정 (Edit2) | 분류 목록 행 | editingGroupId 설정 + 폼에 데이터 로드 |
| 삭제 (Trash2) | 분류 목록 행 | `window.confirm('이 분류를 삭제하시겠습니까?')` → `deleteProductGroup`. 성공: `toast.success('분류가 삭제되었습니다.')` |

### 8. 상태별 화면

| 상태 | 화면 |
|------|------|
| 로딩 | AsyncBoundary isLoading 표시 |
| 빈 데이터 (상품) | Package 아이콘 + "상품이 없습니다." + "첫 상품 등록하기" 링크 (canEditProduct시) |
| 빈 데이터 (분류) | "등록된 분류가 없습니다." |
| 에러 | 별도 에러 UI 없음 (fetch 실패시 빈 목록) |
| 권한별 | canEditProduct=false: "상품 등록" 버튼 숨김, 패널에서 편집 제한. isSuperAdmin=false: "전 지점 배포" 버튼 숨김 |

### [DLG-P001] 전 지점 배포 모달

#### 1. 트리거
상품 관리 > "전 지점 배포" 버튼 클릭 (슈퍼관리자 전용)

#### 2. 필드 테이블

**배포할 상품 선택 섹션:**

| 필드명 | 타입 | 필수 | 옵션 | 유효성 | 에러메시지 |
|--------|------|------|------|--------|------------|
| 상품 체크박스 | checkbox list | Y (1개 이상) | 전체 상품 목록 (이름, 패키지 배지, 가격 표시) | 0개 선택시 | `toast.error('배포할 상품을 선택하세요.')` |
| 전체 선택 | link button | - | 모든 상품 선택 | - | - |

**대상 지점 섹션:**

| 필드명 | 타입 | 필수 | 옵션 | 유효성 | 에러메시지 |
|--------|------|------|------|--------|------------|
| 전체 지점 | checkbox | N | 기본값: true | - | - |
| 지점별 체크박스 | checkbox list | 전체미선택시 Y | branches 테이블에서 로드 (현재 지점 제외) | 0개 선택시 | `toast.error('배포할 지점을 선택하세요.')` |

#### 3. 버튼

| 버튼명 | 동작 | 성공 토스트 | 실패 시 동작 |
|--------|------|-------------|-------------|
| X (닫기) | 모달 닫기 | - | - |
| 배포 (ConfirmFlow) | 중복 체크 → INSERT. 동일 이름 상품은 건너뜀 | `배포 완료: ${successCount}건 추가, ${skipCount}건 건너뜀` | 개별 실패시 건너뜀 |

#### 4. 닫기 조건
- X 버튼 클릭
- 배경 overlay 클릭

## [SCR-P002] 상품 등록/수정 - 레거시 폼

### 1. 화면 목적
레슨북(클래식) 스타일의 상품 등록/수정 폼. `/products/new` (신규), `/products/edit?id=N` (수정). react-hook-form + zod 유효성 검증.

### 2. 접근 경로
- 신규: POS > "상품 등록" 버튼 (`moveToPage(997)`)
- 수정: 상품 목록에서 이동 (`/products/edit?id=N`)
- `/products/edit` 페이지는 `/products/new` 컴포넌트를 재사용 (export default from '../new/page')

### [DLG-P007] 할인 정책 추가/수정 모달

#### 1. 트리거
- 추가: "할인 추가" 버튼 클릭
- 수정: 테이블 행의 수정(Edit2) 아이콘 클릭

#### 2. 필드 테이블

| 필드명 | 타입 | 필수 | 옵션 | 유효성 | 에러메시지 |
|--------|------|------|------|--------|------------|
| 할인명 | text | Y | placeholder: "예: 재등록 10% 할인" | 비어있으면 에러 | `toast.error('할인명을 입력해주세요.')` |
| 할인 유형 | radio | Y | 정률(%), 정액(원) | - | - |
| 할인 값 | number (min=0) | Y | placeholder: 정률시 "예: 10", 정액시 "예: 50000" | 비어있거나 NaN | `toast.error('할인 값을 입력해주세요.')` |
| 최소 계약 기간 | number (min=0) | N | placeholder: "예: 90" | - | - |
| 최대 할인금액 | number (min=0) | N | placeholder: "예: 100000" | - | - |
| 활성 상태 | toggle switch | N | 기본값: true. accent 색상 토글 | - | - |

#### 3. 버튼

| 버튼명 | 동작 | 성공 토스트 | 실패 시 동작 |
|--------|------|-------------|-------------|
| 취소 | 모달 닫기 | - | - |
| 등록 / 수정 저장 | `createDiscountPolicy` 또는 `updateDiscountPolicy`. localStorage에 변경이력 저장 | 등록: `toast.success('할인 정책이 등록되었습니다.')`, 수정: `toast.success('할인 정책이 수정되었습니다.')` | `toast.error('등록/수정에 실패했습니다.')` |

**변경이력 저장 형식:**
```json
{
  "date": "2026/04/12 14:30",
  "title": "등록/수정/삭제: 할인명",
  "description": "10% · 활성"
}
```

#### 4. 닫기 조건
- 취소 버튼 클릭
- 외부 영역(overlay) 클릭
- ESC: (이벤트 전파에 의존)
