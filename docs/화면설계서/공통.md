### 2.3 권한 체크 로직

```typescript
// 접근 불가 시 리다이렉트
if (!hasAccess(currentRole, targetMenu)) {
  router.push('/forbidden');
}
```

- 슈퍼관리자(`isSuperAdmin=true`): 모든 메뉴 접근 가능
- 비인증 사용자: `/login` 으로 리다이렉트
- 권한 설정 화면(SCR-081)에서 메뉴별 CRUD 권한을 세분화 가능

## 3. 공통 UI 패턴

### 3.1 로딩 상태 (Loading)

| 유형 | 적용 대상 | 표시 방식 |
|------|----------|----------|
| 전체 페이지 로딩 | 페이지 첫 진입 | 스켈레톤 카드 + `animate-pulse` (헤더/통계/테이블 영역) |
| 데이터 테이블 로딩 | DataTable `loading={true}` | `emptyMessage="데이터를 불러오는 중..."` |
| 버튼 로딩 | 저장/발송 등 비동기 버튼 | `loading={true}` → 스피너 아이콘 + `"처리 중..."` 텍스트 + `disabled` |
| 인라인 로딩 | 특정 섹션 | 중앙 텍스트 `"불러오는 중..."` |
| 새로고침 | 수동 새로고침 버튼 | `RefreshCw` 아이콘 `animate-spin` |

### 3.2 빈 상태 (Empty)

| 유형 | 표시 방식 |
|------|----------|
| 데이터 없음 | DataTable `emptyMessage="{도메인} 데이터가 없습니다."` |
| 검색 결과 없음 | `Search` 아이콘 + `"검색 결과가 없습니다."` 중앙 표시 |
| 위젯 빈 상태 | 도메인별 빈 메시지 (예: `"등록된 리드가 없습니다."`) |

### 3.3 에러 상태 (Error)

| 유형 | 표시 방식 |
|------|----------|
| API 호출 실패 | `toast.error("{작업}에 실패했습니다.")` |
| 데이터 로드 실패 | `toast.error("{도메인} 데이터를 불러오지 못했습니다.")` + `console.error` |
| 폼 유효성 에러 | 필드 하단 `<p role="alert">` 에러 메시지 + `border-state-error` 테두리 |
| 네트워크 에러 | `toast.error("네트워크 오류가 발생했습니다. 다시 시도해주세요.")` |

### 3.4 토스트 규칙

| 유형 | 메서드 | 색상 | 지속 시간 | 사용 시점 |
|------|--------|------|----------|----------|
| 성공 | `toast.success()` | green | 3초 | CRUD 성공, 상태 변경 성공 |
| 에러 | `toast.error()` | red | 5초 | API 실패, 검증 실패 |
| 경고 | `toast.warning()` | amber | 4초 | 주의 필요 상황 |
| 정보 | `toast.info()` | blue | 3초 | 안내 메시지, 미구현 기능 |

**토스트 메시지 작성 규칙:**
- 성공: `"{대상}이(가) {동작}되었습니다."` (예: `"직원이 성공적으로 등록되었습니다."`)

```typescript
function formatAmount(amount: number): string {
  if (amount >= 100000000) return `${(amount / 100000000).toFixed(1)}억`;
  if (amount >= 10000) return `${Math.floor(amount / 10000).toLocaleString()}만`;
  return amount.toLocaleString();
}
```

## 11. 공통 상태 전환 규칙

### 11.1 회원 상태 라이프사이클

```
미등록(UNREGISTERED) → 활성(ACTIVE) → 만료(EXPIRED)
                                    → 홀딩(HOLDING) → 활성(ACTIVE)
                                    → 정지(SUSPENDED)
                                    → 탈퇴(WITHDRAWN)
```

| 전환 | 트리거 | 자동/수동 |
|------|--------|----------|
| 미등록→활성 | 상품 구매 완료 | 자동 |
| 활성→만료 | 이용권 만료일 경과 | 자동 (runDailySync) |
| 활성→홀딩 | 홀딩 신청 승인 | 수동 |
| 홀딩→활성 | 홀딩 종료일 도달 또는 수동 해제 | 자동/수동 |
| 활성→정지 | 관리자 강제 정지 | 수동 |
| 활성→탈퇴 | 탈퇴 처리 | 수동 |
| 만료→활성 | 재등록 (신규 상품 구매) | 자동 |

### 11.2 직원 상태 라이프사이클

```
활성(ACTIVE) → 휴직(ON_LEAVE) → 활성(ACTIVE)
            → 퇴사예정(RESIGN_SCHEDULED) → 퇴사(RESIGNED)
            → 전보(TRANSFERRED)
            → 잠금(LOCKED)
```

| 전환 | 트리거 | 자동/수동 |
|------|--------|----------|
| 활성→휴직 | 휴직 시작일 도달 | 수동 등록 → 자동 전환 |
| 휴직→활성 | 휴직 종료일 도달 | 자동/수동 |
| 활성→퇴사예정 | 퇴사 예정 등록 (SCR-062) | 수동 |
| 퇴사예정→퇴사 | 퇴사 확정 처리 | 수동 |
| 활성→전보 | 지점 간 이동 | 수동 |
| 활성→잠금 | 보안 사유 잠금 | 수동 |

### 11.3 리드 상태 라이프사이클

```
신규 → 연락완료 → 상담예정 → 방문완료 → 등록완료
                                     → 미전환
                           → 보류
```

| 전환 | 트리거 |
|------|--------|
| 신규→연락완료 | FC가 첫 연락 완료 |
| 연락완료→상담예정 | 상담 일정 잡힘 |
| 상담예정→방문완료 | 회원이 센터 방문 |
| 방문완료→등록완료 | 상품 구매 완료 |
| 방문완료→미전환 | 등록 포기 |
| 어느 단계→보류 | 연락 불가, 시기 미정 |

### 11.4 결제 상태 라이프사이클

```
대기(PENDING) → 완료(COMPLETED) → 환불(REFUNDED)
             → 실패(FAILED)
             → 취소(CANCELLED)
```

### 11.5 쿠폰 상태 라이프사이클

```
활성(active) → 만료(expired)    [validUntil 경과]
            → 소진(exhausted)  [totalUsed >= totalIssued]
            → 비활성(inactive) [관리자 비활성화]
```

## 13. 데이터 검증 규칙

### 13.1 공통 검증 패턴

| 검증 대상 | 규칙 | 에러 메시지 |
|----------|------|-----------|
| 이름 | 필수, 1~30자 | `"이름을 입력하세요."` |
| 전화번호 | `010-XXXX-XXXX` 형식 | `"올바른 전화번호를 입력하세요."` |
| 이메일 | RFC 5322 형식 | `"올바른 이메일 주소를 입력하세요."` |
| 금액 | 0 이상 정수 | `"올바른 금액을 입력하세요."` |
| 날짜 | YYYY-MM-DD 형식, 유효한 날짜 | `"올바른 날짜를 입력하세요."` |
| 필수 Select | 값 선택됨 | `"{필드명}을(를) 선택하세요."` |

### 13.2 전화번호 자동 포맷

```typescript
function formatPhone(value: string): string {
  const digits = value.replace(/\D/g, '');
  if (digits.length <= 3) return digits;
  if (digits.length <= 7) return `${digits.slice(0,3)}-${digits.slice(3)}`;
  return `${digits.slice(0,3)}-${digits.slice(3,7)}-${digits.slice(7,11)}`;
}
```

### 13.3 중복 제출 방지

모든 폼 제출 버튼은 `isSaving`/`isSubmitting` state로 중복 클릭을 방지한다:
- 제출 시작: `setIsSaving(true)` → 버튼 `disabled + loading`
- 제출 완료/실패: `setIsSaving(false)`

## 15. branchId 기준 데이터 격리

### 15.1 규칙

- 모든 데이터 조회/생성은 `getBranchId()` 기준으로 필터링
- 슈퍼관리자가 "전체 지점" 모드일 때는 branchId 필터 미적용
- 지점 전환 시 `localStorage.setItem('branchId', newId)` → 페이지 새로고침

### 15.2 getBranchId 구현

```typescript
const getBranchId = (): number => {
  if (typeof window === 'undefined') return 1;
  const stored = localStorage.getItem('branchId');
  return stored ? Number(stored) : 1;
};
```
