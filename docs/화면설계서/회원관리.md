## SCR-001. 회원 목록 (/members)

### 1. 화면 목적
센터의 전체 회원을 조회, 검색, 필터링하고 일괄 관리하는 메인 화면

### 2. 접근 경로
- 사이드바 > 회원 > 회원 목록
- URL 직접 접근: /members

### 3. 화면 구성 (상단→하단)
```
┌──────────────────────────────────────────────────────┐
│ PageHeader: "회원 목록"   [엑셀 다운로드] [+ 회원 추가]  │
│ "센터의 전체 회원 정보를 조회하고 관리합니다."             │
├──────────────────────────────────────────────────────┤
│ StatCard×4: 전체회원 | 활성회원 | 만료예정(D-30) | 이번달만료 │
├──────────────────────────────────────────────────────┤
│ 메인탭: [회원 전체] [보유상품별] [이용권 목록]              │
├──────────────────────────────────────────────────────┤
│ ┌── 회원 전체 탭 내부 ──────────────────────────────┐ │
│ │ 서브탭(상태): 전체(N) 활성(N) 만료(N) 미등록(N) 홀딩(N) 정지(N) │
│ │                                                  │ │
│ │ 검색: [회원명, 연락처 검색...]                        │ │

│ │ 추가필터: ★관심회원만 | 미방문[▼] | 회원구분[▼] |        │

## SCR-002. 회원 등록/수정 (/members/new, /members/edit)

### 1. 화면 목적
신규 회원을 등록하거나 기존 회원 정보를 수정하는 2단계 폼 화면

### 2. 접근 경로
- 회원 목록 > [+ 회원 추가] 버튼 → /members/new
- 회원 상세 > [수정] 버튼 → /members/edit?id={memberId}
- URL 직접 접근

### 3. 모드 판별
- `/members/edit` 경로 + URL에 `id` 파라미터 → **수정 모드** (isEditMode=true)
- `/members/new` 경로 → **등록 모드** (isEditMode=false)
- 수정 모드 시 supabase에서 해당 회원 데이터를 로드하여 폼에 채움

### 4. 화면 구성
```
┌───────────────────────────────────────────────────┐
│ PageHeader: "신규 회원 등록" / "회원 정보 수정"         │
│ "새로운 회원을 시스템에 등록합니다." / "기존 회원의 정보를 수정합니다." │
│ 우측: [초기화] [취소]                                │
├───────────────────────────────────────────────────┤
│ 스텝 탭: [Step 1. 필수 정보] [Step 2. 추가 정보]      │
├───────────────────────────────────────────────────┤
│ ┌── Step 1 ──────────────────────────────────┐    │
│ │ [프로필 이미지 원형 120px]                      │    │
│ │                                            │    │

### 6. 전화번호 중복 확인 플로우
1. 연락처 입력 (자동 하이픈 포맷팅: formatPhone)
2. 13자 이상 입력 시 [중복확인] 버튼 활성화
3. 클릭 → supabase.from('members').select('id').eq('branchId', branchId).eq('phone', phone).is('deletedAt', null)
4. 수정 모드 시 자기 자신(.neq('id', urlMemberId)) 제외
5. **중복 없음**: 녹색 체크 아이콘 + "사용 가능한 번호입니다."
6. **중복 있음**: 빨간색 + "이미 등록된 번호입니다." → 저장 차단
7. 전화번호 수정 시 중복확인 상태 리셋

| 4 | fat | 체지방량 (kg) | right | primary medium |

### 7. 변화 그래프 탭
- SVG 라인 차트: 최근 6회 데이터
- 3개 메트릭 토글: 체중(#FF7F6E/coral), 골격근량(#48D1CC/mint), 체지방률(#F59E0B/amber)
- 토글 버튼: 선택 시 해당 색상 bg, 미선택 시 surface + line border
- 포인트 호버 → 툴팁 (날짜 + 값)
- 하단 변화 요약: 측정 기간, 체중/골격근/체지방률 변화량 (색상 코딩)

- GoalGauge 컴포넌트 x2
- 프로그레스 바: 달성 시 #10B981(green), 미달성 시 coral/mint
- 달성률 계산: current <= goal → 100%, else → max(0, (1 - (current-goal)/goal) * 100)

### 9. 비만 분석 섹션
- AnalysisRow: BMI (표준 18.5~23 kg/m2), 체지방률 (표준 18~28%)
- 바 차트에 표준 범위 영역 표시 (bg-state-success/15)

|------|---|
