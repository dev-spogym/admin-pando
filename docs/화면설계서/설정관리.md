## SCR-080 센터 설정

### 1. 화면 목적
센터 기본 정보, 알림 설정, 테마 설정, 물품 관리를 4개 탭으로 관리한다. 탭 간 이동 시 미저장 변경사항 경고를 제공한다.

### 2. 접근 경로
`사이드바 > 설정 > 센터 설정` (`/settings`)

### 3. 접근 권한

| 역할 | 접근 | 조회 | 수정 |
|------|:---:|:---:|:---:|
| 슈퍼관리자~센터장 | ● | ● | ● |
| 매니저 이하 | — | — | — |

### 4. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 센터 설정                                   │
│ [⚠ 저장되지 않은 변경사항] / [✅ 저장 완료]             │
├─────────────────────────────────────────────────────────┤
│ [ℹ️ 기본정보] [🔔 알림설정] [🎨 테마설정] [📦 물품관리]   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ (각 탭 콘텐츠 영역)                                      │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                              [저장하기]                   │
└─────────────────────────────────────────────────────────┘
```

### 5. 탭 구조

| 탭 | key | 아이콘 | 설명 |
|----|-----|--------|------|
| 기본정보 | basic | Info | 센터명, 주소, 사업자번호, 연락처, 영업시간, 업종 |
| 알림설정 | notification | Bell | 푸시/이메일/SMS 알림 ON/OFF (8항목) |
| 테마설정 | theme | Palette | 라이트/다크/시스템 모드, 주요 색상, 폰트 크기 |
| 물품 관리 | supplies | Package | 수건/운동복 재고, 일일 제한, 잔여 수량 |

### 6. 기본정보 탭

| 필드명 | key | 타입 | 아이콘 | 필수 |
|--------|-----|------|--------|:---:|
| 센터 로고 | - | 이미지 업로드 (120x120) | - | - |
| 센터명 | name | Input(text) | Building2 | * |
| 사업자등록번호 | businessNumber | Input(text) | - | - |
| 센터 소개 | description | Textarea (rows=2) | - | - |
| 대표 연락처 | phone | Input(text) | Phone | - |
| 주소 | address | Input(text) + [주소검색] | MapPin | - |
| 업종 선택 | sectors | Checkbox 그룹 (11종) | - | - |
| 평일 오픈/마감 | openTime/closeTime | Input(time) | Clock | - |
| 주말 오픈/마감 | weekendOpenTime/weekendCloseTime | Input(time) | Clock | - |

**업종 11종:** 헬스, 필라테스, PT샵, 골프, 요가, 태권도, 크로스핏, 복싱, 수영, 사우나, 기타

### 7. 알림설정 탭

| 카테고리 | 항목 | key | 기본값 |
|----------|------|-----|--------|
| 앱 푸시 | 실시간 입장 알림 | pushEntrance | OFF |
| 앱 푸시 | 회원 만료 임박 알림 | pushExpiry | OFF |
| 앱 푸시 | 결제 완료 알림 | pushPayment | OFF |
| 앱 푸시 | 수업 예약/취소 알림 | pushReservation | OFF |
| 이메일 | 주간 리포트 | emailWeeklyReport | OFF |
| 이메일 | 만료 예정 안내 메일 | emailExpiry | OFF |
| SMS | 결제 완료 SMS | smsPayment | OFF |
| SMS | 만료 임박 SMS | smsExpiry | OFF |

각 항목: ToggleSwitch (ON/OFF) + label + description

### 8. 테마설정 탭

| 필드 | key | 타입 | 옵션/기본값 |
|------|-----|------|------------|
| 모드 | mode | 3개 버튼 | light(Sun)/dark(Moon)/system(Monitor). 기본: light |
| 주요 색상 | primaryColor | Color picker (원형 w-10 h-10) | 기본: #3B82F6 |
| 강조 색상 | accentColor | Color picker | 기본: #10B981 |
| 폰트 크기 | fontSize | 3개 버튼 | sm(작게)/md(기본)/lg(크게) |

### 9. 물품 관리 탭

| 물품 | key | 항목 |
|------|-----|------|
| 수건 (대) | towelLarge | 재고 수량, 일일 제한, 잔여 수량 (각 Input number) |
| 수건 (소) | towelSmall | 동일 |
| 운동복 (상의) | uniformTop | 동일 |
| 운동복 (하의) | uniformBottom | 동일 |

### 10. 저장 버튼 상태

| 상태 | 스타일 |
|------|--------|
| 저장 가능 (isDirty=true) | `bg-accent text-white hover:opacity-90` |
| 저장 불가 (isDirty=false) | `opacity-50 cursor-not-allowed` |
| 저장 중 (isSaving=true) | `animate-pulse` |

### 11. 다이얼로그

#### DLG-080-001 미저장 경고

```
┌──────────────────────────────────────┐
│  ⚠ 저장되지 않은 변경사항              │
│  탭을 이동하면 변경사항이 사라집니다.     │
│                                      │
│    [취소]  [저장 후 이동]  [저장 안 함]  │
└──────────────────────────────────────┘
```

- 트리거: isDirty=true 상태에서 다른 탭 클릭
- beforeunload: 브라우저 이탈 시 네이티브 경고

### 12. API 호출

| 동작 | 테이블 | 쿼리 |
|------|--------|------|
| 설정 로드 | branches + settings | `.select().eq('branchId').single()` |
| 기본정보 저장 | settings + branches | `.update({...}).eq('branchId')` |
| 알림 저장 | settings | `.update({ smsEnabled, autoExpireNotify }).eq('branchId')` |
| 테마 저장 | settings | `.update({ theme }).eq('branchId')` |
| 물품 저장 | settings | `.update({ supplies }).eq('branchId')` |

### 13. 인라인 상태 표시 (토스트 대신)

| 조건 | 표시 |
|------|------|
| isDirty && !saveSuccess | amber 배지: AlertTriangle + "저장되지 않은 변경사항이 있습니다" |
| saveSuccess | accent 배지: CheckCircle2 + "변경사항이 저장되었습니다." (3초 후 해제) |

## SCR-081 권한 설정

### 1. 화면 목적
6개 시스템 역할과 사용자 정의 역할에 대해 메뉴별 권한(접근/조회/생성/수정/삭제)을 매트릭스 형태로 관리한다.

### 2. 접근 경로
`사이드바 > 설정 > 권한 설정` (`/settings/permissions`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────────┐
│  PageHeader: "권한 설정"                                     │
│  [저장되지 않은 변경...] [초기화] [변경 사항 저장]              │
├──────────┬──────────────────────────────────────────────────┤
│ 역할 목록 │  권한 매트릭스                                    │
│ ┌──────┐ │  [모두 허용] [모두 거부]                          │
│ │최고   │ │  ┌─────┬───┬───┬───┬───┬───┐                   │
│ │관리자 │ │  │메뉴  │접근│조회│생성│수정│삭제│                   │
│ ├──────┤ │  ├─────┼───┼───┼───┼───┼───┤                   │
│ │센터장 │ │  │회원  │ ✓ │ ✓ │ ✓ │ ✓ │ ✓ │                   │
│ ├──────┤ │  │매출  │ ✓ │ ✓ │ - │ - │ - │                   │
│ │매니저 │ │  └─────┴───┴───┴───┴───┴───┘                   │
│ ├──────┤ │                                                 │
│ │FC    │ │  [충돌 경고 영역]                                 │
│ ├──────┤ │                                                 │
│ │스태프 │ │  배정된 직원: 홍길동, 김철수                       │
│ ├──────┤ │                                                 │
│ │조회   │ │                                                 │
│ │전용  │ │                                                 │
│ └──────┘ │                                                 │
│  [+ 추가]│                                                 │
└──────────┴──────────────────────────────────────────────────┘
```

### 4. 6개 시스템 역할

| id | code | name | 설명 | 수정 가능 |
|----|------|------|------|:---:|
| 1 | primary | 최고관리자 | 모든 권한 (수정 불가) | ✗ |
| 2 | owner | 센터장 | 경영/매출 전체 접근 | ● |
| 3 | manager | 매니저 | 회원·상품·일정 관리 | ● |
| 4 | fc | 피트니스 코치 | 담당 회원·수업 접근 | ● |
| 5 | staff | 스태프 | 출석 체크·기본 조회 | ● |
| 6 | readonly | 조회전용 | 읽기 전용 전체 | ● |

### 5. 권한 매트릭스: 메뉴 그룹 (9그룹 × 22메뉴 × 5권한)

| 그룹 | 메뉴 |
|------|------|
| 회원 | 회원 목록, 회원 상세, 회원 등록/수정 |
| 수업 | 수업/캘린더 |
| 매출 | 매출 현황, POS 결제 |
| 상품 | 상품 관리 |
| 직원 | 직원 관리 |
| 급여 | 급여 관리 |
| 시설 | 락커 관리, 운동룸 관리, 밴드/카드 관리 |
| 메시지 | 메시지 발송, 쿠폰 관리, 마일리지 관리 |
| 설정 | 센터 설정, 키오스크 설정, IoT 설정, 권한 설정 |

### 6. 체크박스 셀 스타일

| 상태 | 스타일 |
|------|--------|
| 권한 ON | `bg-primary-light text-primary` + Check 아이콘 |
| 권한 OFF | `bg-surface-secondary text-line` + X 아이콘 |
| 변경됨 | `ring-2 ring-amber-600` + 셀 배경 `bg-amber-600/10` |
| 최고관리자 | `cursor-default` + 모든 권한 true 고정, 변경 불가 |

### 7. 충돌 검증 규칙 (3가지)

| # | 조건 | 경고 메시지 |
|---|------|-----------|
| 1 | delete=true && update=false | `"삭제 권한이 있지만 수정 권한이 없습니다."` |
| 2 | (create\|update\|delete) && access=false | `"쓰기 권한이 있지만 접근 권한이 없습니다."` |
| 3 | (create\|update\|delete) && read=false | `"조회 권한 없이 쓰기 권한이 설정되어 있습니다."` |

### 8. 버튼

| 버튼 | 동작 |
|------|------|
| 초기화 | DLG-081-001 → savedPermissions로 복원 |
| 변경 사항 저장 | 충돌 검증 → 경고 시 DLG-081-002 → localStorage 저장 |
| + (역할 추가) | DLG-081-003 역할 생성 모달 |
| 모두 허용 | 전체 메뉴 전체 권한 true |
| 모두 거부 | 전체 메뉴 전체 권한 false |

### 9. 다이얼로그

#### DLG-081-001 초기화 확인
- title: `"권한 설정 초기화"`
- description: `"정말로 ${selectedRole.name}의 권한을 초기화하시겠습니까?\n저장하지 않은 변경 사항은 모두 유실됩니다."`
- variant: danger

#### DLG-081-002 충돌 경고
- 충돌 목록 표시 → [경고 무시하고 저장] / [취소]

#### DLG-081-003 역할 생성
- 필드: 역할 이름(text, *), 역할 코드(text, *), 설명(textarea), 기준 역할 복사(select)

#### DLG-081-004 역할 삭제 확인
- confirmationText: 역할 이름 타이핑 필요
- 시스템 역할(isSystem=true)은 삭제 불가

### 10. 저장 위치

- `localStorage` 키: `permissions_{roleCode}`
- 향후: Supabase `role_permissions` 테이블 마이그레이션 예정

| 섹션 | 항목 | 타입 | 설명 |

## SCR-083 IoT/출입 관리

### 1. 화면 목적
출입문 게이트, CCTV, 환경 센서 등 IoT 장비의 연결 상태와 출입 이력을 관리한다.

### 2. 접근 경로
`사이드바 > 설정 > IoT/출입 관리` (`/settings/iot`)

### 3. 설정 항목

| 섹션 | 항목 | 설명 |
|------|------|------|
| 게이트 연동 | 게이트 장비 목록 | 장비명, IP, 상태(온라인/오프라인) |
| 게이트 연동 | 게이트 테스트 | 열기/닫기 테스트 |
| 출입 정책 | 영업시간 외 출입 차단 | Toggle |
| 출입 정책 | 만료 회원 출입 차단 | Toggle |
| 출입 정책 | 미등록 카드 알림 | Toggle |
| 출입 이력 | 오늘 출입 로그 | 테이블: 시간, 회원명, 방식(RFID/QR/수동), 결과(성공/실패) |
| CCTV | 카메라 목록 | 카메라명, 위치, 상태, 미리보기 |
| 환경 | 온도/습도 센서 | 현재 값, 경고 임계값 설정 |


### 4. 커스텀 역할 생성 프로세스

1. [+ 새 역할 생성] 클릭
2. 역할 이름, 코드, 설명 입력
3. (선택) 기존 역할에서 권한 복사
4. 권한 매트릭스에서 세부 권한 설정
5. 저장 → 직원 배정 시 해당 역할 선택 가능

### 5. 역할 속성

| 필드 | 타입 | 필수 | 설명 |
|------|------|:---:|------|
| 역할 이름 | Input(text) | * | 예: "필라테스 팀장" |
| 역할 코드 | Input(text) | * | 예: "pilates_lead" (영문 소문자+언더스코어) |
| 설명 | Textarea | - | 역할 상세 설명 |
| 기준 역할 | Select | - | 기존 역할의 권한을 복사 |
| 대시보드 접근 | Toggle | - | KPI 대시보드 접근 허용 |
| 급여 정보 열람 | Toggle | - | 급여 관련 메뉴 접근 |
| 보고서 열람 | Toggle | - | 리포트 메뉴 접근 |

### 6. 삭제 규칙

- 시스템 역할(isSystem=true): 삭제 불가
- 직원이 배정된 커스텀 역할: 삭제 전 재배정 필요 (DLG-087-001)
- 배정된 직원 없는 커스텀 역할: 즉시 삭제 가능
