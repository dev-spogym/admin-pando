## SCR-I001 통합 출석 관리

### 1. 화면 목적
회원과 직원의 출석을 한 화면에서 통합 관제하고, 출석 결과에 따라 `입장 성공`, `실패 사유`, `옷 락커 배정 상태`, `고정 물품 락커 보유`, `InBody 측정 권장`, `활동 데이터 연계 상태`까지 운영자가 한 번에 판단할 수 있도록 한다.

### 2. 접근 경로
`사이드바 > 출석 관리` (`/attendance`)

### 3. 핵심 운영 정책

- 기본값은 `출석 후 직원 수동 배정`
- 지점 설정에 따라 `옷 락커 자동 배정` 가능
- 일부 지점은 `출석만 처리` 후 인포데스크에서 별도 키 지급
- 직원도 키오스크에서 동일하게 출석 체크
- 회원/직원 출석은 같은 채널을 쓰더라도 저장 타입은 분리

### 4. 상단 KPI 카드

| 카드 | 설명 |
|------|------|
| 오늘 총 출석 | 회원 + 직원 전체 체크인 수 |
| 회원 출석 | 회원 체크인 수 |
| 직원 출석 | 직원 체크인 수 |
| 락커 미배정 | 출석 성공 후 옷 락커 미배정 건수 |
| 실패 로그 | 인증 실패/입장 거절 건수 |

### 5. 화면 구성

```text
┌────────────────────────────────────────────────────────────────────┐
│ PageHeader: 통합 출석 관리                                          │
│ [수동 출석] [실패 로그] [키오스크 모니터] [엑셀 다운로드]           │
├────────────────────────────────────────────────────────────────────┤
│ [오늘 총 출석] [회원 출석] [직원 출석] [락커 미배정] [실패 로그]   │
├────────────────────────────────────────────────────────────────────┤
│ 필터: [회원/직원] [성공/실패] [지점] [채널] [락커상태] [검색]      │
├────────────────────────────────────────────────────────────────────┤
│ 실시간 스트림                                                      │
│ 시간 | 구분 | 이름 | 채널 | 결과 | 옷 락커 | 고정 락커 | 메모     │
│ 09:01 | 회원 | 김민준 | QR | 성공 | 미배정 | P-12 | 인포 처리 대기 │
│ 09:03 | 직원 | 박서윤 | 얼굴인식 | 성공 | - | - | 출근 처리        │
│ 09:07 | 회원 | 이수현 | QR | 실패 | - | - | 회원권 만료           │
└────────────────────────────────────────────────────────────────────┘
```

### 6. 주요 액션

| 액션 | 조건 | 결과 |
|------|------|------|
| 수동 출석 등록 | 인포 직원 권한 보유 | 출석 로그 생성 |
| 옷 락커 수동 배정 | 회원 출석 성공 + 미배정 상태 | 락커 번호 부여 |
| 실패 사유 확인 | 실패 건 | 인증/회원권/시간제한 사유 확인 |
| 퇴실 처리 | 재실 회원/직원 | check-out 기록 저장 |
| 회원 상세 이동 | 회원 행 클릭 | 회원 상세 화면 이동 |

### 7. 상태값

| 분류 | 값 |
|------|----|
| 사용자 유형 | 회원, 직원, 게스트 |
| 채널 | 앱 QR, 키오스크 QR, 얼굴인식, 수동 |
| 결과 | 성공, 실패, 중복 시도, 퇴실 |
| 옷 락커 상태 | 미배정, 수동배정 완료, 자동배정 완료, 대상 아님 |

### 8. 운영 주의점

- 실패 로그는 `인증 실패`와 `운영 정책 거절`을 구분해야 한다.
- 고정 물품 락커 보유 여부는 출석 처리에는 영향을 주지 않지만, 현장 응대 정보로 즉시 보여줘야 한다.
- 직원 출석은 추후 근태 연동을 고려해 `member attendance`와 다른 저장 타입으로 관리한다.

## SCR-I002 키오스크 설정

### 1. 화면 목적
지점별 키오스크 운영 모드를 설정한다. 출석 채널, 성공/실패 메시지, 옷 락커 후처리, 직원 출석 허용, 예외 처리 정책을 관리한다.

### 2. 접근 경로
`사이드바 > 설정 > 키오스크 설정` (`/settings/kiosk`)

### 3. 핵심 탭

| 탭 | 설명 |
|----|------|
| 기본 설정 | 기기 활성화, 운영 모드, 대기 메시지 |
| 화면 설정 | 배경, 로고, 안내 문구, 배너 |
| 음성/TTS | 성공/실패/대기 음성 |
| 출석 규칙 | 입장 허용 시간, 중복 방지, 만료 회원 정책 |
| 락커 후처리 | 출석만, 수동 배정, 자동 배정 |

### 4. 추가 정책 필드

| 필드 | 타입 | 기본값 |
|------|------|--------|
| 직원 출석 허용 | toggle | ON |
| 출석 후 처리 | select | `수동 배정` |
| 자동 배정 대상 존 | multi select | 없음 |
| 락커 미배정 시 메시지 | textarea | "인포데스크에서 락커를 배정받아 주세요." |
| 얼굴인식 사용 | toggle | 지점별 |
| QR 사용 | toggle | ON |

### 5. 화면 구성

```text
┌──────────────────────────────────────────────────────────────────┐
│ PageHeader: 키오스크 설정                                        │
│ [연결 테스트] [미리보기] [저장]                                  │
├──────────────────────────────────────────────────────────────────┤
│ [기본 설정] [화면 설정] [TTS] [출석 규칙] [락커 후처리]          │
├──────────────────────────────────────────────────────────────────┤
│ 좌측: 설정 폼                                                    │
│ 우측: 키오스크 결과 미리보기                                     │
│                                                                  │
│ 출석 후 처리: ( ) 출석만  (●) 직원 수동 배정  ( ) 자동 배정      │
│ 직원 출석 허용: [ON]                                             │
│ 자동배정 대상 존: [A] [B]                                        │
│ 미배정 안내 문구: "인포데스크에서 락커를 배정받아 주세요."       │
└──────────────────────────────────────────────────────────────────┘
```

### 6. 결과 메시지 예시

| 상황 | 메시지 |
|------|--------|
| 회원 출석 성공 + 수동 배정 | "출석이 완료되었습니다. 인포데스크에서 락커를 배정받아 주세요." |
| 회원 출석 성공 + 자동 배정 | "{이름}님, 오늘의 락커는 A-12번입니다." |
| 직원 출석 성공 | "{이름}님 출근 처리되었습니다." |
| 회원권 만료 | "이용권이 만료되었습니다. 데스크로 문의해 주세요." |

## SCR-I003 IoT 연동 관리

### 1. 화면 목적
출입문, 키오스크, 락커 컨트롤러, InBody 장비의 온라인 상태와 최근 통신 이력을 관리하고, 장애 발생 시 운영자가 즉시 대응할 수 있도록 한다.

### 2. 접근 경로
`사이드바 > 설정 > IoT 설정` (`/settings/iot`)

### 3. 관리 대상

| 장비 유형 | 설명 |
|-----------|------|
| 출입문/게이트 | 원격 개방, 상태 확인 |
| 키오스크 | 출석 단말 상태 확인 |
| 락커 컨트롤러 | 자동 배정/해제용 장비 |
| InBody | 파일 수신/API 연동 장비 |

### 4. 화면 구성

```text
┌──────────────────────────────────────────────────────────────────┐
│ PageHeader: IoT 연동 관리                                         │
│ [기기 추가] [전체 동기화] [장애 로그]                             │
├──────────────────────────────────────────────────────────────────┤
│ [게이트 관리] [IoT 기기] [출입 로그] [출입 규칙] [자동 전원]      │
├──────────────────────────────────────────────────────────────────┤
│ 기기명 | 유형 | 지점 | IP/시리얼 | 상태 | 최근 통신 | 관리       │
│ 메인 출입구 | 출입문 | 강남점 | ... | 온라인 | 09:02 | 테스트     │
│ InBody 270 | 체성분 | 강남점 | ... | 오류 | 08:11 | 로그보기      │
└──────────────────────────────────────────────────────────────────┘
```

### 5. 필수 액션

| 액션 | 설명 |
|------|------|
| 연결 테스트 | 장비 상태 재확인 |
| 원격 게이트 개방 | 출입문 예외 개방 |
| 장비 비활성화 | 점검 시 자동배정 대상 제외 |
| 장애 로그 확인 | 실패 원인/재발 시간 추적 |
| InBody 수신 테스트 | 파일/API 수신 채널 점검 |

### 6. 운영 규칙

- `오류`, `오프라인`, `점검중` 장비는 자동 처리 대상에서 제외
- IoT 설정의 장애 상태는 출석과 락커 화면에도 배지로 노출
- InBody 장비는 `수기`, `업로드`, `실시간 API` 세 채널 상태를 구분 표시

## SCR-I004 옷 락커 운영 관리

### 1. 화면 목적
당일 사용되는 옷 락커를 관리한다. 출석 성공 회원에게 현장 수동 배정을 기본으로 제공하고, 지점 설정에 따라 자동 배정도 지원한다.

### 2. 접근 경로
`사이드바 > 시설관리 > 락커 관리` (`/locker`)

### 3. 정책 요약

- 옷 락커는 `출석 이벤트 기반`으로 배정
- 기본 운영값은 `수동 배정`
- 자동 배정은 지점별 옵션
- 당일 마감 시 미반납 락커 회수 대상 표시

### 4. 화면 구성

```text
┌──────────────────────────────────────────────────────────────────┐
│ PageHeader: 옷 락커 운영 관리                                     │
│ [새로고침] [미배정 회원 보기] [일괄 해제] [엑셀]                 │
├──────────────────────────────────────────────────────────────────┤
│ [전체] [사용중] [미배정 대기] [만료/미반납] [고장]              │
├──────────────────────────────────────────────────────────────────┤
│ 좌측: 락커 존 그리드                                             │
│ 우측: 오늘 출석 회원 중 미배정 목록                              │
│                                                                  │
│ 미배정 회원 선택 -> 빈 락커 클릭 -> [배정]                       │
└──────────────────────────────────────────────────────────────────┘
```

### 5. 미배정 회원 패널

| 컬럼 | 설명 |
|------|------|
| 이름 | 출석 완료 회원명 |
| 출석시간 | 체크인 시각 |
| 회원권 | 일반/PT/GX 등 |
| 우선권 | VIP, 고정존 우선 등 |
| 액션 | 빠른 배정 |

### 6. 상태값

| 상태 | 설명 |
|------|------|
| 빈 락커 | 배정 가능 |
| 사용중 | 정상 사용 |
| 미반납 | 종료시간 초과 |
| 점검중 | 배정 불가 |
| 청소중 | 배정 불가 |

## SCR-I005 고정 물품 락커 관리

### 1. 화면 목적
상품 판매와 연결된 `기본물품(소형) 고정 락커`를 회원별로 장기 배정하고, 만료/연장/회수 관리를 수행한다.

### 2. 접근 경로
`사이드바 > 시설관리 > 사물함 배정` (`/locker/management`)

### 3. 정책 요약

- 출석 시 배정되는 옷 락커와 완전히 분리 관리
- 상품 계약과 연결된 장기 배정 자산
- 만료 예정 회원에게 자동 알림 가능

### 4. 화면 구성

```text
┌──────────────────────────────────────────────────────────────────┐
│ PageHeader: 고정 물품 락커 관리                                   │
│ [배정 등록] [만료 예정] [연장 처리] [회수 대상]                  │
├──────────────────────────────────────────────────────────────────┤
│ [전체] [사용중] [만료 예정] [회수 대기] [점검중]                 │
├──────────────────────────────────────────────────────────────────┤
│ 번호 | 회원명 | 상품명 | 시작일 | 만료일 | 상태 | 관리           │
│ P-12 | 김민준 | 개인 물품 락커 | 04-01 | 04-30 | 사용중 | 상세   │
└──────────────────────────────────────────────────────────────────┘
```

### 5. 주요 액션

| 액션 | 설명 |
|------|------|
| 신규 배정 | 락커 상품 보유 회원에게 번호 부여 |
| 연장 | 기존 계약 연장 |
| 회수 | 미연장 또는 해지 시 회수 |
| 잠금/점검 | 운영 중지 |

## SCR-I006 체성분 통합 관리

### 1. 화면 목적
체성분 데이터를 `수기 입력 보조`, `파일 업로드`, `실시간 API` 세 경로로 수집하고, CRM과 회원앱에서 비교 조회 가능한 단일 운영 화면을 제공한다.

### 2. 접근 경로
`사이드바 > 회원관리 > 체성분 관리` (`/body-composition`)

### 3. 상단 탭

| 탭 | 설명 |
|----|------|
| 측정 결과 | 회원별 최신 결과와 추이 |
| 업로드 센터 | 파일 업로드 및 파싱 결과 |
| 실시간 연동 | InBody API 수신 로그 |
| 오류/검수 | 중복/매칭 실패/보정 필요 데이터 |

### 4. 화면 구성

```text
┌──────────────────────────────────────────────────────────────────┐
│ PageHeader: 체성분 통합 관리                                      │
│ [수기 등록] [파일 업로드] [실시간 연동 로그]                      │
├──────────────────────────────────────────────────────────────────┤
│ [측정결과] [업로드 센터] [실시간 연동] [오류/검수]                │
├──────────────────────────────────────────────────────────────────┤
```

## 공통 정책 및 연동 원칙

### 1. Health Connect 원칙

- 순수 모바일 웹앱 직접 연동이 아니라 Android 하이브리드 앱을 통해 연동
- 회원앱 UI는 웹앱 중심 유지
- 동기화 대상은 걸음수, 활동량, 운동기록
- iOS는 초기 범위 제외

### 2. 락커 원칙

- `옷 락커`와 `고정 물품 락커`는 운영 목적과 수명주기가 다르므로 화면과 정책을 분리
- 기본 배정 모드는 수동
- 자동 배정은 지점 설정 기반 옵션

### 3. InBody 원칙

- 수집 경로 3종을 모두 지원
- 수기 수정 이력과 원본 소스를 추적 가능하게 유지
- 회원 요약에는 검수 완료 데이터만 반영
