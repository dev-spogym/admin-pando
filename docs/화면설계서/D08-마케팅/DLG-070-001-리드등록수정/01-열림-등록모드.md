## 상태 진입 조건
- `editTarget === null` 이고 `open === true`
- 역할 ∈ {owner, manager, fc}

## 상태 고유 비주얼 델타
- 타이틀: **"리드 등록"**
- 폼 전체 빈 값 (문의일만 오늘 날짜 프리필)
- 상태 pill `신규` 활성, 유입경로 `전화문의` 기본
- 저장 버튼 라벨 **"저장"**, 활성

## 역할별 차이
공통 UI. trainer/staff/front/readonly는 트리거 버튼 비노출.

## 상태 고유 인터랙션
1. 오픈 직후 이름 input 오토포커스
2. 문의일 = `new Date().toISOString().slice(0,10)` 주입
3. Tab 순서: 이름 → 연락처 → 유입경로 → 상태 pill → 담당FC → 문의일 → 후속일 → 메모 → 취소 → 저장
4. ESC/배경/X → `onClose()` (dirty 시 confirm 선택)

## 상태 고유 비즈니스 룰 델타
- `branchId = getBranchId()` 자동 주입
- createdAt은 서버 기록
- 동일 연락처 중복 검사는 경고만 (저장 차단 X)

## 에지 케이스
- 무권한 상태에서 URL 직접 접근 → 트리거 숨김
- localStorage `branchId` 누락 → `toast.error("지점 정보가 없습니다.")` 후 중단
- 유입경로 옵션 비어있음(비정상) → `"기타"` 폴백
