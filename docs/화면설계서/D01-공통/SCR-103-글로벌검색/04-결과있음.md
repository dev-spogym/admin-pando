## 🧩 바이브코딩 프롬프트
```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-103 글로벌 검색 — 상태: 결과 있음

파일: src/components/common/GlobalSearchOverlay.tsx (results.length > 0 분기)

레이아웃:
- 동일 오버레이 패널 (max-h-[600px] overflow-y-auto)
- 그룹별 결과 표시

컴포넌트 구조:
1. 검색 입력 (query 유지)
2. 결과 그룹 (도메인별 섹션):
   [회원] 그룹:
   - 그룹 레이블 "회원 (N)" (text-xs text-gray-400 px-4 py-2)
   - 결과 항목: User 아이콘 + 이름(bold) + 연락처(gray) + 상태 배지
   [수업] 그룹:
   - 그룹 레이블 "수업 (N)"
   - 결과 항목: BookOpen 아이콘 + 수업명 + 강사명
   [직원] 그룹:
   - 그룹 레이블 "직원 (N)"
   - 결과 항목: Briefcase 아이콘 + 이름 + 역할
3. 키보드 네비게이션 (↑↓ 선택, Enter 이동)
4. 하단 단축키 안내

데이터:
- results: { members: Member[]; classes: Class[]; staff: Staff[] }
- selectedIndex: number (키보드 네비게이션)
- 검색어 하이라이트: query 매칭 부분 <mark> 처리

인터랙션:
- 항목 클릭/Enter → 해당 상세 화면 이동 + 검색어 최근 목록 저장 + 오버레이 닫기
- ↑↓ 키 → selectedIndex 변경
- ESC → 닫기

사용 유틸:
- useAuthStore, moveToPage, lucide-react (User, BookOpen, Briefcase, ChevronRight)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 검색 API 응답 결과 1개 이상 존재

### 필수 데이터
| 그룹 | 테이블 | 검색 필드 |
|------|--------|----------|
| 회원 | `members` | name, phone |
| 수업 | `classes` | title, instructor_name |
| 직원 | `staff` | name, role |

### 인터랙션 (User Actions)
1. 항목 클릭 → 해당 상세 화면 + 최근 검색 저장
2. ↑↓ 키보드 → 항목 선택 이동
3. Enter → 선택 항목 이동
4. 추가 입력 → `03-검색중` 재진입

### 비즈니스 룰
- 그룹별 최대 5개 표시 (더보기는 없음, 검색어 구체화 유도)
- 검색어 매칭 부분 하이라이트 표시
- 역할별 결과 그룹 필터 (staff → 재무 관련 제외)
- 결과 클릭 시 localStorage 최근 검색어 업데이트

### 에지 케이스
- 결과 30개 초과 → 각 그룹 5개 제한
- 검색어 변경 → 즉시 `03-검색중` 재진입

### 접근성 (A11y)
- 결과 목록 `role="listbox"` `aria-label="검색 결과"`
- 선택 항목 `aria-selected="true"`
- 그룹 레이블 `role="group"` `aria-labelledby`

### 연결 화면
- 회원 항목 → SCR-M003 회원 상세
- 수업 항목 → SCR-C003 수업 상세
- 직원 항목 → SCR-ST001 직원 상세
