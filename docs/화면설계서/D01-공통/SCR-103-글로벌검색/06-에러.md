## 📝 디스크립션

### 사용 시점 (Trigger)
- 검색 API 500/502/504 응답
- 네트워크 오류로 API 요청 실패

### 필수 데이터
- query (재시도 시 사용)
- retrySearch 함수

### 인터랙션 (User Actions)
1. 재시도 버튼 → 동일 쿼리로 재검색
2. 검색어 수정 → 새 검색 실행
3. ESC → 닫기

### 비즈니스 룰
- 에러는 토스트 미표시 (오버레이 내 인라인 표시)
- 재시도 횟수 제한 없음

### 에지 케이스
- 오프라인 상태 에러 → 동일 에러 UI (메시지만 변경 가능)

### 접근성 (A11y)
- 에러 영역 `role="alert"` `aria-live="assertive"`
- 재시도 버튼 `aria-label="검색 다시 시도"`
