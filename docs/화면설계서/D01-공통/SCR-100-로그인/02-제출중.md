## 상태 고유 인터랙션
- **모든 키보드/마우스 입력 차단** (Tab으로 포커스 이동은 허용하되 액션 없음)
- **Enter 키**: `preventDefault()` — 중복 제출 방지
- **브라우저 뒤로가기**: 허용 → 폼 초기화, `01-기본` 복원(요청은 abort)
- **페이지 새로고침**: 허용 → 요청 취소 후 `01-기본` 복원

## 상태 고유 비즈니스 룰 델타
- **타임아웃**: API 호출에 `AbortController` + 10초 타임아웃
- **타임아웃 초과**: `03-에러` 전환, errorCode = `'NETWORK'`, 메시지 "요청 시간이 초과되었습니다"
- **중복 제출 방지**: `isSubmitting=true` 동안 `onSubmit` 호출 자체를 가드
- **응답 분기**:
  - 200 + `needs2FA: true` → `04-2FA입력` 전환
  - 200 + `needs2FA: false/undefined` → 세션 저장 → 역할별 `moveToPage()` → Redirect
  - 401/423/500/503 → `03-에러` 또는 `05-서비스점검`
  - fetch 실패(네트워크) → `03-에러` errorCode='NETWORK'

## 에지 케이스
- **2초 초과 응답 지연**: 스피너 유지 (UX 안내 없음). 5초 초과 시 선택적으로 "서버가 응답하지 않습니다. 잠시만 기다려주세요" 보조 텍스트 표시 가능
- **응답 지연 중 사용자가 새로고침**: 서버 측에서 세션이 이미 생성되었을 수 있음 → 다음 진입 시 세션 확인 후 자동 리다이렉트
- **동일 브라우저 다른 탭에서 먼저 로그인 성공**: `storage` 이벤트 감지 → 현재 요청 취소 후 대시보드로 이동

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: login-submitting (제출 중)

이 상태에서만 적용할 변경점:
- isSubmitting = true (useForm의 formState.isSubmitting 또는 별도 state)
- Button: disabled + <Loader2 className="animate-spin size-4 mr-2"/> 아이콘 prepend
- Button 텍스트: '로그인 중...'
- Button className에 disabled:bg-blue-400 disabled:cursor-not-allowed 적용
- 모든 Input에 readOnly 속성 추가 + className 'bg-gray-50 text-gray-600 cursor-wait'
- 비밀번호 토글 버튼: disabled + 'opacity-50 pointer-events-none'
- Checkbox: disabled
- Link: aria-disabled='true' className 'opacity-50 pointer-events-none' tabIndex={-1}
- aria-live 영역 추가: <div role="status" aria-live="polite" className="sr-only">로그인 처리 중입니다</div>

API 호출 구현:
  const abortRef = useRef<AbortController>();
  const onSubmit = async (values) => {
    if (isSubmitting) return;
    setIsSubmitting(true);
    setErrorCode(null);
    abortRef.current = new AbortController();
    const timeout = setTimeout(() => abortRef.current?.abort(), 10_000);
    try {
      const { data, error } = await supabase.auth.signInWithPassword({
        email: values.email, password: values.password
      });
      if (error) { /* 상태 코드별 매핑 → 03-에러 */ }
      else if (data.needs2FA) { setStep('2fa'); /* 04-2FA입력 */ }
      else { useAuthStore.setSession(data.session, data.user);
             moveToPage(roleToPath(data.user.role)); }
    } catch (e) {
      if (e.name === 'AbortError') setErrorCode('NETWORK');
      else setErrorCode('NETWORK');
    } finally {
      clearTimeout(timeout);
      setIsSubmitting(false);
    }
  };

접근성:
- role="status" aria-live="polite" 로 "로그인 처리 중" 공지
- button aria-busy="true" aria-disabled="true"
```
