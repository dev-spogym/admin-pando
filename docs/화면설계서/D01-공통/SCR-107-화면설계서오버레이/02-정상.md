## 🧩 바이브코딩 프롬프트
```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-107 화면설계서 오버레이 — 상태: 정상

파일: src/components/common/SpecOverlay.tsx (isLoaded=true 분기)

레이아웃:
- 우측 슬라이드인 패널 (w-[600px], 또는 전체화면 전환 가능)
- 헤더 + 툴바 + 설계서 뷰어

컴포넌트 구조:
1. 오버레이 헤더 (h-14 border-b flex items-center justify-between px-4):
   - 설계서 제목 (screenId 기반)
   - 툴바: 전체화면(Maximize2) + 새탭열기(ExternalLink) + 닫기(X)
2. 뷰어 툴바 (h-10 border-b bg-gray-50):
   - 줌 컨트롤 (ZoomIn/ZoomOut/100%)
   - 페이지 네비게이션 (이전/다음)
   - 다운로드 버튼
3. 설계서 뷰어 (flex-1 overflow-auto):
   - Markdown 렌더링 또는 PDF/이미지 표시
   - 상태별 섹션 하이라이트 (현재 화면 상태 강조)
4. 하단 메타 (업데이트 시간, 버전)

데이터:
- specContent: string (Markdown) 또는 specUrl: string (PDF/이미지)
- screenId: string
- currentState?: string (현재 화면 상태, 하이라이트용)
- zoomLevel: number = 100

인터랙션:
- 전체화면(Maximize2) → `05-전체화면`
- 새탭열기 → window.open(specUrl)
- 닫기(X) / ESC → 오버레이 닫힘
- 줌 컨트롤 → zoomLevel 변경
- 다운로드 → 파일 다운로드

사용 유틸:
- supabase, lucide-react (Maximize2, ExternalLink, X, ZoomIn, ZoomOut, Download)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 설계서 파일 로드 완료

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 설계서 콘텐츠 | Supabase Storage 또는 DB | screenId 기반 |
| 현재 상태 | props | 하이라이트용 (옵셔널) |

### 인터랙션 (User Actions)
1. 전체화면 → `05-전체화면`
2. 새 탭 열기 → 새 브라우저 탭
3. 다운로드 → 파일 다운로드
4. 줌 조절 → 확대/축소
5. 닫기/ESC → 오버레이 닫힘

### 비즈니스 룰
- 설계서는 읽기 전용 (편집 불가)
- 버전 관리: 최신 버전만 표시
- 현재 화면 상태 자동 하이라이트 (선택적)

### 에지 케이스
- PDF 렌더링 불가 브라우저 → 다운로드 링크로 폴백
- 대용량 설계서(> 10MB) → 로딩 지연 안내

### 접근성 (A11y)
- `role="dialog"` `aria-modal="true"`
- 닫기 버튼 `aria-label="화면설계서 닫기"`
- 줌 컨트롤 `aria-label="확대/축소"`

- 전체화면: `05-전체화면`
