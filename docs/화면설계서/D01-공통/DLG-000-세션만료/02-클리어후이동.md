- "로그인 화면으로" 버튼 클릭
- `handleGoLogin()` 함수 시작 ~ `router.replace('/login…')` 완료 직전까지의 짧은 구간

| 요소 | 변경 |

## 상태 고유 인터랙션
- 버튼 비활성 → 중복 클릭 차단
- 키보드 Enter/Space 입력 무시 (disabled 버튼)
- ESC는 여전히 차단
- 이 상태는 **보통 200~600ms** 짧게 지속 (네트워크에 따라 최대 2s)

## 역할별 차이
- 동일. 단, supabase signOut 은 서버 세션이 이미 무효이면 immediate resolve.

## 상태 고유 비즈니스 룰 델타
- 진행 순서(Promise 체인):
  ```ts
  setClearing(true);
  try {
    await Promise.race([
      supabase.auth.signOut(),
      new Promise((_,rej) => setTimeout(() => rej(new Error('timeout')), 2000)),
    ]);
  } catch {} // best-effort
  useAuthStore.getState().clear();
  useBranchStore.getState().clear();
  ['authToken','refreshToken','branchId','last_email']
    .forEach(k => localStorage.removeItem(k));
  sessionStorage.clear();
  try { localStorage.setItem('logout_broadcast', String(Date.now())); } catch {}
  useSessionStore.getState().close();
  const target = (!redirectPath || /^\/(login|logout)/.test(redirectPath))
    ? '/' : redirectPath;
  router.replace(`/login?redirect=${encodeURIComponent(target)}`);
  ```
- `supabase.auth.signOut()` 실패(네트워크 오류 포함) 해도 클라이언트 클리어는 진행
- `router.replace()` 성공 후 다이얼로그는 언마운트 → `02-` 상태 종료

## 에지 케이스
- **`signOut()` 2초 이상 지연**: 2초 타임아웃으로 Promise.race 처리, 그래도 클리어 진행
- **redirectPath가 `/logout`, `/login`** 인 경우: `/` 로 대체
- **redirectPath가 현재 role에 접근 불가**: 로그인 성공 후 역할별 기본 대시보드로 자동 대체(로그인 측 책임)
- **동시 탭 로그아웃**: 이미 다른 탭이 `/login` 이면 이 탭의 `router.replace` 와 경합 없이 안전 처리
