## 상태 진입 조건
- `DELETE` API → 403/404/409/500/타임아웃 응답
- 네트워크 단절로 요청 실패

## 상태 고유 비즈니스 룰 델타
- **에러 맵 기반 분기**:
  ```ts
  const ERR_MAP: Record<string, { message: string; closeDialog: boolean; refetchList: boolean }> = {
    E403001: { message: '삭제 권한이 없습니다',            closeDialog: true,  refetchList: true },
    E404100: { message: '이미 삭제된 회원입니다',           closeDialog: true,  refetchList: true },
    E409100: { message: '이 회원은 사용 중이어서 삭제할 수 없습니다', closeDialog: true, refetchList: false },
    E500001: { message: '일시적인 오류가 발생했습니다',      closeDialog: false, refetchList: false },
    NETWORK: { message: '네트워크 오류가 발생했습니다',      closeDialog: false, refetchList: false },
  };
  ```
- **낙관적 업데이트 롤백**: `onError` 에서 `queryClient.setQueryData(prevSnapshot)`
- **중복 에러 토스트 방지**: 같은 errorCode 의 토스트가 3초 내 재발 시 dedup

## 에지 케이스
- **응답 본문 파싱 실패**: JSON 이 아닌 응답 → "일시적인 오류" 기본 메시지로 대체
- **3회 연속 500**: 다이얼로그 자동 닫기 + "잠시 후 다시 시도해주세요" 토스트 (악순환 방지)
- **일괄 삭제 부분 실패는 `04`가 아닌 `03`에서 warning 토스트**로 처리 (분기 유의)
- **E401 은 이 상태로 오지 않음** — 인터셉터에서 DLG-000 으로 우선 분기

onError 핸들러:
function onError(err: { errorCode?: string } & Error) {
  const code = err.errorCode ?? (err.message === 'timeout' ? 'TIMEOUT' : 'NETWORK');
  const info = ERR_MAP[code] ?? { message: '삭제에 실패했습니다', closeDialog: false, refetchList: false };
  if (code === 'TIMEOUT') toast.error('요청 시간이 초과되었습니다');
  else toast.error(info.message);
  if (info.refetchList) qc.invalidateQueries({ queryKey:['resource'] });
  if (info.closeDialog) setOpen(false);
  setLoading(false);
}
