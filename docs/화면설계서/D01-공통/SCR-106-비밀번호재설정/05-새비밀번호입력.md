## 📝 디스크립션

### 사용 시점 (Trigger)
- `/reset-password?token=...` 접근 후 서버 토큰 검증 성공

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| token | URL 파라미터 | 서버 검증 완료 |
| password | 사용자 입력 | 8자 이상, 영문+숫자 |
| confirmPassword | 사용자 입력 | password와 동일 |

### 인터랙션 (User Actions)
1. 비밀번호 입력 → 강도 표시기 업데이트 + 조건 체크
2. 확인 입력 → 일치 여부 실시간 검증
3. "비밀번호 변경" → `06-변경중`

### 비즈니스 룰
- 비밀번호 최소 조건: 8자 이상, 영문+숫자 포함
- 이전 비밀번호와 동일 불가 (서버 검증)
- 토큰은 제출 후 즉시 무효화

### 에지 케이스
- 조건 미충족 제출 → 인라인 에러 (서버 요청 없음)
- 제출 중 토큰 만료 → `04-토큰만료-무효`

### 접근성 (A11y)
- 비밀번호 강도 `role="status"` `aria-live="polite"`
- 조건 목록 `role="list"` + 각 조건 `aria-label="{조건} 충족/미충족"`
- 눈 아이콘 `aria-label="비밀번호 표시/숨기기"`

### 연결 화면
- 제출: `06-변경중`
- 토큰 에러: `04-토큰만료-무효`
