## 📝 디스크립션

### 사용 시점 (Trigger)
- 서버/API → 503 Service Unavailable 응답
- 미들웨어에서 점검 플래그 감지
- SCR-100 로그인 시도 중 503 발생

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| 점검 종료 예정 | 503 응답 `Retry-After` 헤더 | 옵셔널 |
| retryCountdown | 클라이언트 | 30초 카운트다운 |

### 인터랙션 (User Actions)
1. "지금 재시도" → 즉시 헬스체크
2. 30초 카운트다운 만료 → 자동 헬스체크
3. 성공 → "/" 이동 + "서비스가 재개되었습니다" 토스트

### 비즈니스 룰
- 자동 재시도 간격: 30초
- 연속 실패 시 카운트다운 재시작 (무한 반복)
- 점검 종료 예정 시각 없으면 "점검 시간 미정" 미표시

### 에지 케이스
- 점검 종료 예정 시각 과거 → 숨김 처리
- 브라우저 탭 비활성 중 → Page Visibility API로 카운트다운 일시 정지

### 접근성 (A11y)
- 카운트다운 `aria-live="polite"` `aria-label="재시도까지 N초"`
- 재시도 버튼 `aria-label="서비스 연결 재시도"`
