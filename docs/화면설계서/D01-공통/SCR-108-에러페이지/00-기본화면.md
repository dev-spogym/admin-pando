> 이 문서는 **화면 마스터 스펙**입니다. `01~04` 상태 문서(404/403/500/503)는 이 문서를 상속(override/delta)합니다.

## 1. 화면 목적 (Why)

예측 못한 상태(요청 오류/권한 없음/서버 장애/점검) 발생 시 **사용자가 길을 잃지 않게** 명확히 안내하고, 가능한 복구 경로를 제시한다.
- 404/403: 클라이언트 행동(잘못 입력/권한 부족)의 결과 안내 + 홈 복귀.
- 500: 서버 문제. 재시도 및 문의 경로 제공.
- 503: 예정된 점검. 카운트다운 + 자동 재시도.
- 모든 케이스에서 레이아웃/세션 유지, 역할별 복귀 대시보드 분기.

## 7. 데이터 계약

### 7.1 Props 타입

```ts
// src/components/error/ErrorPage.tsx
export type ErrorVariant = '404' | '403' | '500' | '503';

interface ErrorPageProps {
  variant: ErrorVariant;
  errorCode?: string;        // 'E403001', 'E500001' ...
  requestId?: string;        // x-request-id (서버 응답 헤더에서)
  reset?: () => void;        // Next.js error boundary reset
  eta?: number;              // 503 점검 예상 잔여 초
  redirectPath?: string;     // 로그인 후 복귀 경로(403에서 /login 이동 시)
}
```

### 7.2 Next.js App Router 바운더리

| 파일 | 역할 | 기본 variant |
|---|---|---|
| `src/app/not-found.tsx` | 매칭 실패 라우트 | `404` |
| `src/app/error.tsx` | 런타임 에러 (500 포함) | `500` |
| `src/app/(error)/forbidden/page.tsx` | 수동 리다이렉트 | `403` |
| `src/app/(error)/maintenance/page.tsx` | 503 점검 | `503` |

```tsx
// src/app/error.tsx
'use client';
import { ErrorPage } from '@/components/error/ErrorPage';
export default function Error({ error, reset }: { error: Error & { digest?: string }; reset: () => void }) {
  // Sentry 전송
  useEffect(() => { captureException(error, { tags: { route: 'error-boundary' } }); }, [error]);
  return <ErrorPage variant="500" errorCode="E500001" requestId={error.digest} reset={reset} />;
}
```

### 7.3 503 카운트다운

```ts
// 헬스체크 엔드포인트
GET /api/health
→ 200 { status: 'ok' }
→ 503 { status: 'maintenance', etaSeconds: 120 }

// 30초마다 재시도
useEffect(() => {
  const id = setInterval(async () => {
    const r = await fetch('/api/health');
    if (r.ok) window.location.reload();
  }, 30_000);
  return () => clearInterval(id);
}, []);
```

### 7.4 상태 관리
- 전역 스토어 불필요 (변수 없음; 각 바운더리 파일에서 variant/props 주입).
- 403 복구 목적지는 `useAuthStore(s => s.user?.role)` 로 계산.

## 8. 비즈니스 룰

1. **단일 컴포넌트 + variant**: 4가지 상태 모두 `ErrorPage` 단일 컴포넌트의 variant prop 으로 분기. 레이아웃/애니메이션 통일.
2. **404 우선순위**: Next.js 매칭 실패 → `not-found.tsx` 자동. 명시적 `notFound()` 호출 시에도 동일.
3. **403 복귀 분기**: 역할별 대시보드로 이동. `useAuthStore` 미초기화 시 `/login` 으로.
4. **500 전송**: `error.tsx` 에서 `captureException` 즉시 실행. `error.digest` 를 `requestId` 로 노출.
5. **503 자동 재시도**: 30초 주기 헬스체크. 성공 시 `window.location.reload()` 로 원래 페이지 복귀 시도.
6. **에러코드 복사**: `errorCode + timestamp + requestId` 를 한 줄로 모아 클릭 시 클립보드 복사 + 토스트 "참조 코드가 복사되었습니다".
7. **뒤로가기 동작**: `history.length > 1` 일 때만 노출, 아니면 "대시보드로"로 대체.
8. **세션 만료 우선**: 401 인터셉터가 먼저 DLG-000 오픈. 이 페이지로 오지 않음.
9. **감사 로그**: 403 발생 시 `AUDIT.ACCESS_DENIED` (path, userId, role, attemptedResource).
10. **SEO**: 404/403 페이지는 `robots: noindex`, 500/503 도 `noindex`.
11. **비로그인 케이스**: 403에서 `!user` 이면 자동 `/login?redirect=<path>` 로 2초 후 리다이렉트.
12. **reset() 훅**: Next.js `error.tsx` 의 `reset` prop 을 "재시도" 버튼에 연결 (500 variant).

## 12. 진입/이탈 연결

### 진입
- Next.js 라우트 매칭 실패 → `not-found.tsx` (404)
- 컴포넌트 `throw` → `error.tsx` (500)
- 권한 가드 실패 → `router.push('/forbidden')` 수동 이동 (403)
- API 503 + 전역 인터셉터 → `router.push('/maintenance')` (503)
- 비로그인 상태에서 보호 라우트 접근 → `/login?redirect=...` (이 페이지 거치지 않고 로그인으로)

### 이탈
| variant | 액션 | 목적지 |
|---|---|---|
| 404 | "대시보드로" | 역할별 대시보드 |
| 404 | "뒤로가기" | `router.back()` |
| 403 | "대시보드로" | 역할별 대시보드 |
| 403 | (비로그인) | `/login?redirect=<path>` |
| 500 | "재시도" | `reset()` (같은 페이지 재렌더) |
| 500 | "문의하기" | `mailto:support@fitgenie.com?subject=E500001+reqId` |
| 503 | 자동 재시도 성공 | `window.location.reload()` |

  requestId?: string;

export function ErrorPage({ variant, errorCode, requestId, reset, eta }: Props) {
  const router = useRouter();
  const role = useAuthStore(s => s.user?.role);
  const homePath = getRoleDashboard(role);
  const C = CONTENT[variant];
  const ts = new Date().toISOString().slice(0,19).replace('T',' ');

  // 503 자동 재시도
  const [remain, setRemain] = useState(eta ?? 30);

        const next = r - 1;
