## 상태 진입 조건

- 페이지 마운트 직후 `useBodyCompositions(memberId)` pending + 캐시 없음

## 역할별 차이

- 모든 접근 가능 역할(super/primary/owner/manager/fc/trainer) 동일 스켈레톤.
- `canAdd(role)=false` 역할(현재는 없음 — trainer도 가능)은 측정 추가 스켈레톤 미렌더.
- staff/front/readonly: 이 상태 미도달 (F1 진입 단계에서 `/forbidden` 리다이렉트).

## 상태 고유 인터랙션

| 액션 | 동작 |
|---|---|
| MemberSelect 변경 | 기존 fetch abort + 새 fetch 시작 (AbortController), 스켈레톤 유지 |
| + 측정 추가 클릭 | 무시 (disabled) |
| 메트릭 토글 | 무시 |
| 테이블 행 클릭 | 무시 |
| ← 뒤로가기 | 즉시 이탈 (fetch abort) |

## 상태 고유 비즈니스 룰 델타

1. **캐시 정책**: 첫 진입 → 스켈레톤. 같은 memberId 재진입 + 캐시 fresh(`staleTime:30s`) → 즉시 렌더 후 background refetch (이 경우 01이 아닌 02~04 직진).
2. **loadTimeout = 10s** 초과 시 `toast.info('서버 응답이 지연되고 있습니다')`.
3. **병렬 쿼리**: `useBodyCompositions` + `useMember(memberId)` 동시 발사. 둘 다 pending이면 스켈레톤.
4. **DLG 저장 후 refetch** 중에는 `isFetching && !isLoading` → 스켈레톤이 아니라 우측 상단 미니 `<Loader2 className="animate-spin w-4 h-4 text-gray-400" />` 만 표시.
5. `prefers-reduced-motion: reduce` → `animate-pulse` 제거, 정적 회색 블록 유지.

## 에지 케이스

- 401 즉시 실패 → 전역 인터셉터로 `/login?redirect=/body-composition?memberId=...`
- 403 즉시 실패 → `/forbidden`
- 오프라인(`navigator.onLine=false`) → 스켈레톤 유지 + 상단 "오프라인 상태입니다" 인포 배너
- memberId 미지정 → 스켈레톤이 아닌 MemberSelect 프롬프트 상태 (이는 00 기본화면 안내)

// 저장 후 background refetch 스피너
{isFetching && !isLoading && (
