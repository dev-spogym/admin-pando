## 상태 고유 비주얼 델타

### 성공 (201)
- Toast(success): `"출석이 기록되었습니다"`
- 버튼: idle 복귀 (emerald 기본)
- 부모 출석 탭: 신규 행 prepend (Realtime 또는 refetch)
- 프로필 헤더 "오늘 출석" 배지 업데이트

## 상태 고유 인터랙션

1. **성공 토스트 후**: 버튼 자동 idle 복귀 (1초 내)
2. **실패 토스트**: 에러 코드별 메시지
3. **재시도**: idle 복귀 후 다시 클릭 가능 (단 E422101/E422401 등은 disabled 유지 필요 → 부모가 판단)
4. **포커스**: 버튼에 포커스 유지

## 역할별 차이

- 모든 역할 동일

## 상태 고유 비즈니스 룰 델타

- **감사로그**: 성공/실패 모두 서버 기록. 실패는 `AUDIT.ATTENDANCE_MANUAL_FAILED` (errorCode 포함)
- **SCR-I001 Realtime 반영**: 성공 시 `attendances` 채널에 INSERT 이벤트 발생 → 통합출석 화면 즉시 업데이트
- **성공 연계 부가 액션**: 없음 (단순 출석 기록만). 만약 "쿠폰 자동 발급" 같은 후속 작업이 필요하면 서버 훅에서 처리

## 에지 케이스

- **응답 이후 회원 상태 변경**: UI는 부모 데이터 기반이므로 다음 refetch에서 반영
- **토스트 중복**: 연속 클릭 시 마지막 토스트만 유지 (`toast.dismiss()` 후 새 토스트)
- **성공 후 네트워크 끊김**: 이미 성공 응답 받음 → idle 복귀, 다음 refetch만 지연

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: manual-attend-result

onSuccess: () => {
  qc.invalidateQueries({queryKey:['attendances', memberId]});
  qc.invalidateQueries({queryKey:['member', memberId]});
  toast.success('출석이 기록되었습니다');
  onSuccess?.();
},

onError: async (e:any) => {
  const err = await (typeof e?.then === 'function' ? e : Promise.resolve(e));
  const code = err?.errorCode ?? 'E500001';
  const msg = {
    E403400: '오늘 이미 출석하셨습니다',
    E403401: '영업시간이 아닙니다',
    E404400: '회원을 찾을 수 없습니다',
    E422400: '유효한 이용권이 없습니다',
    E422401: '이용이 정지된 회원입니다',
    E422101: '탈퇴한 회원입니다',
    E403001: '권한이 없습니다',
    E500001: '일시적 오류가 발생했습니다',
  }[code] ?? '출석 처리에 실패했습니다';
  toast.error(msg);
  // E401002는 전역 인터셉터가 DLG-000 처리
},
```
