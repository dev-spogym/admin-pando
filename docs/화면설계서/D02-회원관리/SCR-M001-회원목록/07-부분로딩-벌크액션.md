### Progress 표시
- 동기 일괄: "처리 중... N명" + Loader2
- 비동기 청크: "처리 중... x/y (xx%)" + Progress bar
- 구현: `useMutation` + onMutate + progress 콜백

## 역할별 차이

- 동일한 UX. 역할은 어떤 버튼이 enabled인지만 차이.
- readonly: BulkActionBar 자체 비노출 → 본 상태 미도달.

## 상태 고유 비즈니스 룰 델타

1. **트랜잭션**: 서버 단일 트랜잭션 `PATCH /members/bulk/status`. 부분 실패 정책 택 1:
   - 전체 롤백
   - 성공/실패 건수 리턴 (현재 기본)
2. **청크 처리**: 1000명 초과 시 500명씩 순차.
3. **응답 분기**:
   - 전체 성공 → `toast.success("${n}명의 상태를 '${label}'(으)로 변경했습니다.")` + refetch + clearSelection
   - 부분 성공 → `toast.warning("${s}명 성공, ${f}명 실패")` + `FailuresModal`
   - 전체 실패 → `toast.error` + 선택 유지 + `05-에러` 배너
4. **낙관적 업데이트**(선택): 대상 행 즉시 회색 처리 → 실패 시 롤백
5. **Audit**: `AUDIT.BULK_STATUS_UPDATE` (actorId, count, from, to)
6. **중복 제출 방지**: `isPending` + request id로 서버 idempotent
7. **타임아웃**: 60s. 초과 시 "처리가 오래 걸립니다. 새로고침 해주세요" 안내 (백그라운드 지속).

## 에지 케이스

- 동시성 충돌 (E409101) → 실패 목록 모달.
- 대상 soft-delete → 서버 건너뜀 + 실패 포함.
- 처리 중 권한 박탈(403) → 중단 → `06-권한없음`.
- 네트워크 단절 중 제출 → 즉시 실패 + 선택 유지 (재시도 가능).
- 선택 후 refetch로 id 유실 → 서버 검증 결과만 토스트.

## 🧩 바이브코딩 프롬프트 (델타)

```
※ 델타: 벌크 pending UX + 경고 모달

const bulkStatus = useMutation({
  mutationFn: (v: {ids:number[]; status:MemberStatus}) => bulkUpdateStatus(v),
  onMutate: (v) => {
    const prev = queryClient.getQueryData(['members', params]);
    queryClient.setQueryData(['members', params], (old:any) => ({
      ...old,
      data: old.data.map((m:any) => v.ids.includes(m.id) ? { ...m, _pending: true } : m)
    }));
    return { prev };
  },
  onError: (_e, _v, ctx) => {
    queryClient.setQueryData(['members', params], ctx?.prev);
    toast.error('상태 변경에 실패했습니다.');
  },
  onSuccess: (res, v) => {
    if (res.failures?.length) {
      toast.warning(`${res.successes}명 성공, ${res.failures.length}명 실패`);
      openFailuresModal(res.failures);
    } else {
      toast.success(`${v.ids.length}명의 상태를 '${LABEL[v.status]}'(으)로 변경했습니다.`);
    }
    setSelectedRows(new Set());
    refetchAll();
  },
});

<div role="toolbar" aria-live="polite"
     className="fixed bottom-4 left-1/2 -translate-x-1/2 z-40
                flex items-center gap-3 bg-gray-900 text-white
                rounded-full shadow-lg px-4 h-12">
  {bulkStatus.isPending ? (
    <>
      <Loader2 className="w-4 h-4 animate-spin" />
      <span className="text-sm">처리 중... ({done}/{total})</span>
    </>
  ) : (
    <span className="text-sm">{selected.size}명 선택됨</span>
  )}
  <Button disabled={bulkStatus.isPending} onClick={openDLG_M001}>상태 변경</Button>
  <Button disabled={bulkStatus.isPending} onClick={goToMessageSend}>전송하기</Button>
  <Button disabled={bulkStatus.isPending} onClick={() => bulkAttend.mutate({ids})}>출석 처리</Button>
  <Button disabled={bulkStatus.isPending} onClick={() => bulkFavorite.mutate({ids})}>관심회원</Button>
  <Button variant="ghost" disabled={bulkStatus.isPending} onClick={clearSelection}>선택 취소</Button>
</div>

<tr className={cx('transition-opacity',
  row._pending && 'opacity-60 animate-pulse pointer-events-none')}>

</tr>
```
