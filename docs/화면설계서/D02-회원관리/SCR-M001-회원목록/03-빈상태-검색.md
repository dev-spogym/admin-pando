## 상태 진입 조건

- `getMembers` 200 + `data.length === 0`
- **필터 조건 1개 이상 활성**: `search !== ''` OR `statusTab !== 'all'` OR 필터 중 기본값 아닌 항목 1개 이상
- 전체 필터 해제 시엔 `04-빈상태-전체`로 분기

## 역할별 차이

- fc: "담당 회원에서 찾을 수 없습니다." (role=fc && staffId=me일 때)
- primary/super: 지점 전환이 필터에 포함되면 "해당 지점에 결과가 없습니다" 추가 안내

## 상태 고유 인터랙션

| 액션 | 동작 |
|---|---|
| [필터 초기화] | 모든 filter/search/statusTab/sort 리셋 → URL 초기화 → `01-로딩` |
| [검색어 지우기] | search만 ""로 초기화, 필터 유지 → `01-로딩` |
| chip × 클릭 | 해당 필터만 해제 → `01-로딩` |
| 검색어 재입력 | 300ms 디바운스 → `01-로딩` |
| StatusTab 변경 | `01-로딩` |
| StatCard 클릭 | 해당 탭 전환 |

## 상태 고유 비즈니스 룰 델타

1. 활성 필터 판정: 기본값과 다른 값이면 활성.
2. 활성 필터 chip 최대 5개 노출, 초과 시 "+N" 더보기.
3. [필터 초기화] 시 sort도 `createdAt DESC`로 리셋.
4. `role="status" aria-live="polite"` "검색 결과가 없습니다." 공지.
5. URL 쿼리스트링 유지 — 뒤로가기로 이전 검색 복원.
6. analytics `search_no_result` 이벤트 기록 (query, filterSnapshot).

## 에지 케이스

- 공백 검색어 trim → 필터 미적용 효과 → `04-빈상태-전체` 또는 정상목록.
- 긴 검색어(>50자) → `maxLength=50` 강제.
- 지점 전환 직후 빈 지점 문구 확장.
- 한국어 초성 미지원 시 그대로 빈 상태.
