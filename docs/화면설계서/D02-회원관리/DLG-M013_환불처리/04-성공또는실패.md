## 상태 진입 조건
- `mutation.isPending === false`
- `mutation.isSuccess` or `mutation.isError`

## 상태 고유 비주얼 델타

### 성공 (SUCCESS)
- 토스트 `toast.success("환불 처리가 완료되었습니다.")` 3초
- 모달 즉시 닫힘
- 결제이력 테이블에서 해당 행 `status` 배지 "환불" 로 갱신
- 환불 레코드(음수) 신규 행 표시

### 실패 (FAIL)
- 치명(409/404/403) → 닫기 + 토스트 + refetch
- 복구(500/503/NET) → 유지 + 토스트 + 버튼 활성 복귀

## 상태 고유 인터랙션

### 성공
- 자동 정리 (액션 없음)

### 실패 복구
1. **재시도**: "환불 처리" 재클릭 → `03-제출중`
2. **취소**: 닫힘
3. **치명 자동 닫기**: 2초 후 onClose

## 상태 고유 비즈니스 룰 델타

### 성공 후처리
```ts
onSuccess: () => {
  qc.invalidateQueries({ queryKey: ['sales', memberId] });
  qc.invalidateQueries({ queryKey: ['member', memberId] });
  qc.invalidateQueries({ queryKey: ['sales-kpi'] });    // D03 대시보드
  qc.invalidateQueries({ queryKey: ['sales-monthly'] }); // D03 월간
  toast.success('환불 처리가 완료되었습니다.');
  onClose();
}
```

### 실패 매핑
| errorCode | 액션 |
|---|---|
| E400301 | 토스트 "환불 금액 확인" + 유지 |
| E403001 | 토스트 "환불 권한 없음" + 닫기(2s) |
| E404300 | 토스트 "매출 없음" + refetch + 닫기 |
| E409300 | 토스트 "이미 환불된 매출" + refetch + 닫기 |
| E500001 | 토스트 "일시 오류" + 유지 |
| E503001/E503002 | 토스트 "결제 서비스 오류" + 유지 |
| NETWORK | 토스트 + 유지 |
| E401002 | DLG-000 우선, 본 모달 정리 |

### 알림(옵션)
- 성공 시 회원에게 "환불 완료" 알림톡 발송 이벤트 발행

## 역할별 차이
- `manager`: 500k 초과 403 시 "센터장 승인 필요" 메시지 커스터마이즈
- `super/primary`: 원문 메시지

## 에지 케이스
- **PG 성공 + DB 실패(희박)**: 서버가 PG 환불 취소 보상 트랜잭션. 클라이언트는 `E500001` 수신
- **트랜잭션 부분 실패**: 서버가 전체 롤백 → `E500001`
- **응답 body 누락**: 안전 모드 refetch + 닫기

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14
상태: refund-done

onSuccess(data):
  invalidateQueries(['sales', memberId])
  invalidateQueries(['member', memberId])
  invalidateQueries(['sales-kpi'])
  invalidateQueries(['sales-monthly'])
  toast.success('환불 처리가 완료되었습니다.')
  onClose()

onError(e):
  switch (e.errorCode) {
    case 'E403001': toast.error('환불 권한이 없습니다'); setTimeout(onClose, 2000); break;
    case 'E404300': toast.error('매출을 찾을 수 없습니다'); invalidateQueries(['sales', memberId]); onClose(); break;
    case 'E409300': toast.error('이미 환불 처리된 매출입니다'); invalidateQueries(['sales', memberId]); onClose(); break;
    case 'E503001':
    case 'E503002': toast.error('결제 서비스에 일시적인 문제가 발생했습니다'); break;
    default: toast.error(e.message ?? '환불 처리 실패');
  }
```
