## 상태 고유 인터랙션

### 공통
- X/Esc → 모달 닫기

### success 전용
- **리다이렉트**: single 모드 → `router.push('/members')`
- **bulk 모드**: 부모 화면 `selectedRows` 초기화 + 목록 refetch
- **토스트**: 4초 자동 해제

### error 전용
- "다시 시도" → `mutation.mutate()` 재호출 (confirmText 유지)
- "취소" → 모달 닫기

## 비즈니스 룰 델타
1. **성공 시 감사 로그**: `AUDIT.MEMBER_DELETE` 확정 기록 (memberId, userId, branchId, deletedAt)
2. **실패 시 롤백 없음**: 서버에서 트랜잭션 롤백하므로 UI는 상태 복원만 처리
3. **errorCode별 분기**:
   - E403001 → 토스트 + 모달 닫기 (권한 없음, 재시도 무의미)
   - E404100 → 부모 refetch + 모달 닫기 (이미 삭제됨)
   - E422101 → 인라인 배너 (탈퇴 회원은 별도 경로 필요)
   - E500001 / NETWORK → 재시도 버튼 유지

## 에지 케이스
- **성공 후 뒤로가기**: `/members` 유지 (브라우저 히스토리 replace 권장)
- **bulk 부분 실패**: 복수 삭제는 DLG-M002가 아닌 BulkActionBar 경로 사용 → 해당 DLG 별도 처리
- **토스트 스팸**: 재시도 실패 N회 → 토스트 대신 인라인 에러로 전환
- **쿼리 캐시**: `queryClient.removeQueries({ queryKey: ['members', member.id] })` (상세 캐시 제거)

## 바이브코딩 프롬프트 (델타)
```
mutation.onSuccess = () => {
  toast.success('회원이 삭제되었습니다.');
  queryClient.invalidateQueries({ queryKey: ['members'] });
  queryClient.removeQueries({ queryKey: ['members', member.id] });
  onDeleted?.();
  onOpenChange(false);
  if (mode === 'single') router.replace('/members');
};
mutation.onError = (err) => {
  const code = err.code ?? (navigator.onLine ? 'E500001' : 'NETWORK');
  setErrorCode(code);
  if (code === 'E403001' || code === 'E404100') {
    toast.error(ERROR_MAP[code]);
    onOpenChange(false);
    if (code === 'E404100') queryClient.invalidateQueries({ queryKey: ['members'] });
  }
};
재시도 버튼 (error):
  <button onClick={handleDelete} ... >🗑 다시 시도</button>
```
