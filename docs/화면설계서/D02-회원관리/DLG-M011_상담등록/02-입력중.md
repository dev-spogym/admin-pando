## 상태 진입 조건
- 사용자가 최소 1개 필드 변경 → `form.formState.isDirty=true`

## 상태 고유 비주얼 델타
- 필드 에러 실시간 노출: `aria-invalid="true"` + `text-xs text-rose-600`
- 필수 미충족 시 "저장" 버튼 `disabled + bg-blue-300`
- 검증 통과 시 "저장" 활성 `bg-blue-600`
- pill 상태 버튼: 선택된 항목 `bg-primary text-white`, 나머지 `border-gray-300`
- 연결 매출 옵션 로드 완료 (5건~50건)

## 상태 고유 인터랙션
1. **필드 입력/변경**: `onChange` → `form.trigger(name)` 실시간 검증
2. **pill 키보드**: `←→` 이동 / `Space` 선택 / `Tab` 다음 필드로
3. **저장 클릭**:
   - 검증 통과 → `03-제출중` 전환
   - 검증 실패 → 첫 에러 필드로 포커스, 모달 유지
4. **취소/ESC**: dirty 상태이므로 DLG-002 이탈경고 오픈 → 확정 시 닫힘

## 상태 고유 비즈니스 룰 델타
- **필수 3개**: `consultedAt`, `type`, `status` 중 1개라도 비어있으면 "저장" 비활성
- **내용/후속조치 글자수**: 각 2,000자 / 1,000자 초과 시 인라인 에러
- **연결 매출 동기화**: 매출 선택 시 `linkedSaleId` 즉시 반영
- **pill 상태 전환 시 결과 옵션 힌트**: 상태=완료 & 유형=상담 → 결과 default suggest "등록" (UI only, 자동 선택 금지)

## 역할별 차이
- `fc`: 담당자 필드 readonly, 값 변경 시도 시 인라인 힌트 "담당자 고정"
- `manager+`: 담당자 search/select UI (옵션)

## 에지 케이스
- **입력 중 sales 재로드**: 옵션 순서 바뀌면 선택한 값 유지(id 매칭)
- **과거 시간**: `consultedAt < 3개월 전` 시 경고 배너 `bg-amber-50` (서버가 거부할 수 있음)
- **다른 탭 동일 회원 편집**: 현재 폼 유지, 서버 version 체크는 저장 시점
