## 상태 진입 조건
- `status !== undefined` (라디오 하나 이상 선택)
- WITHDRAWN 선택 시: 경고 배너 표시
- 확인 버튼 `enabled`

| 취소 버튼 | 동일 | — |

## 역할별 차이
- 마스터 §5와 동일. `manager`가 WITHDRAWN 선택은 옵션 미노출로 불가능.
- `owner`도 WITHDRAWN 선택 가능 (§7 전이 규칙 준수).

## 상태 고유 인터랙션
1. **다른 RadioItem 클릭** → `setStatus(newValue)` → `02-선택중` 유지 (내부 전이)
2. **textarea 입력** → `setReason(value)` → UI 변경 없음
3. **"변경 확인" 클릭** → 유효성 검증 → `03-제출중`
4. **Esc/취소/X** → 모달 닫기 (제출 전이므로 허용)

## 상태 고유 비즈니스 룰 델타
1. WITHDRAWN 선택 시 경고 배너 즉시 노출 (CSS transition)
2. WITHDRAWN 선택 시 `reason` 필드에 포커스 자동 이동 (사유 권장 UX, 강제는 아님)
3. 동일 상태(예: ACTIVE) 선택 시에도 버튼 활성 (서버가 no-op 처리, 감사로그 기록)
4. `reason.length > 500` 시 textarea `maxLength` 하드 제한

## 에지 케이스
- **빠른 연속 클릭**: 라디오 간 변경은 즉시 반영, 버튼 스팸 클릭은 `isSubmitting` 가드
- **선택 후 selectedIds 변경**: 부모 컴포넌트에서 `selectedRows` 변경 시 `selectedIds.length` 재확인
- **WITHDRAWN 선택 후 manager 역할 복귀 시**: 옵션 자체 제거되므로 `status=undefined`로 자동 리셋

## 바이브코딩 프롬프트 (델타)
```
상태: selected
조건: status !== undefined
UI 변경:
  - 선택된 radio: border-{opt.danger ? 'red-300 bg-red-50' : 'blue-600 bg-blue-50'}
  - 확인 버튼: disabled={false} → bg-blue-600
  - status==='WITHDRAWN' → <WarnBanner />
handleSubmit:
  if (!status) return;
  const parsed = bulkStatusChangeSchema.safeParse({ memberIds: selectedIds, status, reason });
  if (!parsed.success) { setErrorCode('E400001'); return; }
  setIsSubmitting(true);
  mutation.mutate(parsed.data);
```
