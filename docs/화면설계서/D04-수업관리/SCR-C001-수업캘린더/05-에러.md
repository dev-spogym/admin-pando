## 역할별 차이
모든 역할 동일. 재시도·뒤로 버튼은 모든 역할에서 노출.

## 상태 고유 인터랙션
1. **[🔄 재시도]**: `refetch()` 호출 → `01-로딩` 전환
2. **[← 뒤로]**: `router.back()` 호출 (또는 `/` 대시보드)
3. **10초 후 자동 재시도 (옵션)**: `retry: 3, retryDelay: exponentialBackoff`
4. **TabNav 클릭**: 다른 탭 이동 가능 (화면 이탈)
5. **오프라인 감지**: 에러 중 `navigator.onLine=false` → `06-오프라인` 전환

## 상태 고유 비즈니스 룰 델타
- **에러 코드 매핑**:
  - E500001 → "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요"
  - E503001 → "서비스 점검 중입니다. 잠시 후 다시 시도해주세요" (warning 톤)
  - E404500 → "수업을 찾을 수 없습니다" (위젯 수준, 전체 에러는 아님)
  - TIMEOUT → "응답 시간이 초과되었습니다"
  - NETWORK → `06-오프라인`으로 전환
  - 기타 → "오류가 발생했습니다"
- **Sentry 전송**: `captureException(err, { tags: { screen:'SCR-C001' } })`
- **재시도 쿨다운**: 3초 (스팸 방지)

## 에지 케이스
- E401 (세션 만료): 전역 인터셉터 → `/login?redirect=/calendar`
- E403 (권한): `/forbidden`
- 부분 실패 (events만 성공, staff 실패): 전체 에러 아님, 필터 드롭다운만 비활성
- 재시도 무한 실패: 3회 후 "관리자 문의" 강조
- 503 점검: 에러 톤 warning (amber)
