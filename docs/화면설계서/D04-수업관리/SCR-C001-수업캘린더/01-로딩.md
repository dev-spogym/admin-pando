## 역할별 차이
모든 역할 동일. trainer는 로그인 직후 `/calendar`로 redirect되는 시점부터 이 상태가 즉시 시작됨.

## 상태 고유 인터랙션
1. 등록/일괄변경 버튼 비활성 (`pointer-events-none aria-disabled=true`)
2. TabNav 정상 (다른 화면 이탈 가능)
3. ESC 키 동작 없음 (네트워크 자연 대기)
4. 10초 초과 시 slow network info 배너 "네트워크가 느립니다..."
5. 30초 초과 시 자동으로 `05-에러`로 전환

## 상태 고유 비즈니스 룰 델타
- Query key: `['classes', branchId, from, to, filters]`
- **중복 요청 방지**: React Query dedup
- **SWR 패턴**: `placeholderData: keepPreviousData` → 이전 데이터 있으면 그대로 유지 + 상단 `<ProgressBar indeterminate />`
- 풀 스켈레톤은 `initialData=undefined`일 때만
- 새로고침 중에도 60초 쿨다운 유지

## 에지 케이스
- 쿼리 4개 중 일부만 완료: progressive 렌더 (완료 섹션 먼저)
- placeholderData 있음: 스켈레톤 대신 기존 데이터 + 상단 progress
- 10초 초과: info 배너
- 30초 초과: 자동 에러 전환 `errorCode: 'TIMEOUT'`

## 🧩 바이브코딩 프롬프트 (델타)
```

상태: calendar-loading

델타:

    ))}
  </div>
- 우측 패널:
  <div className="space-y-2">
    {Array.from({length:3}).map((_,i)=>(
      <div key={i} className="h-16 bg-gray-100 rounded-md animate-pulse" />
    ))}
  </div>
- 버튼 disabled:
  <Button disabled aria-disabled>+ 수업 등록</Button>
- placeholderData:
  useQuery({ queryKey, queryFn, placeholderData: keepPreviousData })
  → 이전 데이터 있으면 <ProgressBar indeterminate />
- 10초 타임아웃:
  useEffect(() => {
    if (!isLoading) return;
    const t = setTimeout(() => setSlowNetwork(true), 10000);
    return () => clearTimeout(t);
  }, [isLoading]);
- 30초 → 에러:
  onError: (e) => setErrorCode(e.code ?? 'TIMEOUT')
- reduced-motion: animate-pulse 제거
```
