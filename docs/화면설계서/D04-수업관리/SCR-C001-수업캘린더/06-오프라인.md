## 역할별 차이
모든 역할 동일 (오프라인은 권한과 무관).
캐시 데이터는 각 역할별 이전 view(branchId, filter)만 보존.

## 상태 고유 인터랙션
1. **[네트워크 재시도]** / **[🔄 다시 시도]**: `fetch /health` 시도 → 성공 시 01 전환
2. **편집 시도**: 모든 mutation → toast "오프라인 상태입니다. 네트워크 복구 후 다시 시도해주세요."
3. **online 이벤트 감지**: 자동으로 `refetch()` + 토스트 "네트워크가 복구되었습니다."
4. **TabNav 클릭**: 다른 탭 이동 가능 (해당 탭도 오프라인 상태로 렌더)

## 상태 고유 비즈니스 룰 델타
- **Service Worker 캐시**: `next-pwa` 또는 Workbox `StaleWhileRevalidate` 전략
- **React Query persistQueryClient**: localStorage 또는 IndexedDB 저장 → 재진입 시 stale 데이터 복원
- **mutation 차단**: `onMutate`에서 `navigator.onLine` 체크 → 즉시 실패 반환 + toast
- **캐시 타임스탬프**: `query.dataUpdatedAt` 활용하여 "캐시: HH:mm" 표시
- **자동 재연결**: `window.addEventListener('online')` → 5초 debounce 후 `refetch()`
- **오프라인 토스트**: `navigator.onLine=false` 진입 시 1회 표시

## 에지 케이스
- Service Worker 미지원 브라우저: 캐시 없음 → 케이스 B
- 반복된 online/offline 플래핑: 5초 debounce로 토스트 스팸 방지
- 캐시 데이터 24시간 초과: "캐시가 오래되었습니다" 추가 배너
- 오프라인 중 탭 전환: 모든 SCR에서 동일 오프라인 배너
- 오프라인 → 에러(5xx) 전환: 네트워크 복구됐지만 서버 문제 → `05-에러`

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14
상태: calendar-offline

델타:
- 네트워크 감지:
  const [online, setOnline] = useState(() => typeof navigator!=='undefined' ? navigator.onLine : true);
  useEffect(() => {
    const onOn = () => setOnline(true);
    const onOff = () => setOnline(false);
    window.addEventListener('online', onOn);
    window.addEventListener('offline', onOff);
    return () => {
      window.removeEventListener('online', onOn);
      window.removeEventListener('offline', onOff);
    };
  }, []);

- 자동 재조회 (debounce):
  useEffect(() => {
    if (!online) return;
    const t = setTimeout(() => {
      qc.invalidateQueries({ queryKey:['classes'] });
      toast.info('네트워크가 복구되었습니다.');
    }, 5000);
    return () => clearTimeout(t);
  }, [online]);

- 캐시 체크:
  const cachedData = qc.getQueryData<ScheduleEvent[]>(['classes', branchId, from, to, filters]);
  const lastUpdated = qc.getQueryState(['classes',...])?.dataUpdatedAt;

- 배너 (케이스 A):
  {!online && cachedData && (
    <div role="status" aria-live="polite"
         className="flex items-center justify-between gap-2 rounded-lg

      <div className="flex items-center gap-2 text-sm">

      </div>
      <Button size="sm" variant="ghost" onClick={()=>qc.invalidateQueries(['classes'])}>
        네트워크 재시도
      </Button>
    </div>
  )}

- 전체 빈 오프라인 (케이스 B):
  {!online && !cachedData && (
    <div role="alert" className="bg-white rounded-xl shadow-sm ring-1 ring-gray-100 p-10 text-center space-y-3">
      <WifiOff className="size-12 mx-auto text-slate-400" aria-hidden />
      <h2 className="text-base font-semibold">네트워크 연결을 확인해주세요</h2>
      <p className="text-sm text-gray-600">인터넷 연결이 복구되면 자동으로 다시 시도합니다.</p>
      <Button onClick={()=>qc.invalidateQueries(['classes'])}>다시 시도</Button>
    </div>
  )}

- mutation 차단:
  const guardedMutation = (fn: ()=>Promise<any>) => async (...args: any[]) => {
    if (!navigator.onLine) {
      toast.warning('오프라인 상태입니다. 네트워크 복구 후 다시 시도해주세요.');
      return;
    }
    return fn(...args);
  };

- FullCalendar 오프라인:
  editable={online && canCreateClass(role) && !isViewOnly(role)}

- React Query persist (앱 전역):
  import { persistQueryClient } from '@tanstack/react-query-persist-client';
  import { createSyncStoragePersister } from '@tanstack/query-sync-storage-persister';
  persistQueryClient({
    queryClient: qc,
    persister: createSyncStoragePersister({ storage: window.localStorage }),
    maxAge: 1000*60*60*24, // 24h
  });
```
