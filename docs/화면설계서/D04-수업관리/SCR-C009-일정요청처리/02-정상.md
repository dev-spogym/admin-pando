## 상태 고유 비즈니스 룰 델타

1. **낙관적 업데이트 금지**: 승인 시 서버에서 시간 충돌/정원 초과 검증이 필요하므로 응답 후 refetch.
2. **pending 외 행 액션 숨김**: `status !== 'pending'`은 서버가 거부하므로 UI에서도 숨긴다.
3. **행 정렬**: 기본 createdAt DESC. 사용자가 컬럼 헤더 클릭 시 서버 정렬 토글(asc/desc).
4. **pendingCount 갱신**: mutation 성공 후 배너도 즉시 감소(optimistic). 실패 시 원복.
5. **highlight 딥링크**: `?highlight={id}` 있으면 자동 스크롤 + 2초 노란 강조 → 쿼리 제거.
6. **30초 무상호작용**: 포커스/마우스 움직임 없는 30초 후 1회 자동 refetch(silent, 인디케이터만).

## 에지 케이스

- **다른 탭에서 동시 승인**: 서버가 409 E409500 → 행 refetch + toast.warning "이미 처리된 요청입니다"
- **승인 도중 세션 만료**: 401 → 전역 인터셉터 → `/login?redirect=/schedule-requests`
- **정원 초과**: 400 E400502 → toast.error + "대기열로 이동" 액션 버튼 (`/waitlist?classId=${...}`)
- **시간 충돌**: 400 E400501 → toast.error + "캘린더 보기" 액션 (`/calendar?date=${...}`)
- **pageSize 변경 시 페이지 초과**: 서버가 items=[] 반환 → `03-빈상태` 전환하지 않고 현재 상태 유지 + "마지막 페이지"로 자동 이동.
- **장시간 유휴 후 복귀**: `refetchOnWindowFocus=true`로 재 fetch, 단 이 상태 유지(스켈레톤 미표시).

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: normal (정상)

조건 렌더:

trainer 본인 외 행 처리:
if (role === 'trainer' && row.targetStaffId !== userId) {
  // 회원명 회색
  // 액션 컬럼 ''
}

행 pending(mutation) 표시:
<tr className={cn(
  'hover:bg-gray-50 transition-colors',
  mutatingId === row.id && 'opacity-70 cursor-wait',
  highlightId === row.id && 'bg-yellow-50 ring-1 ring-yellow-200'
)}>

RefetchIndicator:
isFetching && data && (
  <div aria-hidden className="h-0.5 bg-blue-500/70 animate-pulse" />
)

approve mutation:
const approveMut = useMutation({
  mutationFn: (id:number) => api.patch(`/api/schedule-requests/${id}/approve`),
  onMutate: (id) => setMutatingId(id),
  onSuccess: () => {
    toast.success('일정이 승인되었습니다.');
    qc.invalidateQueries(['schedule-requests']);
  },
  onError: (err) => {
    const code = err?.response?.data?.errorCode;
    if (code === 'E409500') toast.warning('이미 처리된 요청입니다.');
    else if (code === 'E400501') toast.error('해당 시간에 이미 수업이 등록되어 있습니다.', {
      action: { label: '캘린더 보기', onClick: () => router.push(`/calendar?date=${row.startTime}`) }
    });
    else if (code === 'E400502') toast.error('수업 정원이 초과되었습니다.', {
      action: { label: '대기열로 이동', onClick: () => router.push(`/waitlist?classId=${row.classId}`) }
    });
    else toast.error('처리에 실패했습니다.');
  },
  onSettled: () => setMutatingId(null),
});

highlight 처리:
useEffect(() => {
  const id = searchParams.get('highlight');
  if (id) {
    setHighlightId(Number(id));
    const t = setTimeout(() => {
      setHighlightId(null);
      router.replace(pathname);
    }, 2000);
    return () => clearTimeout(t);
  }
}, [searchParams]);
```
