## 1. 상태 진입 조건

- `SCR-C004` PageHeader `+ 템플릿 등록` 클릭
- `mode === 'create'`, `template === undefined`, `isOpen === true`

## 2. 비주얼 델타

- **Header 제목**: "그룹수업 템플릿 등록"
- **Submit 버튼 라벨**: "저장"
- **폼 필드**: 모두 `defaultValues` 상태 (빈 템플릿명, `category=GX`, `duration=60`, `capacity=10`, `color=#6366f1`, `repeatDays=[]`, `isActive=true`)
- **isValid**: 기본 false → 필수 필드 입력 전까지 `저장` 버튼 disabled

```
┌─────────────────────────────────────┐
│ 그룹수업 템플릿 등록           [×] │
├─────────────────────────────────────┤
│ 템플릿명 *  [ (empty) ]              │
│ 분류 *       [ GX ▼ ]                │

│ 색상  ●○○○○○○○○○                    │
│ 반복 요일 [일][월][화][수][목][금][토] │
│ 설명 []                               │

├─────────────────────────────────────┤
│                [ 취소 ] [ 저장:disabled ] │
```

## 3. 역할별 차이

- `trainer`: 진입 자체 차단 (부모 페이지에서 `+ 템플릿 등록` 버튼 숨김)
- `front/readonly`: 동일
- `manager/owner/superAdmin`: 전체 가능

## 4. 상태 고유 인터랙션

- **첫 포커스**: 템플릿명 인풋 (Radix `autoFocus` 기본 동작)
- **Enter 키**: Submit (disabled면 무시)
- **색상 팔레트 선택**: `shouldDirty: true`로 `isDirty` 전환
- **반복 요일 토글**: 빈 배열에서 인덱스 push/splice
- **실시간 유효성**: `mode:'onChange'`로 필수 필드 입력 직후 버튼 활성화

## 5. 비즈니스 룰 델타

1. **지점 자동 주입**: 제출 시 `session.branchId` 첨부
2. **성공 토스트 메시지**: `"템플릿이 등록되었습니다."`
3. **API**: `POST /api/lesson-templates`

## 6. 에지 케이스

- 입력 중 서버 점검 발생 시: Submit에서 E503 → 에러 토스트 + 폼 유지
- 네트워크 끊김 후 재연결: 동일 폼 재제출 가능 (멱등성 없음 → 중복 생성 주의: 409로 방지)
- 아주 긴 템플릿명(50자 초과) 붙여넣기: `maxLength=50` + Zod `max(50)` 이중 방어

## 7. 바이브코딩 프롬프트 (델타)

```
모드: create
useForm defaultValues: { name:'', category:'GX', duration:60, capacity:10, color:'#6366f1', repeatDays:[], description:'', isActive:true }
Title: "그룹수업 템플릿 등록"
Submit label: "저장"
Submit action:
  await api.post('/api/lesson-templates', { ...data, branchId: session.branchId })
  toast.success("템플릿이 등록되었습니다.")
  queryClient.invalidateQueries(['lesson-templates', branchId])
  onSuccess(); onClose()
```
