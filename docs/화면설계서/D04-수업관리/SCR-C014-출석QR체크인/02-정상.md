## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-C014 출석QR체크인 — 상태: 정상

파일: src/app/(classes)/qr-checkin/page.tsx

레이아웃 (데스크톱 2열):
- AppLayout + PageHeader (제목 "출석 QR 체크인")

좌측 (QR 생성 패널 w-1/2 border-r p-6):
- 수업 Select (오늘 수업 목록):
  "09:00 PT - 홍길동 강사" 형식
- QR 코드 생성:
  · qrcode 라이브러리로 QR 생성
  · QR 데이터: { lesson_schedule_id, branch_id, expires_at: +5분 }
  · QR 이미지 (w-48 h-48 border rounded-lg mx-auto)
  · 유효시간 카운트다운: "유효시간 M:SS" (text-sm text-gray-500)
  · "QR 갱신" Button outline (5분마다 자동 갱신 + 수동 갱신)

우측 (체크인 이력 w-1/2 p-6):
- 오늘 체크인 목록 (실시간 갱신):
  · 각 행: 회원명 | 체크인 시각 | 지각여부
  · 지각: text-amber-600 "지각 (+N분)"
  · 정상: text-green-600 "정상"
- 실시간 구독 (supabase realtime)

수동 체크인 (QR 스캔 불가 시):
- 회원명 검색 input + "체크인" Button
- POST /api/qr-checkins { lesson_schedule_id, member_id, checkin_type: 'manual' }

사용 유틸:
- qrcode 또는 react-qr-code
- useEffect (5분 interval QR 갱신)
- supabase realtime channel
- format from 'date-fns'
- differenceInMinutes for 지각 계산
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 후 오늘 수업 있음

### 필수 데이터
| 블록 | 테이블 |
|------|--------|
| 오늘 수업 | `lesson_schedules` |
| 체크인 이력 | `qr_checkins` (신규) |
| 회원 정보 | `users` |

### 인터랙션 (User Actions)
1. 수업 Select → 해당 수업 QR 생성
2. QR 갱신 버튼 → 새 QR + 타이머 리셋
3. 수동 체크인 → 회원 검색 + 체크인 처리
4. 실시간 체크인 알림 (supabase realtime)

### 비즈니스 룰
- QR 유효시간: 5분 (보안)
- 수업 시작 시간 기준 지각 판단 (N분 초과)
- 중복 체크인 방지: 동일 회원/수업 2회 불가

### 에지 케이스
- QR 만료 후 스캔: "QR이 만료되었습니다. 갱신 후 다시 시도하세요."
- 미예약 회원 QR 스캔: 경고 토스트

### 연결 화면
- 수업 없음: `03-빈상태`
