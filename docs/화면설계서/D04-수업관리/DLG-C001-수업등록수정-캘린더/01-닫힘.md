## 상태 고유 인터랙션

- 캘린더 기본 상호작용으로 복귀
- 부모 SCR-C001에서 진입 트리거 대기
- Trainer 이하 권한 사용자가 `isOpen=true` 이벤트 수신 시 권한 검사로 차단

## 상태 고유 비즈니스 룰 델타

- 모달 관련 훅 언마운트 → `useForm` 내부 state 초기화
- `document.body.style.overflow` 복원
- 진행 중이던 `searchMembers` debounce timeout 취소
- 저장 성공 직후라면 `onSaved(saved)` 콜백이 이 상태보다 **먼저** 호출되어 부모 캘린더 갱신됨
- `selectedEvent=null`, `selectedDate=null` 초기화

## 에지 케이스

- **저장 API 응답 지연 중 강제 닫힘 시도**: `isSaving=true`인 동안은 닫힘 차단
- **전환 애니메이션**: 200ms fade-out 후 완전 닫힘 (`motion-reduce` 시 즉시)
- **다중 인스턴스 방지**: 부모에서 하나의 `selectedEvent`만 허용, 중복 오픈 금지
- **키보드 포커스 복귀**: 직전 트리거(등록 버튼/캘린더 이벤트)로 포커스 이동

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 전체 적용
상태: lc-closed

핵심:
- if (!isOpen) return null;
- useEffect cleanup:
   document.body.style.overflow = '';
   clearTimeout(searchDebounceRef.current);
- onClose 호출 직전 selectedEvent=null, form.reset()
- 저장 성공 플로우: setIsSaving(false) → onSaved(saved) → onClose() 순서

부모 측 사용 패턴:
const [isOpen, setIsOpen] = useState(false);
const [mode, setMode] = useState<'create'|'edit'>('create');
const [selectedEvent, setSelectedEvent] = useState<ScheduleEvent|null>(null);

<LessonCalendarModal
  isOpen={isOpen}
  mode={mode}
  initial={selectedEvent ?? { date: selectedDate, startTime, endTime }}
  events={events} localEvents={localEvents}
  instructors={instructors} templates={CLASS_TYPES} rooms={ROOMS}
  onClose={() => { setIsOpen(false); setSelectedEvent(null); }}
  onSaved={(saved) => { setEvents(prev => merge(prev, saved)); }}
/>
```
