## 🧩 바이브코딩 프롬프트

```
Next.js 14 App Router + TypeScript + Tailwind CSS 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-063 직원 근태 관리 — 상태: 일별 뷰

파일: src/app/staff/attendance/page.tsx

조건: viewMode='daily', isLoadingData=false

상단 구성:
- TabNav: [일별 (active) | 월별]
- DatePicker: 날짜 선택 (onChange → 해당 날짜 근태 재fetch)
- 우측: "출근 처리" 버튼 (+ 출근 기록 모달 or 직접 처리)

통계 카드 (4개, grid-cols-4):
  1. 재직 중 전체: staffList.length
  2. 출근: attendance.filter(a => a.clock_in).length
  3. 미출근: staffList.length - 출근 수
  4. 퇴근 완료: attendance.filter(a => a.clock_out).length

DataTable 컬럼:
- 이름
- 부서
- 직책
- 출근 시간 (clock_in, HH:mm 형식 또는 "-")
- 퇴근 시간 (clock_out, HH:mm 형식 또는 "-")
- 근무 시간 (clock_out - clock_in, "X시간 Y분" 또는 "-")
- 상태 (StatusBadge: 출근중/퇴근/미출근/휴가/외근)
- 액션:
  · 미출근 → "출근 처리" 버튼 (primary small)
  · 출근중 → "퇴근 처리" 버튼 (outline small)
  · 퇴근 → "-"

출근 처리 핸들러:
  const handleClockIn = async (staffId: string) => {
    const now = new Date().toISOString()
    const { error } = await supabase.from('staff_attendance').upsert({
      staff_id: staffId,
      date: selectedDate,
      clock_in: now,
      status: 'WORKING',
    })
    if (error) toast.error('출근 처리에 실패했습니다.')
    else { toast.success('출근 처리되었습니다.'); fetchAttendance() }
  }

퇴근 처리 핸들러:
  const handleClockOut = async (staffId: string) => {
    const now = new Date().toISOString()
    const { error } = await supabase.from('staff_attendance')
      .update({ clock_out: now, status: 'DONE' })
      .eq('staff_id', staffId).eq('date', selectedDate)
    if (error) toast.error('퇴근 처리에 실패했습니다.')
    else { toast.success('퇴근 처리되었습니다.'); fetchAttendance() }
  }

사용 컴포넌트:
- AppLayout, PageHeader, DataTable, TabNav, StatusBadge, Button
- supabase, toast, formatNumber
- useAuthStore
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `isLoadingData=false && viewMode='daily'`
- 날짜 변경 후 재조회 완료

### 필수 데이터
| 테이블 | 조건 |
|--------|------|
| `staff` | `staff_status='ACTIVE'`, center_id |
| `staff_attendance` | `date=selectedDate`, center_id |

### 인터랙션 (User Actions)
1. DatePicker 날짜 변경 → 해당 날짜 근태 재fetch + 로딩 스피너
2. 탭 "월별" 클릭 → `03-월별뷰`
3. "출근 처리" 버튼 → `handleClockIn(staffId)` → `04-출근완료`
4. "퇴근 처리" 버튼 → `handleClockOut(staffId)` → `04-퇴근완료`
5. 직접 시간 수정 (관리자): 셀 인라인 편집 (더블클릭 → time input)

### 비즈니스 룰
- 동일 날짜 중복 출근 방지 (upsert 처리)
- 미래 날짜 출근 처리 불가 (DatePicker max=today)
- 퇴근 시간 > 출근 시간 검증

### 에지 케이스
- 휴일/주말 날짜 선택: 데이터 없음 (빈 테이블) + 안내 메시지 없음
- 근태 데이터 없는 직원: "-" 표시 (미출근)

### 접근성 (A11y)
- StatusBadge `aria-label="근태 상태: {status}"`
- DatePicker `aria-label="날짜 선택"`
- 통계 카드 `role="region"` + `aria-label="{제목}"`

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-월별뷰` / `04-출근완료` / `04-퇴근완료`
