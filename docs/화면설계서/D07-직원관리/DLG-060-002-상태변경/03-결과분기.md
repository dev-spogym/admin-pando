## 4. Payload 빌더

```ts
function buildPayload(v: StatusChangeFormValues): Partial<Staff> {
  const base = { staffStatus: v.status, updatedAt: new Date().toISOString() };
  switch (v.status) {
    case 'ACTIVE':
      return { ...base, leaveStartAt: null, leaveEndAt: null, leaveReason: null };
    case 'ON_LEAVE':
      return { ...base, leaveStartAt: v.startAt, leaveEndAt: v.endAt, leaveReason: v.reason };
    case 'TRANSFERRED':
      return {
        ...base,
        isActive: false,
        transferReason: v.reason,
        transferredFromBranchId: getBranchId(),
        transferredAt: new Date().toISOString(),
      };
    case 'LOCKED':
      return { ...base, isActive: false, lockReason: v.reason };
  }
}
```

## 5. 비즈니스 룰 델타

- **전이 매트릭스**: 서버 트리거로 금지된 전이(예: RESIGNED→*)는 E409 반환. 클라이언트는 배너만.
- **감사 로그**: `AUDIT.STAFF_STATUS_CHANGE`(actor, targets, from→to, reason, branchId).
- **알림**: `ON_LEAVE`/`TRANSFERRED` 성공 시 해당 직원에게 시스템 알림 발송 (서버 측 훅).
- **세션 시나리오**: `LOCKED` 성공 시 해당 직원이 로그인 중이면 다음 요청에서 강제 로그아웃 (서버가 토큰 무효화).

## 7. 에지 케이스

- **부분 성공**: `count < targets.length` → warning 토스트로 명시
- **409 동일 상태**: 다른 탭에서 미리 변경된 경우. 목록 재조회 권유
- **500 연속 3회**: 10초 쿨다운 (버튼 disabled + 카운트다운)
- **네트워크 단절**: 배너 "NETWORK" + 재시도 가능

## 8. 바이브코딩 프롬프트 (델타)

```
상태: dlg-statuschange-result
변경점:
- mutation 구현:
  const mutation = useMutation({
    mutationFn: async (v) => {
      const payload = buildPayload(v);
      const { data, count, error } = await supabase
        .from('staff')
        .update(payload, { count:'exact' })
        .in('id', targets.map(t=>t.id))
        .eq('branchId', getBranchId())
        .select('id');
      if (error) throw error;
      return { affected: count ?? data?.length ?? 0, from: targets[0]?.staffStatus, to: v.status };
    },
    onSuccess: ({ affected, from, to }) => {
      qc.invalidateQueries({ queryKey: ['staff'] });
      clearSelection();
      auditLog('STAFF_STATUS_CHANGE', { affected, from, to });
      if (affected === targets.length) toast.success(`${affected}명의 상태가 변경되었습니다.`);
      else if (affected > 0) toast.warning(`${affected}/${targets.length}명 변경. 일부 제외`);
      else setServerError({ code: 'PARTIAL_ZERO', message: '변경된 대상이 없습니다' });
      if (affected > 0) onClose();
    },
    onError: (err:any) => {
      const code = mapSupabaseError(err);
      setServerError({ code, message: ERROR_MESSAGE[code] ?? err.message });
      if (code === 'E403001') { toast.error(ERROR_MESSAGE[code]); onClose(); }
      if (code === 'E401001') redirectToLogin();
    },
  });

- 실행 중 UI:
  readOnly 입력 / 모든 close action 가드
  loading={mutation.isPending}

- 쿨다운:
  if (consecutive5xx >= 3) setCooldown(Date.now() + 10000);
```
