## 역할별 차이
- 공통. 권한 부재 시 여기까지 오지 못함 (마스터 가드).

## 고유 인터랙션
- 취소: DLG-061-001 → 목록.
- Esc: 취소.
- 입력 불가 (모든 필드 disabled).

## 비즈니스 룰 델타
1. 단일 요청 `getStaffById(editId)` retry:1.
2. 10s 타임아웃 → 실패 + toast.
3. 타 지점 직원 id → 403 → 목록.
4. `staffStatus==='RESIGNED'` → toast "퇴사 처리된 직원은 수정할 수 없습니다." + 목록.
5. `ON_LEAVE`: 일부 필드만 수정 허용 (기본급 등). 배지 표시.

## 에지 케이스
- 잘못된 id(숫자 아님) → 400 → 목록.
- 삭제된 id → 404 → 목록.
- 동시 편집: 락 없음. 낙관적 concurrency 선택(updatedAt).

## 바이브코딩 프롬프트 (델타)
```
editId && isLoadingStaff === true:
- input/select/textarea: disabled + "animate-pulse bg-gray-100"
- 저장 버튼: disabled

useEffect(() => {
  if (!editId) return;
  setIsLoadingStaff(true);
  (async () => {
    try {
      const { data, error } = await supabase.from('staff').select('*').eq('id', editId).single();
      if (error || !data) throw error ?? new Error('not found');
      if (data.staffStatus === 'RESIGNED') {
        toast.error('퇴사 처리된 직원은 수정할 수 없습니다.'); moveToPage(974); return;
      }
      reset({
        name: data.name ?? '', role: ROLE_DB_TO_KEY[data.role] ?? 'staff',
        contact: data.phone ?? '',
        joinDate: data.hireDate ? data.hireDate.slice(0,10) : '',
        email: data.email ?? '', salary: data.salary ?? '', memo: '',
      });
    } catch (e) {
      toast.error('직원 정보를 불러오지 못했습니다.'); moveToPage(974);
    } finally { setIsLoadingStaff(false); }
  })();
}, [editId]);
```
