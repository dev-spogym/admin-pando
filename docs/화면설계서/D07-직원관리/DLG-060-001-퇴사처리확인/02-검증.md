## 1. 상태 진입 조건

- `open === true`

## 2. 비주얼 델타

### 2.1 불일치 상태 (`confirmText !== '퇴사처리'`)

```
┌──────────── Modal ────────────┐
│ ...                            │
│ 확인을 위해 "퇴사처리"를 입력.   │
│ ┌────────────────────────────┐ │
│ │ 퇴사                       │ │  ← 입력 중(불일치)
│ └────────────────────────────┘ │  border-red-300 (onBlur 후)
│ ⚠ "퇴사처리"를 정확히 입력하세요 │  role="alert" (onBlur 시 표시)
│ ...                            │
│         [취소]  [퇴사 처리]    │  disabled=true
└────────────────────────────────┘
```

### 2.2 일치 상태 (`confirmText === '퇴사처리'`)

```
┌──────────── Modal ────────────┐
│ ...                            │
│ ┌────────────────────────────┐ │
│ │ 퇴사처리                    │ │  border-gray-300 (기본)
│ └────────────────────────────┘ │
│   ↑ 힌트 메시지 제거            │
│ ...                            │
│         [취소]  [퇴사 처리]    │  disabled=false ✅
└────────────────────────────────┘
```

| 요소 | 불일치(onBlur) | 일치 | 비고 |
|---|---|---|---|
| input border | `border-red-300` | `border-gray-300` | `aria-invalid` 동적 |
| input focus ring | `ring-red-500` | `ring-red-500` | danger variant 유지 |
| 힌트 | `"퇴사처리"를 정확히 입력하세요` | 미표시 | `role="alert"` |
| 확정 버튼 | `disabled` | enabled | |

## 3. 검증 로직

```ts
const EXPECTED = '퇴사처리';
const isValid =
  confirmText.trim() === EXPECTED &&
  targets.length > 0 &&
  !targets.some(t => t.staffStatus === 'RESIGNED');

const validate = () => {
  if (confirmText.length === 0) setConfirmError(null);
  else if (confirmText.trim() !== EXPECTED) {
    setConfirmError('"퇴사처리"를 정확히 입력하세요');
  } else {
    setConfirmError(null);
  }
};
```

### 3.1 타이밍
| 이벤트 | 동작 |
|---|---|
| onChange | error 초기화(즉시), isValid 재계산 |
| onBlur | `validate()` 호출, 불일치 시 힌트 노출 |
| Enter (isValid=true) | `handleConfirm()` → `03-결과분기` |
| Enter (isValid=false) | `preventDefault`, 힌트 노출 |

### 3.2 거부 규칙
| 입력 | 결과 |
|---|---|
| `"퇴사"` | 불일치 |
| `"퇴사 처리"` (중간 공백) | 불일치 |
| `" 퇴사처리 "` (양끝 공백) | **일치** (`.trim()`) |
| `"退社処理"` / 영문 | 불일치 |
| `"퇴사처리퇴사처리"` | 불일치 |

## 5. 인터랙션 델타

- 입력 중 지속적으로 `isValid` 재계산 → 버튼 활성화/비활성 토글
- onBlur 시에만 에러 힌트 표시 (입력 중 과도한 에러 노출 방지)
- 입력 삭제(backspace)로 빈 문자열 → 힌트 즉시 제거

---

## 6. 비즈니스 룰 델타

- **IME 조합 상태**: 한글 조합 중에는 `trim()` 결과가 일시적으로 불일치 → onBlur 또는 조합 종료(`compositionend`) 기준으로만 힌트 표시.
- **클립보드 붙여넣기**: `"퇴사처리"` 붙여넣기 시 즉시 일치. 붙여넣기 탐지 로그는 생략.

---

## 7. 에지 케이스

- **한글 자음/모음 단독**: `ㅌ`, `ㅚ` 등 조합 미완성 상태 → 불일치로 유지하되 힌트는 `compositionend` 후에만 표시.
- **자동 채우기**: 브라우저 autofill로 이전 값이 들어올 가능성 낮음(type="text") → 무시.
- **다중 언어 키보드**: 영문 상태에서 `xhtkdlfl` 입력 → 불일치, 한글 전환 안내 문구 미추가 (과설계 방지).

## 8. 바이브코딩 프롬프트 (델타)

```
상태: dlg-resign-validating
변경점:
- isComposing 상태 추가 (IME 대응)
  const [isComposing, setIsComposing] = useState(false);
  onCompositionStart={()=>setIsComposing(true)}
  onCompositionEnd={(e)=>{ setIsComposing(false); setConfirmText(e.currentTarget.value); }}

- 힌트 노출 조건:
  (confirmError !== null && !isComposing)

- onBlur 시 validate() 호출
- Enter 키:
  onKeyDown={(e) => {
    if (e.key === 'Enter' && isValid && !isProcessing) {
      e.preventDefault();
      handleConfirm();
    }
  }}

- aria-invalid={confirmError !== null}
- aria-describedby="resign-hint"
```
