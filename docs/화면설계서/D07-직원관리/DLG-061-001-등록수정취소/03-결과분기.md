## 🧩 바이브코딩 프롬프트

```
결과 분기 핸들러:

  // "계속 작성하기" 선택
  const handleKeepWriting = () => {
    setCancelModalOpen(false)
    // 폼 포커스 복귀 (첫 번째 빈 필드 또는 마지막 편집 필드)
    firstInputRef.current?.focus()
    // 입력 데이터 완전히 유지 (form.reset 호출 안 함)
  }

  // "네, 취소합니다" 선택
  const handleCancelConfirm = () => {
    setCancelModalOpen(false)
    form.reset()  // 폼 데이터 초기화
    moveToPage('/staff')  // SCR-060으로 이동
  }

결과 분기:
  "계속 작성하기":
    → 모달 닫힘
    → SCR-061 폼 상태 그대로 유지 (isDirty=true 유지)
    → 사용자가 마지막 편집하던 위치로 포커스 복귀

  "네, 취소합니다":
    → form.reset() (입력값 모두 초기화)
    → moveToPage('/staff') (SCR-060 목록)
    → toast 없음 (사용자가 의도적으로 선택)
```

## 📝 디스크립션

### 분기 결과
| 선택 | 처리 | 결과 |
|------|------|------|
| 계속 작성하기 | 모달 닫힘, 폼 유지 | SCR-061 폼 복귀 |
| 네, 취소합니다 | form.reset() + navigate | SCR-060 목록 |

### 비즈니스 룰
- "계속 작성하기": 어떤 데이터도 잃지 않음
- "네, 취소합니다": 모든 입력 데이터 폐기 (복구 불가)
- 취소 후 SCR-060 진입: 마지막 선택 탭 유지

### 에지 케이스
- "계속 작성하기" 후 브라우저 뒤로가기: beforeunload 이벤트 (향후)
