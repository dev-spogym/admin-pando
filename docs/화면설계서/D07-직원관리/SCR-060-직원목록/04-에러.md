- `useStaffList.error != null`
- HTTP 5xx, 타임아웃, 기타 예외
- 주요 코드: E500001, E503001, E404200

## 비주얼 델타

```
┌──────────────────────────────────────────────────────────┐
│ PageHeader [+ 직원 등록]                                 │
├──────────────────────────────────────────────────────────┤
│ 🔴 ErrorBanner (상단 고정)                               │
│ "⚠ 직원 데이터를 불러오지 못했습니다." (E500001)        │
│ [다시 시도]  [지점 변경] (primary만)                     │
├──────────────────────────────────────────────────────────┤
│ StatCardGrid  [—] [—] [—] [—]                            │
├──────────────────────────────────────────────────────────┤
│ DataTable emptyMessage="데이터를 불러오지 못했습니다."    │
│ 또는 stale 데이터 유지 + 상단 배너                        │
└──────────────────────────────────────────────────────────┘
```

| 변경 요소 | From → To |
|---|---|
| 상단 | — → `<ErrorBanner tone="error" />` |
| StatCard values | 숫자 → `—` |
| DataTable | row → stale fallback **or** `emptyMessage` |
| 액션 버튼 | 활성 → 전부 disabled (stale 있으면 엑셀만 허용 옵션) |
| 재시도 | — → 배너 우측 `[다시 시도]` |

## 역할별 차이
- primary/superAdmin: "다른 지점으로 전환" 제안.
- owner/manager: "다시 시도" + 지원팀 링크(선택).

## 고유 인터랙션
- "다시 시도" → `refetch()` → `01-로딩`.
- 배너 닫기(X): 닫기만, stale 데이터로 계속 탐색.
- 자동 재시도: 네트워크 에러 한정 3초 뒤 1회 (React Query `retry:1`).
- toast.error `"직원 데이터를 불러오지 못했습니다."` 진입 1회.

## 비즈니스 룰 델타
1. **에러 분류**:
   - `NETWORK`/offline → `06-오프라인` 전이.
   - `E500001` → 서버 일반, 재시도 권장.
   - `E503001` → 점검, "서비스 점검" 문구, 30s 쿨다운.
   - `E401*` → `/login?redirect=/staff`.
   - `E403001/E403003` → `/forbidden` (`05-권한없음`).
2. **stale-while-error**: `keepPreviousData`로 이전 성공 유지.
3. 반복 5xx → Sentry 로깅 (throttle).

## 에지 케이스
- **403 (지점)**: `?branch=` 조작 시 → `/forbidden`.
- **타임아웃**: `retry:1` + exponential backoff.
- **JSON 파싱 에러**: "응답 형식이 올바르지 않습니다" 대체 문구.
