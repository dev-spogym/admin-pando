## 고유 인터랙션
- 검색 입력: 즉시 filter 재계산 + `currentPage = 1`.
- 필터 Select 변경: 동일 리셋.
- 정렬 헤더: 3-state 순환 (null→asc→desc→null).
- 체크박스: `07` ↔ `08` 전이.
- 직원명 ghost: `moveToPage(998,{id})` → SCR-061 수정.
- MoreVertical: 컨텍스트 메뉴 (상세/수정/휴직/퇴사).
- Tab 복귀: `isRefetching=true` 스피너, 데이터 유지.

## 비즈니스 룰 델타
1. **파생값** `total/active/onLeave/resigned` useMemo. status는 `staffStatus + isActive` 조합.
2. **정렬 안정성**: 동률 시 `id asc` 2차 정렬.
3. **검색**: `name || contact`. 공백 trim. debounce 없음.
4. **페이지 리셋**: 검색/필터/정렬 변경 시 1페이지로.

## 에지 케이스
- **데이터 1건**: 통계 `1/1/0/0`, 페이지네이션 숨김.
- **다중 페이지 축소**: 현재 페이지 > lastPage → lastPage로 조정.
- **leave date null + staffStatus='ON_LEAVE'**: "휴직" 배지 + tooltip "기간 미지정".
- **한글/영문 role 공존**: ROLE_CONFIG 양쪽 키 등록.
