## 📝 디스크립션

### API 분기 결과
| 응답 | 처리 |
|------|------|
| 200 OK | toast.success + 닫힘 + SCR-064 갱신 (전원확정) |
| 400 | toast.error('입력 오류') + 모달 유지 |
| 403/RLS | toast.error('권한 없음') + 닫힘 |
| 409 이미확정 | toast.error('이미 확정') + 닫힘 + 갱신 |
| 500 | toast.error('서버 오류') + 모달 유지 + 재시도 |

### 성공 후 처리
1. `toast.success('급여가 확정되었습니다. 명세서가 생성됩니다.')`
2. `setConfirmModalOpen(false)` — 모달 닫힘
3. `fetchPayrollList()` — SCR-064 `04-전원확정` 상태 전환
4. payroll_statements 자동 생성 (Edge Function 또는 DB trigger)

### 비즈니스 룰
- 확정은 PENDING 전원 일괄 처리 (부분 확정 불가)
- 확정 후 SCR-064 "급여 확정" 버튼 → disabled
- 확정 후 SCR-065 명세서 목록에 자동 반영

### 에지 케이스
- 확정 중 네트워크 끊김: catch → toast.error + isConfirming=false
- 일부 직원만 확정 성공(부분 실패): SCR-064 재조회로 실제 상태 확인
- manager가 API 직접 호출 시도: 서버 RLS에서 403 차단

### 접근성 (A11y)
- 확정 중 `aria-busy="true"`
- 성공 toast `role="status"`, 에러 toast `role="alert"`
