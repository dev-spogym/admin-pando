## 상태 고유 인터랙션

- ESC/배경/X: **허용** (로딩 취소 → Query abortController로 요청 취소)
- 재요청 버튼: 없음 (로딩 중)
- 키보드: Tab → 닫기 가능

## 상태 고유 비즈니스 룰 델타

- `query.isFetching = true` 동안 skeleton 유지
- 타임아웃 15초 초과 시 자동으로 `04-에러` 전이 (NETWORK)
- `aria-busy="true"` 컨텐트에 부여, `role="status" aria-live="polite"` 로딩 공지
- 메타 폼은 쿼리와 독립적으로 즉시 사용 가능 (별도 `useMutation`)

## 에지 케이스

- 모달 닫힘 직후 재오픈: React Query 캐시 hit → skeleton 건너뛰고 즉시 `02` 전이 가능
- 네트워크 플래킹: `retry: 1` + 15s 타임아웃
- 부모 락커 ID 변경(다른 셀 기록 클릭): `queryKey` 변경 → 새 요청 시작, skeleton 재표시
- 세션 만료: DLG-000 우선, 모달 자동 정리

## 바이브코딩 프롬프트 (델타)

```
{isLoading 분기 렌더}
<section aria-busy="true" role="status" aria-live="polite" className="space-y-1.5">
  <span className="sr-only">이력 불러오는 중</span>
  {Array.from({length: 5}).map((_, i) => (
    <div key={i} className="h-12 rounded bg-gray-100 animate-pulse motion-reduce:animate-none" />
  ))}
</section>

{useEffect 타임아웃}
useEffect(() => {
  if (!query.isFetching) return;
  const t = setTimeout(() => query.cancel?.(), 15_000);
  return () => clearTimeout(t);
}, [query.isFetching]);
```
