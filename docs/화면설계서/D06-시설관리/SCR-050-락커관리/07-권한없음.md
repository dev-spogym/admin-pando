# SCR-050 락커 관리 — 07. 권한 없음 (델타)

> 마스터 `00-기본화면.md` 상속. 변경점만 기술.
> fc/trainer 또는 타 지점 접근 시 표시. RBAC 차단.

## 메타

| 항목 | 값 |
|---|---|
| 상태코드 | `locker-forbidden` |
| 부모 화면 | SCR-050 |
| 다이어그램 노드 | `F7_권한.md` → `STATE_FORBIDDEN` |
| 이전 상태 | `01-로딩`(권한 검사), 직접 URL 접근 |
| 다음 상태 | 대시보드로 리다이렉트, 로그아웃 |

## 상태 진입 조건

- `canAccessLocker(role) === false` (fc, trainer)
- 또는 서버 응답 403 `E403001` (권한 없음) / `E403003` (타 지점 접근)
- 또는 URL `?branch=<타지점>` 조작 시 지점 접근 권한 실패
- readonly 역할이 수정 페이지 직접 접근 시도 (조회만 가능)

### ForbiddenState 컴포넌트
```tsx
<ForbiddenState
  icon={<ShieldOff size={80} className="text-gray-400"/>}
  title="락커 관리에 접근할 권한이 없습니다"
  description={ROLE_FORBIDDEN_DESC[role] ?? "이 화면은 센터장/매니저/스탭 권한이 필요합니다."}
  errorCode="E403001"
  currentRole={roleLabel}
  actions={[
    <Button variant="primary" onClick={() => moveToPage('/')}>대시보드로 이동</Button>,
    <Button variant="ghost" onClick={contactSupport}>지원 문의</Button>,
  ]}
/>
```

## 역할별 차이

| 역할 | 진입 원인 | 안내 |
|---|---|---|
| fc | 시설은 상담 역할 범위 외 | "FC는 상담 기능을 이용해주세요" + [고객 리드 이동] 버튼 |
| trainer | 시설은 수업 역할 범위 외 | "트레이너는 캘린더/수업 관리를 이용해주세요" + [캘린더 이동] |
| readonly | 기본 읽기 전용 | "조회만 가능합니다. 센터 관리자에게 문의하세요" |
| 타 지점 접근 | URL 조작 | "해당 지점에 대한 접근 권한이 없습니다 (E403003)" + [내 지점으로 이동] |
| 세션/role 손상 | JWT 위변조 등 | "인증 정보가 유효하지 않습니다" + 강제 로그아웃 |

## 상태 고유 인터랙션

- **[대시보드로 이동]**: `moveToPage('/')` → SCR-090
- **[캘린더 이동] (trainer)**: `moveToPage('/calendar')`
- **[고객 리드 이동] (fc)**: `moveToPage('/leads')` (없으면 `/consultations`)
- **[내 지점으로 이동] (타지점)**: `moveToPage('/')` + `useBranchStore.current = user.branchId` 복원
- **[지원 문의]**: `mailto:support@fitgenie.com?subject=Access%20Denied%20E403001`
- **역할 변경 시점**: 세션 갱신 후 페이지 리로드 시 재검사

## 비즈니스 룰 델타

1. **하드 리다이렉트 정책**: fc/trainer가 직접 URL로 `/locker` 접근 시 3초 후 자동 대시보드 리다이렉트 (UI에 카운트다운)
2. **서버 강제 스코프**: 클라이언트 가시성과 무관하게 서버가 403 강제 → 이 상태 표시
3. **감사 로그**: 접근 시도 기록 (`AUDIT.ACCESS_DENIED` + role + targetBranchId)
4. **브루트포스 방어**: 동일 role이 1분 내 5회 이상 권한 거부 요청 → 경고 로그 + alert 관리자

## 에지 케이스

| 상황 | 처리 |
|---|---|
| 권한 변경 중 (업그레이드) | 세션 새로고침 안내 toast "권한이 변경되었습니다. 새로고침하세요" |
| 권한 강등 (downgrade) | 자동 이 상태 전환 + "권한이 변경되었습니다" 배너 |
| 토큰 만료와 403 동시 | 401 우선 → `/login` 리다이렉트 |
| 슈퍼 관리자 "지점 위장" 모드 | 위장된 branch의 권한 기준으로 판정 |
| 페이지 직접 북마크 접근 | 로그인 후 즉시 이 상태 |
| iframe 내부 접근 | `X-Frame-Options` 차단 우선 |
