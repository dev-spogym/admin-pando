## 📝 디스크립션

### 사용 시점 (Trigger)
- `E_F6_OK_01`: API 200 + cleaning_schedules.length > 0

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 스케줄 | `cleaning_schedules` | `branch_id = ?`, 오늘 범위 |
| 담당자 | `members` / `staff` | JOIN on assignee_id |

### 인터랙션 (User Actions)
1. "스케줄 추가" 버튼 → DLG-058-001 등록 모달
2. PENDING 행 "처리" 버튼 → 체크리스트 모달 → IN_PROGRESS 전환
3. IN_PROGRESS 행 체크리스트 완료 → DONE 전환 + 완료시간 기록
4. ISSUE 행 "점검등록" → 점검 이슈 등록 모달
5. 날짜 이동 버튼 (< >) → 다른 날짜 스케줄 조회

### 비즈니스 룰
- 전체 진행률: DONE / total * 100%
- ISSUE 상태: 관리자에게 자동 알림 발송
- 예정시간 30분 초과 후 PENDING → 자동 ISSUE 경고 표시

### 에지 케이스
- 전체 DONE: 상단 "오늘 청소 완료! 수고하셨습니다" 성공 배너 (bg-green-50)
- 담당자 미배정: "미배정" 표시 (amber 강조)

### 접근성 (A11y)
- 진행률 바 `role="progressbar"` + `aria-valuenow` + `aria-valuemax`
- ISSUE 행 `role="alert"` + `aria-label="점검 필요"`
