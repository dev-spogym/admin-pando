## 📝 디스크립션

### 사용 시점 (Trigger)
- `E_F6_OK_01`: API 200 + rooms.length > 0

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 운동룸 | `rooms` | `branch_id = ?` |
| 통계 | `rooms` | 상태별 COUNT |

### 인터랙션 (User Actions)
1. "룸 등록" 버튼 → DLG-053-001 등록 모달
2. 카드 "수정" 버튼 → DLG-053-001 수정 모달
3. 카드 "삭제" 버튼 → DLG-053-002 삭제 확인 모달
4. 상태 TabNav → 해당 상태 룸만 표시
5. 상태 클릭으로 ACTIVE ↔ MAINTENANCE 토글 (owner/manager만)

### 비즈니스 룰
- BROKEN 룸: 예약 생성 불가
- MAINTENANCE 룸: 슬롯 바 비활성 표시
- UNUSED: 관리 목적 보존, 예약 불가

### 에지 케이스
- 전체 룸 BROKEN/MAINTENANCE: 경고 배너 "사용 가능한 룸이 없습니다"
- 룸명 30자 초과 → truncate

### 접근성 (A11y)
- 카드 `role="article"` + `aria-label="{룸명} - {상태}"`
- StatusBadge `aria-label="상태: {상태명}"`
