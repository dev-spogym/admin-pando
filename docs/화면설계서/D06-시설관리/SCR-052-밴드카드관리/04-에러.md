## 상태 진입 조건
- `GET /rfid/cards` 4xx/5xx
- `navigator.onLine === false` (오프라인)
- 로딩 타임아웃 (>8초)
- mutation 실패(등록/분실/삭제)는 본 상태 전환 없이 inline 에러 처리

### 에러 유형별 메시지 분기

| 타입 | 제목 | 본문 | CTA |
|---|---|---|---|
| NETWORK (offline) | 네트워크 연결이 끊어졌습니다 | Wi-Fi 또는 이더넷을 확인해 주세요. | [재시도] |
| 401 만료 | 세션이 만료되었습니다 | 다시 로그인해 주세요. | [로그인으로] (자동 3초 리다이렉트) |
| 403 권한 | 접근 권한이 없습니다 | 이 지점의 RFID 관리 권한이 없습니다. | [대시보드로] |
| 404 지점 없음 | 지점을 찾을 수 없습니다 | 선택된 지점이 삭제되었거나 접근 불가합니다. | [지점 전환] (primary/super) |
| 500/502/503 | 서버 오류가 발생했습니다 | 잠시 후 다시 시도해 주세요. `{errorCode}` | [재시도] |
| TIMEOUT | 응답이 지연됩니다 | 네트워크가 느릴 수 있습니다. | [재시도] |

## 역할별 차이
| 역할 | 차이 |
|---|---|
| primary/super | 403 시 [지점 전환] CTA 우선 노출 |
| owner/manager/staff | 403 시 [대시보드로] CTA만 |
| 전 역할 | 401은 전역 인터셉터가 로그인으로 리다이렉트 |

## 상태 고유 인터랙션
- **[재시도]**: `refetch()` → `01-로딩` 전환, 3회 연속 실패 시 exponential backoff 안내
- **[대시보드로]**: `router.push('/')` 대시보드 이동
- **[로그인으로]**: `router.push('/login?redirect=/rfid')`
- **온라인 복구 감지**: `window.addEventListener('online')` → 자동 refetch 1회
- **5분 후 자동 재시도**: silent retry 1회(배너 그대로 유지)

## 상태 고유 비즈니스 룰 델타
- 페이지네이션 비활성
- 엑셀 버튼 disabled
- 신규 등록 버튼은 enable 유지(DLG-052-001 내부에서 별도 에러 처리)
- `Sentry.captureException(error, { extra:{ branchId, filters } })` 로깅
- `toast.error` 중복 방지: 배너가 있으면 toast는 생략

## 에지 케이스
- 부분 실패(통계 API만 성공): StatCard는 값 렌더, Table만 에러 상태
- 네트워크 복구 직후 연속 실패: 재시도 버튼 3회 후 disabled + "잠시 후 다시 시도해 주세요"
- 403 권한 변경(관리자가 중도에 권한 박탈): 배너 표시 후 5초 뒤 자동 `/forbidden`
- 401 + 백그라운드 탭: 포커스 복귀 시 재검증 → 갱신 실패 시 로그인 이동

## 바이브코딩 프롬프트 (델타)
```
isError=true 분기:
- ErrorBanner 상단 sticky
- ErrorState 중앙 (아이콘+제목+본문+CTA)
- errorCode 매핑 함수 getErrorMessage(error):
  { title, description, cta }
- onRetry: refetch() + 3회 실패 시 exponential backoff (1s → 2s → 4s)
- window.online 이벤트 → 자동 refetch
- 401: 전역 인터셉터가 처리(본 화면에서는 toast만)
- StatCard 값은 '-' text-gray-400
- 부분 실패: Promise.allSettled → 성공 쿼리만 렌더, 실패만 ErrorState
```
