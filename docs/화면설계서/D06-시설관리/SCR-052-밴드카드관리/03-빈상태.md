## 상태 진입 조건
- `GET /rfid/cards` 200 + `items.length === 0` + 필터 비어있음(`q=''`, `status='ALL'`, `type='ALL'`).
- 신규 지점으로 최초 진입 시 기본 상태.

| 요소 | 델타 |

| StatCard 전체/활성/기타 | 값 모두 `0` (gray-400 텍스트) |

| CTA | `Button variant="primary" size="lg"` |

### 조건부 분기 (검색 결과 0건 vs 초기 0건)
| 케이스 | 표시 |
|---|---|
| 초기 0건 (필터 비어있음) | 아이콘 + "등록된 RFID 카드가 없습니다" + [+ 신규 등록] |
| 검색 결과 0건 (필터 적용) | 아이콘 + "검색 조건에 맞는 카드가 없습니다" + [검색 초기화] |

## 역할별 차이
| 역할 | 차이 |
|---|---|
| primary/super/owner/manager/staff | CTA 노출 (+ 신규 등록 버튼 활성) |
| (접근 불가 역할) | 본 상태 도달 불가 (사이드바 메뉴 숨김 + 403) |

## 상태 고유 인터랙션
- **[+ 신규 등록] CTA**: 헤더 버튼과 동일하게 DLG-052-001 오픈
- **[검색 초기화] CTA**(검색 결과 0건 시): `setQ('')` + `setStatus('ALL')` + `setType('ALL')` → `01-로딩` 재진입
- 테이블 영역: 키보드 tab으로 CTA 버튼까지 직접 도달

## 상태 고유 비즈니스 룰 델타
- 페이지네이션 미렌더
- 엑셀 다운로드 버튼 disabled (`total === 0`)
- "최근 등록된 카드" 힌트/샘플 데이터 노출 금지(프로덕션)

## 에지 케이스
- 삭제 직후 0건 도달: toast.info "마지막 카드가 삭제되었습니다"와 함께 본 상태 진입
- primary/super가 신규 지점 선택 시: 즉시 본 상태 (BranchSwitcher 안내 toast "강남점으로 전환됨 · 카드 없음")
- 검색어가 있으나 DB 결과 0건: 분기 2(검색 결과 0건) 표시

## 바이브코딩 프롬프트 (델타)
```
items.length===0 분기:
<EmptyState
  icon={<CreditCard className="h-16 w-16 text-gray-300"/>}
  title={hasFilter ? "검색 조건에 맞는 카드가 없습니다" : "등록된 RFID 카드가 없습니다"}
  description={hasFilter ? "필터를 바꾸거나 초기화해 보세요" : "신규 등록 버튼을 눌러 첫 카드를 등록하세요"}
  action={
    hasFilter
      ? <Button variant="ghost" onClick={resetFilters}>검색 초기화</Button>
      : <Button variant="primary" size="lg" onClick={openAdd}>+ 신규 등록</Button>
  }
/>
- 엑셀 버튼 disabled={total===0}
- StatCard 값 0 → text-gray-400
```
