## SCR-070 리드 관리

### 1. 화면 목적
잠재 고객(문의/상담) 파이프라인을 목록 뷰와 칸반 뷰 두 가지 방식으로 관리한다. 리드 등록, 수정, 삭제 및 통계 확인 기능을 제공한다.

### 2. 접근 경로
`사이드바 > 마케팅 > 리드 관리` (`/leads`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 리드 관리                   [+ 리드 등록]   │
│ "잠재 고객(문의/상담) 파이프라인을 관리합니다."           │
├─────────────────────────────────────────────────────────┤
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐                     │
│ │전체 │ │등록 │ │전환율│ │대기 │ │미전환│                    │
│ │128건│ │23건│ │18% │ │45건│ │22% │                     │
│ └────┘ └────┘ └────┘ └────┘ └────┘                     │
├─────────────────────────────────────────────────────────┤
│ 상태별 분포: [신규 45건 35%] [상담예정 30건 23%] ...     │
├─────────────────────────────────────────────────────────┤
│ [상태 ▼] [유입경로 ▼]              [📋 목록] [🔲 칸반]   │
├─────────────────────────────────────────────────────────┤
│ (목록 뷰 또는 칸반 뷰)                                   │
└─────────────────────────────────────────────────────────┘
```

### 4. 통계 카드 (StatCardGrid cols=5)

| 카드 | 값 | 아이콘 | variant |
|------|-----|--------|---------|
| 전체 리드 | `{total}건` | Users | default |
| 등록 전환 | `{converted}건` | UserPlus | mint |
| 전환율 | `{conversionRate}%` | TrendingUp | peach |
| 대기 중 | `{pending}건` | Phone | default |
| 미전환율 | `{missedRate}%` | AlertCircle | default |

### 5. 상태별 분포 패널
stats.total > 0 일 때 표시. LEAD_STATUSES 중 count > 0 인 항목만 그리드(2~4열) 배치.

### 6. 뷰 모드

#### 목록 뷰 (viewMode = "list")

**테이블 컬럼:**

| 컬럼 | key | width | align | 렌더링 |
|------|-----|-------|-------|--------|
| 문의일 | inquiryDate | 100 | left | YYYY-MM-DD (mono 12px) |
| 이름 | name | auto | left | 볼드 텍스트 |
| 연락처 | phone | auto | left | mono 12px |
| 유입경로 | source | auto | left | StatusBadge variant=info |
| 상태 | status | auto | center | StatusBadge dot (상태별 variant) |
| 담당 FC | assignedFc | auto | left | 텍스트 |
| 후속일 | followUpDate | 100 | left | YYYY-MM-DD (mono 12px) |
| 메모 | memo | auto | left | 12px 잘림 (max-w-160px) |
| 관리 | actions | auto | center | [수정] [삭제] ghost 버튼 |

**리드 상태 variant 매핑:**

| 상태 | variant |
|------|---------|
| 신규 | info |
| 연락완료 | default |
| 상담예정 | warning |
| 방문완료 | info |
| 등록완료 | success |
| 미전환 | error |
| 보류 | default |

#### 칸반 뷰 (viewMode = "kanban")

**5개 파이프라인 컬럼:**

| 순서 | status | 표시 라벨 | 헤더 색상 |
|------|--------|----------|----------|
| 1 | 신규 | 신규 | blue |
| 2 | 상담예정 | 상담예약 | amber |
| 3 | 방문완료 | 상담완료 | purple |
| 4 | 연락완료 | 등록의향 | orange |
| 5 | 등록완료 | 등록완료 | green |

각 카드: 이름(13px bold) + 전화번호(11px mono) + 유입경로 배지 + 메모 미리보기. 클릭 → 수정 모달.

### 7. 필터 옵션

| 필터 | key | type | 옵션 |
|------|-----|------|------|
| 상태 | filterStatus | select | 전체, 신규, 연락완료, 상담예정, 방문완료, 등록완료, 미전환, 보류 |
| 유입경로 | filterSource | select | 전체, 간판, 인터넷, 전단지, 추천, SNS, 카카오톡, 전화문의, 방문, 기타 |
| 검색 | searchValue | text (DataTable 내장) | "이름, 연락처, 담당자 검색..." |

### 8. 버튼 → 동작 매핑

| 버튼 | 동작 |
|------|------|
| 리드 등록 (Primary + Plus) | DLG-070 등록 모달 표시 |
| 수정 (ghost Edit) | DLG-070 수정 모달 표시 (editTarget 세팅) |
| 삭제 (ghost Trash2) | DLG-071 삭제 확인 다이얼로그 |

## SCR-071 메시지 발송

### 1. 화면 목적
회원들에게 알림톡, SMS/LMS, 앱 푸시를 수동 발송하고, 발송 이력을 조회한다. 자동 알림 설정 미리보기도 제공한다.

### 2. 접근 경로
`사이드바 > 마케팅 > 메시지 발송` (`/message`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 메시지 발송                                 │
├─────────────────────────────────────────────────────────┤
│ [✉️ 메시지 전송]  [📋 발송 이력]                          │
├──────────────────────────┬──────────────────────────────┤
│ [보유 포인트: 52,480 P]  │ 발신 설정: 02-1234-5678      │
│ SMS 2,624건 발송 가능    │ [알림톡 활성] [SMS 가능]      │
├──────────────────────────┼──────────────────────────────┤
│ [수동 메시지 발송]        │ [자동 알림 설정]              │
│                          │                              │
│ 수신자 선택 *            │ 고객 관련:                    │
│ [전체][활성회원][만료임박]│ [☐ 계약완료시] [ON]           │
│ [PT회원][장기미출석]     │ [☐ 생일자]    [OFF]           │
│ [수신자 선택 영역]       │ [☐ 신규등록시] [ON]           │
│ [대상 검색]              │ ...                           │
│                          │                              │
│ 발송 채널 *              │ 상품 관련:                    │
│ [알림톡] [SMS/LMS] [푸시]│ [☐ 수강권만료시] [ON]         │
│ SMS: 90자/LMS: 2,000자  │ [☐ 회원권만료전] [ON]         │
│                          │ ...          [상세 설정 >]   │
│ 예상 비용: 8원/건 × 5명  │                              │
│ = 40원                   │                              │
│                          │                              │
│ 메시지 내용 *  450/1,000 │                              │
│ [████████████████░░░░░]  │                              │
│ [textarea 6행]           │                              │
│ #{이름} #{만료일} #{상품명}│                              │
│ #{잔여횟수} #{센터명}     │                              │
│                          │                              │
│ [☐ 예약 발송] [datetime] │                              │
│              [메시지 발송]│                              │
├──────────────────────────┴──────────────────────────────┤
│ [최근 발송 이력 DataTable]                               │
└─────────────────────────────────────────────────────────┘
```

### 4. 탭 구조

| 탭 | key | 아이콘 | 내용 |
|----|-----|--------|------|
| 메시지 전송 | send | Send | 수동 발송 폼 + 자동 알림 미리보기 + 최근 이력 |
| 발송 이력 | history | History | 전체 이력 테이블 (selectable, 엑셀 다운로드) |

### 5. 채널 설정 (CHANNEL_CONFIG)

| 채널 | label | 최대 글자 | 건당 비용 | 비고 |
|------|-------|----------|----------|------|
| kakao | 알림톡 | 1,000자 | 8원 | - |
| sms | SMS/LMS | 2,000자 | SMS: 70원 (≤90자), LMS: 30원 (>90자) | 자동 SMS/LMS 구분 |
| push | 앱 푸시 | 500자 | 0원 (무료) | - |

### 6. 수신자 선택

**그룹 빠른 선택 버튼:** 전체, 활성 회원, 만료임박 (7일), PT 회원, 장기 미출석

| 동작 | 설명 |
|------|------|
| "전체" 클릭 | useAllMembers 토글 → 전체 회원 수 표시 |
| 그룹 클릭 | toast.info("그룹 선택") (향후 구현) |
| 대상 검색 | DLG-071-R 수신자 검색 모달 표시 |
| 수신자 태그 X | 개별 수신자 제거 |

### 7. 메시지 에디터

| 기능 | 설명 |
|------|------|
| 글자수 표시 | `{현재}/{최대}자` (SMS면 "(SMS)" 또는 "(LMS)" 추가) |
| 초과 경고 | 90% 이상: amber, 초과: error + "초과" 텍스트 |
| 게이지 바 | 하단 프로그레스 바 (accent/amber/error) |
| 변수 삽입 | `#{이름}`, `#{만료일}`, `#{상품명}`, `#{잔여횟수}`, `#{센터명}` 클릭 시 본문 삽입 |

### 8. 발송 이력 테이블 컬럼

| 컬럼 | key | width | align | 렌더링 |
|------|-----|-------|-------|--------|
| 발송일시 | sentAt | 160 | left | "YYYY-MM-DD HH:mm" |
| 유형 | type | 100 | center | StatusBadge variant=info |
| 수신자 수 | recipients | 100 | right | "{N}명" |
| 예약일시 | scheduledAt | 140 | left | "YYYY-MM-DD HH:mm" |
| 상태 | status | 100 | center | SENT=녹색체크 "완료", SCHEDULED=주황경고 "예약", FAILED=빨강X "실패" |

### 9. 자동 알림 미리보기 (메시지 발송 페이지 우측)

고객 관련 5종 + 상품 관련 4종 토글 표시. [상세 설정 >] 클릭 → moveToPage(992) → 자동 알림 설정 페이지.

## SCR-072 자동 알림 설정

### 1. 화면 목적
회원 이벤트 발생 시 자동으로 메시지를 발송하는 13종 알림 규칙을 관리한다. 규칙별 활성/비활성 토글, 채널/시점/대상/템플릿 편집을 지원한다.

### 2. 접근 경로
`메시지 발송 > 자동 알림 상세 설정` (`/message/auto-alarm`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 자동 알림 설정                              │
│    [메시지 발송] [모두 사용] [+ 설정 추가]               │
├─────────────────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌────────────────────────┐    │
│ │ 발신 번호 │ │보유포인트 │ │ 활성화된 알림: 9/13종  │    │
│ │02-1234.. │ │125,400 P│ │     ● 정상 작동 중     │    │
│ └──────────┘ └──────────┘ └────────────────────────┘    │
├─────────────────────────────────────────────────────────┤
│ [고객 관련 자동 알림 (7종)]                               │
│ ┌──────────────────────┐ ┌──────────────────────┐       │
│ │ ● 만료 D-7 알림  [ON]│ │ ● 만료 D-3 알림  [ON]│       │
│ │ 알림톡 | [7] 일 전   │ │ 알림톡 | [3] 일 전   │       │
│ └──────────────────────┘ └──────────────────────┘       │
│ ┌──────────────────────┐ ┌──────────────────────┐       │
│ │ ○ 만료 D-1 알림 [OFF]│ │ ● 생일 축하 알림 [ON]│       │
│ │ SMS | [1] 일 전      │ │ 알림톡              │       │
│ └──────────────────────┘ └──────────────────────┘       │
│ ...                                                     │
├─────────────────────────────────────────────────────────┤
│ [상품 관련 자동 알림 (6종)]                               │
│ (동일 카드 레이아웃)                                     │
└─────────────────────────────────────────────────────────┘
```

### 5. 알림 규칙 카드 (RuleCard) 구조

## SCR-075 전자계약

### 1. 화면 목적
5단계 위자드를 통해 회원 선택 → 상품 선택 → 기간/금액 확정 → 결제 → 서명 확인의 전자계약 프로세스를 완료한다.

### 2. 접근 경로
`사이드바 > 마케팅 > 전자계약` (`/contracts/new`)

### 3. 위자드 5단계 개요

```
①──────②──────③──────④──────⑤
회원     상품    기간/    결제    확인
선택     선택    금액
```

### 4. Step 1: 회원 선택

**목적:** 계약 대상 회원을 검색하여 선택

| 요소 | 설명 |
|------|------|
| 검색 입력 | "이름 또는 전화번호로 검색" + Search 아이콘 |
| 신규 회원 등록 링크 | moveToPage(986) |
| 에러 표시 | "계약 대상 회원을 선택해주세요." (미선택 시) |
| DataTable | 이름, 연락처, 상태(ACTIVE/EXPIRED/HOLDING/INACTIVE/SUSPENDED 배지), 보유상품, [선택] 버튼 |
| 선택 확인 | accent-light 배너: "{이름} ({전화번호}) 회원이 선택되었습니다." |
| UX-12 | 회원 선택 시 확인 영역으로 자동 스크롤 |

### 5. Step 2: 상품 선택

**목적:** 계약할 상품을 카테고리별로 선택하여 장바구니에 추가

| 요소 | 설명 |
|------|------|
| 카테고리 탭 | 시설이용(facility), 1:1수업(pt), 그룹수업(gx), 옵션(option) |
| 상품 카드 | 상품명 + 기간(일) + 가격(formatKRW) + [선택/선택됨] |
| 장바구니 (우측 sticky) | 선택된 상품 목록 + 개별 삭제 + 총 합계 |
| 에러 표시 | "최소 한 개의 상품을 선택해주세요." |

**상품 카테고리 매핑 (DB → 탭):**

| DB category | 탭 key | 탭 라벨 |
|-------------|--------|---------|
| MEMBERSHIP | facility | 시설이용 |
| PT | pt | 1:1수업 |
| GX | gx | 그룹수업 |
| 기타 | option | 옵션 |

### 6. Step 3: 기간/금액

**목적:** 계약 기간, 할인, 서비스 일수를 설정하고 최종 금액을 확정

| 필드 | key | 타입 | 필수 | 설명 |
|------|-----|------|------|------|
| 시작일 | startDate | date | Y | 기본값: 오늘 |
| 종료일 | - | date (readonly) | - | 시작일 + 최대 상품 기간 + 서비스일수 자동 계산 |
| 서비스 일수 추가 | serviceDays | number | N | 0 이상 |
| 메모 | memo | textarea | N | 특약사항 |

**할인 설정:**

| 필드 | key | 타입 | 옵션/유효성 |
|------|-----|------|------------|
| 할인 유형 | discountType | select | 재등록, 신규, 이벤트, 관리자 재량 |
| 할인율 (%) | discountRate | number | 0~50% (최대 50% 제한 + 에러 메시지) |
| 할인 사유 | discountReason | text | 자유 입력 |

**금액 계산 로직:**
```
totalPrice = 선택 상품 합계
discountAmount = totalPrice × (discountRate / 100)
finalPrice = totalPrice - discountAmount
```

### 7. Step 4: 결제

**목적:** 결제 수단을 선택하고 금액을 분배

**결제 수단 (PAYMENT_METHODS):**

| id | label | 아이콘 |
|----|-------|--------|
| card | 카드 | CreditCard |
| cash | 현금 | Banknote |
| mileage | 마일리지 | Coins |
| transfer | 계좌이체 | ArrowRightLeft |

**복합 결제:** 여러 수단 선택 가능, 각 수단별 금액 분배 입력

**담당 직원 선택:** staffs 목록에서 담당 직원 선택 (Select)

### 8. Step 5: 확인 (서명)

**목적:** 최종 계약 내용 확인 및 전자 서명

| 요소 | 설명 |
|------|------|
| 요약 카드 | 회원 정보 + 상품 목록 + 기간 + 금액 + 할인 + 결제 수단 |
| 고객 서명 | SignaturePad 컴포넌트 → dataUrl 저장 |
| 센터장 서명 | SignaturePad 컴포넌트 → dataUrl 저장 |
| 서명 에러 | "고객 서명과 센터장 서명을 모두 완료해주세요." |

**저장 프로세스:**
1. 중복 결제 확인 (checkDuplicatePayment)
2. 서명 이미지 Supabase Storage 업로드 (contracts/{branchId}/{id}_{type}_{timestamp}.png)
3. contracts 테이블 INSERT (branchId, memberId, memberName, productName, amount, startDate, endDate, signatureUrl, managerSignatureUrl, signedAt, status='서명완료')
4. sales 테이블 INSERT (매출 레코드)
5. updateMembershipPeriod (이용권 기간 설정 + 회원 상태 ACTIVE)
6. accruePoints (마일리지 자동 적립, 결제금액의 1%)
7. toast.success("계약이 완료되었습니다.") → moveToPage(985, {id: memberId})

### 9. 네비게이션

| 버튼 | 위치 | 동작 |
|------|------|------|
| 이전 | 좌측 하단 | prevStep() (step > 1) |
| 다음 | 우측 하단 | validateStep(step) → nextStep() |
| 계약 완료 (Step 5) | 우측 하단 | handleContractSave() |
