## SCR-070 리드 관리

### 1. 화면 목적
잠재 고객(문의/상담) 파이프라인을 목록 뷰와 칸반 뷰로 관리한다. 리드 등록, 수정, 삭제, 통계 확인 기능을 제공한다.

### 2. 접근 경로
`사이드바 > 마케팅 > 리드 관리` (`/leads`)

| FC/코치 | ● | ● | ● | ● | — |

### 5. 통계 카드 (StatCardGrid cols=5)

| 카드 | 값 | 아이콘 | variant |
|------|-----|--------|---------|
| 전체 리드 | `{total}건` | Users | default |
| 등록 전환 | `{converted}건` | UserPlus | mint |
| 전환율 | `{conversionRate}%` | TrendingUp | peach |
| 대기 중 | `{pending}건` | Phone | default |
| 미전환율 | `{missedRate}%` | AlertCircle | default |

**통계 계산:**
- `converted`: 등록완료 건수
- `conversionRate`: `Math.round((converted / total) * 100)`
- `pending`: 신규 + 연락완료 건수
- `missedRate`: `Math.round((미전환 / total) * 100)`

## SCR-071 메시지 발송

### 1. 화면 목적
회원에게 알림톡/SMS/LMS/앱 푸시를 발송하고, 예약 발송 및 발송 이력을 관리한다.

### 2. 접근 경로
`사이드바 > 마케팅 > 메시지 발송` (`/message`)

### 3. ASCII 와이어프레임

```
┌──────────────────────────────────────────────────────────┐
│  PageHeader: "메시지 발송"                                │
├──────────────────────────────────────────────────────────┤
│  [메시지 전송]  [발송 이력]                                │
├──────────────────────────────────────────────────────────┤
│  ┌──────────┐ ┌────────────────────────────────────┐     │
│  │보유 포인트│ │발신 설정 [02-1234-5678 ▾]  알림톡활성│     │
│  │52,480 P  │ │                                    │     │
│  └──────────┘ └────────────────────────────────────┘     │
├──────────────────────────────────────────────────────────┤
│  ┌──────────────────────┐ ┌──────────────────────┐      │
│  │  수동 메시지 발송      │ │  자동 알림 설정        │      │
│  │  수신자 선택 *         │ │  고객 관련 (5종 토글)  │      │
│  │  [전체][활성][만료]... │ │  상품 관련 (4종 토글)  │      │
│  │  발송 채널 *           │ │        [상세 설정 →]   │      │
│  │  [알림톡][SMS/LMS][푸시]│ │                      │      │
│  │  예상 비용: {N}원      │ │                      │      │
│  │  메시지 내용 *         │ │                      │      │
│  │  [변수 삽입 버튼들]     │ │                      │      │
│  │  □ 예약 발송           │ │                      │      │
│  │          [메시지 발송]  │ │                      │      │
│  └──────────────────────┘ └──────────────────────┘      │
├──────────────────────────────────────────────────────────┤
│  최근 발송 이력 (5건 DataTable)                           │
└──────────────────────────────────────────────────────────┘
```

### 4. 발송 채널 설정

| 채널 | label | 아이콘 | 최대 글자 | 건당 비용 |
|------|-------|--------|----------|----------|
| kakao | 알림톡 | MessageSquare | 1,000 | 8원 |
| sms | SMS/LMS | Smartphone | 2,000 | SMS: 70원 (90자 이하), LMS: 30원 (90자 초과) |
| push | 앱 푸시 | Bell | 500 | 0원 (무료) |

### 5. 수신자 선택

**그룹 빠른 선택 버튼 5개:**
| 그룹 | 동작 |
|------|------|
| 전체 | `useAllMembers` 토글 |
| 활성 회원 | 활성 상태 회원 필터 |
| 만료임박 (7일) | D-7 이내 만료 회원 |
| PT 회원 | PT 이용권 보유 회원 |
| 장기 미출석 | 30일+ 미출석 회원 |

**수신자 검색 모달 (DLG-071-001):**
- 회원 목록에서 체크박스 다중 선택
- 이름/전화번호 검색
- 전체 선택/해제
- 선택 완료 → 수신자 목록 업데이트

### 6. 변수 삽입

| 변수 | 치환 |
|------|------|
| `#{이름}` | 회원명 |
| `#{만료일}` | 이용권 만료일 |
| `#{상품명}` | 보유 상품명 |
| `#{잔여횟수}` | PT 잔여 횟수 |
| `#{센터명}` | 센터 이름 |

### 7. 발송 미리보기 모달 (DLG-071-002)

- 채널 StatusBadge + 수신자 수 + 메시지 내용
- [취소] / [발송하기]

### 8. 발송 이력 탭

| 컬럼 | 렌더링 |
|------|--------|
| 발송일시 | YYYY-MM-DD HH:MM |
| 유형 | StatusBadge info |
| 수신자 수 | `${N}명` |
| 예약일시 | YYYY-MM-DD HH:MM 또는 "-" |
| 상태 | SENT=CheckCircle2(success)+"완료", SCHEDULED=AlertCircle(accent)+"예약", 기타=XCircle(error)+"실패" |

### 9. 비즈니스 로직

- **SMS/LMS 자동 전환**: 90자 이하=SMS(70원), 90자 초과=LMS(30원)
- **예상 비용**: `수신자수 × 건당비용`
- **예약 발송**: status='SCHEDULED', sentAt=null, scheduledAt=예약시간
- **즉시 발송**: status='SENT', sentAt=현재시간

## SCR-072 자동 알림 설정

### 2. 접근 경로
