## SCR-060 직원 목록

### 1. 화면 목적
센터에 등록된 전체 직원 정보를 조회하고, 역할별/상태별 필터링, 일괄 퇴사 처리, 메시지 전송, 엑셀 다운로드 등의 관리 기능을 제공한다.

### 2. 접근 경로
`사이드바 > 직원 관리 > 직원 목록` (`/staff`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 직원 관리                    [+ 직원 등록]  │
│ "센터의 직원 정보를 관리합니다."                          │
├─────────────────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│ │ 전체 직원 │ │ 재직 중   │ │ 휴직 중   │ │ 퇴사     │    │
│ │  12명     │ │  10명     │ │   1명     │ │   1명     │    │
│ └──────────┘ └──────────┘ └──────────┘ └──────────┘    │
├─────────────────────────────────────────────────────────┤
│ [🔍 직원 이름, 연락처 검색] [역할 ▼] [재직 상태 ▼]      │
├─────────────────────────────────────────────────────────┤
│ [메시지 전송] [상태 변경] [퇴사 처리] [명단 다운로드]     │
├─────────────────────────────────────────────────────────┤
│ ☐ │ No │ 직원명  │ 역할     │ 연락처       │ 입사일    │ │
│ ☐ │ 1  │ 김민수  │ 트레이너 │ 010-1234-... │ 2024-03  │ │
│ ☐ │ 2  │ 이서연  │ FC       │ 010-5678-... │ 2024-05  │ │
│ ☐ │ 3  │ 박지훈  │ 매니저   │ 010-9012-... │ 2023-11  │ │
│   │    │ ...     │          │              │          │ │
├─────────────────────────────────────────────────────────┤
│                    < 1 2 3 ... >                         │
└─────────────────────────────────────────────────────────┘
```

### 4. 통계 카드 (StatCardGrid cols=4)

| 카드 | 값 | 아이콘 | variant | description |
|------|-----|--------|---------|-------------|
| 전체 직원 | `{total}명` | Users | default | 등록된 총 인원 |
| 재직 중 | `{active}명` | UserCheck | mint | 현재 근무 인원 |
| 휴직 중 | `{onLeave}명` | Clock | peach | 휴직자 현황 |
| 퇴사 | `{resigned}명` | UserMinus | default | 퇴사 처리 인원 |

| 버튼 | 조건 | 동작 |
|------|------|------|

| 메시지 전송 (Outline) | selectedRows > 0 | moveToPage(980) → 메시지 발송 |
| 상태 변경 (Outline) | selectedRows > 0 | 상태 변경 기능 (미구현) |
| 퇴사 처리 (Danger) | selectedRows > 0 | DLG-060 퇴사 확인 다이얼로그 |
| 명단 다운로드 (Outline) | 항상 활성 | exportToExcel(전체 필터 데이터, '직원목록') |
| 직원명 클릭 | - | moveToPage(998, {id: row.id}) → 직원 상세 |
| 엑셀 다운로드 (DataTable 내장) | - | exportToExcel(필터된 데이터, '직원목록') |

### 8. 상태별 화면

| 상태 | 표시 |
|------|------|

| 에러 | toast.error("직원 데이터를 불러오지 못했습니다.") |

## SCR-061 직원 등록/수정

### 1. 화면 목적
새 직원 정보를 입력하여 등록하거나, 기존 직원 정보를 수정한다. 역할 선택 시 해당 역할의 권한 미리보기를 제공한다.

### 2. 접근 경로
- 등록: `직원 목록 > [+ 직원 등록]` (`/staff/new`)
- 수정: `직원 목록 > 직원명 클릭` (`/staff/new?id={staffId}`)

```
├─────────────────────────────────────────────────────────┤

│ ┌─────────────────────────────────────────────┐         │
│ │ 이름 *        [👤 홍길동                   ] │         │
│ │ 역할 *        [▼ 역할 선택                 ] │         │
```

## SCR-062 직원 퇴사 처리

### 1. 화면 목적
4단계 위자드를 통해 직원 퇴사를 단계적으로 처리한다. FC 직원의 경우 담당 회원 재배정, 미래 스케줄 처리까지 포함한다.

### 2. 접근 경로
`직원 목록 > 퇴사 처리 버튼` 또는 `직접 URL` (`/staff/resignation?staffId={id}`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 직원 퇴사 처리                              │
│ "단계별로 퇴사 정보를 입력하고 처리합니다."               │
├─────────────────────────────────────────────────────────┤
│   ①──────②──────③──────④                               │
│  직원·    담당 회원  스케줄   최종                         │
│  퇴사정보  재배정    확인    확인                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  (각 스텝 콘텐츠 영역)                                   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│ [< 이전]                                    [다음 >]     │
└─────────────────────────────────────────────────────────┘
```

### 4. 위자드 4단계 상세

#### Step 1: 직원·퇴사 정보

**목적:** 퇴사 대상 직원 선택 및 퇴사 사유 입력

| 필드명 | key | 타입 | 필수 | 유효성 | 에러 메시지 |
|--------|-----|------|------|--------|------------|
| 직원 | selectedStaffId | select (전체 직원 목록) | Y | 선택 필수 | "퇴사 처리할 직원을 선택하세요." |
| 퇴사 예정일 | resignDate | date (min=오늘) | Y | 오늘 이후 | "퇴사 예정일을 입력하세요." |
| 퇴사 사유 | resignReason | textarea (3행) | N | - | - |

**선택된 직원 카드:** 이름, 역할 배지, 입사일, 담당 회원 수 (FC인 경우) 표시

**네비게이션:**
- FC 직원 → Step 2로 이동 (담당 회원 로드)
- FC 외 직원 → Step 2 스킵, Step 3으로 이동

#### Step 2: 담당 회원 재배정 (FC 전용)

**목적:** 퇴사 FC의 담당 회원을 다른 FC에게 재배정

| 필드명 | 타입 | 설명 |
|--------|------|------|
| 일괄 배정 FC | select | 활성 FC 목록 (퇴사 대상 제외) |
| 미배정 회원에 적용 | button | 미배정 회원에 일괄 FC 지정 |
| 개별 재배정 담당자 | select (회원별) | 각 회원마다 개별 FC 선택 가능 |

**테이블 컬럼 (SimpleTable):**

| 컬럼 | key | 렌더링 |
|------|-----|--------|
| 회원명 | name | 폰트 medium |
| 연락처 | phone | 텍스트 |
| PT 잔여 | hasPtRemaining | "PT잔여" 에러 배지 또는 "-" |
| 재배정 담당자 | assignedStaffId | Select 드롭다운 (FC 목록) |

**검증:** PT 잔여 회원 중 미배정 건수 > 0이면 "다음" 차단 + toast.warning

#### Step 3: 미래 스케줄 확인

**목적:** 퇴사 예정일 이후 예정된 스케줄 확인 및 처리 방법 결정

**스케줄 없는 경우:** 체크 아이콘 + "퇴사 예정일 이후 예정된 스케줄이 없습니다."

**스케줄 있는 경우:**
- 테이블: 스케줄명, 유형(PT/GX 배지), 날짜
- 처리 옵션 (RadioGroup vertical):
  - `transfer`: 후임에게 일괄 이관 (재배정된 담당자에게 스케줄을 이관)
  - `cancel`: 일괄 취소 (모든 스케줄을 취소)

#### Step 4: 최종 확인

**목적:** 입력 내용 전체 요약 및 최종 확인

**경고 배너:** bg-error/5 + "퇴사 예정 등록 후에는 수정이 어려울 수 있습니다."

**요약 카드 항목:**

| 라벨 | 값 |
|------|-----|
| 퇴사 대상 | {이름} ({역할}) |
| 퇴사 예정일 | YYYY년 M월 D일 |
| 퇴사 사유 | 입력값 또는 "(미입력)" |
| 담당 회원 재배정 (FC만) | {N}명 완료 |
| 미래 스케줄 | "예정된 스케줄 없음" / "{N}건 후임 이관" / "{N}건 일괄 취소" |

**버튼:**
- [이전]: Step 3으로
- [퇴사 예정 등록] (bg-error): handleSubmit → reassignMembers API → scheduleResignation API → toast.success → moveToPage(974)

### 5. 상태별 화면

| 상태 | 표시 |
|------|------|
| 제출 중 | "처리 중..." + disabled |
| 재배정 실패 | toast.error("담당 회원 재배정 실패: {message}") |
| 퇴사 등록 실패 | toast.error("퇴사 예정 등록 실패: {message}") |
| 처리 중 오류 | toast.error("처리 중 오류가 발생했습니다.") |

## SCR-063 직원 근태 관리

### 2. 접근 경로

### 3. ASCII 와이어프레임


## SCR-065 급여 명세서

### 1. 화면 목적
직원별 월간 급여 명세서를 상세 조회하고, PDF 저장/이메일 발송/일괄 인쇄 기능을 제공한다.

### 2. 접근 경로
`급여 관리 > 명세서 바로가기` (`/payroll/statements`)

### 3. ASCII 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│ [PageHeader] 급여 명세서                 [일괄 인쇄]     │
├─────────────────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐                  │
│ │총 실지급액│ │ 지급완료  │ │  미지급   │                  │
│ │28,500,000│ │   10건   │ │   2건    │                  │
│ └──────────┘ └──────────┘ └──────────┘                  │
├──────────────────────────┬──────────────────────────────┤
│ [명세서 조회]             │ [2026년 4월 전체 지급 현황]   │
│ 직원 선택: [▼ 김민수]    │                              │
│ 지급 월:   [▼ 2026년 4월]│ (DataTable: 직원별 요약)      │
│                          │                              │
│ ┌──────────────────────┐ │                              │
│ │ 김민수 / 트레이너     │ │                              │
│ │ ● 지급완료 / 4/25    │ │                              │
│ ├──────────┬───────────┤ │                              │
│ │ + 지급항목│ - 공제항목│ │                              │
│ │ 기본급  2.8M│ 국민연금│ │                              │
│ │ 인센티브 320K│ 건강보험│ │                              │
│ │ 합계  3.12M│ 합계 85K│ │                              │
│ ├──────────┴───────────┤ │                              │
│ │  → 실지급액 3,035,000│ │                              │
│ ├──────────────────────┤ │                              │
│ │ [이메일 발송][PDF 저장]│ │                              │
│ └──────────────────────┘ │                              │
└──────────────────────────┴──────────────────────────────┘
```

### 4. 통계 카드 (3열)

| 카드 | 값 | 아이콘 | variant |
|------|-----|--------|---------|
| 총 실지급액 | formatKRW(totalNet) | FileText | default |
| 지급완료 | `{N}건` | CheckCircle2 | mint |
| 미지급 | `{N}건` | Clock | peach |

### 5. 명세서 상세 구조

**지급 항목 (earnings):**
- 기본급, 직무수당, 상여금, 인센티브 등 동적 항목
- 항목별 금액 + 지급 총액 합산 (녹색 bold)

**공제 항목 (deductions):**
- 국민연금, 건강보험, 고용보험, 소득세 등 동적 항목
- 항목별 금액 (빨강) + 공제 총액 합산 (빨강 bold)

**실지급액:** 지급총액 - 공제총액 (28px bold, accent 색상)

### 6. 버튼 → 동작 매핑

| 버튼 | 동작 |
|------|------|
| 일괄 인쇄 | window.print() |
| 이메일 발송 | toast.info("이메일 발송 기능은 추후 지원 예정입니다.") |
| PDF 저장 | toast.info → setTimeout → window.print() |
| 상세보기 (테이블 내) | 모달(DLG-065)에 해당 직원 명세서 표시 |

### 7. 상태별 화면

| 상태 | 표시 |
|------|------|
| 명세서 없음 | FileText 아이콘 + "명세서 없음" + "선택한 직원 및 월의 급여 데이터가 없습니다." |
| 로딩 중 | "데이터를 불러오는 중..." |
| readonly 사용자 | 직원 선택 드롭다운 disabled (본인만 조회) |
