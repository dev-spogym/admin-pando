### 7. 필터 옵션

| 필터 | key | type | 옵션 |
|------|-----|------|------|
| 검색 | searchQuery | text | placeholder: "직원 이름, 연락처 검색" |
| 역할 | role | select | 센터장, 매니저, FC, 트레이너, 스태프 |
| 재직 상태 | status | select | 재직(active), 휴직(leave), 퇴사(resigned) |

**필터링 로직:**
```typescript
let data = staffData.filter(s => {
  const matchSearch = !searchQuery || s.name.includes(searchQuery) || s.contact.includes(searchQuery);
  const matchRole   = !filterValues.role   || s.role   === filterValues.role;
  const matchStatus = !filterValues.status || s.status === filterValues.status;
  return matchSearch && matchRole && matchStatus;
});
```

**정렬:** 3-state 토글 (`null` → `asc` → `desc` → `null`). 정렬 가능 컬럼: `name`, `role`, `joinDate`, `status`.

## SCR-061 직원 등록/수정

### 1. 화면 목적
새로운 직원을 등록하거나 기존 직원 정보를 수정하는 폼 화면. `react-hook-form` + `zod` 스키마 기반 유효성 검증.

### 2. 접근 경로
- 등록: `직원 목록 > + 직원 등록` (`/staff/new`)

## SCR-062 직원 퇴사 처리

### 1. 화면 목적
4단계 위자드를 통해 직원 퇴사를 체계적으로 처리한다. FC 역할은 담당 회원 재배정 단계가 추가된다.

### 2. 접근 경로
`직원 목록 > 퇴사 처리 버튼` (`/staff/resignation?staffId={id}`)

### 3. 위자드 4단계

| 단계 | 이름 | 설명 | FC 외 |
|------|------|------|------|
| Step 1 | 직원·퇴사 정보 | 퇴사 대상 직원 선택 + 퇴사 예정일/사유 입력 | 동일 |
| Step 2 | 담당 회원 재배정 | 담당 회원을 다른 FC에 재배정 | **건너뜀** |
