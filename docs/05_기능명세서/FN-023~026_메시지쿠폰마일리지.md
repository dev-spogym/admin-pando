# FN-023~026: 메시지, 마케팅, 마일리지

## FN-023: 메시지 발송

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-023 |
| **기능명** | 메시지 발송 |
| **분류** | 메시지/마케팅 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-070 |
| **관련 API ID** | API-025 (미정) |

### 기능 설명
SMS, LMS, 카카오톡 메시지를 발송합니다. 변수 치환(예: #{이름})을 지원하며, 발송 이력을 관리합니다.

### 상세 요구사항
1. 수신자 선택 (개별/그룹/전체)
2. 메시지 유형 선택 (SMS/LMS/카카오톡)
3. 메시지 본문 입력 + 변수 삽입
4. 발송 비용 사전 표시
5. 예약 발송 (선택사항)
6. 발송 이력 조회
7. 발송 실패 건 재발송

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 수신자 | String/UUID[] | 필수 | "개별" \| "그룹" \| "전체" \| [회원ID, ...] |
| 메시지유형 | String | 필수 | "SMS" (90자) \| "LMS" (2000자) \| "카카오톡" |
| 본문 | String | 필수 | 유형별 문자수 제한 (변수 포함) |
| 변수 | Object | 선택 | {회원명: "#{이름}", 남은일수: "#{days_left}"} |
| 예약발송 | Boolean | 선택 | false (기본값) |
| 발송예정시간 | DateTime | 조건부 | 예약 발송 시 필수 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 발송 결과 | Object | {success_count, fail_count, total_cost} |
| 발송 이력 | Array | [{id, recipient, message, type, status, sent_at}, ...] |

### 비즈니스 규칙
1. **문자수 제한**:
   - SMS: 90자 이내 (90자 초과 시 LMS로 자동 변경)
   - LMS: 2000자 이내
   - 카카오톡: 1000자 이내
2. **발송 비용**:
   - SMS: 30원/건
   - LMS: 60원/건
   - 카카오톡: 15원/건
3. **변수 치환**: 각 회원별로 변수 자동 치환
4. **예약 발송**: 최대 30일 후까지 (과거 시간 불가)
5. **발송 이력**: 모든 발송 기록 저장 (성공/실패 포함)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 수신자 없음 | "수신자를 선택해주세요" |
| 본문 미입력 | "메시지 본문을 입력해주세요" |
| 문자수 초과 | "{{유형}}: 최대 {{제한}}자입니다 (현재 {{현재}}자)" |
| 예약 시간 과거 | "예약 발송 시간은 현재 시간 이후여야 합니다" |
| 발송 비용 부족 | "발송 비용이 부족합니다. 충전 후 발송해주세요" (선택사항) |
| 발송 실패 | "메시지 발송에 실패했습니다. 재시도하시겠습니까?" |

### Web 구현 사항
- **수신자 선택**: Radio (개별/그룹/전체) → 상세 선택 모달
- **메시지 입력**: Textarea (문자수 카운터 표시)
- **문자수 체크**: onChange 이벤트로 실시간 계산 및 유형 자동 변경
- **변수 삽입**: 드롭다운 메뉴 (#{이름}, #{연락처} 등)
- **발송 비용**: 카드로 표시 (수신자 수 × 비용)
- **예약 발송**: DateTimePicker (미래 날짜만)
- **발송 이력**: 테이블 (날짜, 수신자, 메시지 미리보기, 상태)

## FN-024: 자동 알림 설정

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-024 |
| **기능명** | 자동 알림 설정 |
| **분류** | 메시지/마케팅 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-070 |
| **관련 API ID** | API-026 (미정) |

### 기능 설명
만료/생일/미출석 등 특정 이벤트 발생 시 자동으로 알림을 발송하도록 규칙을 설정합니다.

### 상세 요구사항
1. 자동 알림 규칙 설정 (ON/OFF)
2. 규칙 유형:
   - 만료 D-7 알림
   - 만료 D-3 알림
   - 만료 D-1 알림
   - 생일 축하 알림
   - 장기 미출석(30일) 알림
3. 각 규칙별 메시지 템플릿 설정
4. 발송 시간 설정 (기본: 오전 9시)
5. 규칙별 통계 (발송 건수, 마지막 발송 시간)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 규칙ID | String | 필수 | "expire_d7" \| "expire_d3" \| "expire_d1" \| "birthday" \| "inactive30" |
| 활성화 | Boolean | 필수 | true \| false |
| 메시지템플릿 | String | 필수 | 변수 포함 (#{이름}, #{days_left}) |
| 발송시간 | Time | 선택 | HH:MM (기본값: 09:00) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 규칙 목록 | Array | [{rule_id, name, enabled, template, send_time, last_sent}, ...] |
| 통계 | Object | {rule_id: {sent_count, last_sent_at}} |

### 비즈니스 규칙
1. **발송 스케줄**: 매일 오전 9시(또는 설정 시간)에 자동 실행
2. **중복 발송 방지**: 동일 회원, 동일 규칙 1회/일 (중복 발송 불가)
3. **메시지 템플릿**: 필수 필드, 변수는 자동 치환
4. **발송 대상 필터**:
   - 만료 D-7: 이용권 만료일이 오늘 + 7일인 회원
   - 만료 D-3: 이용권 만료일이 오늘 + 3일인 회원
   - 생일: 오늘이 생년월일인 회원
   - 장기 미출석: 지난 30일 출석 0회인 회원
5. **템플릿 미입력**: 규칙 활성화 불가

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 템플릿 미입력 | "메시지 템플릿을 입력해주세요" + 규칙 활성화 불가 |
| 발송 실패 | 로그에 기록, 다음 실행 시 재시도 |

### Web 구현 사항
- **규칙 목록**: 카드 또는 테이블 (규칙명, 활성화 토글, 마지막 발송 시간)
- **규칙 상세**: 모달 (템플릿 에디터, 발송 시간 설정)
- **템플릿 에디터**: Textarea + 변수 삽입 버튼
- **통계**: 각 규칙 카드에 발송 건수/마지막 발송 시간 표시
- **발송 시간**: TimePicker

## FN-025: 쿠폰 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-025 |
| **기능명** | 쿠폰 관리 |
| **분류** | 메시지/마케팅 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-072 |
| **관련 API ID** | API-027 (미정) |

### 기능 설명
할인 쿠폰을 생성, 발급, 관리합니다. 유효기간과 사용 한도를 설정할 수 있습니다.

### 상세 요구사항
1. 할인율/금액 쿠폰 생성
2. 유효기간 설정
3. 사용 한도 설정 (무제한 가능)
4. 발급 대상 선택 (개별/그룹/전체)
5. 쿠폰 코드 자동 생성
6. 사용 이력 추적
7. 만료 쿠폰 자동 비활성화
8. 사용 한도 도달 시 자동 마감

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 할인타입 | String | 필수 | "금액" \| "할인율" |
| 할인값 | Number | 필수 | 금액 또는 % |
| 유효기간시작 | Date | 필수 | YYYY-MM-DD |
| 유효기간종료 | Date | 필수 | 시작일보다 나중 |
| 사용한도 | Number | 선택 | 무제한은 null (기본값) |
| 발급대상 | String/UUID[] | 필수 | "개별" \| "그룹" \| "전체" |
| 쿠폰명 | String | 필수 | 1~50자 |
| 설명 | String | 선택 | 최대 200자 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 쿠폰 정보 | Object | {id, name, code, discount_type, discount_value, valid_from, valid_to, usage_limit, issued_count, used_count} |
| 사용 이력 | Array | [{id, coupon_id, user_id, used_at, amount_discounted}, ...] |

### 비즈니스 규칙
1. 만료 쿠폰 자동 비활성화 (유효기간 종료 시)
2. 사용 한도 도달 시 자동 마감
3. 쿠폰 코드 자동 생성 (예: COUPON-20260311-ABC123)
4. 사용 이력: 쿠폰 사용 시 자동 기록
5. 할인값:
   - 금액: 0 이상의 정수
   - 할인율: 0~100%

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 만료 쿠폰 사용 | "기간이 만료된 쿠폰입니다" |
| 한도 초과 | "사용 한도에 도달한 쿠폰입니다" |
| 유효기간 역순 | "유효기간 종료일이 시작일보다 나중이어야 합니다" |
| 할인값 음수 | "0 이상의 숫자를 입력해주세요" |

### Web 구현 사항
- **쿠폰 목록**: DataTable (쿠폰명, 코드, 할인값, 유효기간, 사용 현황)
- **생성/수정 모달**: Form (할인타입, 할인값, 기간, 한도)
- **발급 대상**: Radio (개별/그룹/전체) → 상세 선택
- **사용 이력**: 탭 또는 모달 (사용자, 사용 일시, 할인액)
- **통계**: 카드 (총 발급/사용/잔여)

## FN-026: 마일리지 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-026 |
| **기능명** | 마일리지 관리 |
| **분류** | 메시지/마케팅 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-072 |
| **관련 API ID** | API-028 (미정) |

### 기능 설명
회원의 마일리지를 수동으로 적립/차감하고, 잔액을 조회하며, 이력을 관리합니다.

### 상세 요구사항
1. 회원 선택
2. 마일리지 적립/차감
3. 사유 입력 (자동/수동, 프로모션 등)
4. 현재 잔액 표시
5. 마일리지 이력 조회 (날짜별)
6. 자동 적립률 설정 (결제 시)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 회원ID | UUID | 필수 | 검색으로 선택 |
| 금액 | Number | 필수 | 0 이상 (적립) / 0 이상 (차감) |
| 유형 | String | 필수 | "적립" \| "차감" \| "수동조정" |
| 사유 | String | 필수 | "자동적립" \| "프로모션" \| "관리자조정" \| "기타" |
| 설명 | String | 선택 | 최대 100자 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 마일리지 잔액 | Number | 현재 보유 마일리지 |
| 마일리지 이력 | Array | [{date, type, reason, amount, balance, note}, ...] |

### 비즈니스 규칙
1. **잔액 제약**: 차감 후 잔액이 음수가 되면 불가
   - 에러: "마일리지 잔액이 부족합니다 (부족액: {{amount}})"
2. **자동 적립**: 결제 완료 시 자동 적립률에 따라 적립
   - 설정 예: 결제액의 1% = 마일리지
3. **유효기간** (선택사항): 마일리지 적립 후 1년 유효
4. **이력 기록**: 모든 적립/차감 기록 저장
5. **음수 입력 방지**: Number Input으로 자동 제어

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 잔액 부족 | "마일리지 잔액이 부족합니다. 현재 잔액: {{balance}}, 필요액: {{amount}}" |
| 음수 입력 | "0 이상의 숫자를 입력해주세요" |
| 회원 미존재 | "존재하지 않는 회원입니다" |
| 조정 실패 | "마일리지 조정에 실패했습니다. 다시 시도해주세요" |

### Web 구현 사항
- **회원 선택**: Combobox (현재 잔액 표시)
- **마일리지 입출금**: 탭 또는 Radio (적립/차감)
- **금액 입력**: Number Input
- **사유 선택**: Select 드롭다운
- **현재 잔액**: 큰 텍스트로 강조 표시 (파랑)
- **이력 테이블**: TanStack Table (날짜, 유형, 사유, 금액, 잔액)
- **통계**: 총 적립액, 총 사용액, 현재 잔액
