# FN-002: 대시보드 통계 조회

## 기본 정보

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-002 |
| **기능명** | 대시보드 통계 조회 (지점별 KPI 요약) |
| **분류** | 대시보드 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-002 |
| **관련 API ID** | API-003 |

---

## 기능 설명

로그인 후 첫 화면에서 지점별 핵심 KPI를 한눈에 볼 수 있는 대시보드입니다. 회원 현황, 생일자, 미납 회원, 만료 예정 이용권, 당월 매출 차트 등을 표시하여 관리자가 빠르게 지점 상황을 파악할 수 있습니다.

### 주요 시나리오
- 출근 시 대시보드 접속 → 오늘의 주요 지표 확인
- 회원 카드 클릭 → 해당 회원 목록 화면으로 이동
- 매출 그래프 클릭 → 매출 상세 조회 화면으로 이동
- 새로고침 → 최신 데이터 조회

---

## 상세 요구사항

1. 회원 통계 카드 (총수/활성/만료임박/만료)
2. 오늘 생일 회원 리스트 (최대 5명, 이름/연락처)
3. 미납 회원 리스트 (최대 5명, 이름/미납액)
4. 만료 예정 회원 (D-7 이내, 최대 5명)
5. 당월 매출 차트 (일별 막대 그래프)
6. 매출 요약 (총액/현금/카드/마일리지)
7. 최근 가입 회원 (최대 5명)
8. 출석 현황 (오늘 출석자 수/비율)
9. 자동 새로고침 (5분 주기, 선택사항)
10. 날짜별 필터 (오늘/이번주/이번달 선택)

---

## 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 지점 ID | UUID | 필수 | JWT에서 추출, 사용자가 선택한 지점 |
| 기간 필터 | String | 선택 | "today" \| "week" \| "month" (기본값: "today") |
| 자동 새로고침 | Boolean | 선택 | 5분 주기 자동 갱신 여부 (기본값: false) |

---

## 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 총 회원 수 | Number | 활성 + 비활성 + 만료 모두 포함 |
| 활성 회원 수 | Number | 이용권 유효 중인 회원 |
| 만료임박 회원 수 | Number | D-7 이내 회원 |
| 만료 회원 수 | Number | 이용권 만료된 회원 |
| 생일 회원 목록 | Array | [{name, phone, birthdate}, ...] |
| 미납 회원 목록 | Array | [{name, phone, amount}, ...] |
| 만료 예정 목록 | Array | [{name, membership_end_date}, ...] |
| 당월 매출 데이터 | Array | [{date, amount}, ...] (최근 30일) |
| 총 매출 | Number | 당월 전체 매출액 |
| 현금 매출 | Number | 현금 결제 매출 |
| 카드 매출 | Number | 카드 결제 매출 |
| 마일리지 매출 | Number | 마일리지 사용 매출 |
| 오늘 출석자 수 | Number | 금일 체크인 회원 수 |
| 출석률 | Number | (출석자 / 활성 회원) * 100 (%) |
| 최근 가입 목록 | Array | [{name, email, reg_date}, ...] |

## 비즈니스 규칙

1. **데이터 격리**: branch_id 기준으로 해당 지점 데이터만 표시
   - 다지점 관리자는 각 지점별 선택 후 조회

2. **만료임박 정의**: D-7 이내 (오늘 기준 이용권 만료일 <= 7일 남음)

3. **매출 차트 범위**: 최근 30일 (오늘로부터 과거 30일)

4. **활성 회원 판단**: 이용권.end_date >= 오늘

5. **데이터 새로고침**:
   - 수동 새로고침: 즉시 최신 데이터 조회
   - 자동 새로고침: 5분 주기 (백그라운드 갱신, 사용자 현재 화면 유지)

6. **리스트 정렬 순서**:
   - 생일자: 오늘 기준 생일 임박순
   - 미납: 미납액 큰 순서
   - 만료 예정: 만료일 빠른 순서
   - 최근 가입: 가입일 최신순

7. **연락처 표시 형식**:
   - 마지막 4자리만 표시: "010-****-1234"
   - 관리자 권한 있으면 전체 표시 가능

## 예외 처리

| 상황 | 오류 코드 | 대응 방법 |
|------|----------|---------|
| 데이터 로딩 실패 | LOAD_ERROR | 섹션별 에러 표시: "데이터를 불러올 수 없습니다. 새로고침을 시도해주세요" |
| 부분 실패 (일부 섹션만) | PARTIAL_LOAD | 성공한 섹션은 표시, 실패한 섹션만 에러 표시 |
| 네트워크 오류 | NETWORK_ERROR | "인터넷 연결을 확인해주세요" + 재시도 버튼 |
| 데이터 없음 | NO_DATA | 각 섹션에 "데이터가 없습니다" 빈 상태 UI 표시 |
| 권한 없음 | FORBIDDEN | "조회 권한이 없습니다. 관리자에게 문의하세요" |
| 서버 오류 | SERVER_ERROR | "일시적 오류가 발생했습니다. 잠시 후 다시 시도해주세요" |
| 토큰 만료 | UNAUTHORIZED | 로그인 화면으로 이동 |

## Web 구현 사항

### 사용 라이브러리
- **데이터 페칭**: React Query (parallel queries 지원)
- **차트**: Recharts (BarChart, LineChart)
- **UI 컴포넌트**: Card, Grid, Skeleton (로딩 상태)
- **상태 관리**: Zustand (필터 상태)
- **날짜 처리**: date-fns 또는 dayjs

### 구현 세부사항

#### 1. 레이아웃 구조
```
대시보드
├── 헤더 (지점명, 새로고침 버튼, 날짜 필터)
├── 통계 카드 영역 (4개: 총수/활성/만료임박/만료)
├── 2열 그리드
│   ├── 왼쪽
│   │   ├── 생일 회원 카드
│   │   ├── 미납 회원 카드
│   │   └── 만료 예정 카드
│   └── 오른쪽
│       ├── 매출 차트
│       ├── 매출 요약 카드
│       └── 출석 현황 카드
└── 하단
    └── 최근 가입 회원 리스트
```

#### 2. React Query 설정
```typescript
// Parallel queries (모든 데이터 동시 로딩)
const { data: members } = useQuery('dashboard-members', ...);
const { data: birthdays } = useQuery('dashboard-birthdays', ...);
const { data: overdue } = useQuery('dashboard-overdue', ...);
// ...

// 새로고침 주기 (5분)
const queryClient = useQueryClient();
useEffect(() => {
  const interval = setInterval(() => {
    if (autoRefresh) {
      queryClient.refetchQueries('dashboard-*');
    }
  }, 5 * 60 * 1000);
  return () => clearInterval(interval);
}, [autoRefresh, queryClient]);
```

#### 3. 에러 처리 (부분 실패 지원)
```typescript
// 섹션별 독립적 에러 상태 관리
const sections = {
  members: { data: null, error: null, isLoading: true },
  birthdays: { data: null, error: null, isLoading: true },
  // ...
};

// 일부 실패해도 다른 섹션은 표시
<Section key="birthdays" status={sections.birthdays} />
```

#### 4. 차트 구성 (Recharts)
```
BarChart:
- X축: 날짜 (MM-DD 형식)
- Y축: 매출액
- 막대 색상: 현금(파랑)/카드(초록)/마일리지(보라)
- Tooltip: 날짜, 각 항목별 금액, 합계

LineChart (대체):
- 일일 매출 추이선
- 마우스 hover 시 상세 정보 표시
```

#### 5. 카드 클릭 네비게이션
```
- 생일 회원 클릭 → /members?filter=birthday
- 미납 회원 클릭 → /members?filter=overdue
- 만료 예정 클릭 → /members?filter=expiring
- 매출 클릭 → /sales
- 출석 클릭 → /attendance
```

#### 6. 로딩 상태 UX
- 초기 로딩: Skeleton placeholders (각 카드/리스트)
- 새로고침: 기존 데이터 유지 + 백그라운드 갱신 (로딩 인디케이터 표시)
- 부분 로딩: 완료된 섹션만 먼저 표시

#### 7. 반응형 디자인
- 데스크톱: 2열 그리드
- 태블릿: 1열 그리드 (넓이 조정)
- 모바일: 단열 스택 레이아웃

## 화면 요소

### 헤더 영역
- 지점명 (큰 타이틀)
- 오늘 날짜 표시
- 새로고침 버튼 (아이콘 + 로딩 인디케이터)
- 날짜 필터 (버튼그룹: 오늘/주/월)
- 자동 새로고침 토글 (선택사항)

### 통계 카드 (4개)
- 카드1: 총 회원 수 (큰 숫자 + 색상 배경)
- 카드2: 활성 회원 (그린 강조)
- 카드3: 만료임박 (오렌지 강조)
- 카드4: 만료 회원 (레드 강조)
- 각 카드 클릭 시 해당 필터 적용 후 회원 목록으로 이동

### 생일 회원 카드
- 헤더: "생일" 타이틀 + "{{ count }}명"
- 리스트: 이름 + "오늘 생일입니다" 또는 "D-{{ daysUntil }}"
- 각 항목 클릭 시 회원 상세 페이지로 이동
- 비어있으면: "생일자가 없습니다"

### 미납 회원 카드
- 헤더: "미납" 타이틀 + "{{ count }}명"
- 리스트: 이름 + 미납액 (빨강 강조)
- 각 항목 클릭 시 회원 상세 페이지 → 결제 탭으로 이동
- 비어있으면: "미납 회원이 없습니다"

### 만료 예정 카드
- 헤더: "만료 예정" 타이틀 + "{{ count }}명"
- 리스트: 이름 + "D-{{ daysUntil }}"
- 각 항목 클릭 시 회원 상세 페이지 → 이용권 탭으로 이동

### 매출 차트
- BarChart 또는 LineChart
- 제목: "당월 매출 추이"
- Tooltip on hover: "{{ date }} {{ amount | currency }}"
- 범례: 현금/카드/마일리지

### 매출 요약 카드
- 총 매출: 가장 큰 텍스트 (파랑 강조)
- 현금: 작은 텍스트
- 카드: 작은 텍스트
- 마일리지: 작은 텍스트
- 각 항목 클릭 시 매출 상세 조회로 이동

### 출석 현황 카드
- "오늘 출석자 수"
- 큰 숫자 (파랑 강조)
- "활성 회원의 {{ attendanceRate }}%"
- 원형 프로그레스 바 (선택)
- 클릭 시 출석 조회 화면으로 이동

### 최근 가입 리스트
- 헤더: "최근 가입 회원"
- 테이블: 이름 | 이메일 | 가입일 | 연락처
- 각 행 클릭 시 회원 상세 페이지로 이동
