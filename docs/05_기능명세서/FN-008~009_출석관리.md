# FN-008~009: 출석관리

## FN-008: 출석 기록 조회

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-008 |
| **기능명** | 출석 기록 조회 |
| **분류** | 출석관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-020 |
| **관련 API ID** | API-013 |

### 기능 설명
회원의 출석 현황을 조회합니다. 일별/주별/월별 뷰로 유연하게 조회할 수 있으며, 출석 유형(일반/PT/GX)과 타지점 출석을 필터링할 수 있습니다.

### 상세 요구사항
1. 날짜 필터: 일/주/월 뷰 전환
2. 출석 유형 필터: 일반/PT/그룹운동(GX) 선택 (다중 가능)
3. 타지점 출석 표시 옵션 (ON/OFF)
4. 키오스크/앱 체크인 구분 표시
5. 캘린더 히트맵 (일별 색상으로 출석 여부 표시)
6. 출석 통계: 월별 출석 횟수, 평균 출석 간격
7. 엑셀 다운로드

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 회원ID | UUID | 필수 | URL 파라미터에서 추출 |
| 날짜범위 | String | 선택 | "day" \| "week" \| "month" \| "custom" |
| 시작일 | Date | 조건부 | custom 선택 시 필수 |
| 종료일 | Date | 조건부 | custom 선택 시 필수 |
| 출석유형 | String[] | 선택 | "일반" \| "PT" \| "GX" (다중 선택) |
| 타지점포함 | Boolean | 선택 | 기본값: false |
| 정렬 | String | 선택 | "date DESC" (기본값) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 출석 목록 | Array | [{date, check_in_time, check_out_time, type, device, branch}, ...] |
| 월별 통계 | Object | {total_count, avg_interval_days, attendance_rate} |
| 히트맵 데이터 | Array | [{date, attended: true/false}, ...] |

### 비즈니스 규칙
1. branch_id 기준 필터링 (자동)
2. 타지점 출석 포함 옵션: ON 시 다른 지점의 출석도 함께 표시
3. 출석 유형별 색상 구분:
   - 일반: 파랑
   - PT: 초록
   - GX: 보라
4. 키오스크/앱 구분: 아이콘으로 표시
5. 출석률 계산: (실제 출석 횟수 / 기간 중 영업일 수) * 100

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 출석 데이터 없음 | "출석 기록이 없습니다" 빈 상태 UI |
| 권한 없음 | "조회 권한이 없습니다" |
| 데이터 로드 실패 | "데이터를 불러올 수 없습니다. 새로고침을 시도해주세요" |

### Web 구현 사항
- **캘린더**: react-calendar 또는 FullCalendar (히트맵 모드)
- **테이블**: TanStack Table
- **필터**: 다중 선택 Checkbox
- **통계**: 카드 컴포넌트
- **차트**: Recharts (선택사항, 월별 추이)
- **데이터 페칭**: React Query

## FN-009: 일정 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-009 |
| **기능명** | 일정 관리 (PT/GX 캘린더) |
| **분류** | 출석관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-021 |
| **관련 API ID** | API-014 (미정) |

### 기능 설명
PT(개인운동) 및 GX(그룹운동) 일정을 캘린더 형식으로 관리합니다. 드래그앤드롭으로 일정을 이동할 수 있으며, 트레이너별 색상 구분과 충돌 감지 기능을 제공합니다.

### 상세 요구사항
1. 월/주/일 캘린더 뷰 전환
2. 드래그앤드롭 일정 이동 (같은 날짜 내)
3. 트레이너별 색상 구분
4. 일정 충돌 감지 (같은 시간 같은 트레이너 또는 같은 룸)
5. 룸 선택 (GX: 필수, PT: 선택)
6. 일정 CRUD (생성/조회/수정/삭제)
7. 예약된 회원 목록 표시

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 일정제목 | String | 필수 | 1~100자 |
| 유형 | String | 필수 | "PT" \| "GX" |
| 시작시간 | Time | 필수 | HH:MM 형식 |
| 종료시간 | Time | 필수 | 시작시간보다 나중 |
| 날짜 | Date | 필수 | YYYY-MM-DD |
| 트레이너ID | UUID | 필수 | PT/GX 담당 트레이너 |
| 회원ID[] | UUID[] | 조건부 | PT는 1명, GX는 다중 |
| 룸ID | UUID | 조건부 | GX는 필수, PT는 선택 |
| 최대인원 | Number | 조건부 | GX는 필수 (1~30) |
| 메모 | String | 선택 | 300자 이내 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 일정 목록 | Array | [{id, title, date, start_time, end_time, trainer, members, room}, ...] |
| 충돌 알림 | Object | {message, conflicting_schedule} |

### 비즈니스 규칙
1. **중복 불가**:
   - 같은 시간에 같은 트레이너는 중복 불가
   - 같은 시간에 같은 룸은 중복 불가

2. **일정 수정 제약**:
   - 과거 일정 수정 불가 (읽기 전용)
   - 현재 진행 중인 일정 수정 불가 (완료 후)

3. **캔슬 규칙**:
   - 일정 24시간 전: 자유 캔슬 가능
   - 24시간 이내: "정말 캔슬하시겠습니까?" 확인 필요
   - 이미 시작된 일정: 캔슬 불가

4. **트레이너 배정**:
   - PT: 1명의 트레이너 지정 (다른 시간에 중복 가능)
   - GX: 1명 이상의 트레이너 지정 (공동 진행 가능)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 일정 충돌 감지 | "경고: 이 시간에 이미 일정이 있습니다. {{시간}} {{트레이너명}} ({{룸명}}) - 계속하시겠습니까?" |
| 과거 일정 수정 | 수정 버튼 비활성화, "과거 일정은 수정할 수 없습니다" |
| 필수 필드 미입력 | 필드별 에러 메시지 |
| 잘못된 시간 범위 | "종료 시간이 시작 시간보다 나중이어야 합니다" |
| 일정 조회 실패 | "일정을 불러올 수 없습니다. 새로고침을 시도해주세요" |
| 권한 없음 | "이 일정을 수정할 권한이 없습니다" |

### Web 구현 사항
- **캘린더**: FullCalendar React (@fullcalendar/react)
- **드래그앤드롭**: @fullcalendar/interaction (기본 포함)
- **충돌 감지**: 캘린더 API + 서버 검증
- **폼**: React Hook Form + Zod
- **컬러**: 트레이너별 고정 색상 (팔레트)
- **모달**: 일정 생성/수정/상세 조회
- **기간 선택**: DatePicker 컴포넌트

### 트레이너별 색상 팔레트
```
트레이너1: #3B82F6 (파랑)
트레이너2: #10B981 (초록)
트레이너3: #F59E0B (주황)
트레이너4: #EF4444 (빨강)
트레이너5: #8B5CF6 (보라)
... (추가 트레이너는 순환)
```

## 출석관리 공통 사항

### 권한 검증
| 기능 | 센터장 | PT담당자 | GX담당자 | 조회전용 |
|------|--------|---------|---------|---------|
| FN-008 출석 조회 | 전체 | 자신의 PT | 자신의 GX | 전체 |
| FN-009 일정 관리 | CRUD | 자신 관련만 | 자신 관련만 | 읽기만 |

### 데이터 격리
- 모든 출석/일정 데이터는 branch_id 기준
- 타지점 데이터 조회는 별도 권한 필요

### 공통 기술 스택
- **캘린더**: FullCalendar React
- **테이블**: TanStack Table
- **폼**: React Hook Form + Zod
- **데이터 페칭**: React Query
- **날짜**: date-fns, dayjs
- **UI**: 커스텀 컴포넌트
