# FN-019~022: 직원관리 및 급여

## FN-019: 직원 목록 조회

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-019 |
| **기능명** | 직원 목록 조회 |
| **분류** | 직원관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-060 |
| **관련 API ID** | API-022 |

### 기능 설명
직원 목록을 검색, 필터링하여 조회합니다. 역할별/상태별 필터를 제공합니다.

### 상세 요구사항
1. 이름/연락처 검색
2. 역할 필터 (센터장/트레이너/스태프/조회전용)
3. 상태 필터 (활성/휴직/퇴직)
4. 테이블 정렬
5. 직원 클릭 → 상세/수정 페이지로 이동

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 검색어 | String | 선택 | 이름/연락처 |
| 역할 | String[] | 선택 | "센터장" \| "트레이너" \| "스태프" \| "조회전용" |
| 상태 | String[] | 선택 | "활성" \| "휴직" \| "퇴직" |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 직원 목록 | Array | [{id, name, phone, role, status, hire_date}, ...] |

### 비즈니스 규칙
1. branch_id 기준
2. 센터장 이상만 직원 관리 페이지 접근 가능
3. 퇴직 직원도 이력 목적으로 조회 가능하나 기본 미표시 (필터로 선택 가능)

### Web 구현 사항
- **테이블**: TanStack Table
- **필터**: MultiSelect (역할, 상태)
- **검색**: Debounce 300ms

## FN-020: 직원 등록/수정

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-020 |
| **기능명** | 직원 등록/수정 |
| **분류** | 직원관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-060 |
| **관련 API ID** | API-023 |

### 기능 설명
신규 직원을 등록하거나 기존 직원 정보를 수정합니다. 역할에 따른 기본 권한이 자동 배정되며, 관리자 계정이 자동 생성됩니다.

### 상세 요구사항
1. 이름/역할/연락처/입사일 필수 입력
2. 이메일 (관리자 계정용) 필수
3. 역할별 기본 권한 자동 배정
4. 수정 시 권한 변경 가능
5. 권한 미리보기 (수정 폼에서 확인)
6. 직원 수정 폼에서 권한도 함께 관리 (조회만)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 이름 | String | 필수 | 1~50자 |
| 역할 | String | 필수 | "센터장" \| "트레이너" \| "스태프" |
| 연락처 | String | 필수 | 전화번호 (010-0000-0000) |
| 입사일 | Date | 필수 | YYYY-MM-DD |
| 이메일 | String | 필수 | 이메일 형식, 중복 불가 |
| 상태 | String | 선택 | "활성" (기본값) |
| 비고 | String | 선택 | 최대 200자 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 등록된 직원 정보 | Object | 직원 정보 + 자동 생성된 관리자 계정 정보 |
| 기본 권한 | Array | 역할에 따른 기본 권한 목록 |

### 비즈니스 규칙
1. 역할별 기본 권한 자동 배정
   - **센터장**: 모든 메뉴/기능 권한
   - **트레이너**: 회원/출석/일정 관련만
   - **스태프**: 회원/매출/상품 관련만
2. 관리자 계정 자동 생성 (임시 비밀번호 발급)
3. 이메일 중복 불가 (전사 관리자 중복 체크)
4. 입사일은 과거 또는 오늘 (미래 입사일 불가)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 중복 이메일 | "이미 사용 중인 이메일입니다. 다른 이메일을 입력해주세요" |
| 필수 미입력 | 필드별 에러 메시지 |
| 연락처 형식 오류 | "올바른 전화번호 형식을 입력해주세요" |
| 이메일 형식 오류 | "올바른 이메일 형식을 입력해주세요" |
| 미래 입사일 | "입사일은 오늘 이전 날짜여야 합니다" |

### Web 구현 사항
- **Form**: React Hook Form + Zod
- **권한 미리보기**: 별도 섹션 (읽기 전용)
- **역할 선택**: Select (역할 변경 시 기본 권한 자동 갱신)
- **입사일**: DatePicker (과거 날짜만 가능)

## FN-021: 급여 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-021 |
| **기능명** | 급여 관리 |
| **분류** | 직원관리 |
| **플랫폼** | Web |
| **우선순위** | High |
| **상태** | 확정 |
| **화면 ID** | SCR-062 |
| **관련 API ID** | API-024 (미정) |

### 기능 설명
월별 급여를 관리합니다. 기본급, 인센티브, 공제를 입력하고, 실지급액을 자동 계산합니다.

### 상세 요구사항
1. 월별 급여 테이블 (직원별)
2. 기본급/인센티브/공제 항목별 입력
3. 실지급액 자동 계산 (기본급 + 인센티브 - 공제)
4. 지급 상태 관리 (미지급/지급대기/지급완료)
5. 지급 완료 시 수정 불가
6. 합계/평균 요약
7. 월 선택 필터

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 연월 | String | 필수 | "YYYY-MM" 형식 |
| 직원ID | UUID | 필수 | 테이블에서 선택 |
| 기본급 | Number | 필수 | 0 이상 |
| 인센티브 | Number | 선택 | 0 이상 (기본값: 0) |
| 공제 | Number | 선택 | 0 이상 (기본값: 0) |
| 지급상태 | String | 선택 | "미지급" \| "지급대기" \| "지급완료" |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 급여 목록 | Array | [{id, name, basic, incentive, deduction, net_pay, status}, ...] |
| 총합 | Object | {total_basic, total_incentive, total_deduction, total_net} |
| 평균 | Object | {avg_basic, avg_net} |

### 비즈니스 규칙
1. 실지급액 = 기본급 + 인센티브 - 공제
2. 지급 완료 상태 후 수정 불가 (읽기 전용)
3. 음수값 입력 불가 (기본급/공제 모두)
4. 월별로 신규 행 자동 생성 (직원별)
5. 합계/평균은 자동 계산

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 지급 완료 후 수정 | "지급 완료된 급여는 수정할 수 없습니다" (수정 버튼 비활성화) |
| 음수 입력 | "0 이상의 숫자를 입력해주세요" |
| 실지급액 음수 | "경고: 실지급액이 음수입니다. 공제 금액을 확인해주세요" |
| 월 선택 후 데이터 없음 | "{{연월}}의 급여 데이터가 없습니다" |

### Web 구현 사항
- **테이블**: TanStack Table (월별 급여 데이터)
- **월 선택**: DatePicker (년월만)
- **입력 필드**: Number Input (기본급, 인센티브, 공제)
- **자동 계산**: useMemo (실지급액)
- **Footer**: 합계/평균 행 (자동 계산)
- **상태 표시**: Badge (미지급/대기/완료)

## FN-022: 급여명세서 조회

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-022 |
| **기능명** | 급여명세서 조회 |
| **분류** | 직원관리 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-062 |
| **관련 API ID** | API-024 |

### 기능 설명
직원별 급여명세서를 상세 조회하고 PDF로 다운로드합니다. 본인 명세서만 조회 가능하나, 센터장 이상은 전체 조회 가능합니다.

### 상세 요구사항
1. 직원/월 선택
2. 명세서 상세 표시 (회사명, 직원명, 기간, 항목별 금액)
3. PDF 다운로드
4. 본인 명세서만 조회 (권한 제한)
5. 센터장 이상은 모든 직원 명세서 조회

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 직원ID | UUID | 필수 | 본인 또는 권한 범위 내 직원 |
| 연월 | String | 필수 | "YYYY-MM" 형식 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 명세서 | Object | {회사명, 센터명, 직원명, 급여기간, 항목별 금액, 발급일, 서명 등} |

### 비즈니스 규칙
1. 본인 명세서만 조회 가능
   - 본인 확인: JWT의 user_id와 요청 직원ID 비교
   - 센터장 이상: 모든 직원 조회 가능
2. 명세서는 지급 완료 상태에서만 조회 가능

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 명세서 없음 | "{{연월}} 급여명세서가 없습니다" |
| 권한 없음 | "다른 직원의 명세서를 조회할 수 없습니다" |
| 지급 미완료 | "지급 완료 후 명세서를 조회할 수 있습니다" |
| PDF 생성 실패 | "PDF 생성에 실패했습니다. 다시 시도해주세요" |

### Web 구현 사항
- **명세서 템플릿**: HTML 컴포넌트 (인쇄 레이아웃)
- **PDF 변환**: html2pdf 또는 pdfkit
- **직원 선택**: Select (권한에 따라 필터링)
- **월 선택**: DatePicker (년월만)
- **미리보기**: 모달 또는 별도 페이지
- **다운로드 버튼**: "PDF 다운로드" 버튼
