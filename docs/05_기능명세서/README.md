# 스포짐 CRM 기능명세서

## 개요

스포짐 CRM(고객관계관리) 시스템은 피트니스 센터의 회원, 매출, 직원, 시설을 통합 관리하는 웹 기반 플랫폼입니다.

- **총 기능 수**: 33개
- **주요 분류**: 인증, 대시보드, 회원관리, 출석관리, 매출관리, 상품관리, 시설관리, 직원관리, 메시지/마케팅, 계약/구독, 설정
- **플랫폼**: Web (모바일 앱 추후 확장)
- **개발 상태**: 확정

## 주요 기술 스택

### Frontend
- **Framework**: React
- **상태관리**: React Query, Zustand
- **Form 관리**: React Hook Form + Zod validation
- **테이블**: TanStack Table
- **차트**: Recharts
- **캘린더**: FullCalendar
- **UI 컴포넌트**: 커스텀 컴포넌트 (Pencil.dev 디자인시스템 기반)

### Backend Integration
- **인증**: JWT 토큰 기반
- **API 통신**: RESTful API
- **데이터 포맷**: JSON
- **결제**: 제3자 결제 게이트웨이 연동

### 구현 기술
- **파일 업로드**: 이미지 (JPG/PNG), 엑셀 (XLSX)
- **보고서**: PDF 생성 (html2pdf)
- **통신**: SMS/LMS/카카오톡 메시지 API
- **IoT**: RFID 리더, 출입문, 체성분 기기 연동

## 문서 구조

각 기능 문서는 다음 항목을 포함합니다:

1. **기능 기본 정보**: ID, 이름, 분류, 플랫폼, 우선순위, 상태
2. **기능 설명**: 주요 목적 및 사용자 시나리오
3. **상세 요구사항**: 단계별 요구사항 목록
4. **입력 항목 테이블**: 항목명, 타입, 필수여부, 설명
5. **출력 항목 테이블**: 반환되는 데이터 항목
6. **비즈니스 규칙**: 세부 규칙 및 제약사항
7. **예외 처리**: 오류 상황별 대응 방법
8. **관련 API**: 호출되는 API ID 및 엔드포인트
9. **Web 구현 사항**: 사용 기술 및 라이브러리

## 용어 정의

| 용어 | 정의 |
|------|------|
| **지점** | 피트니스 센터 또는 지사 (멀티테넌트 구분 기준) |
| **회원** | 센터 이용 고객 |
| **이용권** | 회원이 구매한 이용 기간/횟수 (월권, PT, GX 등) |
| **담당자** | 회원을 담당하는 FC(Front Counselor) |
| **매출** | 상품/이용권 판매로 인한 수익 |
| **RFID** | Radio Frequency Identification (출입통제용 카드) |
| **GX** | 그룹운동 (Group Exercise) |
| **PT** | 개인운동 (Personal Training) |
| **마일리지** | 포인트 적립 (결제 시 자동 적립, 다음 결제 사용) |
| **쿠폰** | 할인권 (금액/할인율, 만료일/사용 한도 설정) |
