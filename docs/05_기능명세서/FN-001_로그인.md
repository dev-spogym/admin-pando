## 기능 설명

사용자가 이메일과 비밀번호를 입력하여 시스템에 로그인하고, 지점을 선택한 후 JWT 토큰을 발급받는 기능입니다. 보안을 위해 연속 실패 시 계정을 일시적으로 잠금하고, 비밀번호 만료를 추적합니다.

### 주요 시나리오
- 신규 접속자: 이메일 입력 → 비밀번호 입력 → 지점 선택 → 로그인
- 로그인 실패 시: 실패 횟수 증가 → 5회 이상 실패 시 30분 계정 잠금
- 비밀번호 만료: 로그인 후 비밀번호 변경 안내 화면 노출

---

## 상세 요구사항

1. 이메일/비밀번호 입력 폼 제공
2. 지점 선택 필수 (로그인 후 또는 로그인 전)
3. JWT 토큰 발급 및 localStorage 저장
4. 로그인 실패 횟수 서버에서 추적
5. 5회 연속 실패 시 30분 계정 잠금
6. 비밀번호 90일 경과 시 변경 안내 메시지 표시
7. 동시 세션 1개 제한 (재로그인 시 기존 토큰 무효화)
8. "로그인 유지" 옵션 (선택사항)
9. 비밀번호 찾기 링크 제공
10. 에러 메시지 명확한 구분 (계정 오류 vs 비밀번호 오류 vs 계정 잠금)

## 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 이메일 | String | 필수 | 사용자 이메일 주소 |
| 비밀번호 | String | 필수 | 사용자 비밀번호 (마스킹 표시) |
| 지점 ID | UUID | 필수 | 선택한 지점의 고유 ID |
| 로그인 유지 | Boolean | 선택 | 자동 로그인 설정 여부 (기본값: false) |

## 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| JWT 토큰 | String | Bearer token (localStorage 저장) |
| 사용자 정보 | Object | 사용자명, 이메일, 프로필 이미지 |
| 권한 목록 | Array | 사용자의 메뉴/기능 접근 권한 |
| 지점 정보 | Object | 선택된 지점의 이름, 주소, 연락처 |
| 비밀번호 만료일 | Date | 비밀번호 변경 필요 여부 판단용 |

## 비즈니스 규칙

1. **계정 잠금**: 5회 연속 실패 시 30분 자동 잠금
   - 잠금 해제 조건: 30분 경과 또는 관리자 수동 해제
   - 잠금 중 로그인 시도: "계정이 잠금되었습니다. 30분 후 재시도하세요" 메시지

2. **비밀번호 유효성**:
   - 최소 8자 이상
   - 영문 대문자, 소문자, 숫자, 특수문자 포함
   - 90일마다 변경 권고

3. **동시 세션 제한**:
   - 동일 계정으로 새로운 로그인 시 기존 토큰 무효화
   - 기존 세션 사용자에게 "다른 곳에서 로그인되어 로그아웃됩니다" 알림

4. **지점 선택 의무**:
   - 지점 선택 없이는 로그인 완료 불가
   - 사용자가 속한 지점만 선택 가능 (관리자 제외)

5. **토큰 유효성**:
   - 토큰 유효기간: 24시간
   - 리프레시 토큰 지원 (선택사항)
   - 로그아웃 시 토큰 서버에서 블랙리스트 처리

6. **보안 로깅**:
   - 모든 로그인 시도 기록 (성공/실패)
   - 실패 시 IP 주소 및 실패 사유 기록

## 예외 처리

| 상황 | 오류 코드 | 대응 방법 |
|------|----------|---------|
| 이메일 미입력 | VALIDATION_ERROR | "이메일을 입력해주세요" 메시지, 필드 강조 |
| 비밀번호 미입력 | VALIDATION_ERROR | "비밀번호를 입력해주세요" 메시지, 필드 강조 |
| 이메일 형식 오류 | VALIDATION_ERROR | "올바른 이메일 형식을 입력해주세요" |
| 존재하지 않는 계정 | AUTH_FAILED | "이메일 또는 비밀번호가 일치하지 않습니다" (구체적으로 표시 금지) |
| 비밀번호 불일치 | AUTH_FAILED | "이메일 또는 비밀번호가 일치하지 않습니다" |
| 계정 잠금 | ACCOUNT_LOCKED | "계정이 잠금되었습니다. 30분 후 재시도하세요" + 해제 안내 |
| 탈퇴 계정 | ACCOUNT_DELETED | "탈퇴한 계정입니다. 고객지원팀에 문의해주세요" |
| 지점 미선택 | VALIDATION_ERROR | "지점을 선택해주세요" |
| 서버 오류 | SERVER_ERROR | "일시적 오류가 발생했습니다. 잠시 후 다시 시도해주세요" + 재시도 버튼 |
| 네트워크 오류 | NETWORK_ERROR | "인터넷 연결을 확인해주세요" + 재시도 버튼 |

## Web 구현 사항

### 사용 라이브러리
- **Form 관리**: React Hook Form + Zod validation
- **상태 관리**: Zustand (인증 상태)
- **HTTP 클라이언트**: Axios
- **UI 컴포넌트**: 커스텀 Input, Button, Card

### 구현 세부사항

#### 1. 폼 검증 (Zod)
```
이메일: 유효한 이메일 형식
비밀번호: 최소 8자, 필수
지점 ID: UUID 형식, 필수
```

#### 2. 상태 관리 (Zustand)
```
- authStore: { token, user, branch, isLoading, error }
- 로그인 성공 시 token, user, branch 저장
- localStorage에 token 영속화
```

#### 3. 토큰 저장 위치
```
localStorage 키: 'auth_token'
형식: 'Bearer {jwt_token}'
```

#### 4. 보안 고려사항
- 비밀번호 필드에서 password 입력 타입 사용
- HTTPS 전송만 허용 (프로덕션)
- 토큰을 httpOnly 쿠키에 저장하는 옵션도 지원

#### 5. UX 개선사항
- 로딩 중 로그인 버튼 비활성화
- 실패 시 자동 포커스를 이메일 필드로 이동
- "로그인 유지" 옵션은 보안 주의 문구 표시
- 비밀번호 찾기는 별도 페이지로 분리

#### 6. 로직 흐름
```
1. 폼 제출 → Zod 검증
2. 검증 실패 → 필드 에러 표시
3. 검증 성공 → API-001 호출
4. API 응답 성공 → 토큰/사용자 정보 저장 → 지점 선택 화면 또는 대시보드로 이동
5. API 응답 실패 → 에러 메시지 표시 + 실패 횟수 추적
6. 5회 연속 실패 → 계정 잠금 안내 표시
```

## 화면 요소

### 로그인 폼
- 이메일 입력 필드 (라벨: "이메일", placeholder: "your@email.com")
- 비밀번호 입력 필드 (라벨: "비밀번호", type="password", 비밀번호 표시/숨김 토글)
- 지점 선택 드롭다운 또는 라디오 버튼
- "로그인 유지" 체크박스 (선택사항)
- "로그인" 버튼 (primary, 로딩 상태 표시)
- "비밀번호 찾기" 링크 (secondary)

### 피드백 영역
- 오류 메시지 (red, 아이콘 포함)
- 성공 메시지 (green, 아이콘 포함)
- 로딩 스피너 (로그인 버튼 내)

### 보안 안내
- "로그인 유지 사용 시 공공 컴퓨터에서 사용하지 마세요" 경고 문구
- 비밀번호 입력 시 비밀번호 강도 표시 (선택사항)
