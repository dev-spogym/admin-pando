## 3. 연계 원칙

- 모듈 7은 `회원 전용 단일 앱`이 아니라 `회원 중심 통합 모바일 앱`으로 정의한다.
- 회원은 앱의 기본 사용자이며, `trainer`, `golf_trainer`, `fc`, `staff`는 같은 앱 셸을 공유하는 역할 확장이다.
- 회원앱은 독립 데이터 원장을 새로 만들지 않는다. CRM 각 도메인의 데이터를 모바일에서 조회/실행하는 채널이다.
- 현재 단계의 가입 방식은 `CRM 회원 선등록 후 앱 연동`이 기본이다.
- 공개형 온라인 가입은 즉시 앱 계정 생성으로 연결하지 않고, `리드/회원 전환 후 앱 연동` 경로로 수렴한다.
- 리워드 개념은 상위 비즈니스 용어로 유지하되, 현재 구현/기획 기준의 실체는 `마일리지 + 쿠폰`이다.
- OT는 비즈니스 분류상 `상담/오리엔테이션`에 속하지만, 회원앱에서는 예약 가능한 시간 슬롯으로 노출될 수 있다.

## 4. 안맞는 부분과 연결 기준

| 항목 | 기존 프로젝트 정의 | 현재 회원앱 정의 | 연결 기준 | 현재 처리 |
|------|--------------------|------------------|-----------|-----------|
| 모듈 7 제품 포지션 | 모든 회원이 쓰는 회원용 앱 | 역할 기반 통합 앱 | `회원 중심 + 현장 역할 확장`으로 재정의 | 즉시 적용 |
| 가입 방식 | 온라인 회원가입 가능 | CRM 등록 회원만 앱 연동 | 현재는 `센터/CRM 선등록`, 공개 가입은 후속 리드 퍼널 | 현재 기준 확정 |
| 역할 범위 | 회원 중심 표현만 존재 | member / trainer / golf_trainer / fc / staff | 기본 사용자는 회원, 나머지는 동일 앱의 role branch | 즉시 적용 |
| OT 표현 | 상담/캘린더 분류 | 회원 예약 타입 OT1/OT2 노출 | CRM 저장은 상담/일정 분류, 앱 노출은 예약 슬롯 분기 | 즉시 적용 |
| 골프 예약 | 골프 타석 관리가 시설 도메인에 존재 | 회원앱에서 골프 예약/타석 선택 사용 | 시설 도메인은 자원 마스터, 회원앱은 예약 UI | 즉시 적용 |
| 대기열 | 수업관리/시설관리 중심 | 회원앱에서 대기 등록/자동 확정 확인 | 대기열 원장은 수업관리, 회원앱은 대기 상태 노출/취소 | 즉시 적용 |
| 리워드 | 포인트/리워드/게이미피케이션 | 마일리지, 쿠폰 중심 | 현재는 `마일리지+쿠폰`, 포인트/배지/챌린지는 후속 | 단계 분리 |
| E-chart / 체형 리포트 | 운동 기록, E-chart 강조 | 체성분 차트 + FMS/체형 리포트 | 현재는 체성분/FMS까지 포함, 운동 기록은 역할별 확장 | 단계 분리 |
| 커뮤니티 | 챌린지, 랭킹, 그룹 활동 포함 | 현재 범위에서 제외 | 모듈 7 로드맵 유지, MVP/1차 계약 범위에서는 제외 | 후속 |
| 적용 상태 | 모듈 7 미구현 | 회원앱 문서 상세화 완료 | 상태는 `기획 상세화 완료 (개발 전)`로 표기 | 즉시 적용 |

## 5. 도메인 소유 기준

| 영역 | 원천 도메인 | 대표 엔티티/경로 | 회원앱 역할 |
|------|-------------|------------------|-------------|
| 회원 마스터 | 모듈 2 회원관리 | `members`, `memberships`, `consultations` | 본인 정보/담당 회원 정보 노출 |
| 예약/대기/노쇼 | 모듈 4 수업관리 | `lesson_schedules`, `lesson_bookings`, `waitlist` | 예약 실행, 대기 상태 조회, 후기 진입 |
| 골프 자원 | 시설관리 + 모듈 4 | `golf-bays`, booth resource | 타석 선택 UI, 예약 가능 자원 조회 |
| 결제/이용권/구독 | 모듈 5 결제·정산 + 상품관리 | `sales`, `payments`, `products`, `subscription` | 결제, 이용권 조회, 자동결제 관리 |
| 쿠폰/마일리지/알림 | 모듈 3 마케팅 | `coupon`, `mileage`, `message`, `push` | 혜택 조회, 캠페인 수신, 재등록 유도 |
| 체성분/FMS/운동 기록 | 회원관리 + 수업관리 | `body_composition`, trainer records | 회원 조회, 트레이너 기록, 성과 추적 |
| KPI/성과 | 모듈 6 분석 | KPI 집계 테이블/리포트 | 역할별 요약 카드 노출 |

| 역할 운영 | 회원/트레이너/골프강사/FC/스태프 | 센터장/매니저 뷰 추가 검토 가능 |

## 7. 계약/개발 기준 사용법

- 계약 기준 화면 수와 기능 범위는 `APP-IA.md`, 회원앱 `화면설계서`, 회원앱 `기능명세서`를 사용한다.
- 루트 문서의 모듈 표는 사업 설명용 상위 요약으로 본다.
- 상위 요약과 회원앱 세부 설계가 다를 때는 본 문서의 연결 기준을 따라 해석한다.
- 후속 기능은 `현재 범위 포함`으로 오해되지 않도록 반드시 `후속/로드맵`으로 표기한다.
