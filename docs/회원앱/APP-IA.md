## 핵심 컨셉

**역할 기반 통합 앱 (Role-based Navigation)**

- 한 사람 = 하나의 역할 (role)
- 로그인 후 서버에서 role을 확인하여 네비게이션 분기
- 기본 역할: 일반 회원 (Member)

**지원 역할 (Role)**

| Role | 설명 |
|------|------|
| member | 일반 회원 |
| trainer | 트레이너 (PT/그룹) |
| golf_trainer | 골프강사 (트레이너 확장) |
| fc | FC (Fitness Consultant, 영업 상담) |
| staff | 스태프 (프론트데스크) |

## 5. 역할 전환 로직

### 로그인 흐름 및 role 분기

```
앱 실행
  └─> 스플래시 / 온보딩 (MA-004)
        └─> 토큰 유효성 확인
              ├─> [토큰 없음 / 만료] → 로그인 (MA-001)
              │     └─> 인증 성공
              │           └─> GET /auth/me → { role, centerId, ... }
              └─> [토큰 유효] → GET /auth/me → { role, centerId, ... }

role 확인 후 RootNavigator 분기:
  ├─> role === 'member'        → MemberNavigator   (MA-1XX 탭)
  ├─> role === 'trainer'       → TrainerNavigator  (MA-2XX 탭)
  ├─> role === 'golf_trainer'  → GolfNavigator     (MA-2XX + MA-3XX 탭)
  ├─> role === 'fc'            → FCNavigator       (MA-4XX 탭)
  └─> role === 'staff'         → StaffNavigator    (MA-5XX 탭)
```

### RootNavigator 구조 (React Navigation 기준)

```typescript
// RootNavigator.tsx
const role = useAuthStore((s) => s.role);

return (
  <NavigationContainer>
    {!isAuthenticated ? (
      <AuthStack />          // MA-001, MA-002, MA-003, MA-004
    ) : role === 'member' ? (
      <MemberTabNavigator />
    ) : role === 'trainer' ? (
      <TrainerTabNavigator />
    ) : role === 'golf_trainer' ? (
      <GolfTabNavigator />
    ) : role === 'fc' ? (
      <FCTabNavigator />
    ) : role === 'staff' ? (
      <StaffTabNavigator />
    ) : (
      <ErrorScreen message="알 수 없는 역할입니다." />
    )}
  </NavigationContainer>
);
```

### 주요 규칙
- role은 로그인 시 서버 응답에서 단 한 번 결정되며, 앱 내에서 임의 변경 불가
- 역할 변경이 필요한 경우 로그아웃 후 재로그인
- 골프강사는 트레이너의 모든 화면을 상속하며, 수업 탭에만 전용 화면 추가
- 스태프의 회원 관련 화면은 모두 읽기전용(read-only)으로, 수정 API 호출 차단
