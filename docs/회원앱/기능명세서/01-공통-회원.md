# FitGenie 회원앱 기능명세서 — 공통 · 회원

> **작성일**: 2026-04-17 | **버전**: 2.0 | **상태**: 기획 중
> **범위**: 공통 기능 (MFN-001~008), 회원 기능 (MFN-101~125)
> **역할 체계**: member(회원), trainer(트레이너), golf_trainer(골프강사), fc(FC), staff(스태프)

## 비즈니스 맥락

### 공통 기능이 해결하는 문제
- **로그인/연동**: 기존에 센터 회원이 앱을 쓰려면 별도 가입이 필요했음 → CRM 데이터 매칭으로 "이름+연락처+SMS" 만으로 연동, 별도 가입 절차 없음
- **역할 분기**: 하나의 앱에서 회원/강사/FC/스태프 모두 지원 → 앱 2~3개 만들 필요 없음, 배포/유지보수 비용 절감
- **푸시 알림**: 예약 리마인더, 이용권 만료, 서명 요청 등 능동적 알림 → 전화/문자 발송 비용 절감, 회원 참여율 향상

### 회원 기능이 해결하는 문제
- **QR 체크인**: 기존 수기 출석 → 자동화, 출석 통계 실시간 집계, 부정 출석 방지
- **수업 예약**: 전화 예약 → 앱 예약, 프론트 업무 감소, 24시간 예약 가능
- **이용권 조회**: "내 이용권 며칠 남았어요?" 전화 문의 → 앱에서 즉시 확인, D-day 알림으로 재등록 유도
- **온라인 결제**: 센터 방문 없이 이용권/상품 구매, 영업시간 외 매출 발생 가능
- **체성분 조회**: 종이 기록/구두 전달 → 앱에서 변화 추이 차트, 동기부여 + 트레이너 전문성 신뢰도 향상

### 핵심 비즈니스 규칙 요약
| 규칙 | 내용 | 이유 |
|------|------|------|
| 예약 취소 마감 | 수업 시작 2시간 전 (센터별 설정) | 다른 회원의 예약 기회 보호 |
| 노쇼 3회 누적 | 예약 1주 제한 | 노쇼 남용 방지 |
| QR 중복 체크인 불가 | 당일 1회만 | 출석 통계 왜곡 방지 |
| 온라인 결제 후 이용권 즉시 생성 | 결제 완료 → 자동 | 즉시 이용 가능해야 함 |
| 환불은 CRM에서만 | 앱에서 신청 → CRM 승인 | 무분별 환불 방지 |

### MFN-001. 로그인

| 항목 | 내용 |
|------|------|
| 기능 ID | MFN-001 |
| 기능명 | 로그인 |
| 분류 | 공통 |
| 대상 역할 | all |
| 우선순위 | Critical |
| 화면 ID | MA-001 |
| CRM 연관 | `members` 테이블 (phone, password_hash, role, is_active) |

#### 기능 설명
연락처(010-xxxx-xxxx 형식)와 비밀번호를 입력하여 앱에 로그인한다. 인증 성공 시 JWT Access Token과 Refresh Token을 발급하며, role 값에 따라 적절한 홈 화면으로 분기한다.

#### 상세 요구사항
1. 연락처 입력 필드는 숫자만 허용하며 자동으로 010-xxxx-xxxx 형식으로 포맷한다.
2. 비밀번호는 마스킹 처리하며, 표시/숨기기 토글 버튼을 제공한다.
3. 로그인 성공 시 서버 응답의 `role` 필드를 기준으로 MFN-005 역할 분기를 수행한다.
4. 로그인 실패 시 연속 5회 실패하면 계정을 30분간 잠근다.
5. 자동 로그인 체크박스가 활성화된 경우 MFN-004 처리를 수행한다.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| phone | string | Y | 연락처 (010-xxxx-xxxx) |
| password | string | Y | 비밀번호 (8자 이상) |
| auto_login | boolean | N | 자동 로그인 여부 (기본값: false) |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| access_token | string | JWT Access Token (유효기간 1시간) |
| refresh_token | string | JWT Refresh Token (유효기간 30일) |
| role | string | 사용자 역할 (member/trainer/golf_trainer/fc/staff) |
| member_id | string | 회원 UUID |
| name | string | 회원 이름 |

#### 비즈니스 규칙
1. `is_active = false`인 계정은 "비활성화된 계정입니다. 센터로 문의하세요." 메시지와 함께 로그인 거부.
2. 탈퇴 처리된 계정(`deleted_at IS NOT NULL`)은 로그인 불가.
3. 비밀번호는 bcrypt 해시로 저장, 평문 전송 금지 (HTTPS 필수).
4. 연속 5회 로그인 실패 시 계정 잠금, 30분 후 자동 해제.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 연락처/비밀번호 불일치 | "연락처 또는 비밀번호가 올바르지 않습니다." 토스트 |
| 계정 잠금 상태 | "계정이 잠겼습니다. 30분 후 다시 시도해 주세요." |
| 비활성 계정 | "비활성화된 계정입니다. 센터로 문의하세요." |
| 네트워크 오류 | "네트워크 연결을 확인해 주세요." 재시도 버튼 표시 |
| 서버 오류 (5xx) | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." |

#### 앱 구현 사항
- 키보드가 올라올 때 레이아웃이 밀리지 않도록 `KeyboardAvoidingView` 적용.
- 로그인 버튼 탭 후 로딩 스피너 표시, 중복 탭 방지.
- 로그인 성공 후 애니메이션 트랜지션으로 홈 화면 이동.
- Secure Storage에 Refresh Token 저장 (iOS: Keychain, Android: EncryptedSharedPreferences).

### MFN-002. 앱 연동/가입

| 항목 | 내용 |
|------|------|
| 기능 ID | MFN-002 |
| 기능명 | 앱 연동/가입 |
| 분류 | 공통 |
| 대상 역할 | all |
| 우선순위 | Critical |
| 화면 ID | MA-002 |
| CRM 연관 | `members` 테이블 (name, phone, app_linked_at), `sms_verifications` 테이블 |

#### 기능 설명
CRM에 이미 등록된 회원이 처음 앱을 사용할 때 이름과 연락처를 입력하고 SMS 인증을 완료하여 앱 계정을 연동한다. CRM 회원 데이터와 매칭 후 초기 비밀번호를 설정한다.

#### 상세 요구사항
1. 이름과 연락처 입력 후 CRM `members` 테이블에서 일치하는 회원을 조회한다.
2. 일치하는 회원이 존재하면 해당 연락처로 6자리 SMS 인증번호를 발송한다.
3. 인증번호 유효시간은 3분이며, 재발송은 60초 쿨다운 후 가능하다.
4. SMS 인증 완료 후 초기 비밀번호 설정 화면으로 이동한다.
5. 비밀번호 설정 완료 시 `app_linked_at` 컬럼을 현재 시각으로 업데이트한다.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| name | string | Y | 회원 이름 (CRM 등록명과 일치해야 함) |
| phone | string | Y | 연락처 (010-xxxx-xxxx) |
| sms_code | string | Y | SMS 인증번호 (6자리) |
| password | string | Y | 설정할 비밀번호 (8자 이상, 영문+숫자 조합) |
| password_confirm | string | Y | 비밀번호 확인 |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| member_id | string | 연동된 회원 UUID |
| app_linked_at | string | 앱 연동 완료 시각 (ISO 8601) |

#### 비즈니스 규칙
1. CRM에 존재하지 않는 이름+연락처 조합은 연동 거부.
2. 이미 앱 연동된 계정(`app_linked_at IS NOT NULL`)은 재연동 불가, 로그인 화면으로 안내.
3. 비밀번호는 8자 이상, 영문+숫자 조합 필수.
4. SMS 인증번호는 3분 초과 시 만료, 재발송 필요.
5. 인증번호 5회 연속 오류 시 해당 세션 무효화.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| CRM 미등록 회원 | "등록된 회원 정보를 찾을 수 없습니다. 센터로 문의하세요." |
| 이미 연동된 계정 | "이미 앱이 연동된 계정입니다. 로그인 화면으로 이동합니다." |
| SMS 발송 실패 | "인증번호 발송에 실패했습니다. 잠시 후 재시도해 주세요." |
| 인증번호 만료 | "인증번호가 만료되었습니다. 재발송해 주세요." |
| 인증번호 불일치 | "인증번호가 올바르지 않습니다. (N/5)" |
| 비밀번호 불일치 | "비밀번호가 일치하지 않습니다." |

#### 앱 구현 사항
- SMS 인증번호 입력은 6자리 자동완성 OTP 컴포넌트 사용.
- 타이머 카운트다운(3:00 → 0:00) UI 표시.
- 재발송 버튼은 60초 쿨다운 동안 비활성화, 잔여 시간 표시.
- 비밀번호 강도 표시기(Weak/Medium/Strong) 실시간 피드백.

### MFN-003. 비밀번호 찾기

| 항목 | 내용 |
|------|------|
| 기능 ID | MFN-003 |
| 기능명 | 비밀번호 찾기 |
| 분류 | 공통 |
| 대상 역할 | all |
| 우선순위 | High |
| 화면 ID | MA-003 |
| CRM 연관 | `members` 테이블 (phone, password_hash), `sms_verifications` 테이블 |

#### 기능 설명
비밀번호를 분실한 사용자가 연락처 SMS 인증을 통해 본인 확인 후 새 비밀번호를 재설정한다. CRM에 등록된 연락처로만 인증이 가능하다.

#### 상세 요구사항
1. 연락처 입력 후 해당 연락처가 앱 연동된 계정인지 확인한다.
2. 확인된 경우 SMS 인증번호 6자리를 발송한다.
3. 인증 완료 후 새 비밀번호 설정 화면으로 이동한다.
4. 새 비밀번호 저장 시 기존 Access Token/Refresh Token을 모두 무효화한다.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| phone | string | Y | 연락처 (010-xxxx-xxxx) |
| sms_code | string | Y | SMS 인증번호 (6자리) |
| new_password | string | Y | 새 비밀번호 (8자 이상, 영문+숫자) |
| new_password_confirm | string | Y | 새 비밀번호 확인 |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| success | boolean | 재설정 성공 여부 |

#### 비즈니스 규칙
1. 앱 미연동 계정(`app_linked_at IS NULL`)은 비밀번호 찾기 불가, 앱 연동 먼저 안내.
2. 새 비밀번호는 최근 3개 비밀번호와 동일 불가.
3. 재설정 완료 시 모든 기기의 세션을 무효화.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 미등록/미연동 연락처 | "앱 연동 이력이 없습니다. 앱 연동 후 이용해 주세요." |
| SMS 인증 실패 | "인증번호가 올바르지 않습니다." |
| 최근 비밀번호 재사용 | "최근에 사용한 비밀번호는 사용할 수 없습니다." |

#### 앱 구현 사항
- 재설정 완료 후 로그인 화면으로 자동 이동, 성공 토스트 표시.
- 비밀번호 찾기 완료 시 다른 기기에서 자동 로그아웃 처리.

### MFN-004. 자동 로그인

| 항목 | 내용 |
|------|------|
| 기능 ID | MFN-004 |
| 기능명 | 자동 로그인 |
| 분류 | 공통 |
| 대상 역할 | all |
| 우선순위 | High |
| 화면 ID | MA-001 |
| CRM 연관 | — (토큰 기반, 서버 세션 테이블 또는 JWT Blacklist) |

#### 기능 설명
Refresh Token을 Secure Storage에 저장하여 앱 재실행 시 자동으로 Access Token을 갱신하고 로그인 상태를 유지한다. Refresh Token 유효기간은 30일이다.

#### 상세 요구사항
1. 자동 로그인 활성화 시 Refresh Token을 Secure Storage에 저장한다.
2. 앱 실행 시 Secure Storage에서 Refresh Token을 읽어 Access Token 갱신을 시도한다.
3. 갱신 성공 시 로그인 화면을 거치지 않고 바로 홈 화면으로 이동한다.
4. Refresh Token 만료 또는 무효 시 로그인 화면으로 이동한다.
5. 비밀번호 재설정, 로그아웃, 강제 로그아웃 시 Secure Storage의 토큰을 삭제한다.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| refresh_token | string | Y | Secure Storage에서 읽은 Refresh Token |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| access_token | string | 갱신된 JWT Access Token |
| refresh_token | string | 갱신된 Refresh Token (Rolling 방식) |

#### 비즈니스 규칙
1. Refresh Token은 Rolling 방식으로 갱신 시마다 새 토큰 발급.
2. 30일 비활성 시 Refresh Token 만료, 재로그인 필요.
3. 계정 잠금/비활성화 시 Refresh Token을 서버에서 즉시 무효화.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| Refresh Token 없음 | 로그인 화면으로 이동 |
| Refresh Token 만료/무효 | Secure Storage 토큰 삭제 후 로그인 화면 이동 |
| 네트워크 오류 | 스플래시에서 재시도, 3회 실패 시 로그인 화면 이동 |

#### 앱 구현 사항
- 스플래시 화면에서 자동 로그인 처리 수행 (사용자에게 노출 최소화).
- iOS: Keychain Services API, Android: EncryptedSharedPreferences 사용.
- Access Token 만료 전 자동 갱신을 위한 API Interceptor 구현 (만료 5분 전 선제 갱신).

### MFN-007. 푸시 알림 수신

| 항목 | 내용 |

#### 기능 설명
FCM(Android) / APNs(iOS)를 통해 역할별 맞춤 푸시 알림을 수신한다. 앱 설치 후 최초 실행 시 알림 권한을 요청하며, 알림 탭 시 관련 화면으로 딥링크 이동한다.

| type | string | 알림 유형 |
| deep_link | string | 이동할 화면 경로 |
| payload | object | 알림 추가 데이터 |

#### 비즈니스 규칙

회원이 카메라로 센터 입구의 QR 코드를 스캔하여 출석을 체크인한다. 체크인 성공 시 출석 기록이 생성되고 이용권 잔여 횟수가 차감된다.

#### 상세 요구사항
1. 카메라 권한 확인 후 QR 스캐너 화면을 활성화한다.
2. QR 코드 인식 시 센터 코드 및 유효성 검증.
3. 검증 성공 시 출석 기록 생성 및 이용권 차감.
4. 성공/실패 피드백을 애니메이션과 함께 표시.
5. 당일 이미 체크인한 경우 중복 체크인 방지.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| qr_code | string | Y | 스캔된 QR 코드 값 |
| member_id | string | Y | 회원 UUID |
| scanned_at | string | Y | 스캔 시각 (ISO 8601) |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| attendance_id | string | 생성된 출석 기록 UUID |
| checked_in_at | string | 체크인 시각 |
| remaining_count | number | 체크인 후 잔여 횟수 |

#### 비즈니스 규칙
1. QR 코드는 센터 단위로 발급, 유효한 센터 코드만 허용.
2. 유효한 이용권(횟수형 또는 기간형)이 없으면 체크인 거부.
3. 당일 동일 센터 중복 체크인 불가 (기간: 당일 00:00~23:59).
4. 횟수형 이용권은 체크인 시 1회 차감.
5. QR 코드 스캔 후 5초 이내 서버 응답 없으면 타임아웃 처리.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 카메라 권한 거부 | "카메라 권한이 필요합니다. 설정에서 허용해 주세요." + 설정 이동 버튼 |

#### 앱 구현 사항
- 성공 시: 초록색 체크마크 애니메이션 + "체크인 완료!" 텍스트 + 진동 피드백.

### MFN-106. 예약 취소

|------|------|

1. 내 예약 목록(MA-122)에서 취소할 예약 선택.

4. 취소 완료 시 확인 알림 발송.

| cancel_reason | string | N | 취소 사유 (선택) |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| success | boolean | 취소 성공 여부 |
| penalty_applied | boolean | 페널티 적용 여부 |
| penalty_description | string | 페널티 내용 설명 |

#### 비즈니스 규칙
1. 수업 시작 2시간 전까지는 페널티 없이 취소 가능.
2. 취소 마감 이후: 결석 횟수 1회 증가 (센터 정책에 따라 다를 수 있음).
3. 수업 시작 후에는 취소 불가 (완료 처리됨).
4. 한 달 내 3회 이상 페널티 취소 시 예약 기능 7일 제한 (센터 정책).

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 수업 시작 후 취소 시도 | "이미 시작된 수업은 취소할 수 없습니다." |

#### 앱 구현 사항
- 취소 확인 다이얼로그에 페널티 내용 명시.

1. 프로필 사진 영역 (기본 아바타 이미지).
2. 기본 정보: 이름, 연락처 (010-****-xxxx 마스킹), 성별, 생년월일, 가입일.
3. 회원등급 배지 표시 (일반/VIP/VVIP 등).
4. [프로필 수정] 버튼으로 MFN-109 이동.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| member_id | string | Y | 회원 UUID |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| name | string | 이름 |
| phone_masked | string | 마스킹된 연락처 |
| gender | string | 성별 (M/F) |
| birth_date | string | 생년월일 (YYYY-MM-DD) |
| joined_at | string | 가입일 |
| member_grade | string | 회원등급 |

#### 비즈니스 규칙
1. 연락처는 중간 4자리를 `****`로 마스킹하여 표시.
2. 생년월일은 나이(만 나이)로 변환하여 함께 표시.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 데이터 로드 실패 | "프로필 정보를 불러올 수 없습니다." 재시도 버튼 |

#### 앱 구현 사항

3. 결제 실패 시 3일 이내 재시도, 재시도 모두 실패 시 구독 일시 중단.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 구독 없음 | "활성 구독이 없습니다." |
| 이미 해지된 구독 | "이미 해지 신청된 구독입니다." |

#### 앱 구현 사항
- 해지 확인 다이얼로그에 서비스 이용 종료일 명시.
- 일시 중단 구독은 별도 섹션에 표시.

### MFN-124. 설정

| 항목 | 내용 |
|------|------|
| 기능 ID | MFN-124 |
| 기능명 | 설정 |
| 분류 | 회원 |
| 대상 역할 | member |
| 우선순위 | High |
| 화면 ID | MA-155 |
| CRM 연관 | `member_settings` (member_id, push_enabled, push_types, auto_login) |

#### 기능 설명
알림 ON/OFF, 자동 로그인, 비밀번호 변경, 로그아웃, 회원 탈퇴 등 앱 사용 설정을 관리한다.

#### 상세 요구사항
1. 알림 설정: 전체 알림, 유형별 알림(예약/출석/이벤트/공지) ON/OFF 토글.
2. 자동 로그인 ON/OFF 토글.
3. [비밀번호 변경]: 현재 비밀번호 확인 후 새 비밀번호 설정.
4. [로그아웃]: MFN-006 처리.
5. [회원 탈퇴]: 탈퇴 확인 후 계정 비활성화 처리.
6. 앱 버전 정보 표시.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| push_enabled | boolean | N | 전체 알림 ON/OFF |
| push_types | object | N | 유형별 알림 설정 |
| auto_login | boolean | N | 자동 로그인 ON/OFF |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| settings | object | 저장된 설정 값 |

#### 비즈니스 규칙
1. 전체 알림을 OFF하면 유형별 설정과 무관하게 모든 알림 미수신.
2. 회원 탈퇴 시 `is_active = false`, `deleted_at = 현재시각` 설정 (Soft Delete).
3. 탈퇴 후 30일간 재가입 불가 (동일 연락처).
4. 탈퇴 시 개인정보는 관련 법령에 따라 일정 기간 보관 후 삭제.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 현재 비밀번호 불일치 | "현재 비밀번호가 올바르지 않습니다." |
| 탈퇴 취소 | 다이얼로그 닫기, 설정 화면 유지 |

#### 앱 구현 사항
- 회원 탈퇴 확인 다이얼로그: "탈퇴 시 모든 데이터를 복구할 수 없습니다." + [취소] [탈퇴하기].
- 알림 설정 변경 시 서버에 즉시 저장.

### MFN-125. 수업 완료 서명 응답

| 항목 | 내용 |
|------|------|
| 기능 ID | MFN-125 |
| 기능명 | 수업 완료 서명 응답 |
| 분류 | 회원 |
| 대상 역할 | member |
| 우선순위 | Medium |
| 화면 ID | 푸시 알림으로 진입 |
| CRM 연관 | `class_signatures` (id, reservation_id, member_id, trainer_id, signed_at, signature_data) |

#### 기능 설명
골프 수업 완료 후 강사가 CRM에서 서명을 요청하면, 회원은 푸시 알림을 받고 앱에서 전자 서명을 완료한다. 완료된 서명은 CRM에 저장된다.

#### 상세 요구사항
1. 강사가 CRM에서 서명 요청 → 회원 앱에 푸시 알림 발송.
2. 알림 탭 시 서명 화면으로 진입 (딥링크).
3. 서명 화면: 수업 정보 (날짜, 수업명, 강사명) 확인 영역 + 서명 패드.
4. 서명 패드에서 터치로 서명 입력.
5. [완료] 버튼 탭 시 서명 이미지를 Base64로 변환하여 서버 전송.
6. 저장 완료 시 강사에게 서명 완료 알림 발송.

#### 입력 항목
| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| signature_request_id | string | Y | 서명 요청 UUID |
| member_id | string | Y | 회원 UUID |
| signature_data | string | Y | 서명 이미지 (Base64 PNG) |
| signed_at | string | Y | 서명 완료 시각 (ISO 8601) |

#### 출력 항목
| 필드명 | 타입 | 설명 |
|--------|------|------|
| signature_id | string | 저장된 서명 UUID |
| signed_at | string | 서명 완료 시각 |

#### 비즈니스 규칙
1. 서명 요청은 수업 완료 후 24시간 이내에만 유효.
2. 이미 서명 완료된 요청은 재서명 불가.
3. 서명 거부 기능은 제공하지 않음 (강사에게 직접 문의).
4. 서명 데이터는 CRM에 영구 보관.

#### 예외 처리
| 상황 | 대응 |
|------|------|
| 서명 요청 만료 (24시간 초과) | "서명 요청이 만료되었습니다. 강사에게 문의하세요." |
| 이미 서명 완료 | "이미 서명이 완료된 수업입니다." |
| 서명 데이터 없음 | "서명을 입력해 주세요." 토스트 |
| 서버 저장 실패 | "서명 저장에 실패했습니다. 다시 시도해 주세요." |

#### 앱 구현 사항
- 서명 패드: `react-native-signature-canvas` 또는 `react-native-draw` 사용.
- [다시 쓰기] 버튼으로 서명 초기화.
- 서명 완료 후 "서명이 완료되었습니다." 확인 화면 표시.
- 딥링크 URL: `fitgenie://signature/{signature_request_id}`
