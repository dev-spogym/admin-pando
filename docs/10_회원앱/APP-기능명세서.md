## 상세 기능 명세

### MFN-001: 회원 로그인

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-001 |
| **기능명** | 회원 로그인 |
| **분류** | 인증 |
| **플랫폼** | App (iOS/Android) |
| **우선순위** | Critical |
| **화면 ID** | MA-001 |
| **CRM 연관** | FN-001 |

#### 기능 설명
회원이 연락처(전화번호)와 비밀번호로 로그인합니다. Supabase Auth를 사용하며, 자동 로그인 옵션을 제공합니다.

#### 상세 요구사항
1. 연락처(010-xxxx-xxxx) + 비밀번호 입력
2. 자동 로그인 토글 (토큰 유지 30일)
3. 로그인 성공 시 홈(MA-010)으로 이동
4. 5회 연속 실패 시 계정 잠금 안내
5. 비밀번호 찾기 링크
6. 앱 연동(회원가입) 링크

#### 입력 항목

| 항목명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 연락처 | String | 필수 | 010-xxxx-xxxx 형식 |
| 비밀번호 | String | 필수 | 8자 이상 |
| 자동로그인 | Boolean | 선택 | 기본값: false |

#### 출력 항목

| 항목명 | 타입 | 설명 |
|---|---|---|
| 액세스 토큰 | String | Supabase JWT |
| 회원 정보 | Object | {id, name, phone, branchId} |

#### 비즈니스 규칙
1. 연락처 기반 인증 (CRM의 이메일과 달리 전화번호 사용)
2. 5회 실패 시 30분 잠금 (CRM과 동일)
3. 자동 로그인: refresh token 30일 유지
4. 비활성/탈퇴 계정 접근 불가
5. 회원 role만 허용 (관리자 role 접근 차단)

#### 예외 처리

| 상황 | 대응 |
|---|---|
| 잘못된 자격증명 | "연락처 또는 비밀번호를 확인해주세요" |
| 계정 잠금 | "5회 실패로 계정이 잠겼습니다. 30분 후 재시도하거나 비밀번호를 찾아주세요" |
| 미연동 계정 | "앱 연동이 필요합니다. 회원가입을 진행해주세요" |
| 탈퇴 계정 | "탈퇴 처리된 계정입니다" |
| 네트워크 오류 | "인터넷 연결을 확인해주세요" |

#### App 구현 사항
- Supabase Auth signInWithPassword
- SecureStorage에 토큰 저장
- 자동 로그인: 앱 시작 시 토큰 유효성 검증
- 연락처 → {phone}@member.spogym.local 형태로 Auth 이메일 매핑

### MFN-002: 앱 연동 (최초 가입)

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-002 |
| **기능명** | 앱 연동 (최초 가입) |
| **분류** | 인증 |
| **플랫폼** | App |
| **우선순위** | Critical |
| **화면 ID** | MA-002 |
| **CRM 연관** | FN-004 |

#### 기능 설명
센터에서 등록된 회원이 앱을 처음 설치 후 본인 확인(SMS 인증)을 거쳐 앱 비밀번호를 설정합니다.

#### 상세 요구사항
1. Step 1: 이름 + 등록된 연락처 입력
2. Step 2: SMS 인증번호 발송 → 입력 (3분 유효)
3. Step 3: 앱 비밀번호 설정 (8자 이상, 영문+숫자+특수문자)
4. Step 4: 연동 완료 → 홈으로 이동

#### 입력 항목

| 항목명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 이름 | String | 필수 | CRM에 등록된 이름과 매칭 |
| 연락처 | String | 필수 | CRM에 등록된 연락처 |
| 인증번호 | String | 필수 | 6자리 숫자 |
| 비밀번호 | String | 필수 | 8자 이상 |
| 비밀번호 확인 | String | 필수 | 비밀번호와 동일 |

#### 비즈니스 규칙
1. CRM에 등록된 이름+연락처 조합이 일치해야 진행
2. SMS 인증번호: 6자리, 3분 유효, 최대 5회 재발송/일
3. 이미 연동된 회원: "이미 앱에 등록된 회원입니다. 로그인해주세요"
4. CRM 미등록 회원: "등록된 회원 정보가 없습니다. 센터에 문의해주세요"
5. 연동 완료 시 Supabase Auth에 회원 계정 자동 생성

#### 예외 처리

| 상황 | 대응 |
|---|---|
| 이름/연락처 불일치 | "등록된 회원 정보와 일치하지 않습니다" |
| 인증번호 만료 | "인증번호가 만료되었습니다. 재발송해주세요" |
| 인증번호 불일치 | "인증번호가 올바르지 않습니다" |
| 이미 연동됨 | "이미 앱에 등록된 회원입니다" + 로그인 이동 |
| 미등록 회원 | "등록된 회원 정보가 없습니다. 센터에 문의해주세요" |

### MFN-004: 홈 대시보드 조회

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-004 |
| **기능명** | 홈 대시보드 조회 |
| **분류** | 홈 |
| **플랫폼** | App |
| **우선순위** | Critical |
| **화면 ID** | MA-010 |
| **CRM 연관** | FN-002, FN-006 |

#### 기능 설명
회원의 핵심 정보를 홈 화면에 요약합니다.

#### 상세 요구사항
1. 인사말 카드: "안녕하세요, {이름}님!"
2. 이용권 상태 카드: 활성 이용권 + 만료 D-day
3. 오늘 예약 카드: 예약된 PT/GX 수업 (시간, 트레이너)
4. 이번 달 출석: 출석 횟수 + 미니 캘린더
5. 공지사항 배너: 최신 1건 (슬라이드)
6. QR 체크인 플로팅 버튼 (하단 고정)

#### 출력 항목

| 항목명 | 타입 | 설명 |
|---|---|---|
| 회원 이름 | String | 인사말용 |
| 활성 이용권 | Array | [{name, end_date, days_left}, ...] |
| 오늘 예약 | Array | [{class_name, time, trainer}, ...] |
| 월간 출석 횟수 | Number | 이번 달 출석 일수 |
| 최신 공지 | Object | {title, content_preview, date} |
| 보유 쿠폰 수 | Number | 사용 가능 쿠폰 수 |
| 마일리지 잔액 | Number | 현재 보유 마일리지 |

#### 비즈니스 규칙
1. 이용권 만료 D-7 이내: 빨강 강조 + "곧 만료됩니다"
2. 이용권 없음: "이용권이 없습니다. 구매하러 가기" 버튼
3. 오늘 예약 없음: "오늘 예약이 없습니다" 회색 카드
4. 모든 데이터 본인(member_id) 것만 조회
5. Pull-to-refresh로 새로고침

### MFN-005: QR 체크인

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-005 |
| **기능명** | QR 체크인 |
| **분류** | 출석 |
| **플랫폼** | App |
| **우선순위** | Critical |
| **화면 ID** | MA-030 |
| **CRM 연관** | FN-008 |

#### 기능 설명
회원이 앱에서 QR코드를 생성하고, 센터의 키오스크/리더기에서 스캔하여 출석 체크합니다.

#### 상세 요구사항
1. 내 QR코드 생성 (member_id + timestamp + 서명)
2. QR코드 60초마다 자동 갱신 (보안)
3. 밝기 자동 최대 (QR 표시 시)
4. 체크인 완료 시 진동 + 성공 애니메이션
5. 또는: 센터 QR을 스캔하여 체크인 (역방향)

#### 입력 항목

| 항목명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 회원 ID | UUID | 자동 | 로그인 세션에서 추출 |
| 지점 ID | UUID | 자동 | 등록된 지점 |

#### 출력 항목

| 항목명 | 타입 | 설명 |
|---|---|---|
| QR코드 데이터 | String | 암호화된 회원식별 + 타임스탬프 |
| 체크인 결과 | Object | {success, check_in_time, attendance_type} |

#### 비즈니스 규칙
1. 활성 이용권 있어야 QR 생성 가능
2. QR 유효시간: 60초 (만료 후 자동 갱신)
3. 당일 중복 체크인 허용 (체크아웃 후 재입장)
4. QR 데이터: `{member_id}:{branch_id}:{timestamp}:{hmac_signature}`
5. 서버에서 서명 검증 후 출석 처리

#### 예외 처리

| 상황 | 대응 |
|---|---|
| 이용권 없음 | "활성 이용권이 없습니다. 이용권을 구매해주세요" + 구매 이동 |
| 이용권 만료 | "이용권이 만료되었습니다. 연장해주세요" |
| QR 인식 실패 | "QR 인식에 실패했습니다. 다시 시도해주세요" |
| 서버 오류 | "체크인 처리 중 오류가 발생했습니다" |

### MFN-007: 수업 목록 조회

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-007 |
| **기능명** | 수업 목록 조회 |
| **분류** | 예약 |
| **플랫폼** | App |
| **우선순위** | Critical |
| **화면 ID** | MA-040 |
| **CRM 연관** | FN-009 |

#### 기능 설명
날짜별 예약 가능한 PT/GX 수업 목록을 조회합니다.

#### 상세 요구사항
1. 상단: 주간 날짜 스크롤 (좌우 스와이프)
2. PT/GX 탭 필터
3. 트레이너별 필터 (선택)
4. 수업 카드: 시간, 수업명, 트레이너, 잔여석/정원, 예약 상태
5. 정원 초과 시 "마감" 뱃지
6. 이미 예약한 수업 "예약됨" 표시

#### 입력 항목

| 항목명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 날짜 | Date | 필수 | 기본값: 오늘 |
| 수업유형 | String | 선택 | "PT" \| "GX" \| "전체" |
| 트레이너ID | UUID | 선택 | 필터 |

#### 출력 항목

| 항목명 | 타입 | 설명 |
|---|---|---|
| 수업 목록 | Array | [{id, title, type, date, start_time, end_time, trainer, room, max_capacity, current_count, is_reserved}, ...] |

#### 비즈니스 규칙
1. 과거 날짜 수업: 목록에 표시하되 예약 불가 (회색 처리)
2. PT: 담당 트레이너 수업만 표시 (또는 전체 + 담당자 강조)
3. GX: 잔여석 = max_capacity - current_count
4. 예약 마감: 수업 시작 1시간 전까지만 예약 가능
5. 자신의 이용권 유형에 맞는 수업만 활성화

### MFN-008: 수업 예약

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-008 |
| **기능명** | 수업 예약 |
| **분류** | 예약 |
| **플랫폼** | App |
| **우선순위** | Critical |
| **화면 ID** | MA-041 |
| **CRM 연관** | FN-009 |

#### 기능 설명
회원이 PT/GX 수업을 예약합니다.

#### 상세 요구사항
1. 수업 상세 정보 표시 (시간/트레이너/룸/잔여석)
2. "예약하기" 버튼 → 확인 팝업 → 예약 처리
3. PT: 잔여 횟수 1회 차감 표시 ("PT 12회 중 9회 → 8회 남음")
4. GX: 잔여석 확인 ("잔여 5석")
5. 예약 완료 시 성공 팝업 + 캘린더 등록 제안

#### 입력 항목

| 항목명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 수업 ID | UUID | 필수 | 선택한 수업 |
| 회원 ID | UUID | 자동 | 로그인 세션 |

#### 비즈니스 규칙
1. PT 이용권 잔여 횟수 > 0 이어야 예약 가능
2. GX 잔여석 > 0 이어야 예약 가능
3. 동일 시간대 중복 예약 불가
4. 수업 시작 1시간 전까지만 예약 가능
5. 예약 완료 시 CRM 일정(FN-009)에 자동 반영

#### 예외 처리

| 상황 | 대응 |
|---|---|

### MFN-021: 온라인 결제 (이용권 구매)

| 항목 | 내용 |
|---|---|
| **기능 ID** | MFN-021 |
| **기능명** | 온라인 결제 (이용권 구매) |
| **분류** | 구매 |
| **플랫폼** | App |
| **우선순위** | High |
| **화면 ID** | MA-062 |
| **CRM 연관** | FN-027 |

#### 기능 설명
앱에서 이용권/PT/GX 상품을 온라인 결제로 구매합니다.

#### 상세 요구사항
1. 주문 요약: 상품명, 기간, 원가
2. 쿠폰 적용: 보유 쿠폰 선택 → 할인액 반영
3. 마일리지 사용: 잔액 표시 + 사용 금액 입력
4. 최종 결제금액 = 원가 - 쿠폰할인 - 마일리지
5. 결제수단: 신용카드/간편결제(카카오페이/네이버페이/토스)
6. PG 결제 모듈 호출
7. 결제 완료 → 이용권 자동 활성화 + 영수증

#### 입력 항목

| 항목명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 상품 ID | UUID | 필수 | 선택한 상품 |
| 쿠폰 ID | UUID | 선택 | 적용할 쿠폰 |
| 마일리지 사용액 | Number | 선택 | 0 이상, 잔액 이하 |
| 결제수단 | String | 필수 | "card" \| "kakaopay" \| "naverpay" \| "tosspay" |

#### 출력 항목

| 항목명 | 타입 | 설명 |
|---|---|---|
| 결제 결과 | Object | {success, transaction_id, amount} |
| 이용권 정보 | Object | {id, start_date, end_date, status} |
| 영수증 | Object | {receipt_number, items, total} |

#### 비즈니스 규칙
1. 최종 결제금액 0원 이상
2. 쿠폰: 유효기간 내 + 미사용 쿠폰만 선택 가능
3. 마일리지: 잔액 초과 사용 불가
4. 결제 완료 시: 이용권 활성화 + 매출 등록 + 마일리지 적립 (자동)
5. 마일리지 자동 적립: 결제액의 1% (센터 설정에 따라)
6. 결제 실패 시 이용권 생성 안 함 (트랜잭션)

#### 예외 처리

| 상황 | 대응 |
|---|---|
| 결제 실패 | "결제에 실패했습니다. 다시 시도해주세요" |
| 쿠폰 만료 | "선택한 쿠폰이 만료되었습니다" (쿠폰 해제) |
| 마일리지 부족 | "마일리지 잔액이 부족합니다. 현재 잔액: {balance}" |
| PG 모듈 오류 | "결제 시스템에 일시적인 문제가 있습니다" |
| 이용권 생성 실패 | 결제 자동 취소 + "이용권 생성에 실패했습니다. 결제가 취소됩니다" |

## 공통 사항

### 인증/보안
- 모든 API 호출: Supabase Auth JWT 필수
- RLS: member_id = auth.uid() 조건으로 본인 데이터만 접근
- 토큰 만료 시 자동 갱신 (refresh token)
- 갱신 실패 시 로그인 화면으로 이동

### 오프라인 처리
- 네트워크 미연결 시 "인터넷 연결을 확인해주세요" 배너
- 캐시된 데이터 표시 (최신 데이터 아님을 표시)
- QR 체크인: 오프라인 불가 (실시간 검증 필수)

### 데이터 격리
- 모든 데이터: branch_id + member_id 기준 격리
- 회원은 자신의 데이터만 조회/수정 가능
- 관리자 데이터(매출/직원/설정 등) 접근 불가
