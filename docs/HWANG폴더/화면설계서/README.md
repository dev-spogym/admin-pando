# FitGenie CRM 화면설계서 — HWANG 운영 우선본

## 목적

- 이 폴더는 기존 `docs/화면설계서`를 수정하지 않고, 실제 운영과 퍼블리싱 기준으로 필요한 핵심 화면만 다시 정리한 화면설계서다.
- 원본처럼 모든 상태 파일을 세분화하지 않고, 운영 우선 화면의 마스터 스펙 중심으로 작성했다.

## 사용 원칙

- 디자인, 퍼블리싱, 화면 QA는 이 폴더를 우선 기준으로 본다.
- 기능 정의는 `../기능명세서/`와 함께 본다.
- 자동화 정책은 본사용 화면과 지점용 화면을 분리해서 본다.

## 문서 구조

- 각 화면은 `00-기본화면.md` 마스터 파일로 작성한다.
- 필요 시 상태 파일을 추가 확장할 수 있다.
- 이 버전은 운영 기준 확정이 목적이므로, 불필요한 상태 분해는 생략했다.
- 도메인별 `다이얼로그_카탈로그.md`는 해당 도메인의 전체 다이얼로그 인벤토리와 목적을 정리한 문서다.
- `원본_대응표.md`는 기존 `docs/화면설계서`와 `HWANG폴더` 문서의 대응 관계를 설명한다.
