# SCR-H201 회원 목록 — 기본화면

## 1. 화면 목적

- 회원 운영의 허브 화면이다.
- 만료예정, 홀딩, 비활성, 미등록을 한눈에 나누어 보고 상세 화면으로 진입한다.

## 2. 주요 사용자

- `manager`, `fc`, `staff`

## 3. 레이아웃

```text
PageHeader
StatCardGrid
StatusTab
SearchFilter
MemberTable
Pagination
```

- 상단 통계 카드: 전체, 활성, 만료예정, 금일 상담예정
- 상태 탭: 전체, 활성, 홀딩, 만료예정, 만료, 비활성, 미등록
- 메인 테이블 핵심 컬럼: 이름, 연락처, 담당자, 상태, 대표 이용권, 만료일, 최근 방문일

## 4. 핵심 기능

- 검색어, 상태, 담당자, 만료일, 최근 방문일 필터
- 회원 상세 진입
- 엑셀 다운로드
- 선택 회원 일괄 액션

## 5. 상태/배지 규칙

- `미등록`은 회색 배지, `비활성`과 분리
- `만료예정`은 경고 배지
- `홀딩`은 별도 강조 배지

## 6. 권한 차이

- `readonly`는 조회만 가능
- `staff`는 등록/수정 가능
- `manager` 이상만 상태 일괄 변경 가능

## 7. 연계 화면

- 회원 상세
- 결제 처리
- 상담 등록

## 8. 퍼블리싱 포인트

- 표보다 필터와 상태 인지가 중요하다.
- 테이블 1행만 봐도 운영 판단이 가능해야 한다.
