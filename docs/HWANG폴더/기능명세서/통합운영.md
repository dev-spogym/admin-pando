# 통합운영 기능명세서

## 1. 목적

- 개별 도메인에 흩어져 있는 현장 운영 기능을 통합 시점에서 정리한다.
- 출석, 키오스크, IoT, 건강 데이터 같은 현장형 기능의 연결 기준을 정의한다.

## 2. 포함 범위

- 통합 출석 현황
- 키오스크 운영
- IoT/출입 모니터링
- 체성분/건강 데이터 통합 조회

## 3. 핵심 기능

### 3.1 통합 출석 현황

- 앱, 키오스크, 관리자 수동 처리 출석을 한 화면에서 확인
- 예외 출석 케이스 확인

### 3.2 키오스크 운영

- 현재 동작 상태 확인
- 배너/안내 문구 노출 상태 확인
- 장애 또는 오프라인 상태 경고

### 3.3 IoT/출입 모니터링

- 게이트, 출입 규칙, 장비 상태 확인
- 운영 장애 감지

### 3.4 건강 데이터 통합

- 체성분, 운동 기록, 기본 건강지표 연결 조회

## 4. 운영 규칙

- 통합운영은 신규 기준 생성 화면이 아니라, 여러 도메인의 운영 상태를 한 번에 보는 화면으로 본다.
- 출석 원천이 달라도 최종 출석 상태는 하나로 합쳐 보여야 한다.
- 키오스크와 IoT는 설정 화면과 운영 화면을 분리한다.

## 5. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 통합 출석 조회 | staff |
| 키오스크 운영 확인 | staff |
| 출입 장애 대응 | manager |
| 건강 통합 조회 | manager |

## 6. 퍼블리싱 포인트

- 통합운영 화면은 카드 나열보다 운영 예외 탐지가 우선이다.
- 정상 상태보다 장애 상태와 대기 상태가 더 눈에 띄어야 한다.
