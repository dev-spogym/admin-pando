## 4. 유저케이스 상세

### UC-001. 로그인

| 항목 | 내용 |
|------|------|
| **목적** | 시스템에 접속하여 본사 관리 업무를 시작한다 |
| **선행 조건** | 슈퍼관리자 계정이 사전에 등록되어 있어야 한다 |
| **결과** | 로그인 성공 시 대시보드 화면으로 이동, 사이드바에 본사 관리 메뉴 노출 |

**정상 시나리오**:
1. 로그인 화면에서 이메일과 비밀번호를 입력한다
2. [로그인] 버튼을 누른다
3. 인증 성공 → 대시보드(/) 화면으로 이동한다
4. 사이드바 상단에 "지점 전환" 드롭다운과 "본사 관리" 메뉴가 나타난다
5. 하단 프로필 영역에 `[본사] 김본부장 / 슈퍼관리자` 로 표시된다

**예외 시나리오**:
- 비밀번호 오류 → "이메일 또는 비밀번호가 올바르지 않습니다" 안내
- 5회 연속 실패 → 🆕 계정 잠금 + "본사 IT팀에 문의하세요" 안내
- 세션 만료 → 자동으로 로그인 화면으로 이동

### UC-002. 지점 전환

| 항목 | 내용 |
|------|------|
| **목적** | 특정 지점의 데이터만 따로 확인하거나, 다시 전체 지점 통합 뷰로 돌아온다 |
| **선행 조건** | 슈퍼관리자로 로그인된 상태 |
| **결과** | 선택한 지점의 데이터 기준으로 모든 화면이 갱신된다 |

**정상 시나리오**:
1. 사이드바 상단 "지점 전환" 드롭다운을 클릭한다
2. 등록된 전체 지점 목록이 나타난다 (최상단에 "전체 지점 (통합)")
3. 원하는 지점을 선택한다 (예: "강남점")
4. 화면 하단 토스트: "강남점으로 전환되었습니다"
5. 현재 보고 있는 페이지의 데이터가 강남점 기준으로 새로고침된다

**예외 시나리오**:
- 폐점 상태 지점 선택 → 🆕 "폐점된 지점입니다. 조회만 가능합니다" 안내

**🆕 개선 사항**:
- 최근 접속 지점 3개를 드롭다운 상단에 "최근" 영역으로 노출
- 지점 검색 기능 (지점 수가 10개 이상일 때 유용)

3. 지점별 현황 카드에서 각 지점의 상태(운영중/휴업/폐점)와 주요 수치를 확인한다

5. 하단 감사 로그에서 최근 10건의 활동을 확인한다
6. [전체 보기]를 누르면 감사 로그 전체 화면으로 이동한다

**🆕 추가 필요 기능**:
- 지점별 매출 순위 바 차트 (어느 지점이 잘하고 있는지 시각적으로 비교)
- 위험 신호 알림 카드 ("분당점 이번주 매출 전주 대비 -30%", "강남점 만료예정 회원 급증")
- 지점 현황 카드에서 지역별/상태별 필터
- 자동 새로고침 (30초~1분 간격, 토글 ON/OFF)

### UC-004. 지점 관리

| 항목 | 내용 |
|------|------|
| **목적** | 지점을 추가·수정·폐점하고, 지점 간 통합 통계를 비교한다 |
| **화면** | 본사 관리 > 지점 관리 |

**정상 시나리오 — 신규 지점 추가**:
1. [지점 추가] 버튼을 누른다
2. 등록 폼이 열린다:
   - **기본 정보**: 지점명(필수), 지점 코드, 상태, 대표자명
   - **위치/운영**: 주소, 전화번호, 운영시간, 최대 정원
   - **추가 설정**: 메모
3. 필수 항목을 모두 입력하고 [저장]을 누른다
4. "지점이 등록되었습니다" 안내 → 목록에 새 지점 추가

**정상 시나리오 — 지점 폐점**:
1. 해당 지점의 상태를 "폐점"으로 변경한다
2. 🆕 경고 안내: "이 지점에 활성 회원 315명, 재직 직원 7명이 있습니다"
3. 🆕 회원 이관 대상 지점을 선택한다
4. 🆕 직원 재배치 또는 퇴직 처리를 진행한다
5. 최종 확인 후 폐점 처리 완료

**화면에서 확인할 수 있는 정보**:
- 지점 현황 목록: 지점명, 상태, 주소, 전화, 대표자, 회원 수, 직원 수, 월매출
- 지점별 통합 통계: 활성 회원, 만료 회원, 이번달 매출, 직원 수, 출석률
- 회원 지점 이동 이력: 날짜, 회원명, 이전 지점 → 이동 지점, 처리자
- 각 테이블별 엑셀 다운로드 가능

**🆕 추가 필요 기능**:
- 지점 오픈 체크리스트 (필수 설정 항목이 모두 완료되었는지 안내)
- 기존 지점의 상품/정책을 복사해서 새 지점에 적용하는 "설정 복사" 기능
- 지점별 운영시간을 평일/주말/공휴일 구분 설정

### UC-005. 지점 비교 리포트

| 항목 | 내용 |
|------|------|
| **목적** | 지점 간 성과를 비교하여 우수 지점/부진 지점을 파악한다 |
| **화면** | 본사 관리 > 지점 비교 리포트 |

**정상 시나리오**:
1. 비교 기간을 선택한다 (이번달 / 지난달 / 분기 / 커스텀)
2. 비교 대상 지점을 선택한다 (전체 또는 특정 지점 복수 선택)
3. 비교 결과가 표와 차트로 표시된다:
   - 매출 비교 (막대 그래프)
   - 회원 수 추이 (꺾은선 그래프)
   - 신규 등록 / 이탈 비교
   - 출석률 비교
4. 엑셀 다운로드 가능

**🆕 추가 필요 기능**:
- 지점 랭킹 보드 (매출 TOP, 회원 증가율 TOP, 출석률 TOP)
- 지점별 목표 매출 대비 달성률 게이지
- "이 지점은 왜 잘하고 있는가?" 인사이트 코멘트 (🆕 AI 분석)

### UC-006. 감사 로그 조회

| 항목 | 내용 |
|------|------|
| **목적** | 누가, 언제, 어떤 작업을 했는지 추적하여 보안과 운영 투명성을 확보한다 |
| **화면** | 본사 관리 > 감사 로그 |

**필터 옵션**:

| 필터 | 유형 | 선택지 |
|------|------|--------|
| 기간 | 날짜 범위 | 시작일 ~ 종료일 |
| 액션 유형 | 드롭다운 | 전체 / 로그인 / 로그아웃 / 로그인 실패 / 생성 / 수정 / 삭제 / 지점 전환 / 권한 변경 / 퇴직 / 이동 / 휴직 시작 / 휴직 종료 / 회원 이적 / 회원 탈퇴 / 환불 / 내보내기 / 설정 변경 / 지점 생성 / 지점 폐점 / 슈퍼관리자 권한 부여/회수 |
| 사용자 | 텍스트 검색 | 사용자 이름 |
| 지점 | 드롭다운 | 전체 / 등록된 지점 목록 |

**필터 옵션**:

| 필터 | 유형 | 선택지 |
|------|------|--------|

| 성별 | 드롭다운 | 전체 / 남 / 여 |
| 최종만료일 | 날짜 범위 | 시작일 ~ 종료일 |
| 최근방문일 | 날짜 범위 | 시작일 ~ 종료일 |
| 회원구분 | 드롭다운 | 전체 / 일반 / VIP / 법인 등 |
| 유입경로 | 드롭다운 | 전체 / SNS / 소개 / 방문 등 |

**추가 필터 (체크박스)**:
- ☐ 만료 상품 숨기기
- ☐ 관심회원만 보기 (⭐ 즐겨찾기)
- ☐ 미방문 N일 이상 (7일 / 14일 / 30일)

**정렬**: 이름, 등록일, 만료일, 최근방문일 기준 오름차순/내림차순 클릭 정렬

**테이블 컬럼**:
- ☐ (체크박스, 다중 선택), ⭐ (즐겨찾기), 이름, 전화번호, 상태 배지, 회원권, 등록일, 만료일, 담당코치, 잔여횟수

**일괄 액션 (다중 선택 시)**:
- 상태 일괄 변경 모달 → 상태 선택 드롭다운 → 확인
- 메시지 일괄 발송

**정상 시나리오**:
1. 회원 목록 화면에 접속한다
2. 상단 탭과 상태 탭으로 1차·2차 분류를 선택한다
3. 필요 시 필터 드롭다운(계약상품, 성별, 날짜 범위 등)을 적용한다
4. 이름이나 전화번호로 검색한다
5. 컬럼 헤더를 클릭하여 정렬한다
6. 엑셀 다운로드로 전체 데이터를 내보낸다
7. 회원 행을 클릭하면 회원 상세 화면으로 이동한다
8. 체크박스로 다중 선택 후 일괄 액션을 수행한다
9. 페이지네이션으로 페이지를 이동한다 (기본 20건/페이지)

**각 탭의 역할**:

| 탭 | 목적 | 주요 내용 | 가능한 행동 |
|----|------|----------|------------|
| 결제 내역 | 이 회원의 모든 결제 기록 확인 | 결제일, 상품명, 금액, 결제수단, 상태 | 환불 처리, 영수증 조회 |

| 상담 이력 | 상담 기록 관리 | 일자, 유형(상담/OT/체험/재등록), 채널(방문/전화/카카오톡/DM/SNS), 상태, 결과 | 상담 등록, 수정, 삭제 |

| 평가 | 체력 평가 기록 | 평가 항목, 점수, 날짜 | 평가 등록 |
| 운동 프로그램 | 배정된 운동 루틴 | 프로그램명, 구성 운동, 난이도 | 프로그램 배정/변경 |

- **탈퇴**: [탈퇴] 클릭 → 사유 입력, 환불 여부 확인 → 탈퇴 처리
- **삭제**: [삭제] 클릭 → "이 회원의 모든 데이터가 삭제됩니다" 최종 확인

**🆕 추가 필요 기능**:

**정상 시나리오**:
1. 상품 관리 화면에 접속한다
2. 상단 통계 카드에서 전체/활성/비활성 상품 수와 카테고리 수를 확인한다
3. 3단계 필터로 원하는 상품을 빠르게 찾는다:
   - **1단계 — 상품 타입**: 전체 / 회원권 / 수강권 / 대여권 / 일반
   - **2단계 — 분류(카테고리)**: 이용권 / PT / GX / 기타 등 (동적으로 추가 가능)
   - **3단계 — 종목**: 전체 / 헬스 / 필라테스 / 요가 / 수영 / 복싱 / 크로스핏 / 기타
4. 상품명으로 텍스트 검색을 한다
5. 목록에서 각 상품의 핵심 정보를 한눈에 확인한다:
   - 상품명 + 타입 배지(회원권/수강권/대여권/일반)
   - 현금가, 카드가
   - 이용 기간(일), 이용 횟수
   - **요일별 이용 가능 여부** (월~일 각각 ✓/✗ 표시)
   - **이용 가능 시간대** (예: 06:00~22:00 또는 "전체")
   - 홀딩 가능 ✓/✗, 양도 가능 ✓/✗, 포인트 적립 ✓/✗
   - 상태 (사용/미사용)
6. 상품 행을 클릭하면 우측에 상세 패널이 열린다 (마스터-디테일 레이아웃)
7. 엑셀 다운로드로 상품 목록을 내보낸다

2. 상품 등록 패널이 열린다
3. **상품구분**을 먼저 선택한다 → 종류에 따라 입력 폼이 달라진다:
   - **레슨**: 레슨시간, 레슨유효기간, 수업구분(개인/정규클래스), 사용인원 필드 활성

   - **락커**: 기간 + 위치 관련 필드
   - **판매**: 금액만 (일반 물품)
4. **이용구분**(기간/횟수/포인트)을 선택한다 → 기간이면 기간 입력, 횟수면 횟수 입력

6. **요일/시간설정**에서 이용 가능한 요일을 체크하고, 요일마다 시작·종료 시간을 설정한다

8. 하단 설정에서 시설이용범위, 예약시점, 휴회횟수, 예약간격, 휴회기간, 판매유형을 설정한다
9. [등록] 클릭 → "상품이 등록되었습니다" 안내
10. 목록에 즉시 반영된다

**빠른 등록 — 상품정보가져오기**:
1. 하단 [상품정보가져오기] 버튼을 클릭한다
2. 기존 등록된 상품 목록이 검색 모달로 열린다
3. 복사할 상품을 선택한다
4. 해당 상품의 모든 정보가 폼에 자동 채워진다
5. 상품명과 금액만 변경하고 [등록] → 빠르게 유사 상품 추가

**예외 시나리오**:
- 상품명 미입력 → "상품명을 입력하세요" 안내
- 금액 미입력 → "금액을 입력하세요" 안내
- 🆕 동일 상품명 중복 등록 시도 → "같은 이름의 상품이 이미 있습니다" 경고

### UC-022e. 상품 분류(카테고리) 관리

| 항목 | 내용 |
|------|------|
| **목적** | 상품을 분류할 카테고리를 자유롭게 추가·수정·삭제한다 |
| **화면** | 상품 관리 > [분류 관리] 탭 |

센터마다 취급하는 종목이 다릅니다 (헬스+필라테스 vs 골프+수영). 분류 관리를 통해 **해당 센터에 맞는 상품 카테고리를 자유롭게 구성**할 수 있습니다.

**정상 시나리오**:
1. 상품 관리 화면 상단에서 [분류 관리] 탭을 클릭한다
2. 현재 등록된 분류 목록이 테이블로 표시된다:
   - 분류명, 정렬순서, 상태(활성/비활성)
3. 새 분류를 추가한다:
   - 분류명(필수) 입력
   - 정렬순서 입력 (숫자가 작을수록 앞에 표시)
   - 활성 여부 토글
4. [등록] 클릭 → 목록에 추가
5. 기존 분류 수정: [수정 아이콘] → 폼에 기존값 채움 → [수정 저장]
6. 분류 삭제: [삭제 아이콘] → 확인 → 삭제

**분류 예시**:

| 순서 | 분류명 | 상태 | 용도 |
|:---:|--------|:---:|------|
| 1 | 이용권 | 활성 | 기본 센터 이용권 (1개월, 3개월, 6개월, 12개월) |
| 2 | PT | 활성 | 개인 트레이닝 수강권 |
| 3 | GX | 활성 | 그룹 운동 수강권 (요가, 필라테스 등) |
| 4 | 골프 | 활성 | 골프 타석 이용권 |
| 5 | 락커 | 활성 | 사물함 대여 상품 |
| 6 | 운동복 | 활성 | 운동복 대여 상품 |
| 7 | 판매 | 활성 | 보충제, 음료, 용품 등 일반 물품 |

**🆕 추가 필요 기능**:
- 분류별 아이콘/색상 지정 (시각적 구분)

### UC-022f. 할인 정책 관리

| 항목 | 내용 |
|------|------|
| **목적** | 상품 판매 시 적용할 수 있는 할인 정책을 등록·관리한다 |
| **화면** | 설정 > 할인 설정 |

**정상 시나리오**:
1. 할인 설정 화면에 접속한다
2. 등록된 할인 정책 목록이 표시된다:
   - 정책명, 할인 유형(정률/정액), 할인값, 최소 등록 기간, 최대 할인 한도, 상태
3. [할인 추가] 버튼 클릭 → 모달:
   - **정책명**: 예) "조기등록 10% 할인", "가족 2만원 할인"
   - **할인 유형**: 정률(%) / 정액(원)
   - **할인값**: 10(%) 또는 20,000(원)
   - **최소 등록 기간**: 이 기간 이상 등록해야 적용 (예: 3개월 이상)
   - **최대 할인 한도**: 정률 할인 시 최대 금액 캡 (예: 최대 5만원)
   - **활성 여부**: ON/OFF
4. [저장] → 목록에 추가
5. POS 결제 시 이 정책 목록에서 선택하여 적용

**할인 정책 활용 예시**:

| 정책명 | 유형 | 값 | 조건 | 한도 |
|--------|:---:|:---:|------|:---:|
| 조기등록 할인 | 정률 | 10% | 3개월 이상 등록 | 5만원 |
| 가족 할인 | 정액 | 2만원 | — | — |
| 법인 할인 | 정률 | 15% | 6개월 이상 등록 | 10만원 |
| 재등록 감사 할인 | 정률 | 5% | — | 3만원 |
| 오픈 프로모션 | 정액 | 5만원 | — | — |

**🆕 추가 필요 기능**:
- 할인 정책에 유효기간 설정 (프로모션 기간 한정)
- 할인 정책 적용 대상 상품/카테고리 지정 (전체 or 특정 상품만)
- 중복 할인 규칙 (쿠폰 + 할인 정책 동시 적용 가능 여부)
- 할인 적용 이력 추적 (어떤 정책이 얼마나 사용되었는지)

### UC-022g. 상품과 다른 기능의 연계 관계

상품은 센터 운영의 중심에 있으며, 다음과 같이 여러 기능과 연결됩니다:

```
                         ┌─────────────┐
                         │   상품 관리   │
                         └──────┬──────┘
                                │
        ┌───────────┬───────────┼───────────┬───────────┐
        │           │           │           │           │
  ┌─────▼────┐ ┌────▼─────┐ ┌──▼──────┐ ┌──▼──────┐ ┌──▼──────┐
  │ POS 결제  │ │ 회원 등록 │ │ 쿠폰     │ │ 할인 설정│ │ 매출 통계│
  │ 상품 선택 │ │ 회원권   │ │ 적용 대상│ │ 적용    │ │ 상품별  │
  │ → 결제   │ │ 연결     │ │ 상품    │ │ 상품    │ │ 분석    │
  └──────────┘ └──────────┘ └─────────┘ └─────────┘ └─────────┘
        │           │
  ┌─────▼────┐ ┌────▼─────┐
  │ 환불 관리 │ │ 선수익금 │
  │ 상품 기준 │ │ 상품별   │
  │ 환불 계산 │ │ 인식     │
  └──────────┘ └──────────┘
```

| 연계 기능 | 관계 | 설명 |
|----------|------|------|
| POS 결제 | 상품 → 매출 | 상품 목록에서 선택하여 결제 처리 |
| 회원 등록 | 상품 → 회원권 | 회원 등록 시 상품을 선택하면 기간/횟수 자동 설정 |
| 캘린더/수업 | 상품 → 예약 | 수강권 상품의 잔여 횟수에 따라 예약 가능 |
| 쿠폰 | 상품 ← 쿠폰 | 쿠폰 생성 시 적용 대상 상품/카테고리 지정 |
| 할인 정책 | 상품 ← 할인 | POS 결제 시 해당 상품에 할인 정책 적용 |
| 환불 | 상품 → 환불 | 환불 시 상품 가격·이용 기간 기준으로 환불금 계산 |
| 매출 통계 | 상품 → 분석 | 상품별/카테고리별 매출 비중 분석 |
| 선수익금 | 상품 → 재무 | 기간 상품의 미소진 금액을 선수익금으로 관리 |
| 키오스크 | 상품 → 노출 | 키오스크 노출 ON인 상품만 키오스크 화면에 표시 |
| 온보딩 | 상품 → 추적 | 신규 회원이 어떤 상품으로 시작했는지 추적 |

**메시지 발송 화면**:

*발송 채널*:

| 채널 | 글자수 한도 | 건당 비용 | 비고 |
|------|:---:|:---:|------|

*수신자 선택 모달*:

- 선택된 회원 목록 (이름, 전화번호, 그룹)

- [취소] [확인] 버튼

*메시지 작성 폼*:

| 항목 | 설명 |
|------|------|

**정상 시나리오**:
1. 채널을 선택한다 (알림톡/SMS/푸시)
2. [수신자 선택] → 수신자 모달에서 회원을 검색·선택한다
3. 메시지를 작성한다 (치환 변수 삽입 가능)

6. 확인 다이얼로그 → 발송 완료 토스트

### A-1. 전자계약 위자드 (`/contracts/new`)

5단계 위자드 스텝 바: **① 회원 선택 → ② 상품 선택 → ③ 기간/금액 → ④ 결제 → ⑤ 확인**

**Step 1: 회원 선택**
- 회원 검색 (이름/전화번호)
- 검색 결과 테이블: 이름, 전화번호, 상태, 보유 회원권
- 행 클릭 → 회원 선택 → 선택된 회원 정보 카드 표시
- [← 이전] [다음 단계 →] 버튼

**Step 2: 상품 선택**
- 카테고리 탭: 시설이용 / PT / GX / 옵션
- 상품 테이블: 상품명, 가격, 기간, 횟수
- 체크박스로 다중 선택 가능 (패키지 구성)
- 선택된 상품 목록 하단 표시

**Step 3: 기간/금액 설정**

| 항목 | 비고 |
|------|------|
| 시작일 | 날짜 선택 (기본: 오늘) |
| 종료일 | 자동 계산 (시작일 + 상품 기간) 또는 수동 변경 |
| 정가 합계 | 자동 계산 |
| 할인 유형 | 드롭다운: 재등록 / 신규 / 이벤트 / 관리자 재량 |
| 할인 금액 | 숫자 입력 |

### A-13. 일정 요청 처리 (`/schedule-requests`)

### A-14. 회원 등록/수정 폼 전체 필드 상세 (`/members/new`, `/members/edit`)

### A-17. 데이터 엔티티 관계도

시스템의 핵심 데이터 엔티티와 관계를 나타냅니다:

```
┌─────────┐     ┌──────────┐     ┌──────────┐
│ tenant  │ 1:N │ branches │ 1:N │  users   │
│ (테넌트) │────→│ (지점)   │────→│ (직원)   │
└─────────┘     └────┬─────┘     └────┬─────┘
                     │ 1:N            │ 1:N
                     │                │
                ┌────▼─────┐    ┌─────▼────┐
                │ members  │    │  staff   │
                │ (회원)   │    │ (직원상세)│
                └────┬─────┘    └──────────┘
                     │
      ┌──────────────┼──────────────┬──────────────┐
      │ 1:N          │ 1:N          │ 1:N          │ 1:N
 ┌────▼────┐  ┌──────▼─────┐ ┌─────▼────┐  ┌──────▼──────┐
 │  sales  │  │consultations│ │attendance│  │lesson_bookings│
 │ (매출)  │  │ (상담 이력) │ │ (출석)   │  │ (수업 예약)  │
 └────┬────┘  └────────────┘ └──────────┘  └─────────────┘
      │ 1:1
 ┌────▼────┐
 │products │
 │ (상품)  │←── product_groups (분류)
 └─────────┘←── discount_policies (할인)

 기타 엔티티:
 ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
 │ lockers  │ │  rfid    │ │ clothing │ │  rooms   │
 │ (락커)   │ │(밴드/카드)│ │ (운동복) │ │(운동룸)  │
 └──────────┘ └──────────┘ └──────────┘ └──────────┘

 ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
 │ classes  │ │ notices  │ │audit_logs│ │ coupons  │
 │ (수업)   │ │(공지사항) │ │(감사로그) │ │ (쿠폰)  │
 └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

**핵심 관계 요약**:
- tenant → branches: 1개 테넌트가 N개 지점 보유
- branch → members: 1개 지점에 N명 회원 소속
- branch → staff/users: 1개 지점에 N명 직원 소속
- member → sales: 1명 회원이 N건 결제
- member → consultations: 1명 회원이 N건 상담 이력
- member → attendance: 1명 회원이 N건 출석 기록
- member → lesson_bookings: 1명 회원이 N건 수업 예약
- sales → products: 매출 건이 상품을 참조
- products → product_groups: 상품이 분류에 소속
- lockers ↔ members: 락커-회원 배정 관계

### A-18. 에러 처리 & 예외 시나리오 모음

개발자가 구현해야 하는 전체 에러/예외 케이스:

**인증/세션**:

| 시나리오 | 동작 | UI 반응 |
|----------|------|---------|
| 로그인 실패 (비밀번호 오류) | 감사 로그 LOGIN_FAILED 기록 | 토스트 에러: "이메일 또는 비밀번호가 올바르지 않습니다" |
| 세션 만료 | onAuthStateChange → 자동 로그아웃 | /login 리다이렉트 |
| 권한 없는 페이지 접근 | PrivateRoute에서 차단 | /forbidden 페이지 표시 |
| 🆕 5회 연속 로그인 실패 | 계정 잠금 | "계정이 잠겼습니다. 본사에 문의하세요" |

**데이터 CRUD**:

| 시나리오 | 동작 | UI 반응 |
|----------|------|---------|
| 네트워크 오류 | API 호출 실패 | 토스트 에러: "데이터를 불러오지 못했습니다" |
| 필수 필드 미입력 | 클라이언트 검증 | 필드 하단 빨간 에러 메시지 + 테두리 빨강 |
| 전화번호 중복 | 서버 조회 결과 | "이미 등록된 번호입니다" + 등록 차단 |
| 이용 중 회원 있는 상품 삭제 | 서버 검증 | 🆕 "이용 중인 회원이 N명 있습니다. 비활성화하시겠습니까?" |
| 체성분 동일 날짜 중복 | 서버 조회 | "같은 날짜에 이미 기록이 있습니다. 덮어쓸까요?" |
| 결제 중복 감지 | 서버 조회 | "유사한 결제가 이미 존재합니다. 계속하시겠습니까?" |
| 잔여 횟수 0인 상태에서 예약 | 클라이언트 검증 | 🆕 "잔여 횟수가 없습니다. 추가 결제가 필요합니다." |
| 만료된 회원 출석 시도 | 서버 검증 | 🆕 "회원권이 만료되었습니다. 재등록이 필요합니다." |
| 홀딩 중 회원 예약 시도 | 클라이언트 검증 | 🆕 "현재 홀딩 중입니다. 홀딩 해제 후 예약 가능합니다." |

**결제/환불**:

## 7. 용어 정의

| 용어 | 설명 |
|------|------|
| **슈퍼관리자** | 본사 레벨의 최상위 권한 사용자. 모든 지점을 통합 관리 |
| **최고관리자** | 개별 지점의 최상위 권한 사용자 (=원장) |
| **리드** | 아직 회원이 아닌 잠재 고객. 유입 경로와 상담 과정을 추적 |
| **온보딩** | 신규 회원 등록 후 초기 적응 과정 (OT, 체성분 측정, 프로그램 배정 등) |
| **홀딩** | 회원권 일시정지 (출장, 부상 등 사유). 정지 기간만큼 만료일 연장 |
| **노쇼** | 예약 후 연락 없이 불참. 페널티(횟수 차감) 부과 대상 |
| **MoM** | Month-over-Month. 전월 대비 증감률 |
| **선수익금** | 회원이 미리 납부했지만 아직 서비스가 제공되지 않은 금액 |
| **감사 로그** | 시스템 내 모든 중요 행위의 기록 (누가, 언제, 무엇을 했는지) |
