### UC-001. 로그인

| 항목 | 내용 |
|------|------|
| **목적** | 시스템에 접속하여 본사 관리 업무를 시작한다 |
| **선행 조건** | 슈퍼관리자 계정이 사전에 등록되어 있어야 한다 |
| **결과** | 로그인 성공 시 대시보드 화면으로 이동, 사이드바에 본사 관리 메뉴 노출 |

**정상 시나리오**:
1. 로그인 화면에서 이메일과 비밀번호를 입력한다
2. [로그인] 버튼을 누른다
3. 인증 성공 → 대시보드(/) 화면으로 이동한다
4. 사이드바 상단에 "지점 전환" 드롭다운과 "본사 관리" 메뉴가 나타난다
5. 하단 프로필 영역에 `[본사] 김본부장 / 슈퍼관리자` 로 표시된다

**예외 시나리오**:
- 비밀번호 오류 → "이메일 또는 비밀번호가 올바르지 않습니다" 안내
- 5회 연속 실패 → 🆕 계정 잠금 + "본사 IT팀에 문의하세요" 안내
- 세션 만료 → 자동으로 로그인 화면으로 이동

### UC-002. 지점 전환

| 항목 | 내용 |
|------|------|
| **목적** | 특정 지점의 데이터만 따로 확인하거나, 다시 전체 지점 통합 뷰로 돌아온다 |
| **선행 조건** | 슈퍼관리자로 로그인된 상태 |
| **결과** | 선택한 지점의 데이터 기준으로 모든 화면이 갱신된다 |

**정상 시나리오**:
1. 사이드바 상단 "지점 전환" 드롭다운을 클릭한다
2. 등록된 전체 지점 목록이 나타난다 (최상단에 "전체 지점 (통합)")
3. 원하는 지점을 선택한다 (예: "강남점")
4. 화면 하단 토스트: "강남점으로 전환되었습니다"
5. 현재 보고 있는 페이지의 데이터가 강남점 기준으로 새로고침된다

**예외 시나리오**:
- 폐점 상태 지점 선택 → 🆕 "폐점된 지점입니다. 조회만 가능합니다" 안내

**🆕 개선 사항**:
- 최근 접속 지점 3개를 드롭다운 상단에 "최근" 영역으로 노출
- 지점 검색 기능 (지점 수가 10개 이상일 때 유용)

| **선행 조건** | 지점이 1개 이상 등록되어 있어야 한다 |

3. 지점별 현황 카드에서 각 지점의 상태(운영중/휴업/폐점)와 주요 수치를 확인한다
4. 특정 지점의 [상세 보기]를 누르면 해당 지점으로 전환되어 지점 대시보드로 이동한다
5. 하단 감사 로그에서 최근 10건의 활동을 확인한다
6. [전체 보기]를 누르면 감사 로그 전체 화면으로 이동한다

**🆕 추가 필요 기능**:
- 지점별 매출 순위 바 차트 (어느 지점이 잘하고 있는지 시각적으로 비교)
- 위험 신호 알림 카드 ("분당점 이번주 매출 전주 대비 -30%", "강남점 만료예정 회원 급증")
- 지점 현황 카드에서 지역별/상태별 필터
- 자동 새로고침 (30초~1분 간격, 토글 ON/OFF)

### UC-004. 지점 관리

| 항목 | 내용 |
|------|------|
| **목적** | 지점을 추가·수정·폐점하고, 지점 간 통합 통계를 비교한다 |
| **화면** | 본사 관리 > 지점 관리 |

**정상 시나리오 — 신규 지점 추가**:
1. [지점 추가] 버튼을 누른다
2. 등록 폼이 열린다:
   - **기본 정보**: 지점명(필수), 지점 코드, 상태, 대표자명
   - **위치/운영**: 주소, 전화번호, 운영시간, 최대 정원
   - **추가 설정**: 메모
3. 필수 항목을 모두 입력하고 [저장]을 누른다
4. "지점이 등록되었습니다" 안내 → 목록에 새 지점 추가

**정상 시나리오 — 지점 폐점**:
1. 해당 지점의 상태를 "폐점"으로 변경한다
2. 🆕 경고 안내: "이 지점에 활성 회원 315명, 재직 직원 7명이 있습니다"
3. 🆕 회원 이관 대상 지점을 선택한다
4. 🆕 직원 재배치 또는 퇴직 처리를 진행한다
5. 최종 확인 후 폐점 처리 완료

**화면에서 확인할 수 있는 정보**:
- 지점 현황 목록: 지점명, 상태, 주소, 전화, 대표자, 회원 수, 직원 수, 월매출
- 지점별 통합 통계: 활성 회원, 만료 회원, 이번달 매출, 직원 수, 출석률
- 회원 지점 이동 이력: 날짜, 회원명, 이전 지점 → 이동 지점, 처리자
- 각 테이블별 엑셀 다운로드 가능

**🆕 추가 필요 기능**:
- 지점 오픈 체크리스트 (필수 설정 항목이 모두 완료되었는지 안내)
- 기존 지점의 상품/정책을 복사해서 새 지점에 적용하는 "설정 복사" 기능
- 지점별 운영시간을 평일/주말/공휴일 구분 설정

### UC-005. 지점 비교 리포트

| 항목 | 내용 |
|------|------|
| **목적** | 지점 간 성과를 비교하여 우수 지점/부진 지점을 파악한다 |
| **화면** | 본사 관리 > 지점 비교 리포트 |

**정상 시나리오**:
1. 비교 기간을 선택한다 (이번달 / 지난달 / 분기 / 커스텀)
2. 비교 대상 지점을 선택한다 (전체 또는 특정 지점 복수 선택)
3. 비교 결과가 표와 차트로 표시된다:
   - 매출 비교 (막대 그래프)
   - 회원 수 추이 (꺾은선 그래프)
   - 신규 등록 / 이탈 비교
   - 출석률 비교
4. 엑셀 다운로드 가능

**🆕 추가 필요 기능**:
- 지점 랭킹 보드 (매출 TOP, 회원 증가율 TOP, 출석률 TOP)
- 지점별 목표 매출 대비 달성률 게이지
- "이 지점은 왜 잘하고 있는가?" 인사이트 코멘트 (🆕 AI 분석)

### UC-006. 감사 로그 조회

| 항목 | 내용 |
|------|------|
| **목적** | 누가, 언제, 어떤 작업을 했는지 추적하여 보안과 운영 투명성을 확보한다 |
| **화면** | 본사 관리 > 감사 로그 |

**정상 시나리오**:
1. 감사 로그 화면에 접속한다
2. 전체 활동 로그가 최신순으로 나열된다:
   - **시간**: 2026-04-10 09:28
   - **사용자**: 김매니저
   - **지점**: 강남점
   - **액션**: 로그인 / 생성 / 수정 / 삭제 / 환불 / 지점전환 / 권한변경 등
   - **대상**: 회원 #1248, 상품 #45 등
   - **상세**: 변경 내용 요약
3. 필터를 적용한다:
   - 기간 (시작일 ~ 종료일)
   - 액션 유형 (로그인, 생성, 수정, 삭제, 환불 등 17종)
   - 사용자
   - 지점
4. 페이지를 넘기며 과거 기록을 확인한다

**언제 사용하나?**:
- 회원이 "내가 신청 안 했는데 환불 처리 됐다" 문의 → 환불 처리자와 시점 확인

### UC-013. 회원 상세 조회 & 관리

| 항목 | 내용 |
|------|------|
| **목적** | 특정 회원의 모든 정보를 한 화면에서 확인하고 관리한다 |
| **화면** | 회원 상세 |

**화면 구성**:

```
┌─────────────────────────────────────────────────────────┐
│  ← 회원 목록     김회원 (010-1234-5678)                  │
│                  🟢 활성  |  PT 6개월권  |  잔여 12회    │
│                  등록: 2026-01-15  만료: 2026-07-15      │
│                  담당: 박코치                            │
│                                                         │
│  [수정]  [홀딩]  [탈퇴]  [삭제]  [지점이관]              │
├─────────────────────────────────────────────────────────┤
│  결제내역 | 예약 | 상담이력 | 체성분 | 평가 |             │
│  운동프로그램 | 운동일지 | 변경이력 | 상세이력            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  (선택한 탭의 내용이 여기에 표시됩니다)                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**각 탭의 역할**:

| 탭 | 목적 | 주요 내용 | 가능한 행동 |
|----|------|----------|------------|
| 결제 내역 | 이 회원의 모든 결제 기록 확인 | 결제일, 상품명, 금액, 결제수단, 상태 | 환불 처리, 영수증 조회 |
| 예약 | PT/GX 예약 현황과 출석 이력 | 날짜, 수업명, 강사, 출결 상태 | 예약 등록, 취소, 노쇼 처리 |
| 상담 이력 | 상담 기록 관리 | 일자, 유형(상담/OT/체험/재등록), 채널(방문/전화/카카오톡/DM/SNS), 상태, 결과 | 상담 등록, 수정, 삭제 |
| 체성분 | InBody 등 체성분 측정 기록 | 날짜, 체중, 골격근량, 체지방률, BMI | 측정 데이터 등록, 추이 그래프 확인 |
| 평가 | 체력 평가 기록 | 평가 항목, 점수, 날짜 | 평가 등록 |
| 운동 프로그램 | 배정된 운동 루틴 | 프로그램명, 구성 운동, 난이도 | 프로그램 배정/변경 |
| 운동 일지 | 일별 운동 기록 | 날짜, 운동 내용, 세트/횟수, 무게 | 일지 작성 |
| 변경 이력 | 회원 정보 변경 히스토리 | 변경일, 변경 항목, 이전값 → 변경값, 처리자 | 조회만 가능 |
| 상세 이력 | 전체 활동 타임라인 | 모든 활동(등록, 결제, 출석, 상담 등) 시간순 | 조회만 가능 |

**회원 상태 변경 시나리오**:
- **홀딩**: [홀딩] 클릭 → 사유, 시작일, 예상 복귀일 입력 → 홀딩 처리 (회원권 일시정지)
- **홀딩 해제**: 홀딩 중인 회원 > [홀딩 해제] → 남은 기간만큼 만료일 연장
- **탈퇴**: [탈퇴] 클릭 → 사유 입력, 환불 여부 확인 → 탈퇴 처리
- **삭제**: [삭제] 클릭 → "이 회원의 모든 데이터가 삭제됩니다" 최종 확인

**🆕 추가 필요 기능**:
- 회원 Before/After 사진 관리 (체형 변화 기록)
- 회원에게 바로 메시지 보내기 버튼
- 회원 메모/태그 빠른 편집

### UC-026. 리드(잠재 고객) 관리

| 항목 | 내용 |
|------|------|
| **목적** | 아직 회원이 아닌 잠재 고객을 관리하고, 회원 전환을 추적한다 |
| **화면** | 영업/마케팅 > 리드 관리 |

**정상 시나리오**:
1. 리드 목록을 확인한다 (이름, 연락처, 유입경로, 관심상품, 상태, 담당자)
2. 리드를 등록한다:
   - 이름(필수), 연락처(필수)

   - 관심 상품, 메모
3. 리드에게 연락 후 상담 결과를 기록한다
4. 상담 후 등록 결정 시 → 회원 등록으로 연결 (리드 정보 자동 채움)

**🆕 추가 필요 기능**:

- 리드 자동 할당 (라운드 로빈 방식으로 담당자 배정)
- 리드 후속 조치 알림 (접수 후 D+1, D+3, D+7에 자동 리마인드)
- 유입 경로별 전환율 분석 (어떤 채널에서 온 리드가 회원으로 가장 많이 전환되는지)

### UC-027. 메시지 발송 & 자동 알림 & 쿠폰

| 항목 | 내용 |
|------|------|
| **목적** | 회원에게 메시지를 보내고, 자동 알림을 설정하고, 쿠폰을 배포한다 |

**메시지 발송**:
1. 수신자를 선택한다 (전체 / 조건 필터 / 개별 선택)
2. 메시지 유형을 선택한다 (SMS / 카카오 알림톡 / 카카오 친구톡)
3. 메시지를 작성한다 (템플릿 또는 직접 작성)
4. 즉시 발송 또는 예약 발송
5. 발송 결과를 확인한다 (성공/실패 건수)

**자동 알림 설정**:
- 만료 D-30, D-7, D-1 자동 발송
- 생일 축하 메시지
- 장기 미출석 회원 알림 (N일 이상)
- PT 잔여 횟수 부족 알림
- 예약 리마인드 (수업 시작 1시간 전)
- 각 규칙별 활성/비활성 토글, 템플릿 편집

**쿠폰 관리**:
1. 쿠폰을 생성한다 (이름, 할인유형, 금액/비율, 유효기간, 사용한도)
2. 대상을 선택하여 배포한다
3. 사용 현황을 추적한다

**🆕 추가 필요 기능**:
- 메시지 발송 비용 관리 (잔여 건수 표시, 충전)
- 자동 알림 A/B 테스트 (어떤 문구가 재등록률이 높은지)
- 쿠폰 사용률 분석

## 7. 용어 정의

| 용어 | 설명 |
|------|------|
| **슈퍼관리자** | 본사 레벨의 최상위 권한 사용자. 모든 지점을 통합 관리 |
| **최고관리자** | 개별 지점의 최상위 권한 사용자 (=원장) |
| **리드** | 아직 회원이 아닌 잠재 고객. 유입 경로와 상담 과정을 추적 |
| **온보딩** | 신규 회원 등록 후 초기 적응 과정 (OT, 체성분 측정, 프로그램 배정 등) |
| **홀딩** | 회원권 일시정지 (출장, 부상 등 사유). 정지 기간만큼 만료일 연장 |
| **노쇼** | 예약 후 연락 없이 불참. 페널티(횟수 차감) 부과 대상 |
| **MoM** | Month-over-Month. 전월 대비 증감률 |
| **선수익금** | 회원이 미리 납부했지만 아직 서비스가 제공되지 않은 금액 |
| **감사 로그** | 시스템 내 모든 중요 행위의 기록 (누가, 언제, 무엇을 했는지) |
