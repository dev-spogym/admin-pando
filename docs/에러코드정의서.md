# FitGenie CRM — 에러코드 정의서

> **문서 버전**: v1.0
> **작성일**: 2026-04-14
> **목적**: 시스템 전체 에러코드의 네이밍 컨벤션, 모듈별 범위 할당, HTTP 매핑, 사용자/개발자 메시지 분리 기준을 정의

## 1. 에러코드 네이밍 컨벤션

### 형식

```
E{HTTP상태코드}{일련번호}
```

| 구성 요소 | 설명 | 예시 |
|-----------|------|------|
| `E` | 에러코드 접두사 (고정) | E |
| HTTP 상태코드 | 3자리 HTTP 상태코드 | 400, 401, 403, 404, 409, 500 |
| 일련번호 | 3자리 순번 (모듈 범위 내) | 001, 002, 003 |

**예시:** `E400001` = HTTP 400 Bad Request 계열의 첫 번째 에러

### HTTP 상태코드 분류

| HTTP 상태코드 | 의미 | 사용 시나리오 |
|--------------|------|--------------|
| 400 | Bad Request | 필수값 누락, 형식 오류, 조건 위반 |
| 401 | Unauthorized | 인증 실패, 토큰 만료 |
| 402 | Payment Required | 결제 실패, 결제 수단 오류 |
| 403 | Forbidden | 권한 부족, 접근 제한 |
| 404 | Not Found | 리소스 없음, 데이터 미존재 |
| 409 | Conflict | 중복 데이터, 상태 충돌 |
| 422 | Unprocessable Entity | 비즈니스 로직 위반 |
| 500 | Internal Server Error | 서버 내부 오류 |
| 503 | Service Unavailable | 외부 서비스 연동 실패 |

## 3. API 응답 형식

### 성공 응답

```json
{
  "success": true,
  "data": { ... },
  "message": null,
  "errorCode": null
}
```

### 에러 응답

```json
{
  "success": false,
  "data": null,
  "message": "사용자에게 표시할 메시지 (한글)",
  "errorCode": "E400001"
}
```

### 메시지 분리 원칙

| 구분 | 대상 | 내용 | 예시 |
|------|------|------|------|
| `message` | 사용자 (프론트엔드 표시) | 친절하고 명확한 한글 안내 | "필수 입력 항목을 확인해주세요" |
| 서버 로그 | 개발자 (서버 로그) | 기술적 상세 정보 | "Required field 'phone' is missing in POST /api/members" |

> **원칙:** `message`에는 기술 용어, 스택 트레이스, DB 컬럼명을 절대 노출하지 않는다.

## 5. 프론트엔드 에러 처리 가이드

### 에러 표시 방식

| HTTP 상태코드 | 표시 방식 | 예시 |
|--------------|-----------|------|
| 400 | 인라인 필드 에러 / 토스트 | 입력 필드 하단 빨간 텍스트 |
| 401 | 로그인 페이지 리다이렉트 | 세션 만료 모달 → 로그인 |
| 402 | 모달 다이얼로그 | 결제 실패 안내 팝업 |
| 403 | 토스트 알림 | "접근 권한이 없습니다" |
| 404 | 토스트 알림 | "데이터를 찾을 수 없습니다" |
| 409 | 토스트 알림 + 안내 | "이미 등록된 데이터입니다" |
| 422 | 모달 또는 토스트 | 비즈니스 로직 안내 |
| 500 | 전역 에러 토스트 | "일시적인 오류가 발생했습니다" |
| 503 | 전역 에러 토스트 + 재시도 버튼 | "서비스 연결에 실패했습니다" |

### 에러 처리 흐름

```
API 요청 → 응답 수신
  ├── success: true → 정상 처리
  └── success: false
       ├── HTTP 401 → 토큰 갱신 시도 → 실패 시 로그인 리다이렉트
       ├── HTTP 403 → 에러 메시지 토스트 표시
       ├── HTTP 5xx → 전역 에러 핸들러 → 재시도 안내
       └── 기타 → errorCode 기반 분기 처리 → message 표시
```
