# FitGenie CRM QA 테스트케이스 문서

**문서 버전:** v2.0
**작성일:** 2026년 3월 11일

**플랫폼:** Web

---

## 1. 테스트 전략 요약

### 1.1 테스트 범위
- **기능 테스트:** 인증, 회원관리, 매출관리, 상품관리, 시설관리, 직원관리, 메시지/마케팅, 계약/구독, 설정/권한
- **비기능 테스트:** 보안(데이터 격리, 인증), 성능(목록/대시보드 로딩), 호환성(브라우저), 접근성
- **API 테스트:** 인증, 회원, 기타 API 엔드포인트
- **UI/UX 테스트:** 폼 검증, 키보드 네비게이션, 비밀번호 토글

### 1.2 테스트 환경
| 항목 | 설정 |
|------|------|
| **환경** | Staging |
| **브라우저** | Chrome (Primary), Safari, Firefox |
| **디바이스** | Desktop |
| **OS** | Windows 11, macOS |

### 1.3 테스트 사이클
- **주기:** 1차 (Phase 1)
- **담당자:** QA담당자1, QA담당자2

### 1.4 자동화 전략
| 구분 | 수량 | 비고 |
|------|------|------|
| 자동화 예정 | 12개 TC | Selenium/Playwright 기반 E2E 테스트 |
| 수동 테스트 | 35개 TC | 수동 검증 필수 영역 |
| API 테스트 | 3개 TC | cURL/Postman 기반 API 테스트 |

- **테스트 유형:** 기능 테스트

- **심각도:** Critical
- **자동화 여부:** 수동
