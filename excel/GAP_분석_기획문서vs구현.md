### SCR-001 | 로그인 (`Login.tsx`, 174줄)

#### 구현됨
- 이메일/비밀번호 입력 폼
- 지점 선택 드롭다운
- 비밀번호 마스킹 토글
- 로그인 유지 체크박스
- 로딩 상태 (스피너)
- 에러 메시지 표시 (shake 애니메이션)

#### 미구현 기능
| 기능 | 기획문서 요구사항 | 현재 상태 |
|------|-------------------|-----------|
| JWT 인증 | JWT 토큰 발급, localStorage 저장 | Mock (admin/password123 하드코딩) |
| 계정 잠금 | 5회 연속 실패 시 30분 잠금 | 미구현 |
| 비밀번호 정책 | 90일 경과 시 변경 안내 | 미구현 |
| 세션 관리 | 동시 세션 1개 제한 | 미구현 |
| 이메일 검증 | 이메일 형식 실시간 검증 (regex) | 미구현 (ID 필드로 구현) |
| 비밀번호 찾기 | 비밀번호 찾기 모달 | 버튼만 있음, 기능 없음 |
| 탈퇴 계정 | 탈퇴 계정 접근 불가 안내 | 미구현 |

#### 필요한 세부정책
```
1. 비밀번호 정책
   - 최소 8자, 영문+숫자+특수문자 조합
   - 이전 3개 비밀번호 재사용 금지
   - 90일 경과 시 변경 권장 팝업 (강제 변경 or 건너뛰기)

2. 계정 잠금 정책
   - 5회 연속 실패 → 30분 잠금
   - 잠금 해제: 비밀번호 찾기 또는 관리자 해제
   - 잠금 시 남은 시간 표시

3. 세션 관리 정책
   - 동시 로그인 1개만 허용
   - 새 로그인 시 기존 세션 강제 종료 or 신규 로그인 차단 중 선택
   - 토큰 만료: 2시간 (갱신 가능)
   - 로그인 유지 체크 시: 7일 (refresh token)

4. 지점 전환 정책
   - 로그인 시 선택한 지점만 데이터 접근
   - 지점 전환: 재로그인 없이 헤더에서 변경 가능 여부 결정 필요
```

   - 차트: 페이지 진입 시 1회
   - 알림 리스트: 실시간 (WebSocket or polling 30초)

   - 미납 30일 초과 → 장기미납 표시

4. 통계 카드 클릭 동작
   - 총회원 → /members (전체 필터)
   - 활성회원 → /members?status=active
   - 만료임박 → /members?status=imminent
   - 만료회원 → /members?status=expired
```

### SCR-010 | 회원 목록 (`MemberList.tsx`, 506줄)

#### 구현됨
- 검색 (이름/연락처)
- 상태 탭 필터 (7개: 전체/활성/만료/예정/임박/홀딩/미등록)
- 추가 필터 (회원구분, 계약상품, 최종만료일, 최근방문일, 성별)
- 통계 카드 4개
- DataTable + 페이지네이션
- 행 선택 + 일괄 처리 (상태변경/전송/출석/관심회원)
- 빈 상태 UI
- 탭 (회원/회원권/수강권/락커/운동복/상품별)

#### 미구현 기능
| 기능 | 기획문서 요구사항 | 현재 상태 |
|------|-------------------|-----------|
| 검색 debounce | 300ms debounce | 미구현 (즉시 필터) |
| 컬럼 정렬 | 컬럼 헤더 클릭 정렬 | sortable 플래그만, 미작동 |
| 페이지당 건수 | 20/50/100건 선택 | 미구현 |
| 엑셀 다운로드 | xlsx export | alert 스텁만 |
| 권한별 범위 | 권한별 조회 범위 제한 | 미구현 |
| 삭제 회원 | 기본 미표시, 옵션으로 표시 | 미구현 |
| API 연동 | GET /api/v1/members | Mock 데이터 |

#### 필요한 세부정책
```
1. 검색 정책
   - 이름: 부분일치 (2글자 이상 입력 시 검색)
   - 연락처: 숫자만 추출 후 부분일치
   - 회원번호: 완전일치
   - debounce: 300ms

2. 필터 조합 정책
   - 모든 필터는 AND 조합
   - 탭 필터(상태)와 상세 필터 동시 적용
   - 필터 초기화 버튼 필요

3. 회원 상태 정의
   - 활성: 유효한 이용권 1개 이상 보유
   - 만료: 모든 이용권 만료, 마지막 만료일 기준
   - 예정: 시작일이 미래인 이용권 보유
   - 임박: 만료일 D-30 이내 (설정 가능)
   - 홀딩: 이용권 정지(홀딩) 중
   - 미등록: 회원 등록만, 이용권 없음

4. 일괄 처리 정책
   - 최대 100명 동시 선택
   - 상태 변경: 활성↔비활성, 홀딩 처리
   - 메시지 발송: 선택 회원 SMS/카카오톡
   - 출석 처리: 수동 출석 일괄 등록
```

```
1. 필수 입력 항목

3. 자동 생성 항목
   - 회원번호: 지점코드-YYMMDD-순번

   - first_reg_date: 등록 시점
   - 기본 상태: "미등록" (이용권 구매 시 "활성")

4. 수정 시 변경 이력
   - 변경 전/후 값 저장
   - 변경자 + 변경일시 기록
   - 연락처 변경 시 재중복 체크
