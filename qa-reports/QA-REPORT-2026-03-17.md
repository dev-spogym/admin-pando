## 2. 버그 (Critical/High)

### BUG-01: 지점비교리포트 — admin 403 차단 [Critical]
- **경로**: `/branch-report`
- **현상**: admin(ADMIN=primary) 역할로 접근 시 403 "접근 권한이 없습니다" 페이지 표시
- **기대**: primary 역할은 모든 페이지 접근 가능해야 함
- **원인 추정**: `branchReport` 라우트 권한이 `superadmin only`로 설정되어 있거나, `isSuperAdmin` 체크에 걸리는 것으로 보임
- **스크린샷**: `admin-27-지점비교리포트.png`

### BUG-02: 슈퍼대시보드 — admin 403 차단 [High]
- **경로**: `/super-dashboard`
- **현상**: admin 역할로 접근 시 403 차단
- **비고**: `isSuperAdmin: false`이므로 설계 의도라면 정상이나, ADMIN 역할이 SuperDashboard에 접근 못하는 것이 맞는지 비즈니스 확인 필요
- **스크린샷**: `admin-28-슈퍼대시보드.png`

### BUG-03: 감사로그 조회 실패 [High]
- **경로**: `/audit-log`
- **현상**: 페이지 진입 시 "감사 로그 조회에 실패했습니다" toast 에러 노출, 데이터 0건
- **원인 추정**: `audit_logs` 테이블이 없거나 RLS 정책으로 인해 anon key로 조회 불가
- **스크린샷**: `admin-29-감사로그.png`

### BUG-04: 지점 상태 매핑 오류 [High]
- **경로**: `/branches`
- **현상**: 3개 지점 모두 "폐점" 상태로 표시, 활성 지점 0개
- **기대**: 시드 데이터 기준 강남본점/서초점은 "운영중", 송파점은 "임시휴업"
- **원인 추정**: status 필드 값과 UI 라벨 매핑 불일치 (DB: 'ACTIVE' → UI에서 '폐점'으로 표시)
- **스크린샷**: `admin-26-지점관리.png`

### BUG-05: 락커관리 데이터 미표시 [Medium]
- **경로**: `/locker`
- **현상**: 전체 락커 0개, 검색 결과 없음
- **기대**: 시드 데이터에 50개 락커 존재 (사물함관리 `/locker/management`에서는 50개 정상 표시)
- **원인 추정**: 락커관리와 사물함관리가 다른 테이블/API를 참조하거나, branchId 필터 이슈
- **스크린샷**: `admin-11-락커관리.png` vs `admin-12-사물함관리.png`

## 3. UX 이슈

### UX-01: 트레이너 대시보드 데이터 격리 [Medium]
- **현상**: trainer1(코치)로 로그인 시 대시보드에 전체 회원 23명, 매출 586만 표시
- **기대**: 코치는 자신이 담당하는 회원만 보여야 함 (담당 회원 필터링)
- **비고**: 로그인 직후에는 0명으로 표시되다가 대시보드 재진입 시 전체 데이터 노출 — 데이터 로딩 타이밍 이슈도 있음
- **스크린샷**: `trainer1-00-login-success.png` (0명) vs `trainer1-01-대시보드.png` (23명)

### UX-02: 출석관리 — 데이터 없을 때 빈 화면 [Low]
- **경로**: `/attendance`
- **현상**: 출석 데이터 0건일 때 "조회된 데이터가 없습니다" 메시지는 있으나 빈 공간이 넓음
- **개선**: CTA ("수동 출석" 버튼 강조) 또는 가이드 문구 추가 권장
- **스크린샷**: `admin-03-출석관리.png`

### UX-03: 마일리지 요약 카드 불일치 [Low]
- **경로**: `/mileage`
- **현상**: "전체 발행 마일리지: 0", "전체 사용 마일리지: 0" 이지만 "잔여 마일리지: 395,599"
- **기대**: 발행 총액 = 사용 + 잔여 이므로 발행이 0이면 잔여도 0이어야 함
- **원인**: 요약 카드와 목록 데이터가 다른 API/집계 로직 사용
- **스크린샷**: `admin-04-마일리지관리.png`

### UX-04: 회원이전 직접 접근 시 에러 [Low]
- **경로**: `/members/transfer`
- **현상**: "잘못된 접근입니다. 회원 ID가 없습니다" toast
- **비고**: 회원 상세에서만 접근하도록 설계됨. 사이드바에 메뉴 노출 안 하면 문제없음. 단, URL 직접 입력 시 사용자 혼란 가능
- **스크린샷**: `admin-32-회원이전.png`

### UX-05: 전자계약 — 만료 회원만 표시 [Low]
- **경로**: `/contracts/new`
- **현상**: 전자계약 위저드 1단계에서 모든 회원이 "만료" 상태 배지로 표시
- **비고**: 활성 회원도 계약 갱신이 가능해야 하므로, 상태와 무관하게 선택 가능하면 OK. 다만 "만료" 배지만 보이는 것이 의도인지 확인 필요
- **스크린샷**: `admin-05-전자계약.png`

## 4. 잘 된 점 (Highlights)

### 디자인 & UI
- **일관된 디자인 시스템**: 모든 페이지에서 동일한 색상 토큰, 카드 레이아웃, 테이블 스타일 유지
- **사이드바**: 역할별 메뉴 필터링 완벽. 접힘/펼침 상태, 현재 위치 하이라이트 잘 동작
- **헤더**: 지점명, 사용자명, 역할 라벨, 알림 배지 모두 정상 표시
- **빈 상태(Empty State)**: 대부분 페이지에서 데이터 없을 때 적절한 아이콘 + 안내 문구 표시
- **403 페이지**: 깔끔한 디자인, "대시보드로 이동" / "이전 페이지로" CTA 제공

### 기능 완성도
- **대시보드**: 6개 요약 카드 + 3개 차트(회원분포 도넛, 주간출석 바, 매출추이 바) 잘 구현
- **회원목록**: 상태 필터(전체/활성/만료/미등록/홀딩/정지), 다중 필터, 정렬 모두 동작
- **POS 시스템**: 카테고리별 상품 카드 + 장바구니 패턴 직관적
- **결제 처리**: 카드/현금/마일리지/복합결제 4가지 수단 제공
- **사물함 관리**: 50개 그리드 시각화 + 빈 사물함 추천 기능 훌륭
- **운동룸**: 카드뷰 + 시간대별 예약 히트맵 (여유/혼잡/만석) 시각적으로 우수
- **급여 시스템**: 기본급/인센티브/공제 분리, 명세서 조회 + PDF 다운로드
- **메시지 발송**: 알림톡/SMS/앱푸시 채널 선택, 비용 미리보기
- **자동알림**: 13종 규칙 카드 (D-7/D-3/D-1/생일/미출석 등) 토글 직관적
- **쿠폰관리**: 사용현황 프로그레스 바 시각화
- **키오스크 설정**: 타입 A/B 선택 + 폰 미리보기 훌륭
- **구독관리**: SaaS 플랜 현황 + 사용량 프로그레스 바
- **권한설정**: 역할×메뉴 권한 매트릭스 UI 잘 구현
- **직원등록**: 역할 선택 시 해당 권한 설명 실시간 표시 (좋은 UX)

### RBAC 권한
- **admin → 32/34 페이지 접근**: 슈퍼대시보드/지점비교리포트만 차단 (isSuperAdmin 전용)
- **manager → 18/34 페이지**: 설정/직원/급여/감사로그 등 적절히 차단
- **trainer → 7/34 페이지**: 대시보드/회원목록/출석/캘린더/급여명세서/체성분만 접근 — 최소 권한 원칙 준수
