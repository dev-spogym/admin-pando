# SCR-H301 결제 처리 — 기본화면

## 1. 화면 목적

- 현장 결제를 빠르게 완료하는 화면이다.

## 2. 레이아웃

```text
BuyerPanel
ProductPanel
DiscountPanel
PaymentSummary
ActionFooter
```

## 3. 핵심 기능

- 회원 검색
- 상품 선택
- 할인 적용
- 결제 수단 선택
- 최종 금액 확인
- 결제 완료 또는 미수금 분기

## 4. 상태 규칙

- 결제 상태와 이용권 생성 여부를 분리 안내한다.
- 분할결제중은 정상 결제와 다른 강조 방식을 사용한다.

## 5. 권한 차이

- `staff`는 결제 가능
- 환불/강제 예외 처리는 `manager` 이상

## 6. 퍼블리싱 포인트

- 속도 우선 화면이므로 2~3단계 안에서 끝나야 한다.
- 경고 다이얼로그는 중복 결제, 미수금, 환불 이슈만 강하게 처리한다.
