# HWANG 화면설계서 원본 대응표

## 목적

- 기존 `docs/화면설계서`와 `docs/HWANG폴더/화면설계서`의 대응 관계를 정리한다.
- `HWANG폴더`는 배포용 기준서이므로 일부 원본 화면을 하나의 운영 화면으로 통합한 경우가 있다.

## 통합된 대표 케이스

| 원본 화면 | HWANG 대응 |
| --- | --- |
| 회원등록 + 회원수정 | `SCR-H203-회원등록수정` |
| 센터설정 + 지점 자동화 적용 | `SCR-H902`, `SCR-H901`로 분리 |
| 본사 자동알림 관련 화면 | `SCR-H1001`, `SCR-H901`, `SCR-H806`으로 역할 분리 |
| 키오스크/IoT 관련 설정 | `SCR-H904`, `SCR-H906`, `SCR-H907`, `SCR-H1103`, `SCR-H1104`로 운영/설정 분리 |
| 공간 자산 관리 | `SCR-H603`, `SCR-H606`, `SCR-H607`로 통합/세부 분리 |

## 다이얼로그 처리 원칙

- D01-공통의 글로벌 다이얼로그는 개별 `00-기본화면.md`로 작성했다.
- 각 도메인의 상세 다이얼로그는 `다이얼로그_카탈로그.md`에서 전체 인벤토리와 트리거를 관리한다.
- 추후 특정 다이얼로그가 상세 설계가 더 필요하면 해당 도메인 안에 개별 폴더를 추가 확장한다.

## 사용 기준

- 퍼블리싱과 QA는 `HWANG폴더` 문서를 우선 기준으로 본다.
- 원본 문서와 이름이 1:1로 같지 않아도 운영 역할과 화면 목적이 같으면 대응되는 것으로 본다.
