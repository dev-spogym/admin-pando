# 시설 · 부가서비스 운영기획서

## 1. 문서 목적

- 시설 운영과 부가서비스를 회원 운영과 끊기지 않게 정리한다.
- 락커, 카드, 운동룸, 타석, 운동복 같은 실물 자산을 일관된 기준으로 관리한다.

## 2. 운영 대상

- 락커
- 사물함 배정
- RFID/밴드 카드
- 운동룸
- 골프 타석
- 운동복

## 3. 공통 운영 원칙

- 시설자산은 `자산 마스터`, `현재 상태`, `배정 대상`, `이력`으로 본다.
- 회원 이용권과 연결되는 자산은 만료일과 연동해 운영한다.
- 단발성 점검 상태와 장기 비활성 상태를 같은 개념으로 쓰지 않는다.
- 배정형 자산은 언제 누구에게 연결되었는지 이력이 남아야 한다.

## 4. 자산별 기준

### 4.1 락커

- 락커는 회원 부가 이용권 또는 옵션 상품과 연결한다.
- 상태는 `사용가능 / 사용중 / 만료임박 / 만료 / 고장 / 비활성` 기준으로 본다.
- 만료임박 판정은 자동화 정책과 같은 기준을 따른다.
- 락커 배정과 해제는 회원 상세에서도 진입 가능해야 한다.

### 4.2 RFID/밴드 카드

- 카드 자체 상태와 회원 매핑 상태를 분리한다.
- 카드 분실, 재발급, 비활성 처리는 이력 중심으로 남긴다.
- 체크인/게이트/락커와 연결되더라도 관리 화면에서는 카드 자산 운영 기준을 우선한다.

### 4.3 운동룸/타석

- 공간 자원은 예약형 자산으로 본다.
- 상태는 `사용가능 / 사용중 / 점검중 / 예약대기` 기준을 기본으로 한다.
- 타석은 대기열 자동 승격을 지원한다.
- 운동룸과 타석은 수업 모듈과 분리된 자산이지만 예약 UX는 최대한 동일하게 맞춘다.

### 4.4 운동복

- 운동복은 재고형 자산이다.
- 상태는 `대여가능 / 대여중 / 세탁중 / 점검중 / 폐기예정`으로 관리한다.
- QR 흐름은 속도를 위해 유지하되, 회원 검색 기반 수동 대여도 동일하게 지원한다.

## 5. 화면 운영 기준

- 시설 화면은 시각적 상태 파악이 가장 중요하다.
- 리스트 뷰와 그리드 뷰를 병행하되, 락커와 타석은 그리드가 기본이다.
- 자산 상세 모달에서는 현재 상태, 사용자, 만료일, 메모, 이력을 우선 노출한다.
- 일괄 배정/일괄 해제는 매니저 이상 권한으로 제한한다.

## 6. 회원 운영과의 연결

- 회원 상세에서 아래 부가 정보는 바로 확인 가능해야 한다.
  - 락커 배정 여부
  - 카드 발급 여부
  - 운동복 대여 이력
  - 골프 타석 또는 룸 예약 이력
- 부가서비스 만료는 회원 만료와 별개일 수 있으므로 개별 상태를 유지한다.

## 7. 자동화 연결 기준

- 락커 만료임박, 카드 만료예정, 옵션 만료예정은 자동화 정책 프레임워크 안에서 동일하게 다룬다.
- 자산 상태 변화는 필요 시 회원앱 알림과 연결할 수 있지만, 기본은 관리자 운영 알림을 우선한다.

## 8. 운영 예외 처리

- 고장 자산은 배정 불가 상태로 즉시 전환한다.
- 만료 자산이더라도 회수 전까지는 실제 배정 이력을 유지한다.
- 회원 탈퇴 시 연결 자산 회수 체크리스트를 자동으로 안내한다.

## 9. 퍼블리싱 기준 메모

- 시설 화면은 표보다 상태 가시성이 중요하다.
- 셀 또는 카드 하나만 봐도 현재 사용 가능 여부가 읽혀야 한다.
- 고장, 만료, 대여중 같은 상태는 서로 비슷한 색을 쓰지 않는다.
