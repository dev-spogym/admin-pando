# 공통 기능명세서

## 1. 목적

- 관리자 웹과 회원앱 모두에 공통으로 필요한 진입, 인증, 내비게이션, 알림 구조를 정의한다.
- 개별 도메인 화면이 달라도 공통 경험이 흔들리지 않도록 기준을 고정한다.

## 2. 포함 범위

- 로그인
- 대시보드 진입
- 전역 네비게이션
- 알림 센터
- 프로필/세션 관리
- 공통 에러/권한 없음 화면

## 3. 핵심 기능

### 3.1 로그인

- 계정 인증
- 역할별 진입 분기
- 세션 만료 후 재로그인 유도

### 3.2 대시보드

- 역할별 핵심 KPI 노출
- 오늘 할 일 또는 운영 큐 요약
- 자주 쓰는 화면 바로가기

### 3.3 전역 네비게이션

- 도메인별 메뉴 이동
- 현재 조직 범위 표시
- 권한에 따른 메뉴 노출 제어

### 3.4 알림 센터

- 운영 알림, 자동화 알림, 시스템 알림 확인
- 읽음/미읽음 처리
- 관련 화면 바로가기

## 4. 운영 규칙

- 로그인 이후 첫 화면은 역할 기준으로 달라질 수 있다.
- 본사와 지점 사용자는 같은 메뉴 구조를 쓰지 않는다.
- 전역 알림은 단순 메시지함이 아니라 후속 화면 이동과 연결되어야 한다.
- 공통 에러 화면은 기술 용어보다 운영자가 이해 가능한 문구를 우선한다.

## 5. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 로그인 | 전체 |
| 대시보드 조회 | 로그인 사용자 |
| 조직 전환 | primary 이상 또는 다지점 owner |
| 알림 센터 조회 | 로그인 사용자 |

## 6. 퍼블리싱 포인트

- 대시보드는 예쁜 요약판이 아니라 즉시 액션 진입점이어야 한다.
- 사이드바는 메뉴 과다 노출보다 현재 역할에 필요한 메뉴만 보여줘야 한다.
- 알림 센터는 읽기용 리스트보다 액션 연결이 우선이다.
