# 회원관리 기능명세서

## 1. 목적

- 회원 운영의 기준 화면과 핵심 기능을 정의한다.
- 리드 전환 이후 회원 등록, 상태 관리, 상담, 이관까지를 하나의 흐름으로 본다.

## 2. 포함 범위

- 회원 목록
- 회원 등록/수정
- 회원 상세
- 회원 이관
- 상담/메모
- 회원 상태 변경

## 3. 주요 화면

| 화면 | 목적 |
| --- | --- |
| 회원 목록 | 전체 회원 조회, 필터, 빠른 액션 |
| 회원 등록/수정 | 신규 등록, 기존 정보 수정 |
| 회원 상세 | 이용권, 결제, 예약, 상담 이력 확인 |
| 회원 이관 | 지점 이동 또는 담당자 변경 |

## 4. 핵심 기능

### 4.1 회원 목록

- 상태, 담당자, 만료일, 최근 방문일 기준 검색
- 만료예정 회원 우선 필터
- 관심회원 표시
- 상세 진입
- 엑셀 다운로드

### 4.2 회원 등록/수정

- 기본정보 입력
- 연락처 중복 확인
- 운영정보 설정
- 지점 및 담당자 지정
- 저장 전 검증

### 4.3 회원 상세

- 상단 요약 카드
  - 회원 상태
  - 대표 이용권 상태
  - 만료일
  - 담당자
- 탭 구조
  - 회원정보
  - 이용권
  - 결제이력
  - 예약/출석
  - 상담/메모
- 빠른 액션
  - 홀딩
  - 탈퇴
  - 결제 등록
  - 상담 등록

### 4.4 회원 이관

- 지점 이관 체크리스트
- 담당 회원 및 예약 영향 확인
- 승인 권한 제한

## 5. 상태 기준

| 상태 | 설명 |
| --- | --- |
| 활성 | 정상 이용 회원 |
| 홀딩 | 이용권 일시 정지 |
| 만료예정 | 이용권 만료 임박 |
| 만료 | 이용권 종료 |
| 비활성 | 운영 종료 또는 관리 제외 |
| 미등록 | 정식 등록 미완료 |

## 6. 운영 규칙

- `미등록`과 `비활성`은 같은 상태로 취급하지 않는다.
- 만료예정은 회원 마스터 상태가 아니라 이용권 기준으로 계산한다.
- 홀딩은 이용권 기간 보정과 함께 동작해야 한다.
- 탈퇴는 이력 삭제가 아니라 운영 종료 처리다.
- 동일 연락처가 기존 리드나 회원과 겹치면 저장 전에 경고한다.

## 7. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 회원 조회 | staff |
| 회원 등록/수정 | staff |
| 상태 일괄 변경 | manager |
| 이관 승인 | manager |
| 탈퇴 처리 | manager |
| 전체 다운로드 | owner 이상 |

## 8. 퍼블리싱 포인트

- 목록 화면은 상태 배지와 만료 정보가 우선이다.
- 상세 화면은 탭이 많아도 상단 요약에서 핵심 판단이 끝나야 한다.
- FC 업무를 고려해 만료예정 회원과 상담 이력이 자연스럽게 연결되어야 한다.
