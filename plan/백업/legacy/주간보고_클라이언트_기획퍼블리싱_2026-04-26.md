## 2. 주요 진행 내용

### 2.1 관리자/운영 문서 기준 재정리

- 기존 `docs` 원본은 보존하고, 정리본 패키지로 재구성
- 문서 간 충돌이 있던 상태 체계, 권한 체계, 자동화 기준을 하나의 운영 기준으로 정렬
- 관리자 웹과 회원앱을 각각 독립된 운영 채널로 보고, 연결 기준만 명확히 하는 방향으로 정리

### 2.2 자동화/만료 알림 정책 정렬

- 기존의 고정형 `설정(ex. D-30, D-14, D-7 등)` 해석에서 벗어나, 본사가 정책 세트를 등록하고 지점이 선택 적용하는 구조로 기준 정리
- 지점은 본사가 정의한 정책 범위 안에서만 선택해 사용할 수 있도록 방향 정리
- 이에 따라 관리자 웹, 회원앱, 현장 운영 화면에서 모두 동일한 정책 결과를 기준으로 표시되도록 정리

### 2.3 기능명세서 배포 기준 보완

- `D01 공통`부터 `D12 회원앱`까지 전체 도메인 기능명세를 운영 기준으로 재정리
- 구현 방식이나 기술 메모는 제외하고, 화면 기획과 퍼블리싱에 필요한 정책/역할/상태/업무 흐름 중심으로 문서화
- 본사, 지점, 현장 인력, 회원앱 역할별로 필요한 기능 범위를 다시 정리

### 2.4 화면설계서 퍼블리싱 기준 보완

- 공통 화면, 도메인별 핵심 리스트/상세/설정 화면을 실제 퍼블리싱 기준의 마스터 문서로 재정리
- 총 12개 도메인 기준으로 화면설계 범위를 확장했고, 화면 마스터 기준 136개 문서를 정리
- 상태 배지, CTA 우선순위, 역할별 노출 차이, 빈 상태/제한 상태 기준을 퍼블리싱 판단 기준으로 고정
- 본사용 정책 화면과 지점용 적용 화면을 분리해 운영 흐름이 섞이지 않도록 정리

### 2.5 모바일 앱(B) 기획 연동 정리

- B 문서는 A의 축소판이 아니라 A와 연동되는 앱 채널로 유지하는 방향으로 정리
- 기능을 `유지`, `A 보강 필요`, `정책 연동 필요`, `앱 고유 기능` 기준으로 재분류
- 클라이언트 공유용 문서에서는 라우트 경로, 코드/파일 경로, 구현 상태값 등 내부 기술 요소를 제거 또는 기획 문장으로 치환
