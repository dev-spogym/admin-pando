# 마케팅 기능명세서

## 1. 목적

- 리드 운영, 메시지 발송, 자동화, 쿠폰, 마일리지, 전자계약의 기준을 정리한다.
- 리드 파이프라인과 만료 알림 체계를 운영 기준으로 통일한다.

## 2. 포함 범위

- 리드 관리
- 메시지 발송
- 자동 알림 설정
- 쿠폰 관리
- 마일리지 관리
- 전자계약 위자드

## 3. 리드 상태 기준

| 상태 | 설명 |
| --- | --- |
| 신규문의 | 첫 접촉 전 |
| 연락완료 | 1차 응대 완료 |
| 상담예정 | 상담 일정 또는 진행 중 |
| 방문완료 | 방문/체험 완료 |
| 등록완료 | 회원 전환 완료 |
| 보류 | 추후 재접촉 |
| 미전환 | 종료 |

## 4. 핵심 기능

### 4.1 리드 관리

- 목록/칸반 전환
- 상태 이동
- 담당 FC 지정
- 리드 등록/수정/삭제
- 회원 전환 연결

### 4.2 메시지 발송

- 대상 검색
- 채널 선택
- 미리보기
- 발송 이력 조회

### 4.3 자동 알림 설정

- 본사 정책 라이브러리 기반 정책 선택
- 상품군별 정책 적용
- 허용된 스텝 on/off
- 테스트 발송

### 4.4 쿠폰/마일리지

- 발급, 사용 여부, 만료 관리
- 회원 상세 및 앱 노출과 연계

## 5. 자동화/만료 알림 기준

- 고정 `D-30 / D-14 / D-7 / D-3 / D-1` 전제를 기본 구조로 고정하지 않는다.
- 본사가 정책 세트를 만든다.
- 지점은 세트를 선택 적용한다.
- 지점은 본사가 허용한 범위 안에서만 스텝을 끄거나 채널을 조정할 수 있다.
- 만료 알림은 발송 이력으로 끝나지 않고 FC 후속 액션 큐와 연결된다.

## 6. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 리드 조회/수정 | fc |
| 일괄 메시지 발송 | manager |
| 자동화 정책 적용 | owner 또는 manager |
| 자동화 정책 생성 | primary 이상 |

## 7. 퍼블리싱 포인트

- 리드 화면은 FC 업무 속도가 가장 중요하다.
- 자동화 화면은 본사와 지점 화면을 분리해야 한다.
- 알림 설정을 단순 토글 모음으로 만들지 말고 정책 세트 구조로 보여야 한다.
