# 본사관리 기능명세서

## 1. 목적

- 본사 화면의 역할을 운영 총괄 기준으로 정리한다.
- 전사 KPI, 지점 관리, 정책 라이브러리의 소유권을 명확히 한다.

## 2. 포함 범위

- 본사 대시보드
- 지점 관리
- KPI 대시보드
- 감사 로그
- 전사 자동화 정책 라이브러리

## 3. 핵심 기능

- 전사 현황 모니터링
- 지점 상태 관리
- KPI 비교
- 감사 로그 조회
- 자동화 정책 세트 생성/수정/중지
- 공통 템플릿 관리

## 4. 자동화 정책 라이브러리 기준

- 본사가 상품군별 권장 정책 세트를 등록한다.
- 각 정책 세트는 다수의 스텝을 가진다.
- 스텝은 기준일, 채널, 템플릿, 필수 여부, 지점 토글 허용 여부를 가진다.
- 사용 중지된 세트는 신규 적용만 막고 기존 지점 운영은 별도 정리 절차를 둔다.

## 5. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 본사 현황 조회 | primary |
| 지점 등록/상태변경 | primary |
| 전사 정책 생성/수정 | primary |
| 감사 로그 조회 | superAdmin 또는 primary |

## 6. 퍼블리싱 포인트

- 본사 화면은 숫자보다 의사결정이 중요하다.
- 정책 라이브러리는 테이블보다 정책 카드 + 스텝 리스트 조합이 적합하다.
- 지점용 화면과 혼동되지 않도록 전사 범위 배지와 조직 범위 표시가 필요하다.
