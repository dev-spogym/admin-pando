# 설정관리 기능명세서

## 1. 목적

- 지점 설정과 권한, 센터 운영 설정의 범위를 정리한다.
- 본사 정책과 지점 적용 설정을 분리한다.

## 2. 포함 범위

- 센터 설정
- 권한 설정
- 키오스크 설정
- IoT/출입 설정
- 출석 관리 설정
- 지점 자동화 적용 설정

## 3. 핵심 기능

### 3.1 센터 설정

- 기본정보 관리
- 공지/알림 채널 기본값
- 운영 안내 정보 관리

### 3.2 권한 설정

- 표준 역할 기준 권한 확인
- 커스텀 역할은 표준 역할을 넘지 않는 범위에서 운영

### 3.3 지점 자동화 적용 설정

- 상품군별 정책 세트 선택
- 허용된 스텝 on/off
- 테스트 발송
- 후속 액션 담당자 기준 설정

## 4. 운영 규칙

- 본사 정책 생성과 지점 적용은 같은 화면에서 처리하지 않는다.
- 지점은 정책 세트를 생성하지 않는다.
- 센터 설정은 지점 범위, 권한 설정은 조직 범위에서 본다.

## 5. 권한 기준

| 기능 | 최소 권한 |
| --- | --- |
| 센터 설정 조회 | manager |
| 센터 설정 수정 | owner |
| 권한 설정 수정 | owner 이상 |
| 자동화 적용 설정 | owner 또는 manager |

## 6. 퍼블리싱 포인트

- 설정 화면은 메뉴 수가 많아도 정책 성격에 따라 그룹을 분리해야 한다.
- 본사 소유와 지점 소유 항목이 한 카드 안에 섞이면 안 된다.
- 자동화 적용 화면은 토글 나열보다 정책 요약 중심 구조가 낫다.
