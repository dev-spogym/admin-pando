# 회원앱 운영기획서

## 1. 문서 목적

- 회원앱을 관리자 웹의 부속 화면이 아니라 실제 운영 채널로 정리한다.
- 회원, 트레이너, 골프강사, FC, 스태프가 같은 앱을 쓰더라도 역할별 화면 기준을 명확히 한다.

## 2. 앱의 역할

- 회원에게는 예약, 결제, 체크인, 이용권 확인, 알림 수신 채널
- 트레이너/강사에게는 수업 수행과 회원 관리 보조 도구
- FC에게는 만료예정 고객 후속관리 도구
- 스태프에게는 현장 체크인/안내 보조 도구

## 3. 역할별 핵심 메뉴

### 3.1 member

- 홈
- QR 체크인
- 수업 목록/예약
- 내 예약
- 이용권 조회
- 결제 이력
- 쿠폰/마일리지
- 공지/문의
- 설정

### 3.2 trainer

- 오늘 수업
- 담당 회원
- 수업 완료 처리
- 상담/메모
- 서명 처리

### 3.3 golf_trainer

- 오늘 레슨
- 타석/예약 확인
- 쌍방 서명 처리
- 회원별 레슨 이력

### 3.4 fc

- 만료예정 회원 큐
- 상담 대상 목록
- 재등록 제안
- 메시지 발송 이력 확인

### 3.5 staff

- 체크인 지원
- 공지 확인
- 문의 인입 확인
- 현장 운영 보조

## 4. 앱 운영 원칙

- 앱은 관리자 웹의 모든 기능을 복제하지 않는다.
- 회원이 스스로 처리해야 하는 흐름은 앱으로 내리고, 예외 처리와 승인 행위는 관리자 웹에 둔다.
- 앱에서 보이는 상태 용어는 운영 친화적 한글 라벨을 사용한다.
- 회원앱에서 관리자 전용 용어와 상태값을 직접 노출하지 않는다.

## 5. 관리자 웹과의 연결 기준

- 회원 기본정보, 이용권 상태, 결제 이력, 예약 현황은 관리자 웹과 같은 기준 데이터를 본다.
- 앱에서 발생한 예약, 취소, 체크인, 서명 결과는 관리자 웹 상세 화면에 바로 반영되는 구조를 전제로 한다.
- 앱 알림함은 자동화 발송과 공지 발송 이력을 함께 보여줄 수 있어야 한다.

## 6. 핵심 운영 시나리오

### 6.1 회원

- 앱 가입
- 이용권 확인
- 수업 예약
- 방문 전 알림 수신
- QR 체크인
- 수업 완료 후 후기 또는 다음 예약

### 6.2 트레이너/강사

- 오늘 수업 확인
- 회원 출석 확인
- 수업 완료/서명 처리
- 메모 작성

### 6.3 FC

- 만료예정 알림 대상 확인
- 상담 이력 확인
- 재등록 유도 메시지 발송
- 결제 전환 확인

## 7. 앱에서 꼭 필요한 관리자 연계

- 만료예정 회원 CTA
  - 재등록 문의
  - 상담 예약
  - 상품 보기
- 예약 확정/변경/취소 알림
- 회원별 출석/예약/결제 요약
- 공지 및 1:1 문의

## 8. 운영 예외 처리

- 지점에서 사용하지 않는 모듈은 앱 메뉴에서 숨긴다.
- 회원 권한으로는 관리자용 상담 기록 전체를 보지 않는다.
- 만료된 이용권이 있는 회원도 공지, 문의, 프로필 기능은 사용할 수 있다.
- 앱 결제가 열리지 않은 지점은 상품 보기까지만 허용할 수 있다.

## 9. 퍼블리싱 기준 메모

- 앱은 관리자 웹보다 한 단계 더 단순해야 한다.
- 홈 화면은 역할별로 정보 구조가 달라져야 한다.
- 회원 홈은 예약/이용권/알림 우선, 현장 인력 홈은 오늘 할 일 우선으로 본다.

## 10. 운영 결론

- 회원앱은 CRM의 조회용 미러가 아니라 운영 채널이다.
- 특히 만료예정, 예약, 체크인, 서명은 앱과 관리자 웹이 하나의 흐름으로 이어져야 한다.
