# FN-010~012: 매출관리

## FN-010: 매출 조회/통계

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-010 |
| **기능명** | 매출 조회/통계 |
| **분류** | 매출관리 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-030 |
| **관련 API ID** | API-015 |

### 기능 설명
매출 현황을 조회하고 통계를 제공합니다. 기간별, 유형별, 상태별 필터로 유연한 조회를 지원하며, 엑셀 다운로드 기능을 제공합니다.

### 상세 요구사항
1. 기간 필터 (프리셋: 오늘/이번주/이번달/커스텀)
2. 유형 필터 (이용권/PT/GX/상품/기타 - 다중 선택)
3. 상태 필터 (완료/환불/미납)
4. 통계 카드: 총매출/현금/카드/마일리지
5. 매출 테이블: 날짜/상품명/유형/금액/상태/결제수단
6. 환불 관리: 환불 사유 입력 후 처리
7. 엑셀 다운로드
8. 페이지네이션

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 기간필터 | String | 선택 | "today" \| "week" \| "month" \| "custom" (기본값: "today") |
| 시작일 | Date | 조건부 | custom 선택 시 필수 |
| 종료일 | Date | 조건부 | custom 선택 시 필수 |
| 유형 | String[] | 선택 | "이용권" \| "PT" \| "GX" \| "상품" \| "기타" |
| 상태 | String[] | 선택 | "완료" \| "환불" \| "미납" |
| 결제수단 | String[] | 선택 | "현금" \| "카드" \| "마일리지" \| "복합" |
| 페이지 | Number | 선택 | 기본값: 1 |
| 페이지당건수 | Number | 선택 | 20 \| 50 \| 100 (기본값: 20) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 총 매출 | Number | 환불액 제외한 순 매출 |
| 현금 | Number | 현금 결제 금액 |
| 카드 | Number | 카드 결제 금액 |
| 마일리지 | Number | 마일리지 사용 금액 |
| 환불 | Number | 환불된 총액 |
| 매출 목록 | Array | [{id, date, product, type, amount, status, payment_method}, ...] |
| 총 건수 | Number | 필터 적용된 전체 매출 건수 |

### 비즈니스 규칙
1. branch_id 기준 데이터 격리
2. 환불 건은 매출에서 차감 (음수로 표시)
3. 천단위 콤마 포맷 (1,234,567)
4. 미납 상태 건은 총매출에 미포함
5. 결제 시간 기준 (결제 완료일)
6. 환불 시 환불 사유 반드시 기록

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 데이터 없음 | "해당 기간의 매출이 없습니다" 빈 상태 UI |
| 다운로드 실패 | "다운로드에 실패했습니다. 다시 시도해주세요" + 재시도 버튼 |
| 권한 없음 | "조회 권한이 없습니다" |
| 서버 오류 | "데이터를 불러올 수 없습니다. 새로고침을 시도해주세요" |

### Web 구현 사항
- **테이블**: TanStack Table
- **필터**: DatePicker (기간), MultiSelect (유형/상태)
- **통계 카드**: 큰 숫자 표시 + 색상 구분
- **엑셀 내보내기**: xlsx 라이브러리
- **페이지네이션**: 기본 구현
- **데이터 페칭**: React Query

## FN-011: POS 상품 선택/결제 카트

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-011 |
| **기능명** | POS 상품 선택/결제 |
| **분류** | 매출관리 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-030 |
| **관련 API ID** | API-018 |

### 기능 설명
POS 시스템에서 상품을 카테고리별로 표시하고, 선택하여 장바구니에 추가합니다. 수량 조절, 삭제, 합계 자동 계산을 제공합니다.

### 상세 요구사항
1. 카테고리별 상품 그리드 (이용권/PT/GX/상품)
2. 상품 클릭 → 장바구니 추가
3. 장바구니: 상품명/수량/단가/소계 테이블
4. 수량 조절 (+-버튼)
5. 상품 삭제 (X 버튼)
6. 합계 자동 계산
7. 장바구니 비어있으면 결제 버튼 비활성화
8. 할인 입력 (선택사항)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 상품ID | UUID | 필수 | 상품 클릭 시 |
| 수량 | Number | 필수 | 기본값: 1, 최소 1 |
| 할인금액 | Number | 선택 | 0 이상 |
| 할인률 | Number | 선택 | 0~100 (%) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 장바구니 목록 | Array | [{product_id, name, quantity, unit_price, subtotal}, ...] |
| 소계 | Number | 모든 상품의 합 |
| 할인액 | Number | 적용된 할인액 |
| 총액 | Number | 소계 - 할인액 |

### 비즈니스 규칙
1. 판매중 상품만 표시 (상태 = "판매중")
2. 장바구니 비어있으면 결제 버튼 비활성화
3. 동일 상품 재추가 시 수량 증가 (새로운 항목 추가 아님)
4. 할인: 금액 또는 율 중 하나만 적용 (동시 적용 불가)
5. 할인액 최대값: 소계의 100% (전액 할인 가능하나 주의 안내)

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 재고 없는 상품 | 상품 아이콘에 "품절" 배지 + 클릭 비활성화 |
| 수량 0 입력 | 수량 필드에 "1 이상을 입력하세요" 에러 |
| 음수 입력 | 숫자 입력 필드로 제한 (음수 자동 제거) |
| 할인 초과 | "할인액이 총액을 초과할 수 없습니다" 경고 |

### Web 구현 사항
- **상품 그리드**: CSS Grid 또는 React Grid Layout
- **카테고리 탭**: 탭 컴포넌트
- **장바구니 상태**: Zustand (전역 상태)
- **수량 조절**: Number Input + +-버튼
- **합계 계산**: useMemo로 최적화
- **아이콘**: product-with-discount, sold-out 배지

## FN-012: POS 결제 처리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-012 |
| **기능명** | POS 결제 처리 |
| **분류** | 매출관리 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-032 |
| **관련 API ID** | API-016 |

### 기능 설명
장바구니의 상품을 실제로 결제 처리합니다. 결제 수단 선택, 회원 검색, 마일리지 적용, 영수증 발급을 제공합니다.

### 상세 요구사항
1. 결제 수단 선택 (현금/카드/마일리지/복합)
2. 회원 검색 → 마일리지 잔액 자동 표시
3. 마일리지 사용 금액 입력 (잔액 이상 불가)
4. 복합 결제: 현금 + 마일리지, 카드 + 마일리지 등 조합
5. 결제 확인 모달: 최종 금액/수단/회원 확인
6. 결제 처리 API 호출
7. 영수증 발급 (인쇄/이메일/다운로드)
8. 결제 완료 후 매출 자동 등록
9. 거스름돈 자동 계산 (현금 결제 시)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 결제수단 | String | 필수 | "현금" \| "카드" \| "마일리지" \| "복합" |
| 회원ID | UUID | 조건부 | 마일리지 사용 시 필수 |
| 현금결제액 | Number | 조건부 | 복합 결제 시 필수 |
| 카드결제액 | Number | 조건부 | 복합 결제 시 필수 |
| 마일리지사용액 | Number | 조건부 | "마일리지" 또는 복합 결제 시 |
| 현금수령액 | Number | 조건부 | 현금 결제 시 필수 (거스름돈 계산) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 결제 결과 | Object | {success: true/false, transaction_id, message} |
| 매출 정보 | Object | {amount, payment_method, member_id, created_at} |
| 영수증 | Object | {receipt_number, items, total, member, timestamp} |
| 거스름돈 | Number | 현금 결제 시 (현금수령액 - 총액) |

### 비즈니스 규칙
1. **복합결제 검증**: 현금 + 마일리지 = 총액 (정확히 일치)
2. **마일리지 제약**:
   - 마일리지 사용액 <= 보유 잔액
   - 부족 시 "마일리지 잔액이 부족합니다. 부족액: {{amount}}" 안내
3. **결제 처리**:
   - 결제 완료 시 매출 테이블에 자동 추가
   - 마일리지 사용 시 회원의 마일리지 잔액 차감
4. **영수증**:
   - 영수증 번호: TRX-{{timestamp}}-{{random}}
   - 항상 발급 (인쇄 또는 이메일)
5. **거스름돈**: 실시간 계산 표시

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 결제 실패 | "결제 처리에 실패했습니다. 다시 시도해주세요" + 재시도 버튼 |
| 마일리지 부족 | "마일리지 잔액이 부족합니다. 부족액: {{amount}}" + 다른 수단 제안 |
| 복합결제 금액 불일치 | "결제 금액이 총액과 맞지 않습니다. 다시 확인해주세요" |
| 회원 미존재 | "존재하지 않는 회원입니다" |
| 현금 거스름돈 | "거스름돈: {{amount}}" 명확하게 표시 |
| 네트워크 오류 | "결제 처리 중 오류가 발생했습니다. 연결을 확인해주세요" |

### Web 구현 사항
- **결제 수단 선택**: Radio Button 또는 Button Group
- **회원 검색**: Combobox (검색 가능, 마일리지 표시)
- **금액 입력**: Number Input
- **확인 모달**: 최종 검토 화면
- **거스름돈**: useMemo 실시간 계산
- **영수증**: 별도 컴포넌트 (인쇄 레이아웃)
- **결제 API**: 비동기 처리, 로딩 인디케이터
- **상태 관리**: Zustand (결제 진행 상태)

## 매출관리 공통 사항

### 권한 검증
| 기능 | 센터장 | PT담당자 | 직원 | 조회전용 |
|------|--------|---------|------|---------|
| FN-010 조회 | O | O | O | O |
| FN-010 환불 처리 | O | X | O | X |
| FN-011 상품 선택 | O | O | O | X |
| FN-012 결제 처리 | O | O | O | X |

### 데이터 격리
- 모든 매출 데이터는 branch_id 기준
- 다지점 관리자도 조회만 가능 (처리는 해당 지점에서만)

### 공통 기술 스택
- **테이블**: TanStack Table
- **필터**: MultiSelect, DatePicker
- **상태 관리**: Zustand (장바구니, 결제 상태)
- **데이터 페칭**: React Query
- **결제 처리**: 비동기 API 호출 (try-catch)
- **UI**: 커스텀 컴포넌트

### 결제 보안 고려사항
- 결제 API는 HTTPS만 사용
- 민감 정보(카드번호)는 클라이언트에서 처리하지 않음
- 결제 게이트웨이와 직접 연동 (PCI-DSS 준수)
- 모든 결제 시도 로깅
