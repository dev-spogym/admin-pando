# FN-029~033: 설정 및 권한 관리

## FN-029: 시스템 설정

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-029 |
| **기능명** | 시스템 설정 |
| **분류** | 설정 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-090 |
| **관련 API ID** | API-030 (미정) |

### 기능 설명
센터의 기본 정보와 시스템 설정을 관리합니다. 센터명, 영업시간, 연락처 등 기본 정보 및 알림 기본값을 설정합니다.

### 상세 요구사항
1. 센터 기본 정보 (센터명, 코드, 주소, 연락처)
2. 영업시간 설정 (요일별)
3. 휴무일 설정
4. 알림 기본값 (SMS 수신 동의 등)
5. 테마 설정 (색상, 로고)
6. 설정 변경 즉시 반영
7. 변경 이력 기록

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 센터명 | String | 필수 | 1~100자 |
| 센터코드 | String | 필수 | 영문/숫자, 중복 불가 |
| 주소 | String | 필수 | 도로명 주소 |
| 연락처 | String | 필수 | 전화번호 |
| 영업시간 | Object | 필수 | {월~일: {시작: "HH:MM", 종료: "HH:MM"}} |
| 휴무일 | Array | 선택 | 요일 또는 날짜 |
| SMS동의 | Boolean | 선택 | 기본값: true |
| 로고 | File | 선택 | 이미지 파일 (JPG/PNG, 5MB 이하) |
| 주요색상 | String | 선택 | Hex 컬러 코드 (기본값: #3B82F6) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 설정 정보 | Object | 모든 설정값 저장됨 |
| 변경 이력 | Array | [{changed_by, changed_at, change_details}, ...] |

### 비즈니스 규칙
1. 설정 변경 즉시 반영 (재로그인 불필요)
2. 센터코드는 변경 불가 (초기 생성 시만)
3. 변경 이력 자동 기록 (변경자, 변경 시간, 변경 내용)
4. 로고 이미지: 5MB 이하, JPG/PNG만 허용
5. 영업시간: 요일별로 다르게 설정 가능

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 필수 미입력 | 필드별 에러 메시지 |
| 센터코드 중복 | "이미 사용 중인 센터코드입니다" |
| 로고 용량 초과 | "파일 크기가 5MB를 초과합니다" |
| 로고 형식 오류 | "JPG, PNG 형식만 지원됩니다" |
| 저장 실패 | "설정 저장에 실패했습니다. 다시 시도해주세요" |

### Web 구현 사항
- **레이아웃**: 탭 또는 좌측 메뉴 (기본정보/영업시간/알림/테마)
- **기본정보**: Form (텍스트 입력, 연락처)
- **영업시간**: 요일별 시간 선택 (TimePicker)
- **휴무일**: 캘린더 (클릭으로 선택)
- **로고 업로드**: FileUpload + 미리보기
- **색상 선택**: Color Picker
- **저장 버튼**: 변경된 항목만 POST
- **변경 이력**: 테이블 (변경자, 변경 시간, 내용)

## FN-030: 권한 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-030 |
| **기능명** | 권한 관리 |
| **분류** | 설정 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-090 |
| **관련 API ID** | API-031 (미정) |

### 기능 설명
역할별 메뉴/기능 권한을 세밀하게 설정합니다. 6단계의 역할 계층과 메뉴별/기능별 접근권 매트릭스를 제공합니다.

### 상세 요구사항
1. 6단계 역할 정의 (최고관리자 > 센터장 > 트레이너 > 스태프 > 조회전용 > 손님)
2. 메뉴별 접근 권한 (접근 가능/불가)
3. 기능별 읽기/쓰기/삭제 권한
4. 역할별 기본 권한 템플릿
5. 커스텀 역할 생성 (선택사항)
6. 변경 이력 로그
7. 권한 충돌 감지 및 경고

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 역할ID | UUID | 필수 | 시스템 정의 역할 또는 커스텀 |
| 메뉴 | String[] | 필수 | 메뉴 목록 (체크박스) |
| 읽기권한 | Boolean[] | 필수 | 각 메뉴별 읽기 권한 |
| 쓰기권한 | Boolean[] | 필수 | 각 메뉴별 쓰기 권한 |
| 삭제권한 | Boolean[] | 필수 | 각 메뉴별 삭제 권한 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 권한 설정 | Object | 역할별 권한 매트릭스 |
| 변경 이력 | Array | [{role, changed_by, changed_at, changes}, ...] |

### 역할 정의

| 역할 | 설명 | 권한 |
|------|------|------|
| **최고관리자** | 전사 최상위 관리자 | 모든 권한 (변경 불가) |
| **센터장** | 센터 운영 책임자 | 거의 모든 권한 (전사 설정 제외) |
| **트레이너** | PT/GX 담당 | 회원/일정/출석 관련만 |
| **스태프** | 일반 직원 | 회원/매출/상품 관련 |
| **조회전용** | 조회만 가능 | 모든 메뉴 읽기만 |
| **손님** | 앱 사용자 (선택) | 자신의 정보만 조회 |

### 비즈니스 규칙
1. **최고관리자 권한**: 변경 불가 (최고관리자 전용)
2. **권한 계층**: 하위 역할에 상위 권한 부여 불가
   - 예: PT담당자는 센터장 권한 부여 불가
3. **자기 자신 권한 축소**: 확인 모달 필수
   - "정말 자신의 권한을 축소하시겠습니까? 이후 복구가 필요할 수 있습니다"
4. **권한 충돌 감지**: 읽기 권한 없으면 쓰기/삭제 권한 자동 해제
5. **변경 이력**: 모든 권한 변경 기록

### 메뉴 매트릭스

| 메뉴 | 최고관리자 | 센터장 | 트레이너 | 스태프 | 조회전용 |
|------|-----------|--------|---------|--------|----------|
| 대시보드 | R/W/D | R/W | R | R/W | R |
| 회원관리 | R/W/D | R/W/D | R/W* | R/W/D | R |
| 출석관리 | R/W | R/W | R/W | R | R |
| 매출관리 | R/W/D | R/W/D | X | R/W/D | R |
| 상품관리 | R/W/D | R/W/D | X | R/W | R |
| 시설관리 | R/W/D | R/W/D | X | R/W | R |
| 직원관리 | R/W/D | R/W/D | X | X | R |
| 급여관리 | R/W/D | R/W | X | X | R |
| 메시지 | R/W | R/W | X | R/W | X |
| 설정 | R/W | R/W | X | X | X |

*자신이 담당한 회원만 쓰기 가능

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 읽기 없이 쓰기 선택 | 자동으로 읽기도 체크 |
| 최고관리자 권한 변경 | "최고관리자 권한은 변경할 수 없습니다" (비활성화) |
| 자신의 권한 축소 | "정말 자신의 권한을 축소하시겠습니까?" 확인 모달 |
| 권한 충돌 | "읽기 권한이 필요합니다. 읽기 권한을 먼저 부여하세요" 안내 |
| 저장 실패 | "권한 설정 저장에 실패했습니다. 다시 시도해주세요" |

### Web 구현 사항
- **역할 목록**: 좌측 메뉴 또는 탭 (6가지 역할)
- **권한 매트릭스**: 테이블 (메뉴 × R/W/D 체크박스)
- **권한 충돌 감지**: onChange 이벤트로 자동 체크 및 수정
- **자신의 권한**: 축소 시 확인 모달
- **변경 이력**: 테이블 (역할, 변경자, 변경 시간, 변경 내용)
- **미리보기**: 역할별 권한 요약 (읽기 전용)

## FN-031: 키오스크 설정

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-031 |
| **기능명** | 키오스크 설정 |
| **분류** | 설정 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-090 |
| **관련 API ID** | API-032 (미정) |

### 기능 설명
키오스크(터치 화면) 디스플레이와 동작을 설정합니다. 로고/배경 업로드, 동작 시나리오 설정, 미리보기를 제공합니다.

### 상세 요구사항
1. 로고 이미지 업로드
2. 배경 이미지 업로드
3. 스크린 색상 테마 설정
4. 동작 시나리오 (회원 인증 방식 설정)
5. 미리보기
6. 설정 적용 (실시간)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 로고 | File | 선택 | 이미지 (JPG/PNG, 5MB 이하) |
| 배경 | File | 선택 | 이미지 (JPG/PNG, 10MB 이하) |
| 주요색상 | String | 선택 | Hex 컬러 코드 |
| 인증방식 | String | 필수 | "RFID" \| "QR코드" \| "수동입력" |
| 자동로그아웃시간 | Number | 선택 | 초 단위 (기본값: 300초) |
| 음량 | Number | 선택 | 0~100 (%) |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 키오스크 설정 | Object | 모든 설정값 |

### 비즈니스 규칙
1. 이미지 파일: 5MB (로고) / 10MB (배경) 이하
2. 지원 형식: JPG, PNG만
3. 자동 로그아웃: 300~3600초 범위
4. 설정 변경 실시간 적용

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 로고 용량 초과 | "파일 크기가 5MB를 초과합니다" |
| 배경 용량 초과 | "파일 크기가 10MB를 초과합니다" |
| 형식 오류 | "JPG, PNG 형식만 지원됩니다" |
| 설정 적용 실패 | "설정 적용에 실패했습니다. 다시 시도해주세요" |

### Web 구현 사항
- **레이아웃**: 좌측 설정, 우측 미리보기 (반응형)
- **로고 업로드**: FileUpload + 미리보기 썸네일
- **배경 업로드**: FileUpload + 미리보기
- **색상 선택**: Color Picker
- **인증방식**: Select 또는 Radio
- **자동 로그아웃**: Slider (300~3600초)
- **음량**: Slider (0~100%)
- **미리보기**: 키오스크 화면 레이아웃 시뮬레이션

## FN-032: IoT 설정

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-032 |
| **기능명** | IoT 설정 |
| **분류** | 설정 |
| **플랫폼** | Web |
| **우선순위** | Medium |
| **상태** | 확정 |
| **화면 ID** | SCR-090 |
| **관련 API ID** | API-033 (미정) |

### 기능 설명
출입문, 체성분 기기 등 IoT 장비를 등록하고 연결 상태를 모니터링합니다.

### 상세 요구사항
1. 기기 등록 (출입문, 체성분계 등)
2. 기기 IP/포트 설정
3. 연결 상태 모니터링
4. 연결 테스트
5. 기기별 상태 아이콘 표시
6. 5분 주기 상태 갱신
7. 연결 실패 알림

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 기기명 | String | 필수 | 1~50자 |
| 기기유형 | String | 필수 | "출입문" \| "체성분계" \| "기타" |
| IP주소 | String | 필수 | IPv4 형식 (xxx.xxx.xxx.xxx) |
| 포트 | Number | 필수 | 1~65535 |
| 설명 | String | 선택 | 최대 100자 |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 기기 목록 | Array | [{id, name, type, ip, port, status, last_checked}, ...] |
| 연결 상태 | String | "연결" \| "대기" \| "끊김" \| "오류" |

### 비즈니스 규칙
1. IP/포트 유효성 검증 필수
2. 연결 상태 5분 주기 갱신
3. 연결 실패 시 자동 재시도 (3회)
4. 3회 실패 시 "끊김" 상태로 표시

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| IP 형식 오류 | "올바른 IP 주소를 입력해주세요 (예: 192.168.1.100)" |
| 포트 범위 오류 | "포트는 1~65535 사이의 숫자여야 합니다" |
| 연결 실패 | "기기 연결에 실패했습니다. IP/포트를 확인해주세요" |
| 테스트 실패 | "연결 테스트에 실패했습니다. 기기 전원을 확인해주세요" |

### Web 구현 사항
- **기기 목록**: DataTable (기기명, 유형, IP, 포트, 상태, 마지막 확인)
- **기기 등록**: 모달 (기기명, 유형, IP, 포트)
- **상태 아이콘**:
  - 연결: 녹색 ●
  - 대기: 노란색 ◐
  - 끊김: 빨강 ●
  - 오류: 검정 ●
- **연결 테스트**: 버튼 (클릭 시 즉시 ping 시도)
- **자동 갱신**: useEffect + 5분 주기 실행

## FN-033: 지점 관리

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-033 |
| **기능명** | 지점 관리 |
| **분류** | 설정 |
| **플랫폼** | Web |
| **우선순위** | Critical |
| **상태** | 확정 |
| **화면 ID** | SCR-090 |
| **관련 API ID** | API-027 |

### 기능 설명
다중 지점(멀티테넌트)을 관리합니다. 지점 정보를 CRUD하고, 데이터 격리를 보장하며, 개발 용도로 데이터 초기화 기능을 제공합니다.

### 상세 요구사항
1. 지점 추가 (이름/코드/주소/연락처)
2. 지점 수정
3. 지점 비활성화 (삭제 대신)
4. 지점별 데이터 완전 격리
5. 데이터 초기화 (개발/테스트용)
6. 지점 통계 (회원 수, 매출 등)

### 입력 항목

| 항목명 | 타입 | 필수여부 | 설명 |
|--------|------|---------|------|
| 지점명 | String | 필수 | 1~100자 |
| 지점코드 | String | 필수 | 영문/숫자, 동일 그룹 내 중복 불가 |
| 주소 | String | 필수 | 도로명 주소 |
| 연락처 | String | 필수 | 전화번호 |
| 상태 | String | 선택 | "활성" \| "비활성" (기본값: "활성") |

### 출력 항목

| 항목명 | 타입 | 설명 |
|--------|------|------|
| 지점 목록 | Array | [{id, name, code, address, phone, status, member_count, sales, created_at}, ...] |
| 지점 통계 | Object | {total_members, total_sales, avg_members_per_branch} |

### 비즈니스 규칙
1. **지점 코드**: 고유 (동일 그룹 내)
2. **데이터 격리**: 지점 A의 데이터는 지점 B에서 조회 불가
3. **삭제 대신 비활성화**: 데이터 보존 목적
4. **데이터 초기화**:
   - 개발/테스트용만 사용
   - 2중 확인: "정말 {{지점명}}의 모든 데이터를 삭제하시겠습니까?" + 지점코드 입력 재확인
   - 삭제 후 로그 기록

### 예외 처리

| 상황 | 대응 방법 |
|------|---------|
| 중복 코드 | "이미 사용 중인 지점코드입니다" |
| 필수 미입력 | 필드별 에러 메시지 |
| 데이터 존재 시 삭제 | "데이터가 존재하는 지점은 삭제할 수 없습니다. 초기화하세요" |
| 초기화 실패 | "데이터 초기화에 실패했습니다. 관리자에 문의하세요" |

### Web 구현 사항
- **지점 목록**: 카드 또는 테이블 (이름, 코드, 주소, 상태, 회원 수, 매출)
- **지점 추가**: 모달 (입력 폼)
- **지점 수정**: 모달 (기존 데이터 프리필)
- **상태 변경**: 토글 (활성/비활성)
- **데이터 초기화**: 버튼 (2중 확인 모달: 경고 메시지 + 지점코드 입력)
- **통계**: 카드 (총 지점, 총 회원, 평균 회원/지점, 총 매출)
- **권한**: 최고관리자만 접근 가능
