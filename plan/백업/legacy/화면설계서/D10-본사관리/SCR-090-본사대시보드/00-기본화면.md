## 1. 화면 목적 (Why)

하나의 **지점** 운영 상태를 한눈에 볼 수 있는 **역할별 통합 홈**.
- 로그인 후 첫 화면으로, 사용자의 역할에 따라 위젯 구성·데이터 범위·액션 권한이 다르게 렌더된다.
- 회원/매출/출석/만료/홀딩/감사로그를 종합 표시.
- 멀티테넌트: `branchId` 컨텍스트로 필터링 (본사 사용자는 지점 전환 드롭다운으로 branchId 교체 가능).

| `POST /system/daily-sync` | POST | `{branchId}` | 새로고침 시 실행, EXPIRED 자동 전환 |

**권한별 API 호출 결정**:
- primary/super: `branchId` 제외(전 지점) 또는 선택된 branchId
- owner/manager/fc/staff/front/trainer: `branchId = user.branchId` (서버 강제)
- 서버는 요청 jwt의 role과 branchId로 데이터 스코프 강제. 클라이언트 필터는 UX용.

### 7.3 상태 관리
- **Store**: `useAuthStore` (user/role/branchId), `useBranchStore`(branches 리스트)
- **Fetching**: React Query (또는 Next 15 `fetch` + `use`). 병렬 Query, 각 위젯 독립 loading/error
- **Cache**: `staleTime: 60_000`, `refetchOnWindowFocus: true`
- **Polling (옵션)**: 오늘 출석 카드만 60초 주기

### 7.4 멀티테넌트 branchId 규칙
1. URL에 `?branch=<id>`가 있고 접근 권한이 있으면 우선
2. `useBranchStore.current` 값
3. `user.branchId` (단일 지점 역할의 경우)
4. super/primary가 아직 지점 선택 안 했으면: 전체(`null`) 또는 기본 지점 제안 UI

## 8. 비즈니스 룰

### 8.1 공통
1. **로그인 후 리다이렉트**: 역할에 따라 `/`, `/super-dashboard`, `/calendar`, `/attendance` 분기 (SCR-100 §7).
2. **일일 동기화**: 진입 시 1회 `runDailySync()` 호출(세션당 1회 guard). 이용권 만료/임박 상태 자동 전환.
3. **매출 포맷**: 1억 이상 `"1.2억"`, 1천만 이상 `"N,NNN만"`, 그 외 `"N,NNN,NNN원"` (utils `formatAmount`).
4. **숫자 포맷**: `Intl.NumberFormat('ko-KR')` + 단위 suffix ("명", "건").
5. **갱신 시간 표시**: `마지막 갱신: HH:mm` KST.
6. **새로고침**: 전체 병렬 refetch + `runDailySync`. 60초 쿨다운(스팸 방지).

### 8.2 멀티테넌트/권한
7. `canSwitchBranch(role)` false인 사용자에게 branchSwitcher 미렌더.
8. super/primary가 지점 전환하면 전역 `branchId` 변경 + URL `?branch=` 갱신 + 위젯 일괄 refetch + toast "강남점으로 전환됨".
9. URL 조작으로 다른 지점 branchId 쿼리 시, 서버가 권한 없으면 403 → `/forbidden` 리다이렉트.
10. **readonly** 역할은 위젯 항목 클릭 비활성 (cursor-default, no hover). 클릭 시 toast "읽기 전용 계정입니다".

### 8.3 위젯
11. 만료 예정은 `오늘 ≤ membershipExpiry ≤ 오늘+30일 AND status='ACTIVE'` 기준.
12. 생일자는 `MM-DD` 일치 + 활성 상태.
13. 홀딩은 `status='HOLDING'` + 남은 기간(일) 계산.
14. 미수금은 `unpaidAmount > 0 AND overdueDays ≥ 1`.
15. 위젯 상한: 각 5건(생일자/미수금/홀딩/만료예정), "전체보기" 클릭 시 필터링된 회원 목록으로 이동.

### 8.4 감사로그
16. 최신 10건만 표시. 클릭 시 `/audit-log?filter=...` 이동.
17. `primary/super`는 전 지점·전 이벤트, `owner` 이하는 본인 지점·일부 이벤트.

### 8.5 접근성 / i18n
18. 모든 위젯에 `aria-label` 필수.
19. 차트 위젯은 `aria-description` 또는 `<table class="sr-only">`로 데이터 제공.
20. 현재는 ko-KR, 금액은 KRW `₩` 프리픽스.
