## 상태 진입 조건
- `navigator.onLine === false`
- 또는 fetch 연속 실패 + 네트워크 오류 (fetch reject with TypeError "Failed to fetch")
- `window.online` / `offline` 이벤트

## 역할별 차이 (거의 없음)
오프라인은 역할과 무관하게 동일 UI. 다만 super/primary의 지점 전환, owner의 매출 데이터 접근 등 고급 기능 차단 안내 차이 미미.

## 상태 고유 인터랙션
1. **"다시 시도"** 버튼: `fetch('/api/health')` 호출 — 성공 시 전체 refetch + 이 상태 해제
2. **자동 복구**: `window.online` 이벤트 리스너 → 자동 refetch
3. **사이드바 메뉴 클릭**: 페이지 이동은 시도 가능하지만 데이터는 못 불러옴 (각 페이지가 자체적으로 오프라인 처리)
4. **Esc**: 배너 닫기 불가 (중요 상태)

## 상태 고유 비즈니스 룰 델타
- **Online/Offline 이벤트**: `window.addEventListener('online', ...)` / `offline`
- **Navigator API**: `navigator.onLine` 주기적 체크 (30초 간격)
- **Health Check**: `fetch('/api/health', { cache: 'no-store' })` 5초 타임아웃
- **캐시 fallback**: Service Worker가 있으면 `05-오프라인.md`로 분기 (stale 표시)
- **감지 신뢰성**: `navigator.onLine`은 불완전 → 실제 fetch 시도로 확인
- **재시도 쿨다운**: 10초 (네트워크 복구 중 과도 요청 방지)

## 에지 케이스
- **Wi-Fi 연결되어 있지만 서버 다운**: 배너는 "일시적 연결 문제"로 변경, 서버 헬스체크 계속 시도
- **VPN/프록시로 일부만 끊김**: 핑은 성공하지만 API 실패 → `04-에러-API`로 분류(오프라인 아님)
- **짧은 끊김 (1~2초)**: 배너 띄우지 않고 조용히 재시도 (debounce 2초)
- **모바일 데이터 off**: `offline` 이벤트 즉시 발생
- **성공한 쿼리 일부 보관**: 기존 화면 유지 + 스티키 배너만 추가

function useOffline() {
  const [offline, setOffline] = useState(!navigator.onLine);
  useEffect(() => {
    const on = () => setOffline(false);
    const off = () => setOffline(true);
    window.addEventListener('online', on);
    window.addEventListener('offline', off);
    return () => {
      window.removeEventListener('online', on);
      window.removeEventListener('offline', off);
    };
  }, []);
  return offline;
}

async function tryReconnect() {
  setIsChecking(true);

    const res = await fetch('/api/health', { cache: 'no-store', signal: AbortSignal.timeout(5000) });
    if (res.ok) {
      setOffline(false);

      refetchAll();

}
