### 변경 요소
| 요소 | 변경 |
|---|---|
| 각 KpiCard | `KpiCardSkeleton` (h-24) |
| 게이지 | `<div className="h-3 bg-gray-200 animate-pulse"/>` |
| 코호트 테이블 | 6행 × 7열 skeleton cells |
| 퍼널 | 5단계 bar skeleton (너비 변화) |
| 새로고침 아이콘 | `animate-spin` |
| 목표 설정 버튼 | disabled (primary/owner만 원래 활성) |

## 역할별 차이
- primary/owner: §B 매출 skeleton 포함
- manager: §B 매출 skeleton은 렌더하되 목표 설정 버튼 skeleton 없음

## 인터랙션
- 새로고침 버튼 60초 쿨다운 적용 (이 상태에선 disabled)
- 탭/스크롤 가능하나 값은 모두 skeleton

## 비즈니스 룰 델타
- 18개 쿼리 중 1개라도 pending → 전체 loading 처리 (단순화)
- **빠른 경로**: React Query `placeholderData` 있으면 stale 데이터 + 우상단 작은 스피너
- 15초 초과 → `05-에러` 전이 + toast "KPI 집계가 오래 걸리고 있습니다"

## 에지 케이스
- 일부 쿼리만 pending (나머지 캐시 hit): stale 데이터 + 부분 spinner
- 새로고침 직후: 기존 데이터 유지 + isRefetching 표시
- Supabase 동시 18개 쿼리 rate limit: 자동 backoff

## 바이브코딩 프롬프트 (델타)
```
const queries = useKpiMetricsQueries(branchId);
const isAllLoading = queries.every(q => q.isLoading && !q.data);
const isAnyLoading = queries.some(q => q.isLoading);
const hasCache = queries.every(q => q.data !== undefined);

if (isAllLoading) {
  return (
    <>
      <StatCardGridSkeleton cols={6}/>
      <StatCardGridSkeleton cols={4}/>
      <GaugeSkeleton/>
      <StatCardGridSkeleton cols={4}/>
      <StatCardGridSkeleton cols={4}/>
      <StatCardGridSkeleton cols={4}/>
      <CohortSkeleton rows={6} cols={7}/>
      <FunnelSkeleton stages={5}/>
    </>
  );
}

// stale + 부분 refetch
{isAnyLoading && !isAllLoading && (
  <div className="absolute top-4 right-4">
    <Loader2 className="h-4 w-4 animate-spin text-gray-400"/>
  </div>
)}

// 15초 타이머
useEffect(() => {
  const t = setTimeout(() => {
    if (isAllLoading) {
      toast.warning('KPI 집계가 오래 걸리고 있습니다');
      setErrorState(true);
    }
  }, 15_000);
  return () => clearTimeout(t);
}, [isAllLoading]);
```
