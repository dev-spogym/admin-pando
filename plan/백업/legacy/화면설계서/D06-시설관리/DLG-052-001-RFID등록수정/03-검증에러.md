# DLG-052-001 RFID 등록/수정 — 상태 03: 검증 에러

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.

## 메타
| 항목 | 값 |
|------|----|
| 상태 코드 | `rfid-validation-error` |
| 상태 ID | STATE_VALIDATION_ERR |
| 다이어그램 노드 | `M2_필드검증.md` → ErrCard / ErrDup / ErrType |
| 이전 상태 | `01-열림-등록모드` / `02-열림-수정모드` (저장 버튼 클릭) |
| 다음 상태 | 이전 열림 상태(재입력 후 재저장) / `04-성공` |

## 상태 진입 조건
- 클라이언트 Zod 검증 실패 (형식 위반, 필수 누락)
- 서버 응답 `409` (카드번호 중복) 또는 `422` (형식 오류)
- `GET /rfid?cardNo=...` 사전 체크 충돌

## 상태 고유 비주얼 델타
- 다이얼로그는 **열린 상태 그대로 유지** (닫히지 않음)
- 각 실패 필드: `border-rose-300 focus:ring-rose-500` + `aria-invalid=true`
- 필드 하단: `text-xs text-rose-600` 메시지
- 상단에 전역 에러 배너(다중 에러 시): `role="alert"` bg-rose-50 border-rose-200
- 저장 버튼: idle 상태 복귀 (`isSubmitting=false`), 재시도 가능
- 첫 에러 필드로 포커스 자동 이동

## 상태 고유 인터랙션
1. **저장 클릭 직후**: 에러 발생 → `setTimeout(() => focusFirstError(), 0)`
2. **필드 수정**: 타이핑 시 해당 필드 에러 즉시 클리어(`clearErrors('cardNo')`)
3. **재저장**: 버튼 클릭 시 전체 재검증
4. **Enter 키**: 에러 필드에서 Enter → submit 재시도

## 역할별 차이
- 없음. 역할별 검증 규칙은 이미 RBAC에서 처리(진입 자체가 차단됨).

## 상태 고유 비즈니스 룰 델타

### 필드별 에러 메시지

| 필드 | 조건 | errorCode | 메시지 |
|---|---|---|---|
| cardNo | 미입력 | CLIENT_REQUIRED | 카드 번호를 입력하세요 |
| cardNo | 형식 위반 | E422052 | 형식: RF-00000000 |
| cardNo | 중복 | E409052 | 이미 등록된 카드번호입니다 |
| userType | 미선택 | CLIENT_REQUIRED | 사용자 유형을 선택하세요 |
| memberId | member 모드 미선택 | CLIENT_REQUIRED | 회원을 선택하세요 |
| staffId | staff 모드 미선택 | CLIENT_REQUIRED | 직원을 선택하세요 |
| lockerNo | 형식 위반 | E422053 | 사물함 번호는 10자 이내 (예: A-102) |

### 전역 배너 노출 조건
- 네트워크 에러(NETWORK), 500 에러 → 필드 인라인 대신 상단 배너 `"일시적인 오류가 발생했습니다. 재시도하세요."`

### 자동 복구
- 중복 에러(409) 수신 시 `cardNo` 필드 자동 포커스 + 텍스트 selectAll
- 재스캔 시 에러 자동 클리어

## 에지 케이스
- **다중 에러**: 여러 필드 에러 시 첫 번째 에러로만 포커스 이동(스크린리더 혼란 방지)
- **서버 검증만 실패**: 클라이언트 Zod는 통과했지만 서버 비즈니스 검증 실패 → `errorCode` 기반 필드 매핑
- **타이핑 지연 후 clearErrors**: debounce 200ms — 너무 빠른 클리어 방지
- **AutoComplete 미매칭**: 유효하지 않은 회원 ID 직접 주입 시 422 → 회원 필드 에러

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: rfid-validation-error

에러 매핑 테이블:
const FIELD_ERROR_MAP: Record<string, keyof CardForm> = {
  E422052: 'cardNo',
  E409052: 'cardNo',
  E422053: 'lockerNo',
};

onError:
onError: (e: any) => {
  const field = FIELD_ERROR_MAP[e.errorCode];
  if (field) {
    setError(field, { message: ERROR_MESSAGES[e.errorCode] });
    setTimeout(() => document.getElementById(field)?.focus(), 0);
  } else {
    setGlobalError(e.message || '일시적인 오류가 발생했습니다');
  }
}

타이핑 시 자동 클리어:
<input
  {...register('cardNo', {
    onChange: debounce(() => clearErrors('cardNo'), 200),
  })}
  aria-invalid={!!errors.cardNo}
  aria-describedby={errors.cardNo ? 'err-cardNo' : undefined}
/>
{errors.cardNo && <p id="err-cardNo" role="alert" className="text-xs text-rose-600">{errors.cardNo.message}</p>}

전역 에러 배너:
{globalError && (
  <div role="alert" aria-live="assertive" className="rounded-md bg-rose-50 border border-rose-200 p-3 text-xs text-rose-700 flex items-start gap-2">
    <AlertCircle className="size-3.5 shrink-0 mt-0.5" />
    <span>{globalError}</span>
  </div>
)}
```
