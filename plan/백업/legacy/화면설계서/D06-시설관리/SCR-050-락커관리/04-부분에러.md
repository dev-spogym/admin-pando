## 상태 진입 조건

여러 병렬 API 중 **일부만 실패**:
- `GET /lockers` 성공 + `GET /members?status=ACTIVE` 실패 → 배정 모달 회원 목록만 장애
- IoT 폴링 `/iot/locker-status` 실패 → 오프라인 배지 갱신 불가
- `GET /lockers/:id/history` 실패 → 기록 모달만 에러
- `POST /lockers/:id/assign` 등 액션 실패 → 해당 액션만 실패 (그리드 유지)


## 역할별 차이

| 역할 | 배너 표시 |
|---|---|
| super/primary/owner/manager | 상세 failed queries 목록 노출 |
| staff | 간단한 "일시적 오류" 메시지만 |
| front | 배너 자동 숨김 (액션 불가 역할) |

## 상태 고유 인터랙션

- **[재시도] 버튼**: 실패한 쿼리만 재시도 (`queryClient.refetchQueries(failedKeys)`)
- **[×닫기]**: `dismissed=true` → 세션 중 배너 숨김 (단 새 실패 발생 시 재표시)
- **정상 그리드 액션**: 그대로 가능. 단, 의존 API(members 등)가 필요한 액션은 모달 내부 에러로 표시

## 비즈니스 룰 델타

- 부분 에러 감지 로직: React Query `useQueries` 결과에서 `errors.filter(e=>e!=null)`
- 배너 자동 표시: `errors.length > 0 && !dismissed`
- 3회 재시도 실패 시 → `05-에러` 승격 제안 toast "전체 새로고침을 시도하세요"
- 주요 쿼리(/lockers) 실패 시 → 즉시 `05-에러` (부분 아님)

## 에지 케이스

| 상황 | 처리 |
|---|---|
| 액션 중(예: 배정) 401 발생 | 전역 인터셉터 → `/login` 리다이렉트 |
| 액션 중 403 발생 | toast "권한이 없습니다" + 상태 유지 |
| 액션 중 409 (중복) | toast 해당 에러 메시지 + 모달 내 인라인 에러 |
| 액션 중 500 | 모달 내 에러 표시, 다시 시도 가능 |
| 새로고침 중 또 실패 | 배너 유지, 재시도 카운터 증가 |
| 쿠폰/다른 도메인 에러 전파 | 본 화면 영향 없음 (격리) |
