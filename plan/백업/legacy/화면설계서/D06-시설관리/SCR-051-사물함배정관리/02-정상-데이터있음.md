## 📝 디스크립션

### 사용 시점 (Trigger)
- API 200 OK + lockers.length > 0

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 사물함 | `lockers` | `branch_id = ?` |
| 회원 | `members` | `id IN (locker.member_id)` |

### 인터랙션 (User Actions)
1. available 셀 클릭 → 배정 모달 오픈
2. in_use 셀 클릭 → 우측 패널에 상세 표시
3. overtime 셀 → AnimatedTriangle 경고 + 클릭 시 회수 모달
4. abnormal 셀 → 클릭 비활성 (not-allowed cursor)
5. disabled 셀 → 클릭 비활성

### 비즈니스 룰
- overtime: 만료일 초과 상태 (expires_at < today)
- abnormal: 비정상 상태 (RLS/물리적 이상)
- disabled: 관리자가 임시 비활성화한 사물함

### 에지 케이스
- overtime 셀 5개 이상: 상단에 "초과 사물함 N개" 경고 배너
- abnormal 셀: 우측 패널에 이상 사유 표시

### 접근성 (A11y)
- 각 셀 `role="button"` + `aria-label="사물함 {번호} - {상태}"`
- overtime `aria-label="사물함 {번호} - 기간 초과, 확인 필요"`
