# DLG-M022 수동 출석 등록 — 상태 02: 입력 중 (프리플라이트)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.
> 🔸 본 DLG는 입력 필드가 없으므로 "입력 중"은 **클릭 직후~서버 호출 직전의 매우 짧은 프리플라이트 단계**로 정의한다.

## 메타

| 항목 | 값 |
|------|----|
| 상태 코드 | `manual-attend-preflight` |
| 상태 ID | STATE_PREFLIGHT |
| 다이어그램 노드 | `M1_생명주기.md` → STATE_PREFLIGHT |
| 이전 상태 | `01-열림` (idle 버튼 클릭) |
| 다음 상태 | `03-제출중` (가드 통과) / `01-열림` (가드 실패 즉시 복귀) |

## 상태 진입 조건

- 사용자가 idle 버튼을 클릭한 직후(수십 ms)
- React 상태 전이 단계 (아직 `mut.mutate()` 호출 전)

## 상태 고유 비주얼 델타

- 버튼 `active:bg-emerald-700` pressed 시각
- 아이콘/텍스트 변경 없음 (즉시 `03-제출중`으로 진입)

## 상태 고유 인터랙션

1. **최종 가드 체크**: 클릭 이벤트 핸들러에서 `disabled || mut.isPending` 확인 → 차단 시 이벤트 무시
2. **Payload 구성**: `{ memberId, type:'MANUAL', checkInMethod:'MANUAL', checkInAt: new Date().toISOString() }`
3. **Optimistic 처리 없음**: 금전/출석은 서버 권한 응답 후에만 UI 반영
4. **즉시 `mut.mutate()` 호출 → `03-제출중` 진입**

## 역할별 차이

- 역할 차이 없음 — 권한/회원상태 가드는 `01-열림`에서 이미 처리

## 상태 고유 비즈니스 룰 델타

- 클라 추가 검증:
  - 현재 시각이 지점 영업시간 내인지 재확인(시간 경계 방어)
  - memberId 유효성 (number, > 0)
- 위 검증 실패 시 가드 무효화하고 `01-열림` 복귀 (이 경우는 거의 없음)

## 에지 케이스

- **더블 클릭**: 첫 번째 클릭이 프리플라이트 진입 후 `mut.isPending=true` 되기 전 두 번째 클릭이 들어오면 가드로 무시
- **시계 오차**: 서버가 최종 판단하므로 클라 clock skew는 허용 범위
- **memberId 변경(부모가 다른 회원으로 전환)**: 새 버튼 인스턴스 기준으로 재계산

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: manual-attend-preflight (매우 짧은 중간 단계)

onClick 핸들러:
const onClick = React.useCallback(() => {
  // 가드
  if (mut.isPending) return;
  if (disabled) return;
  if (!memberId || memberId <= 0) return;
  // 즉시 submit
  mut.mutate();
}, [mut, disabled, memberId]);

참고: 이 상태는 별도 렌더 분기 없이 01→03 사이의 논리적 단계.
     상태 기계 문서화 목적으로만 존재.
```
