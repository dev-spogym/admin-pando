# DLG-M012 상담 삭제 확인 — 상태 02: 입력 중(확인 대기)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 델타만 기술.

## 메타
| 항목 | 값 |
|------|----|
| 상태 코드 | `consult-del-confirming` |
| 상태 ID | STATE_CONFIRMING |
| 다이어그램 | `M2_필드검증.md` → DECIDING |
| 이전 상태 | `01-열림` (오픈 유지) |
| 다음 상태 | `03-제출중` (삭제 클릭) / Closed (취소) |

## 상태 진입 조건
- 사용자가 모달을 관찰하며 판단 중(키보드 탐색 포함)
- 별도 입력 필드 없음 → 사실상 `01` 과 시각적 동일

## 상태 고유 비주얼 델타
- 포커스가 "취소" 또는 "삭제" 에 머무는 상태
- hover/focus 링: 각 버튼 `focus:ring-2 focus:ring-rose-500`

## 상태 고유 인터랙션
1. **Tab 이동**: 취소 ↔ 삭제 포커스 순환
2. **Enter/Space**: 포커스된 버튼 실행
3. **마우스 hover**: "삭제" → `bg-rose-700`, "취소" → `bg-gray-50`
4. **ESC/배경/X**: 닫힘

## 상태 고유 비즈니스 룰 델타
- 이 다이얼로그는 타이핑 확인(`confirmationText`)을 사용하지 않음
  (상담 이력은 단건이므로 L2 파괴이나 타이핑까지는 요구 X)
- 연속 클릭 방지는 `03` 상태 진입 시 처리

## 에지 케이스
- **모달을 너무 오래 열어둠(5분)**: 세션 만료 가능 → 확정 시 401 → DLG-000 우선
- **상담이 다른 세션에서 수정됨**: 확정 시 서버는 삭제 그대로 수행(혹은 409 정책)
