| 상태 ID | STATE_OPEN |

| 다음 상태 | `02-입력중` (확인 대기) / `03-제출중` (즉시 확정) / Closed (취소) |

## 상태 진입 조건
- `consultationId` 유효
- `auth.role` ∈ {primary, owner, manager, fc(본인만)}
- 서버 `canDelete` 검증 전이므로 UI 레벨만 판정

## 상태 고유 비주얼 델타
- 마스터 기본 렌더 (danger variant)
- "취소" 버튼 `autoFocus` (DLG-003 정책)
- `loading=false`, 두 버튼 모두 enabled
- backdrop `bg-black/50`

## 상태 고유 인터랙션
1. **오픈 직후**: "취소" 포커스, `Enter` = 취소 (안전)
2. **"삭제" 클릭**: `03-제출중` 전환
3. **"취소"/ESC/배경/X**: 닫힘
4. **Tab**: 취소 → 삭제 → X → 취소

## 상태 고유 비즈니스 룰 델타
- 오픈 시점에 `consultation.counselorId` 캐시에서 확인 (fc 권한)
- 낙관적 제거 비활성 (삭제 실패 가능성 고려)

## 에지 케이스
- **이미 삭제됨**: 다른 탭에서 삭제 → 서버 404 → `04` 에서 refetch
- **오픈 중 권한 변경**: 다이얼로그는 표시, 확정 시 403 응답
