## 상태 진입 조건

- `01-열림` 상태에서 유형/포인트/사유 중 하나라도 값이 변경됨
- `isDirty === true`

## 상태 고유 비주얼 델타

- 필드에 값 입력됨
- "조정" 버튼 활성/비활성 상태가 값에 따라 동적 변경
  - 활성: `isValid === true && !insufficient`
  - 비활성: 나머지
- `insufficient === true` (type=USE + points > balance)인 경우:
  - 포인트 필드 `border-red-300 focus:ring-red-500`
  - 필드 하단 `text-xs text-red-600` 로 "보유 마일리지가 부족합니다 (잔액 {balance} P)"
- 사유 200자 근접 시 카운터 표시 (`{len}/200`, 180자 초과 `text-amber-600`, 200자 도달 `text-red-600`)
- 유형 변경(EARN↔USE) 시 포인트 재검증 트리거

## 상태 고유 인터랙션

1. **실시간 검증**: `mode: 'onChange'`로 필드별 에러 즉시 표시
2. **포인트 숫자 입력**: `valueAsNumber: true`, 음수/소수 차단 (`min=1`, `step=1`)
3. **USE 선택 + balance=0**: 포인트 필드 `disabled` + 경고 배너 "보유 마일리지가 없어 사용할 수 없습니다"
4. **사유 입력**: IME 조합 중에도 `trim().length` 재계산
5. **Enter 키**: 필드 내에서 Enter 시 `handleSubmit(onSubmit)` 호출 (단 `disabled`면 no-op)
6. **취소/ESC**: 입력값 즉시 폐기, 확인 없이 닫음 (가벼운 수정이므로 DLG-002 이탈 경고 불필요)

## 역할별 차이

- 마스터와 동일

## 상태 고유 비즈니스 룰 델타

- USE + points > balance: 클라 사이드에서 제출 차단 (서버 422 방지)
- 사유 trim 후 공백이면 에러
- 포인트 최대값: 정책상 상한 없음 (서버 쪽 `max: 1_000_000`)
- 유형 스위치 시 `points`는 유지되나 `insufficient` 재평가

## 에지 케이스

- **빠른 타이핑**: debounce 없이도 onChange 안정적
- **IME 조합**: 한국어 입력 시 `onCompositionEnd`에서 검증 재실행
- **붙여넣기**: 포인트 필드 숫자 외 문자는 자동 제거

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용

검증 로직:
const points = watch('points');
const type   = watch('type');
const reason = watch('reason');
const insufficient = type === 'USE' && Number(points) > member.mileage;
const canSubmit = isValid && !insufficient && !mut.isPending;

포인트 필드 에러 시각:
<input className={insufficient || errors.points
  ? 'border-red-300 focus:ring-red-500' : 'border-gray-300 focus:ring-blue-500'} />
{insufficient && (
  <p className="text-xs text-red-600 mt-1">
    보유 마일리지가 부족합니다 (잔액 {formatNumber(member.mileage)} P)
  </p>

사유 카운터:
<div className="flex justify-between">
  <label>사유 *</label>

    reason.length > 200 ? 'text-red-600' :
    reason.length > 180 ? 'text-amber-600' :
    'text-gray-400'
  }>{reason.length}/200</span>
</div>

제출 버튼:
<button disabled={!canSubmit}
  className={`h-10 px-4 rounded-lg text-white text-sm
    ${canSubmit ? 'bg-blue-600 hover:bg-blue-700' : 'bg-blue-300 cursor-not-allowed'}`}>
  조정
</button>
```
