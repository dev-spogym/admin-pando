## 상태 고유 비주얼 델타

### 성공 분기 (2xx)
- **즉시 다이얼로그 닫힘** (지연 없음)
- Toast: `success` 배너 상단 우측
  - EARN: `"마일리지 {points}P가 적립되었습니다"`
  - USE: `"마일리지 {points}P가 사용되었습니다"`
- 부모 화면(SCR-M004) 마일리지 서브탭: 신규 로그 행 추가 + 잔액 갱신
- 포커스: 부모의 "수동 조정" 버튼으로 복귀

### 실패 분기 (4xx/5xx)

| errorCode | HTTP | 처리 |
|---|---|---|
| E422100 | 422 | 포인트 필드 에러 "보유 마일리지가 부족합니다" + 다이얼로그 유지 → `02-입력중` |
| E400101 | 400 | 필드별 에러 매핑 + 다이얼로그 유지 |
| E404100 | 404 | 토스트 "회원을 찾을 수 없습니다" + 다이얼로그 닫기 |
| E403001 | 403 | 토스트 "권한이 없습니다" + 다이얼로그 닫기 |
| E422101 | 422 | 토스트 "탈퇴한 회원입니다" + 다이얼로그 닫기 |
| E500001 | 500 | 토스트 "일시적 오류가 발생했습니다. 다시 시도해주세요" + 다이얼로그 유지 |
| NETWORK | — | 토스트 "네트워크 오류" + 다이얼로그 유지 |
| E401002 | 401 | DLG-000 세션만료 오픈, 이 다이얼로그 정리 |

### 실패 유지 시 시각
- 필드 readOnly 해제, 버튼 다시 활성
- 에러 코드별 토스트 상단 공지
- 포커스: 첫 번째 에러 필드 또는 "조정" 버튼

## 상태 고유 인터랙션

1. **성공 시 onSuccess 콜백**: `qc.invalidateQueries`, `toast.success`, `onClose()`
2. **실패 시**: 에러 분기 처리 후 `02-입력중`으로 복귀 또는 `onClose()`
3. **실패 유지 상태에서 재시도**: 필드 수정 후 다시 "조정" 클릭 가능

## 역할별 차이

- `superAdmin/primary`: 타 지점 에러 시 `branchId mismatch` 메시지도 토스트로 노출 가능
- 그 외: 표준 에러 메시지

## 상태 고유 비즈니스 룰 델타

- **성공 시 원자성**: 서버는 트랜잭션으로 `member_mileage_logs` INSERT + `members.mileage` UPDATE를 보장
- **감사로그**: `AUDIT.MEMBER_MILEAGE_ADJUST` 서버 기록 (adjustor userId, type, points, reason, beforeBalance, afterBalance, timestamp)
- **중복 제출 결과**: 422100 외 422101(탈퇴)도 닫기 대상
- **부분 성공 없음**: 금융 트랜잭션 특성상 all-or-nothing

## 에지 케이스

- **응답 파싱 실패**: `catch` 블록에서 `toast.error('응답 처리 중 오류')` + 다이얼로그 유지
- **서버 지연 응답 후 유저 이미 닫음**: 클린업 체크 `if (!mountedRef.current) return;`
- **토스트 스택**: 다중 실패 시 직전 토스트 `toast.dismiss()` 후 새 토스트

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: mileage-adjust-result

const mut = useMutation({
  mutationFn: postMileageAdjust,
  onSuccess: (res, vars) => {
    qc.invalidateQueries({queryKey:['mileage-logs', member.id]});
    qc.invalidateQueries({queryKey:['member', member.id]});
    toast.success(
      vars.type === 'EARN'
        ? `마일리지 ${formatNumber(vars.points)}P가 적립되었습니다`
        : `마일리지 ${formatNumber(vars.points)}P가 사용되었습니다`
    );
    onSuccess?.(res.data.newBalance);
    onClose();
  },
  onError: (e: ApiError) => {
    switch (e.errorCode) {
      case 'E422100':
        setError('points', { message: '보유 마일리지가 부족합니다' });
        toast.error('보유 마일리지가 부족합니다');
        break; // 다이얼로그 유지
      case 'E403001':
        toast.error('권한이 없습니다'); onClose(); break;
      case 'E404100':
        toast.error('회원을 찾을 수 없습니다'); onClose(); break;
      case 'E422101':
        toast.error('탈퇴한 회원입니다'); onClose(); break;
      case 'E500001':
      default:
        toast.error('일시적 오류가 발생했습니다');
        // 다이얼로그 유지
    }
  },
});

포커스 복귀:
onClose() 호출 시 triggerRef.current?.focus()
```
