## 상태 진입 조건
- `form.formState.isValid === true`
- `mutation.mutate(values)` 호출됨 → `mutation.isPending=true`

## 상태 고유 비주얼 델타
- "저장" 버튼: `<Loader2 className="size-4 animate-spin" />` + 텍스트 "저장 중..." + `cursor-not-allowed`
- "취소" 버튼: `disabled + opacity-50`
- X(닫기) 버튼: `disabled + opacity-50`
- 모든 필드 `disabled` (pointer-events: none 대안 허용)
- Backdrop 클릭 차단: `onClick` 가드

## 상태 고유 인터랙션
1. **ESC**: 무시 (guard by `isPending`)
2. **배경 클릭**: 무시
3. **버튼 클릭**: 무시
4. **중복 저장**: disabled 이므로 발생 불가 (2중 방어 가능)

## 상태 고유 비즈니스 룰 델타
- **엔드포인트 분기**:
  - 신규: `POST /api/consultations`
  - 수정: `PATCH /api/consultations/:id`
- **페이로드**: `{ ...values, memberId }` (fc는 서버에서 `counselorId=self` 강제)
- **타임아웃**: 10초 초과 시 AbortController → `04-실패` 경로
- **낙관적 업데이트 금지**: 서버 응답까지 목록 미변경 (에러 롤백 복잡도 회피)

## 역할별 차이

## 에지 케이스
- **네트워크 끊김**: `NETWORK` 에러 → `04-실패` 유지 + 재시도 안내
- **서버 처리 지연**: 10초 후 토스트 "네트워크가 느립니다" (선택)
- **다른 탭에서 동일 상담 수정**: 서버 409 가능 → `04-실패` 토스트
