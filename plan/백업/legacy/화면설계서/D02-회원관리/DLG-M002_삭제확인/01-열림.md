# DLG-M002 회원 삭제 확인 — 상태: 열림 (초기)

> 🔗 **마스터**: [`00-기본화면.md`](./00-기본화면.md) 참조. 이 문서는 **델타만** 기술합니다.

## 메타
| 항목 | 값 |
|---|---|
| 상태 코드 | `open` |
| 상태 ID | DLG-M002-01 |
| 다이어그램 노드 | `DLG_OPEN` |
| 이전 상태 | SCR-M004 위험 구역 "회원 삭제" 클릭 / SCR-M001 "일괄 삭제" |
| 다음 상태 | `02-입력중또는확인중` |

## 상태 진입 조건
- `open === true`
- 역할: `superAdmin | primary | owner` (MANAGER 이하 불가)
- `member.deletedAt === null` (이미 삭제된 회원 대상 재삭제 불가)
- bulk 모드: `selectedRows.size > 0`

| 요소 | 마스터 대비 | 비고 |
|---|---|---|

| input border | `border-gray-300` (기본) | — |
| 힌트 텍스트 | "삭제 텍스트 입력 필요" | aria-live polite |
| "삭제하기" 버튼 | `disabled` + `bg-red-300` | — |
| 경고 배너 | 항상 노출 | bg-red-50 |

## 역할별 차이
| 역할 | 접근 | 비고 |
|---|---|---|
| superAdmin/primary/owner | ● 진입 가능 | — |
| manager 이하 | 트리거 버튼 미노출 | 방어: 직접 URL 접근 시 403 |

## 상태 고유 인터랙션
1. **confirmText input 타이핑** → `02-입력중또는확인중`
2. **Esc/X/취소** → 모달 닫기
3. **Overlay 클릭** → 모달 닫기
4. **Tab** 순서: 닫기X → input → 취소 → 삭제하기

## 비즈니스 룰 델타
- 열림 시 `confirmText`, `errorCode` 초기화
- `member.status === 'WITHDRAWN'`일 경우 안내 문구 변경: "이미 탈퇴한 회원입니다. 삭제하시겠습니까?"
- `member` 정보 최신화를 위해 진입 직전 부모 화면 refetch 권장

## 에지 케이스
- `member === null`: 다이얼로그 렌더링 방지 + toast
- 네트워크 복구/재연결 중 열기: 최신 member 정보 재조회
- 동시성: 다른 세션에서 이미 삭제 → 열 때 404 처리 (부모 refetch)

## 바이브코딩 프롬프트 (델타)
```
상태: open
useEffect(() => {
  if (open) {

```
