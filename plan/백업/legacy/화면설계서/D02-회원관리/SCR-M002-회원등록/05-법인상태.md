## 인터랙션
| 액션 | 동작 |
|---|---|
| 회원구분 "일반" 재선택 | companyName 필드 **숨김 + 값 초기화** |
| 회원구분 "기명법인" → "무기명법인" | companyName 유지 |
| companyName 입력 | 일반 필드처럼 onBlur 검증 (현재 min 검증 없음, 선택) |

## 룰 델타
1. **조건부 렌더링**: `memberType === '기명법인' || memberType === '무기명법인'`일 때만 companyName 노출.
2. 전환 시 값 초기화: 일반으로 돌아가면 `setValue('companyName', '')`.
3. **무기명법인 규칙(🆕)**: 회원 이름이 자동으로 "{회사명} 회원 N"로 제안 (accept/reject).
4. **법인 회원 특수 동작(🆕)**:
   - 가족 회원 기능 비활성
   - 통합 미수금/결제 지원
   - 마일리지 공유 (무기명)
5. 시각적 비활성 아님 — 모든 필드 정상 사용 가능.
6. 회원구분 변경 시 analytics `member_type_changed` 이벤트 기록.

## 에지
- 법인 → 일반 전환 시 companyName 값 삭제 안내: "법인 회사명 정보가 초기화됩니다." confirm (선택 정책).
- 법인 + 법인명 비어있고 저장 시도 → 경고 toast "법인 회사명을 입력하는 것을 권장합니다." (soft warning, 저장은 허용).
- 무기명법인인데 이름 필드가 일반 개인명이면 저장 시 안내 "무기명법인은 회사명으로 관리됩니다." (🆕).

## 🧩 바이브코딩 (델타)
```
const memberType = watch('memberType');
const isCorp = memberType === '기명법인' || memberType === '무기명법인';

// memberType 변경 시 초기화
useEffect(() => {
  if (!isCorp) setValue('companyName', '');
}, [isCorp, setValue]);

{isCorp && (
  <FormField label="법인 회사명" error={errors.companyName?.message}
             htmlFor="companyName">
    <Input id="companyName" {...register('companyName')}
           placeholder="예: ㈜핏지니"
           autoComplete="organization" />
    {memberType === '무기명법인' && (
      <p className="text-xs text-gray-500 mt-1 flex items-center gap-1">
        <Info className="w-3 h-3" /> 무기명법인은 회사명으로 관리되며 이름은 "{watch('companyName')} 회원 N"으로 자동 생성됩니다.
      </p>
    )}
  </FormField>
)}
```
