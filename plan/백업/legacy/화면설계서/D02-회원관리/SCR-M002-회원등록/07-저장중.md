## 룰 델타
1. **중복 제출 방지**: `isSubmitting` 플래그 + RHF 내부 `isSubmitting` 자동 관리.
2. **idempotency**: 서버에 request id (UUID v4) 전송 → 같은 id로 2회 제출 시 서버가 409 또는 기존 리소스 반환.
3. **타임아웃**: 30s. 초과 시 toast.info "서버 응답이 지연됩니다. 잠시만 기다려주세요" (강제 실패 아님).
4. **페이로드 구성**:
   ```ts
   const payload = {
     name, phone, email: email || null,
     gender: gender === 'male' ? 'M' : 'F',
     birthDate: birthDate || null,
     memberType, companyName: isCorp ? companyName : null,
     height: height ? Number(height) : null,
     memo: notes || null,
     status: 'INACTIVE',        // 이용권 구매 시 ACTIVE 자동
     branchId: user.branchId,   // 서버에서 재검증
     profileImage, adConsent: marketingConsent,
     staffId: fc ? Number(fc) : null,
     visitSource: visitPath || null,
     referralSource: referralSource || null,
     exercisePurpose: exerciseGoal || null,
     // 등록 모드 추가
     idempotencyKey: crypto.randomUUID(),
   };
   ```
5. **성공 콜백**:
   - `AUDIT.MEMBER_CREATE` 로그
   - React Query cache invalidate: `members`, `memberStats`
   - draft 제거: `localStorage.removeItem('member:new:draft')`
   - `beforeunload` 리스너 해제
6. **실패 콜백**:
   - 400: `03-유효성에러`로 전환 + 필드 맵핑
   - 409: phone 필드에 "이미 등록된 번호입니다." + phoneChecked=true, phoneDuplicate=true
   - 403: `/forbidden`
   - 5xx: 폼 유지 + toast.error "저장에 실패했습니다. 잠시 후 다시 시도해주세요." → `06-Step2`
   - NETWORK: toast.error "네트워크 연결을 확인해주세요."

## 에지
- 브라우저 백그라운드 슬립 후 복귀 → mutation 상태 유지 여부 확인.
- 5분 이상 pending → auto retry 1회 후 실패 처리.
- 탭 여러 개에서 같은 draft로 저장 → idempotencyKey로 서버가 중복 감지 → 409.
- 이미지 업로드 완료 전 저장 → profileImage=null 또는 경고 (정책 선택).
- 중복확인 통과했으나 저장 직전 다른 사용자가 같은 번호 등록 → 서버 409.

## 🧩 바이브코딩 (델타)
```
const createMember = useMutation({
  mutationFn: (payload: CreateMemberPayload) =>
    fetch('/api/members', {
      method: 'POST', headers: { 'Content-Type':'application/json', 'X-Idempotency-Key': payload.idempotencyKey },
      body: JSON.stringify(payload)
    }).then(r => r.json()),
  onSuccess: (res) => {
    queryClient.invalidateQueries({ queryKey: ['members'] });
    localStorage.removeItem('member:new:draft');
    toast.success('신규 회원이 등록되었습니다.', {
      action: { label: '바로 결제', onClick: () => router.push(`/pos?memberId=${res.id}`) },
      duration: 5000
    });
    router.push(`/members/detail?id=${res.id}`);
  },
  onError: (err) => {
    if (err.errorCode === 'E409100') {
      setError('phone', { message: '이미 등록된 번호입니다.', type: 'server' });
      setPhoneChecked(true); setPhoneDuplicate(true);
    } else if (err.errorCode?.startsWith('E400')) {
      mapServerErrorsToFields(err);
    } else {
      toast.error(err.message ?? '저장에 실패했습니다.');
    }
  }
});

<fieldset disabled={isSubmitting} className="space-y-6">
  {/* 모든 폼 내용 */}
</fieldset>

<Button type="submit" loading={isSubmitting} disabled={isSubmitting}
        aria-busy={isSubmitting}>
  {isSubmitting ? '저장 중...' : '저장'}
</Button>

// 상단 progress
{isSubmitting && <div className="fixed top-0 inset-x-0 h-0.5 bg-blue-500 animate-pulse" aria-hidden />}

// 30s 지연 알림
useEffect(() => {
  if (!isSubmitting) return;
  const t = setTimeout(() => toast.info('서버 응답이 지연됩니다. 잠시만 기다려주세요'), 30_000);
  return () => clearTimeout(t);
}, [isSubmitting]);
```
