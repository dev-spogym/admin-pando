> 🧭 **델타**. 공통 스펙은 [`00-기본화면.md`](./00-기본화면.md) 참조.

## 메타

| 항목 | 값 |
|------|----|
| 상태 코드 | `new-default` |
| 다이어그램 | `STATE_DEFAULT` @ `F6_상태별.md` |
| 이전 | (진입) |
| 다음 | `02-입력중` (첫 입력) / `03-유효성에러` / `05-법인상태` |

## 진입 조건
- `/members/new` 초기 진입, `isDirty === false`, `phoneChecked === false`, `currentStep === 'step1'`
- 모든 필드 `defaultValues` (이름 "", 성별 undefined, 연락처 "", memberType "일반", profileImage null)

## 비주얼 델타
| 요소 | 변경 |
|---|---|
| Step Indicator | `[① 필수 정보]` 활성, `[② 추가 정보]` 비활성 |
| 프로필 이미지 | 카메라 아이콘 placeholder |
| 모든 필드 | empty value + placeholder |
| 에러 | 모두 숨김 (onBlur 전) |
| 중복확인 버튼 | `disabled` (phone 비어있거나 길이 < 13) |
| [초기화] | `disabled` (isDirty=false && phoneChecked=false) |
| [취소] | `enabled` |
| [다음] | 시각적 비활성 (step1CanNext=false) 또는 활성(이름/성별/연락처/회원구분 기본값 방어) |
| [저장] | Step 2 전용이라 미노출 |

## 역할별 차이
- 기본 §5와 동일. 접근 가능한 역할만 본 상태 진입.

## 인터랙션
| 액션 | 동작 |
|---|---|
| 필드 첫 입력 | `02-입력중`로 전환 (isDirty=true) |
| 프로필 이미지 선택 | `04-이미지업로드중` |
| memberType "기명/무기명법인" 선택 | `05-법인상태` (companyName 필드 노출) |
| [취소] | isDirty=false이므로 즉시 `/members` 이동 |

## 룰 델타
1. autoFocus: `name` 필드에 자동 포커스 (접근성).
2. analytics `member_new_enter` 이벤트 (role, branchId).
3. URL 쿼리 `?from=list` 등 traceback 파라미터 유지.
4. `beforeunload` 리스너 비활성 (dirty 아님).

## 에지
- 로컬 스토리지에 이전 draft 존재 → 상단 배너 "이전 입력 내용 복원하시겠습니까?" [복원] [버리기]. (🆕 기능)
- 권한 변동 감지 (JWT 만료 직전) → 세션 연장 또는 /login 이동.

## 🧩 바이브코딩 (델타)
```
※ 델타: autoFocus + draft 복원 배너

useEffect(() => { nameRef.current?.focus(); }, []);

const draft = localStorage.getItem('member:new:draft');
{draft && !isDirty && (
  <DraftRestoreBanner
    preview={JSON.parse(draft)}
    onRestore={() => reset(JSON.parse(draft))}
    onDiscard={() => localStorage.removeItem('member:new:draft')}
  />
)}
```
