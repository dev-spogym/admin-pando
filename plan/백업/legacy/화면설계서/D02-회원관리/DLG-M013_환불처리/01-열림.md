# DLG-M013 환불 처리 — 상태 01: 열림

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 델타만 기술.

## 메타
| 항목 | 값 |
|------|----|
| 상태 코드 | `refund-open` |
| 상태 ID | STATE_OPEN |
| 다이어그램 | `M1_생명주기.md` → DLG_OPEN |
| 이전 상태 | Closed ([환불] 버튼 클릭) |
| 다음 상태 | `02-입력중` (사유 입력) / `03-제출중` (즉시 확정) / Closed (취소) |

## 상태 진입 조건
- `sale.status !== 'REFUNDED'`
- `sale.salePrice > 0`
- `auth.role` ∈ {primary, owner, manager}
- `sale.branchId === auth.branchId` (super/primary 예외)

## 상태 고유 비주얼 델타
- 마스터 danger 레이아웃 초기 렌더
- 요약 카드: 결제일/결제방법/환불액(-150,000원 rose-600)
- 사유 입력 필드: 비어있음, placeholder 없음
- "취소" autoFocus (금전 파괴 보수)
- "환불 처리" 버튼 enabled (사유 없어도 가능, 선택)
- `loading=false`

## 상태 고유 인터랙션
1. **오픈 직후**: 요약 카드 + 사유 입력 필드 동시 표시
2. **Enter**: 포커스된 버튼 실행 (오픈 직후 = 취소)
3. **Tab**: 사유 → 취소 → 환불 처리 → X → 사유
4. **ESC/배경/X**: 닫힘

## 역할별 차이
- `manager`: 500,000원 초과 시 서버 403 가능(정책). UI 경고 배너 선택적 노출:
  ```
  {sale.salePrice > 500000 && role === 'manager' && (
    <p className="text-xs text-amber-700">⚠ 50만원 초과 환불은 센터장 승인 필요</p>
  )}
  ```

## 상태 고유 비즈니스 룰 델타
- `sale` 객체는 부모에서 prop 으로 주입(별도 fetch 없음).
- 요약 카드 값은 `sale` snapshot (재조회 없음).
- 사유 입력 필드 `maxLength={200}`, 비제어 입력도 허용.

## 에지 케이스
- **열람 직후 다른 탭에서 환불**: 확정 시 `E409300` → `04`
- **sale 누락(undefined)**: 모달 오픈 자체 차단 (부모 가드)
- **salePrice=0 무료 상품**: 버튼 비노출, 여기 진입 불가
