## 상태 진입 조건

- getMembers/핵심 API 실패:
  - 500 (E500001), 503 (E503001), 404 (E404001)
  - 네트워크 단절 (`navigator.onLine === false` 또는 AbortError)
  - 타임아웃 (30s)
- **401/403은 제외** — 401→로그인 리다이렉트, 403→`06-권한없음`

## 역할별 차이

- 모든 역할 동일
- fc/staff/front: "관리자에게 문의" 링크가 사내 메신저 딥링크
- DEBUG 모드(`NEXT_PUBLIC_DEBUG=1` 또는 관리자): Error code/Trace ID 노출

## 상태 고유 인터랙션

| 액션 | 동작 |
|---|---|
| [🔄 다시 시도] | `refetchAll()` → `01-로딩` |
| [이전 페이지로] | `router.back()` |
| 자동 재시도 | exponential backoff 1s/3s/7s, 최대 3회 |
| 지점 전환 | 활성 유지 (다른 지점은 정상 가능) |
| 온라인 복구 | 자동 refetch |
| Esc | 이펙트 없음 |

## 상태 고유 비즈니스 룰 델타

1. 에러 분류:
   ```ts
   if (!navigator.onLine) return 'NETWORK';
   if (err.code === 'TIMEOUT')  return 'TIMEOUT';
   if (err.status === 503)      return 'MAINTENANCE';
   if (err.status >= 500)       return 'SERVER';
   return 'UNKNOWN';
   ```
2. 같은 에러코드 10초 내 재토스트 금지.
3. traceId 표기 + Sentry/Datadog 연동.
4. SR `role="alert" aria-live="assertive"` 공지.
5. 부분 실패: members OK + stats 실패 → 정상 유지 + StatCard "-" + inline 재시도.
6. 온라인 복구 리스너 + offline 리스너 장착.

## 에지 케이스

- 반복 실패(3회) → 수동 버튼만.
- CORS/SSL → "서비스 연결에 실패했습니다. 관리자에게 문의해주세요."
- 인증 만료 + 네트워크 동시 → 네트워크 우선 (복구 후 401 처리).
- 재시도 중 새 요청 → 이전 abort.
