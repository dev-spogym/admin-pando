## 상태 고유 비즈니스 룰 델타

1. 온보딩 시나리오: 서버 `isFreshBranch: true` 플래그로 힌트.
2. analytics `members_empty_first_time` (branchId, role) 기록.
3. 모바일 CTA 세로 배치 + full-width.
4. `role="status"` 공지 "등록된 회원이 없습니다. 회원을 추가해보세요."
5. CTA `autoFocus` — 키보드 사용자 편의.

## 에지 케이스

- stats.isError로 인한 가짜 0건은 `05-에러`로 분기 (본 상태 아님).
- deletedAt 모두 존재 → 서버 where `IS NULL`로 0건 판정.
- fc 담당 0명 → "담당 회원이 아직 없습니다. 관리자에게 배정 요청하세요."
- 지점 전환 직후 빈 지점 → primary/super에게 "다른 지점으로 이동하거나 회원을 추가하세요."
