## 역할별 차이

- 모든 역할 동일한 스켈레톤. 기본 §5.1로 숨겨진 컬럼/버튼은 스켈레톤에도 미렌더.
- readonly: 본 상태 미도달 (서버 403 → `06-권한없음`).

## 상태 고유 인터랙션

| 액션 | 동작 |
|---|---|
| 재검색/필터 변경 | 기존 fetch abort + 새 fetch 시작 (AbortController) |
| ESC | 이펙트 없음 |
| 체크박스 클릭 | 무시 (테이블 body skeleton) |
| 페이지 이탈 | React Query abort |

## 상태 고유 비즈니스 룰 델타

1. 캐시 없는 첫 진입 → 스켈레톤. 캐시 존재 → `keepPreviousData`로 이전 데이터 `opacity-60` + 우측 상단 미니 스피너.
2. `loadTimeout = 10s` 초과 시 `toast.info("서버 응답이 지연되고 있습니다")` (강제 에러 아님).
3. 스켈레톤 행 수: 이전 pageSize만큼 (없으면 10).
4. `prefers-reduced-motion: reduce`이면 `animate-pulse` 제거 후 고정 스켈레톤.

## 에지 케이스

- 401 즉시 실패 → `/login?redirect=/members` 이동.
- 403 즉시 실패 → `06-권한없음`.
- 오프라인 → 스켈레톤 유지 + "오프라인 상태입니다" 인포 배너.
- 페이지크기 100 → 스켈레톤 행 수는 10으로 고정 (과도함 방지).
