## 상태 고유 비즈니스 룰 델타

1. 데이터 정렬: 기본 `createdAt DESC`. 컬럼 헤더로 override.
2. 세그먼트 계산: 클라이언트 매 렌더 1회, `useMemo`로 캐시.
3. 위험/경고 세그먼트(`CHURN_RISK`, `EXPIRING`)는 행 왼쪽 2px `border-l-red-400`로 강조.
4. 만료일 D-Day 표시: `membershipExpiry`까지 ≤ 7일이면 셀 옆에 `⚠️ D-n` (빨강), ≤ 30일이면 `D-n` (주황).
5. 관심회원 ON 회원은 테이블 행 좌측 0.5px `bg-yellow-50/50` 라인 힌트 (선택).
6. 상태 배지 dot: `.●` (before pseudo) 크기 6px, 해당 상태 색.
7. 탈퇴 회원(`WITHDRAWN`)은 이름 셀 `text-gray-400 line-through`.
8. 소프트 삭제된(`deletedAt IS NOT NULL`) 회원은 목록에 노출 안 됨(서버 where).

## 에지 케이스

- **stats 단독 실패**: 테이블은 정상, StatCard 값 "-" + tooltip "일시 오류" + [다시] 링크.
- **stats 지연**: StatCard만 skeleton, 테이블은 정상.
- **장기 페이지(p.page = 100)**: total count 대비 over면 server가 `[]` 반환 → `03-빈상태-검색` 전환 대신 page=1 리셋 + toast.info "잘못된 페이지입니다".
- **선택된 행이 refetch 후 사라짐**: 선택 집합에서 자동 제거 + BulkActionBar count 갱신.
- **같은 id 중복**: React key 충돌 감지 시 dev 경고 + row id fallback `id-${index}`.
