## 상태 진입 조건
- 폼 검증(react-hook-form + zod) 통과
- `useSessionStore.open === false`
- 부모 화면의 저장 권한 통과

## 상태 고유 비주얼 델타
- 마스터 기본 렌더
- 변경 미리보기(`diff`) prop 제공 시 본문에 `DiffPreview` 렌더
- "저장" 버튼 `autoFocus` (Primary 액션 관례)
- `loading=false`

## 상태 고유 인터랙션
1. **오픈 직후**: "저장" 포커스, Enter 즉시 확정
2. **ESC / 배경 / X**: 취소, 모달 닫힘
3. **"저장"**: `02-저장중` 전환
4. **"취소"**: 닫힘, 편집 화면 유지(dirty 유지)
5. **Tab 순환**: 취소 → 저장 → X → 취소

## 역할별 차이
- 부모 권한 없는 역할은 "저장" 버튼 자체가 없음(오픈 불가)
- 센터장(`owner`)은 매출 관련 설정 저장 가능, manager 이하는 일부 저장 항목만

## 상태 고유 비즈니스 룰 델타
- **변경 미리보기 구성**: `react-hook-form`의 `formState.dirtyFields` 와 `defaultValues` 를 비교하여 `diff` 배열 생성
  ```ts
  const diff = Object.keys(form.formState.dirtyFields).map(key => ({
    key, from: defaultValues[key], to: values[key],
  }));
  ```
- 변경 항목이 0건이면 저장 버튼 자체가 비활성(다이얼로그 오픈 전 UI에서 차단)
- `confirmationText` 는 사용하지 않음(저장은 비파괴적)
- 낙관적 잠금 정보(`version`, `etag`)는 오픈 시점에 form store 에 snapshot

## 에지 케이스
- **검증 통과 후 값 변경**: 다이얼로그 오픈 직후 부모 폼이 추가 변경되면 저장 시점의 `form.getValues()` 사용(snapshot 아님)
- **권한 중간 변경**: 오픈 후 역할 변경으로 저장 불가 → 서버 403 → `04-저장실패`
- **다른 탭에서 동일 리소스 저장**: 낙관적 잠금 버전이 달라 → `05-충돌`

function DiffPreview({ diff }: { diff: {key:string; from:string; to:string}[] }) {
  if (!diff?.length) return null;
  return (
    <div className="rounded-md bg-gray-50 border border-gray-200 p-3 text-xs space-y-1">
      <p className="font-medium text-gray-900">변경된 항목 {diff.length}건</p>
      <ul className="space-y-1">
        {diff.map(d => (
          <li key={d.key} className="text-gray-600">
            <span className="font-medium text-gray-900">{LABEL[d.key] ?? d.key}</span>:{' '}
            <span className="line-through">{String(d.from)}</span>{' → '}
            <span className="tabular-nums">{String(d.to)}</span>
          </li>
        ))}
      </ul>
    </div>
  );
}
