## 상태 진입 조건
- 서버가 `409 Conflict` + `errorCode: E409xxx` + `data.server: <서버 최신값>` 반환
- 낙관적 잠금 실패(If-Match / version mismatch)

| 요소 | 변경 |
|---|---|
| 아이콘 | `AlertTriangle text-amber-500` (role=alertdialog) |
| 타이틀 | "데이터가 변경되었습니다" |
| 본문 | 충돌 설명 + 차이 미리보기 (`ConflictPanel`) |
| 버튼 1 | "취소" (Secondary) → 닫기 |
| 버튼 2 | "내 입력 유지" (Tertiary) → 다음 저장 재시도로 다시 409 대응 |
| 버튼 3 | "최신 불러오기" (Primary, `bg-blue-600`) → 서버값으로 `form.reset` + 01-열림 복귀 |
| 토스트 | 별도 노출 없음(다이얼로그 내부에서 안내) |

## 상태 고유 인터랙션
1. **"최신 불러오기"**: `form.reset(server)` + `setConflict(null)` + `01-열림` 복귀. 사용자 재편집 후 재저장
2. **"내 입력 유지"**: `setConflict(null)` + `01-열림` 복귀 (다음 저장 시 동일 409 가능)
3. **"취소"**: 전체 닫기, 편집 화면 유지(dirty 유지)
4. **ESC/배경**: 취소와 동일

## 역할별 차이
- 없음. 단 primary/super 의 경우 지점 단위 동시 편집이 잦아 충돌이 더 자주 발생할 수 있음.

## 상태 고유 비즈니스 룰 델타
- **ConflictPanel**: `currentValues` vs `serverValues` 를 키별로 비교하여 차이만 표시
- **최신 불러오기 선택 시**:
  - `form.reset(serverValues)` → dirty=false 리셋
  - `setVersion(newVersion)` → 다음 저장 요청용 최신 버전
  - `setConflict(null)` + `01-열림` 으로 복귀(사용자가 재확인 후 재저장)
- **내 입력 유지**:
  - form 값은 그대로, 서버 버전만 갱신하지 않음
  - 재저장 시 재차 409 발생 가능 → 사용자에게 명시(안내 문구)
- **강제 덮어쓰기 옵션**(선택, 고위험): 관리자 권한에만 노출
- **감사로그**: 충돌 발생 자체는 서버 로그에만 기록

## 에지 케이스
- **`server` 값 미제공(구버전 API)**: ConflictPanel 없이 "최신 불러오기"만 가능, diff 비교 비활성
- **재로드 실패**: 토스트 + 다이얼로그 유지 + 재시도 버튼
- **연속 충돌(3회+)**: 토스트 "다른 사용자가 계속 수정 중입니다. 잠시 후 다시 시도해 주세요" + 다이얼로그 닫기
- **세션 만료 우선**: 401 인터셉트가 우선

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: save-confirm-conflict

ConflictPanel 컴포넌트:
function ConflictPanel({
  current, server, onReload, onKeepMine,
}: { current: any; server: any; onReload: () => void; onKeepMine: () => void }) {
  const keys = Array.from(new Set([...Object.keys(current ?? {}), ...Object.keys(server ?? {})]));
  const diffs = keys.filter(k => JSON.stringify(current?.[k]) !== JSON.stringify(server?.[k]));
  return (
    <div role="alert" aria-live="assertive" className="space-y-3">
      <div className="flex items-start gap-3">
        <span className="flex size-10 items-center justify-center rounded-full bg-amber-50 shrink-0">
          <AlertTriangle className="size-5 text-amber-500" aria-hidden />
        </span>
        <div className="flex-1 text-sm text-gray-600 leading-relaxed">
          <p className="font-medium text-gray-900">데이터가 변경되었습니다</p>
          <p>다른 사용자가 이 데이터를 수정했습니다. 최신 데이터를 불러온 뒤 다시 저장해 주세요.</p>
        </div>
      </div>
      {diffs.length > 0 && (
        <ul className="rounded-md bg-gray-50 border border-gray-200 p-3 text-xs space-y-2">
          {diffs.map(k => (
            <li key={k} className="space-y-1">
              <p className="font-medium text-gray-900">{LABEL[k] ?? k}</p>
              <p className="text-gray-600">
                내 입력: <span className="tabular-nums">{String(current?.[k])}</span><br/>
                서버값: <span className="tabular-nums">{String(server?.[k])}</span>
              </p>
            </li>
          ))}
        </ul>
      )}
      <div className="flex flex-col sm:flex-row sm:justify-end gap-2 pt-2">
        <button onClick={onKeepMine}
          className="h-10 px-4 rounded-lg text-gray-700 hover:bg-gray-50 text-sm font-medium">
          내 입력 유지
        </button>
        <button onClick={onReload} autoFocus
          className="h-10 px-4 rounded-lg bg-blue-600 hover:bg-blue-700 text-white text-sm font-medium">
          최신 불러오기
        </button>
      </div>
    </div>
  );
}

부모 사용:
const [conflict, setConflict] = useState<{current:any; server:any} | null>(null);

mutationFn onError 에서:
  if (err.__conflict) setConflict({ current: form.getValues(), server: err.server });

ConfirmDialog children 에 주입:
{conflict && (
  <ConflictPanel
    current={conflict.current}
    server={conflict.server}
    onReload={() => {
      form.reset(conflict.server);
      setVersion(res.version);
      setConflict(null); // 01-열림 으로 복귀
    }}
    onKeepMine={() => setConflict(null)}
  />
)}
```
