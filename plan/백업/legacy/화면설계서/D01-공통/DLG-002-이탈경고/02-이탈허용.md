## 상태 진입 조건
- 사용자가 "저장 안 함" 버튼 클릭
- `confirmLeave()` 호출 → pendingUrl 경로 리다이렉트

## 상태 고유 비주얼 델타
- 이 상태는 **거의 순간적(즉시 언마운트)** → 별도 UI 변경 없음
- pendingUrl 이동 직전에 다이얼로그가 자연스럽게 사라지도록 처리(페이드아웃 없이 즉시 unmount)

## 상태 고유 인터랙션
- 없음(사용자 입력 수용 안 함, 자동 처리)

## 역할별 차이
- 없음

## 상태 고유 비즈니스 룰 델타
- `confirmLeave()` 로직 순서:
  1. `setPendingUrl(null)` → 다이얼로그 언마운트
  2. 분기:
     - pendingUrl === `'__back__'` → `history.back()`
     - 그 외 → `router.push(pendingUrl)`
- **isDirty 상태는 변경하지 않음**: 저장 안 하고 이동만 함. 다음 화면에서 form unmount 시 자동 소멸.
- 이동 후에는 `useBlockNavigation` 가 `beforeunload`/`popstate` 리스너를 해제(훅 클린업)

## 에지 케이스
- **pendingUrl 이 현재 URL**: router.push 는 리렌더만 발생(브라우저 URL 변화 없음). 의도된 동작
- **동일 라우트로 이동**: Next.js 15가 컴포넌트 재사용 시 form 상태 유지될 수 있음 → 사용자가 의도적 확인한 이탈이므로 리셋 권장(부모 폼 책임)
- **popstate 플로우에서 뒤로갈 곳 없음**: `history.back()` 이 no-op → 브라우저 기본 시작 페이지로 폴백 (브라우저 동작에 위임)
- **이동 중 세션만료(401)**: DLG-000 이 오픈되며 DLG-002 는 이미 사라진 상태

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: leave-warning-discard (이탈 허용)

confirmLeave 구현:
const confirmLeave = () => {
  const u = pendingUrl;
  setPendingUrl(null);
  if (u === '__back__') history.back();
  else if (u) router.push(u);
};

UI: 다이얼로그 즉시 언마운트(onDiscard 클릭 → confirmLeave → pendingUrl=null → isOpen=false)
부가 로직:
- 저장 안 함 이벤트 로컬 로그 (선택): console.debug('[leave] discarded', pendingUrl)
- i18n: 사용자 피드백용 토스트는 띄우지 않음(기대된 동작)
```
