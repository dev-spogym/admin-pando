## 🧩 바이브코딩 프롬프트
```
Next.js 15 App Router + TypeScript + Tailwind + Supabase 기반 'use client' 컴포넌트를 작성하라.
화면: SCR-103 글로벌 검색 — 상태: 검색 중

파일: src/components/common/GlobalSearchOverlay.tsx (isSearching=true 분기)

레이아웃:
- 동일 오버레이 패널
- 검색 입력 영역 + 스피너

컴포넌트 구조:
1. 검색 입력 (입력 중인 query 표시, 우측에 Loader2 스피너 animate-spin)
2. 구분선
3. 스켈레톤 결과 영역:
   - 그룹 레이블 스켈레톤 (h-4 w-20 bg-gray-100 animate-pulse)
   - 결과 항목 스켈레톤 3개 (h-12 bg-gray-50 animate-pulse rounded-lg mb-2)

데이터:
- isSearching: boolean = true
- query: string (2자 이상)
- debounceDelay: 300ms
- API: GET /api/search?q={query}&role={role}

인터랙션:
- 추가 입력 → debounce 리셋 (새 검색 요청)
- 응답 도착 → `04-결과있음` 또는 `05-결과없음`
- 에러 → `06-에러`
- ESC → 닫기

사용 유틸:
- useAuthStore, supabase, lucide-react (Loader2)
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 검색어 2자 이상 입력 후 300ms debounce 경과
- 최근 검색어 클릭

### 필수 데이터
| 항목 | 출처 | 비고 |
|------|------|------|
| query | 사용자 입력 | 2자 이상 |
| role | useAuthStore | 검색 범위 필터 |

### 인터랙션 (User Actions)
1. 입력 추가 → debounce 재시작 (이전 요청 취소)
2. API 완료 → 결과 상태 전환
3. ESC → 닫기

### 비즈니스 룰
- Debounce 300ms (연속 입력 시 마지막 요청만 실행)
- AbortController로 이전 요청 취소
- 역할별 검색 범위: staff는 재무 데이터 검색 제외

### 에지 케이스
- 검색 중 ESC → 요청 취소 + 닫기
- 검색 중 쿼리 삭제(1자 이하) → 요청 취소 + `02-초기-힌트`

### 접근성 (A11y)
- 스피너 `aria-label="검색 중"` `role="status"`
- 결과 영역 `aria-live="polite"` (결과 도착 시 공지)

### 연결 화면
- 성공: `04-결과있음`
- 없음: `05-결과없음`
- 에러: `06-에러`
