## 1. 화면 목적 (Why)

사용자가 이메일과 비밀번호로 자신을 인증하여 CRM 시스템에 접근하기 위한 **유일한 진입점**.
- 인증 후 역할(superAdmin/owner/manager/fc/trainer/staff/front)에 따라 전용 대시보드로 라우팅한다.
- 2FA 활성 계정은 추가 인증 단계를 통과해야 세션이 발급된다.
- 계정 잠금/서비스 점검 등 예외 상황을 사용자에게 명확히 전달한다.

## 6. 데이터 계약

### 6.1 폼 스키마 (Zod)
```ts
// src/schemas/auth.ts
export const loginSchema = z.object({
  email:    z.string().email('이메일 형식이 올바르지 않습니다'),
  password: z.string().min(1, '비밀번호를 입력하세요'),
  remember: z.boolean().default(false),
});
export type LoginForm = z.infer<typeof loginSchema>;
```

### 6.2 API 계약

| 항목 | 값 |
|---|---|
| 엔드포인트 | `POST /auth/login` (Supabase Auth signInWithPassword 래핑) |
| 요청 | `{ email: string, password: string, remember?: boolean }` |
| 성공(200) | `{ success: true, data: { session, user: { id, role, name, branchId }, needs2FA: boolean } }` |
| 실패(401) | `{ success: false, errorCode: 'E401001', message: '이메일 또는 비밀번호가 올바르지 않습니다' }` |
| 실패(423) | `{ success: false, errorCode: 'E423001', message: '계정이 잠겼습니다. 관리자에게 문의하세요' }` |
| 실패(500) | `{ success: false, errorCode: 'E500001', message: '서버 오류가 발생했습니다' }` |
| 실패(503) | `{ success: false, errorCode: 'E503001', message: '서비스 점검 중입니다' }` |

### 6.3 상태 관리
- **Store**: `useAuthStore` (Zustand) — `user`, `session`, `role`, `setSession()`, `logout()`
- **Form**: `react-hook-form` + `zodResolver(loginSchema)`
- **Local state**: `isSubmitting`, `errorCode`, `showPassword`, `cooldownUntil(5회 실패 후)`
- **Persistence**: `remember=true` 시 세션 `localStorage`, false 시 `sessionStorage`

## 8. 비즈니스 룰

1. **자동 리다이렉트**: 유효 세션 존재 시 이 화면은 렌더되지 않고 역할별 대시보드로 즉시 이동한다.
2. **중복 제출 방지**: `isSubmitting=true` 동안 버튼 `disabled`, 입력 필드 `readOnly`, Enter 키 무시.
3. **2FA 분기**: API 응답 `needs2FA=true`면 카드 내용을 `04-2FA입력` 상태로 전환(같은 라우트).
4. **재시도 쿨다운**: 5회 연속 401 발생 시 클라이언트 쿨다운 30초. 버튼 `disabled` + 카운트다운 표시.
5. **자동 로그인**: `remember=true` 저장 + 다음 진입 시 이메일 자동 채움(비밀번호 제외).
6. **세션 만료 리다이렉트**: 기타 화면에서 401 수신 시 `?redirect=...`로 복원 경로를 쿼리에 실어 이 화면으로 이동.
7. **점검 모드**: `GET /health` 503 응답 또는 feature flag `maintenance=true`면 `05-서비스점검` 상태만 표시.
8. **감사 로그**: 성공 시 `AUDIT.LOGIN_SUCCESS`, 실패 시 `AUDIT.LOGIN_FAILURE`(사유코드 포함) 기록.
