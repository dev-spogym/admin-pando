# SCR-100 로그인 — 상태 01: 기본 (폼 대기)

> 상속: [`00-기본화면.md`](./00-기본화면.md) — 이 문서는 **변경점(델타)**만 기술합니다.
> 마스터 문서의 모든 레이아웃/토큰/컴포넌트/데이터/접근성 규칙이 기본값으로 적용됩니다.

## 메타
| 항목 | 값 |
|------|----|
| 상태 코드 | `login-default` |
| 상태 ID | STATE_NORMAL |

| 이전 상태 | 세션 없음/만료 감지, 로그아웃 직후, `03-에러` 재시도 복원 |
| 다음 상태 | `02-제출중` (로그인 클릭) / `05-서비스점검` (503 수신) |
| 스크린샷 레퍼런스 | `figma:SCR-100/state-default` (예정) |

## 상태 진입 조건
- `/login` 접근 시 세션이 없거나 만료됨 → 폼 렌더
- `03-에러` 에서 사용자가 입력값을 수정하여 에러 배너가 해제된 경우
- `05-서비스점검` 해제(503 → 200) 후 자동 복귀

## 상태 고유 비주얼 델타
**변경 없음.** 마스터(00) 레이아웃 기본 상태 그대로 렌더링.
- 에러 배너: 비표시
- 로딩 스피너: 비표시
- 입력 필드: 활성(enabled), 기본 `border-gray-300`
- 로그인 버튼: 활성(enabled), `bg-blue-600`
- 재설정 링크: 활성

## 상태 고유 인터랙션
1. **이메일 blur** → RFC 5322 형식 검증 → 실패 시 필드 인라인 에러 (상태 유지, 배너는 표시 안 함)
2. **비밀번호 토글** → `show` 상태 플립 → 아이콘/타입/aria-label 전환
3. **로그인 버튼 클릭** 또는 **Enter 키** → `loginSchema.parse()` 성공 시 → `02-제출중` 전환
4. **체크박스 "로그인 상태 유지"** → 토글 → `remember` 값 변경 (제출 시에만 사용)
5. **비밀번호 재설정 링크 클릭** → `/reset-password` 이동(SCR-106)

## 상태 고유 비즈니스 룰 델타
- 최초 진입 시 이전 세션의 `email`이 localStorage에 남아있다면 이메일 필드에 자동 주입(`autofill`)
- 비밀번호 자동 주입은 하지 않음 (보안)
- URL 쿼리 `?redirect=<경로>` 있으면 state에 보관 → 로그인 성공 후 해당 경로로 이동

## 에지 케이스
- **빈 이메일 제출**: `email` 필드 인라인 에러 "이메일을 입력하세요", API 호출 안 함
- **빈 비밀번호 제출**: `password` 필드 인라인 에러 "비밀번호를 입력하세요"
- **이메일 형식 오류**: 필드 인라인 에러 "이메일 형식이 올바르지 않습니다"
- **캡스락 ON 감지 (선택)**: 비밀번호 입력 중 `KeyboardEvent.getModifierState('CapsLock')` → 헬퍼 문구 "Caps Lock이 켜져 있습니다"
- **페이지 복귀**: 뒤로가기로 돌아와도 폼 값 보존(remember=true일 때 이메일만)

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: login-default (기본)

이 상태에서만 적용할 변경점:
- errorCode = null (에러 배너 렌더 안 함)
- isSubmitting = false (버튼/입력 모두 활성)
- step = 'credentials' (2FA 아님)
- 초기 렌더 시 이메일 필드에 focus() 호출
  useEffect(() => { emailRef.current?.focus() }, [])

추가 로직:
- URL 쿼리에 ?redirect=... 가 있으면 useRef에 저장
- localStorage에서 last_email 읽어 defaultValues.email로 주입
- onSubmit 시작 시점에 상태를 'submitting'으로 전환 (02-제출중)
```
