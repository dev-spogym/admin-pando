## 상태 진입 조건
API 응답이 다음 중 하나:
- 401 → `E401001` 잘못된 자격증명
- 423 → `E423001` 계정 잠금
- 500 → `E500001` 서버 오류
- 503 → `05-서비스점검`으로 빠짐 (이 상태 아님)
- 네트워크/타임아웃 → `NETWORK`
- 429 → `E429001` 쿨다운(5회 연속 실패)

### 에러 코드별 UI 분기
| errorCode | 배너 tone | 비밀번호 필드 | 재설정 링크 | 버튼 | 추가 UI |
|---|---|---|---|---|---|
| `E401001` | error | 값 초기화, `border-red-300` | 보통 | 활성 | — |
| `E423001` | error | `disabled` 처리 | **강조** (`text-red-600 font-semibold underline`) | 활성 | "계정 잠금 해제 문의" 보조 링크 |
| `E500001` | error | 유지 | 보통 | 활성 | "재시도" 보조 텍스트 "문제가 지속되면 관리자에게 문의" |
| `NETWORK` | warn | 유지 | 보통 | 활성 | 재시도 아이콘 |
| `E429001` | warn | `disabled` | 보통 | `disabled` + 카운트다운 `"30초 후 재시도 (NN)"` |

## 상태 고유 인터랙션
1. **에러 배너 표시 즉시** → 배너 DOM으로 포커스 이동 (`ref.focus()`)
2. **재입력 시작** → 에러 배너 **유지** (입력 중 숨기지 않음), 필드 `border` 만 `border-red-300` 유지
3. **재제출** → `02-제출중` 전환, 이때 에러 배너 제거 + 필드 테두리 기본 복원
4. **423 계정 잠금 링크 클릭** → `/reset-password?lock=true` 이동
5. **Esc 키** → 에러 배너 닫기(선택, 배너는 닫아도 필드 테두리는 유지)

## 상태 고유 비즈니스 룰 델타
- 에러 코드별 **다른 시각적 톤** 사용 (에러 vs 경고)
- 401 발생 횟수 카운트: `useRef(failCount)` 5회 도달 시 → `E429001` 쿨다운 전환(30초)
- 쿨다운 중에는 버튼 `disabled` + 카운트다운 라벨 "{N}초 후 재시도"
- 감사 로그: 각 실패마다 `AUDIT.LOGIN_FAILURE`(사유코드+IP+UA) 서버에서 기록
- **비밀번호 자동 지움**: 401 수신 시 `setValue('password', '')` → 재입력 유도
- 성공(`02`) 전환 시 `failCount = 0` 리셋

## 에지 케이스
- **2개 이상 에러 동시 발생 불가** (서버가 단일 errorCode 반환)
- **페이지 새로고침** → 에러 상태 휘발, `01-기본`으로 복원
- **엔터 연타로 연속 실패** → 디바운스로 200ms 가드
- **네트워크 일시 단절 → 복구**: 재시도 버튼 자동 노출
- **423 잠금 해제 대기**: 비밀번호 재설정 링크 강조 / 클라이언트에서 잠금 시간 알 수 없음 → 안내만

## 🧩 바이브코딩 프롬프트 (델타)
```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: login-error (에러)

이 상태에서만 적용할 변경점:
- errorCode !== null (union type 'E401001'|'E423001'|'E500001'|'E429001'|'NETWORK')
- isSubmitting = false (재시도 가능)

에러 배너 컴포넌트 렌더:
<div role="alert" aria-live="assertive" ref={bannerRef}
     className={cn(
       'flex items-start gap-2 rounded-lg border p-3 text-sm',
       tone === 'error' && 'bg-red-50 border-red-200 text-red-700',
       tone === 'warn'  && 'bg-amber-50 border-amber-200 text-amber-800'
     )}>
  <AlertCircle className="size-4 shrink-0 mt-0.5" />
  <div>
    <p className="font-medium">{ERROR_MESSAGES[errorCode]}</p>
    {errorCode === 'E423001' && (
      <Link href="/reset-password?lock=true"
            className="text-red-700 font-semibold underline mt-1 inline-block">
        비밀번호 재설정으로 잠금 해제
      </Link>
    )}
    {errorCode === 'E429001' && (
      <p className="text-xs mt-1">{cooldownLabel}초 후 재시도 가능합니다</p>
    )}
  </div>
</div>

메시지 맵:
const ERROR_MESSAGES = {
  E401001: '이메일 또는 비밀번호가 올바르지 않습니다.',
  E423001: '계정이 잠겼습니다. 관리자에게 문의하세요.',
  E500001: '서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.',
  E429001: '로그인 시도가 너무 많습니다. 잠시 후 다시 시도해주세요.',
  NETWORK: '네트워크 연결을 확인해주세요.',
};

필드 처리:
- 입력 필드 className에 errorCode 있을 때 'border-red-300 focus:ring-red-500' 추가
- 비밀번호 필드: errorCode === 'E401001' → useEffect(() => setValue('password',''), [errorCode])
- 비밀번호 필드: errorCode === 'E423001' || errorCode === 'E429001' → disabled
- aria-invalid={errorCode != null}
- aria-describedby="login-error-banner"

포커스 이동:
useEffect(() => {
  if (errorCode) {
    bannerRef.current?.focus();
    // 또는 passwordRef.current?.focus() for E401001
  }
}, [errorCode]);

쿨다운 UI (E429001):
useEffect(() => {
  if (errorCode !== 'E429001') return;
  const id = setInterval(() => {
    const remain = Math.ceil((cooldownUntil - Date.now()) / 1000);
    if (remain <= 0) { setErrorCode(null); clearInterval(id); }
    else setCooldownLabel(remain);
  }, 500);
  return () => clearInterval(id);
}, [errorCode, cooldownUntil]);
```
