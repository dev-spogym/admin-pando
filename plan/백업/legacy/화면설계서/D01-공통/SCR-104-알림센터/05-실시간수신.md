## 상태 진입 조건

- Supabase Realtime 채널 `notifications:user:{userId}` 에서 `INSERT` 이벤트 수신
- `payload.new` 가 현재 사용자 `user_id` 기준 필터 통과
- `payload.new.branchId` 가 접근 가능 범위(멀티테넌트) 내 — 서버에서 필터링 후 전송
- 알림 센터가 포커스되어 있거나 백그라운드 탭이어도 구독 활성

### 애니메이션 시퀀스 (총 600ms)

| 구간 | 시간 | 효과 |
|---|---|---|
| T0~T100ms | 진입 | 헤더/탭 카운트 `+1` (transition `all 200ms ease-out`) |
| T0~T400ms | slideDown | 신규 항목 `translateY(-100%) → 0`, `opacity 0 → 1`, `height 0 → auto` |
| T0~T300ms | Toast | 우상단에서 `slide-in` (sonner 기본) |
| T0~T1000ms | Badge pulse | 헤더/사이드바 미읽음 뱃지 `animate-pulse` |
| T400ms~T600ms | Ring glow | 신규 항목 `ring-2 ring-blue-400` → 페이드아웃 |
| T600ms | 종료 | 일반 `02-정상` 스타일로 안정화 |

### 변경 요소

| 영역 | 정상 | 실시간 수신 |
|---|---|---|
| 헤더 카운트 | 정적 | `animate-pulse` + 화살표 `↑` |
| TabNav 카운트 | 정적 | transition 애니메이션 |
| 신규 항목 | 일반 bg-blue-50 | `animate-[slideDown_400ms] + ring-2 ring-blue-400` (600ms) |
| Toast | 없음 | 우상단 토스트 (4초 자동 닫힘) |
| 뱃지 (헤더/사이드바) | 정적 숫자 | `animate-pulse` (1s) |

### 포커스 상태별 표현

| 상태 | 동작 |
|---|---|
| 알림 센터 포커스 중 | 목록 상단 삽입 + 뱃지 +1 + 토스트 **미표시**(중복 방지) |
| 다른 페이지(앱 내) | 목록 상단 삽입(백그라운드) + 뱃지 +1 + 토스트 **표시** |

## 상태 고유 인터랙션

1. **WS INSERT 수신** → `payload.new` 수령 → QueryClient 캐시 `prepend`
2. **Zustand 뱃지 갱신** → `useNotificationStore.getState().incrementUnread()`
3. **토스트 노출** (포커스 상태에 따라): 다른 페이지/숨김 상태 시만 표시
4. **사운드 재생** (사용자 설정 + reduced-motion 체크 + 야간 모드 체크)
5. **Browser Notification** (권한 허용 시): `audit.security`, `payment.failed`만 표시
6. **연속 수신 throttle**: 100ms 내 다수 INSERT → debounce로 한 번만 토스트

## 상태 고유 비즈니스 룰 델타

- **reconnect 시 중복 방지**: WS 재연결 직후 놓친 알림 fetch 시 `id`로 dedupe
- **최대 목록 길이**: 20개 초과 시 오래된 항목 제거(페이지 1 캐시 기준)
- **`activeTab === 'read'` 에서 INSERT**: 미읽음이므로 목록 미추가(카운트만 갱신)
- **`activeTab === 'unread'` 에서 INSERT**: 바로 목록 상단 추가
- **사용자 설정 존중**: 채널별 ON/OFF에 따라 토스트/사운드 제어
- **감사 로그**: `audit.security` 타입은 slideDown 대신 `animate-[shake]` + 빨간 ring
- **중복 수신 방지**: `item.id` set 으로 중복 체크 (30초 캐시)

## 에지 케이스

- **오프라인 복구 시 대량 INSERT 수신** → 배치 처리 (50개 단위) + "새 알림 N개 도착" 요약 토스트
- **prefers-reduced-motion** → slideDown/pulse/shake 모두 off, `opacity 0→1` 페이드만
- **탭 중복 오픈** → BroadcastChannel로 중복 알림 전역 dedupe
- **WS 연결 전 INSERT** → 폴링으로 보충 (30초 내 반영)
- **사용자가 수동으로 탭 전환 직후 INSERT** → 새 탭의 쿼리 캐시에만 반영
- **토스트 오픈 중 항목 클릭** → 토스트 자동 dismiss + 낙관적 읽음 처리
- **"모두 읽음" 직후 INSERT** → 신규 항목은 미읽음으로 추가(정상 동작)

실시간 구독 훅:
export function useNotificationRealtime(userId: number, branchId: number | null) {
  const qc = useQueryClient();
  const router = useRouter();
  const setUnread = useNotificationStore(s => s.incrementUnread);
  const dedupe = useRef(new Set<string>());

  useEffect(() => {
    const channel = supabase
      .channel(`notifications:user:${userId}`)
      .on('postgres_changes',
        { event: 'INSERT', schema: 'public', table: 'notifications',
          filter: `user_id=eq.${userId}` },
        (payload) => {
          const item = payload.new as Notification;
          if (dedupe.current.has(item.id)) return;
          dedupe.current.add(item.id);
          setTimeout(() => dedupe.current.delete(item.id), 30_000);

          // 캐시 업데이트 (1페이지 기준, 현재 활성 탭)
          const updateFn = (old: any) => old && ({
            ...old,
            items: [item, ...old.items].slice(0, 20),
            total: old.total + 1,
            unreadCount: old.unreadCount + 1,
          });
          qc.setQueryData(['notifications', 'all', 1, branchId], updateFn);
          qc.setQueryData(['notifications', 'unread', 1, branchId], updateFn);
          setUnread();

          // 토스트 조건부
          const isOnThisPage = window.location.pathname === '/notifications';
          const isHidden = document.visibilityState === 'hidden';
          if (!isOnThisPage || isHidden) {
            toast(item.title, {
              description: item.body,
              duration: 4000,
              action: item.linkUrl
                ? { label: '상세보기', onClick: () => router.push(item.linkUrl!) }
                : undefined,
            });
          }
          if (item.type.startsWith('audit.security')) {
            toast.error(item.title, { description: item.body, duration: 10_000 });
          }

          // Browser Notification
          if (isHidden && 'Notification' in window && Notification.permission === 'granted'
              && ['audit.security', 'payment.failed'].includes(item.type)) {
            new Notification(item.title, { body: item.body, icon: '/favicon.ico' });
          }
        })
      .subscribe();
    return () => { supabase.removeChannel(channel); };
  }, [userId, branchId, qc, router, setUnread]);
}
