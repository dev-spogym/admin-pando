## 역할별 차이

- `superAdmin`: 스켈레톤 6행(전 지점 기본 조회량)
- `primary/owner/manager`: 스켈레톤 6행
- `fc/trainer/staff/front`: 스켈레톤 5행(본인 지점 평균 표시량)
- `readonly`: 스켈레톤 3행

## 상태 고유 인터랙션

1. **재시도 버튼(부모에서 호출)** → `refetch()` → `isLoading=true` 유지
2. **탭 전환 중 새 로딩** → 이전 데이터 `keepPreviousData=false` 기본(스켈레톤 재표시)
3. **키보드 Tab** → 헤더 액션 버튼까지만 포커스, 리스트 영역 `tabIndex=-1`
4. **취소 불가** — 사용자가 로딩을 중단할 수 없음(진행 중 탭 전환 시 이전 요청 abort)

## 상태 고유 비즈니스 룰 델타

- **최소 노출 시간**: 스켈레톤은 **최소 200ms** 노출(깜빡임 방지). 200ms 미만 응답 시에도 200ms까지 유지.
- **최대 대기 시간**: 10초 초과 시 `04-에러`(타임아웃) 자동 전환, `NETWORK` errorCode로 분기.
- **이전 요청 abort**: 탭/페이지 전환 시 `AbortController`로 기존 요청 취소.
- **실시간 구독은 유지**: WebSocket 채널은 로딩 중에도 활성(INSERT 수신 시 로드 완료 후 합산).

## 에지 케이스

- **캐시된 데이터 존재 시** React Query `placeholderData` 사용 → 스켈레톤 대신 stale 데이터 + 상단 프로그레스 바(`h-0.5 bg-blue-500 animate-pulse`)
- **오프라인 상태** → 스켈레톤 대신 즉시 `04-에러` (NETWORK)
- **WebSocket으로 INSERT 수신** → 로딩 중에도 큐잉, 완료 후 병합
- **slow-3G 환경** → 10초 제한 내 완료 시도, 초과 시 에러

최소 노출 가드:
const [showSkel, setShowSkel] = useState(true);
useEffect(() => {
  if (isLoading) { setShowSkel(true); return; }
  const t = setTimeout(() => setShowSkel(false), 200);
  return () => clearTimeout(t);
}, [isLoading]);

useEffect(() => {
