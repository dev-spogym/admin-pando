## 상태 진입 조건
- 모든 필수 쿼리 성공(200)이지만 `stats.totalMembers===0 && 위젯 items 모두 []`
- 신규 지점 초기 상태 또는 기간 필터 범위에 데이터 없음


| 요소 | 변경 |
|---|---|
| KPI 카드 | 값 `0` 또는 `-` 로 렌더(스켈레톤 아님) |
| 차트 | 중앙 "데이터 없음" + sr-only 대체 텍스트 |
| 리스트 위젯 | 빈 상태 메시지 + 위젯별 CTA |
| 감사로그 | 비표시 또는 "활동 기록이 없습니다" |
| EmptyState 카드 | 메인 영역 중앙에 통합 안내 카드(옵션) |
| 아이콘 | `LayoutDashboard size-16 text-gray-300 mx-auto` |

## 상태 고유 인터랙션
1. "회원 등록하기" → `/members/new` (권한 있는 역할만)
2. "상품 만들기" → `/products/new` (super/primary/owner만)
3. "출석 관리" → `/attendance` (front)
4. 기간 필터 제공 화면이면 `"다른 기간 보기"` 유도 문구 포함

## 역할별 차이
| 역할 | EmptyState CTA |
|---|---|
| superAdmin/primary/owner | 회원 등록 + 상품 만들기 + 지점 초기화 가이드 |
| manager | 회원 등록 + 출석 관리 |
| fc | 회원 등록 + 상담 예약 |
| trainer | 본인 담당 수업이 없으므로 "수업 배정을 기다리는 중" 안내 |
| staff | 회원 목록 보기 |
| front | 출석 관리 바로가기 |

## 상태 고유 비즈니스 룰 델타
- 빈 상태 판정: `stats.totalMembers===0 && weeklyAttendance.every(d=>d.count===0)`
- 신규 지점 전용 가이드(옵션): super/primary 에게 "온보딩으로 이동" 버튼 노출 (`/onboarding`)
- CTA 권한: 권한 없는 역할은 해당 CTA 숨김
- 감사로그: 신규 지점은 로그인/생성 이벤트만 존재 → 그대로 표시

## 에지 케이스
- **기간 필터 '오늘'만 0건, '이번달'은 있음**: "다른 기간에 데이터가 있습니다" 토스트 유도
- **데이터 생성 직후**: 낙관적 업데이트가 없으면 새로고침 유도
- **i18n**: 문구는 ko-KR 고정
