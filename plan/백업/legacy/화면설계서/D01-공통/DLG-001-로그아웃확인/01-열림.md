## 상태 진입 조건
- `isOpen === true`
- `processing === false`
- `useSessionStore.open === false` (세션만료 모달이 열려 있지 않음 — 우선순위 충돌 방지)

## 상태 고유 비주얼 델타
**마스터 기본 렌더 그대로**.
- 모달 카드 정상 렌더
- "로그아웃" 버튼: 활성(`bg-rose-600`), 오토포커스
- "취소" 버튼: 활성
- X 버튼: 활성

## 상태 고유 인터랙션
1. **오픈 직후**: Primary "로그아웃" 포커스 (Enter 즉시 진행 가능)
2. **ESC**: `onClose()` 호출 → 닫힘
3. **배경 클릭**: `onClose()` 호출 → 닫힘
4. **X 버튼**: `onClose()` 호출 → 닫힘
5. **"취소" 버튼**: `onClose()` 호출 → 닫힘
6. **"로그아웃" 버튼 / Enter**: `02-로그아웃처리중` 전환 (`handleConfirm()`)
7. **Tab 순환**: 취소 ↔ 로그아웃 ↔ X (focus trap)

## 역할별 차이
- 차이 없음(모든 역할 동일 UI/카피)

## 상태 고유 비즈니스 룰 델타
- 오픈 시 `document.body.style.overflow = 'hidden'`
- `useEffect` 클린업에서 복원
- DLG-000(세션만료) 오픈 중이면 이 모달은 render 자체를 생략(`if (sessionOpen) return null;`)
- DLG-002(이탈경고)가 뜨려고 하면 DLG-001 오픈 중 취소 → 로그아웃 의도 우선

## 에지 케이스
- **모바일 키보드 상단 영역**: 모달이 항상 center 정렬 유지 → 키보드 열림 시 뷰포트 resize 대응
- **트리거 직후 네트워크 끊김**: 오픈은 정상, 처리 단계에서 실패 처리
- **여러 번 빠른 연속 클릭**: `isOpen` 가드로 중복 오픈 방지
