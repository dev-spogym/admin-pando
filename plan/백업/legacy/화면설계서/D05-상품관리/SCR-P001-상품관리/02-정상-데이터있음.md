## 상태 고유 인터랙션
- **행 클릭** → `06-패널열림` 전환, 테이블 컬럼 13→6 축소.
- **컬럼 헤더 클릭**(상품명/현금가/카드가/기간/횟수/판매) → 정렬 토글(오름/내림/해제), `aria-sort` 갱신.
- **+ 상품 등록** → `06-패널열림`(신규 모드), `selectedProduct=null`.
- **전지점 배포** → DLG-P001-전지점배포 오픈.
- **Excel** → 파일 다운로드 + 토스트 `${N}건 엑셀 다운로드 완료`.
- **필터 변경** → `filteredData` 갱신, 빈 결과면 "검색 조건에 맞는 상품이 없습니다" 인라인(별도 상태 전환 없음).
- **검색 debounce** 200ms.
- **키보드**: 행에 Tab 포커스 → Enter/Space → 패널 오픈.

## 비즈니스 룰 델타
- `salesCountMap.get(product.name) ?? 0` → 판매 컬럼 렌더.
- 패키지 감지: `JSON.parse(description)?.isPackage === true` (try/catch, 실패 시 false).
- 타입 배지 미존재(`productType===null`) → 배지 없음 + 이름만 표시.
- usage_restrictions 없음 → 월~일 전부 ✓, 시간 "전체".
- `pointAccrual !== false` → ✓ (null 관대 처리).
- 정렬 우선순위: 사용자 클릭 정렬 > 기본(상품명 ASC).
- 필터 변경은 URL 쿼리에 반영 X (상태는 local).

## 에지 케이스
- filteredData 길이 0 (필터 결과 없음) → 테이블 빈 내부에 "선택한 조건에 맞는 상품이 없습니다." + [필터 초기화] 버튼. `02-정상` 상태 유지 (빈 상태 전환 X).
- salesCountMap fetch 실패 → 판매 컬럼 `"-"`, 경고 토스트 없음.
- 같은 이름 상품이 두 개(레거시) → 판매 카운트 공유(알려진 제약).
- usage_restrictions JSON 깨짐 → 전부 ✓ fallback.
