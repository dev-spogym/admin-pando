## 상태 진입 조건

- `/products/edit?id=N` 진입
- `GET /products/:id` 200 응답 수신 + `applyProductToForm(form, data)` 완료
- `isDirty=false` (prefill 직후), 이후 사용자 변경 시 `isDirty=true`

## 비주얼 델타

```
헤더: [📦] 상품 수정   [ACTIVE 배지 | INACTIVE 배지]
sticky footer primary: [수정]
prefill 후 모든 필드에 기존 값 반영
```

| 요소 | 수정 모드 값 |
|---|---|
| 헤더 타이틀 | "상품 수정" |
| 활성/비활성 배지 | `isActive=true → 활성 (bg-emerald-100 text-emerald-700)`, `false → 비활성 (bg-gray-100 text-gray-600)` |
| primary 버튼 라벨 | "수정" |
| 상품구분 라디오 | 기존 값 선택 (변경 가능) |
| 요일 체크박스 | 기존 weekdays 반영 |

## 상태 고유 인터랙션

- 가격 변경 시 내부 플래그 `priceChanged=true` → 저장 시 `audit_log` INSERT 트리거
- 상품명 변경 시 제출 직전 `check-name?excludeId=${id}` 호출 (자신 제외 중복 검사)
- `신규` 버튼 → DLG-P002 → "돌아가기" 시 `/products/new` 이동 (edit 컨텍스트 종료)

## 상태 고유 비즈니스 룰 (델타)

1. 저장 성공 시 **PATCH** 경로.
2. 가격 변경 있으면 `audit_log` INSERT (target=product, action=UPDATE, before/after price).
3. 상품명 + branchId 기존 일치 (자기 자신) → 중복 아님으로 처리.
4. `isActive` 토글 변경(true→false) 시 레거시 폼에서는 즉시 저장되지 않고 제출 시 반영. (SCR-P003 패널은 DLG-P004 연계)

## 에지 케이스

- 다른 탭에서 같은 상품 동시 편집 → 낙관적 락 없음 → 마지막 저장이 덮어씀. 추후 `updatedAt` 기반 충돌 감지 고려.
- prefill 중 사용자가 입력 시작 → prefill 완료 후 사용자 값 유지 (`applyProductToForm`은 `shouldDirty:false`로 호출)

## 바이브코딩 프롬프트 (델타)

```
// 마스터 프롬프트를 기본으로 하되, 다음만 오버라이드
mode = 'edit'
id = searchParams.get('id')
useQuery(['product', id], () => getProduct(id), { enabled: !!id, staleTime: 0 })
onPrefill: applyProductToForm(form, data, { shouldDirty: false })
header title = '상품 수정'
headerBadge = <StatusBadge status={product.isActive ? 'ACTIVE' : 'INACTIVE'} />
submit button label = '수정'
onSubmit: updateProduct(id, values).then(() => {
  if (priceChanged(prevPrice, values.priceCash)) {
    createAuditLog({ target:'product', action:'UPDATE', id,
                     before:{price:prevPrice}, after:{price:values.priceCash} });
  }
});
```
