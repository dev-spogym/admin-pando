# DLG-C009 템플릿 등록/수정 — 상태: 수정 폼

> 이 문서는 `00-기본화면.md` 마스터의 **델타**만 기술합니다.

## 0. 메타

| 항목 | 값 |
|---|---|
| 상태 코드 | `form-edit` |
| 다이어그램 노드 | `STATE_EDIT` |
| 이전 상태 | `01-닫힘` |
| 다음 상태 | `05-저장결과` / `04-미저장확인` |

## 1. 상태 진입 조건

- `SCR-C004` 행의 `Edit2` 버튼 클릭
- `mode === 'edit'`, `template` 객체 존재, `isOpen === true`
- `GET /api/lesson-templates/{id}` 성공 응답으로 `template` 채워짐

## 2. 비주얼 델타

- **Header 제목**: "그룹수업 템플릿 수정"
- **Submit 버튼 라벨**: "수정 저장"
- **폼 필드**: `reset(template)`으로 프리필
  - 템플릿명/분류/시간/정원/색상/반복요일/설명/활성 — 기존 값 표시
- **isDirty=false 시 Submit disabled** (변경 없으면 저장 불가)


## 3. 역할별 차이

- `manager/owner/superAdmin`: 전체 필드 수정 가능
- `trainer`: 본인 지점 템플릿만 수정 가능 (서버 측 RLS 이중 검증)
- `fc/staff/front/readonly`: 진입 불가

## 4. 상태 고유 인터랙션

- **초기 프리필**: `useEffect(() => { if (mode==='edit' && template) reset(template) }, [mode, template])`
- **isDirty 감지**: 모든 인풋 변경 시 true
- **취소 동작**: `isDirty=true` → `04-미저장확인`, false → 즉시 `01-닫힘`
- **Submit**: `isDirty=true && isValid=true`일 때만 활성

## 5. 비즈니스 룰 델타

1. **Optimistic Lock 방지**: 서버는 `updated_at` 기반 ETag 검증 (E409 충돌 시 `05-저장결과` 실패 분기)
2. **성공 토스트 메시지**: `"템플릿이 수정되었습니다."`
3. **API**: `PUT /api/lesson-templates/{id}`
4. **비활성 토글**: `isActive=false` 저장 시 캘린더에서 더 이상 템플릿으로 노출되지 않음 (기존 수업은 유지)
5. **감사 로그**: 변경 필드 diff 포함 (`AUDIT.LESSON_TEMPLATE.UPDATE`)

## 6. 에지 케이스

- **동시 수정 충돌**: 다른 사용자가 먼저 수정 → 서버 409 → "다른 관리자가 먼저 수정했습니다. 새로고침 후 다시 시도해주세요." 토스트 + 모달 유지
- **삭제된 템플릿 편집 시도**: 404 → "삭제된 템플릿입니다." 토스트 + 모달 닫힘
- **부분 입력 후 네트워크 단절**: 로컬 상태 유지 → 재연결 후 저장 재시도 가능
- **수정 직전 비활성된 기존 템플릿 편집**: 가능 (`isActive` 토글로 재활성화)

## 7. 바이브코딩 프롬프트 (델타)

```
모드: edit
props: template: ClassTemplate (필수)

useEffect(() => {
  if (mode === 'edit' && template) {
    reset({
      name: template.name,
      category: template.category,
      duration: template.duration,
      capacity: template.capacity,
      color: template.color ?? '#6366f1',
      repeatDays: template.repeatDays ?? [],
      description: template.description ?? '',
      isActive: template.isActive,
    })
  }
}, [mode, template, reset])

Title: "그룹수업 템플릿 수정"
Submit label: "수정 저장"
Submit disabled: !isDirty || !isValid || isSubmitting

Submit action:
  await api.put(`/api/lesson-templates/${template.id}`, { ...data })
  toast.success("템플릿이 수정되었습니다.")

에러 처리:
  409 (동시 충돌): toast.warning("다른 관리자가 먼저 수정했습니다. 새로고침 후 다시 시도해주세요.")
  404 (삭제됨):   toast.error("삭제된 템플릿입니다.") + onClose()
```
