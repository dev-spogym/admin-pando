# DLG-C009 템플릿 등록/수정 — 상태: 미저장 확인

> 이 문서는 `00-기본화면.md` 마스터의 **델타**만 기술합니다.

## 0. 메타

| 항목 | 값 |
|---|---|
| 상태 코드 | `unsaved-confirm` |
| 다이어그램 노드 | `STATE_UNSAVED_CONFIRM` |
| 이전 상태 | `02-신규폼` / `03-수정폼` (isDirty=true + 닫기 시도) |
| 다음 상태 | 버리기 → `01-닫힘` / 계속편집 → 이전 상태 복귀 |

## 1. 상태 진입 조건

- `formState.isDirty === true`인 상태에서 아래 중 하나 발생:
  - 취소 버튼 클릭
  - X 버튼 클릭
  - Esc 키
  - Overlay 클릭

## 2. 비주얼 델타

**확인 다이얼로그가 본 모달 위에 겹쳐서 표시** (중첩 AlertDialog, z-index 1상승):

```
┌────────────────────────────────────┐
│ [기존 DLG-C009 폼이 뒤에 흐리게 보임] │
│  ┌─────────────────────────────┐   │
│  │ 저장하지 않고 닫으시겠습니까?│   │
│  │                             │   │
│  │ 변경한 내용이 있습니다.       │   │
│  │ 닫으면 입력한 내용이 사라집니다.│ │
│  │                             │   │
│  │  [ 계속 편집 ] [ 버리고 닫기 ] │   │
│  └─────────────────────────────┘   │
└────────────────────────────────────┘
```

- AlertDialog 크기: `w-[360px]`
- 배경 블러: 본 모달 `opacity-50` + pointer-events-none
- "버리고 닫기" 버튼: `variant="destructive"` (빨강)
- "계속 편집" 버튼: `variant="outline"` (기본 포커스)

## 3. 역할별 차이

- 동일 (모든 역할에서 같은 UX)

## 4. 상태 고유 인터랙션

- **기본 포커스**: "계속 편집" 버튼 (안전한 기본값)
- **Enter 키**: 포커스된 버튼 실행
- **Esc 키**: AlertDialog 닫기 + 본 모달 유지 (계속 편집과 동일)
- **Overlay 클릭**: 동일하게 계속 편집으로 처리
- **Tab**: 두 버튼 사이 순환

## 5. 비즈니스 룰 델타

1. **트리거 조건**: `isDirty === true` 일 때만 표시. false면 바로 `01-닫힘`.
2. **"버리고 닫기"**: `reset(defaultValues); setShowUnsaved(false); onClose()` — 본 모달 닫힘.
3. **"계속 편집"**: `setShowUnsaved(false)` — 본 모달 유지, 이전 상태(`02-신규폼` 또는 `03-수정폼`) 복귀, 포커스는 마지막 필드로 복귀.
4. **자동 저장 없음**: 입력값 localStorage에 보존하지 않음(민감도 고려).

## 6. 에지 케이스

- 연속 Esc: 첫 Esc로 AlertDialog 열림, 두 번째 Esc로 AlertDialog 닫힘(계속 편집). 본 모달은 유지.
- Submit 중 Esc: `isSubmitting=true`일 때 Esc 무시(의도치 않은 취소 방지).
- 브라우저 닫기(`beforeunload`): 추가 네이티브 확인 트리거 (`event.preventDefault(); event.returnValue = ''`).
- 라우트 변경 시도: Next.js `router.events` 인터셉트 또는 `useBeforeUnload`.

## 7. 바이브코딩 프롬프트 (델타)

```
컴포넌트: AlertDialog (Radix) — 본 모달 내부에서 conditional render

state:
  const [showUnsaved, setShowUnsaved] = useState(false)

트리거:
  const handleClose = () => {
    if (formState.isDirty) setShowUnsaved(true)
    else onClose()
  }
  <Dialog.Content
    onEscapeKeyDown={(e) => { if(isDirty){ e.preventDefault(); setShowUnsaved(true) } }}
    onPointerDownOutside={(e) => { if(isDirty){ e.preventDefault(); setShowUnsaved(true) } }}
  />
  <button onClick={handleClose}>취소</button>

AlertDialog:
  <AlertDialog.Root open={showUnsaved} onOpenChange={setShowUnsaved}>
    <AlertDialog.Portal>
      <AlertDialog.Overlay className="fixed inset-0 bg-black/30" />
      <AlertDialog.Content
        className="fixed left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2
                   w-[360px] bg-white rounded-xl shadow-2xl p-6 space-y-4 z-[60]">
        <AlertDialog.Title className="text-base font-semibold text-gray-900">
          저장하지 않고 닫으시겠습니까?
        </AlertDialog.Title>
        <AlertDialog.Description className="text-sm text-gray-600">
          변경한 내용이 있습니다. 닫으면 입력한 내용이 사라집니다.
        </AlertDialog.Description>
        <div className="flex justify-end gap-2">
          <AlertDialog.Cancel asChild>
            <button autoFocus className="h-9 px-4 rounded-lg border border-gray-300 text-sm hover:bg-gray-50">
              계속 편집
            </button>
          </AlertDialog.Cancel>
          <AlertDialog.Action asChild>
            <button
              onClick={() => { reset(defaultValues); setShowUnsaved(false); onClose() }}
              className="h-9 px-4 rounded-lg bg-red-600 hover:bg-red-700 text-white text-sm font-medium">
              버리고 닫기
            </button>
          </AlertDialog.Action>
        </div>
      </AlertDialog.Content>
    </AlertDialog.Portal>
  </AlertDialog.Root>

beforeunload 차단 (추가):
  useEffect(() => {
    if (!isDirty) return
    const handler = (e: BeforeUnloadEvent) => { e.preventDefault(); e.returnValue = '' }
    window.addEventListener('beforeunload', handler)
    return () => window.removeEventListener('beforeunload', handler)
  }, [isDirty])
```
