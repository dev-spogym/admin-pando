## 상태 고유 인터랙션
1. **[처리하기] 클릭** → `/schedule-requests` 이동 (SCR-C009)
2. **캘린더 pending 이벤트 클릭** → `DLG-C002` 오픈 (승인/거절 버튼 노출)
3. **DLG-C002에서 승인/거절** → `pendingCount` decrement → count=0 시 배너 자동 제거 (02 상태로)
4. **Bell 아이콘 클릭** → 토스트 "미승인 일정은 [처리하기] 버튼으로 확인할 수 있습니다."

## 상태 고유 비즈니스 룰 델타
- **count 폴링**: `refetchInterval: 60_000` (60초마다 재조회)
- **count 계산 범위**:
  - super/primary: 전 지점 (branchId=null)
  - owner/manager/fc: `branchId=user.branchId`
  - trainer: `branchId=user.branchId AND instructorId=user.id`
- **승인/거절 성공 후 mutation onSuccess**: `invalidate(['pendingCount'])` + `invalidate(['classes'])`
- **count=0 즉시 배너 제거**: React `{pendingCount>0 && <Banner/>}`

## 에지 케이스
- pendingCount 조회 실패 → 배너 미표시 (graceful fallback, 에러 토스트 없음)
- pendingCount >= 100 → "99+ 건" 표시
- trainer + 본인 대상 pending 없음 → 배너 숨김
- 배너와 다른 알림 동시 노출 → 세로 스택
- 탭 전환 후 복귀 → 최신 count로 자동 refetch
