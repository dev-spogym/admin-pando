## 상태 고유 인터랙션
1. 상세 (Eye) 클릭 → `DLG-C005` (RecordDetailModal)
2. DLG-C005 내 [수업 시작] → PATCH lesson_status='in_progress' → 토스트
3. DLG-C005 내 [완료 처리] → `DLG-C006` (SignatureModal)
4. DLG-C006 서명 → Storage 업로드 → lesson_status='completed'
5. [노쇼] → PATCH lesson_status='no_show' + policy.autoDeduct ON이면 session -1
6. [취소] → PATCH lesson_status='cancelled'
7. 엑셀 다운로드 (있으면) → `exportToExcel(records)`

## 상태 고유 비즈니스 룰 델타
- **정렬**: 일시 내림차순 기본 (최신 먼저)
- **페이지네이션**: 기본 20건, 20/50/100 선택 가능
- **상태 필터**: (선택 옵션) 상단 필터 바 `[전체|예정|진행중|완료|노쇼|취소]`
- **실시간 갱신**: 상태 변경 후 invalidate(['records']) → 해당 행만 optimistic 업데이트
- **서명 upsert**: 이미 서명 있으면 대기→완료 직접 전환 가능 (관리자 overrides)
- **노쇼 세션 차감**:
  ```
  policy.autoDeduct = true이면:
    PATCH /lesson-counts/:memberSessionId/deduct { reason:'no_show', classId }
  ```

## 에지 케이스
- 수업 시작 10분 전 [수업 시작] 버튼 노출 (그 전엔 disabled)
- 종료 시간 경과 후 자동 [완료 처리] 제안 (policy.autoCompleteAfter)
- 회원 없는 수업 (그룹): 회원명 "-", 서명 생략 가능 (수업 기록만)
- 대량 completed → 엑셀 다운로드 시 `N>10000` 경고
