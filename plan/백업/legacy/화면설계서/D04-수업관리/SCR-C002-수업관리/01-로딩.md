## 역할별 차이
모든 역할 동일 스켈레톤. staff/front는 버튼이 원래 숨김이므로 델타 없음.

## 상태 고유 인터랙션
1. 모든 버튼/행 클릭 비활성
2. TabNav 클릭 가능 (탭 전환 시 새 로딩 시작)
3. 10초 초과 → slow network 배너
4. 30초 초과 → `06-에러` 자동 전환

## 상태 고유 비즈니스 룰 델타
- 3개 쿼리 (`lessons`, `records`, `stats`) 병렬 호출
- stats만 먼저 완료 시 카드부터 채움 (progressive)
- `placeholderData: keepPreviousData` 사용
- 탭 전환 시 해당 탭 쿼리만 `isFetching=true`

## 에지 케이스
- stats만 실패 → 테이블만 정상 (카드에 "—" 표시)
- records 첫 페이지만 로딩, 이후 infinite scroll → 하단 `<Spinner>`
- 빠른 응답 (<200ms) → 스켈레톤 깜빡임 방지 (`useDeferredValue`)

