## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 삭제 확인 다이얼로그를 작성하라.
화면: DLG-C002 일정상세 — 상태: 삭제 확인

파일: src/components/classes/LessonDetailModal.tsx (내부 상태)

삭제 확인 다이얼로그 (showDeleteConfirm=true):
- 모달 위에 중첩된 소형 다이얼로그 (z-60):
<div className="absolute inset-0 bg-black/30 flex items-center justify-center rounded-xl">
  <div className="bg-white rounded-lg p-6 w-80 shadow-xl">
    <h3 className="font-semibold text-gray-800">수업을 삭제하시겠습니까?</h3>
    <p className="mt-2 text-sm text-gray-500">
      삭제된 수업은 복구할 수 없습니다.
    </p>
    <div className="mt-4 flex justify-end gap-3">
      <Button variant="outline" onClick={() => setShowDeleteConfirm(false)}>
        취소
      </Button>
      <Button variant="destructive" onClick={handleDelete} disabled={isDeleting}>
        {isDeleting ? <Loader2 className="animate-spin w-4 h-4" /> : '삭제'}
      </Button>
    </div>
  </div>
</div>

handleDelete:
const handleDelete = async () => {
  setIsDeleting(true)
  try {
    await fetch(`/api/lessons/${lessonId}`, { method: 'DELETE' })
    toast.success('수업이 삭제되었습니다.')
    onSuccess()  // 캘린더 갱신
    onClose()
  } catch (error) {
    toast.error('삭제에 실패했습니다.')
  } finally {
    setIsDeleting(false)
    setShowDeleteConfirm(false)
  }
}
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- `03-정상-상세표시`에서 삭제 버튼 클릭

### 인터랙션 (User Actions)
1. "취소" → `03-정상-상세표시` 복귀
2. "삭제" → DELETE /api/lessons/:id → 성공 시 `01-닫힘` + 캘린더 갱신

### 비즈니스 룰
- 삭제 중 isDeleting=true: 버튼 disabled + Loader2 스피너
- 403: toast.warning "권한이 없습니다."
- 500: toast.error "삭제에 실패했습니다." + 모달 유지

### 연결 화면
- 확인: `01-닫힘` + SCR-C001 갱신
- 취소: `03-정상-상세표시`
