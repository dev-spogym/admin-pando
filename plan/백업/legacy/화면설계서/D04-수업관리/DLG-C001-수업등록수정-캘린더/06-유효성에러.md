## 🧩 바이브코딩 프롬프트

```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 유효성 에러 UI를 작성하라.
화면: DLG-C001 — 상태: 유효성 에러

파일: src/components/classes/LessonCalendarModal.tsx

Zod 스키마:
const lessonSchema = z.object({
  title: z.string().min(1, "수업명을 입력해주세요.").max(50, "50자 이내로 입력해주세요."),
  type: z.enum(['PT','GX','pilates','yoga','etc'], { required_error: "수업 분류를 선택해주세요." }),
  instructorId: z.string().min(1, "강사를 선택해주세요."),
  date: z.date({ required_error: "날짜를 선택해주세요." })
    .refine(d => d >= startOfDay(new Date()), "과거 날짜는 선택할 수 없습니다."),
  startTime: z.string().min(1, "시작 시간을 선택해주세요."),
  duration: z.number().min(10, "수업 시간은 최소 10분 이상이어야 합니다."),
  capacity: z.number().min(1).max(100, "정원은 1~100명 사이로 입력해주세요."),
})

인라인 에러 표시:
- 각 필드 하단: <p className="mt-1 text-xs text-red-500">{errors.field?.message}</p>
- 에러 필드 border: border-red-500 focus:ring-red-500
- 저장 버튼 클릭 시 첫 에러 필드로 자동 포커스 (setFocus)

시간 충돌 확인 (서버 사전 검증):
- 저장 전 GET /api/lesson-schedules/conflict?instructorId=?&start=?&end=?
- 충돌 시: confirm 다이얼로그
  "해당 강사에게 동일 시간대 수업이 있습니다. 계속 진행하시겠습니까?"
  - 확인: API 강제 저장 (forceCreate=true)
  - 취소: 폼 유지
```
