## 역할별 차이

| 역할 | 차이 |
|---|---|
| trainer | `lesson.instructorId === user.id` 아니면 진입 시 403 → 토스트 후 닫힘 |
| trainer | 담당 강사 변경 disabled (본인 고정) |
| 관리자 | 모든 필드 편집 가능 |
| readonly | 진입 불가 |

## 상태 고유 인터랙션

1. 프리필 값 편집 → `isDirty=true` → "수업 수정 완료" 버튼 활성
2. 필드 원래 값으로 복귀 시 `isDirty=false` → 버튼 disabled
3. 시간 변경 → 저장 시 시간 충돌 재검사 (자기 자신 `id` 제외)
4. 참여자 기존 목록 표시 + ✕ 제거 + 신규 추가
5. 좌석 행/열 0으로 변경 → 기존 좌석 배정 유지 여부 경고(백엔드 처리, UI는 안내만)
6. 반복 요일 변경 → 저장 시 "시리즈 전체 적용" vs "이 수업만" 선택 다이얼로그(기능명세 외, Phase 2)
7. "수정" 진입 → DLG-C002 자동 닫힘

## 상태 고유 비즈니스 룰 델타

- **이전 값 복원**: `reset(toFormValues(initial), { keepDirty:false })`
- **동시성(낙관적 잠금)**: `initial.updatedAt` 같이 전송 → 서버에서 불일치 시 409 "다른 사용자가 수정 중입니다" → toast + 최신값 refetch 후 diff 안내
- **시간 충돌 검사**: 자기 자신(`id===initial.id`) 제외
- **편집 가능 판정**: `isEventEditable(initial.start)` 규칙 (기능명세 §1.19). false면 저장 불가 + 배너 표시
- **참조 무결성**: 참여자 목록은 서버에서 `class_participants` 테이블과 diff 처리
- **로컬 이벤트**: `initial.id.startsWith('local-')`이면 DB 호출 없이 `localEvents` 배열만 업데이트
- **감사 로그**: 변경 필드 diff를 `AUDIT.CLASS_UPDATE` payload로 서버 기록

## 에지 케이스

- **완료/취소/노쇼 상태 수업 수정 시도**: 저장 버튼 disabled + 안내 "완료/취소된 수업은 수정할 수 없습니다"
- **삭제된 강사 표시**: Select에 `"(삭제됨) 김강사"` stale 옵션 제공, 변경 강제
- **사용자가 다른 탭에서 삭제한 수업**: 저장 시 404 → 토스트 "이미 삭제된 수업입니다" + 자동 닫힘 + 캘린더 refetch
- **참여자 5명 초과 기존 데이터**: 읽기 전용 표시, 추가 버튼 disabled
- **시간대 변경(DST)**: 서버 UTC 저장, 클라이언트 KST 렌더. 경계 시간(02:00~03:00) 경고
- **in_progress 수업의 강사 변경**: 서버에서 경고, 확인 후 진행
