## 상태 고유 인터랙션
1. **포커스**: 오픈 완료 시 "닫기"(Primary)에 오토포커스. 로딩 단계에서 이미 포커스가 있었다면 유지.
2. **키보드**:
   - `Esc` → 닫기
   - `Tab` → (X) → 회원상세 → 결제취소 → 영수증 → 닫기 → (X) 순환
   - `Enter` on 결제취소 → `/payment-cancel?saleId=...`
3. **회원 상세 클릭**: 모달 닫기 없이 `moveToPage(985, { id: buyerId })` 로 라우트 변경 → 다이얼로그 unmount.
4. **결제 취소 클릭**:
   - 사전 가드: `status === 'REFUNDED'` 면 토스트 "이미 환불된 매출입니다" (E409300)
   - 정상 흐름: `router.push('/payment-cancel?saleId=...')` 이동
5. **영수증 재발행 클릭**:
   - 팝업 열기 → receipt HTML 주입 → `print()` → 팝업 닫기
   - 성공 토스트 / 실패 토스트
   - 감사 로그 `AUDIT.RECEIPT_REPRINT` 전송(fire-and-forget)
6. **외부 데이터 변경(부모 테이블 mutate)**: React Query invalidate 시 silent update.

## 상태 고유 비즈니스 룰 델타
- 섹션 노출 순서: 구매 → 금액 → 결제 → (카드) → 서비스 → 비고. 이 순서 **변경 금지**.
- 환불/부분환불 매출은 "비고" 섹션 하단에 **"환불 내역 보기"** 링크를 추가 노출(→ DLG-S006 환불 상세로 전환).
- 메모 내용에 URL 이 포함되어도 **자동 링크화 금지**(XSS/피싱 방지) — plain text 렌더.
- 민감정보 복사 방지: 카드 정보 섹션의 셀은 `user-select: none` 옵션(정책 선택).

## 에지 케이스
- **금액 필드 NaN/null**: 서버에서 0 으로 기본값 보정 + 프론트 `Number.isFinite` 가드. null → 0.
- **할인 > 정가**: 이론상 불가(에러코드 E400302). 발생 시 붉은 경고 인라인 "데이터 이상 — 관리자에게 문의".
- **`cardNumber` 마스킹 실패(raw 노출)**: 개발용 검증 플래그 `NEXT_PUBLIC_DEV` 외에는 해당 필드를 `-` 로 표시(보안 우선).
- **네트워크 재연결 후 stale cache**: React Query refetch → 조용히 갱신, 사용자 커서/스크롤 위치 유지.
- **동시에 다른 사용자가 환불 처리**: 현재 모달 내용은 정상→환불로 바뀔 수 있음. `refetchOnWindowFocus: true` 로 대응. 재렌더 시 "결제 취소" 버튼 disabled 즉시 반영.
