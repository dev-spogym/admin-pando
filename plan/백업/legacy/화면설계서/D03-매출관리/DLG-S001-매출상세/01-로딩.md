## 역할별 차이
- 역할별 차이 없음(로딩 스켈레톤은 모든 역할 동일).

## 상태 고유 인터랙션
1. 오픈 직후 "닫기" 버튼에 오토포커스(로딩 중에도 닫기 가능).
2. ESC 키로 언제든 닫기 가능(fetch abort).
3. Tab 순환: 닫기 ↔ X 버튼 (다른 버튼은 disabled → focusable=false).
4. 로딩이 **300ms 미만**이면 스켈레톤 생략(깜빡임 방지) — `setTimeout(showSkeleton, 300)` 패턴.
5. 로딩이 **5초 초과**이면 하단에 "시간이 걸립니다" 캡션 노출(선택).

## 상태 고유 비즈니스 룰 델타
- 닫기 클릭 시 `AbortController.abort()` 로 fetch 중단.
- 동일 `saleId` 의 React Query 캐시가 있으면 `initialData` 로 로딩 생략하고 곧바로 `02-열림` 진입.
- 로딩 중에는 영수증 재발행/결제 취소/회원 상세 모두 **가드**(버튼 disabled + 클릭 이벤트 차단).

## 에지 케이스
- **saleId === null**: 다이얼로그 오픈 자체 차단(개발 실수 방지).
- **네트워크 지연 + 사용자 닫기**: 이후 fetch 완료해도 setState 가드로 렌더 무시.
- **캐시 stale + refetch**: 캐시 데이터로 즉시 `02-열림` 렌더 + 백그라운드 refetch → silent update.
- **부모 행 클릭 연타**: saleId 변경 시 이전 fetch abort, 마지막 saleId 기준으로 로딩 재시작.

## 🧩 바이브코딩 프롬프트 (델타)

```
상속: 00-기본화면.md §14 프롬프트 전체 적용
상태: sale-detail-loading (로딩)

이 상태에서만 적용할 변경점:
- q.isLoading === true 인 분기에서 <SaleDetailSkeleton /> 렌더
- 헤더 날짜/배지도 스켈레톤으로 치환
- footer의 회원상세/결제취소/영수증 버튼 disabled=true
- 닫기 버튼만 활성 + autoFocus

컴포넌트:
function SaleDetailSkeleton() {
  return (
    <div className="animate-pulse space-y-5" aria-busy="true" role="status" aria-label="매출 상세 불러오는 중">
      {[
        { title: '구매 정보', rows: 4 },
        { title: '금액 정보', rows: 3 },
        { title: '결제 내역', rows: 4 },
        { title: '서비스',    rows: 3 },
        { title: '비고',      rows: 1 },
      ].map(({ title, rows }) => (
        <section key={title} className="space-y-2">
          <div className="h-3 w-20 rounded bg-gray-200" aria-hidden />
          {Array.from({ length: rows }).map((_, i) => (
            <div key={i} className="grid grid-cols-[120px_1fr] gap-y-2">
              <div className="h-4 w-16 rounded bg-gray-200" aria-hidden />
              <div className="h-4 w-40 rounded bg-gray-200" aria-hidden />
            </div>
          ))}
        </section>
      ))}
    </div>
  );
}

접근성:
- 컨테이너에 aria-busy="true", role="status", aria-label="매출 상세 불러오는 중"
- Skeleton 요소는 aria-hidden="true"
```
