## 3. 상태 고유 비주얼 델타

### 3.1 UI 플로우

```
  02-처리중 ─(201 OK)─→ 03-성공
                         │
                         ├─ onSuccess?.()         (부모 refetch 트리거)
                         ├─ onClose()             (모달 언마운트)
                         ├─ toast.success(...)    (우상단 토스트 3s)
                         └─ SCR-S009 `02-정상`    (새 행 하이라이트 옵션)
```

### 3.2 변경 요소 표

| 요소 | 값 |
|---|---|
| 모달 렌더 | **언마운트** (`isOpen=false`) |
| 토스트 | `toast.success('할부가 등록되었습니다.')`, 자동 소멸 3s |
| 부모 상태 | `installments` 리스트 재조회 → 새 행 표시 |
| `isProcessing` | 컴포넌트 언마운트로 의미 없음 |
| 포커스 복귀 | SCR-S009 상단 [할부 등록] 버튼(부모 관리) |

### 3.3 Toast 스타일

```

          rounded-lg px-4 py-3 shadow
```

## 5. 상태 고유 인터랙션

| 이벤트 | 시점 | 결과 |
|---|---|---|
| `E_REG_DONE_01` | supabase 응답 | 3개 액션 연속 실행(아래 순서) |
| 1) 토스트 출력 | 즉시 | react-hot-toast `success` |
| 2) `onSuccess?.()` | 즉시 | 부모 `fetchInstallments()` 호출 |
| 3) `onClose()` | 즉시 | 모달 닫힘 |

> 순서 권장: toast → onSuccess → onClose. 언마운트 후 setState 경고 방지.

---

## 6. 상태 고유 비즈니스 룰 델타

1. **목록 갱신**: 부모는 `fetchInstallments()` 를 `useEffect` 의존성으로 실행하거나 React Query `queryClient.invalidateQueries(['installments', branchId])` 사용.
2. **감사로그**: 서버에서 `AUDIT.INSTALLMENT_CREATE` 레코드 생성. 클라이언트는 별도 기록 없음.
3. **연결 자동화(옵션)**: Phase 2에서 첫 회차 납입 예정 알림을 회원 앱에 푸시 발송하는 트리거 서버측 처리.
4. **대시보드 KPI 갱신**: `SCR-S011 매출예측` / `SCR-S006 선수익금조회`가 구독 중이면 자동 반영.

## 7. 에지 케이스

| 케이스 | 처리 |
|---|---|
| 성공 직후 부모가 이미 언마운트됨 | `onSuccess?.()` 안전 호출(옵셔널 체이닝) |

| 새 행 하이라이트 | SCR-S009가 `justAddedId` 로 2초 highlight (옵션) |

## 8. 바이브코딩 프롬프트 (델타)

```
상태: DLG-S009 — 03-성공

// 성공 분기 (02-처리중 onSubmit 내부)
if (!error) {
  toast.success('할부가 등록되었습니다.', { id: 'installment-create-success' });
  onSuccess?.();     // 부모 목록 refetch 또는 QueryClient invalidate
  onClose();         // 모달 언마운트
  return;
}

// 부모(SCR-S009)측 패턴
const queryClient = useQueryClient();
<InstallmentRegisterModal
  isOpen={openRegister}
  onClose={() => setOpenRegister(false)}
  onSuccess={() => {
    queryClient.invalidateQueries({ queryKey: ['installments', branchId] });
  }}
  branchId={branchId}
/>
```
