```
Next.js 15 App Router + TypeScript + Tailwind v4 기반 'use client' 컴포넌트를 작성하라.
화면: DLG-S005 메모편집 — 상태: 열림 (기존 메모 있음)

파일: src/app/unpaid/page.tsx 내 MemoEditModal

<Modal isOpen={isOpen} onClose={onClose} title="메모 편집">
  <div className="p-6">
    <label className="block text-sm font-medium text-gray-700 mb-2">
      메모 내용
    </label>
    <Textarea
      value={memoText}
      onChange={(e) => setMemoText(e.target.value)}
      placeholder="미수금 관련 메모를 입력하세요..."
      rows={5}
      maxLength={500}
      className="w-full border rounded-lg p-3 text-sm resize-none focus:ring-2 focus:ring-blue-500"
    />
    <div className="text-right text-xs text-gray-400 mt-1">

    <div className="flex gap-3 mt-4">
      <Button variant="outline" className="flex-1" onClick={onClose}>
        취소
      </Button>
      <Button
        variant="primary"
        className="flex-1"
        onClick={handleSave}
        disabled={isSaving}
      >
        {isSaving ? <Loader2 className="h-4 w-4 animate-spin mx-auto" /> : '저장'}
      </Button>
    </div>
  </div>
</Modal>

저장 로직:
const handleSave = async () => {
  setIsSaving(true);
  const { error } = await supabase
    .from('sales')
    .update({ memo: memoText })
    .eq('id', unpaidId);

  if (error) {
    toast.error('메모 저장에 실패했습니다.');
    console.error('[DLG-S005]', error);
  } else {
    toast.success('메모가 저장되었습니다.');
    onClose();
    refreshList(); // SCR-S008 목록 갱신
  }
  setIsSaving(false);
};

사용 유틸:
- Textarea from '@/components/ui/Textarea'
- Loader2 from 'lucide-react'
- supabase, toast
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- E_LOAD_OK_01: 기존 메모 있음 → 텍스트영역에 기존 내용 표시

### 필수 데이터
| 항목 | 설명 |
|------|------|
| memoText | 기존 메모 (미리 채워짐) |
| unpaidId | 해당 미수금 건 ID |

### 인터랙션 (User Actions)
1. 메모 수정 → memoText 업데이트
2. [저장] → PATCH API → 성공 토스트 + 닫힘
3. [취소] / X / ESC → 닫힘 (수정 미반영)

### 비즈니스 룰
- 최대 500자 (실시간 카운터)
- 빈 문자열 저장 가능 (메모 삭제 효과)

### 에지 케이스
- 저장 실패: 모달 유지 + 에러 토스트
- 저장 성공: SCR-S008 목록 갱신 (refreshList)

### 연결 화면
- 저장 성공 → 닫힘 → SCR-S008 `02-정상` (갱신)
