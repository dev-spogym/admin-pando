## 상태 고유 인터랙션
- [결제 확인] 클릭 → `setShowConfirm(true)` → **DLG-S003 결제 확인 모달** 오픈
- DLG-S003 "결제 완료" → `handlePaymentConfirm()`
  - `checkDuplicatePayment` isDuplicate=true → DLG-S004 전환
  - 아니면 `executePayment()` → `03-처리중`
- Enter(버튼 포커스 시) = 결제 확인 모달 오픈
- 결제수단 재클릭 → `01-대기`
- 회원 해제(마일리지 결제 중) → `05-마일리지부족`

## 상태 고유 비즈니스 룰 델타
- 결제 확인 버튼 활성 = `isValid && !isProcessing`
- DLG-S003 표시: 상품 목록, 합계, 할인, 최종, 결제수단, 회원명
- 중복결제 검사는 "결제 완료" 클릭 시점에만 수행 (사전 검사 없음)

## 에지 케이스
- Enter 연타 → 모달 1회만 오픈(상태 guard)
- DLG-S003 오픈 직전 마일리지 잔액 변동 → 내부 재검증 실패시 toast "마일리지 잔액이 변경되었습니다." + 모달 닫고 `05`
- 복합결제 분배가 재변경되어 불일치 → 즉시 `06`으로 전이
