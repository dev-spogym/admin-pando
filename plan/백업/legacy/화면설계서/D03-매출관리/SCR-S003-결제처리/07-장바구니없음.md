## 상태 고유 인터랙션
- [POS 판매로 이동] 클릭 → `moveToPage(971)` → `/pos`
- 사이드바 메뉴 클릭 → 다른 도메인 이동 가능
- PageHeader의 [초기화] 버튼은 disabled 처리(리셋할 상태 없음)
- 3초 후 자동 리다이렉트(옵션) — 설정값 `autoRedirectOnNoCart`:true 시 `toast.info('POS 판매로 이동합니다')` 후 3초

## 상태 고유 비즈니스 룰 델타
- 진입 시 `useEffect` 내 체크:
  ```ts
  useEffect(() => {
    const raw = sessionStorage.getItem('posCart');
    if (!raw) { setIsCartRestored(false); return; }
    try {
      const cart = JSON.parse(raw);
      if (!Array.isArray(cart) || cart.length === 0) { setIsCartRestored(false); return; }
      setCartItems(cart); setIsCartRestored(true);
    } catch { setIsCartRestored(false); }
  }, []);
  ```
- `posBuyer`는 cart 복원 성공 후에만 읽음
- 결제 완료 후 `calculateReset` 시 sessionStorage가 이미 비워져 있으므로 [계속 판매하기] 클릭 시 이 상태로 진입 가능

## 에지 케이스
- `posCart`에 빈 배열만 존재 → 동일 처리(NoCart)
- `posCart`에 잘못된 JSON 형식 → catch → NoCart
- 새로고침으로 완료 화면에서 NoCart로 이동 → 정상(의도된 동작)
- URL 직접 진입 `/pos/payment` → 가장 빈번한 케이스
- 사용자가 브라우저 탭을 닫았다가 재오픈 → sessionStorage는 탭 단위이므로 비어있음 → NoCart
