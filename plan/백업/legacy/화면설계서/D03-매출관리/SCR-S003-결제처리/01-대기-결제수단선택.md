## 상태 진입 조건
- sessionStorage `posCart` 복원 성공(`cartItems.length >= 1`)
- `paymentMethod === null` 또는 유효성 미통과지만 부족/불일치도 아님
- `isProcessing === false`, `isComplete === false`

## 역할별 차이
| 역할 | 델타 |
|---|---|
| trainer | `DiscountPanel` 자체 미렌더 |
| manager 이상 | 전체 노출 |
| staff/front/readonly | 진입 전 `/forbidden` |

## 상태 고유 인터랙션
- 결제수단 카드 클릭 → `setPaymentMethod(v)` → 유효성 재계산 → 다음 상태(`02`/`05`/`06`)
- 동일 결제수단 재클릭 → `paymentMethod=null` 토글 해제
- 마일리지 선택 + 회원 없음 → info 배너("마일리지 결제는 회원 검색 후 이용 가능합니다.") + `01` 유지
- 마일리지 선택 + 회원 있으나 잔액 부족 → `05-마일리지부족`
- 복합결제 선택 → `MixedPaymentPanel` 렌더 → 합계 불일치면 `06-복합결제-불일치`
- BuyerSelect.onOpen → DLG-S002 구매자 검색 모달

## 상태 고유 비즈니스 룰 델타
- 결제수단 미선택 시 `isValid=false`
- 할인으로 `finalAmount < 0` → DiscountPanel inline error "할인 금액이 상품가를 초과합니다." (E400302)
- trainer 진입 시 서버 `discount=0` 강제 (클라도 UI 숨김)

## 에지 케이스
- 카트의 상품 1개가 서버에서 삭제(E404301) → toast "상품 정보가 유효하지 않습니다." + `handleReset`
- 회원 선택 직후 → selectedMember 세팅만, 상태는 `01` 유지
- URL 직접 진입 시 branchId 컨텍스트 없으면 `/forbidden`
