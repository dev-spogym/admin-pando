## 역할별 차이
이 상태는 fc 전용. 다른 역할은 해당 없음.

## 상태 고유 인터랙션
- POS 버튼 클릭 → `/pos` (SCR-S002) 정상
- 환불/미수 탭 클릭 → 조회 가능, 상세 모달 열림, 편집성 액션만 숨김
- 상세 모달에서 [회원 상세] 이동 가능, [결제 취소]는 서버 403으로 막혀 있고 클라에서도 버튼 숨김
- InfoBanner X 버튼 클릭 → `sessionStorage.setItem('fc.sales.banner.dismissed', '1')` 저장
  - 다음 세션/다른 기기 진입 시 기본값 `false` (세션 내 X만 보장)
- URL 직접 `?tab=TAB-006` 접근 → 쿼리 존중 + 읽기 UI

## 상태 고유 비즈니스 룰 델타
- **읽기 + POS 실행 혼합**: FC는 신규 결제는 가능, 환불/세금계산서/할부 등은 조회만
- **엑셀 차단**: 개인정보(회원명/전화) 포함된 대량 내보내기 차단 (Compliance)
- **미수금 카드**: 값은 표시하지만 클릭 시 읽기 전용 페이지로 이동 (상태 변경 불가)
- **환불 탭 클릭 카운트**: 환불 탭 클릭은 허용, 상세 보기 가능, 환불 처리 버튼은 숨김
- **상세 모달 버튼 가시성**: `canCancelPayment('fc')=false`, `canReprintReceipt('fc')=true`, `canGotoMember('fc')=true`, `canEditMemo('fc')=false`
- 서버 검증: fc가 환불 API 호출 시 403 반환
- `hasPermission('sales:export')` → false → 엑셀 UI 완전 숨김

## 에지 케이스
- FC가 POS에서 환불성 결제(마이너스) 시도: POS에서 차단 + 센터장 요청 안내
- FC + trainer 겸직(미지원 역할): 서버는 primary role 기준, 클라도 primary만 사용
- FC가 본인이 등록한 매출의 상세를 열었을 때도 [결제 취소]는 여전히 숨김 (권한 우선)
- 배너 dismiss 후 새 세션: 다시 표시됨 (`sessionStorage` 사용)
- FC가 `/sales?tab=TAB-007` 쿼리 진입: 미납 탭 열리되 액션만 숨김

## 바이브코딩 프롬프트 (델타)
```
상태: sales-fc-scope. 마스터 프롬프트에 아래 override.
- if (role === 'fc') 분기
- PageHeader: [신규 결제] 활성, [엑셀] 조건 `canExportSalesExcel(role)=false` → 렌더 X (display:none)
- InfoBanner dismissible:

  const dismiss = () => { setBannerOpen(false); sessionStorage.setItem('fc.sales.banner', '1'); }
  {bannerOpen && (
    <div role="note" aria-label="제한된 접근 권한 안내"
         className="flex items-start gap-2 rounded-lg border border-yellow-200 bg-yellow-50 p-3 text-sm text-yellow-800">
      <ShieldOff className="size-4 mt-0.5" />
      <div className="flex-1">

      </div>
      <button onClick={dismiss} aria-label="배너 닫기"><X className="size-4" /></button>
    </div>
  )}
- TabNav tabs 전체 노출 (환불/미납은 readOnly)

  footer={
    <>
      {canGotoMember(role) && <Button onClick={gotoMember}>회원 상세</Button>}

      <Button variant="outline" onClick={close}>닫기</Button>
    </>
  }
```
