## 상태 진입 조건
- `data.length >= 1` (전체 데이터는 존재)
- 현재 탭의 파생 데이터 `dataByTab[activeTab].length === 0`
  - 예) TAB-006 환불내역: `filteredData.filter(s => s.status === '환불').length === 0`
  - 예) TAB-007 미납내역: `filteredData.filter(s => s.unpaid > 0).length === 0`
  - 예) TAB-001 + roundFilter='휴면복귀' + 검색어 조합으로 0건
  - 예) 매출유형 + 상태 + 날짜 복합 필터로 0건

## 상태 고유 비주얼 델타

```
┌─ StatCardGrid (전체 기준 값 유지, 0 아님) ──────────────────────────┐
│ [순매출 ₩128M] [카드 ₩80M] [현금 ₩20M] [미수금 ₩10M]                 │
├─ TabNav (배지는 실 건수 유지) ─────────────────────────────────────┤
│ [매출내역(150)][기간별][상품별][결제수단][담당자][환불(0,active)]     │
├─ DataTable empty ──────────────────────────────────────────────────┤
│                                                                     │
│            ┌──────────────────┐                                     │
│            │  🔍 (Filter)      │  80px, text-gray-300                │
│            └──────────────────┘                                     │
│       조건에 맞는 결과가 없습니다                                      │
│  현재 탭 및 필터 조합으로 표시할 데이터가 없습니다.                      │
│                                                                     │
│     [🔄 필터 리셋]  [↶ 전체 매출내역 보기]                            │
│                                                                     │
├─ SummaryBar (전체 기준, 0 아님) ───────────────────────────────────┤
└─────────────────────────────────────────────────────────────────────┘
```

| 변경 요소 | 델타 |
|---|---|
| StatCard | **전체 데이터 기준 실값 유지** (탭과 무관) |
| TabNav 배지 | 각 탭의 건수 유지 (`(0)`은 현재 탭만) |
| DataTable body | `emptyMessage={<TabEmptyState />}` 렌더 |
| 매출유형 필터 | 렌더 유지, 현재 선택값 강조 |
| SummaryBar | 전체 기준 실값 유지 (현재 탭 0건이어도 전체 계산은 유효) |

## 탭별 메시지 테이블

| 탭 | 아이콘 | 제목 | 설명 |
|---|---|---|---|
| TAB-001 매출내역 | Filter | 조건에 맞는 매출이 없습니다 | 매출유형/검색/날짜 조건을 조정해보세요 |
| TAB-002 기간별 | CalendarRange | 해당 기간의 매출이 없습니다 | 기간 단위를 변경하거나 날짜를 늘려보세요 |
| TAB-003 상품별 | Package | 해당 조건의 상품 매출이 없습니다 | 필터를 조정해 보세요 |
| TAB-004 결제수단별 | CreditCard | 해당 조건의 결제수단별 매출이 없습니다 | 필터를 조정해 보세요 |
| TAB-005 담당자별 | Users | 해당 조건의 담당자별 매출이 없습니다 | 필터를 조정해 보세요 |
| TAB-006 환불내역 | RotateCcw | 환불 내역이 없습니다 | 선택 기간에 환불된 매출이 없습니다 |
| TAB-007 미납내역 | AlertCircle | 미납 내역이 없습니다 | 모든 매출이 완납 처리되었습니다 (긍정 톤) |

## 상태 고유 인터랙션
- [필터 리셋] 클릭 → `handleResetFilters()` 호출 (이번달·공백·전체 유형·전체 상태)
- [전체 매출내역 보기] 클릭 → `setActiveTab('TAB-001')` + 매출유형 `''` + 검색어 `''`
- 다른 탭 클릭 시 정상 전환 (배지 숫자 참고)

## 상태 고유 비즈니스 룰 델타
- 필터 리셋 후에도 0건이면 전체 `03-빈상태`로 전이
- 매출유형 필터 버튼은 유지, 동일 값 클릭 시 전체 복귀
- 엑셀 버튼은 **활성** (전체 데이터가 있으므로 전체 기준 다운로드)
  - ⚠️ 엑셀 다운로드 대상은 `filteredData`(현재 필터 적용) — 현재 탭 기준 0건이면 `toast.warning('내보낼 데이터가 없습니다.')`
- SummaryBar는 전체 필터링 결과 기준 (탭과 무관)
- TAB-001(매출내역)이 빈 상태라면 `03-빈상태`와 동일 처리

## 에지 케이스
- TAB-006에서 모든 환불이 이번달 아닌 경우 → 날짜 리셋 힌트
- TAB-007에서 모든 미수금이 회수 완료된 경우 → "모든 매출이 완납 처리되었습니다" 긍정 메시지
- 매출유형 필터 '휴면복귀' 선택 시 0건 → 버튼을 "전체"로 리셋하는 shortcut 링크
- TAB-001 검색어 공백만 입력 → 무시 (전체 표시)

## 바이브코딩 프롬프트 (델타)
```
상태: sales-tab-empty. 마스터 프롬프트 기본값에 아래만 override.
```
