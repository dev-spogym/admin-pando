## 7. 데이터 계약

```ts
interface BodyCompositionRecord {
  id: number;
  memberId: number;
  memberName: string;
  measuredAt: string;
  source: 'inbody' | 'manual';
  deviceId?: string;
  weight: number;         // kg
  bodyFatPct: number;     // %
  muscleMass: number;     // kg
  bmi: number;
  visceralFat?: number;
  bodyWater?: number;
  boneMass?: number;
  basalMetabolicRate?: number;
  staffId?: number;       // 수기 등록 담당자
}

interface TrendPoint {
  date: string;
  weight: number;
  bodyFatPct: number;
  muscleMass: number;
}
```

## 8. 비즈니스 룰
1. InBody 기기 → `source: 'inbody'`, 자동 연동 (WebSocket or HTTP POST)
2. 수기 입력 → `source: 'manual'`, 담당 직원 `staffId` 기록
3. 트레이너는 담당 회원 데이터만 조회 가능
4. 체지방률 전회 대비 변화 → 트렌드 아이콘 (↑ 빨강/↓ 초록)
5. 같은 날 중복 측정 허용 (InBody 자동 + 수기 병용 가능)
6. CSV 내보내기: manager 이상
7. 멀티테넌트 `branchId` 스코프
