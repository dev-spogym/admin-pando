## 📝 디스크립션

### 사용 시점 (Trigger)
- 로딩 완료 + `iot_devices.length > 0`
- 모든 장비 또는 일부 장비 온라인

### 필수 데이터
| 블록 | 테이블 | 조건 |
|------|--------|------|
| 장비 목록 | `iot_devices` | `branch_id = ?` |
| 실시간 상태 | 30초 폴링 | 각 device ping |

### 인터랙션 (User Actions)
1. "재시작" 버튼 → POST /api/iot/restart/:deviceId
2. "장비 추가" → 장비 등록 모달 (향후)
3. 카드 클릭 → 장비 상세 정보

### 비즈니스 룰
- 장비 상태 30초 폴링 자동 갱신
- 오류 장비 카드: 빨간 테두리 `border-red-200`
- 오프라인 장비: 회색 처리 `opacity-60`
- `iot:write` 없으면 재시작/추가 버튼 숨김

### 에지 케이스
- 일부 장비 오류 → `05-장비오류배지` (카드 오류 표시 + 상단 배지)
- 전체 오프라인 → `06-전체오프라인` 경고 배너
- 재시작 실패: toast.error('재시작 실패. 장비를 직접 확인하세요')

### 연결 화면
- 이전: `01-로딩`
- 다음: `03-빈상태` / `05-장비오류배지` / `06-전체오프라인`
