```
Next.js 14 App Router + TypeScript + Tailwind CSS 기반 Modal 컴포넌트를 작성하라.
다이얼로그: DLG-064-002 급여 확정 확인 모달 — 상태: 오픈

트리거: SCR-064 "급여 확정" 버튼 클릭
권한: hasPermission('payroll:confirm') — primary / owner

모달 오픈 즉시 확정 가능 여부 계산:
  const pendingList = payrollList.filter(p => p.status === 'PENDING')
  const totalAmount = pendingList.reduce((sum, p) => sum + p.net_salary, 0)

모달 구조:
  <Modal
    isOpen={confirmModalOpen}
    onClose={() => setConfirmModalOpen(false)}
    variant="danger"
    title="급여 확정"
    size="md"
  >
    <div className="space-y-4">
      {/* 확정 대상 요약 */}
      <div className="rounded-lg bg-blue-50 border border-blue-200 p-4 space-y-2">
        <div className="flex justify-between text-sm">
          <span className="text-gray-600">대상 기간</span>
          <span className="font-medium">{selectedMonth}</span>
        </div>
        <div className="flex justify-between text-sm">
          <span className="text-gray-600">미확정 직원</span>
          <span className="font-medium text-orange-600">{pendingList.length}명</span>
        </div>
        <div className="flex justify-between text-sm">
          <span className="text-gray-600">확정 총 급여액</span>
          <span className="font-bold text-blue-700">{formatNumber(totalAmount)}원</span>
        </div>
      </div>

      {/* 전원 이미 확정 케이스 */}
      {pendingList.length === 0 && (
        <div className="rounded-md bg-yellow-50 border border-yellow-200 p-3">
          <p className="text-sm text-yellow-700">모든 직원이 이미 확정되었습니다.</p>
        </div>
      )}

      {/* 급여 데이터 없음 케이스 */}
      {payrollList.length === 0 && (
        <div className="rounded-md bg-red-50 border border-red-200 p-3">
          <p className="text-sm text-red-700">이 월의 급여 데이터가 없습니다. 확정할 수 없습니다.</p>
        </div>
      )}

      {/* 경고 */}
      {pendingList.length > 0 && (
        <div className="rounded-md bg-yellow-50 border border-yellow-200 p-3">
          <p className="text-sm text-yellow-700">
            급여 확정 후에는 수정이 불가능합니다. 급여 명세서가 자동으로 생성됩니다.
          </p>
        </div>
      )}
    </div>

    <div className="mt-6 flex justify-end gap-3">
      <Button variant="outline" onClick={() => setConfirmModalOpen(false)}>취소</Button>
      <Button
        variant="danger"
        disabled={pendingList.length === 0 || payrollList.length === 0 || isConfirming}
        onClick={handleConfirmPayroll}
      >
        {isConfirming ? '확정 중...' : '급여 확정'}
      </Button>
    </div>
  </Modal>

상태 변수:
- confirmModalOpen: boolean
- isConfirming: boolean
- pendingList: Payroll[] (PENDING 상태)
- totalAmount: number
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-064 "급여 확정" 버튼 클릭 (미확정 있을 때만 활성)
- `hasPermission('payroll:confirm')` true

### 모달 진입 시 확인 사항
| 조건 | 표시 | 확정 버튼 |
|------|------|-----------|
| 미확정 직원 있음 | 인원 + 총액 | 활성 |
| 전원 이미 확정 | 경고 메시지 | disabled |
| 급여 데이터 없음 | 에러 메시지 | disabled |

### 인터랙션 (User Actions)
1. "취소" → 모달 닫힘, SCR-064 유지
2. "급여 확정" (활성) → `03-결과분기`
3. ESC → 닫힘

### 비즈니스 룰
- manager: 이 모달 도달 불가 (SCR-064의 "급여 확정" 버튼 hidden)
- 확정 대상: PENDING 상태 전원 일괄 확정 (개별 확정 불가)
- 확정 후 명세서 자동 생성 안내

### 접근성 (A11y)
- `role="dialog"` + `aria-modal="true"`
- 확정 요약 영역 `role="region"` + `aria-label="급여 확정 요약"`

### 연결 화면
- 이전: SCR-064 `02-정상-미확정있음`
- 다음: `02-검증` / 닫힘
