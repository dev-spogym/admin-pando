## 3. 역할별 차이

| 역할 | 옵션 가시성 |
|---|---|
| superAdmin / owner | 전체 4개 |
| manager | `ACTIVE`, `ON_LEAVE` 만 |
| 기타 | 도달 불가 |

---

## 4. 인터랙션

- **오픈 애니메이션**: `animate-[fadeInScale_150ms]`
- **초기 포커스**: 첫 라디오 (재직)
- **ArrowUp/ArrowDown**: 라디오 간 이동 (Radix RadioGroup 기본)
- **Esc/Overlay click**: 닫힘
- 라디오 선택 시 조건부 필드(사유·기간) 애니메이션으로 펼침 (`transition-all duration-150`)

---

## 5. 비즈니스 룰 델타

- `useEffect(() => { if (open) form.reset(); }, [open])` — 오픈마다 초기화
- `targets` 중 `user.staffId` 포함 시:
  ```ts
  if (targets.some(t => t.id === user.staffId)) {
    toast.warning('자기 자신의 상태는 변경할 수 없습니다');
    onClose();
    return;
  }
  ```
- 복수 대상 N>1 시: 현재 상태 뱃지 숨김, 안내 문구 "복수 직원은 현재 상태가 다를 수 있습니다."

---

## 6. 에지 케이스

- **오픈 중 targets 변경**: SCR-060에서 selectedRows가 바뀌면 form reset + 제목 N 업데이트
- **권한 변동**: 모달 오픈 중 세션 갱신으로 role 강등 → 다음 API 호출에서 403 처리
- **창 320px 이하**: 라디오 카드 세로 유지, description 말줄임(`truncate`) 적용

---

## 7. 바이브코딩 프롬프트 (델타)

```
상태: dlg-statuschange-open
변경점(마스터 대비):
- 오픈 시 초기화:
  useEffect(() => {
    if (!open) return;
    form.reset({ status: undefined });
    if (targets.some(t => t.id === user.staffId)) {
      toast.warning('자기 자신의 상태는 변경할 수 없습니다');
      onClose();
    }
  }, [open]);
- 복수 대상 안내:
  {targets.length > 1 && (
    <p className="text-xs text-gray-500">복수 직원은 현재 상태가 다를 수 있습니다.</p>
  )}
- manager 옵션 필터:
  options = STATUS_OPTIONS.filter(o =>
    user.role === 'manager' ? ['ACTIVE','ON_LEAVE'].includes(o.value) : true
  );
- 초기 포커스: onOpenAutoFocus → 첫 라디오
```
