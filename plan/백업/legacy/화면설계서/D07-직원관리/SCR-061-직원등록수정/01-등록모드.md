## 진입 조건
- `/staff/new` 진입 + `searchParams.id` 없음
- `editId === undefined`
- defaultValues: `{ name:'', role:'trainer', contact:'', joinDate: today, email:'', memo:'', salary:'' }`

## 비주얼 델타
| 변경 요소 | From → To |
|---|---|
| title | 동적 | "직원 등록" |
| description | 동적 | "새로운 직원 정보를 입력하고 역할을 설정합니다." |
| save button label | 동적 | "저장하기" + `<Save size={16}/>` |
| defaultValues.joinDate | — | 오늘 날짜 ISO 앞 10자 |
| defaultValues.role | — | 'trainer' (가장 흔한 역할) |
| Preview 박스 | 조건부 | trainer 권한 기본 렌더 (`08` 참조) |
| 이름 필드 | — | 마운트 시 autofocus |

## 역할별 차이
- manager: "센터장(owner)" 옵션 disabled.
- owner/primary/superAdmin: 전 옵션 활성.
- primary는 "최고관리자" 옵션 본 폼에서 미표시 (내부 관리 전용).

## 고유 인터랙션
- 이름 필드 autofocus.
- 연락처 `onChange`: `formatPhone` + `trigger('contact')`.
- 역할 변경: `watchedRole` → 08 재렌더.
- Enter: 기본 제출 (이름 필드에서 Enter는 다음 필드 이동 옵션).
- 취소: DLG-061-001 (isDirty 무관 등록 모드에서는 항상 확인).

## 비즈니스 룰 델타
1. 기본값 정책: 오늘 날짜 + `trainer`.
2. 중복 방지: `isSaving` 동안 Enter 무시.
3. 휴직/퇴사 필드 비노출 (ACTIVE 상태로 INSERT).
4. branchId: 서버 jwt 강제 + 클라 `getBranchId()` 보조.

## 에지 케이스
- 동명이인 허용(DB 제약 없음).
- 전화번호 중복 E409100 → 인라인 에러.
- 예약어/특수문자 이름: 검증 없음.
- 뒤로가기: `isDirty` true 시 beforeunload.

## 바이브코딩 프롬프트 (델타)
```
editId === undefined:
<PageHeader title="직원 등록"
  description="새로운 직원 정보를 입력하고 역할을 설정합니다."
  actions={<Button type="submit" variant="primary" disabled={isSaving}>
    <Save size={16}/> {isSaving ? '저장 중...' : '저장하기'}
  </Button>} />

useForm 기본값:
defaultValues: { name:'', role:'trainer', contact:'',
  joinDate: new Date().toISOString().split('T')[0],
  salary:'', email:'', memo:'' }

useEffect(() => { document.getElementById('name')?.focus(); }, []);

// manager에게 owner 옵션 disabled
<option value="owner" disabled={role==='manager'}>센터장</option>
```
