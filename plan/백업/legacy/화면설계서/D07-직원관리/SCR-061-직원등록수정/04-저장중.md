## 진입 조건

- `handleSubmit(onSubmit)` 실행 중, 서버 응답 대기

## 비주얼 델타
| 변경 요소 | From → To |
|---|---|
| 저장 버튼 | "저장하기"/"수정 저장" → "저장 중..." + `<Loader2 className="animate-spin"/>` |
| 저장 버튼 disabled | false → true + `aria-busy="true"` |
| 취소 버튼 | 활성 | disabled (중단 불가) |

## 역할별 차이
- 공통.

## 고유 인터랙션
- 중단 불가. 모든 키보드 이벤트 preventDefault.
- 창 닫기 시도: `beforeunload` 경고.

## 비즈니스 룰 델타
1. `isSaving=true` 전역 guard → 중복 제출 차단.
2. 30s 타임아웃 → AbortError + toast.
3. 낙관적 업데이트 없음. 성공 응답 후 반영.
4. 감사 로그는 성공 후에만.
5. Supabase 에러 코드 매핑: `unique_violation`→E409*, `check_violation`→E400*, `foreign_key`→E404*.

## 에지 케이스
- 중간 네트워크 단절 → `07-에러`.
- DB 트랜잭션 타임아웃 → 서버 롤백 → `07`.
- 감사 로그 실패: 콘솔 warning만, 사용자 UX는 성공.
- 빠른 더블 클릭: guard로 2번째 차단.

## 바이브코딩 프롬프트 (델타)
```
isSaving === true:
<Button type="submit" variant="primary" disabled aria-busy="true">
  <Loader2 className="animate-spin" size={16}/> 저장 중...
</Button>
<Button variant="secondary" disabled>취소</Button>

<form className={isSaving ? 'pointer-events-none opacity-60' : ''}
      onKeyDown={(e)=>{ if (isSaving && (e.key==='Enter'||e.key==='Escape')) e.preventDefault(); }}>

// 30s 타임아웃
const controller = new AbortController();
const timer = setTimeout(() => controller.abort(), 30_000);
try {
  const { error } = await supabase.from('staff').update(payload).eq('id', editId).abortSignal(controller.signal);
  clearTimeout(timer);
  if (error) throw error;
  ...
} catch (e) {
  if (e.name === 'AbortError') toast.error('요청 시간이 초과되었습니다. 다시 시도해주세요.');
  else toast.error((editId ? '수정 실패: ' : '저장 실패: ') + e.message);
}
```
