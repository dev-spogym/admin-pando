## 🧩 바이브코딩 프롬프트

```
Next.js 14 App Router + TypeScript + Tailwind CSS 기반 Modal 컴포넌트를 작성하라.
다이얼로그: DLG-064-001 급여 상세/편집 모달 — 상태: 오픈 (데이터 로드 중)

트리거: SCR-064 테이블 행 클릭 → selectedPayrollId 설정 → detailModalOpen=true

모달 오픈 즉시 데이터 fetch:
  useEffect(() => {
    if (detailModalOpen && selectedPayrollId) {
      fetchPayrollDetail(selectedPayrollId)
    }
  }, [detailModalOpen, selectedPayrollId])

  const fetchPayrollDetail = async (id: string) => {
    setIsLoadingDetail(true)
    const { data, error } = await supabase
      .from('payroll')
      .select('*, staff(name, department, position, hire_date)')
      .eq('id', id)
      .single()

    if (error) {
      toast.error('급여 정보를 불러오지 못했습니다.')
      setDetailModalOpen(false)
      return
    }
    setPayrollDetail(data)
    form.reset({
      base_salary: data.base_salary,
      allowances: data.allowances,
      deductions: data.deductions,
    })
    setIsLoadingDetail(false)
  }

로딩 중 모달 UI:
  <Modal isOpen={detailModalOpen} title="급여 상세" size="lg">
    <div className="flex items-center justify-center py-12">
      <Spinner className="h-8 w-8 animate-spin text-blue-500" />
      <p className="ml-3 text-sm text-gray-500">급여 정보를 불러오는 중...</p>
    </div>
  </Modal>

상태 변수:
- detailModalOpen: boolean
- selectedPayrollId: string | null
- payrollDetail: Payroll | null
- isLoadingDetail: boolean
- isSaving: boolean
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- SCR-064 테이블 행 클릭 → `selectedPayrollId` 설정 → 모달 오픈
- Supabase GET payroll detail API 호출 중

### 권한
- payroll:view: primary / owner / manager (모달 오픈 가능)
- payroll:edit: primary / owner (편집 필드 활성)
- manager: 조회만 가능 (편집 필드 disabled)

### 인터랙션 (User Actions)
- 로딩 중: 없음 (닫기 버튼만 활성)

### 에지 케이스
- fetch 실패: toast.error + 모달 즉시 닫힘
- selectedPayrollId null: 모달 렌더링 방지 (guard)

### 접근성 (A11y)
- `role="dialog"` + `aria-modal="true"` + `aria-label="급여 상세 정보"`
- 로딩 중 `aria-busy="true"`
