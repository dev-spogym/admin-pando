## 2. 비주얼 델타 — 3단계

### 2.1 실행 중 (Executing)

```
┌──────────── Modal ────────────┐
│ ...                            │
│ ┌────────────────────────────┐ │
│ │ 퇴사처리                    │ │  readOnly (mutation.isPending)
│ └────────────────────────────┘ │
│ ...                            │
│   [취소]          [⏳ 처리 중..] │
│    disabled        disabled+spinner
└────────────────────────────────┘
```

| 요소 | 값 | 비고 |
|---|---|---|
| 취소 버튼 | `disabled` | 실행 중 닫기 차단 |
| 확정 버튼 | `loading=true, disabled` | Loader2 아이콘 spin |
| 입력 필드 | `readOnly` | 값 고정 |
| Esc / Overlay | **무시** | 닫힘 차단 |

### 2.2 성공 (Success, affected === N)

```
→ 모달 즉시 닫힘
→ SCR-060 갱신 (queryClient.invalidate)
→ toast.success(`{N}명의 퇴사 처리가 완료되었습니다.`)
```

### 2.3 부분 성공 (Partial, 0 < affected < N)

```
→ 모달 즉시 닫힘
→ SCR-060 갱신
→ toast.warning(`{affected}/{N}명 처리. 일부는 권한/상태 문제로 제외되었습니다.`)
```

### 2.4 실패 (Failure)

```
┌──────────── Modal (유지) ──────┐
│ ...                            │
│ ┌─ ErrorBanner ─────────────┐  │
│ │ ⚠ E500001 서버 오류         │  │ role="alert"
│ │ 잠시 후 다시 시도해주세요.   │  │
│ └───────────────────────────┘  │
│         [취소]  [퇴사 처리 재시도] │  enabled (재시도 가능)
└────────────────────────────────┘
```

| 요소 | 값 |
|---|---|
| 모달 | 유지 (닫히지 않음) |
| 입력값 | 유지 (재제출 가능) |
| 확정 버튼 | `enabled` (재시도) |
| 에러 배너 | `bg-red-50 border-red-200 text-red-700` |

## 3. 결과 코드 매핑

| HTTP | errorCode | 분기 | UI |
|---|---|---|---|
| 200 | — | Success (affected===N) | toast.success + 닫힘 |
| 200 | — | Partial (affected<N) | toast.warning + 닫힘 |
| 400 | E400200 | 실패 | 배너 + 모달 유지 |
| 401 | E401001 | 세션 만료 | DLG-000 위임 |
| 403 | E403001 | 권한 없음 | toast.error + 강제 닫힘 |
| 404 | E404200 | 직원 없음 | 배너 + 목록 재조회 안내 |
| 409 | E409200 | 이미 퇴사 | 배너 + 해당 ID 제외 재시도 안내 |
| 500 | E500001 | 서버 오류 | 배너 + 재시도 활성 |
| — | NETWORK | 네트워크 단절 | 배너 + 재시도 활성 |

## 5. 인터랙션 델타

- **실행 중**: 모든 닫기 액션(Esc, Overlay, 취소) 무시
- **성공 후**: 모달 `onClose()` 자동 호출 + 트리거 버튼으로 포커스 복귀
- **실패 후**: 입력 필드로 포커스 복귀, 재시도 가능

---

## 6. 비즈니스 룰 델타

- **React Query invalidate**: 성공·부분 성공 모두 `queryClient.invalidateQueries({ queryKey: ['staff'] })`
- **SCR-060 selectedRows 초기화**: 성공·부분 성공 후 `setSelectedRows(new Set())`
- **통계 카드 갱신**: SCR-060의 `재직중/퇴사` 카운트 자동 재집계 (invalidate로 트리거)
- **재시도 쿨다운**: 500/NETWORK 연속 3회 실패 시 확정 버튼 10초 disabled + 카운트다운
- **감사 로그**:
  - 성공 → `AUDIT.STAFF_RESIGN_SUCCESS`(ids, affected)
  - 부분 → `AUDIT.STAFF_RESIGN_PARTIAL`(ids, affected, diff)
  - 실패 → `AUDIT.STAFF_RESIGN_FAILURE`(ids, errorCode)

---

## 7. 에지 케이스

- **네트워크 복구 중 재시도**: 이전 mutation이 pending인지 확인 (`isPending` 가드).
- **403 수신 + 다른 탭에서 로그아웃**: 강제 닫힘 + `/login?redirect=/staff` 리다이렉트.
- **409 after Success (race)**: 한 탭에서 성공 → 다른 탭에서 동일 ID 처리 시 409. 안내 메시지 노출 후 목록 재조회로 해결.
- **partial=0**: 실패와 동일하게 처리 (모달 유지, 에러 배너).
- **500 → 500**: 3회 연속 실패 시 쿨다운 트리거.

## 8. 바이브코딩 프롬프트 (델타)

```
상태: dlg-resign-result
변경점(마스터 대비):
- 실행 중 UI:
  <Input readOnly={mutation.isPending} />
  <Button disabled={mutation.isPending}>취소</Button>
  <Button loading={mutation.isPending} disabled={mutation.isPending || !isValid}>
    {mutation.isPending ? <><Loader2 className="animate-spin size-4" /> 처리 중...</> : '퇴사 처리'}
  </Button>
  onOpenChange={(v) => { if (!v && !mutation.isPending) onClose(); }}

- 결과 분기:
  onSuccess: ({ affected }) => {
    qc.invalidateQueries({ queryKey: ['staff', branchId] });
    clearSelection();
    if (affected === targets.length) toast.success(`${affected}명의 퇴사 처리가 완료되었습니다.`);
    else if (affected > 0) toast.warning(`${affected}/${targets.length}명 처리. 일부 제외`);
    else setServerError({ code: 'PARTIAL_ZERO', message: '처리된 대상이 없습니다' });
    if (affected > 0) onClose();
  },
  onError: (err:any) => {
    const code = mapSupabaseError(err);
    setServerError({ code, message: ERROR_MESSAGE[code] ?? err.message });
    if (code === 'E403001') { toast.error(ERROR_MESSAGE[code]); onClose(); }
    if (code === 'E401001') { redirectToLogin(); }
    failureCount.current += 1;
    if (failureCount.current >= 3) setCooldown(Date.now() + 10000);
  },

- 에러 배너:
  {serverError && (
    <div role="alert" className="rounded-lg border border-red-200 bg-red-50 p-3 text-sm text-red-700">
      <strong>{serverError.code}</strong> {serverError.message}
    </div>
  )}

- 쿨다운:
  const [cooldownUntil, setCooldown] = useState<number|null>(null);
  const remaining = cooldownUntil ? Math.max(0, cooldownUntil - now) : 0;
  disabled: !isValid || mutation.isPending || remaining > 0
  label:    remaining > 0 ? `${Math.ceil(remaining/1000)}초 후 재시도` : '퇴사 처리'
```
