> **델타 문서**: 마스터(`00-기본화면.md`) 상속. 변경점만 기술.

## 1. 상태 진입 조건

- `open === true` (부모가 SCR-060에서 제어)
- `targets.length >= 1`
- `user.role ∈ {superAdmin, owner}` (서버/클라이언트 양쪽 가드)
- `targets` 중 `staffStatus === 'RESIGNED'` 포함 시 진입 차단 (선처리: `toast.error` 후 모달 오픈 금지)

## 3. 역할별 차이

| 역할 | 비고 |
|---|---|
| superAdmin | 대상 목록에 `branchId` 뱃지 추가 (cross-branch 시) |
| owner | 기본 UI |
| 그 외 | **도달 불가** (버튼 hidden) |

---

## 4. 인터랙션

- **autoFocus**: 입력 필드로 포커스 이동 (Radix `onOpenAutoFocus` 기본)
- **Tab 트랩**: 입력 → 취소 → 확정 → 입력 순환
- **Esc**: `onClose()` 호출 (SCR-060 유지, 선택 유지)
- **Overlay 클릭**: `onClose()` (옵션 유지)
- 입력 onChange 시 → `02-검증` 진입

---

## 5. 비즈니스 룰 델타

- `targets.length === 0` 이면 render 금지 (부모가 open=true로 넘겨도 early return)
- 오픈 직전 RESIGNED 포함 검증:
  ```ts
  if (targets.some(t => t.staffStatus === 'RESIGNED')) {
    toast.error('이미 퇴사 처리된 직원이 포함되어 있습니다');
    return; // 모달 미오픈
  }
  ```
- 오픈 시 감사 로그: `AUDIT.STAFF_RESIGN_MODAL_OPEN`(actor, targetIds, branchId)

---

## 6. 에지 케이스

- **targets 변경 중 오픈**: SCR-060의 selectedRows가 바뀌면 `useEffect`로 최신값 반영. 변경 시 confirmText 초기화.
- **빠른 재오픈**: 직전 상태 잔여(confirmText) 없이 항상 빈 상태로 시작.
- **네트워크 단절 화면**: 오픈은 가능하나 Confirm 클릭 시 NETWORK 에러 대응.
- **창 크기 ≤320px**: 여백 16px 유지, 입력 가로 축소되지만 가독성 유지.

## 7. 바이브코딩 프롬프트 (델타)

```

    }
```
