## 역할별 차이
- 공통. primary/superAdmin는 actor 지점 + target 지점 모두 로그 기록.

## 고유 인터랙션
- 자동 리다이렉트 `moveToPage(974)`. 0~300ms 체크마크 노출 후 이동(선택).
- 목록에서 해당 row 3초 `bg-primary/10 animate-pulse` 강조.

## 비즈니스 룰 델타
1. **감사 로그**: `action:'STAFF_RESIGN_SCHEDULE'`, afterValue `{resignScheduledAt, resignReason}` + (FC) `reassignedCount`.
2. **staff.staffStatus**: 여전히 `ACTIVE` (예정일 도달 시 cron으로 `RESIGNED` 전환).
3. **staff.resignScheduledAt**: UPDATE됨.
4. **members.staffId**: 재배정된 회원 전부 UPDATE (FC).
5. **목록 캐시 invalidate**: `queryClient.invalidateQueries(['staff-list'])`.
6. **알림 (선택)**: 본사 관리자/해당 직원에게 알림 발송 — 추후.

## 에지 케이스
- 감사 로그 INSERT 실패: 콘솔 warning만. UX 성공.
- `/staff` 이동 실패: `router.push('/staff')` fallback.
- 재배정 수 0(FC지만 담당 회원 0명): 로그에 `reassignedCount:0` 기록.
- 퇴사 예정일이 오늘인 경우: cron이 즉시 RESIGNED로 전환할 수 있음. UI에서는 "오늘 예정"으로 표기.

## 바이브코딩 프롬프트 (델타)
```
성공 후:
toast.success('퇴사 예정이 등록되었습니다.');
try {
  await createAuditLog({
    action: 'STAFF_RESIGN_SCHEDULE',
    targetType: 'staff',
    targetId: selectedStaffId!,
    afterValue: {
      resignScheduledAt: resignDate,
      resignReason: resignReason,
      reassignedCount: isFC ? members.filter(m => m.assignedStaffId).length : 0,
      scheduleAction: futureSchedules.length ? scheduleAction : null,
    },
  });
} catch { /* swallow */ }

queryClient.invalidateQueries({ queryKey:['staff-list'] });
moveToPage(974, { highlight: selectedStaffId });
```
