| 항목 | 값 |
|---|---|
| 상태 코드 | `staff-list-unselected` |
| 상태 ID | 07 |
| 다이어그램 노드 | F6 STATE_NO_SELECTION / F3 BTN_DISABLED |
| 이전 상태 | `02-정상` (기본) / `08-선택있음`(해제) |

## 진입 조건
- `isLoadingData === false`
- `filtered.length >= 1`
- `selectedRows.size === 0`

| 메시지 전송 | 활성 → disabled + tooltip |

| 행 스타일 | — | 선택된 행 없음, `row.selected` 없음 |

## 역할별 차이
- 역할 무관 (선택 로직은 공통). 단, 역할에 따라 버튼 disabled 사유가 추가될 수 있음:
  - manager: 퇴사 버튼은 "선택 후에도" disabled (역할 제약 병합)
  - 제약 우선순위: 권한 제약 > 선택 제약 (tooltip 메시지도 권한 쪽 우선)

## 고유 인터랙션
- 체크박스 클릭 → `08-선택있음`으로 전이.
- 비활성 버튼 클릭 시도 → 반응 없음 (CSS `pointer-events-none` 또는 disabled 속성).
- 키보드: Tab으로 활성 버튼(등록/엑셀/필터)으로만 이동.

## 비즈니스 룰 델타
1. **기본 상태**: 데이터 로드 직후 `selectedRows = new Set()`.
2. **페이지 이동 시**: 선택 유지 정책 결정.
   - 정책: **페이지 이동 시 선택 해제**(현재) — 복잡도 최소.
   - 대안: 전역 선택 유지 (Set 크기 표시) — 미래 확장.
3. **필터 변경 시**: filtered에서 제외된 id는 `selectedRows`에서도 제거.
4. **헤더 "전체 선택"**: 현재 페이지 rows만 대상. 50건 중 현재 페이지 10건만 선택.

## 에지 케이스
- **필터 변경 → 일부 선택 사라짐**: 자동 해제 + 살짝 shake 애니메이션 (선택).
- **새로고침 후**: 선택 초기화.
- **새 직원 등록 후 리턴**: 새 row 추가되어도 선택 초기화 유지.

## 바이브코딩 프롬프트 (델타)
```
selectedRows.size === 0 (조건):
const hasSel = selectedRows.size > 0;

        aria-disabled={!hasSel}

        onClick={()=>setIsStatusDialogOpen(true)}>
  <Settings2 size={16}/> 상태 변경
</Button>

<Button variant="danger" disabled={!hasSel || !canRetire(role)}
        title={!hasSel ? '먼저 직원을 선택하세요' : !canRetire(role) ? '센터장 권한 필요' : ''}
        onClick={()=>setIsRetireDialogOpen(true)}>
  <UserMinus size={16}/> 퇴사 처리
</Button>

<Button variant="outline" onClick={exportAll}>
  <Download size={16}/> 명단 다운로드
</Button>

// 필터 변경 시 선택 정리
useEffect(()=>{
  setSelectedRows(prev => new Set([...prev].filter(id => filtered.some(r => r.id === id))));
}, [filtered]);
```
