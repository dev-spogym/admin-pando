# DLG-061-001 등록수정취소 — 상태: 변경 여부 확인

## 메타

| 항목 | 값 |
|------|----|
| 화면 ID | DLG-061-001 |
| 상태 코드 | `dirty-check` |
| 역할 | primary / owner / manager |
| 우선순위 | P1 |
| 이전 상태 | 취소 버튼 클릭 직후 (모달 오픈 전 분기) |
| 다음 상태 | dirty=true → `01-모달오픈` / dirty=false → 즉시 SCR-060 |

## 🧩 바이브코딩 프롬프트

```
변경 여부 확인 로직 (모달 오픈 전):

  const handleCancelClick = () => {
    const dirty = formState.isDirty

    if (!dirty) {
      // 변경 없음 → 모달 없이 즉시 이동
      moveToPage('/staff')
      return
    }

    // 변경 있음 → 경고 모달 오픈
    setCancelModalOpen(true)
  }

변경 있음 판단 기준 (react-hook-form):
  formState.isDirty === true
  → 하나 이상의 필드가 기본값(defaultValues)과 다른 경우

변경 없음 (isDirty=false) 케이스:
  - 등록 모드 진입 직후 (아무것도 입력 안 함)
  - 수정 모드에서 데이터 로드 직후 (변경 안 함)
  - 모든 필드를 원래 값으로 되돌린 경우

경고 UI (isDirty=true일 때 모달 내부):
  <div className="rounded-lg bg-yellow-50 border border-yellow-200 p-4">
    <p className="text-sm font-medium text-yellow-800">
      작성 중인 내용이 저장되지 않고 사라집니다.
    </p>
  </div>

즉시 이동 (isDirty=false):
  // 모달 없이 바로 이동
  moveToPage('/staff')
```

## 📝 디스크립션

### isDirty 분기
| isDirty | 처리 |
|---------|------|
| true | DLG-061-001 모달 오픈 + 경고 메시지 표시 |
| false | 모달 생략, 즉시 `/staff` 이동 |

### 비즈니스 룰
- react-hook-form의 `formState.isDirty` 기준
- 수정 모드에서 defaultValues와 완전히 동일하면 isDirty=false

### 연결 화면
- 이전: 취소 버튼 클릭
- 다음: `01-모달오픈` (dirty) / SCR-060 (not dirty)
