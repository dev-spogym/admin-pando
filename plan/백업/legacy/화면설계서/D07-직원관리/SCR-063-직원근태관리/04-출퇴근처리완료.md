## 🧩 바이브코딩 프롬프트

```
출근 처리 성공 UI:
  toast.success('{직원명}님 출근 처리되었습니다. (HH:mm)')
  - 해당 직원 행 즉시 갱신:
    · clock_in: 현재 시각
    · 상태 배지: "출근중" (green)
    · 액션 버튼: "퇴근 처리"로 변경

퇴근 처리 성공 UI:
  toast.success('{직원명}님 퇴근 처리되었습니다. (HH:mm, 총 N시간 M분)')
  - 해당 직원 행 즉시 갱신:
    · clock_out: 현재 시각
    · 근무 시간: 계산 표시
    · 상태 배지: "퇴근" (gray)
    · 액션 버튼: "-"

낙관적 업데이트 (Optimistic UI):
  - API 호출 전 즉시 UI 갱신
  - API 실패 시 이전 상태로 롤백
  - toast.error('처리에 실패했습니다.')

통계 카드 즉시 갱신:
  - 출근 수 +1 / 미출근 수 -1 (출근 처리)
  - 퇴근 완료 수 +1 (퇴근 처리)

사용:
  - toast (sonner)
  - 낙관적 상태 관리: setAttendanceList(prev => prev.map(...))
```

## 📝 디스크립션

### 사용 시점 (Trigger)
- 출근 처리: `handleClockIn(staffId)` API 성공
- 퇴근 처리: `handleClockOut(staffId)` API 성공

### 인터랙션 (User Actions)
- 자동 처리 (사용자 추가 액션 불필요)
- toast 3초 후 자동 닫힘

### 비즈니스 룰
- 낙관적 업데이트로 즉각적인 피드백 제공
- API 실패 시 롤백 + toast.error
- 동일 직원 중복 출근 시도: API에서 409 → toast.error('이미 출근 처리된 직원입니다.')

### 에지 케이스
- 퇴근 시간 < 출근 시간 (자정 넘김): 다음 날 날짜 기준 처리
- 8시간 초과 퇴근: 초과 근무 표시 (별도 색상 없음)

### 연결 화면
- 이전: `02-일별뷰`
- 다음: `02-일별뷰` (즉시 복귀, UI 갱신)
