## 📝 디스크립션

### 사용 시점 (Trigger)
- `payrollList.every(p => p.status === 'CONFIRMED')`
- DLG-064-002 급여 확정 성공 후 목록 갱신

### 인터랙션 (User Actions)
1. "명세서 확인" → `/payroll/statements`
2. MonthPicker → 다른 월 선택
3. 행 클릭 → DLG-064-001 (읽기 전용)

### 비즈니스 룰
- 확정 후 급여 수정 불가 (DLG-064-001 편집 필드 disabled)
- 확정 취소 기능 없음 (센터장 이상 지원팀 요청 필요)
- 명세서 자동 생성: 급여 확정과 동시에 payroll_statements 레코드 생성

### 에지 케이스
