## 상태 진입 조건
- `editTarget !== null` 이고 `open === true`
- 역할 ∈ {owner, manager, fc} + 본인 지점 리드(RLS)

## 상태 고유 비주얼 델타
- 타이틀: **"리드 수정"**
- 모든 필드 `editTarget` 프리필
- 상태 pill: 기존 상태 활성
- 저장 라벨 **"저장"**

## 역할별 차이
- super/primary: 타지점 리드 수정 가능
- owner/manager/fc: 본 지점만 (RLS 보강)

## 상태 고유 인터랙션
1. `editTarget` 변경 시 useEffect로 form 재동기화
2. 이름 input 오토포커스
3. 변경 없어도 저장 가능(멱등)
4. ESC/X → dirty confirm (선택)

## 상태 고유 비즈니스 룰 델타
- `updateLead(id, payload)` 호출 시 branchId 변경 불가 (서버 무시)
- `convertedMemberId`는 이 모달에서 수정 안함
- `status='등록완료'` 변경 시 별도 전환 플로우 알림(선택)

## 에지 케이스
- 동시 삭제 → 저장 시 404 → `05-API오류`
- stale editTarget → 서버 최신으로 강제 리프레시
- 타지점 RLS 403 → `toast.error("권한이 없습니다.")`

## 🧩 바이브코딩 프롬프트 (델타)
```

useEffect(() => {
  if(open && editTarget) {
    setForm({ ...editTarget }); nameRef.current?.focus();
  }
}, [open, editTarget]);
헤더 타이틀: '리드 수정'
저장 호출: updateLead(editTarget.id, form)
```
