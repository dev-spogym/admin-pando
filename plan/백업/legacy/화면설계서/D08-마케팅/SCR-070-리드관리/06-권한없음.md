## 상태 고유 인터랙션
1. **[대시보드로 이동] 클릭** → 역할별 홈 리다이렉트
2. **[관리자에게 문의] 클릭** → mailto 링크 (`mailto:admin@{branch}.kr?subject=권한요청`)
3. **자동 리다이렉트(옵션)**: 5초 카운트다운 후 자동 이동, ESC로 취소 가능
4. **브라우저 뒤로가기**: 이전 페이지 복귀
5. **URL 직접 재입력**: 동일 화면 재표시

## 상태 고유 비즈니스 룰 델타
- 진입 시 감사 로그 `AUDIT.ACCESS_DENIED(path='/leads', role, branchId, userId)` 기록
- Next.js middleware에서 사전 차단 — `/forbidden` 리다이렉트로 URL 자체 차단
- 클라이언트 단독 체크 금지: 반드시 서버 RLS/API 403 이중 검증
- 사이드바 메뉴 동일 `canAccessLeads()`로 필터링 — 해당 역할에 "리드 관리" 메뉴 미표시
- `readonly`는 이 상태에 진입하지 않음 — 정상 화면에 진입하여 모든 액션 disabled

## 에지 케이스
- **세션 중 역할 변경**: admin이 역할 강등 → refetch 시 403 → 이 상태
- **URL 쿼리 조작**: `?branch=` 다른 지점 시도 → 서버 403 → `/forbidden`
- **middleware 우회**: 클라이언트 SPA 라우팅으로 도달 시도 → page.tsx 내부 `canAccessLeads()` 재확인
- **자동 리다이렉트 중 사용자 입력**: 카운트다운 중 어떤 버튼이든 클릭하면 카운트다운 취소

import { NextResponse } from 'next/server';
import { getRoleFromJwt } from '@/lib/auth';

const DENY_ROLES_FOR_LEADS = ['trainer', 'staff', 'front'];

export function middleware(req) {
  const { pathname } = req.nextUrl;
  if (pathname.startsWith('/leads')) {
    const role = getRoleFromJwt(req);
    if (DENY_ROLES_FOR_LEADS.includes(role)) {
      return NextResponse.redirect(new URL('/forbidden?from=/leads', req.url));
    }
  }
  return NextResponse.next();
}
export const config = { matcher: ['/leads/:path*'] };
