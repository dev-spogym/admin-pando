## 상태 고유 인터랙션

### 목록 뷰 (list)
1. **행 hover** → `bg-gray-50`, cursor-pointer
2. **행 클릭**(옵션) → 리드 상세 미니 모달 또는 수정 모달로 바로 이동
3. **[수정 ✏] 클릭** → `openEdit(lead)` → DLG-070-001 (수정모드)
4. **[삭제 🗑] 클릭** (owner+만) → `openDelete(lead)` → DLG-070-002
5. **`convertedMemberId != null` 행** → 이름 옆 `ExternalLink` 아이콘 → 회원 상세
6. **컬럼 헤더 클릭**(옵션) → 정렬 토글(`createdAt`만 기본 지원)

### 칸반 뷰 (kanban)
1. **카드 드래그**(Phase 2, 기본 비활성) → 다른 컬럼 드롭 시 `status` 변경 + PATCH
2. **카드 클릭** → 수정 모달 오픈
3. **컬럼 스크롤**: 카드 많으면 컬럼 내부 세로 스크롤

### 공용
- **필터 변경**: 즉시 `useMemo` 재계산, 테이블/칸반 리프레시(애니메이션 없음)
- **검색 디바운스**: 300ms
- **새로고침**(옵션 `⟳` 버튼): `refetch()` → 상단 `isRefetching` 인디케이터

## 상태 고유 비즈니스 룰 델타
- 정렬 기본: `createdAt DESC` (최신 리드가 맨 위)
- Pagination: 25/페이지 (모바일은 무한 스크롤 대체 가능)
- 검색어 입력 시 `searchValue` 유지(URL 쿼리 `?q=` 반영 옵션)
- 칸반에서 `미전환/보류` 리드는 기본 숨김, "기타 리드 N건" 접이식 섹션
- 통계 카드는 **필터 적용 전 전체** 기준 (필터링된 결과의 통계가 아님)

## 에지 케이스
- **대량 데이터(>1,000건)**: 가상 스크롤(react-virtuoso) 고려(Phase 2)
- **검색 결과 0건**: 테이블 영역에 "검색 결과가 없습니다. 필터를 초기화하세요." + [필터 초기화] 버튼
- **mutation 직후 낙관적 업데이트**: 즉시 UI 반영 후 실패 시 롤백 + 에러 toast
- **동일 리드 다중 탭 수정**: Supabase realtime subscribe로 다른 탭도 업데이트(옵션)
- **status='등록완료'**이지만 `convertedMemberId == null`: 경고 배지 "미연결" 표시
