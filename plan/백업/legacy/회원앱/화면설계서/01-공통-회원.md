## SCR-MA-001. 로그인

### 1. 화면 목적

비로그인 사용자가 연락처 + 비밀번호를 입력하여 앱에 로그인한다. 스플래시(SCR-MA-004) 이후 로그인 세션이 없을 때 진입하는 첫 화면이다.

### 2. 접근 경로

| 접근 방법 | 경로 |
|---|---|
| 앱 최초 실행 (비로그인) | 스플래시 → 로그인 |
| 로그아웃 후 | 설정 → 로그아웃 → 로그인 |
| 세션 만료 | 임의 화면 → 세션 만료 토스트 → 로그인 |
| 자동 로그인 실패 | 스플래시 → 자동 로그인 실패 → 로그인 |

### 3. 역할별 접근

| 역할 | 접근 가능 | 비고 |
|---|---|---|
| 비로그인 | O | 기본 진입 화면 |
| member | X | 이미 로그인 상태이면 홈으로 리다이렉트 |
| staff / admin | X | CRM 앱 별도 진입 |

### 4. 와이어프레임

```
┌─────────────────────────────┐
│                             │
│      [FitGenie 로고]        │
│                             │
│  ┌─────────────────────┐    │
│  │ 연락처 (010-xxxx-xxxx)│   │
│  └─────────────────────┘    │
│                             │
│  ┌─────────────────────┐    │
│  │ 비밀번호       [👁] │    │
│  └─────────────────────┘    │
│                             │
│  ☐ 자동 로그인              │
│                             │
│  ┌─────────────────────┐    │
│  │      로그인         │    │
│  └─────────────────────┘    │
│                             │
│  앱 연동/가입  |  비밀번호 찾기│
│                             │
└─────────────────────────────┘
```

### 5. 구성 요소 상세

| 요소 | 타입 | 상세 |
|---|---|---|
| FitGenie 로고 | Image | 중앙 정렬, 상단 1/3 위치 |
| 연락처 입력 | TextInput | 숫자만 허용, 자동 하이픈(010-XXXX-XXXX), keyboardType=phone-pad |
| 비밀번호 입력 | TextInput | secureTextEntry=true, 우측 눈 아이콘으로 토글 |
| 자동 로그인 | Checkbox | 기본값 OFF, 로컬 저장 |
| 로그인 버튼 | Button | Primary, 연락처+비밀번호 입력 시 활성화 |
| 앱 연동/가입 링크 | TextButton | SCR-MA-002 이동 |
| 비밀번호 찾기 링크 | TextButton | SCR-MA-003 이동 |

### 6. 인터랙션

- 연락처 입력: 숫자 11자리 자동 하이픈 삽입 (01012345678 → 010-1234-5678)
- 비밀번호 눈 아이콘 탭: secureTextEntry 토글
- 자동 로그인 체크: AsyncStorage에 저장, 다음 앱 실행 시 자동 로그인 시도
- [로그인] 탭: 로딩 스피너 표시 → API 호출 → 결과 처리
- 키보드 표시 시: 화면 스크롤 또는 버튼 위치 조정 (KeyboardAvoidingView)

### 7. 상태별 화면

**기본 상태 (입력 전)**
- 로그인 버튼: 비활성(회색)

**입력 중 상태**
- 연락처/비밀번호 모두 입력 시 로그인 버튼 활성화(Primary 색상)

**로딩 상태**
- 로그인 버튼 내 스피너 표시, 버튼 비활성화
- 입력 필드 비활성화

**오류 상태 (유효성 검증 실패)**

| 오류 유형 | 메시지 | 위치 |
|---|---|---|
| 연락처 형식 불일치 | "올바른 연락처를 입력해주세요" | 연락처 필드 하단 Red |
| 비밀번호 미입력 | "비밀번호를 입력해주세요" | 비밀번호 필드 하단 Red |
| 로그인 실패 (잘못된 정보) | "연락처 또는 비밀번호가 올바르지 않습니다" | 상단 토스트 또는 필드 하단 |
| 계정 잠김 (5회 실패) | "계정이 잠겼습니다. 비밀번호 찾기를 이용해주세요" | 모달 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" | 상단 토스트 |

**성공 상태**
- 홈 화면(SCR-MA-100)으로 이동

## SCR-MA-002. 앱 연동/가입

### 1. 화면 목적

CRM에 등록된 회원이 회원앱 계정을 처음 생성(연동)한다. Step 형태로 진행한다.

### 2. 접근 경로

| 접근 방법 | 경로 |
|---|---|
| 로그인 화면 | "앱 연동/가입" 링크 탭 |

### 3. 역할별 접근

| 역할 | 접근 가능 | 비고 |
|---|---|---|
| 비로그인 | O | 신규 연동 전용 |
| member | X | 이미 가입 완료 상태 |

### 4. 와이어프레임

**Step 1 — 정보 입력**

```
┌─────────────────────────────┐
│  ← 뒤로      앱 연동/가입   │
│                             │
│  ● ─────── ○ ─────── ○     │
│  Step1     Step2    Step3   │
│  정보입력  SMS인증  비밀번호  │
│                             │
│  이름                       │
│  ┌─────────────────────┐    │
│  │ 홍길동              │    │
│  └─────────────────────┘    │
│                             │
│  연락처                     │
│  ┌──────────────┐ [인증요청]│
│  │ 010-xxxx-xxxx│          │
│  └──────────────┘          │
│                             │
│  ┌─────────────────────┐    │
│  │      다음           │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

**Step 2 — SMS 인증**

```
┌─────────────────────────────┐
│  ← 뒤로      앱 연동/가입   │
│                             │
│  ● ─────── ● ─────── ○     │
│  Step1     Step2    Step3   │
│                             │
│  010-1234-5678로            │
│  인증번호를 발송했습니다.    │
│                             │
│  인증번호                   │
│  ┌──────────────────┐       │
│  │  _ _ _ _ _ _    │ 04:59 │
│  └──────────────────┘       │
│  [인증번호 재발송]           │
│                             │
│  ┌─────────────────────┐    │
│  │      확인           │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

**Step 3 — 비밀번호 설정**

```
┌─────────────────────────────┐
│  ← 뒤로      앱 연동/가입   │
│                             │
│  ● ─────── ● ─────── ●     │
│  Step1     Step2    Step3   │
│                             │
│  [CRM 매칭 결과]            │
│  홍길동님, 환영합니다!      │
│  회원번호: MEM-001234       │
│                             │
│  새 비밀번호                │
│  ┌─────────────────────┐    │
│  │ 8자 이상, 영문+숫자 │    │
│  └─────────────────────┘    │
│                             │
│  비밀번호 확인              │
│  ┌─────────────────────┐    │
│  └─────────────────────┘    │
│                             │
│  ☐ 이용약관 동의 (필수)     │
│  ☐ 개인정보처리방침 (필수)  │
│  ☐ 마케팅 수신 동의 (선택)  │
│                             │
│  ┌─────────────────────┐    │
│  │    가입 완료        │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

### 5. 구성 요소 상세

| 요소 | 타입 | 상세 |
|---|---|---|
| 스텝 인디케이터 | StepBar | 현재 단계 강조, 완료 단계 채움 |
| 이름 | TextInput | 한글/영문, 최대 20자 |
| 연락처 | TextInput | 자동 하이픈, phone-pad |
| [인증요청] | Button | Secondary, 연락처 10-11자리 입력 시 활성화 |
| 인증번호 | TextInput | 6자리 숫자, OTP 자동완성 지원 |
| 타이머 | Text | MM:SS 카운트다운, 0:00 도달 시 재발송만 가능 |
| 비밀번호 | TextInput | 8자 이상, 영문+숫자 필수 조합 |
| 비밀번호 확인 | TextInput | 위와 일치 검증 |
| 약관 체크박스 | Checkbox | 필수 2개 모두 동의 시 가입 버튼 활성화 |

### 6. 인터랙션

- Step 1 → Step 2: [다음] 탭 시 SMS 발송 API 호출, 성공 시 Step 2 이동
- 타이머: 5분 카운트다운, 만료 시 인증번호 입력 불가 + 재발송 안내
- Step 2 → Step 3: 인증 성공 시 CRM 회원 매칭 결과 표시
- CRM 미매칭: 오류 모달 ("센터에 등록된 회원 정보가 없습니다. 센터로 문의해주세요")
- 약관 전체동의 체크박스: 하위 3개 일괄 선택/해제

### 7. 상태별 화면

**CRM 매칭 실패**
```
┌─────────────────────────────┐
│  [!] 회원 정보 미확인       │
│  입력하신 정보로 등록된     │
│  회원을 찾을 수 없습니다.   │
│  센터에 문의해주세요.       │
│  [확인]                     │
└─────────────────────────────┘
```

**인증번호 불일치**
- 인증번호 필드 하단: "인증번호가 올바르지 않습니다" (Red)

**인증 시간 초과**
- 타이머 Red 표시, 입력 필드 비활성화, "인증번호가 만료되었습니다. 재발송해주세요"

**가입 완료**
- 성공 모달: "가입이 완료되었습니다!" → [확인] → 홈 이동

## SCR-MA-003. 비밀번호 찾기

### 1. 화면 목적

비밀번호를 잊은 회원이 SMS 인증을 통해 새 비밀번호를 설정한다.

### 2. 접근 경로

| 접근 방법 | 경로 |
|---|---|
| 로그인 화면 | "비밀번호 찾기" 링크 탭 |

### 3. 역할별 접근

| 역할 | 접근 가능 | 비고 |
|---|---|---|
| 비로그인 | O | |
| member | O | 로그인 상태에서도 설정에서 접근 가능 |

### 4. 와이어프레임

**Step 1 — 연락처 입력**

```
┌─────────────────────────────┐
│  ← 뒤로    비밀번호 찾기    │
│                             │
│  ● ─────── ○ ─────── ○     │
│  연락처인증  새비밀번호  완료 │
│                             │
│  가입 시 등록한 연락처를    │
│  입력해주세요.              │
│                             │
│  ┌──────────────┐ [인증요청]│
│  │ 010-xxxx-xxxx│          │
│  └──────────────┘          │
│                             │
│  인증번호                   │
│  ┌──────────────────┐       │
│  │                  │ 05:00 │
│  └──────────────────┘       │
│                             │
│  ┌─────────────────────┐    │
│  │      확인           │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

**Step 2 — 새 비밀번호 설정**

```
┌─────────────────────────────┐
│  ← 뒤로    비밀번호 찾기    │
│                             │
│  ● ─────── ● ─────── ○     │
│                             │
│  새 비밀번호                │
│  ┌─────────────────────┐    │
│  │ 8자 이상, 영문+숫자 │    │
│  └─────────────────────┘    │
│                             │
│  비밀번호 확인              │
│  ┌─────────────────────┐    │
│  └─────────────────────┘    │
│                             │
│  ┌─────────────────────┐    │
│  │    비밀번호 변경     │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

### 5. 구성 요소 상세

| 요소 | 타입 | 상세 |
|---|---|---|
| 연락처 | TextInput | 자동 하이픈 |
| [인증요청] | Button | Secondary |
| 인증번호 | TextInput | 6자리, OTP 자동완성 |
| 타이머 | Text | 5분 카운트다운 |
| 새 비밀번호 | TextInput | 8자+, 영문+숫자 조합 |
| 비밀번호 확인 | TextInput | 일치 검증 |

### 6. 인터랙션

- 등록되지 않은 연락처: "가입된 회원 정보가 없습니다" 토스트
