### C. 데이터 모델

**타입**: `SaleItem`

```typescript
type SaleItem = {
  id: number;
  no: number;
  purchaseDate: string;     // YYYY-MM-DD (saleDate에서 slice)
  type: string;             // 유형 (이용권, PT, 상품, 기타)
  productName: string;      // 상품명
  manager: string;          // 담당자 (staffName)
  buyer: string;            // 구매자 (memberName)
  buyerId: number;          // 회원 ID (memberId)
  round: string;            // 매출유형 (신규, 재등록, 휴면복귀, 종목추가, 업그레이드)
  quantity: number;         // 수량 (기본값 1)
  originalPrice: number;    // 정가 (originalPrice 또는 amount)
  salePrice: number;        // 판매가 (salePrice 또는 amount)
  discountPrice: number;    // 할인금액
  paymentMethod: string;    // 결제수단 영문 (CARD, CASH, TRANSFER, MILEAGE)
  paymentType: string;      // 결제유형
  paymentTool: string;      // 결제수단 한글 (카드, 현금, 계좌이체, 마일리지)
  cash: number;             // 현금 결제액
  card: number;             // 카드 결제액
  mileage: number;          // 마일리지 사용액 (mileageUsed)
  cardCompany: string;      // 카드사
  cardNumber: string;       // 카드번호
  approvalNo: string;       // 승인번호
  unpaid: number;           // 미수금
  serviceDays: number;      // 서비스 일수 (현재 항상 0)
  serviceCount: number;     // 서비스 횟수 (현재 항상 0)
  servicePoints: number;    // 서비스 포인트 (현재 항상 0)
  status: string;           // 상태 한글 (완료, 미납, 환불, 대기)
  category: string;         // 카테고리 (현재 빈 문자열)
  memo: string;             // 메모
};
```

**DB 상태 영문-한글 매핑** (`STATUS_KO`):

| DB 값 (영문) | 표시 값 (한글) |
|-------------|--------------|
| `COMPLETED` | `완료` |
| `UNPAID` | `미납` |
| `REFUNDED` | `환불` |
| `PENDING` | `대기` |

**결제수단 영문-한글 매핑** (`PAYMENT_KO`):

| DB 값 (영문) | 표시 값 (한글) |
|-------------|--------------|
| `CARD` | `카드` |
| `CASH` | `현금` |
| `TRANSFER` | `계좌이체` |
| `MILEAGE` | `마일리지` |

### F. 버튼/액션

| 버튼 | 위치 | 조건 | 동작 |

**SearchFilter 컴포넌트 필터 옵션** (`filters`):

| key | label | type | options |

**장바구니 추가 로직** (`handleAddToCart`):
1. 동일 상품(동일 `id`, `priceType === 'cash'`)이 이미 있으면 → `quantity + 1`
2. 없으면 → 새 CartItem 추가 (`priceType: 'cash'`, `quantity: 1`, `cartId: \`${product.id}-${Date.now()}\``)

**결제 이동 로직**:
1. `cart`를 `{ id, name, category, price, quantity }` 형태로 변환
2. `price`는 `priceType === 'cash' ? cashPrice : cardPrice`
3. `sessionStorage.setItem('posCart', JSON.stringify(cartData))`
4. `buyer`가 있으면 `sessionStorage.setItem('posBuyer', JSON.stringify(buyer))`
5. `buyer`가 없으면 `sessionStorage.removeItem('posBuyer')`
6. `moveToPage(982)` → 결제 페이지 이동

### J. API 호출

**결제 처리** (`executePayment`):

```
supabase.from('sales').insert({
  branchId: getBranchId(),
  memberId: selectedMember?.id ?? null,
  memberName: selectedMember?.name ?? '비회원',
  type: 'POS',
  productName: cartItems.map(i => i.name).join(', '),
  amount: subtotal,
  salePrice: subtotal,
  originalPrice: subtotal,
  paymentMethod: paymentMethodMap[paymentMethod],
  card: paymentMethod === 'card' ? subtotal : (paymentMethod === 'mixed' ? mixedAmount.card : 0),
  cash: paymentMethod === 'cash' ? subtotal : (paymentMethod === 'mixed' ? mixedAmount.cash : 0),
  saleDate: new Date().toISOString(),
  status: 'COMPLETED',
  memo: mixedMemo,  // 복합결제 시에만
})
```

**결제수단 DB 매핑** (`paymentMethodMap`):

| PaymentMethod | DB 값 |
|--------------|-------|
| `card` | `CARD` |
| `cash` | `CASH` |
| `mileage` | `MILEAGE` |
| `mixed` | `CARD` (memo에 상세 기록) |

**마일리지 차감** (조건: `selectedMember && (mileage결제 || mixed에서 mileage > 0)`):
- `deductPoints(selectedMember.id, mileageToDeduct)` 호출
- 실패 시: `toast.error(deductResult.message ?? '마일리지 차감에 실패했습니다.')`

**마일리지 적립** (조건: `selectedMember && paymentMethod !== 'mileage'`):
- `accruePoints(selectedMember.id, subtotal)` 호출 (1% 적립)
- 성공 시: `toast.info(\`${pointResult.accrued}P 마일리지가 적립되었습니다.\`)`

**중복 결제 확인** (`handlePaymentConfirm`):
- `checkDuplicatePayment(selectedMember.id, subtotal)` 호출
- 중복 감지 시: `setDupMessage()` + `setShowDupConfirm(true)`
- 중복 미감지 시: `executePayment()` 직접 호출

### L. 비즈니스 로직

**결제 처리 플로우**:

```
1. handlePaymentConfirm() 호출
   ├─ 회원 선택됨? → checkDuplicatePayment()
   │   ├─ 중복 → 경고 다이얼로그 표시 → 사용자 확인 → executePayment()
   │   └─ 미중복 → executePayment()
   └─ 비회원 → executePayment()

2. executePayment() 내부
   ├─ isProcessing 체크 (중복 실행 방지)
   ├─ 마일리지 차감 (해당 시)
   ├─ sales 테이블 INSERT
   ├─ 마일리지 적립 (해당 시, 결제금액의 1%)
   └─ 완료 → isComplete = true
```

**복합결제 메모 생성**:
- `paymentMethod === 'mixed'` 일 때만
- 형식: `"복합결제 - 카드: {N}원, 현금: {N}원, 마일리지: {N}P"`

1. `setSelectedMember(null)`
2. `setMemberSearch('')`
3. `setPaymentMethod('card')`

5. `setIsComplete(false)`

### D. 함수 상세

#### `getSales(params?)`

**파라미터**:

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `page` | `number` | `1` | 페이지 번호 |
| `size` | `number` | `20` | 페이지 크기 |
| `startDate` | `string` | - | 시작일 필터 |
| `endDate` | `string` | - | 종료일 필터 |
| `memberId` | `number` | - | 회원 ID 필터 |

**쿼리**:
- `select('*', { count: 'exact' })`
- `.eq('branchId', branchId)`
- `.order('saleDate', { ascending: false })`
- `.range(from, to)`

**에러**: `throw new Error(error.message)`

#### `createSale(payload)`

**자동 설정 필드**:
- `saleDate`: `payload.saleDate ?? new Date().toISOString()`
- `status`: `payload.status ?? 'COMPLETED'`
- `branchId`: `getBranchId()`

**감사 로그**: `action: AUDIT_ACTIONS.CREATE, targetType: 'sale', targetId: data.id`
**성공 메시지**: `'매출이 등록되었습니다.'`

#### `updateSale(id, payload)`

**감사 로그**: `action: AUDIT_ACTIONS.UPDATE, targetType: 'sale', targetId: id`
**성공 메시지**: `'매출이 수정되었습니다.'`

#### `deleteSale(id)`

**감사 로그**: `action: AUDIT_ACTIONS.DELETE, targetType: 'sale', targetId: id`
**성공 메시지**: `'매출이 삭제되었습니다.'`

#### `getSalesStats(params?)`

**쿼리 2개 병렬 실행**:
1. 전체 통계: `status === 'COMPLETED'` + 날짜 범위
2. 이번달 통계: `status === 'COMPLETED'` + 이번달 범위 (1일~말일)

**반환 계산**:
- `totalAmount`: 전체 amount 합산
- `totalCount`: 전체 건수
- `monthlyAmount`: 이번달 amount 합산
- `monthlyCount`: 이번달 건수
- `byPaymentMethod`: paymentMethod별 amount 합산 (Record)

### B. 매출 관련 함수

#### `getNetSalesStats(params?)`

순매출 계산 (환불 건 반영):

| 반환 필드 | 계산 |
|-----------|------|
| `grossSales` | `COMPLETED` 상태 매출의 `amount` 합산 |
| `refundAmount` | `REFUNDED` 상태 매출의 `amount` 합산 |
| `netSales` | `grossSales - refundAmount` |
| `completedCount` | `COMPLETED` 건수 |
| `refundCount` | `REFUNDED` 건수 |

#### `accruePoints(memberId, paymentAmount, rate?)`

마일리지 자동 적립:
- 기본 적립율: `1%` (`rate = 0.01`)
- 적립액: `Math.floor(paymentAmount * rate)`
- 적립액 0 이하 시 스킵
- `members` 테이블 `mileage` 필드 업데이트

| 반환 | 설명 |
|------|------|
| `success: true, accrued: N, newBalance: M` | 성공 |
| `success: false, accrued: 0` | 회원 조회 실패 |
| `success: false, accrued: N` | 업데이트 실패 |

#### `deductPoints(memberId, amount)`

마일리지 차감:
- 잔액 부족 시: `{ success: false, message: '마일리지 부족 (보유: N, 필요: M)' }`
- 성공 시: `{ success: true, newBalance: N }`
- 조회 실패: `{ success: false, message: '회원 정보 조회 실패' }`
- 업데이트 실패: `{ success: false, message: '마일리지 차감 실패' }`

#### `checkDuplicatePayment(memberId, amount)`

중복 결제 확인 (최근 5분 이내):

```
supabase
  .from('sales')
  .select('id, saleDate')
  .eq('branchId', branchId)
  .eq('memberId', memberId)
  .eq('amount', amount)
  .eq('status', 'COMPLETED')
  .gte('saleDate', fiveMinAgo)
  .limit(1)
```

| 반환 | 조건 |
|------|------|
| `{ isDuplicate: true, message: '최근 5분 이내 동일한 금액의 결제가 이미 처리되었습니다. 중복 결제인지 확인해주세요.' }` | 동일 회원+금액+5분 이내 결제 존재 |
| `{ isDuplicate: false }` | 없음 또는 에러 |

#### `updateMembershipPeriod(memberId, startDate, endDate, productName?)`

결제 후 이용권 시작/종료일 자동 설정:
- `members` 테이블의 `membershipStart`, `membershipExpiry`, `membershipType`, `status` 업데이트
- `status`를 `ACTIVE`로 변경
- 성공: `'이용권 기간이 설정되었습니다.'`
- 실패: `'이용권 기간 설정 실패: ' + error.message`

선수익금 목록 조회:
