## 1. 직원 목록 (`/staff`)

> 소스: `src/app/staff/page.tsx`

### A. 화면 개요

센터에 등록된 모든 직원의 정보를 테이블 형태로 조회하고, 검색·필터·정렬을 통해 빠르게 탐색할 수 있는 화면이다. 선택한 직원에 대해 메시지 전송, 상태 변경, 퇴사 처리를 수행할 수 있다.

- 컴포넌트: `StaffList` (default export)
- 라우트: `/staff`
- 레이아웃: `AppLayout` > `PageHeader` + `StatCardGrid` + `SearchFilter` + 액션 버튼 바 + `DataTable`
- 페이지네이션: `PAGE_SIZE = 10`

### G. 모달/팝업

#### G-1. 퇴사 처리 확인 (ConfirmDialog)

- `open={isRetireDialogOpen}`
- `title="직원 퇴사 처리"`
- `description`:
  ```
  선택한 ${selectedRows.size}명의 직원을 퇴사 처리하시겠습니까?
  퇴사 처리 시 해당 직원의 계정 접근이 제한됩니다.
  ```
- `confirmLabel="퇴사 처리"`
- `variant="danger"`
- `confirmationText="퇴사처리"` (텍스트 입력 확인)
- onConfirm:
  1. `const ids = Array.from(selectedRows)`
  2. `supabase.from("staff").update({ isActive: false }).in("id", ids)`
  3. 성공: `toast.success(${ids.length}명의 퇴사 처리가 완료되었습니다.)`
  4. 실패: `toast.error("퇴사 처리에 실패했습니다.")`
  5. `setIsRetireDialogOpen(false)` + `setSelectedRows(new Set())` + `fetchStaff()`
- onCancel: `setIsRetireDialogOpen(false)`

### I. 필터/검색/정렬

#### SearchFilter 컴포넌트

- `searchPlaceholder="직원 이름, 연락처 검색"`
- `searchValue={searchQuery}`
- `onSearchChange={setSearchQuery}`

#### 필터 정의

| key | label | type | options |
|-----|-------|------|---------|
| `role` | 역할 | select | `owner(센터장)`, `manager(매니저)`, `fc(FC)`, `trainer(트레이너)`, `staff(스태프)` |
| `status` | 재직 상태 | select | `active(재직)`, `leave(휴직)`, `resigned(퇴사)` |

#### 필터링 로직

```typescript
let data = staffData.filter(s => {
  const matchSearch = !searchQuery || s.name.includes(searchQuery) || s.contact.includes(searchQuery);
  const matchRole   = !filterValues.role   || s.role   === filterValues.role;
  const matchStatus = !filterValues.status || s.status === filterValues.status;
  return matchSearch && matchRole && matchStatus;
});
```

#### 정렬

- 3-state 토글: null → `asc` → `desc` → null (해제)
- 정렬 가능 컬럼: `name`, `role`, `joinDate`, `status`
- 정렬 방식: `aVal.localeCompare(bVal, "ko")`
- 정렬 아이콘:
  - 미정렬: `ChevronsUpDown` (13, opacity-50)
  - asc: `ChevronUp` (13, text-primary)
  - desc: `ChevronDown` (13, text-primary)

#### 필터 리셋

```typescript
const handleResetFilters = () => {
  setFilterValues({ role: "", status: "" });
  setSearchQuery("");
  setCurrentPage(1);
};
```

```typescript
const handleBack = () => {
  if (currentStep === 3 && !isFC) {
    setCurrentStep(1);  // FC 아닌 경우 Step 2 건너뜀
  } else {
    setCurrentStep((prev) => Math.max(1, prev - 1));

};
```

```typescript
const applyBulkAssign = () => {

};
```

```typescript
    const value = `${d.getFullYear()}-${String(d.getMonth() + 1).padStart(2, "0")}`;
    const label = `${d.getFullYear()}년 ${d.getMonth() + 1}월`;
    months.push({ value, label });
  }
  return months;
}
```

## 부록 B. 직원 라이프사이클 상태 전이

```
                    ┌──────────┐
                    │  ACTIVE  │ ← 최초 등록 / 복직
                    └────┬─────┘
                         │
            ┌────────────┼────────────┐
            │            │            │
            ▼            ▼            ▼
     ┌───────────┐ ┌──────────┐ ┌───────────┐
     │ ON_LEAVE  │ │ RESIGNED │ │TRANSFERRED│
     │  (휴직)   │ │  (퇴사)  │ │  (전보)   │
     └─────┬─────┘ └──────────┘ └───────────┘
           │
           ▼
     ┌───────────┐
     │  ACTIVE   │ (복직)
     └───────────┘
```

- `ACTIVE` → `ON_LEAVE`: `startStaffLeave()` — `isActive: false`
- `ON_LEAVE` → `ACTIVE`: `endStaffLeave()` — `isActive: true`
- `ACTIVE` → `RESIGNED`: `confirmResignation()` — `isActive: false`
- `ACTIVE` → `TRANSFERRED`: `transferStaff()` — 기존 레코드 비활성화, 새 지점에 새 레코드 생성

### 주의: 프론트엔드 매핑 한계

현재 `staff/page.tsx`에서는 `isActive === false`이면 모두 `resigned`로 매핑한다. `staffStatus` 필드를 사용하여 `ON_LEAVE`, `TRANSFERRED` 상태를 별도로 표시하는 기능은 미구현이다.

```typescript
// 현재 매핑 (한계)
status: s.isActive === false ? "resigned" : "active"

// 이상적 매핑 (미구현)
status: s.staffStatus === 'ON_LEAVE' ? "leave"
      : s.staffStatus === 'RESIGNED' ? "resigned"
      : s.staffStatus === 'TRANSFERRED' ? "transferred"
      : "active"
```
