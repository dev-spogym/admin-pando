# X11 — 회원 탈퇴 → 개인정보 마스킹 → 감사로그 적재

## 1. 시나리오 개요

회원이 탈퇴 요청 → 매니저가 DLG-M005 탈퇴 모달에서 확인 → 이용권/미수금 확인 → 개인정보 마스킹 처리 → 감사로그 적재까지의 탈퇴 플로우. 개인정보보호법 준수를 위한 마스킹이 핵심.

| 항목 | 내용 |
|------|------|
| 트리거 | 회원 탈퇴 요청 (방문/전화) |
| 종료 조건 | 회원 WITHDRAWN 상태 + 개인정보 마스킹 + 감사로그 |
| 참여 도메인 | 회원관리(D2), 본사관리(D10) |

## 2. 전제조건

- 매니저 계정 로그인 상태 (탈퇴 처리 권한)
- 개인정보 마스킹 정책이 설정되어 있음
- 감사로그 시스템 활성화 상태

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원 | 탈퇴 요청자 |
| 매니저 | 탈퇴 처리 승인 권한 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 감사로그서비스 | 변경 이력 적재 |
| 알림서비스 | 탈퇴 완료 알림 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor MB as 회원
    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    participant AUD as 감사로그서비스
    participant N as 알림서비스

    MB->>M: 1. 탈퇴 요청 %% E_MB_WITHDRAW_REQUEST_01
    M->>API: 2. GET /members/{id}/withdraw-check %% E_M_WITHDRAW_CHECK_01
    API->>DB: 3. SELECT 활성 이용권, 미수금, 예약 현황
    DB-->>API: 4. 탈퇴 전 확인 사항
    API-->>M: 5. 탈퇴 전 체크 결과 { active_memberships, unpaid, upcoming_reservations }
    Note over M: DLG-M005 탈퇴 모달 열기

    alt 미수금 또는 활성 이용권 존재
        M->>MB: 6. 미수금 정산 / 이용권 환불 처리 먼저 안내
        Note over M: 탈퇴 불가 — 선행 처리 필요
    else 탈퇴 조건 충족
        M->>API: 7. POST /members/{id}/withdraw { reason, confirm } %% E_M_WITHDRAW_01
        API->>DB: 8. UPDATE members SET status=WITHDRAWN, withdrawn_at=NOW()
        API->>DB: 9. UPDATE memberships SET status=TERMINATED (잔여 이용권)
        API->>DB: 10. DELETE/UPDATE reservations (예약 취소)
        API->>DB: 11. 개인정보 마스킹 처리 %% E_API_MASK_PII_01
        Note over DB: name → "탈퇴회원", phone → "***-****-****", email → "***@***"
        DB-->>API: 12. OK
        API->>AUD: 13. 감사로그 적재 { action=MEMBER_WITHDRAW, operator_id, member_id, timestamp } %% E_API_AUDIT_01
        AUD->>DB: 14. INSERT audit_logs { masked_before, action, operator }
        DB-->>AUD: 15. log_id
        AUD-->>API: 16. OK
        API->>N: 17. 탈퇴 완료 알림 (마스킹 전 연락처로 발송) %% E_API_WITHDRAW_NOTIFY_01
        N-->>MB: 18. SMS — "탈퇴가 완료되었습니다"
        API-->>M: 19. 200 OK { withdrawn_at }
        Note over M: 회원 목록에서 제거 (또는 탈퇴회원 필터로 표시)
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 2 | GET /withdraw-check | 탈퇴 전 미수금/활성이용권/예약 존재 여부 확인 |
| 11 | 개인정보 마스킹 | 이름/전화/이메일/주소 마스킹. 결제 이력은 법적 의무 보존 기간 동안 유지 |
| 13 | 감사로그 적재 | 누가, 언제, 어떤 회원을 탈퇴 처리했는지 불변 기록 |
| 17 | 탈퇴 완료 알림 | 마스킹 처리 전 저장된 연락처로 발송 (발송 후 마스킹) |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 미수금 보유 | 탈퇴 불가, 먼저 미수금 정산 안내 |
| 활성 이용권 존재 | 환불 처리 후 탈퇴 또는 이용권 포기 확인 |
| readonly 권한 | 탈퇴 처리 불가, 매니저 이상 권한 필요 |
| 마스킹 실패 | 트랜잭션 롤백, 탈퇴 미처리 상태 유지 |
