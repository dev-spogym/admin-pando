# X06 — 환불 요청 → 매니저 승인 → 정산 반영

## 1. 시나리오 개요

회원이 이용권 환불을 요청 → 매니저가 환불 처리 모달에서 환불 금액 확인 및 승인 → PG 취소 요청 → 이용권 비활성화 → 정산 반영까지의 환불 플로우.

| 항목 | 내용 |
|------|------|
| 트리거 | 회원 환불 요청 (방문/전화) |
| 종료 조건 | 환불 완료 + 이용권 REFUNDED + 정산 반영 |
| 참여 도메인 | 매출관리(D3), 회원관리(D2) |

## 2. 전제조건

- 매니저 계정 로그인 상태 (환불 처리 권한 보유)
- 환불 대상 결제건이 존재하며 환불 가능 상태
- PG 연동 정상 상태
- 환불 정책(위약금/기간별 공제율) 설정 완료

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원 | 환불 요청자 |
| 매니저 | 환불 승인 권한 보유 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| PG | 결제 대행사 (카드 취소 처리) |
| 알림서비스 | 환불 완료 알림 발송 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor MB as 회원
    actor M as 매니저
    participant API as CRM API
    participant DB as DB
    participant PG as PG사
    participant N as 알림서비스

    MB->>M: 1. 환불 요청 (이용권 사용 기간 안내) %% E_MB_REFUND_REQUEST_01
    M->>API: 2. GET /members/{id}/sales?status=PAID %% E_M_GET_SALES_01
    API->>DB: 3. SELECT sales WHERE member_id=? AND refundable=true
    DB-->>API: 4. 환불 가능 결제 목록
    API-->>M: 5. 결제 이력
    Note over M: SCR-M004 결제이력 탭 확인

    M->>API: 6. GET /sales/{sale_id}/refund-preview %% E_M_REFUND_PREVIEW_01
    API->>DB: 7. SELECT refund_policy, membership 사용 기간 계산
    DB-->>API: 8. 환불 정책 + 공제 금액
    API-->>M: 9. 환불 가능 금액 미리보기 { original, deduction, refundable }
    Note over M: DLG-M013 환불 처리 모달 열기

    alt 매니저 환불 승인
        M->>API: 10. POST /refunds { sale_id, reason, refund_amount } %% E_M_APPROVE_REFUND_01
        API->>PG: 11. 카드 취소 요청 (카드 결제 건)

        alt PG 취소 성공
            PG-->>API: 12. 취소 승인 응답 %% E_PG_CANCEL_OK_01
            API->>DB: 13. INSERT refunds { sale_id, amount, status=COMPLETED }
            API->>DB: 14. UPDATE sales SET status=REFUNDED
            API->>DB: 15. UPDATE memberships SET status=REFUNDED
            DB-->>API: 16. OK
            API->>N: 17. 환불 완료 알림 트리거 %% E_API_NOTIFY_REFUND_01
            N-->>MB: 18. SMS/카카오톡 — 환불 완료 + 환불 금액 안내
            API-->>M: 19. 200 OK { refund_id, refunded_amount }
            Note over M: SCR-S007 환불 관리에 기록 반영
        else PG 취소 실패
            PG-->>API: 20. 취소 실패 응답 %% E_PG_CANCEL_FAIL_01
            API-->>M: 21. PG 취소 실패 안내
            Note over M: 수동 환불(계좌이체) 처리 유도
        end

    else 매니저 환불 거절
        M->>API: 22. POST /refunds { sale_id, status=REJECTED, reason } %% E_M_REJECT_REFUND_01
        API->>DB: 23. INSERT refunds { status=REJECTED }
        DB-->>API: 24. OK
        API->>N: 25. 환불 거절 알림 트리거
        N-->>MB: 26. 환불 거절 사유 안내
        API-->>M: 27. 200 OK
    end
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 6 | GET /refund-preview | 결제 원금 - 사용 기간 공제 - 위약금 = 환불 가능 금액 계산 |
| 10 | POST /refunds | refund_amount는 미리보기와 다를 수 있음 (매니저 수동 조정 가능) |
| 11 | PG 카드 취소 | 현금 결제는 PG 취소 없이 계좌이체로 수동 환불 |
| 14 | UPDATE sales REFUNDED | 매출 레코드 환불 상태로 변경 → 매출 통계에서 차감 |
| 15 | UPDATE memberships REFUNDED | 이용권 즉시 비활성화 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 현금 결제 환불 | PG 취소 없이 계좌이체 정보 입력 후 수동 처리 |
| 부분 환불 | refund_amount < original_amount 허용, PARTIAL_REFUNDED 상태 |
| 이미 환불된 건 | refundable=false 체크 → 환불 불가 안내 |
| 환불 가능 기간 초과 | 정책에 따라 환불 불가 또는 위약금 100% |
| PG 취소 실패 | 수동 계좌이체 환불로 fallback |
