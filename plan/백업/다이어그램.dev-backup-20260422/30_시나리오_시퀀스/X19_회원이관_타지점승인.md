# X19 — 회원 이관 → 타지점 승인 → 이력 동기화

## 1. 시나리오 개요

회원이 거주지 이전 등의 사유로 타지점으로 이관 요청 → 원지점 매니저가 이관 신청 → 타지점 매니저가 승인 → 회원 소속 변경 및 이력 동기화까지의 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 회원의 지점 이관 요청 |
| 종료 조건 | 타지점 소속 변경 완료 + 이력 동기화 |
| 참여 도메인 | 회원관리(D2), 본사관리(D10) |

## 2. 전제조건

- 원지점 매니저 계정 로그인 상태
- 이관 대상 지점이 동일 CRM 시스템 내 존재
- 회원에게 미수금이 없고 환불 처리가 완료된 상태

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 회원 | 이관 요청자 |
| 원지점매니저 | 이관 신청 처리 |
| 타지점매니저 | 이관 승인 처리 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 알림서비스 | 이관 요청/승인 알림 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor MB as 회원
    actor OM as 원지점매니저
    actor TM as 타지점매니저
    participant API as CRM API
    participant DB as DB
    participant N as 알림서비스

    MB->>OM: 1. 지점 이관 요청 (사유 설명) %% E_MB_TRANSFER_REQUEST_01
    OM->>API: 2. GET /members/{id}/transfer-check %% E_OM_TRANSFER_CHECK_01
    API->>DB: 3. SELECT 미수금, 활성이용권, 예약 현황
    DB-->>API: 4. 이관 전 체크 결과
    API-->>OM: 5. 이관 가능 여부 { transferable, issues }

    alt 이관 조건 충족
        OM->>API: 6. POST /member-transfers { member_id, from_branch, to_branch, reason } %% E_OM_CREATE_TRANSFER_01
        API->>DB: 7. INSERT member_transfers { status=PENDING, requested_at=NOW() }
        DB-->>API: 8. transfer_id
        API->>N: 9. 타지점 이관 요청 알림 %% E_API_TRANSFER_REQUEST_NOTIFY_01
        N-->>TM: 10. 앱/웹 알림 — "이관 승인 요청: 회원 OOO"
        API-->>OM: 11. 200 OK { transfer_id }

        Note over TM,API: [타지점 매니저 승인]
        TM->>API: 12. GET /member-transfers/{id} %% E_TM_GET_TRANSFER_01
        API-->>TM: 13. 이관 요청 상세 (회원 정보, 이용권, 이력 요약)
        TM->>API: 14. POST /member-transfers/{id}/approve %% E_TM_APPROVE_TRANSFER_01
        API->>DB: 15. UPDATE members SET branch_id=to_branch
        API->>DB: 16. UPDATE memberships SET branch_id=to_branch (이용권 이관)
        API->>DB: 17. UPDATE member_transfers SET status=APPROVED, approved_at=NOW()
        DB-->>API: 18. OK
        API->>N: 19. 이관 완료 알림 %% E_API_TRANSFER_DONE_01
        N-->>MB: 20. SMS — "타지점 이관이 완료되었습니다"
        N-->>OM: 21. 앱 알림 — "이관 승인 완료"
        API-->>TM: 22. 200 OK

    else 이관 조건 미충족 (미수금 등)
        API-->>OM: 23. 이관 불가 사유 안내 %% E_API_TRANSFER_BLOCKED_01
        Note over OM: 선행 처리 후 재요청
    end

    Note over TM,API: [이관 거절]
    TM->>API: 24. POST /member-transfers/{id}/reject { reason } %% E_TM_REJECT_TRANSFER_01
    API->>DB: 25. UPDATE member_transfers SET status=REJECTED
    API->>N: 26. 이관 거절 알림
    N-->>OM: 27. 이관 거절 사유 알림
    N-->>MB: 28. SMS — "이관 요청이 거절되었습니다"
    API-->>TM: 29. 200 OK
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 2 | GET /transfer-check | 미수금/예약/이용권 현황 확인. 이관 조건 검증 |
| 15 | UPDATE members branch_id | 회원 소속 지점 변경. 접근 권한도 자동 변경 |
| 16 | UPDATE memberships branch_id | 이용권 귀속 지점 변경. 매출 집계에 반영 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 미수금 보유 | 이관 불가, 선행 정산 안내 |
| 타지점 정원 초과 | 타지점 매니저가 거절 사유로 정원 초과 선택 |
| 이관 취소 | PENDING 상태에서만 취소 가능 |
| 이용권 잔여 기간 | 원 지점 매출에서 제외, 타 지점 매출로 이관 |
