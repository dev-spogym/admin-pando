# X24 — 급여 지급일 → 자동 명세서 → 직원 수신

## 1. 시나리오 개요

X08에서 계산된 급여가 매니저 승인 완료 → 급여 지급일에 직원별 명세서 자동 생성 → SMS/이메일로 발송 → 직원이 SCR-065에서 열람하는 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 매니저 급여 승인 완료 또는 지급일 스케줄러 |
| 종료 조건 | 명세서 발송 + 직원 열람 |
| 참여 도메인 | 직원관리(D7) |

## 2. 전제조건

- X08 시나리오에서 급여 계산 완료
- 매니저가 급여 승인 처리
- 직원 이메일/SMS 연락처 등록

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 스케줄러 | 급여 지급일 명세서 발송 크론잡 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 매니저 | 급여 승인 담당 |
| 직원 | 명세서 수신 및 열람 |
| 알림서비스 | 명세서 발송 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor M as 매니저
    participant SCH as 스케줄러
    participant API as CRM API
    participant DB as DB
    actor E as 직원
    participant N as 알림서비스

    Note over M,API: [매니저 급여 최종 승인 — X08 연속]
    M->>API: 1. GET /payrolls?month={month}&status=CALCULATED %% E_M_GET_PAYROLLS_01
    API-->>M: 2. 직원별 급여 목록
    Note over M: SCR-064 급여 관리 화면에서 검토
    M->>API: 3. POST /payrolls/{id}/approve { comment } %% E_M_APPROVE_PAYROLL_01
    API->>DB: 4. UPDATE payrolls SET status=APPROVED, approved_at=NOW(), approved_by=?
    DB-->>API: 5. OK
    API-->>M: 6. 200 OK

    Note over SCH,DB: [급여 지급일 명세서 발송 크론잡]
    SCH->>API: 7. 급여 명세서 발송 트리거 { pay_date } %% E_SCH_PAYSLIP_01
    API->>DB: 8. SELECT payrolls WHERE pay_date=TODAY AND status=APPROVED
    DB-->>API: 9. 지급 대상 급여 목록

    loop 각 직원별
        API->>DB: 10. SELECT payroll_detail WHERE payroll_id=? %% E_API_GET_DETAIL_01
        DB-->>API: 11. 상세 항목 (기본급, 초과근무, 인센티브, 4대보험, 세금, 실지급)
        API->>API: 12. 명세서 PDF 생성 %% E_API_GEN_PAYSLIP_01
        API->>DB: 13. INSERT payslips { payroll_id, pdf_url, generated_at }
        DB-->>API: 14. payslip_id
        API->>N: 15. 명세서 발송 { employee_id, payslip_url } %% E_API_SEND_PAYSLIP_01
        N-->>E: 16. SMS — "급여 명세서가 발행되었습니다 {url}"
        N-->>E: 17. 이메일 — 명세서 PDF 첨부
    end
    API-->>SCH: 18. 발송 완료 { count }

    Note over E,API: [직원 명세서 열람]
    E->>API: 19. GET /payslips?month={month} %% E_E_GET_PAYSLIP_01
    API->>DB: 20. SELECT payslips WHERE employee_id=? AND month=?
    DB-->>API: 21. 명세서 URL
    API-->>E: 22. 명세서 정보
    Note over E: SCR-065 급여 명세서 화면
    E->>API: 23. GET /payslips/{id}/pdf %% E_E_DOWNLOAD_PAYSLIP_01
    API-->>E: 24. PDF 파일 스트림
    API->>DB: 25. UPDATE payslips SET viewed_at=NOW()
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 3 | POST /payrolls/{id}/approve | 개별 또는 일괄 승인. 승인 후 pay_date 기준 명세서 발송 |
| 11 | 상세 항목 조회 | 기본급, 초과근무수당, 인센티브, 국민연금/건강/고용보험, 소득세 공제 내역 |
| 12 | PDF 생성 | 서버에서 PDF 렌더링. payslip_url에 S3 등 스토리지 URL 저장 |
| 25 | UPDATE viewed_at | 직원이 명세서를 열람한 시각 기록 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 급여 미승인 상태 | pay_date 도달해도 미승인 급여 발송 skip, 매니저 경고 알림 |
| 이메일 없는 직원 | SMS만 발송, 이메일 발송 skip |
| PDF 생성 실패 | 재시도 큐 등록, URL 대신 웹 뷰 링크 제공 |
| 퇴사 직원 | 재직 기간 일할 급여만 포함, 마지막 명세서 발송 |
