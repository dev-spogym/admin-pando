# X30 — 알림 센터 클릭 → 해당 화면 이동 → 읽음 처리

## 1. 시나리오 개요

사용자가 알림 센터 아이콘 클릭 → 미읽음 알림 목록 조회 → 알림 클릭 시 관련 화면으로 이동 → 읽음 처리까지의 시나리오.

| 항목 | 내용 |
|------|------|
| 트리거 | 사용자의 알림 센터 접근 |
| 종료 조건 | 알림 읽음 처리 + 관련 화면 이동 |
| 참여 도메인 | 공통(D1) |

## 2. 전제조건

- 사용자가 로그인 상태
- 시스템에 발송된 알림이 존재함
- 알림에 딥링크(target_url)가 설정되어 있음

## 3. 참여 액터

| 액터 | 설명 |
|------|------|
| 사용자 | 알림 수신 및 확인 |
| CRM API | FitGenie CRM 백엔드 |
| DB | 데이터베이스 |
| 라우터 | 프론트엔드 라우팅 |

## 4. 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor U as 사용자
    participant API as CRM API
    participant DB as DB
    participant RT as 프론트라우터

    Note over U,API: [알림 센터 열기]
    U->>API: 1. GET /notifications?status=UNREAD&page=1 %% E_U_GET_NOTIF_01
    API->>DB: 2. SELECT notifications WHERE user_id=? AND status=UNREAD ORDER BY created_at DESC
    DB-->>API: 3. 미읽음 알림 목록 (type, message, target_url, created_at)
    API-->>U: 4. 알림 목록 { items, unread_count }
    Note over U: SCR-104 알림 센터 드롭다운/패널

    Note over U,API: [전체 알림 조회 — 더보기]
    U->>API: 5. GET /notifications?page=1&limit=20 %% E_U_GET_ALL_NOTIF_01
    API->>DB: 6. SELECT notifications WHERE user_id=? ORDER BY created_at DESC LIMIT 20
    DB-->>API: 7. 전체 알림 목록 (읽음/미읽음 포함)
    API-->>U: 8. 전체 알림 목록

    Note over U,RT: [알림 클릭 → 화면 이동]
    U->>API: 9. POST /notifications/{id}/read %% E_U_READ_NOTIF_01
    API->>DB: 10. UPDATE notifications SET status=READ, read_at=NOW() WHERE id=? AND user_id=?
    DB-->>API: 11. OK
    API->>DB: 12. UPDATE users SET unread_notification_count = unread_notification_count - 1
    DB-->>API: 13. new_count
    API-->>U: 14. 200 OK { unread_count: new_count }

    U->>RT: 15. 딥링크 이동 { target_url } %% E_U_DEEPLINK_01
    Note over RT: target_url 예시: /members/123, /sales/456, /lessons/789

    alt 유효한 target_url
        RT->>API: 16. GET {target_url} (권한 포함) %% E_RT_FETCH_TARGET_01
        API->>DB: 17. 해당 리소스 조회 + 권한 확인
        DB-->>API: 18. 리소스 데이터
        API-->>RT: 19. 200 OK { data }
        RT-->>U: 20. 해당 화면 렌더링
    else target_url 없음 (단순 알림)
        Note over U: 알림 내용만 확인, 화면 이동 없음 %% E_U_NOTIF_NO_LINK_01
    else 리소스 삭제됨 또는 권한 없음
        API-->>RT: 21. 404 또는 403 %% E_RT_TARGET_MISSING_01
        RT-->>U: 22. "해당 정보를 찾을 수 없습니다" 토스트
    end

    Note over U,API: [전체 읽음 처리]
    U->>API: 23. POST /notifications/read-all %% E_U_READ_ALL_01
    API->>DB: 24. UPDATE notifications SET status=READ, read_at=NOW() WHERE user_id=? AND status=UNREAD
    API->>DB: 25. UPDATE users SET unread_notification_count=0
    DB-->>API: 26. OK
    API-->>U: 27. 200 OK { unread_count: 0 }
    Note over U: 알림 뱃지 카운트 0으로 초기화

    Note over U,API: [알림 삭제]
    U->>API: 28. DELETE /notifications/{id} %% E_U_DELETE_NOTIF_01
    API->>DB: 29. UPDATE notifications SET status=DELETED
    DB-->>API: 30. OK
    API-->>U: 31. 200 OK
```

## 5. 주요 메시지 설명

| 번호 | 메시지 | 설명 |
|------|--------|------|
| 1 | GET /notifications?status=UNREAD | 헤더 뱃지용 미읽음 목록. 최신 5개만 표시 |
| 9 | POST /{id}/read | 클릭과 동시에 읽음 처리. 화면 이동 전 호출 |
| 12 | unread_count 업데이트 | 헤더 뱃지 숫자 실시간 감소 |
| 15 | 딥링크 이동 | target_url은 알림 생성 시 설정. 해당 리소스의 상세 화면으로 이동 |
| 24 | 전체 읽음 처리 | 한 번에 모든 미읽음 알림 읽음 처리 |

## 6. 예외/분기

| 상황 | 처리 방법 |
|------|-----------|
| 알림 없음 | 빈 상태 화면 표시 "알림이 없습니다" |
| 딥링크 대상 삭제 | 404 처리, "삭제된 항목" 안내 |
| 권한 없는 화면으로 이동 | 403 처리, 권한 없음 토스트 |
| 오프라인 상태 | 로컬 캐시 알림 표시, 온라인 복구 후 동기화 |
