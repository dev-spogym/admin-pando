# A12 — 감사로그 아카이빙

## 1. 개요

| 항목 | 내용 |
|------|------|
| 트리거 | 크론 — 매월 1일 04:00 |
| 대상 엔티티 | AuditLog |
| 조건 | 2개월 이전 감사로그 (기본 보관 정책) |
| 결과 | Hot DB → Cold Storage (S3) 이관 후 DB 정리 |
| 관련 화면 | SCR-097 감사로그, SCR-090 본사 대시보드 |

## 2. 발생 조건

- `AuditLog.createdAt < 2개월 전 (설정 가능)`
- `AuditLog.archived = false`
- 압축(gzip) 후 S3 버킷 저장
- DB에는 메타데이터 + S3 경로만 유지

## 4. 복구/재시도 전략

| 상황 | 전략 |
|------|------|
| S3 업로드 실패 | 최대 3회 재시도, 실패 시 배치 스킵 |
| DB 삭제 실패 | S3는 유지, 삭제 실패 로그, 다음 크론에서 재시도 |
| 무결성 검증 실패 | 운영팀 긴급 알림, 수동 복구 |
| 크론 중단 | 다음 달 크론에서 미처리 대상 포함 |
