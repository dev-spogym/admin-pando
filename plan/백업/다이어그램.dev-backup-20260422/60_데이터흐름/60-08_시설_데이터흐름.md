## 1. 엔티티 개요

락커(`Locker`)는 번호 단위로 관리되며, 배정(`LockerAssignment`)으로 회원과 연결. 운동복(`Uniform`)도 대여 이력 관리. S11 LockerStatus 참조.

## 3. 쓰기 경로 (락커 배정)

```mermaid
flowchart LR
  UI[SCR-050 락커관리] -->|E_DF08_01| API[POST /api/lockers/:id/assign]
  API --> Valid{가용 확인}
  Valid -->|OCCUPIED| Rej[/409/]
  Valid -->|OK| Txn[(BEGIN)]
  Txn --> DB1[(UPDATE lockers SET status=OCCUPIED)]
  DB1 --> DB2[(INSERT locker_assignments)]
  DB2 --> Pay[결제 연동 SCR-S003]
  Pay --> Commit[(COMMIT)]
```

## 4. 자동 만료 (크론)

```mermaid
flowchart LR
  Cron[[매일 00:00]] --> Job[만료 판정]
  Job --> SEL[(SELECT assignments WHERE end_date < today)]
  SEL --> UPD[(UPDATE lockers SET status=OVERDUE)]
  UPD --> Notif[알림: 회원+스태프]
```

## 5. 주요 필드

| 필드 | 비고 |

## 6. 인덱스/제약

- `UNIQUE(locker_id) WHERE status IN ('RESERVED','OCCUPIED')`
- `INDEX(end_date)` — 만료 크론
