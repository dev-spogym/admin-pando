## 1. 목적

DLG-M021 마일리지 수동 조정 다이얼로그의 열기/닫기/완료 생명주기를 명세한다. 🆕 미구현 기능.

## 2. 트리거/전제조건

- 상세내역 탭 > 마일리지 서브탭 > "수동 조정" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([마일리지 수동 조정 버튼 클릭])

  TRIGGER -->|E_OPEN_01| DLG_OPEN[DLG-M021 마일리지 수동 조정 🆕]
  DLG_OPEN --> INIT[유형=EARN 포인트/사유 빈값]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소| CLOSE[모달 닫기]
  USER_ACT -->|E_SUBMIT_01 저장 필수 충족| API_CALL[POST /api/members/:id/mileage 수동 조정]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success 마일리지가 조정되었습니다))
  TOAST_OK -->|E_CLOSE_OK_01| POST_OK[모달 닫기 + refetch]

  API_CALL -->|E_FAIL_01 오류| TOAST_ERR((toast.error 조정에 실패했습니다))
  TOAST_ERR -->|E_STAY_01| DLG_OPEN

  classDef newFeature fill:#F3E5F5,stroke:#9C27B0,stroke-dasharray:5 5
  classDef success    fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20

  classDef system     fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT newFeature
  class TOAST_OK,POST_OK success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_OPEN_01 | 수동 조정 버튼 | 모달 열기 | - |
| E_SUBMIT_01 | 저장 | API | 포인트+사유 충족 |
| E_SUCCESS_01 | API | toast.success | 200 |
| E_FAIL_01 | API | toast.error | 오류 |
