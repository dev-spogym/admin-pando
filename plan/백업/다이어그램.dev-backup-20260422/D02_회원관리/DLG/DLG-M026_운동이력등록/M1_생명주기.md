## 1. 목적

DLG-M026 운동 이력 추가 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 운동이력 탭 > "운동 추가" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([운동이력 탭 운동 추가 버튼 클릭])

  TRIGGER -->|E_OPEN_01| DLG_OPEN[DLG-M026 운동 이력 추가 max-w-480px]
  DLG_OPEN --> INIT[날짜=오늘 운동명/기타 빈값]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소| CLOSE[모달 닫기]
  USER_ACT -->|E_SUBMIT_01 저장 logDate+exerciseName 있음| API_CALL[POST /api/exercise-logs INSERT]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success 운동 이력이 저장되었습니다))
  TOAST_OK -->|E_CLOSE_OK_01| POST_OK[모달 닫기 + refetch]

  API_CALL -->|E_FAIL_01 오류| TOAST_ERR((toast.error 저장 실패 error))
  TOAST_ERR -->|E_STAY_01| DLG_OPEN

  classDef modal   fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error   fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system  fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT modal
  class TOAST_OK,POST_OK success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_OPEN_01 | 운동 추가 | 모달 열기 | - |
| E_SUBMIT_01 | 저장 | API | logDate+exerciseName 있음 |
| E_SUCCESS_01 | API | toast.success | 200 |
| E_FAIL_01 | API | toast.error | 오류 |
