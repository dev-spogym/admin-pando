## 1. 목적

DLG-M002 회원 삭제 확인 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- 회원 상세 > 위험 구역 > "회원 삭제" 버튼 클릭

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([회원 상세 위험 구역 회원 삭제 클릭])

  TRIGGER -->|E_OPEN_01| DLG_OPEN[DLG-M002 열기 ConfirmDialog]
  DLG_OPEN --> INIT[confirmationText 빈값 초기화]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소 버튼| CLOSE[모달 닫기]
  USER_ACT -->|E_X_01 X 버튼| CLOSE
  USER_ACT -->|E_SUBMIT_01 삭제하기 클릭 삭제 입력됨| API_CALL[DELETE /api/members/:id soft delete]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success 회원이 삭제되었습니다))
  TOAST_OK -->|E_REDIRECT_01| REDIRECT[/members 이동]

  API_CALL -->|E_FAIL_01 4xx/5xx| TOAST_ERR((toast.error 삭제에 실패했습니다))
  TOAST_ERR -->|E_STAY_01| DLG_OPEN

  classDef modal   fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error   fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system  fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT modal
  class TOAST_OK,REDIRECT success
  class TOAST_ERR error
  class API_CALL system
```

## 4. 엣지 설명

| 엣지 ID | 출발 | 도착 | 조건 |
|---------|------|------|------|
| E_OPEN_01 | 삭제 버튼 | 모달 열기 | - |
| E_CANCEL_01 | 취소 | 모달 닫기 | - |
| E_SUBMIT_01 | 삭제하기 | API 호출 | confirmationText==="삭제" |
| E_SUCCESS_01 | API | toast.success | 200 |
| E_REDIRECT_01 | toast | /members 이동 | - |
| E_FAIL_01 | API | toast.error | 오류 |
