## 1. 목적

DLG-M001 상태 일괄 변경 다이얼로그의 열기/닫기/완료 생명주기를 명세한다.

## 2. 트리거/전제조건

- SCR-M001 BulkActionBar > "상태 변경" 버튼 클릭
- selectedRows.size > 0

## 3. 다이어그램

```mermaid
flowchart TD
  TRIGGER([SCR-M001 BulkActionBar 상태 변경 클릭])

  TRIGGER -->|E_OPEN_01 selectedRows > 0| DLG_OPEN[DLG-M001 열기 sm 중앙]
  DLG_OPEN --> INIT[초기 상태 radio=ACTIVE 설정]

  INIT --> USER_ACT{사용자 액션}

  USER_ACT -->|E_CANCEL_01 취소 버튼| CLOSE[모달 닫기]
  USER_ACT -->|E_ESC_01 ESC 키| CLOSE
  USER_ACT -->|E_X_01 X 버튼| CLOSE
  USER_ACT -->|E_SUBMIT_01 변경 확인 클릭| API_CALL[PUT /api/members/bulk-status]

  API_CALL -->|E_SUCCESS_01 200| TOAST_OK((toast.success N명의 상태를 변경했습니다))
  TOAST_OK -->|E_AFTER_OK_01| POST_ACTION[모달 닫기 + refetch + selectedRows 초기화]

  API_CALL -->|E_FAIL_01 4xx/5xx| TOAST_ERR((toast.error 상태 변경에 실패했습니다))
  TOAST_ERR -->|E_STAY_01| DLG_OPEN

  classDef modal   fill:#FFF3E0,stroke:#E65100,color:#BF360C
  classDef success fill:#E8F5E9,stroke:#2E7D32,color:#1B5E20
  classDef error   fill:#FFEBEE,stroke:#C62828,color:#B71C1C
  classDef system  fill:#EDE7F6,stroke:#5E35B2

  class DLG_OPEN,INIT modal
  class TOAST_OK,POST_ACTION success
  class TOAST_ERR error
  class API_CALL system
```
