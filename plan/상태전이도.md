## 1. 회원 상태 (MemberStatus)

### 1.1 상태 정의

| 상태값 | 한글명 | 설명 | UI 색상 | 비고 |
|--------|--------|------|---------|------|
| `ACTIVE` | 활성 | 유효한 이용권 보유, 기간 내 | #4CAF50 (녹색) | 정상 이용 가능 |
| `INACTIVE` | 비활성 | 비활성 처리된 회원 | #9E9E9E (회색) | 관리자 수동 처리 |
| `EXPIRED` | 만료 | 모든 이용권 기간 종료 | #F44336 (빨강) | 재구매 시 활성 전환 |
| `HOLDING` | 홀딩 | 기간정지 중 | #9C27B0 (보라) | 남은 기간 보존 |
| `SUSPENDED` | 정지 | 이용 제한 (징계 등) | #FF5722 (주황) | 관리자만 해제 가능 |
| `WITHDRAWN` | 탈퇴 | 자발적 탈퇴 | #795548 (갈색) | 개인정보 마스킹 대상 |
| `DORMANT` | 휴면 | 장기 미방문 (90일 이상) | #607D8B (청회색) | 자동 전환 |
| `TRANSFERRED` | 이관 | 타지점으로 이관 완료 | #2196F3 (파랑) | 이관 이력 보존 |

**추가 표시 상태 (DB 미저장, UI 계산)**

| 표시 상태 | 한글명 | 판정 조건 | UI 색상 |
|-----------|--------|-----------|---------|
| `unregistered` | 미등록 | 이용권 미보유 | #BDBDBD (연회색) |
| `scheduled` | 예정 | 이용권 시작일 미도래 | #03A9F4 (하늘색) |
| `imminent` | 임박 | 만료일 N일(기본 30일) 이내 | #FF9800 (주황) |

> **복수 이용권 판정 규칙**: 여러 이용권 보유 시 가장 좋은 상태를 대표 상태로 표시.
> 우선순위: 활성 > 임박 > 예정 > 홀딩 > 만료

| 탈퇴 | 재가입 | 미등록 | 관리자 수동 처리 | MANAGER 이상 | previousMemberId 연결 |

### 1.4 자동 전환 규칙

| 규칙 | 실행 주기 | 조건 | 대상 상태 | 후속 처리 |
|------|-----------|------|-----------|-----------|
| 예정→활성 | 매일 00:00 | `membershipStart <= today` | ACTIVE | - |
| 활성→임박 | 매일 00:00 | `membershipExpiry - today <= N일` | (UI 계산) | 만료 임박 알림 발송 |
| 임박→만료 | 매일 00:00 | `membershipExpiry < today` | EXPIRED | 만료 알림 발송 |
| 활성→휴면 | 매일 00:00 | `lastVisitAt < today - 90일` | DORMANT | 휴면 안내 발송 |
| 홀딩 자동해제 | 매일 00:00 | 홀딩 종료일 도래 | ACTIVE/EXPIRED | 해제 알림 발송 |

### 1.5 개인정보 처리

| 상태 | 개인정보 보존 기간 | 마스킹 처리 |
|------|-------------------|------------|
| WITHDRAWN (탈퇴) | 탈퇴 후 30일 | `privacyMaskedAt` 기록, 이름/전화번호/이메일 마스킹 |
| DORMANT (휴면) | 휴면 전환 후 1년 | 미복귀 시 자동 탈퇴 처리 고려 |

## 2. 매출 상태 (SaleStatus)

### 2.1 상태 정의

| 상태값 | 한글명 | 설명 | 비고 |
|--------|--------|------|------|
| `COMPLETED` | 결제완료 | 정상 결제 처리 완료 | 기본 최종 상태 |
| `PENDING` | 대기 | 결제 처리 중 / PG 응답 대기 | 일시적 상태 |
| `UNPAID` | 미수금 | 결제 미완료 (외상/분할) | 미수금 잔액 > 0 |
| `REFUNDED` | 환불 | 환불 처리 완료 | 부분/전체 환불 |

### 2.2 상태 전이 다이어그램

```
                         ┌──────────────┐
                         │              │
     결제 요청            ▼              │ PG 결제 실패 (재시도)
 ────────────────▶ ┌──────────┐         │
                   │ PENDING  │─────────┘
                   │ (대기)    │
                   └────┬─────┘
                        │
              ┌─────────┼──────────┐
              │         │          │
         결제 성공   미수금 발생   결제 포기
              │         │          │
              ▼         ▼          ▼
        ┌──────────┐ ┌──────────┐  (삭제)
        │COMPLETED │ │ UNPAID   │
        │(결제완료) │ │(미수금)   │
        └────┬─────┘ └────┬─────┘
             │             │
        환불 요청      미수금 완납
             │             │
             ▼             ▼
        ┌──────────┐  ┌──────────┐
        │ REFUNDED │  │COMPLETED │
        │ (환불)    │  │(결제완료) │
        └──────────┘  └──────────┘
```

### 2.3 상태 전이 매트릭스

| 변경 전 상태 | 이벤트 | 변경 후 상태 | 트리거 | 권한 | 비고 |
|-------------|--------|-------------|--------|------|------|
| - | 결제 요청 (POS/전자계약) | PENDING | 사용자 결제 시작 | STAFF 이상 | 단말기/PG 연동 시 |
| PENDING | PG/단말기 결제 성공 | COMPLETED | 결제 응답 수신 | 시스템 | 이용권 자동 생성 |
| PENDING | 수기/현금 결제 | COMPLETED | 관리자 수동 확인 | STAFF 이상 | 수기 결제 시 즉시 완료 |
| PENDING | 미수금 발생 (부분 결제) | UNPAID | 관리자 등록 | STAFF 이상 | `unpaid` 필드에 잔액 기록 |
| PENDING | 결제 실패/포기 | (삭제) | 결제 취소 | 시스템 | 매출 레코드 미생성 |
| UNPAID | 미수금 완납 | COMPLETED | 추가 결제 처리 | STAFF 이상 | `unpaid = 0` |
| COMPLETED | 환불 요청 | REFUNDED | 관리자 환불 처리 | MANAGER 이상 | 환불 사유 필수 |
| UNPAID | 환불 요청 | REFUNDED | 관리자 환불 처리 | MANAGER 이상 | 미수금 포함 정리 |

### 2.4 환불 규칙

| 규칙 | 조건 | 처리 | 에러 코드 |
|------|------|------|-----------|
| 환불 금액 검증 | 환불금 > 결제금 | 거부 | E400301 |
| 이용권 연동 | 환불 시 | 연결된 이용권 비활성화 | - |
| 부분 환불 | 일부 금액만 환불 | 매출 금액 조정, 상태 유지(COMPLETED) | - |
| 전체 환불 | 전액 환불 | REFUNDED 전환 | - |

## 4. 이용권/수강권 상태

DB에 별도 enum이 아닌 기간/횟수 기반으로 계산되는 표시 상태.

### 4.1 상태 정의

| 표시 상태 | 한글명 | 판정 조건 | UI 색상 |
|-----------|--------|-----------|---------|
| `scheduled` | 예정 | 시작일 > 오늘 | #03A9F4 (하늘색) |
| `active` | 활성 | 시작일 ≤ 오늘 ≤ 만료일, 잔여횟수 > 0 | #4CAF50 (녹색) |
| `imminent` | 임박 | 만료일 - 오늘 ≤ N일 또는 잔여횟수 ≤ 3 | #FF9800 (주황) |
| `holding` | 홀딩 | 기간정지 중 | #9C27B0 (보라) |
| `expired` | 만료 | 만료일 < 오늘 또는 잔여횟수 = 0 | #F44336 (빨강) |

### 4.2 상태 전이 다이어그램

```
 결제 완료           시작일 도래         만료 N일 전         만료일 경과
───────────▶ [예정] ───────────▶ [활성] ───────────▶ [임박] ───────────▶ [만료]
              scheduled          active              imminent           expired
                                   │                   │                  │
                              홀딩 신청            홀딩 신청          재구매
                                   │                   │                  │
                                   ▼                   ▼                  ▼
                                [홀딩]              [홀딩]           [예정/활성]
                                holding             holding
                                   │                   │
                              홀딩 해제            홀딩 해제
                                   │                   │
                                   ▼                   ▼
                                [활성]              [임박]
```

### 4.3 상태 전이 매트릭스

| 변경 전 상태 | 이벤트 | 변경 후 상태 | 트리거 | 비고 |
|-------------|--------|-------------|--------|------|
| - | 결제 완료 (시작일 미래) | 예정 | POS/전자계약 | 이용권 레코드 생성 |
| - | 결제 완료 (시작일 오늘) | 활성 | POS/전자계약 | 즉시 이용 가능 |
| 예정 | 시작일 도래 | 활성 | 배치 스케줄러 | 자동 전환 |
| 활성 | 만료일 N일 전 | 임박 | 배치 계산 | UI 표시용 |
| 활성 | 홀딩 신청 | 홀딩 | 관리자 처리 | 기간정지 제한 적용 |
| 활성 | PT 횟수 소진 | 만료 | 수업 완료 시 차감 | 잔여횟수 = 0 |
| 임박 | 만료일 경과 | 만료 | 배치 스케줄러 | 자동 전환 |
| 임박 | 홀딩 신청 | 홀딩 | 관리자 처리 | - |
| 홀딩 | 홀딩 해제 | 활성/임박 | 관리자 또는 자동 | 남은 기간 재계산 |

## 5. 수업 승인 상태 (Class.approvalStatus)

| pending | 관리자 거절 | rejected | 수동 거절 | MANAGER 이상 |

## 6. 상담 상태 (Consultation.status)

| 상태값 | 한글명 | 설명 |
|--------|--------|------|
| `scheduled` | 예정 | 상담 예약 완료 (기본값) |

## 9. 지점 상태 (BranchStatus)

### 9.1 상태 정의

| 상태값 | 한글명 | 설명 | 비고 |
|--------|--------|------|------|
| `ACTIVE` | 운영중 | 정상 운영 | 기본값 |
| `PAUSED` | 일시중단 | 임시 휴업 | pausedAt, resumeAt 기록 |
| `CLOSING` | 폐점진행 | 폐점 절차 진행 중 | BranchClosureLog 연동 |
| `INACTIVE` | 폐점 | 폐점 완료 | 모든 리소스 이관/정리 완료 |

### 9.2 상태 전이 다이어그램

```
 지점 개설                 임시 휴업          재개
───────────▶ [ACTIVE] ──────────────▶ [PAUSED] ────────▶ [ACTIVE]
             (운영중)                  (일시중단)
                │
                │ 폐점 공지
                ▼
            [CLOSING] ─────────────▶ [INACTIVE]
            (폐점진행)   모든 처리 완료    (폐점)
                │
                │ 폐점 취소
                ▼
            [ACTIVE]
```

### 9.3 상태 전이 매트릭스

| 변경 전 상태 | 이벤트 | 변경 후 상태 | 트리거 | 권한 | 비고 |
|-------------|--------|-------------|--------|------|------|
| - | 지점 개설 | ACTIVE | 관리자 등록 | OWNER 이상 | maxBranches 제한 확인 |
| ACTIVE | 임시 휴업 | PAUSED | 관리자 설정 | OWNER 이상 | pausedAt, resumeAt 기록 |
| PAUSED | 재개 | ACTIVE | 관리자 또는 resumeAt 도래 | OWNER 이상 | - |
| ACTIVE | 폐점 공지 | CLOSING | 관리자 폐점 시작 | OWNER 이상 | BranchClosureLog 생성 |
| CLOSING | 모든 회원 이관/환불 + 직원 정산 완료 | INACTIVE | 관리자 최종 확인 | OWNER 이상 | - |
| CLOSING | 폐점 취소 | ACTIVE | 관리자 취소 | OWNER 이상 | BranchClosureLog.status = CANCELLED |

### 9.4 폐점 처리 체크리스트

| 순서 | 항목 | 상태 필드 | 완료 조건 |
|------|------|-----------|-----------|

| 2 | 회원 이관 처리 | transferredMembers | 대상 회원 전원 이관 |
| 3 | 미이관 회원 환불 | refundedMembers | 잔여 회원 환불 처리 |
| 4 | 직원 정산/이관 | totalStaff | 전원 퇴사 또는 이관 |
| 5 | 최종 폐점 확정 | status = COMPLETED | 관리자 최종 확인 |
